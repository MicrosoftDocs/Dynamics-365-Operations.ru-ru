# <a name="retail-home-page"></a>Домашняя страница розничной торговли

[!include[banner](includes/banner.md)]

Dynamics 365 для операций – Retail обеспечивает полный опыт по покупке и адаптируемых решение односторонний канал, современное мобильных и глобальные. Можно развернуть начала до конца, с возможностями современное точки продажи (POS) программных средств, операций магазина, торговли электронной коммерции, центра обработки вызовов, маркетинг и поддержки заказчиков, для предоставления цепочки, финансовой информации и т.д., таким же образом, оптимальным для вашего бизнеса.

Эта страница указывает на ресурсы, которые помогут использовать Microsoft Dynamics 365 для операций - розничных более эффективно. На левой панели выберите выберите функциональной области для получения дополнительных сведений о нем. 

## <a name="set-up-retail-channels"></a>Настройка розничных каналов
При подготовке к розничной торговли будет решить, как определить и настроить хранилище физических (называемый *розничного магазина* в Dynamics 365 для операций) и интерактивного магазина операций. 

Каждый розничный магазин могут быть свой собственный способов оплаты, ценовые группы, POS решений, счета доходов, расходные счета и персонала. После создания магазин розничной торговли, назначаются продукты, которые должны храниться в магазине. Кроме того, необходимо назначить сотрудников, кассы и клиентов для магазина. Наконец, следует добавить новый магазин в организационную иерархию.

Когда вы создаете Интернет-магазин, необходимо настроить компоненты, такие как ассортиментов, атрибуты, методы оплаты и способах поставки. Можно также определить цены, рекламные акции, скидки и условия доставки, относящиеся к Интернет-магазина.

-   [Определение каналов розничной торговли](define-maintain-retail-channels.md)
-   [Online store overview](online-stores.md)

## <a name="set-up-a-call-center"></a>Настройка центра обработки вызовов
Можно выполнить настройку центра обработки вызовов как тип канала розничной торговли. В центре обработки вызовов работники принимают заказы от клиентов по телефону и создают заказы на продажу. Чтобы помочь сотрудникам приема заявок и обработки обслуживания клиентов в процессе выполнения заказа, можно настроить заказ на продажу будет содержать, как будет управлять ценообразования и доступных данных о клиентах, продукты и цены. Для каждого центра обработки вызовов можно также настроить пользователей, способы оплаты, группы цен, финансовых аналитик и способы поставки.

-   [Call center functionality](call-center-functionality.md)
-   [Create call center catalogs](create-call-center-catalogs.md)

## <a name="define-your-pos-and-mpos-solutions"></a>Определение POS и MPOS решений
Retail POS (MPOS современное) – POS-приложение для ПК, планшетные ПК и телефоны. Сотрудники отдела продаж можно обработать проводки по продажам, заказа клиентом и выполнения ежедневных операций и управление запасами с мобильных устройств в любом магазине, а также контрольно-кассовых машин, основанные на ПК. Cloud POS-терминала является POS-терминала на основе веб-обозревателя. 

Несколько компонентов, которые необходимо настроить для определения отношений между магазина, ККМ POS или каналов в хранилище и периферийных устройств розничной торговли, которые их регистрирует или каналы используется для обработки проводок. Будет также определяют характеристики определенному экземпляру POS, включая настройку для периферийных устройств розничной торговли, которые будут использоваться для пользователя, который выполняет вход в систему, регистрация на ККМ, который сопоставлен ККМ магазина и повышается или профиль оборудования.

-   [Определение и поддержку клиентов канала, ККМ и станции оборудования](define-maintain-channel-clients-registers-hw-stations.md)
-   [Настроить и установить MPOS](retail-modern-pos-device-activation.md)
-   [Автономные функциональности POS](pos-offline-functionality.md)

## <a name="set-up-a-customer-loyalty-program"></a>Настройки программы лояльности клиент
Можно настроить программу лояльности так, чтобы клиенты могли зарабатывать баллы на основе определенных критериев и баллы могут рассчитываться разными способами. Программы лояльности могут быть простыми или сложными и применяются во всей системе вашей юридические лица в любой канала розничной торговли.
-   [Настройки программы лояльности клиент](set-up-customer-loyalty-program.md)

## <a name="set-up-retail-products"></a>Настройка розничных продуктов
Прежде чем можно будет предлагать продукты для продажи в ваших каналах розничной торговли, необходимо создать и настроить продукты в Dynamics 365 для операций. Можно создать продукты, определить свойства и атрибуты продукции и назначить продукты иерархиям розничных категорий. Чтобы сделать продукты доступными розничным каналам и добавить их в ассортимент активной, затем будет выпущена продуктов юридическим лицам, в которых они доступны.
