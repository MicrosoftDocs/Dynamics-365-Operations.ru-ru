# <a name="pos-application-and-user-language-settings"></a>Настройки приложения POS и языка пользователя

В этом разделе описывается, как изменить языковые параметры в Retail POS (MPOS современное) и Cloud POS.

<a name="overview"></a>Обзор
========

Retail POS (MPOS современное) и Cloud POS поддержки сред, где языковые параметры и переводы, может принимать значения от параметров магазина и пользователя. Например магазин может находиться в регионе, на английском наиболее часто для своих клиентов, когда некоторые работники предпочитают использовать приложения с переводы на французском языке.

## <a name="data-language"></a>Язык данных
Вне зависимости от параметров пользователя MPOS и Cloud POS всегда используют магазина языковые параметры для определения переводов, используемых для сохранения данных. Это гарантирует, что всех пользователей и все клиенты будут иметь согласованную работу.  Примеры данных.

-   Товары
-   Атрибуты и значения
-   Названия категорий
-   Напечатанные или отправленные по электронной почте приходы по транзакциям
-   Названия способа оплаты
-   Сообщения строкового дисплея

Магазина язык будет также использоваться для главный экран входа в систему POS, поскольку пользователь неизвестен перед входом систему. Если отсутствует перевод для языка для магазина, POS вернется к языка компании.

### <a name="configuring-the-stores-language-setting"></a>Конфигурация настройки языка магазина

Языковые настройки магазина устанавливается из **все розничные магазины** на **Retail Store** странице под ** Общие &gt;региональных настроек &gt;языка. ** Воспользуйтесь выпадающими Выбор языка для каждого магазина.

## <a name="user-interface-language"></a>Язык пользовательского интерфейса
Параметр языка пользователя POS определяет переводы, которые используются в пользовательском интерфейсе приложения. Это включает в себя все метки, меню и списков, которые не рассматриваются в данных. Единственным исключением является текст, который отображается на сетки кнопок POS. Сетки кнопок не поддерживают переводы, поэтому они всегда будет отображать текст, как определено на кнопке. Для обеспечения поддержки переведенный кнопок, необходимо скопировать и ведение сеток кнопок отдельные и назначить пользователей соответствующим образом.

### <a name="configuring-the-users-language-setting"></a>Конфигурация настройки языка пользователя

Параметр языка пользователя POS устанавливается из **всех работников** на **рабочий** странице под **Retail &gt;язык**.  На вкладке "основной" профиля не выбрано.  Этот параметр не используется в POS. Если язык пользователя не задан или задан как язык, для которого недоступны переводы, POS перейдет к использованию языка магазина.  

|             |                            |                                                                   |
|-------------|----------------------------|-------------------------------------------------------------------|
| ** **       | **Язык пользовательского интерфейса** ** **      | **Язык данных (продукты, форматы поступлений, строковый дисплей и т. д.)** |
| **Компания** | По умолчанию                    | По умолчанию                                                           |
| **Магазин**   | Переопределяет компанию          | Переопределяет компанию                                                 |
| **User**    | Переопределяет магазин или компанию | Никогда                                                             |
