# <a name="retail-peripherals-overview"></a>Обзор периферийных устройств для розничной торговли

В этом разделе объясняется понятиях, связанных с периферийных устройств розничной торговли. В ней описывается различными способами, что периферийные устройства могут быть связаны с точки продажи (POS) и компонентов, которые ответственным за настройку соединения с POS.

<a name="concepts"></a>Концепции
--------

### <a name="pos-registers"></a>POS-регистраторы

Переходов: Нажмите кнопку **розничной торговли и электронной коммерции**&gt;**настройку канала**&gt;**настройки POS**&gt;**регистрирует**. Объект, используемый для определения характеристик конкретного экземпляра терминал важно регистра продаж (POS). Эти характеристики включают профиль оборудования или настройка периферийных устройств розничной торговли, которые будут использоваться для пользователя, вход на данной ККМ в ККМ магазина, который сопоставлен ККМ и повышается.

### <a name="devices"></a>Оборудование

Переходов: Нажмите кнопку **розничной торговли и электронной коммерции**&gt;**настройку канала**&gt;**настройки POS**&gt;**устройств**. Устройство — это объект, представляющий физический экземпляр устройства, которое сопоставлено с регистром POS. При создании устройства, отображаемый в POS ККМ. Объект устройства отслеживает сведения об активации регистра POS, типе используемого клиента и пакете приложения, развернутом на определенном устройстве. Устройства могут быть сопоставлены для следующих типов приложений: Cloud Retail POS, Retail POS современное, современное Retail POS – Windows Phone, Retail POS современное – Android и Retail POS современное — операций ввода-вывода.

### <a name="retail-modern-pos"></a>Retail Modern POS

Современное POS является программой POS-терминала для Microsoft Windows. Его можно развернуть в операционных системах Windows 10 (ОС).

### <a name="cloud-pos"></a>Cloud POS

Cloud POS-терминала является версией обозревателя программы современное POS, доступную в веб-обозревателе.

### <a name="modern-pos-for-ios"></a>Современное POS для операций ввода-вывода

Современное POS для операций ввода-вывода является более операций ввода-вывода версии программы современное POS, может быть развернут на устройствах операций ввода-вывода.

### <a name="modern-pos-for-android"></a>Современное POS для Android

Современное POS для Android является более Android версии программы современное POS, которая может развертываться на Android устройств.

### <a name="pos-peripherals"></a>Периферийные устройства POS

Периферийные устройства POS могут устройств, которые явно для функций POS. Эти периферийные устройства обычно разделяются на определенных классов. Дополнительные сведения об этих классах см. раздел «Классов устройств» данного раздела.

### <a name="hardware-station"></a>Станция оборудования

Переходов: Нажмите кнопку **розничной торговли и электронной коммерции**&gt;**каналов**&gt;**розничные магазины**&gt;**все розничные магазины**. Выберите магазин и перейдите на экспресс-вкладку **Станции оборудования**. **Станции оборудования** параметр является параметром уровня канала, который используется для определения экземпляров, где развертывается логика периферийных устройств розничной торговли. Этот параметр на уровне канала используется для определения характеристик оборудования станции. Он также используется для списка оборудования станции, доступные для экземпляра в магазине заданной современное POS. На станции оборудования встроены в программу современное POS для Windows. Станции оборудования также могут развертываться независимо друг от друга как отдельной программы Microsoft Internet Information Services (IIS). В этом случае он доступен по сети.

### <a name="hardware-profile"></a>Профиль оборудования

Переходов: Нажмите кнопку **розничной торговли и электронной коммерции**&gt;**настройку канала**&gt;**настройки POS**&gt;**профилей POS**&gt;**профили оборудования**. Профиль оборудования представляет собой список устройств, которые настроены для ККМ POS или станцию оборудования. В профиле оборудования могут быть сопоставлены непосредственно к ККМ POS или станцию оборудования.

## <a name="devices-classes"></a>Классы устройств
Периферийные устройства POS обычно разделяются на классы. В этом разделе приводится описание и обзор устройств, поддерживаемых современное POS.

### <a name="printer"></a>Принтер

Принтеры включают традиционной принтеры чеков POS и принтерам целой страницы. Принтер поддерживаются через объект OLE, для интерфейсов драйвер Retail POS (OPOS) и Microsoft Windows. В то же время может использоваться до два принтера. Эта возможность поддерживает сценарии, где cash-and-carry клиента чеки печатаются на принтеры чеков, тогда как заказов клиентов, которые содержат дополнительные сведения, печатаются на принтере целой страницы. Принтеры чеков можно подключен непосредственно к компьютеру через USB, подключенных к сети через Ethernet или подключается с помощью Bluetooth.

### <a name="scanner"></a>Сканер

До двух штрих-кода можно использовать в то же время сканеров. Благодаря этому обеспечивается поддержка случаи, требующие сканер, который мобильных для поиска вирусов в больших или большой элементами, в то время как основных внедренных сканера используется для большинства элементов стандартного размера, чтобы ускорить время извлечения. Сканеры могут поддерживаться через интерфейсы клавиатуры сектору, OPOS или универсальные платформы Windows (UWP). USB или Bluetooth могут использоваться для подключения к компьютеру сканера.

### <a name="msr"></a>MSR / считыватель магнитных карт

Один USB считыватель магнитных карт (MSR) могут настраиваться с помощью драйверы OPOS. Если вы хотите использовать автономный MSR для платежных проводок электронных платежей система электронных платежей, MSR должна управляться соединителя платежей. Автономный MSR может использоваться для ввода программы лояльности клиент вход сотрудника и запись подарочные карты, независимо от соединителя платежей.

### <a name="cash-drawer"></a>Денежный ящик

Два денежные ящики могут поддерживаться в профиле оборудования. Эта возможность позволяет каждой ККМ, которые должны быть доступны в то же время в две смены активного. В случае смены общих или кассовый лоток, используемый нескольких мобильных устройств POS-терминала в то же время для каждого профиля оборудования допускается только один кассовый лоток. Денежные ящики можно подключен непосредственно к компьютеру через USB, подключенных к сети или подключен принтер чеков, через интерфейс RJ12. В некоторых случаях денежные ящики может быть подключен через Bluetooth.

### <a name="line-display"></a>Строковый дисплей

Отображает строки используются для отображения продуктов, сальдо проводки и другие полезные сведения для клиента во время проводки. Один строковый дисплей может быть подключен к компьютеру через USB с помощью драйверы OPOS.

### <a name="signature-capture"></a>Считывание рукописных подписей

Подпись устройства записи может быть подключен непосредственно к компьютеру через USB с помощью драйверы OPOS. Во время настройки подписи записи клиента будет предложено выйти на устройстве. После подписи, он будет отображаться кассир для принятия.

### <a name="scale"></a>Шкала

Шкал подключаются к компьютеру через USP используется водители OPOS. При добавлении продукта, помеченном как «Weighed» продукт в проводку, POS считывает вес из шкалы, добавляет продукт в проводку и использует количество, предоставляемых шкалы.

### <a name="pin-pad"></a>Клавиатура для набора ПИН-кода

Персональный идентификационный номер (PIN) прокладок поддерживаются с помощью OPOS, но должно осуществляться через соединитель платежей.

### <a name="secondary-display"></a>Дополнительный дисплей

Во время настройки дополнительного экрана Windows номер 2 используется для отображения основных сведений. Дополнительный дисплей предназначено для поддержки независимых поставщиков Программных расширение, поскольку по умолчанию не настраиваемой дополнительный дисплей, а также отображение ограниченного содержимого.

### <a name="payment-device"></a>Платежный терминал

Поддержка устройств оплаты реализуется через соединитель платежей. Устройства обработки платежей можно выполнить одно или много функций, которые предоставляют других классов устройств. Например устройство платежа может функционировать как считыватель Магнитных/карт, строковый дисплей, устройство записи подписи или ввода ПИН-кодов. Поддержка реализуется устройства обработки платежей, независимо от поддержки отдельного устройства, который указывается для других устройств, которые включены в профиле оборудования.

## <a name="supported-interfaces"></a>Поддерживаемые интерфейсы
### <a name="opos"></a>OPOS

Помогают обеспечить, что широкий выбор устройств может использоваться с Microsoft Dynamics 365 для операций - розничной торговли, OLE для POS промышленного стандарта является первичной розничной торговли платформы периферийные устройства поддерживаемый Microsoft Dynamics 365 для операций — Retail. OLE для POS стандартных было произведено по национальной Retail федерации (NRF), задает протоколы связи промышленный стандарт для периферийных устройств розничной торговли. OPOS — это широко распространенных реализация OLE для стандартных POS. Он был разработан в середине-90-х и была обновлена несколько раз, затем. OPOS предоставляет архитектуру драйвера устройства, позволяющий интегрирование оборудования POS с систем кассовых Терминалов на основе Windows. OPOS управляет маркер связи совместимого оборудования и программного обеспечения POS. Элемент управления OPOS состоит из двух частей:

-   **Объект элемента управления** — объект элемента управления для устройств класса (например, строке будет показано) предоставляет интерфейс для программного обеспечения. Консультационных услуг Monroe ([www.monroecs.com](http://www.monroecs.com/)) предоставляет стандартизированный набор объектов OPOS элемента управления, которые называются компонент common control objects (CCOs). CCOs используются для проверки компонента системы Microsoft Dynamics 365 для операций — Retail POS. Таким образом, такая проверка помогает гарантировать, что, если Microsoft Dynamics 365 for Operations - Retail поддерживает класс устройств через большое количество разных типов устройств OPOS могут поддерживаться, приобретенного объект обслуживания, который создан для OPOS. Не нужно явно тестирование устройств каждого типа.
-   **Объект обслуживания** — объект обслуживания обеспечивает связь между объект элемента управления (CCO) и устройства. Как правило объект обслуживания для устройства предоставляется производителем устройства. Однако в некоторых случаях может потребоваться загрузить объект обслуживания см. Например объект обслуживания более позднюю становятся доступны. Чтобы найти адрес веб-сайта производителя, обратитесь к документации.

[![Управление и обслуживание объектов](./media/retail_peripherals_overview01.png)](./media/retail_peripherals_overview01.png) поддержки для реализации OPOS OLE для POS помогает гарантировать, что, если производители устройств и издатели POS правильно реализовать стандартной, системы POS-терминала и поддерживаемых устройствах совместной работы, даже если они не были тестирование ранее друг с другом. **Примечание:** OPOS поддержка не гарантирует поддержкой всех устройств, для которых установлены драйверы OPOS. Microsoft Dynamics 365 для операций - сначала должен поддерживать розничной торговли, этот тип устройства или класса, с помощью OPOS. Кроме того службы объектов не всегда может быть курсе последнюю версию CCOs. Также следует иметь в виду, что, как правило, качество объектов обслуживания может варьироваться.

### <a name="windows"></a>Окна

Печать чека в POS-оптимальный OPOS. OPOS, как правило, гораздо быстрее, чем печать через Windows. Таким образом рекомендуется использовать OPOS, особенно в средах розничной торговли, где столбец 40 чеки печатаются и время транзакции должны быть быстро. Для большинства устройств будет использовать управление OPOS. Однако некоторые принтеры чеков OPOS также поддерживают драйверов Windows. С помощью драйвера Windows, можно получить доступ к последней шрифты и сети один принтер для нескольких ККМ. Тем не менее есть недостатки с помощью драйверов Windows. Вот несколько примеров эти недостатки:

-   При использовании Windows водители изображения выводятся перед выполнением печати. Таким образом печати обычно медленнее, чем на принтерах, использующих элементы управления OPOS.
-   Устройства, подключенные через принтер («последовательно подключенных») может не работать правильно при использовании драйверов Windows. К примеру денежный ящик не может открыть или принтера накладной не может word должным образом.
-   OPOS также поддерживает более широкий набор переменных, относящихся к принтеры чеков розничной торговли, таких как снижение бумаги или печати отборочных накладных.

Если OPOS элементы управления доступны для используемого принтера Windows, принтер будет продолжать работать правильно с Microsoft Dynamics 365 для операций - розничной торговли.

### <a name="universal-windows-platform"></a>Платформа Windows в формате универсального

UWP, в случае периферийных устройств розничной торговли, относится поддержка самонастраивающихся устройств в Windows. При подключении самонастраиваемого устройства к версии операционной системы Windows, которая поддерживает этот тип устройства, драйвер не является обязательным для оборудования, которое будет использоваться как планировалось. Например, если Windows обнаруживает устройство Bluetooth динамика, операционная система знает, что устройство имеет **динамика** типа класса. Таким образом и он считает динамика данному устройству. Дополнительная настройка не требуется. В случае Торговых терминалов можно подключить многие USB-устройства и Windows распознает их как HID-устройства (HIDs). Тем не менее он не может быть возможность определить возможности, предоставляемые устройства, так как устройство не указывает класс или тип устройства. Классы устройств для сканеров штрих-кода и MSR были добавлены Windows 10. Таким образом Если устройство объявляет себя Windows 10 как устройство с одним из этих классов, Windows будет отслеживать события от устройства в соответствующее время. Современное POS поддерживает MSR UWP и сканеры. Таким образом когда он готов для входных данных из одного из этих устройств и подключено устройство, к которому относится к одному из этих классов, можно использовать устройство. Например если сканер штрих-кодов UWP подключен к компьютеру Windows 10, а вход штрих-код настроен для современного POS, сканер штрих-кодов станет активной на экране входа. Дополнительная настройка не требуется. Дополнительные классы пункта обслуживания UWP устройств добавляются в систему. Эти классы включают классы, денежные ящики и принтеры чеков. Поддержка этих новых классов устройств в современной POS находится в состоянии ожидания.

### <a name="keyboard-wedge"></a>Сектору клавиатуры

Клавиатуры сектору отправки данных на компьютер, как если бы эти данные были введены на клавиатуре. Таким образом по умолчанию поля, является активным в POS-получит данные, отсканированы или считан. В некоторых случаях это может привести к неправильный тип данных, просматриваемых в том поле. Например в поле, предназначены для ввода данных кредитной карты сканирования штрих-кода. Во многих случаях нет логики на POS, определяет, является ли данные, отсканированы или считан штрих-код или считывание карты. Таким образом обрабатываются данные правильно. Однако при устройств настроены как OPOS вместо клавиатуры сектору, имеется больший контроль над как данные из этих устройств могут потребляться, более «поскольку известно, что» об устройстве, рассчитанная на основе данных. Например данные из сканер штрих-кодов автоматически считались штрих-кода и связанные записи в базе данных найдена, легче и быстрее, чем если использовались поиск общей строки, как в случае сектору клавиатуры.

### <a name="native-printer"></a>Встроенная принтера

Встроенная (или «Устройство» в качестве типа указан в профиле оборудования) принтеры могут быть настроены для предложить пользователю выбрать принтер, настроенный для компьютера. Когда принтер из **устройство** тип настраивается современное POS обнаруживает команды print, пользователю будет предложено выбрать принтер в списке. Это поведение отличается от поведения для драйверов Windows, поскольку **Windows** тип принтера в профиле не отображает список принтеров. Вместо этого его требуется именованный принтера в **имя устройства** поле.

### <a name="windows"></a>Окна

**Windows** тип устройства используется только для принтеров. Если принтера Windows настроен в профиле оборудования, необходимо указать имя конкретного принтера. Современное POS встретится события печати, если принтер Windows настроен, события будут передаваться указанного принтера Windows. Пользователю не будет предложено выбрать принтер.

### <a name="network"></a>Сеть

Сети адресом денежные ящики, принтеры чеков и терминалы оплаты используется через сеть, непосредственно через станции оборудования Interprocess обмена данными (IPC), встроенный в приложении современное POS для Windows или с помощью станции оборудования IIS для других клиентов современное POS.

## <a name="hardware-station-deployment-options"></a>Параметры развертывания станции оборудования
### <a name="ipc-built-in"></a>IPC (встроенные)

На станции оборудования Interprocess обмена данными (IPC), встроенная в приложение современное POS для Windows. Чтобы использовать станцию IPC оборудования, назначьте профиль оборудования регистр, который будет использоваться современное POS для Windows application. Затем создайте станцию оборудования из **выделенном** типа для магазина, где будет использоваться ККМ. При запуске современное POS IPC оборудования станции будут активны, и периферийного оборудования POS-терминала, которые были настроены будет готово к работе. Если временно не требуется локальная оборудования для какой-либо причине, используйте **оборудования станции управления** операции для отключения возможности станции оборудования. Современное POS станции IPC оборудования можно также использовать для прямой связи с периферийных устройств сети.

### <a name="iis"></a>IIS

Можно использовать экземпляр сервера IIS или отдельного станции оборудования двумя способами. Дескриптор «Службы IIS» предполагает, что POS-приложение подключается к станции оборудования через службы IIS. POS-приложение подключается к IIS оборудования станции через веб-службы, выполняющиеся на компьютере, где подключения устройств. При использовании IIS периферийных устройств розничной торговли, подключенных к станции оборудование может использоваться с любой ККМ POS, в сети, в которой на станции оборудования служб IIS. Поскольку только современное POS для Windows включает встроенную поддержку периферийного оборудования для розничной торговли, все остальные современное POS-приложения должны использовать станцию оборудования IIS для взаимодействия с POS периферийных устройств, которые настраиваются в профиле оборудования. Таким образом каждый экземпляр станции оборудования IIS требует компьютер, использующий веб-службы и приложения, которое обменивается данными с устройствами. На станции оборудования IIS является обязательным для всех приложений современное POS не - Windows.

#### <a name="dedicated"></a>Выделенный

Современное POS используется оборудование станции **выделенном** типа, чтобы обнаружить, что периферийные устройства подключены непосредственно на компьютер, где используется приложение. Тем не менее **выделенном** тип также может использоваться для станции оборудования служб IIS. В традиционных, фиксированная POS сценарий, использующий Cloud POS как POS-приложение **выделенном** для станции оборудования служб IIS, которые развертываются на том же компьютере, на котором выполняется Cloud POS используется тип станции оборудования. С точки зрения периферийных устройств розничной торговли специальное оборудование станции IIS лучше retail периферийных устройств поддержку традиционных, основных сценариях POS. Специальное оборудование станции поддерживают все периферийные устройства, которые поддерживаются в профиле оборудования.

#### <a name="shared"></a>Общие

Общие оборудования станции предназначены для использования нескольких Торговых терминалов посредством курса дня. Оборудование, которое станции оптимизированы для поддержки только денежные ящики, принтеры чеков и терминалы оплаты в общем доступе. При подключении непосредственно сканеры автономным штрих-код, MSR, строке будет показано, шкал или других устройств. В противном случае конфликтов произойдет при нескольких Торговых терминалов для выдачи этих периферийных устройств, в то же время. Ниже приводится порядок управления конфликтов для поддерживаемых устройств:

-   **Кассовый лоток** — открытия денежного ящика через события, отправляемые на устройство. Единственная проблема, возникающих при вызове денежного ящика возникает, если уже открыт в кассовом лотке. В случае станции общего оборудования, должно быть указано в кассовом лотке **Shared** в профиле оборудования. Этот параметр отключает проверку, кассовый лоток открыта ли уже при передаче команд открыть POS.
-   **Принтера чеков** — Если два прихода команды печати отправляются на станцию оборудования в то же время, одну из команд могут быть потеряны, в зависимости от устройства. Некоторые устройства имеют внутреннюю память или группировки в пул, можно предотвратить эту проблему. Если команды print не успешно, кассир получает сообщение об ошибке и может повторить команду печати в POS.
-   **Терминал оплаты** — если кассир пытается платежное средство, если проводка по оплате терминала, которая уже используется, сообщение уведомляет кассиру, что терминала, используется и выводится запрос кассиру о повторите попытку позже. Как правило кассиры обнаружила, что терминал уже используется и будет ожидать, пока другая транзакция должна быть завершена до попытке платежное средство еще раз.

В будущих версиях, чтобы обнаружить, настроены ли устройств не поддерживается для профиля оборудования, отображаемый в станцию общей оборудования планируется проверки. Если обнаружены какие-либо устройства не поддерживается, пользователь получит сообщение о том, что устройства не поддерживаются для станции общего оборудования. В случае станции общего оборудования **выберите после оплата** был установлен в **Да** на уровне ККМ. Затем пользователей POS предлагается выбрать станцию оборудования при выборе платежного средства для проводки на POS. При выборе оборудования станции только во время платежного средства, выбор станции оборудования добавляется непосредственно к POS workflow-процесс для мобильных сценариев. Как дополнительное преимущество строковый дисплей на терминал оплаты не используется для общих сценариев. Если терминалу оплаты используется как дисплей, другим пользователям может быть заблокирован с использованием этого терминала до завершения транзакции. В мобильных ПК строки могут быть добавлены проводки за длительный период. Таким образом **выберите после оплата** параметр необходим для обеспечения доступности устройства оптимальный.

### <a name="network-peripherals"></a>Периферийные устройства в сети

Обозначение устройства в профиле оборудования, сети позволяет денежные ящики принтеры чеков и терминалы оплаты должны быть подключены через сетевое подключение.

#### <a name="modern-pos-for-windows"></a>Современное POS для Windows

Можно указать IP-адреса для периферийных устройств сети в двух местах. Современное POS клиент использует один набор периферийных устройств в сети, нужно устанавливать IP-адреса для этих устройств с помощью **конфигурации IP** параметр в области действий для регистра сам. В случае сетевых устройств, которые будут использоваться совместно между ККМ POS профиль оборудования с сетевых устройств, связанных с ним может быть сопоставлен непосредственно станцию общей оборудования. Для назначения IP-адреса выберите станции оборудования на **розничные магазины**, а затем **конфигурации IP** параметр в **станции оборудования** раздел, чтобы указать сетевых устройств, назначенных для станции оборудования. Для оборудования станции, которые имеют только сетевые устройства не нужно развертывать станции оборудования, сам. В этом случае станции оборудования нужно только по существу группировки адресации сетевых устройств в соответствии с их расположением в розничного магазина.

#### <a name="cloud-pos-modern-pos-for-ios-and-modern-pos-for-android"></a>Современное POS для Android, современное POS для операций ввода-вывода и Cloud POS

Логики, которая управляет физически подключен и сетевой адресации периферийных устройств, содержащихся в станции оборудования. Таким образом для всех клиентов POS, за исключением современное POS для Windows, станции оборудования служб IIS должно быть развернутой и активных для включения POS взаимодействует с периферийных устройств, вне зависимости от состояния этих периферийных устройств физически подключен к станции оборудования устранить по сети.

### <a name="modern-pos-for-windows-with-an-ipc-built-in-hardware-station"></a>Современное POS для Windows с помощью станции оборудования IPC (встроенные)

Такая настройка является наиболее типичная конфигурация для обычных, основных ККМ POS. В этом случае сведения о профиле оборудования сопоставляется непосредственно сам ККМ. Номер терминала электронных ПЛАТЕЖЕЙ, также должен быть установлен на сам ККМ. Чтобы настроить эту конфигурацию, выполните следующие действия.

1.  Создание профиля оборудования, где настроены все необходимые периферийные устройства.
2.  Сопоставьте профиля оборудования ККМ POS.
3.  Создание станцию оборудования из **выделенном** типа для розничного магазина, где будет использоваться ККМ POS. Описание не является обязательным. **Примечание:** нет необходимости установите другие свойства на станции оборудования. Все другие необходимые сведения, например, в профиле, будут взяты из самого ККМ.
4.  Щелкните **розничной торговли и электронной коммерции**&gt;**Retail ИТ**&gt;**график распределения**.
5.  Выберите **1090** график распределения для синхронизации новый профиль оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
6.  Выберите **1040** график распределения для синхронизации новую станцию оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
7.  Установка и активация современное POS для Windows.
8.  Запустите современное POS для Windows и начать использовать подключенных периферийных устройств.

### <a name="all-modern-pos-clients-that-have-a-dedicated-iis-hardware-station"></a>Все клиенты современное POS, имеющие специальное оборудование станции IIS

Эта конфигурация может использоваться для всех клиентов современное POS, имеющих станцию оборудования, используемый исключительно по одной POS регистрации. Чтобы настроить эту конфигурацию, выполните следующие действия.

1.  Создание профиля оборудования, где настроены все необходимые периферийные устройства.
2.  Создание станцию оборудования из **выделенном** типа для розничного магазина, где будет использоваться ККМ POS.
3.  На станции специальное оборудование задайте следующие свойства:
    -   **Имя узла** — имя компьютера, на которой будет выполняться станции оборудования. **Примечание:** должен иметь возможность разрешать Cloud POS **localhost** для определения локального компьютера, где выполняется Cloud POS. Однако сертификат, который нужно сопоставить с помощью станции оборудования Cloud POS должно быть «Localhost» как имя компьютера. Чтобы избежать проблем, рекомендуется указать экземпляр каждой станции специальное оборудование для магазина, при необходимости. Для каждой станции оборудования имя узла должны быть определенное имя компьютера для развертывания станции оборудования.
    -   **Порт** — порт для станции оборудования для связи с клиентом современное POS.
    -   **Профиль оборудования** — Если профиль оборудования не условии на сам станции оборудования, будет использоваться профиль оборудования, который назначается ККМ.
    -   **Номер POS для электронных ПЛАТЕЖЕЙ** — код терминала электронных ПЛАТЕЖЕЙ для использования при отправке электронных ПЛАТЕЖЕЙ авторизации. Этот код предоставляется процессора кредитных карт.
    -   **Имя пакета** – пакет станции оборудования для использования при развертывании станции оборудования.

4.  Щелкните **розничной торговли и электронной коммерции**&gt;**Retail ИТ**&gt;**график распределения**.
5.  Выберите **1090** график распределения для синхронизации новый профиль оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
6.  Выберите **1040** график распределения для синхронизации новую станцию оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
7.  Установите станцию оборудования. Дополнительные сведения о том, как установить станцию оборудования см. в разделе [Retail конфигурация оборудования и установки](retail-hardware-station-configuration-installation.md).
8.  Установка и активация современное POS. Дополнительные сведения об установке современное POS-терминала см. в разделе [Retail POS современное конфигурации и установки](retail-modern-pos-device-activation.md).
9.  входа в систему POS современное и выберите **выполнять операции, не являющихся лоток**.
10. Запуск **оборудования станции управления** операции.
11. Щелкните **управления**.
12. На странице управления станции оборудования параметра для включения станции оборудования.
13. Выберите станцию оборудования, а затем щелкните **паре**.
14. После паре станции оборудования, нажмите кнопку **закрыть**.
15. На странице Выбор станции оборудования щелкните станции последний выбранный оборудования, чтобы сделать его активным.

### <a name="all-modern-pos-clients-that-have-a-shared-iis-hardware-station"></a>Все клиенты современное POS, имеющие станцию оборудования общих служб IIS

Эта конфигурация может использоваться для всех клиентов современное POS, использующих станции оборудования с другими устройствами. Чтобы настроить эту конфигурацию, выполните следующие действия.

1.  Создание профиля оборудования, где требуется периферийных устройств настроены.
2.  Создание станцию оборудования из **Shared** типа для розничного магазина, где будет использоваться ККМ POS.
3.  На станции общего оборудования задайте следующие свойства:
    -   **Имя узла** — имя компьютера, на которой будет выполняться станции оборудования.
    -   **Описание** — текст, который поможет определить станции оборудования, например, **возвращает** или **титульной хранилища**.
    -   **Порт** — порт для станции оборудования для связи с клиентом современное POS.
    -   **Профиль оборудования** — для станции общего оборудования каждой станции оборудования должна иметь профиль оборудования. Профили оборудования могут совместно использоваться станции оборудования, но они должны быть сопоставлены с каждой станции оборудования. Кроме того Корпорация Майкрософт рекомендует использовать общий смены, при нескольких устройств с помощью одной станции общего оборудования. Чтобы настроить общие смены, щелкните **розничной торговли и электронной коммерции**&gt;**настройку канала**&gt;**настройки POS**&gt;**профилей POS**&gt;**профили оборудования**. Для каждого профиля оборудования общей выберите кассовый лоток и **ящика смены Shared** параметр для **Да**.
    -   **Номер POS для электронных ПЛАТЕЖЕЙ** — код терминала электронных ПЛАТЕЖЕЙ для использования при отправке электронных ПЛАТЕЖЕЙ авторизации. Этот код предоставляется процессора кредитных карт.
    -   **Имя пакета** – пакет станции оборудования для использования при развертывании станции оборудования.

4.  Повторите шаги 2 и 3 для каждой станции дополнительное оборудование, необходимое в магазине.
5.  Щелкните **розничной торговли и электронной коммерции**&gt;**Retail ИТ**&gt;**график распределения**.
6.  Выберите **1090** график распределения для синхронизации новый профиль оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
7.  Выберите **1040** график распределения для синхронизации новую станцию оборудования для магазина в целом. Щелкните **запустить сейчас** для синхронизации изменений в POS.
8.  Установите станцию оборудование на каждом компьютере узла, который настроен на шагах 2 и 3. Дополнительные сведения о том, как установить станцию оборудования см. в разделе [Retail конфигурация оборудования и установки](retail-hardware-station-configuration-installation.md).
9.  Установка и активация современное POS. Дополнительные сведения об установке современное POS-терминала см. в разделе [Retail POS современное конфигурации и установки](retail-modern-pos-device-activation.md).
10. входа в систему POS современное и выберите **выполнять операции, не являющихся лоток**.
11. Запуск **оборудования станции управления** операции.

12. Щелкните **управления**.
13. На странице управления станции оборудования параметра для включения станции оборудования.
14. Выберите станцию оборудования, а затем щелкните **паре**.
15. Повторите шаг 14 для каждой станции оборудования, который будет использовать современное POS.
16. После сопоставляются все станции требуемого оборудования, нажмите кнопку **закрыть**.
17. На странице Выбор станции оборудования щелкните станции последний выбранный оборудования, чтобы сделать его активным. **Примечание:** Если устройства часто используют станции другое оборудование, рекомендуется настроить современное POS для запроса кассирам для выбора станцию оборудования, когда они начать процесс платежного средства. Щелкните **розничной торговли и электронной коммерции**&gt;**настройку канала**&gt;**настройки POS**&gt;**регистрирует**. Выбор формы регистрации, а затем установите **выберите после платежное средство** параметр для **Да**. Используйте **1090** график распределения, чтобы синхронизировать изменения, внесенные в базу данных канала.

## <a name="security"></a>Контроль доступа
В соответствии с текущим стандартам безопасности, в рабочей среде следует использовать следующие параметры: **Примечание:** станции оборудования программа установки автоматически вносит эти изменения реестра во время установки через дистанционное обслуживание.

-   Следует отключить Secure Sockets Layer (SSL).
-   Только безопасность TLS (Transport Layer) версии 1.2 (или текущая версия самый высокий) должна включено и использоваться. **Примечание:** по умолчанию отключены все версии TLS, за исключением 1,2 TLS и SSL. Чтобы изменить или включить эти значения, выполните следующие действия:
    1.  Сочетания клавиш Windows клавиш + R, чтобы открыть **выполнения** окно.
    2.  В **откройте** введите **Regedit**, а затем щелкните **OK**.
    3.  Если **контроль учетных записей** появляется окно сообщения, нажмите кнопку **Да**.
    4.  В **редактора реестра** окно, перейдите к **HKEY\_локальной\_MACHINESystemCurrentControlSetSecurityProvidersSCHANNELProtocols**. Следующие разделы были автоматически вводится только для TLS 1,2 обеспечения:
        -   TLS 1.2Server: включен = 1
        -   TLS 1.2Server:DisabledByDefault = 0
        -   TLS 1.2Client: включен = 1
        -   TLS 1.2Client:DisabledByDefault = 0
        -   TLS 1.1Server: включен = 0
        -   TLS 1.1Client: включен = 0
        -   TLS 1.0Server: включен = 0
        -   TLS 1.0Client: включен = 0
        -   SSL 3.0Server: включен = 0
        -   SSL 3.0Client: включен = 0
        -   SSL 2.0Server: включен = 0
        -   SSL 2.0Client: включен = 0
-   Нет дополнительных сетевых портов должен быть открыт, если они необходимы для известных, указанным причинам.
-   Совместное использование ресурсов между происхождения должен быть отключен и необходимо указать допустимых источников, которые принимаются.
-   Только доверенных центров сертификации можно использовать для получения сертификатов, которые будут использоваться на компьютерах под управлением станции оборудования.

**Примечание:** очень важно изучить рекомендации по безопасности для служб IIS и требования индустрии платежных карт (PCI).

## <a name="troubleshooting"></a>Устранение неполадок
### <a name="modern-pos-can-detect-the-hardware-station-in-its-list-for-selection-but-it-cant-complete-the-pairing"></a>Современное POS можно обнаружить станции оборудования в списке для выбора, но она не может выполнить связывания

**Решение:** проверьте потенциальной точки сбоя описанные ниже:

-   Компьютер, на котором выполняется современное POS доверия сертификата, который используется на компьютере, на котором выполняется на станции оборудования.
    -   Для проверки этой настройки в веб-обозревателе, перейдите к следующему URL-АДРЕСУ: https://&lt;имя компьютера&gt;:&lt;номер порта&gt;/HardwareStation/ping.
    -   Этот URL-адрес использует ping для убедитесь, что компьютер может осуществляться и обозреватель указывает, является ли сертификат доверенных. (Например, в обозревателе Internet Explorer появляется значок замка в адресной строке. Если щелкнуть этот значок, Internet Explorer проверяет, является ли в настоящее время доверенного сертификата. Можно установить сертификат на локальном компьютере при просмотре сведений о сертификате, которая отображается.)
-   На компьютере, выполняемый станции оборудования в брандмауэре открыт порт, который будет использоваться на станции оборудования.
-   Станции оборудование установлено правильно сведения о счете продавца продавца сведения о программе установки, выполняемый в конце станции оборудования программа установки.

### <a name="modern-pos-cant-detect-the-hardware-station-in-its-list-for-selection"></a>Современное POS не удается определить станции оборудования в списке для выбора

**Решение:** любой из следующих факторов может вызвать эту проблему:

-   На станции оборудования еще не настроены в центральный офис. Описанные ранее в этом разделе для проверки того, правильно ввести станции в профиле оборудования и станции оборудования.
-   Чтобы обновить конфигурацию канала еще не запущено заданий. В этом случае следует запустите задание 1070 для настройки канала.

### <a name="modern-pos-doesnt-reflect-new-cash-drawer-settings"></a>Современное POS не отражают новые параметры денежного ящика

**Решение:** закрывается текущий пакет. Изменения в кассовый лоток не обновляются современное POS до закрытия текущего пакета.

### <a name="modern-pos-is-reporting-an-issue-with-a-retail-peripheral"></a>Современное POS сообщение о проблеме с периферийных устройств розничной торговли

**Решение:** ниже приведены некоторые типичные причины этой проблемы:

-   Убедитесь, что закрыты других служебных программ настройки драйвера устройства. Если открыты эти служебные программы, они могут препятствовать современное ЗАКАЗЫ на покупку или на станцию оборудования информация по получению устройства.
-   Периферийных устройств розничной торговли используется совместно с нескольких Торговых терминалов, убедитесь, что он принадлежит к одной из следующих категорий:
    -   Денежный ящик
    -   Принтера чеков
    -   Платежный терминал 

    Если периферийных устройств не принадлежит к одной из этих категорий, станции оборудования не обеспечивает возможность периферийных устройств, совместно используемых несколькими устройствами POS.
-   В некоторых случаях драйвер устройства может привести к компонент common control objects (CCOs) работает неправильно. Если устройство недавно был установлен, но не работает должным образом или появиться другие проблемы, часто можно устранить проблему, переустановки CCOs. Загрузить CCOs <http://monroecs.com/oposccos_current.htm>.
-   Если изменения вносятся часто периферийных устройств во время проверки или устранения неполадок, может потребоваться сбросить IIS вместо ожидания для кэша для обновления сам. Чтобы сбросить IIS, выполните следующие действия:
    1.  От **начать** меню, тип **CMD**.
    2.  В результатах поиска, щелкните правой кнопкой мыши **командной строки**, а затем щелкните **Запуск от имени администратора**.
    3.  В **командной строки** окно, тип **iisreset /Restart** и нажмите клавишу ВВОД.
    4.  После перезапуска IIS перезапуск современное POS.
-   Время выполнения часто происходят изменения для периферийных устройств, если также часто запуска и выхода из клиента POS, процесс dllhost из предыдущего сеанса POS могут мешать текущего сеанса. В этом случае устройство может стать недоступной до закрытия узла библиотеки динамической компоновки (DLL), который управляет предыдущего сеанса. Чтобы закрыть узел библиотеки DLL, выполните следующие действия:
    1.  От **начать** меню, тип **диспетчера задач**.
    2.  В результатах поиска, щелкните **диспетчера задач**.
    3.  В диспетчере задач на **сведения о**, щелкните заголовок столбца под названием **имя** Сортировка в таблице в алфавитном порядке по имени.
    4.  Прокрутите вниз, пока не найдете dllhost.exe.
    5.  Выберите каждый узел библиотеки DLL, а затем щелкните **завершить задачу**.
    6.  После закрытия узлов DLL перезапустите современное POS.
