# <a name="personalized-product-recommendations-overview"></a>Обзор рекомендаций индивидуальными продукции

В Dynamics 365 для операций рекомендации по продукции могут отображаться на кассе устройства продаж (POS). Рекомендации по это номенклатуры, которые клиент может быть заинтересован в на основе их История приобретений, номенклатур в список планируемых покупок и номенклатуры, приобретенные остальных клиентов в интерактивном режиме, а также и физическими магазинами. Для предприятий розничной торговли с большими каталоги рекомендации помочь пользователю с обнаружения продукции. Демонстрации продуктов, предназначенных для клиента интересы и график, дающей, рекомендации по продукции может помочь предприятий розничной торговли и увеличения продаж и перекрестные продажи и повысить удержания клиента. В Dynamics 365 for Operations рекомендации по продукции на базе нарушений распознавания службы и машинного обучения Microsoft Azure.

<a name="scenarios"></a>Сценарии
---------

Рекомендации по продукции, включаются в следующих сценариях POS. Они доступны в облаке ЗАКАЗЫ на покупку или современное POS (MPOS).

1.  На **подробные сведения о** страницы:

-   Если магазин связать визиты **подробные сведения о** странице при просмотре предыдущие проводки на разных каналах, ядро рекомендаций предлагает дополнительные элементы, скорее всего, будут покупаться вместе.
-   Если связать хранилище клиента будут добавлены в проводку, а затем физическое посещение **подробные сведения о** страницы, ядро рекомендаций рекомендации индивидуальными с помощью истории проводок клиента.

[![proddetails](./media/proddetails.png)](./media/proddetails.png)

1.  На **проводки** страницы:

-   Ядро рекомендаций рекомендует номенклатуры на основе весь список номенклатур в корзине.
-   Если связать хранилище клиента будут добавлены в проводку, ядро рекомендаций рекомендации личные с помощью истории проводок клиента и список номенклатур в корзине.

**Примечание** для отображения рекомендации на **проводки** необходимых розничной торговли веб-странице, для обновления в выходную форму экрана в Dynamics 365 для операций. **Рекомендации** в систему, необходимо удалить элемент управления **проводки** страницы. [![transactionscreenmultipleproductslargemessengersbag-5](./media/transactionscreenmultipleproductslargemessengersbag-5.jpg)](./media/transactionscreenmultipleproductslargemessengersbag-5.jpg)

1.  На **сведения о клиенте** страницы:
    -   Ядро рекомендаций рекомендует номенклатуры на основе идентификатора пользователя и номенклатур в список планируемых покупок клиента.

[![customerdetailsrecommendations](./media/customerdetailsrecommendations.png)](./media/customerdetailsrecommendations.png)

## <a name="configure-dynamics-365-for-operations-to-enable-pos-recommendations"></a>Настройка Dynamics 365 для операций, чтобы включить рекомендации POS
Чтобы настроить рекомендации по продукции, необходимо выполнить следующие.

1.  Убедитесь, что заданы правильные **компании**.
2.  Перейдите к **объекта хранилища**, выберите **розничной продажи**и нажмите кнопку **обновить**. ** ** это использовать демонстрационные данные (или данных) из базы данных рабочих и перемещать их в положение объекта хранилища.
3.  Необязательно: Чтобы отобразить рекомендации на экране проводки, перейдите на ** макет экрана **выбрать макет экрана, запуска **конструктор макета экрана**,** **, а затем удалите ** управления рекомендации ** при необходимости.
4.  Последовательно выберите пункты **параметры Retail**, выберите **машинного обучения**, выберите ** Да ** под **включить POS рекомендации**.
5.  Чтобы просмотреть рекомендации на POS-Терминале, запустите задание Глобальная конфигурация **1110**. Для отражения изменений, внесенных в конструктор макета экрана POS, запустить задание конфигурации канала **1070**.

## <a name="how-does-it-work"></a>[]()Как это работает?
При обновлении **объекта хранилища** объекта, следующие действия выполняются.

-   Данные в формате, требуемом нарушений распознавания службами извлечения из Dynamics 365 для операций операционной базы данных и отправленных в магазин объекта.
-   Данные используется фабрики данных Azure (ADF) для очистки данных с помощью сценариев куст как часть ПОДАЧИ мероприятий. Очищенным данные хранятся в хранилище двоичных объектов.
-   Данные из хранилища используется службами нарушений распознавания API для обучения модели рекомендаций.

При следующем включении **включить рекомендации** и выполнение заданий, конфигурации, выполняются следующие операции.

-   Модель учетные данные и код кот API и хранящихся в Dynamics 365 для операций операционной базы данных, в файл web.config для AOS, а также на сервере розничной торговли.
-   Модель учетные данные и код становятся доступными для CRT таким образом, может учитываться вызовов для рекомендации продукта от Cloud POS и MPOS в интерактивном режиме.
