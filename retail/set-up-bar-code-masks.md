# <a name="set-up-bar-code-masks"></a>Настройка масок штрих-кодов

В этом разделе описывается настройка символов маски штрих-кода, маски штрих-кодов и каким образом выполняется назначение штрих-код маски штрих-кодов.

<a name="set-up-bar-code-mask-characters"></a>Настройка символов маски штрих-кода
-------------------------------

Маски штрих-кодов используются для создания штрих-кодов и для быстрого определения штрих-кодов, проверяются в точки продажи (POS). Маски состоят из символов, которые действуют как местозаполнители, которые определяют формат для штрих-кодов, которые будут созданы. Для настройки маски штрих-кода, необходимо настроить символы маски штрих-кода. Последовательно выберите пункты **розничной торговли и электронной коммерции**&gt;**Управление запасами**&gt;**штрих-кодов и меток**&gt;**символов маски**. Щелкните **создать** для создания символов маски штрих-кода. Символы маски могут создаваться для указания следующие данные штрих-кода.

|                      |                                                                                                                 |
|----------------------|-----------------------------------------------------------------------------------------------------------------|
| **Field**            | **Description**                                                                                                 |
| **Product**          | Заполнитель для идентификатора продукта.                                                                                     |
| **Any number**       | Используется для указания номера, которые будут жестко закодированные в штрих-коды.                                                  |
| **Check digit**      | Указывает, что формат штрих-кода в маске штрих-кода использует контрольный разряд, чтобы подтвердить действительность штрих-кода. |
| **Size digit**       | Указывает размер в штрих-коде, созданные для варианта продукта, включая размер.                                 |
| **Color digit**      | Указывает цвет в штрих-коде, созданные для варианта продукта, включая цвет.                               |
| **Style digit**      | Указывает стиль в штрих-коде, созданные для варианта продукта, который включает стиль.                             |
| **EAN license code** | Заполнитель для лицензии EAN выпущен для кодов лицензии EAN.                                                       |
| **Цена**            | Указывает цены цены встроенных штрих-кодов.                                                                   |
| **Quantity**         | Указывает количество в поле Кол-во/случайной веса embedded штрих-кодов.                                                |
| **Employee**         | Указывает сегмент штрих-кода по номеру сотрудника, используемый для входа в систему POS штрих-кода.                                  |
| **Customer**         | Указывает сегмент кода клиента.                                                                                  |
| **Ввод данных**       | *Еще не реализовано.*                                                                                          |
| **Discount code**    | Указывает код скидки для штрих-код, используемый для добавления скидку на момент продажи проводки             |
| **Подарочный сертификат**        | Указывает номер подарочного сертификата при выдаче или оплаты подарочным сертификатом.                                               |
| **Loyalty card**     | Добавляет постоянного клиента в проводку и может использоваться при оплате по программе лояльности.                             |

## <a name="define-bar-code-masks"></a>Определение маски штрих-кодов
После символов маски штрих-кода указаны для масок необходимости штрих-кода, перейдите к **розничной торговли и электронной коммерции**&gt;**Управление запасами**&gt;**штрих-кодов и меток**&gt;**настройка маски штрих-кода**. На этой странице можно определить маски штрих-кодов, которые используют ранее указанных символов. Эти штрих-код маски будет использоваться при создании штрих-кодов и будет также помочь в определении штрих-кодов сканирования на POS.

1.  Щелкните **создать** для создания новой маски штрих-кода.
2.  Введите значения в **код маски** и **описание** поля, а затем выберите тип маски штрих-кода **типа** поле.
3.  В **Общие** выберите значение в **стандарт штрих-кода** поле, а затем укажите префикс штрих-кода, если это требуется.
4.  В **сегмент маски штрих-кода** разделе, добавлять сегменты штрих-кода, которые будут использоваться в штрих-коде должна быть создана.

Например для создания маски штрих-кода с код маски «Продукт», можно выполнить следующие:

1.  Создание новой маски штрих-кода и Выбор типа «Продукт».
2.  Выберите стандартный штрих-код, например, «Code 39".
3.  Предоставить префикс, используемый для идентификации штрих-кода. Например "22".
4.  Добавьте сегмент маски. Сегмент маски «Продукт» будет установлен.
5.  Обеспечивают длину сегмента продукта, например, "10". Длины должны совпадать длина идентификатора продукта, обычно используемые в магазине. Маска будет отображаться как предварительный просмотр в **Общие** разделе под **маски**.

## <a name="assign-bar-code-masks-to-bar-codes"></a>Назначение штрих-кодов маски штрих-кодов
Маски штрих-кодов должна быть назначена штрих-кодов, прежде чем их можно будет использовать. В приведенном в предыдущем примере, чтобы назначить штрих-код маски штрих-кода, выполните следующие:

1.  Последовательно выберите пункты **управления организацией**&gt;**установки**&gt;**штрих-кодов**. Щелкните **создать** для создания нового штрих-кода.
2.  Введите значения в **штрих-код****установки** и ** настройки ** поля.
3.  В **Общие** в разделе **тип штрих-кода** выберите «Code 39". В **маски****идентификатор** выберите ранее созданных маски «Продукт».
4.  В разделе **размер**, введите "12".
5.  Click **Save**.

Маски штрих-кода используется для создания штрих-кодов для продуктов. Описанные выше шаги являются примерами создания маски штрих-кодов для продуктов, но они также показывают, как создавать маски штрих-кода для любых других типов поддерживаемых штрих-кода. Маски штрих-кодов, типов и длин следует настроить для использования в конкретной среде.
