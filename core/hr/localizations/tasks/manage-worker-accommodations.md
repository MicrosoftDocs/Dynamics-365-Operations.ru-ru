# <a name="manage-worker-accommodations"></a>Управление проживанием работников

[!include[task guide banner](../../../includes/task-guide-banner.md)]

Эта процедура демонстрирует шаги, которые необходимо предпринять для настройки типов удобств, таких как эргономичные стулья или периодические перерывы. В ней также показано, как назначить эти типы удобств работнику и либо утвердить запрос на удобства этого типа, либо отказать в нем. В качестве компании с демонстрационными данными для создания этой процедуры используется USMF.


## <a name="setup-accommodations"></a>Настройка удобств
1. Перейдите в раздел "Управление персоналом" > "Настройка" > "Типы помещений".
2. Щелкните "Создать".
3. В поле "Тип проживания" введите название для удобства. Пример: "Клавиатура".
4. В поле "Описание" введите описание для удобства. Пример: "Эргономичная клавиатура".
5. В поле "Примечание" введите какую-либо информацию, которая помогает понять, в чем суть удобства.
6. Нажмите кнопку "Сохранить".
7. Закройте страницу.

## <a name="assign-accommodations"></a>Назначение удобств
1. Перейдите в раздел "Управление персоналом" > "Работники" > "Сотрудники".
2. Выберите в списке сотрудника.
3. Щелкните имя сотрудника, чтобы просмотреть сведения о сотруднике, который запрашивает удобство.
4. Разверните экспресс-вкладку "Личные данные".
5. Щелкните "Проживание".
6. Нажмите Создать.
7. В поле "Тип проживания" выберите соответствующее удобство. Пример: "Клавиатура"
8. В поле "Задача задания" выберите рабочую задачу, для которой необходимо запрашиваемое удобство.
9. В поле "Статус" введите соответствующий статус. Пример: "Разумно"
    * Предусмотрены следующие статусы. Статус "Запрошено" означает, что запрос на удобство создан, но еще не рассмотрен. Статус "Разумно" означает, что запрос на удобство является разумным и будет удовлетворен. Статус "Чрезмерные трудности" означает, что удовлетворение запроса на удобство будет связано с чрезмерными трудностями для работодателя, поэтому в запросе отказано. В этом примере запрос был утвержден немедленно, поскольку запрос на удобство минимальным.  
10. В поле "Принято" выберите пользователя, который принял запрос на удобство.
11. Щелкните Выбрать.
12. В поле "Дата и время принятия" введите дату и время принятия запроса на удобство.
13. Разверните раздел "Примечание".
14. В поле "Финансовое" введите какую-либо информацию или затраты на ресурсы, которые помогают понять стоимость удобства.
    * Если в запросе на проживание отказано, в поле "Ответ" можно указать, почему в запросе было отказано.  
15. Нажмите кнопку "Сохранить".
