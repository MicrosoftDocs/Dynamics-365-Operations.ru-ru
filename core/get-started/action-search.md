# <a name="action-search"></a>Поиск действий

В этой статье описываются функции поиска действий в Microsoft Dynamics 365 для операций. Поиск действий служит для поиска и выполнения действия на странице.

<a name="introduction"></a>Введение
------------

Страниц в Microsoft Dynamics 365 для операций в первую очередь предоставлять команды на панели операций, стандартной области действий, которая появляется в верхней части страницы и панелей инструментов, которые отображаются в различных разделах страницы. В предыдущих версиях функция советы ключ позволяют быстро получить доступ к любой кнопки на панели операций, нажав клавиши Alt, а затем набора букв. 

[![keyTipsAX6](./media/keytipsax6.png)](./media/keytipsax6.png) тем не менее, в текущей версии Dynamics 365 для операций, ключ советы становятся недоступными, но были заменены функцию поиска действий. Эта новая функция позволяет быстро искать и наживать кнопки на любой видимой панели операций.

## <a name="using-action-search"></a>Использование поиска действий
Для использования функции поиска действий выполните следующие действия.

1.  На панели операций щелкните в поле **поиска действий**. (В поле **поиска действий** находится значок в виде лупы.)
2.  Введите имя или часть имени кнопки, которое следует выполнить. Можно выполнять поиск, используя слова из кнопки «путем.» (Дополнительные сведения см. в следующем разделе этой статьи.) Как правило кнопки отображаются в верхней части списка результатов после ввода двух до четырех символов.
3.  Найдите и нажмите кнопку в результатах (с помощью мыши или клавиатуры).

После нажатия кнопки фокус возвращается туда, где вы до этого находились на странице, чтобы вы могли продолжить работу. 

[![действия поиска](./media/action-search-field.png)](./media/action-search-field.png)

Запустить поиск действий можно также путем нажатия сочетаний клавиш Ctrl+/ или Alt+Q. Нажмите сочетание клавиш еще раз, чтобы фокус вернулся на то место на странице, где вы раньше находились.

## <a name="understanding-the-results-list"></a>Что представляет собой список результатов
Как правило в Dynamics 365 для операций, необходимо знать местоположения и контекст кнопку, чтобы полностью понимать назначение этой кнопки. Таким образом, Дополнительные сведения отображаются для каждого элемента в списке результатов, чтобы точно понять кнопок, отображаемых в списке. В частности, отображается "путь" к кнопке. В этот путь могут входить метки следующих элементов пользовательского интерфейса:

-   Вкладка панели операций
-   Группа кнопок
-   Кнопка меню (если кнопка находится внутри кнопки меню)
-   Разделитель меню (если кнопка находится внутри именованной группы внутри кнопки меню)
-   Группа или вкладка на странице (например, имя экспресс-вкладки)

Предположим, что вы ввели в поле **поиска действий** буквы **ито** и теперь изучаете список результатов. Первая запись для кнопки с именем **итоги**, выделяется. Кнопка путь **заказ на продажу**&gt;**представление** также отображается. **Заказ на продажу** соответствует части пути **заказ на продажу** вкладке в области действий и **представление** соответствует части пути **представление** на этой вкладке. Аналогичным образом, путь к файлу **общей скидки** кнопку (**продать**&gt;**рассчитать**) проинформирует вас о том, что эта кнопка находится в **рассчитать** на **продать** на вкладке панели операций. Таким образом эта информация помогает определить точно какая кнопка инициируемых действие поиска (при нажатии этой кнопки в списке результатов). 

[![действия поиска поле с данных](./media/action-search-field-with-data.png)](./media/action-search-field-with-data.png) 

В предыдущем примере поиск действия показал результаты из стандартной панели действий в верхней части страницы. Однако поиск действий показывает также результаты с видимых панелей инструментов, которые находятся в других местах на странице. Например, выполняя поиск **запасы в наличии**, расположенную на **строк заказа на продажу** экспресс-вкладку. В данном случае кнопку путь, указанный в списке результатов (**строк заказа на продажу**&gt;**складских**&gt;**представление**) проинформирует вас о том, что эта кнопка находится в разделе **представление** заголовок на **складских** на кнопку меню **строк заказа на продажу** экспресс-вкладку. 

[![на имеющихся запасов](./media/on-hand-inventory.png)](./media/on-hand-inventory.png)

## <a name="action-search-vs-navigation-search"></a>Поиск действий и навигационный поиск
А поиска действий предназначена для поиска и выполнять действия на странице, существует механизм отдельный поиска для поиска и навигации по страницам в Dynamics 365 для операций. Дополнительные сведения об этой функции см. в разделе [поиска переходов](navigation-search.md) статьи.
