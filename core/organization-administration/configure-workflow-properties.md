# <a name="configure-the-properties-of-a-workflow"></a>Настроить свойства workflow-процесса

[!include[banner](../includes/banner.md)]


В этом разделе описывается, как настроить различные свойства workflow-процесса.

Чтобы настроить свойства workflow-процесса, откройте workflow-процесс в редакторе workflow-процессов. Щелкните холст редактора workflow-процессов, а затем щелкните **Свойства**, чтобы открыть страницу **Свойства**. Можно использовать следующие процедуры для настройки различных свойств workflow-процесса.

## <a name="name-the-workflow"></a>Указать название документооборота
Чтобы ввести название workflow-процесса, необходимо выполнить следующие действия.

1.  В левой области нажмите **Основные настройки**.
2.  В поле **Имя** введите уникальное название документооборота. Например, если требуется создать workflow-процесс заявки на покупку для каждой страны или региона, в которых работает компания, можно назвать workflow-процесс заявки на покупку **Заявки на покупку: Дания** или **Заявки на покупку: Испания**.

## <a name="specify-the-workflow-owner"></a>Указать владельца документооборота
Владелец workflow-процесса — это лицо, которое будет управлять данным workflow-процесс и вести его. Чтобы указать владельца workflow-процесса, необходимо выполнить следующие действия.

1.  В левой области нажмите **Основные настройки**.
2.  Из списка **Владелец** выберите имя сотрудника, который будет управлять workflow-процессом.

## <a name="select-an-email-template"></a>Выберите шаблон электронной почты.
Выполните следующие шаги для выбора шаблона электронной почты, который будет использоваться для создания уведомлений о workflow-процессе.

1.  В левой области нажмите **Основные настройки**.
2.  В списке **Шаблон электронной почты для уведомлений workflow-процесса** выберите шаблон.

## <a name="enter-instructions-for-users"></a>Ввести инструкции для пользователей
Можно добавить инструкции для пользователей, представляющих документы на обработку и утверждения. Эти пользователи также называются *инициаторами*. Например, вы создаете workflow-процесс заявки на покупку и вводите инструкции. Эти инструкции смогут просматривать пользователи, которые вводят заявки на покупку на странице **Заявки на покупку**. Исходный пользователь может щелкнуть значок в строке сообщений workflow-процесса, чтобы просмотреть инструкции. Чтобы ввести инструкции для пользователей, необходимо выполнить следующие действия.

1.  В левой области нажмите **Основные настройки**.
2.  В поле **Инструкции по отправке** введите инструкции.
3.  Для того, чтобы персонализовать инструкции можно вставить заполнители. Заполнители заменяются соответствующей информацией, когда инструкции отображаются пользователям. Выполните следующие действия, чтобы вставить заполнитель:
    1.  Щелкните в поле **Инструкции по отправке**, чтобы указать, где должен находиться заполнитель.
    2.  Щелкните **Вставить заполнитель**.
    3.  В открывшемся списке выберите заполнитель, который необходимо вставить.
    4.  Нажмите **Вставить**.

4.  Для добавления переводов инструкций выполните следующие действия:
    1.  Щелкните **Переводы**.
    2.  На открывшейся странице щелкните **Добавить**.
    3.  В отображаемом списке выберите язык, на котором будет вводиться текст.
    4.  В поле **Переведенный текст** введите текст.
    5.  Для того, чтобы персонализовать текст можно вставить заполнители. Инструкции по вводу заполнителя см. на шаге 3.
    6.  Нажмите кнопку **Закрыть**.

## <a name="specify-when-this-workflow-is-used"></a>Указать, когда данный workflow-процесс используется
Можно создать несколько workflow-процессов на основе одного и того же типа. Например, вы создаете workflow-процесс заявки на покупку для каждой страны или региона, в которых работает компания, например "Заявки на покупку: Дания" и "Заявки на покупку: Испания". Если в основе нескольких workflow-процессов лежит один и тот же тип, необходимо указать, когда следует использовать каждое workflow-процесс. В приведенном выше примере вы указали следующие условия:

-   Заявки на закупку: Дания используется, когда страна (регион) = DK
-   Заявки на закупку: Испания используется, когда страна (регион) = ES

Чтобы указать, когда следует использовать настраиваемый workflow-процесс, необходимо выполнить следующие действия.

1.  В левой области щелкните **Активация**.
2.  Установите флажок **Установка условий для запуска этого workflow-процесса**.
3.  Щелкните **Добавить условие**.
4.  Введите условие.
5.  Введите все необходимые дополнительные условия.
6.  Чтобы убедиться, что введенные условия настроены верно, выполните следующие действия.
    1.  Щелкните **Тест**.
    2.  На странице **Проверить условие workflow-процесса** в области **Проверить условие** выберите запись.
    3.  Щелкните **Тест**. Система оценит запись и определит, соответствует ли она определенным вами условиям. Например, если вы создаете workflow-процесс заявки на покупку для Испании, в области **Проверка условия** страницы будет отображаться список заявок на покупку. Если нажать кнопку **Тест**, система проверит выбранную заявку на покупку, чтобы определить, является ли страной/регионом ES.
    4.  Нажмите кнопку **OK** или **Отмена** для возврата на страницу **Свойства**.

## <a name="specify-when-notifications-are-sent"></a>Задать условия отправки уведомлений
Когда документ представлен на обработку, создается экземпляр workflow-процесса. При запуске, выполнении, завершении или отмене экземпляров workflow-процесса (созданных на основе данного workflow-процесса), а также при их остановке из-за ошибки пользователям могут быть отправлены уведомления. Чтобы указать, когда следует отправлять уведомления, необходимо выполнить следующие действия.

1.  В левой области щелкните **Уведомления**.
2.  Установите флажок для каждого события, запускающего уведомления:
    -   **Запущено** — отправка уведомлений при запуске экземпляра workflow-процесса.
    -   **Остановлено** — уведомления будут отправляться, если экземпляр workflow-процесса остановлен из-за ошибки.
    -   **Выполнено** — уведомления будут отправляться при выполнении экземпляра workflow-процесса.
    -   **Неустранимо** — уведомления будут отправляться, если экземпляр workflow-процесса остановлен из-за неустранимой ошибки.
    -   **Завершено** — уведомления будут отправляться при завершении экземпляра workflow-процесса.

3.  Выберите строку для события, выбранного на шаге 2.
4.  На вкладке **Текст уведомления** введите текст уведомления.
5.  Для того, чтобы персонализовать текст можно вставить заполнители. Заполнители заменяются соответствующей информацией, когда текст отображается пользователям. Выполните следующие действия, чтобы вставить заполнитель:
    1.  Щелкните в поле ,чтобы указать, где должен находиться заполнитель.
    2.  Щелкните **Вставить заполнитель**.
    3.  В открывшемся списке выберите заполнитель, который необходимо вставить.
    4.  Нажмите **Вставить**.

6.  Для добавления переводов текста выполните следующие действия:
    1.  Щелкните **Переводы**.
    2.  На открывшейся странице щелкните **Добавить**.
    3.  В отображаемом списке выберите язык, на котором будет вводиться текст.
    4.  В поле **Переведенный текст** введите текст.

    6.  Нажмите кнопку **Закрыть**.

## <a name="enter-comments-about-the-changes-that-you-made-to-the-workflow"></a>Ввод комментариев об изменениях, внесенных в workflow-процесс
Чтобы ввести комментарии об изменениях, внесенных в workflow-процесс, необходимо выполнить следующие действия.

1.  В левой области щелкните **Примечания**.
2.  В поле **Введите комментарии о workflow-процессе** введите комментарии.
3.  Проверьте комментарии. После добавления комментариев их невозможно изменить.
4.  Нажмите кнопку **Добавить** для добавления комментариев в область **История комментариев**.
