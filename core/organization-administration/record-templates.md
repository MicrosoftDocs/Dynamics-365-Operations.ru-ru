# <a name="create-record-templates"></a>Создание шаблонов записей

[!include[banner](../includes/banner.md)]


Эта статья вводит понятие шаблонов записей и описывает порядок, как их можно использовать для создания записей, совместно использующих сведения.

Шаблоны записей позволяют быстрее создавать записи в системе Microsoft Dynamics 365 for Operations. Шаблоны записей могут быть созданы только для определенных типов записей в Microsoft Dynamics 365 for Operations. Например, **** представьте, что вы вводите арендную информацию для фирмы по прокату автомобилей, которая расположено в Сан-Франциско. В виду того что большинство клиентов, скорее всего, будут из региона Сан-Франциско, было бы славно, если вы могли автоматически заполнить значения для полей **Штат**, **Страна** и **Город** в форме аренды. **Примечание** Можно применять шаблоны только для тех областей Microsoft Dynamics 365 for Operations, к которым имеется доступ. Все заголовки шаблонов видны пользователю при создании новой записи, а также другим пользователям, если пользователь создает шаблоны, которые будут доступны для всех пользователей. Имейте это в виду, присваивая названия шаблонам. Например, следует избегать использования названий, включающих такие слова, как "комиссия", если не все пользователи знают, что некоторые сотрудники компании получают заработную плату на комиссионной основе. Когда один или несколько шаблонов, к которым пользователь имеет доступ, создаются для определенной формы, а пользователь пытается создать в форме новую запись, отображается страница **Поиск шаблона для…**. При выборе шаблона из списка, на его базе создается новая запись, содержащая информацию по умолчанию. Если вы не хотите видеть список шаблонов при создании новых записей, установите флажок **Повторно не спрашивать** на странице **Выбрать шаблон для**. Чтобы открыть диалоговое окно выбора шаблонов, щелкните правой кнопкой мыши любую запись, выберите **Сведения о записи**, а затем щелкните **Показать выбор шаблона**.
