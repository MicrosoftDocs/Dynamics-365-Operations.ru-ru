# <a name="ship-orders-as-direct-deliveries"></a>Отгрузка заказов как прямых поставок

[!include [task guide banner](../../includes/task-guide-banner.md)]

В этом разделе показано, как создать прямую поставку для заказа на продажу. Прямая поставка используется, если необходимо отгрузить товары клиенту непосредственно от поставщика вместо предварительной их отгрузки на свой склад. Чтобы выполнить эту процедуру, используйте компанию с демонстрационными данными USMF или собственные данные. Для успешного завершения второй подзадачи "Создание прямых поставок на рабочем месте" убедитесь, что выбираемая номенклатура по заказу на продажу содержит поставщика по умолчанию, указанного на экспресс-вкладке "Покупка" шаблона выпущенного продукта.

## <a name="set-an-individual-order-for-direct-delivery"></a>Настройка отдельного заказа для прямой поставки
1. Перейдите к **Область перехода > Модули > Расчеты с клиентами > Заказы > Все заказы на продажу**.
2. Выберите **Создать**.
3. Введите или выберите значение в поле **Счет клиента**, затем выберите **OK**
4. Введите или выберите значения в поле **Код номенклатуры** и **Сайт**, затем выберите **Сохранить**.
5. На панели операций выберите **Заказ на продажу**, затем выберите **Прямая поставка**. На странице "Создание поставки" перечислены все открытые строки заказа на продажу, скопированные из заказа на продажу. Можно просмотреть сведения о заказе и при необходимости можно изменять сведения, такие как количество по покупке и условия ценообразования, прежде чем вы создаете прямую поставку.  
6. Выберите значение **Да** в поле **Включить все**.
    - Если требуется создать прямую поставку только для подмножества строк заказа на продажу, установите это по отдельности.  
    - В поле **Счет поставщика** может быть указан или не указан номер поставщика. Если поставщик по умолчанию настроен для продукта (в соответствующем покрытии номенклатуры), то этот поставщик будет скопирован в строку. В противном случае следует указать поставщика вручную. В этом примере выберем нового поставщика на следующем этапе, даже если поле заполнено.   
7. Введите или выберите значение в поле **Счет поставщика**, затем выберите **OK**. Сообщение указывает, что заказ на покупку создан.   
8. Разверните раздел **Сведения о строке**.
9. Выберите вкладку **Поставка** и убедитесь, что поле **Прямая поставка** установлено на **Да**.
10. На панели операций выберите **Общее**.
11. Выберите **Связанные заказы**.
12. Выберите ссылку в поле **Заказ на покупку**.
13. Разверните раздел **Сведения о строке** и выберите вкладку **Адрес**.
    - Адрес поставки для этой строки заказа на покупку является адресом поставки клиента, а не адресом вашей компании.  
    - При изменении адреса поставки для строки заказа на покупку или исходной строки заказа на продажу адрес для соответствующей строки заказа автоматически обновится.  
14. Выберите вкладку **Поставка**.
    - Как и строка заказа на продажу, связанный тип строки заказа на покупку также задан как "Прямая поставка".  
    - Дата поставки строки заказа на покупку и дата подтвержденной поставки заданы как "Запрошенная дата поступления" и "Подтвержденная дата поступления" исходной строки заказа на продажу соответственно.   
    - При обновлении любой из этих дат в строке покупки или строке продажи даты соответствующего заказа будут автоматически обновлены.     
    - Заказ на покупку, заданный для поставки товаров непосредственно клиенту, связан с исходным заказу на продажу посредством специальной связи. Эта связь подразумевает правило, что обновление отборочной накладной заказа на продажу нельзя сделать из самого заказа на продажу, а следует сделать с помощью заказа на покупку. Однако выставление накладных для клиента необходимо выполнить из заказа на продажу.  
15. На панели операций выберите **Покупка**.
16. Выберите **Подтверждение**.
17. Нажмите **ОК**.
18. На панели операций выберите **Получить**.
19. Выберите **Поступление продуктов**.
20. В поле **Поступление продуктов** введите значение.
21. Нажмите **ОК**.
22. На панели операций выберите **Общее**.
23. Выберите **Связанные заказы** и выделите требуемую запись.
    - После обновления заказа на покупку как "получено" или, другими словами, после того, как поставщик отгрузил товары по адресу клиента, статус исходного заказа на продажу автоматически обновляется на "поставлено".  
    - Теперь можно выставить накладную по заказу на продажу.    
24. Нажмите **ОК**.
25. Закройте страницу.
26. Нажмите **ОК**. Закройте страницы и вернитесь на домашнюю страницу.

## <a name="create-direct-deliveries-from-the-workbench"></a>Создание прямых поставок на рабочем месте
1. Перейдите к **Навигация > Модули > Расчеты с клиентами > Заказы > Все заказы на продажу**.
2. Выберите **Создать**.
3. Введите или выберите значение в поле **Счет клиента**, затем выберите **OK**.
4. Введите или выберите значение в поле **Код номенклатуры** и **сайт**.
5. Разверните раздел **Сведения о строке**, затем выберите вкладку **Поставка**. Вместо создания прямой поставки в рамках обработки заказа на продажу как в предыдущей процедуре можно выбрать передачу этой задачи специалисту по закупкам. Для включения строки заказа на продажу в процесс обработки прямой поставки необходимо отметить строку для прямой поставки.  
6. Выберите **Да** в поле **Прямая поставка**.
    - Если номенклатура уже была настроена для прямой поставки по умолчанию, поле будет автоматически заполнено как "Да" при вводе строки заказа. Можно настроить номенклатуру для прямой поставки в мастере выпущенного продукта, установив параметр "Прямая поставка" как "Да" и выбрав склад по умолчанию "Прямая поставка".  
    - Поскольку заказ на покупку еще не создан, статус «Прямая поставка» задан как «Для прямой поставки».   
7. Нажмите **Сохранить**.
8. Закройте страницы, пока вы не вернетесь на домашнюю страницу.
9. Введите `Direct delivery` в строке поиска.
    - Страница прямой поставки является по сути рабочим местом, где специалист по закупкам просматривает все строки заказа на продажу, которые должны быть поставлены напрямую, и где можно разрешить создание соответствующих заказов на покупку. Кроме того, они могут просмотреть открытые заказы прямой поставки и подтвержденные заказы на вкладках "Подтверждение" и "Поставка".  
    - После создания заказа с прямой поставкой он автоматически перемещается на вкладку "Подтверждение". Можно подтвердить заказ прямо на этой странице. Когда покупка подтверждена, она автоматически переместится на вкладку "Поставка", откуда можно зарегистрировать ее поступление.
