# <a name="register-payments-automatically-for-intercompany-customer-invoices"></a>Автоматическая регистрация платежей для накладных по внутрихолдинговым накладным клиентов

[!include [banner](../../includes/banner.md)]

Microsoft Dynamics 365 Supply Chain Management создает проводку по клиенту при разноске внутрихолдинговой накладной клиента. Эта проводка по клиенту остается открытой до тех пор, пока она не будет сопоставлена, что означает, что по ней уплачено. Когда соответствующий внутрихолдинговый заказ на покупку обновляется с помощью накладной, создается проводка по поставщику, соответствующая проводке по клиенту. Эта проводка по поставщику также остается открытой до тех пор, пока она не будет сопоставлена. Для уменьшения риска разницы, журнал платежей по расчетам с клиентами можно автоматически создавать и разносить, когда журнал платежей по расчетам с поставщиками разнесен.

1. Перейдите в раздел **Продажи и маркетинг \> Клиенты \> Все клиенты**.
1. На панели операций откройте вкладку **Общие** и выберите **Внутрихолдинговый**.
1. Чтобы настроить внутрихолдинговые платежи по расчетам с клиентами на странице **Внутрихолдинговые** для заказов на продажу, выберите ссылку **Политики заказов на продажу**.
1. В поле **Журнал платежей** выберите журнал платежей по расчетам с клиентами, в котором нужно зарегистрировать внутрихолдинговые платежи поставщикам. Это настраивается на странице **Параметры модуля расчетов с клиентами**.
1. Если необходимо выполнять разноску в этот журнал автоматически, установите флажок **Автоматическая разноска журнала**.
