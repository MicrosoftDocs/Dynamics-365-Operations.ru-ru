# <a name="rebate-reduction-principles"></a>Принципы сокращения ретробонусов

[!include [banner](../includes/banner.md)]

Можно сгруппировать сделки управления ретробонусами в *принципы сокращения* для уменьшения другого обеспечения, связанных с номенклатурой, и/или для уменьшения результирующих значений в расчетах ретробонусов. После настройки принципов уменьшения ретробонусов их можно при необходимости назначить строкам сделок управления ретробонусами.

Правила принципа уменьшения ретробонусов применяются только в случае перекрытия сделок с ретробонусами. Таким образом, они могут предоставлять несколько ретробонусов в тех случаях, когда несколько ретробонусов не причитаются.

## <a name="manage-rebate-reduction-principles"></a>Управление принципами сокращения ретробонусов

Чтобы работать с принципами сокращения ретробонусов, перейдите в раздел **Управление ретробонусами \> Настройка \> Принципы уменьшения ретробонусов**. Затем используйте кнопки на панели операций для добавления и удаления принципов сокращения по мере необходимости. Для каждого принципа задайте значения в полях, как описано в следующей таблице.

| Поле | описание |
|---|---|
| Принцип сокращения ретробонусов | Введите уникальное имя для принципа сокращения ретробонусов. |
| описание | Введите описание принципа сокращения ретробонусов. |
| Применить сокращение | Установите этот флажок, чтобы применить базис сокращения для ретробонусов, относящихся к этому принципу сокращения ретробонусов. |
| Базис сокращения | Если установлен флажок **Применить сокращение**, выберите базис сокращения (*Обеспечение*, *Ретробонусы* или *Оба*). Если выбрано значение *Оба*, и если для обеспечение и ретробонусы были разнесены для предыдущей сделки, будут применены только ретробонусы. |
| Исключить из сокращения | Если этот флажок установлен, обеспечения и ретробонусы, принадлежащие этому принципу сокращения ретробонусов, будут исключены из последующих сокращений. Задание поле **Базис сокращения** не применяется к этому параметру. |

## <a name="examples-of-rebate-reduction-principle-setups"></a>Примеры настроек принципа сокращения ретробонусов

В следующей таблице показаны некоторые типичные примеры настроек принципа сокращения ретробонусов. Для каждого из этих принципов сокращения ретробонусов значение поля **Описание** описывает назначение принципа сокращения ретробонусов.

| Принцип сокращения ретробонусов | описание | Применить сокращение | Базис сокращения | Исключить из сокращения |
|---|---|---|---|---|
| Действие отложено | Сократить ретробонус | Да | Оба | Нет |
| Exclreb | Исключить ретробонусы | Да | Компенсационные выплаты | Да |
| Кратность | Несколько ретробонусов | Да | Оба | Да |
| Не допускается | Только обеспечение и ретробонусы | Нет | Оба | Да |
| Подготовка | Только обеспечение | Да | Подготовка | Нет |
| Компенсационные выплаты | Только ретробонусы | Да | Компенсационные выплаты | Да |

## <a name="examples-of-rebate-reduction-principle-calculations"></a>Примеры расчетов принципа сокращения ретробонусов

Имеется заказ на продажу, содержащий одну строку заказа на продажу. Эта строка имеет значение 1000 долларов США. Следующие сделки применяются к заказу:

- **Сделка 1:** 10 процентов на 1000 долларов США
- **Сделка 2:** 15 процентов на 1000 долларов США
- **Сделка 3:** 20 процентов на 1000 долларов США
- **Сделка 4:** 25 процентов на 1000 долларов США

Порядок обработки очень важен. Порядок обработки базируется на способе вашей работы, как описано в разделе [Обработка, просмотр и разноска ретробонусов](process-review-post.md).

Например, в следующей таблице приводится сводка результатов, если используется порядок *1, 2, 3, 4* и выполняется обработка только обеспечения.

| Сделка | Описание и настройки | Результат подготовки |
|---|---|---|
| 1 | <p>Классификация не проверяет другие сделки для сокращений. Проверка выполняется как для обеспечения, так и для ретробонусов.</p><ul><li>**Применить сокращение:** *Нет*</li><li>**Базис сокращения:** *Оба*</li><li>**Исключить из сокращения:** *Нет*</li></ul> | 1000 долларов США × 10% = 100 долларов США |
| 2 | <p>Классификация проверяет другие сделки для сокращений, но помечается так, чтобы она сама пропускается. Проверка выполняется только для ретробонусов.</p><ul><li>**Применить сокращение:** *Да*</li><li>**Базис сокращения:** *Ретробонусы*</li><li>**Исключить из сокращения:** *Да*</li></ul> | 1000 долларов США × 15% = 150 долларов США |
| 3 | <p>Классификация проверяет другие сделки на сокращения. Проверка выполняется как для обеспечения, так и для ретробонусов.</p><ul><li>**Применить сокращение:** *Да*</li><li>**Базис сокращения:** *Оба*</li><li>**Исключить из сокращения:** *Нет*</li></ul> | (1000 долларов США – Сделка 1) × 20% = 180 долларов США |
| 4 | <p>Классификация проверяет другие сделки на сокращения. Проверка выполняется как для обеспечения, так и для ретробонусов.</p><ul><li>**Применить сокращение:** *Да*</li><li>**Базис сокращения:** *Оба*</li><li>**Исключить из сокращения:** *Нет*</li></ul> | (1000 долларов США – Сделка 1 – Сделка 3) × 25% = 180 долларов США |

В следующей таблице приводятся примеры того, где обеспечение обрабатывается в других заказах, и где, следовательно, получаются другие итоговые значения. Расхождение в обеспечениях зависит от того, в каком порядке система обрабатывает эти сделки. Важно понимать, как порядок обработки влияет на окончательную сумму ретробонуса.

| Последовательность | Расчет |
|---|---|
| 1<br>(1, 2, 3, 4) | <ul><li>**Сделка 1:** 1000 долларов США × 10% = 100 долларов США</li><li>**Сделка 2:** 1000 долларов США × 15% = 150 долларов США</li><li>**Сделка 3:** (1000 долларов США – Сделка 1) × 20% = 180 долларов США</li><li>**Сделка 4:** (1000 долларов США – Сделка 1 – Сделка 2) × 25% = 180 долларов США</li><li>**Общее обеспечение:** 610 долларов США</li></ul> |
| 2<br>(4, 3, 2, 1) | <ul><li>**Сделка 4:** 1000 долларов США x 25% = 250 долларов США</li><li>**Сделка 3:** (1000 долларов США – Сделка 4) × 20% = 150 долларов США</li><li>**Сделка 2:** 1000 долларов США x 15% = 150 долларов США</li><li>**Сделка 1:** 1000 долларов США x 10% = 100 долларов США</li><li>**Общее обеспечение:** 650 долларов США</li></ul> |
| 3<br>(3, 2, 1, 4) | <ul><li>**Сделка 3:** 1000 долларов США x 20% = 200 долларов США</li><li>**Сделка 2:** 1000 долларов США x 15% = 150 долларов США</li><li>**Сделка 1:** 1000 долларов США x 10% = 100 долларов США</li><li>**Сделка 4:** (1000 долларов США – Сделка 3 – Сделка 1) × 25% = 175 долларов США</li><li>**Общее обеспечение:** 625 долларов США</li></ul> |
| 4<br>(2, 4, 1, 3) | <ul><li>**Сделка 2:** 1000 долларов США × 15% = 150 долларов США</li><li>**Сделка 4:** 1000 долларов США × 25% = 250 долларов США</li><li>**Сделка 1:** 1000 долларов США × 10% = 100 долларов США</li><li>**Сделка 3:** (1000 долларов США – Сделка 4 – Сделка 1) × 20% = 130 долларов США</li><li>**Общее обеспечение:** 630 долларов США</li></ul> |
