# <a name="product-dimensions"></a>Аналитики продуктов

[!include [banner](../includes/banner.md)]

Предусмотрено пять аналитик продукта: цвет, конфигурация, размер, стиль и версия. Аналитики продукта объединяются в группы аналитик, а группы аналитик назначаются шаблонам продуктов. Комбинации аналитик продукта определяют, как определяются варианты продуктов.

Аналитики продукта — это характеристики, которые позволяют определить вариант продукта. Вы можете использовать комбинации аналитик продукта для определения вариантов продуктов. Необходимо определить хотя бы одну аналитику продукта для шаблона продукта, чтобы создать вариант продукта.

## <a name="product-variants"></a>Варианты продуктов

Варианты продуктов также называются номенклатурой. Номенклатура — это материальный продукт в отличие от услуги. Можно также определить шаблон продукта типа "Услуга". Используя тип "Услуга", можно указать варианты продукта, включающие услуги. Например, можно указать шаблон продукта для консультаций и варианты продуктов для работы, которая выполняется старшими и младшими консультантами.

## <a name="product-dimensions"></a>Аналитики продуктов

Вариант продукта можно создать на основе значений аналитик продукта.

Значения аналитик продукта для аналитики размера, цвета и стиля могут быть созданы в следующих местоположениях:

- Страница **размер** (**Управление сведениями о продукте \> Настройки \> Группы аналитик и вариантов \> размеры**)
- Страница **Цвет** (**Управление сведениями о продукте \> Настройки \> Группы аналитик и вариантов \> Цвета**)
- Страница **Стиль** (**Управление сведениями о продукте \> Настройки \> Группы аналитик и вариантов \> Стили**)

Значения аналитик продуктов для аналитики "Конфигурация" обычно создаются с помощью конфигуратора продукции или конфигуратора на основе аналитик. 

Версии продукта обычно создаются для конкретной версии по мере развития продукта во время его жизненного цикла. Версии продукта подробно описаны ниже в этом разделе.

Аналитики продуктов можно также создавать и поддерживать на странице **Аналитики продуктов**, которую можно открыть из следующих местоположений:

- Перейдите в раздел **Управление сведениями о продукте \> Продукты \> Шаблоны продукта**. В области операций выберите **Аналитики продуктов**.
- Перейдите к **Управление сведениями о продукте \> Продукты \> Все продукты и шаблоны продуктов**. Выберите шаблон продукта. В области операций выберите **Аналитики продуктов**.
- Перейдите к **Управление сведениями о продукте \> Выпущенные продукты**. Выберите шаблон продукта. В области действий на вкладке **Продукт** в группе **шаблон продукта** выберите **Аналитики продуктов**.

Число вариантов, которые можно создать для номенклатуры, ограничено числом возможных комбинаций аналитик продукта.

> [!TIP]
> При использовании продукта, например, в строке заказа, вы выбираете аналитики продукта для указания варианта продукта, с которым требуется работать.

## <a name="example"></a>Пример

Компания продает джинсы из денима. Для номенклатуры *Джинсы* используются аналитики "Цвет" и "Размер". Джинсы продаются в трех различных цветовых решениях и могут быть шести различных размеров. Имеются синие, черные и коричневые цвета. Размеры: XS, S, M, L, XL и XXL. Не все размеры доступны во всех трех цветах. Если бы все комбинации были доступны, будет создано 18 разных типов джинсов. Однако в этом примере создаются только следующие девять комбинаций вариантов продукта.

| Цвет | Размер |
|-------|------|
| Голубой  | XS   |
| Голубой  | В    |
| Голубой  | М    |
| Черный | М    |
| Черный | L    |
| Черный | XL   |
| Коричневый | L    |
| Коричневый | XL   |
| Коричневый | XXL  |

## <a name="the-version-product-dimension"></a>Аналитика версии продукта

Версия — это аналитика продукта, которая предназначена для поддержки и отслеживания нескольких версий продукта в рамках цепи поставок. Отслеживание версий очень важно для успеха производителей, работающих в мире непрерывного сокращения жизненного цикла продукта, повышенных требований к качеству и надежности и повышенного внимания к безопасности продукта.

В качестве стандартной аналитики продукта версия будет действовать аналогично существующим аналитикам продукта (размеру, стилю, цвету и конфигурации). Таким образом, ее можно использовать для других целей помимо отслеживания версий продукта.

### <a name="turn-on-the-version-dimension"></a><a name="enable-version-dim"></a>Включение аналитики версии

#### <a name="before-you-turn-on-the-version-dimension"></a>Перед включением аналитики версии

При включении аналитики версии некоторые функции могут прерываться или перестать работать ожидаемым образом, если установлены другие решения, добавляющие настройки в складские аналитики. Для обеспечения полной функциональности аналитики версии может потребоваться обновить эти решения, чтобы они включали аналитику версии в их ссылки на складские аналитики.

При тестировании решений на совместимость с аналитикой версий необходимо искать следующие элементы:

1. **Функциональность:** что более важно, все настройки, затрагивающие складские аналитики, должны оцениваться, чтобы гарантировать, что они могут работать в сочетании с аналитикой версии.
1. **Ссылки на складские аналитики:** выводятся ссылки на складские аналитики (то есть, места, куда явно ссылаются аналитики). Ссылки на `InventDimId`, которые должны работать из коробки, но ищут ссылки на стиль или цвет. Например, убедитесь в наличии следующих элементов:

    - Вызовы API в расширенных классах
    - Все ссылки на отдельные складские аналитики в коде расширения (этот код должен содержать аналитику версий вместе с аналитиками стиля, цвета и размера.)

1. **Ссылки на устаревшие вызовы API:** при вводе аналитики версии Microsoft пыталась сделать как можно меньше интерфейсов API устаревшими. При включении конфигурационного ключа **Аналитика продукта — версия** несколько интерфейсов API, которые стали устаревшими, будут выдавать предупреждение. Прежде чем включать аналитику версий в производственной системе, необходимо исправить вызовы к этим API в расширенных решениях. Устаревшие интерфейсы API для данной версии:

    - RetailTransactionServiceInventory::getProductRecordId
    - EcoResProductNumberIdentifiedProductVariantEntity::find
    - EGAISAlcoholProduction_RU::findByItemDim
    - PCVariantConfiguration::findByProductMasterAndDimensions

1. **Maps:** Если какие-либо сопоставления используют складские аналитики, необходимо обновить соответствующее сопоставление отношений с этими сопоставлениями, чтобы они включали аналитику версии. В расширенной модели или расширениях таблиц обратите внимание на таблицы, в которых поля содержат складские аналитики.
1. **Функциональность Microsoft Dynamics 365 Commerce:** после ее включения аналитика версий будет отображаться в коде, относящемся к Commerce в Dynamics 365 Supply Chain Management. Однако аналитика версии еще не поддерживается базой данных канала Commerce или в приложениях POS-терминала или электронной коммерции. Эти специальные приложения Commerce не поддерживают пользователей, осуществляющих продажу/отгрузку или возврат/получение запасов по аналитикам версий. Функции подстановки наличия запасов не различают запасы по аналитикам версий в приложениях Commerce. Это поведение напоминает текущее поведение аналитики конфигурации в Commerce.

#### <a name="turn-on-the-version-dimension"></a>Включение аналитики версии

Прежде чем использовать аналитику версии, она должна быть включена в системе. Для выполнения этой задачи требуются разрешения администратора.

1. Перейдите в раздел **Администрирование системы \> Рабочие области \> Управление функциями**.
1. Включите функцию с именем *версия аналитики продукта*. (Дополнительные сведения см. в разделе [Управление функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).)
1. Переведите систему в [режим обслуживания](../../fin-ops-core/dev-itpro/sysadmin/maintenance-mode.md).
1. Перейдите в раздел **Администрирование системы \> Настройка \> Конфигурация лицензии**.
1. На вкладке **конфигурационные ключи** разверните **торговля** и установите флажок для **Аналитика продукта — версия**.
1. Отключение [режима обслуживания](../../fin-ops-core/dev-itpro/sysadmin/maintenance-mode.md).

### <a name="areas-where-the-version-dimension-isnt-supported"></a>Области, в которых аналитика версии не поддерживается

Следующие области не поддерживают аналитику версий (эти области все равно можно использовать, но в них нельзя будет добавить версии продуктов (продукты, для которых используется аналитика версий)). Например, нельзя добавить номенклатуру с версиями в каталог поставщика. Это объясняется тем, что добавление продуктов с аналитикой версий в эти области приведет к критическим изменениям.

- Ежемесячный отчет по объекту затрат
- Кэш выписки объекта затрат
- Статистика продаж многоканальной розничной торговли по номенклатуре
- Каталоги поставщиков
- EcoResProductDimensionGroupEntity

Кроме того, функции создания заказов и обработки заказов в Commerce (например, для POS, центра обработки вызовов и заказов электронной коммерции) не поддерживают аналитику версии. Нет никаких подтвержденных сроков относительно того, когда заказы Commerce будут улучшены для ее поддержки.

### <a name="functional-characteristics-of-the-version-dimension"></a>Функциональные характеристики аналитики версии

Аналитика версии функционирует так же, как и другие аналитики продукта. Однако из-за особенностей, и поскольку она предназначена для ведения и отслеживания нескольких версий продукта, она ведет себя несколько иначе. Ниже приведены некоторые различия и сходства:

- **Нет группы версий.**

    Несмотря на то что концепция групп существует для размера, цвета и стиля (группа цветов, группа размеров и группа стилей), концепция группы версий не существует. Группы позволяют заранее определить применимые значения, чтобы, например, при назначении цветовой группы продукту, продукт мог использовать все цвета в этой группе цвета. Эта концепция неприменима к аналитике версии, так как версии продукта не являются предопределенными при создании продукта. Вместо этого они создаются в течение жизненного цикла продукта, по мере необходимости. Обычно если форма, размер и функция продукта остаются прежними, создается новая версия вместо нового продукта.

- **Предложения вариантов продукта работают так же, как и в настоящее время.**

    Предложения вариантов продукта будут создавать предложения для всех значений аналитики версии, как и для других аналитик. Процесс создания и выпуска версий продуктов такой же, как и для продуктов, использующих другие аналитики. При создании версии продукта создается первая версия (V1), которая будет использоваться как аналитика продукта, и будут выпущены варианты. После изменения продукта и необходимости в новой версии будет добавлено новое значение версии (V2), и будут выпущены необходимые варианты. Нет никаких ожиданий, что все версии (V1, V2, и V3) будут созданы заранее для продукта заранее.

> [!IMPORTANT]
> При включении и использовании аналитики версии некоторые решения, которые ссылаются на складские аналитики, могут перестать работать надлежащим образом. Чтобы подтвердить эти ошибки и устранить их, обратитесь к независимым поставщикам программного обеспечения для соответствующих решений. Дополнительные сведения см. в разделе [Включение аналитики версии](#enable-version-dim).
