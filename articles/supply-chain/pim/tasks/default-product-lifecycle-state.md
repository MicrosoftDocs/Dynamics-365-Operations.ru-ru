# <a name="create-a-default-product-lifecycle-state"></a>Создание состояния жизненного цикла продукта по умолчанию

[!include [task guide banner](../../includes/task-guide-banner.md)]

В этой процедуре показано, как создать состояние жизненного цикла продукта по умолчанию, а также связать состояние по умолчанию с выпущенными продуктами.


## <a name="create-a-default-lifecycle-state"></a>Создание состояния жизненного цикла по умолчанию
1. Перейдите в раздел "Управление сведениями о продукте" > "Настройка" > "Состояние жизненного цикла продукта".
2. Щелкните "Создать".
3. В поле "Состояние" введите значение.
4. Выберите "Да" в поле "По умолчанию при выпуске юридическому лицу".
5. В поле "Описание" введите значение.
6. Выберите "Нет" в поле "Активен для планирования".

> [!NOTE]
> Если новый выпущенный продукт не следует включать в сводное планирование, выберите "Нет". Если он должен быть включен в сводное планирование, оставьте для элемента управления значение по умолчанию "Да".  

## <a name="create-a-new-released-product"></a>Создание нового выпущенного продукта
1. Закройте страницу.
2. Щелкните "Управление сведениями о продукте" > "Продукты" > "Выпущенные продукты".
3. Щелкните "Создать".
4. В поле "Номер продукта" введите значение.
5. В поле "Имя продукта" введите значение.
6. В поле "Краткое наименование" введите значение.
7. В поле "Группа номенклатурных моделей" введите или выберите значение.
8. В поле "Номенклатурная группа" введите или выберите значение.
9. В поле "Группа аналитик хранения" введите или выберите значение.
10. В поле "Группа аналитик отслеживания" введите или выберите значение.
11. Нажмите кнопку "OК".

> [!NOTE]
> Состояние жизненного цикла продукта по умолчанию является глобальным определением.  

## <a name="change-the-product-to-an-active-state"></a>Изменение состояния продукта на "Активно"
1. В поле "Состояние жизненного цикла продукта" введите или выберите значение.

> [!NOTE]
> Предположим, что вы задали активное состояние, теперь вы можете выбрать активное состояние, чтобы разрешить использовать продукт в сводном планировании и расчете уровня спецификации. Очевидно, что имеет смысл указать все сведения о продукте, необходимые для согласованного планирования.
