# <a name="product-lifecycle-state"></a>Состояние жизненного цикла продукта 

[!include[banner](../includes/banner.md)]


Состояние жизненного цикла продукта документирует состояние жизненного цикла выпущенного продукта или варианта продукта. Состояния жизненного цикла продукта определяются пользователем, обычно менеджером по продуктам или менеджером справочника по продуктам. На определенные бизнес-процессы, такие как сводное планирование, может влиять конкретное состояние жизненного цикла.   
 
Выпущенный продукт или вариант продукта может быть связан с состоянием жизненного цикла продукта, которое документирует, в каком состоянии жизненного цикла находится в настоящий момент конкретный продукт или вариант продукта. Можно определить любое число состояний жизненного цикла продукта, назначая имя и описание состояния. Можно выбрать одно состояние жизненного цикла как состояние по умолчанию для новых продуктов, запущенных в производство. Варианты выпущенного продукта наследуют состояние жизненного цикла от своего шаблона выпущенного продукта при создании. При изменении состояния жизненного цикла в выпущенном шаблоне продукта можно выбрать обновление всех существующих вариантов, которые имеют такое же исходное состояние.  

## <a name="create-a-new-product-lifecycle-state"></a>Создание нового состояния жизненного цикла продукта 
 
- Чтобы создать новое состояние жизненного цикла продукта, воспроизведите или прочитайте проводник по задаче **Создание нового состояния жизненного цикла продукта**. 

-  Чтобы создать состояние жизненного цикла продукта по умолчанию, воспроизведите или прочитайте проводник по задаче **Создание состояния жизненного цикла продукта по умолчанию**.   

## <a name="associate-product-lifecycle-states-to-released-products"></a>Связывание состояний жизненного цикла продукта с выпущенными продуктами  

Существует несколько способов связать состояние жизненного цикла продукта с выпущенными продуктами или вариантами продукта.

-  При создании нового выпущенного продукта ему автоматически назначается **Состояние жизненного цикла продукта** по умолчанию. 
-  При выпуске продукта для юридического лица автоматически назначается **Состояние жизненного цикла продукта** по умолчанию. 
-  Для выпуске варианта продукта для юридического лица новому варианту автоматически назначается **Состояние жизненного цикла продукта**, связанное с выпущенным шаблоном продукта в данном юридическом лице. 

Состояние жизненного цикла продукта можно обновить вручную с помощью: 

-    Страница списка **Выпущенные продукты** или **Представление сведений**. 
-  Страница списка **Используемые варианты продукта** или **Представление сведений**. 
-  Найдите устаревшие продукты или варианты продуктов на основе спроса и свяжите состояние жизненного цикла.  

Чтобы получить подробные сведения о связывании состояний жизненного цикла продукта, запустите воспроизведение или почитайте два проводника по задачам.

-  Чтобы связать состояние жизненного цикла продукта с шаблоном выпущенного продукта, запустите воспроизведение или прочитайте проводник по задаче **Назначение состояния жизненного цикла продукта шаблону выпущенного продукта**. 

-  Чтобы связать состояние жизненного цикла продукта с выпущенным продуктом, запустите воспроизведение или прочитайте проводник по задаче **Назначение состояния жизненного цикла продукта выпущенному продукту**. 

## <a name="impact-on-master-planning"></a>Влияние на сводное планирование 

Состояние жизненного цикла продукта имеет только один флаг управления: **Активен для планирования**. По умолчанию он имеет значение **Да** для всех созданных состояний жизненного цикла продукта, однако его можно изменить на **Нет**. Если задано значение **Нет**, связанные выпущенные продукты или варианты выпущенных продуктов: 

-  Исключаются из сводного планирования. 
-  Исключаются из расчета на уровне спецификаций. 

Подробные сведения об использовании состояние жизненного цикла продукта для исключения продуктов из сводного планирования и расчета на уровне спецификации можно получить, запустив воспроизведение или прочитав руководство по задаче **Состояние состояния жизненного цикла продукта для исключения продуктов из сводного планирования**.

> [!NOTE]
> Из соображений производительности настоятельно рекомендуется связать все устаревшие выпущенные продукты или варианты продукта, особенно при работе с вариантами конфигурации продуктов, не допускающих повторного использования, с состоянием жизненного цикла продукта, которое отключено для сводного планирования.  
 
## <a name="default-migration-import-and-export"></a>Перенос, импорт и экспорт по умолчанию 

Состояния жизненного цикла продукта не поддерживаются информационными объектами, и состояние жизненного цикла не может быть заданно переменному состоянию с помощью информационных объектов выпущенных продуктов.

-  При миграции из предыдущих выпусков состояние жизненного цикла всех продуктов и вариантов продукта будет пустым.  
-  При импорте выпущенных продуктов через информационный объект при создании будет применено состояние жизненного цикла по умолчанию.  
-  При импорте вариантов выпущенного продукта через информационный объект будет импортировано состояние жизненного цикла продукта шаблона выпущенного продукта.   
 
## <a name="find-obsolete-products-and-products-variants"></a>Поиск устаревших продуктов и вариантов продуктов 
 
Можно выполнить анализ моделирования, чтобы найти устаревшие используемые продукты или варианты продукта, а затем обновить их состояние жизненного цикла продукта. Чтобы найти устаревшие продукты, запустите воспроизведение или прочитайте руководство по задаче **Поиск вариантов устаревших продуктов и назначение состояния жизненного цикла продукта**. В данном руководстве по задаче показано, как находить устаревшие используемые продукты или варианты продуктов, а также как связать состояние жизненного цикла продукта с устаревшими продуктами. В нем также показано, как просмотреть результаты моделирования и оценить, сколько продуктов и вариантов продукта будут связаны с новым состоянием жизненного цикла продукта при выполнении обновления без моделирования.  
 
Путем выполнения анализа в режиме моделирования продукты и варианты продуктов, которые определены как устаревшие, отображаются в конкретной форме, где их можно легко просматривать. Анализ производит поиск транзакций и конкретного справочника для идентификации продуктов, не пользовавшихся спросом в течение переменного периода и не имеющих справочника, который может привести к спросу. Новые продукты, выпущенные в течение переменного периода, могут быть исключены из анализа. При моделирование анализа возвращает ожидаемый результат, пользователь может запустить анализ и установить новое состояние жизненного цикла продукта для всех продуктов, которые определены как устаревшие в ходе анализа.  
 
> [!NOTE]
> Обратите внимание, что весь анализ и обновления должны выполняться в одном и том же юридическом лице.  
 
## <a name="criteria-to-select-and-update-released-products-or-product-variants"></a>Критерии для выбора и обновления выпущенных продуктов или вариантов продуктов 
 
Используйте следующие критерии для выбора и обновления выпущенных продуктов и вариантов продуктов: 

-    Состояние жизненного цикла продукта или варианта продукта должно отличаться от нового желаемого состояния. 
-  Продукт или вариант продукта был создан некоторое число дней назад на основе количества дней, которое введено в диалоговом окне выбора. 
-  Открытые производственные заказы отсутствуют (= состояние < завершено) для продукта или варианта продукта. 
-  Отсутствуют открытые складские проводки (= состояние отпуска ReservPhysical для QuotationIssue или состояние получения Зарегистрировано для QuotationReceipt) для продукта или варианта продукта. 
-  Не существует складских проводок в течение последнего числа дней для продукта или варианта продукта. 
-  Отсутствует будущий спрос или прогноз поставки для продукта или варианта продукта.  
-  Минимальный уровень запасов не установлен в покрытии номенклатуры для продукта или варианта продукта. 
-  Нет активных правил фиксированного канбана для продукта или варианта продукта.  
-  Нет строки заказа на сервисное обслуживание для продукта или варианта продукта. 
-  Не активных или будущих строк продажи или строк договор покупки для продукта или варианта продукта. 
-  Продукт или вариант продукта не используется в спецификации, которая связана с действующей утвержденной версией спецификации для продукта или варианта продукта, который активен для планирования.
