# <a name="product-identifiers"></a>Идентификаторы продуктов

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

[!include [banner](../includes/banner.md)]

В этом разделе представлены сведения о различных типах идентификаторов продуктов и объясняется, как можно добавить идентификаторы продуктов в данных продукции.

При работе с продуктами в цехе или на складе в Microsoft Dynamics ERP или Microsoft Dynamics CRM необходимо иметь хорошую стратегию для идентификации этих продуктов и вариантов продуктов.

## <a name="unique-product-numberproduct-id"></a>Уникальный номер продукта/идентификатор продукта

В Dynamics 365 Supply Chain Management основной идентификатор продукта — это номер продукта (т. е., уникальный идентификатор продукта). Этот номер может быть автоматически создан с помощью номерной серии или вручную связан с продуктом. Для вариантов продуктов номера можно определить через шаблон номенклатуры продуктов.

Во многих случаях номер продукта изначально не создается в Dynamics 365 Supply Chain Management. Вместо этого он связывается с продукта в систему управления жизненным циклом продуктов (PLM) или системе управления данными продуктов (PDM). В этом случае используются информационные объекты для импорта продуктов и вариантов продукта. Затем Supply Chain Management использует эти номера во всех операциях.

При реализации Supply Chain Management следует уделять особое внимание стратегии для номеров продуктов. Хорошая система нумерации улучшает потоки снабжения и позволяет избежать ошибок. Хороший идентификатор продукта содержит не более 15 символов. В идеальном случае он содержит менее 10 символов и включает не более пяти классификационных символов. Можно также использовать краткие наименования для обеспечения быстрого поиска. Краткое наименование — это дополнительное имя, представляющее классификации продукта.

При использовании Microsoft Dataverse номер продукта в Supply Chain Management также является номером продукта в Microsoft Dataverse. Варианты продуктов синхронизируются в службе Dataverse как уникально идентифицируемые продукты.

## <a name="item-number-and-product-dimensions"></a>Код номенклатуры и аналитики продукта

Код номенклатуры является идентификатором продукта, используемым для конкретного юридического лица. В идеальном случае код номенклатуры должен быть идентичен номеру продукта. Если номенклатуры отличаются у разных юридических лиц, становится трудно отслеживать продукт по цепочке поставок, и вводятся обременительные процессы перемаркировки и создания ссылок. Для совместимости с ранними версиями (то есть, с Microsoft Dynamics AX 2009 и ранее) мы сохранили эту модель. Тем не менее по возможности рекомендуется исключить идентификаторы, которые относятся к конкретным юридическим лицам, и вместо этого использовать уникальный номер продукта в качестве основного идентификатора.

Кроме того, вариант продукта нельзя однозначно идентифицировать по коду номенклатуры. Для него всегда требуется комбинация кода номенклатуры и всех аналитик продукта, которые определены в шаблоне продукта. Это требование может стать достаточно громоздким и может замедлить процессы идентификации. Также и по этой причине по возможности рекомендуется использовать уникальный номер продукта вместо кода номенклатуры.

Многие страницы все еще содержат код номенклатуры и аналитики продукта как первичные идентификаторы. Однако номера продуктов могут использоваться для поиска. В пункте **Продажи и маркетинг** &gt; **Настройка** &gt; **Поиск** &gt; **Параметры поиска** можно изменить подстановку поиска, чтобы в ней в качестве основной стратегии поиска использовались номера продуктов, а не коды номенклатур. Если для параметра **Включить поиск для поиска продуктов** задано значение **Да**, в подстановке будут отображаться не только шаблоны продуктов, но и варианты продуктов. Подробнее см. в разделе [Поиск продуктов и вариантов продуктов во время ввода заказа](search-products-product-variants.md)

Кроме того, можно будет выполнять поиск и фильтрацию по номеру продукта, названию продукта и описанию и по идентификаторам аналитик продукта для варианта продукта. При выборе варианта соответствующий код номенклатуры и все идентификаторы аналитик продукта будут выбраны. Таким образом будет легче искать и выбирать правильный вариант. Этот параметр настоятельно рекомендуется при использовании вариантов продуктов и уникального номера продукта в качестве первичных идентификаторов для продуктов. Единственным исключением может быть индустрия моды, где бизнес-процессы часто требуют выбора основного продукта перед выбором варианта. Этот параметр следует внимательно оценить, прежде чем внедрять систему нумерации.

> [!NOTE]
> Код номенклатуры для продукта нельзя изменить, если для этого продукта существует одна или несколько проводок.

## <a name="product-name-and-description"></a>Наименование и описание продукта

Наименование продукта и описание являются удобными для восприятия человеком идентификаторами продукта и могут поддерживаться на нескольких языках. По умолчанию клиент Supply Chain Management отображает все сведения о продукте на языке компании по умолчанию, а не на языке пользователя. Тем не менее переведенные имена и описания продуктов используются во всех сообщениях клиентам и поставщикам. Переводы основаны на коде языка для счетов клиентов и поставщиков.

Для вариантов продуктов наименование продукта может быть создано с помощью шаблона номенклатуры продуктов. Так как это не обязательно, чтобы имена продуктов быть уникальными, может оказаться нескольких продуктов с одинаковым именем.

## <a name="product-and-item-search-names"></a>Краткие наименования продуктов и номенклатуры

Supply Chain Management предлагает дополнительное краткое наименование для продуктов, а также для номенклатур (запущенных в производство продуктов). Это краткое наименование не обязательно должны быть уникальными, и оно может быть изменено после создания продукта или варианта продукта. Рекомендуется использовать краткое наименование для поиска продуктов по категориям. Краткие наименования обеспечивают быстрый поиск, особенно в процессах продажи и покупки.

Краткое наименование также может содержать идентификатор продукта у клиента или поставщика или некоторый другой внешний идентификатор продукта, если этот внешний код является первичным критерием поиска для продукта.

## <a name="external-product-identifiers-customer-and-vendor-identifiers"></a>Внешние идентификаторы продуктов (идентификаторы у клиента и у поставщика)

Для выпущенных продуктов можно поддерживать коды номенклатур, наименования номенклатур и описания номенклатур, используемые клиентом или поставщиком. Эти ссылки отображаются на внешних документах, таких как заказы на продажу, заказы на покупку, отборочные накладные и накладные. В текущей версии Supply Chain Management внешние ссылки не отображаются на страницах основных операций. Единственное исключение — код номенклатуры поставщика. Этот номер отображается в диалоговом окне **Сведения о продукте**, если поставщик по умолчанию определен для выпущенного продукта.

Можно поддерживать внешние идентификаторы продуктов по выпущенным продуктам, вариантам выпущенных продуктов, клиентам или группам клиентов, поставщикам или группам поставщиков.

На странице **Выпущенные продукты** выполните одно из следующих действий.

- Для клиентов на вкладке **Продажа** в группе **Связанные сведения** выберите **Внешнее описание номенклатуры**.
- Для поставщиков на вкладке **Покупка** в группе **Связанные сведения** выберите **Внешнее описание номенклатуры**.

На странице **Внешние описания номенклатуры** можно связать код продукта у клиента или поставщика с выпущенным продуктом. Эта связь должна быть выполнена для каждого юридического лица. Может быть записана следующая информация. К сожалению, метки могут немного вводить в заблуждение в текущей версии Supply Chain Management. Тем не менее, эти метки может быть изменен в будущей версии.

| Поле | Соответствующая информация по клиенту | Соответствующая информация по поставщику |
|-------|------------------------------------|----------------------------------|
| Внешний код номенклатуры | Код номенклатуры, назначенный клиентом | Код номенклатуры, назначенный поставщиком |
| описание | Имя, которое клиент связывается с номенклатурой | Имя, которое поставщик связывается с номенклатурой |
| Внешний текст номенклатуры | Описание номенклатуры,составленное клиентом | Описание номенклатуры, составленное поставщиком |

Если несколько клиентов или поставщиков используют одинаковые номера номенклатуры (как в случае, например, ассоциации по закупками или коммерческой группы), можно создать группы клиентов или поставщиков для упрощения обслуживания внешней информации о продуктах.

- Для групп клиентов перейдите к пункту **Продажи** &gt; **Настройка** &gt; **Номенклатуры** &gt; **Внешнее описание номенклатуры** для создания и ведения групп и связанных кодов номенклатур. Для связи клиентов с группой откройте **Расчеты с клиентами** &gt; **Клиенты** &gt; **Все клиенты**, а затем на экспресс-вкладке **Значения заказов на продажу по умолчанию** укажите значение в поле **Номенклатура - Группы клиентов**.
- Для групп поставщиков перейдите к пункту **Закупки и источники** &gt; **Настройка** &gt; **Группа внешних описаний номенклатур** для создания и ведения групп и связанных кодов номенклатур. Для связи поставщиков с группой откройте **Расчеты с поставщиками** &gt; **Поставщики** &gt; **Все поставщики**, а затем на экспресс-вкладке **Значения заказа на покупку по умолчанию** укажите значение в поле **Группа номенклатура-поставщик**.
 
## <a name="bar-codes"></a>Штрихкоды

Если требуется использовать сканер штрих-кодов для идентификации продуктов, идентификатор продукта должны удовлетворять требованиям используемого стандарта штрих-кода. Таким образом, штрих-коды обычно содержат не номера продуктов в необработанном виде, а число, созданное специально для выбранной технологии штрих-кода. Можно поддерживать несколько штрих-кодов по типу штрих-кода. Можно даже связать один и тот же штрих-код с несколькими продуктами, а затем выбрать фактическую активную связь при сканировании штрих-кода.

Перед определением штрих-кодов можно определить одну или несколько настроек штрих-кодов. Настройки штрих-кода могут помочь проверить, что штрих-коды удовлетворяют необходимым рекомендациям и, следовательно, возможна их эффективная печать и сканирование. Также можно поддерживать специальные штрих-коды для конкретных количеств продукта.

Рекомендуется использовать настройку штрих-кода, чтобы поддерживать коды Global Trade Item Number (GTIN) или International Article Number (EAN).

Для поддержки штрих-кодов на странице **Выпущенные продукты** на вкладке **Управление складскими запасами** в группе **Склад** выберите **Штрих-коды**.

## <a name="gtin-codes"></a>Коды GTIN

В электронной коммерции очень важно, чтобы все стороны говорили на общем языке и ссылались на продукты с помощью общего набора идентификаторов. Поэтому в некоторых отраслях используется система [GTIN](https://www.gs1.org/id-keys/gtin), которая является глобальной системой нумерации номенклатур, созданной GS1.

Рекомендуется поддерживать GTIN в качестве штрих-кода. Тем не менее поддержка также возможна на странице **Номенклатура — GTIN**. Чтобы открыть эту страницу, на странице **Выпущенные продукты** на вкладке **Управление складскими запасами** в группе **Склад** выберите **Коды GTIN**. GTIN не поддерживается как глобальный номер. Вместо этого он поддерживается юридическим лицом.

В Supply Chain Management вы определяете варианты упаковки в складских операциях, определяя конкретные единицы измерения. Например, номенклатура может храниться в штуках, упаковками по шесть, лотками по 18 или полными палетами. Для каждого из этих вариантов упаковки определяется определенная единица измерения. Поскольку GTIN обычно относится к единице упаковки продукта, страница **Номенклатура - GTIN** позволяет поддерживать несколько кодов GTIN для каждого продукта и единицы измерения. Тем не менее нельзя более одного раза использовать один и тот же код GTIN для различных номенклатур или вариантов продукта юридического лица.

Чтобы поддерживать **Коды GTIN**, на странице **Выпущенные продукты** на вкладке **Управление складскими запасами** в группе **Склад** выберите **GTIN**.

## <a name="external-codes"></a>Внешние коды

Для многих объектов можно определить внешние коды. Например, можно определить внешние коды для идентификации продуктов и выпущенных продуктов. Эти внешние коды могут использоваться для связи статистических кодов или налоговых кодов с выпущенными продуктами или выпущенными вариантами продуктов. Внешние коды определяются по юридическому лицу и типу кода. Они должны быть уникальными по юридическому лицу, типу кода и ссылку на таблицу.

К сожалению, нет стандартных функций, позволяющих искать продукты по внешним кодам.

### <a name="product-and-item-number-sequences"></a>Номерные серии продуктов и номенклатур

Можно определить две различные номерные серии:

- Номерная серию **Номер продукта** для глобального номера продукта.
- Номерная серия **Код номенклатуры** для номера номенклатуры для юридического лица

> [!NOTE]
> Код номенклатуры следует использовать как отдельный идентификатор только при миграции различных юридических лиц из различных источников, в которых имеются различные системы нумерации. Следует всегда пытаться использовать идентификатор продукта, который является уникальным для всех юридических лиц. Таким образом, следует установить для параметра **Вручную** значение **Да** для номерной серии **Код номенклатуры**. Таким образом код номенклатуры будет следовать номеру продукта при создании. Если Supply Chain Management не является ведущей системой для новых номеров продуктов, следует установить для параметра **Вручную** значение **Да** как для номерной серии **Код номенклатуры**, так и для номерной серии **Номер продукта**.

При использовании объекта **Создание выпущенного продукта V2** для создания продуктов несколько настроек могут влиять на то, как номерные серии используются для создания кода номенклатуры и номера продукта:

- Параметры номерной серии **Номер продукта**
- Параметры номерной серии **Код номенклатуры**
- Сопоставление кода номенклатуры 
- Сопоставление номера продукта

## <a name="product-entity-identifier-export-all-product-identifiers"></a>Идентификатор объекта продукта (Экспорт всех идентификаторов продуктов)

Модель идентификатора объекта продукта была создана, чтобы в версии 1.0 Dataverse можно было подготовить все идентификаторы, которые используются для ссылки на продукт. Чтобы упростить эту задачу, все идентификаторы собираются в одной глобальной таблице идентификаторов, чтобы их можно было экспортировать как одну модель. Обратите внимание, что эта версия Dataverse не использует модель идентификаторов продуктов. Таким образом, объект **Объект идентификатора объекта продукта в службе Common Data Service** и этот процесс имеют ограниченное практическое использование и скорее всего будут изменяться в будущем.

Таблица идентификаторов продуктов является глобальной таблицей, которая заполняется из всех таблиц ссылок основного юридического лица через повторяющееся пакетное задание. Необходимо выбрать юридическое лицо и иерархию категорий продуктов как определение глобальной области действия шаблонов продуктов. Создание глобальной таблицы идентификаторов продуктов ограничивается продуктами, выпущенными для выбранного юридического лица, и продуктами, которые являются членами иерархии продуктов, выбранной для роли **Common Data Service** в иерархии категорий продуктов.

Этот процесс предполагает, что данные шаблона продукта в первую очередь ведутся в одном центральном юридическом лице. (Тем не менее, это юридическое лицо может быть виртуальным юридическим лицом, которое используется только для поддержки глобальных основных данных.)

Выполните следующие действия, чтобы настроить среду.

1. Выбор иерархию категорий для службы Dataverse. На странице **Ассоциации роли иерархии категорий**, если никакая иерархия не связана с ролью **Common Data Service**, необходимо создать новую связь. Выберите роль **Common Data Service**, затем свяжите иерархию категорий, которая представляет ассортимент продуктов, которые должны быть синхронизированы со службой Dataverse.
2. Выберите юридическое лицо для глобальных данных шаблонов продуктов. На странице **Параметры модуля "Управление сведениями о продукте"** на вкладке **Атрибуты продуктов** выберите основную компанию, где идентификаторы продуктов и номенклатур в основном поддерживаются.
3. Определите типы кодов идентификаторов и коды, которые должны быть экспортированы. Откройте **Управление сведениями о продукте** &gt; **Настройка** &gt; **Коды идентификаторов продуктов**. Чтобы создать типы кодов идентификаторов, выберите **Создать коды**. Запись типа кода создается для каждого типа идентификатора, который находится в выбранном юридическом лице.

    Для штрих-кодов тип кода создается для каждой настройки штрих-кода. Для внешних кодов тип кода создается для каждого класса внешних кодов.

    Теперь можно поддерживать список типов кодов. Можно изменить код, наименование и описание. Можно также удалять типы кодов. Типы кода, которые вы удалили, не будут использоваться для заполнения глобальных таблиц идентификаторов объектов продуктов.

4. Завершив определение типов кодов идентификаторов продуктов, можно создать идентификаторы в глобальной таблице, запустив задание **Создание идентификаторов объектов продуктов** на странице **Коды идентификаторов объектов продуктов**. Это задание необходимо запускать в пакете. Это задание следует настроить как периодическое пакетное задание таким образом, чтобы таблица заполнялась в соответствии с новыми записями.

Теперь можно использовать информационные объекты **Объект идентификатора объекта продукта в службе Common Data Service**, **Код идентификаторов объектов продуктов** и **Область идентификаторов объектов продуктов** для экспорта идентификаторов для любой целевой системы.
