# <a name="flushing-principle-settings-on-bom-lines-arent-respected"></a>Настройка принципа пополнения по спецификации для строк спецификации не учитывается

Номер статьи базы знаний: 4612725

## <a name="symptoms"></a>Симптомы

Эта проблема возникает в том случае, когда поле **Автопотребление по спецификации** на вкладке **Автоматическое обновление** на странице **Параметры управления производством** имеет значение *Принцип пополнения по спецификации*. Этот параметр указывает, что все строки спецификации должны автоматически потребляться при получении заказа на покупку. Если значение поля **Принцип пополнения по спецификации** в строках спецификации явно задан *Вручную*, можно предположить, что строки спецификации не потребляются автоматически. Однако в этом случае строки спецификации, для которых в поле **Принцип пополнения по спецификации** используется значение *Вручную*, все равно потребляются автоматически.

## <a name="resolution"></a>Приказ

Если вы столкнулись с этой проблемой, параметры управления производством могут быть настроены для переопределения настройки **Принцип пополнения по спецификации** для строк спецификации. На странице **Параметры управления производством** на вкладке **Автоматическое обновление** проверьте значение поля **Автопотребление по спецификации**. Если значение параметра равно *Всегда*, система будет игнорировать настройку **Принцип пополнения по спецификации** для всех строк спецификации и всегда будет пополнять строки по спецификации. Чтобы система соблюдала настройку **Принцип пополнения по спецификации** для строк спецификации, измените значение в поле **Автопотребление по спецификации** на *Принцип пополнения по спецификации*.
