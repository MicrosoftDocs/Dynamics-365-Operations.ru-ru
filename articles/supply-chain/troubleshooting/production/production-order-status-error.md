# <a name="error-when-changing-status-of-production-order-from-reported-as-finished-to-end"></a>Ошибка при изменении статуса производственный заказа с "Принято" на "Завершено"

## <a name="symptoms"></a>Симптомы

Когда статус производственного заказа изменяется с "Принято" на "Окончание", вы получите одно из следующее сообщений об ошибке:

> Конфликт при обновлении. В результате обновления стандартные затраты не соответствуют финансовой величине запасов.

> Критическая ошибка в функции InventCostMovement.checkVariance.

## <a name="cause"></a>Причина

Эта проблема возникает из-за того, что базовые данные были изменены другим процессом. Процесс будет пытаться обновлять данные до пяти раз. Если конфликт по-прежнему существует после пяти попыток, вы получите одно из сообщений, перечисленных выше.

## <a name="resolution"></a>Решение

Такое поведение предусмотрено разработчиками. Чтобы устранить эту проблему, возобновите пакетное задание. Оно должно завершить выполнение.

Если после возобновления пакетного задания происходят постоянные сбои, убедитесь, что точность округления валюты по умолчанию в главной книге соответствует округлению, применяемому к значениям в таблице InventSum. Если точность округления была изменена на несоответствующее значение, вероятно, необходимо изменить его обратно на соответствующее значение. Найдите **ModifiedDateTime**. В этом случае значение обычно показывает, что точность округления была недавно изменена.
