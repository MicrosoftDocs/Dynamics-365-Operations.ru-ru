# <a name="quantity-not-valid-when-theres-picking-work-with-multiple-lps-in-one-location"></a>Недопустимое количество при работе комплектации с несколькими НЗ в одном местоположении

## <a name="symptoms"></a>Симптомы

Когда идет работа по комплектации с несколькими грузоместами в одном местоположении, количество недействительно для единицы измерения *шт*, и появляется следующее сообщение об ошибке:

> Недопустимое количество для единицы измерения 1%.

## <a name="resolution"></a>Решение

Убедитесь, что поля **Код группы упорядочения единиц** и **Пересчеты единиц измерения** для выпущенного продукта или варианта продукта установлены правильно.

Обратите внимание, что сообщение об ошибке было улучшено в версии 10.0.15, чтобы показать ожидаемое количество (см. статью базы знаний [KB 4581627](https://fix.lcs.dynamics.com/Issue/Details/?bugId=486531)). Новое сообщение об ошибке:

> Недопустимое количество. Ожидалось %1 %2.
