# <a name="master-planning-generates-planned-orders-for-phantom-products"></a>Сводное планирование создает спланированные заказы для виртуальных продуктов

Номер статьи базы знаний: 4614729

## <a name="symptoms"></a>Симптомы

Спланированные заказы для виртуальных продуктов создаются после выполнения сводного планирования.

## <a name="resolution"></a>Приказ

Настройка параметра **Виртуальный** для выпущенных продуктов определяет тип строки по умолчанию для спецификации. Если для параметра **Виртуальный** установлено значение *Да*, система все равно создаст спланированные заказы для номенклатуры, но для каждого спланированного заказа параметр **Непосредственно производное требование** будет установлено значение *Да*. Поэтому спланированный производственный заказ не может быть утвержден самостоятельно. Вместо этого он всегда будет автоматически включено при утверждении родительского производственного заказа. Кроме того, строки спецификации спланированного виртуального заказа будут включены в спецификацию родительского производственного заказа.
