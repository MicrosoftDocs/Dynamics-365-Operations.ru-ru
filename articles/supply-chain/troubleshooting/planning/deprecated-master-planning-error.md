# <a name="you-receive-an-error-when-running-the-built-in-master-planning-engine"></a>Вы получаете ошибку при запуске встроенной подсистемы сводного планирования

Код ошибки: ReqCalcScheduleItemTablePlanningOptimizationFitError

## <a name="symptoms"></a>Симптомы

При запуске сводного планирования с помощью устаревшей подсистемы сводного планирования вы получаете следующее сообщение об ошибке:

> Это сообщение об ошибке выводится из-за того, что встроенный механизм сводного планирования использовался для сценариев, поддерживаемых оптимизацией планирования. Сейчас следует выполнить миграцию на оптимизацию планирования, поскольку текущее встроенное сводное планирование будет устаревшим. Обратите внимание, что выполнение этого сводного планирования успешно завершено. В случае, когда миграция имеет сильную зависимость от ожидающих функций, может быть запрошено исключение для продолжения использования встроенного механизма сводного планирования. Заполните следующую анкету, чтобы приступить к работе, и, если требуется, запросить исключение из миграции на оптимизацию планирования. [Миграция оптимизации планирования и анкета об исключениях](https://go.microsoft.com/fwlink/?linkid=2144962)

## <a name="cause"></a>Причина

Если вы используете планирование и не используете функции управления производством, следует рассмотреть возможность перехода на оптимизацию планирования. Встроенная подсистема сводного планирования в настоящее время устаревает. Поэтому, если вы хотите продолжать использовать ее без получения сообщения об ошибке, вы должны подать заявку на исключение от корпорации Майкрософт.

## <a name="resolution"></a>Решение

Для получения дополнительной информации о том, как перейти к оптимизации планирования или как подать заявку на исключение, чтобы продолжать использовать устаревающую подсистему сводного планирования, см. [Миграция на оптимизацию планирования для сводного планирования](/dynamics365/supply-chain/master-planning/new-master-planning-engine).
