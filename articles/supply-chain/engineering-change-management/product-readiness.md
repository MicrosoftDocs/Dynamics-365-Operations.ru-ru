# <a name="product-readiness"></a>Готовность продукта

[!include [banner](../includes/banner.md)]

Вы можете использовать проверки готовности, чтобы гарантировать, что все требуемые основные данные указаны для продукта, прежде чем они будут использованы в проводках. При использовании проверок готовности пользователь или рабочая группа назначаются ответственными за проверку определенных заранее определенных данных, связанных с продуктом. Если имеется открытая проверка готовности продукта, продукт не может быть выпущен или использован в проводках.

Флажок **Активно** для технологического продукта, варианта или версии доступен только после ввода и проверки всех необходимых данных, и после того, как все проверки готовности обработаны. На этом этапе продукт, версия или вариант могут быть выпущены в другие компании и использоваться в проводках. Можно создавать проверки готовности для новых продуктов, новых вариантов и новых технологических версий.

## <a name="types-of-readiness-checks"></a>Типы проверок готовности

Есть три типа проверок готовности:

- **Проверка системы** — система проверяет, имеется ли действительная запись. Например, запись может быть активной спецификацией.
- **Ручная проверка** — пользователь проверяет, является ли запись действительной. Например, для проверки готовности может требоваться проверка настроек по умолчанию для заказа. В некоторых случаях, например, когда продукт по-прежнему разрабатывается и поэтому не будет помещаться на склад, не требуется настройка заказа по умолчанию. Однако настройки заказа по умолчанию могут потребоваться для другого продукта того же типа, поскольку продукт может храниться на складе. Пользователь несет ответственность за знание того, как правильно определить необходимость проверки готовности.
- **Контрольный список** — пользователь отвечает на ряд вопросов из контрольного списка, и система определяет, отвечают ли эти ответы ожиданиям. Контрольный список может иметь любую тематику. Например, его можно использовать, чтобы определить, готовы ли маркетинговые материалы или документация по продукту.

## <a name="how-readiness-checks-are-created-for-a-new-product-variant-or-version"></a>Создание проверок готовности для нового продукта, варианта или версии

При создании нового инженерного **продукта** система определяет, была ли политика проверки готовности настроена для категории инженерных продуктов. (Политики проверки готовности могут быть применены на уровне выпущенного продукта, на уровне выпущенного варианта и на уровне инженерной версии.) Если политика была настроена, происходят следующие события:

- Для продукта создаются проверки готовности согласно применяемой политике.
- Инженерная версия устанавливается в неактивное состояние для блокирования использования продукта. Все версии для данного продукта, которые затронуты, устанавливаются в неактивное состояние.

Если для продукта создан новый **вариант**, система проверяет, настроены ли проверки готовности в категории инженерных продуктов. (Проверки готовности могут быть применены на уровне выпущенного варианта и на уровне инженерной версии.) Если проверка готовности была настроена, происходят следующие события:

- Для продукта создаются проверки готовности.
- Инженерная версия устанавливается в неактивное состояние для блокирования использования продукта.

Если для продукта создана новая инженерная **версия**, система проверяет, настроены ли проверки готовности в категории инженерных продуктов. (Проверки готовности могут быть применены на уровне инженерной версии.) Если проверка готовности была настроена, происходят следующие события:

- Для продукта создаются проверки готовности.
- Инженерная версия устанавливается в неактивное состояние для блокирования использования продукта.

## <a name="view-readiness-checks"></a>Просмотр проверок готовности

### <a name="view-open-readiness-checks-for-a-product-or-product-version"></a>Просмотр открытых проверок готовности для продукта или версии продукта

Чтобы найти открытые проверки готовности для продукта, откройте страницу **Сведения о выпущенных продуктах**. Затем в область действий на вкладке **Продукт** в группе **Проверки готовности** выберите **Проверки готовности**.

Чтобы найти открытые проверки готовности для версии продукта, откройте страницу **Инженерные версии** для выпущенного продукта и выберите версию. Затем в область действий на вкладке **Продукт** в группе **Контрольный список** выберите **Проверки готовности**.

### <a name="view-open-readiness-checks-that-are-assigned-to-you"></a>Просмотр открытых проверок готовности, которые назначены вам

Чтобы просмотреть назначенные вам открытые проверки готовности, выполните одно из следующих действий.

- Перейдите к пункту **Управление техническими изменениями \> Общее \> Готовность продуктов \> Мои открытые проверки готовности**.
- Перейдите к пункту **Управление сведениями о продукте \> Рабочие области \> Готовность продукта для дискретного производства**.

Настройка, в которой указывается, кому назначена проверка готовности, выполняется для категории технологических продуктов. Проверки готовности могут быть назначены сотруднику или рабочей группе. Если для рабочей группы была назначена проверка готовности, в рабочей группе имеется один сотрудник, который должен выполнить проверку готовности. Дополнительные сведения см. в разделе [Инженерные версии и категории технологических продуктов](engineering-versions-product-category.md).

## <a name="process-open-readiness-checks"></a>Обработка открытых проверок готовности

### <a name="process-system-and-manual-readiness-checks"></a>Обработка системных и ручных проверок готовности

После открытия страницы **Проверки готовности** можно просмотреть тему о системных и ручных проверках готовности, выбрав **Просмотр связанной информации** на панели операций. Затем можно подготовить и/или проверить данные для проверки готовности. Открытые проверки готовности имеют в поле **Статус** значение *Ожидание*. Этот статус указывает, что проверка готовности должна быть все еще обработана. Чтобы обработать проверку готовности, выполните одно из следующих действий.

- На панели операций выберите **Проверка/завершение** для просмотра и выполнения проверки готовности. По завершении поле **Статус** обновляется до значения *Пройдено*.
- В области действий выберите **Пропустить**, если хотите пропустить проверку готовности, которая не является обязательной. Например, можно настроить проверку готовности для расчета цены. Однако вы решили пропустить эту проверку, пока продукт все еще находится в стадии проектирования. В этом случае поле **Статус** обновляется на значение *Пропущено*.

В зависимости от конфигурации политики готовности, когда значение поля **Статус** для проверки готовности обновляется до значения *Пройдено*, для утверждения проверки готовности может потребоваться дополнительный шаг. В этом случае выберите **Утверждение** для завершения проверки готовности. Этот шаг утверждения всегда является обязательным, если проверка готовности пропущена.

Когда все открытые проверки готовности для нового продукта, варианта или версии были обработаны и утверждены в соответствии с требованиями, номенклатура автоматически становится активной и, таким образом, готова к использованию.

### <a name="process-checklist-readiness-checks"></a>Обработка контрольных списков проверок готовности

Чтобы открыть контрольный список, откройте страницу **Проверки готовности**, затем на панели операций выберите **Начать контрольный список**. После завершения контрольного списка система проверяет, пройдена ли проверка готовности на основе настроек в анкете. Если контрольный список пройден, поле **Статус** обновляется до значения *Пройдено*. Если проверка готовности не является обязательной, ее можно пропустить. В этом случае поле **Статус** обновляется на значение *Пропущено*.

В зависимости от конфигурации политики готовности, когда значение поля **Статус** для проверки готовности обновляется до значения *Пройдено*, для утверждения проверки готовности может потребоваться дополнительный шаг. В этом случае выберите **Утверждение** для завершения проверки готовности. Этот шаг утверждения всегда является обязательным, если проверка готовности пропущена.

Когда все открытые проверки готовности для нового продукта, варианта или версии были обработаны и утверждены в соответствии с требованиями, номенклатура автоматически становится активной и, таким образом, готова к использованию.

## <a name="create-and-manage-product-readiness-policies"></a>Создание политик готовности продукта и управление ими

Политики готовности продукта используются для управления проверками готовности, которые применяются к продукту. Поскольку политика готовности назначается инженерной категории, все проверки политики готовности применяются ко всем инженерным продуктам, основанным на инженерной категории. Дополнительные сведения см. в разделе [Инженерные версии и категории технологических продуктов](engineering-versions-product-category.md).

Каждая политика готовности содержит набор проверок готовности. Когда политика готовности назначается категории технологических продуктов, всем продуктам, созданным из этой категории технологических продуктов, будут иметь проверки готовности, указанные в политике готовности.

Чтобы работать с политиками готовности продукта, перейдите в раздел **Управление техническими изменениями \> Настройка \> Политики готовности продуктов**. Затем выполните одно из следующих действий.

- Чтобы создать новую политику, в области действий выберите **Создать**, затем настройте поля, как описано в следующих подразделах.
- Чтобы изменить существующую политику, выберите ее на панели списка, выберите **Правка** на панели операций, затем задайте поля, как описано в следующих подразделах.
- Чтобы удалить существующую политику, выберите ее в панели списка, выберите **Изменить** на панели операций, а затем на экспресс-вкладке **Общие** убедитесь, что в параметре **Активный** установлено значение *Нет*. Затем в области действий выберите **Удалить**.

### <a name="header"></a>Верхний колонтитул

Задайте следующие поля в заголовке политики готовности продуктов.

| Поле | описание |
|---|---|
| Наименование | Введите имя для политики. |
| описание | Введите описание политики. |

### <a name="general-fasttab"></a>Экспресс-вкладка "Общее"

Задайте следующие поля на экспресс-вкладке **Общее** политики готовности продуктов.

| Поле | описание |
|---|---|
| Тип продукта | Выберите, применяется ли политика к продуктам типа *Номенклатура* или *Услуга*. Вы не можете изменить этот параметр после сохранения записи. |
| Активные сотрудники | Этот параметр используется для ведения политик готовности. Установите значение *Да* для всех используемых политик готовности. Установите значение *Нет*, чтобы пометить политику готовности неактивной, если она не используется. Обратите внимание, что невозможно деактивировать политику готовности, назначенную для категории технологического продукта, и можно удалить только неактивные политики выпуска. |

### <a name="readiness-control-fasttab"></a>Экспресс-вкладка управления готовностью

Для каждого типа проверки готовности, которую необходимо включить в политику, добавьте строку на экспресс-вкладке **Управление готовностью**. Следующие кнопки на панели инструментов экспресс-вкладки используются для добавления и удаления строк, как требуется:

- **Добавить проверку** — добавление стандартной проверки готовности к политике. При нажатии этой кнопки отображается диалоговое окно **Добавление проверки**. Там можно выбрать из списка доступных проверок.
- **Добавление существующей анкеты** — добавление пустой строки в сетку. После этого можно назначить существующую анкету, задав поля в следующей таблице.
- **Копировать** — добавление копии выбранной строки в сетку.
- **Удалить** — удаление выбранной строки из сетки.

Для каждой добавляемой строки установите следующие поля.

| Поле | описание |
| --- | --- |
| Область процесса | Выберите область, с которой связана проверка. |
| Вид | Выберите, является ли проверка системной проверкой, ручной проверкой или контрольным списком (анкетой). |
| Наименование | Если проверка является контрольным списком, введите его имя. Для системных и ручных проверок это поле устанавливается автоматически. |
| описание | Если проверка является контрольным списком, введите описание. Для системных и ручных проверок это поле устанавливается автоматически, и описание объясняет фокус проверки. |
| Применить проверки к | Укажите, должна ли строка создавать проверки готовности в ответ на новый выпущенный продукт, выпущенный вариант или выпущенную версию. |
| Выполнение в | Выберите, должны ли проверки готовности, создаваемые строкой, применятся для всех компаний или для одной компании. |
| Организация | Если в поле **Выполнить в** указано значение *Одна компания*, выберите компанию. |
| Тип владельца | Выберите, должны ли проверки готовности, которые создает данная строка, быть назначены для лица или рабочей группы. |
| Ответственный | Выберите, лицо или рабочую группу, которым должны быть назначены проверки готовности, которые создает данная строка. |
| Анкета | Выберите анкету, которая будет использоваться для контрольного списка. Контрольный список является локальным контрольным списком в компании, в которой выполняется проверка готовности. Система должна иметь возможность оценить, правильно ли даны ответы на контрольный список. Поэтому контрольный список должен быть настроен таким образом, чтобы оценка выполнялась на основе правильных ответов. Дополнительные сведения о создании анкет см. в разделе [Использование анкет](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/using-questionnaires) и его соответствующих темах. |
| Автоматическое утверждение | Записи проверки готовности включают флажок **Утверждено**, который указывает статус утверждения. Установите флажок **Автоматическое утверждение** для проверок, которые должны быть установлены в утвержденное состояние сразу после того, как назначенный пользователь выполнит их. Снимите этот флажок, чтобы в качестве дополнительного шага требовалось явное утверждения. |
| Обязательно | Установите этот флажок для проверок, которые должны быть выполнены назначенным пользователем. Обязательные проверки не могут быть пропущены. |
