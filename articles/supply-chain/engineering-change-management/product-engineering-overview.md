# <a name="engineering-change-management-overview"></a>Обзор управления техническими изменениями

[!include [banner](../includes/banner.md)]

## <a name="feature-summary"></a>Сводка по функции

Современным производителям требуются надежное управление данными о продукции, управление версиями и управления техническими изменениями, чтобы добиться успеха в мире непрерывного сокращения жизненных циклов продуктов, повышения качества и требований к надежности и увеличивающегося внимания к безопасности продукции.

Управление инженерными изменениями приносит структуру и дисциплину в процесс управления данными о продуктах и позволяет определять, выпускать и изменять продукты в контролируемой манере, поддерживаемой рабочими процессами. С помощью версий продуктов и управления техническими изменениями можно документировать, оценить воздействие и применить технические изменения на протяжении всего жизненного цикла продукта.

Управление техническими изменениями помогает в планировании и управлении версиями продуктов, а также в управлении жизненными циклами продуктов и техническими изменениями. Ниже приводится список основных функций:

- Управление версиями продукта
- Улучшенная функциональность выпуска продукта, которая позволяет хранить основные данные продукта в одном юридическом лице (инженерной компании) и публиковать полностью настроенный выпущенный продукт для других юридических лиц
- Правила проверки того, что перед активацией версии продукта (проверки готовности) были введены требуемые сведения
- Улучшенное управление жизненным циклом продуктов с помощью детального контроля над тем, когда выпущенный продукт может использоваться в конкретных бизнес-процессах
- Запросы на инженерное изменение, поддерживаемые рабочими процессами
- Заказы на инженерное изменение, поддерживаемые рабочими процессами

## <a name="turn-on-the-engineering-change-management-features-for-your-system"></a>Включение функций управления изменениями в проектировании в системе

Перед началом использования управления изменениями разработки следует включить функцию *Управление изменениями разработки* и ее конфигурационный ключ. Если необходимо также отследить аналитику продуктов в проводках (необязательно), необходимо включить функцию *Аналитики версии продукта* и ее конфигурационный ключ. После настройки необходимых условий вы сможете включить дополнительные функции для управления изменениями в проектировании.

### <a name="turn-on-the-basic-engineering-change-management-features"></a>Включение базовых функций управления изменениями в проектировании

Сначала включите эти функции, выполнив следующие действия.

1. Откройте рабочую область [Управление функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).
1. Проверьте наличие обновлений.
1. Включите функцию с именем *Управление техническими изменениями*.
1. Если вы хотите использовать ее, включите также функцию с именем *Версия аналитики продукта*.

### <a name="turn-on-the-required-configuration-keys"></a>Включите необходимые ключи конфигурации

Затем включите ключи конфигурации, выполнив следующие действия.

1. Переведите систему в режим обслуживания, как описано в разделе [Режим обслуживания](../../fin-ops-core/dev-itpro/sysadmin/maintenance-mode.md).
1. Перейдите в раздел **Администрирование системы \> Настройка \> Конфигурация лицензии**.
1. Разверните узел **Торговля**.
1. Активируйте конфигурационный ключ для основной функции, установив флажок **Управление изменениями разработки**.
1. Разверните узел **Управление технологическими изменениями** и установите или снимите следующие флажки (в зависимости от функций, которые хотите использовать):

    - **Поиск атрибутов** — установите этот флажок, чтобы включить [функцию поиска атрибутов](engineering-attributes-and-search.md). Мы рекомендуем включить эту функцию, но можно снять этот флажок, если она не будет использоваться.
    - **Управление изменениями для непрерывного производства** — этот флажок устанавливается, если необходимо использовать функции управления технологическими изменениями в формулах для непрерывного производства. Если нет необходимости управлять формулами, этот флажок можно снять. Дополнительные сведения см. в разделе [Управление изменениями в формулах и их ингредиентах](manage-formula-changes.md).

1. Если также необходимо использовать аналитику версий, установите флажок **Аналитика продукта — версия**. (Этот флажок находится ниже по списку, он не вложен в узел **Управление технологическими изменениями**.)
1. Выключите режим обслуживания, как описано в разделе [Режим обслуживания](../../fin-ops-core/dev-itpro/sysadmin/maintenance-mode.md).

> [!IMPORTANT]
> Начиная с апреля 2022, ключи лицензии для **Управление изменениями разработки** и **Аналитика продукта — версия** будут включены по умолчанию для всех новых установок, но их по-прежнему можно отключить, если это необходимо.

### <a name="turn-on-additional-engineering-change-management-features"></a>Включение дополнительных функций управления изменениями в проектировании

После включения базовых функций управления изменениями и включения конфигурационных ключей в управлении функциями добавляются несколько дополнительных и опциональных функций управления изменениями в проектировании. Каждая из этих функций указана в модуль **Управление изменениями в проектировании**. В следующей таблице описаны все дополнительные возможности и приведены ссылки для получения дополнительной информации.

| Имя функции в управлении функциями | описание |
|---|---|
| Включение управления изменениями для существующих продуктов | <p>Эта функция позволяет преобразовывать существующие продукты в продукты разработки, чтобы можно было начать управлять ими с помощью управления изменениями в проектировании.</p><p>Дополнительные сведения см. в разделе [Включить управление изменениями для существующих продуктов](change-management-existing-products.md).</p> |
| Уведомления разработки для производства | <p>При изменении продукта при разработке может быть важно уведомить производство об этих изменениях. Таким образом, производственные работники могут предпринять соответствующие действия, такие как замена компонентов, замена спецификации или замена маршрута. Эта функция позволяет оповещать производство об изменениях производимых продуктов.</p><p>Дополнительные сведения см. в разделе [Управления изменениями для технологических продуктов](engineering-change-management.md).</p> |
| Улучшенное наследование атрибутов для управления изменениями разработки | <p>Эта функция упрощает управление атрибутами для готовой продукции или промежуточной номенклатуры. Если эта функция включена, легче определить все атрибуты, относящиеся к номенклатуре, и можно выбрать атрибуты, которые должны быть распространены из этого номенклатуры в родительскую номенклатуру. Эта функция полезна, если, например, один компонент готовой продукции является хрупким, токсичным или огнеопасным, так как можно легко определить хрупкий, токсичный или огнеопасный атрибут и распространить его на готовый товар.</p><p>Дополнительные сведения см. в разделе [Инженерные атрибуты и поиск инженерных атрибутов](engineering-attributes-and-search.md).</p> |
| Проверки готовности продукта | <p>Эта функция позволяет вам настроить проверки для стандартных (неинженерных) продуктов. Используйте проверки готовности продукта, чтобы убедиться, что каждый продукт полностью определен и все необходимые политики настроены до того, как продукт будет сделан доступным и использоваться в проводках. Если отключить эту функцию после того, как она использовалась в течение определенного момента, все существующие проверки готовности для стандартных продуктов будут удалены.</p><p>Дополнительные сведения см. в разделе [Готовность продуктов](product-readiness.md).</p> |
| Управление изменениями в формулах и их ингредиентах | <p>Эта функция позволяет отслеживать изменения ингредиентов формулы, сопутствующих и побочных продуктов.</p><p>Дополнительные сведения см. в разделе [Управление изменениями в формулах и их ингредиентах](manage-formula-changes.md).</p> |
| Создание вариантов для продуктов разработки | <p>Эта функция позволяет создавать варианты для продуктов разработки на основе доступных значений аналитик.</p><p>Дополнительные сведения см. в разделе [Создание вариантов для продуктов разработки](engineering-variants.md).</p> |
