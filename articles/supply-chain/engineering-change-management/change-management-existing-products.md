# <a name="enable-change-management-on-existing-products"></a>Включить управление изменениями для существующих продуктов

[!include [banner](../../includes/banner.md)]

В этой теме объясняется, как можно включить управление изменениями для существующих продуктов. Здесь также описываются ситуации, в которых возможность включения управления изменениями ограничивается.

При включении управления изменениями для существующего продукта можно создавать версии этого продукта и отслеживать изменения, внесенные в него в течение своего жизненного цикла. Таким образом, эти изменения можно отслеживать с помощью заказов на изменение. Чтобы включить управление изменениями, необходимо преобразовать соответствующие продукты в *инженерные элементы* (также называемые инженерными продуктами). Инженерные продукты — это продукты с управлением версиями и общим управлением с помощью системы управления изменениями. Предусмотрен мастер, помогающий выполнить процесс преобразования.

## <a name="turn-on-the-feature-in-your-system"></a>Включите функцию в системе

Для использования этой возможности необходимо выполнить следующие задачи:

1. Включите функцию "Управление изменениями разработки" и ее конфигурационный ключ, как описано в [Обзор управления изменениями разработки](product-engineering-overview.md).
1. Включите функцию *Включить управление изменениями для существующих продуктов* в управлении функциями. Дополнительные сведения см. в разделе [Обзор управления функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

## <a name="restrictions-and-limitations"></a>Ограничения

Не все типы продуктов могут быть преобразованы во все остальные типы. Действуют следующие ограничения:

- При преобразовании продукта в инженерный продукт он остается *продуктом*. Он не становится *шаблоном продукта*.
- При преобразовании шаблона продукта с определенным набором аналитик эти аналитики сохраняются после изменения. Например, при преобразовании шаблона продукта с аналитикой размера, он сохранит аналитику размера.

Таким образом, если есть уникально идентифицируемый продукт, его можно изменить только на инженерский продукт, который не отслеживает аналитику продукта в проводках (то есть аналитика версии не используется). Дополнительные сведения по этим вопросам см. в оставшихся разделах в этой теме.

## <a name="prepare-for-conversion-by-creating-all-required-engineering-product-categories"></a>Подготовка к преобразованию путем создания всех необходимых категорий инженерных продуктов

*Категория инженерных продуктов* должна быть назначена каждому инженерному продукту. Это можно сделать при выполнении мастера **Преобразовать в инженерный продукт**. *Прежде чем* можно будет преобразовать эти продукты, необходимо, чтобы категории инженерных продуктов были созданы для всех соответствующих стандартных продуктов.

Категория инженерных продуктов предоставляет основу для создания инженерного продукта и устанавливает набор значений и политик по умолчанию. Категория инженерного продукта должна соответствовать продукту, для которого выполняется назначение. Например, тип продукта и группа аналитик должны соответствовать как к продукту, так и категории инженерных продуктов. Дополнительные сведения см. в разделе [Инженерные версии и категории технологических продуктов](engineering-versions-product-category.md).

> [!IMPORTANT]
> Мастер **Преобразовать в инженерный продукт** может преобразовать продукт только в инженерные продукты, в которых данная версия не отслеживается в проводках. Таким образом, параметр **Отслеживать версии в проводках** должен быть установлен как *Нет* для категорий инженерных продуктов, которые создаются для преобразования существующих продуктов.

Сведения о способах создания инженерных продуктов см. в разделе [Инженерные версии и категории инженерных продуктов](engineering-versions-product-category.md).

## <a name="run-the-convert-to-engineering-product-wizard"></a>Запуск мастера "Преобразовать в инженерный продукт"

Мастер **Преобразовать в инженерный продукт** помогает преобразовать один или несколько существующих продуктов в инженерные продукты. Он конвертирует продукты в инженерные продукты (продукты с версиями), в которых версия не отслеживается в проводках (то есть аналитика версии не используется). После завершения преобразования можно управлять продуктами, используя управления изменениями разработки.

Преобразование необратимо. Таким образом, его невозможно вернуть. 

Каждый преобразованный инженерный продукт будет продолжать выпускаться в тех же компаниях, в которых был выпущен исходный продукт. Однако спецификация разработки (BOM) и маршруты не будут автоматически выпущены в эти компании.

Выполните следующие действия, чтобы запустить мастер **Преобразовать в инженерный продукт** и преобразовать продукт в инженерный продукт.

1. Выберите **Управление сведениями о продукте \> Продукты \> Выпущенные продукты**.
1. В сетке установите флажок для каждого продукта, который следует преобразовать.
1. В области действий на вкладке **Инженер** в группе **Управление техническими изменениями** выберите **Преобразовать в инженерный продукт** для открытия мастера.
1. На первой странице мастера находится страница **Приветствие**. Если вы еще не знакомы с процессом преобразования, внимательно прочитайте информацию на этой странице. Когда вы будете готовы к продолжению, выберите **Далее**.
1. На странице **Выбор сведений для преобразуемых продуктов** показан каждый продукт, который был выбран до того, как был открыт мастер. Проверьте список. Для добавления или удаления продуктов по мере необходимости используйте кнопки **Создать** и **Удалить** на панели инструментов.
1. Используйте следующие поля в верхней части сетки для назначения значений по умолчанию каждому из перечисленных продуктов. (Вы сможете изменить эти значения для отдельных продуктов после завершения преобразования.) Значения по умолчанию не будут назначаться продуктам, для которых соответствующее значение уже было назначено.

    - **Категория разработки по умолчанию** — выберите исходную категорию инженерных продуктов для назначения каждому продукту в списке. Выбранная категория будет применена только к совместимым с ней продуктам.
    - **Версия по умолчанию** — введите исходную версию продукта, которая будет назначена для всех продуктов в списке. У каждого инженерного продукта есть по крайней мере одна инженерная версия.
    - **Состояние жизненного цикла по умолчанию** — выберите исходное состояние жизненного цикла продукта, которое будет назначено для всех продуктов в списке.
    - **Текущая спецификация будет частью инженерного продукта** — установите этот флажок, если текущая спецификация всех продуктов в списке должна использоваться в качестве спецификации для инженерного продукта.

    Дополнительные сведения о параметрах продуктов см. в следующем пункте.

1. Просмотрите все продукты в сетке и оцените, насколько хорошо назначенные значения по умолчанию к нему применимы. Для каждой строки изучите следующую информацию и задайте соответствующие поля:

    - **Номер продукта** — номер продукта.
    - **Имя продукта** — имя продукта.
    - **Категория разработки** — выберите категорию инженерных продуктов, к которой должен принадлежать продукт после преобразования. Для каждого продукта должна существовать соответствующая категория, как было объяснено в предыдущем разделе данной темы. Необходимо назначить категорию каждому продукту.
    - **Версия** — введите версию продукта, которая будет назначена продукту после преобразования. Например, можно выбрать номер, который указан в номерной серии, которая уже используется вашей категорией. В каждой из инженерных версий хранятся относящиеся к проектированию данные, относящиеся к этой версии. Дополнительные сведения см. в разделе [Инженерные версии и категории технологических продуктов](engineering-versions-product-category.md).
    - **Состояние жизненного цикла продукта** — выберите состояние жизненного цикла продукта, в котором он должен быть после преобразования. Состояние жизненного цикла продукта позволяет управлять тем, какие транзакции разрешены для данной инженерной версии. Дополнительные сведения см. в разделе [Состояния жизненного цикла продукта и проводки](product-lifecycle-state-transactions.md).
    - **Есть спецификация** — установленный флажок указывает, что у продукта есть спецификация. Установка этого флажка может помочь в принятии решения о том, как настроить флажок **Текущая спецификация будет частью инженерного продукта**.
    - **Текущая спецификация будет частью инженерного продукта** — установите этот флажок, если текущая спецификация продукта должна использоваться в качестве спецификации для инженерного продукта. После этого данная спецификация будет управляться системой управления изменениями разработки. Если у продукта нет спецификации или если вы хотите вручную создать спецификацию для преобразованного продукта, уберите этот флажок.
    - **Есть ошибки** — установленный флажок указывает, что в настройке продукта есть одна или несколько ошибок. Например, тип продукта или группа аналитик могут не совпадать в категории. Продукты, содержащие ошибки, будут пропущены и не будут преобразованы.

1. По завершении нажмите кнопку **Проверить** на панели инструментов, чтобы проверить настройку продукта. Для каждой строки флажок **Есть ошибки** будет обновлен, чтобы указать статус продукта. Корректируйте значения, пока в настройке всех продуктов не будет ошибок.
1. После того, как все продукты будут настроены правильно, выберите **Далее** для продолжения.
1. На странице **Подтверждение выбора** отображается количество продуктов, для которых отсутствуют ошибки в настройке, и поэтому они готовы к преобразованию. В нем также показано количество продуктов, которые будут пропущены из-за ошибок. Чтобы выполнить преобразование как пакетное задание, установите для параметра **Выполнить в пакетном режиме** значение *Да*.
1. Выберите **Готово**, чтобы применить настройки и начать преобразование продуктов в инженерные продукты.

> [!NOTE]
> Если система настроена на ручное принятие продуктов до их выпуска, необходимо принять каждый преобразованный продукт, используя страницу **Открыть использование продуктов** в соответствующих компаниях. Дополнительные сведения см. в разделе [Проверка и приемка продукта перед его выпуском в местной компании](engineering-scenarios.md#accept).
