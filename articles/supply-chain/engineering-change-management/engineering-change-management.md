# <a name="manage-changes-to-engineering-products"></a>Управление изменениями в технологических продуктах

[!include [banner](../includes/banner.md)]

Управление техническими изменениями предоставляет структурированные процессы для управления изменениями в технологических продуктах. Можно использовать процесс *запроса технических изменений*, чтобы предложить и запросить изменения, а затем использовать процесс *заказа технических изменений* для фактического выполнения этих изменений. Пользователи могут создавать запросы на технические изменения или заказы на технические изменения, а затем обрабатывать их для анализа и утверждения, оценивать их влияние на существующие проводки и выполнять дальнейшие действия с ними.

## <a name="engineering-change-requests"></a>Запросы на изменение разработки

Процесс запроса инженерного изменения позволяет собирать запросы на изменения от всех подходящих отделов в компании. Не имеет значения, работаете ли вы инженером или в отделе производства, снабжения, на складе или в отделе сбыта: любой может использовать запрос на техническое изменение для запроса изменения. Это изменение может стать идеей нового продукта, проблемой, обнаруженной при работе с существующим продуктом, предложением по улучшению существующего продукта или чем-то другим.

После отправки запроса на изменение процесс *рассмотрения и утверждения* управляется рабочим процессом, который определяет, кто должен утверждать изменение (например, владельца продукта).

Чтобы настроить рабочий процесс для заказов на техническое изменение или запросов на техническое изменение, перейдите в пункт **Управление техническими изменениями \> Рабочие процессы инжиниринга**. Выберите **Создать**, выберите, будет ли рабочий процесс использоваться для рассмотрения заказов на техническое изменение или запросов на техническое изменение, а затем настройте рабочий процесс.

Дополнительные сведения о работе с рабочими процессами см. в разделе [Обзор системы рабочих процессов](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md). Дополнительные сведения о владельцах продуктов см. в разделе [Владельцы продуктов](product-owner.md).

### <a name="create-a-new-engineering-change-request"></a>Создать новый запрос на изменение разработки

Чтобы создать запрос на техническое изменение, выполните одно из следующих действий.

- Перейдите к пункту **Управление техническими изменениями \> Общее \> Управление техническими изменениями \> Запросы на техническое изменение**, затем выберите **Создать** на панели операций.
- Откройте страницу **Сведения о продукте** для существующего инженерного продукта. Затем на панели действий на вкладке **Инженер** в группе **Управление техническими изменениями** выберите **Запрос на техническое изменение \> Создать запрос на техническое изменение**.

Создается новый запрос на изменение. Теперь можно задать поля на каждой экспресс-вкладке, как описано в следующих подразделах.

#### <a name="general-fasttab"></a>Экспресс-вкладка "Общее"

Экспресс-вкладка **Общие** позволяет предоставить базовое описание запроса на изменение. В следующей таблице описаны поля на этой экспресс-вкладке.

| Поле | описание |
|---|---|
| Запрос на изменение | Введите имя для запроса технического изменения. |
| Название должности | Введите текст, кратко описывающий или идентифицирующий изменения в запросе. |
| Приоритет | Выберите значение, чтобы обозначить степень приоритета изменения. При необходимости можно настроить доступные значения для компании. (Дополнительные сведения см. в разделе [Определение общих значений для управления техническими изменениями](engineering-change-management-setup.md).) |
| Категория | Выберите значение, чтобы описать тип запрошенного изменения. При необходимости можно настроить доступные значения для компании. (Дополнительные сведения см. в разделе [Определение общих значений для управления техническими изменениями](engineering-change-management-setup.md).) |
| Строгость | Выберите значение, чтобы указать серьезность проблемы, которая должна быть исправлена при реализации запроса. При необходимости можно настроить доступные значения для компании. (Дополнительные сведения см. в разделе [Определение общих значений для управления техническими изменениями](engineering-change-management-setup.md).) |
| Автор запроса | Имя пользователя, который создал запрос. |
| Вкл | Дата создания запроса. |
| Состояние | Статус запроса. При персом создании запроса статус равен *Создано*. После утверждения запроса статус изменяется на *Утверждено*. Если для запроса был создан связанный заказ на изменение, статус изменится на *Обработано*. |
| Заявка на изменение | Номер заказа на изменение, если запрос на изменение был обработан путем создания заказа на изменение. |

#### <a name="information-fasttab"></a>Экспресс-вкладка информации

Экспресс-вкладка **Информация** позволяет добавлять дополнительные сведения о запросе.

Чтобы добавить строку в сетку, выберите **Создать** на панели инструментов над сеткой, затем выберите один из следующих вариантов:

- **Файл** — отправка файла.
- **Изображение** — отправка файла изображения.
- **Заметка** — введите заметку непосредственно в сетке.
- **URL-адрес** — введите URL-адрес непосредственно в сетке.

В следующей таблице описаны поля в каждой строке.

| Поле | описание |
|---|---|
| Дата и время создания | Дата и время создания строки. |
| Вид | Тип сведений, для которых была создана строка (файл, изображение, заметка или URL-адрес). |
| описание | Введите описание для строки. |
| Ограничение | Значение, указывающее, были ли добавленные данные получены из внутреннего или внешнего источника. |
| Вложен | Установленный флажок указывает, что строка включает вложение (файл или изображение). Чтобы загрузить вложение, выберите строку, затем выберите **Открыть** на панели инструментов над сеткой. |

#### <a name="products-fasttab"></a>Экспресс-вкладка продуктов

Экспресс-вкладка **Продукты** позволяет перечислить каждый продукт, на который влияет запрос на изменение. Вы можете использовать кнопки на панели инструментов, чтобы добавить продукты в сетку или удалить продукты.

Этот список предназначен только для информационных целей. Таким образом можно добавить столько связанных продуктов, сколько необходимо. Если вы создаете запрос на изменение со страницы **Сведения о продукте** для существующего продукта, этот продукт должен быть перечислен на экспресс-вкладке **Продукты** после сохранения записи запроса.

#### <a name="source-fasttab"></a>Экспресс-вкладка источника

Экспресс-вкладка **Источник** позволяет отслеживать начальную точку запроса на изменение. Это полезно, если, например, необходимо просмотреть, был ли запрос на изменение создан из заказа на продажу, кто создал его, а также в какой компании он был создан.

### <a name="evaluate-the-business-impact-of-a-change-request"></a>Оценка влияния запроса на изменение на бизнес

При просмотре запроса на изменение можно выполнить поиск зависимостей. Таким образом можно оценить влияние запрошенного изменения на открытые проводки, такие как заказы на продажу, производственные заказы и запасы в наличии.

1. Перейдите в раздел **Управление техническими изменениями \> Общее \> Управление техническими изменениями \> Запросы технических изменений**.
1. Либо откройте существующий запрос на изменение, либо выберите **Создать** на панели операций, чтобы создать новый запрос на изменение.
1. На панели действий на вкладке **Запрос на изменение** в группе **Влияние на бизнес** выберите одну из следующих кнопок:

    - **Поиск** — сканирует все открытые проводки, а затем открывается диалоговое окно **Бизнес-воздействие на открытые проводки**, в котором перечислены все проводки, которые будут затронуты этим изменением.
    - **Просмотр предыдущего поиска** — открывается диалоговое окно **Бизнес-воздействие на открытые проводки**, в котором перечисляются результаты предыдущего поиска. (Новый поиск не выполняется.)

1. Если проблема, требующая изменения, является критической, можно заблокировать открытые проводки или уведомить ответственного пользователя, используя кнопки на панели инструментов в диалоговом окне **Бизнес-воздействие на открытые проводки**.

### <a name="create-a-change-order-from-a-change-request"></a>Создание заказа на изменение из запроса на изменение

Инженер, просматривающий запрос на техническое изменение, может создать заказ на техническое изменение непосредственно со страницы **Запросы на технические изменения**. В области действий на вкладке **Запрос изменения** в группе **Заказ на техническое изменение** выберите **Копировать ссылку и продукты**.

## <a name="engineering-change-orders"></a>Заявки на изменение разработки

Заказы на технические изменения предоставляет структурированный процесс для внесения изменений в технологические продукты. Вы предлагаете изменения, используя копию данных, связанных с инжинирингом. Реальные основные данные не затрагиваются. Дополнительные сведения о данных, относящихся к проектированию, см. в разделе [Инженерные версии и категории технологических продуктов](engineering-versions-product-category.md).

Можно создать заказ на техническое изменение на основе утвержденного запроса на техническое изменение. Инженеры могут также создавать заказы на техническое изменение "с нуля". Можно включить несколько продуктов в один заказ на техническое изменение, выполнив следующие шаги:

- Выберите продукты вручную.
- Используйте спецификацию для включения продуктов, которые находятся ниже в структуре продукта (то есть, дочерних).
- Поиск по использованию используется для включения продуктов, находящихся выше в структуре продукта (то есть, родительских).

После завершения предложения по изменениям процесс рассмотрения и утверждения будет обрабатываться рабочим процессом. Можно настроить разные рабочие процессы на основе приоритета и важности.

Чтобы настроить рабочий процесс для заказов на техническое изменение или запросов на техническое изменение, перейдите в пункт **Управление техническими изменениями \> Рабочие процессы инжиниринга**. Выберите **Создать**, выберите, будет ли рабочий процесс использоваться для рассмотрения заказов на техническое изменение или запросов на техническое изменение, а затем настройте рабочий процесс.

Дополнительные сведения о работе с рабочими процессами см. в разделе [Обзор системы рабочих процессов](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md).

Ниже приведены некоторые типичные заинтересованные лица, возможно, должны будут утвердить заказ на техническое изменение:

- **Владельцы продуктов** — дополнительные сведения о владельцах продуктов см. в разделе [Владельцы продуктов](product-owner.md).
- **Ответственный ведущий рабочей группы** — поле **Инженер** в представлении **Заголовок** заказа на технические изменения показывает инженера, который создал заказ на техническое изменение. Если инженер принадлежит к рабочей группе, которая определена в системе, в поле **Ответственный** отображается лидер этой рабочей группы.
- **Финансовый отдел** — финансовый отдел должен проанализировать случаи, в которых изменение включает высокие затраты.

Можно выбрать, следует ли обрабатывать заказ на техническое изменение непосредственно после его утверждения, как часть рабочего процесса, или следует ли выполнять обработку позднее в качестве ручного шага. Во время обработки заказа на техническое изменение обновляются инженерные данные по фактическому продукту.

При проверке запроса на изменение в области действий на вкладке **Запрос на изменение** в группе **Влияние на бизнес** выберите **Поиск**, чтобы оценить влияние предлагаемого изменения на открытые проводки, такие как заказы на продажу, производственные заказы и запасы в наличии. Результаты отображаются в диалоговом окне **Бизнес-влияние на открытые проводки**, где можно выбрать затронутые проводки, а затем с помощью команд на панели инструментов просмотреть дополнительные сведения, уведомить ответственного пользователя или заблокировать проводку.

### <a name="engineering-change-orders-in-engineering-or-operational-companies"></a>Заказы на техническое изменение в инжиниринговых или рабочих компаниях

Как описано в разделе [Инжиниринговые компании и правила владения данными](engineering-org-data-ownership-rules.md), данные о продуктах, которые можно редактировать, могут различаться в зависимости от типа юридического лица, в котором ведется работа (инжиниринговая компания или операционная компания). Правила владения данными применяются также к заказам на технические изменения. Таким образом, в зависимости от юридического лица, в котором создается заказ на техническое изменение, могут быть сделаны различные типы изменений. Далее приводятся некоторые примеры.

- Для создания заказов на изменение в **инжиниринговой компании** можно внести основные изменения в инженерные данные. Например, можно создавать новые версии продукта, изменять структуру продукта с помощью спецификации и изменять значения инженерных атрибутов. Для каждого затронутого продукта выберите одно из следующих значений в поле **Влияние**:

    - **Нет** — обновление существующей версии продукта (обновление внутри версии).
    - **Новая версия** — создается новая версия, основанная на выбранной версии продукта.
    - **Новый продукт** — создается полностью новый продукт или вариант продукта, который основан на выбранной версии продукта.

- Для заказов на техническое изменение в **операционной компании** можно изменить логистические данные продукта. Например, можно расширить существующую спецификацию параметрами для источников, добавить локальные маршруты или местные спецификации, а также расширить спецификацию, добавляя новые строки спецификации для местных упаковочных материалов, смазочных жидкостей или инструкций на местном языке. Обогащения, вносимые пользователями в операционной компании, будут сохраняться при отправке новых обновлений из инжиниринговой компании. Дополнительные сведения см. в разделе [Инжиниринговые компании и правила владения данными](engineering-org-data-ownership-rules.md).

    Когда заказы на технические изменения обрабатываются в инжиниринговой компании, продукты создаются и/или обновляются только в технологической компании. Таким образом, если основные данные продукта также должны быть обновлены, необходимо также выпустить продукты для операционных компаний.

- Продукты можно выпустить непосредственно из заказов на технические изменения. Откройте заказ на изменение, затем в области действий на вкладке **Заказ на изменение** в группе **Выпуски продукта** выберите **Структура выпуска продуктов**. Этот процесс работает так же, как и при выпуске продуктов со страницы **Выпущенные продукты**. Дополнительные сведения см. в разделе [Структуры выпуска продуктов](release-product-structure.md).
- Продукты автоматически выпускаются из заказов на техническое изменение на основе следующих факторов:

    - Повторные выпуски для компаний, в которых продукты ранее были выпущены. Выберите **Поиск** для сканирования всех предыдущих выпусков, затем выберите **Просмотр** для просмотра результатов. На странице **Просмотр** отображаются предыдущие выпуски продуктов, а также можно выбрать продукты, которые требуется повторно выпустить. Затем закройте страницу **Просмотр** и выберите **Процесс** для повторного выпуска выбранных продуктов.
    - Параметры автоматического выпуска в модуле управления выпусками категории инжиниринговых продуктов. Этот выпуск можно выполнить как часть рабочего процесса. Когда используется блок **сбор предложений выпуска**, предложение по выпуску будет заполнено предложениями повторного выпуска (см. предыдущий элемент списка), а продукты будут выпущены в компаниях, если установлен флажок **Автовыпуск** в модуле управления выпуском категории технологических продуктов. Можно выбрать **Просмотр** для просмотра результатов, как описано в предыдущем элементе списка. Продукты также будут повторно выпущены при использовании блока **обработка предложения по выпуску**. Если выбрано только собирать предложение по выпуску в рамках рабочего процесса, можно запустить выпуск вручную, выбрав **Процесс**, как описано в предыдущем элементе списка.

## <a name="follow-up-on-an-engineering-change-request-via-an-engineering-change-order"></a>Последующие действие с запросом на техническое изменение через заказ на техническое изменение

Как только заказ на техническое изменение утвержден, можно выполнить дальнейшее действие с ним через заказ на техническое изменение. Можно объединять несколько запросов на технические изменения в один заказ на техническое изменения. Один заказ на техническое изменение может содержать даже несколько продуктов. (Обычно этот подход применяется, когда к нескольким продуктам должно применяться одно и то же изменение.) Однако невозможно создать несколько заказов технических изменений из одного запроса на техническое изменение.

Для дальнейшей обработки запроса на изменение с помощью заказа на изменение откройте запрос на изменение, затем на панели действий на вкладке **Заказ на изменение** в группе **Заказ на техническое изменение** выберите **Копировать ссылку и продукты**. Затем можно выбрать существующий заказ на техническое изменение, которому требуется подключить запрос на изменение, или можно создать новый заказ на техническое изменение для этого конкретного запроса.

## <a name="engineering-change-order-report"></a>Отчет по заявке на изменение разработки

Отчеты о заказах на технические изменения описывают изменения, которые были сделаны в заказе на технические изменения. Они полезны и во время, и после процесса рассмотрения и утверждения.

Чтобы просмотреть отчет о заказе на технические изменения, откройте соответствующий заказ на изменение, затем на панели действий на вкладке **Заказ на изменение** в группе **Просмотреть** выберите **Отчет о заказе на технические изменения**.

## <a name="fields-on-an-engineering-change-order"></a>Поля заказа на техническое изменение

Большая часть полей в заказах на техническое изменение совпадает с полями выпущенных продуктов, инженерных версий, документов, спецификаций и маршрутов (операций). Однако поля в следующей таблице являются уникальными для заказов на изменение.

| Поле | описание |
|---|---|
| Причины изменения разработки | Выберите причину изменения затронутого продукта. |
| Изменение описания | Введите описание изменения. |
| Требуемое специальное средство | Укажите, требуется ли специальная оснастка для применения изменения. |
| Обработка технологических материалов | Выберите код методов обработки материалов для любого отхода, производимого при применении изменения. |
| Требуется утверждение клиента | Укажите, требуется ли утверждение клиента, прежде чем изменение может быть применено. |
| Получено утверждение клиента | Укажите статус утверждения клиента. |
| Охрана окружающей среды и охрана труда | Укажите, применимы ли к данному изменению правила охраны окружающей среды и правила безопасности. Если это так, можно выбрать соответствующие правила. |

Можно использовать кнопку **Поддержка/копирования информации об изменении**, чтобы скопировать информацию об изменениях между затрагиваемыми продуктами.
