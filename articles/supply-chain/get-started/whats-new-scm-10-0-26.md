# <a name="preview-of-dynamics-365-supply-chain-management-10026-may-2022"></a>Предварительная версия Dynamics 365 Supply Chain Management 10.0.26 (май 2022 г.)

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2022wave1/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное).

Если вы хотите включить или отключить какие-либо из этих функций, необходимо сделать это в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Модуль | Имя функции в управлении функциями | Дополнительные сведения |
|---|---|---|
| Закупки и источники | Разноска зарегистрированных значений учитываемых в запасах продуктов и остатков не учитываемых в запасах продуктов для поступлений и накладных поставщиков | Эта функция позволяет изменить порядок разноски количества не учитываемых в запасах продуктов (например, услуг) при обработке накладных поставщика и входящих отгрузок по заказам на покупку. Функция изменяет поведение параметра количества *Зарегистрированное количество и службы* для разноски приходов и накладных поставщиков, изменяя их в соответствии с поведением параметра *Зарегистрированное количество и не учитываемых в запасах продукты*, который уже был предоставлен при разноске количеств для отборочных накладных по продажам.<br><br>При разноске поступления продуктов или накладной поставщика с помощью параметра количества *Зарегистрированное количество и услуги*, система выполняет разноску зарегистрированного количества продуктов, учитываемых в запасах, и разносит остаток количества не учитываемых в запасах продуктов (включая услуги и не услуги). Без этой функции система все равно разносит зарегистрированное количество учитываемых в запасах продуктов (включая услуги, настроенные как номенклатуры в запасах), но всегда разносит полностью заказанное количество не учитываемых в запасах продуктов сервисных услуг (и игнорирует не учитываемые в запасах продукты, которые не относятся к типу *Услуга*). |
| Закупки и источники | Синхронизация аналитик отслеживания в строках внутрихолдинговых заказов на продажу и на покупку | Эта функция позволяет управлять синхронизацией аналитик отслеживания серийных номеров и номеров партий между строками внутрихолдинговых заказов на продажу и покупку. Он добавляет новые параметры на обе вкладки **Политики заказов на покупку** и **Политики заказов на продажу** страницы **Внутрихолдинговых** настроек для клиентов и поставщиков. Он также обновляет имена нескольких связанных соседних параметров для ясности.<br><br>Если используется расширенное управление складом (WMS), то помните, что эта функция будет синхронизировать номера партий и серийные номера только тогда, когда эти аналитики находятся выше расположения в иерархии резервирования целевых мест назначения. |
| Управление сведениями о продукте | Удаление значений атрибутов продуктов | Эта функция добавляет периодическую задачу под названием **Удаление значений атрибутов продуктов**, предназначенную для удаления записей значений атрибутов продуктов, которые больше не связаны ни с одним продуктом посредством категории продуктов. |
| Управление запасами и складами | (Россия) Предотвращение расхождений при выпуске ГТД для заказов на покупку с номенклатурами, для которых включена WMS | Эта функция доступна только для локализации для России. Это предотвращает расхождения, происходящие при выдаче номеров российских таможенных деклараций (ГТД) для импорта заказов на покупку, включающих номенклатуры, включенные в расширенные складские операции (WMS). Процесс создания ГТД изменяет некоторые значения складских аналитик в соответствующих складских проводках для накладных, включенных в настраиваемый журнал, что приводит к несоответствию между рабочими записями для заказа на покупку и складскими проводками для покупки. Если эта функция включена, процесс формирования ГТД создает работу по корректировке, которая устраняет такие несоответствия. |
| Управление складом | Расширенный анализатор для штрихкодов GS1 | Эта функция добавляет расширенный синтаксический анализатор для данных символов GS1. Новый синтаксический анализатор реализует алгоритм общей спецификации GS1 для разбора символов GS1 и обеспечивает более надежную проверку данных. Дополнительные сведения см. в разделе [Сканирование штрих-кодов GS1](../warehousing/gs1-barcodes.md). |
| Управление складом | Новые страницы рабочего места планирования загрузки | Добавляет две новые страницы рабочего места планирования загрузки: **Рабочее место планирования входящей загрузки** и **Рабочее место планирования исходящей загрузки**. |
| Управление складом | Приложение Warehouse Management — пустая ГТД | Эта функция доступна только для локализации для России. Он позволяет работникам, работающим с мобильным приложением Warehouse Management, оставлять номера таможенных деклараций для России (ГТД) пустым, когда это необходимо. Если аналитика отслеживания ГТД настроена на разрешение пустых значений, система будет принимать пустые значения для ГТД для складских операций при условии доступных заказов в наличии. |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
