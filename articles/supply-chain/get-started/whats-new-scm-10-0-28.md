# <a name="whats-new-or-changed-in-dynamics-365-supply-chain-management-10028-august-2022"></a>Что нового и что изменилось в Dynamics 365 Supply Chain Management 10.0.28 (август 2022 г.)

[!include [banner](../includes/banner.md)]

В этой статье перечислены новые и измененные компоненты Microsoft Dynamics 365 Supply Chain Management версии 10.0.28. Эта версия имеет номер сборки 10.0.1264 и доступна по следующему графику:

- **Предварительная версия выпуска:** май 2022 г.
- **Общая доступность выпуска (самостоятельное обновление):** июль 2022 года
- **Общая доступность выпуска (автоматическое обновление):** июль 2022 года

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2022wave1/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное).

Если вы хотите включить или отключить какие-либо из этих функций, необходимо сделать это в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Модуль | Имя функции в управлении функциями | Дополнительные сведения |
|---|---|---|
| Управление запасами и складами | Включить параметр "Внутрихолдинговые запасы в наличии" для отображения наличия только тех запасов, количество которых отличается от нуля | Эта функция позволяет выбрать, должны ли номенклатуры с нулевым количеством в наличии включаться в список внутрихолдинговых запасов в наличии. Можно управлять этим параметром, используя параметр **Не показывать номенклатуры с нулевым количеством в наличии в списке внутрихолдинговых запасов в наличии**, который добавляется к странице **Параметры управления запасами и складами**. |
| Управление запасами и складами | (Индия) Игнорировать местонахождение для правил цен переноса, если для параметра "Из кода склада" установлено значение "Все" | <p>Эта функция применяется только для локализации Индии. Процесс настройки трансферных цен для номенклатур на складских перемещениях становится более наглядным.</p><p>Для настройки трансферных цен необходимо настроить каждую номенклатуру с правилами ценообразования для перемещения. Одним из способов выполнения этой конфигурации является добавление строки правила, в которой для поля **кода склада** указано значение *все*. Этот параметр указывает, что цена перемещения, определенная по строке, должна применяться независимо от склада, из которого отбирается номенклатура. Если эта функция включена, правила цены для перемещения, для которых значение поля **От кода склада** равно *Все*, будут игнорировать настройку **Местоположения**. Поэтому правило будет применяться независимо от местоположения, указанного в заказе на перемещение. Такое поведение, вероятно, является ожидаемым, поскольку местоположение находится ниже склада в иерархии измерений хранения.</p><p>Без этой функции система будет применять правила этого типа только в том случае, если местоположение в заказе на перемещение точно соответствует местоположению, заданному для правила. (Если для правила задано пустое местоположение, система будет применять правило только к заказам на перемещение, которые также имеют пустое значение для местоположения.)</p> |
| Управление запасами и складами | Очистка данных отчета о наличии запасов | Эта функция позволяет удалить данные, которые используются для создания отчетов *Хранилище отчетов о запасах в наличии*. |
| Управление производством | Назначение мероприятий по проекту для строк договора на обслуживание и заказа на обслуживание | Эта функция добавляет поле с именем **Мероприятия по проекту** к соглашению об обслуживании и строкам заказа на обслуживание, чтобы вы могли установить для них проектную деятельность. Эта функция поможет предотвратить блокирующие ошибки при публикации журналов проекта управления услугами, требующих установки действия проекта.  |
| Управление складом | Служба комплектации строк перемещения вручную для администратора или подобных доверенных пользователей | Эта функция позволяет администраторам вручную выбирать операции с запасами, связанные с линиями передачи. Эти строки включают строки, которые уже были выпущены на склад. Администраторы должны делать это только в исключительных случаях, например, когда система находится в поврежденном состоянии. |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В статье [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в статье [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
