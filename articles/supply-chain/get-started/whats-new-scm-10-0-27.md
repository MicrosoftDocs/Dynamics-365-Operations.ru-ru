# <a name="preview-of-dynamics-365-supply-chain-management-10027-july-2022"></a>Предварительная версия Dynamics 365 Supply Chain Management 10.0.27 (июль 2022 г.)

[!include [banner](../includes/banner.md)]

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2022wave1/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное).

Если вы хотите включить или отключить какие-либо из этих функций, необходимо сделать это в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Модуль | Имя функции в управлении функциями | Дополнительные сведения |
|---|---|---|
| Сводное планирование | Учитывать время упреждения запасов при создании плановых заказов на перемещение | При создании планового заказа на перемещение эта функция позволяет учитывать время упреждения запасов, указанное в параметрах заказа номенклатуры по умолчанию, когда она вычисляет дату получения запланированного заказа на перемещение для настройки контроля даты доставки в качестве времени упреждения типа *Нет* или *Продажи*. Когда указано время упреждения, его значение будет использоваться для расчета даты получения, а дни транспортировки не будут учитываться. |
| Закупки и источники | Налоговая информация по контракту брокера по умолчанию в строках накладной поставщика | Эта функция вводит логику для задания значений по умолчанию для полей **Налог** и **Налог номенклатуры** в строках накладной поставщика в контракте брокера. Эта логика применяется только в том случае, если типом накладных расходов в строке контракта брокера является ГК/ГК. Налоговая группа номенклатур будет принимать значение по умолчанию из категории закупаемой продукции брокера (если она настроена) или из типа накладных расходов. Налоговая группа получает свое значение по умолчанию из счета поставщика. |
| Закупки и источники | Ограничение числа строк заказов на покупку на пакетную задачу | Эта функция позволяет оптимизировать производительность системы при разноске подтверждений и поступлений продуктов. Он добавляет новый параметр, который называется **Строк на задачу**, на вкладку **Доставка** страницы **Параметры модуля "Закупки и источники"**. Эта новая настройка позволяет ограничить число строк заказа на покупку, обрабатываемых каждой пакетной задачей. Таким образом, это может помочь предотвратить замедление работы системы большими пакетными задачами. |
| Управление сведениями о продукте | Заполнение значений атрибутов продуктов | Эта функция добавляет периодическую задачу с именем *Заполнение значений атрибутов продуктов*. Эта новая периодическая задача создает отсутствующие записи значений атрибутов продуктов для атрибутов, которые связаны с продуктами через категорию продуктов. |
| Управление производством | Дополнительная настройка интерфейса выполнения производственного цеха | <p>Эта функция добавляет следующие параметры на странице **Настроить управление производственным участком**:</p><ul><li>**Автоматически открывать диалоговое окно запуска после завершения поиска** — если этот параметр включен, диалоговое окно **Запустить задание** автоматически открывается, когда работники используют строку поиска для поиска задания.</li><li>**Автоматически открывать диалоговое окно хода производства после завершения поиска** — если этот параметр включен, диалоговое окно **Проверить ход выполнения** для задания автоматически открывается, когда работники используют строку поиска для поиска задания.</li><li>**Оставшееся количество по умолчанию в диалоговом окне хода производства** — если этот параметр включен, в диалоговом окне **Проверить ход выполнения** отображается оставшееся количество для отчета по производственному заданию.</li></ul><p>Дополнительные сведения см. в разделе [Настройка интерфейса выполнения производственного цеха](../production-control/production-floor-execution-configure.md).</p> |
| Управление производством | Производственные группы в интерфейсе выполнения производственного цеха | <p>Когда одному рабочему заданию назначается несколько сотрудников, они могут назначить одного сотрудника как руководителя. Остальные работники автоматически становятся помощниками для этого руководителя. Для конечной рабочей группы только руководитель должен зарегистрировать статус задания, в то время как записи времени применяются ко всем участникам рабочей группы. Эта функция также поддерживает сценарий с именем *ресурс помощи*. В этом сценарии работник может зарегистрироваться в качестве помощника для другого сотрудника, который затем становится руководителем для новой рабочей группы.</p><p>Дополнительные сведения см. в разделе [Как работники используют интерфейс выполнения производственного цеха](../production-control/production-floor-execution-use.md).</p> |
| Управление производством | Отчет по планированию номенклатур в интерфейсе выполнения производственного цеха | Эта функция упрощает процесс составления отчетов по заказам партий для планирования номенклатур в интерфейсе выполнения производственного цеха. При создании заказа партии для продукта, который имеет тип производства *Плановая номенклатура*, приемка может быть выполнена только для побочных и попутных продуктов для этого заказа партии. Заказы партий для плановых номенклатур обычно используются для поддержки процессов разборки, когда продукт сырья разбирается на несколько отдельных продуктов. Когда работник сообщает о завершении заказа партии для плановой номенклатуры, в диалоговом окне **Проверить ход выполнения** будут отображаться только побочные и сопутствующие продукты. Ранее также отображалась плановая номенклатура, и это поведение могло запутать работников. |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
