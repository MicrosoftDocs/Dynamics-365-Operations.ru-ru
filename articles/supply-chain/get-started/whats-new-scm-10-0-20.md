# <a name="preview-of-dynamics-365-supply-chain-management-10020-august-2021"></a>Предварительная версия Dynamics 365 Supply Chain Management 10.0.20 (август 2021 года)

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой теме перечислены новые и измененные компоненты предварительной версии Microsoft Dynamics 365 Supply Chain Management выпуска 10.0.20. Эта версия имеет номер сборки 10.0.886 и доступна следующим образом:

- **Предварительная версия выпуска:** май 2021 г.

- **Общая доступность выпуска (автоматическое обновление):** август 2021 года

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2021wave1/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное). Если вы хотите использовать какие-либо из этих функций, необходимо явно включить их в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Область компонентов | Имя&nbsp;функции в&nbsp;управлении&nbsp;функциями | Дополнительные сведения |
|---|---|---|
| Сводное планирование | Отрицательные дни для оптимизации планирования | Эта предварительная версия функции позволяет оптимизировать планирование с учетом допуска задержки в зависимости от параметра **Отрицательные дни**, определенного в группах покрытия. |
| Сводное планирование | Параллельная авторизация скорректированного прогноза спроса | Эта функция разрешает параллельную авторизацию скорректированного прогноза спроса со страницы **Скорректированный прогноз спроса**. Цель этой функции — повысить производительность, когда авторизуется большее количество прогнозов. При выполнении авторизации пользователь может указать **Количество потоков** в диалоговом окне авторизации. |
| Сводное планирование | (Предварительная версия) Подтверждение и консолидация в пакетном режиме для спланированных оптовых заказов и партионных заказов на упаковку | Эта функция позволяет использовать пакетные задания для подтверждения и консолидации спланированных оптовых заказов и заказов на упаковку. |
| Управление производством | Копировать стандартные маршруты | Эта функция расширяет функцию копирования маршрута, чтобы позволить пользователям копировать маршруты, не связанные с номенклатурой. Она позволяет системе обновлять все соответствующие сведения (такие как узел, маршрутная группа, потребности в ресурсах и различные времена) после использования функции копирования маршрута для перезаписи маршрута, еще не назначенного номенклатуре. |
| Управление производством | Обновить связанные потребности в ресурсах при изменении операции маршрута | Благодаря этой функции система может обновить связанные потребности в ресурсах после того, как пользователь изменит операцию существующего шага маршрута. |
| Управление сведениями о продукте | Предварительная обработка отчета по спецификации для исключения тайм-аута | Эта функция вызывает предварительную обработку отчета по спецификациям. Это позволяет избежать ошибок времени ожидания, связанных с большой загрузкой данных для отчета. |
| Закупки и источники | Включить сброс workflow-процессов, связанных с закупками | Эта предварительная версия функции позволяет сбросить следующие рабочие процессы в статус черновика: заказ на покупку, изменение поставщика и заявки на закупку. |
| Управление транспортировкой | Включить создание журнала накладных поставщика при отмене векселя фрахта | Если эта функция включена, соответствующий журнал накладных поставщиков будет создан только для причин выверки при использовании параметра платежа поставщику. В противном случае журнал накладных будет создан всегда. |
| Управление складом | Проверить шаблоны, выбранные для заданий пополнения | Эта функция позволяет гарантировать, что пользователи смогут выбирать допустимые шаблоны пополнения при настройке задания пополнения. Она не позволяет пользователям создавать задание пополнения без шаблона и выбирать шаблоны для типа *Потребность волны*, которые не создадут никакой работы пополнения и займут много времени для обработки. |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
