# <a name="whats-new-or-changed-in-dynamics-365-supply-chain-management-10029-october-2022"></a>Что нового и что изменилось в Dynamics 365 Supply Chain Management 10.0.29 (октябрь 2022 г.)

[!include [banner](../includes/banner.md)]

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2022wave1/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное).

Если вы хотите включить или отключить какие-либо из этих функций, необходимо сделать это в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Модуль | Имя функции в управлении функциями | Дополнительные сведения |
|---|---|---|
| Управление затратами | Оптимизация расчета ожидающей цены сопутствующего продукта | Эта функция устраняет конфликт, иногда возникающий, когда цены побочных продуктов рассчитываются с использованием нескольких потоков. В результате система проверяет, чтобы каждая цена побочного продукта была рассчитана только один раз. Результат этого расчета используется в качестве входных данных для всех остальных расчетов. Если ожидающая цена уже существует, используется эта цена. |
| Сводное планирование | Группировать проводки при оптимизации планирования | Эта функция позволяет сократить число спланированных заказов, созданных для поставки одной строки заказа на продажу при использовании оптимизации планирования. Если эта функция включена, оптимизация планирования будет группировать все проводки по запасам для строки заказа в одно требование в полном количестве. (Это поведение соответствует встроенному поведению механизма планирования.) Поставки и спрос группируются по отдельности. Таким образом, функция помогает уменьшить объем проводок при наличии разделенных проводок и при использовании аналитик (таких как номера партии или серийные номера), которые не являются аналитиками покрытия. |
| Закупки и источники | Заблокировать для поставщика заказы на покупку | Эта функция позволяет заблокировать поставщика для заказов на покупку. Добавляет новый тип блокировки *Заказ на покупку*, который помечает поставщика как заблокированного для заказов на покупку. Вы не сможете создавать новые заказы на покупку для поставщиков, заблокированных для заказов на покупку, но по-прежнему сможете продолжать работать с любыми открытыми накладными или платежами для этих поставщиков. |
| Продажи и маркетинг | Рассчитать чистую сумму по строке для импорта | Эта функция позволяет контролировать, должна ли система пересчитывать итоговые значения по строкам при импорте данных через сущности *Строки заказов на продажу*, *Строки предложения по продаже* или *Строки заказа на возврат* с помощью OData или двойной записи. Она действует лишь в случае, когда также имеются действующие политики оценки торговых договоров, которые ограничивают изменения поля **Чистая сумма** для строк заказов на продажу, строк предложений по продаже и (или) строк заказов на возврат. Она добавляет настройку с названием **Рассчитать чистую сумму по строке** на страницу **Расчеты с клиентами > Настройки > Параметры модуля расчетов с клиентами**. Если для этой настройки установлено значение *Да*, система будет всегда пересчитывать суммы строк при необходимости (таким образом игнорируя политику оценки торговых договоров для чистой суммы по строке). Если для этой настройки установлено значение *Нет*, система не будет автоматически рассчитывать чистую сумму по строке, даже если входящие изменения цены, количества и (или) скидки в строке будут подразумевать перерасчет чистой суммы по строке. Эта функция включена по умолчанию и изначально устанавливает для параметра **Рассчитать чистую сумму по строке** значение *Да*. Значение *Нет* соответствует поведению системы до версии 10.0.23 и в основном предназначено для поддержки устаревших сценариев интеграции.<br><br>Дополнительные сведения см. в разделе [Пересчет чистых сумм по строкам при импорте заказов на продажу, предложений и возвратов](../sales-marketing/calc-line-net-amounts-import.md). |
| Продажи и маркетинг | Рассчитать итоги продаж с помощью нескольких потоков | Эта функция помогает повысить производительность системы, позволяя системе использовать параллельную обработку при расчете итогов продаж в пакетном режиме. Эта функция добавляет новое поле **Количество потоков** в диалоговое окно **Рассчитать итоги продаж**. Если выбрать выполнение расчета в пакетном режиме, это поле также можно использовать для задания максимального количества потоков. Если задать значение *0* (ноль) или *1*, будет использоваться один поток. Значения выше 1 обеспечивают многопоточность. |
| Продажи и маркетинг | Обновить цены и скидки, введенные вручную для внутрихолдинговых операций | Эта функция обеспечивает возможность изменения политик вручную для внутрихолдинговых заказов. Она включает в себя перенос политик изменения вручную между внутрихолдинговыми продажами и заказами на покупку. Ранее это было возможно только для заказов, не являющихся внутрихолдинговыми. Когда эта функция включена, после сохранения изменений во внутрихолдинговые заказы отображается диалоговое окно "Обновление цен и скидок". Это диалоговое окно может использоваться для применения новых сведений о ценах и скидках ко внутрихолдинговому заказу или для того, чтобы оставить заказ без изменений. |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В статье [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в статье [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
