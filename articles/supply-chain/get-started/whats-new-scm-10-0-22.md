# <a name="preview-of-dynamics-365-supply-chain-management-10022-november-2021"></a>Предварительная версия Dynamics 365 Supply Chain Management 10.0.22 (ноябрь 2021 г.)

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой теме перечислены новые и измененные компоненты предварительной версии Microsoft Dynamics 365 Supply Chain Management выпуска 10.0.22. Эта версия имеет номер сборки 10.0.995 и доступна следующим образом:

- **Предварительная версия выпуска:** сентябрь 2021
- **Общая доступность выпуска (самостоятельное обновление):** октябрь 2021 г.
- **Общая доступность выпуска (автоматическое обновление):** ноябрь 2021 г.

## <a name="feature-enhancements-included-in-this-release"></a>Улучшения функций, включенные в данный выпуск

В этой таблице перечислены улучшения функций, включенные в этот выпуск. Каждая из них обеспечивает дополнительное усовершенствование существующей функции. Поскольку они только улучшения, они не перечисляются в [плане выпуска](/dynamics365-release-plan/2021wave2/finance-operations/dynamics365-supply-chain-management/planned-features). Однако чтобы гарантировать, что эти улучшения не будут конфликтовать с существующими настройками или предпочтениями, каждое из них отключена по умолчанию (если не указано иное). Если вы хотите использовать какие-либо из этих функций, необходимо явно включить их в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

| Область компонентов | Имя функции в управлении функциями | Дополнительные сведения |
|---|---|---|
| Управление затратами | Создание соответствующих ваучеров для переоценки округления стандартной себестоимости | <p>При выполнении финансовой разноски запасов (например, по накладной заказа на продажу или складской проводке) эта функция заставляет систему создать отдельный ваучер для любой связанной с ней переоценки округления стандартной себестоимости и привязать ее к ваучеру финансовой разноски в качестве связанного ваучера.</p><p>Если эта функция не выбрана, система записывает переоценки округления стандартной себестоимости по одному и тому же ваучеру разноски. Это может привести к возникновению конфликтов с датой, поскольку переоценка использует дату сеанса или системную дату, тогда как финансовые разноски используют дату разноски.</p> |
| Распределенная гибридная топология | *(Управление функциями не является обязательным.)* | <p>В этом выпуске разворачивается возможность планирования исходящей загрузки для рабочей нагрузки управления складом для облачных и пограничных единиц масштабирования.</p><p>Дополнительные сведения см. в разделе [Рабочие нагрузки управления складом для облачных и пограничных единиц масштабирования](../cloud-edge/cloud-edge-workload-warehousing.md).</p> |
| Управление изменениями разработки | Создание вариантов для продуктов разработки | <p>Эта функция позволяет создать несколько вариантов для технологического продукта на основе аналитик его цвета, размера, стиля или конфигурации.</p><p>Дополнительные сведения см. в разделе [Создание вариантов для продуктов разработки](../engineering-change-management/engineering-variants.md).</p> |
| Управление запасами и складами | Интеграция видимости запасов со смещением резервирования | <p>Эта возможность доступна только после включения возможности функции *Интеграция видимости запасов*. Она предоставляет функциональность для корр. резервирований, сделанных по в видимости запасов.</p><p>Дополнительные сведения см. в разделе [Резервирования видимости запасов](../inventory/inventory-visibility-reservations.md).</p> |
| Продажи и маркетинг | Ограничить число заказов на продажу, которые можно выбрать для разноски | <p>Данная функция включена автоматически. Она добавляет поле **Макс. число заказов на продажу для разноски** на странице **Параметры модуля расчетов с клиентами**. Это поле позволяет определить максимальное число заказов на продажу, которые могут быть выбраны при разноске подтверждений, листов подбора, отборочных накладных и накладных на странице списка заказов на продажу. Значение по умолчанию — *100*.</p><p>Данная функция позволяет повысить производительность страницы список заказа на продажу при выборе значительного количества заказов на продажу. Она не влияет на количество заказов на продажу, которые могут быть обработаны периодическим заданием.</p> |

### <a name="removed-and-deprecated-supply-chain-management-features"></a>Удаленные и устаревшие функции Supply Chain Management

В теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) описываются функции, которые запланированы к удалению или которые планируется удалить или объявить устаревшими для Supply Chain Management.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в теме [Удаленные или устаревшие функции в Dynamics 365 Supply Chain Management](removed-deprecated-features-scm-updates.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
