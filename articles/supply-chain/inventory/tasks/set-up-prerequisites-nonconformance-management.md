# <a name="set-up-prerequisites-for-nonconformance-management"></a>Настройка предварительных требований для управления несоответствиями

[!include [task guide banner](../../includes/task-guide-banner.md)]

Используйте эту тему, чтобы включить процесс управления несоответствиями. Несоответствие описывает процедуру или номенклатуру, имеющую проблемы качества, причем это описание включает сведения об источнике и типе проблемы. В этой процедуре используется компания с демонстрационными данными USMF. Обычно эта процедура выполняется менеджером по контролю качества.


## <a name="enable-quality-management-processes-within-the-company"></a>Включение процессов управления качеством в компании
1. В области перехода выберите **Модули > Управление запасами > Настройка > Параметры модуля Управление запасами и складами**.
2. Выберите вкладку **Управление качеством**.
3. Выберите **Да** в поле **Использовать управление качеством**, чтобы включить процессы управления качеством для компании.
4. В поле **Почасовая ставка** введите число в местной валюте. Почасовая ставка используется для расчета затрат на операции, связанные с несоответствием. Почасовая ставка и рассчитанные затраты предоставляют справочные сведения для несоответствия, и они не взаимодействуют с другими функциями.  
5. Выберите **Настройка отчетов**, чтобы определить типы примечаний отчетов по контролю качества, которые будут использоваться для различных типов отчетов по управлению качеством.

## <a name="enable-user-for-nonconformance-processing"></a>Включение пользователя для обработки несоответствия
1. В области перехода выберите **Модули > Администрирование системы > Пользователи > Пользователи**. 
2. Используйте быстрый фильтр для поиска пользователя, который будет утверждать или отклонять записи несоответствия. Например, отфильтруйте поле **Имя** по значению `Ricardo`. Чтобы обработать утверждение несоответствия, пользователь, который утверждает или отклоняет несоответствия, должен иметь значение "Имя", назначенное на странице **Пользователи**. Для использования примечаний к документам у пользователя также должна быть активирована функция "Работа с документами" (в параметрах пользователя).  
3. Пометьте строку с нужной записью.
4. Выберите **Параметры пользователя**.
5. Выберите вкладку **Параметры**.
6. Выберите значение **Да** в поле **Разрешить работу с документами**.

## <a name="define-diagnostic-types-for-nonconformance-processing"></a>Определение типов диагностики для обработки несоответствия
1. В области перехода перейдите к **Модули > Управление запасами > Настройка > Управление качеством > Типы диагностики**. Используйте страницу **Типы диагностики**, чтобы определить классификацию диагностических действий. Коррекция устанавливает, какой тип диагностического действия должен быть предпринят для утвержденного несоответствия, кто его должен выполнять, а также запрошенную и планируемую дату завершения.  
2. Выберите **Создать**.
3. В поле **Диагностика** введите значение.
4. В поле **Описание** введите значение.

## <a name="define-quality-charges-for-nonconformance-processing"></a>Определение расходов по контролю качества для обработки несоответствия
1. В области перехода перейдите к **Модули > Управление запасами > Настройка > Управление качеством > Расходы по контролю качества**. Используйте страницу **Расходы по контролю качества**, чтобы определить классификацию расходов, которые будут использоваться в операциях, связанных с несоответствиями.  
2. Выберите **Создать**.
3. В поле **Код накладных расходов** введите значение.
4. В поле **Описание** введите значение.

## <a name="define-the-operations-for-nonconformance-processing"></a>Определение операций для обработки несоответствия
1. В области перехода перейдите к **Модули > Управление запасами > Настройка > Управление качеством > Операции**. Используйте страницу **Операции**, чтобы определить классификацию работы, которая может быть выполнена для утвержденного несоответствия. При связи операции с несоответствием имеется возможность определить сведения о связанном материале, часах трудозатрат и накладных расходах, требуемых для выполнения операции. Эти сведения составляют основу для расчета оценочной стоимости выполнения операции.  
2. Выберите **Создать**.
3. В поле **Операция** введите значение.
4. В поле **Описание** введите значение.

## <a name="define-problem-types-for-nonconformance-processing"></a>Определение типов проблем для обработки несоответствия
1. В области перехода перейдите к **Модули > Управление запасами > Настройка > Управление качеством > Типы проблем**. Используйте страницу **Типы проблем**, чтобы определить классификацию проблем качества, возникающих для различных типов несоответствий. Имеются следующие типы несоответствий: **Внутренний**, **Клиент**, **Поставщик**, **Запрос на обслуживание**, **Производство** и **Производство сопутствующих продуктов**. Один тип проблемы можно связать с несколькими типами несоответствий.  
2. Выберите **Создать**.
3. В поле **Тип проблемы** введите значение.
4. В поле **Описание** введите значение.
5. Выберите **Типы несоответствия**. Используйте страницу **Типы несоответствия** для указания типа проблемы, который будет использоваться для одного или нескольких типов несоответствия. Например, тип проблемы, относящийся к коду брака, может применяться ко всем типам несоответствия, в то время как тип проблемы о жалобах клиента может применяться только к типам несоответствия о клиентах или запросах на обслуживание.  
6. Выберите **Создать**.
7. В поле **Тип несоответствия** в следующей строке выберите параметр.

## <a name="define-quarantine-zones-for-nonconformance-processing"></a>Определение зон карантина для обработки несоответствия
1. В области перехода перейдите к **Модули > Управление запасами > Настройка > Управление качеством > Зоны карантина**.
2. Выберите **Создать**.
3. В поле **Зона карантина** введите значение.
4. В поле **Описание** введите значение.
5. Закройте страницу.
