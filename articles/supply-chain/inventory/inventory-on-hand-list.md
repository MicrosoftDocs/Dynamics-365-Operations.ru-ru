# <a name="inventory-on-hand-list"></a>Список запасов в наличии

[!include [banner](../includes/banner.md)]

В этом разделе описывается, как использовать страницу **списка запасов в наличии** для проверки сведений о запасах в наличии. В нем показаны несколько способов совместной работы различных параметров фильтрации и сортировки, а также способы, с помощью которых эти возможности могут получить неожиданные результаты при их комбинировании.

## <a name="query-your-on-hand-inventory"></a>Запрос запасов в наличии

Для проверки доступности запасов перейдите в раздел **Управление запасами \> Запросы и отчеты \> Список количества в наличии**.

Страница **Список количества в наличии** автоматически обновляется, когда проводки выполняются на складе. Этими проводками могут быть прогнозы, физические или финансовые проводки.

Следующие средства используются для поиска требуемых продуктов:

- В области действий выберите [**Аналитики**](#dimensions), чтобы открыть диалоговое окно, в котором можно добавлять или удалять столбцы, отображаемые в сетке **В наличии**.
- В области [**Фильтры**](#filters-pane) введите значения для некоторых полей, чтобы отобразить только те записи, которые соответствуют этим значениям. Обратите внимание, что фильтры, определяемые здесь, применяются к исходным таблицам, которые позже могут быть объединены в соответствии с аналитиками, выбранными для отображения. Сведения о том, как это поведение может влиять на результаты, см. в [примерах](#examples) ниже в этом разделе.
- На панели **Фильтры** выберите **Применить** для создания списка соответствующих запасов в наличии в сетке **В наличии**.
- В сетке **В наличии** выберите любой заголовок столбца для сортировки или фильтрации значений в этом столбце. Дополнительные параметры фильтрации представлены в быстром фильтре в верхней части сетки. Эти фильтры применяются к результатам, а не к исходным таблицам. Сведения о том, как это поведение может влиять на результаты, см. в [примерах](#examples) ниже в этом разделе.

Для каждой соответствующей номенклатуры сетка **В наличии** содержит следующие столбцы сведений о запасах.

| Столбец | описание |
|---|---|
| Физические запасы | Физическое количество, доступное на складе. |
| Физически зарезервировано | Общее количество, которое было физически зарезервировано. |
| Физ. доступно | Доступное (не зарезервированное количество) количество в физических запасах.<p>**Физически доступно** — это вычисляемое поле. Значение равно значению **Физические запасы** минус значение **Физически зарезервировано**.</p> |
| Физически доступно в дополнительных аналитиках | Доступное физическое количество для всех аналитик, отображаемых в сетке. |
| Всего заказано | Общее количество, которое включается в состав входящих заказов или имеет положительное количество в различных журналах запасов. |
| Заказано | Общее количество, которое включается в состав исходящих заказов или имеет отрицательное количество в различных журналах запасов. |
| Зарезервировано в заказанных | Общее количество, зарезервированное в заказанных приходах. Значение в этом поле представляет общее количество номенклатур в исходящих проводках, имеющих статус _Зарезервировано в заказанных_. Номенклатуры, зарезервированные как заказанные, физически недоступны в запасах. Следовательно, их невозможно немедленно скомплектовать и доставить. |
| Доступно для резервирования | Общее количество запасов в наличии, которое доступно для резервирования.<p>**Примечание.** Если установлен флажок **Зарезервировать заказанную номенклатуру** на странице **Параметры модуля "Управление запасами и складами"**, значение в этом поле включает ожидаемые приходы. Если этот флажок снят, значение не включает ожидаемые приходы.</p> |
| Доступное общее количество | Общее доступное количество.<p>**Всего доступно** — это вычисляемое поле. Это значение равно значению **Физически доступно** плюс значение **Всего заказано** минус значение **В заказе**.</p> |

## <a name="apply-filters-to-find-the-records-that-youre-looking-for"></a><a name="filters-pane"></a>Применение фильтров для поиска нужных записей

Используйте панель **Фильтры** для фильтрации списка запасов в наличии, чтобы он включал только записи, в которых значения полей соответствуют критериям фильтрации. Чтобы определить фильтр, выполните следующие действия.

1. На панели **Фильтры** найдите поле, для которого необходимо выполнить фильтрацию.
2. В поле под именем целевого поля выберите логический оператор (например, *начинается с*, *равно* или *больше чем*).
3. Введите или выберите значение для поиска.

> [!IMPORTANT]
> Страница **Список запасов в наличии** составляется из подробной таблицы запасов в наличии, которая включает все доступные аналитики. Однако список на этой странице является сводным. Таким образом, это может быть сочетание строк из исходной таблицы путем агрегирования значений в соответствии с отображаемыми аналитиками.
>
> Фильтры, определяемые на панели **Фильтры**, применяются к исходной таблице, а не к сводному списку. Такое поведение может иногда привести к непредсказуемым результатам. Сведения о том, как это поведение может влиять на результаты, см. в [примерах](#examples) ниже в этом разделе.
> 
> Однако, [указанные в сетке фильтры](#grid-filters) *применяются* к сводному списку. К этим фильтрам относятся QuickFilter в верхней части сетки и фильтр для каждого заголовка столбца.

Можно изменить набор фильтров, доступных на панели **Фильтры**, выполнив следующие шаги.

- Чтобы удалить фильтр с панели, выберите его кнопку **Закрыть** (**X**).
- Чтобы добавить фильтр, выберите **Добавить** в верхней части панели **Фильтры**. Откроется диалоговое окно **Добавление полей фильтра**, в котором отображается список доступных полей. Кроме того, в нем отображаются сведения о типе данных и таблице для каждого поля. Используйте заголовки столбцов, чтобы отфильтровать и отсортировать список по мере необходимости, а затем установите флажок для каждого поля, которое необходимо добавить на панель **Фильтр**. По завершении нажмите кнопку **Вставить**, чтобы применить изменения.

## <a name="select-which-dimensions-to-show"></a><a name="dimensions"></a>Выбор отображаемых аналитик

Аналитики позволяют узнать больше о каждой номенклатуре в списке запасов в наличии и предоставить дополнительные возможности сортировки и фильтрации списка. Аналитики, выбранные для отображения, также влияют на то, как строки суммируются на странице **Список количества в наличии**. Это агрегирование, в свою очередь, может влиять на объединение строк из исходных таблиц в результаты, которые вы видите. Сведения о том, как это поведение может влиять на результаты, см. в [примерах](#examples) ниже в этом разделе.

Чтобы настроить отображаемый набор складских аналитик, выполните следующие действия.

1. В области действий выберите **Аналитики**.

    Откроется диалоговое окно **Отображение аналитики**, в котором приводятся все измерения.

2. Установите флажок для каждой аналитики, которую следует включить в сетку.
3. Если выбор должен использоваться по умолчанию при следующем открытии страницы **Список количества в наличии**, задайте для параметра **Сохранить настройку** значение **Да**. Если для этого параметра установить значение **Нет**, выбранный вариант будет использоваться только в течение текущего сеанса. Поэтому при следующем открытии страницы будет использоваться текущий выбранный вариант по умолчанию.
4. Выберите **ОК**, чтобы закрыть диалоговое окно и применить изменения.

## <a name="filter-on-the-output-of-the-inventory-on-hand-list"></a><a name="grid-filters"></a>Фильтрация по выходным данным списка запасов в наличии

Можно выбрать любой заголовок столбца в сетке **В наличии** для сортировки или фильтрации значений в этом столбце. Дополнительные параметры фильтрации представлены в быстром фильтре в верхней части сетки. Эти фильтры применяются к результатам, а не к исходным таблицам. Сведения о том, как это поведение может влиять на результаты, см. в [примерах](#examples) ниже в этом разделе.

> [!NOTE]
> Фильтрация и сортировка по всем столбцам невозможны. Большинство столбцов количества не включают в себя элементы управления сортировкой и фильтрацией, поскольку они являются вычисляемыми полями. Столбец **В заказе** является исключением.

## <a name="examples"></a><a name="examples"></a>Примеры

В системе имеется подробная (неагрегированная) таблица запасов, в которой показаны следующие запасы в наличии.

| Код номенклатуры | Сайт | Склад | Физические запасы | Физически доступно |
|---|---|---|---|---|
| IA0001 | 1 | 1 | 1 | 1 |
| IA0001 | 1 | 2 | 2 | 2 |
| IA0001 | 1 | 3 | 1 | 1 |

### <a name="scenario-1"></a>Сценарий 1

Страница **Список количества в наличии** настроена для отображения следующих окончательных аналитик:

- Код номенклатуры
- Сайт
- Склад

На панели **Фильтры** настраиваются следующие критерии фильтрации:

- **Код номенклатуры** \| **точно** \| _IA0001_
- **Доступно физически** \| **меньше или равно** \| _1_

Здесь полученный результат.

| Код номенклатуры | Сайт | Склад | Физические запасы | Физически доступно |
|---|---|---|---|---|
| IA0001 | 1 | 1 | 1 | 1 |
| IA0001 | 1 | 3 | 1 | 1 |

### <a name="scenario-2"></a>Сценарий 2

Страница **Список количества в наличии** настроена для отображения следующих окончательных аналитик:

- Код номенклатуры
- Сайт

На панели **Фильтры** настраиваются следующие критерии фильтрации:

- **Код номенклатуры** \| **точно** \| _IA0001_
- **Доступно физически** \| **меньше или равно** \| _1_

Здесь полученный результат.

| Код номенклатуры | Сайт | Физические запасы | Физически доступно |
|---|---|---|---|
| IA0001 | 1 | 2 | 2 |

Обратите внимание, что настройки на панели **Фильтры** применимы к подробной (неагрегированной) таблице запасов, которая отображается в начале этого раздела. Таким образом, критерий **Физически доступно** \| **меньше или равно** \| _1_ находит две строки из этой таблицы (первая и третья строки, каждая из которых отображает значение **Физически доступно** _1_). Однако в этом случае страница **Список количества в наличии** не настроена на отображение аналитики **Склад**. Таким образом, две исходные строки суммируются в одну результирующую строку, поскольку обе строки имеют одинаковые значения во всех отображаемых аналитиках. Эта строка нарушает критерий фильтрации, так как значение **Физически доступно** отображается как _2_. Однако результат является правильным, поскольку настройки на панели **Фильтры** применимы к исходной таблице, а не к агрегированной таблице, которая отображается на странице **Список количества в наличии**.
