# <a name="quality-management-test-groups"></a>Группы проверки контроля качества

[!include [banner](../includes/banner.md)]

В этой теме описывается, как создавать группы проверки, чтобы можно было использовать несколько проверок с заказами контроля качества в Microsoft Dynamics 365 Supply Chain Management.

Используйте страницу **Группы проверки** для настройки, изменения и просмотра групп проверки и отдельных проверок, назначенных им. В верхней области странице отображаются группы проверки, а в нижней области отображаются проверки, включенные в выбранную группу проверки.

Группе проверки назначаются несколько политик, включая план выборочного контроля, допустимый уровень качества и потребности для проверки с разрушением. Затем когда вы назначаете отдельную проверку для группы проверки, вы определяете дополнительную информацию, такую как последовательность, документы и даты срока действия. Для количественного испытания информация, которую вы определяете, также включает приемлемые значения измерений. Для качественного испытания информация включает переменную испытания и результат по умолчанию.

Группа проверок, назначенная заказу на контроль качества, определяет начальный набор испытаний, которые должны выполняться для указанной номенклатуры. Однако можно в заказе для контроля качества можно добавлять, удалять или изменять проверки. Результаты проверок указываются для каждой проверки в заказе контроля качества.

При определении группы проверки можно дополнительно указать выборку номенклатур. Выборки номенклатур используются для определения суммы продукта, которую необходимо проверить. Дополнительные сведения см. в разделе [Выборка номенклатур управления качеством](quality-item-sampling.md). Можно также указать, являются ли проверки в группе проверки разрушительными. При проверках с разрушением выборка номенклатур уничтожается, и количество удаляется из запасов в наличии.

## <a name="example-of-a-test-group"></a>Пример группы проверки

Производственная компания определяет группу испытаний для каждой разновидности своих инструкций по обеспечению качества. Разные группы испытаний отражают различия планов выборочного контроля, наборов испытаний, которые должны быть выполнены совместно, приемлемых уровней качества, а также других коэффициентов. Для испытаний с количественными результатами также имеются различия в приемлемых значениях измерений. Чтобы применить инструкции по обеспечению качества, компания назначает группу проверки каждому правилу, используемому для автоматического создания заказов контроля качества на странице **Сопоставления контроля качества**. Также назначается группа проверки для созданных вручную заказов контроля качества.

## <a name="create-a-test-group"></a>Создание проверочной группы

Чтобы создать группу проверки, выполните следующие действия.

1. Перейдите **Управление запасами \> Настройка \> Контроль качества \> Группы проверки**.
1. В области действий выберите **Создать**, чтобы добавить строку в сетку в верхней части страницы. Затем задайте следующие поля для новой строки:

    - **Группа проверки** — введите уникальный код или имя группы проверки.
    - **Описание** — введите подробное описание группы проверки.
    - **Допустимый уровень качества** — введите суммарный процент результатов проверок, который должен считаться успешно пройденным для группы проверки.
    - **Выборка номенклатуры** — выберите выборку номенклатуры. Дополнительные сведения см. в разделе [Выборка номенклатур управления качеством](quality-item-sampling.md).
    - **Разрушительный** — установите этот флажок, чтобы указать, что группа проверки уничтожит проверяемые номенклатуры.

1. Когда новая строка все еще выбрана, выберите вкладку **Общее** в верхней части страницы. Здесь повторяются некоторые настройки для группы проверки, выбранной на вкладке **Обзор**. Кроме того, можно задать следующие поля для группы:

    - **Обновить атрибут партии запасов** — при выборе значения *Да* для каждой новой проверки, добавляемой в группу проверки в нижней части страницы, будет установлено значение *Да*.
    - **Обновить расстановку партии** — когда для этого параметра устанавливается значение *Да*, если проверяемая номенклатура является контролируемой по партии, расстановка партии автоматически обновляется на значение, выбранное в поле **Расстановка партии завершившегося неудачно заказа на контроль качества** или **Расстановка партии завершившегося удачно заказа на контроль качества**.
    - **Расстановка партии завершившегося неудачно заказа на контроль качества** — выберите код расстановки партии, который должен быть назначен, когда заказ контроля качества, включающий данную группу проверки, заканчивается неудачей, поскольку он не соответствует AOL.
    - **Расстановка партии завершившегося удачно заказа на контроль качества** — выберите код расстановки партии, который должен быть назначен, когда заказ контроля качества, включающий данную группу проверки, заканчивается удачно, поскольку он соответствует AOL.
    - **Обновить статус запасов** — при выборе значения *Да*, если **Статус запасов** включен в группе аналитик хранения для проверяемой номенклатуры, статус будет автоматически обновляться на статус, выбранный в поле **Статус завершившегося неудачно заказа на контроль качества** или **Статус завершившегося удачно заказа на контроль качества**.
    - **Статус завершившегося неудачно заказа на контроль качества** — выберите статус запасов, который должен быть назначен номенклатуре, когда заказ контроля качества, включающий данную группу проверки, заканчивается неудачей, поскольку он не соответствует AOL.
    - **Статус завершившегося удачно заказа на контроль качества** — выберите статус запасов, который должен быть назначен номенклатуре, когда заказ контроля качества, включающий данную группу проверки, заканчивается удачно, поскольку он соответствует AOL.

## <a name="add-a-qualitative-test-to-a-test-group"></a>Добавление качественной проверки в группу проверки

Чтобы добавить качественную проверку в группу проверки, выполните следующие действия.

1. Перейдите **Управление запасами \> Настройка \> Контроль качества \> Группы проверки**.
1. На вкладке **Обзор** в верхней части страницы выберите группу проверки, в которую необходимо добавить проверку.
1. В нижней части страницы на вкладке **Обзор** выберите **Добавить** на панели инструментов, чтобы добавить строку в сетку. Затем задайте следующие поля для новой строки:

    - **Порядковый номер** — введите целое число, определяющее порядок, в котором должны выполняться проверки. Проверки, для которых указаны меньшие порядковые номера, будут выполняться перед проверками с большими порядковыми номерами.

        > [!NOTE]
        > Рекомендуется оставлять промежуток между порядковыми номерами. Например, установите для поля **Порядковый номер** *10* для первой проверки, а затем увеличивайте значение на 10 для каждой дополнительной проверки. Таким образом можно добавить новые проверки позже без удаления и повторного создания строк, чтобы организовать правильный порядок.

    - **Проверка** — выберите проверку, которое необходимо выполнить. Для качественной проверки выберите проверку, в которой поле **Тип** задано как *Параметр*.
    - **Действует с** — выберите первую дату, когда проверка действительна. Если оставить это поле пустым, ограничения нет.
    - **Истечение срока** — выберите последнюю дату, когда проверка действительна. Если оставить это поле пустым или задать для него значение *Никогда*, ограничение отсутствует.
    - **Определение значения для проверки** — выберите способ определения AOL при записи нескольких результатов для одной и той же проверки. Для качественной проверки можно выбрать только *Вручную*. При выборе одного из других значений (*Среднее*, *Минимальное* или *Максимальное*) при сохранении группы проверки будет выведено сообщение об ошибке.
    - **Атрибут** — если требуется записать результаты проверки для атрибута партии, выберите здесь атрибут и установите флажок **Обновить атрибут партии запасов**.
    - **Обновить атрибут партии запасов** — установите этот флажок для записи результатов проверки для атрибута партии, который выбран в поле **Атрибут**.

1. В нижней части страницы выберите вкладку **Общее**. Здесь повторяются некоторые параметры проверки, которые выбраны на вкладке **Обзор**. Кроме того, можно задать следующие поля для проверки:

    - **Отчет по сертификату анализа** — установите этот параметр как *Да*, чтобы указать, что результаты проверок должны быть распечатаны в сертификате анализа (CoA). Этот отчет может быть создан из заказа контроля качества.
    - **Операция при отказе** — выберите *Принять* или *Ошибка*, чтобы указать, должна ли проверка считаться успешно пройденной, если для нее не соблюдается AOL.
    - **Допустимый уровень качества** — введите суммарный процент результатов проверок, который должен считаться успешно пройденным для проверки.

1. В нижней части страницы на вкладке **Проверка** задайте следующие поля:

    - **Переменная** — выберите переменную проверки для записи результатов качественной проверки.
    - **Результат по умолчанию** — выберите результат по умолчанию, который должен быть записан для результатов проверки.
    - **Инструмент проверки** — выберите устройство, которое должно использоваться для проверки. Если инструмент проверки задан, он автоматически вводится для проверки в группе проверки.

1. Закройте страницу.

## <a name="add-a-quantitative-test-to-a-test-group"></a>Добавление количественной проверки в группу проверки

Чтобы добавить количественную проверку в группу проверки, выполните следующие действия.

1. Перейдите **Управление запасами \> Настройка \> Контроль качества \> Группы проверки**.
1. На вкладке **Обзор** в верхней части страницы выберите группу проверки, в которую необходимо добавить проверку.
1. В нижней части страницы на вкладке **Обзор** выберите **Добавить** на панели инструментов, чтобы добавить строку в сетку. Затем задайте следующие поля для новой строки:

    - **Порядковый номер** — введите целое число, определяющее порядок, в котором должны выполняться проверки. Проверки, для которых указаны меньшие порядковые номера, будут выполняться перед проверками с большими порядковыми номерами.

        > [!NOTE]
        > Рекомендуется оставлять промежуток между порядковыми номерами. Например, установите для поля **Порядковый номер** *10* для первой проверки, а затем увеличивайте значение на 10 для каждой дополнительной проверки. Таким образом можно добавить новые проверки позже без удаления и повторного создания строк, чтобы организовать правильный порядок.

    - **Проверка** — выберите проверку, которое необходимо выполнить. Для количественной проверки выберите проверку, в которой поле **Тип** задано как *Дробь* или *Целое число*.
    - **Действует с** — выберите первую дату, когда проверка действительна. Если оставить это поле пустым, ограничения нет.
    - **Истечение срока** — выберите последнюю дату, когда проверка действительна. Если оставить это поле пустым или задать для него значение *Никогда*, ограничение отсутствует.
    - **Определение значения для проверки** — выберите способ определения AOL при записи нескольких результатов для одной и той же проверки. Доступны варианты *Среднее*, *Минимум*, *Максимум* и *Вручную*.
    - **Атрибут** — если требуется записать результаты проверки для атрибута партии, выберите здесь атрибут и установите флажок **Обновить атрибут партии запасов**.
    - **Обновить атрибут партии запасов** — установите этот флажок для записи результатов проверки для атрибута партии, который выбран в поле **Атрибут**.

1. В нижней части страницы выберите вкладку **Общее**. Здесь повторяются некоторые параметры проверки, которые выбраны на вкладке **Обзор**. Кроме того, можно задать следующие поля для проверки:

    - **Отчет по сертификату анализа** — установите этот параметр как *Да*, чтобы указать, что результаты проверок должны быть распечатаны в CoA. Этот отчет может быть создан из заказа контроля качества.
    - **Операция при отказе** — выберите *Принять* или *Ошибка*, чтобы указать, должна ли проверка считаться успешно пройденной, если для нее не соблюдается AOL.
    - **Допустимый уровень качества** — введите суммарный процент результатов проверок, который должен считаться успешно пройденным для проверки.

1. В нижней части страницы на вкладке **Проверка** задайте следующие поля:

    - **Стандарт** — введите сумму (целое число или дробь), ожидаемую для результатов проверки. Вводимое вами значение будет введено по умолчанию в результаты проверки. Кроме того, оно будет автоматически введено в качестве значения по умолчанию в полях **Мин** и **Макс**.
    - **Мин** — введите минимально допустимое значение для результатов проверки. Введенное значение должно быть меньше суммы, которая задана в стандартном поле **Стандарт**. Когда обновляется поле **Мин**, поле **Макс. допуск (%)** обновляется автоматически. Аналогично, когда обновляется поле **Мин. допуск %**, поле **Мин** обновляется автоматически.
    - **Макс** — введите максимально допустимое значение для результатов проверки. Введенное значение должно быть больше суммы, которая задана в стандартном поле **Стандарт**. Когда обновляется поле **Макс**, поле **Макс. допуск (%)** обновляется автоматически. Аналогично, когда обновляется поле **Макс. допуск %**, поле **Макс** обновляется автоматически.
    - **Инструмент проверки** — выберите устройство, которое должно использоваться для проверки. Если инструмент проверки задан, он автоматически вводится для проверки в группе проверки.

1. Закройте страницу.
