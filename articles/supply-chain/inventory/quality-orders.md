# <a name="quality-orders"></a>Заказы контроля качества

[!include [banner](../includes/banner.md)]

В этой теме описывается, как вручную или автоматически создавать заказы контроля качества и как работать с ними для проведения проверок и записи результатов проверок в Microsoft Dynamics 365 Supply Chain Management.

## <a name="automatically-created-quality-orders"></a>Автоматически созданные заказы контроля качества

Можно настроить систему таким образом, чтобы она автоматически создавала заказы контроля качества на основе правил выборки номенклатуры. Дополнительные сведения см. в разделе [Выборка номенклатур управления качеством](quality-item-sampling.md).

## <a name="manually-create-quality-orders"></a><a name="manual-quality-orders"></a>Создание заказов контроля качества вручную

Чтобы вручную создать заказ контроля качества, выполните следующие действия.

1. Перейдите в раздел **Управление запасами \> Периодические задачи \> Управление качеством \> Заказы для контроля качества**.
1. Выберите **Создать**.
1. В диалоговом окне **Заказы контроля качества** в поле **Тип ссылки** выберите ссылку на запасы, с которой будет связан заказ контроля качества. Описание типов ссылок, доступных для выбора, см. в разделе [Типы ссылок заказа контроля качества](#ref-types) далее в этой теме.

    > [!NOTE]
    > Запасы, которые относятся к выбранной ссылке, должны быть доступны. Если нет доступных запасов для комбинации типа ссылки, количества и складских аналитик, которые выбраны, будет выведено сообщение об ошибке.

1. Выполните одно из следующих действий в зависимости от значения, выбранного в поле **Тип ссылки**.

    - Если выбрано **Запасы**, дополнительные сведения о ссылке не требуются.
    - Если выбрано **Продажи** или **Покупка**, укажите следующие сведения:

        - **Выбор счета** — ссылка на номер клиента или поставщика.
        - **Номер ссылки** — ссылка на номер заказа на продажу или заказа на покупку.
        - **Ссылки на лот**— ссылка на конкретную строку заказа на продажу или заказа на покупку.

    - Если выбрано **Производство**, **Карантин** или **Производство сопутствующих продуктов**, в поле **Номер ссылки** следует ссылаться на номер производственного заказа, заказа партии или заказа карантина.
    - Если выбрано **Операция маршрута**, укажите следующие сведения:

        - **Номер ссылки** — ссылка на номер производственного заказа или заказа партии.
        - **№ опер.** — Ссылка на конкретный номер операции маршрута.
        - **Операция** — ссылка на конкретную операцию маршрута.
        - **Ресурс** — ссылка на конкретный ресурс, который должен использоваться с операцией маршрута.

1. В поле **Группа проверки** выберите группу выполняемых проверок.
1. В поле **Количество** введите проверяемое количество номенклатур.
1. В разделе **Складские аналитики** введите любые дополнительные складские аналитики, которые необходимы для выбранной номенклатуры.
1. Нажмите **ОК**.

После создания заказа контроля качества можно начать проверку запасов и записывать результатов проверок. Дополнительные сведения о записи и проверке результатов проверок см. в разделе [Проверка качества товаров](tasks/inspect-quality-goods.md).

## <a name="quality-order-reference-types"></a><a name="ref-types"></a>Типы ссылок заказа контроля качества

Заказы контроля качества используются для отслеживания сведений о проверках и результатах проверок для конкретной номенклатуры на складе. Заказ контроля качества должен включать ссылку, представляющую источник проверяемых запасов. Заказы контроля качества могут быть связаны со следующими типами ссылок:

- **Запасы** — этот тип ссылки указывает на то, что выполняется проверка запасов в наличии. Проверки этого типа обычно являются случайными или незапланированными и выполняются, когда работник склада ищет проблему.
- **Продажи** — этот тип ссылки указывает на то, что выполняется проверка запасов, связанных с данным заказом на продажу. Проверки этого типа обычно связаны с спецификациями клиентов или с созданием сертификата анализа (CoA), который должен быть предоставлен клиенту в рамках отгрузки. (Сертификат CoA иногда называют сертификатом соответствия \[CoC\].)
- **Покупка** — этот тип ссылки указывает на то, что выполняется проверка запасов, связанных с данным заказом на покупку. Проверки этого типа часто используются для проверки входящих товаров перед их размещением в запасах или для потребления в производственных процессах.
- **Производство** — этот тип ссылки указывает на то, что выполняется проверка запасов, связанных с производственным заказом. Проверки этого типа часто используются для проверки готовых товаров перед их размещением в запасах.
- **Карантин** — этот тип ссылки указывает на то, что выполняется проверка запасов, связанных с карантинным заказом. Карантинные заказы представляют собой особый тип заказа, который позволяет отслеживать запасы на отдельном складе или в отдельной области на складе. Проверки этого типа часто используются для проверки товаров, возвращенных клиентом или помещенных в карантин для дальнейшего анализа. Карантинные заказы могут создаваться из заказов контроля качества. Кроме того, они могут быть созданы из других источников, а затем заказы контроля качества могут быть связаны с карантинными заказами.
- **Операция маршрута** — этот тип ссылки указывает на то, что производится проверка запасов, относящаяся к конкретному шагу маршрута для производственного заказа. Проверки этого типа обычно используются для анализа незавершенного производства (НЗП) продукта перед переходом к следующему этапу производственного процесса.
- **Производство сопутствующих продуктов** — этот тип ссылки указывает на то, что выполняется проверка запасов, связанных с сопутствующим продуктом производственного заказа. Проверки этого типа обычно используются для проверки сопутствующего продукта заказа партии до того, как сопутствующий продукт добавляется в запасы.

## <a name="view-and-create-quality-orders-from-various-parts-of-the-system"></a>Просмотр и создание заказов контроля качества из различных частей системы

Заказы контроля качества могут быть созданы вручную. Кроме того, они могут быть созданы автоматически на основе определяемых сопоставлений контроля качества. Дополнительные сведения о создании и управлении автоматическим созданием заказов контроля качества см. в разделе [Сопоставления контроля качества](quality-associations.md).

Можно использовать страницу заказа контроля качества, чтобы вручную создать новый заказ контроля качества или просмотреть статус заказа контроля качества, который связан с другой записью. Есть несколько способов доступа к странице заказа контроля качества.

### <a name="from-the-quality-orders-page"></a>Со страницы заказов контроля качества

Чтобы вручную создать заказы контроля качества и просмотреть все существующие заказы контроля качества, перейдите в раздел **Управление запасами \> Периодические задачи \> Управление качеством \> Заказы контроля качества**. В остальных разделах этой главы приводятся сведения о том, как работать со страницей **Заказы контроля качества**.

### <a name="from-sales-orders"></a>Из заказов на продажу

Для работы с заказами контроля качества, которые относятся к заказам на продажу, перейдите **Продажи и маркетинг \> Все заказы на продажу \> Все заказы на продажу**, а затем выполните любое из следующих действий:

- Откройте заказ на продажу или выберите его в сетке. Затем на панели операций на вкладке **Комплектация и упаковка** в группе **Управление качеством** выберите **Заказы контроля качества**, чтобы открыть страницу **Заказы контроля качества**. Здесь можно просмотреть, создать и обновить заказы контроля качества, которые относятся к данному заказу на продажу.
- Откройте заказ на продажу, а затем на вкладке **Заголовок** выберите экспресс-вкладку **Общее**. В поле **Статус заказа контроля качества** отображается общий статус всех заказов контроля качества, соответствующих данному заказу на продажу.
- Откройте заказ на продажу, а затем на вкладке **Строки** выберите экспресс-вкладку **Строки заказа на продажу**. В столбце **Статус заказа контроля качества** отображается статус каждой строки заказа на продажу.

### <a name="from-purchase-orders"></a>Из заказов на покупку

Для работы с заказами контроля качества, которые относятся к заказам на покупку, перейдите **Закупки и источники \> Заказы на покупку \> Все заказы на покупку**, а затем выполните любое из следующих действий:

- Откройте заказ на покупку или выберите его в сетке. Затем на панели операций на вкладке **Получение** в группе **Управление качеством** выберите **Заказы контроля качества**, чтобы открыть страницу **Заказы контроля качества**. Здесь можно просмотреть, создать и обновить заказы контроля качества, которые относятся к данному заказу на покупку.
- Откройте заказ на покупку, а затем на вкладке **Заголовок** выберите экспресс-вкладку **Общее**. В поле **Статус заказа контроля качества** отображается общий статус всех заказов контроля качества, соответствующих данному заказу на покупку.
- Откройте заказ на покупку, а затем на вкладке **Строки** выберите экспресс-вкладку **Строки заказа на покупку**. В столбце **Статус заказа контроля качества** отображается статус каждой строки заказа на покупку.

### <a name="from-production-orders"></a>Из производственных заказов

Для работы с заказами контроля качества, которые относятся к производственным заказам, перейдите **Управление производством \> Производственные заказы \> Все производственные заказы**, а затем выполните любое из следующих действий:

- Откройте производственный заказ или выберите его в сетке. Затем на панели операций на вкладке **Вид** в группе **Управление качеством** выберите **Заказы контроля качества**, чтобы открыть страницу **Заказы контроля качества**. Здесь можно просмотреть, создать и обновить заказы контроля качества, которые относятся к данному производственному заказу.
- Откройте производственный заказ или выберите его в сетке. Затем в области действий на вкладке **Производственный заказ** в группе **Производственные сведения** выберите **Маршрут**, чтобы открыть страницу **Маршрут производства**. Чтобы просмотреть заказы контроля качества, которые связаны с операцией маршрута, выполните одно из следующих действий:

    - Выберите операцию целевого маршрута в сетке. Затем в области действий выберите **Запросы \> Заказы контроля качества**.
    - Выберите значение в поле **№ операции** для целевой операции маршрута, чтобы открыть страницу сведений **Маршрут производства**. На экспресс-вкладке **Общее** в поле **Статус заказа контроля качества** отображается статус заказов контроля качества, которые связаны с операцией маршрута.

- Откройте производственный заказ и выберите экспресс-вкладку **Общее**. В поле **Статус заказа контроля качества** отображается статус заказов контроля качества, которые связаны с производственным заказом.

### <a name="from-quarantine-orders"></a>Из карантинных заказов

Для работы с заказами контроля качества, которые относятся к карантинным заказам, перейдите **Управление запасами \> Периодические задачи \> Управление качеством \> Карантинные заказы**, а затем выполните любое из следующих действий:

- Проверьте значения в столбце **Статус заказа контроля качества**. Таким образом можно узнать общий статус всех заказов контроля качества, которые связаны с каждым карантинным заказом в сетке.
- Выберите карантинный заказ в сетке, а затем в области действий выберите **Заказы контроля качества**, чтобы просмотреть, создать или обновить заказы контроля качества, которые относятся к карантинному заказу.

## <a name="advanced-actions-for-quality-orders"></a>Расширенные действия по заказам контроля качества

Для управления статусом, созданием документов и запросами дополнительных сведений можно выполнить несколько действий в заказах контроля качества.

### <a name="reopen-a-quality-order"></a>Повторное открытие заказа контроля качества

По умолчанию больше нельзя редактировать или обновлять заказ контроля качества после его проверки и завершения тестов. Однако в некоторых случаях может потребоваться повторно открыть заказ контроля качества после его завершения.

Чтобы повторно открыть заказ контроля качества, выполните следующие действия.

1. Перейдите в раздел **Управление запасами \> Периодические задачи \> Управление качеством \> Заказы для контроля качества**.
1. Откройте проверенный заказ контроля качества или выберите его в сетке.
1. На панели операций выберите **Повторно открыть заказ контроля качества**.

### <a name="create-a-certificate-of-analysis-for-a-quality-order"></a>Создание сертификата анализа для заказа контроля качества

CoA — это отчет, который создается группой обеспечения качества в организации. Он проверяет соответствие продукта определенным правилам или требованиям. Требования клиентов или нормативные требования в вашем географическом регионе могут требовать CoA. Они также могут потребоваться в зависимости от отрасли и типа продуктов, которые вы обрабатываете, закупок, производства или продаж.

Supply Chain Management позволяет создать CoA из заказа контроля качества. Отчет будет содержать результаты любых проверок в заказе контроля качества, для которых параметр **Отчет для сертификата анализа** задан как *Да*. Этот параметр может быть установлен по умолчанию на основе проверки, определенной на странице **Проверки**. Однако можно переопределить этот параметр для определенных проверок для конкретного заказа контроля качества.

Чтобы создать CoA для заказа контроля качества, выполните следующие действия.

1. Перейдите в раздел **Управление запасами \> Периодические задачи \> Управление качеством \> Заказы для контроля качества**.
1. Выберите заказ контроля качества, для которого необходимо создать CoA.
1. На панели операций выберите **Запросы \> Сертификат анализа**.
1. На странице **Сертификат анализа** на панели операций выберите **Создать**.
1. Необязательно: в поле **Контакт** выберите контактное лицо, которому должен быть направлен сертификат.
1. В области действий выберите **Печать**.
1. В диалоговом окне **Сертификат анализа** определите, как должен печататься отчет. Затем нажмите **ОК**, чтобы распечатать отчет на принтере, на экране, в файл или в электронную почту.
1. Закройте страницы.

### <a name="block-or-unblock-inventory-for-a-quality-order"></a>Блокирование и разблокирование запасов для заказа контроля качества

Когда заказ контроля качества создается автоматически на основе сопоставления контроля качества, выборку номенклатуры, назначенной данному сопоставлению контроля качества, можно настроить так, чтобы она блокировала полное количество запасов для проверяемой ссылки. Для получения дополнительных сведений о выборках номенклатур см. раздел [Выборка номенклатуры управления качеством](quality-item-sampling.md).

Если не используется полная блокировка или вручную создается заказ контроля качества, система автоматически создает запись блокировки запасов для количества номенклатуры, которое проверяется в заказе контроля качества. В записи, которая создается на странице **Блокировка запасов** поле **Тип блокировки запасов** задается как *Заказ контроля качества*.

Для просмотра и изменения блокировки запасов для заказа контроля качества, выбранного на странице **Блокировка запасов**, выберите **Запросы \> Блокировка запасов** на панели операций. Для получения дополнительной информации о см. раздел [Блокировка запасов](inventory-blocking.md).

### <a name="inquire-about-the-details-of-a-quality-order"></a>Запрос сведений о заказе контроля качества

Используйте следующие кнопки на панели операций на странице **Заказы контроля качества** для просмотра дополнительной информации о заказе контроля качества или связанных с ним сведений:

- **Запросы \> Сведения о работе** — открывается страница, на которой можно просматривать работу склада, связанную с этим заказом контроля качества.
- **Запросы \> Несоответствия** — открывается страница, на которой можно просмотреть любые несоответствия, связанные с этим заказом контроля качества.
- **Запасы** — команды в этом меню являются общими для всех проводок по складу. Их можно использовать для просмотра или обновления сведений о проводках, запасах в наличии, резервированиях и маркировке.

### <a name="create-cases-related-to-quality-orders"></a>Создание обращений, связанных с заказами контроля качества

Можно создавать обращения, которые относятся к заказам контроля качества. Таким образом можно отследить сведения о проблемах и работать над их разрешением. Затем можно использовать функции workflow-процесса управления обращениями для маршрутизации обращения с помощью предопределенного бизнес-процесса для получения дополнительных утверждений или получения дополнительных сведений по конкретном вопросу. Кроме того, можно использовать функцию статей базы знаний для создания базы знаний с целью решения наиболее распространенных проблем.

## <a name="case-management-examples-for-quality-management"></a>Примеры управления обращениями для управления качеством

### <a name="example-1"></a>Пример 1

Вы работаете для производственной компании, которая должна следовать требованиям строгого законодательства, которые связаны с производством поднадзорных продуктов, таких как продукты питания. Заказы контроля качества используются для записи и отслеживания подробных сведений о качестве номенклатур в ходе производственного процесса. Если заказ контроля качества не проходит проверку, возможно, проблема связана с оборудованием. Обращения используются для отслеживания бизнес-процесса и эскалации проблемы правильным инженерам, чтобы можно было определить основную причину. Для облегчения работы с бизнес-процессами компания ведет базу знаний общих вопросов, связанных с заказами контроля качества и проблемами с оборудованием.

### <a name="example-2"></a>Пример 2

Вы работаете для компании-дистрибьютора, которая поставляет продукты, которые могут быть скорректированы для различных стран и регионов. У некоторых клиентов есть строгие спецификации, которые необходимо соблюдать. В противном случае могут быть выставлены претензии и возвраты или возвратные платежи. Заказы контроля качества используются для отслеживания сведений о каждой проверке и результатах, соответствующих требованиям клиента. Обращения используются для проверки и утверждения сведений о CoA перед созданием документа и его вложением с другими документами по отгрузке.
