# <a name="quality-management-for-warehouse-processes"></a>Управление качеством для складских процессов

Функция _Управление качеством для складских процессов_ позволяет интегрировать элементы управления выборкой номенклатуры в процесс складской приемки с помощью расширенного управления складом. Работа склада может быть создана автоматически для перемещения запасов в местоположение контроля качества на основе процентного значения или фиксированного количества, либо на основе каждого *n*-го грузоместа. После того как заказ контроля качества завершен, работа может быть создана автоматически для перемещения запасов в следующее местоположение в процессе, в зависимости от результатов качества.

Функция _Управление качеством для складских процессов_ расширяет возможности функции базового управления качеством. Она предоставляет возможность создавать заказы контроля качества для запасов, которые отправляются в местоположение контроля качества, хотя заказы контроля качества не всегда требуются. Таким образом, она позволяет использовать упрощенный процесс контроля качеством на основе работы склада.

## <a name="turn-on-the-quality-management-for-warehouse-processes-feature"></a>Включение функции управления качеством для складских процессов

Прежде чем использовать эту функцию, она должна быть включена в системе. Администраторы могут использовать параметры [управления компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление складом*
- **Название функции:** *Управление качеством для складских процессов*

## <a name="key-benefits"></a>Ключевые преимущества

Функция _Управления качеством для складских процессов_ автоматически создает работу как часть процесса приемки, чтобы переместить количество запасов, необходимое для контроля качества, в местоположение управления качеством. Если полученное количество превышает количество, необходимое для контроля качества (в соответствии с настройкой выборки номенклатур), излишнее количество перемещается во входящее местоположение, определенное в настройке директивы местоположения. После проверки заказа контроля качества, работа создается автоматически для перемещения количества для заказа контроля качества в новое входящее местоположение или местоположение возврата на основе результатов проверки и настройки директивы местоположения. Автоматическое создание работы, имеющее только количество, которое необходимо переместить в контроль качества или из него, обеспечивает интегрированную работу процесса.

> [!NOTE]
> Когда функция _Управления качеством для складских процессов_ включена, все равно можно воспользоваться преимуществом ручного процесса. В ручном процессе перемещение запасов и перемещение по шаблону используются для того, чтобы рабочий склада запустил создание складских работ для перемещения запасов из местоположения контроля качества в новое местоположение. Имеется также возможность настроить директиву входящего местоположения, которая переносит запасы полностью с местоположения приемки в местоположение контроля качества без учета настройки выборки номенклатуры.

## <a name="enablers-the-quality-item-sampling-and-quality-order-work-order-types"></a>Средства включения: типы заказов на работу "Выборка номенклатуры для контроля качества" и "Контроль качества"

Функция _Управление качеством для складских процессов_ вводит два типа заказов на работу, которые позволяют выполнять процесс создания работы:

- **Выборка номенклатуры для контроля качества** — этот тип заказа на работу используется для создания работы, которая перемещает зарегистрированные запасы для контроля качества.
- **Заказ контроля качества** — этот тип заказа на работу используется для создания работы, которая перемещает запасы из контроля качества в новое местоположение на основе настройки директивы местоположения.

### <a name="work-classes-location-directives-and-work-templates"></a>Классы работы, директивы местоположения и шаблоны работ

Типы заказов на работу _Выборка номенклатуры для контроля качества_ и _Заказ для контроля качества_ используются в директивах местоположения, классах работ и шаблонах работ.

Перед тем как работа склада может быть создана автоматически для перемещения запасов для управления качеством, необходимо выполнить следующие действия для настройки системы.

1. Создайте отдельные классы работ для типов заказов на работу _Выборка номенклатуры для контроля качества_ и _Заказ для контроля качества_. Таким образом можно гарантировать, что соответствующие работы могут быть автоматически созданы на основе двух типов заказов на работу, и эта работа может быть затем запущена с помощью приложения склада.
1. Настройте шаблон работы для каждого типа заказа на работу:

    - Настройте шаблон работы, который использует тип заказа на работу _Выборка номенклатур для контроля качества_ для автоматического перемещения зарегистрированных запасов в местоположение контроля качества.
    - Настройте шаблон работы, который использует тип заказа на работу _заказ контроля качества_ для перемещения запасов из местоположения контроля качества после завершения контроля качества.

1. Для каждого типа заказа на работу настройте директивы местоположения, которые применяются к правильным местоположениям контроля качества, в которые должны быть перемещены запасы. После завершения контроля качества директива местоположения для типа заказа на работу _Заказ для контроля качества_ позволяет гарантировать, что будет выбрано новое местоположение назначения, чтобы запасы можно было переместить из местоположения контроля качества.
1. Настройте соответствующие пункты меню мобильного устройства для поддержки перемещения полученных запасов в местоположение контроля качества и перемещения запасов, которые прошли или не прошли контроль качества из местоположения контроля качества в новое местоположение.

Пошаговый пример, в котором показано, как выполнить настройку, см. в [примере сценария](#example-scenario) в конце этого раздела.

## <a name="enable-a-warehouse-for-quality-management"></a>Включение склада для управление качеством

Прежде чем функция _Управление качеством для складских процессов_ может быть применена к конкретному складу, необходимо выполнить следующие действия, чтобы сделать функцию доступной для данного склада.

1. Перейдите в раздел **Управление складом \> Настройка \> Склад \> Склады**.
1. Выберите склад для включения управление качеством.
1. На экспресс-вкладке **Склад** установите для параметра **Включить заказ для контроля качества для складских процессов** значение _Да_. (Обратите внимание, что для этого параметра можно установить значение _Да_ только для складов, использующих процессы управления складом.)

Когда для параметра **Включить заказ для контроля качества для складских процессов** установлено значение _Да_, настройка сопоставления качества определяет, будет ли фактически применяться функция _Управление качеством для складских процессов_ для выбранного склада. В любое время можно изменить значение этого параметра на _Нет_. В этом случае функция больше не будет применяться к складу, независимо от настройки сопоставления качества.

## <a name="quality-control"></a>Контроль качества

Функция _Управление качеством для складских процессов_ управляет несколькими ключевыми настройками для сопоставлений контроля качества и выборки номенклатуры.

### <a name="quality-associations"></a>Сопоставления контроля качества

Каждая [запись сопоставления качества](enable-quality-management.md) определяет также набор тестов, приемлемый уровень качества (AQL) и план выборочного контроля, применимые для создаваемых автоматически заказов для контроля качества. Чтобы настроить запись сопоставления контроля качества, выполните следующие действия.

1. Перейдите в раздел **Управление запасами \> Настройка \> Контроль качества \> Сопоставления контроля качества**.
1. Создайте или выберите запись сопоставления контроля качества для номенклатуры или группы, с которой вы работаете, или для всех номенклатур.
1. На экспресс-вкладке **Условия** в поле **Применимый тип склада** задайте одно из следующих значений:

    - **Управление качеством только для складских процессов** — активирует функцию _Управление качеством для складских процессов_. Это значение можно выбрать только в том случае, если типом ссылки является *Покупка* или *Производство*.
    - **Все** — выключение функции _управления качеством для складских процессов_. Выберите это значение для всех типов ссылок, за исключением *Покупка* и *Производство*.

> [!NOTE]
> Функция _Управления качеством для складских процессов_ применяется только в том случае, если номенклатура в строке документа-источника использует расширенные процессы управления складом , и если для склада в строке документа-источника для параметра **Включить заказ для контроля качества для складских процессов** установлено значение _Да_.

По мере регистрации каждой номенклатуры (или ее приемки) система определяет, какие сопоставления контроля качества применяются к ней.

Когда функция _Управления качеством для складских процессов_ включена, соответствующий тип склада логически вставляется в четвертую группу поиска в иерархии поиска сопоставления контроля качества. В следующей таблице представлено логическое представление иерархии поиска.

| Группа поиска | описание |
|---|---|
| Группа 1 | Для каждого сопоставления контроля качества проверьте значения **Ссылочный тип**, **Тип события** и **Соответствие выполнения** для номенклатуры. Если имеется соответствие по строке документа-источника, перейдите в группу 2. |
| Группа 2 | Для каждого сопоставления контроля качества проверьте значение **Код номенклатуры** (_Таблица_, _Группа_ или _Все_) относительно номенклатуры. _Таблица_ является более специфической ,чем _Группа_, и _Группа_ является более специфической, чем _Все_. Если имеется соответствие для _Таблица_ (определенная номенклатура), перейдите в группу 3. Если нет соответствия для _Таблица_, выполните поиск совпадения для _Группа_. Если нет соответствия для _Группа_, применяется _Все_. Если имеется соответствие, перейдите в группу 3. |
| Группа 3 | Для каждого сопоставления контроля качества проверьте значения **Код счета** и **Код ресурса** по отношению к номенклатуре. Применяемая логика напоминает логику, которая применяется для значения **Код номенклатуры**. |
| Группа 4 | Для каждого сопоставления контроля качества проверьте значение **Применимый тип склада** (_Управление качеством только для складских процессов_ или _Все_) относительно номенклатуры. Если для параметра **Включить заказ для контроля качества для складских процессов** задано значение _Да_ для склада в исходном документе, а номенклатура в строке исходного документа настроена для _Использовать процессы управления складом_, то обе связи, где существует соответствие для _Управления качеством только для складских процессов_ и связи, для которых имеется соответствие для значения _Все_, будут применяться параллельно, если они имеются. Если для параметра **Включить заказ для контроля качества для складских процессов** установлено значение _Нет_ для склада в строке документа-источника, и для номенклатуры в строке документа-источника задано значение _Использовать процессы управления складом_, будет применяться только управление качеством. |

Например, был определен склад, в котором параметр **Включить заказ для контроля качества для складских процессов** имеет значение _Да_, и имеется два сопоставления контроля качества, которые определены для типа ссылки *Покупка*: один для всех номенклатур, а другой для типа события *Регистрация*. Единственным различием между двумя сопоставлениями контроля качества является значение параметра **Применимый тип склада**: оно устанавливается на _Все_ для одного сопоставления качества и на _Управление качеством только для складских процессов_ для другого. В этом случае оба сопоставления качества являются одинаково специфичными и оба будут применимы.

Значение поля **Группа проверки** для сопоставлений контроля качества также является фактором. Это поле определяет процедуру проверки, которая должна быть применена. Если значение **Группа проверки** одинаково для обоих ассоциаций, будет создан только один заказ для контроля качества, для сопоставления контроля качества, в котором значение **Применимый тип склада** равно _Управление качеством только для складских процессов_. Если значение **Группа проверки** для обеих связей не совпадает, будут созданы два заказа для контроля качества. Первый заказ для контроля качества будет создан для сопоставления контроля качества, в котором значение **Применимый тип склада** равно _Управление качеством только для складских процессов_. Второй заказ для контроля качества будет создан для сопоставления контроля качества, в котором значение **Применимый тип склада** равно _Все_.

> [!NOTE]
> Значение _Управление качеством только для складских процессов_ считается более специфическим, чем _Все_, когда критерии для сопоставлений контроля качества для групп 1 и 2 одинаковы, и если группа проверок совпадает. Два заказа для контроля качества будут созданы только в том случае, если группы проверки отличаются друг от друга.

#### <a name="reference-types"></a>Типы ссылок

Если значение **Тип ссылки** равно _Покупка_, а значение **Применимый тип склада** равно _Управление качеством только для складских процессов_, в поле **Тип события** на экспресс-вкладке **Процесс** должно быть задано значение _Регистрация_. _Регистрация_ — это единственный поддерживаемый тип события для типа ссылки _Покупка_ при использовании функции _Управление качеством для складских процессов_.

#### <a name="quality-processing-policy"></a>Политика обработки качества

Функция _Управления качеством для складских процессов_ позволяет создавать работу на основе только выборки номенклатуры. Таким образом, она позволяет использовать упрощенный процесс. Запасы, для которых создается работа, зависят от выборки номенклатуры, связанной с сопоставлением контроля качества. Когда используется облегченный процесс, после того как работник поместит количество в местоположение контроля качества, подразделение контроля качества можно вручную создать заказ для контроля качества, если заказ для контроля качества является обязательным.

Поле **Политика обработки контроля качества** на экспресс-вкладке **Процесса заказов для контроля качества** определяет, будет ли заказ для контроля качества также создан во время создания работы для перемещения номенклатуры в местоположение контроля качества. В этом поле может быть задано значение _Создание заказа для контроля качества_ или _Создание только работы_. Значение по умолчанию: _Создание заказа для контроля качества_.

> [!NOTE]
> Независимо от того, созданы ли заказы для контроля качества вручную или автоматически, система автоматически создает работу, чтобы переместить номенклатуры из местоположения контроля качества, когда заказ для контроля качества помечен как проверенный.

Создание работы для контроля качества не связано с настройкой сопоставления контроля качества. Если существует шаблон работы, в котором параметр **Тип заказа на работу** имеет значение *Заказ для контроля качества*, и если критерии запроса выполняются для этого шаблона работы, то проверка заказа для контроля качества запускает создание заказа для контроля качества.

#### <a name="referenced-item-sampling"></a>Выборка указанных номенклатур

Каждое сопоставление контроля качества должно ссылаться на выборку номенклатуры. Выборка номенклатуры определяет количество, которое будет отправлено для контроля качества. Ее можно настроить так, чтобы она применялась только к сопоставлениям контроля качества, в которых значение **Применимый тип склада** равно _Управление качеством только для складских процессов_. Если значением **Область выборки** для выборки номенклатуры является _Загрузка_ или _Отгрузка_, либо параметр **Спецификация количества** имеет значение _Полное грузоместо_, на выборку номенклатур могут ссылаться только сопоставления контроля качества, для которых значение **Применимый тип склада** равно _Управление качеством только для складских процессов_.

Если определяется выборка номенклатуры, которая использует применимый тип склада _Управление качеством только для складских процессов_, при попытке ссылки на него из сопоставления контроля качества, для которого не используется функция _Управление качеством для складских процессов_, будет получено сообщение об ошибке.

> [!NOTE]
> Выборка номенклатуры, использующая полную блокировку, не поддерживается для сопоставлений контроля качества, в которых для поля **Применимый типа склада** задано _Управление качеством только для складских процессов_.

### <a name="item-sampling"></a>Выборка элементов

Выборка номенклатуры определяет частоту отправки номенклатур для контроля качества. Функция _Управления качеством для складских процессов_ представляет концепцию _области выборки номенклатур_. Область выборки номенклатуры используется системой при оценке того, должны ли быть созданы заказы для контроля качества и/или работа выборки номенклатур для контроля качества и работа заказа для контроля качества и как они должны создаваться.

Чтобы настроить выборку номенклатуры, перейдите к **Управление запасами \> Настройка \> Контроль качества \> Выборка номенклатуры** и задайте в поле **Область выборки** одно из следующих значений:

- **Заказ** — строка исходного документа будет основой для оценки того, должны ли быть созданы заказы для контроля качества и/или работа выборки номенклатур для контроля качества и работа заказа для контроля качества и как они должны создаваться. Это значение является значением по умолчанию и, когда оно выбрано, система работает так же, как если бы функция _Управление качеством для складских процессов_ не была включена.
- **Загрузка** — загрузки будут использоваться в качестве основы для оценки того, создается ли заказ и/или работа для контроля качества и как они создаются. Это значение доступно только в том случае , если включена функция _Управление качеством для складских процессов_.
- **Отгрузка** — отгрузки будут использоваться в качестве основы для оценки того, создается ли заказ и/или работа для контроля качества и как они создаются. Это значение доступно только в том случае , если включена функция _Управление качеством для складских процессов_.

> [!NOTE]
> Если для поля **Область выборки** установлено значение *Загрузка* или *Отгрузка*, то будут использоваться объект загрузки или объекты отгрузка, если они доступны. Если они недоступны, будет использоваться объект заказа.

Функция _Управления качеством для складских процессов_ также вводит значение *Полное грузоместо* для поля **Спецификация количества**. Это значение поддерживает создание работы по заказу для контроля качества и работы выборки номенклатуры для контроля качества на основе грузомест. При выборе этого значения происходят следующие изменения:

- Становятся доступными параметр **Разбить количество по номенклатурам** и поле **Для каждого n-го грузоместа** на экспресс-вкладке **Обработка**.
- Поле **Значение** на экспресс-вкладке **Количество выборки** становится недоступным.
- Для всех параметров **На обновленное количество**, **Местоположение** и **Грузоместо** устанавливается значение *Да*, и эти настройки изменить нельзя.

Параметр **Разбить количество по номенклатурам** определяет, оценивается ли количество грузомест но номенклатурам или для всех номенклатур в области выборки. Варианты продукта рассматриваются как одна и та же номенклатура. Этот параметр также определяет, сбрасывать ли для количество грузомест для каждой номенклатуры.

Значение поля **Для каждого n-го грузоместа** определяет, как часто создаются заказы для контроля качества относительно количества зарегистрированных номенклатур. Например, значение *3* соответствует отправке каждой третьей номенклатуры на контроль качества, начиная с первой номенклатуры. Значение должно быть больше нуля.

В то время как работники получают номенклатуры с помощью приложения склада, система проверяет, настроено ли сопоставление контроля качества для каждой входящей номенклатуры. Если сопоставление контроля качества настроено, система использует запись выборки номенклатуры, настроенную для этого сопоставления качества, чтобы определить, как будут создаваться заказы для контроля качества, работа по выборке номенклатуры для контроля качества и работа заказа на покупку.

> [!NOTE]
> Когда регистрация прихода выполняется в веб-клиенте (с помощью небольшой страницы регистрации или журнал прибытия номенклатуры для строк заказа на покупку), не будет создано никаких работ выборки номенклатур для контроля качества или работ заказов на покупку, независимо от настройки. Вместо этого для номенклатур, соответствующих сопоставлению контроля качества, будет использоваться выборка номенклатуры, на которую имеется ссылка, для управления созданием только заказов для контроля качества.

Можно повторно открыть заказ для контроля качества, который уже был проверен, при условии, что работа заказа для контроля качества, которая связана с перемещением запасов из местоположения для контроля качества, не имеет в поле **Статус работы** значения *Закрыто* или *Выполняется*.

## <a name="process-insights-when-multiple-quality-associations-coexist"></a>Аналитические сведения, когда несколько сопоставлений контроля качества сосуществуют

Для одной и той же строки документа-источника можно определить и применить несколько сопоставлений контроля качества, а поле **Применимый тип склада** можно настроить как _Управление качеством только для складских процессов_ для некоторых из этих сопоставлений контроля качества и как _Все_ для других.

В следующем примере значение **Тип ссылки** равно _Покупка_.

1. Первое сопоставление контроля качества настраивается следующим образом:

    - **Применимый тип склада:** *Управление качеством только для складских процессов*
    - **Код номенклатуры:** *A0001*
    - **Код счета:** *All*
    - **Проверочная группа:** *Корпус*
    - **Выборка номенклатуры:** *5 шт.*

1. Второе сопоставление контроля качества настраивается следующим образом:

    - **Применимый тип склада:** *Все*
    - **Код номенклатуры:** *Все*
    - **Код счета:** *All*
    - **Проверочная группа:** *Корпус*
    - **Выборка номенклатуры:** *1 шт.*

1. Третье сопоставление контроля качества настраивается следующим образом:

    - **Применимый тип склада:** *Управление качеством только для складских процессов*
    - **Код номенклатуры:** *Все*
    - **Код счета:** *104*
    - **Проверочная группа:** *Импеданс*
    - **Выборка номенклатуры:** *Каждое второе грузоместо* (Эта настройка означает, что полученные первое, третье, пятое и т. д. грузоместа создают заказ для контроля качества.)

1. Четвертое сопоставление контроля качества настраивается следующим образом:

    - **Применимый тип склада:** *Все*
    - **Код номенклатуры:** *Все*
    - **Код счета:** *All*
    - **Проверочная группа:** *Импеданс*
    - **Выборка номенклатуры:** *5 шт.*

1. Пятое сопоставление контроля качества настраивается следующим образом:

    - **Применимый тип склада:** *Все*
    - **Код номенклатуры:** *Все*
    - **Код счета:** *All*
    - **Проверочная группа:** *Конус*
    - **Выборка номенклатуры:** *10%*

Теперь для поставщика 104 создается заказ на покупку в количестве 10 штук номенклатуры A0001. Затем строка заказа на покупку, имеющая количество 10, регистрируется как полученная на одном грузоместе с помощью приложения склада. Вот результат:

- Имеется один заказ для контроля качества от первого сопоставления контроля качества для группы испытаний *Корпус*. Количество равно 5. Нет заказа для контроля качества, связанного со вторым сопоставлением контроля качества, поскольку критерии для первого сопоставления контроля качества являются более специальными по отношению к группе проверки *Корпус*.
- Имеется один заказ для контроля качества от третьего сопоставления контроля качества для группы испытаний *Импеданс*. Количество равно 10. Нет заказа для контроля качества, связанного с четвертым сопоставлением контроля качества, поскольку критерии для первого сопоставления контроля качества являются более специальными по отношению к группе проверки *Импеданс*.
- Имеется один заказ для контроля качества от пятого сопоставления контроля качества для группы испытаний *Конус*. Количество равно 1.

В связи с созданием одного заказа контроля качества для каждого из трех сопоставлений контроля качества создается также работа выборки номенклатур для контроля качества. Зарегистрированное количество равно только 10. Однако вследствие настройки выборки номенклатур сумма количеств по заказам контроля качества, созданных для применимого типа склада *Управление качеством только для складских процессов* равно 16, что превышает физическое зарегистрированное количество, равное 10. Таким образом, работа не будет создаваться для заказов контроля качества с полным количеством (16), поскольку для перемещения в место контроля качества физически доступны только 10. Приоритет, который используется для создания работы выборки номенклатур для контроля качества, соответствует порядку создания заказов для контроля качества:

- **Первый заказ для контроля качества (количество = 5):** работа выборки номенклатур для контроля качества создается для 5 шт. Количество 5 (10 – 5) теперь остается для последующего создания работы выборки номенклатуры для контроля качества.
- **Второй заказ для контроля качества (количество = 10):** работа выборки номенклатур для контроля качества создается для 5 шт. Количество 0 (ноль) теперь остается для последующего создания работы выборки номенклатуры для контроля качества.
- **Третий заказ для контроля качества (количество = 1):** работа выборки номенклатур для контроля качества не создается.

В процессе создания заказов для контроля качества создается блокировка запасов для количества, равного 10. Данная блокировка запасов осуществляется в соответствии с каждым из трех заказов для контроля качества. Сумма количества по заказам для контроля качества равна 16.

Когда заказы для контроля качества проверяются, система пытается создать работу по заказу для контроля качества для каждого проверенного заказа для контроля качества. Поскольку сумма количеств по заказам для контроля качества превышает количество, которое фактически заблокировано и, следовательно, доступно для создания работы, невозможно создать заказ для контроля качества для полного количества заказов для контроля качества, как показано здесь. (В этом примере продолжается предыдущий пример.)

1. **Проверьте второй созданный заказ для контроля качества (количество = 10). Работа по заказу для контроля качества создается для количества, равного 4.**

    Создание заказа для контроля качества запускается путем изменения количества блокировки запасов. Поскольку сумма заказов для контроля качества была равна 16, проверка количества 10 приведет к проверке остающихся количеств заказа для контроля качества как равных 6. Количество блокировки запасов уменьшается с 10 до 6. Уменьшенное количество 4 выделяется для создания заказа для контроля качества.

2. **Проверьте первый созданный заказ для контроля качества (количество = 5). Работа по заказу для контроля качества создается для количества, равного 5.**

    Создание заказа для контроля качества запускается путем изменения количества блокировки запасов. Поскольку сумма заказов для контроля качества была равна 6, проверка количества 5 приведет к проверке остающихся количеств заказа для контроля качества как равных 1. Количество блокировки запасов уменьшается с 6 до 1. Уменьшенное количество 5 выделяется для создания заказа для контроля качества.

3. **Проверьте третий созданный заказ для контроля качества (количество = 1). Работа по заказу для контроля качества создается для количества, равного 1.**

    Создание заказа для контроля качества запускается путем изменения количества блокировки запасов. Поскольку сумма заказов для контроля качества была равна 1, проверка количества 1 приведет к проверке остающихся количеств заказа для контроля качества как равных 0 (нулю). Блокировка запасов удаляется (то есть, количество блокировки запасов уменьшается с 1 до 0). Уменьшенное количество 1 выделяется для создания заказа для контроля качества.

> [!NOTE]
> Создание работы заказа для контроля качества зависит от количества блокировки запасов, которое связано с одним или несколькими заказами для контроля качества. Если сумма количеств по заказам для контроля качества превышает количество блокировки запасов, на которое имеется ссылка, то порядок, в котором проверяются заказы для контроля качества, определяет создание работы заказа для контроля качества.

## <a name="canceling-quality-item-sampling-work"></a>Отмена работы по выборке номенклатуры для контроля качества

Можно отменить работу, созданную для выборки номенклатуры для контроля качества. Чтобы контролировать, что происходит при отмене этой работы, выполните следующие действия.

1. Перейдите в раздел **Управление складом \> Настройка \> Параметры управления складом**.
1. На вкладке **Общие** на экспресс-вкладке **Работа** установите для параметра **Отмена регистрации поступления при отмене работы** одно из следующих значений:

    - **Да** — при отмене работы выборки номенклатуры для контроля качества связанный заказ для контроля качества удаляется, и регистрация запасов отменяется.
    - **Нет** — при отмене работы выборки номенклатуры для контроля качества связанный заказ для контроля качества не удаляется, и регистрация запасов не отменяется.

## <a name="cross-docking"></a>Кросс-докинг

Можно настроить сопоставление контроля качества, которое создает работу выборки номенклатуры. Однако, когда кросс-докинг существует параллельно с сопоставлением контроля качества, которое создает работу по отбору номенклатур для контроля качества, при наличии достаточного количества только для удовлетворения кросс-докинга, создается только работа выборки номенклатуры. В случаях, когда параметр **Включить заказ для контроля качества для складских процессов** имеет значение _Да_ для склада приемки, а в поле **Применимый тип склада** задано значение _Управление качеством только для складских процессов_, создание работы выборки номенклатуры для контроля качества имеет больший приоритет, чем создание работы кросс-докинга. Если количество превышает потребность в кросс-докинге, система все равно создает только работу выборки номенклатуры.

## <a name="destructive-testing"></a>Тестирование с разрушением

Можно определить группу испытаний, которая производит испытания с разрушением. В случае испытания с разрушением предполагается, что, независимо от результата испытания, количество тестируемой номенклатуры будет уничтожено как часть испытания. Способ, с помощью которого функция _Управление качеством для складских процессов_ поддерживает разрушительное тестирование, похож на способ, которым система управления качеством поддерживает его, когда эта функция не включена. Прежде чем заказ для контроля качества может быть проверен, контроллер качества должен указать местоположение комплектации для количества, которое было уничтожено. Зарегистрировать комплектацию можно на странице заказа для контроля качества, выбрав на панели операций **Запасы \> Комплектация**. После регистрации комплектации количества по заказу для контроля качества можно выполнить проверку.

### <a name="quality-in-setup--move-to-the-quality-control-location"></a>Настройка поступления на контроль качества — перемещение в местоположение контроля качества

Теперь необходимо подготовить базовую настройку, которая позволит системе поддерживать функцию _Управление качеством для складских процессов_ для склада 51. (Демонстрационные данные определяют местоположение управления качеством с именем *QMS*. В этом сценарии это местоположение упоминается несколько раз.) Будут подготовлены следующие элементы, как описано в подразделах этого раздела:

- Класс работы
- Шаблон работы
- Директива местоположения
- Выборка элементов
- Сопоставление контроля качества
- Пункты меню мобильного устройства

#### <a name="work-class-for-quality-in"></a>Класс работы для поступления на контроль качества

1. Перейдите в раздел **Управление складом \> Настройка \> Работа \> Классы работы**.
1. Создайте класс работы и задайте следующие значения:

    - **Код класса работы:** _QualityIn_
    - **Описание:** _Выборка номенклатуры для контроля качества_
    - **Тип заказа на работу:** _Выборка номенклатуры для контроля качества_

#### <a name="work-template"></a>Шаблон работы

1. Перейдите в раздел **Управление складом \> Настройка \> Работа \> Шаблоны работ**.
1. Задайте в поле **Тип заказа на работу** значение _Выборка номенклатуры для контроля качества_.
1. Создайте шаблон работы и задайте следующие значения:

    - **Шаблон работы:** _51 Контроль качества_
    - **Описание шаблона работы:** _51 Контроль качества_

1. Добавьте строку в шаблон работы и задайте следующие значения:

    - **Тип работы:** _Комплектация_
    - **Код класса работы:** _QualityIn_

1. Добавьте вторую строку в шаблон работы и задайте следующие значения:

    - **Тип работы:** _Размещение_
    - **Код класса работы:** _QualityIn_

#### <a name="location-directive"></a>Директива местоположения

1. Перейдите в раздел **Управление складом \> Настройка \> Директивы местонахождения**.
1. Задайте в поле **Тип заказа на работу** значение _Выборка номенклатуры для контроля качества_.
1. Создайте директиву местонахождения и задайте следующие значения:

    - **Имя:** _51 на контроль качества_
    - **Тип работы:** _Размещение_
    - **Сайт:** 5
    - **Склад:** _51_

1. Добавьте строку для директивы местонахождения и задайте следующие значения:

    - **Количество "От":** _1_
    - **Количество "До":** _1000000_

1. Создайте действие директивы местонахождения и задайте следующее значение:

    - **Имя:** _Контроль качества_

1. Для нового действия директивы местонахождения выберите **Изменить запрос** и укажите запись **Диапазон** со следующими значениями:

    - **Таблица:** *Местонахождения*
    - **Поле:** _Код профиля местонахождения_
    - **Критерии:** *QMS*

1. Нажмите кнопку **ОК**, чтобы сохранить запрос, и сохраните новую директиву местонахождения.

Затем необходимо изменить последовательность существующих директив местоположения заказов на покупку для склада 51. Демонстрационные данные включают две директивы местоположения, которые имеют для параметра **Тип заказа на работу** значение _Покупка_: одна называется _51 QMS_, другая называется _51 Прямая по заказу на покупку_. Чтобы гарантировать, что функция *Управление качеством для складских процессов* применяется для склада 51, следует обеспечить, чтобы не применялась директива местоположения _51 QMS_. Однако вместо удаления этой директивы местоположения (так как ее можно будет использовать в будущем) можно просто изменить последовательность.

1. Перейдите в раздел **Управление складом \> Настройка \> Директивы местонахождения**.
1. Задайте в поле **Тип заказа на работу** значение _Заказ на покупку_.
1. В списке последовательности выберите порядковый номер 5 для директивы местонахождения _51 Прямая по заказу на покупку_.
1. Переместите выбранную последовательность вверх на порядковый номер 4.
1. Убедитесь, что порядковый номер директивы местонахождения _51 QMS_ теперь как минимум 5.

#### <a name="item-sampling"></a>Выборка элементов

Функция _Управление качеством для складских процессов_ добавляет новые возможности выборки номенклатуры. Значением параметра **Область выборки** теперь может быть _Заказ_, _Отгрузка_ или _Загрузка_, а значением параметра **Количество выборки** может быть _Полное грузоместо_.

1. Перейдите в раздел **Управление запасами \> Настройка \> Контроль качества \> Выборка номенклатуры**.
1. Создайте запись выборки номенклатуры и установите следующие значения:

    - **Выборка номенклатуры:** _3-е грузоместо_
    - **Описание:** _Каждое третье грузоместо_
    - **Область выборки:** _Заказ_

1. На экспресс-вкладке **Количество выборки** задайте в поле **Спецификация количества** значение _Полное грузоместо_.
1. На экспресс-вкладке **Обработка** задайте для поля **Для каждого n-го грузоместа** значение _3_.
1. В разделе **По аналитике хранения** включите **Склад** и **Состояние запасов**.

#### <a name="quality-associations"></a>Сопоставления контроля качества

Создайте сопоставление контроля качества, которое будет использовать новую выборку номенклатуры.

1. Перейдите в раздел **Управление запасами \> Настройка \> Контроль качества \> Сопоставления контроля качества**.
