# <a name="quality-management-test-variables"></a>Переменные тестирования управления качеством

[!include [banner](../includes/banner.md)]

В этой статье описывается, как создавать переменные проверки, чтобы можно было использовать для качественных проверок с заказами контроля качества в Microsoft Dynamics 365 Supply Chain Management.

Для каждой качественной проверки, заданной на странице **Проверка**, следует определить переменную проверки и ее возможные результаты. (Для качественных проверок поле **Тип** на странице **Проверки** задается как *Параметр*.)

Используйте страницу **Переменные проверки** для настройки, изменения и просмотра возможных результатов переменной проверки, связанной с качественной проверкой. Для каждого результата назначается статус результата *Пройдено* или *Ошибка*, чтобы указать, пройдена ли проверка или нет, при выборе этого результата в качестве результата проверки. Используйте страницу **Группы проверки** для назначения переменной проверки и ее результата по умолчанию для отдельной качественной проверки.

Для каждой переменной проверки рекомендуется определить по крайней мере два результата: один со статусом *Пройдено*, а второй *Ошибка*. Нет ограничения на общее число переменных или результатов, которые могут быть определены. Кроме того, несколько проверок могут использовать одни и те же переменные проверки для записи результатов.

## <a name="examples-of-test-variables"></a>Примеры переменных проверки

### <a name="example-1"></a>Пример 1

Производственная компания выполняет две проверки произведенных материалов. В одной проверке уровень pH связан с цветной полоской. Приемлемые уровни pH — более светлые цвета, а неприемлемые уровни pH — более темные цвета. В ходе другой проверки выполняется несколько визуальных проверок, и работники контроля качества используют свое мнение для определения того, проходит ли номенклатура визуальную проверку или нет.

### <a name="example-2"></a>Пример 2

Производственная компания, выпускающая печенье, использует инспекционный тест для готовой продукции. Этот инспекционный тест имеет несколько переменных. Одна из переменных — вкус, при этом возможные результаты — хороший или плохой. Вторая переменная — цвет, возможные результаты — "слишком темное", "слишком светлое" и "правильный".

## <a name="create-a-test-variable"></a>Создание переменной проверки

Чтобы создать переменную проверки, выполните следующие действия.

1. Перейдите **Управление запасами \> Настройка \> Контроль качества \> Переменные проверки**.
1. На панели операций выберите **Создать**, чтобы добавить строку в сетку. Затем задайте следующие поля для новой строки:

    - **Переменная** — введите уникальный код или имя переменной.
    - **Описание** — введите подробное описание переменной.

1. Когда новая строка все еще выбрана, выберите **Результаты** на панели операций.
1. На странице **Результаты переменной проверки** в области действий выберите **Создать**, чтобы добавить строку в сетку. Затем задайте следующие поля для новой строки:

    - **Результат** — введите уникальный код или имя результата.
    - **Описание** — введите подробное описание результата.
    - **Статус результата** — выберите *Пройдено* или *Ошибка*, чтобы указать, пройдена ли проверка или нет, при выборе результата в качестве результата проверки.

1. Повторите предыдущий шаг для каждого дополнительного результата. Убедитесь, что по крайней мере один результат имеет статус *Пройдено*, а по крайней мере один имеет статус *Ошибка*.
1. Закройте страницы.
