# <a name="manage-voyages"></a>Управление рейсами

[!include [banner](../../includes/banner.md)]

Рейсом обычно является судно. Однако, в зависимости от методик и процедур, это может быть поставщик, заказ на покупку или другая номенклатура, которая используется в вашей организации.

Страница **Все рейсы** содержит сведения о ходе исполнения, сведения о доставке и затратах для рейса, а также информацию о номенклатурах, заказах на покупку и заказах на перемещение. Чтобы открыть страницу **Все рейсы**, перейдите **Стоимость на складе \> Рейсы \> Все рейсы**. На этой странице отображается список всех текущих рейсов. Можно использовать кнопки на панели операций для создания, удаления и работы с рейсами. Выберите любой рейс в списке, чтобы просмотреть сведения о нем.

> [!NOTE]
> Контейнеры отгрузки и листы связаны с рейсом. Строки покупки связаны с контейнером отгрузки. Если контейнеры отгрузки и листы отключены, они также могут быть связаны непосредственно с рейсом. Кроме того, введенные здесь затраты распределяются по всем связанным строкам покупки.

## <a name="action-pane"></a>Панель операций

Панель операций на странице **Рейсы** содержит кнопки, позволяющие работать с выбранным рейсом. Каждая кнопка выполняет одно действие. Панель операций также включает вкладки, каждая из которых, в свою очередь, содержит набор соответствующих кнопок. За исключением оговоренных случаев, все кнопки и вкладки доступны как в представлении списка на странице **Все рейсы**, так и в подробном представлении для одного выбранного рейса.

### <a name="buttons-that-appear-directly-on-the-action-pane"></a>Кнопки, появляющиеся непосредственно на панели операций

В следующей таблице описаны кнопки непосредственно на панели операций.

| Кнопка | описание |
|---|---|
| Новое | Создайте рейс. <!-- KFM: Link to scenario --> |
| Удаление | Удалите текущий рейс. Могут быть удалены только рейсы со статусом *Подтверждено*. После выхода рейса из порта и обработки товаров в пути его нельзя удалить. |
| Редактор поездки | Откройте страницу **Редактор рейсов**, где можно добавлять или удалять строки покупок для рейса, создавать новые контейнеры и изменять сведения о самом рейсе. |
| Затраты на поездку | Откройте страницу **Затраты на рейс**, на которой можно просмотреть и добавить затраты на рейс для всех товаров в рейсе. Когда затраты на рейс добавляются в рейс вручную, они автоматически добавляются на страницу **Запрос затрат** и распределяются по всем товарам в соответствии с методом, указанным на странице **Затраты на рейс**. |

### <a name="buttons-on-the-voyage-tab"></a>Кнопки на вкладке "Рейс"

В следующей таблице описаны кнопки на панели операций вкладки **Рейс**. Эта вкладка доступна только при работе в представлении списка страницы **Все рейсы**.

| Кнопка | описание |
|---|---|
| Грузовой контейнер | Открывает страницу, отображающую все контейнеры отгрузки, связанные с выбранным рейсом. Может быть один контейнер или несколько контейнеров. |
| Лист | Открывает страницу, отображающую все листы, связанные с выбранным рейсом. Может быть один лист или несколько листов. |

### <a name="buttons-on-the-manage-tab"></a>Кнопки на вкладке "Управление"

В следующей таблице описаны действия на панели операций вкладки **Управление**.

| Кнопка | описание |
|---|---|
| Документы получены | Обновите рейс таким образом, чтобы параметр **Полученные документы** был установлен как *Да*. Эту кнопку можно использовать для блокировки номенклатуры и/или строки покупки, чтобы дальнейшее обновление было невозможно. |
| В пути | Обновите поле **Статус рейса** на статус "в пути", которое устанавливается на странице **[Параметры стоимости на складе](landed-cost-parameters.md)**. Дальнейшая логика по этому процессу отсутствует. Статус рейса также может быть автоматически обновлен до статуса "в пути" на основе параметров в [центре управления отслеживанием](delivery-information-setup.md).
| Готово к учету затрат | Обновите поле **Статус рейса** на статус "готово к учету затрат", которое устанавливается на странице **[Параметры стоимости на складе](landed-cost-parameters.md)**. Затраты на рейс могут быть оценены после того, как все накладные обработаны (как накладные на запасы и накладные на затраты на рейс) и товары были получены. Если оценочная стоимость, связанная с рейсом, не рассчитана по затратам, то при попытке обработать учет затрат на рейс произойдет ошибка. |
| Произведен расчет себестоимости | Удалите все отклонения в учете затрат после появления накладной для всех заказов на покупку и затрат на рейс. При нажатии этой кнопки отображается диалоговое окно **Обновление рейса — Произведен расчет себестоимости**. Здесь можно выбрать разноску на стандартную финансовую дату или указать дату разноски, а затем выполнить действие. Можно повторно выполнить действие любое требуемое число раз. Можно также воспользоваться диалоговым окном **Обновление рейса — Произведен расчет себестоимости**, чтобы задать план выполнения действия как периодической задачи (пакетное задание). Рекомендуется регулярно выполнять действие, настроив его как пакетное задание. |
| Разноска списка прихода | Разноска списка поступлений для всех строк заказа на покупку в рейсе.  |
| Разноска поступления продуктов | Разнесите поступление продукта для всех строк заказа на покупку в рейсе. Процесс поступления продуктов для строк заказа на покупку, связанных с рейсом, будет использоваться только в том случае, если товары **не** проходят через обработку товаров в пути. Если товары будут проходить через обработку товаров в пути, появится сообщение об ошибке при попытке разноски поступления продуктов для строки заказа на покупку.  |
| Разноска накладной | Разнесите накладную для всех строк заказа на покупку в рейсе. Если товары в рейсе будут проходить через обработку товаров в пути, по строкам заказа на покупку будут выставлены накладные, прежде чем процесс поступления будет завершен. Когда для исходного заказа на покупку выставляется накладная, будут созданы заказы товаров в пути, связанные с исходными строками заказа на покупку. Затем эти заказы могут быть получены складом.  |
| Отгрузить заказ на перемещение | Разнесите рейс заказа на перемещение для всех строк заказа на перемещение в рейсе. Если эта кнопка выбрана, только заказы на перемещение будут доступны для обновления. |
| Получить заказ на перемещение | Разнесите поступление заказа на перемещение для всех строк заказа на перемещение в рейсе. |
| Получить товары в пути | Получите все строки заказа, которые находятся в пути для рейса. Эта кнопка является одним из трех вариантов, доступных для получения товаров в пути в рейсе. (Два других параметра — кнопка **Создать журнал прибытия**, которая описывается ниже в этой таблице, и мобильное приложение управления складом.) Этот параметр является простейшим параметром и будет обрабатывать товары в пути из склада товаров в пути в последний склад назначения. Если требуется более полный контроль над процессом, используйте журнал прибытия или мобильное устройство для обработки поступления товаров. |
| Поиск автоматических затрат | Найдите соответствующие затраты на рейс. Если эти затраты уже были найдены или обновлены, появится следующее сообщение: "Обнаружены строки затрат без накладных. Перезаписать их?" Будут найдены все затраты, которые не связаны с рейсом в момент создания. Затраты на рейс, приложенные к рейсу и по которым были выставлены накладные, не будут перезаписаны. |
| Создать журнал прибытия | <p>Откройте диалоговое окно **Создание журнала прибытия**, в котором можно создать журнал прибытия, в котором указывается местоположение. Диалоговое окно содержит следующие варианты:</p><ul><li>**Создание из товаров в пути** или **Создание заказа на перемещение** — метка для этого параметра меняется в зависимости от того, используется ли процесс товаров в пути. Установите значение *Да*, чтобы открыть страницу журнала прибытия, которая позволяет обработать стандартный журнал прибытия для товаров в пути, связанных с данным рейсом. Если номенклатура уже была получена на конечном целевом складе, она не будет добавлена в строки журнала прибытия.</li><li>**Инициализировать количество** — установите этот флажок как *Да*, чтобы инициализировать количество, которое будет получено, на основе количества товаров, указанных в строке рейса. Если строка рейса была частично получена, это количество будет равно количеству остатка. Рекомендуется оставить для этого параметра значение *Да*.</li><li>**Создать из строк заказа** — установите для этого параметра значение *Да*, чтобы получить значение из строк заказа.</li></ul><p>Эта кнопка является одним из трех вариантов, доступных для получения товаров в рейсе. (Другие параметры — кнопка **Получить товары в пути**, которые были описаны ранее в этой таблице, и мобильное приложение управления складом.)</p> |
| Начислить затраты | Можно выполнить начисление затрат, где у типа затрат есть счет ГК, указанный для дебета. Эта кнопка обычно используется, когда запасы находятся в пути или когда товары были получены и по ним выставлены накладные. |
| Агрегировать затраты | Переместите затраты с уровня контейнера отгрузки на уровень рейса. Эту кнопку можно использовать в сценарии общих служб/отгрузки, где несколько объектов совместно используют контейнер отгрузки или место для упаковки. Например, в рейсе есть 40-футовый контейнер отгрузки и 20-футовой контейнер отгрузки, а распределение выполняется по объему. В этом случае товары/объекты, которые совместно используют или просто используют место в 20-футовом контейнере отгрузки, могут быть штрафными. Чтобы честно распределить затраты, в некоторых организациях может возникнуть необходимость переноса затрат в рейс и распределения их на основе метода распределения на уровне рейса. |
| Изменить шаблон пути | Откройте диалоговое окно, где вы можете изменить шаблон пути. После изменения шаблона затраты на рейс будут удалены. Таким образом, может потребоваться выбрать **Найти автоматические затраты** (см. описание ранее в этой таблице) или вручную добавить затраты снова. |

### <a name="buttons-on-the-general-tab"></a>Кнопки на вкладке "Общее"

В следующей таблице описаны кнопки на панели операций вкладки **Общее**.

| Кнопка | описание |
|---|---|
| Список прихода | Откройте список поступлений продуктов для всех строк заказа на покупку в рейсе.  Если списки поступлений продуктов не были обработаны, эта кнопка недоступна. |
| Поступление продуктов | Откройте запись о поступлении продуктов для строк заказа на покупку, связанных с рейсом, если данная запись используется. Если поступления продуктов были разнесены, эта кнопка недоступна. Процесс поступления продуктов не будет использоваться, если используется обработка товаров в пути. |
| Прибытие номенклатуры | Откройте журнал прибытия номенклатуры, если он используется. |
| Отслеживание | Откройте страницу **Входящая трассировка**, на которой можно обновить ожидаемую дату прибытия товаров в контейнере отгрузки и рейсе, а затем обновить ожидаемые даты доставки по строкам заказа на покупку. |
| Запрос затрат | <p>Откройте страницу **Запрос затрат**, на которой отображаются все затраты на рейс.</p><p>Выберите **Вид** в области действий для изменения представления. Можно просмотреть любой из уровней, плюс номенклатуру и код типа затрат.</p><p>На странице **Запрос затрат** отображаются только коды типов затрат, которые непосредственно связаны с проводками по запасам. Эта возможность может быть полезной при использовании размеров и цветов.</p><p>На странице **Запрос затрат** отображаются только коды типов затрат, для которых в поле **Дебет** указано значение *Номенклатура*.</p> |
| Заказы товаров в пути | Откройте **Заказы товаров в пути**, на которой показаны записи о товарах в пути только для выбранного рейса. |

## <a name="lines-view"></a>Представление строк

Чтобы открыть представление **Строки**, откройте рейс, а затем выберите вкладку **Строки** в верхнем правом углу заголовка рейса.

### <a name="information-on-the-voyage-header-fasttab"></a>Сведения на экспресс-вкладке заголовка рейса

Экспресс-вкладка **Заголовок рейса** в представлении **Строки** рейса содержит базовые сведения, описывающие рейс. Многие поля, отображаемые на этой экспресс-вкладке, также отображаются в представлении **Заголовок**, как описано ниже в этом разделе.

### <a name="information-on-the-voyage-lines-fasttab"></a>Сведения на экспресс-вкладке строк рейса

Экспресс-вкладка **Строки рейса** в представлении **Строки** рейса связана со сведениями о рейсе и сведениями о затратах, которые применяются на уровне рейса. Здесь можно просмотреть контейнеры отгрузки, листы и номенклатуры, связанные с рейсом. Также показываются цена и количество номенклатур в данном рейсе. Можно просмотреть любой контейнер отгрузки или лист, который отображается в списке, выбрав соответствующую ссылку.

### <a name="information-on-the-line-details-fasttab"></a>Сведения на экспресс-вкладке сведений строки

Экспресс-вкладка **Сведения строки** в представлении **Строки** содержит более подробную информацию о выбранной строке на экспресс-вкладке **Строки рейса**. Обратите внимание, что эта экспресс-вкладка содержит сведения о положении, которое каждая строка занимает в контейнере, и его объявленное количество.

## <a name="header-view"></a>вид заголовка

Чтобы открыть представление **Заголовок**, откройте рейс, а затем выберите вкладку **Заголовок** в верхнем правом углу заголовка рейса.

### <a name="settings-on-the-general-fasttab"></a>Настройки на экспресс-вкладке "Общее"

В следующей таблице описаны поля, доступные на экспресс-вкладке **Общее** в представлении **Заголовок** рейса.

| Поле | описание |
|---|---|
| Поездка | Введите номер рейса судна или самолета. |
| Ссылка на резервирование | Если грузоотправитель или центр отгрузки предоставляет номер ссылки для определения рейса (т. е. внутренняя ссылка грузоотправителя или центра отгрузки), введите его здесь. |
| Судно | Введите или выберите судно. Если судно отсутствует в списке, можно ввести код судна в качестве свободного текста. В этом случае основная таблица не обновляется, поэтому код судна в этом поле можно выбрать позднее. |
| Внешний ИД поездки | Введите общедоступный код для рейса (например, номер рейса). |
| Основная авиационная грузовая накладная/транспортная накладная | Введите номер путевого листа главной авианакладной или транспортной накладной. Это значение можно указать при отгрузке для авиарейса. |
| Авиационная грузовая накладная/транспортная накладная компании | Введите путевой лист домашней авианакладной или транспортную накладную для контейнера отгрузки. |
| Аренда | Установите этот флажок, чтобы указать, что контейнер является арендным контейнером, который должен быть возвращен. |
| описание | Введите описание рейса, чтобы упростить его распознавание. |
| Статус поездки | Статус рейса.. Значения: *Создано*, *В пути*, *Полученные документы* и *Учтены в затратах*. Это поле не рассчитывается на основе логики. Оно обновляется только при помощи действия разноски. Значение *Учтено в затратах* указывает на то, что накладная для запасов и все затраты на рейс получены. Пока накладная не будет получена, рейс не может получить статус *Учтено в затратах*. |
| Статус заказа на покупку | Наивысший статус, который был достигнут среди всех заказов на покупку, связанных с рейсом. |
| Документы получены | Значение, указывающее, получены ли в компании все документы, связанные с этим рейсом. Чтобы изменить значение, выберите **Полученные документы** в группе **Обновление рейса** на вкладке **Управление** в области действий. |
| Компания по отгрузке | Выберите для этого рейса транспортную компанию. Это поле может использоваться для определения автоматических затрат. |
| ФИО | Название компании, выбранной в поле **Транспортная компания**. |
| Измерение | В этом поле можно указать измерение, задаваемое в автоматических затратах. Измерения часто используются организациями, которые не знают отдельный объем или вес товаров, но требуют более точного распределения по сравнению с суммой или количеством. Экспедитор груза предоставит сведения о весе или мере объема, их можно использовать на уровне номенклатуры или заказа на покупку. Он может быть автоматически обновлен, если выбран параметр, или его можно ввести вручную. |
| Единица измерения | Единица измерения, которая применяется к числу, выбранному в поле **Измерение**. |
| Количество палет | Указывается количество палет в строке рейса. Это поле обновляется автоматически, если параметр **Автоматически обновлять число коробок** задан как *Да* на вкладке **Общее** на странице **Параметры стоимости на складе**. |

## <a name="voyage-update-periodic-tasks"></a>Периодические задачи обновления рейса

Модуль **Стоимость на складе** содержит несколько периодических задач, связанных с рейсом, которые могут выполнить массовое обновление нескольких аспектов рейса. Чтобы выполнить или спланировать эти периодические задачи, перейдите на страницу **Стоимость на складе \> Периодические задачи \> Обновления рейса**, а затем выберите один из следующих типов задач:

- **Полученные документы** — эта задача позволяет установить для **Полученные документы** значение *Да* для нескольких рейсов одновременно. Используйте параметры **Фильтр** для определения набора рейсов, которые необходимо обновить.
- **В пути** — эта задача позволяет установить в поле **Статус рейса** статус "в пути", который задается на странице **[Параметры стоимости на складе](landed-cost-parameters.md)** для нескольких рейсов одновременно. Используйте параметры **Фильтр** для определения набора рейсов, которые необходимо обновить.
- **Готово к учету затрат** — эта задача позволяет установить в поле **Статус рейса** статус "готово к учету затрат", который задается на странице **[Параметры стоимости на складе](landed-cost-parameters.md)** для нескольких рейсов одновременно. Обычно эта задача настраивается для выполнения по регулярному расписанию.
- **Учтено в затратах** — эта задача позволяет установить в поле **Статус рейса** статус "учтено в затратах", который задается на странице **[Параметры стоимости на складе](landed-cost-parameters.md)** для нескольких рейсов одновременно при условии, что они были учтены в затратах, но еще не были обновлены в рейсе. Обычно эта задача настраивается для выполнения по регулярному расписанию.
