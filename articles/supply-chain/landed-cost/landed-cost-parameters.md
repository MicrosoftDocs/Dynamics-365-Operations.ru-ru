# <a name="landed-cost-parameters-setup"></a>Настройка параметров стоимости на складе

[!include [banner](../../includes/banner.md)]

Используйте страницу **Стоимость на складе**, чтобы настроить общую информацию и параметры конфигурации, используемые в модуле **Стоимость на складе** для разноски, обновления статуса, номерных серий и поведения. Настройка параметров используется совместно в юридических лицах и может изменяться администратором.

## <a name="open-the-landed-cost-parameters-page"></a>Открытие страницы параметров стоимости на складе

Чтобы работать с параметрами, перейдите **Стоимость на складе \> Настойка \> Параметры стоимости на складе**, а затем настройте поля, как описано в следующих подразделах.

![Страница параметров стоимости на складе.](media/landed-cost-parameters.png "Страница параметров стоимости на складе")

## <a name="general-tab"></a>Вкладка "Общие"

### <a name="general-fasttab"></a>Экспресс-вкладка "Общее"

В следующей таблице описаны поля, доступные на экспресс-вкладке **Общее** вкладки **Общее** на странице **Параметры стоимости на складе**.

| Параметр | описание |
|---|---|
| Использовать ставку отгрузки | Ставка отгрузки устанавливается для определенного периода и используется для оценочной стоимости товаров с несколькими валютами. Установите для этого параметра значение *Да*, чтобы использовать ставку отгрузки. |
| Тип валютного курса | Набор валютных курсов по умолчанию, используемый для расчетов с несколькими валютами для рейса и затрат на рейс. |
| Обновить количество по заказу на покупку | Выберите, что происходит, если пользователь изменяет количество в строке заказа на покупку:<ul><li>**Принять** — количество рейса автоматически корректируется.</li><li>**Предупреждение** — если строка связана с рейсом, будет выведено предупреждение, но количество по рейсу обновляется.</li><li>**Ошибка** — если строка связана с рейсом, будет выведено сообщение об ошибке, и заказ на покупку невозможно обновить. Поэтому строка заказа должна быть сначала удалена из рейса.</li></ul> |
| Автоматически обновлять число коробок | Если этот параметр задан как *Да*, все коробки добавляются вместе и отображаются на уровнях рейса и контейнера. Если установлено значение *Нет*, число коробок первоначально устанавливается равным 0 (нулю), и при необходимости может быть изменено вручную. |

### <a name="costing-fasttab"></a>Экспресс-вкладка расчета себестоимости

В следующей таблице описаны поля, доступные на экспресс-вкладке **Расчет себестоимости** вкладки **Общее** на странице **Параметры стоимости на складе**.

| Параметр | описание |
|---|---|
| Спецификация разноски | Выберите корректировку стоимости в ГК:<ul><li>**Итого** — итоговое значение разносится в главную книгу.</li><li>**Номенклатурная группа** — корректировка задается по номенклатурной группе.</li><li>**Код номенклатуры** — корректировка задается по номенклатуре. Это значение предоставляет наиболее полные сведения.</li></ul> |
| Разрешить нулевые затраты | Установите для этого параметра значение *Нет* для отображения ошибки, если пользователь пытается разнести оценку затрат для накладной рейса или заказа на покупку, который не содержит значения ожидаемых затрат на рейс. В сообщении об ошибке указывается, что себестоимость 0 (нулевая) не может быть распределена, и разноска накладных завершается ошибкой. В этом случае пользователь может вручную обновить оценку (или перенастроить автоматические затраты), а затем либо внести обновленное значение, либо удалить затраты, если она не применяется.<p>Установите для этого параметра значение *Да*, чтобы разрешить пустые затраты на рейс. В этом случае цена 0 (ноль) будет распределяться в соответствии с областью затрат. После этого накладная затрат поставщика может быть обработана с себестоимостью с нулевой ценой, когда получен рейс.</p><p>Рекомендуется настроить оценку в записи автоматических затрат, чтобы предотвратить появление себестоимости с нулевой ценой. Хотя эта оценка не полностью точна, она должна быть более точной, чем предполагаемая себестоимость с нулевой ценой.</p> |
| Дата разноски корректировки | При разноске накладной затрат на рейс для расчетов с поставщиками, таблица сопоставлений (коррекция запасов) также обновляется. Выберите дату, которая устанавливается по умолчанию на странице **Выбор затрат на рейс** при работе с журналом накладных:<ul><li>**Дата проводки** — используйте дату журнала (дату проводки).</li><li>**Дата накладной заказа на покупку** — используйте дату финансовой разноски накладной запасов (заказ на покупку).</li><li>**Выбранная дата** — пользователь может указать дату разноски. Хотя дата может быть пустой, но если она остается пустой, когда разносится накладная по затратам, пользователь получит сообщение об ошибке.</li></ul> |
| Использовать ваучер накладной по покупке | Если этот параметр имеет значение *Да*, проводки начисления затрат будут использовать тот же код ваучера, который используется для накладной по покупке. Когда установлено значение *Нет*, система использует следующий доступный номер для номерной серии **Ваучер начисления затрат**, которая настроена на вкладке **Номерные серии** на странице **Параметры стоимости на складе**.<p>Этот вариант эффективен, только когда параметр **Выполнить разноску на счет накл. расходов в ГК** имеет значение *Да* на вкладке **Накладная** на странице **Параметры модуля расчетов с поставщиками**.</p> |
| Блокировать разноску вручную на клиринговый счет | Установите для этого параметра значение *Да*, чтобы не допустить разноску на клиринговые счета, где проводка не была связана с рейсом, выбрав **Функции \> Выбор затрат на рейс** на панели операций журнала накладных поставщика. Рекомендуется установить для этого параметра значение *Да*, чтобы рейс и клиринговый счет можно было правильно сопоставить. |
| Использовать счет начисления накладных расходов типа затрат | Когда для этого параметра установлено значение *Да*, счет начисления накладных расходов, который настроен для соответствующего кода типа затрат на странице **Коды типов затрат**, будет использоваться для начисления затрат в качестве расходов.<p>Этот вариант эффективен, только когда параметр **Выполнить разноску на счет накл. расходов в ГК** имеет значение *Да* на вкладке **Накладная** на странице **Параметры модуля расчетов с поставщиками**. |
| Разнести корректировки как отклонение | Если этот параметр задан как *Да*, он переопределяет стандартные функции и заставляет проводки по корректировке запасов, которые связаны с расхождениями между оценочной стоимостью и фактическими затратами, разноситься на счет расхождений.<p>Когда для этого параметра установлено значение *Нет*, проводки корректировки запасов, которые относятся к расхождениям, обрабатываются на основе конфигурации метода учета затрат и кода типа затрат. Для стандартных затрат расхождения по-прежнему будут разноситься на счет расхождений. Для перемещения взвешенного среднего (WMA) отклонения будут разнесены либо на счет отклонении, либо в запасы.</p><p>Этот вариант эффективен, только когда параметр **Выполнить разноску на счет накл. расходов в ГК** имеет значение *Да* на вкладке **Накладная** на странице **Параметры модуля расчетов с поставщиками**.</p> |

### <a name="validation-fasttab"></a>Экспресс-вкладка "Проверка"

В следующей таблице описаны поля, доступные на экспресс-вкладке **Проверка** вкладки **Общее** на странице **Параметры стоимости на складе**.

| Параметр | описание |
|---|---|
| Несколько накладных по затратам | Выбор того, что происходит, если для одних и тех же накладных расходов обрабатывается несколько накладных для рейса, листа или контейнера.<ul><li>**Принять** — система должна разрешать несколько накладных по затратам.</li><li>**Предупреждение** — выводится предупреждение.</li><li>**Ошибка** — выводится сообщение об ошибке.</li></ul> |
| Несколько поставщиков на лист | Выбор того, что происходит, если к листу добавляется более одного заказа на покупку от поставщика.<ul><li>**Принять** — система должна разрешать действие.</li><li>**Предупреждение** — выводится предупреждение, но действие все равно можно выполнить.</li><li>**Ошибка** — отображается сообщение об ошибке, и действие запрещено.</li></ul><p>Таможенный брокер или местное законодательство могут предписывать для этого поля конкретное значение.</p> |
| Отклонение по нескольким режимам доставки | Выбор того, что происходит, если товары из заказа на покупку, в котором используется другой способ доставки, а не рейс, добавляются в этот рейс.<ul><li>**Принять** — система должна разрешать действие.</li><li>**Предупреждение** — выводится предупреждение, но действие все равно можно выполнить.</li><li>**Ошибка** — отображается сообщение об ошибке, и действие запрещено.</li></ul> |
| Отклонение по нескольким складам | Выбор того, что происходит, если рейс содержит несколько строк заказа, которые должен быть доставлен на разные склады. Эти строки заказа могут распределяться по одному или нескольким заказам на покупку.<ul><li>**Принять** — система должна разрешать действие.</li><li>**Предупреждение** — выводится предупреждение.</li><li>**Ошибка** — выводится сообщение об ошибке.</li></ul> |
| Отсутствует объем | Выбор, что происходит, если пользователь добавляет номенклатуру без объема для рейса.<ul><li>**Принять** — система должна разрешить номенклатуру.</li><li>**Предупреждение** — выводится предупреждение.</li><li>**Ошибка** — выводится сообщение об ошибке.</li></ul><p>Если для расчета и распределения затрат используются объемы, рекомендуется выбрать либо *Предупреждение*, либо *Ошибка*.</p> |
| Поставщик услуг без затрат на поездку | Выбор того, что происходит, если пользователь пытается обработать накладную для поставщика услуг, который еще не был связан с рейсом. <ul><li>**Принять** — система должна разрешать действие.</li><li>**Предупреждение** — выводится предупреждение.</li><li>**Ошибка** — выводится сообщение об ошибке.</li></ul><p>Рекомендуется выбрать *Предупреждение*.</p> |

В следующей таблице описаны поля, доступные на экспресс-вкладке **Значения по умолчанию** вкладки **Общее** на странице **Параметры стоимости на складе**.

| Параметр | описание |
|---|---|

| Шаблон пути | Выберите шаблон пути по умолчанию, который будет использоваться для новых арендных морских контейнеров. Обычно выбирается шаблон пути, включающий стоимость аренды. |
| Проводка | Если чрезмерное/недостаточное количество для доставки находится в пределах допуска, журнал перемещения будет обрабатываться автоматически. Выберите используемый журнал перемещения по умолчанию. Поле **Корр. счет** для названия выбранного журнала движения должно быть заполнено. |

| Льготный период получения сверх количества товаров в пути | Укажите число дней после первого поступления контейнера отгрузки, когда дополнительные чрезмерные поступления могут быть по-прежнему выполнены для данного контейнера отгрузки. |

## <a name="status-updates-tab"></a>Вкладка обновлений статуса

Система использует значения статуса, чтобы указать статус каждого рейса. Значения статуса рейса могут автоматически применяться к рейсам с помощью отслеживания рейсов или периодических пакетных заданий. Кроме того, можно применить их вручную, открыв рейс, а затем выбрав статус в группе **Обновление рейсов** на вкладке **Управление** в области действий. 

Может быть создано любое значений статусов рейсов. Однако четыре из них должны определяться как используемые для специальных целей на вкладке **Обновления статуса** на странице **Параметры стоимости на складе**. В следующей таблице описываются поля, которые доступны.

| Параметр | описание |
|---|---|
| Произведен расчет себестоимости | Выбор статуса рейса, который указывает на то, что рейс был завершен. |
| В пути | Выбор статуса рейса, который указывает на то, что рейс находится в пути. |
| Готово к учету затрат | Выбор статуса рейса, который указывает на то, что рейс готов к учету затрат. Этот статус используется, когда все накладные по покупке для запасов и накладные затрат по рейсу, в которых поле **Кредит по затратам на рейс** задано как *Поставщик*, были обработаны для рейса. У рейсов с ошибкой процесса "учтено в затратах" будет статус *Готово к учету затрат*.|
| Произведен предварительный расчет себестоимости | Выбор статуса рейса, который указывает на то, что по рейсу выполняется предварительный учет затрат. Этот статус используется, когда новая проводка затрат добавляется к рейсу после того, как она уже была учтена в затратах. Новые проводки по затратам могут добавляться к рейсам с уже учтенными затратами при получении второй накладной по фрахту или непредвиденных расходах на простои. Этот статус автоматически применяется, когда к рейсу с учтенными затратами добавляются новые затраты на рейс. |

## <a name="voyage-creator-tab"></a>Вкладка создателя рейсов

В следующей таблице описаны разделы на вкладке **Создатель рейса** страницы **Параметры стоимости на складе**.

| Секция | описание |
|---|---|
| Допуски | Поля **Допустимый предел объема** и **Допустимый предел веса** определяют пороговые значения, выше которых товары считаются с чрезмерным объемом и чрезмерным весом. Когда пользователь добавляет товары на странице **Редактор рейсов**, если объем или вес превышает значение, заданное в этом поле, система отображает предупреждение. Значение каждого поля выражается в процентах от максимального объема или веса, заданного для соответствующего типа контейнера отгрузки. Рекомендуется, чтобы значение было 5 до 10 процентов от максимального объема или веса. |
| Настройка создания листа | Во время процесса создания рейса система может создать несколько листов. Этот раздел используется для определения времени создания нового листа. Для каждой строки данного раздела система проверит указанную таблицу и поле и создаст лист для каждого значения уникального поля. |

## <a name="cost-estimates-tab"></a>Вкладка оценки затрат

На вкладке **Оценки затрат** на странице **Параметры стоимости на складе** есть только одно поле: **Версия учета затрат по умолчанию**. Это поле применяется только в том случае, если методом учета затрат является *Стандартный учет затрат*. Выберите версию учета затрат по умолчанию, которая будет использоваться для периодической задачи *Обновление себестоимости номенклатуры*. Может возникнуть необходимость изменить этот параметр при каждом начале нового финансового года.

## <a name="inventory-dimensions-tab"></a>Вкладка складских аналитик

Вкладка **Складские аналитики** на странице **Параметры стоимости на складе** используется для управления, какие доступные складские аналитики должны отображаться по умолчанию на каждой странице стоимости на складе, где используются аналитики.

Выберите аналитику, а затем установите для параметров **Строки рейса**, **Товары в пути** и/или **Оценки затрат** значение *Да* для каждой страницы, на которой эта аналитика должна отображаться по умолчанию. Повторите этот шаг для других аналитик, если это необходимо.

Настройки на этой вкладке определяют аналитики по умолчанию для каждой назначенной страницы среди юридических лиц. Однако пользователи, работающие на одной из указанных страниц, могут переопределить аналитики по умолчанию, выбрав **Запасы \> Отобразить аналитики** в области действий.

## <a name="number-sequences-tab"></a>Вкладка "Номерные серии"

На вкладке **Номерные серии** страницы **Параметры стоимости на складе** перечислены все типы ссылочных номерных серий, которые требуются для стоимости на складе, но они не являются общими для юридических лиц. Для каждой ссылки в списке выберите код номерной серии.

> [!NOTE]
> В конфигурации с несколькими компаниями для каждой компании (юридического лица) должны быть созданы отдельные номерные серии.

## <a name="shared-number-sequences-tab"></a>Вкладка "Общие номерные серии"

На вкладке **Общие номерные серии** страницы **Параметры стоимости на складе** перечислены все типы ссылочных номерных серий, которые являются общими для юридических лиц для стоимости на складе. В настоящее время в списке есть только одна номерная серия. Эта номерная серия используется для кода рейса.

На странице **Все рейсы** пользователи могут просматривать все рейсы для всех юридических лиц. Однако для редактирования и обработки рейса пользователи должны быть в юридическом лице выбранной записи.

## <a name="feature-visibility-tab"></a>Вкладка «Видимость функций»

Стоимость на складе добавляет функции (поля и функции) на несколько часто используемых страниц Microsoft Dynamics 365 Supply Chain Management. Эти страницы содержат страницы, которые относятся к справочнику поставщика, выпущенным продуктам, заказам на покупку, заказам на перемещение и настройке склада. При использовании стоимости на складе необходимо сделать эти функции видимыми везде, и тогда они станут работоспособны. Если стоимость на складе не используется, можно скрыть функции, чтобы они не мешали.

На вкладке **Видимость функций** страницы **Параметры стоимости на складе** установите для параметра **Активировать** значение *Да*, чтобы функции стоимости на складе были видны везде, где они доступны. Задайте значение *Нет* для скрытия функций на общих страницах вне стоимости на складе. Однако даже если для параметра установлено значение *Нет*, сам модуль, включая страницу **Параметры стоимости на складе**, останется доступным для пользователей с необходимыми разрешениями на доступ к этому модулю.
