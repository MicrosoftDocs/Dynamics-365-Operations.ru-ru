# <a name="auto-costs-setup"></a>Настройка автозатрат

[!include [banner](../../includes/banner.md)]

Страницу **Автозатраты** можно использовать для настройки правил затрат для различных областей затрат (таких как рейсы, контейнеры отгрузки, листы, заказы на покупку, номенклатуры или строки заказа на перемещение). На основе правил и полей, выбранных пользователями при создании записей для одной из областей затрат, система рассчитывает затраты и добавляет их автоматически. Поэтому пользователям не придется вручную добавлять затраты.

Для работы с автоматическими затратами перейдите в раздел **Стоимость на складе \> Настройка затрат \> Автозатраты**. Затем настройте правила автоматических затрат, как описано в оставшейся части этого раздела.

## <a name="work-with-cost-areas"></a>Работа с областями затрат

Автоматические затраты действуют, как торговые соглашения или автоматические накладные расходы. Каждая запись автоматических затрат относится к конкретному уровню затрат. Поле **Область затрат** в области списка страницы **Автозатраты** используется для выбора области затрат, с которой необходимо работать. В области списка отображаются только те автоматические затраты, которые относятся к выбранной в данный момент области затрат. Все создаваемые автоматические затраты будут принадлежать выбранной в данный момент области затрат.

Области затрат позволяют системе распределять затраты по строкам заказа на покупку в области затрат. Например, стоимость одного из контейнеров отгрузки будет распределена по всем продуктам в данном контейнере отгрузки. Если имеется два или более контейнеров отгрузки, для каждого из контейнеров отгрузки может быть указан один и тот же тип затрат. Таким образом, тип контейнера может использоваться для поиска правильных автоматических затрат.

## <a name="buttons-on-the-action-pane"></a>Кнопки на панели операций

В следующей таблице описаны кнопки на панели операций страницы **Автозатраты**.

| Кнопка | описание |
|---|---|
| Редактировать | Редактирование существующих автоматических затрат. |
| Новое | Создание автоматических затрат. |
| Удаление | Удаление автоматических затрат. |
| Копировать из | Создайте новую запись автозатрат на основе существующей записи. После этого можно свободно редактировать новую запись. Эта кнопка позволяет быстро создать новые автоматические затраты. |
| Отобразить аналитики | Откройте диалоговое окно, в котором можно выбрать складские аналитики, которые отображаются для проводок затрат, которые вы просматриваете. Если снять флажки, для проводок затрат будет отображаться меньшее количество складских аналитик. Таким образом, для проводки будет отображаться меньшее количество сведений. Если для автозатрат указана складская аналитика, автоматические затраты будут применяться только в том случае, если для номенклатуры в рейсе используется указанная складская аналитика. |

## <a name="settings-on-the-header"></a>Параметры в заголовке

Сочетание параметров в разделе заголовка определяет, когда и как добавляются конкретные автоматические затраты в рейс. Автоматические затраты будут применены к рейсу, контейнеру отгрузки или листу только в том случае, когда сведения об автоматических затратах соответствуют сведениям заголовка этой области затрат. Сумма всех соответствующих автозатрат определяет ожидаемую стоимость на складе для рейса. Эти затраты распределяются по всем номенклатурам на судне или в рейсе.

В следующей таблице описываются все поля, которые доступны в разделе заголовка. Однако отдельные доступные поля могут различаться в зависимости от области затрат и отображаемых аналитик, которые в настоящее время выбраны.

| Поле | описание |
|---|---|
| **Код счета** | <p>Выберите одно из следующих значений:</p><ul><li>**Таблица** — правило затрат применяется только к конкретному поставщику.</li><li>**Группа** — правило затрат применяется только к конкретной группе. Система будет искать [группу типа затрат поставщика](costing-parameters-setup.md), связанную с записью поставщика.</li><li>**Все** — правило затрат применяется ко всем поставщикам. |
| **Компания по отгрузке** | Выберите поставщика или группу поставщиков, к которой относится правило затрат. Это поле доступно, только если выбран вариант *Таблица* или *Группа* в поле **Код счета**. |
| **Таможенный брокер** | Выберите поставщика или группу поставщиков, к которой относится правило затрат. Это поле доступно, только если выбран вариант *Таблица* или *Группа* в поле **Код счета**. |
| **Код номенклатуры** | <p>Выберите одно из следующих значений:</p><ul><li>**Таблица** — правило затрат применяется только к конкретной номенклатуре.</li><li>**Группа** — правило затрат применяется только к конкретной группе номенклатур. Система будет искать [группу типа затрат номенклатуры](costing-parameters-setup.md), связанную с записью номенклатуры.</li><li>**Все** — правило затрат применяется ко всем номенклатурам.|
| **Связь номенклатуры** | Выберите номенклатуру или группу номенклатур, к которой относится правило затрат. Это поле доступно, только если выбран вариант *Таблица* или *Группа* в поле **Код номенклатуры**. |
| **Способ поставки** | Выберите способ доставки (например, по воздуху или морю), к которому применяется правило затрат. |
| **Тип контейнера** | Тип контейнера отгрузки, в который будут отгружены товары. К рейсу могут быть применены автоматические затраты только в том случае, если тип контейнера в настройке автоматических затрат совпадает с типом контейнера рейса. |
| **Из порта** и **В порт** | Исходный порт ("из") и порт назначения ("в") для рейса. (У некоторых контейнеров отгрузки могут быть разные "в", так как рейс может остановиться в нескольких портах.) Автоматические затраты могут быть применены к рейсу только в том случае, если порты "из" и "в" в настройке автоматических затрат соответствуют портам "из" и "в" для рейса. |
| **Номер автоматических затрат** | Уникальный идентификатор правила автоматических затрат. Значение этого поля создается автоматически на основе номерной серии, настроенной на странице **Параметры стоимости на складе**. |
| **Складские аналитики** | Некоторые области затрат позволяют включить одну или несколько номенклатурных аналитик в заголовок (например, размер, цвет, склад и номер партии). Выберите **Отобразить аналитики** на панели операций для выбора аналитик, которые необходимо включить. |

## <a name="settings-on-the-lines-fasttab"></a>Настройки на экспресс-вкладке строк

На экспресс-вкладке **Строки** добавьте строку для каждой валюты, которая может использоваться, когда затраты применяются к строке заказа на покупку в рейсе. 

Для номенклатур и заказов на покупку система будет сопоставлять валюту каждой строки заказа на покупку с валютами, указанными на экспресс-вкладке **Строки**. Будет применена только себестоимость, установленная для соответствующей строки или номенклатуры. Если для валюты строки или номенклатуры не определена ни одна строка, автоматические затраты не будут применяться к этой строке или номенклатуре.

Для других типов областей затрат все строки на экспресс-вкладке **Строки** будут применяться к каждому рейсу, контейнеру отгрузки, листу или строке заказа на перемещение, которые соответствуют значениям, установленным в заголовке.

В следующей таблице описываются все поля, которые доступны в каждой строке. Однако отдельные доступные поля могут различаться в зависимости от области затрат, которая в настоящее время выбрана.

| Поле | описание |
|---|---|
| **Валютное** | Выберите валюту, к которой применяется строка. Эта валюта связана с заказом на покупку. Когда система пытается найти автоматические затраты, которые должны быть применены к рейсу или строкам рейса, она будет выбирать строку с той же валютой, что и в заказе на покупку. Это поле доступно только в том случае, если в поле **Область затрат** установлено значение *Заказ на покупку* или *Номенклатура*. |
| **Код типа затрат** | Тип затрат. |

Имеются следующие варианты:</p><ul><li><p>**Процент** — значение поля **Себестоимость** представляет собой процент, который применяется к общей стоимости товаров.</p><p>Например, если в поле **Себестоимость** установлено значение *10*, а общая стоимость товаров — $800, значение затрат — $80 (= $800 × 10 процентов).</p></li><li>**Количество** — затраты будут распределены в соответствии с количеством товаров.</li><li>**Сумма** — затраты будут распределены в соответствии со стоимостью товаров.</li><li>**Объем** — затраты будут распределены в соответствии с объемом товаров. Объем указан в главной записи номенклатуры.</li><li>**Вес** — затраты будут распределены в соответствии с весом товаров. Вес указан в главной записи номенклатуры.</li><li>**Объемный** — затраты будут распределены в соответствии с мерой объема товаров.</li><li><p>**Измерение** — этот параметр позволяет указать измерение в модуле **Стоимость на складе**. Это часто используется организациями, которые не знают отдельный объем или вес товаров, но требуют более точного распределения по сравнению с вариантами **Сумма** и **Количество**. Экспедитор груза или агент предоставит в организацию сведения о весе или мере объема на уровне номенклатур или на уровне заказа на покупку.</p><p>Например, морской фрахт равняется $900. Номенклатура A имеет вес 800 килограмм (кг) и объем 2 кубических метра (м³). У номенклатуры B вес 100 кг и объем 1 м³. Ниже приведены результаты распределения затрат по весу:</p><ul><li>Номенклатура A = $800</li><li>Номенклатура B = $100</li></ul><p>Ниже приведены результаты распределения затрат по объему:</p><ul><li>Номенклатура A = $600</li><li>Номенклатура B = $300</li></ul><p>**Примечание.** Когда для поля **Метод распределения** выбрано значение *Измерение*, система использует измерения, введенные для обеих областей затрат (контейнер отгрузки) и строк. Эти измерения не обязательно должны добавляться к ожидаемому итогу. Однако, если требуется, чтобы они добавлялись в ожидаемые итоги, можно выполнить проверку, используя статистику для просмотра общего измерения. Настройка запроса измерения и автоматическое обновление измерения на уровне отгрузки или контейнера устанавливаются в заголовке рейса.</p></li></ul> |
| **С даты** | Указывается начало диапазона дат для расчета себестоимости, если есть диапазон дат. Эта дата является первой датой, когда будут применяться автоматические затраты. |

| **Категория** | <p>Выберите метод, который должен использоваться для расчета затрат. Имеются следующие варианты:</p><ul><li>**Постоянные** — затраты будут распределены в соответствии с методом распределения.</li><li>**Штуки** — затраты распределяются по штукам. Поэтому метод распределения не используется.</li><li>**Процент** — будет добавлен процент стоимости товаров. Поэтому метод распределения не используется.</li><li>**Ставка** — выберите этот параметр, если имеются разбивки по количеству. Поле **Метод распределения** для данной строки должно быть установлено как *Измерение*.</li><ul> |
| **Разбивка по количеству** | <p>Установите этот флажок, чтобы кнопка **Разбивки по количеству** появилась на экспресс-вкладке **Строки**. Поле **Категория** для данной строки должно быть установлено как *Ставка*.</p><p>Количество соответствует параметру, выбранному в поле **Метод распределения**. | **Код валюты затрат** | Выберите валюту для затрат. |

В этом случае затраты могут быть автоматически рассчитаны на уровне контейнера отгрузки. Однако их также можно объединять и распределять по всем номенклатурам во всех контейнерах в рейсе вместо всех номенклатур в отдельных контейнерах отгрузки. |

Себестоимость оставшихся компонентов можно затем рассчитать, используя сумму, на которую был скорректирован предыдущий расчет.</p><p>Например, стоимость компонента A часов — $9,50 при пошлине 10 процентов, а стоимость компонента B — $0,50 при пошлине 2 процента. Следовательно, итоговая стоимость — $10,00. Первая запись предназначена для строки 10 процентов. Ее значения:</p><ul><li>**Категория:** *Процент*</li><li>**Себестоимость:** *10*</li><li>**Скорректированная стоимость:** *Сумма корректировки*</li><li>**Скорректированное значение:** *-0,50*</li></ul><p>Эта настройка указывает, что стоимость номенклатуры ($10) следует уменьшить на $0,50 (цена компонента B) до расчета затрат на 10-процентную пошлину. Таким образом, к $9,50 будет применяться 10 процентов.</p><p>Вторая запись относится к строке 2 процентов ($0,50, на которые была скорректирована предыдущая запись). Ее значения:</p><ul><li>**Категория:** *Процент*</li><li>**Себестоимость:** *2*</li><li>**Скорректированное значение:** *Использование*</li><li>**Корректировка:** *0,50*</li></ul><p>Этот параметр используется для расчета остатка пошлины, выплачиваемой по компоненту B.</p> |
