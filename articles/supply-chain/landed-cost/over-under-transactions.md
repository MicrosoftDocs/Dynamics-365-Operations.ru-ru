# <a name="overunder-transactions"></a>Проводки перепоставки/недопоставки

[!include [banner](../../includes/banner.md)]

При обработке заказов в рейсе, система ожидает, что количество номенклатур, полученное в конечном складе назначения, для потребления будет совпадать с количеством, указанным в строках заказа на покупку, связанных с данным рейсом. Однако поскольку точное количество в строках заказа на покупку не всегда получается на складе, в модуле **Стоимость на складе** определяется набор правил, используемых для обработки избыточной и недостаточной приемки товаров. Эти правила особенно важны, поскольку для исходного заказа на покупку была выставлена накладная, которая больше не может быть изменена. После настройки сведений о политиках для чрезмерных/недостаточных проводок система может определить, как управлять чрезмерной обработкой и недостаточной обработкой товаров на момент получения. Можно также вручную управлять чрезмерными и недостаточными запасами с помощью **Чрезмерные/недостаточные проводки**.

## <a name="overunder-tolerances"></a>Допустимые отклонения перепоставки/недопоставки

Вы настраиваете допуски перепоставки и недопоставки, чтобы указать чрезмерные и недостаточные количества или объемы, которые могут быть обработаны в рейсе без возникновения ошибки. Если получение строки рейса находится за пределами этих допусков, оно должно быть изменено и разрешено до того, как строка рейса может быть закрыта для дальнейшей обработки.

Для настройки допусков перейдите **Стоимость на складе \> Настройка чрезмерных/недостаточных значений \> Допуски для чрезмерных/недостаточных значений**. Там можно просматривать, изменять, добавлять и удалять записи допусков. В следующей таблице описываются поля, которые доступны для каждой записи.

| Поле | описание |
|---|---|
| Код счета | <p>Определите область поставщиков, к которой применяется допуск, выбрав одно из следующих значений:</p><ul><li>**Таблица** — допуск чрезмерных/недостаточных допусков применяется только к поставщику, выбранному для строки.</li><li>**Группа** — допуск чрезмерных/недостаточных допусков применяется ко всем поставщикам в группе допусков поставщиков, выбранной для строки.</li><li>**Все** — чрезмерный/недостаточный допуск применяется ко всем поставщикам.</li></ul> |
| Связь счета | Выберите поставщика или группу поставщиков, к которой применяется допуск чрезмерного/недостаточного значения, в зависимости от значения, выбранного в поле **Код счета**. |
| Код номенклатуры | <p>Определите область номенклатур, к которой применяется допуск, выбрав одно из следующих значений:</p><ul><li>**Таблица** — допуск чрезмерных/недостаточных допусков применяется только к номенклатуре, выбранной для строки.</li><li>**Группа** — допуск чрезмерных/недостаточных допусков применяется ко всем номенклатурам в группе допусков номенклатур, выбранной для строки.</li><li>**Все** — чрезмерный/недостаточный допуск применяется ко всем номенклатурам.</li></ul> |
| Связь номенклатуры | Выберите номенклатуру или группу номенклатур, к которой применяется допуск чрезмерного/недостаточного значения, в зависимости от значения, выбранного в поле **Код номенклатуры**. |
| Отклонение суммы | Введите допуск суммы, который должен применяться ко всему заказу на покупку. |
| Отклонение в процентах | Введите допуск процента, который должен применяться к отдельной строке заказа на покупку. |

## <a name="overunder-reasons"></a>Причины перепоставки/недопоставки

Когда чрезмерное или недостаточное количество связано со строкой выполнения, которая получена, возможно, потребуется найти причину чрезмерного или недостаточного количества. В этом случае можно выбрать причину перепоставки или недопоставки в строке приемки при получении товаров.

Чтобы настроить причины перепоставки и недопоставки, перейдите **Стоимость на складе \> Настройка чрезмерных/недостаточных значений \> Причины чрезмерных/недостаточных значений**. Там можно просматривать, изменять, добавлять и удалять доступные причины перепоставки и недопоставки. В следующей таблице описываются поля, которые доступны для каждой причины.

| Поле | описание |
|---|---|
| Причина перепоставки/недопоставки | Введите уникальное имя причины для проводки с перепоставкой или недопоставкой. |
| описание | Введите описание причины чрезмерных/недостаточных значений. |
| Проводка | Выберите журнал проводок, связанный с причиной чрезмерных/недостаточных значений. В этом поле перечислены все доступные журналы, с которыми связан тип журнала *Перемещение* на странице **Наименования журналов запасов** (**Настройка управления запасами \> Наименования журналов \> Запасы**). |

## <a name="item-overunder-tolerance-groups"></a>Группы допустимых отклонений перепоставки/недопоставки номенклатуры

Номенклатуры с аналогичными допусками могут быть сгруппированы вместе. Таким образом можно настроить допуск чрезмерных/недостаточных значений для всех номенклатур в этой группе одновременно.

Чтобы настроить группы допуска чрезмерных/недостаточных значений номенклатур, перейдите **Стоимость на складе \> Настройка чрезмерных/недостаточных значений \> Группы допуска чрезмерных/недостаточных значений номенклатур**. Там можно просматривать, изменять, добавлять и удалять записи групп допуска чрезмерных/недостаточных значений. В следующей таблице описываются поля, которые доступны для каждой записи.

| Поле | описание |
|---|---|
| Группа допустимых отклонений перепоставки/недопоставки | Введите уникальное имя группы. Выберите имя, описывающее допуск, например *1% Var*. |
| описание | Введите описание группы. |

## <a name="vendor-overunder-tolerance-groups"></a>Группы допустимых отклонений перепоставки/недопоставки поставщика

Можно сгруппировать поставщиков, которые регулярно перепоставляют или недопоставляют. Затем можно установить допуск чрезмерных/недостаточных значений для группы.

Чтобы настроить группы допуска чрезмерных/недостаточных значений поставщиков, перейдите **Стоимость на складе \> Настройка чрезмерных/недостаточных значений \> Группы допуска чрезмерных/недостаточных значений поставщиков**. Там можно просматривать, изменять, добавлять и удалять записи групп допуска чрезмерных/недостаточных значений. В следующей таблице описываются поля, которые доступны для каждой записи.

| Поле | описание |

| Групп чрезмерных/недостаточных значений | Введите уникальное имя группы. Выберите имя, которое описывает тип поставщиков, принадлежащих к данной группе. |
| описание | Введите описание группы. |

## <a name="view-and-process-overunder-transactions"></a>Просмотр и обработка проводок чрезмерных/недостаточных значений

Проводки чрезмерных/недостаточных значений создаются, когда количество товаров для размещения не совпадает с инициализированным количеством. Количество в журнале прибытия должно обновляться только с учетом количества размещенного.

Когда обрабатывается приемка строк рейса, для поставщика могут быть настроены допуски чрезмерных/недостаточных значений. Затем номенклатуры будут просмотрены и проверены. Допуск можно установить как процент, локальную сумму или оба этих значения.

Если полученная номенклатура находится в пределах допуска, система создаст журнал перемещения. Этот журнал можно указать на странице параметров рейса. Кроме того, будет создана проводка чрезмерных/недостаточных значений. Например, стоимостью заказа является $100, значение прихода равно $99, и эти значения соответствуют допуску. В этом случае будет создано отрицательный журнал перемещения для недопоставленного количества.

Если полученная номенклатура находится вне допуска, система генерирует проводку чрезмерных/недостаточных значений для разности в количестве.

Для просмотра и обработки проводок чрезмерных/недостаточных значений перейдите **Стоимость на складе \> Запросы \> Проводки чрезмерных/недостаточных значений**. В этом случае проводка чрезмерных/недостаточных значений будет связана со всеми номенклатурами в рейсе, которые перепоставлены или недопоставлены. Для разрешения всех проводок чрезмерных/недостаточных значений, связанных с рейсами, рекомендуется использовать страницу **Проводки чрезмерных/недостаточных значений**. Кроме того, рекомендуется **не** использовать журналы перемещения или журналы инвентаризации для ручного разрешения складских проводок недопоставки. Вместо этого следует использовать страницу **Проводки чрезмерных/недостаточных значений** для управления количествами недопоставки на склад.

### <a name="process-overunder-transactions"></a>Обработка проводок чрезмерных/недостаточных значений

Панель операций на странице **Проводки чрезмерных/недостаточных значений** содержит следующие команды для обработки проводок, выбранных на странице. Каждая команда позволяет выбрать способ обработки каждой проводки.

- **Создать \> Перемещение** — создание и разноска журнала перемещения для выбранной проводки. Для проводок недостаточных значений журнал перемещения автоматически создается и разносится в соответствии с разницей между ожидаемым и фактически полученным количеством. Выберите эту команду для проводок чрезмерных значений, если вы хотите, чтобы поставщик компенсировал разницу в стоимости.
- **Создать \> Заказ на покупку** — создайте заказ на покупку для разницы между ожидаемым и фактически полученным количеством по выбранной проводке. Выберите эту команду для проводок чрезмерных значений, если ваша организация будет компенсировать разницу в стоимости.
- **Создать \> Перенос в место назначения** — эта команда применяется только к заказам на перемещение. Этот флажок необходимо установить, если требуется переместить чрезмерное или недостаточное количество на склад назначения.
- **Создать \> Перенос в место происхождения** — эта команда применяется только к заказам на перемещение. Этот флажок необходимо установить, если требуется переместить чрезмерное или недостаточное количество на исходный склад.
