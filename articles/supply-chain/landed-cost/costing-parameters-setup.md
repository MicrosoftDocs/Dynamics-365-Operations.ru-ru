# <a name="costing-parameter-values-setup"></a>Настройка значений параметров учета затрат

[!include [banner](../../includes/banner.md)]

При настройке модуля **Стоимость на складе** можно определить несколько наборов общих значений и связанных параметров для каждого значения. Эти значения затем будут доступны при выборе конкретных типов значений параметров учета затрат в других частях приложения. В этой теме описан порядок настройки таких наборов значений.

## <a name="set-up-cost-type-codes"></a>Настройка кодов типов затрат

Коды типов затрат определяют тип затрат, которые производятся при выгрузке товаров на склад, или стоимость рейса на складе. Хотя они обычно увеличивают стоимость товаров, они могут также использоваться для начисления сумм в главной книге. Корректировки ГК используются, когда стоимость начисляется со временем или в ходе ряда рейсов и смещения в одной проводке.

> [!NOTE]
> Если таблица типов затрат используется совместно в юридических лицах, план счетов также должен быть с общим доступом среди юридических лиц. В противном случае проводки разноски будут работать неправильно.

Для настройки кодов типов затрат перейдите **Стоимость на складе \> Настройка учета затрат \> Коды типов затрат**. Можно использовать кнопки на панели операций для создания новых кодов типов затрат, изменения существующих кодов или удаления выбранного типа затрат.

В следующей таблице описываются поля, которые доступны для каждого кода типа затрат.

| Поле | описание |
|---|---|
| Код типа затрат | Укажите имя кода типа затрат. |
| описание | Введите описание кода типа затрат. |
| Использовать ставку отгрузки | Установите для этого параметра значение *Да*, если валютный курс рейса (иногда называемый ставкой управления) должен использоваться для расчета стоимости этих затрат. В этом случае используется ставка отгрузки, а не стандартные значения по умолчанию или валютный курс места для накладных в иностранной валюте. |
| Категория отчетности | В этом поле создается категория отчетности для данного типа затрат. Отчеты могут быть распечатаны либо с помощью категорий отчетности, либо по типу затрат. |
| Тип дебетования | Выберите, должен ли тип затрат дебетовать номенклатуру, счет ГК или поставщика. |
| Разноска по дебету | Если для поля **Тип дебета** установлено значение *Счет ГК*, выберите описание разноски. |
| Дебетовый счет | Если для поля **Тип дебета** установлено значение *Счет ГК*, выберите используемую ГК. |
| Тип кредитования | Выберите, должен ли этот тип затрат кредитовать номенклатуру, счет ГК или поставщика. |
| Кредитовая разноска | Если для поля **Тип кредита** установлено значение *Счет ГК*, выберите описание разноски. |
| Кредитовый счет | Если для поля **Тип кредита** установлено значение *Счет ГК*, выберите используемую ГК. |
| Клиринговый счет | Выберите клиринговый счет для типа затрат. Рекомендуется указать отдельный клиринговый счет для каждого типа затрат для упрощения выверки. |
| Тип разноски стандартной себестоимости | Если используется стандартный учет затрат, выберите описание разноски. |
| Счет стандартного отклонения стоимости | <p>Если используется стандартный учет затрат, указанный здесь счет будет использоваться для разноски каких-либо отклонений. Этот счет использует детализацию затрат на складе на странице **Цены номенклатур**. Эта разбивка создается с помощью периодической программы для обновления цен.</p><p>Например, стандартная себестоимость номенклатуры — $15,00, АOB — $13,00, а фрахт — $2,00. При получении накладной для запаса номенклатура поступает при $15,00, но для номенклатуры имеется стандартное отклонение по цене $2,00, поскольку фактический FOB равен $13,00. Это отклонение разносится на стандартный счету отклонения по цене, который настроен в профиле разноски номенклатуры. Так как ожидаемый фрахт составляет $2,00, то при разноске накладной запасов отклонения нет. Однако при получении накладной для фрахта фрахт равен $2,50 за единицу. Таким образом, отклонение $0,50 разносится по указанным затратам. |
| Счет отклонения скользящего среднего | <p>Если используется учет затрат по скользящему среднему, указанный здесь счет будет использоваться для разноски каких-либо отклонений.</p><p>Например, ожидаемый фрахт равен $2,00. Однако при получении накладной для фрахта фрахт равен $2,50 за единицу. Таким образом, следует разнести отклонение $0,50.</p><p>Если параметр **Разносить корректировки как отклонение** задан как *Да* на странице **Параметры стоимости на складе**, все отклонения между ожидаемыми и фактическими затратами на отгрузку будут разнесены на счет отклонения по скользящему среднему, указанный здесь. Если параметр **Разносить корректировки как отклонение** задан как *Нет*, будут использоваться стандартные функции, а отклонение будет применено либо к запасам, либо к счету отклонения по скользящему среднему, который указан здесь, в зависимости от запасов в наличии.</p> |
| Счет начисления накладных расходов | Выберите счет, который используется для начисления оценок затрат при разноске накладной по покупке. Это поле используется только в том случае, если параметр **Использовать счет начисления накладных расходов типа затрат** задан как *Да* на экспресс-вкладке **Учет затрат** на вкладке **Общее** на странице **Параметры стоимости на складе**. |
| Расходный счет | Выберите счет, используемый для сбора входящих транспортных издержек, по которым был выставлен счет поставщиком. Сумма разносится как дебет. Корр. счет является счетом изменения запасов. Эта разноска используется, только когда параметр **Выполнить разноску на счет накл. расходов в ГК** имеет значение *Да* на странице **Параметры модуля расчетов с поставщиками**. |
| Счет отклонения | Счет, который будет использоваться для смещения начислений накладных расходов при разноске накладной по закупке. Это поле используется только в том случае, если параметр **Использовать счет начисления накладных расходов типа затрат** задан как *Да* на экспресс-вкладке **Учет затрат** на вкладке **Общее** на странице **Параметры стоимости на складе**. |

## <a name="vendor-cost-type-groups"></a>Группы типов затрат поставщика

Группы типов затрат поставщика помогают определить, как будут определяться *автоматические затраты* и применяться к рейсу. Поставщики с аналогичными затратами на импорт, связаны друг с другом. Например, все поставщики из развивающихся рынков платят одинаковый процент пошлины для одного и того же типа продукта, который куплен на сформировавшемся рынке.

Группы типов затрат поставщика можно настроить, перейдя в **Стоимость на складе \> Настройка учета затрат \> Группы типов затрат поставщика**. На странице **Группы типов затрат поставщика** есть сетка, в которой перечислены все существующие группы типов затрат поставщика. Можно использовать кнопки на панели операций для добавления, удаления и редактирования строк в сетке.

В следующей таблице описываются поля, которые доступны в каждой строке в сетке.

| Поле | описание |
|---|---|
| Группы типов затрат поставщика | Введите уникальное имя для группы типов затрат поставщика (например, *Развивающиеся рынки*). |
| описание | Введите описание группы типа затрат поставщика. В описании могут быть представлены подробные сведения об уровне или типе накладных расходов, связанных с группой поставщиков. |

## <a name="item-cost-type-groups"></a>Группы типов стоимости номенклатуры

Можно использовать кнопки на панели операций для добавления, удаления и редактирования строк в сетке.

| Группы типов стоимости номенклатуры | Введите уникальное имя для группы типов затрат номенклатуры (например, *Пошлина 5%*). |

## <a name="cost-templates"></a>Шаблоны затрат

Шаблоны затрат используются для задания значений по умолчанию для параметров, которые могут быть неизвестны пользователи, получившим оценку затрат. Шаблоны затрат позволяют упростить процесс оценки путем минимизации вариантов выбора, которые пользователь должен указать для получения точной оценки.

Для работы с шаблонами затрат перейдите **Стоимость на складе \> Настройка учета затрат \> Шаблоны затрат**. На странице **Шаблоны затрат** в области списка слева отображаются все текущие шаблоны затрат. Можно использовать кнопки на панели операций для добавления, удаления и редактирования шаблонов.

В следующей таблице описываются параметры, которые доступны для каждого шаблона.

| Поле | описание |
|---|---|
| Шаблон затрат | Введите уникальное имя шаблона затрат. Название обычно описывает коэффициент или множитель затрат для шаблона. |
| описание | Введите описание шаблона затрат. |
| Компания по отгрузке | Выберите счет поставщика для транспортной компании, связываемый с шаблоном затрат. |
| Способ поставки | Выберите способ доставки, который должен использоваться шаблоном затрат при расчете оценочной стоимости номенклатуры. Это поле помогает определить автоматические затраты, связанные с этими товарами в оценке затрат. |
| Тип грузового контейнера | Выберите тип контейнера отгрузки, связываемый с шаблоном затрат. Это поле помогает определить автоматические затраты, связанные с этими товарами в оценке затрат. |
| Таможенный брокер | Выберите таможенного брокера (поставщика) для связи с шаблоном затрат. Это поле помогает определить автоматические затраты, связанные с этой оценкой затрат. |
| Множитель | Введите коэффициент, который будет применяться к окончательной оценке затрат на товары. Например, чтобы добавить 10 процентов к вычисленной оценке затрат, введите *1,10*. |

## <a name="volumetric-divisors"></a>Волюметрические делители

Объемные делители используются для расчета объемного веса. Каждая транспортная компания или компания-фрахтовщика формулирует свои собственные объемные делители. Кроме того, делители компании обычно изменяются в зависимости от способа доставки. Например, у воздушных и морских перевозок часто очень разные делители. Компания также может усложнить свои правила в зависимости от того, откуда она отгружает товар.

Например, у пакета, отправляемого по воздуху, объем 3 кубических метра (м³). Компания оплачивает расходы по объемному весу и применяет объемный делитель 6. Этот делитель умножается на объем, чтобы определить объемный вес. Таким образом, объемный вес для этого примера составляет 3 × 6 = 18 килограмм (кг).

Для настройки объемных делителей перейдите **Стоимость на складе \> Настройка учета затрат \> Объемные делители**. На странице **Объемные делители** есть сетка, в которой перечислены все существующие объемные делители. Можно использовать кнопки на панели операций для добавления, удаления и редактирования строк в сетке.

В следующей таблице описываются поля, которые доступны в каждой строке в сетке.

| Поле | описание |
|---|---|
| Компания по отгрузке | Выберите счет поставщика транспортной компании, связанный с объемным делителем. |
| Код типа затрат | Выберите код типа затрат, который связан с объемным делителем. Используйте это поле для преобразования типов затрат в сегменты отчетности. Отчеты могут быть распечатаны либо с помощью категорий отчетности, либо по типу затрат. |
| Порт отправления | Выберите исходный порт ("из"), к которому применяется объемный делитель. |
| Волюметрический делитель | Выберите значение объемного делителя, который применяется к строке. Введенное значение будет *умножаться* на объем каждого пакета, чтобы определить объемный вес этого пакета. |
