# <a name="automatic-allocation-of-charges"></a>Автоматическое распределение накладных расходов

[!include [banner](../includes/banner.md)]

На основе клиента, с которым ведется работа, или номенклатуры, которую вы продаете, может потребоваться применение особых накладных расходов. Функция *накладных расходов* в Microsoft Dynamics 365 Supply Chain Management помогает автоматически распределять расходы по заказам на покупку или заказам на продажу.

Автоматические накладные расходы или авторасходы, автоматически применяются при создании заказа на продажу или заказа на покупку. Можно определить расходы для конкретных поставщиков, клиентов, групп поставщиков или номенклатур. Также можно определить автоматические расходы, которые применяются ко всем поставщикам, клиентам или номенклатурам.

## <a name="set-up-parameters"></a>Настройка параметров

На странице **Параметры модуля "Закупки и источники"** есть несколько настроек, которые особенно подходят, если требуется распределять накладные расходы автоматически. Чтобы завершить эту настройку, выполните следующие действия.

1. Выберите **Закупки и источники \> Настройка \> Параметры модуля "Закупки и источники"**.
1. Откройте вкладку **Цены**.
1. На экспресс-вкладке **Цены** выполните следующие настройки:
    - **Поиск автоматических накладных расходов для заголовка** — определяет, должны ли накладные расходы автоматически распределяться по заголовкам заказов на покупку. Установите значение *Да* для использования автоматического распределения накладных расходов.
    - **Поиск автоматических накладных расходов для строки** — определяет, должны ли накладные расходы автоматически распределяться по строкам заказов на покупку. Установите значение *Да* для использования автоматического распределения накладных расходов.

## <a name="set-up-charges-codes"></a>Настройка кодов накладных расходов

Прежде чем распределять расходы, необходимо определить коды накладных расходов.

1. Выполните одно из следующих действий.

    - Для заказов на покупку: перейдите в раздел **Расчеты с поставщиками \> Настройка накладных расходов \> Код накладных расходов**.
    - Для заказов на продажу: перейдите в раздел **Расчеты с клиентами \> Настройка накладных расходов \> Код накладных расходов**.

1. На панели операций выберите **Создать** для создания кода накладных расходов.
1. В заголовке новой записи задайте следующие поля:

    - **Код расходов** — введите код для накладных расходов.
    - **Описание** — введите описание накладных расходов.
    - **Налоговая группа номенклатур** — выберите налоговую группа номенклатур, если применимо.
    - **Пропорциональное распределение** — установите для этого параметра значение *Да*, если требуется пропорциональное распределение накладных расходов. Этот параметр доступен только для заказов на продажу.
    - **Максимальная сумма** — введите максимальную сумму, разрешенную для этого кода расходов. Это поле используется для проверки накладных расходов для накладных поставщиков. Он доступен только для заказов на покупку.

        > [!NOTE]
        > Чтобы включить функцию для проверки накладных расходов по заказам на покупку, перейдите в раздел **Расчеты с поставщиками \> Настройка \> Параметры модуля расчетов с поставщиками**. На экспресс-вкладке **Проверка накладной** в разделе **Проверка накладной** установите для параметра **Включить проверку сопоставления накладных** значение *Да*.

1. Экспресс-вкладка **Разноска** включает разделы **Дебет** и **Кредит**. Установите следующие поля в зависимости от книги учета, в которую требуется разнести накладные расходы:

    - **Тип** — выберите тип счета, на который выполняется разноска (*ГК*, *Клиент* или *Номенклатура*).
    - **Разноска** — выберите тип разноски для создания (например, *Комиссия брокера* или *Сопоставление по клиенту*).
    - **Счет** — выберите счет для разноски накладных расходов.

1. На панели операций выберите **Сохранить**.

## <a name="create-charge-groups"></a>Создание групп накладных расходов

Группы накладных расходов автоматически распределяют отдельные накладные расходы для группы клиентов или поставщиков. В следующих подразделах описывается порядок создания и назначения этих групп расходов.

### <a name="charge-groups-for-purchase-orders"></a>Группы накладных расходов для заказов на покупку

Чтобы создать группы накладных расходов для заказов на покупку, выполните следующие шаги.

1. Перейдите в раздел **Расчеты с поставщиками \> Настройка накладных расходов \> Группа накладных расходов по поставщикам**.
1. На панели операций выберите **Создать**, чтобы добавить строку в сетку, затем задайте следующие поля:

    - **Группа накладных расходов** — введите имя группы накладных расходов.
    - **Описание** — введите описание группы накладных расходов.

1. На панели операций выберите **Сохранить**.
1. Перейдите в раздел **Расчеты с поставщиками \> Поставщики \> Все поставщики** и либо откройте существующего поставщика, либо создайте нового поставщика.
1. На экспресс-вкладке **Значения заказа на покупку по умолчанию** в разделе **Заказ на покупку** задайте в поле **Группа накладных расходов** только что созданную группу накладных расходов.

### <a name="charge-groups-for-sales-orders"></a>Группы накладных расходов для заказов на продажу

Чтобы создать группы накладных расходов для заказов на продажу, выполните следующие шаги.

1. Перейдите в раздел **Расчеты с клиентами \> Настройка накладных расходов \> Группы накладных расходов по клиентам**.
1. На панели операций выберите **Создать**, чтобы добавить строку в сетку, затем задайте следующие поля:

    - **Группа накладных расходов** — введите имя группы накладных расходов.
    - **Описание** — введите описание группы накладных расходов.

1. На панели операций выберите **Сохранить**.
1. Перейдите в раздел **Расчеты с клиентами \> Клиенты \> Все клиенты** и либо откройте существующего клиента, либо создайте нового клиента.
1. На экспресс-вкладке **Значения заказа на покупку по умолчанию** в разделе **Заказ на продажу** задайте в поле **Группа накладных расходов** только что созданную группу накладных расходов.

## <a name="define-auto-charges"></a>Определение автоматических накладных расходов

После настройки кодов накладных расходов выполните следующие действия, чтобы определить автоматические накладные расходы.

1. Выполните одно из следующих действий.

    - Для заказов на покупку: перейдите в раздел **Закупки и источники \> Настройка \> Накладные расходы \> Автоматические списания**.
    - Для заказов на продажу: перейдите в раздел **Расчеты с клиентами \> Настройка \> Настройка накладных расходов \> Автоматические затраты**.

1. В области списка в поле **Уровень** выберите уровень, на который применяются автоматические расходы:

    - *Основной* — применение расходов к заголовку заказа.
    - *Строка* — применение расходов к строкам заказа.

1. Выберите существующие автоматические накладные расходы для редактирования или выберите **Создать**, чтобы определить новый автоматический расход.
1. В списке **Код счета** выберите одно из следующих значений, чтобы указать область счетов, которые будут затронуты:

    - *Таблица* — назначьте расходы конкретному клиенту или поставщику.
    - *Группа* — назначьте расходы группе накладных расходов.
    - *Все* — присвойте расходы всем клиентам или поставщикам.

1. В поле **Связь клиента** или **Связи поставщика** выберите конкретного клиента или поставщика, если вы в поле **Код счета** задали значение *Таблица*. Если в поле **Код счета** выбрано значение *Группа*, выберите группу накладных расходов для клиента или поставщика.
1. В поле **Код номенклатуры** выберите одно из следующих значений, чтобы указать область номенклатур, которые будут затронуты. Можно выбрать код номенклатуры, только если автоматические расходы определены на уровне строк.

    - *Таблица* — назначьте расходы конкретной номенклатуре.
    - *Группа* — назначьте расходы группе расходов по номенклатуре.
    - *Все* — назначьте расходы всем номенклатурам.

1. В поле **Связь номенклатуры** выберите конкретную номенклатуру, если в поле **Код номенклатуры** задано значение *Таблица*. Если в поле **Код номенклатуры** задано значение *Группа*, выберите группу накладных расходов по номенклатурам.
1. **Только для заказов на продажу:** в поле **Код способа поставки** выберите одно из следующих значений, чтобы указать область режимов поставки, которая будет затронута:

    - *Таблица* — назначьте расходы конкретному способу доставки.
    - *Группа* — назначьте расходы группе способов доставки.
    - *Все* — назначьте расходы всем способам доставки.

1. **Только для заказов на продажу:** в поле **Связь способа поставки** выберите конкретный способ поставки, если в поле **Код способа поставки** задано значение *Таблица*. Если в поле **Код способа поставки** задано значение *Группа*, выберите режим группы поставки.
1. На экспресс-вкладке **Строки** определите расходы и ставки расходов, которые будут использоваться при применении текущего автоматического расхода. С помощью панели инструментов на этой экспресс-вкладке можно добавить столько строк, сколько требуется. Для каждой строки установите следующие поля:

    - **Валюта** — выберите валюту, которая должна использоваться для расчета расходов.
    - **Код накладных расходов** — выберите код для расхода.
    - **Категория** — выберите одно из следующих значений:

        - *Фиксированная сумма* — расход вводится в виде фиксированной суммы в строке. Фиксированные расходы могут использоваться как в заголовке заказа, так и в строках заказа.
        - *Шт.* —расход рассчитывается по количеству единиц продукции. Эти расходы можно использовать только в строках заказа. Они будут отображаться при расчете итогового значения заказа.
        - *Процент* — расход вводится в виде процентного отношения по строке. Процентные расходы могут использоваться как в заголовке заказа, так и в строках заказа.
        - *Внутрихолдинговый процент* — расход вводится как процентное отношение в строках внутрихолдинговых заказов. Внутрихолдинговые процентные расходы можно использовать только в строках заказа.
        - *Внешний* — расход рассчитывается сторонней службой, связанной с одним или более перевозчиков.

    - **Значение расходов** — введите сумму накладных расходов на основе выбранной категории.
    - **Код валюты накладных расходов** — укажите валюту для расходов, если нужно использовать валюту, отличную от валюты, указанной в поле **Валюта**. Другую валюту можно использовать только в том случае, если в поле **Тип дебета** или **Тип кредита** задано значение *Счет ГК* или *Номенклатура* для выбранного кода расходов.
    - **Начальная сумма** — укажите начальную сумму, к которой нужно применить автоматический расход. В этом контексте сумма относится к общему количеству заказа.
    - **Конечная сумма** — укажите конечную сумму, к которой нужно применить автоматический расход. В этом контексте сумма относится к общему количеству заказа.
    - **Налоговая группа** — указывается налоговая группа.
    - **Сайт** и **Склад** — укажите сайт и склад,, если накладные расходы должны применяться только для определенного сайта и склада.
    - **Сохранить** — установите этот флажок, чтобы сохранить проводки с расходами после завершения выставления счетов, чтобы расход применялся всякий раз, когда создается новая накладная для выбранного счета клиента.

1. **Только для заказов на продажу:** если требуется рассчитать многоуровневые расходы, см. информацию в разделе [Многоуровневые расходы по заказам на продажу](/dynamicsax-2012/appuser-itpro/about-tiered-charges-on-sales-orders).

## <a name="allocate-charges-from-the-header-to-a-line"></a>Распределение расходов из заголовка в строку

Следующая процедура показывает, как распределять расходы уровня заголовка по строке. Перед началом этой процедуры необходимо, чтобы у вас уже были расходы уровня заголовков для типа *фиксированная сумма* и заказ, в котором применяется накладные расходы. Кроме того, в заказе должен уже быть хотя бы один элемент строки.

1. Откройте заказ на покупку или заказ на расходы.
1. В области действий выполните один из следующих шагов:

    - Для заказов на покупку: на вкладке **Покупка** в группе **Накладные расходы** выберите **Распределить накладные расходы**.
    - Для заказов на продажу: на вкладке **Продажа** в группе **Накладные расходы** выберите **Распределить накладные расходы**.

1. В диалоговом окне **Распределение расходов по строкам заказа** установите следующие поля:

    - **Распределение расходов** — выберите одно из следующих значений, чтобы указать, как должны распределяться накладные расходы:

        - *Чистая сумма* — расходы распределяются в соответствии с суммой каждой строки по отношению к общей чистой сумме.
        - *Количество* — расходы распределяются в соответствии с числом единиц измерения для каждой строки по отношению к общему числу единиц измерения.
        - *По каждой строке* — расходы распределяются поровну среди общего числа строк.

    - **Распределить расходы по строкам** — выберите значение, чтобы указать, должны ли накладные расходы распределяться по всем строкам, только по положительным строкам или по отрицательным строкам.
    - **Распределить все** — установите этот флажок, чтобы распределить расходы по строкам заказа, даже если код накладных расходов имеет тип дебета, отличный от *Номенклатура*.
    - **Получено** — установите этот флажок, чтобы распределить расходы только по строкам полученного заказа.
    - **Учитываемый в запасах** — установите этот флажок, чтобы распределить расходы только по строкам заказа, учитываемых в запасах.
    - **Показать выборки и очистить определенные строки** — установите этот флажок, чтобы исключить отдельные строки из этого распределения. Если этот флажок установлен, открывается сетка **Выберите строки для исключения из распределения**. Эта сетка содержит только те строки, которые соответствуют критериям, определенным параметрами **Распределить накладные расходы по строкам** и **Учитываемый в запасах**. Например, если задать в поле **Распределить накладные расходы по строкам** значение *Положительные строки* и установите флажок **Учитываемый в запасах**, в сетке отображаются только строки, которые одновременно положительные и учитываемые в запасах. Кроме того, сетка автоматически фильтрует все строки, для которых уже было получено полное количество. Когда сетка открыта, снимите флажок **Включить** для каждой строки, которую следует исключить из распределения. 

        > [!IMPORTANT]
        > При работе с сеткой **Выберите строки для исключения из распределения** сетка обязательно должна оставаться открытой, пока вы не выберите **Распределить**. Если вы закроете сетку до выбора пункта **Распределить**, ваши настройки в сетке будут потеряны. Таким образом, расходы будут распределяться на основе ранее определенного критерия.

1. Выберите **Распределить**, чтобы применить настройки и закрыть диалоговое окно.
