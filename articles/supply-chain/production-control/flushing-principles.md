## <a name="description-of-the-flushing-principles"></a><span data-ttu-id="f453d-125">Описание принципов очистки</span><span class="sxs-lookup"><span data-stu-id="f453d-125">Description of the flushing principles</span></span>

### <a name="manual"></a><span data-ttu-id="f453d-126">Руководство</span><span class="sxs-lookup"><span data-stu-id="f453d-126">Manual</span></span>
<span data-ttu-id="f453d-127">Принцип очистки "Вручную" указывает, что регистрация потребления материалов выполняется вручную.</span><span class="sxs-lookup"><span data-stu-id="f453d-127">The Manual flushing principle indicates that the registration of material consumption is a manual operation.</span></span> <span data-ttu-id="f453d-128">Этот принцип применяется, если, например, необходимо иметь возможность отслеживать время и следует учитывать количество потребленных номеров партий или серийных номеров в целях отслеживания.</span><span class="sxs-lookup"><span data-stu-id="f453d-128">This principle is relevant if, for example, you want to be able to track time, and the quantity of consumed batch numbers or serial numbers must be accounted for, for tracking purposes.</span></span> <span data-ttu-id="f453d-129">Потребление вручную регистрируется в журнале ведомостей комплектации производства.</span><span class="sxs-lookup"><span data-stu-id="f453d-129">Manual consumption is registered in a production picking list journal.</span></span> <span data-ttu-id="f453d-130">Для номенклатур, которые включены для расширенных процессов складов, может быть применен ручной поток.</span><span class="sxs-lookup"><span data-stu-id="f453d-130">For items that are enabled for advanced warehouse processes, a hand-held flow can be applied.</span></span>

### <a name="start"></a><span data-ttu-id="f453d-131">Запустить</span><span class="sxs-lookup"><span data-stu-id="f453d-131">Start</span></span>
<span data-ttu-id="f453d-132">Принцип очистки "Начать" указывает, что материал будет автоматически потребляться при запуске производственного заказа.</span><span class="sxs-lookup"><span data-stu-id="f453d-132">The Start flushing principle indicates that material will be automatically consumed when the production order is started.</span></span> <span data-ttu-id="f453d-133">Объем потребляемого материала пропорционален количеству, которое начато.</span><span class="sxs-lookup"><span data-stu-id="f453d-133">The amount of material that is consumed is proportional to the quantity that is started.</span></span> <span data-ttu-id="f453d-134">При использовании принципа очистки "Начать" вместе с системой управления производством он также может использоваться для очистки материалов при запуске операции или задания процесса.</span><span class="sxs-lookup"><span data-stu-id="f453d-134">When the Start flushing principle is used together with the manufacturing execution system, it can also be used to flush materials when an operation or a process job is started.</span></span> <span data-ttu-id="f453d-135">Этот принцип применяется, если, например, отклонение в отклонении низкое, материалы имеют низкую стоимость, нет требований к отслеживанию или время выполнения операций короткое.</span><span class="sxs-lookup"><span data-stu-id="f453d-135">This principle is relevant if, for example, the variance in the consumption is low, the materials are low-value materials, there are no tracking requirements, or there is a short run time on operations.</span></span> 

### <a name="finish"></a><span data-ttu-id="f453d-136">Готово</span><span class="sxs-lookup"><span data-stu-id="f453d-136">Finish</span></span>
<span data-ttu-id="f453d-137">Принцип очистки "Готово" указывает, что материал будет автоматически потребляться, когда принимается производственный заказ или когда завершается операция, которая настроена для потребления материалов.</span><span class="sxs-lookup"><span data-stu-id="f453d-137">The Finish flushing principle indicates that material will be automatically consumed when the production order is reported as finished, or when an operation that is set up to consume the materials is registered as completed.</span></span> <span data-ttu-id="f453d-138">Объем потребляемого материала пропорционален количеству, которое принято.</span><span class="sxs-lookup"><span data-stu-id="f453d-138">The amount of material that is consumed is proportional to the quantity that is reported as finished.</span></span> <span data-ttu-id="f453d-139">При использовании принципа очистки "Готово" вместе с системой управления производством он также может использоваться для очистки материалов при завершении операции или задания процесса.</span><span class="sxs-lookup"><span data-stu-id="f453d-139">When the Finish flushing principle is used together with the manufacturing execution system, it can also be used to flush materials when an operation or a process job is completed.</span></span> <span data-ttu-id="f453d-140">Этот принцип применяется в тех же ситуациях, что и принцип "Начать".</span><span class="sxs-lookup"><span data-stu-id="f453d-140">This principle is relevant in the same situations as the Start principle.</span></span> <span data-ttu-id="f453d-141">Однако принцип "Готово" применяется для операций, которые имеют более длительное время выполнения, в которых материалы не должны устанавливаться как НЗП до завершения операции.</span><span class="sxs-lookup"><span data-stu-id="f453d-141">However, the Finish principle is for operations that have a longer run time, where materials should not be set to WIP before the operation is completed.</span></span> 

### <a name="available-at-location"></a><span data-ttu-id="f453d-142">Доступно в ячейке</span><span class="sxs-lookup"><span data-stu-id="f453d-142">Available at location</span></span>
<span data-ttu-id="f453d-143">Принцип очистки "Доступно в местонахождении" указывает, что материала будет автоматически потребляться, когда он регистрируется как скомплектованный для производства.</span><span class="sxs-lookup"><span data-stu-id="f453d-143">The Available at location flushing principle indicates that the material will be automatically consumed when it's registered as picked for production.</span></span> <span data-ttu-id="f453d-144">Материал регистрируется как скомплектованный из местонахождения, если завершена работа по комплектации сырья или если материал доступен в местонахождении поступлений на производство и строка материала выпущена на склад.</span><span class="sxs-lookup"><span data-stu-id="f453d-144">The material is registered as picked from location when work for the raw material picking is completed, or when material is available on the production input location and the material line is released to the warehouse.</span></span> <span data-ttu-id="f453d-145">Лист подбора, которая создается во время процесса, разносится в пакетном задании.</span><span class="sxs-lookup"><span data-stu-id="f453d-145">The picking list that is generated during the process is posted in a batch job.</span></span> <span data-ttu-id="f453d-146">Этот принцип используется, если, например, имеется много мероприятий комплектации для одного производственного заказа.</span><span class="sxs-lookup"><span data-stu-id="f453d-146">This principle is relevant if, for example, you have many picking activities against one production order.</span></span> <span data-ttu-id="f453d-147">Таким образом, не нужно вручную обновлять лист подбора, и можно получить текущее представление сальдо НЗП.</span><span class="sxs-lookup"><span data-stu-id="f453d-147">In this case, you don't have to update the picking list manually, and you can get a current view of the WIP balance.</span></span>
