# <a name="report-as-finished-to-a-non-license-plate-controlled-location--application-may-2016"></a>Приемка в местонахождении, не находящемся под управлением грузоместа (приложение, май 2016 г.)

[!include [banner](../../includes/banner.md)]

В этом руководстве по задаче показан пример приемки отчетности в местонахождении, которое не управляется грузоместом. Применимая политика работы является обязательным условием для этой задачи. В предыдущем руководстве по задаче показана настройка политики работы. Для выполнения этого руководства по задаче требуется приложение Dynamics AX 7.0.1 или более поздней версии.




## <a name="set-up-an-output-location"></a>Настройка местонахождения выхода
1. Перейдите в раздел "Управление организацией" > "Ресурсы" > "Группы ресурсов".
2. В списке выберите группу ресурсов "5102".
3. Выберите Изменить.
4. В поле "Склад выпуска" введите "51".
5. В поле "Местонахождение выхода" введите "001".
    * Местонахождение 001 не является местонахождением под управлением грузоместа. Местонахождение выхода, которое не находится под управлением грузоместа, можно настроить, только если для местоположения существует соответствующая политика работы.  

## <a name="create-a-production-order-and-report-it-as-finished"></a>Создание производственного заказа и его приемка
1. Закройте страницу.
2. Перейдите в раздел "Управление производством" > "Производственные заказы" > "Все производственные заказы".
3. Щелкните "Новый производственный заказ".
4. В поле "Код номенклатуры" введите "L0101".
5. Щелкните Создать.
6. В области действий щелкните "Производственный заказ".
7. Щелкните "Оценка".
8. Нажмите кнопку OK.
9. Щелкните "Начать".
10. Перейдите на вкладку Общие.
11. В поле "Автопотребление по спецификации" выберите "Никогда".
12. Нажмите кнопку OK.
13. Щелкните Приемка.
14. Перейдите на вкладку Общие.
15. Выберите значение "Да" в поле "Допускать ошибки".
16. Нажмите кнопку OK.
17. В области действий щелкните "Склад".
18. Щелкните "Сведения о работе".
    * При учете производственного заказа не была создана работа размещения. Это происходит, потому что определена политика работы, которая предотвращает создание работы при приемке продукта L0101 в местоположении 001.
