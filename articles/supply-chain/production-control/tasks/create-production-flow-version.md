# <a name="create-a-production-flow-version"></a>Создание версии производственного потока

[!include [banner](../../includes/banner.md)]

В этой процедуре рассматривается создание новой версии производственного потока. Для этой процедуры необходимо определить параметры производства для бережливого производства и единицы измерения для времени класса. Необходимо также определить поток создания ценности и производственную группу. Для получения дополнительных сведений о производственных потоках и мероприятиях в бережливом производстве см. официальные документы по бережливому производству для Microsoft Dynamics AX. В качестве компании с демонстрационными данными для создания этой процедуры используется USMF.


## <a name="create-a-production-flow"></a>Создание производственного потока
1. Перейдите в раздел "Управление производством" > "Настройка" > "Поток бережливого производства" > "Производственные потоки".
2. Щелкните "Создать".
3. В поле "Имя" введите значение.
4. В поле "Описание" введите значение.
5. В поле "Имя" нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
6. В списке перейдите по ссылке в выбранной строке.
    * Выберите операционную единицу потока создания ценности типа. Поток создания ценности является операционной единицей, которая охватывает все мероприятия и потоки создания ценности потока. На этом этапе операционные единицы ограничены юридическим лицом и не имеют других функций.  
7. В поле "Производственная группа" нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
8. В списке перейдите по ссылке в выбранной строке.
    * Выберите производственную группу для производственного потока. Производственные группы позволяют определять материал, трудозатраты и счет НЗП для процесса производства. Чтобы связать контекст учета с производственным потоком, необходимо выбрать производственную группу.  
9. Нажмите кнопку "Сохранить".

## <a name="create-a-production-flow-version"></a>Создание версии производственного потока
1. Нажмите кнопку Добавить.
2. В поле "Дата окончания срока действия" введите дату и время.
    * При необходимости определите дату окончания срока действия для версии. Можно обновить ее в любой момент времени, даже для активных версий. Ее можно использовать для планирования постепенной замены версии.  
3. Нажмите кнопку "OК".
4. В списке пометьте выбранную строку.
5. В поле "Единица измерения" введите значение.
6. Разрешить изменения единицы такта.
7. В поле "Среднее время такта" введите число.
    * Определите среднее время такта версии. Для управления тактами версии производственного потока определите среднее время такта. Такт определен как количество за период времени. В примере время такта составляет 0,2 часа на 10 единиц. Для рабочего времени 8 часов это соответствует ежедневной пропускной способности в 400 единиц.  
8. В поле "Количество в цикле" введите число.
    * Определите количество на цикл, связанный со среднем временем такта.  
9. Переключите развертывание раздела "Сведения о версии".
10. В поле "Минимальное время такта" введите число.
    * Определите минимальное время такта. Минимальное время такта должно быть меньше или равно среднему времени такта.  
11. В поле "Максимальное время такта" введите число.
    * Максимальное время такта должно быть больше или равно среднему времени такта.  
12. Введите число в поле "Период для фактического времени цикла (дни)".
    * Введите количество дней в период для фактического времени цикла. Период для фактического времени цикла — это число дней, из которых суммируются задания по фактической минуте назад для расчета фактического времени цикла. Значение можно изменить в любое время, и оно используется только для расчета фактического времени цикла.  
13. Нажмите кнопку "Сохранить".
