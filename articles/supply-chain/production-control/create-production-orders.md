# <a name="create-production-orders"></a>Создание производственных заказов

[!include [banner](../includes/banner.md)]

При создании производственного заказа инициируется запрос для начала производства номенклатуры. Производственный заказ содержит сведения о том, что будет производиться, в каком количестве, и плановую дату окончания. Он также содержит информацию о том, какие материалы будут потреблены и которому процессу следовать для произведения номенклатуры.

Производственный заказ проходит через этапы жизненного цикла продукции. Когда заказ создается, ему назначается статус **Создано**. Когда заказ завершен, ему назначается статус **Завершено**. Установка параметра на каждом этапе позволяет пользователю настроить каждый этап. Настройку можно сделать для отдельного пользователя или для всех пользователей.

Производственная спецификация и маршрут производства являются главными объектами производственного заказа. Они копируются в производственный заказ на основе выбранной номенклатуры и количества, которое должно быть произведено. Прежде чем начать производственный заказ, можно редактировать производственную спецификацию и маршрут.

Производственный заказ можно создать в следующих сценариях:

-   Созданный сводным планированием на основе потребности в материале.
-   Созданный сразу из строки заказа на продажу или когда создается и оценивается производственный заказ более высокого уровня (фиксированное снабжение).
-   Созданный вручную.
