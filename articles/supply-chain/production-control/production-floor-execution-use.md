# <a name="how-workers-use-the-production-floor-execution-interface"></a>Как работники используют интерфейс выполнения производственного цеха

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

Интерфейс выполнения производственного цеха оптимизирован для сенсорного взаимодействия. Его дизайн обеспечивает визуальную контрастность, которая соответствует требованиям к специальным возможностям в средах производственных цехов. Он предлагает все те же функциональные возможности, что и устройство карты задания. Однако он также позволяет запускать несколько заданий параллельно из списка заданий. (Эта возможность также называется *объединением заданий*.) Кроме того, в списке заданий работники могут открыть руководство, созданное в Microsoft Dynamics 365 Guide. Таким образом они могут получить визуальные инструкции на HoloLens.

## <a name="sign-in-to-the-production-floor-execution-interface-as-a-worker"></a>Вход в интерфейс выполнения производственного цеха в качестве работника

Прежде чем работники смогут начать использовать устройство, супервизором или технический персонал должны подготовить его и открыть правильную страницу в Dynamics 365 Supply Chain Management. Дополнительные сведения о настройке устройства см. в разделе [Настройка устройства для запуска интерфейса выполнения производственного цеха](production-floor-execution-setup.md).

После подготовки устройства на нем появится страница входа в систему. На этой странице отображаются сведения о статусе заданий для локальной производственной ячейки. Эти сведения периодически обновляются. На странице работники используют коды своего бейджа для входа. Хотя работникам не требуется иметь учетную запись пользователя для управления Supply Chain Management, у них должна быть *зарегистрированная по времени* учетная запись работника, которую они могут использовать при входе в систему.

![Страница входа в интерфейс выполнения производственного цеха](media/pfei-sign-in-page.png "Страница входа в интерфейс выполнения производственного цеха")

В остальных разделах этой темы описывается взаимодействие сотрудников с интерфейсом.

## <a name="all-jobs-tab"></a>Вкладка "Все задания"

Вкладка **Все задания** содержит список заданий, в котором отображаются все производственные задания со статусом *Не начато*, *Остановлено* или *Начато*. (Название этой вкладки может настраиваться и отличаться для вашей системы.)

![Вкладка "Все задания"](media/pfei-all-jobs-tab.png "Вкладка "Все задания"")

Список заданий имеет следующие столбцы. Цифры соответствуют номерам на предыдущем рисунке.

1. **Столбец выбора** — крайний левый столбец использует флажки, чтобы указать задания, которые были выбраны работником. Работники могут выбирать в списке несколько заданий одновременно. Чтобы выбрать все задания в списке, установите флажок в заголовке столбца. Если выбрано одно задание, сведения об этом задании отображаются в нижней части страницы.
1. **Столбец состояния задания** — в этом столбце используются символы для указания статуса каждого задания. Задания, у которых нет символа в этом столбце, имеют статус *Не начато*. Зеленый треугольник указывает задания со статусом *Начато*. Две желтые вертикальные линии обозначают задания со статусом *Остановлено*.
1. **Столбец высокого приоритета** — в этом столбце для обозначения заданий с высоким приоритетом используются восклицательные знаки.
1. **Заказ** — в этом столбце отображается номер производственного заказа для задания.
1. **Описание** — в этом столбце отображается описание операции, частью которой является задание.
1. **Запрошено** — в этом столбце отображается количество, которое планируется произвести при выполнении задания.
1. **Запущено** — в этом столбце отображается количество, которое уже было запущено для задания.
1. **Завершено** — в этом столбце отображается количество, которое уже было завершено для задания.
1. **Отбраковано** — в этом столбце отображается количество, которое уже было отбраковано для задания.
1. **Остаток** — в этом столбце отображается количество, которое осталось выполнить для задания.

На вкладках **Активные задания** отображается список всех заданий, которые уже запущены сотрудником, выполнившим вход. (Название этой вкладки может настраиваться и отличаться для вашей системы.)

![Вкладка "Активные задания"](media/pfei-active-jobs-tab.png "Вкладка "Активные задания"")

Чтобы выбрать все задания в списке, установите флажок в заголовке столбца. Если выбрано одно задание, сведения об этом задании отображаются в нижней части страницы.

## <a name="my-machine-tab"></a>Вкладка "Мое оборудование"

На вкладке **Мое оборудование** работники выбирают актив, подключенный к машинному ресурсу, в наборе фильтров на вкладке **Все задания**. Работник может просмотреть состояние и работоспособность выбранного актива, считывая значения до четырех выбранных счетчиков и списки последних запросов на обслуживание и зарегистрированных простоев. Работник может также запрашивать обслуживание для выбранного актива и регистрировать и изменять время простоя оборудования. (Название этой вкладки может настраиваться и отличаться для вашей системы.)
 
![Вкладка "Мое оборудование"](media/pfei-my-machine-tab.png "Вкладка "Мое оборудование"")

На вкладке **Мое оборудование** имеются следующие столбцы. Цифры соответствуют номерам на предыдущем рисунке.

1. **Машинный актив** — выберите машинный актив, который необходимо отследить. Начните ввод имени, чтобы выбрать из списка соответствующих активов, или выберите значок лупы для выбора из списка всех активов, связанных с ресурсами, которые находятся в пределах фильтра списка заданий.

    > [!NOTE]
    > Пользователи Supply Chain Management могут назначить ресурс для каждого актива нужным образом, используя страницу **Все активы** (на вкладке **Основные средства**, используя раскрывающийся список **Ресурсы**). Дополнительные сведения см. в разделе [Создание актива](../asset-management/objects/create-an-object.md).

1. **Параметры** — выберите значок шестеренки, чтобы открыть диалоговое окно, в котором можно выбрать, какие счетчики будут отображаться для выбранного машинного актива. Значения этих счетчиков отображаются в верхней части вкладки **Управление активами**. В меню **Параметры** (см. следующий снимок экрана) можно включить до четырех счетчиков. Для каждого счетчика, который необходимо включить, выберите счетчик в поле подстановки в верхней части плитки. В поле подстановки перечислены все счетчики, связанные с активом, выбранным в верхней части страницы **Управление активами**. Настройте каждый из счетчиков для отслеживания **Суммарного** значения или самого последнего **Фактического** значения счетчика. Например, если установлен счетчик, отслеживающий, сколько часов работало оборудование, следует настроить его как **Суммарный**. При установке счетчика для измерения последней обновленной температуры или давления следует установить значение **Фактическое**. Выберите **ОК**, чтобы сохранить настройки и закрыть диалоговое окно.

    ![Вкладка "Мое оборудование"](media/pfei-my-machine-tab-settings.png "Вкладка "Мое оборудование"")

1. **Запрос обслуживания** — нажмите эту кнопку, чтобы открыть диалоговое окно, в котором можно создать запрос обслуживания. Вы сможете ввести описание и примечание. Запрос будет выведен пользователю Supply Chain Management, который сможет преобразовать запрос обслуживания в заказ на работу обслуживания.
1. **Регистрация времени простоя** — нажмите эту кнопку, чтобы открыть диалоговое окно, в котором можно зарегистрировать время простоя оборудования. Вы сможете выбрать код причины и ввести дату и время периода простоя. Регистрация времени простоя оборудования используется для расчета эффективности актива оборудования.
1. **Просмотр или редактирование** — нажмите эту кнопку, чтобы открыть диалоговое окно, в котором можно изменить или просмотреть существующие записи о времени простоя.


## <a name="starting-and-completing-production-jobs"></a>Запуск и выполнение производственных заданий

Работники запускают производственное задание, выбрав задание на вкладке **Все задания**, а затем выбрав **Запустить задание**, чтобы открыть диалоговое окно **Запуск задания**.

![Диалоговое окно "Запуск задания"](media/pfei-start-job-dialog.png "Диалоговое окно "Запуск задания"")

Работники используют диалоговое окно **Запуск задания** для подтверждения количества в производстве, а затем запуска задания. Работники могут скорректировать количество, выбрав поле **Количество**, затем используя цифровую клавиатуру, которая появляется. Работники затем выбирают **Пуске**, чтобы начать работу над заданием. Диалоговое окно **Запуск задания** закрывается, и задание добавляется на вкладку **Активные задания**.

Работники могут начать задание, которое имеет любой статус. Когда работник запускает задание со статусом *Не начато*, в поле **Количество** в диалоговом окне **Запуск задания** первоначально отображается полное количество. Когда работник запускает задание со статусом *Начато* или *Остановлено*, в поле **Количество** первоначально отображается оставшееся количество.

## <a name="reporting-good-quantities"></a>Отчетность о количестве годных изделий

Когда работник выполняет или частично выполняет задание, он может сообщить о количествах произведенных годных изделий, выбрав задание на вкладке **Активные задания**, затем выбрав пункт **Проверить ход выполнения**. Затем в диалоговом окне **Проверить ход выполнения** сотрудник вводит количество годных изделий с помощью цифровой клавиатуры. По умолчанию количество пустое. После ввода количества сотрудник может обновить статус задания на *Выполняется*, *Остановлено* или *Завершено*.

![Диалоговое окно "Проверить ход выполнения"](media/pfei-report-progress-dialog.png "Диалоговое окно "Проверить ход выполнения"")

## <a name="reporting-scrap"></a>Отчетность об отходах

Когда работник выполняет или частично выполняет задание, он может сообщить об отходах, выбрав задание на вкладке **Активные задания**, затем выбрав пункт **Сообщить об отходах**. Затем в диалоговом окне **Сообщить об отходах** сотрудник вводит количество отходов с помощью цифровой клавиатуры. Работник также выбирает причину (*Нет*, *Машина*, *Оператор* или *Материал*).

![Диалоговое окно "Сообщить об отходах"](media/pfei-report-scrap-dialog.png "Диалоговое окно "Сообщить об отходах"")

## <a name="completing-a-job-and-starting-a-new-job"></a>Завершение задания и запуск нового задания

Обычно работники выполняют задание, выбирая одно или несколько текущих заданий на вкладке **Активные задания**, затем выбирая **Проверить ход выполнения**. Затем они вводят произведенное количество (количество товаров) и устанавливают статус на *Завершено*. Если было выбрано более одного задания, работник использует кнопки **Назад** и **Далее** для перемещения между ними. Для запуска нового задания работник выбирает его на вкладке **Все задания**, затем выбирает **Запустить задание**.

Работник может также начать новое задание, когда еще открыто его предыдущее задание. Снова, работник выбирает новое задание на вкладке **Все задания**, затем выбирает **Запустить задание**. Однако в этом случае диалоговое окно **Запуск задания** информирует сотрудника о том, что в данный момент он работает над заданием, и поэтому он должен остановить или завершить это задание до начала нового задания.

## <a name="working-on-multiple-jobs-in-parallel"></a>Параллельная работа с несколькими заданиями

Один работник может работать с несколькими заданиями одновременно (то есть, параллельно). В этом случае набор заданий, над которыми работает сотрудник, называется *набором заданий*. Работник может добавлять новые задания в набор или завершать одно или несколько заданий в наборе. Следующие два сценария показывают, как работник может работать над заданиями параллельно.

### <a name="scenario-1-a-worker-who-has-no-active-jobs-wants-to-start-two-jobs-and-work-on-them-in-parallel"></a>Сценарий 1. Работнику, у которого нет активных заданий, необходимо запустить два задания и работать с ними в параллельном режиме

Работник выбирает два задания на вкладке **Все задания**, затем выбирает **Запустить задание**. В диалоговом окне **Запуск задания** отображаются оба выбранных задания, и работник может скорректировать количество для запуска для каждого задания. Затем работник подтверждает диалоговое окно и может запустить оба задания.

### <a name="scenario-2-a-worker-who-has-two-active-jobs-that-are-in-progress-wants-to-start-a-third-job-and-work-on-it-in-parallel-with-the-other-two"></a>Сценарий 2. Работник, у которого есть два активных задания, которые выполняются, хочет начать третье задание и работать с ним параллельно с другими двумя

Работник выбирает третье задание на вкладке **Все задания**, затем выбирает **Набор**. В диалоговом окне **Набор** работник может скорректировать количество для запуска. Затем сотрудник подтверждает диалоговое окно, выбрав **Набор**.

## <a name="working-on-indirect-activities"></a>Работа с дополнительными действиями

Дополнительные действия — это действия, не имеющие непосредственного отношения к производственному заказу. Дополнительные действия могут быть гибко определены, как описано в разделе [Настройка дополнительных действий для времени и посещаемости](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/set-up-indirect-activities-for-time-and-attendance).

Например, Шаннон, работник цеха в компании Contoso, хочет посетить собрание компании, а собрания рассматриваются как дополнительные действия. Применяется один из двух следующих сценариев:

- **Шаннон работает над одним или несколькими активными заданиями.** Шаннон выбирает **Действие**, определяет действие (собрание) и подтверждает свой выбор. Появится сообщение, информирующее, что у нее есть задания, которые выполняются. Из этого сообщения Шаннон может выбрать завершение или остановку заданий, над которыми она работает, перед тем, как она уйдет на собрание.
- **У Шаннон нет активных заданий.** Шаннон выбирает **Действие**, определяет действие (собрание) и подтверждает свой выбор. Сейчас она зарегистрирована как собрание.

В обоих сценариях после того как Шаннон подтверждает свой выбор, она переходит на страницу входа или на страницу, которая ждет, когда она подтвердит свое возвращение с дополнительного действия. Отображаемая страница зависит от настройки интерфейса выполнения производственного цеха. (Дополнительные сведения см. в разделе [Настройка интерфейса выполнения производственного цеха](production-floor-execution-configure.md).)

## <a name="registering-breaks"></a>Регистрация перерывов

Работники могут регистрировать перерывы. Перерывы могут быть определены гибко, как описано в разделе [Оплата на основе регистраций](pay-based-on-registrations.md).

Работник регистрирует перерывы, выбрав **Перерыв**, затем выбрав карточку, представляющую тип перерыва (например, обед). После того как работник подтверждает выбор, устройство отображает страницу входа или страницу, которая ждет, пока сотрудник не подтвердит, что он возвратился после перерыва. Отображаемая страница зависит от настройки интерфейса выполнения производственного цеха. (Дополнительные сведения см. в разделе [Настройка интерфейса выполнения производственного цеха](production-floor-execution-configure.md).)

## <a name="opening-instructions"></a>Открытие инструкций

Работники могут открыть документ, прикрепленный к заданию, выбрав **Инструкции**. Кнопка **Инструкции** доступна только в том случае, если документ связан с заданием в основных данных. Например, документ, прикрепленный к продукту на странице **Выпущенные продукты** в Supply Chain Management, будет доступен для открытия сотрудниками в интерфейсе выполнения производственного цеха.

## <a name="opening-mixed-reality-guides-for-hololens"></a>Открытие руководств с использованием смешанной реальности для HoloLens

[Dynamics 365 Guides](https://dynamics.microsoft.com/mixed-reality/guides/) позволяет повысить эффективность работы сотрудников, представляя практическое обучение, использующее смешанную реальность. Можно определить стандартизированные процессы, содержащие пошаговые инструкции, которые позволяют рабочим работать с инструментами и компонентами, которые им необходимы, и показывать сотрудникам, как использовать эти инструменты в реальных условиях работы. Ниже приведен обзор процесса.

1. Каждый раз, когда работник открывает список заданий в интерфейсе выполнения цеха, этот интерфейс находит все соответствующие руководства для показанных заданий.
1. Работник выбирает **Руководства** для просмотра списка руководств.
1. Работник выбирает соответствующее руководство в списке.
1. Интерфейс выполнения производственного цеха показывает QR-код для выбранного руководства.
1. Работник надевает HoloLens и смотрит на QR-код для запуска руководства.
1. Работник знакомится с руководством для изучения задачи.

Дополнительные сведения о создании, назначении и использовании руководств для HoloLens см. в разделе [Предоставление руководств с использованием смешанной реальности для работников на производстве](instruction-guides-in-production-overview.md).
