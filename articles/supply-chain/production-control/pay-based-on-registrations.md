# <a name="pay-based-on-registrations"></a>Оплата на основе регистраций

[!include [banner](../includes/banner.md)]

В этом разделе подробно объясняется, как оплата вычисляется на основе регистраций работника. Он включает примеры, которые показывают, как различные комбинации параметров настройки, которые доступны для расчета, влияют на результат. Вот несколько областей, которые будут рассмотрены:

- Время работы по гибкому графику
- Сверхурочное время
- Перерывы
- Коды подключения
- Пункты соглашения
- Соглашения по зарплате
- Параметры расчета времени и посещаемости
- Отсутствие

## <a name="the-use-of-flex-time"></a>Использование времени работы по гибкому графику

Периоды времени работы по гибкому графику настраиваются в профилях времени, которые используются в модуле посещаемости и времени присутствия. Существует два типа профилей гибкого графика работы: **Переработка по гибкому графику** и **Недоработка по гибкому графику-**. Когда работник регистрирует время в период "Переработка по гибкому графику", баланс работы по гибкому графику работника увеличивается на количество отработанных часов. Работник не получает никакой компенсации за часы, которые были отработаны за период "Переработка по гибкому графику". Однако работник может брать отгулы в периоды "Недоработка по гибкому графику-", и компенсация будет производиться из баланса работы по его гибкому графику. Таким образом, отгулы в периоды гибкого графика работы рассматриваются системой как отсутствие.

## <a name="scenarios-based-on-flex-periods"></a>Сценарии на основе гибких периодов

Два следующих сценария основаны на профиле гибкого графика работы, который представляет рабочий день. В обоих случаях оплата рассчитывается в соответствии с периодом гибкого графика работы, в которые сотрудник регистрирует время прихода и ухода.

### <a name="flex-profile-for-one-workday"></a>Профиль гибкого графика работы для одного рабочего дня

| Тип графика  | Запустить    | Завершение      | День     |
|---------------|----------|----------|---------|
| Сверхурочное время     | 00:00 | 06:00 | Понедельник  |
| Переработка по гибкому графику         | 06:00 | 07:00 | Понедельник  |
| Время прихода      | 07:00 | 07:00 | Понедельник  |
| Нормативное рабочее время | 07:00 | 14:30 | Понедельник  |
| Недоработка по гибкому графику-         | 14:30 | 15:30 | Понедельник  |
| Отметить время ухода     | 15:30 | 15:30 | Понедельник  |
| Сверхурочное время     | 15:30 | 06:00 | Вторник |

### <a name="scenario-1-a-worker-registers-clock-in-during-a-flex-period-and-clock-out-during-a-flex--period"></a>Сценарий 1. Работник регистрирует приход в период переработки по гибкому графику и уход в период недоработки по гибкому графику-

Регистрации работника для этого дня выглядят следующим образом.

| Тип регистрации журнала | Запустить    | Завершение      |
|---------------------------|----------|----------|
| Время прихода                  | 06:30 | 06:30 |
| Производственное задание            | 06:30 | 14:45 |
| Отметить время ухода                 | 14:45 | 14:45 |

Регистрации работника за день рассчитываются и переносятся в зарплату на странице **Утвердить** (**Посещаемость и время присутствия** &gt; **Просмотр и утверждение** &gt; **Утвердить**). После расчета регистраций можно просмотреть результат расчета на вкладке **Время**. 

Чтобы понять этот сценарий, см. следующие поля.

| Переработка по гибкому графику | Недоработка по гибкому графику - | Время | Оплачиваемое время |
|--------|--------|------|----------|
| 0,50   | 0.75   | 8,25 | 8,50     |

#### <a name="calculation-of-flex"></a>Расчет переработки по гибкому графику

В соответствии с профилем гибкого графика работы время между 06:00 и 07:00 является периодом переработки по гибкому графику. Таким образом, если сотрудник регистрирует время прихода в 06:30, он зарабатывает 0,5 ч. Это количество времени добавляется на счет гибкого графика работы работника.

#### <a name="calculation-of-flex-"></a>Расчет недоработки по гибкому графику

В соответствии с профилем гибкого графика работы период недоработки по гибкому графику начинается в 14:30 и заканчивается в 15:30. Таким образом,если работник регистрирует время ухода в 14:45, 45 минут (0,75 часа), которые остаются в периоде недоработки по гибкому графику, регистрируются как оплачиваемое время, и это же количество времени вычитается со счета гибкого графика работы работника. 45 минут включаются в оплачиваемое время, поскольку работнику будет предоставлена оплата за оставшиеся 45 минут в периоде недоработки по гибкому графику. Если сотрудник отсутствует в течение периода недоработки по гибкому графику, 45 минут будут вычитаться с его счета гибкого графика работы.

#### <a name="calculation-of-time"></a>Расчет времени

Время рассчитывается как время между временем прихода и временем ухода, т. е., от 06:30 до 14:45 получается 8,25 часа.

#### <a name="calculation-of-pay-time"></a>Расчет оплачиваемого времени

Оплачиваемое время является временем, в течение которого работнику начисляется оплата. В этом сценарии работник находится на работе в течение 8,25 часа (**Время**). Тем не менее **Оплачиваемое время** рассчитывается как 8,50 часа, поскольку работнику предоставляется оплата в период недоработки по гибкому графику, когда он ушел. Оплачиваемое время равняется запланированным рабочим часам, поскольку время переработки по гибкому графику добавляется к счету гибкого рабочего графика работника, а не к оплачиваемому времени. Время отсутствия в период недоработки по гибкому графику компенсируется оплачиваемым временем и вычитается из счета гибкого графика работы работника.

| Время              | Тип регистрации | Оплачиваемое время (часы)      |
|-------------------|-------------------|-----------------------|
| 6:30 – 7:00 | Переработка по гибкому графику             | 0                     |
| 7:00 – 14:45 | Нормативное рабочее время     | 7,75                  |
| 14:45 – 15:30 | Недоработка по гибкому графику-             | 0,75 (Период отсутствия) |
|                   | Итоговый             | 8,50                  |

### <a name="scenario-2-a-worker-works-during-the-whole-flex--period-and-also-works-overtime"></a>Сценарий 2. Работник работает во время всего периода переработки по гибкому графику, и также работает сверхурочно

Регистрации работника для этого дня выглядят следующим образом.

| Тип регистрации журнала | Запустить    | Завершение      |
|---------------------------|----------|----------|
| Время прихода                  | 06:30 | 06:30 |
| Производственное задание            | 06:30 | 17:00 |
| Отметить время ухода                 | 17:00 | 17:00 |

После расчета регистраций в журнале на странице **Утвердить** можно просмотреть результат расчета на вкладке **Время**. Чтобы понять этот сценарий, см. следующие поля.

| Переработка по гибкому графику | Недоработка по гибкому графику - | Время  | Оплачиваемое время | Оплачиваемые сверхурочные |
|--------|--------|-------|----------|--------------|
| 0,50   | 0.00   | 10,50 | 10.00    | 1.50         |

#### <a name="calculation-of-flex"></a>Расчет переработки по гибкому графику

В соответствии с профилем гибкого графика работы время между 06:00 и 07:00 является периодом переработки по гибкому графику. Таким образом, если сотрудник регистрирует время прихода в 06:30, он зарабатывает 0,5 ч времени переработки по гибкому графику на баланс своего счета гибкого графика работы.

#### <a name="calculation-of-flex-"></a>Расчет недоработки по гибкому графику

Так как работник работает в период недоработки по гибкому графику, недоработка по гибкому графику не вычисляется. Недоработка по гибкому графику вычисляется только в том случае, если работник отсутствовал в период недоработки по гибкому графику. С точки зрения оплаты, если сотрудник работает в течение периода недоработки по гибкому графику, ему предоставляется ставка оплаты, которая определена для стандартного времени. Если сотрудник отсутствует в течение периода недоработки по гибкому графику, 45 минут вычитаются с его счета гибкого графика работы.

#### <a name="calculation-of-time"></a>Расчет времени

Время рассчитывается как время между временем прихода в 06:30 и временем ухода в 17:00, или 10,50 часа.

#### <a name="calculation-of-pay-time"></a>Расчет оплачиваемого времени

В этом сценарии работник работал в течение 10,50 часа (**Время**). Тем не менее **Оплачиваемое время** рассчитывается как 10,00 часа, потому что работнику не предоставляется оплата в период переработки по гибкому графику.

#### <a name="calculation-of-pay-overtime"></a>Расчет оплачиваемого сверхурочного времени

| Время               | Тип регистрации | Оплачиваемое время (часы) |
|--------------------|-------------------|------------------|
| 6:30 – 7:00  | Переработка по гибкому графику             | 0                |
| 7:00 – 14:45  | Нормативное рабочее время     | 7,50             |
| 14:30 – 15:30  | Недоработка по гибкому графику-             | 1.00             |
| 15:30 – 17:00 | Сверхурочное время          | 1.50             |
|                    | Итоговый             | 10.00            |

### <a name="generation-of-pay-items"></a>Создание элементов оплаты

Регистрации работника за день можно перенести со страницы **Утвердить**. В процессе переноса генерируются элементы оплаты и перемещенные регистрации. Элементы оплаты представляют разбивку оплачиваемого времени на стандартное время, сверхурочное время, время оплачиваемого перерыва и т. д.

Чтобы открыть список элементов оплаты, выберите **Посещаемость и время присутствия** &gt; **Просмотр и утверждение** &gt; **Утвердить**. Затем выберите **Запрос** &gt; **Перенесенные регистрации**.

Элементы оплаты являются основой для оплаты труда работника. Можно создать файл, содержащий информацию из элементов оплаты, и перенести этот файл в систему расчета зарплаты.

В процессе перемещения время и затраты из производственных и проектных действий переносятся в журналы производства и проекта для учета затраченного времени и понесенных затрат. Перенесенные регистрации являются основой для времени и себестоимости за час для производственных заказов и проектов. Перемещенные регистрации можно открыть с помощью меню **Запрос** на странице **Утвердить**.

Например, для сценария 2 создаются следующие элементы оплаты.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме  | Общие затраты |
|---------------|----------|-----------|-------|------------|
| Нормативное рабочее время | 1201     | 10,0      | 10    | 100        |
| Сверхурочное время      | 1301     | 1.50      | 5     | 7,50       |
|               |          |           | Итоговый | 107,50     |

Элемент оплаты для стандартного времени имеет единицу оплаты 10 часов, которая охватывает нормативное рабочее время, времени недоработки по гибкому графику и сверхурочное время. Стандартное время, время недоработки по гибкому графику и сверхурочное время объединяются в один элемент оплаты, так как все эти типы рассчитываются как нормативное рабочее время, на основе настройки по умолчанию параметра на странице **Параметры расчета** (**Посещаемость и время присутствия** &gt; **Настройка** &gt; **Параметры расчета**). Сверхурочное время рассчитывается поверх нормативного рабочего времени с дополнительным коэффициентом 5.

Если вы хотите четка отделять нормативное рабочее время и сверхурочное время, чтобы элементы оплаты для типов оплаты охватывали только фактическое время, затраченное на нормативное рабочее время и сверхурочное время, элементы оплаты для нормативного рабочего времени должны рассчитываться как 8,50, а элементы оплаты за сверхурочное время должны рассчитываться как 1,50.

Чтобы настроить систему таким образом, чтобы четко отличать нормативное рабочее время и сверхурочное время, необходимо исключить сверхурочное время из нормативного рабочего времени. Необходимо также изменить настройку типа оплаты сверхурочного времени, таким образом, чтобы ставка оплаты сверхурочного времени охватывала все платежи для часов, потраченных на сверхурочную работу.

### <a name="exclude-overtime-from-the-standard-time"></a>Исключение сверхурочного времени из нормативного рабочего времени

На странице **Параметры расчета** выберите **Сверхурочное время** как тип спецификации профиля и установите для параметра **Оплачиваемое время** значение **Нет**, как показано здесь.

| Спецификация рег. | Тип спецификации профиля | Расчет   | Параметр | Платный         | Параметр |
|--------------------|----------------------------|---------------|-----|--------------|-----|
| Рабочее время       | Сверхурочное время                   | Нормативное рабочее время | Да | Оплачиваемое время     | Нет  |
|                    |                            | Оплачиваемое время      | Да | Оплачиваемые сверхурочные | Да |

После настройки параметров расчета будут созданы следующие элементы оплаты.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме  | Общие затраты |
|---------------|----------|-----------|-------|------------|
| Нормативное рабочее время | 1201     | 8,50      | 10    | 85,0       |
| Сверхурочное время      | 1301     | 1.50      | 15    | 22,50      |
|               |          |           | Итоговый | 107,50     |

> [!NOTE]
> Параметры расчета имеют стандартные рекомендуемые значения. В общем случае будьте внимательны при изменении этих параметров. Чтобы восстановить стандартные рекомендуемые параметры, на странице **Параметры расчета** выберите **Восстановить значения**.

### <a name="allow-a-deviation-from-the-standard-pay-profiles"></a>Разрешение отклонения от стандартных профилей оплаты

На странице **Профили** (**Посещаемость и время присутствия** &gt; **Настройка** &gt; **Профили времени** &gt; **Профили**) можно настроить типы профилей, которые включают коды переключения и перерывы.

### <a name="switch-codes"></a>Коды подключения

Коды переключения можно использовать, чтобы разрешить работникам отклоняться от их типа профиля путем изменения другого типа профиля. Например, можно разрешить работнику изменить время переработки по гибкому графику на сверхурочное время. Работник может добавить код переключения во время регистрации или вы можете назначить задачу добавления кода переключения лицу, утверждающему регистрации.

Прежде чем можно будет использовать код переключения, необходимо определить его как тип дополнительной деятельности. Затем необходимо добавить код переключения к профилю времени для периода, для которого нужно разрешить изменение типа профиля. Например, выполните следующие действия, чтобы создать код переключения, который позволяет изменять период переработки по гибкому графику с 06:00 до 07:00 на сверхурочное время.

1. Создайте код переключения, который называется **OTBCI (Преобразование гибкого графика работы в сверхурочное время до регистрации прихода)**. Выберите **Посещаемость и время присутствия** &gt; **Ведение дополнительных мероприятий** &gt; **Дополнительные задания**.
2. В столбце **Код переключения** добавьте OTBCI в строку "Переработка по гибкому графику" в профиле времени.
3. В столбце **Вторичные** добавьте тип профиля **Сверхурочное время**.

Рассмотрим следующий профиль работы по гибкому графику, представляющий рабочий день.

| Переработка по гибкому графику         | 06:00 | 07:00 | Понедельник  |
| Время прихода      | 07:00 | 07:00 | Понедельник  |
| Нормативное рабочее время | 07:00 | 14:30 | Понедельник  |
| Недоработка по гибкому графику-         | 14:30 | 15:30 | Понедельник  |
| Отметить время ухода     | 15:30 | 15:30 | Понедельник  |
| Сверхурочное время     | 15:30 | 06:00 | Вторник |

На странице **Утвердить** отмените перенос, затем используйте меню **Код переключения**, чтобы применить код переключения **OTBCI**. При переносе регистрации второй раз создаются следующие элементы оплаты.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме  | Общие затраты |
|---------------|----------|-----------|-------|------------|
| Нормативное рабочее время | 1201     | 8,50      | 10    | 80,0       |
| Сверхурочное время      | 1305     | 2.00      | 15    | 30,0       |
|               |          |           | Итоговый | 107,50     |

> [!NOTE]
> При применении код переключения сверхурочное время увеличивается на 0,5 часа, с 1,50 до 2,00. Значение 0,5 часа является преобразованием времени переработки по гибкому графику с 6:30 до 07:00 в сверхурочное время.

### <a name="breaks"></a>Перерывы

Перерывы в работе влияют на расчет оплаты труда работника. Перерывы определяются как тип дополнительной деятельности. Их можно определить как **Оплаченные**, чтобы разрешить добавлять перерывы в оплату труда работника, или **Неоплаченные**, чтобы запретить добавлять перерывы в оплату труда работника. Перерыв может быть также определен как **Запланировано** или **Зарегистрировано**.

#### <a name="planned-breaks"></a>Запланированные перерывы

Если в компании имеется фиксированное время перерыва, такое как перерыв на обед, перерыв может быть заранее определен в профиле времени. В этом случае работнику не требуется регистрировать перерыв на страницах карты задания. Вместо этого перерыв автоматически учитывается при расчете регистраций работника на странице **Утвердить**.

#### <a name="registered-breaks"></a>Регистрируемые перерывы

Если компания не использует запланированные перерывы, работники могут регистрировать перерывы в течение рабочего дня. Зарегистрированные перерывы могут использоваться, например, если работник работает по профилю гибкого рабочего времени, который не имеет определенного времени прихода и ухода. Регистрируемые перерывы определяются как тип дополнительной деятельности. Работник регистрирует перерыв на странице терминала **Карта задания** или на странице устройства **Карта задания**. На обоих этих страницах пользователь может выбрать тип перерыва в списке предопределенных действий перерыва.

#### <a name="paid-and-unpaid-breaks"></a>Оплачиваемые и неоплачиваемые перерывы

Действия перерыва могут быть настроены как **Оплаченные** или **Неоплаченные**. Оплачиваемый перерыв будет включена в расчет оплачиваемого времени, и система использует тип оплаты, который определен в соглашении по оплате для типа регистрации **Перерыв**.

## <a name="how-the-cost-of-paid-breaks-is-allocated-to-projects-and-production-orders"></a>Каким образом стоимость оплачиваемых перерывов разделяется на проекты и производственные заказы

Почасовые затраты на мероприятия по проекту и производственные задания можно настроить таким образом, будет они определялись либо ставками оплаты, которые рассчитываются в модуле посещаемости и времени присутствия, либо категориями затрат, которые определены для действий.

Чтобы настроить категорию затрат, выберите **Управление производством** &gt; **Настройка** &gt; **Управление производством** &gt; **Значения по умолчанию для производственного заказа** и задайте в поле **Категория затрат** значение **Да** или **Нет**.

- **Нет** — затраты рассчитываются на основе ставок оплаты, которые определены для типов регистрации модуля посещаемости и времени присутствия.
- **Да** — затраты рассчитываются на основе категорий затрат для производственных и проектных действий.

### <a name="cost-calculation-based-on-pay-rates-that-are-calculated-in-time-and-attendance"></a>Расчет затрат на основе ставок оплаты, которые рассчитываются в модуле посещаемости и времени присутствия

В следующем примере показано, как вычисляются почасовые затраты, когда стоимость настроена так, что она рассчитываются на основании ставок заработной платы.

В процессе переноса рассчитывается почасовые нормы затрат для производственных заказов и проектов. Чтобы просмотреть почасовую ставку по действиям, откройте страницу **Утвердить** в модуле "Посещаемость и время присутствия", затем выберите **Запрос** &gt; **Перенесенные регистрации**. Почасовые ставки затрат каждой регистрации можно найти на вкладке **Себестоимости**.

Рассмотрим следующие регистрации, использующие тот же профиль времени, что и в предыдущем примере.

| Тип регистрации журнала | Запустить    | Завершение      | Время |

|-----------------------|------|---------------------|
| Время прихода              | 0.00 | 0.00                |
| Процесс (Заказ: 4711) | 4,00 | 10.00               |
| Процесс (Заказ: 4712) | 3,50 | 11,14               |
| Перерыв (Оплачиваемый)          | 0,50 | 0.00                |

Расчет себестоимости в час для оплачиваемого перерыва зависит от настройки для прямых затрат на зарплату. На вкладке **Себестоимость** в разделе **Прямые затраты** в поле **Нормативное рабочее время** можно выбрать **Да**, **Нет** или **Распределение**.

В данном примере это мероприятие является производственным заданием для заказа 4712. Как вы видите, себестоимость в час для оплачиваемого перерыва равна 0 (нолю), и она выделяется заданию, которое выполняется параллельно с перерывом.

## <a name="absence"></a>Отсутствие

Код отсутствия используется для регистрации периода отсутствия работника. Как и перерывы и коды переключения, код отсутствия — это тип дополнительного мероприятия. Время отсутствия может быть плановым или зарегистрированным, и отсутствие может быть как законным, так и незаконным. Примерами законного отсутствия могут быть посещение врача, семинара или исполнение обязанностей присяжного. Незаконное отсутствие — это отсутствие без уважительной причины, например когда работник опаздывает. Как правило, законное отсутствие не приводит к вычетам из зарплаты работника, а незаконное отсутствие вызывает вычет.

### <a name="planned-absence"></a>Запланированное отсутствие

Можно создать плановое отсутствие для работников на странице **Создать запланированное отсутствие** (**Посещаемость и время присутствия** &gt; **Создать запланированное отсутствие**). Здесь запланированное отсутствие регистрируется как задание отсутствия для указанной даты и временного интервала.

Задание основано на запросе. Поэтому вы можете создать запланированное отсутствие для нескольких сотрудников, например сотрудников, принадлежащими к одной группе расчета. Если запланированное отсутствие предназначено для одного сотрудника, регистрацию можно ввести на странице **Посещаемость** или **Регистрация времени - работники**.

- Для ввода регистрации отсутствия со страницы **Посещаемость** выберите **Посещаемость и время присутствия** &gt; **Запросы и отчеты** &gt; **Посещаемость** &gt; **Посещаемость**, затем выберите **Регистрация отсутствия**.
- Для ввода регистрации отсутствия со страницы *<strong><em>Регистрация времени - работники</em></strong>* выберите <strong>Посещаемость и время присутствия</strong> &gt; <strong>Настройка</strong> &gt; <strong>Регистрация времени - работники</strong>, затем на вкладке <strong>Время</strong> в разделе <strong>Назначение времени</strong> выберите <strong>Регистрация отсутствия</strong>.

Можно использовать отчет **Запланированные отсутствия**, чтобы просмотреть обзор спланированного отсутствия для работников. Чтобы открыть этот отчет, выберите **Посещаемость и время присутствия** &gt; **Запросы и отчеты** &gt; **Отчеты об отсутствии** &gt; **Запланированные отсутствия**.

### <a name="registered-absence"></a>Зарегистрированное отсутствие

Как правило, работники считаются отсутствующими на работе, если они не находятся на работе в течение любого периода времени от запланированного времени прихода на работу до запланированного времени ухода с работы. Если работники регистрируют приход на работу позже планового времени или регистрируют время ухода раньше, чем планировалось, им будет предложено выбрать код отсутствия для указания причины отсутствия. Код отсутствия можно настроить, чтобы он был применим к регистрации. В списке будут доступны только применимые коды.

### <a name="scenario-1-the-worker-clocks-in-later-than-planned"></a>Сценарий 1. Сотрудник регистрирует время прихода позже, чем планировалось

Сотрудник регистрирует время прихода в 08:30. Поскольку его плановое время прихода — 07:00, он опоздал на работу на 1,50 часа. Поскольку 1,50 часа рассматривается как время отсутствия, работнику будет предложено выбрать код отсутствия. Работник уходит с работы в 15:30, что является запланированным временем ухода с работы. При расчете и утверждении регистраций работника регистрация отсутствия вместе с кодом отсутствия, выбранным работником при приходе на работу, отображаются для времени между 07:00 и 08:30.

В профиле времени можно настроить тип регистрации **Время прихода**, чтобы был некоторый допуск, когда сотрудники опаздывают на работу. Например, если настроен допуск 5, работнику выдается запрос на ввод кода отсутствия только в том случае, если он пришел на работу позднее 07:05.

В этом случае, поскольку у работника нет уважительной причины опоздания на работу, он выбирает код отсутствия, который определен для недопустимого отсутствия. Код отсутствия считается применимым для недопустимого отсутствия, если включен параметр для вычета сверхурочного времени для группы отсутствия, к которой принадлежит данный код отсутствия. Чтобы задать настройку, выберите **Посещаемость и время присутствия** &gt; **Настройка** &gt; **Группы** &gt; **Группы отсутствия**, затем выберите **Удерживать из сверхурочных**.

### <a name="scenario-2-the-worker-clocks-out-before-the-planned-clock-out-time-during-a-standard-time-period"></a>Сценарий 2. Работник отметил время ухода ранее запланированного времени ухода в период нормативного рабочего времени

Работник выбирает код отсутствия для посещения врача, который определяется как законное отсутствие. Ставка оплаты для законного отсутствия определяется в соглашения по зарплате для типа регистрации **Отсутствие** (**Посещаемость и время присутствия** &gt; **Настройка** &gt; **Заработная плата** &gt; **Соглашения по зарплате**).

### <a name="scenario-3-the-worker-clocks-out-before-the-planned-clock-out-time-during-a-flex--period"></a>Сценарий 3. Работник отметил время ухода ранее запланированного времени ухода в период недоработки по гибкому графику

Время между фактическим временем ухода с работы и запланированным временем ухода не считается отсутствием, и работнику не будет предложено выбрать код отсутствия. Сумма вычитается из счета гибкого графика работника, и работник получает оплату за остальную часть периода недоработки по гибкому графику, с 14:15 по 15:30.

### <a name="scenario-4-the-worker-clocks-in-late-and-clocks-out-after-the-planned-clock-out-time-during-an-overtime-period"></a>Сценарий 4. Работник пришел на работу поздно и ушел после запланированного времени ухода в период сверхурочного времени

Сотрудник пришел на работу в 09:30, и затем, чтобы компенсировать свой поздний приход, он работает в сверхурочное время и уходит с работы в 17:00. Так как работник пришел поздно и компенсировал это более длительной работой, компания не хочет предоставлять работнику оплату сверхурочной работы за время, которое он работал между запланированным временем ухода в 15:30 и его фактическим временем ухода в 17:00, несмотря на то, что этот период определяется как сверхурочные часы в профиле времени.

Для обработки этого сценария, можно настроить код отсутствия, чтобы он уменьшал сверхурочные часы на все время отсутствия без уважительной причины, которое было у работника в этот же день. Выберите **Посещаемость и время присутствия** &gt; **Настройка** &gt; **Группы** &gt; **Группы отсутствия**, и установите флажок **Удерживать из сверхурочных**, чтобы вычитать сверхурочные часы из времени отсутствия без уважительной причины.

Ниже показано, как регистрации работника на этот день выглядят на странице **Утвердить** после расчета.

| Тип регистрации журнала         | Запустить    | Завершение      | Время |
|-----------------------------------|----------|----------|------|
| Отсутствие (недопустимое — опоздание на работу) | 07:00 | 09:30 | 1.5  |
| Время прихода                          | 09:30 | 09:30 |      |
| Производственное задание                    | 09:30 | 17:00 | 7,5  |
| Отметить время ухода                         | 17:30 | 17:30 |      |

Если флажок **Удерживать из сверхурочных** установлен для выбранного кода отсутствия, оплата сверхурочных часов уменьшается на количество часов отсутствия сотрудника без уважительной причины. В этом случае после переноса регистраций создаются следующие элементы оплаты.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме |
|---------------|----------|-----------|------|
| Нормативное рабочее время | 1201     | 9,00      | 10   |
| Сверхурочное время      | 1301     | 0,5       | 15   |

Здесь 1,5 часа недопустимого отсутствия с 07:00 до 09:30 вычитаются из 2,0 часов сверхурочного времени, с 15:30 по 17:30. В результате регистрируются 1,5 часов стандартного времени и 0,5 часа сверхурочного времени.

Напротив, если флажок **Удерживать из сверхурочных** не установлен для выбранного кода отсутствия, работнику оплачиваются сверхурочные часы, несмотря на опоздание и отсутствие без уважительной причины. В этом случае после переноса регистраций создаются следующие элементы оплаты.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме |
|---------------|----------|-----------|------|
| Нормативное рабочее время | 1201     | 7,50      | 10   |
| Сверхурочное время      | 1301     | 2.0       | 15   |

### <a name="scenario-5-the-worker-clocks-out-before-the-planned-clock-out-time-and-can-convert-the-absence-period-to-a-flex--period"></a>Сценарий 5. Работник уходит с работы до спланированного времени ухода с работы и может преобразовать период отсутствия в период недоработки по гибкому графику

В следующем примере показано, как счет гибкого графика работы работника может быть уменьшен путем преобразования периода отсутствия в период недоработки по гибкому графику.

Работник регистрирует приход на работу в 07:00 и уход с работы в 13:00. Работник имеет соглашение о том, что он может уйти домой на выходные, если он вычитает эти часы со своего счета по гибкому графику. Когда сотрудник регистрирует время ухода с работы в 13:00, ему будет предложено выбрать код отсутствия, так как затрагиваемый период отсутствия для оставшейся части рабочего дня не является периодом гибкого графика работы. Чтобы преобразовать оставшуюся часть рабочего дня в период работы по гибкому графику, работник может выбрать код отсутствия, настроенный для уменьшения его счета гибкого графика работы.

Чтобы уменьшить баланс часов работы по гибкому графику для работников, которые регистрируют отсутствие в рабочий день, выберите **Посещаемость и время присутствия** &gt; **Настройка** &gt; **Группы** &gt; **Группы отсутствия**, затем выберите **Уменьшить время работы по гибкому графику**.

Ниже показано, как регистрации работника на этот день выглядят на странице **Утвердить** до расчета.

| Тип регистрации журнала | Запустить    | Завершение      | Время |
|---------------------------|----------|----------|------|
| Время прихода                  | 07:00 | 07:00 |      |
| Производственное задание            | 07:00 | 13:00 | 6,0  |
| Отметить время ухода                 | 13:00 | 13:00 |      |

Если работник выбирает код отсутствия для недопустимого отсутствия, вот как будет выглядеть итоговый элемент оплаты после переноса регистрации.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме |
|---------------|----------|-----------|------|
| Нормативное рабочее время | 1201     | 6,00      | 10   |

Если работник выбирает код отсутствия для отсутствия по уважительной причине, и код отсутствия настроен для уменьшения его счета гибкого графика работы, вот как будут выглядеть результирующие элементы оплаты после переноса регистраций.

| Тип заработной платы     | Вид оплаты | Единицы зарплаты | По норме |
|---------------|----------|-----------|------|
| Нормативное рабочее время | 1201     | 8,50      | 10   |

В этом случае баланс времени работы по гибкому графику работника уменьшается на количество часов между временем фактического ухода с работы и запланированным временем ухода с работы (то есть, 2,5 часа с 13:00 до 15:30).

> [!NOTE]
> Не рекомендуется устанавливать оба флажка **Удерживать из времени работы по гибкому графику** и **Удерживать из сверхурочных** для кода отсутствия, так как эта настройка будет вычитать часы отсутствия без уважительно причины из сверхурочного времени работника и в то же время уменьшать счет гибкого графика работы работника.

### <a name="scenario-6-there-is-no-planned-absence-for-the-day-and-no-worker-attendance-for-the-day"></a>Сценарий 6. Не существует спланированного отсутствия для дня и нет посещаемости работника за этот день

Если работник не пришел на работу в рабочий день и нет запланированного отсутствия для этого работника на этот день, используется код отсутствия по умолчанию для расчета регистраций работника. Чтобы определить коды отсутствия по умолчанию, выберите **Посещаемость и время присутствия** &gt; **Параметры учета посещаемости и времени присутствия**. Затем можно выбрать код отсутствия в следующих полях:

- Автовставка недоработки по гибкому графику
- Автовставка отсутствия

При расчете ежедневных регистраций работника, для которого включен гибкий график работы, кода отсутствия, который указан в поле **Автовставка недоработки по гибкому графику**, используется как код отсутствия по умолчанию. Если для работника не включены часы гибкого графика работы, используется код отсутствия, который указан в поле **Автовставка отсутствия**. Если в компании имеются как работники, для которых разрешен гибкий график работы, и работники, для которых гибкий график работы не разрешен, необходимо настроить оба параметра.
