# <a name="service-templates"></a>Шаблоны обслуживания

[!include [banner](../includes/banner.md)]

Можно определить соглашение на обслуживание в качестве шаблона, и в дальнейшем копировать строки шаблона в другое соглашение на обслуживание или в заказ на обслуживание.

## <a name="example"></a>пример

У обслуживаемого вами клиента существует несколько служебных лифтов в пяти различных местоположениях.

Вам необходимо настроить для данного клиента пять соглашений на обслуживание - по одному на каждое местоположение.
Чтобы сократить повторяющиеся этапы работы по настройке и обеспечить идентичность настроек соглашений на обслуживание, создается одно соглашение на обслуживание в качестве шаблона для работ по обслуживанию лифтов.

Теперь можно копировать строки шаблона в 5 новых соглашений на обслуживание таким образом, чтобы каждое соглашение на обслуживание будет заполнено строками из шаблона.

## <a name="create-a-template"></a>Создать шаблон

При создании шаблона обслуживания создается соглашение на обслуживание, созданные в нем требуемые строки, и присоединяется соглашение на обслуживание к группе шаблона.

> [!NOTE]
> Соглашение на обслуживание можно определить в качестве шаблона, только если оно не имеет приложенных заказов на обслуживание. Кроме того, ни один заказ на обслуживание не создаются из соглашения на обслуживание, которое определено в качестве шаблона.

## <a name="copy-template-lines"></a>Копировать строки шаблона

Строки шаблона можно копировать из шаблона обслуживания в другое соглашение на обслуживание или в заказ на обслуживание.

При копировании строк шаблона в заказы на обслуживание или соглашения на обслуживание группы шаблонов отображаются в виде дерева, в котором каждая группа может быть раскрыта. Такое представление позволяет видеть все шаблоны и строки шаблонов.

Проще определить строки шаблона, которые необходимо скопировать если вы группировали шаблоны по именам, которые отражают использование шаблонов.
