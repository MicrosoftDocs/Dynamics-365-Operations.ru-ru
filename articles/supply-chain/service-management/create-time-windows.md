# <a name="create-time-windows"></a><span data-ttu-id="27848-103">Создание окон времени</span><span class="sxs-lookup"><span data-stu-id="27848-103">Create time windows</span></span>   

[!include [banner](../includes/banner.md)]

<span data-ttu-id="27848-104">Окна времени можно использовать для оптимизации планирования строк заказа на обслуживание.</span><span class="sxs-lookup"><span data-stu-id="27848-104">You can use time windows to optimize the scheduling of service order lines.</span></span>

1. <span data-ttu-id="27848-105">Щелкните **Управление сервисным обслуживанием** \> **Настройка** \> **Соглашения на обслуживание** \> **Окно времени**.</span><span class="sxs-lookup"><span data-stu-id="27848-105">Click **Service management** \> **Setup** \> **Service agreements** \> **Time window**.</span></span>
2. <span data-ttu-id="27848-106">Нажмите комбинацию клавиш **Alt+N**, чтобы создать новое окно времени.</span><span class="sxs-lookup"><span data-stu-id="27848-106">Press **Alt+N** to create a new time window.</span></span>
3. <span data-ttu-id="27848-107">В поле **Окно времени** введите имя для окна времени.</span><span class="sxs-lookup"><span data-stu-id="27848-107">In the **Time window** field, enter a name for the time window.</span></span>
4. <span data-ttu-id="27848-108">В поле **Описание** введите описание для окна времени.</span><span class="sxs-lookup"><span data-stu-id="27848-108">In the **Description** field, enter a description for the time window.</span></span>
5. <span data-ttu-id="27848-109">В поле **Метод** выберите метод для автоматического создания.</span><span class="sxs-lookup"><span data-stu-id="27848-109">In the **Method** field, select the method to be autocreated.</span></span> <span data-ttu-id="27848-110">Если метод относится к типу **Вручную**, укажите количество дней до или после расчетной даты, на которое может быть сдвинута строка заказа на сервисное обслуживание.</span><span class="sxs-lookup"><span data-stu-id="27848-110">If the method is of the **Manual** type, specify the number of days before or after the calculated date that the service order line can be moved.</span></span>
