# <a name="update-standard-costs-for-a-new-manufactured-item"></a><span data-ttu-id="ddbda-103">Обновление стандартной себестоимости для новой произведенной номенклатуры</span><span class="sxs-lookup"><span data-stu-id="ddbda-103">Update standard costs for a new manufactured item</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="ddbda-104">Эта статья содержит рекомендации по обновлению стандартных затрат для новой произведенной номенклатуры.</span><span class="sxs-lookup"><span data-stu-id="ddbda-104">This article provides guidance for updating standard costs for a new manufactured item.</span></span> 

<span data-ttu-id="ddbda-105">В приведенных ниже указаниях предполагается использование подхода на основе двух версий для обновления стандартных затрат.</span><span class="sxs-lookup"><span data-stu-id="ddbda-105">The following guidelines assume that you use a two-version approach to update standard costs.</span></span> <span data-ttu-id="ddbda-106">В этом подходе одна версия цены содержит стандартные затраты, которые исходно были определены для периода "замораживания", а вторая версия учета затрат содержит инкрементные обновления, относящихся к новым произведенным номенклатурам.</span><span class="sxs-lookup"><span data-stu-id="ddbda-106">In this approach, one costing version contains the standard costs that were originally defined for the frozen period, and the second costing version contains the incremental updates that pertain to the new manufactured items.</span></span> <span data-ttu-id="ddbda-107">Инкрементные обновления вводятся как записи себестоимости во второй версии цены и, в конечном счете, они активируются.</span><span class="sxs-lookup"><span data-stu-id="ddbda-107">The incremental updates are entered as cost records in the second costing version, and eventually they are enabled.</span></span> <span data-ttu-id="ddbda-108">Подход на основе двух версий требует определения второй версии цены.</span><span class="sxs-lookup"><span data-stu-id="ddbda-108">The two-version approach requires that you define a second costing version.</span></span> <span data-ttu-id="ddbda-109">Далее приведены инструкции по определению этой версии расчета себестоимости.</span><span class="sxs-lookup"><span data-stu-id="ddbda-109">Here are the guidelines for defining this costing version:</span></span>

-   <span data-ttu-id="ddbda-110">Назначьте тип учета затрат **Стандартные затраты**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-110">Assign a costing type of **Standard cost**.</span></span>
-   <span data-ttu-id="ddbda-111">Назначьте значащий идентификатор, который отражает содержимое версии расчета себестоимости, например **2016-UPDATES**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-111">Assign a significant identifier that indicates the contents of the costing version, such as **2016-UPDATES**.</span></span>
-   <span data-ttu-id="ddbda-112">В группе полей **Разрешить тип цены** убедитесь, что в поле **Себестоимость** задано значение **Да**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-112">In the **Allow price types** field group, make sure that **Cost price** is set to **Yes**.</span></span>
-   <span data-ttu-id="ddbda-113">Разрешите ввод записей затрат для всех узлов (то есть, оставьте поле **Узел** пустым).</span><span class="sxs-lookup"><span data-stu-id="ddbda-113">Allow cost records to be entered for all sites (that is, leave the **Site** field blank).</span></span> <span data-ttu-id="ddbda-114">Если ввести сайт, записи затрат можно ввести только для этого сайта.</span><span class="sxs-lookup"><span data-stu-id="ddbda-114">If you enter a site, cost records can be entered only for that site.</span></span>
-   <span data-ttu-id="ddbda-115">Используйте принцип отката **Активный**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-115">Use a fallback principle of **Active**.</span></span>

<span data-ttu-id="ddbda-116">Чтобы добавить новые произведенные номенклатуры в течение периода "замораживания", необходимо выполнить следующие действия.</span><span class="sxs-lookup"><span data-stu-id="ddbda-116">To add new manufacturing items throughout the frozen period, follow these steps.</span></span>

1.  <span data-ttu-id="ddbda-117">Используйте страницу **Настройка версии калькуляции издержек**, чтобы разрешить ввод записей затрат во вторую версию расчета себестоимости, которая содержит инкрементные обновления.</span><span class="sxs-lookup"><span data-stu-id="ddbda-117">Use the **Costing version setup** page to enable cost records to be entered into the second costing version that contains the incremental updates.</span></span> <span data-ttu-id="ddbda-118">Запретите активацию ожидающей себестоимости, где такая активация разрешена после полного и точного определения ожидающей себестоимости.</span><span class="sxs-lookup"><span data-stu-id="ddbda-118">Prevent the activation of pending costs, where activation is allowed after pending costs have been completely and accurately defined.</span></span> <span data-ttu-id="ddbda-119">Укажите незаполненную начальную дату как политику в этой версии цены, а затем вводите начальную дату при вводе каждой записи себестоимости.</span><span class="sxs-lookup"><span data-stu-id="ddbda-119">Indicate a blank from-date as a policy in the costing version, and then enter the from-date when you enter each cost record.</span></span> <span data-ttu-id="ddbda-120">Эта начальная дата должна представлять собой дату, более раннюю, чем предполагаемые даты приобретения и производства новых номенклатур.</span><span class="sxs-lookup"><span data-stu-id="ddbda-120">The from-date should represent a date before the new items are purchased or manufactured.</span></span>
2.  <span data-ttu-id="ddbda-121">Воспользуйтесь страницей **Цена номенклатуры**, чтобы ввести записи себестоимости для новых приобретенных номенклатур.</span><span class="sxs-lookup"><span data-stu-id="ddbda-121">Use the **Item price** page to enter cost records for new purchased items.</span></span> <span data-ttu-id="ddbda-122">Для каждой записи себестоимости введите версию цены, содержащую добавочные обновления, и используйте начальную дату, которая предшествует ожидаемой дате производства новых номенклатур.</span><span class="sxs-lookup"><span data-stu-id="ddbda-122">For each cost record, enter the costing version that contains incremental updates, and use a from-date that comes before the expected manufacturing date for new manufactured items.</span></span>
3.  <span data-ttu-id="ddbda-123">Рассчитайте себестоимость новых произведенных номенклатур с помощью страницы **Расчет**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-123">Calculate the cost of new manufactured items by using the **Calculation** page.</span></span> <span data-ttu-id="ddbda-124">Откройте страницу **Расчет** со страницы **Поддержка версии калькуляции издержек**, затем выберите версию затрат, содержащую инкрементные обновления.</span><span class="sxs-lookup"><span data-stu-id="ddbda-124">Open the **Calculation** page from the **Costing version maintenance** page, and then select the costing version that contains the incremental updates.</span></span> <span data-ttu-id="ddbda-125">Воспользуйтесь критериями отбора, чтобы указать новую произведенную номенклатуру номенклатуры в качестве компонентов.</span><span class="sxs-lookup"><span data-stu-id="ddbda-126">In a multilevel product structure, you might also have to specify any parent item that contains the new manufactured items as a component.</span></span> <span data-ttu-id="ddbda-127">Введите начальную дату для расчета спецификации, которая соответствует началу производства новых номенклатур.</span><span class="sxs-lookup"><span data-stu-id="ddbda-127">Enter a from-date for the bill of materials (BOM) calculation that corresponds to the start of manufacturing for the new manufactured items.</span></span> <span data-ttu-id="ddbda-128">Начальная дата должна находиться в числе дат действия для версии спецификации номенклатуры и версии маршрута.</span><span class="sxs-lookup"><span data-stu-id="ddbda-128">The from-date should be in the effective dates for the item’s BOM version and route version.</span></span> <span data-ttu-id="ddbda-129">**Примечание.** Отсутствующая запись затрат может означать новую произведенную номенклатуру.</span><span class="sxs-lookup"><span data-stu-id="ddbda-129">**Note:** A missing cost record can indicate a new manufactured item.</span></span> <span data-ttu-id="ddbda-130">Расчеты спецификаций можно ограничить номенклатурами, для которых имеются недостающие затраты.</span><span class="sxs-lookup"><span data-stu-id="ddbda-130">BOM calculations can be limited to items that have missing costs.</span></span>
4.  <span data-ttu-id="ddbda-131">Проверьте полноту и точность расчетной себестоимости для новых произведенных номенклатур.</span><span class="sxs-lookup"><span data-stu-id="ddbda-131">Verify the completeness and accuracy of the calculated costs for new manufactured items.</span></span> <span data-ttu-id="ddbda-132">Используйте страницу **Выполнено** для просмотра рассчитанных затрат для каждой записи затрат по номенклатуре, а также проверки всех предупреждающих сообщений Infolog.</span><span class="sxs-lookup"><span data-stu-id="ddbda-132">Use the **Complete** page to review the calculated costs for each item cost record, and also review any Infolog warning messages.</span></span> <span data-ttu-id="ddbda-133">Или используйте страницу **Расчет**, чтобы просмотреть список рассчитанных затрат.</span><span class="sxs-lookup"><span data-stu-id="ddbda-133">Alternatively, use the **Calculation** page to review the calculated costs.</span></span>
5.  <span data-ttu-id="ddbda-134">Используйте страницу **Настройка версии калькуляции издержек** для изменения блокирующего флага, чтобы активировать записи предстоящих затрат во второй версии расчета себестоимости.</span><span class="sxs-lookup"><span data-stu-id="ddbda-134">Use the **Costing version setup** page to change the blocking flag to allow activation of the pending cost records in the second costing version.</span></span>
6.  <span data-ttu-id="ddbda-135">Используйте страницу **Активировать цены** (которая открывается со страницы **Поддержка версии калькуляции издержек**) для включения всех записей предстоящих затрат в рамках второй версии расчета себестоимости.</span><span class="sxs-lookup"><span data-stu-id="ddbda-135">Use the **Activate prices** page (which you open from the **Costing version maintenance** page) to enable all pending cost records in the second costing version.</span></span> <span data-ttu-id="ddbda-136">Кроме того, записи по ожидаемым затратам для отдельных номенклатур можно включить, нажав кнопку **Активировать** на странице **Цена номенклатуры**.</span><span class="sxs-lookup"><span data-stu-id="ddbda-136">You can also enable the pending cost records for individual items by clicking the **Activate** button on the **Item price** page.</span></span>
7.  <span data-ttu-id="ddbda-137">Используйте страницу **Настройка версии калькуляции издержек** для изменения блокирующих флагов в рамках второй версии расчета себестоимости, чтобы избежать поддержки дополнительных данных.</span><span class="sxs-lookup"><span data-stu-id="ddbda-137">Use the **Costing version setup** page to change the blocking flags in the second costing version to prevent additional data maintenance.</span></span> <span data-ttu-id="ddbda-138">Политики блокирования помогут предотвратить ввод новых предстоящих затрат и активацию предстоящих затрат.</span><span class="sxs-lookup"><span data-stu-id="ddbda-138">The blocking policies prevent the entry of new pending costs and the activation of pending costs.</span></span>
