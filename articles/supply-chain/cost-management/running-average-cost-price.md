<span data-ttu-id="14375-105">Процесс закрытия запасов сопоставляет проводки расходов с проводками приходов на основе метода оценки запасов, который выбирается в группе номенклатурной модели номенклатуры.</span><span class="sxs-lookup"><span data-stu-id="14375-105">The inventory close process settles issue transactions to receipt transactions, based on the inventory valuation method that is selected in the item’s item model group.</span></span> <span data-ttu-id="14375-106">Однако перед запуском закрытия запасов система вычисляет скользящую среднюю себестоимость, которая обычно используется при разноске проводок расхода.</span><span class="sxs-lookup"><span data-stu-id="14375-106">However, before inventory close is run, the system calculates a running average cost price that is typically used when issue transactions are posted.</span></span>

## <a name="avoiding-pricing-amplification"></a><span data-ttu-id="14375-126">Как избежать увеличения цены</span><span class="sxs-lookup"><span data-stu-id="14375-126">Avoiding pricing amplification</span></span>
<span data-ttu-id="14375-127">В редких случаях система определяет себестоимость по нескольким расходам до получения достаточного количества приходов в качестве основы для расчета цены.</span><span class="sxs-lookup"><span data-stu-id="14375-127">On rare occasions, the system prices several issues before it has enough receipts to base the price on.</span></span> <span data-ttu-id="14375-128">В этом сценарии оценки скользящей средней себестоимости могут быть сильно преувеличены.</span><span class="sxs-lookup"><span data-stu-id="14375-128">This scenario can cause estimates of the running average cost price to be overly inflated.</span></span> <span data-ttu-id="14375-129">Однако можно предпринять некоторые шаги, чтобы избежать завышенного ценообразования или смягчить его влияние, если оно все-таки происходит.</span><span class="sxs-lookup"><span data-stu-id="14375-129">However, there are steps that you can take to avoid pricing amplification, or to reduce its impact when it does occur.</span></span> <span data-ttu-id="14375-130">**Сценарий** Следующие проводки происходят для номенклатуры, для которой выбран параметр **Включать физическую стоимость**:</span><span class="sxs-lookup"><span data-stu-id="14375-130">**Scenario** The following transactions occur for an item that you've selected the **Include physical value** option for:</span></span>

1.  <span data-ttu-id="14375-131">Вы финансово получаете количество, равное 100, по цене USD 100,00.</span><span class="sxs-lookup"><span data-stu-id="14375-131">You financially receive a quantity of 100 at USD 100.00.</span></span>
2.  <span data-ttu-id="14375-132">Вы финансово расходуете количество, равное 200.</span><span class="sxs-lookup"><span data-stu-id="14375-132">You financially issue a quantity of 200.</span></span>
3.  <span data-ttu-id="14375-133">Вы физически получаете количество, равное 101, по цене USD 202,00.</span><span class="sxs-lookup"><span data-stu-id="14375-133">You physically receive a quantity of 101 at USD 202.00.</span></span>

<span data-ttu-id="14375-134">При проверке расчетной скользящей средней себестоимости номенклатуры вы ожидаете себестоимость USD 1,51.</span><span class="sxs-lookup"><span data-stu-id="14375-134">When you examine the estimated running average cost price for the item, you expect a cost price of USD 1.51.</span></span> <span data-ttu-id="14375-135">Вместо этого расчетная скользящая средняя оказывается равной USD 102,00 на основе следующей формулы: Расчетная цена = \[202 + (-100)\] ÷ \[101 + (-100)\] = 102 ÷ 1 = 102. Такое увеличение ценообразования происходит потому, что при финансовом расходе 200 единиц номенклатуры на шаге 2 система должна определить цену 100 единиц номенклатуры, прежде чем появятся какие-то соответствующие приходы.</span><span class="sxs-lookup"><span data-stu-id="14375-135">Instead, you find an estimated running average of USD 102.00, which is based on the following formula: Estimated price = \[202 + (-100)\] ÷ \[101 + (-100)\] = 102 ÷ 1 = 102 This pricing amplification occurs because, when 200 items are issued financially in step 2, the system must price 100 of the items before it has any corresponding receipts.</span></span> <span data-ttu-id="14375-136">Это приводит к отрицательной величине запасов.</span><span class="sxs-lookup"><span data-stu-id="14375-136">This situation causes negative inventory.</span></span> <span data-ttu-id="14375-137">Затем система рассчитает цену за единицу равной USD 1,00, как и ожидалось.</span><span class="sxs-lookup"><span data-stu-id="14375-137">The system then estimates a unit price of USD 1.00, as we might expect.</span></span> <span data-ttu-id="14375-138">Однако при получении соответствующего прихода 100 единиц они имеют цену за единицу USD 2,00 каждая.</span><span class="sxs-lookup"><span data-stu-id="14375-138">However, when the corresponding 100 receipts arrive, they are at a unit price of USD 2.00 each.</span></span> 

<span data-ttu-id="14375-139">**Примечание.** Хотя расходы создают отрицательный запас, запасы имеют положительное значение при вычислении цены расхода.</span><span class="sxs-lookup"><span data-stu-id="14375-139">**Note:** Although the issues create negative inventory, inventory is positive when the issue price is calculated.</span></span> <span data-ttu-id="14375-140">Поэтому используется скользящая средняя себестоимость вместо цены по главной записи номенклатуры.</span><span class="sxs-lookup"><span data-stu-id="14375-140">Therefore, the running average cost price is used instead of the price on the item master.</span></span> <span data-ttu-id="14375-141">В этой точке система имеет смещение величины стоимости запасов, равное USD 100,00.</span><span class="sxs-lookup"><span data-stu-id="14375-141">At this point, the system has an inventory value offset of USD 100.00.</span></span> <span data-ttu-id="14375-142">Хотя данное смещение было создано для 100 шт., там где была величина смещения по USD 1,00 для каждой единицы, мы теперь имеем только одну штуку в запасах.</span><span class="sxs-lookup"><span data-stu-id="14375-142">Although that offset was built up over 100 pieces, where there was a unit offset of USD 1.00 each, we now have only one piece in inventory.</span></span> <span data-ttu-id="14375-143">Поэтому смещение USD 100,00 распределяется в эту единственную штуку.</span><span class="sxs-lookup"><span data-stu-id="14375-143">Therefore, the offset of USD 100.00 is allocated to this one piece.</span></span> <span data-ttu-id="14375-144">В результате получается очень сильно преувеличенная расчетная себестоимость.</span><span class="sxs-lookup"><span data-stu-id="14375-144">The result is the overly inflated estimated cost price.</span></span>

<span data-ttu-id="14375-151">Однако если используемая бизнес-модель допускает следующие процедуры, они могут помочь при предотвращении таких отрицательных количеств, которые сделают возможным сценарий увеличения цены:</span><span class="sxs-lookup"><span data-stu-id="14375-151">However, if your business model allows for the following practices, they can help you avoid the negative quantities that make the pricing amplification scenario possible:</span></span>
