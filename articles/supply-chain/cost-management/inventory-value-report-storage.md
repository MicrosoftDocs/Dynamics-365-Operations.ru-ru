# <a name="inventory-value-reports"></a>Отчеты о стоимости запасов

[!include [banner](../includes/banner.md)]

В отчетах о стоимости запасов приводятся подробные сведения о физических и финансовых количествах запасов и суммах. Просматривать отчеты можно различными способами. Например, можно отобразить итоговые значения или проводки или фильтровать по номенклатурам или диапазону времени. Можно просмотреть значения себестоимости проданных товаров (COGS) или суммы незавершенного производства (НЗП) и задать другие параметры.

Отчеты о стоимости запасов позволяют выполнять следующие задачи:

- Выполнять выверку между главной книгой и запасами.
- Изучить запасы в наличии и стоимость за указанный период.
- Создавать конфигурации отчетов, которые адаптированы к конкретному назначению.
- Сохранять конфигурации отчетов, чтобы их можно было использовать несколько раз.
- Добавлять диапазоны для фильтрации данных при выполнении отчета.

## <a name="types-of-inventory-value-report"></a>Типы отчетов о стоимости запасов

Имеется два типа отчетов о стоимости запасов: **Стоимость запасов** (стандартный отчет) и **Сохраненные отчеты о стоимости запасов**.

### <a name="standard-inventory-value-report"></a>Стандартный отчет о стоимости запасов

Стандартный отчет **Стоимость запасов** — это простой отчет, который позволяет выбрать включаемые сведения, а затем отображает эту информацию на экране. Результаты не сохраняются. Он также не предоставляет интерактивных функций для фильтрации, детализации, просмотра или экспорта. По этим причинам рекомендуется вместо этого в большинстве случаев использовать **сохраненный отчет о стоимости запасов**.

### <a name="inventory-value-report-storage-report"></a>Сохраненный отчет о стоимости запасов

Отчет **Сохраненный отчет о стоимости запасов** предоставляет выходные данные либо как интерактивную страницу в Microsoft Dynamics 365 Supply Chain Management, либо в виде экспортированного документа в одном из нескольких форматов.

При просмотре отчета в браузере столбцы и агрегированные балансы динамически корректируются в зависимости от настроенного вами макета. Можно сортировать результаты, фильтровать их, детализировать данные и т. д.

Результаты отчета хранятся в информационном объекте **Стоимость запасов**. Таким образом, можно выполнять фильтрацию и экспорт результатов в таком формате, как значения с разделителями (CSV) или Microsoft Excel.

Отчет **Сохраненный отчет о стоимости запасов** полезен, когда вывод содержит много строк. Например, имеется 50 000 номенклатур, и 300 магазинов были созданы как склады. В этом случае, если вы запрашиваете конечные сальдо запасов по номенклатуре, сайту и складу, на выходе будет содержаться много строк.

> [!NOTE]
> Отчет **Сохраненный отчет о стоимости запасов** не включает промежуточные итоги, определенные в макете отчета. Он также не включает сальдо главной книги, даже если эти сальдо определены в макете отчета. Выверка в главную книгу должна выполняться с использованием пробных балансов. Однако стандартный отчет **Стоимость запасов** включает эти промежуточные суммы и сальдо.

## <a name="turn-the-inventory-value-report-storage-feature-on-or-off"></a>Включение или отключение функции сохраненного отчета о стоимости запасов

В Supply Chain Management версии 10.0.25 эта функция включена по умолчанию. Администраторы могут включать и выключать эту функцию, выполнив поиск функции *Сохраненные отчеты о стоимости запасов* в рабочей области [Управление функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

## <a name="define-inventory-value-report-configurations"></a><a name="report-configuration"></a>Определение конфигураций отчета о стоимости запасов

Страница **Отчеты о стоимости запасов** используется для настройки содержимого, включенного в различные типы отчета о стоимости запасов. Можно определить любое количество типов отчетов. Каждый раз при создании отчета о стоимости запасов любого типа вы будете выбирать тип отчета.

1. Перейдите в раздел **Управление затратами \> Настройка учетных политик запасов \> Отчеты о стоимости запасов**.
1. Выполните одно из следующих действий.

    - Чтобы изменить существующий отчет, выберите его на панели списка, затем выберите **Изменить** на панели операций.
    - Для создания нового отчета выберите **Создать** на панели операций.

1. В заголовке нового или выбранного отчета задайте следующие поля:

    - **ИД** — введите короткий идентификатор для отчета. Это значение должно быть уникальным среди всех конфигураций отчетов о стоимости запасов. После сохранения новой конфигурации она не может быть изменена.
    - **Имя** — введите описательное имя для отчета.

1. При создании новой конфигурации отчета выберите **Сохранить** на панели действий, чтобы сделать доступными остальные поля.
1. На экспресс-вкладке **Общие** настройте следующие поля:

    - **Интервал дат** — выбор заранее определенного интервала дат. Можно переопределить этот интервал дат при выполнении отчета.
    - **Диапазон** — выберите *Дата разноски* или *Время проводки*, в зависимости от даты и времени, которые должны использоваться при получении записей для отчета.
    - **Набор аналитик** — выберите набор аналитик, для которых необходимо выполнить данные. (Аналитики определяются в главной книге.) Например, можно выполнить данные для *счета ГК* или для *счета ГК и подразделений*. Выбранный набор аналитик должен содержать не более двух аналитик. Дополнительные сведения см. в разделе [Наборы финансовых аналитик](../../finance/general-ledger/financial-dimension-sets.md).

1. Задайте следующие поля на экспресс-вкладке **Столбцы**. Эти поля управляют столбцами, содержащими отчет, и типами данных, содержащимися в этих столбцах.

    - **Запасы** — установите для этого параметра значение *Да*, чтобы отобразить значения запасов. После этого можно согласовать эти значения с сальдо по счету главной книги.
    - **НЗП** — установите для этого параметра значение *Да*, чтобы отобразить значения НЗП. После этого можно согласовать эти значения с сальдо по счету НЗП в главной книге. Если для этого параметра установлено значение *Да*, в отчете будут показаны только физические количества и суммы запасов со статусом "НЗП". Производственные заказы со статусом НЗП были скомплектованы или учтены как готовые, но они не были завершены.
    - **Отложенная себестоимость проданных товаров** — установите для этого параметра значение *Да*, чтобы вывести столбец, в котором отображаются физические количества и суммы запасов для отложенной себестоимости проданных товаров. Отложенная себестоимость проданных товаров отображается с использованием физических количеств и сумм, поскольку она смещает количества и суммы по отборочной накладной.
    - **Себестоимость проданных товаров** — установите для этого параметра значение *Да*, чтобы вывести столбец, в котором отображаются финансовые количества и суммы запасов для себестоимости проданных товаров. Себестоимость проданных товаров отображается с использованием финансовых количеств и сумм, поскольку она смещает количества и суммы по накладной.
    - **Прибыли и убытки** — установите для этого параметра значение *Да*, чтобы вывести столбец, в котором отображается финансовая сумма, которая была разнесена на счета прибылей и убытков для запасов.
    - **Печать совокупных значений счетов для сравнения** — установите для этого параметра значение *Да*, чтобы отобразить столбец, в котором отображается сальдо счета ГК. Таким образом, вам не придется проверять пробный баланс. Этот параметр применяется только для стандартного отчета **Стоимость запасов**, но не для отчета **Сохраненные отчеты о стоимости запасов**. После установки для этого параметра значения *Да* необходимо использовать следующие поля, чтобы указать каждый счет ГК, который требуется включить в список, в зависимости от включенных параметров **Финансовое положение**.

        > [!NOTE]
        > При выборе *итогового* счета для любого из этих полей будет отображаться как сумма каждого счета запасов, включенного в итоговый счет, так и сумма итогового счета.

        - **Счет запасов** — укажите счет ГК, для которого необходимо отобразить сведения о запасах. (Для обоих параметров **Запасы** и **Печать совокупных значений счетов для сравнения** должно быть задано значение *Да*.)
        - **Счет НЗП** — укажите счет ГК, для которого необходимо отобразить сведения о НЗП. (Для обоих параметров **НЗП** и **Печать совокупных значений счетов для сравнения** должно быть задано значение *Да*.)
        - **Счет COGS** — укажите счет ГК, для которого необходимо отобразить сведения об отложенной себестоимости проданных товаров. (Для обоих параметров **Отложенная себестоимость проданных товаров** и **Печать совокупных значений счетов для сравнения** должно быть задано значение *Да*.)
        - **Счет COGS** — укажите счет ГК, для которого необходимо отобразить сведения о стоимости проданных товаров. (Для обоих параметров **Себестоимость проданных товаров** и **Печать совокупных значений счетов для сравнения** должно быть задано значение *Да*.)

    - **Суммировать физические и финансовые значения** — установите для этого параметра значение *Да*, чтобы вывести столбец, содержащий общее количество запасов и сумму запасов (сводную информацию о значениях физических и финансовых запасов). Если для этого параметра установлено значение *Нет*, в отчете будут показаны как физические, так и финансовые значения запасов.
    - **Включить не разнесенные в книгу учета** Установите для этого параметра значение *Да*, чтобы отобразить столбец, в котором показаны проводки, которые никогда не были разнесены в главную книгу. Проводки для следующих типов номенклатур не могут быть разнесены в главную книгу:

        - Полученные номенклатуры, по которым еще не выставлены накладные, когда снят флажок **Разносить физические запасы** для соответствующей группы номенклатурных моделей.
        - Полученные номенклатуры, по которым еще не выставлены накладные, если снят флажок **Разнести поступление продуктов в ГК** на экспресс-вкладке **Поступление продуктов** на вкладке **Общие** на странице **Параметры модуля расчетов с поставщиками** (**Расчеты с поставщиками \> Настройка \> Параметры модуля расчетов с поставщиками**).

    - **Расчет средней стоимости единицы измерения** — установите для этого параметра значение *Да*, чтобы отображать столбец, содержащий среднюю стоимость единицы. Средняя стоимость единицы — общая сумма, разделенная на общее количество.
    - **Итоговые количество и стоимость** — установите для этого параметра значение *Да*, чтобы вывести столбцы, содержащие общее количество физических запасов (и финансовые количества) и общую сумму физических запасов (и финансовых сумм). Можно установить для этого параметра значение *Да* только в том случае, если для параметра **Суммировать физические и финансовые значения** установлено значение *Нет*.
    - **Складские аналитики** — в этой сетке установите флажок **Просмотр** для каждой аналитики, которая должна отображаться в отчете. Значения в отчете будут отображаться только для аналитик, для которых включен параметр **Финансовые запасы**. Другие аналитики будут показывать только пустые столбцы. Для тех аналитик, которые выбраны для отображения, можно установить флажок **Итого**, чтобы включить также итоги.
    - **Код ресурса** — установите для параметра **Просмотр** значение *Да*, чтобы отобразить столбец, идентифицирующий номенклатуру для каждой строки. Установите для параметра **Итоги** значение *Да*, чтобы включить также итоговые значения. В зависимости от типа номенклатуры, указанной в каждой строке, в столбце отображаются сведения одного из следующих типов:

        - **Материал** — столбец показывает значение поля `ItemID` для соответствующей записи материала.
        - **Трудозатраты** — столбец показывает значение поля `WorkCenterID` для соответствующей записи трудозатрат.
        - **Косвенные затраты** — столбец показывает значение поля `CodeID` для соответствующей записи затрат.

        Если для параметра **Просмотр** задано значение *Нет* для обоих полей **Код ресурса** и **Группа ресурсов**, будет отображена только общая стоимость запасов, основанная на выбранных складских аналитиках.

    - **Группа ресурсов** — установите для параметра **Просмотр** значение *Да*, чтобы отобразить столбец, идентифицирующий группу ресурсов для каждой строки. Установите для параметра **Итоги** значение *Да*, чтобы включить также итоговые значения. В зависимости от типа номенклатуры, указанной в каждой строке, в столбце отображаются сведения одного из следующих типов:

        - **Материал** — столбец показывает значение поля `ItemGroup` для соответствующей записи материала.
        - **Трудозатраты** — столбец показывает значение поля `WorkcenterGroup` для соответствующей записи трудозатрат.
        - **Косвенные затраты** — столбец показывает значение поля `CostGroup` для соответствующей записи затрат. (Значение `CostGroupType` должно быть *Косвенный*.)

        Если для параметра **Просмотр** задано значение *Нет* для обоих полей **Код ресурса** и **Группа ресурсов**, будет отображена только общая стоимость запасов, основанная на выбранных складских аналитиках.

1. На экспресс-вкладке **Строки** задайте следующие поля. Эти поля позволяют добавлять в отчет соответствующие подразделы, связанные с НЗП, или удалять их.

    - **Материал** — установите для этого параметра значение *Да*, чтобы отобразить сведения о материалах. *Материал* является типом ресурсов по умолчанию, поскольку материалы должны включаться во все конфигурации отчетов для создания надежных выходных данных.
    - **Трудозатраты** — установите для этого параметра значение *Да*, чтобы показать затраты на трудовые ресурсы в НЗП.
    - **Косвенные затраты** — установите для этого параметра значение *Да*, чтобы показать косвенные затраты на НЗП.
    - **Прямой аутсорсинг** — установите для этого параметра значение *Да*, чтобы показать затраты на прямой аутсорсинг в НЗП. Эта информация полезна для субподряда.
    - **Уровень детализации** — выберите параметр просмотра для отчета:

        - *Проводки* — просмотр всех соответствующих проводок в отчете. Обратите внимание, что при просмотре отчетов, содержащих большой объем проводок, могут возникнуть проблемы с производительностью. Поэтому, если вы хотите использовать этот параметр просмотра, рекомендуется использовать отчет **Сохраненный отчет о стоимости запасов**.
        - *Итоги* — просмотр итогового результата.

    - **Включить начальный баланс** — установите для этого параметра значение *Да*, чтобы отобразить начальное сальдо. Этот параметр доступен только в случае, если в поле **Уровень детализации** установлено значение *Транзакции*.

## <a name="generate-an-inventory-value-report-storage-report"></a>Формирование сохраненного отчета о стоимости запасов

Выполните следующие действия, чтобы создать и сохранить отчет **Сохранный отчет о стоимости запасов**.

1. Перейдите **Управление затратами \> Запросы и отчеты \> Хранение отчета о стоимости запасов**.
1. В области действий выберите **Создать**.
1. В диалоговом окне **Стоимость запасов** на экспресс-вкладке **Параметры** установите следующие поля:

    - **Имя** — введите уникальное имя для отчета.
    - **Код** — выберите [конфигурацию отчета о стоимости запасов](#report-configuration) для использования в отчете. Конфигурация устанавливает параметры для столбцов и строк, которые будут включены в отчет.
    - **Интервал дат** — поля в этом разделе используются для определения записей, включаемых в отчет. Чтобы определить интервал дат, можно либо выбрать предварительно определенный диапазон ( относительно даты создания отчета) в поле **Код интервала дат**, либо выбрать определенные даты в полях **Начальная дата** и **Конечная дата**.

1. На экспресс-вкладке **Записи для добавления** настройте фильтры и ограничения, чтобы определить, какие данные включаются в отчет. Выберите **Фильтр**, чтобы открыть стандартное диалоговое окно редактора запросов, в котором можно определить критерии выбора, критерии сортировки и соединения. Поля работают так же, как они работают для других типов запросов в Supply Chain Management. Все эти фильтры будут применяться к складским проводкам, но не к сальдо главной книги. При настройке фильтров учитывайте подобное поведение. В противном случае может появиться расхождение между запасами и главной книгой.
1. На экспресс-вкладке **Выполнять в фоновом режиме** укажите как, когда и как часто создается отчет. Поля работают так же, как они работают для других типов [фоновых заданий](../../fin-ops-core/dev-itpro/sysadmin/batch-processing-overview.md) в Supply Chain Management.

    > [!NOTE]
    > Этот отчет всегда выполняется как часть пакетного задания.

1. Выберите **ОК**, чтобы применить настройки и закрыть диалоговое окно.

После завершения пакетного задания отчет будет указан на странице **Хранение отчета о стоимости запасов**. Возможно потребуется обновить страницу, чтобы увидеть этот отчет.

> [!IMPORTANT]
> В выбранной конфигурации отчета о стоимости запасов можно получить неправильный начальный баланс, если выбрана та же самая дата в поле **Начальная дата** и **Конечная дата**, а также для параметра **Включить начальный баланс** установлено значение *Да*.

## <a name="explore-an-inventory-value-report-storage-report"></a>Изучение сохраненного отчета о запасах

После создания отчета его можно просмотреть и изучить в любое время, выполнив указанные ниже действия.

1. Перейдите **Управление затратами \> Запросы и отчеты \> Хранение отчета о стоимости запасов**.
1. Выберите отчет в списке. На странице отображаются сведения о [конфигурации отчета о стоимости запасов](#report-configuration), которая использовалась для создания выбранного отчета.
1. В области действий выберите **Просмотреть сведения** для отображения содержимого отчета.
1. Просмотрите отчет, выполнив любые из следующих действий:

    - Как и в случае большинства стандартных страниц в Supply Chain Management, можно выбрать практически любой заголовок столбца для сортировки или фильтрации сетки по значениям в этом столбце.
    - Поле **Фильтр** используется для фильтрации отчета по любому значению в любом из нескольких доступных столбцов.
    - Используйте меню представления (над полем **Фильтр**) для сохранения и загрузки избранных комбинаций параметров сортировки и фильтрации.

## <a name="export-an-inventory-value-report-storage-report"></a>Экспорт сохраненного отчета о стоимости запасов

Каждый созданный отчет сохраняется в информационный объекте **Стоимость запасов**. Можно использовать стандартные функции управления данными Supply Chain Management, чтобы экспортировать данные из этого объекта в любой поддерживаемый формат данных, такой как CSV или формат Excel.

В следующем примере показано, как экспортировать отчет **Сохраненный отчет о стоимости запасов**.

1. Перейдите в раздел **Администрирование системы \> Рабочие области \> Управление данными**.
1. В разделе **Импорт/экспорт** выберите плитку **Экспорт**.
1. На странице **Экспорт**, которая появится, вы настроите задание экспорта. Сначала введите имя группы для задания.
1. В разделе **Выбранные объекты** выберите **Добавить объект**.
1. В открывшемся диалоговом окне задайте следующие поля:

    - **Имя объекта** — выберите *Стоимость запасов*.
    - **Формат целевых данных** — выберите формат, в который необходимо экспортировать данные.

1. Выберите **Добавить**, чтобы добавить новую строку, а затем нажмите кнопку **Закрыть**, чтобы закрыть диалоговое окно.
1. Обычно вы будете экспортировать один отчет за раз. Чтобы экспортировать один отчет, настройте фильтр для строки, только что добавленной в диалоговом окне **Запрос**. Таким образом можно определить, какой отчет из объекта **Стоимость запасов** включен в экспорт. Выполните следующие шаги, чтобы задать следующие параметры фильтра для экспорта одного отчета:

    1. На вкладке **Диапазон** выберите **Добавить**, чтобы добавить строку.
    2. Задайте в поле **Таблица** значение *Стоимость запасов*.
    3. Задайте в поле **Производная таблица** значение *Стоимость запасов*.
    4. Задайте в поле **Поле** то поле, по которому необходимо выполнить фильтрацию. Обычно используется поле **Имя выполнения** и/или поле **Время выполнения**.
    5. В поле **Критерии** задайте значение, которое необходимо искать в выбранном поле. (Если на предыдущем шаге было выбрано поле **Имя выполнения**, это значение будет именем отчета. Если было выбрано поле **Время выполнения**, это будет время создания отчета.)
    6. Добавляйте требуемые дополнительные строки на вкладку **Диапазон** до тех пор, пока не будет найден необходимый отчет.

1. Выберите **ОК**, чтобы сохранить настройки и закрыть диалоговое окно.
1. Для сохранения настроек экспорта выберите **Сохранить**.
1. На вкладке **Параметры экспорта** выберите **Экспортировать**, чтобы создать файл экспорта.
1. На открывшейся странице **Сводка по выполнению** можно посмотреть статус задания экспорта и список экспортированных объектов. В разделе **Статус обработки объекта** выберите объект **Стоимость запасов** в списке, затем выберите **Загрузить файл**, чтобы загрузить данные, которые были экспортированы из этого объекта.

Дополнительные сведения об использовании управления данными для экспорта данных см. в разделе [Обзор заданий импорта и экспорта данных](../../fin-ops-core/dev-itpro/data-entities/data-import-export-job.md).

## <a name="generate-a-standard-inventory-value-report"></a>Формирование стандартного отчета о стоимости запасов

Следующая процедура используется для создания стандартного отчета **Стоимость запасов**.

    - **Код** — выберите [конфигурацию отчета о стоимости запасов](#report-configuration) для использования в отчете. Конфигурация устанавливает параметры для столбцов и строк, которые будут включены в отчет.
    - **Интервал дат** — поля в этом разделе используются для определения записей, включаемых в отчет. Чтобы определить интервал дат, можно либо выбрать предварительно определенный диапазон ( относительно даты создания отчета) в поле **Код интервала дат**, либо выбрать определенные даты в полях **Начальная дата** и **Конечная дата**.

1. На экспресс-вкладке **Записи для добавления** настройте фильтры и ограничения, чтобы определить, какие данные включаются в отчет. Выберите **Фильтр**, чтобы открыть стандартное диалоговое окно редактора запросов, в котором можно определить критерии выбора, критерии сортировки и соединения. Поля работают так же, как они работают для других типов запросов в Supply Chain Management.
1. На экспресс-вкладке **Выполнять в фоновом режиме** укажите как, когда и как часто создается отчет. Поля работают так же, как они работают для других типов [фоновых заданий](../../fin-ops-core/dev-itpro/sysadmin/batch-processing-overview.md) в Supply Chain Management.
1. Выберите **ОК**, чтобы применить настройки и закрыть диалоговое окно. Появляется отчет.

## <a name="reading-inventory-value-reports"></a>Чтение отчетов о стоимости запасов

В этом разделе представлено несколько руководств по чтению и пониманию отчета о стоимости запасов.

Supply Chain Management поддерживает два следующих важных понятия, связанных со статусом запасов:

- **Финансово обновленные** — эта концепция указывает, что для складских проводок уже выставлены накладные. Для производственных заказов он указывает на окончание производственного заказа.
- **Физический обновлен** — эта концепция указывает, что для складских проводок еще не выставлены накладные, но они получены или отгружены. Для производственных заказов это свидетельствует о том, что материал был скомплектован или производственный заказ был учтен как завершенный.

Когда вы понимаете эти два понятия, вам будет легче понять следующие столбцы в выходных данных отчета:

- **Запасы: финансовое количество** — количество, которое было финансово обновлено.
- **Запасы: финансовая сумма** — значение суммы запасов, которые были финансово обновлены.
- **Запасы: разнесенное физическое количество** — количество, которое было физически обновлено.
- **Запасы: разнесенная физическая сумма** — значение суммы запасов, которые были физически обновлены.
- **Запасы: неразнесенное физическое количество** — количество, имеющее складские проводки, но еще не разнесенное в главную книгу. Например, имеется группа номенклатурных моделей, в которой флажки **Разносить физические запасы** и **Разносить финансовые запасы** сняты, и имеется номенклатура, связанная с этой группой. Затем вы создаете заказ на покупку, получаете его и выставляете на него накладную. В этот момент, если просмотреть отчет о стоимости запасов для данной номенклатуры, вы увидите, что количество и значение в заказе на покупку отображаются в столбцах **Запасы: неразнесенное физическое количество** и **Запасы: неразнесенная физическая сумма**.
- **Запасы: неразнесенная физическая сумма** — можно настроить отчеты таким образом, чтобы они отображали неразнесенные суммы. Однако, если вы используете отчет для выверки запасов, это значение использовать не следует. В противном случае сумма не разносится в главную книгу.
- **Запасы: количество** — общее количество по всем столбцам количества в отчете.
- **Запасы: сумма** — общее количество по всем столбцам сумм в отчете. При выверке запасов не используйте этот столбец, если отчет включает в себя столбец **Запасы: неразнесенная физическая сумма**. В этом случае необходимо исключить сумму **Запасы: неразнесенная физическая сумма** из общей суммы.
- **Средняя стоимость единицы** — общая сумма, разделенная на общее количество.

Обычно для просмотра стоимости и количества запасов будет использоваться отчет о стоимости запасов. Однако иногда в этом отчете не отображаются все соответствующие складские аналитики. Если ожидаемые аналитики не отображаются, проверьте следующие настройки:

- Проверьте группы аналитик хранения номенклатур и аналитик отслеживания. В отчете могут отображаться только аналитики, для которых включен параметр **Финансовые запасы**.
- Перейдите в раздел **Управление затратами \> Настройка учетных политик запасов \> Отчеты о стоимости запасов**, выберите конфигурацию отчета, использованную для создания отчета, и убедитесь, что в столбце **Просмотр** выбраны необходимые складские аналитики.

Например, имеется номенклатура с номером номенклатуры *A0001*. В группе аналитики хранения для финансовых запасов доступен только сайт. Сайт и склад оба включены для физических запасов. В группе аналитик отслеживания номер партии включен для физических запасов, но не для финансовых запасов. Затем используйте конфигурацию отчета, где выбраны сайт, склад и номер партии. При просмотре отчета значение отображается только для сайта. Столбцы для склада и номера партии пусты. Как показано в этом примере, в отчетах о стоимости запасов могут отображаться только складские аналитики, включенные для финансовых запасов.
