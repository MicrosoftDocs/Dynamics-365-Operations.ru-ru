# <a name="try-out-scale-units-in-a-distributed-hybrid-topology"></a>Попробуйте единицы масштабирования в распределенной гибридной топологии

[!include [banner](../includes/banner.md)]

Процесс опробования распределенной гибридной топологии прост. На первом этапе необходимо проверить настройки, чтобы убедиться в том, что они работают в распределенной топологии. Имеется два варианта.

## <a name="option-1-evaluate-customizations-in-development-environments"></a>Вариант 1. Оценка настроек в средах разработки

Прежде чем приступить к работе со средой "песочницы", рекомендуется изучить единицы масштабирования в настройке разработки, например в односерверной среде (называемой также средой уровня 1), чтобы можно было проверять процессы, настройки и решения. На этом этапе данные и настройки будут применяться к односерверных средам. Можно работать в одной среде, для которой может быть назначена роль как концентратора предприятия, так и единицы масштабирования. В качестве альтернативы можно использовать две среды разработки, одна из которых принимает роль концентратора, а другая — роль единицы масштабирования. Эта настройка обеспечивает лучший способ идентификации и устранения проблем. Для выполнения этого этапа также можно использовать последнюю [сборку с ранним доступом (PEAP)](https://forms.office.com/FormsPro/Pages/ResponsePage.aspx?id=v4j5cvGGr0GRqy180BHbR56j8lZs0FdAvwT75_WNFyxURUFWTjQzTzg0UUk5RkJHMDFEMVlSSDFEQy4u).

Следует использовать [инструменты развертывания единиц масштабирования в односерверных средах разработки](https://github.com/microsoft/SCMScaleUnitDevTools), чтобы устанавливать и поддерживать среды. Эти инструменты позволяют настраивать концентраторы и единицы масштабирования в одной или двух односерверных средах. Эти инструменты предоставляются как в виде двоичного выпуска, так и в исходном коде в GitHub. Изучите вики-сайт проекта, который включает в себя [Пошаговое руководство по использованию](https://github.com/microsoft/SCMScaleUnitDevTools/wiki/Step-by-step-usage-guide), которое описывает использование этих инструментов. При [развертывании пограничных единиц масштабирования на пользовательском оборудовании с использованием локальных бизнес-данных (LBD)](cloud-edge-edge-scale-units-lbd.md) необходимо выполнить другой процесс.

## <a name="option-2-acquire-add-ins-and-deploy-in-your-sandbox-environments"></a>Вариант 2. Получение надстроек и развертывание в средах "песочницы"

Чтобы испытать распределенную гибридную топологию, необходимо иметь две среды "песочницы" (уровень 2), один из которых содержит данные (корпоративный концентратор), а другой предназначен для единицы масштабирования и содержит "пустые данные".

Необходимо приобрести надстройки по крайней мере для одного облака или пограничной единицы масштабирования. После этого соответствующие слоты проекта и среды будут предоставляться в [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/), так что среды с единицей масштабирования можно развернуть с помощью [портала управления единицами масштабирования](https://aka.ms/SCMSUM).

Обратитесь в службу поддержки Майкрософт, чтобы запросить включение сред "песочницы".
