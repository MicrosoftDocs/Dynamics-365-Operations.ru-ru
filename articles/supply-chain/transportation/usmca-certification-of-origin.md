# <a name="usmca-certification-of-origin"></a>Сертификация происхождения USMCA

[!include [banner](../includes/banner.md)]

Эта функция позволяет печатать документы сертификатов происхождения, необходимые для соглашения США-Мексика-Канада (USMCA).

*Документ сертификации происхождения USMCA* содержит минимальные элементы данных, необходимые для объявления. Некоторые элементы данных могут быть предварительно заполнены перед печатью, в то время как другие должны быть заполнены вручную после печати. Чтобы получить тарифные преференции, документ должен быть завершен и находиться в распоряжении импортера в момент подачи декларации. Документ может быть заполнен импортером, экспортером или производителем.

Документ для одной отгрузки можно напечатать со страницы списка **Все отгрузки** или со страницы **Сведения об отгрузке**.

Документ доступен только в том случае, когда страной основного адреса для юридического лица является США.

В зависимости от выбора печати документа, документ может быть заранее заполнен данными из системы. Можно изменять или добавлять данные в печатный документ путем экспорта печатного документа в редактируемый формат, например Microsoft Word. После экспорта можно применить все необходимые изменения перед созданием декларации.

## <a name="turn-on-the-usmca-feature"></a>Включение функции USMCA

Прежде чем использовать функцию USMCA, она должна быть включена в системе. Администраторы могут использовать параметры [управления компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление транспортировкой*
- **Название компонента:** *Документ сертификации происхождения USMCA*

## <a name="document-content"></a>Содержание документа

Документ сертификации происхождения USMCA содержит следующие элементы данных:

- Элементы адреса
- Роль субъекта сертификации
- Отдельная отгрузка
- Накладные
- Срок действия
- Сведения об элементе
- Подпись удостоверяющего
- Число страниц

Дополнительные сведения о каждом из этих элементов и порядке поиска их значений см. в разделах, оставшихся в данной теме.

## <a name="print-a-usmca-certification-of-origin-document"></a>Печать документа сертификации происхождения USMCA

Чтобы распечатать документ сертификации происхождения USMCA для отгрузки, выполните следующие действия:

1. Выполните одно из следующих действий.
    - Перейдите в раздел **Управление транспортировкой > Отгрузки > Все отгрузки** и выберите отгрузку, для которой требуется напечатать документ.
    - Откройте страницу **Сведения об отгрузке** для отгрузки, для которой необходимо напечатать документ (для этого существует несколько способов, включая страницу **Все отгрузки**).
1. На панели операций откройте вкладку **Отгрузки** и в группе **Печать** выберите **Сертификат происхождения USMCA**.
1. Откроется диалоговое окно **Сертификат о происхождении**. Выполните настройки, описанные в следующих подразделах, затем выберите **ОК**, чтобы создать документ.
1. Откроется предварительный просмотр документа. Используйте команды, предоставляемые на панели операций, для печати или экспорта документа, если необходимо.

### <a name="certifying-party"></a>Субъект сертификации

Раскрывающийся список **Субъект сертификации** используется для определения типа субъекта, который печатает документ. Укажите, кем является субъект сертификации: *Экспортер*, *Экспортер и производитель*, *Производитель* или *Импортер*; или оставьте поле пустым, если субъект сертификации не является никем из них. Выбранный параметр определяет, что будет напечатано в разделах адреса документа.

Выбранный **Субъект сертификации** будет включен в печатный документ.

Документ может быть распечатан как для входящих, так и для исходящих отгрузок. Выбирайте *Импортер* в качестве **субъекта сертификации** только для входящих отгрузок.

### <a name="has-various-producers"></a>Разные производители

Раскрывающийся список **Субъект сертификации** определяет текст, который будет использоваться для сведений о производителе в документе. Выберите один из следующих вариантов:

- *Прочие производители* — печать текста "Прочие" в данных производителя.
- *Доступно по запросу* — печатает текст "Доступен по запросу импортирующих органов" в сведениях о производителе.

Если в поле **Субъект сертификации** задано значение *Экспортер и производитель* или *Производитель*, то параметр **Имеет разных производителей** переопределяется, и адресная информация производителя совпадает с адресной информацией сертифицирующей стороны.

### <a name="blanket-period"></a>Срок действия

Используйте **Срок действия с** и **Срок действия по**, чтобы установить срок действия, в течение которого документ будет охватывать несколько отгрузок идентичных товаров, даже если документ печатается только для одной отгрузки. Даты срока действия можно настроить без каких-либо ограничений, и они будут добавлены в документ. Кроме того, эти параметры можно оставить пустыми или настроить их в прошлом.

### <a name="is-single-shipment"></a>В одной отгрузке

В диалоговом окне **Сертификат происхождения** задайте в поле **Является одной отгрузкой** одно из следующих значений:

- *Да* — добавляется "Одна отгрузка: Да" рядом с номером накладной.
- *Нет* — ничего не добавляется.

## <a name="other-information-included-in-the-document"></a>Другие сведения, включенные в документ

В дополнение к необязательным элементам, которые выбираются с помощью диалогового окна **Сертификат происхождения**, документ сертификации происхождения USMCA будет включать информацию и настраиваемые поля, сведенные в следующие подразделы. Часть этой информации должна быть введена вручную после создания документа.

### <a name="invoice-number"></a>Номер накладной

Коды накладных на продажу, связанных с отгрузками, печатаются на документе независимо от срока действия. Номера накладных печатаются независимо от того, что выбрано в поле **Является одиночной отгрузкой**.

### <a name="item-details"></a>Сведения об элементе

Документ содержит несколько разделов, в которых перечислены отдельные сведения о номенклатуре, такие как:

- **Номер SKU**: печатается код номенклатуры выпущенного продукта.

- **Описание**: печать описания или имени выпущенного продукта. Если на языке пользователя имеется описание, оно будет напечатано. Если такое описание отсутствует, печатается название на языке пользователя. Если это имя не существует, то будет напечатано название номенклатуры.

- **Тарифная классификация по HS (гармонизированная система)**: печатает план тарифов гармонизированной системы, связанный с продуктом. Можно настроить эти планы, перейдя в раздел **Управление транспортировкой \> Настройка \> Стандарт транспортировки \> Планы гармонизированных тарифов**.

- **Критерий происхождения:** данные в этом разделе необходимо ввести вручную при первом выпуске документа.

- **Страна происхождения:** печать страны происхождения, которая была применена путем перехода к разделу **Управление сведениями о продукте \> Настройка \> Соответствие продуктов \> Страна происхождения** (см. также раздел [Страна происхождения](../pim/country-of-origin.md)). Код ISO для страны происхождения распечатывается на основе страны/региона назначения в адресе поставки отгрузки и номенклатуры. Если не было настроено никаких данных о стране происхождения, это значение возвращается к параметру, указанному в поле **Выпущенный продукт \> Внешняя торговля \> Происхождение**. Если по-прежнему не найдена информация о стране происхождения, то после создания документа необходимо ввести страну происхождения вручную.

### <a name="certifier-signature-and-date"></a>Подпись и дата сертифицирующей стороны

Это необходимо ввести вручную после создания документа.

### <a name="consists-of-number-of-pages"></a>Состоит из количества страниц

Пользователь, подписавший сертификат, должен вручную ввести число страниц (для проверки) после создания документа.

### <a name="page-numbers"></a>Номера страниц

Текущая страница и число страниц печатаются в нижней части документа.
