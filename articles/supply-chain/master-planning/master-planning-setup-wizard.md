# <a name="master-planning-setup-wizard"></a>Мастер настройки сводного планирования

## <a name="specific-requirements-of-your-company"></a>Конкретные требования вашей компании

Первая страница мастера спрашивает о конкретных требованиях вашей компании. Ваши ответы на эти вопросы не обязательно должны быть точными, но вы должны быть в состоянии приблизительно указать количество номенклатур и спланированных заказов для юридического лица. Ваши ответы используются для настройки параметров, которые применяются к вашему юридическому лицу, а не только к сводному плану, который вы выбрали. В следующих разделах описываются рассчитанные параметры и используемые формулы.

### <a name="number-of-threads"></a>Количество потоков

- **Если вы производите любую из номенклатур:**: Количество потоков = Количество спланированных заказов ÷ 1 000
- **Если вы не производите любую из номенклатур:**: Количество потоков = Количество номенклатур ÷ 1 000

Если количество вычисляемых потоков превышает 75 процентов от имеющегося количества потоков, оно ограничено 75 процентами от количества потоков, доступных для каждого клиента. (Количество доступных потоков будет определено для каждого клиента.)

Дополнительные сведения см. в разделе [Количество потоков](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/master-planning/master-planning-performance#number-of-threads).

### <a name="bundle-size"></a>Размер набора

Размер набора будет установлен на **1**. Это значение часто является наилучшим значением, поскольку оно помогает повысить производительность сводного планирования.

Дополнительные сведения см. в разделе [Число задач в наборе задач помощников](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/master-planning/master-planning-performance#number-of-tasks-in-helper-task-bundle).

### <a name="firming-bundle-size"></a>Размер набора утверждения

- **Если вы производите любую из номенклатур:** Размер набора утверждения будет установлен на большее из этих двух значений: **10** и расчет набора
- **Если вы не производите любую из номенклатур:** Размер набора утверждения будет установлен на большее из этих двух значений: **50** и расчет набора

Расчет набора = (Количество спланированных заказов × (Временная граница утверждения ÷ Временная граница покрытия) ÷ Количество потоков) ÷ 10

### <a name="cache-size"></a>Размер кэша

Размер кэша будет установлен на **Максимум**. Это значение часто является наилучшим значением, поскольку оно помогает повысить производительность сводного планирования.

Дополнительные сведения см. в разделе [Распределение времени между заданиями в наборе заданий](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/allocate-time-jobs-job-bundle).

### <a name="manufacturing-setup"></a>Настройка производства

Если вы производите номенклатуры, страница **Настройка производства** появится позже в мастере.

## <a name="scope-of-the-current-plan"></a>Область действия текущего плана

На насколько **Область действия текущего плана** мастера вы отвечаете на вопросы, связанные с тем, как далеко в будущем будут учитываться и рассчитываться различные требования в сводном планировании. Каждый вопрос спрашивает, хотите ли вы использовать функцию и как вы хотите ее конфигурировать.

Например, для функции прогнозного плана мастер спрашивает: «Вы хотите использовать прогнозный план при сводном планировании, чтобы спланированные заказы предлагались для удовлетворения прогнозируемого спроса?»

Имеются следующие варианты:

- **Нет** — сводное планирование не будет предлагать спланированные заказы для выполнения прогноза. На вкладке **Временные границы** страницы **Сводные планы** (**Сводное планирование \> Настройка \> Планы \> Сводные планы**), мастер установит параметр **Прогнозный план (временная граница)** на **Да** и установит количество дней на **0** (ноль). Эта настройка переопределяет временную границу, указанный в группе покрытия. Поскольку количество дней установлено на **0** (ноль), функция не будет использоваться.
- **Да, как определено в этом сводном плане** — становится доступно поле, где можно ввести количество дней, в течение которых сводное планирование будет предлагать спланированные заказы для удовлетворения прогнозируемого спроса. Мастер установит параметр **Прогнозный план (временная граница)** на **Да** и установит количество дней, равное количеству дней, которое вводится в поле **Прогнозный план** на вкладке **Временные границы** страницы **Сводные планы**. Эта настройка переопределит значения, которые устанавливаются в группах покрытия.
- **Да, как определено в покрытии** — мастер установит параметр **Прогнозный план (временная граница)** на **Нет**. Временные границы, указанные в группе покрытия, будут использоваться для определения того, как долго вы будете планировать прогноз.

Остальные вопросы на этой странице и их ответы следуют той же схеме:

- **Нет**— Параметр **Прогнозный план (временная граница)** будет установлен на **Да**, и количество дней будет установлено на **0** (ноль).
- **Да, как определено в сводном плане** — параметр **Прогнозный план (временная граница)** будет установлен на **Да**. Количество дней, которые вы вводите, будет использовано и переопределит значения, которые установлены в группах покрытия.
- **Да, как определено в группе покрытия** — параметр **Прогнозный план (временная граница)** будет установлен на **Нет**.

Дополнительные сведения см. в разделе [Планирование заданий](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/job-scheduling).

## <a name="scheduling-options"></a>Параметры планирования

Страница **Параметры планирования** появляется только в том случае, если вы ответили **Да** на вопрос «Вы производите какие-либо из спланированных номенклатур?» вопрос на первой странице мастера.

Ваш ответ на первый вопрос на этой странице («Нужно ли планировать операции, разделенные на отдельные задания?») определяет метод планирования на вкладке **Общие** страницы **Сводные планы**.

- **Да** — Планирование заданий будет использоваться.
- **Нет** — Планирование операций будет использоваться.

Дополнительные сведения см. в разделе [Планирование операций](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/operations-scheduling) и [Планирование заданий](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/job-scheduling).

## <a name="updates-of-demand-and-supply"></a>Обновления спроса и предложения

Вопросы на странице **Обновления спроса и предложения** связаны с утверждением, сообщениями по действию и задержками.

Настройка сводного планирования будет обновляться на основе ваших ответов, в соответствии с той же схемой, которая описана в предыдущем разделе:

- **Нет** — Параметр **Временная граница** на странице **Сводные планы** будет установлен на **Да**, и количество дней будет установлено на **0** (ноль).
- **Да, как определено в сводном плане** — параметр **Временная граница** будет установлен на **Да**. Количество дней, которые вы вводите, будет использовано и переопределит значения, которые установлены в группах покрытия.
- **Да, как определено в группе покрытия** — параметр **Временная граница** будет установлен на **Нет**.

Для рассчитанных задержек ответы на вопросы в мастере будут обновлять соответствующие параметры на вкладке **Рассчитанные задержки** страницы **Сводные планы**.

## <a name="summary-of-your-changes"></a>Сводка изменений

На последней странице мастера показаны изменения, которые рекомендуются на основе ваших ответов. Он показывает как значение в вашей настройке (**Текущая настройка**), так и значение, которое рекомендует мастер (**Новая конфигурация**).

Можно также изменить параметры в новой конфигурации. Параметры разделены на параметры, которые применяются к вашему юридическому лицу и параметры, которые применяются только к сводному плану, который вы выбрали. Чтобы просмотреть все параметры настройки, которые могут быть изменены с помощью мастера, выберите **Показать все параметры** в нижней части страницы. Затем можно изменить параметры.

Наконец, при выборе **Готово** применяется новая конфигурация. Если вы выберете **Отмена**, ни одно из изменений не применяется.

## <a name="examples"></a>Примеры

В этом разделе описывается настройка двух вымышленных компаний, чтобы показать, как настройка может измениться в соответствии с потребностями каждого бизнеса.

### <a name="example-1-contoso-manufacturer"></a>Пример 1: Contoso Manufacturer

Contoso Manufacturer является компанией, которая производит динамики. Она закупает различные сырьевые материалы и компоненты, которые используются для готовых динамиков от различных поставщиков. Вот некоторые из характеристик ее поставки и производства:

- Готовые номенклатуры, которые компания производит, имеют структуру спецификации.
- Все готовые номенклатуры и компоненты планируются посредством сводного планирования. Планирование вручную не выполнено.
- Для производства каждой готовой номенклатуры определен маршрут операций.
- Завод производит готовые номенклатуры. Он оснащен определенным количеством фрезерных и сверлильных станков, которые используются для обработки компонентов. Различные компоненты должны быть обработаны этими станками.
- Есть много поставщиков. Среднее время упреждения для номенклатур составляет одну неделю. Группа номенклатур от того же поставщика будет иметь время упреждения в семь недель.

В мастере для Contoso Manufacturer вводятся следующие значения:

- **Покрытие:**

    - **Вопрос:** «Вы хотите указать количество дней горизонта планирования?»
    - **Ответ:** «Да, как это определено в группах покрытия».

    Поскольку время упреждения для номенклатур очень отличается, Contoso не должны планировать все номенклатуры на тот же период в будущем. Создаются группы покрытия для номенклатур. Номенклатуры, которые имеют аналогичное время упреждения, назначаются той же группе покрытия. Горизонт планирования для каждой группы покрытия (т.е. временная граница покрытия) составляет приблизительно время упреждения плюс маржа в одну неделю. Сводное-планирование затем гарантирует, что номенклатуры планируются заранее, в зависимости от их времени упреждения.

    Таким образом, для этого примера будут созданы две группы покрытия. Одна группа покрытия будет иметь временная граница покрытия в две недели, а другая будет иметь временную граница покрытия в восемь недель.

    Если в качестве ответа выбрано **Да, как указано в этом сводном плане**, количество введенных дней должно превышать самое длинное время упреждения всех номенклатур. Однако, поскольку многие номенклатуры не должны быть спланированы настолько заранее, многие спланированные заказы будут рассчитаны, но никогда не использованы. Поэтому время выполнения сводного планирования увеличится.

- **Планирование:**

    - **Вопрос:** «Нужно ли планировать операции, разделенные на отдельные задания?»
    - **Ответ:** «Да».

    Contoso Manufacturing должны планировать и планировать отдельные задания, которые будут выполняться в производственном цеху. Таким образом, он будет использовать планирование заданий.

- **Вместимость:**

    - **Вопрос:** Вы хотите спланировать с использованием вместимости ресурсов?
    - **Ответ:** «Да, как это определено в этом сводном плане». Введено **10 дней**.

    Количество фрезерных и сверлильных станков ограничено. Производственное планирование должно учитывать это ограничение и своевременно упорядочить задания в зависимости от вместимости ресурсов. Другими словами, спланированные задания будут перепланироваться на основе ограниченности ресурсов. Планирование будет использовать время упреждения в дополнение к определенному периоду. (Спланированные производственные заказы могут перекрываться.) Поскольку время производственного цикла номенклатур составляет семь дней, вместимость ресурсов для сводного планирования будет учитываться в течение 10 дней.

- **Последовательность:**

    - **Вопрос:** «Вы хотите определить последовательность спланированных заказов, используя значения последовательности продукта?»
    - **Ответ:** «Да, как это определено в группах покрытия».

    Для производства каждой готовой номенклатуры определен маршрут. Поэтому сводное планирование будет планировать заказы в срок в соответствии с определенными маршрутами.

- **Развертывание:**

    - **Вопрос:** «Вы хотите планировать заказы на все номенклатуры в спецификации (план для родительских и дочерних номенклатур)?»
    - **Ответ:** «Да, как это определено в группах покрытия».

    Все номенклатуры, которые используются для производства, должны быть спланированы. Поскольку номенклатуры имеют очень разные времена упреждения, сводное планирование будет иметь лучшую производительность, когда оно использует группы покрытия. Опять же, маржа в одну неделю может быть введена, и развертывание может быть сделано за то же время, как покрытие.

### <a name="example-2-contoso-retailer"></a>Пример 2: Contoso Retailer

Contoso Retailer является компанией-дистрибьютором в индустрии моды. Оно использует сводное планирование для расчета, когда заказы на покупку должны быть размещены, на основе ее прогнозируемых продаж. Вот некоторые из ее характеристик:

- Contoso Retailer использует прогнозирование спроса для прогнозирования продаж. Заказы на покупку будут планироваться в соответствии с прогнозом.
- Магазины используют заявки для пополнения.
- Время упреждения от основного склада до каждого магазина составляет приблизительно две недели для всех номенклатур.

В мастере для Contoso Retailer вводятся следующие значения:

- **Прогноз спроса:**

    - **Вопрос:** «Вы хотите использовать прогнозный план в сводном планировании, чтобы были предложены спланированные заказы для удовлетворения прогнозируемого спроса?»
    - **Ответ:** «Да, как это определено в этом сводном плане».

    Contoso включила прогноз спроса, чтобы спрогнозировать свои продажи. Поэтому сводное планирование должно рекомендовать спланированные заказы для выполнения прогноза.

- **Утверждение:**

    - **Вопрос:** «Хотите ли вы, чтобы сводное планирование автоматически утвердило спланированные заказы в документах заказа, например, заказы на производство или покупку?»
    - **Ответ:** «Да, как это определено в этом сводном плане». Введено **1 день**.

    Поскольку Contoso Retailer будет создавать заказы на покупку непосредственно из спланированных заказов на покупку, будет полезно, если спланированные заказы на покупку автоматически утверждаются. Поскольку компания использует сводное планирование каждый день, временная граница утверждения за один день будет автоматически утверждать все заказы, которые необходимы на следующий день.

- **Утвержденные заявки:**

    - **Вопрос:** «Вы хотите включить спрос от утвержденных заявок для пополнения розничных магазинов?»
    - **Ответ:** «Да, как это определено в этом сводном плане». Введено **1 день**.

    Contoso использует утвержденные заявки из своих магазинов для создания спланированных заказов на покупку для пополнения этих магазинов. Поскольку сводное планирование проводится каждый день, заявки последнего дня будут включены в планирование.
