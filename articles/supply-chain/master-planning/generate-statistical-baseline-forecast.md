# <a name="generate-a-statistical-baseline-forecast"></a><span data-ttu-id="6b17f-103">Сгенерировать статистический базовый прогноз</span><span class="sxs-lookup"><span data-stu-id="6b17f-103">Generate a statistical baseline forecast</span></span>

[!INCLUDE [banner](../includes/banner.md)]

<span data-ttu-id="6b17f-104">Эта статья содержит информацию о параметрах и фильтрах, которые используются для расчета прогноза спроса.</span><span class="sxs-lookup"><span data-stu-id="6b17f-104">This article provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="6b17f-105">При создании базового прогноза сначала необходимо определить параметры и фильтры, которые используются в расчете.</span><span class="sxs-lookup"><span data-stu-id="6b17f-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="6b17f-106">Например, можно создать базовый прогноз для оценки спроса на основе транзакционных данных за прошлый год для определенной компании, для наступающего месяца и для выбранной группы номенклатур.</span><span class="sxs-lookup"><span data-stu-id="6b17f-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="6b17f-107">Чтобы создать прогноз спроса, перейдите в раздел **Сводное планирование &gt; Прогнозирование &gt; Прогнозирование спроса &gt; Сгенерировать статистический базовый прогноз**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="6b17f-108">Период прогноза можно выбрать во время создания прогноза.</span><span class="sxs-lookup"><span data-stu-id="6b17f-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="6b17f-109">Доступные значения: день, неделя, месяц.</span><span class="sxs-lookup"><span data-stu-id="6b17f-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="6b17f-110">Число периодов, для которых создается прогноз, задается в поле **Горизонт прогнозирования**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="6b17f-111">Если для стратегии прогнозирования задано значение **Копировать поверх исторического спроса**, окончание исторического горизонта игнорируется.</span><span class="sxs-lookup"><span data-stu-id="6b17f-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="6b17f-112">Система копирует число периодов, заданных в поле **Горизонт прогноза**, чтобы спрогнозировать спрос, начиная с даты, заданной в поле **Начальная дата** в разделе **Исторический горизонт**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="6b17f-113">Копируй спрос за прошлые периоды, начиная с определенной даты, планировщики производства могут составить план на следующий квартал двумя способами.</span><span class="sxs-lookup"><span data-stu-id="6b17f-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="6b17f-114">Путем копирования спроса за тот же квартал прошлого года.</span><span class="sxs-lookup"><span data-stu-id="6b17f-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="6b17f-115">Путем копирования спроса за предыдущий квартал.</span><span class="sxs-lookup"><span data-stu-id="6b17f-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="6b17f-116">Чтобы не допустить путаницы в производственных планах, определенное количество периодов прогнозирования можно заблокировать.</span><span class="sxs-lookup"><span data-stu-id="6b17f-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="6b17f-117">Это количество задается в поле **Временная граница блокировки**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="6b17f-118">На странице **Отрегулированный прогноз спроса** ячейки заблокированных периодов неактивны, чтобы визуально указать на то, что изменить эти значения невозможно.</span><span class="sxs-lookup"><span data-stu-id="6b17f-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="6b17f-119">Начальная дата базового прогноза спроса не обязательно должна являться текущей датой или датой в будущем.</span><span class="sxs-lookup"><span data-stu-id="6b17f-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="6b17f-120">Чтобы задать другую начальную дату, воспользуйтесь полем **Начальная дата базового прогноза — начальная дата**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="6b17f-121">Например, в июне пользователи могут создать прогноз на следующий год.</span><span class="sxs-lookup"><span data-stu-id="6b17f-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="6b17f-122">Поскольку периоды прогноза между окончанием спроса за прошлые периоды и началом базового периода отсутствуют, прогноз может быть неточным.</span><span class="sxs-lookup"><span data-stu-id="6b17f-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="6b17f-123">Если используется сервис прогнозирования спроса Microsoft Dynamics 365 for Finance and Operations, существует четыре способа заполнения пробелов.</span><span class="sxs-lookup"><span data-stu-id="6b17f-123">If you are using the Microsoft Dynamics 365 for Finance and Operations Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="6b17f-124">Можно выбрать желаемый метод, задав значение для параметра MISSING\_VALUE\_SUBSTITUTION на странице **Параметры прогнозирования спроса**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

<span data-ttu-id="6b17f-125">В поле **Начальная дата базового прогноза** - **Начальная дата** необходимо задать начало периода прогнозирования. Так, в США воскресенье станет начальной датой периода прогнозирования длительностью неделя.</span><span class="sxs-lookup"><span data-stu-id="6b17f-125">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="6b17f-126">Система автоматически корректирует значение в поле **Начальная дата базового прогноза** - **Начальная дата**, чтобы оно соответствовало началу периода прогнозирования.</span><span class="sxs-lookup"><span data-stu-id="6b17f-126">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="6b17f-127">В поле **Начальная дата базового прогноза** - **Начальная дата** можно задать дату в прошлом.</span><span class="sxs-lookup"><span data-stu-id="6b17f-127">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="6b17f-128">Другими словами, можно создать прогноз спроса в прошлом.</span><span class="sxs-lookup"><span data-stu-id="6b17f-128">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="6b17f-129">Это полезно, поскольку позволяет менять параметры сервиса прогнозирования так, чтобы статистический прогноз, созданный в прошлом, соответствовал фактическому спросу за прошлые периоды.</span><span class="sxs-lookup"><span data-stu-id="6b17f-129">This is useful, because it lets users tweak the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="6b17f-130">Затем пользователи могут использовать эти значения параметров, чтобы создать статистический базовый прогноз на будущее.</span><span class="sxs-lookup"><span data-stu-id="6b17f-130">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="6b17f-131">Сделанные вручную корректировки в предыдущих итерациях прогнозирования спроса можно автоматически применять к новому базовому прогнозу, если флажок **Перенести ручные корректировки в прогноз спроса** установлен.</span><span class="sxs-lookup"><span data-stu-id="6b17f-131">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas**t check box is selected.</span></span> <span data-ttu-id="6b17f-132">Если флажок не установлен, ручные корректировки не добавляются в базовый прогноз, но и не удаляются.</span><span class="sxs-lookup"><span data-stu-id="6b17f-132">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="6b17f-133">Ручные корректировки прогноза можно удалить только на этапе импорта прогноза, сняв флажок **Сохранить ручные корректировки базового прогноза спроса**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-133">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="6b17f-134">Ручные корректировки сохраняются на этапе авторизации.</span><span class="sxs-lookup"><span data-stu-id="6b17f-134">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="6b17f-135">Следовательно, если пользователь вносит ручные корректировки в прогноз, но не авторизует прогноз обратно в Dynamics 365 for Finance and Operations, изменения будут утеряны.</span><span class="sxs-lookup"><span data-stu-id="6b17f-135">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Finance and Operations, the changes are lost.</span></span> <span data-ttu-id="6b17f-136">Дополнительные сведения о ручных корректировках и принципах их работы см. в разделе [Авторизация скорректированного прогноза](authorize-adjusted-forecast.md).</span><span class="sxs-lookup"><span <span data-ttu-id="6b17f-137">Создание прогноза спроса может иметь имя и комментарии, чтобы помочь пользователям идентифицировать созданный прогноз.</span><span class="sxs-lookup"><span data-stu-id="6b17f-137">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="6b17f-138">Эти значения отображаются в истории создания прогнозов на странице **История создания статистических базовых прогнозов**.</span><span class="sxs-lookup"><span data-stu-id="6b17f-138">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="6b17f-139">Внутрихолдинговая группа планирования, ключи распределения номенклатуры и другие фильтры можно применить во время создания прогноза.</span><span class="sxs-lookup"><span data-stu-id="6b17f-139">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="6b17f-140">Их можно использовать для повышения производительности или разделения данных на управляемые блоки.</span><span class="sxs-lookup"><span data-stu-id="6b17f-140">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="6b17f-141">Однако заметьте, что прогноз спроса не создается для участников какого-либо ключа распределения номенклатуры, который не связан с внутрихолдинговой группой планирования, даже если ключ выбран в запросе.</span><span class="sxs-lookup"><span data-stu-id="6b17f-141">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span> 

<span data-ttu-id="6b17f-142">**Подсказка**. Иногда пользователи получают ошибки при создании прогноза спроса или создание прогноза завершается без журнала сеанса.</span><span class="sxs-lookup"><span data-stu-id="6b17f-142">**Tip**: Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="6b17f-143">Это может случиться из-за остатка данных в запросе, который ранее был использован для создания прогноза.</span><span class="sxs-lookup"><span data-stu-id="6b17f-143">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="6b17f-144">Чтобы решить эту проблему, щелкните **Выбрать**, чтобы открыть страницу **Запрос**, щелкните **Сброс** и создайте базовый прогноз повторно.</span><span class="sxs-lookup"><span data-stu-id="6b17f-144">To fix this issue, click **Select** to open the **Query** page, click **Reset**, and then regenerate the baseline forecast.</span></span>
