# <a name="inventory-forecasts"></a>Прогнозы запасов

[!include [banner](../includes/banner.md)]

В этой теме описывается, как просматривать и создавать прогнозы запасов. Можно создавать и просматривать строки прогноза поставок и спроса для номенклатур, номенклатурных групп, ключей распределения номенклатур, счетов клиентов, групп клиентов, счетов поставщиков и групп поставщиков.

Для каждой строки прогноза можно выбрать используемую модель прогноза. Затем можно указать номенклатуру или номенклатурную группу, а также количество или сумму проводки. Также можно задать временные интервалы для распределения количества по прогнозу.

Строки прогноза поставок и спроса создают прогноз запасов для той же комбинации номенклатуры и модели. Этот прогноз запасов отражает баланс между поставками и спросом, которые были введены для той же номенклатуры. Это значение нельзя редактировать. Прогноз запасов позволяет рабочей группе управления запасами просматривать ожидаемые изменения запасов в наличии для номенклатуры в ходе предстоящего периода, для которого был выполнен прогноз.

После ввода спроса и/или поставок можно выполнить планирование прогноза для расчета брутто потребностей для материалов и мощности и создания спланированных заказов.

- [Импорт исторических данных для прогнозов спроса](import-historical-data.md).
- [Создание прогноза с помощью веб-службы машинного обучения Microsoft Azure](demand-forecasting-setup.md).
- [Импорт строк прогноза спроса или поставок с помощью структуры управления данными (сущности данных ForecastDemandForecastEntryStaging и ForecastSupplyForecastEntryStaging)](../../dev-itpro/data-entities/data-entities-data-packages.md).

Как показано в таблице на шаге 1, имеются различные способы доступа к используемым страницам.

1. В зависимости от типа сущности, для которой необходимо создать прогноз, и типа прогноза, который необходимо создать, откройте страницу прогноза поставок, спроса или запасов, как описано в следующей таблице.

    |---|---|

    | Используемые варианты продукта | <ol><li>Выберите **Управление сведениями о продукте \> Продукты \> Используемые варианты продукта**.</li><li>Выберите вариант, для которого необходимо создать прогноз.</li><li>В области действий на вкладке **План** в группе **Прогноз** выберите **Прогноз спроса**, **Прогноз поставок** или **Прогноз запасов**, в зависимости от типа прогноза, с которым предполагается работать.</li></ol> |
    | Номенклатурные группы | <ol><li>Перейдите в раздел **Управление запасами \> Настройка \> Запасы \> Группы номенклатур**.</li><li>Выберите номенклатурную группу, для которой необходимо создать прогноз.</li><li>В области действий выберите **Прогнозирование \> Спрос**, **Прогнозирование \> Поставки** или **Прогнозирование \> Прогноз запасов** в зависимости от типа прогноза, с которым предполагается работать.</li></ol> |

    | Клиенты | <ol><li>Перейдите в раздел **Сводное планирование \> Прогнозирование \> Ручной ввод прогноза \> Клиенты**.</li><li>Выберите клиента, для которого необходимо создать прогноз.</li><li>На панели операций выберите **Определить прогноз спроса**.</li></ol> |
    | Группы клиентов | <ol><li>Перейдите в раздел **Сводное планирование \> Прогнозирование \> Ручной ввод прогноза \> Группы клиентов**.</li><li>Выберите группу клиентов, для которой необходимо создать прогноз.</li><li>На панели операций выберите **Определить прогноз спроса**.</li></ol> |

    | Группы поставщиков | <ol><li>Перейдите в раздел **Сводное планирование \> Прогнозирование \> Ручной ввод прогноза \> Группы поставщиков**.</li><li>Выберите группу поставщиков, для которой необходимо создать прогноз.</li><li>В области действий выберите **Запись**, чтобы открыть страницу **Прогноз поставок**.</li></ol> | 
    | Все строки | <ul><li>Перейдите к пункту **Сводное планирование \> Прогнозирование \> Строки прогноза спроса** или **Сводное планирование \> Прогнозирование \> Строки прогноза поставки**, в зависимости от типа прогноза, с которым предполагается работать.</li></ul> |

    В зависимости от сделанного выбора появляется страница **Прогноз поставок** или **Прогноз спроса**. В нем отображаются все существующие строки прогноза для записи, которая была выбрана перед открытием страницы.

1. В области действий выберите **Создать**, чтобы добавить строку прогноза в сетку в верхней части страницы.
1. В новой строке в поле **Модель** выберите прогнозную модель, которая будет использоваться. Затем введите другие необходимые данные, такие как номенклатура, номенклатурная группа, счет клиента, счет поставщика или группа, количество номенклатуры или общая сумма проводки. Полные сведения о полях, доступных на страницах **Прогноз поставок** и **Прогноз спроса**, см. в последующих разделах этой темы.
1. Чтобы распределить прогноз на период, на вкладке **Обзор** выберите **Распределить прогноз** на панели инструментов.
1. В сетке **Распределение** проверьте временной диапазон и временные интервалы для распределения количеств по прогнозу.

## <a name="supply-forecast-lines"></a>Строки прогноза поставки

Прогноз поставок позволяет создать план для номенклатур, которые должны быть закуплены. Он сообщает служащим, отвечающим за закупки и источники, что им требуется заказать.

Можно ввести прогноз поставок по номенклатуре, номенклатурной группе, ключу распределения номенклатуры, поставщику и группе поставщиков. Сведения обо всех способах открытия страницы **Прогноз поставок** для различных сущностей и записей см. в разделе [Просмотр и ввод вручную строк прогнозов](#manual-entry) ранее в этой теме.

Верхняя часть страницы **Прогноз поставок** содержит сетку строк прогноза поставок и набор вкладок, которые можно использовать для просмотра и настройки дополнительных сведений о выбранной строке прогноза. В нижней части страницы имеется сетка **Распределение**.

В следующих подразделах описываются все поля, доступные на каждой вкладке, и все команды, доступные в области действий на странице и на панели инструментов на вкладке **Обзор**.

### <a name="the-item-tab-on-the-supply-forecast-page"></a>Вкладка номенклатуры на странице прогноза поставок

Кроме того, в нее включено поле **Цена за единицу**, в котором отображается цена покупки для количества единиц, указанных в поле **Единица**. Цена за единицу извлекается автоматически из настройки номенклатуры, но может редактироваться.

В полях под сеткой представлены дополнительные сведения о номенклатуре. Некоторые из этих сведений повторяются на вкладке **Обзор**. В следующей таблице описаны поля, которые являются уникальными для вкладки **Номенклатура**.

| Накладные расходы по покупкам | Фиксированные накладные расходы по цене покупки. Эти расходы не зависят от количества. |

| Сумма скидки | Сумма скидки как сумма для единицы измерения продажи. |
| Валовая сумма | Сумма, когда не применяются скидки. |
| Количество | Количество по проводке в единицах измерения складского учета номенклатуры. |

## <a name="allocate-forecast"></a><a name="allocate-forecast"></a>Распределить прогноз

Следующая процедура используется для обработки выбранных строк проводок по прогнозу. При распределении прогноза количество распределяется таким образом, как указано в строках сетки **распределения**.

1. В зависимости от типа объекта, для которого вы создаете прогноз, и типа прогноза, который вы хотите создать, откройте страницу прогноза спроса или предложения, как описано в [Просмотр и ввод строк прогноза вручную](#manual-entry).
1. На странице прогноз поставок или спроса выберите строку прогноза, а затем на вкладке **Обзор** выберите **распределить прогнозы** на панели инструментов.
1. В диалоговом окне **Распределить прогноз** задайте поля, которые описаны в следующей таблице. (Значение, выбранное в поле **Метод** определяет, что доступны другие доступные поля).

    | Поле | описание |
    |---|---|
    | Метод | <p>Выберите метод, который будет использоваться для распределения проводки прогноза:</p><ul><li>**Нет** — распределения не происходит.</li><li>**Период** — прогнозирование одинакового количества для каждого периода. Если выбрано это значение, укажите количество в поле **На** и единицу времени в поле **Единица измерения**.</li><li>**Ключ** — прогноз распределяется согласно ключу распределения периода, который указывается в поле **Ключ периода**. Можно использовать этот метод, если необходимо учитывать сезонные изменения.</li><ul>|
    | На | <p>Ввод количества временных интервалов в будущем, на которые прогноз распространяется. Это поле доступно только в том случае, если в поле *Метод* выбрано значение **Период**.</p><p>Например, можно выбрать *Период* в поле **Метод**, ввести *1* в поле **На** и выбрать *Месяцы* в поле **Единица**. Затем, в поле **Окончание** укажите дату завершения, которая на один год позже текущей. В этом случае создается одна строка прогноза для каждого месяца наступающего года на основе номенклатуры и количества, указанных в строке заголовка. |
    | Единица | Выберите единицу временного интервала: *Дни*, *Месяцы* или *Годы*. Распределение затем соответствует количеству дней, месяцев или лет, указанному в поле **На**.|
    | Ключ периода | Указывает ключ распределения периода, который используется для распределения прогноза. Дополнительные сведения см. в разделе [Распределение данных бюджетного планирования](../../finance/budgeting/budget-planning-data-allocation.md). |
    | Окончание | Указывается дата окончания, которая применяется к параметрам в полях **на** и **единица измерения**. |

1. Выберите **ОК**, чтобы подтвердить параметры.
1. Результаты на вкладке **распределение** можно просмотреть для одной и той же строки.

## <a name="bulk-update-forecast-transactions"></a><a name="bulk-update"></a>Массовое обновление проводок прогноза

Эта процедура используется для обработки существующих строк проводок по прогнозу. Строки проводок по прогнозу можно копировать, изменять и удалять.

1. На странице строк прогноза поставок или спроса выберите **Массовое обновление**.
1. В диалоговом окне выберите **Фильтр**.
1. На вкладке **Диапазон** выберите критерии, определяющие исходные данные прогноза, которые требуется скопировать, изменить или удалить. В эти данные могут входить прогнозная модель, код номенклатуры и счет клиента.
1. Выберите **ОК**, чтобы подтвердить выбор.

    После выбора данных можно использовать диалоговое окно **Редактировать проводки прогноза**, чтобы определить, как необходимо копировать, изменять или удалять.

    > [!NOTE]
    > Шаг фильтрации является необходимым условием для использования функции **Массовое изменение**. Если вы пропустите этот флажок, программа выберет и обновит **все** строки прогноза. Таким образом, можно непреднамеренно вызвать дублирование или удаление проводок.

1. В разделе **Управление** выберите, следует ли скопировать, обновить или удалить выбранные проводки прогноза.
1. Используйте раздел **Модификации в поле**, чтобы изменить параметры прогноза. Установите флажок параметра и выберите один из доступных вариантов. Например, установите флажок **Модель**, затем выберите код модели прогноза. Существующие строки прогноза копируются в выбранную прогнозную модель.
1. Используйте раздел **Изменить период**, чтобы переместить даты начала и завершения прогноза вперед или назад. Установите флажок, затем используйте поля количества и периода, чтобы определить, как нужно переместить даты прогноза. Можно ввести отрицательное количество, чтобы переместить дату начала вперед (то есть, сделать ее более ранней).

    Например, чтобы задержать начальную дату прогнозной проводки на шесть месяцев, установите флажок, введите *6* в качестве количества и выберите *месяцы* в качестве периода.

1. Используйте раздел **Корректировать поле**, чтобы обновить фактические данные прогноза. В поле **Поле** выберите критерий, который требуется изменить. В поле **Коэффициент** введите коэффициент умножения, чтобы применить его к выбранному критерию, либо введите константу для добавления или вычитания. Например, выберите *Количество* в качестве критерия и введите коэффициент *1,5*, чтобы умножить количество номенклатуры на 1,5. Можно также ввести константу *-25*, чтобы уменьшить количество номенклатуры на 25 единиц.
1. Используйте раздел **Финансовые аналитики** для обновления финансовых аналитик строк прогноза. Выберите финансовые аналитики, которые необходимо изменить, затем введите значение для применения к выбранным аналитикам.
1. Выберите **ОК**, чтобы применить изменения.

## <a name="use-forecasts-with-master-planning"></a>Использование прогнозов в сводном планировании

После ввода прогноза спроса и/или прогноза поставок можно включить прогнозы в процессе сводного планирования, чтобы учесть ожидаемый спрос и/или поставки в выполнении сводного планирования. Когда прогнозы включаются в сводное планирование, рассчитываются брутто-потребности для материалов и емкости, и создаются спланированные заказы.

### <a name="set-up-a-master-plan-to-include-an-inventory-forecast"></a>Настройка сводного плана для включения прогноза запасов

Чтобы настроить сводный план так, чтобы он включал прогноз запасов, выполните следующие действия.

1. Выберите **Сводное планирование \> Настройка \> Планы \> Сводные планы**.
1. Выберите существующий план или создайте новый план.
1. На экспресс-вкладке **Общие** настройте следующие поля:

    - **Прогнозная модель** — выберите прогнозную модель для применения. Эта модель будет учитываться при создании предложений поставки для текущего сводного плана.
    - **Включить прогноз поставки** — установите для этого параметра значение *Да*, чтобы включить прогноз поставки в текущий сводный план. Если задать для него значение *Нет*, проводки прогноза поставки не включаются в сводный план.
    - **Включить прогноз спроса** — установите для этого параметра значение *Да*, чтобы включить прогноз спроса в текущий сводный план. Если задать для него значение *Нет*, проводки прогноза спроса не включаются в сводный план.
    - **Метод, используемый для сокращения потребностей по прогнозу** — выберите метод, который должен использоваться для снижения требований прогноза. Дополнительные сведения см. в разделе [Ключи сокращения прогнозов](planning-optimization/demand-forecast.md#reduction-keys).

1. На экспресс-вкладке **Временные границы в днях** можно настроить следующие поля, чтобы указать период, в течение которого включается прогноз:

    - **Прогнозный план** — установите этот параметр в значение *Да*, чтобы переопределить временную границу прогнозного плана, созданную на основе отдельных групп покрытия. Установите для этого параметра значение *Нет*, чтобы использовать значения из отдельных групп покрытия для текущего сводного плана.
    - **Период времени прогноза** — если для параметра **Прогнозный план** установлено значение *Да*, укажите количество дней (начиная с сегодняшней даты), к которым должен применяться прогноз спроса.

    > [!IMPORTANT]
    > Параметр **Прогнозный план** еще не поддерживается с оптимизацией планирования.

### <a name="run-a-master-plan-that-includes-an-inventory-forecast"></a>Выполнение сводного плана, включающего прогноз запасов

Чтобы выполнить сводный план так, включающий прогноз запасов, выполните следующие действия.

1. Перейдите **Сводное планирование \> Рабочие области \> Сводное планирование**.
1. В поле **Сводный план** введите или выберите сводный план, который был настроен в предыдущей процедуре.
1. На плитке **Сводное планирование** выберите **Выполнить**.
1. В диалоговом окне **Сводное планирование** задайте для параметра **Отслеживать время обработки** значение *Да*.
1. В поле **Количество потоков** введите число.
1. На экспресс-вкладке **Записи для добавления** выберите **Фильтр**.
1. Появляется диалоговое стандартного редактора запросов. На вкладке **Диапазон** выберите строку, в которой в поле **Поле** задано как *Код номенклатуры*.
1. В поле **Критерии** выберите код номенклатуры для включения в план.
1. Нажмите **ОК**.

Для просмотра рассчитанных требований откройте страницу **Брутто-потребность**. Например, на странице **Выпущенные продукты** на панели действий на вкладке **План** в группе **Требования** выберите **Брутто-потребность**.

Чтобы просмотреть созданные спланированные заказы, перейдите к пункту **Сводное планирование \> Общие \> Спланированные заказы** и выберите подходящий прогнозный план.
