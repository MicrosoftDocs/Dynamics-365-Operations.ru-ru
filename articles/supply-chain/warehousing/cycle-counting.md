# <a name="cycle-counting"></a>Цикличный подсчет

[!include [banner](../includes/banner.md)]

Эта статья описывает, как можно использовать подсчет циклов с решением складирования, которое доступно в модуле управления складом. Эта статья не относится к решению складирования, доступному в модуле управления запасами.

Подсчет циклов — это процесс склада, который можно использовать для аудита номенклатур запасов в наличии. Процесс подсчета циклов можно описать в три шага:

1.  **Создание работы подсчета циклов**. Работа подсчета циклов может создаваться автоматически на основе параметров порога для номенклатур или с помощью плана подсчета циклов. Вы можете также создать работу подсчета циклов вручную путем использования параметров номенклатуры или склада на странице **Работа подсчета циклов по номенклатуре** или **Работа подсчета циклов по месту хранения**.
2.  **Обработка подсчета циклов**. После создания работы подсчета циклов можно выполнить работу подсчета циклов, подсчитав номенклатуры в местонахождении склада и введя результаты в Dynamics 365 Supply Chain Management с помощью мобильного устройства. Либо можно подсчитать номенклатуры в местонахождении склада, не создавая работу подсчета циклов. Этот процесс называется *подсчет циклов наличия*.
3.  **Устранение разницы значений подсчета циклов**. После подсчета циклов все номенклатуры с отличным значением подсчета циклов будут иметь статус работы **Ожидает рассмотрения** на странице **Все работы**. Вы можете устранить эти разницы на странице **Работа подсчета цикла ожидает просмотра**.

На следующем рисунке показан процесс подсчета циклов. ![Последовательность операций для подсчета циклов](./media/performcyclecountinginawarehouselocation.jpg)

## <a name="automatically-create-cycle-counting-work"></a>Автоматическое создание работы подсчета циклов
Предусмотрено два метода для планирования повторяющегося создания работы подсчета циклов: настройка пороговых значений подсчета циклов или настройка планов подсчета циклов.

-   Порог подсчета циклов определяет предельное количество или процент складируемых номенклатур. Работа подсчета циклов создается автоматически при превышении лимита порога.
-   План подсчета циклов создает работу подсчета циклов либо немедленно, либо периодически с помощью пакетного задания. При создании работы подсчета циклов строка подсчета циклов содержит сведения о местонахождении для инвентаризации. Запасы в наличии, связанных с этим местонахождением, не блокируются и поэтому доступны для резервирования и исходящей обработки даже при наличии открытой работы инвентаризации.

### <a name="create-cycle-counting-work-based-on-threshold-parameters-for-items"></a>Создание работы подсчета циклов на основе параметров порога для номенклатур

Работу подсчета циклов можно создать, когда число номенклатур становится меньше определенного порогового значения для местонахождения. Например, имеется 60 номенклатур в местонахождении, для которого пороговое значением подсчета циклов равно 40. Во время проводки заказа на продажу 25 номенклатур было скомплектовано из этого местонахождения и помещено в местонахождение промежуточного хранения. Поскольку новое количество номенклатур (35) меньше порогового количества, автоматически создается работа подсчета циклов для местонахождения.

### <a name="schedule-cycle-counting-work"></a>Планирование работы подсчета циклов

Планы подсчета циклов можно запланировать для создания работы подсчета циклов незамедлительно или время от времени. Настраивая планы подсчета циклов, можно контролировать пул работ, для которого создается работа подсчета циклов, максимальное количество подсчетов цикла, созданных для номенклатур в различных местонахождениях, и количество дней до следующей инвентаризации местонахождения склада. Например, номенклатура доступна в трех местонахождениях на складе и максимальное количество подсчетов циклов установлено равным **2**. В этом случае при запуске плана подсчета циклов будут созданы два подсчета циклов для двух местонахождений, в которых имеется номенклатура. Как другой пример, вы задали число дней между подсчетами цикла, равное **5**. В этом случае работа подсчета циклов создается каждые пять дней. Однако если работа подсчета циклов обрабатывается на третий день, следующая работа подсчета циклов будет создана спустя пять дней после последней обработки подсчета циклов, то есть на восьмой день.

## <a name="create-cycle-counting-work-manually"></a>Создание работы подсчет циклов вручную
Чтобы вручную создать работу подсчета циклов, можно использовать страницу **Работа подсчета циклов по номенклатуре** или **Работа подсчета циклов по месту хранения**. Можно указать максимальное количество создаваемых подсчетов циклов. Например, если менеджер склада указывает значение **5**, работа подсчета циклов создается для пяти местонахождений, даже если номенклатура присутствует в 10 различных местонахождениях. Можно также выбрать код пула работ, которому назначаются созданные коды работ подсчета циклов. Когда код пула работ обрабатывается для подсчета циклов, коды работ подсчета циклов, назначенные этому пулу работ, обрабатываются в группе.

## <a name="perform-a-cycle-count-by-using-a-mobile-device"></a>Выполнение подсчета циклов с помощью мобильного устройства
Существует несколько способов обработки работы подсчета циклов с помощью Supply Chain Management на мобильном устройстве:

-   **Под управлением пользователя** — работник может указать код работы подсчета циклов со статусом **Открыто**.
-   **Под управлением системы** — Supply Chain Management назначает код работы подсчета циклов работнику.
-   **Группировка подсчетов циклов** — работник может сгруппировать коды работ подсчета циклов, относящиеся к определенному местонахождению, зоне или пулу работ.
-   **Подсчет циклов наличия** — работник может подсчитать номенклатуры в местонахождении склада в любое время, не создавая работу подсчета циклов. Для выполнения подсчета циклов наличия в местонахождении работник вводит код местонахождения.

Ниже приводится пример, показывающий, как можно выполнить точечный подсчета циклов с помощью мобильного устройства. Инструкции, которые работник видит на устройстве, зависят от настройки пункта меню для подсчета циклов наличия.

1.  На мобильном устройстве выберите пункт меню для обработки работы подсчета циклов.
2.  Зарегистрируйте местонахождение, для которого требуется выполнить подсчет циклов наличия.
3.  Зарегистрируйте и подтвердите код номенклатуры и фактическое количество номенклатуры. **Примечание.** Статус работы подсчета циклов изменится на **Ожидает рассмотрения** или **Закрыто** на странице **Все работы** в зависимости от параметров, заданных на странице **Работник**.
4.  (Необязательно) Повторите шаг 3 для оставшихся номенклатур в местонахождении и убедитесь, что не осталось дополнительных номенклатур для подсчета.

## <a name="resolve-cycle-counting-differences"></a>Устранение разницы подсчета циклов
Разница подсчета циклов возникает в следующих сценариях, если для параметра **Является супервизор по подсчету циклов** задано значение **Нет** для идентификатора пользователя работы.

-   Подсчитанное значение выходит за пределы отклонения, указанные в полях **Максимальный лимит процента** или **Максимальный лимит количества** на странице **Пользователь работы**. Например, количество запасов в наличии в местоположении равно 50, а предел отклонения для пользователя работы равен 10. Если пользователь работы вводит значение, которое не находится в диапазоне от 40 до 60, возникает разница.
-   Подсчитанное значение отличается от количества запасов в наличии, и пределы отклонения не заданы.

Можно скорректировать значения разницы в подсчитанном значении, а затем принять подсчитанное значение на странице **Ожидающий просмотра подсчет циклов**. Можно проверить измененный подсчет количества номенклатуры на странице **В наличии по местонахождению**. Подсчитанное значение отклоняется, если разницу невозможно утвердить.
