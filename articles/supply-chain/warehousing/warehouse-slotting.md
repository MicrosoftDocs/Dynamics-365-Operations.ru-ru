# <a name="warehouse-slotting"></a>Слоттинг на складе

[!include [banner](../includes/banner.md)]

Слоттинг на складе позволяет консолидировать спрос по номенклатуре и единицам измерения из заказов, имеющих статус *Заказанный* , *Зарезервированный* или *Выпущенный*. Созданный спрос затем может быть применен к ячейкам, которые будут использоваться для комплектации, на основе количества, единицы измерения, физических аналитик, фиксированных местоположений и т. д. После установки плана слоттинга можно создать работу по пополнению запасов для перемещения соответствующего объема запасов в каждое местоположение.

Эта функция помогает менеджерам складов интеллектуально планировать местонахождения комплектации до того, как они выпускают заказы на склад и создают работу по комплектации.

## <a name="turn-on-the-warehouse-slotting-feature"></a>Включение функции слоттинга на складе

Прежде чем использовать эту функцию, она должна быть включена в системе. Администраторы могут использовать параметры [управления компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения, если это требуется. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление складом*
- **Имя функции:** *Функция слоттинга на складе*

## <a name="set-up-warehouse-slotting"></a>Настройка слоттинга на складе

Чтобы использовать слоттинг на складе, необходимо настроить следующие элементы в вашей системе.

### <a name="create-unit-of-measure-tiers-for-slotting"></a><a name="unit-tiers"></a>Создание уровней единиц измерения для слоттинга

Уровни единиц измерения позволяют группировать вместе несколько единиц измерения для целей слоттинга. Например, если несколько размеров коробок все выбраны из одной области комплектации, можно создать один уровень для всех размеров. **Необходимо создать строку для каждой единицы измерения, которая должна быть частью уровня.**

1. Перейдите к пункту **Управление складом \> Настройка \> Пополнение \> Уровни единиц измерения слоттинга**.
1. Выберите **Создать**.
1. В заголовке установите следующие значения:

    - **Уровень единиц измерения:** *EaBoxPl*
    - **Описание:** *Штуки коробки паллеты*

1. Нажмите **Сохранить**.
1. На экспресс-вкладке **Единицы измерения** выберите **Создать** , чтобы добавить строку в сетку.
1. В новой строке установите следующие значения:

    - **Единица измерения:** *Ящик*
    - **Описание:** оставьте это поле пустым. Оно будет заполнено автоматически при сохранении изменений.
    - **Класс единиц измерения:** *Количество*

1. Выберите **Создать** , чтобы добавить вторую строку в сетку.
1. В новой строке установите следующие значения:

    - **Ед. изм:** *шт.*
    - **Описание:** оставьте это поле пустым. Оно будет заполнено автоматически при сохранении изменений.
    - **Класс единиц измерения:** *Количество*

1. Выберите **Создать** , чтобы добавить третью строку в сетку.
1. В новой строке установите следующие значения:

    - **Единица измерения:** *PL*
    - **Описание:** оставьте это поле пустым. Оно будет заполнено автоматически при сохранении изменений.
    - **Класс единиц измерения:** *Количество*

1. Выберите **Сохранить** , чтобы сохранить уровень.

### <a name="create-a-directive-code-for-slotting"></a>Создание кода директивы для слоттинга

Вы должны выбрать код директивы, который должен быть связан с шаблоном.

1. Перейдите в раздел **Управление складом \> Настройка \> Коды директив**.
1. В области действий выберите **Создать**.
1. В поле **Код директивы** введите *Слоттинг*.
1. В поле **Описание директивы** введите *Слоттинг*.

### <a name="set-up-slotting-templates"></a>Настройка шаблонов слоттинга

Каждый шаблон слоттинга определяет способ назначения запасов местонахождениям для конкретного склада. Каждый шаблон должен содержать строку для каждой из спецификаций слоттинга. Используйте процедуры данного раздела для настройки шаблонов слоттинга.

1. Перейдите в раздел **Управление складом \> Настройка \> Пополнение \> Шаблоны слоттинга**.
1. Выберите **Создать** для создания шаблона.

Затем необходимо настроить заголовок шаблона, спецификации слоттинга и директивы местоположения, как описано в следующих подразделах.

#### <a name="set-up-a-slotting-template-header"></a>Настройка заголовка шаблона слоттинга

1. В заголовке шаблона задайте следующие значения:

    - **Шаблон слоттинга:** _61_
    - **Описание:** _61_
    - **Тип спроса:** *Заказ на продажу*

        В настоящее время *Заказ на продажу* — это единственный поддерживаемый тип спроса.

    - **Стратегия спроса:** _Заказано_

        В поле доступны следующие значения:

        - **Заказано** — полное заказанное количество в заказе на продажу должно рассматриваться как спрос.
        - **Зарезервировано** — только количества по строке заказа на продажу, которые зарезервированы (физические и заказанные), должны рассматриваться как спрос.

    - **Склад:** _61_
    - **Разрешить волне спроса использовать незарезервированные количества:** _Да_

Можно также указать запрос для сужения области спроса, который оценивается.

#### <a name="set-up-slotting-specifications-for-each-template"></a>Настройка спецификаций слоттинга для каждого шаблона

Для каждого создаваемого шаблона выполните следующие шаги, чтобы добавить строку для каждой из спецификаций слоттинга.

1. На экспресс-вкладке **Сведения о шаблоне слоттинга** выберите **Создать** , чтобы создать строку шаблона.
1. В новой строке установите следующие значения:

    - **Последовательность:** _1_
    - **Описание:** _Фиксированное местоположение_
    - **Минимальное количество:** _1_

        Это поле определяет минимальное количество спроса, необходимое для строки.

    - **Максимальное количество:** _1000000_

        Это поле определяет максимальное количество спроса, допустимое для строки.

    - **Единица измерения:** оставьте это поле пустым.

        Это поле определяет единицу измерения для минимального и максимального количества.

    - **Уровень единиц измерения:** _EaBoxPl_

        Это поле определяет единицы измерения спроса, допустимые для строки. (Дополнительные сведения см. в разделе [Настройка уровней единиц измерения для слоттинга](#unit-tiers) ранее в этой теме.)

    - **Назначение критериев слота:** _Учитывать количество_

        В поле доступны следующие значения:

        - **Считать пустым** — система должна предполагать, что все ячейки в области комплектации пусты, и не должна проверять эти ячейки на запасы.
        - **Учитывать количество** — система должна проверять ячейки в области комплектации на наличие запасов, и должна пропускать все ячейки, которые не являются пустыми.

    - **Код директивы:** _Слоттинг_

        Это поле определяет директиву местоположения, используемую для нахождения ячейки комплектации работы пополнения.

    - **Местоположение переполнения:** оставьте это поле пустым.

        Это поле определяет местоположение, в которое помещаются запасы, если для количества, обрабатываемого в строке, невозможно найти местоположение.

    - **Разрешить освобождение:** _Да_

        Если для этого параметра установлено значение *Да* , то если какой-либо спрос не может быть включен в слот, будет создана работа по перемещению, чтобы взять запасы из местоположений, где имеются запасы, но где не было использован слоттинг. Затем шаблон запускается снова. На этот раз он игнорирует запасы в ячейках. Эта функция лучше всего работает в том случае, когда в поле **Критерии назначения слота** задано значение _Учитывать количество_.

    - **Использование фиксированного местонахождения:** _Только фиксированные местонахождения для продукта_

        В поле доступны следующие значения:

        - **Фиксированные и нефиксированные местонахождения** — система не должна ограничиваться использованием только фиксированных местоположений.
        - **Только фиксированные местонахождения для продукта** — система должна использовать слоттинг только в местонахождениях, которые являются фиксированными местонахождениями для продукта.
        - **Только фиксированные местонахождения для варианта продукта** — система должна использовать слоттинг только в местонахождениях, которые являются фиксированными местонахождениями для варианта продукта.

1. Нажмите **Сохранить**.
1. Выберите **Создать** , чтобы создать вторую строку шаблона.
1. В новой строке установите следующие значения:

    - **Последовательность:** _2_
    - **Описание:** _Другое_
    - **Минимальное кол-во:** _1_
    - **Максимальное кол-во:** _1000000_
    - **Единица измерения:** оставьте это поле пустым.
    - **Уровень единиц измерения:** _EaBoxPl_
    - **Назначение критериев слота:** _Учитывать количество_
    - **Код директивы:** _Слоттинг_
    - **Местоположение переполнения:** оставьте это поле пустым.
    - **Разрешить освобождение:** _Да_
    - **Использовать фиксированные местонахождения:** _Фиксированные и нефиксированные местонахождения_

    В запросе для второй строки будет указан критерий, который используется для определения того, в каких местоположениях может использоваться слоттинг для спроса по этой строке.

1. Выберите строку, в которой в поле **Последовательность** указано значение *2*.
1. Выберите **Изменить запрос**.
1. На вкладке **Диапазон** выберите **Добавить** , чтобы добавить строку в сетку.
1. В новой строке установите следующие значения:

    - **Таблица:** *Местонахождения*
    - **Производная таблица:** *Местонахождения*
    - **Поле:** *Код профиля местонахождения*
    - **Критерии:** *Комплектация-06* (Выберите знак двойного плюса \[**++**\] в поле, чтобы развернуть список, затем выберите *Комплектация-06* - *Сайт комплектации 6*.)

1. Нажмите **ОК**.

#### <a name="set-up-location-directives"></a>Настройка директив для места хранения

По крайней мере одна директива местоположения должна быть настроена для поддержки комплектации со слоттингом. С помощью процедур данного раздела можно настроить новую *директиву местоположения пополнения* для комплектаций со слоттингом.

1. Перейдите в раздел **Управление складом \> Настройка \> Директивы местонахождения**.
1. В левой области в поле **Тип заказа на работу** выберите *Пополнение*.
1. В области действий выберите **Создать**.
1. В заголовке для новой директивы местоположения в поле **Имя** введите *61 Комплектация слоттинга*.

##### <a name="configure-the-location-directives-fasttab"></a>Экспресс-вкладка настройки директив местоположения

1. На экспресс-вкладке **Директивы местоположения** установите следующие значения. Примите для всех остальных полей значения по умолчанию.

    - **Тип работы:** _Комплектация_
    - **Сайт:** _6_
    - **Склад:** _61_
    - **Код директивы:** _Слоттинг_

1. Выберите **Сохранить** , чтобы сделать доступной экспресс-вкладку **Строки**.

##### <a name="configure-the-lines-fasttab"></a>Экспресс-вкладка настройки строк

1. На экспресс-вкладке **Строки** выберите **Создать** , чтобы создать новую строку.
1. В новой строке установите следующие значения. Примите для всех остальных полей значения по умолчанию.

    - **Количество "От":** _0_
    - **Количество "До":** _1000000_

1. Выберите **Сохранить** , чтобы сделать доступной экспресс-вкладку **Действия директивы местоположения**.

##### <a name="configure-the-location-directive-actions-fasttab"></a>Экспресс-вкладка настройки действий директив местоположения

1. На экспресс-вкладке **Действия директивы местоположения** выберите **Создать** , чтобы создать строку.
1. В новой строке установите следующие значения. Примите для всех остальных полей значения по умолчанию.

    - **Имя:** _Сборное_
    - **Стратегия:** _Нет_

1. Выберите **Сохранить** , чтобы сделать доступной кнопку **Правка запроса**.

##### <a name="edit-the-query"></a>Изменение запроса

1. На экспресс-вкладке **Действия директивы местоположения** выберите **Изменить запрос**.
1. На вкладке **Диапазон** выберите **Добавить** , чтобы добавить строку в сетку.
1. В новой строке установите следующие значения:

    - **Таблица:** *Местонахождения*
    - **Производная таблица:** *Местонахождения*
    - **Поле:** *Код зоны*
    - **Критерии:** *Сборная* (Выберите знак двойного плюса \[**++**\] в поле, чтобы развернуть список, затем выберите *Сборная*.)

1. Нажмите **ОК**.

## <a name="scenario"></a>Сценарий

### <a name="set-up-the-scenario"></a>Настройка сценария

Для этого сценария используйте встроенный пример данных и создайте записи, которые описаны в этом разделе.

#### <a name="use-the-usmf-sample-data"></a>Использование примера данных USMF

Для работы с этим сценарием с помощью образцов записей и значений, указанных здесь, необходимо использовать систему, в которой установлены стандартные [демонстрационные данные](../../fin-ops-core/dev-itpro/deployment/deploy-demo-environment.md). Дополнительно перед началом необходимо выбрать юридическое лицо **USMF**.

#### <a name="create-demand"></a>Создание спроса

Выполните следующие действия, чтобы создать спрос, к которому будет применяться слоттинг.

1. Перейдите в раздел **Продажи и маркетинг \> Заказы на продажу \> Все заказы на продажу**.
1. Выберите **Создать** , чтобы создать заказ на продажу.
1. В диалоговом окне **Создать заказ на продажу** в поле **Счет клиента** выберите _US-007_.
1. В поле **Склад** выберите _61_.
1. Нажмите **ОК**.
1. Открывается новый заказ на продажу. Он включает пустую строку на экспресс-вкладке **Строки заказа на продажу**. В этой строке установите следующие значения:

    - **Номенклатура:** _L0101_
    - **Количество:** _20_

1. Выберите **Добавить строку** , чтобы добавить новую строку, и установите следующие значения:

    - **Номенклатура:** _T0100_
    - **Количество:** _8_

1. Нажмите **Сохранить**.
1. Выберите **Создать** , чтобы создать второй заказ на продажу.
1. В диалоговом окне **Создать заказ на продажу** в поле **Счет клиента** выберите _US-008_.
1. В поле **Склад** выберите _61_.
1. Открывается новый заказ на продажу. Он включает пустую строку на экспресс-вкладке **Строки заказа на продажу**. В этой строке установите следующие значения:

    - **Номенклатура:** _T0100_
    - **Количество:** _1_

1. Нажмите **Сохранить**.

### <a name="walk-through-a-typical-slotting-scenario"></a>Пошаговое выполнение типичного сценария слоттинга

После того, как все необходимые элементы готовы, как описано в предыдущем разделе, можно попробовать испытать функцию, выполнив каждое из упражнений данного раздела.

#### <a name="generate-demand"></a>Сгенерировать спрос

1. Перейдите в раздел **Управление складом \> Настройка \> Пополнение \> Шаблоны слоттинга** и выберите шаблон слоттинга, созданный ранее.
1. На панели операций выберите **Создать спрос**. Эта команда оценивает весь спрос, который находится в системе, и который соответствует запросу шаблона слоттинга. Общий спрос по всем заказам затем консолидируется в одну строку для количества/единицы измерения. По завершении процесса выводится информационное сообщение.

#### <a name="slotting-demand"></a>Спрос слоттинга

*Спрос сплоттинга* отображает результаты создания спроса на основе настройки шаблона слоттинга.

- На панели операций выберите **Спрос слоттинга** для просмотра результатов из команды **Создать спрос**. Строки в спросе слоттинга могут быть изменены. Можно удалить строку, добавить новую строку или отредактировать сведения строки.

> [!NOTE]
> Можно редактировать спрос вручную или импортировать его из внешней системы с помощью управления данными. Все, что есть в спросе слоттинга, будет использоваться на следующем шаге, независимо от того, откуда он был получен.

#### <a name="locate-demand"></a>Найти спрос

После того как спрос был создан, следует воспользоваться командой **Найти спрос** для создания *плана слоттинга*.

- На панели операций выберите **Найти спрос**. Будет запущен процесс слоттинга. По завершении процесса выводится информационное сообщение.

#### <a name="slotting-plan"></a>План слоттинга

План слоттинга показывает местоположение, которому была назначена каждая номенклатура/количество, используется ли переполнение, была ли создана работа освобождения и строку шаблона, которая была использована. **Любой спрос, для которого невозможно использовать слоттинг, выделяется красным цветом.**

- На панели операций выберите **План слоттинга** , чтобы просмотреть результаты.

#### <a name="create-replenishment"></a>Создание пополнения

После создания плана слоттинга необходимо создать *работу пополнения* на основе плана.

- В области действий выберите **Выполнить пополнение**. По завершении процесса выводится информационное сообщение. В этом сообщении указывается количество заголовков, созданных для кода сборки работы.

Директивы местоположения, которые будут использоваться, определяются на основе кода директив, указанного в каждой строке шаблона.

## <a name="tips"></a>Советы

### <a name="set-up-automatic-slotting"></a>Настройка автоматического слоттинга

После того как все необходимые элементы установлены, можно настроить автоматическое выполнение слоттинга, выполнив следующие действия.

1. Перейдите в раздел **Управление складом \> Пополнение \> Выполнение слоттинга**.
1. Укажите шаги слоттинга для выполнения. Выберите один или несколько следующих шагов слоттинга:

    - Сгенерировать спрос
    - Найти спрос
    - Создать работу пополнения

    > [!NOTE]
    > Шаги слоттинга прогрессивные. Если требуется выбрать *Поиск спроса* , необходимо сначала выбрать *Создать спрос*.

1. Укажите используемый шаблон слоттинга.
1. При необходимости настройте повтор для автоматического запуска.

Для упражнений в сценарии **не** настраивайте автоматический слоттинг.
