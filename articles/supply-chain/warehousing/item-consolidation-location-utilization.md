# <a name="item-consolidation---location-utilization"></a>Консолидация номенклатур — использование местонахождения

[!include [banner](../includes/banner.md)]

В этой теме приводятся сведения о функциональности, облегчающей менеджерам складов просмотр и фильтрацию объемного использования ячеек на складе. Менеджеры могут выбирать местоположения и создавать работу по перемещению запасов непосредственно со страницы **Консолидация номенклатур** для консолидации номенклатур и, таким образом, лучше использовать пространство на складе.

## <a name="turn-on-the-features"></a>Включение функций

Прежде чем можно будет использовать функции, описанные в этой теме, их необходимо включить в системе. Администраторы могут использовать рабочую область [Управление компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса этих функций и их включения, если это требуется. Включите обе следующих функции в том порядке, в котором они перечислены. (Обе функции относятся к модулю **Управление складом**.)

1. Статус местонахождения на складе
2. Использование местонахождения консолидации номенклатур

## <a name="warehouse-location-status"></a>Статус местонахождения на складе

Функция *Статус места хранения на складе* добавляет четыре новых поля на страницу **Местонахождения** для отслеживания дополнительной информации о текущем состоянии местонахождения:

- **Код номенклатуры** — номенклатура, которая в настоящее время находится в местоположении. Если местоположение содержит несколько номенклатур, это поле будет пустым.
- **Дата и время последнего действия** — отметка времени последней складской проводки, которая была выполнена для этого местоположения.
- **Дата распределения по срокам** — дата, когда запасы в местонахождении поступили на склад. Эта дата рассчитывается на основе даты распределения по срокам грузоместа. Хотя эта дата точна для местоположений, которые отслеживаются по грузоместу, она может быть неточной для местоположений, не отслеживаемых по грузоместу.
- **Статус местонахождения** — статус местонахождения. Доступны четыре значения:

    - **Не определено** — профиль местоположения не отслеживает статус. Поэтому текущий статус неизвестен.
    - **Пусто** — в данный момент нет запасов в местонахождении.
    - **Комплектация** — исходящие проводки для местоположения были выполнены после того как оно было пустым.
    - **Хранение** — только входящие проводки были выполнены с момента, когда оно было пустым.

Эти поля позволяют менеджерам склада лучше просматривать статус местоположений на складе. Они также позволяют выполнять более расширенную отчетность и фильтрацию.

## <a name="set-up-item-consolidation-and-location-utilization"></a>Настройка консолидации номенклатур и использования местонахождения

В этом разделе описывается, как подготовить систему для использования консолидации номенклатур и использования местоположений. В процедурах используются образцы значений из стандартных демонстрационных данных. Если планируется работа с примером сценария, представленного далее в этом разделе, выберите юридическое лицо **USMF** (которое содержит стандартные демонстрационные данные) и создайте каждую запись, описанную в этом разделе. Если вы не планируете работать с примером сценария, предоставляемые здесь значения могут рассматриваться как примеры типов настройки, которую необходимо выполнить для использования этих функций.

### <a name="released-product"></a>Используемый продукт

1. Выберите **Управление сведениями о продукте \> Продукты \> Выпущенные продукты**.
1. В поле **Код номенклатуры** выберите *M9201* и откройте страницу сведений.
1. На панели действий на вкладке **Управление складскими запасами** в группе **Склад** выберите **Физические размеры**.
1. На странице **Физические размеры** в области действий выберите **Создать**.

    В сетку добавляется новая строка. Поле **Код номенклатуры** является предварительно установленным.

1. В поле **Единица измерения** выберите *шт.* Оставшиеся поля в этой строке задаются автоматически.
1. Выберите **Сохранить** и закройте страницу.

### <a name="location-profile"></a>Профиль местонахождения

1. Перейдите в раздел **Управление складом \> Настройка \> Склад \> Профили местонахождения**.
1. В списке профилей местоположения выберите **ЭТАЖ-05**.
1. На панели операций выберите **Правка**.
1. На экспресс-вкладке **Общие** убедитесь, что для обоих следующих параметров установлено значение *Да*:

    - Включить номенклатуру в месте хранения
    - Включить статус места хранения

1. Нажмите **Сохранить**.

    > [!IMPORTANT]
    > Если для обоих параметров **Включить номенклатуру в месте хранения** и **Включить статус места хранения** уже было задано значение *Да*, переходите дальше к инструкциям по настройке экспресс-вкладки **Аналитики** на шаге 10. Если для этих параметров еще не было задано значение *Да*, необходимо выполнить проверку согласованности для модуля **Управление складом** после того, как они будут установлены вручную. В этом случае перейдите к следующему шагу.

1. Чтобы выполнить проверку согласованности, перейдите к пункту **Администрирование системы \> Периодические задачи \> База данных \> Проверка целостности**.
1. В диалоговом окне **Проверка целостности** задайте следующие значения:

    - **Модуль:** *Управление складом*
    - **Режим проверки:** *Проверка*
    - **Начальная дата:** оставьте это поле пустым.
    - **Проверка согласованности статуса мест хранения на складе:** установите этот флажок.

1. Нажмите **ОК**.

    > [!TIP]
    > Когда проверка целостности завершена, вы получите уведомление. Откройте [центр уведомлений](../../fin-ops-core/fin-ops/get-started/user-interface-elements.md#notifications) для просмотра сообщения. Выберите **Сведения о сообщении** для просмотра сведений.
    >
    > Если в сообщении для проверки согласованности указано состояние "Найдена неверная информация о статусе места хранения для места хранения XXXX на складе XX", необходимо снова выполнить проверку согласованности. На этот раз задайте в поле **Режим проверки** значение *Исправление ошибок*. Просмотрите сообщения, чтобы убедиться, что не было найдено никаких ошибок.

1. Теперь необходимо завершить настройку профиля местоположения. Вернитесь к пункту **Управление складом \> Настройка \> Склад \> Профили местонахождения**, выберите профиль местонахождения **ЭТАЖ-05**, затем на панели операций выберите **Правка**.
1. На экспресс-вкладке **Аналитика** установите следующие значения:

    - **Процент использования объема:** *100*
    - **Волюметрический метод, используемый для поиска местонахождения:** *Использовать объем местонахождения*
    - **Фактическая высота места хранения:** *10*
    - **Фактическая ширина места хранения:** *10*
    - **Фактическая глубина местонахождения:** *10*
    - **Максимальный вес:** *100*

1. Нажмите **Сохранить**.

### <a name="mobile-device-menu-items"></a>Пункты меню мобильного устройства

1. Перейдите в раздел **Управление складом \> Настройка \> Мобильное устройство \> Пункты меню мобильного устройства**.
1. На панели операций выберите **Создать** для создания пункта меню для сортировки.
1. В заголовке установите следующие значения:

    - **Наименование пункта меню:** *Входящая корректировка*
    - **Заголовок:** *Входящая корректировка*
    - **Режим:** *Работа*
    - **Использовать существующую работу:** *Нет*

1. На экспресс-вкладке **Общие** установите следующие значения.

    - **Процесс создания работы:** *Входящая корректировка*
    - **Типы корректировки запасов:** *Входящая корректировка*

1. Нажмите **Сохранить**.

### <a name="mobile-device-menu"></a>Меню мобильного устройства

1. Перейдите в раздел **Управление складом \> Настройка \> Мобильное устройство \> Меню мобильного устройства**.
1. В списке меню выберите **Запасы**.
1. На панели операций выберите **Правка**.
1. В списке **Доступные меню и пункты меню** найдите и выберите пункт меню **Входящая корректировка**.
1. Выберите кнопку со стрелкой вправо, чтобы переместить пункт **Входящая корректировка** в списке **Структура меню**. Таким образом, вы добавляете новый пункт меню в выбранное меню.
1. Нажмите **Сохранить**.

### <a name="movement-types"></a>Типы перемещения

1. Перейдите в раздел **Управление складом \> Настройка \> Запасы \> Типы перемещения**.
1. На панели действий выберите **Создать**, затем задайте следующие значения:

    - **Код типа перемещения:** *КОНСОЛИДИРОВАТЬ*
    - **Описание:** *Консолидация мест хранения*
    - **Код класса работы:** *InvMov*

1. Нажмите **Сохранить**.

## <a name="example-scenario"></a>Пример сценария

В следующем сценарии для выполнения *входящей корректировки* запасов в двух местах склада используется приложение склада на мобильном устройстве.

### <a name="add-inventory-to-locations"></a>Добавление запасов в места хранения

1. Выполните вход на мобильном устройстве в качестве пользователя, который настроен для склада *51*.
1. Перейти к пункту **Запасы \> Входящая корректировка**.

    Теперь будет введена первая корректировка местонахождения.

1. В задаче **Входящая корректировка** выберите местонахождение, для которого требуется выполнить корректировку запасов. В поле **МЕСТОНАХОЖДЕНИЕ** выберите *LP-001*.
1. Подтвердите местонахождение.
1. Создайте код грузоместа для номенклатуры, которая будет добавлен в местонахождение. В поле **Грузоместо** введите *LP00101*.
1. Подтвердите грузоместо.
1. Введите номенклатуру, которая будет добавлена на грузоместо. В поле **ITEM** введите *M9201*.
1. Подтвердите номенклатуру.
1. Введите количество номенклатуры, которое будет добавлено. В поле **КОЛИЧЕСТВО** введите *10*.
1. Подтвердите количество.

    Вы получите сообщение "Работа завершена". Теперь будет введена вторая корректировка местонахождения.

1. В задаче **Входящая корректировка** выберите местонахождение, для которого требуется выполнить корректировку запасов. В поле **МЕСТОНАХОЖДЕНИЕ** выберите *LP-002*.
1. Подтвердите местонахождение.
1. Создайте код грузоместа для номенклатуры, которая будет добавлен в местонахождение. В поле **Грузоместо** введите *LP00201*.
1. Подтвердите грузоместо.
1. Введите номенклатуру, которая будет добавлена на грузоместо. В поле **ITEM** введите *M9201*.
1. Подтвердите номенклатуру.
1. Введите количество номенклатуры, которое будет добавлено. В поле **КОЛИЧЕСТВО** введите *15*.
1. Подтвердите количество.

    Вы получите сообщение "Работа завершена".

1. Выберите кнопку меню (иногда называется "гамбургер" или "кнопка-гамбургер"), затем выберите команду **Отмена**, чтобы выйти из задачи **Входящая корректировка**.

### <a name="consolidate-locations"></a>Консолидация мест хранения

1. Перейдите к пункту **Управление складом \> Периодические задачи \> Консолидация номенклатур**.
1. В заголовке выберите склад, для которого выполняется консолидация. В поле **Склад** введите *51*.

    Запись отображается для каждой ячейки, в которой номенклатура *M9201* была скорректирована. В столбце **Процент использования** указано объемное использование для каждого местонахождения.

1. Для консолидации запасов выберите все места хранения, которые необходимо консолидировать, затем в области действий выберите **Консолидировать запасы**.
1. В диалоговом окне **Консолидация запасов** укажите местонахождение и тип перемещения, которые должны использоваться для создания работы по перемещению запасов. Задайте следующие значения:

    - **Местонахождение:** *LP-001*
    - **Код типа перемещения:** *КОНСОЛИДИРОВАТЬ*

1. Нажмите **ОК**.
1. Вы получаете информационное сообщение, которое показывает работу перемещения, которая была создана. Запишите код работы перемещения.

### <a name="view-movement-work"></a>Просмотр работы перемещения

1. Перейдите в раздел **Управление складом \> Работа \> Сведения о работе**.
1. Просмотрите созданную работу. Для фильтрации или поиска в сетке работ используйте код работы перемещения из консолидации запасов.

    В этом сценарии в качестве местонахождения консолидации запасов использовалось существующее местонахождение, содержащее запасы. Поэтому был создан только один код работы.

    > [!NOTE]
   > Система создает один код работы для каждой операции перемещения, которая должна быть завершена. При указании местоположения, которое уже содержит запасы, создается только один код работы. При указании нового местоположения создаются два кода работы.
