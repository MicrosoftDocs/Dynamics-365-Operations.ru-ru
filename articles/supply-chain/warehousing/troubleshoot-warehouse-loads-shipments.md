# <a name="troubleshoot-load-building-and-shipments"></a>Устранение неполадок формирования загрузок и отгрузок

[!include [banner](../includes/banner.md)]

В этой теме описывается устранение распространенных проблем, которые могут встретиться при работе с формированием загрузок и отгрузок в Microsoft Dynamics 365 Supply Chain Management.

## <a name="i-receive-the-following-error-message-you-cant-create-a-load-line-for-this-order-line-because-it-contains-inventory-dimensions-that-are-invalid"></a>Получено следующее сообщение об ошибке: "Невозможно создать строку загрузки для данной строки заказа, так как она содержит недопустимые складские аналитики..."

### <a name="issue-description"></a>Описание проблемы

При попытке выпуска возврата заказа на продажу на склад появляется следующее сообщение об ошибке:

> Невозможно создать строку загрузки для данной строки заказа, так как она содержит недопустимые складские аналитики. Нельзя ссылаться на складские аналитики, расположенные под аналитикой местоположения в иерархии резервирования. Удалите недопустимые аналитики из строки заказа.

### <a name="issue-resolution"></a>Устранение проблемы

К сожалению, продукт не поддерживает обработку загрузки для процесса возврата продажи. Поэтому невозможно выпустить возврат заказа на продажу на склад.

В транзакциях заказа на продажу нельзя ссылаться на складские аналитики, расположенные под аналитикой **Местоположение** в иерархии резервирования. Для разрешения неполадки удалите недопустимые аналитики из строки заказа.

## <a name="i-receive-the-following-error-message-one-of-the-lines-is-already-on-a-load-unable-to-release-to-warehouse"></a>Получено следующее сообщение об ошибке: "Одна из строк уже находится в загрузке. Невозможно выпустить на склад."

### <a name="issue-description"></a>Описание проблемы

При создании вручную загрузок или при настройке процесса таким образом, чтобы загрузка уже была создана до ввода строки заказа на продажу, предполагается, что последующий выпуск будет выполнен вручную, и будут использоваться маршрут и рейтинг из загрузки.

В другом возможном сценарии выполняется попытка автоматического выпуска на склад, но процессу волны не удалось создать работу. Поэтому открытая отгрузка или загрузка все еще создаются. Эта открытая отгрузка или загрузка затем блокирует последующие попытки автоматического выпуска заказа до тех пор, пока не будет удалена открытая отгрузка или загрузка или не будет вручную выполнена повторная обработка волны.

### <a name="issue-resolution"></a>Устранение проблемы

Можно выполнить выпуск со страницы заказа на продажу, или автоматический выпуск может быть выполнен со страницы выпуска заказа на продажу, только если загрузка не существовала перед выпуском на склад. Загрузка будет создана автоматически после обработки волны.

## <a name="i-receive-the-following-error-message-the-delivery-note-correction-cant-be-processed-the-delivery-note-only-contains-items-that-are-subject-to-warehouse-management-processes-as-these-are-not-supported-by-delivery-note-correction"></a>Получено следующее сообщение об ошибке: "Исправление уведомления о поставке не может быть обработано. Уведомление о поставке содержит только номенклатуры, которые подлежат процессам управления складом, так как они не поддерживаются исправлением уведомления о поставке."

### <a name="issue-description"></a>Описание проблемы

После разноски уведомления о поставке его невозможно отменить, так как кнопка **Отмена** недоступна. Вы также не можете скорректировать уведомление о поставке. Если вы попробуете, появится следующее сообщение об ошибке.

### <a name="issue-resolution"></a>Устранение проблемы

Для корректировки разнесенных отборочных накладных для номенклатур, включенных для расширенного управления складом (WMS), разноска должна выполняться из загрузки, а не напрямую из заказа.

## <a name="how-can-i-create-work-from-outbound-loads-instead-of-waves"></a>Как можно создавать работу на основе исходящих загрузок вместо волн?

### <a name="issue-description"></a>Описание проблемы

Вот один из способов воспроизвести эту проблему.

1. Создайте исходящую загрузку с помощью заказа на продажу или заказа перемещения.
2. Запустить загрузку в складе.
3. Обратите внимание, что еще не было создано ни одной работы комплектации.

### <a name="issue-resolution"></a>Устранение проблемы

Если работа должна быть создана сразу после выпуска загрузки, необходимо соответствующим образом настроить шаблон волны. В шаблоне волны задайте для следующих параметров значение *Да*:

- Создавать волны автоматически
- Обрабатывать волну перед запуском на склад
- Автоматический запуск волны

## <a name="i-cant-re-release-a-partially-shipped-load"></a>Невозможно повторно выпустить частичную загрузку.

### <a name="issue-description"></a>Описание проблемы

Невозможно выпустить частично отгруженную загрузку на склад. При выполнении выпуска на склад появляется сообщение "Операция завершена", но ничего не происходит, и работа для оставшегося количества не создается. Эта проблема возникает при использовании функциональности транспортной загрузки и неполного резервирования.
