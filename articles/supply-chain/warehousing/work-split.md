# <a name="work-split"></a>Разделение работ

Функция разделения работ позволяют разделять коды крупных работ (т. е., заказы на работу, содержащие несколько строк) на несколько кодов меньших работ, которые затем можно назначить нескольким складским сотрудникам. Таким образом, один и тот же номер создания работы может быть скомплектован сразу нескольким складским работникам.

> [!IMPORTANT]
> Можно разбить только те заказы на работу, которые имеют статус *Открыто* или *В обработке*.

## <a name="turn-on-the-work-split-functionality"></a>Включение функции разделения работ

Прежде чем можно будет использовать функцию разделения работ, необходимо включить эту функцию и ее обязательные функции в системе. Администраторы могут использовать параметры [управления компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функций и их включения, если это требуется.

Прежде всего включите требуемую функцию *Блокирование работы на уровне организации*, если она еще не включена. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление складом*
- **Имя функции:** *Блокировка работ для всей организации*

> [!NOTE]
> Когда эта функция активирована, обновление данных применяется автоматически после включения функции для всех юридических лиц.

Затем включите функцию *Разделение работ*, которая указана следующим образом:

- **Модуль:** *Управление складом*
- **Имя функции:** *Разделение работ*

## <a name="enhancements-to-the-work-details-and-all-work-pages"></a>Улучшения страниц сведений о работе и всех работ

Функция *Разделение работ* добавляет следующие две кнопки на вкладку **Работы** в области действий страниц **Сведения о работе** и **Вся работа**:

- **Разделить работу** — разделение кода текущей работы на несколько кодов меньших работ, которые могут обрабатываться разными работниками.
- **Отменить сеанс разделения работ** — отменяет сеанс разделения работ и делают работу доступной для обработки.

![Кнопки разделения работы и отмена сеанса разделения работы](media/Work_split_buttons.png "Кнопки разделения работы и отмена сеанса разделения работы")

> [!IMPORTANT]
> Кнопка **Разделить работу** будет недоступна, если выполняется любое из следующих условий:
>
> - Статус работы отличается от *Открыто* или *Выполняется*.
> - Код контейнера связан с кодом работы. (Контейнер не может быть систематически разделен, так как для него требуются физические операции.)
> - Работа связана с кластером.
> - Тип заказа на работу не относится к одному из следующих типов:
>
>    - Заказы на продажу
>    - Комплектация сырья
>    - Выдача на перемещение
>
> - Работа в данный момент разделяется другим пользователем. При попытке открыть страницу разделения для работы, которая уже разделяется другим пользователем, появляется следующее сообщение об ошибке: "Работа с кодом \#\#\#\# в данный момент разделена. Повторите попытку через несколько минут. Если вы продолжаете получать это сообщение, обратитесь к супервизору."

Новая причина блокировки работы, *Разделение работы*, указывает, когда код работы находится в процессе разделения. Это отображается на странице **Разделение работы** и в приложении склада, если пользователь пытается выполнить работу. При использовании причин блокировки имя поля **Заблокированная волна** из кода работы изменяется на **Заблокировано**.

## <a name="initiate-a-work-split"></a>Инициализация разделения работы

Функция добавляет новую страницу **Разделение работы**, позволяющую пользователям разбивать строки работы из кода работы. При первом открытии страницы отображаются строки, имеющие статус работы *Открыто* и доступные для разбиения. На панели операций выберите **Разделить работу**, чтобы обработать выбранную работу.

Чтобы разделить работу, выполните следующие действия.

1. Откройте одну из следующих страниц работы:

    - **Сведения о работе** (**Управление складом \> Работа \> Сведения о работе**)
    - **Вся работа** (**Управление складом \> Работа \> Вся работа**)

1. В сетке выберите код работы, который необходимо разделить. В поле **Тип заказа на работу** должно быть указано одно из следующих значений:

    - Заказы на продажу
    - Комплектация сырья
    - Выдача на перемещение

1. В области действий на вкладке **Работа** в группе **Работа** выберите **Разделить работу**.

    Отображается страница **Разделить работу**, в которой отображаются открытые и доступные для разделения строки работ. По умолчанию отображаются только доступные строки работ. Чтобы просмотреть все строки для кода работы (например, строки, имеющих тип работы *Разместить*), установите флажок **Показать все строки** над сеткой.

    Отображается следующее сообщение: "Пользователи не могут обрабатывать строки работы до завершения разделения и закрытия этой страницы."

    В поле **Причина блокировки работ** для текущей работы будет установлено значение *Разделить работу*, и работа будет заблокирована.

    ![Причина блокировки](media/Blocking_reason.png "Причина блокировки")

1. Выберите строки, которые необходимо удалить из текущего кода работы, и добавить к новому коду работы. Имеют место следующие события:

    - При разделении работы выбранная строка или строки исходного кода работы отменяются, а затем копируются в новый код работы.
    - Будет сохранена существующая структура шаблонов работ и местоположение размещения (и также будущие пары комплектации/размещения). Значения для следующих полей кода работ копируются из исходной работы в новую работу:

        - Код загрузки
        - Код отгрузки
        - Тип заказа на выполнение работ
        - Порядковый номер
        - Место
        - Склад
        - Приоритет работы
        - Код пула работ
        - Код волны
        - Номер создания работы

    - Следующие поля не копируются в новый код работы:

        - **Код работы** — новый код работы создается из соответствующей номерной серии.
        - **Статус работы** — для этого поля установлено значение *Открыто*.
        - **Заблокировано** — для этого поля первоначально установлено пустое значение.
        - **Код целевого грузоместа** — это поле остается пустым.
        - **Дата и время создания** — в этом поле задается текущая дата и время.
        - **Заблокированная волна/заморожено** — это поле пересчитывается для исходного кода работы и нового кода работы.

1. В панели операций выберите **Разделить работу**.

В процессе разделения работы отображается следующее сообщение: "Обработка операции — разделение работы". Пока это сообщение отображается, можно отменить операцию, выбрав **Отмена** в окне сообщения.

Если флажок **Показать все строки** снят, строка, которая была разбита и отменена, больше не будет отображаться в сетке. Если установлен флажок, вы должны увидеть, что значение поля **Статус работы** для этой строки изменилось на *Отменено*.

Отображается следующее уведомление, указывающее на то, что новый код работы был создан: "Работа \#\#\#\# была создана путем разделения из исходной работы \#\#\#\#."

Другие строки работы из исходного кода работы (например, строки *Разместить*) будут скорректированы по мере необходимости для отображения строк отмененной работы. Например, если у исходного кода работы имелась строка *Разместить* для количества 15, а строки *Комплектация* с общим количеством 10 были отменены, новое количество *Разместить* в исходном коде работы теперь будет иметь значение 5.

Новая работа не будет сразу назначена никакому пользователю. Однако при необходимости теперь можно назначить ее пользователю, используя стандартную функциональность страницы **Сведения о работе**.

> [!IMPORTANT]
> Можно разбить только коды работ, содержащие две или более доступных строк работы. Если выбрать **Разделить работу**, когда имеется только одна строка работы, будет выведено следующее сообщение об ошибке: "По крайней мере одна строка работы должна оставаться в исходной работе." В этом случае разбиение производиться не будет.

## <a name="finish-a-work-split"></a>Завершение разделения работы

Для завершения разделения работы должна быть удалена причина блокировки *Разделить работу*. Существует два способа выполнения этого шага:

- Пользователь, выполняющий разбиение работы, закрывает страницу **Разделить работу**, выбирая кнопку **Закрыть** (**X**) в верхнем правом углу. При закрытии страницы причина блокировки *Разделение работы* будет удалена. Состояние *Заблокировано* этой работы будет пересчитано, и, если для этой работы нет оставшихся причин блокировки, работа будет разблокирована.
- Любой пользователь открывает код работы и выбирает кнопку **Отмена сеанса разделения работы** на панели операций. Причина блокировки *Разделение работы* будет удалена, и состояние *Заблокировано* этой работы будет пересчитано, точно так же, как и при закрытии страницы **Разделить работу**.

После удаления причины блокирования *Разделение работы* можно выполнить работу на мобильном устройстве, при условии, что состояние **Заблокировано** установлено на значение *Нет* в коде работы.

## <a name="user-blocking-on-the-warehouse-app"></a>Блокировка пользователя в приложении склада

При попытке использовать приложение склада для выполнения работы комплектации по коду работы, которая была разделена, вы получите следующее сообщение об ошибке: "Работа с кодом \#\#\#\# в данный момент разделена." Если получено это сообщение, выберите **Отмена**. После этого можно продолжить обработку другой работы.

## <a name="other-blocked-operations"></a>Другие заблокированные операции

Любые операции, которые изменяют строки работы, проводки по запасам работы или ссылки пополнения, которые связаны с разделяемой работой, не будут работать, и будет показано следующее сообщение об ошибке: "Работа с кодом \#\#\#\# в данный момент разделена."
