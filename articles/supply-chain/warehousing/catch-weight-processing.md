# <a name="catch-weight-product-processing-with-warehouse-management"></a>Обработка учета продуктов в двух единицах измерения с помощью управления складом

[!include [banner](../includes/banner.md)]

## <a name="feature-exposure"></a>Действие функции

Для использования системы управления складом для обработки учета в двух единицах измерения продуктов необходимо использовать лицензионный ключ конфигурации, чтобы включить эту функцию. (Перейдите в раздел **Администрирование системы \> Настройка \> Конфигурация лицензии**. Затем на вкладке **Конфигурационные ключи** разверните **Торговля \> Управление складом и транспортировкой** и установите флажок **Учет в двух единицах измерения для склада**.)

> [!NOTE]
> Как конфигурационный ключ лицензии **Управление складом и транспортировкой**, так и конфигурационный ключ лицензии **Распределение процессов \> Учет в двух единицах измерения** также должны быть включены.

После включения лицензионного ключа при создании выпущенного продукта можно выбрать **Учет в двух единицах измерения**. Выпущенный продукт можно также связать с группой аналитик хранения, для которой выбран параметр **Использовать процессы управления складом**.

Перед использованием продукта в управлении складом необходимо выполнить некоторые базовые настройки конкретного продукта для учета в двух единицах измерения.

- Определите номинальное преобразование веса между вторичной единицей учета (коробка) и единицей складского учета (килограмм \[кг\]) как часть определения преобразования единиц измерения.
- Определите минимальный и максимальный допуск веса как часть настройки учета в двух единицах измерения.
- Настройте группу упорядочения единиц измерения, где вторичная единица измерения определяется как низшая единица складского учета (SKU).
- Настройте политику обработки номенклатур, учитываемых в двух единицах измерения.

Дополнительные сведения см. в разделе [Настройка и ведение номенклатур с учетом в двух единицах измерения](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/setting-up-and-maintaining-catch-weight-items).

## <a name="transaction-adjustments"></a>Корректировки проводок

Так как вес запасов при поступлении на склад может отличаться от веса при выдаче запасов со склада, при обработке продуктов с учетом в двух единицах измерения необходимо корректировать имеющиеся запасы.

**Пример 1**

Во время производственного процесса **Приемка** входящий вес грузоместа, содержащего восемь коробок продукта с учетом в двух единицах измерения регистрируется как 80,1 кг. Грузоместо затем хранится в области готовой продукции и в течение периода хранения часть веса будет потеряна в воздухе.

Позже, в качестве части процесса комплектации заказа на продажу, вес этого же грузоместа регистрируется как 79,8 кг. Таким образом, в системе появится разница веса как часть набора физической аналитики.

В этом случае система автоматически корректирует разницу путем разноски проводки для отсутствующих 0,3 кг.

**Пример 2**

В определении продукта он настроен для допуска минимального веса 8 кг и максимального веса 12 кг для единицы **Коробка** учета в двух единицах измерения.

Имеется две коробки продукта, и у них есть зарегистрированный вес 16 кг. Если работник склада выбирает и взвесит одну из коробок и вес окажется равным 9 кг, остальная коробка будут весить 7 кг. Однако потому, что 7 кг меньше минимального веса, система выполняет автоматическую коррекцию для повышения веса складских запасов в наличии на 1 кг.

Чтобы настроить счета, на которые разносятся эти корректировки, выберите **Управление затратами \> Настройка политик интеграции главной книги \> Разноска**. Затем на вкладке **Запасы** определите следующие счета:

- Счет убытков во вторичной единице измерения
- Счет прибылей во вторичной единице измерения

## <a name="catch-weight-item-handling-policy"></a>Политика обработки номенклатур, учитываемых в двух единицах измерения

Политика обработки номенклатур с учетом в двух единицах измерения определяет два основных потока управления складом для номенклатур: время регистрации веса номенклатур и порядок его регистрации.

Можно определить, когда вес регистрируется для обработки заказов на продажу и на перемещение. Вес может быть зафиксирован во время любого из следующих процессов:

- **Комплектация** — вес регистрируется во время строк первоначальной работы по подбору для обработки заказа.
- **Упаковка** — вес регистрируется во время упаковки вручную. (Номенклатуры необходимо отправить на станцию упаковки.)

Если фактический вес регистрируется на станции упаковки во время процесса упаковки контейнера, работникам склада не предлагается зарегистрировать вес во время работ по комплектации. Вместо этого средний вес физических запасов будет использоваться как вес скомплектованных запасов, которые отправляются в зону упаковки.

Для внутренних складских процессов управления, таких как инвентаризации и корректировка исправлений, можно определить, требуется ли фиксировать вес. Если не фиксируется, номинальный вес будет использоваться.

Можно также определить, как вес регистрируется. В одном из двух основных потоков метки учета в двух единицах измерения отслеживаются и используются для регистрации веса. В другом потоке метки веса учета в двух единицах измерения не отслеживаются.

- **Да** — номенклатура использует метки учета в двух единицах измерения. Каждое число метки указывает одну единицу учета в двух единицах измерения (коробка), и вес и другие сведения связываются с меткой. Для исходящих процессов используется вес, который связан с меткой.
- **Нет** — номенклатура не использует метки учета в двух единицах измерения. Входящие и исходящие процессы взвешивания основаны на фактическом весе, зарегистрированном во время каждого процесса.

Процесс отслеживания меток учета в двух единицах измерения можно использовать для номенклатур, вес которых не изменяется в течение периода хранения. Вес будут записан только во входящем процессе склада. Во время исходящего процесса метки учета в двух единицах измерения будут просто сканироваться, и значения вес, которые связаны с тегами, будут использоваться для обработки исходящих транзакций.

### <a name="how-to-capture-catch-weight"></a>Как регистрировать учет в двух единицах измерения

При использовании отслеживания меток учета в двух единицах измерения тег должен всегда создаваться для каждой единицы учета в двух единицах измерения, и каждый тег всегда должен быть связан с весом.

Например, **Коробка** представляет собой единицу измерения учета в двух единицах измерения, и вы получаете одну палету с восемью коробками. В этом случае восемь уникальных тегов учета в двух единицах измерения должны быть созданы, и вес должен быть связан с каждым тегом. В зависимости от маркера входящего учета в двух единицах измерения вес всех восьми коробок может быть зафиксирован и средний вес можно распределить на каждую коробку, или уникальный вес может быть зафиксирована для каждой коробки.

При использовании отслеживания маркера учета в двух единицах измерения вес может регистрироваться для каждого набора аналитик (например, для каждого грузоместа и аналитики отслеживания). Альтернативно вес может быть захвачен на основании общего уровня, такого как пять грузомест (палет).

Для методов регистрации исходящего веса можно определить, выполняется ли взвешивание для каждой вторичной единицы измерения (то есть, за упаковку), или ли вес регистрируются на основании количества, которое будет комплектоваться (например, из трех упаковок). Обратите внимание, что для процесса комплектации строки производства средний вес будет использоваться, если используется параметр **Не регистрируется**.

## <a name="supported-scenarios"></a>Поддерживаемые сценарии

Не все workflow-процессы поддерживают обработку учета продуктов в двух единицах измерения с управлением склада. В настоящее время применяются следующие ограничения.
 
### <a name="configuring-catch-weight-products-for-warehouse-management-processes"></a>Регистрация продуктов учета в двух единицах измерения для процессов управления складом

- Для продуктов учета в двух единицах измерения нельзя изменить группу аналитик хранения для номенклатур (чтобы процессы управления складом можно было использовать для них).
- Для продуктов с учетом в двух единицах измерения поддерживается только обработка формулы (не спецификация).
- Продукты с учетом в двух единицах измерения не могут быть связаны с группой аналитик отслеживания с помощью аналитики ответственного.
- Продукты с учетом в двух единицах измерения не могут использоваться как службы.
- Продукты с учетом в двух единицах измерения можно использовать в качестве "учитываемые в запасах продуктов" только в рамках группы моделей номенклатуры.
- Продукты с учетом в двух единицах измерения не могут использоваться вместе с функциональными возможностями для отслеживания "Активно в процессе продаж".
- Продукты с учетом в двух единицах измерения не могут использоваться вместе с функциональными возможностями для регистрации серийных номеров. Таким образом, продукты нельзя перенести из "пусто" в серийный номер в процессе комплектации/упаковки.
- Продукты с учетом в двух единицах измерения не могут использоваться вместе с функциональными возможностями для регистрации серийных номеров перед потреблением.
- Продукты с учетом в двух единицах измерения, для которых доступны варианты, не могут использоваться вместе с функциональной возможностью для преобразования единиц измерения вариантов.
- Продукт с учетом в двух единицах измерения не могут помечаться как розничный "комплект продуктов".
- Продукты с учетом в двух единицах измерения можно использовать только с группой номерных серий, которая имеет единицы обработки учета в двух единицах измерения, и которая имеет вторичную единицу измерения как самую низкую последовательность.
- Для продуктов с учетом в двух единицах измерения единицу складского учета можно преобразовать в вторичную единицу учета в двух единицах измерения только в том случае, если преобразование создает номинальное количество, которое превышает 1.
- Настройка штрих-кодов для продуктов с учетом в двух единицах измерения не поддерживает настройку переменного веса.
 
### <a name="order-processing"></a>Обработка заказа

- Обработка внутрихолдинговых заказов не поддерживается.
- Создание предварительного уведомления об отгрузке (ASN/упаковочные структуры) не поддерживает сведения о весе.
- Количество заказа должно быть основано на единице учета в двух единицах измерения.
 
### <a name="inbound-warehouse-processing"></a>Входящая обработка на складе

- Получение грузомест требует, чтобы веса был присвоены во время регистрации, поскольку сведения о весе не поддерживаются как часть предварительного уведомления об отгрузке. Когда тег учета в двух единицах измерения используется, необходимо вручную назначить номер маркера на единицу учет в двух единицах измерения.
- Получение грузомест со смешанными номенклатурами не поддерживается для продуктов, учитываемых в двух единицах измерения.
 
### <a name="inventory-and-warehouse-operations"></a>Операции запасов и складов

- Создание вручную карантинных заказов не поддерживается для продуктов с учетом в двух единицах измерения.
- Перемещение запасов вручную, которое связанно с работой, не поддерживается для продуктов, учитываемых в двух единицах измерения.
- Консолидация грузомест не поддерживается для продуктов, учитываемых в двух единицах измерения.
- Изменения в состоянии складских запасов как часть периодической задачи не поддерживаются для продуктов с учетом в двух единицах измерения.
- Изменения статуса складских запасов, которые определены по запросу, не поддерживаются для продуктов с учетом в двух единицах измерения. (Изменения статуса складского заказа контроля качества также не поддерживаются.)
- Для продуктов с учетом в двух единицах измерения статус складских запасов не может изменяться со страницы **В наличии по местонахождению**.
- Для продуктов с учетом в двух единицах измерения статус складских запасов невозможно изменить как часть работы перемещения склада приложения.
- Загрузка грузоместа для инициализации запасов на складе не поддерживается для продуктов с учетом в двух единицах измерения.
- Процессы балансировки партий не поддерживаются для продуктов с учетом в двух единицах измерения.
- Обработка отрицательных физических остатков не поддерживается для продуктов с учетом в двух единицах измерения.
- Маркировка запасов не может использоваться для продуктов с учетом в двух единицах измерения.
 
### <a name="outbound-warehouse-processing"></a>Исходящая обработка на складе

- Функция комплектации кластера не поддерживается для продуктов с учетом в двух единицах измерения.
- Обработка комплектации и упаковки на складе не поддерживается для продуктов с учетом в двух единицах измерения.
- Для продуктов с учетом в двух единицах измерения работа не может быть выполнена со страницы **Работа**.
- Для продуктов с учетом в двух единицах измерения работа, которая определена в шаблоне работы, может запускаться автоматически.
- Функция обращения работы не поддерживается для продуктов с учетом в двух единицах измерения.
- Для продуктов с учетом в двух единицах измерения ручная обработка на отборочной станции, когда работа создается после закрытия контейнера, не поддерживается.
- Функция поштучного сканирования не поддерживается для продуктов с учетом в двух единицах измерения.
 
### <a name="production-processing"></a>Обработка производства

- Для продуктов с учетом в двух единицах измерения поддерживаются только заказы партий для продуктов по формуле.
- Функция канбан не поддерживается для продуктов с учетом в двух единицах измерения.
- Для продуктов с учетом в двух единицах измерения серийные номера не могут регистрироваться до потребления.
- Функция обращения грузомест не поддерживается для продуктов с учетом в двух единицах измерения.
- Для продуктов с учетом в двух единицах измерения приемку можно зарегистрировать по серийному номеру.

### <a name="transportation-management-processing"></a>Обработка управления транспортировкой

- Обработка на рабочем месте формирования загрузки не поддерживается для продуктов с учетом в двух единицах измерения.
- Строки запросов на транспорт не поддерживаются для продуктов с учетом в двух единицах измерения.
 
### <a name="other-restrictions-and-behaviors-for-catch-weight-product-processing-with-warehouse-management"></a>Другие ограничения и поведения для обработки продуктов с учетом в двух единицах измерения при управлении складом

- При регистрации меток учета в двух единицах измерения в составе обработки в приложении склада пользователь не может выплнить отмену вне workflow-процесса.
- Во время процессов комплектации, в которых пользователю не отображается подсказка для идентификации аналитик отслеживания, назначение веса осуществляется на основании среднего веса. Это поведение происходит, например, когда комбинация аналитик отслеживания используется в одном месте и, после обработки комплектации пользователем, только одно значение аналитики отслеживания остается в местонахождении.
- Когда запасы резервируются для продукта с учетом в двух единицах измерения, который настроен для процессов управления складом, резервирование выполняется на основании минимального веса, который определен, даже если это количество является последним количеством обработки в наличии. Это поведение отличается от поведения для номенклатур, которые не настроены для процессов управления складом.
- Процессы, использующие вес как часть расчетах мощности (пороговые значения волны, максимальные рабочие перерывы, максимальные значения контейнера, загрузка производственных мощностей местоположения и т. д.), не используют фактический вес запасов. Вместо этого процессы основаны на весе физической обработки, который определяется для продукта.
- В общем случае, функция розничной торговли не поддерживается для продуктов с учетом в двух единицах измерения.
 
### <a name="catch-weight-tags"></a>Теги учета в двух единицах измерения

Возможность тегов учета в двух единицах измерения в настоящее время поддерживается только в следующих случаях:

- При обработке получения заказа на покупку в приложении склада.
- При обработке получения груза через приложение склада.
- Для получения грузоместа, которое относится к загрузке заказа на покупку, назначение веса запрашивается во время процесса получения. Напротив, для получения заказа на перемещение вес используется из данных отгрузки для заказа на перемещение.
- Для получения номенклатуры и строки заказа на перемещение, который поступает со склада, не использующего процесс управления складом.
- При обработке приемки заказов на возврат продажи можно записать теги учета в двух единицах измерения, но обработка не будет проверяться, если теги не совпадают с тегами, которые первоначально были отгружены для конкретной строки заказа на продажу.
- При обработке изменения статуса запасов с помощью приложения склада.
- При выполнении перемещения склада с помощью приложения склада.
- При обработке корректировки прихода и ухода через приложение склада.
- При обработке работы по комплектации для заказов на продажу и перемещение. (Обратите внимание, что теги учета в двух единицах измерения не могут быть записаны для комплектации производственных компонентов.)
- Когда скомплектованные количества уменьшаются из строк загрузки, независимо от того, используются ли контейнеры.
- Когда продукты упакованы в контейнеры на станции упаковки.
- Когда контейнеры открываются повторно.
- При приемки продуктов с формулами с помощью приложения склада.
- При обработке транспортных грузов с помощью приложения склада.
