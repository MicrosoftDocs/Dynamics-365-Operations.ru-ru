# <a name="system-directed-cluster-picking"></a>Комплектация кластера, управляемая системой

[!include [banner](../includes/banner.md)]

Комплектации кластера — это процесс выбора номенклатур, который позволяет комплектовать номенклатуры для нескольких заказов одновременно путем кластеризации их в кластеры комплектации. Затем вы должны посетить местонахождение комплектации только один раз. Как правило, эта функциональность используется при небольшой комплектации заказов или при количествах, которые меньше количества обращений.

При настройке комплектации кластера под управлением системы можно создавать заголовки работ комплектации по кластеру на основе создаваемого системой кластера. Система осуществляет комплектацию по кластерам для заказов до числа позиций, указанных в профиле кластера. Таким образом, можно комплектовать несколько заказов одновременно без необходимости вручную создавать кластер.

Комплектация кластера под управлением системы является альтернативой ручного создания кластеров, где для создания кластера используется профиль кластеров. Перед использованием в профиле кластера следует определить некоторые связанные с настройкой строки:

- **Число штатных единиц** — количество заказов, которые будут помещены в кластер. Таким образом, это соответствует количеству общего.
- **Разделить кластер** определяет необходимость разделения кластера.
- **Создать идентификатор кластера** управляет, будет код кластера создан системой или введен пользователем.
- **Тип проверки сортировки** определяет, требуется ли проверка позиции.

Новый пункт меню мобильного устройства используется для комплектации кластеров под управлением системы. Код профиля кластера должен быть указан для параметра **Кем управляется**. Кроме того, заказ запросов под управлением системы может группировать заказы на основе конкретных критериев компании. Таким образом, можно дополнительно оптимизировать назначение заказов на работу, указав настроенные критерии сортировки в заказе запроса под управлением системы.

Когда комплектация кластера под управлением системы завершена, работникам складов предоставляется кластер, в котором заказы комплектации уже заданы для позиций кластера. Таким образом, работники могут начать комплектацию номенклатуры для нескольких заказов на работу, посетив место комплектации только один раз. Процесс комплектации кластера под управлением системы такой же, как и процесс комплектации кластера под управлением пользователя.

## <a name="set-up-system-directed-cluster-picking"></a>Настройка комплектации кластера, управляемой системой

### <a name="create-cluster-profiles"></a>Создание профилей кластеров

Профили кластеров управляют, как система создает каждый кластер. Если необходимы разные кластеры, для каждого пункта меню мобильного устройства должен быть создан другой профиль кластера.

1. Перейдите **Управление складом \> Настройка \> Мобильное устройство \> Профили кластера**.
2. Выберите **Создать**.
3. В поле **Код профиля кластера** введите **Позиция 2**.
4. В поле **Имя** введите **Позиция 2**.
5. На экспресс-вкладке **Общие** настройте следующие поля:

    - **Создание кода кластера:** Установите этот параметр как **Да**. Этот параметр определяет, автоматически ли код кластера создается системой или же пользователь создаст его в начале комплектации.
    - **Активация параметров:** задайте значение **Да**. Этот параметр определяет, автоматически ли создаются имена позиций на основе настройки имени позиции. Если параметр задан **Нет**, используется код грузоместа для позиции.
    - **Число штатных единиц:** введите **2**. Это поле определяет максимальное количество позиций, которые может иметь кластер (т.е. максимальное количество коробок, общего и так далее).
    - **Имя позиции:** выберите **числовые**, чтобы позиции были названы с помощью непрерывных чисел. Если вы выберете **алфавит**,то позиции будут названы в алфавитном порядке.
    - **Разделить кластер в:** выберите **Разместить**. Это поле определяет, когда кластер разделяется.
    - **Тип проверки сортировки:** выберите **Скан позиции**. Это поле определяет, проверяется ли шаг размещения должности.

6. На экспресс-вкладке **Сортировка кластера** можно определить критерии сортировки, создав новую строку и установив следующие поля:

    - **Порядковый номер:** это поле определяет последовательность, которую сортирует система. Значение вводится автоматически, но вы можете изменить его, как вам требуется.
    - **Имя поля:** выберите **WMSLocationId**. Это поле определяет поле, которое строка использует для критериев сортировки.
    - **Сортировка:** выберите **Восходящий**. Это поле определяет, осуществляется ли сортировка в восходящем или убывающем порядке.

### <a name="create-a-mobile-device-menu-item"></a>Создание элемента меню мобильного устройства

Чтобы создать новый пункт меню мобильного устройства для комплектации кластера под управлением системы и связать код профиля кластера с пунктом меню мобильного устройства, выполните эти действия.

1. Перейдите в раздел **Управление складом \> Настройка \> Мобильное устройство \> Пункты меню мобильного устройства**.
2. Выберите **Создать**.
3. В поле **Имя пункта меню** введите **Кластер SD**.
4. В поле **Заголовок** введите **кластер SD**.
5. В поле **Режим** выберите **Работа**.
6. Укажите для **Использовать существующую работу** вариант **Да**.
7. На экспресс-вкладке **Общие** настройте следующие поля:

    - **Кем управляется:** выберите **Управляется системой**.
    - **Создать грузоместо:** задайте как **Да**.
    - **Код профиля кластера:** выберите **позиция 2**.

8. На экспресс-вкладке **Классы работы** настройте допустимый класс работы для этого пункта меню мобильного устройства, задав следующие поля:

    - **Код класса работы:** убедитесь, что введено **Продажи**.
    - **Тип класса работы:** убедитесь, что введено **Заказы на работу**.

9. Выполните следующие действия, чтобы указать новый запрос последовательности работы под управлением системы:

    1. Выберите **Создать**.
    2. В поле **Порядковый номер** введите значение **1**.
    3. В поле **Описание** выберите **Приоритет работы — код работы**.

10. В меню выберите **Изменить запрос**.
11. На вкладке **Сортировка** заполните следующие поля:

    - **Таблица:** выберите **Работа**.
    - **Производная таблица:** выберите **Работа**.
    - **Поле:** выберите **Приоритет работы**.
    - **Направление поиска:** выберите **Восходящий**.
    - **Таблица:** выберите **Работа**.
    - **Производная таблица:** выберите **Работа**.
    - **Поле:** выберите **Код работы**.
    - **Направление поиска:** выберите **Восходящий**.

Теперь система будет сортировать коды работы в кластере, сначала по приоритету работы, а затем по коду работы.

### <a name="set-up-a-mobile-device-menu"></a>Настройка меню мобильного устройства

1. Перейдите в раздел **Управление складом \> Настройка \> Мобильное устройство \> Меню мобильного устройства**.
2. Добавьте только что созданный пункт меню в нужное меню.

## <a name="example"></a>Пример

### <a name="create-picking-work"></a>Создать работы комплектации

Прежде чем вы сможете настроить комплектацию кластера под управлением системы, необходимо создать некоторые подходящие исходящие работы. Профиль кластера, созданный ранее, определяет две позиции кластера. Таким образом, необходимо создать как минимум два кода работы.

1. Перейдите в раздел **Продажи и маркетинг \> Заказы на продажу \> Все заказы на продажу**.
2. Выберите **Создать**, чтобы создать заказ на продажу.
3. В поле **Счет клиента** выберите любого клиента.
4. На экспресс-вкладке **Общее** в поле **Склад** выберите склад **62**.
5. Нажмите **ОК**.
6. На экспресс-вкладке **Строки заказа на продажу** выберите **Добавить строку**.
7. В поле **Код номенклатуры** выберите **A0001**.
8. В поле **Количество** введите **1**.
9. Выберите **Добавить строку**, чтобы добавить вторую строку.
10. В поле **Код номенклатуры** выберите **A0002**.
11. В поле **Количество** введите **3**.
12. Повторите шаги 2–6.
13. В поле **Код номенклатуры** выберите **A0001**.
14. В поле **Количество** введите **4**.
15. Выберите **Добавить строку**, чтобы добавить вторую строку.
16. В поле **Код номенклатуры** выберите **A0002**.
17. В поле **Количество** введите **2**.

Зарезервируйте запасы и выпустите на склад. Создаются два разных кода работы. Каждый код работы имеет две строки комплектации.

### <a name="run-the-mobile-device-flow"></a>Запуск потока мобильных устройств

1. На мобильном устройстве выберите меню, включающее новый пункт меню мобильного устройства.
2. Выберите пункт меню **Кластер SD** и инициируйте комплектацию. Создается кластер, и к нему прикрепляются два кода работы, созданные ранее. Если вы создали более двух кодов работы, в кластер добавляются только первые два. Обратите внимание, что коды работы добавляются в кластер в порядке возрастания, как вы указали в настройке запроса.

    > [!NOTE]
    > Новый кластер автоматически создается только в том случае, если ранее было создано достаточное количество дополнительных кодов работы. В противном случае отображается следующее сообщение: «Для кластера не найдено достаточно работы».

    После выбора меню появляется первый экран комплектации. Система агрегирует все соответствующие строки комплектации из двух кодов работы и направляет вас посетить место комплектации один раз, так что вы можете выполнить оба заказа с помощью одной комплектации. Этот процесс выполняется так же, как и процесс комплектации кластера под управлением пользователя.

3. Подтвердите первую комплектацию. Затем необходимо ввести имя позиции, чтобы подтвердить, что элементы были поставлены в правильное положение.
4. Повторите этот процесс до тех пор, пока все элементы не будут выбраны и встанут в правильное положение.
5. Последним шагом на мобильном устройстве является размещение кластера в окончательном месте. Когда операция размещения подтверждена, кластер закрывается и разбивается в зависимости от значения, которое устанавливается для поля **Разделить кластер в** в профиле кластера. Коды работы также закрыты.
6. На мобильном устройстве отображается сообщение "Кластер готов".
