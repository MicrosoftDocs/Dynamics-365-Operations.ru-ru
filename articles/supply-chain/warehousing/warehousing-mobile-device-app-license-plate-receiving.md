# <a name="license-plate-receiving-via-the-warehouse-app"></a>Получение грузоместа через мобильное приложение склада

[!include [banner](../includes/banner.md)]

В этой теме объясняется, как настроить мобильное приложение склада, чтобы оно поддерживало использование процесса получения грузоместа для получения физических запасов.

Можно использовать эту функцию для быстрой регистрации прихода входящих запасов, которые относятся к предварительному уведомлению об отгрузке (ASN). Система автоматически создает ASN, когда процессы управления складом используются для отгрузки заказа на перемещение. Для процесса заказа на покупку ASN может быть записано вручную, либо оно может быть автоматически импортировано с помощью входящего процесса информационного объекта ASN.

Данные ASN связаны с загрузками и отгрузками через *упаковочные структуры*, где палеты (родительские грузоместа) могут содержать ящики (вложенные грузоместа).

> [!NOTE]
> Для уменьшения количества складских проводок при использовании структур упаковки, для которых используются вложенные грузоместа, система записывает физические запасы в наличии на родительском грузоместе. Для запуска перемещения физических запасов в наличии с родительского грузоместа во вложенные грузоместа, основанного на данных о структуре упаковки, на мобильном устройстве должен содержаться пункт меню, основанный на процесса создания работы *Упаковывать по вложенным грузоместам*.

## <a name="warehousing-mobile-device-app-processing"></a>Обработка приложения мобильного устройства складского учета

Когда работник сканирует идентификатор поступающего грузоместа, система запускает процесс получения грузоместа. На основе этих сведений содержимое грузоместа (данные, поступающие из ASN) физически регистрируется в местоположении дебаркадера приемки. Последующие потоки будут зависеть от потребностей бизнес-процесса.

## <a name="work-policies"></a>Политики работы

Например, как и процесс пункта меню мобильного устройства *Приемка*, процесс получения грузоместа поддерживает несколько бизнес-правил на основе заданной настройки.

### <a name="work-policies-with-work-creation"></a>Политики работы с созданием работ

При регистрации входящих номенклатур с помощью политики работы, которая создает работу, система создает и сохраняет записи работы размещения для каждой регистрации. Если используется процесс работы *Получение грузоместа и размещение*, регистрация и размещение обрабатываются как единая операция с помощью одного пункта меню мобильного устройства. Если используется процесс *Получение грузоместа*, процессы получения и размещения обрабатываются как две разные складские операции, каждая со своим собственным пунктом меню мобильного устройства.

### <a name="work-policies-without-work-creation"></a>Политики работы без создания работ

Процесс получения грузоместа можно использовать без создания работы. При определении политик работы с типом заказа на работу *Приход переноса* и/или *Заказы на покупку* и использовании процесса *Получение грузоместа (и размещение)* следующие два процесса мобильного приложения складского учета не смогут создавать работу. Вместо этого они просто регистрируют входящие физические запасы для грузоместа в входящем дебаркадере приемки.

- *Получение грузоместа*
- *Получение и складирование номерного знака*

> [!NOTE]
> - В разделе **Места хранения** необходимо определить по крайней мере одно место хранения для политики работы. Нельзя указывать одно и то же местоположение для нескольких политик работы.
> - Параметр **Печать этикетки** для пунктов меню мобильного устройства складского учета не печатает этикетку грузоместа без создания работы.

Чтобы эта функция была доступна в вашей системе, необходимо включить функцию *Улучшения получения грузомест* в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

### <a name="receive-inventory-on-a-location-that-doesnt-track-license-plates"></a>Получение запасов в месте, которое не отслеживает грузоместа

Можно использовать местонахождение склада, назначенное профилю местонахождения даже в том случае , если **Использовать отслеживание грузоместа** не включено. Таким образом, при получении запасов можно непосредственно зарегистрировать запасы в наличии в местонахождении без создания работ.

## <a name="add-mobile-device-menu-items-for-each-receiving-location-in-a-warehouse"></a>Добавление пунктов меню мобильного устройства для каждого места приемки на складе

Функция *Улучшения получения грузомест* позволяет получать в любое местоположение на складе, добавив пункты меню получения (и размещения) грузоместа с учетом местоположения в мобильное приложение складского учета. Раньше система поддерживала получение только в местоположение по умолчанию, определенное для каждого склада. Однако, если эта функция включена, пункты меню мобильного устройства для получения грузомест (и размещения) теперь содержат параметр **Использовать данные по умолчанию**, что позволяет выбрать настраиваемое местоположение получения для каждого пункта меню. (Этот параметр уже доступен для некоторых других типов пунктов меню.)

Чтобы эта функция была доступна в вашей системе, необходимо включить функцию *Улучшения получения грузомест* в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

## <a name="show-or-skip-the-receiving-summary-page"></a>Отображение или пропуск страницы сводки получения

Можно использовать функцию *Управление отображением страницы сводки получения на мобильном устройстве*, чтобы воспользоваться преимуществом дополнительного подробного потока приложения склада в рамках процесса получения грузомест.

Если эта функция включена, пункты меню мобильного устройства для получения грузомест или получения и размещения грузомест необходимо задать параметр **Отображать страницу сводки получения**. Эта настройка имеет следующие параметры:

- **Отображать подробную сводку** — во время получения грузоместа работники увидят дополнительную страницу, которая показывает полные сведения ASN.
- **Пропустить сводку** — сотрудники не будут видеть полные сведения ASN. Работники склада не смогут задавать код метода обработки или добавлять исключения в ходе процесса приемки.

Чтобы эта функция была доступна в вашей системе, необходимо включить функцию *Управление отображением страницы сводки получения на мобильном устройстве* в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

## <a name="prevent-transfer-ordershipped-license-plates-from-being-used-at-warehouses-other-than-the-destination-warehouse"></a>Блокировать использование грузомест, отправленных по заказам на перемещение, на складах, отличных от склада назначения.

Процесс получения грузомест не может использоваться, если ASN содержит идентификатор грузоместа, уже существующий и имеющий данные о физическом наличии в местоположении склада, отличном от местоположения склада, где выполняется регистрация этого грузоместа.

Для сценариев заказов на перемещение, когда транзитный склад не отслеживает грузоместа (и, таким образом, также не отслеживает физические запасы в наличии для каждого грузоместа), можно использовать функцию *Блокировать использование грузомест, отправленных по заказам на перемещение, на складах, отличных от склада назначения*, чтобы предотвратить физические обновления запасов в наличии для грузомест в процессе транзита.

Чтобы эта функция была доступна в вашей системе, необходимо включить функцию *Блокировать использование грузомест, отправленных по заказам на перемещение, на складах, отличных от склада назначения* в [управлении функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

Чтобы управлять функциональностью, когда эта функция доступна, выполните следующие действия.

1. Перейдите в раздел **Управление складом \> Настройка \> Параметры управления складом**.
1. На вкладке **Общие** на экспресс-вкладке **Грузоместа** задайте в поле **Политика грузомест на транзитном складе** одно из следующих значений:

    - **Разрешить повторное использование неотслеживаемых грузомест** — система работает так же, как и при недоступности функции *Блокировать использование грузомест, отправленных по заказам на перемещение, на складах, отличных от склада назначения*. Это значение является значением по умолчанию при первой активации функции.
    - **Запретить повторное использование неотслеживаемых грузомест** — только обновления запасов в наличии, которые относятся к отгруженному грузоместу, будут разрешены на складе назначения до тех пор, пока не будет получен заказ на перемещение.
