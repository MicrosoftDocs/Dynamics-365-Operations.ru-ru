# <a name="troubleshoot-outbound-warehouse-operations"></a>Устранение неполадок исходящих операций склада

[!include [banner](../includes/banner.md)]

В этой теме описывается устранение распространенных проблем, которые могут встретиться при работе с исходящими операциями склада в Microsoft Dynamics 365 Supply Chain Management.

## <a name="i-receive-the-following-error-message-sales-order-could-not-be-released"></a>Получено следующее сообщение об ошибке: "Заказ на продажу не может быть выпущен."

### <a name="issue-description"></a>Описание проблемы

Эта проблема может возникнуть по нескольким причинам. Например, заказ находится на блокировке в системе управления кредитами, и отгрузки невозможно создать до тех пор, пока не будет введен действительный почтовый адрес для всех строк продаж, связанных с заказом на продажу. Также возможна блокировка заказа, которая должна быть устранена, прежде чем заказ может быть выпущен на склад. Эта блокировка может быть связана с определенным заказом, или она может находиться на счете клиента.

### <a name="issue-resolution"></a>Устранение проблемы

Добавьте или обновите адрес в заказе на продажу и строках заказа, а затем выпустите заказ на склад. Заказы могут быть выпущены на склад только в том случае, если они имеют допустимый адрес поставки (согласно настройке формата адреса в модуле **Администрирование организации**).

Изучите блокировку заказа и устраните проблемы. Затем удалите блокировку из заказа или клиента и выпустите заказ на склад.

## <a name="i-receive-the-following-message-the-shipment-for-load-1-has-been-confirmed-however-no-lines-are-posted"></a>Получено следующее сообщение: "Была подтверждена отгрузка для загрузки 1%." Однако никакие строки не разносятся.

### <a name="issue-description"></a>Описание проблемы

Отгрузка по загрузке подтверждена, но дальнейшая разноска не выполнена.

### <a name="issue-resolution"></a>Устранение проблемы

Подтверждение отгрузки не влияет на разноску. Оно просто обновляет отгрузку и статус загрузки. Разноска должна выполняться в отдельном процессе.

## <a name="i-receive-the-following-error-message-direct-delivery-is-not-able-to-process-for-warehouse-1-as-it-has-warehouse-management-enabled-please-specify-another-warehouse-that-is-not-enabled-for-warehouse-management"></a>Получено следующее сообщение об ошибке: "Невозможна обработка прямой поставки для склада %1, так как для него включено управление складом. Задайте другой склад, для которого управление складом не включено."

### <a name="issue-description"></a>Описание проблемы

Номенклатура добавляется в строку продаж для прямой поставки со склада, который включен для управления складом (WMS). Это сообщение об ошибке выводится при обновлении строки продаж. 

### <a name="issue-resolution"></a>Устранение проблемы

Корпорация Майкрософт проверила эту проблему и определила, что она является ограничением функции. В настоящее время WMS не поддерживает прямую поставку. Таким образом, для использования прямой поставки необходимо выбрать номенклатуру и склад, для которых не используется WMS.
