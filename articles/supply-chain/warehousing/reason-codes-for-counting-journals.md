# <a name="reason-codes-for-inventory-counting"></a>Коды оснований для проведения инвентаризации запасов

[!include [banner](../includes/banner.md)]

Коды оснований позволяют анализировать результаты процесса инвентаризации и любых несоответствий, возникающих во время этого процесса. Можно указать причину для выполнения инвентаризации, например сломанная палета или корректировка запасов, которая основана на образцах запасов.

## <a name="recommendation"></a>Рекомендации

Перед настройкой системы рекомендуется определить стратегию для работы с кодами оснований. Например, попробуйте ответить на следующие вопросы:

- Должны ли коды оснований быть обязательными на складе?
- Должны ли коды оснований быть обязательными или необязательными для некоторых номенклатур?
- Как много кодов оснований требуется?
- Как должны пользователи сканеров штрихкода использовать коды оснований? Должны ли коды оснований быть предустановленными, обязательными или недоступны для изменения?
- Требуется ли для складских работников другое поведение кодов причин для мобильных сканеров? Если вы ответили "Да", можно создать дополнительные пункты меню и назначить их разным сотрудникам.

## <a name="where-reason-codes-apply"></a>Когда применяются коды оснований

Можно создать несколько политик кодов оснований, и каждая политики кодов оснований может иметь две политики кодов оснований инвентаризации. Политики кодов оснований инвентаризации могут использоваться на уровне склада или на уровне номенклатуры.

## <a name="set-up-reason-code-policies"></a>Настройка политик кодов оснований

1. Выберите **Управление запасами** \> **Настройка** \> **Запасы** \> **Политики кодов оснований для проведения инвентаризации** и создайте новую политику кодов оснований.
2. В поле **Тип кода основания для проведения инвентаризации** выберите **Обязательный** или **Необязательный** для указания, является ли выбора кода основания необязательным или обязательным действием в одном из следующих журналов инвентаризации:

    - Цикличный подсчет (мобильное устройство)
    - Подсчет наличия (мобильное устройство)
    - Пороговый подсчет (мобильное устройство)
    - Входящая корректировка (мобильном устройстве)
    - Исходящая корректировка (мобильном устройстве)
    - Инвентарный журнал (полный клиент)

Можно также настроить коды оснований для отдельных складов и продуктов. Настройка кода оснований для продуктов можно не принимать во внимание настройку для складов.

## <a name="mandatory-reason-codes"></a>Обязательные коды оснований

Если задан параметр **Обязательный** в конфигурации кодов оснований для складов или номенклатур, журнал инвентаризации не может быть завершен и закрыт, пока не предоставлен код основания.

### <a name="set-up-reason-codes-for-warehouses"></a>Настройка кодов оснований для складов

1. Выберите **Управление запасами** \> **Настройка** \> **Разделение запасов** \> **Склады**.
2. На вкладке **Склад** в поле **Политика кодов оснований для проведения инвентаризации** выберите один из следующих вариантов:

    - **Пустой** — параметр, настроенный для номенклатуры, используется, чтобы определить, являются ли журналы инвентаризации обязательными для продукта.
    - **Обязательный** — код основания всегда необходим для журналов инвентаризации для склада.
    - **Необязательный** — код основания не требуется для журналов инвентаризации для склада.

### <a name="set-up-reason-codes-for-products"></a>Настройка кодов оснований для продуктов

1. Выберите **Управление сведениями о продукте** \> **Продукты** \> **Выпущенные продукты**.
2. На вкладке **Продукт** выберите **Политика кодов оснований для проведения инвентаризации**, затем выберите один из следующих вариантов:

    - **Пустой** — параметр, настроенный для склада, используется, чтобы определить, являются ли журналы инвентаризации обязательными для продукта.
    - **Обязательный** — код основания всегда необходим для журналов инвентаризации для продукта. Эта настройка переопределяет любую настройку кода основания на уровне склада.
    - **Необязательный** — код основания не требуется для журналов инвентаризации для продукта. Эта настройка переопределяет любую настройку кода основания на уровне склада.

### <a name="use-reason-codes-in-counting-journals"></a>Использование кодов оснований в журналах инвентаризации

В журнале инвентаризации можно добавить коды оснований для инвентаризации следующих типов:

- Цикличный подсчет
- Подсчет наличия
- Пороговый подсчет
- Входящая корректировка
- Исходящая корректировка

Коды оснований добавляются в строки журнала в журналах инвентаризации типа **Инвентарный журнал**.

1. Выберите **Управление запасами** \> **Записи журнала** \> **Инвентаризация номенклатуры** \> **Инвентаризация**.
2. В сведения о строке журнала инвентаризации в поле **Код основания для проведения инвентаризации** выберите вариант.

### <a name="view-the-counting-history-as-its-recorded-by-reason-codes"></a>Просмотр истории инвентаризации в том виде, в котором она зарегистрирована по кодам оснований

- Выберите **Управление запасами** \> **Запросы и отчеты** \> **Инвентарный журнал**, а затем в поле **Код основания для проведения инвентаризации** просмотрите историю инвентаризации, которая была записана с использованием кода основания.

### <a name="use-a-reason-code-for-a-quantity-adjustment"></a>Использование кода основания для корректировки количества

1. На странице **Запасы в наличии** выберите **Скорректировать количество**. Страницу **Запасы в наличии** можно открыть несколькими способами. Например, выберите **Управление запасами** \> **Запросы и отчеты** \> **Запасы в наличии**.
2. Выберите **Скорректировать количество**, а затем в поле **Код основания для проведения инвентаризации** выберите код основания.

### <a name="configure-reason-codes-for-mobile-device-menu-items"></a>Настройка кодов оснований для пунктов меню на мобильном устройстве

Можно настроить коды оснований для любого типа инвентаризации в пункте меню на мобильном устройстве. Конфигурация пункта меню мобильного устройства включает следующие сведения:

- Отображается ли код основания на мобильном устройстве работника во время инвентаризации.
- Код основания по умолчанию, который отображается в пункте меню на мобильном устройстве.
- Может ли пользователь изменять код основания.

### <a name="set-up-reason-codes-on-a-mobile-device"></a>Настройка кодов оснований на мобильном устройстве

1. Выберите **Управление складом** \> **Настройка** \> **Мобильное устройство** \> **Пункты меню мобильного устройства**.
2. На вкладке **Цикличный подсчет** выберите **Цикличный подсчет**.
3. В поле **Код основания для проведения инвентаризации по умолчанию** задайте код основания по умолчанию, который должен быть записан, когда инвентаризация выполняется с помощью пункта меню мобильного устройства.
4. В поле **Показать код основания для проведения инвентаризации** выберите **Строка** для отображения кода основания после записи каждого отклонения. Вместо этого можно выбрать **Скрыть**, если код основания не должен отображаться.
5. Задайте для параметра **Изменить код основания для проведения инвентаризации** значение **Да** или **Нет**. Если для этого параметра задать значение **Да**, работник может изменить код основания, когда он будет отображаться на мобильном устройстве во время инвентаризации.

> [!NOTE]
> Кнопку **Цикличный подсчет** можно включить на любом пункте меню мобильного устройства, на котором может выполняться инвентаризация. Примеры включают пункты меню для проверки наличия, работы по указанию пользователя или работы по указанию системы.

## <a name="cycle-count-approvals"></a>Утверждения цикличного подсчета

Перед утверждением инвентаризации пользователь может изменить код основания, связанный с инвентаризацией. После утверждения инвентаризации код основания вводится в строки журнала инвентаризации.

### <a name="modify-cycle-count-approvals"></a>Изменение утверждений цикличного подсчета

1. Выберите **Управление складом** \> **Цикличный подсчет** \> **Работа цикличного подсчета ожидает рассмотрения**.
2. Выберите **Цикличный подсчет**, а затем в поле **Код основания** выберите новый код основания.

### <a name="modify-the-mobile-device-menu-item-for-adjustment-in-and-adjustment-out"></a>Изменение пункта меню мобильного устройства для входящей корректировки и исходящей корректировки

1. Выберите **Управление складом** \> **Настройка** \> **Мобильное устройство** \> **Пункты меню мобильного устройства**, затем выберите **Входящая и исходящая корректировка**.
2. Задайте для параметра **Использовать существующую работу** значение **Нет**.
3. В поле **Процесс создания работы** выберите **Входящая корректировка**.

Следующие поля будут добавлены в пункт меню мобильного устройства, когда выбрана **Входящая корректировка** или **Исходящая корректировка** в процессе создания работы:

- Код основания для проведения инвентаризации по умолчанию
- Показать код основания для проведения инвентаризации
- Изменить код основания для проведения инвентаризации
