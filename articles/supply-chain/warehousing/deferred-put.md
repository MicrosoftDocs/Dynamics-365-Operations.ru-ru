# <a name="deferred-processing-of-warehouse-work"></a>Отложенная обработка работы склада

[!include [banner](../includes/banner.md)]

Эта тема описывает функциональность, которая делает отложенную обработку операций размещения для работы склада доступной в Dynamics 365 Supply Chain Management.

Функциональность отложенной обработки позволяет работникам склада продолжать выполнять другую работу, пока операция размещения обрабатывается в фоновом режиме. Отложенная обработка полезна, когда многие строки работ должны быть обработаны, и работник может позволить, чтобы эта работа обрабатывалась асинхронно. Это также полезно, когда сервер может иметь специальное или незапланированное увеличение времени обработки, и увеличение времени обработки может повлиять на производительность пользователя.

Фоновая обработка достигается с помощью платформы SysOperation. Для получения дополнительных сведений см. раздел [Обзор платформы SysOperation](/dynamicsax-2012/developer/sysoperation-framework-overview).

## <a name="configuring-the-work-processing-policies"></a>Настройка политик обработки работ

Чтобы использовать отложенную обработку, необходимо настроить и использовать политику обработки работ.

Политики настраиваются на странице **Политики обработки работ**. В следующей таблице описаны поля на этой странице.

| Поле                           | Описание |
|---------------------------------|-------------|
| Имя политики обработки работы     | Название политики обработки работ. |
| Тип заказа на выполнение работ                 | Тип заказа на работу, к которому применяется политика. |
| Операция                       | Операция, обрабатываемая с помощью политики. |
| Метод обработки работы          | Метод, используемый для обработки строки работ. Если метод настроен на **Немедленно**, поведение напоминает поведение, когда для обработки строки не используются никакие политики обработки работ. Если метод настроен на **Отложенная**, используется отложенная обработка с помощью пакетной платформы. |
| Порог отложенной обработки   | Значение **0** (ноль) показывает, что порог не существует. В этом случае используется отложенная обработка, если ее можно использовать. Если расчет конкретного порога ниже порога, используется метод "Немедленно". В противном случае используется метод отложенной обработки, если его можно использовать. Для работ, связанных с продажами и перемещениями, порог рассчитывается как количество связанных строк исходной нагрузки, которые обрабатываются для работы. Для работы по пополнению порог рассчитывается как количество строк работ, которые пополняются работой. Устанавливая порог, например, **5** для продаж, меньшие работы, которые имеют менее пяти начальных строк исходной загрузки, не будут использовать отложенную обработку, но более крупные работы будут использовать ее. Порог действует только в том случае, если метод обработки работы задан как **Отложенная**. |
| Группа партий отложенной обработки |Пакетная группа, используемая для обработки. |

Для отложенной обработки размещения поддерживаются следующие типы заказов на работу: заказ на продажу, выдача по заказу на перемещение и пополнение.

## <a name="assigning-the-work-creation-policy"></a>Назначение политики создания работ

Политика создания работ может быть назначена в параметрах управления складами. Она также может быть назначена на уровне отдельных складов. Если политика назначена складу, она применяется только к работе, созданной для этого склада. Если никакая политика не назначена на уровне склада, применяется политика из параметров управления складами.

## <a name="viewing-the-deferred-put-processing-tasks"></a>Просмотр задач отложенной обработки размещения

Вы можете просмотреть отложенные задачи обработки размещения на странице **Задачи отложенной обработки размещения**.

По умолчанию отображаются **Завершенные** задачи.

| Поле            | Описание |
|------------------|-------------|
| Состояние           | Статус задачи. |
| Статус пакетного задания | Статус соответствующего пакетного задания. Если пакетное задание обработано, история пакетов содержит журнал и информацию из пакетного задания. |

Ниже приведено объяснение возможных состояний:

- **В ожидании** — соответствующее пакетное задание ожидает обработки на пакетном сервере.
- **Сбой** — обработка не удалось. Задача может быть заново обработана с помощью действия **Обработать отложенное размещение** или может быть отменена с помощью действия **Отменить отложенное размещение**.
- **Завершено** — задание было завершено.

## <a name="impact-on-closed-work-dates"></a>Влияние на даты закрытия работ

При использовании отложенной обработки размещения дата закрытия работы устанавливается как дата и время создания задач отложенной обработки размещения. Дата закрытия работы используется, потому что это лучшая оценка того, когда операция размещения была завершена.

## <a name="reprocessing-a-failed-task"></a>Повторная обработка сбойной задачи

Можно повторно обработать невыполненную задачу, выбрав ее на странице, затем выбрав **Обработать отложенное размещение**. Повторная обработка соответствует ситуации, когда пользователь завершает откладывание с мобильного устройства, как если бы оно было настроено на немедленную обработку.

## <a name="canceling-failed-tasks"></a>Отмена сбойных задач

Можно отменить только сбойные задачи. При отмене задачи можно назначить ее новому пользователю. Кроме того, задача может оставаться назначенной пользователю, который обработал работу. Отмена соответствует ситуации, когда работа возвращается на мобильное устройство, как если бы последний шаг комплектации был только что завершен, и работа должна быть отложена. Тем не менее, пользователь сможет определить, что работа "другая", потому что она будет иметь только шаг размещения, и местоположение будет соответствовать местоположению, которое первый пользователь, который обработал работу, имел в качестве окончательного местоположения размещения.

При отмене задачи работа больше не блокируется по причине блокировки отложенной обработки размещения. Она может быть заново обработана с помощью мобильного устройства.

Запись задачи удаляется при отмене задачи.

## <a name="configuring-the-mobile-device-menu-to-skip-the-deferred-processing-policy"></a>Настройка меню мобильного устройства для пропуска политики отложенной обработки

Можно настроить пункт меню мобильного устройства таким образом, чтобы политика отложенной обработки не использовалась. Затем работа обрабатывается так, как если бы политика отложенной обработки не использовалась. Эта функция позволяет руководителю использовать определенный пункт меню, в котором отложенное размещение не используется. Чтобы настроить его, задайте в поле **Политика отложенной обработки размещения** значение **Переопределить настройки и обрабатывать размещение синхронно**. 

## <a name="restrictions-when-the-deferred-put-processing-isnt-applied"></a>Ограничения, когда отложенная обработка размещения не применяется

Существует несколько сценариев, в которых отложенная обработка размещения не применяется, даже если политика настроена. Далее приводятся некоторые примеры.

- Используется ручное завершение работы.
- Работы завершена с помощью автоматического завершения.
- Используются шаблоны аудита.


## <a name="monitoring-the-deferred-processing-tasks-from-the-outbound-work-monitoring-workspace"></a>Мониторинг задач отложенной обработки из рабочей области отслеживания исходящей работы

Рабочая область **Отслеживание исходящей работы** имеет две плитки, которые помогают контролировать задачи отложенной обработки:

- **Сбойные задачи отложенный обработки размещения** — эта плитка показывает количество сбойных задач. При наличии сбойных задач менеджер склада должен либо заново обработать их, либо отменить, поскольку они не будут повторно обрабатываться автоматически.
- **Ожидающие задачи отложенной обработки размещения** — эта плитка показывает количество задач, которые находились в статусе **Ожидание** более 10 минут. Если плитка показывает число, это может означать, что проблема возникла во время пакетной обработки. Вы можете вручную обработать задачи, находящиеся в состоянии **Ожидание**. Если пакетное задание для задачи будет обработано позже, оно просто закончится сбоем, поскольку оно уже обработано. Это не оказывает никакого негативного воздействия.

## <a name="deleting-completed-tasks"></a>Удаление завершенных задач

Можно удалить задачи отложенной обработки размещения, которые были завершены, выбрав их и удалив на странице.
