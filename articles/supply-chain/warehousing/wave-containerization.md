# <a name="containerization"></a>Контейнеризация

[!include [banner](../includes/banner.md)]

В этой статье описывается, как автоматизировать контейнеризацию загрузок. Автоматизированная контейнеризация создает контейнеры и работу комплектации для отгрузок при обработке волны.

Чтобы настроить контейнеризацию, создайте следующее.

- **Тип контейнера** ─ определение физических характеристик контейнеров. Используйте типы контейнеров для упаковки складируемых номенклатур в упаковку определенного типа и размера, например ящики или палеты.
- **Группы контейнеров** ─ создание групп аналогичных типов контейнеров. Например, группа контейнеров может включать типы контейнеров, которые имеют одинаковые аналитики размера. Группа контейнеров определяет последовательность, в которой упаковываются контейнеры, и процент заполнения каждого контейнера.
- **Шаблон создания контейнера** ─ создание шаблонов, определяющих правила для контейнеризации. Например, правила объединения запасов и другие стратегии упаковки.
- **Шаблоны волны** — настройка одного или нескольких шаблонов волны, чтобы создать работу комплектации для контейнеризации.

## <a name="create-wave-templates-for-containerization"></a>Создание шаблонов волны для контейнеризации

Необходимо настроить один или несколько шаблонов волны отгрузки для контейнеризации. Шаблоны волны включают критерии, определяющие следующее:

- Способ обработки волн. Обработку можно выполнять вручную или автоматически.
- Способ создания работы комплектации. Способ создания зависит от методов волны. Шаблон волны должен включать метод **контейнеризации**.
- Способ сопоставления номенклатур или строк распределения с волной.

Дополнительные сведения см. в разделе [Шаблоны волн](wave-templates.md).

## <a name="create-container-types"></a>Создание типов контейнеров

Используйте типы контейнеров для создания описаний контейнеров, включая максимальные значения для физических аналитик размера и грузоподъемности. При обработке контейнерной волны контейнеры создаются на основе сведений о типе контейнера.

Чтобы настроить тип контейнера, выполните следующие действия.

1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Контейнеры** \> **Типы контейнеров**.
1. Выберите **Создать** для создания нового типа контейнера.
1. Введите уникальный идентификатор и описание для типа контейнера.
1. В поле **Вес тары** введите фактический или расчетный вес контейнера.
1. В поле **Максимальный вес** введите максимальный вес, который может выдержать контейнер.
1. В полях **Максимальный объем для контейнеризации**, **Максимальная длина для контейнеризации**, **Максимальная ширина для контейнеризации** и **Максимальная высота для контейнеризации** укажите физические размеры контейнера.

    > [!NOTE]
    > Значения длины и ширины являются взаимозаменяемыми. Это означает, что номенклатуры можно поворачивать вбок или по оси Х, если это необходимо. Например, если длина составляет 2 метра, а ширина — 1 метр, можно изменить значение длины на 1 метр, а ширины — на 2 метра. Обратите внимание, что это не применяется к аналитике высоты. Невозможно изменить значение высоты, чтобы повернуть номенклатуру вертикально.

1. Выберите пользовательские значения атрибутов для контейнера в полях атрибутов. Атрибуты представляют собой пользовательские значения, используемые для фильтрации или сортировки номенклатур на основе значения, которое в противном случае будет недоступно. Например, если необходимо упаковать номенклатуры для определенного клиента, можно создать атрибут для имени клиента. Эти атрибуты создаются на странице **Атрибуты контейнера**.

## <a name="create-container-groups"></a>Создание групп контейнеров

Можно настроить логические группы типов контейнеров. Для каждой группы можно указать последовательность, в которой упаковываются контейнеры, и процент заполнения каждого контейнера. Аналитики размера номенклатуры определяют, поместится ли она в контейнер. Используется контейнер, размеры которого наиболее подходят к аналитикам размера номенклатуры. При наличии нескольких типов контейнеров в группе рекомендуется упорядочивать последовательность по размеру, чтобы самый большой контейнер был в последовательности первым, а самый маленький — последним.

Чтобы настроить группу контейнеров, выполните следующие действия:

1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Контейнеры** \> **Группы контейнеров**.
1. Выберите **Создать**, чтобы создать группу контейнеров.
1. Введите уникальный идентификатор и описание для группы контейнеров.
1. На экспресс-вкладке **Сведения** выберите **Создать**, чтобы добавить тип контейнера в группу.
1. В поле **Тип контейнера** выберите тип контейнера.
1. Выберите **Переместить вверх** или **Переместить вниз**, чтобы указать последовательность, в которой упаковываются типы контейнеров.

## <a name="create-container-build-templates"></a>Создание шаблонов сборки контейнеров

Можно настроить правила процесса контейнеризации, такие как правила объединения запасов и другие стратегии упаковки. Например, можно настроить правило, чтобы работники не могли упаковывать строки распределения, которые представляют заказы на продажу от разных клиентов, в один и тот же контейнер.

Чтобы настроить шаблон сборки контейнера, выполните следующие действия.

1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Контейнеры** \> **Шаблон создания контейнера**.
1. Создайте новый шаблон сборки контейнера.
1. В полях **Имя контейнеризации** и **Порядковый номер** введите имя шаблона контейнеризации и заказа, который соответствует строкам распределения.

    > [!NOTE]
    > Система применяет первый шаблон, критериям запроса которого соответствует строка распределения. Таким образом, рекомендуется помещать шаблон с самыми узкими критериями наверх списка. Чем шире критерии, тем больше вероятность того, что строка распределения будет им отвечать. Это может привести к тому, что строки будут назначены неправильному контейнеру. Если требуется, можно выбрать **Переместить вверх** или **Переместить вниз**, чтобы изменить последовательность шаблонов.

1. В поле **Код группы контейнеров** выберите группу контейнеров, из которой требуется создать контейнеры.
1. В поле **Типы базовых запросов** выберите тип запроса, который будет определять, что упаковывать и на чем основывать запрос фильтра. Имеются следующие варианты:

      - **Строка распределения продаж** — упаковка строк распределения, созданных для заказов на продажу.
      - **Строка распределения перемещений** — упаковка строк распределения, созданных для заказов на перемещение.
      - **Контейнер** — упаковка контейнера, созданного в процессе контейнеризации. Например, этот параметр используется для вложения контейнеров.

        > [!NOTE]
        > Для использования вложения контейнеров необходимо сделать метод контейнеризации повторяемым. Дополнительные сведения см. в разделе [Шаблоны волн](wave-templates.md).

1. На экспресс-вкладке **Общие** в поле **Код шага волны** введите уникальный идентификатор метода обработки волны, который связывает шаблон сборки контейнера с действиями в шаблоне волны.
1. Установите флажок **Разрешить разделение комплектации**, чтобы разрешить работникам упаковывать номенклатуры из заказа на выполнение работ в отдельные контейнеры. Для этого требуется, чтобы все количество вошло в контейнер. Всегда используется самая большая единица измерения в строке распределения.
1. В поле **Упаковка по единице измерения** выберите единицу измерения, представляющую контейнер. Например, можно указать, что контейнером является короб. При упаковке в соответствии с единицей измерения можно также указать группу контейнеров, поскольку единицей измерения является контейнер.
1. В поле **Стратегия упаковки контейнера** выберите стратегию упаковки для использования. Имеются следующие варианты:

    > [!NOTE]
    > Стратегия применяется только к строкам распределения продаж и строкам распределения перемещения.

      - **Упаковать во все открытые контейнеры** — система оценивает, соответствует ли строка распределения любому контейнеру, созданному во время цикла контейнеризации.
      - **Упаковать только в текущий контейнер** — система оценивает только то, соответствует ли строка распределения последнему созданному контейнеру.

    Для получения дополнительной информации и примеров, которые показывают, как работать со стратегиями упаковки контейнеров, см. [Стратегии упаковки в контейнеры](container-packing-strategy-overview.md).

1. Чтобы настроить правила для упаковки строк распределения в контейнеры, выберите **Перерывы логики смешивания**. Например, можно создать правило, которое позволит работникам упаковывать строки распределения для двух различных номенклатур в один и тот же контейнер. Чтобы определить правило объединения, выполните следующие действия.

    1. На странице **Перерывы логики смешивания** выберите **Создать**.
    1. В поле **Таблица** выберите таблицу, содержащую поле, которое будет использоваться в качестве критерия для перерыва по логике смешивания.
    1. В поле **Выбор поля** выберите поле, которое будет использоваться в качестве критерия для перерыва по логике смешивания.
    1. Повторяйте эти шаги до тех пор, пока не будут добавлены все критерии для перерыва по логике смешивания.

    > [!NOTE]
    > При использовании логики смешивания рекомендуется выполнить сортировку по одинаковым полями в запросе критериев фильтра. Это уменьшит количество создаваемых контейнеров.
