# <a name="configure-number-sequences-for-warehouse-flows"></a>Настройка номерных серий для складских потоков

[!include [banner](../includes/banner.md)]

Функция *Расширения номерных серий* добавляют новую страницу конфигурации для настройки номерных серий. Она позволяет гибко настроить GS1-регулируемые коды, включая префиксы GS1 и контрольные цифры (по модулю 10), и накладывает ограничение на длину существующих номерных серий.

Сегменты стандартной номерной серии не подходят для реализации GS1, так как контрольный разряд не рассчитывается, а префикс GS1 компании должен быть обновлен вручную.

Эта функция добавляет следующую функциональность:

- Коды транспортной накладной (BOL) могут создаваться заранее.
- Для порядковых номеров кодов контейнеров отгрузки (SSCC) может быть создана уникальная номерная серия.
- Для номеров BOL и SSCC могут быть созданы номерные серии, соответствующие GS1. Эта функция добавляет готовую поддержку для кодов грузомест, кодов контейнеров, кодов этикеток волны и кодов BOL.
- Настройка номеров кодов грузомест является гибкой. Например, можно включить или исключить идентификаторы приложений (ИИ), например, ведущие нули (00).

Эта функция позволяет более эффективно поддерживать маркировку упаковок и корректировать новые номера, создаваемые системой.

## <a name="turn-on-the-number-sequence-extensions-feature"></a>Включение функции расширений номерных серий

Прежде чем использовать эту функцию, она должна быть включена в системе. Администраторы могут использовать рабочую область [Управление функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения, если это требуется. В этом случае функция указана следующим образом:

- **Модуль:** *Управление складом*
- **Имя функции:** *Расширения номерной серии*

## <a name="set-up-the-feature"></a>Настройка функции

Чтобы настроить расширения номерной серии в системе, выполните следующие действия.

1. Перейдите в раздел **Управление складом \> Настройка \> Параметры управления складом**.
1. На вкладке **Общие** в поле **Префикс компании GS1** введите префикс компании GS1. Это значение повлияет на все номерные серии, для которых префикс GS1 включен в качестве сегмента.
1. Если необходимо создать номера BOL для этикеток волны, на вкладке **Отчеты** установите флажок **Генерировать номер BOL при печати этикеток волны**.

    > [!NOTE]
    > Этот флажок доступен только в том случае, если функция [печати этикеток волны](configure-wave-label-printing.md) включена.

1. Перейти к пункту **Расширения номерной серии** \> **Настройка** \> **Расширения номерной серии**
1. В панели операций выберите **Создать настройку по умолчанию**. Номерная серия BOL, соответствующая GS1, и три типа номерных серий SSCC создаются. Все эти номерные серии учитывают длину префикса компании GS1.

    Дополнительные сведения о настройке этих номерных серий по умолчанию и/или о добавлении новых последовательностей см. в следующем разделе. Кроме того, можно удалить любую из этих последовательностей, если они не нужны.

1. Вернитесь в раздел **Управление складом \> Настройка \> Параметры управления складом**.
1. На вкладке **Номерные серии** выберите соответствующее расширение номерной серии, которое будет использоваться для создания номеров для кодов грузомест, кодов этикеток волны, кодов контейнеров (в этом случае выберите соответствующую **номерную серию SSCC-18**) и/или коды BOL (в этом случае выберите последовательность **BOL**). По умолчанию расширения номерной серии поддерживаются только для этих четырех типов кодов.

При следующем создании нового номера для одной из этих номерных серий будет использоваться новая логика.

> [!NOTE]
> Если не выбрано расширение номерной серии для кода грузоместа, будут использоваться текущие правила создания кодов для грузомест. В противном случае будет использована выбранная номерная серия. Другие коды будут использовать обычную номерную серию, пока для них не будет применено расширение номерной серии.

## <a name="create-and-edit-number-sequences"></a>Создание и редактирование номерных серий

В предыдущем разделе был создан набор номерных серий по умолчанию. В этом разделе объясняется, как определяется каждая номерная серия. В нем также объясняется, как создавать настраиваемые последовательности и как изменять стандартные или настраиваемые последовательности.

Чтобы создать и изменить номерные серии, выполните следующие действия.

1. Перейти к пункту **Расширения номерной серии** \> **Настройка** \> **Расширения номерной серии**.
1. В области действий выберите **Создать**.
1. В поле **Расширение номерной серии** введите имя новой последовательности. В поле **Описание** введите описание.
1. На экспресс-вкладке **Сегменты** используйте кнопки на панели инструментов, чтобы собрать формат нумерации путем добавления, удаления и упорядочивания сегментов. В поле **Сегмент** для каждой строки назначьте тип сегмента, чтобы определить назначение и содержание этого сегмента. В следующей таблице описываются типы сегментов, которые доступны.

    | Тип сегмента | описание |
    |---|---|
    | Константа | Этот тип сегмента добавляет один и тот же постоянный текст для каждого создаваемого номера в последовательности. В поле **Значение** введите требуемый текст. Поле **Длина** автоматически обновляется до длины текста, введенного в поле **Значение**. |
    | Номерная серия | В поле **Значение** введите знак решетки (*\#*) для каждого символа, который должен отображаться в созданной последовательности. Сама номерная серия может генерировать более длинные номера, но будут отображаться только самые правые символы. Поле **Длина** автоматически обновляется до количество знаков чисел, введенных в поле **Значение**.<p>Чтобы обеспечить соответствие требованиям GS1 для номеров SSCC-18, убедитесь, что длина этого сегмента равна 16 знакам минус длина префикса GS1.</p> |
    | Префикс GS1 | Этот тип сегмента добавляет значение, заданное в поле **Префикс компании GS1** на странице **Параметры управления складом**. В поле **Значение** отображается значение, заданное на странице **Параметры управления складом**, а в поле **Длина** отображается число символов в значении. Как поле **Значение**, так и поле **Длина** доступны только для чтения. |
    | Идентификатор приложения | В поле **Значение** введите код приложения, как указано соответствующей политикой GS1 для данного типа номерной серии. Например, введите *00* для SSCC или *420* для BOL. Поле **Длина** автоматически обновляется до длины кода, введенного в поле **Значение**. |
    | Тип упаковки | Для номенклатур, которые могут быть ясно идентифицированы, этот тип сегмента добавляет значение поля из соответствующей группы последовательностей единиц измерения (со страницы **Группы упорядочения единиц**). (Это поведение соответствует существующей логике для кодов грузомест.) Для грузомест, содержащих несколько единиц складского учета (SKU), данный тип сегмента добавляет *0* (нуль) по умолчанию. Для этого типа сегмента поле **Значение** всегда устанавливается равным *P*, а поле **Длина** всегда устанавливается равным *1*.|
    | Контрольный разряд | Этот тип сегмента добавляет контрольный разряд, который является расчетом по модулю 10. (Это поведение соответствует существующей логике для кодов грузомест.) Для этого типа сегмента поле **Значение** всегда устанавливается равным каретке (*^*), а поле **Длина** всегда устанавливается равным *1*. |

1. Чтобы просмотреть пример окончательного формата номеров, проверьте поле **Формат** в нижней части экспресс-вкладки **Сегменты**.
