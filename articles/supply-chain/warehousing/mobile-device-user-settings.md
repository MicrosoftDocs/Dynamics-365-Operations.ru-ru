# <a name="mobile-device-user-settings"></a>Параметры пользователя мобильного устройства

[!include [banner](../../includes/banner.md)]

В новом мобильном приложении "Управление складом" есть набор параметров, относящихся к приложению, которые помогают адаптировать взаимодействие с пользователем. Так как приложение может использоваться на устройствах различных размеров и конфигураций экрана (например, планшеты, телефоны и др.), может быть полезно централизованно управлять этими параметрами в Microsoft Dynamics 365 Supply Chain Management.

Функция *настроек пользователя мобильного устройства* позволяет определить глобальные параметры пользователя, которые будут использоваться на всех устройствах. Кроме того, можно определить более детализированные пользовательские настройки для отдельных торговых марок устройств, моделей устройств и/или сотрудников. Когда работник выполняет вход в мобильное приложение "Управление складом", приложение получает и применяет наиболее подходящий профиль параметров, который хранится в Supply Chain Management для соответствующих торговой марки, устройства и/или ИД пользователя.

Эта функция помогает сотрудникам быстрее начать работу, когда они начинают использовать новое устройство. Далее приводятся некоторые примеры.

- Администраторы могут установить стандартный набор предпочтений, которые наилучшим образом подходят для устройств от конкретного производителя и/или для конкретной модели устройств. Таким образом, работники могут начать работу с новым устройством без изменения настроек.
- Если у вашей компании есть набор идентичных устройств, которые являются общими для работников, работники будут видеть предпочитаемую настройку каждый раз, когда они выполняют вход, даже если они никогда не использовали определенное физическое устройство, выбранное в данный день.
- В Supply Chain Management администраторы могут просматривать и редактировать все сохраненные настройки, даже для отдельных работников. Эта возможность может помочь в устранении неполадок или тонкой настройке новых функций.

> [!IMPORTANT]
> Функция *пользовательских настроек мобильного устройства* применяется только к новому мобильному приложению "Управление складом". Она не работает со старым приложением склада.

## <a name="turn-on-the-mobile-device-user-settings-feature"></a>Включите функцию пользовательских настроек мобильного устройства

Прежде чем использовать эту функцию, она должна быть включена в системе. Администраторы могут использовать параметры [управления компонентами](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление складом*
- **Название компонента:** *Параметры пользователя, значки и названия шагов для нового приложения склада*

## <a name="create-and-manage-user-settings"></a>Создание пользовательских настроек и управление ими

Страница **Пользовательские настройки мобильного устройства** используется для создания, просмотра и управления профилями настроек на всех уровнях детализации. При первом изменении пользователем своих пользовательских настроек на новом устройстве автоматически добавляется новый профиль на этой странице, если он еще не существует. Затем этот профиль обновляется каждый раз, когда работник вносит изменения.

Кроме того, можно определить профиль настроек для всех торговых марок устройств, моделей устройств и/или сотрудников. Затем можно увеличить детализацию, применяя более конкретные настройки для отдельных торговых марок, моделей и/или сотрудников.

Выполните следующие действия, чтобы создать пользовательские настройки для мобильных устройств и управлять ими.

1. Перейдите в раздел **Управление складом \> Мобильное устройство \> Пользовательские параметры мобильного устройства**.
1. Выберите существующий профиль пользовательских параметров в области списка, чтобы открыть его запись. Или выберите **Создать** в области действий, чтобы создать новый профиль.

    Каждый профиль в панели списка помечен для обозначения торговой марки, модели и/или ИД пользователя, к которым применяется данный профиль. У более общих профилей есть значение *Все* для некоторых или всех этих характеристик.

1. В разделе заголовков записи для нового или выбранного профиля настроек задайте следующие поля:

    - **Название торговой марки устройства** — выберите название торговой марки устройства, к которому должен применяться данный профиль. Если профиль должен применяться ко всем торговым маркам, оставьте это поле пустым. Список значений включает все торговые марки, которые были определены в системе. Сведения об определении списка торговых марок см. в следующем разделе.
    - **Код модели устройства** — выберите модель устройства, к которому должен применяться данный профиль. Если профиль должен применяться ко моделям выбранной торговой марки, оставьте это поле пустым. Список значений включает все модели, которые были определены для торговой марки, выбранной в поле **Название торговой марки устройства**. Сведения об определении списка моделей для каждой торговой марки см. в следующем разделе.
    - **ИД пользователя** — выберите ИД пользователя складского работника, к которому должен применяться этот профиль настройки. Если профиль должен применяться ко всем сотрудникам, оставьте это поле пустым.

1. На экспресс-вкладке **Общие** настройте следующие поля:

    - **Положение кнопки** — выберите способ расположения кнопок на устройстве. Элементы в приложении будут перемещены, чтобы лучше соответствовать предпочтению или удобству работника. Доступные параметры — *Нижний правый (по умолчанию)*, *Нижний левый*, *Верхний правый* и *Верхний левый*.
    - **Ориентация дисплея** — выберите ориентацию дисплея, которая должна применяться по умолчанию в приложении.
    - **Сканировать камерой** — установите этот параметр как *Да*, чтобы использовать камеру устройства для сканирования полей ввода, если предпочтительным режимом ввода является *Сканирование*. Если устройство оснащено встроенным сканером, установите для этого параметра значение *Нет*, чтобы использовать сканер.
    - **Показать фотографию продукта** — выберите, должны ли отображаться фотографии продуктов, если они доступны для выпущенного продукта. Дополнительные сведения о добавлении изображений продуктов см. в разделе [Добавление изображения к продукту](../pim/tasks/add-image-product.md).
    - **Цветовая тема дисплея** — выберите цветовую тему для устройства.
    - **Уровень громкости** — выберите уровень громкости для устройства. Выберите значение от 0 (ноль) до 10. Значение *0* означает "без звука", а значение *10* — максимальная громкость. Значение по умолчанию — *4*.
    - **Уровень вибрации** — выберите уровень вибрации для устройства. Выберите значение от 0 (ноль) до 5. Значение *0* означает "без вибрации", а значение *5* — максимальная вибрация. Значение по умолчанию — *1*.
    - **Процент масштабирования текста** — укажите размер текста в процентах от стандартного размера. Введите значение от 70 до 400. Значение *70* означает наименьший масштаб текста, а значение *400* — наибольший масштаб текста. Значение по умолчанию — *100*.
    - **Процент масштабирования кнопок** — укажите размер кнопок в процентах от стандартного размера. Введите значение от 50 до 200. Значение *50* означает наименьший масштаб кнопок, а значение *200* — наибольший масштаб кнопок. Значение по умолчанию — *100*.

## <a name="create-and-manage-mobile-device-brands"></a>Создание торговых марок мобильных устройств и управление ими

Страница **Торговые марки мобильного устройства** предназначена для просмотра, создания и управления торговыми марками и моделями устройств, которые могут использоваться с профилями настроек. Мобильное приложение автоматически получает и передает имя локальной торговой марки и код модели при первом изменении параметров на данном устройстве работником. Поэтому большая часть этих записей обычно будет создаваться автоматически. Однако можно также управлять всеми записями на этой странице. Например, можно присвоить более подробное описание каждой торговой марки и модели, чтобы отличать аналогичные и криптографически закрытые коды моделей.

Выполните следующие действия, чтобы создать пользовательские торговые марки и модели мобильных устройств и управлять ими.

1. Перейдите в раздел **Управление складом \> Мобильное устройство \> Торговые марки мобильного устройства**.
1. В области списка выберите торговую марку мобильного устройства, чтобы открыть его запись. Или выберите **Создать** в области действий, чтобы создать новую торговую марку.
1. В разделе заголовков записи для новой или выбранной торговой марки устройства задайте следующие поля:

    - **Торговая марка устройства** — название торговой марки устройства, например, *корпорация Майкрософт*.
    - **Описание** — можно ввести описание, которое поможет различать названия торговой марки.

1. На экспресс-вкладке **Модели мобильных устройств** отображаются все модели для выбранной торговой марки устройства. Используйте кнопки на панели инструментов, чтобы добавить строки в сетку или удалить строки. Для каждой строки установите следующие поля:

    - **Код модели устройства** — код модели устройства, такой как *Surface Book 2*.
    - **Описание** — можно ввести описание, которое поможет различать коды моделей.
