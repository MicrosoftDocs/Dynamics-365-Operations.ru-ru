# <a name="auto-release-shipment-for-cross-docking"></a>Автоматический запуск отгрузки для кросс-докинга

[!include [banner](../includes/banner.md)]

В этом разделе описывается стратегия кросс-докинга, которая позволяет автоматически выпустить заказ спроса на склад, когда производственный заказ, предоставляющий количество спроса, считается завершенным. Таким образом, количество, необходимое для выполнения заказа на спрос, перемещается непосредственно из местонахождения производства в исходящее местоположение.

Кросс-докинг — это поток обработки склада, в котором количество, требуемое для выполнения исходящего заказа, направляется в дебаркадер отгрузки или промежуточный участок заказа из местоположения, в котором был получен входящий заказ. (Входящий заказ может быть заказом на покупку, заказом на перемещение или производственным заказом). В то время как расширенная функция кросс-докинга поддерживает все заказы на поставку и запросы, и требует, чтобы исходящий спрос был запущен до того, как будет определена возможность кросс-докинга, функция автозапуска по отгрузке имеет следующие характеристики:

- Она поддерживает только производственные заказы как поставки, а только заказы на продажу и заказы на перемещение в качестве спроса.
- Операция кросс-докинга может быть запущена даже в том случае, если заказ спроса не был передан на склад до регистрации квитанции о поставке (то есть до того, как производство было принято).

Функция кросс-докинга имеет два преимущества:

- Складские операции могут пропустить шаг помещения количества готовых изделий в обычное место складского хранения, если эти количества будут только снова отобраны для выполнения исходящего заказа. Вместо этого количества могут быть перемещены один раз из выходного местонахождения в местонахождение упаковки/отгрузки. Таким образом, функция позволяет свести к минимуму количество попыток обработки запаса и, в конечном счете, помогает максимизировать экономию времени и пространства в цехе для склада.
- Складские операции могут откладывать выпуск заказов на продажу и заказов на перемещение на склад до тех пор, пока не будет учтена приемка готовой продукции для связанного производственного заказа. Это преимущество могут быть особенно актуальным в производственных средах создания на заказ, где время производства может превышать время производства в средах производства на склад.

## <a name="prerequisites"></a>Необходимые условия

| Необходимые условия | Описание |
|---|---|
| Элемент | Для процессов управления складом должна быть включена номенклатура.<p>**Примечание:** в процессах кросс-докинга невозможно включить номенклатуры, учитываемые в двух единицах измерения.</p> |
| Склад | Для процессов управления складом должен быть включен склад. |
| Шаблоны кросс-докинга | Для данного склада должен быть настроен как минимум один шаблон кросс-докинга, который использует политику выпуска спроса **получение по поставке**. |
| Класс работы | Для типа заказа на работу **кросс-докинг** должен быть создан код класса работ кросс-докинга. |
| Шаблоны работ | Рабочие шаблоны типа заказа на работу **кросс-докинг** необходимы для создания работы комплектации и размещения кросс-докинга. |
| Директивы для мест хранения | Директивы местоположения типа заказа на работу **кросс-докинг** должны быть использованы для сопровождения работы размещения в местоположения, в которых будут упакованы и отгружены количества по заказу на продажу. |
| Маркировка между заказом на спрос и производственным заказом | Складская система может активировать автоматический запуск отгрузки исходящего заказа и создавать работу кросс-докинга из выходного местонахождения при действии "приемка", только если заказы на продажу и заказы на перемещение зарезервированы и отмечены в отношении производственного заказа. |

## <a name="example-cross-docking-flow"></a>Пример потока кросс-докинга

Типичный поток кросс-докинга состоит из следующих основных шагов.

1. Составитель заказа на продажу создает заказ на продажу для продукта производства под заказ. Обычно запрошенное количество не имеется в наличии и должно быть сначала произведено.
2. Составитель заказа на продажу создает производственный заказ непосредственно из строки заказа на продажу. Таким образом, составитель заказа на продажу создает резервирует и помечает количество по заказу на продажу с учетом количества по производственному заказу. 

    В качестве альтернативы, планировщик производства указывает, что маркировку необходимо обновить при утверждении спланированных заказов.

3. Производственный заказ проходит через следующие шаги:

    1. Планировщик производства оценивает и отпускает производственный заказ. (Оценка включает резервирование сырья, а выпуск включает выпуск на склад).
    2. Работник склада начинает и завершает комплектацию сырья из расположения хранилища в местонахождении поступлений на производство в соответствии с работой комплектации производства.
    3. Оператор цеха начинает производственный заказ.
    4. В последней операции оператор машины использует мобильное устройство для сообщения о завершении производственного заказа.

4. Система использует настройку для идентификации события кросс-докинга для двух связанных заказов, а затем выполняет следующие задачи:

    1. Автоматический выпуск связанного заказа на продажу на склад, чтобы создать отгрузку.
    2. Автоматическое создание работы кросс-докинга, в котором представлены инструкции по комплектации готовой продукции из выходного местонахождения и ее перемещению в исходящее местоположение, которое назначено директивами местонахождения кросс-докинга для заказа на продажу.
    3. Предложите оператору машины завершить работу кросс-докинга сразу же после приемки производственного заказа.

5. После завершения работы по кросс-докингу и загрузки количеств на транспортные средства планировщик склада отгрузки подтверждает отгрузку продаж.

## <a name="example-scenario"></a>Пример сценария

Для этого сценария необходимо установить демонстрационные данные, а также необходимо использовать компанию **USMF** с демонстрационными данными.

### <a name="set-up-cross-docking-that-uses-the-auto-release-shipment-feature"></a>Настройка кросс-докинга, в котором используется функция автозапуска по отгрузке

#### <a name="cross-docking-template"></a>Шаблон кросс-докинга

1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Работа** \> **Шаблоны кросс-докинга**.
2. Выберите **Создать**.
3. В поле **Порядковый номер** введите значение **1**.
4. В поле **код шаблона кросс-докинга** введите имя, например, **XDock\_RAF**.
5. В поле **политика выпуска спроса** выберите **получение по поставке**.
6. В поле **склад** введите номер склада, на котором требуется настроить процесс кросс-докинга. В этом примере выберите **51**.

    > [!NOTE]
    > После выбора **получение по поставке** в качестве политики выпуска спроса для шаблона все остальные поля на странице становятся недоступными. Аналогичным образом невозможно определить источники поставок. Это происходит потому, что кросс-докинг, использующий функцию автоматического выпуска отгрузки, поддерживает только производственные заказы в качестве источников поставок, и для этого требуется наличие маркировки между заказами на продажу и производственными заказами. Если выбрать **До поступления поставки** в качестве политики выпуска спроса, поля в таблицах **Планирование** и **источники поставок** доступны и могут быть изменены.

#### <a name="work-classes"></a>Классы работы

1. Перейдите к **Управление складом** \> **Настройка** \> **Работа** \> **Классы работы**.
2. Выберите **Создать**.
3. В поле **код класса работы** введите имя, например **CrossDock**.
4. В поле **Тип заказа на работу** выберите **кросс-докинг**.

Чтобы ограничить типы местоположений, в которые могут быть размещены готовые товары с кросс-докингом, можно указать один или несколько допустимых типов местоположений.

#### <a name="work-templates"></a>Шаблоны работ

1. Перейдите к **Управление складом** \> **Настройка** \> **Работа** \> **Шаблоны работ**.
2. В поле **Тип заказа на работу** выберите **кросс-докинг**.
3. Выберите **Создать**.
4. В поле **Порядковый номер** введите значение **1**.
5. В поле код **шаблон работы** введите имя, например **CrossDock\_51**.
6. Нажмите **Сохранить**.
7. В разделе **Сведения шаблона работы** выберите **Создать**.
8. Для новой строки, в поле **Тип работы** выберите **Комплектовать**. В поле **Код класса работы** введите **CrossDock**.
9. Выберите **Создать**.
10. Для новой строки, в поле **Тип работы** выберите **Разместить**. В поле **Код класса работы** введите **CrossDock**.

#### <a name="location-directives"></a>Директивы для мест хранения

Стандартный процесс размещения для готовой продукции требует использования директивы **Разместить** для сопровождения перемещения скомплектованных количеств продукции в обычное пространство хранения. Аналогичным образом, необходимо настроить директиву местоположения кросс-докинга **Разместить**, чтобы поручить работе поместить готовое количество в назначенное исходящее местоположение, которое обслуживает отгрузку связанного заказа на продажу.

Для кросс-докинга, как для обычного размещения готовой продукции, нет необходимости создавать директиву местоположения для действия работы по комплектации, потому что указывается выходное местоположение. Кроме того, ожидается, что это выходное местоположение будет настроено либо как выходное местоположение по умолчанию в одной из записей, относящихся к ресурсам (т.е. ресурсы, отношения группы ресурсов или группы ресурсов), либо в качестве местоположения готовой продукции производства по умолчанию для склада.

1. Перейдите к **Управление складом** \> **Настройка** \> **директивы местонахождений**.
2. В поле **Тип заказа на работу** выберите **кросс-докинг**.
3. Выберите **Создать**.
4. В поле **Порядковый номер** введите значение **1**.
5. В поле **имя** введите имя, например **Поднимающаяся дверь**.
6. В поле **Тип работы** выберите **Поместить**.
7. В поле **Сайт** выберите **5**.
8. В поле **Склад** выберите **51**.
9. На экспресс-вкладке **Строки** выберите **Создать**.
10. В поле **Количество "До"** введите максимальное количество диапазона, например **1000000**.
11. Нажмите **Сохранить**.
12. На экспресс-вкладке **Действия директив местоположения** выберите **создать**.
13. В поле **имя** введите имя, например **Поднимающаяся дверь**.
14. Нажмите **Сохранить**.
15. Можно использовать стандартное средство запроса, чтобы ограничить местонахождения комплектации одним или несколькими отдельными местонахождениями. Выберите **редактировать запрос**и выберите **51** в качестве критерия для поля **Склад** в таблице **Местоположения**.

### <a name="cross-dock-finished-goods-to-the-outbound-location"></a>Кросс-докинг готовой продукции в исходящее местоположение

Чтобы выполнить кросс-докинг количества готовой продукции в исходящее местоположение связанного заказа на продажу, выполните следующие действия.

1. Перейдите в раздел **Продажи и маркетинг** \> **Заказы на продажу** \> **Все заказы на продажу**.
2. Выберите **Создать**.
3. В заголовке заказа на продажу выберите счет клиента **US-001** и склад, настроенный для кросс-докинга, в котором используется функция автозапуска по отгрузке.
4. Добавьте строку для готовой продукции и введите **10** в качестве количества.
5. В разделе **строки заказа на продажу**, выберите **Продукт и поставки** \> **производственный заказ**.
6. В диалоговом окне **Создание производственного заказа** просмотрите значения по умолчанию, а затем выберите **создать**. Новый производственный заказ создается и связывается с заказом на продажу (то есть он зарезервирован и помечен).
7. Необязательно: измените значение в поле **количество** так, чтобы оно было больше чем значение, которое требуется для выполнения заказа на продажу. Когда количество продукции принято, система создает работу кросс-докинга для помеченного количества и работу размещения для оставшегося количества в соответствии с обычной процедурой обработки размещения готовой продукции.

    Как упоминалось ранее, между заказом на продажу и производственным заказом должна существовать маркировка. В противном случае кросс-докинг не работает. Маркировка может быть создана несколькими способами:

    - Система может автоматически создать маркировку в следующих ситуациях:

        - Производственный заказ вручную создается непосредственно из строки заказа на продажу (см. шаг 6).
        - Спланированные производственные заказы утверждаются, а в поле **Обновить маркировку** установлено значение **стандартное**.

    - Пользователь может вручную создать маркировку, открыв страницу **маркировка** из строки заказа на спрос.

    > [!NOTE]
    > Маркировка не может быть создана вручную для номенклатур, которые настроены для использования в качестве складских моделей стандартного и средневзвешенного значения в качестве складских моделей.

8. На странице **производственный заказ** в области действий на вкладке **производственный заказ** в группе **процесс** выберите **Оценка**, а затем нажмите кнопку **ОК**. Этот заказ оценен, а количество сырья резервируется для производства.
9. в области действий на вкладке **производственный заказ** в группе **процесс** выберите **Оценка**, а затем нажмите кнопку **ОК**. Работа комплектации по складу создается для сырья.
10. Откройте и проверьте работу. в области действий на вкладке **Склад** в группе **Общие** выберите **Сведения о работе**. Запишите код работы.
11. Откройте и войдите в приложение Dynamics 365 for Finance and Operations — Warehousing, чтобы запустить работу на складе 51.
12. Перейдите к **Production** \> **Комплектация для производства**.
13. Ввод кода работы для начала и завершения комплектации сырья. 

    После того, как работа принята, количество сырья доступно в местонахождении поступлений на производство (**005** в демонстрационных данных USMF), и выполнение производственного заказа может быть начато.

14. На странице **производственный заказ** в области действий на вкладке **производственный заказ** в группе **процесс** выберите **Начать**, а затем нажмите кнопку **ОК**.
15. В приложении перейдите к **Производство** \> **приемка и размещение**.
16. В поле **код продукта** введите номер производственного заказа и другие обязательные сведения, а затем нажмите кнопку **ОК**.

Обратите внимание, что происходят следующие события:

- Выпуск на склад запускается для связанного заказа на продажу.
- На основе выпуска создается отгрузка и работа по кросс-докингу. Эта работа указывает оператору склада отобрать количества, необходимые для выполнения строки заказа на продажу, и поместить их в исходящее местоположение, указанное в директиве местонахождения кросс-докинга.
- Если количество в производственном заказе превышает количество, необходимое для заказа на продажу, создается обычная работа размещения. Эта работа указывает оператору склада отобрать количества готовой продукции, которая остается после кросс-докинга, и поместить их в обычное хранение в соответствии с директивой местоположения.
