# <a name="put-to-wall---put-to-store"></a>Поместить на стену — поместить в магазин

[!include [banner](../includes/banner.md)]

Функция *Поместить на стену — поместить в магазин* позволяет обрабатывать сценарии, в которых необходимо консолидировать продукт в промежуточной области предварительной упаковки, используя настраиваемые критерии. Так как эта функция позволяет комплектацию на одном целевом грузоместе и может использовать больше позиций размещения, чем комплектация кластера, компании, которые отгружают в магазины или обрабатывают небольшие номенклатуры, смогут получить преимущества от уменьшения времени комплектации.

Рабочий процесс для этой функциональности направляет отобранный продукт на место сортировки для распределения по различным типам контейнеров. Обычно каждое местоположение сортировки содержит несколько позиций сортировки. Каждая позиция сортировки назначается в соответствии с критерием, заданным бизнес-процессом. Обычно в качестве критерия используется конечная точка, отгрузка или загрузка. После прибытия продукта он распределяется по каждой позиции сортировки на основе количества, связанного с заказом, пунктом назначения, отгрузкой или загрузкой. Когда контейнер заполнен или завершен, он перемещается в промежуточное местоположение или место отгрузки для дальнейшей обработки, в зависимости от бизнес-процесса.

Эти функции склада также называются другими названиями, такими как световые подсказки и перерыв.

## <a name="turn-on-the-outbound-sorting-feature"></a>Включение функции исходящей сортировки

Прежде чем можно будет использовать функцию *Поместить на стену — поместить в магазин*, в системе должна быть включена функция *исходящей сортировки*. Администраторы могут использовать рабочую область [Управление функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) для проверки статуса функции и ее включения, если это требуется. В этом случае функция указана следующим образом:

- **Модуль:** *Управление складом*
- **Название компонента:** *Исходящая сортировка*

Функция *исходящей сортировки* может использоваться в сочетании с функцией *кода этапа волны для всей организации*, если она включена. Кроме того, необходимо включить эту функцию, если будут использоваться заранее определенные коды, настроенные в кодах этапов волн. В рабочей области **Управление функциями** эта функция перечисляется следующими способами:

- **Модуль:** *Управление складом*
- **Имя функции:** *Код этапа волны для всей организации*

## <a name="setup"></a>Настройка

Для этой демонстрации используются стандартные данные Contoso и склад *62*. Кроме того, будут использованы некоторые добавления, отмеченные позже.

### <a name="location-type"></a>Тип местоположения

1. Перейдите в раздел **Управление складом \> Настройка \> Склад \> Типы местонахождения**.
1. На панели операций выберите **Создать** для создания типа местонахождения для сортировки.
1. Задайте следующие значения:

    - **Тип местоположения:** *СОРТИРОВКА*
    - **Описание:** *Сортировка*

1. Нажмите **Сохранить**.

### <a name="warehouse-management-parameters"></a>Параметры управления складом

1. Перейдите в раздел **Управление складом \> Настройка \> Параметры управления складом**.
1. На вкладке **Общие** на экспресс-вкладке **Типы местонахождения** введите в поле **Тип местонахождения сортировки** значение *СОРТИРОВКА*.
1. Нажмите **Сохранить**.

### <a name="location-profile"></a>Профиль местонахождения

1. Перейдите в раздел **Управление складом \> Настройка \> Склад \> Профили местонахождения**.
1. На панели операций выберите **Создать** для создания профиля местонахождения для местонахождения сортировки.
1. В заголовке установите следующие значения:

    - **Код профиля местонахождения:** *Сортировка*
    - **Имя:** *Сортировка*

1. На экспресс-вкладке **Общие** установите следующие значения.

    - **Формат местонахождения:** *УПАКОВКА*
    - **Тип местоположения:** *СОРТИРОВКА*
    - **Использовать отслеживание грузомест:** *Да*
    - **Разрешить смешанные номенклатуры:** *Да*
    - **Разрешить смешанные статусы запасов:** *Да*

1. Нажмите **Сохранить**.

### <a name="locations"></a>Ячейки

1. Перейдите в раздел **Управление складом \> Настройка \> Склад \> Местонахождения**.
1. Снимите флажок **Создать контрольные разряды для местонахождения**.
1. На панели действий выберите **Создать**, затем задайте следующие значения:

    - **Склад:** *62*
    - **Местонахождение:** *Сортировка*
    - **Код профиля местонахождения:** *Сортировка*

1. Нажмите **Сохранить**.

### <a name="packing-profiles"></a>Профили упаковки

1. На панели действий выберите **Создать**, затем задайте следующие значения:

    - **Код профиля упаковки:** *Сортировка*
    - **Описание:** *Сортировка*
    - **Политика упаковки контейнеров:** оставьте это поле пустым.
    - **Режим кода контейнера:** *Авто*
    - **Тип контейнера:** *ПАЛЕТА 48X48*
    - **Автоматически создавать контейнер при закрытии контейнера:** оставьте это поле пустым.

1. Нажмите **Сохранить**.

### <a name="wave-step-codes"></a>Коды шагов волны

Если вы включили функцию *Код этапа волны для всей организации*, настройте следующий код.

1. Перейдите в раздел **Управление складом \> Настройка \> Волны \> Коды этапов волны**.
1. На панели действий выберите **Создать**, затем задайте следующие значения:

    - **Код этапа волны:** *Сортировка*
    - **Описание этапа волны:** *Сортировка*
    - **Тип этапа волны:** *Шаблон сортировки*

1. Нажмите **Сохранить**.

### <a name="outbound-sorting-template"></a>Шаблон исходящей сортировки

Шаблон сортировки определяет, создаются ли позиции сортировки, какие критерии используются и другие атрибуты процесса сортировки.

1. Выберите **Управление складом \> Настройка \> Упаковка \> Шаблон исходящей сортировки**.
1. На панели операций выберите **Создать** для создания шаблона сортировки.
1. В заголовке нового шаблона задайте следующие значения:

    - **Код шаблона исходящей сортировки:** *Сортировка волны*
    - **Описание:** *Сортировка волны*
    - **Тип шаблона сортировки:** *Спрос волны*

        Это поле определяет процесс, для которого используется шаблон сортировки. Доступны следующие значения:

        - **Спрос волны** — шаблон сортировки используется для процесса *Поместить на стену*. Этот тип шаблона используется для обработки запасов прямо из волны, минуя станцию упаковки. Этот тип можно использовать только в том случае, если метод обработки волны **сортировка** включен в шаблон волны.
        - **Контейнер** — шаблон сортировки используется для процесса *Создание палеты после упаковки*. Этот тип шаблона используется для обработки контейнеров, которые закрыты на станции упаковки и должны быть отсортированы на палеты.

    - **Склад:** *62*
    - **Местонахождение:** *Сортировка*

1. На экспресс-вкладке **Общие** установите следующие значения.

    - **Проверка сортировки:** *Скан позиции*

        Это поле определяет проверку, требуемую при сортировке. Доступны следующие значения:

        - Не допускается
        - Скан грузоместа
        - Скан позиции

    - **Создание работы при закрытии позиции:** *Да*

        Если для этого параметра задано значение *Да*, при закрытии позиции будет создаваться работа для перемещения запасов в конечное местоположение отгрузки. Если для него задано значение *Нет*, запасы будут немедленно скомплектованы в заказ при закрытии позиции.

    - **Назначение позиции:** *Вручную*

        Это поле определяет тип назначения позиции. Доступны следующие значения:

        - **Вручную** — пользователь должен всегда указывать, в какую позицию запасов должна быть выполнена сортировка.
        - **Автоматически** — система автоматически направляет запасы в позицию, когда это возможно, на основе перерывов шаблона сортировки.

    - **Критерии назначения позиции сортировки:** *Использовать только пустую позицию*

        Это поле определяет, будут ли запасы, уже имеющиеся в позициях сортировки, учитываться при назначении позиции для спроса. Доступны следующие значения:

        - **Использовать только пустую позицию** — позиции, для которых уже имеются связанные с ними запасы, будут учтены
        - **Считать позицию пустой** — все запасы, уже находящиеся в этой позиции, не будут учитываться во время назначения. Будут использованы все доступные позиции.

    - **Код этапа волны:** *Сортировка*

        Если функция *Код этапа волны для всей организации* включена, код этапа волны *Сортировка* должен быть также настроен в кодах этапов волны.

    - **Автоматическое закрытие позиции сортировки:** *Да*

        Если для этого параметра задано значение *Да*, позиция сортировки автоматически закрывается после завершения всей работы, идущей в эту позицию.

    - **Число позиций сортировки:** *3*

        В этом поле определяется максимальное число позиций сортировки, которое будет создавать система.

    - **Префикс позиции сортировки:** *SP-*

        Это поле определяет префикс, который система назначает перед номером позиции.

    - **Автоматическая упаковка позиции сортировки:** *Да*

        Если для этого параметра задано значение *Да*, запасы на позиции сортировки будут упакованы в контейнер, когда позиция будет закрыта.

    - **Код профиля упаковки:** *Сортировка*

        Это поле определяет профиль упаковки, который будет использоваться при упаковке позиции сортировки в контейнер.

1. В области действий выберите **Изменить запрос**, чтобы указать критерии, которые используются для этого шаблона сортировки.
1. В диалоговом окне запрос на вкладке **Сортировка** выберите **Создать**, чтобы добавить строку, затем установите следующие значения:

    - **Таблица:** *Сведения о загрузке*
    - **Производная таблица:** *Сведения о загрузке*
    - **Поле:** *Код отгрузки*
    - **Направление поиска:** *Восходящее*

1. Нажмите **ОК**.
1. Может появиться следующее сообщение: "Группирование будет сброшено, продолжить?" Выберите **Да**.

    Становится доступной кнопка **Разбиения шаблона исходящей сортировки** на панели операций.

1. На панели операций выберите **Разбиения шаблона исходящей сортировки**.
1. Установите флажок **Поле для группировки**, чтобы сгруппировать по коду отгрузки.

    Этот параметр создаст одну позицию сортировки для каждой отгрузки, которая является контейнером в волне.

1. Нажмите **ОК**.

### <a name="wave-process-methods"></a>Методы обработки волн

1. Перейдите в раздел **Управление складом \> Настройка \> Волны \> Методы обработки волн**.
1. В области операций выберите **Восстановить методы**.

    Метод **сортировка** добавляется в список доступных методов, и для него выбирается тип шаблона волны *Отгрузка*.

### <a name="wave-templates"></a>Шаблоны волны

Измените шаблон волны, который используется для сортировки спроса волны.

1. Перейдите в раздел **Управление складом \> Настройка \> Волны \> Шаблоны волны**.
1. В поле **Тип шаблона волн** выберите *Отгрузка*.
1. Выберите существующий шаблон **62 Отгрузка по умолчанию**.
1. На панели операций выберите **Правка**.
1. На экспресс-вкладке **Общие** сделайте следующие изменения:

    - Установите для параметра **Обрабатывать волну перед запуском на склад** значение *Нет*.
    - Установите для параметра **Назначить открытым волнам** значение *Да*.

1. На экспресс-вкладке **Методы** настройте метод **сортировка**:

    1. В сетке **Оставшиеся методы** выберите **сортировка**.
    2. Выберите кнопку со стрелкой вправо, чтобы переместить метод **сортировка** в сетку **Выбранные методы**.
    3. В сетке **Выбранные методы** выберите **сортировка**.
    4. Задайте в поле **Код этапа волны** значение *Сортировка*.

1. Нажмите **Сохранить**.

### <a name="location-directives"></a>Директивы для мест хранения

Необходимо создать директивы местоположения, чтобы направлять работу, созданную после завершения сортировки.

1. Перейдите в раздел **Управление складом \> Настройка \> Директивы местонахождения**.
1. В поле **Тип заказа на работу** выберите *Комплектация сортированных запасов*.
1. В области действий выберите **Создать**.
1. В заголовке установите следующие значения:

    - **Последовательность:** *1*
    - **Имя:** *Поместить к двери отсека*

1. На экспресс-вкладке **Директивы местоположения** установите следующие значения:

    - **Тип работы:** *Размещение*
    - **Сайт:** *6*
    - **Склад:** *62*
    - **Код директивы:** оставьте это поле пустым.
    - **Несколько SKU:** *Нет*

1. Выберите **Сохранить**, чтобы сделать доступной экспресс-вкладку **Строки**.
1. На экспресс-вкладке **Строки** выберите **Создать**, затем установите следующие значения. Примите для всех остальных полей значения по умолчанию.

    - **Порядковый номер**: *1*
    - **Количество "От":** *0*
    - **Количество "До":** *1000000*

1. Выберите **Сохранить**, чтобы сделать доступной экспресс-вкладку **Действия директивы местоположения**.
1. На экспресс-вкладке **Действия директивы местонахождения** выберите **Создать**, затем установите следующие значения. Примите для всех остальных полей значения по умолчанию.

    - **Порядковый номер**: *1*
    - **Имя:** *Поднимающаяся дверь*

1. Выберите **Сохранить**, чтобы сделать доступной кнопку **Изменить запрос** на экспресс-вкладке **Действия директив местоположения**.
1. На экспресс-вкладке **Действия директивы местоположения** выберите **Изменить запрос**.
1. В диалоговом окне запросов на вкладке **Диапазон** найдите строку, в которой поле **Поле** имеет значение *Местонахождение*. Задайте в поле **Критерии** для этой строки значение *Дверь отсека*.
1. Выберите **ОК**, чтобы подтвердить изменение.

## <a name="example-scenario"></a>Пример сценария

Этот сценарий имитирует ситуацию, в которой на складе хранятся небольшие номенклатуры в ячейках, и их необходимо упаковать в ящики до того, как они отгружаются, и когда функциональность станции упаковки не является действительно подходящей. Работники могут скомплектовать заказы для нескольких клиентов и разных адресов одновременно, чтобы увеличить скорость комплектации. После завершения комплектации работники прибывают на место сортировки, где подобранные номенклатуры могут быть отсортированы по соответствующим коробкам на основе требуемых критериев. В этом примере код отгрузки будет использоваться для определения нужной коробки, поскольку каждая отгрузка имеет свой адрес. После того, как все номенклатуры из загрузки упаковываются и сортируются по отгрузке, коробки будут перемещены в промежуточную область и в конечном счете загружаются на грузовик.

Прежде чем начать создание сценария, убедитесь, что все стандартные функции склада правильно настроены для склада. Для этой цели используется стандартный склад Contoso *62*. Стандартные конфигурации не были изменены, кроме описанных в настройке.

### <a name="sortingput-to-wall"></a>Сортировка/размещение на стене

После того, как все запасы будут помещены в местонахождение сортировки, они должны быть отсортированы в правильной позиции сортировки.

1. Войдите в мобильное приложение.
1. В главном меню выберите **Исходящее**.
1. В меню **Исходящие** выберите **Сортировка** для начала сортировки номенклатур.
1. В поле **Грузоместо/контейнер** введите целевое грузоместо для работы скомплектованного заказа на продажу.
1. Подтвердите ввод.
1. Ввод код номенклатуры для сортировки в первую очередь.
1. Система определяет первую позицию сортировки, которая должна отображаться. Подтвердите позицию сортировки.
1. Вам будет предложено назначить грузоместо для позиции сортировки. Выберите поле **Грузоместо**, введите номер грузоместа, затем подтвердите ввод.

    Поскольку позиция сортировки связана с кодом отгрузки, отобранные номенклатуры будут отсортированы в грузоместо, относящееся к исходящей отгрузке и заказу на продажу.

    На следующей странице отображается код номенклатуры, количество, код позиции сортировки и коды грузоместа "из" (комплектация) и "в" (сортировка). Сведения на этой странице дают указание отсортировать указанную номенклатуру и количества из грузоместа комплектации в грузоместо сортировки.

1. Подтвердите позицию сортировки.
1. Отсортируйте номенклатуры в первой позиции сортировки. После завершения процедуры система направляет вас к следующей позиции сортировки.
1. Повторите эту процедуру для всех скомплектованных строк в заказе на работу. Этот процесс следует использовать также при наличии нескольких строк комплектации, имеющих одинаковый код номенклатуры.

    При повторном выполнении этого процесса для всех номенклатур система оценивает критерии из следующей отсканированной номенклатуры (строка работы) и определяет, в какую позицию сортировки ее следует поместить. Вы автоматически направляетесь, чтобы поместить номенклатуру в нужную позицию сортировки. В зависимости от настройки подтверждения вы также направляетесь для подтверждение этого действия, вводя номер позиции или код грузоместа.

    > [!NOTE]
    > Если автоматическая сортировка включена, переопределение вручную недоступно.

1. После завершения в Microsoft Dynamics 365 Supply Chain Management откройте страницу **Назначение позиций исходящей сортировки** для просмотра статуса позиций.

    - Если позиции закрываются автоматически, выберите **Показать закрытые**, чтобы отобразить закрытые позиции.
    - Обратите внимание, что проводки позиции сортировки отображаются. Отображаются номенклатура и количество, которые были обработаны по данной позиции.

    При настройке шаблона исходящей сортировки вы задали для параметра **Автоматическое закрытие позиции сортировки** значение *Да*. Таким образом, позиция автоматически закрывается после того, как в нее помещаются последние ожидаемые запасы. Позиции сортировки имеют статус **Закрыто**, и работа была создана для перемещения отсортированных складских запасов в местонахождение *Дверь отсека*.

1. Выполните работу комплектации сортированных запасов, чтобы переместить запасы в место отгрузки. Когда запасы готовы, подтвердите их для отгрузки.

> [!NOTE]
> Для правильной обработки работы комплектации сортированных запасов пункт меню мобильного устройства с кодом класса работы, в котором в поле **Тип заказа на работу** настроено значение *Комплектация сортированных запасов*, следует использовать для процесса перемещения и загрузки.

### <a name="manually-close-a-position-optional"></a>Закрытие позиции вручную (необязательно)

Если позиции сортировки должны быть закрыты вручную, параметр **Автоматическое закрытие позиции сортировки** для шаблона исходящей сортировки должен иметь значение *Нет*, и закрытие должно быть выполнено до того, как запасы могут быть перемещены в область двери отсека. Позиции могут быть закрыты различными способами:

- Через мобильное приложение управления складом:

    - Пользователь может отсканировать одну из номенклатур, уже расположенных в данной позиции, затем нажать кнопку **Закрыть**, чтобы закрыть эту позицию.
    - Если пользователь сканирует контейнер, который уже отсортированный контейнер, выводится сообщение об ошибке. Однако пользователь все равно может продолжать закрытие этой позиции.

- На странице **Назначения позиций исходящей сортировки** в Microsoft Dynamics 365 Supply Chain Management:

    - Пользователь может выбрать запись позиции исходящей сортировки, затем выбрать **Закрыть позицию** на панели операций.

## <a name="tips"></a>Советы

- Номенклатуры не могут быть перемещены между позициями после того, как они были назначены для одной позиции. Система оценивает количество каждой номенклатуры, которое должно попасть в определенную позицию.
- Шаблон сортировки можно настроить для автоматического создания контейнеров, когда позиции закрываются. Этот подход создаст структуру кода стандартного контейнера, которая базируется на указанном профиле упаковки.

> [!IMPORTANT]
> После создания работы по перемещению из местоположения сортировки вы не должны отменять работу. В противном случае позиция и контейнеры в ней будут удалены из системы и недоступны для дальнейшей обработки. Запасы также будут удалены.
