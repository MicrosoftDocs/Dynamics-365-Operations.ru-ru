# <a name="manual-update-of-asset-counters"></a>Ручное обновление счетчиков основных средств

[!include [banner](../../includes/banner.md)]



Счетчики используются для создания регистраций для актива, например регистрации количества часов, в течение которых актив был в эксплуатации, или количества, которое было произведено.

Для типа счетчика, выбранного для счетчика, можно настроить наследование значений счетчиков. Другими словами, параметр **Наследовать значения счетчика актива** имеет значение **да** на экспресс-вкладке **общие** на странице **счетчики** (**Управление активами** > **Настройка** > **Типы активов** > **Счетчики**). В этом случае при создании новой строки счетчиков этого типа все дочерние активы, использующие один и тот же тип счетчика, автоматически обновляются.

На странице **Все активы** создаются регистрации счетчика времени или количества для актива на основе показаний для актива.

1. Выберите **Управление активами** > **Общие** > **Активы** > **Все активы**.

2. Выберите актив, затем в области действий на вкладке **Актив** в группе **Профилактическое** выберите **Счетчики**. Страница **Счетчики актива** отображает список всех предыдущих регистраций счетчиков, которые были сделаны для выбранного актива.

3. Выберите **Создать** для создания регистрации. Код актива автоматически вводится в поле **Актив**.

4. В поле **Счетчик** выберите соответствующий счетчик. Для выбора доступны только счетчики, соответствующие типу актива, выбранному в активе. Связанная единица измерения автоматически вводится в поле **Единица измерения**.

5. В поле **зарегистрировано** выберите дату и время для регистрации счетчика.

6. В поле **значение** введите номер с момента последней регистрации счетчика. В качестве альтернативы в поле **Общее значение** введите общее число счетчика.

Обратите внимание на следующие аспекты:

- При физической установке нового счетчика для актива необходимо зарегистрировать изменение в активе на странице **Счетчики актива**. Затем необходимо создать две строки регистрации с одинаковыми метками времени. Первая строка должна относиться к заменяемому счетчику. В строке, которая относится к новому счетчику, установите флажок **Сброс счетчика**. В поле **Итого** общее число счетчиков представляет собой сумму всех итоговых зарегистрированных значений для этого типа счетчика.

- Если установлен флажок **Сброс счетчика** для актива с использованием плана обслуживания с типом интервала "Один раз с..." или "После достижения...", счетчик по-прежнему активен в новой строке счетчика, так как создается отдельная строка счетчика, которая начинается с нового счетчика.

На приведенном ниже рисунке показан пример страницы **Счетчики актива**.

![Рисунок 1](media/11-work-orders.png)

На странице **Счетчики актива** (**Управление активами** > **Запросы** > **Активы** > **Счетчики актива**), можно выполнять регистрацию счетчика на нескольких активах одновременно, при необходимости.

>[!NOTE]
>Можно настроить диапазон для определения отклонений в регистрациях счетчиков вручную. Можно также указать тип сообщения, которое будет отображаться, если регистрации выходят за пределы определенного диапазона. Дополнительные сведения о настройке счетчиков см. в разделе [Счетчики](../setup-for-objects/counters.md).
