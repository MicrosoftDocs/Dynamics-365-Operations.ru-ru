# <a name="maintenance-downtime"></a>Простой из-за обслуживания

[!include [banner](../../includes/banner.md)]


Можно создавать регистрации времени простоя из-за обслуживания для актива, выбранного в заказе на работу. Эта возможность полезна, если необходимо зарегистрировать время простоя из-за обслуживания на одном или нескольких станках в области производства. Сначала вы создаете коды причин простоя из-за обслуживания, которые вы хотите использовать, например, **поломка** и **плановая остановка**. Этот шаг выполняется на странице **Коды причин простоя из-за обслуживания**. Далее можно создать регистрации простоя из-за обслуживания на странице **Простой из-за обслуживания** и добавить соответствующие коды причин простоя из-за обслуживания.

## <a name="create-maintenance-downtime-reason-codes"></a>Создание кодов причин простоя из-за обслуживания

1. Выберите **Управление активами** > **Настройка** > **Заказы на работу** > **Коды причин простоя из-за обслуживания**.

2. Выберите **Создать**.

3. В поле **код причины простоя при обслуживании** введите код причины простоя из-за обслуживания.

4. В поле **Имя** введите имя.

5. Установите флажок **Включить в КПЭ**, если код причины должен включаться в расчеты ключевых показателей эффективности (КПЭ) для актива. Обычно плановые остановки производства не должны включаться в расчеты ключевых показателей эффективности, поскольку они не влияют на ожидаемую производительность.

6. Нажмите **Сохранить**.

На рисунке ниже показан пример страницы **Коды причин простоя из-за обслуживания**.

![Рисунок 1](media/15-work-orders.png)

После создания кодов причин простоя из-за обслуживания, которые вы хотите использовать, можно создать регистрации простоя из-за обслуживании для заказов на работу и активов.


## <a name="create-maintenance-downtime-registrations"></a>Создание регистраций простоя из-за обслуживания

1. Щелкните **Управление активами** > **Общее** > **Заказы на работу** > **Все заказы на работу** или **Активные заказы на работу**.

2. Выберите заказ на работу, а затем в области действий на вкладке **Заказ на работу** в группе **Актив** выберите **Простой из-за обслуживания**.

3. Выберите **Создать**.

4. Определите интервал дат и времени для регистрации простоя из-за обслуживания в полях **От** и **До**.

>[!NOTE]
>При выходе из поля **До** длительность в часах автоматически вставляется в поле **Продолжительность**.

5. Выберите код причины в поле **Код причины простоя из-за обслуживания**.

6. Повторите шаги 3–5, чтобы добавить другие регистрации.

7. Нажмите **Сохранить**.

Иллюстрация ниже показывает пример регистрации простоя из-за обслуживания.

![Рисунок 2](media/16-work-orders.png)

Календарь, используемый для расчета регистрации простоя из-за обслуживания, зависит от выбора в настройке активов и параметрах. Если ресурс выбран для актива в поле **Ресурс** на экспресс-вкладке **основное средство** страницы **Все активы**, используется календарь, настроенный для связанной группы ресурсов, как показано на следующем рисунке.

![Рисунок 3](media/17-work-orders.png)

Если никакой ресурс не выбран для актива, используется стандартный календарь, выбранный на странице **Параметры управления активами**, как показано на следующем рисунке.

![Рисунок 4](media/18-work-orders.png)

Щелкните **Управление активами** > **Запросы** > **Простой из-за обслуживания**, чтобы просмотреть обзор всех регистраций простоя из-за обслуживания.

>[!NOTE]
>Все календари, используемые в модуле **Управление активами**, настраиваются в разделе **Администрирование организации** > **Настройка** > **Календари** > **Календари**.
