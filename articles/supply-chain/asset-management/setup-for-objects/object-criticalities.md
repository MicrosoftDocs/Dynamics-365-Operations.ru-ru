# <a name="asset-criticality-types"></a>Типы критичности активов

[!include [banner](../../includes/banner.md)]

 

В этой статье объясняются типы критичности актива в «Управлении активами». Критичность активов связана с активами и передается в заказы на работу. Она не может быть изменена в заказе на работу. Критичность активов используется для расчета критичности заказа на работу при планировании заказа на работу. Другими словами, она используется для расчета степени, в которой задание обслуживания по активу влияет на план производства и производительность в вашей компании. Для получения дополнительной информации о настройке, связанной с расчетом рейтинговых оценок для планирования заказа на работу, см. раздел [Параметры управления активами](../setup-for-objects/enterprise-asset-management-parameters.md).

Для настройки критичности сначала создаются типы критичности, которые следует использовать в настройке актива. Затем вы настраиваете критичности активов.

## <a name="set-up-criticality-types"></a>Настройка типов критичности

1. Выберите **Управление активами** \> **Настройка** \> **Активы** \> **Типы критичности**.
2. Выберите **Создать**, чтобы создать запись.
3. В поле **Критичность** введите число, указывавщее критичность.
4. В поле **Имя** введите имя типа критичности.
5. В поле **Коэффициент** введите коэффициент. Этот коэффициент используется при расчете планировании заказа на работу для определения записи критичности, которая должна быть использована. (Всегда используется запись с самым высоким коэффициентом). Этот параметр актуален, если, как показано на следующем рисунке, создаются строки критичности, которые имеют одинаковое значение критичности.

    ![Страница типов критичности.](media/23-setup-for-objects.png)

## <a name="set-up-asset-criticalities"></a>Настройка критичностей актива

1. Выберите **Управление активами** \> **Настройка** \> **Критичности активов**.
2. Выберите **Создать**, чтобы создать запись.
3. В зависимости от требуемого уровня детализации для критичности актива сделайте соответствующие выборы в полях **Функциональное местоположение**, **Тип актива**, **Производитель**, **Модель**, **Актив**, **Категория типа задания**, **Тип задания**, **Вариант типа задания**, и **Потребность в задании**.

    > [!NOTE]
    > При выборе критичности актива «Управление активами» проходит все записи критичности актива, чтобы проверить возможное соответствие. Она всегда проверяет наиболее конкретные комбинации в первую очередь. Другими словами, «Управление активами» сначала проверяет **Потребность в задании**. Если совпадение не найдено, оно проверяет **Вариант типа задания**. Если совпадение не найдено, оно проверяет **Тип задания** и т.д. Как вы можете видеть в макете страницы, это поведение означает, что, чтобы найти наиболее конкретную комбинацию, «Управление активами» проверяет каждую запись справа налево на совпадение. Если совпадение не найдено, используется запись "по умолчанию", которая не имеет выбора.

4. В поле **Критичность** выберите один из значений критичности, созданных в предыдущей процедуре.

### <a name="notes-about-criticality-setup"></a>Заметки о настройке критичности

- Если вы измените критичность актива в этой настройке после того, как вы уже использовали ее в заказе на работу, критичность в заказе на работу не обновляется соответствующим образом.
- Критичность в заказе на работу пересчитывается каждый раз при добавлении или удалении строки в заказ на работу или удаления из заказа на работу.
- Если заказ на работу содержит несколько заданий заказа на работу, наивысшая критичность, согласно полю **Коэффициент** на странице **Типы критичности**, всегда используется в заказе на работу.
- Как правило, критичность активв может меняться в течение определенного периода. На критичность может повлиять покупка нового оборудования, реконструкции и так далее. Рассмотрите возможность переоценки критичностей актива через регулярные промежутки времени (например, один раз в год или каждые два года), чтобы убедиться, что ваши определения критичности соответствуют текущей настройке производства.
