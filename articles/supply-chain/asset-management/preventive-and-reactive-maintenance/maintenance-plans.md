# <a name="maintenance-plans"></a>Планы обслуживания

[!include [banner](../../includes/banner.md)]

 

План обслуживания определяет, когда должно быть выполнено запланированное задание профилактического обслуживания для актива. Планы обслуживания могут быть связаны с активами, типами активов, функциональными местоположениями или типами функциональных местоположений, но сначала создаются планы обслуживания для использования в компании.

План обслуживания может иметь несколько строк плана обслуживания. Тип заданий обслуживания и интервалы указаны в строке плана обслуживания. Имеется два типа строк плана обслуживания:

- Время  
- Счетчик  

Строки плана обслуживания типа "Время" используются для периодического планового обслуживания, основанного на фиксированном интервале времени. Строки плана обслуживания с типом "Счетчик" используются для планового обслуживания или оперативного обслуживания на основе регистраций счетчика актива. План обслуживания может включать несколько строк плана обслуживания обоих типов.

>[!NOTE]
>Если значения счетчика не были зарегистрированы для типа счетчика в активе, строки плана обслуживания пропускаются.

Сначала создаются планы обслуживания, необходимые для выполнения заданий профилактического обслуживания, и выбираются типы активов, активы, типы функциональных местоположений и функциональные местоположения, которые должны быть связаны с каждым планом обслуживания. Впоследствии, если это необходимо, можно также добавить планы обслуживания в актив или функциональное местоположение, что делается на экспресс-вкладке **Все активы** > выберите актив > **Планы обслуживания актива** или на экспресс-вкладке **Все функциональные местоположения** > выберите функциональное местоположение > **Планы обслуживания**.

Если план обслуживания добавляется к типам активов или типам функциональных местоположений, это означает, что при создании новых активов или функциональных местоположений с данными типами активов или типами функционального местоположения актив или функциональное местоположение будут автоматически добавлены в план обслуживания. Датой начала связи с планом обслуживания будет текущая дата, которая может потребоваться корректировки.

## <a name="set-up-maintenance-plans"></a>Настройка планов обслуживания

В этом разделе описывается, как настроить строки плана обслуживания и приводятся примеры их использования.

1. Щелкните **Управление активами** > **Настройка** > **Профилактическое обслуживание** > **Планы обслуживания**.

2. Нажмите кнопку **Создать**, чтобы создать новую последовательность.

3. Вставьте идентификатор в поле **Последовательность обслуживания** и имя в поле **Имя**.

4. В поле **Дата плана** введите начальную дату, с которой планирование может быть выполнено в плане обслуживания. Обратите внимание, что строки плана обслуживания на основе времени могут иметь другие даты плана.

5. Выберите "Да" в переключателе **Активен**, чтобы активировать план обслуживания.

>[!NOTE]
>Если вы отключите план обслуживания, в расписании обслуживания не будут создаваться записи расписания при выполнении задания расписания плана обслуживания.

6. Поля **Допустимое количество дней до** и **Допустимое количество дней после** относятся к строкам плана обслуживания, в которых установлен флажок **Подавить перекрывающиеся задания обслуживания** (см. шаг 17). Поля "Допуск" используются для расширения интервала в днях, в котором, если несколько планов обслуживания перекрываются, наиболее полное/самое большое задание создается как строка графика обслуживания при планировании плана обслуживания, в то время как более частые перекрывающиеся задания пропускаются при планировании плана обслуживания. Вставьте количество дней в поле **Допустимое количество дней до**, например "2".

7. Если введено значение в поле **Допустимое количество дней до**, также вставьте число дней в поле **Допустимое количество дней после**, например "2".

>[!NOTE]
>Пример, описанный в этом и предыдущем шаге, означает, что если несколько строк плана обслуживания перекрываются, и флажок **Подавить перекрывающиеся задания обслуживания** установлен для одной или нескольких строк, период пропуска строк графика обслуживания увеличивается всего на пять дней (ожидаемая дата начала в строке графика обслуживания *и* два дня до *и* два дня после этой даты).

8. В полях в группе **Сведения** на экспресс-вкладке **Сведения** отображается число строк плана обслуживания, настроенных в плане обслуживания, а также количество активов и функциональных местоположений, связанных с планом обслуживания.

9. На экспресс-вкладке **Строки** щелкните **Добавить строку времени** или **Добавить строку счетчика актива** для создания новой строки плана обслуживания.

10. Введите описание строки в поле **Описание заказа на работу**. Описание переносится в соответствующие заказы на работу.

11. В поле **Тип задания обслуживания** выберите тип задания, с которым связана строка плана обслуживания.

12. В полях **Варианты типа заданий обслуживания** и **Специальность** выберите вариант и специальность, связанные с типом заданий обслуживания.

13. В полях **Время завершения, дней** и **Время завершения, часы** можно вставить ожидаемую конечную дату в днях или часах. Ожидаемая дата окончания вводится относительно ожидаемой даты начала, которая рассчитывается при создании строк графика обслуживания. Например, можно вставить "7" в поле **Время завершения, дней**, чтобы указать, что связанное задание должно быть завершено в течение недели с ожидаемой даты начала.

14. В поле **Тип интервала** выберите тип интервала, который должен использоваться в строке плана обслуживания, например, "Повторяется..." или "Однократно...". Описание отношений между типами интервала и типами строки см. в таблице [Обзор типов интервалов](## Interval types overview) ниже.

15. Поле **Период** относится только к типам строк, основанным на времени. Выберите тип периода, который имеет отношение к частоте периода.

16. В поле **Частота периода** введите, сколько раз данная строка должна использоваться для планирования заданий профилактического обслуживания. Пример: если была создана строка с типом "Счетчик", а счетчик является произведенным количеством, и в этом поле введено число "20000", новые строки последовательности обслуживания создаются в процессе планирования профилактического обслуживания каждый раз, когда ожидается выпуск еще 20 000 единиц номенклатуры.

17. Флажок **Подавить перекрывающиеся задания обслуживания** относится к типам строк на основе времени и на основе счетчиков. Установите флажок, чтобы удалить записи графика обслуживания, созданные в одну дату. Это имеет значение, если, например, была создана строка инспекции через 1 месяц, строка инспекции через 6 месяцев и строка инспекции через 1 год. Для проверки каждый 1 год требуется, чтобы была выполнена только эта инспекция, а не две другие инспекции, которые также попадают в этот промежуток времени. Чтобы правильно настроить этот пример, необходимо настроить строку инспекции через 1 год как первую строку, строку 6 месяцев как вторую строку, а строку 1 месяца как третью строку, и установить флажок **Подавить перекрывающиеся задания обслуживания** для строк 1 месяца и 6 месяцев. Таким образом будет обеспечено, что при достижении отметки 1 год инспекции за один месяц и шесть месяцев пропускаются, и строка графика обслуживания создается только для строки инспекции за 1 год.

>[!NOTE]
>Пример, описанный на этом шаге, показывает, что наиболее полное задание, содержащее наибольшее число задач и выполняющееся не столь часто, всегда должно быть вставлено как первая строка. Более часто выполняемые задания затем вставляются в виде отдельных строк в порядке частоты, помещая самое частое задание в низ списка.

18. Поле **Счетчик** относится только к типам строк, основанным на счетчике. Выберите тип счетчика для использования в строке. Если тип счетчика не активен для соответствующего актива, строка плана обслуживания пропускается.

19. Поле **Временная граница счетчика актива в днях** относится только к типам строк, основанным на счетчиках. Вставьте число, определяющее, на сколько дней назад выполняется проверка регистраций счетчика при планировании плана обслуживания. Это означает, насколько далеко назад данные (существующие регистрации счетчиков) используются в качестве основы для расчета тенденции, которая определяет, сколько строк графика обслуживания создается.

>*Пример.* Если регистрации счетчиков предполагается выполнять один раз в месяц, в это поле можно вставить число "365", поскольку планирование плана обслуживания всегда будет основываться на последних 12 месяцах и, следовательно, создаются строки графика обслуживания на основе тенденции прошлого года. С другой стороны, если ввести в это поле число "10", ожидается, что регистрации счетчиков будет выполняться более часто, например, ежедневно. Это означает, что при планировании плана обслуживания регистрации счетчиков за последние 10 дней используются в качестве основы для планирования строк графика обслуживания.

20. Поле **Дата плана** относится только к типам строк, основанным на времени. Если строка плана обслуживания имеет другую дату планирования, чем весь план обслуживания, выберите дату в поле **Дата плана** в строке.

21. В поле **Уровень обслуживания** можно выбрать уровень обслуживания заказов на работу в качестве дополнительного разграничения по строке плана обслуживания, которое будет использоваться как уровень обслуживания в заказах на работу.

22. Установите флажок **Автосоздание**, если требуется автоматически создавать заказ на работу в соответствии с выбранной строкой плана обслуживания при планировании планов обслуживания.

23. Если установлен флажок **Автосоздание**, можно выбрать тип заказа на работу для автоматически создаваемого заказа на работу в поле **Тип заказа на работу**. Если установлен флажок **Автосоздание** и в этом поле не выбран тип заказа на работу, то используется тип заказа на работу, выбранный в поле **Управление активами** > **Настройка** > **Параметры управления активами** > ссылка **Заказы на работу** > **Тип профилактического заказа на работу**.

24. Поля **Сезон с** и **Сезон до** используются для создания повторяющейся строки плана обслуживания на основе времени в течение 12-месячного периода. *Пример.* Оборудование, используемое для содержания зеленых насаждений, требует обслуживания каждую весну в заранее определенном периоде. Вставьте дату начала периода, который должен повторяться, в поле **Сезон с**.

25. Вставьте дату окончания периода, который должен повторяться, в поле **Сезон до**.

26. В поле **Получающийся период** отображается текущий период, который необходимо повторить. После прохождения текущего периода и начала нового года период, показанный в этом поле, будет обновлен для отображения следующего периода в последовательности повторов.

27. На экспресс-вкладке **Активы** выберите активы, которые должны быть связаны с планом обслуживания.

28. На экспресс-вкладке **Типы активов** выберите типы активов, которые должны быть связаны с планом обслуживания.

29. На экспресс-вкладке **Функциональные местоположения** выберите функциональные местоположения, которые должны быть связаны с планом обслуживания. Если необходимо, можно сделать настройку более подробной, выбрав соответствующий тип актива, производителя и модель.

30. На экспресс-вкладке **Типы функциональных местоположений** выберите типы функциональных местоположений, которые должны быть связаны с планом обслуживания.

>[!NOTE]
>Когда заказы на работу создаются вручную по активам, которые покрываются гарантией поставщика, отображается диалоговое окно, чтобы пользователь знал о гарантии. После этого создание заказа на работу может быть отменено. Проверка гарантийных отношений пропускается для автоматически создаваемых заказов на работу.

## <a name="add-a-maintenance-plan-to-an-asset"></a>Добавление плана обслуживания в актив

1. Нажмите **Управление активами** > **Общее** > **Активы** > **Все активы** или **Активные активы**.

2. Выберите актив, для которого необходимо настроить план обслуживания, и щелкните **Изменить**.

3. На экспресс-вкладке **Планы обслуживания актива** выберите **Добавить строку**, чтобы добавить план обслуживания в актив.

4. В поле **План обслуживания** выберите соответствующий план обслуживания.

5. В поле **Дата начала** выберите дату, с которой может быть выполнено планирование заданий профилактического обслуживания. 

6. Нажмите кнопку **Сохранить**. Поле **Активен** обновляется автоматически.
