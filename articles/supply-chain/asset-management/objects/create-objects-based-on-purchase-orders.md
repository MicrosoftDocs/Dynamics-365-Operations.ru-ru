# <a name="create-assets-based-on-purchase-orders"></a>Создание активов на основе заказов на покупку

[!include [banner](../../includes/banner.md)]

 

Эта тема объясняет, как можно создать список элементов активов, которые могут быть использованы в качестве основы для создания активов для заданий на обслуживание в управлении активами. На основе номенклатур активов можно просмотреть список строк заказа на покупку, созданного на этих номенклатурах. Цель этой функциональности заключается в том, чтобы легко создать актив в управлении активами на основе заказа на покупку.

Сначала вы настраиваете номенклатуры, которые будут использоваться для создания активов из заказа на покупку, в разделе **Номенклатуры активов**. После создания строки заказа на покупку вы создаете активы в разделе **Активы в ожидании**. Можно решить, на какой стадии заказа на покупку актив должен быть создан.


## <a name="select-asset-items"></a>Выбор номенклатур активов

1. Щелкните **Управление активами** > **Настройка** > **Активы** > **Номенклатуры**.
2. Щелкните **Создать**, чтобы создать новую номенклатуру актива.
3. Выберите номенклатуру в поле **Код номенклатуры**. При выходе из этого поля имя номенклатуры автоматически вставляется в поле **Наименование продукта**.
4. На экспресс-вкладке **Общее** выберите тип актива для номенклатуры.
5. Выберите производителя и модель актива для номенклатуры.
6. Сохраните номенклатуру.


## <a name="create-assets-from-pending-assets"></a>Создание активов из отложенных активов

1. Щелкните **Управление активами** > **Общие** > **Активы** > **Активы в ожидании**.
2. Вы увидите обновленный список заказов на покупку на основе номенклатур, выбранных в разделе **Номенклатуры активов**.
3. Можно отфильтровать состояние заказов на покупку, чтобы выбрать, в каком состоянии жизненного цикла должен быть создан актив. Например, вы можете создавать активы только в том случае, если поступление продуктов было разнесено по заказу на покупку.
4. Выберите ссылку **Номер ссылки** в строке заказа на покупку, чтобы просмотреть подробную информацию о товаре.
5. Если вы хотите изменить аналитики, которые отображаются на экспресс-вкладке **Складские аналитики**, нажмите кнопку **Показать аналитики** и сделайте свой выбор.
6. Если требуется создать актив на основе строки заказа на покупку, установите флажок в столбце **Пометка** для этой строки на странице списка и нажмите **Создать актив**. Отображается сообщение, информирующее вас об идентификаторе актива.
7. Выберите актив в списке **Все активы** и нажмите кнопку **Изменить**, чтобы добавить дополнительную информацию в актив.
8. Если в разделе **Активы в ожидании** вы хотите удалить строку, потому что не хотите создавать актив, установите флажок в столбце **Пометка** для этой строки и нажмите **Удалить активы в ожидании**. Отображается сообщение, информирующее вас об идентификаторе актива. Вы не удаляете заказ на покупку или заказ на продажу, удаляется только запись, из которой вы могли бы создать актив в модуле **Управление активами**.

>[!NOTE]
>Все аналитики продукта (размер, цвет, конфигурация и т. д.) автоматически передаются атрибутам актива. Аналитики отслеживания (серийный номер) хранятся непосредственно в активе при создании актива.


## <a name="find-pending-assets"></a>Поиск отложенных активов

Можно запустить **Подсчет активов в ожидании** для проверки ожидающих активов. Например, эта функция может использоваться для получения уведомления каждый раз, когда актив в ожидании готов к созданию в качестве актива.

1. Щелкните **Управление активами** > **Периодические операции** > **Активы** > **Подсчет активов в ожидании**.
2. Нажмите **OK**, чтобы запустить задание и обновить список **Активы в ожидании**.
3. Можно настроить это задание для выполнения в качестве пакетного задания, например, один раз в день.

**Внимание.** Если данные изменяются в заказе на покупку *после* создания актива на основе соответствующей номенклатуры, эти изменения не будут отражены в активе.
