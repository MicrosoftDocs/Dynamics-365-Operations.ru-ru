# <a name="provision-microsoft-dynamics-365-for-talent"></a>Подготовка среды Microsoft Dynamics 365 for Talent

[!include[banner](includes/banner.md)]

[!include[banner](includes/banner.md)]

В этом разделе содержится пошаговое описание процесса подготовки новой производственной среды для Microsoft Dynamics 365 for Talent. В этом разделе предполагается, что вы приобрели Talent в соответствии с соглашением поставщика облачных решений (CSP) или архитектуры предприятия (EA). Если имеется существующая лицензия Microsoft Dynamics 365, которая уже включает план обслуживания Talent, но вы не можете выполнить действия, описанные в этом разделе, обратитесь в службу поддержки.

Сначала глобальный администратор должен войти в систему [Microsoft Dynamics Lifecycle Services](http://lcs.dynamics.com) (LCS) и создать новый проект Talent. Если у вас нет проблем с лицензированием, которые мешают подготовить Talent, помощь службы поддержки или представителей службы разработки Dynamics (DSE) не потребуется.

## <a name="create-an-lcs-project"></a>Создание проекта LCS
Чтобы использовать LCS для управления средами Talent, сначала необходимо создать проект LCS.

1. Выполните вход в [LCS](https://lcs.dynamics.com/Logon/Index) с помощью учетной записи, используемой для подписки на Talent.
2. Выберите знак "плюс" (**+**) для создания проекта.
3. Выберите **Microsoft Dynamics 365 for Talent** как имя продукта и версию продукта.
4. Выберите методологию **Dynamics 365 for Talent**.
5. Выберите **Создать**.

Сведения о том, как начать работу с Talent, см. в методологии **Talent**, созданной в вашем новом проекте. После создания проекта выполните следующую процедуру, чтобы подготовить среду Talent.

## <a name="provision-a-talent-project"></a>Подготовка проекта Talent
После создания проекта LCS можно подготовить Talent в среде.

1. В проекте LCS выберите плитку **Управление приложением Talent**.
2. Talent всегда предоставляется в среде Microsoft PowerApps, чтобы включить интеграцию и расширения PowerApps. Если у вас еще нет среды PowerApps среде, прежде чем продолжить, выполните шаги из пункта "Создание новой среды PowerApps (если требуется)" в этом разделе.

    > [!NOTE]
    > Чтобы просмотреть существующие среды или создать новые среды, администратору клиента, который подготавливает Talent, должна быть назначена лицензия PowerApps P2. Если ваша организация не имеет лицензии PowerApps P2, ее можно получить у CSP или на [странице цен на PowerApps](https://powerapps.microsoft.com/en-us/pricing/).

3. Выберите **Добавить**, затем выберите среду, в которой требуется подготовить Talent.
4. Выберите **Да**, чтобы принять условия соглашения и начать развертывание.

    Новая среда отображается в списке сред в области перехода в левой части окна. Однако невозможно начать использовать эту среду до тех пор, пока статус развертывания не обновится до **Развернуто**. Обычно этот процесс занимает всего несколько минут. Если процесс подготовки завершается с ошибкой, необходимо обратиться в службу поддержки.

6. Выберите **Войти в приложение Talent** для использования в новой среде.

> [!NOTE]
> Если вы еще не закончили работу над окончательными требованиями, можно развернуть тестовый экземпляр Talent в проекте. Затем этот экземпляр можно использовать для проверки решения до его утверждения. Если для тестирования используется новая среда, необходимо повторить эту процедуру для создания рабочей среды.

> [!NOTE]
> Среды Talent, которые подготавливаются через LCS, не содержат демонстрационных данных, настроенных для задач отдела кадров (HR) или относящихся специально к Talent. Если требуется среда, которая содержит демонстрационные данные, рекомендуется подписаться на бесплатную 60-дневную [пробную среду Talent](https://dynamics.microsoft.com/en-us/talent/overview/). Хотя пробная среда принадлежит пользователю, запросившему ее, можно пригласить других пользователей через функцию системного администрирования для среды Core HR. Пробная версия среды содержит вымышленные данные, которые можно использовать для изучения программы безопасным образом. Они не предназначены для использования в производственных средах. Обратите внимание, что по истечении срока действия пробной среды через 60 дней все данные в ней удаляются, и восстановить их будет невозможно. Вы может подписаться на новую пробную среду после истечения срока действия существующей среды.

## <a name="create-a-new-powerapps-environment-if-required"></a>Создание новой среды PowerApps (если требуется)
Интеграция между средами Talent и PowerApps позволяет интегрировать и расширить использование данных Talent с помощью средств PowerApps. Понимание назначения сред PowerApps поможет создать приложения, которые соответствую требованиям по расширению Talent. Сведения о средах PowerApps, включая область действия среды, доступ к среде, а также создание и выбор среды, см. в разделе [Представляем среды PowerApps](https://powerapps.microsoft.com/en-us/blog/powerapps-environments/). Хотя каждый клиент автоматически подготовлен в среде PowerApps по умолчанию, это может быть не лучшая среда для использования для развертывания Talent. Стратегии интеграции данных и тестирования должны учитываться на этом шаге, поэтому рекомендуется рассмотреть различные соображения, которые могут повлиять на развертывание, так как некоторые решения будет сложно изменить в дальнейшем. 

Хотя каждый клиент автоматически подготовлен в среде PowerApps по умолчанию, эта может быть не лучшая среда для использования для развертывания Talent. Стратегии интеграции данных и тестирования должны учитываться на этом шаге. Поэтому рекомендуется рассмотреть различные последствия для развертывания, поскольку впоследствии изменить среду PowerApps будет сложно.

1. В LCS выберите **Управление средами**. Вы перейдете в [Центр администрирования PowerApps](https://preview.admin.powerapps.com/environments), где можно просмотреть существующие среды и создавать новые среды.
2. Выберите **Создать среду**.
3. Введите уникальное имя для среды и выберите местоположение для развертывания.

    > [!NOTE]
    > Talent доступно не во всех регионах. Поэтому обязательно проверьте доступность, прежде чем выбирать местоположение для вашей среды.

4. Когда появится запрос, требуется ли создать базу данных, выберите **Создать базу данных** для создания базы данных Common Data Service (CDS), в которой должна быть размещена часть ваших данных Talent. Создав базы данных, можно также интегрировать приложения PowerApps с приложением Talent.
5. Будет предложено выбрать уровень доступа для использования для базы данных. Рекомендуется выбрать **Ограничение доступа**, так как этот вариант не позволяет пользователям Talent напрямую обращаться к конфиденциальным данным с помощью приложения PowerApps.
6. Создаваемая база данных CDS содержит демонстрационные данные, которые добавляют в производственную среду неактивных сотрудников и вымышленные адреса, а также другую информацию. Чтобы удалить демонстрационные данные, выполните следующие действия после создания базы данных CDS:

    > [!IMPORTANT]
    > Если вы ранее создали базу данных CDS и ввели в нее какие-либо производственные данные своей компании, эти действия удаляют **все** данные из выбранной базы данных, даже производственные данные вашей компании.

    1. Выполните вход в [PowerApps](https://preview.web.powerapps.com/home).
    2. В раскрывающемся списке в правом верхнем углу выберите среду, созданную на шаге 2.
    3. В левой области навигации разверните **Common Data Service**, затем выберите **Объекты**.
    4. В правой части страницы выберите кнопку с многоточием (**...**), затем выберите пункт **Очистить все данные**.
    5. Выберите **Удалить данные** для подтверждения удаления данных. Это действие удаляет все демонстрационные данные, которые включены в CDS по умолчанию. Оно также удаляет все другие данные, которые были введены в выбранную базу данных.

Теперь можно использовать новую среду.

## <a name="grant-access-to-the-environment"></a>Предоставление доступа к среде
По умолчанию, глобальный администратор, который создал среду, имеет к ней доступ. Однако дополнительным пользователям приложения необходимо явно предоставить доступ. Чтобы предоставить доступ, [добавьте пользователей](../dev-itpro/sysadmin/tasks/create-new-users.md) и [присвоить им соответствующие роли](../dev-itpro/sysadmin/tasks/assign-users-security-roles.md) в среде Core HR. Также необходимо добавить этих пользователей в среду PowerApps, чтобы они имели доступ к приложениям Attract и Onboard. Здесь описана процедура. Если требуется справка по выполнению этих шагов, см. раздел [Введение в центр администрирования PowerApps](https://powerapps.microsoft.com/en-us/blog/introducing-admin-center-for-powerapps/) в записи блога.

Эта процедура выполняется глобальным администратором, который выполнил развертывание среды Talent.

1. Откройте [центр администрирования PowerApps](https://preview.admin.powerapps.com/environments).
2. Выберите соответствующие среды.
3. На вкладке **Безопасность** добавьте необходимых пользователей для роли **Создатель среды**.

Обратите внимание, что это последний шаг, в котором требуется добавление пользователей к среде PowerApps вручную, является временным. В конечном счете он будет выполняться автоматически при добавлении пользователей в среде Core HR.
