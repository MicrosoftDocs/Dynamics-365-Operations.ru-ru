# <a name="whats-new-or-changed-in-dynamics-365-for-talent-september-10-2019"></a>Что нового и что изменилось в Dynamics 365 for Talent (10 сентября 2019 г.)

В этой теме описываются новые и измененные компоненты Dynamics 365 for Talent.

## <a name="changes-in-attract"></a>Изменения в Attract

### <a name="candidate-e-mail-login"></a>Вход кандидатов по адресу эл. почты

Кандидаты теперь могут использовать любой адрес электронной почты для создания учетной записи и входа на сайт вакансий Talent, чтобы подавать заявления на должность и проверять статус своих заявлений. Это в дополнение к тому, что они уже могли входить на сайт вакансий Talent, используя учетные записи социальных сетей или учетные данные своих организаций.

### <a name="job-activation-and-posting"></a>Активация и размещение объявлений о вакансиях

Мы внесли несколько изменений в активацию и размещение объявлений о вакансиях. Необходимо активировать вакансии перед их размещением на сайте вакансий Talent или на любых внешних досках вакансий, включая LinkedIn или Broadbean.

## <a name="changes-in-onboard"></a>Изменения в Onboard

Этот выпуск содержит исправления незначительных ошибок для Dynamics 365 Talent: Onboard.

## <a name="changes-in-core-hr"></a>Изменения в Core HR

Изменения, описанные в этом разделе, относятся к сборке номер 8.1.2482. Числа в скобках в некоторых заголовках относятся к номерам поддержки в службе Microsoft Dynamics Lifecycle Services (LCS).

### <a name="you-can-enable-preview-features-in-sandbox-and-trial-environments"></a>Предварительные версии функций можно включить в песочнице и в пробных средах

При подготовке нового экземпляра Talent можно указать, имеет ли данный экземпляр тип Производство или Песочница. Экземпляры типа Песочница позволяют заблаговременно тестировать новые возможности. Если необходимо обновить один из существующих экземпляров до типа экземпляра Песочница, обратитесь в службу поддержки, чтобы инициировать запрос на изменение.

Дополнительные сведения о порядке публикации изменений см. в разделе [Подготовка Talent](./provisioning-talent.md).

### <a name="platform-update-29"></a>Обновление платформы update 29

Дополнительные сведения об обновлении платформы Platform update 29 см. в разделе [Предварительные версии функций в Dynamics 365 for Finance and Operations Platform Update 29 (октябрь 2019)](https://docs.microsoft.com/dynamics365/unified-operations/fin-and-ops/get-started/whats-new-platform-update-29).

### <a name="new-job-base-entity-available-in-data-management-framework-347202"></a>Новый базовый объект вакансии доступен в структуре управления данными (347202)

В этом выпуске новый базовый объект вакансии доступен для импорта или экспорта данных. 

### <a name="worker-advanced-security-policy-incorrectly-displays-positions-in-position-hierarchy-354868"></a>Политика дополнительной безопасности работника"неправильно отображает должности в иерархии должностей (354868)

В этом выпуске должности больше не будут отображаться открытыми с "пустым" работников, когда пользователи имеют ограниченный доступ.

### <a name="job-form-close-box-wont-close-form-in-certain-situations-342467"></a>Кнопка закрытия формы вакансии не закрывает форму в определенных ситуациях (342467)

Этот выпуск включает изменение, которое позволяет закрыть форму вакансии во всех сценариях.

### <a name="new-case-on-employee-record-is-locked-for-human-resources-manager-role-337111"></a>Новое обращение на записи сотрудника заблокировано для роли менеджера по управлению персоналом (337111)

В этом изменении форма управления обращениями больше не блокируется при доступе к ней из формы сотрудника.

### <a name="employment-end-date-always-defaults-to-235959-351492"></a>Дата увольнения по умолчанию всегда равна 23:59:59 (351492)

В этом изменении можно изменить дату приема на работу на время, отличное от 23:59:59, при ручном окончании трудоустройства.

### <a name="unable-to-set-up-expiration-date-on-an-earning-code-through-data-management-336604"></a>Невозможно настроить дату истечения срока действия кода дохода через управление датой (336604)

В этом выпуске можно настроить коды дохода, срок действия которых истекает, через объект **PayrollWorkerPositionEarningCodeEntity**.

### <a name="employee-development-analytic-report-doesnt-display-data-348737"></a>Аналитический отчет по развитию сотрудников не показывает данные (348737)

В выпуске за эту неделю аналитика для навыков сотрудников была обновлена, чтобы обеспечить точную отчетность.

### <a name="terms-of-employment-datetime-dont-default-to-beginning-of-day-349101"></a>Условия даты и времени трудоустройства по умолчанию не устанавливаются на начало дня (349101)

С этим изменением дата и время начала теперь по умолчанию устанавливается на начало дня, а дата и время окончания по умолчанию устанавливается на конец дня.

### <a name="changing-the-employment-end-date-on-worker-action-form-displays-an-error-354092"></a>При изменении даты окончания занятости в форме действия работника отображается ошибка (354092) 

Это изменение исправляет ошибку при изменении даты окончания занятости в рамках действия сотрудника.

### <a name="applying-onboarding-checklists-across-companies-338433"></a>Применение контрольных списков адаптации в нескольких компаниях (338433)

В данном выпуске теперь имеется возможность применить контрольные списки для сотрудников, работающих в юридических лицах, отличных от юридического лица, в которое осуществлен вход.

## <a name="in-preview"></a>В режиме предварительного просмотра

### <a name="streamlined-employee-entry-and-navigation"></a>Упрощенный вход и навигация сотрудников

Эта функция теперь доступна в изолированных средах. Чтобы включить эту функцию, выберите **Системный администратор > Ссылки > Настройка > Системные параметры > Предварительные версии функций**. Выберите **Расширенные форма и навигация по сотрудникам**. Это позволит включить эти изменения для всех пользователей. Этот параметр можно отключить в любое время.

Дополнительные сведения см. в разделе [Упрощенный вход и навигация сотрудников](./streamlined-employee-entry.md).
