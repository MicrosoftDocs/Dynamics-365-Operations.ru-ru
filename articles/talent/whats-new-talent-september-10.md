# <a name="whats-new-or-changed-in-dynamics-365-for-talent-core-hr-september-10-2018"></a>Что нового и что изменилось в Dynamics 365 for Talent Core HR (10 сентября 2018 г.)

[!include [banner](includes/banner.md)]

**Сборка 8.1.138.0**

В этой теме описываются новые и измененные компоненты Microsoft Dynamics 365 for Talent Core HR.

## <a name="allow-specific-time-of-day-on-time-off-requests-half-days"></a>Возможность указания конкретного времени суток в запросах на отгулы (на половину дня)

Если модуль "Управление отпусками и отсутствиями" настроен так, что продолжительность отгулов указывается в днях, теперь можно включить возможность указания половины дня. После этого при отправке запросов на отгулы пользователи могут указать, когда они хотят взять отгул — в первую половину для или во вторую.

По умолчанию эта возможность отключена. Чтобы сотрудники могли запрашивать отгулы в первой или второй половине дня, необходимо включить эту возможность в области **Отпуска и отсутствия** параметров управления персоналом.

Привилегия безопасности, которая для этого требуется, — "Ведение параметров управления персоналом".

## <a name="validation-of-leave-and-absence-entries"></a>Проверка записей отпусков и отсутствий

В зависимости от того, как настроены отпуска, сотрудники, пытающиеся отправить запрос на отгул, длительность которого превышает их рабочий день, получают предупредительное сообщение. Иными словами, система предупреждает их, когда они пытаются взять отгул продолжительностью больше целого дня на отдельно взятую дату.

Такая проверка всегда включена. Каждый раз, сотрудники превышают заданное пороговое значение, они получают предупреждение при отправке запроса на отгул.

## <a name="additional-fields-for-conditional-statements-in-workflows"></a>Дополнительные поля для условных операторов в workflow-процессах

В условные операторы и заполнители для нескольких workflow-процессов в Core HR были добавлены дополнительные поля.

В workflow-процессы компенсации, увольнения и перемещения были добавлены следующие поля:

- EmploymentType (Тип трудоустройства)
- LegalEntity (Юридическое лицо)
- AdjustedWorkerStartDate (Скорректированная дата приема работника на работу)
- EmployerNoticeAmount (Сумма в уведомлении работодателя)
- EmployerUnitOfNotice (Подразделение для уведомления работодателя)
- TransitionDate (Дата перевода)
- WorkerNoticeAmount (Сумма в уведомлении работника)
- WorkerStartDate (Дата приема работника на работу)
- WorkerUnitOfNotice (Подразделение для уведомления работника)
- ProbationEndDate (Дата окончания испытательного срока)
- Position (Занимаемая должность)
- Union (Профсоюз)
- Подразделение (Department)
- PositionType (Тип должности)
- CompLocation (Местоположение для компенсации)
- Title (Название должности)
- Job (Должность)
- JobType (Тип должности)
- JobFamily (Семейство должностей)
- JobFunction (Функциональные обязанности)

В workflow-процессы должности были добавлены следующие поля:

- Position (Занимаемая должность)
- Union (Профсоюз)
- Department (Подразделение)
- PositionType (Тип должности)
- CompLocation (Местоположение для компенсации)
- Title (Название должности)
- Job (Должность)
- JobType (Тип должности)
- JobFamily (Семейство должностей)
- JobFunction (Функциональные обязанности)

Поля в условных операторах и заполнителях доступны для всех пользователей, имеющих доступ к настройке упомянутых выше workflow-процессов.

## <a name="navigation-to-attract-from-personnel-management"></a>Переход из Attract в управление персоналом

В управлении персоналом, если модуль Attract не настроен, в разделе **Кандидаты для приема на работу** пользователям предлагается начать работу с Attract, вместо отображения сообщения "Данные для отображения не найдены".

## <a name="other-changes"></a>Другие изменения

Этот выпуск содержит несколько дополнительных исправлений ошибок:

- При найме подрядчика вкладка **Компенсация** не должна быть доступна на странице запроса/действия.
- В процессе запроса увольнения нельзя продолжить, пока все обязательные поля не будут содержать данные.
- Решены проблемы с порядком сортировки и отображения дат в аналитике управления персоналом.
