# <a name="set-up-offer-management"></a>Настройка управления предложениями 

[!include [banner](includes/banner.md)]

При перемещении кандидата на стадию предложения в Dynamics 365 for Talent: Attract необходимо убедиться, что предложения может быть быстро создано для кандидата, одобрено по необходимости и отправлено кандидату. Поскольку большинство предложений являются стандартными, они могут быть созданы на основе шаблонов для повторного использования. В Attract все предложения упакованы в пакет предложений, который представляет собой набор из одного или нескольких документов для предложения. 

В этом разделе перечислены все действия, которым администратору Attract нужно следовать, чтобы настроить шаблоны пакета как часть возможности управления предложениями в Attract. Пользователи с ролями, не являющими администраторами, не будут иметь доступ к этим возможностям.

>[!NOTE]
> Возможности управления предложениями доступны в рамках надстройки Comprehensive Hiring.

## <a name="offer-data"></a>Данные предложения 

Данные предложений являются наименьшей единицей в шаблоне пакета предложений. Типичное предложение состоит из стандартного текста и набора значений. Наборы значений являются единственными элементами, которые могут изменяться между предложениями. При создании предложения самый важный аспект, на котором создатель предложения может сосредоточиться, — это список заполнителей данных предложений, имеющиеся в шаблоне пакета предложения. Чтобы настроить данные предложения, выполните следующие действия.

1.  Перейдите в **Offer management**.

1.  Разверните раздел **Библиотека** в левой области навигации, затем перейдите к **Данные предложения**.

    >[!NOTE]
    > На странице **Данные предложения** имеются разделы **Сведения о кандидате** и **Сведения о вакансии**. Attract предоставляет несколько готовых заполнителей данных предложения.
    
    > На странице имеются разделы для организации различных заполнителей данных предложения вместе в логические группы. Эти разделы могут помочь в ведении данных предложения и заполнения данных во время создания предложения.

1.  Чтобы создать новый раздел данных предложения, щелкните **Добавить раздел** и введите уникальное имя для раздела.

1.  Чтобы добавить заполнители данных предложение к какому-либо разделу, нажмите **Добавить данные предложения** и введите уникальное имя для заполнителя.

1.  Чтобы изменить имя любого раздела, наведите указатель мыши на имя раздела и обновите имя.

1.  Чтобы изменить имя любого существующего заполнителя данных предложения, сначала убедитесь, что заполнитель еще не является частью какого-либо шаблона. Затем наведите указатель мыши на имя заполнителя данных предложения и обновите имя при необходимости.

1. Чтобы удалить существующий заполнитель данных предложения, сначала убедитесь, что заполнитель еще не является частью какого-то другого шаблона. Затем наведите указатель мыши на заполнитель и, когда появится значок корзины, щелкните корзину для удаления заполнителя данных предложения.
    >[!NOTE]
    > Можно увидеть, в сколько шаблонов входит заполнитель данных предложений, но числовому индикатору рядом с каждым элементом данных предложения. 

1. Чтобы удалить любой раздел, наведите указатель мыши на имя. Если ни один заполнитель данных предложения внутри раздела не используется ни в одном шаблоне, можно щелкнуть значок корзины для его удаления. 
    >[!NOTE]
    > Удаление раздела приведет к удалению всех заполнителей данных предложений в этом разделе.

После того как заполнители данных предложений были настроены, их можно использовать в любом шаблоне документа. Заполнители данных предложений не ограничены определенным шаблоном — тот же набор может использоваться во всех шаблонах.

## <a name="offer-data-rules"></a>Правила данных в приложении

Большинство организации предусматривают правила в отношении того, как должны создаваться предложения. Например, организация может требовать, чтобы комбинация максимальной ежегодной зарплаты для определенного местоположения и уровень трудового стажа находились в пределах определенного диапазона, или чтобы были возможны только несколько конкретных значений для уровня предложение для найма новых сотрудников. Attract в настоящее время поддерживает все такие правила данных с использованием файла CSV.

Чтобы подготовить CSV-файл с правилами данных, выполните следующие действия.

1.  Определите заполнитель данных предложения для набора правил. Например **Ежегодная зарплата**.

1.  Укажите значения заполнителей зависимых данных предложения. Например, **Местонахождение должности** и **Уровень**.

1.  Укажите столбцы 1 и 2 в качестве **Местонахождение должности** и **Уровень**.

1.  Чтобы отправить значение диапазона, сделайте столбцы 3 и 4 **Ежегодная зарплата**. Чтобы отправить определенное значение, а не диапазон, только сделайте столбец 3 **Ежегодная зарплата**.

1.  Заполните файл Microsoft Excel в зависимости от требуемых ролей.

1.  Убедитесь, что каждая строка содержит уникальное сочетание всех значений, объединенных вместе.

1.  Сохраните файл в формате CSV.

Чтобы отправить файл с правилами данных предложения, выполните следующие действия.

1.  Перейдите в **Offer management**.

1.  Разверните раздел **Библиотека** в левой области навигации, затем перейдите к **Правила данных в предложении**.

1.  Щелкните **Создать набор данных** для отображения диалогового окна **Отправить**.

1.  Укажите уникальное имя для набора правил, затем отправьте сохраненный CSV-файл.

1.  Нажмите кнопку **Добавить**.
    Набор правил будут обрабатываться в фоновом режиме, и вы будете оповещены в приложения и по электронной почте после завершения.

    Вы будете оповещены в случае ошибок при обработке отправки. Затем можно загрузить журнал ошибок, исправить файл и отправить его еще раз.

1.  Используйте параметр кнопки с многоточием (**...**), если хотите загрузить набор правил и отправить набор значений. После обновления можно отправить файл еще раз.

1.  Можно удалить существующую загрузку набора правил, если удаляемый заполнитель не используется в другом шаблоне документа.

>[!ПРИМЕЧАНИЯ]
> - Каждый заполнитель может иметь только один уникальный набор столбцов, от которых он зависит. Например, если **Ежегодная зарплата** зависит от **Местонахождение должности** и **Уровень**, невозможно отправить другой набор правил, где **Ежегодная зарплата** является зависимой от другого набора столбцов.

> - Можно загрузить примеры наборов правил данных предложений на вкладке **Примеры** на странице **Правила данных в предложении**.

> - Когда создатель предложения переопределяет значения, которые рекомендуются правилами данных предложения, предложение помечается как нестандартное и будет обязателен workflow-процесс утверждения.

## <a name="document-templates"></a>Шаблоны документов

Шаблон документа предложения может помочь администраторам создавать письма с предложениями. Шаблон документа предложения представляет собой сочетание текста, который будет частью предложения, а также определенных заполнителей данных предложения. 

Чтобы создать шаблон документа предложения, выполните следующие действия.

1.  Перейдите в **Offer management**.

1.  Разверните раздел **Библиотека** в левой области навигации и перейдите к пункты **Шаблоны**.

    На странице **Шаблоны** будет отображаться список шаблонов документов, которые уже были определены.

1.  Чтобы создать новый шаблон документа, щелкните **Создать шаблон**.

1.  Введите уникальное имя для шаблона и щелкните **Создать**.

1.  С помощью редактора форматированного текста вставьте или отредактируйте содержимое документа предложения. Можно вставлять таблицы, изображения и гиперссылки, используя параметры, доступные в верхней части текстового редактора.

1.  В документе шаблона предложения можно вставить заполнители данных предложения следующими способами:

    - Перетаскивание из правой области.

    - Вставка хэштега заполнителя данных предложения непосредственно в нужное место. Введите **\#**, затем начинайте вводить имя заполнителя данных предложения. В раскрывающемся списке отображаются параметры. Щелкните или нажмите кнопку или клавишу **ВВОД**, чтобы вставить заполнитель данных предложения.

    >[!ПРИМЕЧАНИЯ]
    > - Чтобы связать заполнитель в шаблон документа предложения, не показывая его значение кандидату, наведите указатель мыши на заполнитель данных предложения и щелкните значок **Прикрепить**. В результате заполнитель будет перемещен в раздел **Закрепленные данные предложения** шаблона документа предложения. Чтобы отменить закрепление, следуйте тем же инструкциям, но выберите **Отменить закрепление** в списке заполнителей данных предложения.

    > - Чтобы просмотреть список активных заполнителей данные предложения, переключитесь на вкладку **Активные** в правой области.

    > - При вставке заполнителя, который обрабатывается набором правил данных предложения, отображаются зависимости этого заполнителя данных предложения от других значений.

    > - Кроме того, имеется возможность **Импортировать** содержимое из DOCX-файла на локальном компьютере и внести требуемые изменения. Таким образом, не нужно вводить все содержимое в редакторе.

1. Для предварительного просмотра документа предложения используйте параметр **Предварительный просмотр** в верхней части страницы.

1. Для управления тем, может ли создатель предложения редактировать содержимое документа предложения, используйте параметр **Управление разрешением** в верхней части страницы. Если требуется, чтобы создатель предложения мог только вставлять значения заполнителей, но не редактировать текст, задайте значение разрешения **Нет**.

1. Щелкните **Сохранить** для сохранения хода работы. Шаблон будет сохранен в состоянии черновика.

1. Чтобы завершить работу и пометить документ как опубликованный, нажмите кнопку **Готово**.

1. Можно просмотреть, какие шаблоны документов в данный момент активные, в режиме черновика и архивированы и больше не используются в библиотеке шаблонов документов приема сотрудников.

>[!NOTE]
> Можно удалить любой шаблон документа предложения, который не является частью существующего шаблона пакетов предложений.


## <a name="offer-package-templates"></a>Шаблоны пакетов предложений

Пакеты предложений — это артефакты предложений, которые используются совместно с кандидатом и состоят из одного или нескольких шаблонов документов предложения. Чтобы создать пакет предложений, выполните следующие действия.

1.  Перейдите в **Offer management**.

1.  Выберите пункт **Пакеты** на левой навигационной панели.

    Отображается список активных шаблонов пакетов, доступных для использования создателями предложений.

1.  Чтобы создать новый пакет предложений, щелкните **Создать пакет**.

1.  Введите уникальное имя и щелкните **Создать**.

1.  Щелкните **Добавить шаблон**.

    >[!ПРИМЕЧАНИЯ]
    > - Можно создать новый шаблон или выбрать один из существующих.

    > - Если вы решите добавить существующий шаблон, необходимо убедиться в том, что шаблон документа предложения был сохранен, завершен и помечен как активный.
    
    > - Можно выбрать любое требуемое число шаблонов документов. 
    
1.  Щелкните **Готово**, чтобы вернуться к **Управление пакетами**.

    Можно настроить последовательность документов и также отметить, является ли конкретный документ предложения необходимым при создании предложения. Создатель предложения будет иметь возможность удалить документы, которые помечены как **Не требуется**.

1. Чтобы сохранить шаблон пакета предложения, чтобы его могли использовать все создатели предложений, щелкните **Сохранить и опубликовать**.

   Для просмотра черновиков шаблонов пакетов предложений перейдите на вкладку **Черновики**. При внесении изменений в шаблон пакета предложения его необходимо сохранить и снова опубликовать.

## <a name="configure-an-offer-process"></a>Настройка процесса предложения

Существует несколько частей процесса создания предложения, которые может настраивать администратор Attract.

- **Утверждения предложений.** Выберите, требуется ли утверждать предложения перед отправкой кандидатам. Администратор может также решить, должны ли все утверждения выполняться последовательно или в параллельном workflow-процессе утверждения. Можно также задать, должны ли лица, утверждающие предложение, добавлять комментарий вместе со своим утверждением предложения. Утверждения предложений являются обязательными для всех нестандартных предложений.

- **Параметры предложения для кандидатов.** С правами администратора можно указать, должны ли все предложения иметь дату окончания срока действия, и, если должны, какое смещения по умолчанию для даты окончания срока годности должно быть. Можно также настроить, могут ли кандидаты отклонить предложение.

- **Электронные подписи.** В настоящее время единственным доступным вариантом электронной подписи для кандидатов является ввод своего имени в пакета предложения при принятии предложения. В будущем мы представим партнерскую интеграцию с другими поставщиками электронной подписи.


Дополнительные сведения о процессе создания предложения см. в разделе [Создание, утверждение и подписание предложений](./creating-offers.md).
