# <a name="whats-new-or-changed-in-dynamics-365-talent---core-hr-december-14-2018"></a>Что нового и что изменилось в Dynamics 365 Talent — Core HR (14 декабря 2018 г.)

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

**Сборка 8.1.2085**

В этой теме описываются новые и измененные компоненты Core HR.

## <a name="changes"></a>Изменения

### <a name="us---aca-affordable-care-act-support-for-tax-year-2018-1095-b-and-1095-c-forms"></a>США — Поддержка ACA (закон Affordable Care) для форм 1095-B и 1095-C для налогового года 2018

Каждый год соответствующие крупные работодатели (ALE) должны предоставить каждому сотруднику с полной занятостью форму 1095-C. Кроме того, если работодатель предоставляет самостоятельное покрытие, он обязан предоставить формы 1095-C (если он является крупным работодателем (ALE)) и 1095-B (если он является небольшим работодателем) для любого сотрудника, с полной или частичной занятостью, застрахованного в соответствии с одним из предложенных планов медицинского страхования. Эта функциональная возможность обеспечивает печатные варианты форм 1095-C и 1095-B.

### <a name="during-import-submittedbypersonid-field-on-hcmperfjournalentity-is-ignored"></a>Во время импорта поле SubmittedByPersonId в HcmPerfJournalEntity игнорируется

При импорте или экспорте записей журнала производительности поле **Кем отправлено** игнорируется. В результате этого изменения значение **импортировано/экспортировано** будет отражать значение в таблице во время экспорта, при импорте система обновляется со значением, указанным в файле импорта.

### <a name="analytics-tab-on-leave-and-absence-workspace-displays-openconnectionerror-error-for-non-system-admin-roles"></a>На вкладке аналитики в рабочей области "Отпуск и отсутствие" отображается ошибка "OpenConnectionError" для ролей, отличных от администратора

В этом обновлении никакие ошибки не отображаются при открытии вкладки **Аналитика** в рабочей области **Отпуск и отсутствие**.

### <a name="employee-self-service-position-hierarchy-drill-down-from-tile-fails-to-get-parent-node"></a>Детализация самообслуживания сотрудников, иерархия штатных единиц, из плитки не получает родительский узел

Внесено изменение для исправления ошибки "Не удалось получить родительский узел", когда иерархия штатных единиц настраивается для отображения в существующей рабочей области и выбирается должность в иерархии.  

### <a name="must-be-system-admin-to-see-the-payroll-tab-in-the-position-page"></a>Для просмотра вкладки заработной платы на странице должностей необходимо быть системным администратором

Был внесено изменение для включения правильных элементов безопасности в существующую привилегию: "Вести сведения о работнике на зарплате и его должности". После этого изменения по умолчанию администратор заработной платы будут иметь доступ к полям заработной платы на странице должности.

### <a name="error-when-submitting-performance-review-to-manager-and-the-reviewsperfperiod-placeholder-is-used-in-the-submission-instructions"></a>Ошибка, когда при отправке оценки производительности руководителю используется заполнитель %Reviews.PerfPeriod% в инструкциях по отправке

Внесено изменение, которое исправляет ошибку "Пустая ссылка" при использовании заполнителя %Reviews.PerfPeriod% в инструкциях по отправке.

### <a name="workforce-power-bi-report-shows-error-when-worker-seniority-date-is-a-leap-day"></a>В отчете Power BI о трудовых ресурсах отображается ошибка, когда дата начала трудового стажа работника приходится на високосный день

После этого изменения високосные дни теперь поддерживаются в Power BI.

### <a name="integration-between-core-hr-and-attract"></a>Интеграцию между модулями Core HR и Attract

Внесено изменение для обновления интеграции между модулями Core HR и Attract, связанной с кандидатами для приема на работу. Чтобы кандидаты для приема на работу отображались в рабочей области **Управление персоналом**, используются следующие сущности Common Data Service:

Заявление о приеме на работу
- Для причины состояния необходимо задать значение "Предложение принято"
-   Предоставляет кандидата и вакансию

Кандидат
-   Представляет сведения о кандидате

Вакансия
-   Предоставляет идентификатор вакансии и участников на вакансию

Участники на вакансию
-   Представляет специалиста по комплектации штата

При добавлении кандидата в модуль "Управление персоналом" кандидата теперь также можно отклонить с помощью нового параметра, доступного на карточке кандидата.

## <a name="coming-soon"></a>Скоро

### <a name="leave-and-absence-future-leave-and-forecasting-leave-balances"></a>Отпуск и отсутствие: будущий отпуск и прогноз сальдо отпуска

С изменениями, вносимыми, чтобы сотрудники могли прогнозировать отгулы и подавать запросы на будущие отгулы, не влияя на текущее сальдо отгулов, также изменяется способ отображения сальдо отгулов. 

Отображаемое в настоящее время доступное сальдо — это время отгула, доступное для запроса, включая начисления за сегодняшний день и все утвержденные отпуска до конца времен. 

Когда возможность прогноза будет выпущена, отображаемое сальдо изменится на текущий баланс отгулов, включая начисления до сегодняшнего дня и запросы на сегодня. Сотрудники и менеджеры увидит эти обновленные сальдо в службе самообслуживания сотрудников и менеджера на карте **Отгулы** и в окне **Сальдо отгулов**. Менеджеры отдела кадров увидят эти обновленные сальдо в рабочей области **Люди** и в окне **Назначенные планы отпусков** сотрудника.

## <a name="known-issue"></a>Известная проблема

### <a name="mapping-errors-in-the-integration-with-finance"></a>Ошибки сопоставления в интеграции с Finance

Следующие проблемы были выявлены в текущем шаблоне для интеграции Talent с Dynamics 365 Finance. Новый шаблон будет опубликован в ближайшее время и будет применяться ко всем новым создаваемым проектам интеграции. Для существующих проектов интеграции сопоставления задач можно обновить. Обновленные сопоставления см. в следующей таблице. 

>[!NOTE]
> Задача назначения должностей для родительских должностей не интегрирует данные. Это ошибка, которая в настоящее время исследуется. Не имеет временного решения в текущем сопоставлении. 

В задаче сопоставления отделов с операционными единицами должны быть обновлены следующие сопоставления.

| Существующее поле источника          | Новое поле источника |
| -------------------------------|------------------|
| cdm_description (Описание)  | cdm_name (Имя)  |

Также необходимо добавить дополнительные сопоставления. Выберите последнее поле **Нет**, чтобы добавить следующее сопоставление.

| Поле источника                   | Поле назначения    |
| -------------------------------|----------------------|
| cdm_description (Описание)  | NAMEALIAS (NAMEALIAS)|

Обновленные сопоставления должна выглядеть, как на следующем рисунке.

![Задача сопоставления подразделений с операционными единицами](./media/DepartmentMapping.png)


В задаче сопоставления должностей со сведения о должности должны быть обновлены следующие сопоставления.

| Существующее поле источника          | Новое поле источника                   |
| -------------------------------|------------------------------------|
| cdm_name (Имя)                | cdm_description (Описание)      |
| cdm_name (Описание)         | cdm_jobdescription(Описание должности)|


Обновленные сопоставления должна выглядеть, как на рисунке ниже.

![Задача сопоставления должностей со сведения о должности](./media/JobMapping.png)

В задаче сопоставления сотрудников с должностными обязанностями должны быть обновлены следующие сопоставления.

| Существующее поле источника                 | Новое поле источника                               |
| --------------------------------------|------------------------------------------------|
| cdm_emailaddress1 (Адрес электронной почты 1)   | cdm_primaryemailaddress (Основной адрес электронной почты |
| cdm_telephone1 (Телефон 1)          | cdm_primarytelephone (Основной телефон)       |

Также необходимо обновить преобразование поля "Пол". Выберите тип сопоставления **fn** (функция) для поля "Пол" и обновите следующие сопоставления значений.

| Значение Common Data Service                   | Значение Finance and Operations                     |
| ----------------------------|--------------------------------------------------|
| 75440000                    | Мужской                                             |
| 75440001                    | Женский                                           |
| 75440002                    | Не допускается                                             | 
| 75440003                    | Не указан                                      |

Обновленные сопоставления должна выглядеть, как на следующих рисунках.
