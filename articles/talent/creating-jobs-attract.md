# <a name="create-a-job-in-attract"></a>Создание вакансии в Attract

[!include [banner](includes/banner.md)]

В этом разделе описаны элементы должности в Microsoft Dynamics 365 Talent: Attract. Также объясняется, как создать должность.

## <a name="job-creation"></a>Создание должности

Администраторы, агенты по найму кадров и менеджеры по найму могут создавать должности. При создании должности предлагается выбрать вашу роль в процессе: менеджер или агент по найму персонала. После выбора роли будет предложено выбрать шаблон процесса. При выборе **Пропустить** используется шаблон по умолчанию. Дополнительные сведения о шаблонах процессов см. в разделе [Создание шаблона процесса в Attract](./process-templates-attract.md).

Должность в Attract содержит сведения о должности, группу найма, процесс найма, публикации объявлений о вакансии и аналитику.

## <a name="job-details"></a>Сведения о вакансии

Вкладка **Сведения о вакансии** содержит подробные сведения о должностных обязанностях и атрибутах. Поля для названия должности, описания должности и местоположения должности являются обязательными. Остальные поля являются необязательными.

По умолчанию в поле **Количество вакансий** установлено значение **1**. Однако это значение можно изменить. Когда предложение было подготовлено для должности, значение поля **Количество доступных вакансий** уменьшается.

Если включено управление должностями в центре администрирования, доступно поле подстановки **Обновить должности**. Это поле подстановки считывает объект JobPosition в Common Data Service и возвращает список штатных единиц, которые могут быть выбраны для должности. Если выбранное количество штатных единиц превышает количество открытых вакансий, вы получаете предупреждение. Вы также получите предупреждение, если штатная единица используется для нескольких должностей.

> [!NOTE]
> Управление штатными единицами доступно с помощью надстройки Comprehensive Hiring.

В зависимости от настроек, определенных в действии "Предложение" процесса найма на работу, номер штатной единицы может использоваться дважды в предложении. Дополнительные сведения см. в разделе [Мероприятия в процессах найма](./activities-attract.md).

Attract включает набор по умолчанию для параметра **Навыки**. Эти навыки отображаются как предложения по мере ввода. Можно добавить дополнительные навыки, введя новый текст навыков в поле и нажав клавишу ВВОД.

Attract включает набор по умолчанию для параметра **Должностные функции**. Можно добавить до трех дополнительных должностных обязанностей, введя новую должностную функцию в поле и нажав клавишу ВВОД.

Attract включает набор по умолчанию для параметра **Отрасль компании**. Можно добавить до трех дополнительных отраслей компании, введя новую отрасль компании в поле и нажав клавишу ВВОД.

## <a name="hiring-team"></a>Группа комплектации штата

Вкладка **Группа комплектации штата** содержит список сотрудников, которые будут заниматься этой должностью. При добавлении пользователей в группу комплектации штата им необходимо назначить роль в этой группе. Роль определяет данные, к которым пользователи имеют доступ, и уведомления, которые они получают. Для выбора доступны роли **Специалист по найму**, **Специалист по комплектации штата**, **Делегат** и **Проводящий собеседование**. Дополнительные сведения о правах роли см. в документе "Управление ролями". Специалисты и менеджеры по найму могут назначить одного или нескольких делегатов для работы от их имени. Дополнительные сведения о делегатах см. в разделе [Безопасность и управление ролями в Attract](./security-attract.md).

Группу комплектации штата можно обновить после активации должности.

## <a name="process"></a>Обработка

Информация по умолчанию о процессе найма основана на шаблоне процесса, который был выбран при создании должности. Если на том этапе не был выбран конкретный шаблон, используется шаблон по умолчанию. При определении процесса найма можно добавить или удалить различные стадии, за исключением стадий "Соискатель", "Заявление" и "Предложение". Хотя стадия "Соискатель" не может быть удалена, она может быть отключена. На каждой стадии можно добавить или удалить одно или несколько предварительно определенных действий.

Дополнительные сведения о действиях, которые могут быть добавлены к процессу найма, см. в разделе [Мероприятия в процессах найма](./activities-attract.md).

> [!NOTE]
> Процесс приема на работу невозможно обновить после активации должности.

## <a name="postings"></a>Разноска

После активации должности можно подать объявление о вакансии. Подавать объявления о вакансии на должность могут только агенты по найму и администраторы. Объявление о вакансии можно опубликовать в Talent Careers (сайт вакансий Dynamics 365 Talent) или в LinkedIn. Группа Attract постоянно работает над расширением партнерства с агрегаторами досок объявлений о вакансиях. Этот список будет расширяться со временем. Когда объявление о вакансии размещается только как внутреннее, кандидатам необходима учетная запись AAD для просмотра должности и подачи заявления на нее. Если должность указана как общедоступная, кандидаты могут просматривать должности и подавать заявления на них с использованием любых вариантов проверки подлинности. 

Дополнительные сведения о публикации объявлений о вакансиях см. в разделе [Настройка сайта карьеры в Microsoft Dynamics 365 Talent — Attract](career-site.md).

> [!NOTE]
> Функция публикации объявлений о вакансиях доступна только с надстройкой Comprehensive Hiring для Attract.

## <a name="activate"></a>Активировать

После активации должности можно публиковать объявления о вакансии, а также добавлять в нее перспективных кандидатов и соискателей. Возможность добавлять соискателей к должности задается в действии "Соискатель" процесса найма.

> [!NOTE]
> Процесс приема на работу невозможно обновить после активации должности.

## <a name="prospects-and-applicants"></a>Соискатели и кандидаты

Возможность добавлять соискателей к должности задается в [Мероприятия в процессах найма](./activities-attract.md#prospect-activity) в процессе найма. Этот параметр должен быть установлен перед активацией должности. После активации должности можно добавлять в нее перспективных кандидатов и соискателей.

## <a name="approvals"></a>Утверждения

Должности Attract можно направлять для утверждения. Не все должности требуют утверждения. Требование устанавливается на уровне шаблона. По умолчанию в шаблоне утверждения отключены. Для настройки утверждений перейдите в шаблон процесса и установите в поле **Утверждение** значение по умолчанию. Затем выберите этот шаблон при создании должности.

После сохранения должности она может быть отправлена на утверждение. В следующей таблице перечислены статусы документа, который использует утверждения.

| Состояние   | Область, край                                                               |
|----------|---------------------------------------------------------------------|
| Черновой    | Должность была сохранена, но еще не был отправлена в workflow-процесс. |
| Ожидание  | Должность отправлена утверждающим.                            |
| Одобрено | Должность утверждена, но еще не была активирована.            |
| Отклонено | Должность была отклонена, и ее невозможно активировать.               |
| Активные   | Должность утверждена и активирована.                            |

В списке должностей можно выполнить фильтрацию по статусам должностей.

Утверждения могут отправлять любому пользователю Microsoft Azure Active Directory (Azure AD) в компании. Утверждения отправляются в параллельном режиме всем пользователям, указанным как утверждающие. Все утверждающие лица должны утвердить должность, прежде чем можно будет перейти к следующему этапу. Если один утверждающий отклоняет должность, должность будет отображаться с состоянием **Отклонено**. После утверждения должности ее можно активировать.

Если пользователь редактирует должность после того как она утверждена, но не активирована, статус должности будут сброшен на **Черновик**, и должность следует заново представить для утверждения. После активации утвержденной должности ее невозможно изменить.

Люди, которые указаны как утверждающие, будут получать уведомление в Attract и по электронной почте, информирующее о том, что у него есть элемент на утверждении.  В сообщении электронной почты утверждающие лица могут щелкнуть ссылку, чтобы открыть должность, просмотреть сведения и либо утвердить, либо отклонить. После того как статус должности установлен на **Утверждено** или **Отклонено**, отправитель уведомляется в Attract и получает сообщение электронной почты. Кроме того, утверждающие лица будут получать оповещения по электронной почте, если они не ответили на запрос утверждения в течение 24 часов.

> [!NOTE]
> Можно создать пользовательские шаблоны электронной почты для сообщений электронной почты об утверждении. Дополнительные сведения см. в разделе [Создание шаблонов электронной почты и управление ими](https://docs.microsoft.com/dynamics365/unified-operations/talent/email-templates).

## <a name="create-a-job"></a>Создание задания

Чтобы создать должность, выполните следующие действия.

1. Перейти в раздел **Должности**.
2. Выберите **Создать**.
3. В поле **Название должности** введите название должности. В поле **Роль** введите свою роль.
4. В поле **Шаблон** выберите шаблон. Или же выберите **Пропустить**. При выборе **Пропустить** используется шаблон, который помечен как шаблон по умолчанию.

    Если документ должен пройти через процесс утверждения, выберите шаблон, в котором в поле **Процесс утверждения** установлено значение **По умолчанию**.

5. На вкладке **Сведения** введите сведения о должности. Поля **Название**, **Описание должности** и **Местоположение** являются обязательными.
6. Выберите **Сохранить**.
7. На вкладке **Группа комплектации штата** добавьте менеджера по найму, сотрудника по найму или сотрудника, проводящего собеседования.
8. Выберите **Сохранить**.
9. На вкладке **Процесс** добавьте или удалите стадии в соответствии с требованиями:

    - Чтобы добавить стадию выберите **+ Создать стадию**.
    - Чтобы удалить стадию, наведите указатель на удаляемую стадию, затем выберите кнопку корзины, которая появится.

        > [!NOTE]
        > Стадии "Соискатель", "Заявление" и "Предложение" удалить невозможно.

10. Добавьте или удалите действия по мере необходимости:

    - Чтобы добавить действие, перетащите его из списка справа на соответствующую стадию. Можно также дважды щелкнуть действие, затем выбирать стадию, в которую его требуется добавить.
    - Чтобы удалить действие, разверните действие, затем выберите кнопку корзины в заголовке действия.

11. Выберите **Сохранить**.
12. Если вы выбрали применение процесса утверждения, выполните следующие действия:

    1. Выберите **+ Добавить утверждающего**, затем введите пользователь, имеющего учетную запись Azure AD. Можно добавить несколько утверждающих лиц.
    2. Выберите **Отправить утверждающим**.

    В поле **Статус должности** для должности установлено значение **Ожидание**. После того как значение в поле **Статус должности** изменится на **Утверждено**, должность может быть активирована.

13. Чтобы активировать должность, выберите **Активировать**.
14. Чтобы разместить объявление о вакансии, перейдите к **Объявления**, затем выберите **Опубликовать** для сайта карьеры Talent или LinkedIn.
