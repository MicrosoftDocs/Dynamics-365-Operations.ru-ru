# <a name="set-user-permissions-in-attract"></a>Установка разрешений для пользователя в Attract

[!include [banner](includes/banner.md)]

Microsoft Dynamics 365 Talent: Attract использует безопасность на основе ролей. Другими словами, доступ предоставлен не для отдельных пользователей, а для ролей безопасности, которые назначены пользователям. Пользователь, которому назначена роль безопасности, имеет доступ к набору полномочий, связанный с этой ролью.

В Attract есть пять базовых ролей пользователей:

- Администратор
- Специалист по комплектации штата
- Специалист по найму
- Проводящий собеседование
- Только для чтения

Роль администратора — единственная роль, имеющая разрешение на добавление других пользователей и изменение их разрешений.

- **Добавить** — в центре администрирования на вкладке **Разрешения пользователя** выберите **Назначить роли**, найдите пользователей, которых требуется добавить, затем назначьте разрешения для этого пользователя.
- **Изменить** — выполните поиск пользователя или найдите пользователя в списке, затем выберите **Изменить**, чтобы изменить его разрешения.
- **Удалить** — при удалении разрешений пользователя вы не удаляете пользователя из системы. Тем не менее, вы ограничиваете доступ пользователя и права в Attract. Например, Хильде была назначена роль менеджера по приему на работу, и она добавлена к должности как менеджер по приему на работу. Если впоследствии у Хильды была удалена роль менеджера по приему на работу, она остается менеджеров по приему на работу для этой конкретной должности и по-прежнему имеет доступ к этой должности. Однако она не может создавать другие должности.

В следующих разделах приведено высокоуровневое описание каждой роли. В таблицах далее в этом разделе представлены более подробные сведения.

> [!NOTE]
> Некоторые функции доступны только с надстройкой Comprehensive Hiring для Attract.

## <a name="administrator"></a>Администратор

Пользователи, которым назначена роль администратора, имеют доступ ко всем данным в Attract и могут изменять эти данные. Администраторы могут создавать, читать, обновлять и удалять данные. Они также имеют доступ к центру администрирования, где могут настроить Attract и настроить сведения о пользователе. Рекомендуется назначить роль администратора хотя бы одному сотруднику. По умолчанию администратор среды в Microsoft Power Apps устанавливается как администратор в Attract. Если вы зарегистрировались для пробной версии Attract, вам автоматически назначается роль администратора. В настоящее время для создания должностей пользователи, которым назначена роль администратора, должны также иметь роль агента по найму персонала или роль менеджера по найму персонала.

## <a name="hiring-manager"></a>Специалист по комплектации штата

Пользователи, которым назначена роль специалиста по комплектации штата, могут создавать должности и обновлять должности, которые они ранее создали. Специалисты по комплектации штата могут выполнять только ограниченный набор действий по должности и заявлениям, которые связаны с этой должностью. Только те пользователи, которым назначена роль специалиста по комплектации штата, могут быть добавлены в группу найма на работу в качестве специалистов по комплектации штата.

## <a name="recruiter-role"></a>Роль специалиста по найму

Пользователи, которым назначена роль специалиста по найму, имеют полные права чтения, создания, обновления и удаления для должностей, которые они создали. Они также имеют полные права на создание, чтение, обновление и удаление для заявлений, которые связаны с принадлежащим им должностями. Только те пользователи, которым назначена роль специалиста по найму, могут быть добавлены в группу найма на работу в качестве специалистов по найму.

## <a name="interviewer"></a>Проводящий собеседование

Любого пользователя, имеющего учетную запись Microsoft Azure Active Directory (Azure AD) в организации, можно добавить в группу найма в качестве проводящего собеседование. Пользователи, которым назначена роль проводящего собеседование, могут просматривать сведения о должности и данные кандидата для должностей, для которых они включены в группу найма. Для этих должностей сотрудники, проводящие собеседование, могут также давать рекомендации по найму и предоставлять отзывы о кандидатах. Тем не менее, они не могут обновлять сведения о должности или данные кандидата.

## <a name="read-only"></a>Только для чтения

Пользователи, которым назначена роль только для чтения, имеют доступ только для чтения ко всем данным в среде Attract. Тем не менее, они не могут создавать или изменять какие-либо данные.

## <a name="delegated-roles"></a>Делегированные роли

Для каждой должности, для которой они включены в группу найма, агенты и менеджеры по найму могут назначить одного или несколько представителей для себя. Тем не менее, они не могут назначить делегатов для других пользователей в группе найма на работу.

Делегаты имеют те же права, что и сотрудник, назначивший их. Например, если менеджер по найму назначает делегата для себя для должности, делегат будет иметь те же права, что и менеджер по найму на работу для этой должности. Делегаты не могут удалять других делегатов из команды по найму. Они также не могут удалить пользователя, который назначил их как делегатов.

## <a name="job-details-and-actions"></a>Сведения о должности и действия

Пользователи, имеющие роль агента или менеджера по найму персонала, могут создавать должности. Следующие привилегии применяются к сведениям о должности и действиям, которые можно выполнить для должностей.

| Данные или действие    | Специалист по найму | Специалист по комплектации штата | Проводящий собеседование |
|-------------------|-----------|----------------|-------------|
| Сведения о вакансии       | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения |
| Группа комплектации штата       | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения |
| Объявление о приеме на работу       | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения | Только для чтения |
| Обработка           | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения |
| Добавить кандидатов    | Добавление кандидатов для должностей, для которых пользователь входит в группу найма | Добавление кандидатов для должностей, для которых пользователь входит в группу найма | Не разрешенный |
| Добавить соискателей     | Добавление соискателей для должностей, для которых пользователь входит в группу найма | Параметр конфигурации в [настройке действий для соискателей](./activities-attract.md#prospect-activity) управляет тем, могут ли сотрудники, проводящие собеседование, добавлять и просматривать соискателей. | Не разрешенный |
| Активировать должность      | Активирование должностей, для которых пользователь входит в группу найма | Активирование должностей, для которых пользователь входит в группу найма | Не разрешенный |
| Закрыть должность         | Закрытие должностей, для которых пользователь входит в группу найма | Не разрешенный | Не разрешенный |
| Удалить должность        | Удаление должностей, для которых пользователь входит в группу найма | Только в том случае, если нет кандидатов, добавленных к должности | Не разрешенный |
| Удалить данные кандидатов | Удаление соискателей, если для них пользователь входит в группу найма | Не разрешенный | Не разрешенный |

## <a name="application-details-and-actions"></a>Сведения о заявлении и действия

Следующие привилегии применяются к данным о должности для соискателей и действиям, которые можно выполнить для соискателей.

| Данные или действие          | Специалист по найму | Специалист по комплектации штата | Проводящий собеседование |
|-------------------------|-----------|----------------|-------------|
| Документы заявлений   | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения |
| Примечания к заявлениям       | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Создание, чтение, обновление и удаление для должностей, для которых пользователь входит в группу найма на работу | Только для чтения|
| Действие по заявлению    | Просмотр, если пользователь входит в группу найма | Просмотр, если пользователь входит в группу найма | Только для чтения |
| Отзыв на заявление    | Добавление и просмотр отзывов,если пользователь входит в группу по найму | Добавление и просмотр отзывов,если пользователь входит в группу по найму | Можно добавить отзыв\*\* |
| Отклонить заявление      | Может отклонить, если пользователь входит в группу найма | Не разрешенный | Не разрешенный |
| Перейти на следующий этап           | Может отклонить, если пользователь входит в группу найма | Может перевести на следующий этап, если пользователь входит в группу найма | Не разрешенный |
| Запуск управления предложением | Можно запустить управление предложением | Имеется параметр конфигурации для действия предложения. | Не разрешенный |


\*\* Параметр конфигурации в [настройке действия отзыва](./activities-attract.md) управляет тем, могут ли сотрудники, проводящие собеседование, видеть отзывы друг друга.


## <a name="process-templates"></a>Шаблоны процесса

Следующие привилегии применяются к шаблонам процесса приема на работу. Способность участников группы создавать и изменять шаблоны настраивается в разделе **Управление шаблонами** в центре администрирования. Если включено управление шаблонами, агенты и менеджеры по найму кадров могут создавать и изменять свои собственные шаблоны процессов. Если управление шаблонами отключено, только администраторы могут создавать и изменять шаблоны процессов. Следующая таблица предполагает, что управление шаблонами было включено.

| Данные              | Специалист по найму                                           | Специалист по комплектации штата                                      | Проводящий собеседование |
|-------------------|-----------------------------------------------------|-----------------------------------------------------|-------------|
| Шаблоны процесса | Полные права для шаблонов, создаваемых пользователем | Полные права для шаблонов, создаваемых пользователем | Нет доступа   |

## <a name="email-and-email-templates"></a>Электронная почта и шаблоны электронной почты

Следующие привилегии применяются к шаблонам электронной почты и действиям, которые можно выполнить для сообщений электронной почты. Только администраторы могут создавать и изменять шаблоны электронной почты.

| Данные или действие     | Специалист по найму          | Специалист по комплектации штата     | Проводящий собеседование |
|--------------------|--------------------|--------------------|-------------|
| Шаблоны сообщений эл. почты    | Доступ только для чтения   | Доступ только для чтения   | Нет доступа   |
| Отправить сообщение электронной почты         | Отправить в соответствии с действием  | Отправить в соответствии с действием  | Нет доступа   |
| Изменить содержимое сообщения электронной почты | Изменить содержимое сообщения электронной почты | Изменить содержимое сообщения электронной почты | Нет доступа   |

## <a name="talent-pools"></a>Кадровые пулы

Следующие привилегии применяются к кадровым пулам. Кадровые пулы видны только лицу, создавшему их, если только этот пользователь не откройте общий доступ к ним. Поиск кандидатов может использоваться для поиска кандидатов, которые еще не были добавлены в именованный пул.

| Данные или действие   | Специалист по найму                                       | Специалист по комплектации штата                                  | Проводящий собеседование |
|------------------|-------------------------------------------------|-------------------------------------------------|-------------|
| Именованный пул       | Полные права для пулов, создаваемых пользователем | Полные права для пулов, создаваемых пользователем | Нет доступа   |
| Общий доступ к пулу       | Только пулы, созданные пользователем                | Только пулы, созданные пользователем                | Нет доступа   |
| Поиск кандидатов | Полные возможности поиска                        | Полные возможности поиска                        | Нет доступа   |

## <a name="candidates"></a>Кандидаты

Кандидаты — это люди, которые были добавлены в кадровый пул, но не связаны с должностью.

| Данные                        | Специалист по найму                        | Специалист по комплектации штата                   | Проводящий собеседование |
|-----------------------------|----------------------------------|----------------------------------|-------------|
| Профиль — сведения о кандидате | Создание, чтение, обновление и удаление | Создание, чтение, обновление и удаление | Нет доступа   |
| Документы                   | Создание, чтение, обновление и удаление | Создание, чтение, обновление и удаление | Нет доступа   |
