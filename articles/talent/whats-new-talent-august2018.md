# <a name="whats-new-or-changed-in-dynamics-365-talent-core-hr-august-2018"></a>Что нового и что изменилось в Dynamics 365 Talent: Core HR (август 2018 г.)

[!include [banner](includes/banner.md)]

**Сборка 8.1.104**

В этой теме описываются новые и измененные компоненты Dynamics 365 Talent: Core HR.

## <a name="view-expiring-records-in-manager-self-service"></a>Просмотр записей с истекающим сроком действия на портале самообслуживания менеджеров

На портале самообслуживания менеджеров теперь можно просматривать записи с истекающим сроком действия. Предусмотрены новые параметры для настройки того, какая информация будет доступна для просмотра менеджерами. К ним относятся:

-   Сертификаты

-   Идентификационные номера

-   Периоды надзора

-   Отборы

-   Тесты

Эта функция также дает возможность указать диапазон дней, в пределах которого следует искать записи с истекающим сроком действия.

## <a name="configurable-transfer-actions"></a>Конфигурируемые действия перемещения

Можно сконфигурировать по ролям параметры, доступные при вводе запроса на перемещение. Это обеспечивает дополнительную гибкость для различных ролей в организации.

Например, руководители, которые запрашивают перемещения сотрудников, могут не иметь доступа, чтобы предложить или ввести суммы компенсации или выбрать списки задач, которые будут связаны с запросом на перемещение. В этом случае руководители могут создавать и отправлять запросы на перемещения, но вводить компенсации или назначать списки задач им не разрешено. В этой же конфигурации HR-специалисты смогут назначить новые значения компенсации, а также назначить дополнительные контрольные списки для заполнения в результате выполнения перемещения.

По умолчанию новые параметры конфигурации заданы так, чтобы не изменять возможности, имевшиеся до этого обновления.

## <a name="leave-and-absence"></a>Отпуск и отсутствие

В компоненте "Отпуск и отсутствие" теперь есть дополнительные поля дат.

С их помощью можно задавать основание периода начисления на уровне плана так, чтобы использовать конкретные даты, связанные с самим сотрудником. Это позволяет использовать в процессе начисления отпусков даты, отличные от начальной даты плана. Параметры дат, связанных с сотрудниками, включают в себя следующие значения:

-   Произвольная (было доступно до этого обновления)

-   Дата годовщины

-   Дата первоначального приема на работу

-   Дата начала стажа

-   Скорректированная дата приема работника на работу

-   Дата приема работника на работу

Будучи настроенным для использования одной из дат, связанных с конкретным сотрудником, процесс регистрации будут использовать указанную дату для расчета периодов начислений. Основание периода начисления также отображается в регистрации сотрудника в плане, чтобы вы понимали, что именно используется в процессе начисления.

## <a name="microsoft-word-integration"></a>Интеграция с Microsoft Word

В Core HR теперь можно использовать шаблоны документов. Это позволяет создавать письма и отчеты на основе созданных вами шаблонов Word.

Дополнительные сведения об этой возможности доступны в [учебнике по интеграции с Office](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/office-integration/office-integration-tutorial?toc=/dynamics365/unified-operations/talent/toc.json).
