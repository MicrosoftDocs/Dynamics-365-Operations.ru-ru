# <a name="vendor-payment-overview"></a><span data-ttu-id="404f6-103">Обзор платежей поставщикам</span><span class="sxs-lookup"><span data-stu-id="404f6-103">Vendor payment overview</span></span>

[!include [banner](../../includes/banner.md)]

<span data-ttu-id="404f6-104">В этом руководстве по задаче демонстрируются различные методы создания платежей поставщикам, в том числе с использованием предложения по оплате и путем ввода разового платежа.</span><span class="sxs-lookup"><span data-stu-id="404f6-104">This task guide will walk you through various methods used to create vendor payments, including how to use a payment proposal or manually entering a one-off payment.</span></span> <span data-ttu-id="404f6-105">В данной процедуре используется демонстрационная компания USMF.</span><span class="sxs-lookup"><span data-stu-id="404f6-105">This procedure uses the USMF demo company.</span></span>

1. <span data-ttu-id="404f6-106">Выберите **Область перехода > Модули > Расчеты с поставщиками > Платежи > Журнал платежей**.</span><span class="sxs-lookup"><span data-stu-id="404f6-106">Go to **Navigation pane > Modules > Accounts payable > Payments > Payment journal**.</span></span>
2. <span data-ttu-id="404f6-107">Нажмите кнопку **Создать**.</span><span class="sxs-lookup"><span data-stu-id="404f6-107">Click **New**.</span></span>
3. <span data-ttu-id="404f6-108">Выберите журнал платежей, в котором будут сохранены платежи поставщикам.</span><span class="sxs-lookup"><span data-stu-id="404f6-108">Select the payment journal in which to save the vendor payments.</span></span> 
4. <span data-ttu-id="404f6-109">Выберите журнал или введите его вручную.</span><span class="sxs-lookup"><span data-stu-id="404f6-109">Select the journal or manually enter it.</span></span>
5. <span data-ttu-id="404f6-110">Выберите **Строки**.</span><span class="sxs-lookup"><span data-stu-id="404f6-110">Click **Lines**.</span></span>
6. <span data-ttu-id="404f6-111">На **панели операций** щелкните **Предложение по оплате**.</span><span class="sxs-lookup"><span data-stu-id="404f6-111">On **Ation pane**, click **Payment proposal**.</span></span>
7. <span data-ttu-id="404f6-112">Щелкните **Создать предложение по оплате**.</span><span class="sxs-lookup"><span data-stu-id="404f6-112">Click **Create payment proposal**.</span></span> <span data-ttu-id="404f6-113">Предложение по оплате — это запрос, используемый для выбора накладных для оплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-113">The payment proposal is a query used to select invoices for payment.</span></span> <span data-ttu-id="404f6-114">Список оплачиваемых накладных можно изменить, прежде чем создавать или автоматически формировать платежи поставщикам.</span><span class="sxs-lookup"><span data-stu-id="404f6-114">You can edit the list of invoices to pay before creating or generating the vendor payments.</span></span>
8. <span data-ttu-id="404f6-115">Выберите накладные для оплаты по сроку оплате, скидке по оплате или обоим этим параметрам.</span><span class="sxs-lookup"><span data-stu-id="404f6-115">Select invoices for payment by due date, cash discount, or both.</span></span> 
9. <span data-ttu-id="404f6-116">Введите дату для сравнения со сроком оплаты или скидкой по оплате.</span><span class="sxs-lookup"><span data-stu-id="404f6-116">Enter the date for comparing to the due date or cash discount.</span></span> 
10. <span data-ttu-id="404f6-117">Необязательно: введите дату оплаты, используемую для всех платежей.</span><span class="sxs-lookup"><span data-stu-id="404f6-117">Optional: Enter a payment date used for all payments.</span></span> <span data-ttu-id="404f6-118">Введенная здесь дата будет датой оплаты для всех созданных платежей, независимо от срока оплаты или срока скидки по оплате.</span><span class="sxs-lookup"><span data-stu-id="404f6-118">The date entered here will be the payment date for all payments created, regardless of the due date or cash discount date.</span></span>  
11. <span data-ttu-id="404f6-119">Необязательно: введите минимальную дату платежа, которая может использоваться в качестве даты платежа.</span><span class="sxs-lookup"><span data-stu-id="404f6-119">Optional: Enter a minimum payment date which may be used as the payment date.</span></span> <span data-ttu-id="404f6-120">Минимальной датой платежа будет самая ранняя дата, используемая при создании платежей.</span><span class="sxs-lookup"><span data-stu-id="404f6-120">The minimum payment date will be the earliest date used when creating payments.</span></span> <span data-ttu-id="404f6-121">Например, если дата наступления срока оплаты накладной позже, чем минимальная дата платежа, в качестве даты платежа будет использоваться дата наступления срока оплаты, а не минимальная дата платежа, чтобы накладная была оплачена как можно позже.</span><span class="sxs-lookup"><span data-stu-id="404f6-121">For example, if an invoice has a due date after the minimum payment date, the due date will become the payment date instead of the minimum payment date in order to pay the invoice on the latest possible date.</span></span>
12. <span data-ttu-id="404f6-122">Введите дополнительные ограничения на запрос в разделе **Записи для добавления**.</span><span class="sxs-lookup"><span data-stu-id="404f6-122">Enter additional query restrictions under **Records to include** section.</span></span> <span data-ttu-id="404f6-123">Фильтр часто используется для ограничения выбираемых для оплаты накладных по группе поставщиков или по способу оплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-123">The filter is often used to restrict the invoices selected for payment by vendor group or method of payment.</span></span> <span data-ttu-id="404f6-124">Например, можно добавить фильтр, чтобы в рамках этой выплаты были оплачены только накладные с оплатой чеком.</span><span class="sxs-lookup"><span data-stu-id="404f6-124">For example, you may add a filter to only pay invoices by check in this pay run.</span></span>
13. <span data-ttu-id="404f6-125">Введите дополнительные ограничения на запрос или данные платежа по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="404f6-125">Enter additional query restriction or payment defaults.</span></span> <span data-ttu-id="404f6-126">Дополнительные параметры можно использовать для определения валюты платежа или для включения централизованных платежей для этой выплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-126">The additional parameters can be used to define the payment currency or to enable centralized payments for this pay run.</span></span>  
14. <span data-ttu-id="404f6-127">Щелкните **OK**.</span><span class="sxs-lookup"><span data-stu-id="404f6-127">Click **OK**.</span></span> <span data-ttu-id="404f6-128">После нажатия кнопки **ОК** появятся результаты запроса.</span><span class="sxs-lookup"><span data-stu-id="404f6-128">After clicking **OK**, the results of the query will appear.</span></span> <span data-ttu-id="404f6-129">Если предварительно просматривать список выбранных для оплаты накладных не требуется, можно вернуться на экспресс-вкладку **Параметры** и изменить значение параметра **Создать платежи без предварительного просмотра накладной** на "Да".</span><span class="sxs-lookup"><span data-stu-id="404f6-129">If you don't want to preview the list of invoices selected to pay, you can go back to the **Parameters** fast tab and change the setting **Create payments without invoice preview** to "Yes".</span></span>  
15. <span data-ttu-id="404f6-130">Нажмите кнопку **Показать обзор платежей**, чтобы просмотреть платежи, которые будут созданы для поставщика по выбранной накладной.</span><span class="sxs-lookup"><span data-stu-id="404f6-130">Choose the **Show payment overview** button to view the payments that will be created for the vendor on the invoice selected.</span></span>
16. <span data-ttu-id="404f6-131">Нажмите **Скрыть обзор платежей**, чтобы скрыть платежи.</span><span class="sxs-lookup"><span data-stu-id="404f6-131">Choose the **Hide payment overview** button to hide the payments.</span></span> 
17. <span data-ttu-id="404f6-132">Щелкните **Создать платежи**.</span><span class="sxs-lookup"><span data-stu-id="404f6-132">Click **Create payments**.</span></span> <span data-ttu-id="404f6-133">Прежде чем нажимать кнопку **Создать платежи**, можно щелкнуть на сетке правой кнопкой мыши и экспортировать список накладных в Excel.</span><span class="sxs-lookup"><span data-stu-id="404f6-133">Before choosing **Create payments**, you can right click on the grid and export the list of invoices to Excel.</span></span> <span data-ttu-id="404f6-134">При нажатии кнопки **Создать платежи** создаются платежи поставщикам в журнале платежей.</span><span class="sxs-lookup"><span data-stu-id="404f6-134">The **Create payments** button will create the vendor payments in the payment journal.</span></span>  
18. <span data-ttu-id="404f6-135">Просмотрите платежи и убедитесь, что для всех платежей указан способ оплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-135">Scan your payments and make sure the method of payment is defined for all payments.</span></span> <span data-ttu-id="404f6-136">При формировании платежей, например печати чека или создании электронного платежа, необходимо определить способ оплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-136">If you generate the payments, such as printing a check or creating an electronic payment, the method of payment must be defined.</span></span> <span data-ttu-id="404f6-137">Способ оплаты также будет определять банковский счет, с которого по умолчанию будет совершаться платеж.</span><span class="sxs-lookup"><span data-stu-id="404f6-137">The method of payment will also default the bank account from the payment will be made.</span></span>  
19. <span data-ttu-id="404f6-138">Щелкните **Создать**, чтобы создать разовый платеж.</span><span class="sxs-lookup"><span data-stu-id="404f6-138">Click **New** to create a one-off payment.</span></span> <span data-ttu-id="404f6-139">Разовый платеж можно добавить в журнал платежей в любое время до разноски.</span><span class="sxs-lookup"><span data-stu-id="404f6-139">A one-off payment can be added to a payment journal at any time prior to posting.</span></span> <span data-ttu-id="404f6-140">Это можно сделать, нажав кнопку **Создать** и добавив информацию о платеже вручную, а не помощью пункта **Предложение по оплате**.</span><span class="sxs-lookup"><span data-stu-id="404f6-140">This is done by clicking the **New** button and adding the payment information manually, rather than using the **Payment proposal**.</span></span>  
20. <span data-ttu-id="404f6-141">Выберите поставщика, оплату котором требуется произвести.</span><span class="sxs-lookup"><span data-stu-id="404f6-141">Select the vendor to whom the payment will be made.</span></span>
21. <span data-ttu-id="404f6-142">Если есть накладная для оплаты, выберите **Сопоставление проводок**, чтобы выбрать накладную для оплаты.</span><span class="sxs-lookup"><span data-stu-id="404f6-142">If an invoice exists to pay, select **Settle transactions** to select the invoice for payment.</span></span> <span data-ttu-id="404f6-143">Если это предоплата, этот шаг является необязательным.</span><span class="sxs-lookup"><span data-stu-id="404f6-143">If this is a prepayment, this step is optional.</span></span> <span data-ttu-id="404f6-144">Можно создать платеж без выбора какой-либо накладной.</span><span class="sxs-lookup"><span data-stu-id="404f6-144">You can create the payment without selecting any invoice.</span></span> 
22. <span data-ttu-id="404f6-145">Пометьте все накладные, которые будут оплачены.</span><span class="sxs-lookup"><span data-stu-id="404f6-145">Mark any invoices that will be paid.</span></span> <span data-ttu-id="404f6-146">Если для выбора накладных для оплаты используется **Сопоставление проводок**, сумма платежа будет автоматически рассчитана на основании того, какие накладные вы пометили для оплаты, а также какую сумму вы ввели в поле **Сумма сопоставления**.</span><span class="sxs-lookup"><span data-stu-id="404f6-146">If you use the **Settle transactions** to select the invoices for payment, the payment amount will automatically be calculated based on what invoices you mark for payment, and what amount you enter in the **Amount to settle**.</span></span>
23. <span data-ttu-id="404f6-147">Щелкните **OK**.</span><span class="sxs-lookup"><span data-stu-id="404f6-147">Click **OK**.</span></span>
24. <span data-ttu-id="404f6-148">Если необходимо удалить платеж, отметьте соответствующую строку.</span><span class="sxs-lookup"><span data-stu-id="404f6-148">If you want to delete a payment, mark the row.</span></span>
25. <span data-ttu-id="404f6-149">Нажмите кнопку **Удалить**.</span><span class="sxs-lookup"><span data-stu-id="404f6-149">Click **Delete**.</span></span> <span data-ttu-id="404f6-150">При удалении платежа будет удален только платеж.</span><span class="sxs-lookup"><span data-stu-id="404f6-150">Deleting a payment will only delete the payment.</span></span> <span data-ttu-id="404f6-151">Все накладные, помеченные для оплаты, останутся, и их можно будет оплатить другим платежом.</span><span class="sxs-lookup"><span data-stu-id="404f6-151">Any invoices marked for payment will still be available to be paid by another payment.</span></span>
26. <span data-ttu-id="404f6-152">Нажмите кнопку **Да**.</span><span class="sxs-lookup"><span data-stu-id="404f6-152">Click **Yes**.</span></span>
27. <span data-ttu-id="404f6-153">Выберите **Создание платежей**, чтобы напечатать чеки или создать файл электронного платежа.</span><span class="sxs-lookup"><span data-stu-id="404f6-153">Choose **Generate payment** to print checks or create the electronic payment file.</span></span>
28. <span data-ttu-id="404f6-154">Выберите способ оплаты для формируемых платежей.</span><span class="sxs-lookup"><span data-stu-id="404f6-154">Select the method of payment that you want to generate.</span></span> <span data-ttu-id="404f6-155">Журнал платежей может содержать и чековые, и электронные платежи, однако за один раз можно сформировать платежи только одного типа.</span><span class="sxs-lookup"><span data-stu-id="404f6-155">The payment journal can contain payments for both Checks and electronic payments, but you can only generate one payment type at a time.</span></span>
29. <span data-ttu-id="404f6-156">Выберите банковский счет, с которого будут осуществлены платежи.</span><span class="sxs-lookup"><span data-stu-id="404f6-156">Select the bank account from which to generate the payments.</span></span>
30. <span data-ttu-id="404f6-157">Щелкните **OK**.</span><span class="sxs-lookup"><span data-stu-id="404f6-157">Click **OK**.</span></span> <span data-ttu-id="404f6-158">Платежи будут сформированы только для платежей, которые соответствуют выбранным параметрам **Способ оплаты** и **Банковский счет**.</span><span class="sxs-lookup"><span data-stu-id="404f6-158">Payments will only be generated for payments that match the **Method of payment** and **Bank account** you selected.</span></span>
31. <span data-ttu-id="404f6-159">Если вы формируете **Чеки**, выберите **Документ**, чтобы проверить правильность выбора устройства печати для чеков.</span><span class="sxs-lookup"><span data-stu-id="404f6-159">If you are generating **Checks**, choose **Document** to ensure the correct print destination for the checks.</span></span>
32. <span data-ttu-id="404f6-160">Щелкните **OK**.</span><span class="sxs-lookup"><span data-stu-id="404f6-160">Click **OK**.</span></span>
33. <span data-ttu-id="404f6-161">Нажмите **ОК**, чтобы сформировать платежи.</span><span class="sxs-lookup"><span data-stu-id="404f6-161">Click **OK** to generate the payments.</span></span>
34. <span data-ttu-id="404f6-162">Щелкните **Разнести**, если все платежи утверждены и сформированы.</span><span class="sxs-lookup"><span data-stu-id="404f6-162">Click **Post** if all the payments are approved and generated.</span></span>
