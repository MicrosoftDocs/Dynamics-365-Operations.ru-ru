# <a name="invoice-capture-solution-dashboard"></a>Панель мониторинга решения Invoice Capture

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В Invoice Capture панель мониторинга включает диаграммы, предоставляющие обзор импортированных накладных. Эти диаграммы могут помочь менеджеру расчетов с поставщиками (AP) анализировать производительность процесса создания накладных. Менеджер AP может просматривать статус процесса создания накладных и, используя различные фильтры, также может просматривать подробные сведения.

## <a name="available-charts"></a>Доступные диаграммы

На панели мониторинга доступны следующие диаграммы:

- **Все записанные накладные по статусу** — на этой диаграмме показаны следующие статусы записанной накладной. Выбор статуса позволяет пользователям отфильтровать записанные накладные в подробном списке.

    - Записано
    - Завершить
    - На рассмотрении
    - Устарело

- **Общий объем записанных накладных** — на этой диаграмме показано число записанных накладных по периодам. Пользователи могут изменить период, используя раскрывающийся список.
- **Записанная накладная от лучших поставщиков** — на этой диаграмме показано общее число накладных по поставщикам. В верхней части отображаются поставщики с наибольшим числом накладных.
- **Дней для завершения по накладным с исключениями** — на этой диаграмме показано среднее число дней, необходимое для регистрации одной накладной. Эта информация может помочь менеджеру AP анализировать время обработки накладной, когда требуется вмешательство вручную. Если имеется тенденция возрастания, пользователи могут просмотреть сведения о процессе и скорректировать настройки, чтобы уменьшить количество требуемых дней.
- **Незавершенные накладные по времени ожидания** — на этой диаграмме показано количество дней, в течение которых записанная накладная не была создана в системе планирования ресурсов предприятия (ERP). Чем больше число дней ожидания, тем дольше процесс создания накладной. Менеджер AP может детализировать сведения в детализированных записанных накладных и создать накладные в системе ERP.
