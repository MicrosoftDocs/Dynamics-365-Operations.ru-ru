# <a name="single-voucher-with-multiple-customer-or-vendor-records"></a>Один ваучер с несколькими записями клиента или поставщика

[!include [banner](../includes/banner.md)]

В этой статье содержится обзор того, что происходит при разноске одного ваучера с несколькими записями клиентов или поставщиков. Эта функция не будет поддерживаться в будущих версиях, поэтому не рекомендуется использовать этот метод разноски из-за влияния учета на обработку сопоставления. 

Некоторые общие примеры, когда один ваучер используется для нескольких клиентов или поставщиков, включают перемещения сальдо между клиентами и взаимозачет сальдо между клиентами и поставщиками в одной организации. 

Ваучер, содержащий более одного клиента или поставщика, можно ввести одним из следующих способов:

-   Используя журнал с выбранным параметром **Только один код ваучера**, чтобы каждая добавленная строка в журнал включалась в один и тот же ваучер.
-   Используя ваучер с несколькими строками, где отсутствует корр. счет ГК, с более чем одним клиентом или поставщиком.
-   Введя ваучер со счетом и корр. счетом, относящимися к поставщику/поставщику, клиенту/клиенту, поставщику/клиенту или клиенту/поставщику.

В этой статье показано, как будет обработано сопоставление, когда будет разнесен один ваучер с несколькими записями клиентов или поставщиков. Кроме того, в этой статье содержатся временные решения, которые помогут понять, как избежать использования одного ваучера с несколькими клиентами или поставщиками. В частности, имеются примеры, которые иллюстрируют два общих сценария сопоставления, на которые влияет использование одного ваучера с несколькими клиентами или поставщиками:

-   Учет скидки по оплате
-   Учет переоценки

## <a name="how-does-settlement-impact-single-voucher-usage"></a>Как сопоставление влияет на использование одного ваучера
При разноске ваучера, содержащего несколько записей клиента или поставщика, один ваучер учета создается, который содержит несколько сальдо расчетов с клиентами или расчетов с поставщиками. Во время процесса сопоставления исходные учетные записи используются для создания учетных записей для скидок по оплате, внереализационных прибылей и убытков, реализованных прибылей и убытков и освобождения итогового счета исходного документа. Например, если учитывается скидка по оплате при сопоставлении платежа поставщику по накладной, учет скидки по оплате должен разноситься на счет ГК расчетов с поставщиками из исходной накладной. Если исходная накладная была разнесена в ваучер, содержащей записи нескольких поставщиков, исходный учет суммируется. В этом случае, поскольку нет возможности получать доступ к детальной учетной записи для каждой проводки поставщика в одном ваучере, нет возможности определить, как пользователь намеревался учитывать скидку при оплате.

### <a name="one-voucher-with-multiple-vendors-and-the-impact-on-cash-discount-accounting"></a>Один ваучер с несколькими поставщиками и влияние на учет скидки по оплате

В следующем примере нескольких накладных поставщиков записаны в ГК в одном ваучере на странице **Общий журнал**. Эти накладные распределены между несколькими аналитиками счета.

| GNJL001     | Главная книга           | 606300-001-- | INV1            |   50,00   |            |
| GNJL001     | Главная книга           | 606300-002-- | INV1            |   50,00   |            |

| GNJL001     | Главная книга           | 606300-004-- | INV3            | 300,00    |            |

| Ваучер | Счет  | Тип разноски | Сумма в валюте проводки |

Нет возможности указать, что дебет 50,00 для 606300-001 и дебет 50,00 для 606300-002 предназначены для зачета записи сальдо поставщика 200110-001. Это связано с тем, что пользователь ввел несколько записей поставщиков в один ваучер.

Это связано с тем, что один ваучер был использован для разноски вышеуказанной накладной, без указания того, как пользователь хотел связать распределение расходов с сальдо поставщика в одном ваучере.

## <a name="one-voucher-for-balance-transfers-and-netting-scenarios"></a>Один ваучер для сценариев перемещения сальдо и взаимозачета
Два часто используемых сценария, который использует один ваучер, который содержит несколько клиентов или поставщиков, включает перемещение сальдо от одного клиента или поставщика к другому клиенту или поставщику, и взаимозачеты клиента и поставщика из одной организации. Следующие два примера иллюстрируют предпочтительный метод ввода этих сценариев в качестве альтернативы вводу их в один ваучер. 

*Перемещение сальдо* представляет собой один ваучер с несколькими клиентами, введенный для переноса сальдо от одного клиента к другому клиенту (то же самое и для поставщиков). Этот сценарий может возникать, когда ответственность за оплату накладной передается другой стороне, например дочерняя компания передает ответственность компании-учредителю. 

В этом примере предполагается продажа, в которой клиент имеет право на скидку при оплате, если платеж осуществляется в течение 10 дней. В этом примере клиент использует страховую компанию, которая будет ответственна за оплату счета. Страховая компания настроена как второй клиент в системе. Сальдо исходного клиента перемещается в страховую компанию, которая оплачивает счет, с учетом 2% скидки при оплате в период скидки. 

Чтобы проиллюстрировать, предположим, что произведена следующая продажа клиенту АКМЕ. Следующие учетные записи представляют продажу.

| Счет ГК | Тип разноски | по дебету | По кредиту |
|--------------------|------------------|-----------|------------|
| 401100-002-023-    | Реализация          |           | 100        |
| 130100-002-        | Сальдо по клиенту | 100       |            |

Затем пользователь переносит дебетовое сальдо из АКМЕ в страховую компанию в одном ваучере в журнале платежей по расчетам с клиентами. Страховая компания настраивается в качестве страхования клиента.

| Ваучер | Тип счета | Счет | описание | по дебету | По кредиту | Тип корр. счета | Корр. счет |
|-------------|------------------|-------------|-----------------|-----------|------------|-----------------|--------------------|
| ARPAYM001   | Клиент         | ACME        | Перевод        |           | 100,00     | Клиент        | Страхование          |

Обратите внимание, что вышеуказанная запись содержится в одном ваучере. Этот ваучер содержит два записи клиента. Следующий ваучер создается, когда вышеуказанная выше запись главной книги разнесена.

| Ваучер | Счет | Тип разноски | Сумма в валюте проводки |
|-------------|-------------|------------------|------------------------------------|
| ARPAYM001   | 130100-002- | Сальдо по клиенту | 100,00                             |
| ARPAYM001   | 130100-002- | Сальдо по клиенту | -100,00                            |

Далее предположим, что вы получили платеж от клиента страховой компании в сумме 98,00 и решили сопоставить этот платеж с накладной, созданной перемещением сальдо. Это приведет в разноске следующего ваучера. Могут быть ожидания, что сопоставление использует финансовые аналитики из первоначальной накладной, но это невозможно, поскольку нет документа накладной для клиента страховой компании. Заметьте, что по умолчанию аналитики распределения по скидке по оплате возникают из проводки клиента, созданной из перемещения, а не со счета выручки первоначальной накладной. По умолчанию используется результат использования одного ваучера для переноса сальдо.

| Ваучер | Счет | Тип разноски | по дебету | По кредиту |
|-------------|-------------|------------------|-----------|------------|
| ARPAYM002   | 110110-002- | Денежные средства             | 98.00     |            |
| ARPAYM002   | 130100-002- | Сальдо по клиенту |           | 98.00      |

В соответствующем ваучере для скидке по оплате значение по умолчанию для финансовой аналитики берется из проводки клиента, созданной из перемещений, так как перемещение имеется более одного клиента.

| Ваучер | Счет | Тип разноски       | по дебету | По кредиту |
|-------------|-------------|------------------------|-----------|------------|
| ARP-00001   | 403300-002- | Скидка по оплате по клиенту | 2.00      |            |
| ARP-00001   | 130100-002- | Сальдо по клиенту       |           | 2.00       |

Если пользователь недоволен со значением по умолчанию для финансовых аналитик для скидки по оплате, вместо одного ваучера следует использовать несколько ваучеров для записи перемещения сальдо. Этот сценарий должен быть выполнен путем создание кредитного авизо для клиента, ОТ которого перемещается сальдо, и дебетового авизо или накладной, созданных для клиента, НА которого переносится сальдо. ## <a name="one-voucher-with-a-netting-for-multiple-customers-and-vendors"></a>Один ваучер с взаимозачетом для нескольких клиентов и поставщиков
Взаимозачет может быть полезен, когда организация производит закупки и продажи внутри одной компании. Вместо оплаты накладных поставщика и ожидания получения платежа за накладные клиента производится взаимозачет накладных поставщика и клиента. Проводка взаимозачета сопоставляется с неуплаченными сальдо. 

Чтобы проиллюстрировать, предположим, что поставщик 1001 и клиент US-008 представляют собой один объект, поэтому организации требуется выполнить взаимозачет сальдо по расчетам с клиентами и по расчетам с поставщиками перед оплатой или получением оставшегося сальдо. Предположим, что запись клиента имеется долг 75,00 EUR, а запись поставщика имеет долг 100,00 EUR; это означает, что вы предпочитаете произвести взаимозачет сальдо и оплатить поставщику только 25,00 EUR. Далее предположим, что валюта учета USD. В этом случае проводка взаимозачета вводится в один ваучер в журнале платежей расчетов с поставщиками.

| Ваучер | Тип счета | Счет | описание | по дебету | По кредиту | Тип корр. счета | Корр. счет |
|-------------|------------------|-------------|-----------------|-----------|------------|-----------------|--------------------|
| APPAYM001   | Поставщик           | 1001        | Взаимозачет         |  75,00    |            | Клиент        | US-008             |

Чтобы избежать нежелательного создания будущих сопоставлений для данной проводки, вместо использования одного ваучера требуется ввести несколько ваучеров в журнал для записи проводок взаимозачета. Обратите внимание, что сальдо клиента и поставщика корректируются с одним клиринговым счетом, чтобы избежать использования одного ваучера, содержащего несколько сальдо клиента и поставщика.

| Ваучер | Тип счета | Счет | описание | по дебету | По кредиту | Тип корр. счета | Корр. счет |
|-------------|------------------|-------------|-----------------|-----------|------------|-----------------|--------------------|
| 001         | Клиент         | US-008      |                 |           |  75,00     | Главная книга          | 999999---          |
| 002         | Поставщик           | 1001        |                 |  75,00    |            | Главная книга          | 999999---          |
