# <a name="vendor-invoice-entry-workspace"></a>Рабочая область записей накладных поставщиков

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой теме объясняется, как настроить рабочую область, которая относится к накладным поставщиков и отображает сведения, доступные в Microsoft Power BI. Сведения о накладной поставщика в этой рабочей области фильтруются для определенных пользователей и отображаются в графическом формате.

## <a name="overview"></a>Обзор

В рабочей области **Запись накладной поставщика** отображается информация, относящаяся к обработке накладных поставщиков. Это включает представление **Моя работа** и страницу **Аналитика — все компании**. В представлении **Моя работа** отображаются сводные плитки, таблицы проводок по поставщикам, а также связанная информация о поставщиках. Страница **Аналитика — все компании** использует возможности Power BI для визуализаций, которые относятся к накладным поставщиков.

## <a name="set-up-the-workspace-to-show-power-bi-content"></a>Настройка рабочей области для отображения содержимого Power BI

Эту настройку необходимо выполнить до отображения данных в визуализациях Power BI в рабочей области **Запись накладной поставщика**.

1. В рабочей области **Управление функциями** отфильтруйте список, чтобы найти функцию **Автоматизации накладных поставщиков**.
3. Выберите **Включить**.
4. Чтобы гарантировать, что накладные могут быть обработаны от начала и до конца без необходимости вмешательства пользователя в ручном режиме, настройте рабочий процесс накладной поставщика. Чтобы настроить рабочий процесс, перейдите в раздел **Расчеты с поставщиками \> Настройка \> Рабочие процессы для расчетов с поставщиками**.
5. Перейдите в раздел **Расчеты с поставщиками \> Настройка \> Параметры расчетов с поставщиками** и выберите вкладку **Автоматизация накладных поставщиков**. Дополнительные сведения см. в разделе [Настройка параметров для автоматизации накладных поставщиков](vnd-invoice-set-up-options.md).
6. Задайте для параметра **Автоматическая отправка импортированных накладных в рабочий процесс** значение **Да**.
7. Если поступления продуктов должны быть автоматически сопоставлены, установите для параметра **Автоматически сопоставлять приходы продуктов со строками накладных** значение **Да**.
8. Просмотрите остальные, необязательные параметры и настройте их в соответствии с требованиями организации.
9. Перейдите в раздел **Администрирование системы \> Настройка \> Системные параметры** и задайте поля **Системная валюта** и **Валютный курс системы**.
10. Перейдите в раздел **Главная книга \> Настройка \> Главная книга** и задайте поля **Валюта учета** и **Тип валютного курса**.
11. Перейдите в раздел **Главная книга \> Валюты \> Валютные курсы** и введите валютные курсы между валютой проводки и валютой учета и между валютой учета и валютой системы.
12. Перейдите в раздел **Администрирование системы \> Настройка \> Хранилище объектов** и найдите пункт **Мера автоматизации накладных поставщиков**. Выберите **Обновить**.

Для просмотра информации, отображаемой в рабочей области, необходимо иметь роль безопасности менеджера расчетов с поставщиками или сотрудника по расчетам с поставщиками.

## <a name="my-work-view"></a>Представление "Моя работа"

### <a name="company-selection"></a>Выбор компании

Когда включена функция **Автоматизации накладных поставщиков** , поле **Компания** отображается в верхней части рабочей области. Выбор в поле **Компания** влияет на все данные, которые отображаются в рабочей области. По умолчанию в представлении отображаются сведения о компании, в которую вы выполнили вход. При выборе другой компаний в поле **Компания** может отображаться информация для этой компании в рабочей области. Затем можно выбрать плитку в рабочей области, чтобы перейти на соответствующую страницу в выбранной компании.

### <a name="summary-tiles"></a>Плитки сводки

Плитки в разделе **Сводка по ожидающим накладным** в представлении **Моя работа** дают обзор состояния накладных поставщиков. Можно просмотреть журналы, которые еще не разнесены, и накладных, которые находятся на удержании. Кроме того, имеются четыре плитки, связанные с функцией автоматизации накладных поставщиков:

- Требуется сопоставление поступлений вручную
- Не удалось выполнить проверку сопоставления
- Накладные не отправлены в workflow-процесс
- Накладные не импортированы

(Эти четыре плитки требуют, чтобы функция автоматизации накладных поставщиков была включена в управлении функциями.)

Чтобы использовать плитку **Восстановить накладные поставщиков** , эта функция должна быть включена в параметрах модуля расчетов с поставщиками. Перейдите в раздел **Расчеты с поставщиками \> Параметрам расчетов с поставщиками** , затем на вкладке **Накладная** установите для параметра **Разрешить восстановление накладной поставщика** значение **Да**.

Если эта функция включена, вы также сможете сгруппировать три плитки в рабочую область в разделе под названием **Журналы**. Плитки являются плитками **Журналы** , **Журналы — назначенные мне** и **Пул накладных**. 

Сведения в разделе **Сводка по ожидающим накладным** указаны для компании, которая настроена в качестве компании по умолчанию для входа в систему.

### <a name="creating-new-records"></a>Создание новых записей

Чтобы создать новую запись накладной, выберите **Создать** , затем выберите один из следующих типов записей в списке:

- Накладная поставщика
- Журнал накладных
- Глобальный журнал накладных
- Реестр накладных
- Утверждение накладной

Обратите внимание, что созданная запись базируется на фильтре компании, а не на компании, в которую выполнен вход. Например, вы выполнили вход в компанию **UMSF** , но фильтр компании имеет значение **GBSI**. В этом случае если выбрать пункт **Создать** , затем выбрать тип записи в списке, запись создается в компании GBSI.

### <a name="documents-not-invoiced-grids"></a>Сетки документов, по которым не выставлены накладные

Раздел **Документы, по которым не выставлены накладные** содержит сетки, отображающие документы, ожидающие накладных поставщиков.

В сетке **Открытые заказы на покупку** отображаются все заказы на покупку, по которым еще не были полностью выставлены накладные. Можно использовать кнопку **Выставить накладную сейчас** для создания накладной поставщика для заказа на покупку. Можно использовать кнопку **Счет на предоплату сейчас** для создания накладной предоплаты для заказа на покупку.

В сетке **Поступления продуктов** отображаются транзакции поступления продуктов, по которым еще не были полностью выставлены накладные. Можно использовать кнопку **Выставить накладную сейчас** для создания накладной поставщика для заказа на покупку.

В сетке **Накладные поставщиков, ожидающие обработки** отображаются все накладные поставщиков, которые не были отправлены в систему рабочего процесса. Для поиска конкретной накладной поставщика можно использовать поле **Поиск** и/или фильтр компании. Для редактирования проводки, отображаемой в сетке, можно использовать кнопку **Изменить**.

В сетке **Поиск заказа на покупку** можно использовать поле **Поиск** для поиска конкретного заказа на покупку.

### <a name="related-information"></a>Связанные сведения

Сведения о разнесенных накладных можно просмотреть, используя ссылки в правой части рабочей области. Эти ссылки включают **Открытые накладные поставщиков** , **Журнал накладных** и **Журнал накладных и соответствующая информация**. В разделе **Поставщики** можно получить доступ к отфильтрованному списку, в котором показаны все заблокированные поставщики, или же можно использовать ссылку **Все поставщики**. Также доступны ссылки **Все заказы на покупку** и **Открыть предоплаты**.

### <a name="analytics--all-companies-page"></a>Страница "Аналитика — все компании"

Если параметр **Автоматически отправлять импортированные накладные в рабочий процесс** имеет значение **Да** , на странице **Параметры модуля расчетов с поставщиками** можно просмотреть аналитику автоматизации. Страница **Аналитика — все компании** предоставляет важные метрики, такие как накладные поставщиков, которые в данный момент утверждены утверждающим лицом или компанией.
