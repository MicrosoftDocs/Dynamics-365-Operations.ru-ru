# <a name="configure-standard-costs-for-labor-and-expenses"></a>Настройка стандартной стоимости труда и расходов

[!include [banner](../../includes/banner.md)]

В этой теме показано, как настроить стандартные затраты на труд и расходы по проекту. В этой задаче используется набор данных компании USSI.

1. В области переходов выберите **Модули > Управление и учет по проектам > Настройка > Цены > Себестоимость (час)**.
2. Выберите **Создать**.
3. В поле **Действует с** введите дату.
4. В поле **Себестоимость** введите число. Можно настроить стандартную себестоимость категории проекта или настроить себестоимость по коду работника, номеру проекта, категории, дате или любой их комбинации. Применяемая себестоимость будет себестоимостью при самом высоком уровне детализации.  
5. Нажмите **Сохранить**.
6. В области переходов выберите **Модули > Управление и учет по проектам > Настройка > Цены > Цена продажи (время)**.
7. Выберите **Создать**.
8. В поле **Действует с** введите дату.
9. В поле **Допустимо для** выберите один из вариантов.
10. В поле **Ценообразование** введите число. Можно настроить стандартную цену продажи для почасовых проводок или категории проектов. Можно также настроить цены продажи по коду работника, номеру проекта, категории, дате проводки или любой их комбинации. Фактическая цена продажи, которая применяется, когда работник вводит проводку в журнале регистрации времени, является ценой продажи с самым высоким уровнем детализации. Например, если существует общая цена продажи и цена продажи для работника, то будет использоваться цена продажи для работника.  
11. Нажмите **Сохранить**.
12. Закройте страницу.
13. В области переходов выберите **Модули > Управление и учет по проектам > Настройка > Цены > Себестоимость (расходы)**.
14. Выберите **Создать**.
15. В поле **Действует с** введите дату.
16. В поле **Себестоимость** введите число. Можно заполнить несколько полей, но это минимум, необходимый для сохранения записи.  
17. Нажмите **Сохранить**.
18. В области переходов выберите **Модули > Управление и учет по проектам > Настройка > Цены > Цена продажи (расходы)**.
19. Выберите **Создать**.
20. В поле **Действует с** введите дату.
21. В поле **Допустимо для** выберите один из вариантов.
22. В поле **Ценообразование** введите число. Фактической ценой продажи, которая используется, когда работник заносит проводки в журнал затрат, является цена продажи с самым высоким уровнем детализации. Например, если существует общая цена продажи и цена продажи для работника, то будет использоваться цена продажи для работника.  
23. Нажмите **Сохранить**.
