# <a name="set-up-project-resources"></a>Настройка ресурсов проектов

[!include [banner](../includes/banner.md)]

Необходимо настроить календарь и связать его с сотрудником или работником. Календарь используется для планирования проекта и рабочего времени ресурсов, которые зарезервированы для проекта. При настройке календаря менеджеры проектов могут выполнять выравнивание ресурсов как часть оптимизации ресурсов. На основе расписания календаря на ресурсы можно накладывать ограничения. Можно настроить календарь на странице **Календари**.

При настройке работника в качестве ресурса проекта можно выбирать из сотрудников, работающих в компании, для которой выполняется настройка ресурсов. Также можно выбирать работников из других компаний в вашей организации. Эти работники называются внутрихолдинговыми ресурсами.

Следующие процедуры показывают, как настроить работника в качестве ресурса проекта в рамках вашей компании, а также как настроить внутрихолдинговый ресурс проекта.

## <a name="set-up-a-worker-as-a-project-resource"></a>Настройка работника в качестве ресурса проекта

1. На странице **Работники** в списке **Работники** выберите работника, добавленного как ресурс проекта, и откройте запись работника.
2. В области действий выберите **Проект** &gt; **Настройка** &gt; **Настройка проекта**.
3. Выберите календарь, затем закройте страницу.

Можно также указать проекты по умолчанию для ресурса в качестве типа предварительного назначения. Предварительные назначения можно использовать, когда менеджер ресурсов или менеджер проекта заранее знают, над какими проектами будет работать ресурс. Предварительные назначения могут также основываться на запросе спонсора проекта или клиента. Чтобы задать предварительные назначения для проекта, на странице **Назначение проектов** на вкладке **Проекты** в списке **Остальные проекты** выберите соответствующий проект.

## <a name="set-up-an-intercompany-resource"></a>Настройка внутрихолдингового ресурса

При настройке работника в качестве внутрихолдингового ресурса необходимо выполнить настройку и в компании-арендодателе и в заимствующей компании.

### <a name="in-the-lending-company"></a>В компании-арендодателе

1. В Finance убедитесь, что выбрана компания-арендодатель, затем выполните процедуру, описанную в предыдущем разделе, "Настройка работника в качестве ресурса проекта".
2. На странице **Внутрихолдинговый учет** выберите **Создать**.
3. В поле **Код информации о компании** выберите компанию-арендодателя. Требуемым образом заполните оставшиеся поля и нажмите кнопку **Сохранить**.
4. На странице **Трансферная цена** выберите **Создать**.
5. В поле **Получающая информация о компании** выберите заимствующую компанию.
6. Чтобы передать в аренду заимствующей компании только ресурс, созданный в начале этого раздела, в поле **Ресурс** выберите имя ресурса, который вы создали. Чтобы все ресурсы в компании-арендодателе были доступны заимствующей компании, оставьте поле **Ресурс** пустым.
7. На странице **Параметры модуля "Управление и учет по проектам"** на вкладке **Внутрихолдинговый** выберите в поле **Включить планирование и графики внутрихолдинговых ресурсов** значение **Да**.

### <a name="in-the-borrowing-company"></a>В заимствующей компании

- На странице **Список ресурсов** в фильтре поиска введите имя ресурса, созданного для компании-арендодателя, чтобы убедиться, что это имя включено в список ресурсов для заимствующей компании.

## <a name="request-project-resources"></a>Запрос ресурсов проекта
Функциональность для планирования ресурсов проекта позволяет менеджерам ресурсов только распределять укомплектованные ресурсы по обязательствам или проектам. Чтобы включить эту функциональность, выполните следующие задачи или убедитесь, что они уже выполнены:

- Настройка номерных серий.
- Настройте workflow-процессы управления и учета по проектам.
- Включение workflow-процессов запроса ресурса.

После выполнения предыдущих задач можно выполнить необходимые из следующих задач:

- Создание запроса ресурса из предварительно зарезервированного укомплектованного ресурса.
- Мониторинг запросов ресурсов.
- Обеспечение запросов ресурсов.
- Запрос укомплектованного ресурса из СДР.
- Резервирование ресурсов для проекта без запроса укомплектованного ресурса.
