# <a name="project-resource-scheduling-performance"></a>Производительность планирования ресурсов проекта

[!include [banner](../includes/banner.md)]
[!include [banner](../includes/preview-banner.md)]


Проблемы с производительностью, связанные с планированием ресурсов, могут возникать, когда число проектов достигает тысяч. Для повышения производительности планирования ресурсов доступна функция, позволяющая пользователям сократить время, необходимое для запуска формы доступности ресурсов. В частности, это приведет к удалению процесса синхронизации свертки емкости ресурсов и использует таблицу **ResProjectResource** для ускорения поиска ресурсов. Обратите внимание, что таблица **ResRollup** больше не будет использоваться.

> [!IMPORTANT]
> Если имеется зависимость от процесса синхронизации по свертке емкости ресурсов или таблицы **ResProjectResource**, не используйте эту функцию.

## <a name="enable-resource-scheduling-performance-enhancement"></a>Включение улучшения производительности планирования ресурсов
Чтобы включить улучшение производительности планирования ресурсов, выполните следующие действия.

1. Выберите **Управление функциями** > **Все** и в списке функций найдите функцию **Включение улучшения производительности планирования ресурсов проектов**.
2. Выберите **Включить**.

> [!NOTE]
> Если вы не можете найти функцию в списке, выберите **Проверить наличие обновлений**, чтобы обновить список.

3. Обновите веб-браузер, затем перейдите в раздел **Управление и учет по проектам** > **Периодические задачи** > **Ресурсы проекта** > **Синхронизация мощностей для календарей ресурсов во всех компаниях**.
4. Установите для параметра **Удаление существующих записей о мощностях** значение **Да**, чтобы удалить предыдущие данные. Если требуется генерировать инкрементные данные, установите для него значение **Нет**.
5. В поле **Код периода** выберите период, в котором должны быть созданы данные. При выборе кода периода дату начала и окончания не требуется определять.
6. Если оставить поле **Код периода** пустым, выберите для создания данных конкретные даты начала и окончания.
7. Нажмите **ОК**.

 > [!NOTE]
 > При этом общие данные будут распределены в таблице **ResCalendarCapacity** по всем компаниям в вашей среде, поэтому пакетное задание должно быть выполнено только в одном юридическом лице. Данные в этом пакетном задании необходимы для расчета емкости ресурсов через связанный календарь.

8. Выберите **Управление и учет по проектам** > **Периодические операции** > **Ресурсы проекта** > **Заполнить ресурсы проекта по всем компаниям**, затем выберите **ОК**. Это сценарий обновления данных для общих данных в таблицах **ResProjectResource**, **ResCalendarDateTimeRange** и **ResEffectiveDateTimeRange**. Значения для поля **PSAPRojSchedRole.RootActivity** также обновляются. Если это не было выполнено, при попытке выполнить операции по планированию ресурсов будет выводиться предупреждение.
 
## <a name="turn-off-resource-scheduling-performance-enhancement"></a>Выключение улучшения производительности планирования ресурсов

1. Выберите **Управление функциями** > **Все** и найдите функцию **Включение улучшения производительности планирования ресурсов проектов**.
2. Выберите функцию, затем выберите кнопку **Отключить**.
3. Обновите веб-браузер.
4. Выберите **Управление и учет по проектам** > **Периодические операции** > **Синхронизация мощностей** > **Синхронизация сверток емкости ресурсов**.
5. На странице **Синхронизация свертки емкости** установите для параметра **Удалить существующие записи о емкости** значение **Да**, чтобы удалить предыдущие данные. Если требуется генерировать инкрементные данные, установите для него значение **Нет**.
6. В поле **Код периода** выберите период, в котором должны быть созданы данные. При выборе кода периода дату начала и окончания не требуется определять.
7. Если оставить поле **Код периода** пустым, выберите для создания данных конкретные даты начала и окончания.
8. Нажмите **ОК**.

> [!NOTE]
> При этом общие данные будут распределены в таблице **ResRollup** по всем компаниям в вашей среде, поэтому пакетное задание должно быть выполнено только в одном юридическом лице. Это пакетное задание требуется для всех представлений **Доступность ресурсов**. Если пакетное задание не выполняется, данные **ResRollup** будут создаваться на ходу, что может занять некоторое время.
