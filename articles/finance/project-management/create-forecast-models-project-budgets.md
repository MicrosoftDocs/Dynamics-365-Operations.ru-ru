# <a name="create-forecast-models-for-project-budgets"></a>Создание прогнозных моделей для бюджетов проекта 

[!include [banner](../includes/banner.md)]

В этом разделе описывается создание прогнозной модели для оставшихся бюджетов. Проект, подлежащий бюджетному контролю, использует два типа бюджетов: исходный и остаток. При создании бюджета проекта необходимо указать модели прогноза исходного бюджета и остатка бюджета, которые были созданы в на странице **Прогнозные модели**. Бюджеты проекта на основе указанных моделей создаются при отправке бюджета проекта.

> [!NOTE]
> Прогнозная модель, используемая для бюджетного контроля, не может иметь подмодели, и ее нельзя использовать в качестве подмодели.

1. Выберите **Управление и учет по проектам** > **Настройка** > **Прогнозы**  > **Прогнозные модели**.
2. Выберите **Создать** , чтобы создать новую прогнозную модель, а затем введите ИД модели и ее имя. 
3. Для параметра **Остановлено** введите значение **Да**, чтобы исключить любые изменения в строках прогноза для прогнозной модели. 
4. Если строки прогноза, с которыми связана модель, должны создавать прогнозы движения денежных средств в главной книге, для параметра **Включить бюджеты в прогнозы движения денежных средств** установите значение **Да**. 
5. Чтобы использовать дату проекта в качестве даты накладной, для параметра **Прогнозировать дату накладной** установите значение **Да**. 
6. В поле **Тип бюджета** выберите один из следующих типов моделей:

   - **Исходный бюджет** - используйте этот тип модели для сумм первоначального бюджета, которые отправляются, когда создается и утверждается исходный бюджет.
   - **Остаток бюджета** - этот тип модели предназначен для сумм остатков бюджета во время срока службы проекта. Сальдо в этой модели прогноза уменьшаются на размер фактических проводок и увеличиваются или уменьшаются версиями бюджета.
   - **Перенесенный бюджет** - этот тип предназначен для сумм перенесенного бюджета для проекта. Перенос — необязательный процесс, который можно выполнить для переноса неиспользованных сумм бюджета из одного финансового года в другой.

7. При необходимости задайте следующие параметры:

   - Включите **Прогнозировать дату накладной**, чтобы использовать дату проекта в качестве даты накладной.
   - Включите **Прогноз с НЗП**, чтобы строить прогноз на основе незавершенного производства (НЗП), а затем выберите тип НЗП. 
   - В поле **Тип бюджета** можно оставить значение **Нет** по умолчанию или ввести новый тип. Тип бюджета нельзя изменить после изменения записи.     
    > [!NOTE]
    > При изменении типа бюджета по умолчанию все остальные параметры будут недоступны для обновления, и вы не сможете повторно использовать эту прогнозную модель.
