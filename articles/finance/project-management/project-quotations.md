# <a name="project-quotations"></a>Предложения по проекту

[!include [banner](../includes/banner.md)]

В этой статье вводится понятие предложений по проекту, которые можно использовать для того, чтобы сделать привлекательное предложение клиенту в качестве первого шага фазы проекта. Предложение по проекту может включать сведения о продуктах или услугах, общую контактную информацию, специальные торговые соглашения и скидки, возможные налоги и сборы. 

Возможность отслеживать, просматривать и контролировать процесс продаж с помощью предложений по проекту и заказов является важной частью управления проектом. Существует множество средств, позволяющих справиться с поставленными задачами. Например, правильные определения ссылочных данных (типов предложений, источников предложений, а также их прогнозирования и вероятности) помогают анализировать процесс продаж. Описанные средства можно использовать для классификации причин успеха или неудачи предложения по проекту и определения возможной прибыли по предложению. 

В предложении по проекту указываются услуги, основная контактная информация, специальные торговые соглашения и скидки, а также расчетные налоги и надбавки по проекту. Можно выбрать мероприятия или задачи для проекта и создать иерархию задач и подзадач. Для каждого действия можно указать сведения о времени и продолжительности его выполнения, навыках и опыте, необходимых для выполняющих это действие сотрудников. 

Предложение по проекту — это неофициальная оценка работы, которую предстоит выполнить. Однако, когда информация в предложении копируется в проект и проект связывается с контрактом по проекту, информация становится частью обязывающего договора между двумя сторонами. 

Если клиент утверждает предложение по проекту, можно скопировать информацию из предложения по проекту в проект. Одновременно можно скопировать информацию предложения по проекту в прогноз проекта.
