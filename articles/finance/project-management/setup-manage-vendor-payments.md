# <a name="set-up-and-use-pay-when-paid-vendor-payments"></a>Настройка и использование платежей поставщику после получения оплаты от клиента

[!include [banner](../includes/banner.md)]

Если для работы в качестве субподрядчика утверждается поставщик, платежи поставщику при необходимости можно задержать до момента получения оплаты от клиента за проект. Чтобы поддержать этот сценарий, можно настроить условия режима "оплата после получения оплаты" (pay-when-paid, PWP) при настройке заказа на покупку (PO) с поставщиком.

Например, если клиент оплачивает сумму по накладной проекта, можно отправить часть или всю сумму по накладной поставщика. Просто настройте условия PWP, которые указывают, что поставщик получит оплату после получения процента от связанного платежа от клиента. После получения частичной оплаты от клиента можно вручную отправить на оплату некоторых из связанных накладных поставщика.

В следующем примере показан процесс, когда используются условия PWP.

## <a name="example"></a>Пример

Ваша организация соглашается предоставить клиенту 100 компьютеров, на которых установлено программное обеспечение, по цене 150,00 долларов США за компьютер. Затем вы нанимаете поставщика, чтобы предоставить вам компьютеры с установленным программным обеспечением. В соответствии с соглашением, клиент должен проверить качество компьютеров, прежде чем он перечислит оплату вашей организации. Политика вашей организации заключается в том, что она задерживает платежи поставщикам до тех пор, пока клиент не заплатит вам. Поэтому проект настраивается таким образом, чтобы у него был процент PWP, равный 100 процентам.

Поставщик посылает вам 100 компьютеров с установленным программным обеспечением и счет на 10 000,00 долларов США. Поскольку условия PWP настроены для проекта, вы не платите поставщику при получении компьютеров. Вместо этого вы отправляете компьютеры клиенту вместе со счетом на 15 000,00. Клиент проверяет компьютеры и подтверждает полную суммы счета по проекту.

После получения полной оплаты от клиента вы выплачиваете поставщику 10 000,00 на всю сумму по накладной поставщика.

## <a name="set-up-pwp-terms-for-a-project"></a>Настройка условий PWP для проекта

При настройке условий PWP для проекта необходимо указать минимальную сумму в процентах, которую клиент должен заплатить за проект, прежде чем вы заплатите поставщику. Пороговая сумма автоматически вычисляется по накладным клиента для проекта. Выполните следующие действия, чтобы настроить пороговый процент для условий PWP.

1. Перейдите в раздел **Управление и учет по проектам** \> **Проекты** \> **Все проекты**.
2. Найдите и откройте проект, для которого требуется настроить условия PWP.
3. На экспресс-вкладке **Договора с поставщиками** выберите **Добавить строку**.
3. В поле **Код счета** выберите один из следующих параметров:

    - **Таблица** — условия PWP применяются к одному поставщику.
    - **Группа** — условия PWP применяются ко всем поставщикам в группе поставщиков.
    - **Все** — условия PWP применяются ко всем поставщикам.

4. Если на предыдущем шаге был выбран вариант **Таблица** или **Группа**, в поле **Поставщик/группа поставщиков** выберите поставщика или группу поставщиков, к которой применяются условия PWP. Если на предыдущем шаге выбрано значение **Все**, поле **Поставщик/группа поставщиков** нельзя изменить.
5. Если в проекте настроены условия удержания оплаты поставщику для поставщика, в поле **Условия удержания поставщика** выберите код правила для условий удержания.
6. В поле **Процент порогового уровня PWP** укажите процент порогового уровня для проекта. Указанный для проекта процент определяет минимальную сумму, которую должен оплатить клиент, чтобы вы заплатили поставщику.

## <a name="create-a-po-that-has-pwp-terms"></a>Создание заказа на покупку с условиями PWP

При разноске накладной от поставщика, если к нему применяются условия PWP, эти условия отображаются в строках заказа на покупку. Чтобы создать заказ на покупку, имеющий условия PWP, выполните следующие действия.

1. Перейдите в раздел **Закупки и источники** \> **Заказы на покупку** \> **Все заказы на покупку**.
2. В области действий выберите **Создать**. Затем в диалоговом окне **Создание заказа на покупку** введите требуемые сведения и выберите **OK**.

    Также можно открыть существующий заказ на покупку на странице списка **Все заказы на покупку**.

4. На странице **Заказ на покупку** на экспресс-вкладке **Строки заказа на покупку** просмотрите сведения о строке заказа на покупку для поставщика. Параметр **Оплата после получения оплаты** выбирается автоматически, и значение в поле **Процент порогового уровня PWP** автоматически копируется из поля **Процент порогового уровня PWP** на странице **Проекты**.
6. Если вы не хотите применять условия PWP для поставщика для строки заказа на покупку, снимите отметку у параметра **Оплата после оплаты**. В этом случае поле **Процент порогового уровня PWP** для строки заказа на покупку будет сброшено на значение 0 (ноль).

## <a name="update-a-customer-payment-and-pay-the-vendor"></a>Обновление платежа клиента и оплата поставщику

Если поставщик завершил работу с проектом и выставил накладную, необходимо просмотреть статус проекта и накладные клиента, чтобы определить, выполнены ли условия PWP для проекта. Если условия PWP для поставщика были выполнены, можно определить подлежащие оплате строки накладной поставщика с учетом платежей клиента для данного проекта. Если принято решение выполнить оплату поставщику даже при невыполнении условий PWP, эти условия можно переопределить на странице **Накладные поставщика с "оплатой после получения оплаты"**.

1. Выберите **Управление и учет по проектам** \> **Запросы и отчеты** \> **Запросы удержания** \> **Накладные поставщика с "оплатой после получения оплаты"**.
2. На странице **Накладные поставщика с "оплатой после получения оплаты"** в поле поиска укажите значения для поиска накладной поставщика, которую нужно просмотреть, затем выберите **Поиск**.
3. На экспресс-вкладке **Строки накладной поставщика** выберите строки, которые хотите изменить.
4. Если для строки накладной выполняются условия **Оплата после получения оплаты**, выберите **Освобождение платежа поставщика**. Параметр **Оплата после получения оплаты** очищается, и значение поля **Готово к оплате** изменится на **Да**.
