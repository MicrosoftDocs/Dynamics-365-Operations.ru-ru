# <a name="set-up-value-models"></a>Настройка моделей стоимости

[!include [banner](../../includes/banner.md)]
[!include [preview banner](../../includes/preview-banner.md)]

В этой процедуре показано, как создать новую модель стоимости основных средств и связать ее с группой ОС.

## <a name="create-a-book"></a>Создайте книгу
1. Перейдите в раздел **Основные средства \> Настройка \> Книги**.
2. Выберите **Создать**.
3. В поле **Книга** введите значение.
4. В поле **Описание** введите значение.
5. Установите параметр **Расчет амортизации** на значение **Да**.

    Если для параметра **Расчет амортизации** задано значение **Да**, связанный журнал основных средств будет включен в предложения по амортизации. Если задано значение **Нет**, журнал основных средств не будет амортизироваться автоматически.

6. В поле **Профиль амортизации** введите или выберите значение.

    * Альтернативный метод амортизации также называется методом переключения амортизации. Предложение по амортизации переключается на этот метод, когда альтернативный метод рассчитывает сумму амортизации, которая больше или равна методу амортизации по умолчанию.
    * Метод дополнительной амортизации используется для дополнительной амортизации основного средства в необычных условиях. Например, можно использовать это для записи амортизации в результате стихийного бедствия.
    * Если выбран параметр **Создание переоценки амортизации с основными поправками**, корректировки амортизации создаются автоматически при обновлении стоимости основного средства. В противном случае обновленное значение ОС будет влиять только на расчеты будущей амортизации.

7. Установите для параметра **Создание переоценки амортизации с основными поправками** значение **Да**.

    * По умолчанию проводки журнала ОС разносятся в главную книгу. Однако можно отключить разноску в главную книгу для этой книги, установив в параметре **Разносить в главную книгу** значение **Нет**. Книги, которые не разносятся в главную книгу, обычно используются для налоговой отчетности. Этот параметр дает большую гибкость для удаления исторических проводок из журнала ОС, поскольку эти проводки не были зафиксированы в главной книге.
    * По умолчанию в поле **Слой разноски** установлено значение **Текущий слой**, если книга разносится в главную книгу, или значение **Нет**, если книга не разносится в главную книгу. Обновите значение поля **Слой разноски**, если проводки для данной книги должны разноситься в другой слой.

8. Расчет положительной амортизации.

    * По умолчанию для параметра **Расчет положительной амортизации** установлено значение **Нет**. Этот параметр указывает, что амортизация будет кредитоваться в выбранный журнал ОС. Кроме того, для обоих параметров **Разрешить остаточную стоимость выше затрат на приобретение** и **Разрешить отрицательную остаточную стоимость** установлено значение **Нет**, и настройки могут быть изменены независимо друг от друга. 
    * Для расчета положительной амортизации установите для параметра **Расчет положительной амортизации** значение **Да**. Этот параметр указывает, что амортизация будет дебетоваться в журнал ОС. Когда для параметра **Расчет положительной амортизации** задано значение **Да**, для параметров **Разрешить остаточную стоимость выше цены приобретения** и **Разрешить отрицательную остаточную стоимость** автоматически будет установлено значение **Да**, и они будут заблокированы. Эта блокировка позволяет гарантировать, что положительная амортизация будет применяться только к основным средствам, которые были приобретены с отрицательной суммой (кредит). 

10. В поле **Календарь** введите или выберите значение.

    Производные книги будут разносить проводки по различным книгам одновременно. Вы создаете проводки в основной книге, и во время разноски точные копии проводки разносятся в производной книге. Пересчет с использованием проводок в производной книге не производится, поэтому она не должна использоваться для проводок амортизации.

## <a name="associate-the-book-with-a-fixed-asset-group"></a>Связь книги с группой основных средств

1. Выберите **Группы основных средств**.
2. В поле **Группа ОС** введите или выберите значение.
3. В поле **Срок службы** введите число.

    * Периоды амортизации рассчитывается после ввода срока службы актива.
    * Соглашение по амортизации можно настроить, как требуется для налогообложения.
    * Для основных средств, связанных с арендой, значение в поле **Срок службы** будет переопределено меньшим из срока аренды в журнале активов и в сроке эксплуатации актива. Если в параметре **Перемещение собственности** для книги аренды указано значение **Да**, то значением поля **Срок службы** всегда будет срок службы актива.
