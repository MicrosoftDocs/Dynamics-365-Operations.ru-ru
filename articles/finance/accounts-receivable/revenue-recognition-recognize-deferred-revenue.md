# <a name="recognize-deferred-revenue"></a>Признание выручки будущих периодов

[!include [banner](../includes/banner.md)]

> [!NOTE]
> Функцию признания выручки невозможно включить через механизм управления функциями. В данный момент это можно сделать только с помощью конфигурационных ключей.

В этом разделе описывается процесс признания выручки в графике признания выручки. После разноски накладной для заказа на продажу создается график признания выручки для каждой строки заказа на продажу, имеющей график признания выручки. График признания выручки в строке используется, чтобы определить, должна ли выручка по строке откладываться.

## <a name="view-revenue-recognition-schedule-details"></a>Просмотр подробных сведений о графике признания выручки

Есть два способа получить доступ к подробным сведениям о графике признания выручки.

- Можно открыть график признания выручки непосредственно из заказа на продажу, по которому выставлена накладная. В этом случае информация в графике признания выручки фильтруется, чтобы показать подробности только для выбранного заказа на продажу. Этот подход полезен при проверке подробных сведений о графике для заказа на продажу.
- Можно открыть график признания выручки на странице **Признание выручки \> Периодические задачи**. Такой подход часто используется, когда выручка признается в конце периода. При первом открытии страницы никакая информация не отображается. Используйте фильтры, расположенные над сеткой, чтобы определить критерии для подробных сведений о графике, которые необходимо отобразить. Можно отфильтровать даты накладной, введя диапазон дат, заказ на продажу, клиента, код проекта или статус.

[![Страница графиков признания выручки](./media/revenue-recognition-rev-revenue-schedules.png)](./media/revenue-recognition-rev-revenue-schedules.png)

На экспресс-вкладке **Финансовая аналитика** под сеткой отображаются финансовые аналитики строки заказа на продажу. Эти аналитики были учтены во время разноски для выручки будущих периодов. Кроме того, они учитываются при признании выручки. Используемые значения аналитики зависят от структуры счета, которая назначена для счетов ГК выручки и выручки будущих периодов.

## <a name="recognize-revenue"></a>Признание выручки

Выручка признается путем запуска процесса **Создать журнал** на странице **Признание выручки**. Можно открыть эту страницу либо из заказа на продажу, либо из **Периодических задач**. Если процесс запускается из заказа на продажу, выручка признается только по выбранному заказу на продажу. Обычно процесс запускается из **Периодических задач**, чтобы он признавал выручку для всех разнесенных накладных по заказам на продажу.

Чтобы определить критерии для выбора и разноски выручки, выберите **Создать журнал**, чтобы открыть диалоговое окно **Создать журнал**.

[![Параметры создания журнала](./media/revenue-recognition-create-journal.png)](./media/revenue-recognition-create-journal.png)

В диалоговом окне используйте параметры в группе полей **Дата обработки**, чтобы определить дату разноски, используемую при признании выручки. Если выбрать **Выбранная дата**, можно ввести дату разноски в поле **Дата проводки**. Если выбрать **Дата графика признания выручки**, дата проводки не используется. Вместо этого в качестве даты разноски используется значение поля **Дата признания** по каждой строке графика.

Затем в поле **Состояние на дату** введите дату "по состоянию на" для признания выручки. Все строки графика, в которых указана дата признания на момент или до даты "по состоянию на", будут разнесены, при условии, что они не заблокированы.

После определения дат нажмите кнопку **ОК** в диалоговом окне, чтобы создать журнал. Вы получите информационное сообщение, в котором отображается число созданных проводок и журнал, в котором они были созданы. Журнал не разносится автоматически. Таким образом, менеджер по признанию выручки располагает временем, позволяющие проверить, какие строки графика признаны.

После запуска процесса строки в графике, которые были перенесены в журнал, помечаются как **обработанные**. Флаг **обработанные** указывает, что строки были перенесены в журнал, но они могут быть разнесенными или не разнесенными. После разноски журнала признания выручки остается флаг **обработанные**. Если журнал признания выручки или строка удаляется, то флаг **обработанные** также снимается. Таким образом, строка может быть признана при повторном выполнении процесса **Создать журнал**.

[![Страница графиков признания выручки](./media/revenue-recognition-rev-recog-schedule-02.png)](./media/revenue-recognition-rev-recog-schedule-02.png)

На странице **Журнал признания выручки** (**Признание выручки \> Записи в журнале \> Журнал признания выручки**), откройте **Строки**, чтобы просмотреть сведения о том, что будет признано. Для каждой строки признанного графика всегда создается отдельная проводка, даже если все строки разнесены на одну и ту же дату, используя одинаковые счета учета.

[![Страница ваучера журнала](./media/revenue-recognition-journal-voucher.png)](./media/revenue-recognition-journal-voucher.png)

В столбце **Счет** отображается счет учета выручки будущих периодов. Этот счет учета не может быть изменен. Это ограничение помогает гарантировать, что освобождается нужный счет учета выручки будущих периодов. Этот счет учета не проверяется в отношении структуры счета, поскольку она могла измениться при последней разноске на указанный счет учета выручки.

В столбце **Корр. счет** отображается счет учета выручки. По умолчанию счет учета выручки берется из профилей разноски запасов, а финансовые аналитики берутся из строки заказа на продажу. Этот счет учета проверяется в отношении структуры текущего счета. Однако он может быть изменен, если структура счета была изменена, и требуются дополнительные финансовые аналитики.

Сумма по умолчанию берется из соответствующей строки графика и не может быть изменена.

По умолчанию, если заказ на продажу является многовалютным заказом на продажу, валютный курс устанавливается на валютный курс из накладной. Это позволяет гарантировать, что валюта учета и суммы валюты отчетности будут полностью освобождены. В связи с округлением, валютный курс последней строки графика может немного отличаться от курса по накладной.

После разноски журнала признания выручки, ваучер вводится в график. Если для одной и той же строки графика имеется более одного ваучера, в этой строке появится символ звездочки (\*). Чтобы просмотреть ваучеры, которые были разнесены для этой строки, выберите **Проводки по ваучеру**.

## <a name="modify-the-revenue-recognition-schedule-details"></a>Изменение подробных сведений о графике признания выручки

Большинство данных в составе графика признания выручки не могут быть изменены. Новые строки не могут быть добавлены в график, и существующие строки не могут быть удалены. Подробные сведения о графике признания выручки для каждой строки заказа на продажу должны поддерживаться, чтобы гарантировать, что организация признает ту же сумму, которая была отложена.

### <a name="edit-schedule-lines"></a>Редактирование строк графика

Некоторые изменения разрешены для строк графика. В строках могут быть изменены следующие поля:

- **Заблокировано** — этот флаг может быть установлен или снят перед обработкой строки. Чтобы снять этот флаг, выберите строку, а затем выберите команду **Снять блокировку**. Выручка не может быть признана по заблокированным строкам. Строки могут быть автоматически переведены в блокировку, если для графика признания выручки настроена автоматическая блокировка.

    [![Графики признания выручки — изменение строк графика](./media/revenue-recognition-rev-revenue-schedules.png)](./media/revenue-recognition-rev-revenue-schedules.png)

- **Дата признания** — дату признания можно изменить перед обработкой строки. Когда запускается процесс, создающий журнал для признания выручки, в поле **признать выручку по состоянию на** вводится дата. Эта дата сравнивается с датой, указанной в поле **Дата признания**, чтобы определить, какие строки должны быть признаны.
- **Освобождаемая сумма** — сумма, которая будет освобождена, может быть изменена перед обработкой строки. Можно уменьшить сумму признаваемой выручки, но нельзя увеличить ее. Это поле позволяет организации признавать часть выручки на дату признания. Если сумма изменена, сумма в поле **Сумма остатка** отображает объем выручки, которая также подлежит признанию.
- **Освобождаемое количество** — если график выручки настроен для одного вхождения или одного месяца, в поле **освобождаемое количество** отображается количество для строки заказа на продажу. Это поле также может быть изменено и предоставляет другой способ признать часть выручки. Например, если количество в строке равно 5, можно переопределить количество, чтобы оно было меньше 5. Сумма в поле **Освобождаемая сумма** обновляется пропорционально.

### <a name="update-contract-terms"></a>Обновление условий контракта

Подробные сведения о графике признания выручки создаются на основе графика признания выручки, назначенного строке заказа на продажу при разноске накладной. Если график признания выручки в строке заказа на продажу является неправильным, он не может быть изменен в заказе на продажу после выставления накладной по заказу на продажу. Вместо этого для изменения графика признания выручки необходимо использовать кнопку **Обновление условий контракта**. График признания выручки можно изменить до или после того, как будет признана выручка.

Чтобы изменить график, выберите любую строку графика для изменяемой номенклатуры. На следующем рисунке выбрана строка для номенклатуры S0008, которая была разнесена с помощью графика признания выручки на 12 месяцев. При выборе **обновления условий контракта** в диалоговом окне отображаются даты начала и окончания действия контракта, а также график признания выручки.

[![Даты начала и окончания действия контракта](./media/revenue-recognition-rev-revenue-schedule-update-cntrct-dates-schedule.png)](./media/revenue-recognition-rev-revenue-schedule-update-cntrct-dates-schedule.png)

Измените даты начала и окончания действия контракта так, чтобы они соответствовали правильному диапазону дат. При изменении диапазона дат значение в поле **Количество вхождений** должно соответствовать графику признания выручки, который был определен в системе. В этом примере, поскольку контракт был изменен на 24-месячный контракт, должен быть настроен график признания выручки для 24 месяцев. Так как график признания выручки для 24 месяцев существует, он вводится по умолчанию, и контракт может быть изменен. Если график признания выручки с соответствующим количеством вхождений не существует, контракт нельзя изменить. После обновления условий контракта и графика признания выручки нажмите кнопку **ОК** в диалоговом окне, чтобы сохранить изменения.

[![Обновленный диапазон дат контракта](./media/revenue-recognition-rev-revenue-schedule-update-cntrct-dates-schedule-02.png)](./media/revenue-recognition-rev-revenue-schedule-update-cntrct-dates-schedule-02.png)

Изменения контракта оказывают следующее воздействие на подробные сведения о графике признания выручки:

- Если выручка для продукта не была признана, все предыдущие сведения о графике удаляются и заменяются новыми сведениями о графике признания выручки. Например, номенклатура S0008 первоначально имела 12 строк в подробных сведениях графика. Эти 12 строк удаляются и заменяются 24 строками на основе нового графика признания выручки.
- Если выручка для продукта была признана, часть выручки была признана неправильно, поскольку признание было основано на неправильном графике признания выручки. Эти строки должны быть реверсированы и снова признаны на основе нового графика. В этом случае создаются новые строки графика признания выручки, которые имеют отрицательные суммы на исходную дату признания. Затем создаются новые строки для признания сумм на основе нового графика признания выручки. Например, на 8 августа 2019 года была признана выручка для 10,53 долларов США. 8 сентября 2019 года была признана выручка для 13,16 долларов США. Таким образом, создаются две новые строки для одних и тех же дат. Одна строка для -10,53 долларов США, другая строка — для -13,16 долларов США. Затем создаются двадцать четыре строки и общая сумма выручки будущих периодов в 160,61 долларов США распределяется по ним. Можно разнести строки сторнирования, запустив процесс **Создать журнал**.

[![График признания выручки](./media/revenue-recognition-rev-recog-schedule-03.png)](./media/revenue-recognition-rev-recog-schedule-03.png)
