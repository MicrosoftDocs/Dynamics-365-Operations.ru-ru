# <a name="credit-limit-adjustments"></a>Корректировки кредитного лимита 

[!include [banner](../includes/banner.md)]

Корректировка кредитного лимита позволяет менеджерам по кредитам обновлять кредитные лимиты и даты истечения срока действия отдельного клиента, группы клиентов или всех клиентов через процесс разноски. Можно добавить записи корректировки кредитного лимита для обновления клиентов и кредитных групп клиентов, либо можно использовать их для расчета автоматических кредитных лимитов. Записи можно просмотреть, отправить на утверждение в рамках workflow-процесса и разнести на счета клиентов.

## <a name="set-up-credit-limit-adjustments"></a>Настройка корректировок кредитного лимита

Можно создать записи в журнале корректировки кредитного лимита на странице **Корректировка кредитного лимита** (**Управление кредитом \> Корректировки кредитного лимита \> Корректировки кредитного лимита**).

1. Выберите **Создать**. Создается новая группа записей с номером корректировки кредитного лимита.
2. Выбор типа корректировки кредитного лимита:

    - Выберите **Кредитный лимит**, чтобы изменить кредитный лимит клиента.
    - Выберите **Временный кредитный лимит**, чтобы создать временный кредитный лимит вместо изменения текущего кредитного лимита клиента. Временные кредитные лимиты переопределяют кредитный лимит клиента для указанного периода. По окончании этого периода кредитный лимит клиента используется снова.
3. Введите описание. 

Если установлен флажок **Разнесено**, кредитные лимиты применены. Поле **Статус утверждения** показывает статус workflow-процесса для журнала. Workflow-процесс необязателен.

### <a name="add-credit-limit-adjustments"></a>Добавление корректировок кредитного лимита

Чтобы вручную добавить корректировки кредитного лимита, выберите **Строки**, а затем выполните следующие действия.

1. Чтобы добавить корректировку кредитного лимита для клиента, воспользуйтесь меню **Корректировки клиента**. Чтобы добавить кредитный лимит для кредитной группы клиентов, выберите **Корректировки кредитной группы клиентов**.
2. Введите счет клиента для счета клиента по накладной, который должен быть обновлен с учетом нового кредитного лимита. Если на шаге 1 были выбраны **Корректировки кредитной группы клиентов**, введите кредитную группу клиентов. В одной строке журнала нельзя вводить счет клиента и код кредитной группы клиентов.

    Отображается текущий кредитный лимит и имя автоматически.

3. Введите новый кредитный лимит, который должен заменять текущий кредитный лимит при разноске записи кредитного лимита.
4. Введите дату для определения новой даты окончания срока действия для кредитного лимита клиента. При создании корректировки кредитного лимита необходимо ввести дату окончания срока действия кредитного лимита.

Поле **Статус утверждения** показывает статус workflow-процесса для строки журнала.

Если необходимо автоматически создавать корректировки кредитного лимита, можно использовать меню **Создать** на панели операций.
 
### <a name="add-temporary-credit-limit-adjustments"></a>Добавление корректировок временного кредитного лимита

Чтобы вручную добавить корректировки временного кредитного лимита, выполните следующие действия для строк журнала.

1. Чтобы добавить корректировку кредитного лимита для клиента, воспользуйтесь меню **Корректировки клиента**. Чтобы добавить кредитный лимит для кредитной группы клиентов, выберите **Корректировки кредитной группы клиентов**.
2. Введите счет клиента для счета клиента по накладной, который должен быть обновлен с учетом нового кредитного лимита. Если на шаге 1 были выбраны **Корректировки кредитной группы клиентов**, введите кредитную группу клиентов. В одной строке журнала нельзя вводить счет клиента и код кредитной группы клиентов.

    Если активный или будущий временный кредитный лимит уже существует, текущий временный кредитный лимит и диапазон дат появятся для каждого временного кредитного лимита. Имя появится автоматически.

3. Введите новый кредитный лимит, который должен заменять текущий кредитный лимит.
4. В полях **Создать с даты** и **Создать до даты** укажите период, в течение которого действителен расширенный кредитный лимит. При создании журнала корректировки кредитного лимита необходимо ввести даты окончания срока действия кредитного лимита.

Поле **Статус утверждения** показывает статус workflow-процесса для строки журнала.

## <a name="generate-credit-limit-adjustments"></a>Создание корректировок кредитного лимита

Можно также автоматически корректировать кредитные лимиты. На панели операций выберите **Создать** и выберите один из следующих вариантов.

- Из существующего клиента
- Из существующей кредитной группы клиентов
- Автоматические кредитные лимиты

### <a name="from-existing-customer"></a>Из существующего клиента

Можно создавать строки журнала из существующих клиентов. При выборе **Создать \> Из существующего клиента** появится диалоговое окно, в котором можно ввести критерии для выбора клиентов и расчета новых лимитов.

1. Ввод значения корректировки для сложения или вычитания суммы из кредитного лимита. Введите отрицательное значение для уменьшения текущего кредитного лимита или положительное значение, чтобы увеличить его.
2. В поле **Тип значения** выберите, как значение, введенное на шаге 1, должно использоваться для расчета нового кредитного лимита:

    - Выберите **Фиксированное значение**, чтобы изменить кредитный лимит на сумму.
    - Выберите **Процент**, чтобы изменить кредитный лимит на процент.

3. Введите значение, которое используется для округления рассчитанного кредитного лимита. Например, введите **10,00** для округления кредитного лимита до ближайших 10,00 единиц валют.
4. Задайте поле **Способ округления**, чтобы указать, должен ли остаток быть округлен вверх или вниз.
5. Выберите метод, используемый для корректировки дат.

    - Если выбрано значение **Абсолютный**, можно ввести даты, задающие диапазон дат для кредитных лимитов.
    - Если выбрано значение **Относительный**, можно ввести даты смещения для диапазона. Текущий диапазон дат для кредитного лимита будет скорректирован по смещению.

6. Используйте экспресс-вкладку **Записи для добавления** для фильтрации списка включенных клиентов. Если фильтры не включены, записи кредитного лимита будут создаваться для всех клиентов.
7. Нажмите **ОК**, чтобы создать записи корректировки кредитного лимита.

### <a name="from-existing-customer-credit-group"></a>Из существующей кредитной группы клиентов

Можно создавать строки журнала из существующих кредитных групп клиентов. При выборе **Создать \> Из существующей кредитной группы клиентов** появится диалоговое окно, в котором можно ввести критерии для выбора кредитных групп клиентов и расчета новых лимитов. Критерии — это те же критерии, которые используются для создания строк журнала из существующих клиентов. См. шаги в предыдущем разделе.

### <a name="automatic-credit-limits"></a>Автоматические кредитные лимиты

Можно создавать автоматические кредитные лимиты для определения и обновления кредитных лимитов клиента. Автоматические кредитные лимиты создаются для группы риска, и они основываются на указанных значениях, используемых в группах показателей. Можно использовать эти автоматические кредитные лимиты для создания записей кредитного лимита. Если клиент назначен определенной группе риска, а кредитные данные клиента соответствуют критериям автоматического кредитного лимита, создается запись корректировки кредитного лимита.

#### <a name="create-automatic-credit-limits"></a>Создание автоматических кредитных лимитов

Создание автоматических кредитных лимитов для клиентов через корректировки кредитного лимита. При выборе **Создать \> Автоматические кредитные лимиты** появляется диалоговое окно, в котором можно задать дату окончания срока действия для новых кредитных лимитов, которые будут созданы на основе групп риска, которым назначены клиенты. После завершения нажмите **ОК**, чтобы создать строки корректировки кредитного лимита.

### <a name="post-adjustments"></a>Разноска корректировок

После создания строк корректировки кредитного лимита можно использовать кнопку **Разноска** на панели операций для разноски записей и обновления кредитных лимитов клиента. Однако если настроен workflow-процесс, необходимо выбрать **Workflow-процесс \> Отправить** на панели действий, чтобы отправить журнал на утверждение.

### <a name="credit-limit-adjustments-workflows"></a>Workflow-процессы корректировок кредитного лимита

Workflow -процессы **Корректировки кредитного лимита** можно использовать для отправки корректировок кредитного лимита с помощью процесса утверждения workflow-процесса. На странице **Workflow-процесс управления кредитованием** можно создать два workflow-процесса (**Управление кредитом \> Настройка \> Workflow-процесс управления кредитованием**):

- **Корректировки кредитного лимита** — этот workflow-процесс может использоваться для утверждения записей на уровне заголовка.
- **Строка корректировки кредитного лимита** — этот workflow-процесс можно использовать для утверждения записей корректировки таким образом, чтобы эти записи могли быть утверждены другими лицами на основе критериев в workflow-процессе.

> [!NOTE]
> При создании workflow-процесса **Корректировки кредитного лимита** его можно настроить таким образом, чтобы корректировки автоматически разносились после утверждения строк. Просто включите в workflow-процесс задачу **Автоматическая разноска журнала**.
