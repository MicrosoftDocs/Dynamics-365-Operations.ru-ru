# <a name="collections-in-accounts-receivable"></a>Сборы в расчетах с клиентами

[!include [banner](../includes/banner.md)]

Управление сведениями по сбору задолженностей осуществляется централизованно с помощью страницы Сборы Microsoft Dynamics 365 Finance. Менеджеры по кредитам и сборам могут использовать его для управления сборами задолженностей. Агенты по сбору задолженностей могут начать сборы по спискам клиентов, созданным с помощью заранее определенных критериев по сборам, или на странице "Клиенты".

Перед настройкой или работой со сборами следует понять следующие основные принципы.
-   Снимки распределения по срокам клиента содержат сведения о сальдо с распределением по срокам на определенный момент времени
-   Клиентские пулы сборов помогают организовать работу
-   У агентов по сбору задолженностей могут быть собственные клиентские пулы
-   На страницах списков упорядочены сборы, клиенты, мероприятия и обращения
-   Вся информация о сборах для клиента находится на одной странице, и действие можно выполнить с этой страницы
-   Отказ, восстановление или реверсирование процента и сборов можно выполнить за один шаг
-   Создание проводок списания можно выполнить за один шаг
-   Обработка платежей, не обеспеченных денежным покрытием, может быть выполнена за один шаг

В следующих разделах описан каждый из этих принципов.

## <a name="customer-aging-snapshots"></a>Снимки распределения по срокам оплаты для клиента
Снимок распределения по срокам содержит рассчитанные сальдо с распределением по срокам для клиента на определенный момент времени. Эта информация отображается на странице "Сальдо с распределением по срокам" и на странице "Сборы". Прежде чем можно будет просматривать информацию на страницах списков сборов, необходимо создать снимок распределения по срокам. 

Для каждого клиента, снимок распределения по срокам содержит заголовок снимка распределения по срокам и записи сведений, которые соответствуют каждому периоду распределения по срокам в определении периода распределения по срокам. 

Заголовок снимка распределения по срокам содержит общую сумму, подлежащую оплате, кредитный лимит, сумму по отборочной накладной, сумму по заказу на продажу, количество оспариваемых проводок и общую сумму по оспариваемым проводкам для счета клиента. Все проводки для клиента включаются в расчет этих сумм. Общая сумма, подлежащая оплате, кредитный лимит, сумма по отборочной накладной и сумма по заказу на продажу показаны в информационном поле "Сведения о кредитовании" на странице "Сборы". 

Для каждого периода распределения по срокам в определении периода распределения по срокам создается запись со сведениями о снимке распределения по срокам. Каждая запись со сведениями о снимке распределения по срокам содержит код периода распределения по срокам и общую сумму по проводкам, которые входят в период распределения по срокам. Проводки, назначенные для периода распределения по срокам, например 30 просроченных дней. Дата по отношению к дате "Распределение по срокам оплаты на дату", которая указывается при создании снимка распределения по срокам. Эта информация отображается на странице "Сальдо с распределением по срокам" и в информационном поле "Сальдо по срокам" на странице "Сборы".

## <a name="collections-customer-pools"></a>Клиентские пулы сборов
Клиентские пулы являются запросами, которые определяют группу записей клиента, которые могут отображаться и настраиваться для сборов или распределений по сроку. Используйте клиентские пулы для фильтрации информации на страницах списков "Сальдо по срокам", "Мероприятия по сборам" и "Обращения по сборам". Можно также использовать клиентские пулы для фильтрации счетов клиентов, которые включаются при создании снимков распределения по срокам.

## <a name="collections-agents"></a>Агенты по сбору задолженностей
По умолчанию пользователи могут просматривать всю информацию по клиенте на страницах списков сборов. Эти записи по агентам по сбору задолженностей можно использовать для определения клиентские пулов, доступных для фильтрации информации на страницах сборов и на странице "Сборы". 

Агент по сбору задолженностей — это лицо, работающее с клиентами с целью гарантии своевременной оплаты. Агенты по сборам — это работники, которые назначены потребителям на странице "Настройка пользователя".

## <a name="collections-list-pages"></a>Страницы списков сборов
Следующие страницы помогают организовать сведения о сборах.
-   Сальдо по срокам — в столбцах на странице списков показано сальдо клиента и суммы по периодам распределения по срокам. Эта информация хранится в снимке распределения по срокам. Распределение по срокам осуществляется в соответствии с используемым определением периода распределения по срокам. Определение периода распределения по срокам берется из клиентского пула, если таковое было задано для запроса пула. Если пул не имеет определения периода распределения по срокам, используется определение периода распределения по срокам по умолчанию, которое указано на странице "Параметры модуля расчетов с клиентами". Если определение периода распределения по срокам по умолчанию не указано, используется первое определение периода распределения по срокам на странице "Определения периодов распределения по срокам".
-   Мероприятия по сборам — столбцы на странице списков содержат мероприятия, которые определены как мероприятия по сборам. Эти мероприятия создаются с использованием страницы "Сборы". Мероприятия используются для отслеживания проделанной работы по сбору задолженностей.
-   Обращения по сборам — в столбцах на странице списков показана информация для обращений с категорией обращения с типом обращения "Сборы".

> [!NOTE]
> Прежде чем можно будет просматривать информацию на этих страницах списков, необходимо создать снимок распределения по срокам. Приводятся сведения только для клиентов, для которых создан снимок распределения по срокам. Записи, которые отображаются на странице списка, можно дополнительно фильтровать следующим образом.
> <li>По умолчанию пользователь управления финансами и операциями имеет доступ ко всем клиентам со снимком распределения по срокам.</li>
> <li>Если имеется клиентский пул, пользователь должен быть настроен как агент по сбору задолженностей для использования пулов для фильтрации информации на страницах списков сборов. Информация ограничена для клиентов, которые включены в выбранный клиентский пул.</li>
> <li>Если пользователь настроен как агент по сбору задолженностей, на странице списка доступны только пулы, которые выбраны для данного агента по сбору задолженностей. Если выбран флажок "Разрешить агенту просматривать все клиентские пулы" на странице "Агент по сбору задолженностей" для агента по сбору задолженностей, то все пулы доступны для того агента.</li>


## <a name="collections-page"></a>Страница "Сборы"
Используйте страницу "Сборы" для просмотра, управления и использования сведений о сборах, мероприятий и обращений для клиента. 

В верхней области показаны случаи для выбранного клиента. Средняя область показывает проводки для клиента. В нижней области показаны мероприятия для клиента. Можно создать обращения сборов, чтобы отслеживать информацию о сборах для одной или нескольких проводок и мероприятий. Информация в верхней и нижней областях может быть отфильтрована по обращениям. 

В информационном поле показаны сальдо с распределением по срокам и сведения по кредитному лимиту для выбранного клиента. Эта информация хранится в снимке распределения по срокам. Если необходимо, можно обновить снимок распределения по срокам, используя текущую информацию. 

На панели операций находятся кнопки действий, которые отображают соответствующую информацию для выбранного клиента, обращения, проводки или мероприятия. Можно также выполнить обычные действия, такие как изменение статуса сборов по проводке, отправка корреспонденции по электронной почте за счет интеграции с вашим поставщиком услуг электронной почты, возмещение средств клиентам, обработка платежей с недостаточным финансированием и списание безнадежного сальдо.

## <a name="waive-reinstate-or-reverse-interest-and-fees"></a>Отказ, восстановление или отмена процентов и сборов
Можно отказаться, восстановить или реверсировать заполненные процент-ноты или сборы и процент по проводке, которые входят в процент-ноту. Это можно сделать на вкладке "Собрать" на панели операций на странице списка "Все клиенты", щелкнув "Процент-нота", "Процент по проводке" или "Сбор". 

Эти корректировки влияют только на процент-ноты и входящие в них процент и сборы. Используйте шаги в разделе "Создание проводок списания за один шаг", чтобы списать все накладные расходы, имеющиеся у клиента.

Дополнительные сведения см. в разделах [Создание кода процента с диапазоном](tasks/create-interest-code-range.md) и [Обработка процента](tasks/process-interest.md). 

## <a name="create-writeoff-transactions"></a>Создание проводок списания
Вы можете написать безнадежные долги, нажав "Списание" в форме "Сборы", а также на страницах списков "Сальдо по срокам", "Клиенты" и "Открытые накладные клиента". 

При списании проводок для клиента все проводки для клиента автоматически помечаются для сопоставления. Списываемая сумма зависит от чистой суммы помеченных проводок. Проводка списания создается в общем журнале и может содержать до 3 типов строк журнала.

-   Первый тип строки журнала содержит запись списания по клиенту. Если помеченные проводки содержат несколько комбинаций кода валюты, аналитики и профиля разноски, для каждой комбинации создается отдельная строка журнала.
-   Второй тип строки журнала содержит запись списания главной книги. Если помеченные проводки содержат несколько комбинаций кода валюты, аналитики и профиля разноски, для каждой комбинации создается отдельная строка журнала.
-   Третий тип строки журнала содержит данные по списанию главной книги для налогов. Эта строка журнала создается, только если выбран флажок "Отдельный налог" на странице "Параметры модуля расчетов с клиентами". Если помеченные проводки содержат несколько комбинаций счета исходящего налога, аналитики и кода налога, для каждой комбинации создается отдельная строка журнала.

Проводка списания создается в валюте проводки.

Дополнительные сведения см. в разделе [Создание журнала списания для клиента](tasks/create-write-off-journal-customer.md)

## <a name="process-not-sufficient-funds-nsf-payments"></a>Обработка платежей, не обеспеченных денежным покрытием 

Вы можете обрабатывать платежи с недостаточным финансированием на странице "Сборы". При нажатии этой кнопки платеж отменяется. Если к клиенту применим штраф за необеспеченность денежными средствами, в журнале платежей создается проводка по накладным расходам. Сумма штрафа зависит от настроек автоматических накладных расходов. Автоматические взыскания, применимые к необеспеченным денежными средствами платежам, указаны выбранной на странице "Банковские счета" группой накладных расходов для соответствующего банковского счета.
