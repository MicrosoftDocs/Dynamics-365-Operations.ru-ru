# <a name="collections-management-key-concepts"></a>Основные понятия управления сборами

[!include [banner](../includes/banner.md)]

Перед началом настройки или работы со сборами следует понять следующие основные принципы.

- Снимки распределения по срокам клиента содержат сведения о сальдо с распределением по срокам на определенный момент времени.
- Клиентские пулы сборов помогают организовать работу.
- У агентов по сбору задолженностей могут быть собственные клиентские пулы.
- На страницах списков упорядочены сборы, клиенты, мероприятия и обращения.
- Вся информация о сборах для клиента находится на одной странице, и действие можно выполнить с этой страницы.
- Отказ, восстановление или реверсирование процента и сборов можно выполнить за один шаг.
- Создание проводок списания можно выполнить за один шаг.
- Обработка платежей с недостаточным финансированием может быть выполнена за один шаг.

В этой теме описывается каждая из этих концепций.

## <a name="customer-aging-snapshots"></a>Снимки распределения по срокам оплаты для клиента

Снимок распределения по срокам содержит рассчитанные сальдо с распределением по срокам для клиента на определенный момент времени. Эта информация отображается на странице **Сальдо по срокам** и на странице **Сборы**. Снимок распределения по срокам должен быть создан до того, как можно будет просматривать информацию на страницах списков сборов (**Сальдо по срокам**, **Мероприятия по сборам** и **Обращения по сборам**).

Для каждого клиента, снимок распределения по срокам содержит заголовок снимка распределения по срокам и записи сведений, которые соответствуют каждому периоду распределения по срокам в определении периода распределения по срокам.

Заголовок снимка распределения по срокам содержит общую сумму, подлежащую оплате, кредитный лимит, сумму по отборочной накладной, сумму по заказу на продажу, количество оспариваемых проводок и общую сумму по оспариваемым проводкам для счета клиента. Все проводки для клиента включаются в расчет этих сумм. Общая сумма, подлежащая оплате, кредитный лимит, сумма по отборочной накладной и сумма по заказу на продажу показаны в информационном поле **Сведения о кредитовании** на странице **Сборы**.

Для каждого периода распределения по срокам в определении периода распределения по срокам создается запись со сведениями о снимке распределения по срокам. Каждая запись со сведениями о снимке распределения по срокам содержит код периода распределения по срокам и общую сумму по проводкам, которые входят в период распределения по срокам. Проводки, назначенные для периода распределения по срокам, например 30 просроченных дней. Дата по отношению к значению даты **Распределение по срокам оплаты на дату**, которая указывается при создании снимка распределения по срокам. Эта информация отображается на странице **Сальдо с распределением по срокам** и в информационном поле **Сальдо по срокам** на странице **Сборы**.

## <a name="collections-customer-pools"></a>Клиентские пулы сборов

Клиентские пулы — это запросы, которые определяют группу записей клиентов. Эти сгруппированные записи можно использовать для просмотра информации о включенных счетах клиента, а также для управления сборами или процессами распределения по срокам для них. Используйте клиентские пулы для фильтрации информации на страницах список сборов. Можно также использовать их для фильтрации счетов клиентов, которые включаются при создании снимков распределения по срокам.

## <a name="collections-agents"></a>Агенты по сбору задолженностей

По умолчанию пользователи могут просматривать всю информацию по клиенте на страницах списков сборов. Записи по агентам по сбору задолженностей можно использовать для определения клиентские пулов, доступных для фильтрации информации на страницах сборов и на странице **Сборы**.

Агенты по сбору задолженностей работают с клиентами с целью гарантии своевременной оплаты. Это работники, которые назначены потребителям на странице **Настройка пользователя**.

## <a name="collections-list-pages"></a>Страницы списков сборов

Следующие страницы помогают организовать сведения о сборах:

- **Сальдо по срокам** — в столбцах на странице списков показано сальдо клиента и суммы по периодам распределения по срокам. Эта информация хранится в снимке распределения по срокам. Распределение по срокам осуществляется в соответствии с используемым определением периода распределения по срокам. Определение периода распределения по срокам берется из клиентского пула, если клиентский пул был задан для запроса пула. Если пул не имеет определения периода распределения по срокам, используется определение периода распределения по срокам по умолчанию, которое указано на странице **Параметры модуля расчетов с клиентами**. Если определение периода распределения по срокам по умолчанию не указано, используется первое определение периода распределения по срокам на странице **Определения периодов распределения по срокам**.
- **Мероприятия по сборам** — столбцы на странице списков содержат мероприятия, которые определены как мероприятия по сборам. Эти мероприятия создаются с использованием страницы **Сборы**. Мероприятия используются для отслеживания проделанной работы по сбору задолженностей.
- **Обращения по сборам** — в столбцах на странице списков показана информация для обращений с категорией обращения с типом обращения **Сборы**.

### <a name="aging-snapshots"></a>Снимки распределения по срокам

Прежде чем можно будет просматривать информацию на страницах списков сборов, необходимо создать снимок распределения по срокам. Приводятся сведения только для клиентов, для которых создан снимок распределения по срокам. Записи, которые отображаются на странице списка, можно дополнительно фильтровать следующим образом:

- По умолчанию у пользователей есть доступ ко всем клиентам со снимком распределения по срокам.
- Если имеется клиентский пул, пользователь должен быть настроен как агент по сбору задолженностей для использования пулов для фильтрации информации на страницах списков сборов. Информация ограничена для клиентов, которые включены в выбранный клиентский пул.
- Если пользователь настроен как агент по сбору задолженностей, на странице списка сборов доступны только пулы, которые выбраны для данного агента по сбору задолженностей. Если выбран параметр **Разрешить агенту просматривать все клиентские пулы** на странице **Агент по сбору задолженностей** для агента по сбору задолженностей, то все пулы доступны для того агента.

## <a name="collections-page"></a>Страница "Сборы"

Можно использовать страницу **Сборы** для просмотра, управления и использования сведений о сборах, мероприятий и обращений для клиента.

В верхней части страницы показаны обращения для выбранного клиента, в среднем разделе показаны проводки для клиента, а в нижней области показаны мероприятия для клиента. Можно создать обращения сборов, чтобы отслеживать информацию о сборах для одной или нескольких проводок и мероприятий. Информация в верхнем и нижнем разделах быть отфильтрована по обращениям.

В информационном поле показаны сальдо с распределением по срокам и сведения по кредитному лимиту для выбранного клиента. Эта информация хранится в снимке распределения по срокам. Если необходимо, можно обновить снимок распределения по срокам, используя текущую информацию.

На панели операций находятся кнопки действий, которые отображают соответствующую информацию для выбранного клиента, обращения, проводки или мероприятия. Можно также выполнить обычные действия, такие как изменение статуса сборов по проводке, отправка по электронной почте за счет интеграции с вашим поставщиком услуг электронной почты, возмещение средств клиентам, обработка платежей с недостаточным финансированием и списание безнадежного сальдо.

## <a name="waiving-reinstating-or-reversing-interest-and-fees"></a>Отказ, восстановление или отмена процентов и сборов

Можно отказаться, восстановить или реверсировать заполненные процент-ноты или сборы и процент по проводке, которые входят в процент-ноту. Это можно сделать на вкладке **Собрать** на панели операций на странице списка **Все клиенты**, щелкнув **Процент-нота**, **Процент по проводке** или **Сбор**.

Эти корректировки влияют только на процент-ноты и входящие в них процент и сборы. Сведения о порядке списания задолженностей клиента см. в разделе [Создание проводок списания](#creating-write-off-transactions) данного раздела.

Дополнительные сведения см. в разделах Создание кода процента с диапазоном и Обработка процента.

## <a name="creating-write-off-transactions"></a>Создание проводок списания

Можно списать неправильные задолженности, выбрав на страницах списков сборов **Списать** на странице **Сборы**.

При списании проводок для клиента все проводки для клиента автоматически помечаются для сопоставления. Списываемая сумма зависит от чистой суммы помеченных проводок. Проводка списания создается в общем журнале и может содержать до 3 типов строк журнала:

- Первый тип строки журнала содержит запись списания по клиенту. Если помеченные проводки содержат несколько комбинаций кодов валюты, аналитик и профилей разноски, для каждой комбинации создается отдельная строка журнала.
- Второй тип строки журнала содержит запись списания главной книги. Если помеченные проводки содержат несколько комбинаций кодов валюты, аналитик и профилей разноски, для каждой комбинации создается отдельная строка журнала.
- Третий тип строки журнала содержит данные по списанию главной книги для налогов. Эта строка журнала создается, только если выбран параметр **Отдельный налог** на странице **Параметры модуля расчетов с клиентами**. Если помеченные проводки содержат несколько комбинаций счетов исходящего налога, аналитик и кодов налога, для каждой комбинации создается отдельная строка журнала. Проводка списания создается в валюте проводки. Дополнительные сведения см. в разделе Создание журнала списания для клиента

## <a name="process-nsf-payments"></a>Обработка платежей с недостаточным финансированием

Вы можете обрабатывать платежи с недостаточным финансированием, выбрав **Платеж с недостаточным финансированием** на странице **Сборы**. При нажатии этой кнопки платеж отменяется. Если к клиенту применим штраф за необеспеченность денежными средствами, в журнале платежей создается проводка по накладным расходам. Сумма штрафа зависит от настроек автоматических накладных расходов. Автоматические взыскания, применимые к необеспеченным денежными средствами платежам, указаны выбранной на странице **Банковские счета** группой накладных расходов для соответствующего банковского счета.
