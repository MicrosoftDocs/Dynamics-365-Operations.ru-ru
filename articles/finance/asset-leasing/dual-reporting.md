# <a name="dual-reporting"></a>Двойная отчетность

[!include [banner](../includes/banner.md)]

В этой теме рассматривается пример, показывающий, как можно выполнить требования как для международной финансовой отчетности по стандарту МСФО, так и для регламентной отчетности в аренде активов. Знакомство с уровнями разноски в Microsoft Dynamics 365 Finance является обязательным и облегчит понимание этого примера.

## <a name="example"></a>Пример

Следующий пример счетов для аренды в соответствии с итальянскими нормативными отчетами с использованием обоих методов расчета наличных денег и способов отчетности МСФО. Компания должна создать три книги для учета как для итальянских регламентных требований, так и требований МСФО 16. Для МСФО 16 требуется одна книга, одна книга требуется которой для регламентных требований, и одна книга требуется для автоматической реверсирования регламентных разносок. Книги настраиваются так, как показано в следующих таблицах.

**Книга МСФО 16**

Книга МСФО 16 настроена таким образом, что она соответствует стандарту отчетности МСФО 16. Все записи, учитываемые в этой книге, будут разнесены в пользовательский слой.

| Наименование                                    | описание    |
|-----------------------------------------|----------------|
| Тип книги                               | МСФО 16        |
| Бухгалтерская запись                        | МСФО 16        |
| Слой разноски                           | Пользовательский слой 1 |
| Тип аренды                              | Finance        |
| Платформа учета                    | МСФО 16        |
| Настройка отношения срока аренды к сроку службы         | 0.00           |
| Настройка отношения "Текущая стоимость / Реальная стоимость актива" | 0.00           |
| Порог краткосрочной аренды                    | 12             |
| Порог аренды малоценного актива                     | 5,000.00       |
| Оплата поставщику                           | Нет             |

**Регламентная книга**

Регламентная книга — это книга на основе наличных, в которой компания будет учитывать расходы на аренду в качестве суммы денежных средств, выплачиваемых ежемесячно за аренду. Эта книга не будет создавать актив в форме права пользования или арендного обязательства.

| Настройка отношения "Текущая стоимость / Реальная стоимость актива" | 0.00        |
| Порог краткосрочной аренды                    | 0           |
| Порог аренды малоценного актива                     | 0           |
| Оплата поставщику                           | Нет          |

**Книга регламентного сторнирования**

Книга регламентного сторнирования настраивается так же, как и регламентная книга.

Как показано в предыдущей таблице, три книги требуются для обеспечения как регламентной отчетности, так и отчетности МСФО.

- Регламентная книга записывает платеж по аренде в соответствии с правилами учета денежных средств в текущем слое.
- В книге по регламентному сторнированию отменяются записи регламентного журнала.
- Книга МСФО 16 создает записи журнала, которые необходимы в соответствии с МСФО 16.

Аренда должна быть введена только один раз. Затем можно открыть страницу **Книги**, чтобы просмотреть все книги, связанные с арендой.

> [!NOTE]
> При создании книг все три из них должны быть связаны с одной и той же записью аренды.

Первая запись журнала записывает расходы на аренду в рамках регламентной книги. Платежи можно создавать либо пакетом, либо путем выбора графика оплаты в регламентной книге.

В данном примере следующая запись журнала создается для регламентной книги из графика оплаты.

Если вы хотите использовать цифры МСФО 16 для запуска того же пробного баланса, записи журнала регламентного учета должны быть реверсированы, а записи журнала МСФО 16 должны быть разнесены. Для реверсирования записей регламентного журнала этот пример включает регламентную книгу сторнирования с той же настройкой, что и у регламентной книги, но с противоположным профилем разноски. Например, регламентная книга *дебетовала* на счет расходов на аренду, но книга сторнирования будет *кредитовать* этот счет. Эти связи легко определить в счетах разноски аренды активов на странице **Параметры аренды активов** (**Аренда активов \> Настройка \> Параметры аренды активов**).

При использовании того же процесса, который использовался для регламентной книги, используется для книги сторнирования, будет создана следующая запись журнала. Разница заключается в том, что книга сторнирования резервирует записи сторнирования из регламентной книги. Операции сторнирования также выполняются в пользовательском слое. Когда пробный баланс запускается на текущем слое, операции журнала сторнирования не включаются.
