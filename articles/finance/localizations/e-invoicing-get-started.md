# <a name="get-started-with-electronic-invoicing"></a>Начало работы с электронным выставлением накладных

[!include [banner](../includes/banner.md)]

В этой теме приводятся сведения, которые помогут приступить к работе с электронным выставлением накладных. В этой теме описываются общие этапы настройки в Regulatory Configuration Services (RCS) и Dynamics 365 Finance и описываются шаги, которые необходимо выполнить, чтобы отправить деловые документы и просмотреть результаты обработки.

## <a name="prerequisites"></a>Необходимые условия

Перед выполнением процедур из данной темы необходимо выполнить следующие предварительные требования:

- Настройте Microsoft Dynamics Lifecycle Services (LCS), Regulatory Configuration Service (RCS), а также среду Microsoft Dynamics 365 Finance или Dynamics 365 Supply Chain Management. Дополнительные сведения см. в разделе [Начало работы с администрированием службы электронного выставления накладных](e-invoicing-get-started-service-administration.md).
- Создайте поставщика конфигурации для организации. Дополнительные сведения см. в разделе [Создание поставщика конфигураций и пометка его как активного](../../fin-ops-core/dev-itpro/analytics/tasks/er-configuration-provider-mark-it-active-2016-11.md).

## <a name="import-an-electronic-invoicing-feature-from-the-microsoft-configuration-provider"></a>Импорт функции электронного выставления накладных из поставщика конфигурации Microsoft 

1. Выполните вход в учетную запись Regulatory Configuration Services (RCS).
2. В рабочей области **Функции глобализации** в разделе **Функции** выберите плитку **Электронное выставление накладных**.
3. Выберите **Импорт** и затем выберите **Синхронизировать**.
4. Отфильтруйте столбец **Поставщик конфигураций** по условию **Microsoft**.
5. Выберите имя функции электронного выставления накладных из таблицы, а затем выберите **Импорт**.

## <a name="create-an-electronic-invoicing-feature-under-your-organization-provider"></a>Создание функции электронного выставления накладных в вашем поставщике организации

1. В RCS в разделе **Функции** рабочей области **Функции глобализации** выберите плитку **Электронное выставление накладных**.
2. Выберите **Добавить** > **На основе существующей функции**, и в поле **Имя** введите имя функции электронного выставления накладных.
3. В поле **Описание** введите описание функции.
4. В поле **Базовая функция** выберите импортированную функцию электронного выставления накладных из поставщика конфигурации Microsoft.
5. Выберите **создать функцию**.

## <a name="country-specific-configuration-for-electronic-invoicing-feature"></a>Конфигурация функции электронного выставления накладных для определенной страны или региона

В зависимости от страны или региона, функция электронного выставления накладных может потребовать специальной настройки. 

Специальные шаги см. в документации "Начало работы", доступной для вашей страны или региона.

## <a name="country-specific-configuration-of-application-setup"></a>Зависящая от страны или региона конфигурация настройки приложения

В зависимости от страны или региона, настройка приложения может потребовать специальной конфигурации. 

Специальные шаги см. в документации "Начало работы", доступной для вашей страны или региона.

## <a name="deploy-the-electronic-invoicing-feature-to-service-environment"></a>Развертывание функции электронного выставления накладных в среде обслуживания

1. На вкладке **Версии** выберите версию функции электронного выставления накладных, которую необходимо развернуть.
2. Выберите **Изменить статус** \> **Завершить**.
3. Выберите **Изменить статус** \> **Опубликовано**.
4. Выберите **Развернуть**.
5. Установите для параметра **Развернуть в подключенное приложение** значение **Нет**.
6. Установите для параметра **Развернуть в среду службы** значение **Да**.
7. В поле **Среда службы** выберите среду службы электронного выставления накладных, в которой необходимо развернуть функцию электронного выставления накладных.
8. В поле **Дата начала** выберите дату, когда функция электронного выставления накладных должна быть эффективной для электронного выставления накладных.
9. Нажмите **ОК**.

## <a name="deploy-the-electronic-invoicing-feature-to-connected-application"></a>Развертывание функции электронного выставления накладных в подключенном приложении

1. На вкладке **Версии** выберите версию функции электронного выставления накладных, которую необходимо развернуть.
2. Выберите **Развернуть**.
3. Установите для параметра **Развернуть в подключенное приложение** значение **Да**.
4. В поле **Подключенное приложение** выберите подключение, связанное с вашим экземпляром Finance или Supply Chain Management.
5. Установите для параметра **Развернуть в среду службы** значение **Нет**.
6. Нажмите **ОК**.

## <a name="issue-electronic-invoices"></a>Выпуск электронных накладных

1. Перейдите в раздел **Администрирование организации** \> **Периодические задачи** \> **Электронные документы** \> **Отправка электронных документов**.
2. На экспресс-вкладке **Записи для добавления** выберите **Фильтр**.
3. Выберите **Добавить**, чтобы добавить имя таблицы в фильтр запроса.
4. Выберите таблицу, содержащую накладные.

    > [!NOTE]
    > Имя таблицы должно быть указано в таблице в разделе [Настройка настройки приложения](#configure-the-application-setup) ранее в этом разделе.

5. Выберите имя поля из таблицы, которую необходимо запросить.
6. Введите имя таблицы и имя поля для критерия, по которому хотите выполнить запрос.
7. Повторите шаги 5 и 6, чтобы добавить дополнительные поля и критерии в запрос, затем выберите **ОК**.
8. Нажмите **ОК**.

## <a name="view-submission-logs"></a>Просмотр журналов отправки

1. Перейдите в раздел **Администрирование организации** \> **Периодические задачи** \> **Электронные документы** \> **Журнал отправки электронных документов**.
2. В поле **Тип документа** выберите таблицу, которая содержит накладные.

    > [!NOTE]
    > Имя таблицы должно быть указано в таблице в разделе [Настройка настройки приложения](#configure-the-application-setup) ранее в этом разделе.

3. Выберите накладную в сетке, затем выберите **Запрос** \> **Сведения об отправке**.
