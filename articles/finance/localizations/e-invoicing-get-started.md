# <a name="get-started-with-the-electronic-invoicing-add-on"></a>Приступая к работе с дополнительным компонентом электронных накладных

[!include [banner](../includes/banner.md)]

В этой теме приводятся сведения, которые помогут приступить к работе с дополнительным компонентом электронного выставления накладных. Во-первых, она поможет выполнить шаги настройки в службах Microsoft Dynamics Lifecycle Services (LCS), Regulatory Configuration Services (RCS) и в Dynamics 365 Finance. Затем описывается процесс отправки документов через службу с помощью Dynamics 365 Finance или Dynamics 365 Supply Chain Management. Также будет рассмотрено, как интерпретировать журналы отправки.

## <a name="availability"></a>Доступность

Дополнение электронных накладных изначально доступно для нескольких стран. Дополнение поддерживает создание электронных накладных и отправку следующих деловых документов:

| Страна/регион  | Бизнес-документ                          |
|-----------------|--------------------------------------------|
| Австрия         | Накладные по продаже и по проекту                 |
| Бельгия         | Накладные по продаже и по проекту                 |
| Бразилия          | Модель электронных финансовых документов 55 (NF-e) |
| Дания         | Накладные по продаже и по проекту                 |
| Эстония         | Накладные по продаже и по проекту                 |
| Финляндия         | Накладные по продаже и по проекту                 |
| Франция          | Накладные по продаже и по проекту                 |
| Германия         | Накладные по продаже и по проекту                 |
| Италия           | Накладные по продаже и по проекту                 |
| Мексика          | Счет CFDI                               |
| Голландия     | Накладные по продаже и по проекту                 |
| Норвегия          | Накладные по продаже и по проекту                 |
| Испания           | Накладные по продаже и по проекту                 |
| Европа          | Накладные PEPPOL по продаже и по проекту          |
    
## <a name="licensing"></a>Лицензирование

Можно использовать дополнение электронных накладных с текущей лицензией. Дополнительные лицензии для использования этой службы не требуются.

## <a name="prerequisites"></a>Необходимые условия

Перед выполнением шагов из данной темы необходимо выполнить следующие предварительные требования:

- Доступ к вашей учетной записи LCS.
- Проект развертывания LCS, который включает в себя Finance или Supply Chain Management 10.0.13 или более поздней версии.
- Доступ к вашей учетной записи RCS.
- Включите функцию глобализации для вашей учетной записи RCS с помощью модуля **Управление функциями**. Дополнительные сведения см. в разделе [Regulatory Configuration Services (RCS) — функции глобализации](rcs-globalization-feature.md)
- Создайте ресурс Key Vault и учетную запись хранилища в Azure. Дополнительные сведения см. в разделе [Создание учетной записи хранения Azure и Key Vault](e-invoicing-create-azure-storage-account-key-vault.md).

## <a name="overview"></a>Обзор

На следующем рисунке показаны пять основных шагов, которые необходимо выполнить в этой теме.

![Обзор пяти шагов данной темы](media/e-invoicing-services-get-started-overview-5-steps.png)

1. **Настройка ресурсов Azure:** настройка хранилища Azure и отправка цифровых сертификатов в Azure Key Vault.
2. **Настройка LCS:** установите надстройку для микрослужб.
3. **Настройка RCS:** настройте среды, доступ пользователя и функций электронного выставления счетов.
4. **Настройка клиента:** настройте связь между клиентом и дополнением электронных накладных, а также отключите старые функции отправки и получения ответов для электронных документов.
5. **Отправка накладных:** отправка электронных документов с помощью дополнения электронных накладных и получение ответов.

> [!NOTE]
> Некоторые шаги настройки в этом разделе являются общими и независимыми от стран/регионов. Шаги и процедуры настройки, зависящие от страны/региона, описываются в разделах, посвященных конкретным странам/регионам.

## <a name="lcs-setup"></a>Настройка LCS

1. Войдите в учетную запись LCS.
2. Выберите плитку **Управление компонентами предварительной версии** , затем в группе поле **Компоненты общей предварительной версии** выберите **BusinessDocumentSubmission**.
3. Отметьте поле **Включенные компоненты предварительной версии**.
4. Выберите проект развертывания LCS. Прежде чем можно будет выбрать проект, он должен быть установлен и запущен.
5. На экспресс-вкладке **Надстройки сред** выберите **Установить новую надстройку**.
6. Выберите **Отправка деловых документов**.
7. В диалоговом окне **Настроить надстройку** в поле **Код приложения AAD** введите **091c98b0-a1c9-4b02-b62c-7753395ccabe**. Это значение является фиксированным значением.
8. В поле **Код клиента AAD** введите идентификатор вашей учетной записи подписки Azure.

    ![Диалоговое окно настройки надстройки в LCS](media/e-invoicing-services-get-started-lcs-addin-setup.png)

9. Установите этот флажок, чтобы принять условия.
10. Выберите **Установить**.

## <a name="rcs-setup"></a>Настройка RCS

Во время настройки RCS будут выполнены следующие задачи:

1. Настройте хранилище ключей в RCS.
2. Настройте интеграцию RCS с дополнительным сервером электронного выставления накладных.
3. Создайте среду дополнения электронных накладных для вашей организации.

### <a name="set-up-the-key-vault-in-rcs"></a>Настройка хранилища ключей в RCS

1. Войдите в учетную запись RCS.
2. В рабочей области **Функции глобализации** в разделе **Среды** выберите плитку **Электронные накладные**.
3. Выберите **Среды обслуживания**.

    ![Выбор сред обслуживания](media/e-invoicing-services-get-started-select-service-environments.png)

> [!NOTE]
> Параметр **Подсоединенные приложения** предоставляет доступ для автоматической настройки дополнения электронных накладных в Finance или Supply Management через RCS. Однако в настоящее время эта функция все еще находится в разработке.

4. На панели операций выберите **Параметры Key Vault**.

    ![Выбор параметра Key Vault](media/e-invoicing-services-get-started-select-key-vault-parameters.png)

5. На панели операций выберите **Создать** для добавления хранилища ключей.
6. В поле **URI хранилища ключей** введите значение атрибута **Имя DNS** для ресурса хранилища ключей, настроенного в Azure. Сведения о том, где искать значение атрибута **Имя DNS** , см. в разделе [Создание учетной записи хранилища Azure и хранилища ключей](e-invoicing-create-azure-storage-account-key-vault.md).

    ![Поле URI хранилища ключей](media/e-invoicing-services-get-started-enter-key-vault-uri.png)

7. На экспресс-вкладке **Сертификаты** выберите **Добавить** , чтобы ввести все имена цифровых сертификатов и секреты хранилища ключей, необходимые для установления доверительных соединений. В столбце **Тип** можно указать, является ли он сертификатом или секретом. Оба набора значений настраиваются в ресурсе хранилища ключей в Azure.

    ![Добавление сертификатов](media/e-invoicing-services-get-started-add-digital-certificates.png)

8. Если в накладной для конкретной страны/региона требуется цепочка сертификатов для применения цифровой подписи, выберите **Цепочка сертификатов** на панели операций, затем введите последовательность сертификатов или секретов ключа хранилища, образующих цепочку.

### <a name="set-up-the-rcs-integration-with-the-electronic-invoicing-add-on-server"></a>Настройка интеграции RCS с дополнительным сервером электронного выставления накладных

1. В рабочей области **Функции глобализации** в разделе **Связанные параметры** выберите ссылку **Параметры электронной отчетности**.
2. Выберите **Щелкните здесь, чтобы подключиться к службе Lifecycle Service**. Если вы не хотите подключаться к LCS, выберите **Отмена**.
3. На вкладке **Службы выставления электронных накладных** в поле **URI конечной точки службы** введите значение в соответствии с доступными географическими положениями: `https://businessdocumentsubmission.us.operations365.dynamics.com/` или `https://businessdocumentsubmission.eu.operations365.dynamics.com/`.
4. В поле **Код приложения** проверьте, что отображается код **0cdb527f-a8d1-4bf8-9436-b352c68682b2**. Это значение является фиксированным значением.
5. В поле **Код среды LCS** введите идентификатор вашей учетной записи подписки LCS.

![Ввод параметров дополнения электронных накладных](media/e-invoicing-services-get-started-enter-e-invoicing-parameters.png)

### <a name="add-an-electronic-invoicing-add-on-environment"></a>Добавление среды дополнения электронных накладных

Можно создать различные среды для дополнения электронных накладных, например, среды разработки, тестирования или производства.

1. В рабочей области **Функции глобализации** в разделе **Среды** выберите плитку **Электронные накладные**.
2. Выберите **Создать** для создания среды.
3. В поле **Учетная запись токена SAS** введите имя секрета хранилища ключей, настроенного в хранилище ключей в RCS.

    ![Поле учетной записи токена SAS хранилища](media/e-invoicing-services-get-started-enter-sas-token-secret.png)

4. На экспресс-вкладке **Пользователи** выберите **Создать** , чтобы предоставить пользователям доступ к этой среде.

    ![Добавление пользователей служб](media/e-invoicing-services-get-started-enter-service-users.png)

5. На панели операций выберите **Опубликовать** для публикации среды на сервере дополнения электронных накладных.

    ![Кнопка "Опубликовать"](media/e-invoicing-services-get-started-publish-service-environment.png)

### <a name="e-invoicing-feature-setup"></a>Настройка функции электронных накладных

"Функция электронных накладных" — это универсальное имя для ресурса, который настраивается и публикуется для использования сервера дополнения электронных накладных. Настройка функции электронных накладных объединяет, помимо всего прочего, использование форматов конфигурации электронной отчетности для создания настраиваемых файлов экспорта и импорта и использования действий и потоков действий для разрешения создания настраиваемых правил для отправки запросов, импорта ответов и обработки содержимого ответов.

Из-за изменений в форматах накладных и в потоках действий настройка функции электронного выставление накладных зависит от страны/региона.

## <a name="set-up-electronic-invoicing-add-on-integration-in-finance-or-supply-chain-management"></a>Настройка интеграции дополнения электронных накладных в Finance или Supply Chain Management 

Во время этой настройки вы выполните следующие задачи:

1. Открытие функции фокус-тестирования.
2. Включение функции интеграции дополнения электронного выставления накладных, чтобы включить интеграцию с Finance.
3. Настройка URL-адреса конечной точки дополнения электронного выставления накладных
4. Импорт конфигурации электронной отчетности, которые относятся к функции электронного выставления накладных, связанных с конкретной страной/регионом.
5. Включение требуемой функции электронного выставления накладных для конкретной страны или региона.
6. Импорт конфигураций электронной отчетности и настройка типов откликов, которые необходимы для обновления документа накладной, предназначенной для конкретной страны/региона, в результате процесса отправки.

### <a name="open-flighted-feature"></a>Открытие функции фокус-тестирования
Функция интеграции электронных накладных включается через фокус-тестирование. Фокус-тестирование — это концепция, которая позволяет по умолчанию включить или выключить функцию. Следующие шаги позволяют включить фокус-тестирование в непроизводственной среде. 

1. Выполните следующую команду SQL:

    INSERT INTO SYSFLIGHTING (FLIGHTNAME, ENABLED) VALUES ('BusinessDocumentSubmissionServiceEnabled', 1)
    
    INSERT INTO SYSFLIGHTING (FLIGHTNAME, ENABLED) VALUES ('ElectronicInvoicingServiceIntegrationFeature', 1)
    
2. После выполнения предыдущего изменения выполните IISReset на всех AOS

### <a name="turn-on-the-electronic-invoicing-add-on-integration-feature"></a>Включение функции интеграции дополнения электронных накладных

1. Войдите в Finance или Supply Chain Management.
2. В рабочей области **Управление функциями** выполните поиск новой функции **Настраиваемая интеграция дополнения электронных накладных**. Если эта функция все еще не отображается на странице управления функциями, выполните функцию **Проверка наличия обновлений**
3. Выберите эту функцию, затем выберите **Включить сейчас**.

### <a name="set-up-the-service-endpoint-url"></a>Настройка URL-адреса конечной точки службы

1. Перейдите в раздел **Администрирование организации \> Настройка \> Параметры электронных документов**.
2. На вкладке **Служба отправки** в поле **URL конечной точки службы** введите `https://businessdocumentsubmission.us.operations365.dynamics.com/`.
3. В поле **Среда** введите название среды дополнения для электронного выставления накладных, которая была создана во время настройки RCS.

![Ввод параметров службы](media/e-invoicing-services-get-started-enter-service-endpoint.png)

### <a name="import-the-er-configurations"></a>Импорт конфигураций электронной отчетности

Чтобы включить сбор и отправку бизнес-данных в дополнение электронного выставления накладных, необходимо импортировать модель данных электронной отчетности и конфигурацию модели данных электронной отчетности, которые относятся к используемой функции электронного выставления накладных, связанных с выбранной страной/регионом.

1. В рабочей области **Электронная отчетность** в разделе **Поставщики конфигурации** выберите плитку **Microsoft**. Убедитесь, что этот поставщик конфигурации имеет значение **Активный**. Дополнительные сведения о том, как задать для поставщика состояние **Активный** см. в разделе [Создание поставщиков конфигураций и пометка их как активных](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/analytics/tasks/er-configuration-provider-mark-it-active-2016-11).
3. Выберите **Репозитории**.
4. Выберите **Глобальный ресурс** , затем выберите **Открыть**.
5. В диалоговом окне **Подключение к Lifecycle Services** выберите **Щелкните здесь, чтобы подключиться к Lifecycle Services**.
6. В зависимости от страны или региона, в которых предполагается использовать функцию электронных накладных, необходимо импортировать подходящую модель данных, сопоставление модели данных и форматы. Сведения о конфигурациях электронной отчетности, которые необходимо импортировать, см. в разделе "Начать работу с дополнением электронных накладных" для конкретной страны/региона.
7. Импортируйте **Модель контекста накладной клиента**. Эта модель содержит дополнительные параметры, которые описывают, среди прочего, среду в Finance, используемую для дополнения электронного выставления накладных при отправке коммерческих данных.

### <a name="turn-on-countryregion-specific-e-invoicing-features"></a><a name="region-specific"></a>Включение функций электронного выставления накладных для конкретной страны или региона

Чтобы включить функции электронного выставления накладных, специфичные для страны/региона, чтобы они работали с дополнением электронного выставления накладных, необходимо включить функцию в каждом юридическом лице, в котором необходимо его использовать. После этого старую интеграцию электронного выставления накладных больше использовать нельзя, а интеграция с новым дополнением электронных накладных будет включена.

1. Перейдите в раздел **Администрирование организации \> Настройка \> Параметры электронных документов**.
2. На вкладке **Функции** в строке функции, относящейся к функции электронного выставления накладных, связанной с вашей страной/регионом, установите флажок в столбце **Включено**. Сведения о том, какую функцию следует включить, см. в разделе "Начать работу с дополнением электронных накладных" для конкретной страны/региона.

![Включение функции выставления накладных в электронном виде](media/e-invoicing-services-get-started-enable-invoicing-feature.png)

> [!NOTE]
> Если имеется несколько юридических лиц, настроенных для различных стран или регионов, можно включить функцию электронного выставления накладных для конкретной страны/региона для каждого юридического лица.

### <a name="import-er-configurations-and-set-up-the-response-types-to-update-your-countryregion-specific-invoice-document"></a>Импорт конфигураций электронной отчетности и настройка типов откликов для обновления документа накладных для конкретной страны/региона

Если отправленный документ накладной требует обновления после ответа на отправку правительственным службам авторизации, необходимо импортировать специальную модель и конфигурации данных электронной отчетности, чтобы включить статус документа накладной или любое другое дополнительное поле для обновления.

1. В рабочей области **Электронная отчетность** в разделе **Поставщики конфигурации** выберите плитку **Microsoft**.
2. Выберите **Репозитории**.
3. Выберите **Глобальный ресурс** , затем выберите **Открыть**.
4. Импортируйте **Модель ответного сообщения** , **Формат импорта ответного сообщения** , **Сопоставление модели сообщения ответа с пунктом назначения** и **Формат импорта содержимого файла**.
5. Перейдите в раздел **Администрирование организации \> Настройка \> Параметры электронных документов**.
6. На вкладке **Электронный документ** выберите **Добавить** , чтобы ввести имя таблицы, которая связана с документом накладной, связанным с конкретной страной/регионом. Сведения о том, какие имена таблиц следует выбрать, см. в разделе "Начать работу с дополнением электронных накладных" для конкретной страны/региона.
7. Выберите **Типы откликов** для настройки типов откликов. Сведения о том, какие имена таблиц следует выбрать, см. в разделе "Начать работу с дополнением электронных накладных" для конкретной страны/региона.

## <a name="electronic-invoice-processing-in-finance-and-supply-chain-management"></a>Обработка электронных накладных в Finance и Supply Chain Management

Во время обработки будут выполнены следующие задачи:

1. Отправка делового документа (накладной) через дополнение электронных накладных
2. Просмотр журналов выполнения отправки.

### <a name="submit-business-documents"></a>Отправка бизнес-документов

В процессе обычной отправки обмен данными между клиентом и дополнением электронных накладных является двунаправленным. Цель состоит в выполнении двух основных задач при отправке электронных документов:

1. Отправка всех электронных документов, которые ожидают отправки из Finance, и которые имеют правильный статус для отправки и удовлетворяют критериям выбора.
2. Импорт в Finance ответа, который дополнение электронного выставления накладных возвращает для ранее отправленных электронных документов. После импорта отклики анализируются, а статус деловых документов обновляется соответствующим образом.

Деловые документы можно отправлять либо вручную, либо в соответствии с вашими требованиями к планированию.

1. Перейдите в раздел **Администрирование организации \> Периодические задачи \> Электронные документы \> Отправка электронных документов**.
2. Для первой отправки любого документа обязательно установите для параметра **Повторно отправить документы** значение **Нет**. Если необходимо повторно отправить документ через службу, установите для этого параметра значение **Да**.
3. На экспресс-вкладке **Включаемые записи** выберите **Фильтр** , чтобы открыть диалоговое окно **Запрос** , в котором можно построить запрос для выбора повторно отправляемых документов.

![Диалоговое окно отправки электронных документов](media/e-invoicing-services-get-started-submission-form.png)

### <a name="filter-query"></a>Запрос фильтрации

1. В диалоговом окне **Запрос** на вкладке **Диапазон** введите критерий фильтра, используя поля **Таблица** , **Производная таблица** , **Поле** и **Критерии**.
2. Выберите **Добавить** , чтобы добавить столько дополнительных критериев, сколько требуется для выбора деловых документов.

    ![Настройка критериев фильтрации отправки](media/e-invoicing-services-get-started-set-up-submission-filter-criteria.png)

3. Выберите **ОК** , чтобы закрыть диалоговое окно **Запрос**.
4. Нажмите кнопку **ОК** , чтобы отправить выбранные деловые документы в дополнение электронных накладных.

    > [!NOTE]
    > При первой попытке отправить документ через службу вам будет предложено подтвердить связь с дополнительной функцией электронных накладных. Выберите **Щелкните здесь для подключения к службе отправки электронных документов**.
    >
    > ![Подключение к окну сообщения службы отправки электронных документов](media/e-invoicing-services-get-started-dialog-form-connect-e-Invoicing-services.png)
    >
    > Если подключение выполнено успешно, появится сообщение с запросом подтверждения.
    >
    > ![Подтверждение подключения к дополнительному модулю электронных накладных](media/e-invoicing-services-get-started-confirmation-connection-e-invoicing-services.png)

5. Закройте диалоговое окно.

> [!NOTE]
> После каждой отправки в центре действий отображается число отправленных документов.
>
> ![Сообщения центра действий](media/e-invoicing-services-get-started-view-action-center-messages.png)

### <a name="submission-by-batch"></a>Пакетная отправка

Вместо того, чтобы вручную отправлять документы, можно автоматизировать процесс отправки и выполнить его в фоновом режиме на основе настроенной частоты пакетного выполнения.

1. В диалоговом окне **Отправка электронных документов** на экспресс-вкладке **Выполнять в фоновом режиме** установите для параметра **Пакетная обработка** значение **Да**.
2. На вкладке **Повторение** настройте частоту пакетной обработки.

![Настройка пакетной отправки](media/e-invoicing-services-get-started-set-up-submission-batch.png)

### <a name="view-all-submission-logs"></a>Просмотр всех журналов отправки

1. Перейдите в раздел **Администрирование организации \> Периодические задачи \> Электронные документы \> Журнал отправки электронных документов**.
2. В поле **Тип документа** выберите тип документа, по которому необходимо выполнить фильтрацию.

    ![Выбор типа документа для просмотра журналов отправки](media/e-invoicing-services-get-started-select-document-type-for-viewing-submission-log.png)

    > [!IMPORTANT]
    > Значение, отображаемое в столбце **Состояние отправки** , представляет статус, который связан с завершением самого процесса отправки. Он указывает, был ли поток действий, настроенный в RCS, выполнен до конца, независимо от того, был ли электронный документ утвержден или отклонен. Значение в столбце **Статус отправки** не представляет статус отправленного документа. Можно просмотреть статус отправленного документа (то есть, был ли документ утвержден или отклонен) на экспресс-вкладке **Журнал действий обработки** в данных журнала отправки, как описано далее.

3. В области действий выберите **Запросы \> Сведения об отправке**.
4. Просмотрите данные журнала отправки.

    ![Данные журнала отправки](media/e-invoicing-services-get-started-view-submission-log-form.png)

Результаты, которые отображаются в журнале отправки, зависят от того, как функция электронных накладных была настроена в RCS. Однако вне зависимости от настройки журнал отправки всегда имеет три экспресс-вкладки:

- **Действия по обработке** — на этой экспресс-вкладке отображается журнал выполнения для действий, настроенных в версии функции, которая была настроена в RCS. Столбец **Состояние** показывает, было ли действие успешно выполнено.
- **Файлы действий** — на этой экспресс-вкладке отображаются промежуточные файлы, созданные во время выполнения действий. Можно выбрать **Просмотр** , чтобы загрузить файл и просмотреть его содержимое.
- **Журнал действий обработки** — на этой экспресс-вкладке отображаются результаты взаимодействия между дополнением электронных накладных и целевой веб-службой. Он также показывает, что было возвращено веб-службой обработки.
