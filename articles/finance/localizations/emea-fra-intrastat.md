# <a name="french-intrastat"></a>Интрастат для Франции

[!include[banner](../includes/banner.md)]

Компании во Франции, которые зарегистрированы для налога на добавленную стоимость (НДС) и которые работают с другими странами Европейского Союза (ЕС), должны декларировать обмен товарами и услугами во Францию и из Франции. Declaration d'exchanges de Biens (Декларация о торговле товарами или DEB) является комбинацией отчета ЕС по НДС и отчета Интрастат. Необходимо ежемесячно представлять этот отчет для всех продаж товаров внутри сообщества.

Можно создать отчет DEB в одном из двух электронных форматов: SAISUNIC 330 или INTRACOM.

### <a name="set-up-intrastat"></a>Настройка отчета Интрастат

1.  В [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/Logon/Index) в общей библиотеке активов загрузите последние версии следующих конфигураций электронной отчетности (ER) для декларации Интрастат:

-   Модель Интрастат

-   Отчет Интрастат

-   INTRACOM Интрастат (FR)

-   SAISUNIC Интрастат (FR)

Дополнительные сведения см. в разделе [Загрузка конфигураций электронной отчетности из Lifecycle Services](../../fin-ops-core/dev-itpro/analytics/download-electronic-reporting-configuration-lcs.md).

2.  В Dynamics 365 Finance выберите **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Параметры внешней торговли** и выполните следующие действия:

    1.  На вкладке **Интрастат** на экспресс-вкладке **Электронная отчетность** в поле **Сопоставление форматов файлов** выберите **Интрастат INTRACOM (FR)** или **Интрастат SAISUNIC (FR)**.

    2.  В поле **Сопоставление форматов отчетов** выберите **Отчет Интрастат**.

    3.  На экспресс-вкладке **Иерархия кодов товаров** в поле **Иерархия категорий** выберите **Интрастат**.

    4.  На экспресс-вкладке **Общие** в поле **Код проводки** выберите код, используемый для переносов товаров.

    5.  В поле **Кредит-нота** выберите код, используемый для возврата товаров.

    6.  В поле **Уровень обязательств для экспорта** введите уровень детализации для отчета по экспорту. Выбранный уровень влияет на строки, отображаемые в отчете. Для получения дополнительных сведений см. таблицы в начале данной темы.

3.  Перейдите к пункту **Администрирование организации** &gt; **Организации** &gt; **Юридические лица**, выберите компанию и выполните следующие действия:

    1.  На экспресс-вкладке **Номера регистраций** в поле **Код NAF** введите свой код NAF. Дополнительные сведения см. в разделе [FR-00003 Коды NAF и номера Siret](tasks/fr-00003-naf-codes-siret-numbers.md).

    2.  На экспресс-вкладке **Внешняя торговля и логистика** в разделе **Интрастат** установите поля **Экспорт кода налогового освобождения от НДС** и **Импорт кода налогового освобождения от НДС**.

    3.  На экспресс-вкладке **Регистрация налога** в поле **Налоговый регистрационный номер** введите налоговый регистрационный номер вашей компании.

4.  Чтобы указать коды NAF и номера налогового освобождения для клиентов, перейдите в раздел **Расчеты с клиентами** &gt; **Клиенты** &gt; **Все клиенты**, затем выполните следующие действия:

    1.  Выберите клиента.

    2.  На экспресс-вкладке **Накладная и поставка** в разделе **Налог** в поле **Код налогового освобождения** введите код налогового освобождения клиента.

    3.  На экспресс-вкладке **Демографические данные продаж** в поле **Siret (Франция)** введите номер Siret компании.

    4.  В поле **Код NAF** введите код NAF компании.

    5.  Повторите эти шаги для других клиентов.

5.  Чтобы указать коды NAF и номера налогового освобождения для поставщиков, перейдите в раздел **Расчеты с поставщиками** &gt; **Поставщики** &gt; **Все поставщики**, затем выполните следующие действия:

    1.  Выберите поставщика.

    2.  На экспресс-вкладке **Накладная и поставка** в разделе **Налог** в поле **Код налогового освобождения** введите код налогового освобождения поставщика.

    3.  На экспресс-вкладке **Демографические данные покупок** в поле **Siret (Франция)** введите номер Siret компании.

    4.  В поле **Код NAF** введите код NAF компании.

    5.  Повторите эти шаги для других поставщиков.

6.  Перейдите в раздел **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Сжатие Интрастат** и выберите поля, которые должны сравниваться, когда суммируются сведения Интрастат. Для французского Интрастат выберите **Статистическая процедура**, **Регион происхождения**, **Страна/регион происхождения**, **Условия поставки**, **Транспортировка**, **Корректировка**, **Страна/регион**, **Район происхождения/назначения**, **Направление**, **страна/регион отправителя**, **Район отправителя**, **Район**, **Код проводки** и **Товар**.

7.  Перейдите в раздел **Управление складом** &gt; **Настройка** &gt; **Склад** &gt; **Склады**, выберите склад, затем выполните следующие действия:

    1.  На экспресс-вкладке **Адреса** выберите **Добавить** или **Изменить**.

    2.  В диалоговом окне в поле **Город** выберите город склада. Регион города будет использоваться в качестве района для отчета DEB.

### <a name="ngp-codes"></a>Коды NGP

В отчете DEB кодификация продуктов состоит из следующих элементов:

1.  Код CN8 с восемью цифрами, который представляет собой тариф и статистическую номенклатуру ЕС.

2.  Если применимо, национальный код номенклатуры Nomenclature Générale des Produits (NGP) из одной цифры.

В 2021 году NGP применяется только к трем типам продуктов:

-   Некоторые продукты из коров, овец и коз

-   Военное оборудование

-   Французские вина

Необходимо настроить коды NGP и назначить их для соответствующих продуктов. Поле **NGP** затем автоматически настраивается на странице **Журнал Интрастат**.

#### <a name="set-up-an-ngp-code"></a>Настройка кода NGP

1.  Перейдите в раздел **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Коды NGP**.

2.  На панели операций выберите **Создать** для создания кода NGP.

3.  В поле **NGP** введите код из 1 цифры.

4.  В поле **Описание** введите описание для кода.

#### <a name="assign-an-ngp-code-to-a-product"></a>Назначение кода NGP продукту

1.  Щелкните **Управление сведениями о продукте** &gt; **Продукты** &gt; **Выпущенные продукты**.

2.  В сетке выберите продукт.

3.  На экспресс-вкладке **Внешняя торговля** в разделе **Интрастат** в поле **NGP** выберите соответствующий код NGP.

## <a name="intrastat-journal"></a>Журнал Интрастат

Перейдите в раздел **Налог** &gt; **Декларации** &gt; **Внешняя** **торговля** &gt; **Интрастат** для управления проводками, применимыми к внешней торговле со странами ЕС. Дополнительные сведения см. в разделе [Обзор Интрастат](emea-intrastat.md).

Столбец **NGP** относится только к Франции. Он показывает код NGP для продукта. Если NGP не применяется к продукту, отображается значение **0** (ноль). Код NGP можно откорректировать. Выберите проводку, затем на вкладке **Общие** в разделе **Коды** в поле **NGP** выберите необходимый код NGP.

### <a name="intrastat-transfer"></a>Перенос Интрастат

На странице **Интрастат** в области действий можно выбрать **Перенос** для автоматического переноса информации о торговле внутри сообщества из заказов на продажу, накладных с произвольным текстом, заказов на покупку, накладных поставщиков, поступлений продуктов поставщиков, накладных по проектам и заказов на перемещение. Будут перенесены только те документы, для которых в качестве страны или региона назначения (для отправок) или консигнации (для поступлений) указана страна ЕС.

Поскольку отчет DEB является комбинацией отчета ЕС по НДС и отчета Интрастат, он также включает в себя *трехсторонние* проводки, где прямая поставка выполняется из одной страны ЕС (субъект A) в другую страну ЕС (субъект C), а французское юридическое лицо (сторона B) находится с середине трехсторонней сделки.

### <a name="generate-a-deb-intrastat-report"></a>Создание отчета DEB (Интрастат)

1.  Перейдите в раздел **Налог** &gt; **Декларации** &gt; **Внешняя торговля** &gt; **Интрастат**.

2.  В области действий выберите **Вывод** &gt; **Отчет**.

3.  В диалоговом окне **Отчет Интрастат** задайте следующие поля.

| Поле            | описание                                                                                                                         |
|------------------|-------------------------------------------------------------------------------------------------------------------------------------|
| С даты        | Выберите начальную дату для отчета.                                                                                               |
| По дату          | Выберите конечную дату для отчета.                                                                                                 |
| Создать файл    | Установите для этого параметра значение **Да**, чтобы создать файл TXT.                                                                                 |
| Имя файла        | Введите имя файла TXT для отчета Интрастат.                                                                          |
| Создать отчет  | Установите для этого параметра значение **Да**, чтобы создать файл XML.                                                                                |
| Имя файла отчета | Введите требуемое имя.                                                                                                            |
| Только корректировки | Установите для этого параметра значение **Да**, чтобы сообщать только о корректировках. Установите значение **Нет** для включения в отчет как обычных проводок, так и исправлений.         |
| Направление        | Выберите **Поступления** для отчета о поступлениях внутри сообщества. Выберите **Отправки** для отчета об отправках внутри сообщества. |

## <a name="example"></a>Пример

В следующем примере показано, как настроить коды NGP и работать с ними. В нем используется юридическое лицо **DEMF**.

1.  В [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/Logon/Index) в общей библиотеке активов загрузите последние версии следующих конфигураций электронной отчетности для формата декларации Интрастат:

-   Модель Интрастат

-   Отчет Интрастат

-   INTRACOM Интрастат (FR)

2.  В модуле Finance настройте код проводки:

    1.  Перейдите в раздел **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Коды проводки**.

    2.  В области действий выберите **Создать**.

    3.  В поле **Код проводки** введите **11**.

    4.  В поле **Имя** введите **Покупки или Продажи**.

    5.  На панели операций выберите **Сохранить**.

3.  Настройте иерархию продуктов:

    1.  Перейдите в раздел **Управление сведениями о продукте** &gt; **Настройка** &gt; **Категории и атрибуты** &gt; **Иерархии категорий**.

    2.  В сетке выберите **Интрастат**. Затем в области действий на вкладке **Иерархия категорий** в группе **Ведение** выберите **Изменить**.

    3.  На панели операций выберите **Новый узел категории**.

    4.  В поле **Имя** введите **Autres Libournais**.

    5.  В поле **Код** введите **2204 21 42**

    6.  На панели операций выберите **Сохранить**.

4.  Перейдите в раздел **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Параметры внешней торговли** и выполните следующие действия:

    1.  На вкладке **Интрастат** на экспресс-вкладке **Общие** в поле **Код проводки** выберите **11**.

    2.  На экспресс-вкладке **Электронная отчетность** в поле **Сопоставление форматов файлов** выберите **Интрастат INTRACOM (FR)**.

    3.  В поле **Сопоставление форматов отчетов** выберите **Отчет Интрастат**.

    4.  На экспресс-вкладке **Иерархия кодов товаров** в поле **Иерархия категорий** выберите **Интрастат**.

5.  Настройка кода NGP:

    1.  Перейдите в раздел **Налог** &gt; **Настройка** &gt; **Внешняя торговля** &gt; **Коды NGP**.

    2.  В области действий выберите **Создать**.

    3.  В поле **NGP** введите **8**.

    4.  В поле **Имя описания** введите **NGP 8**.

    5.  На панели операций выберите **Сохранить**.

6.  Назначьте код NGP продукту:

    1.  Щелкните **Управление сведениями о продукте** &gt; **Продукты** &gt; **Выпущенные продукты**.

    2.  В сетке выберите **0001**.

    3.  На экспресс-вкладке **Внешняя торговля** в поле **NGP** выберите **8**.

    4.  В поле **Товар** выберите **2204 21 42**.

    5.  На панели операций выберите **Сохранить**.

7.  Создайте заказ на продажу, включающий новый продукт:

    1.  Перейдите в раздел **Расчеты с клиентами** &gt; **Заказы** &gt; **Все заказы на продажу**.

    2.  В области действий выберите **Создать**.

    3.  В диалоговом окне **Создать** **заказ** **на продажу** в разделе **Клиент** в поле **Счет** **клиента** выберите **100001**.

    4.  В разделе **Адрес** в поле **Адрес поставки** выберите знак "плюс" (**+**), чтобы создать адрес.

    5.  В диалоговом окне **Создать адрес** в поле **Имя описания** введите **Германия**.

    6.  В поле **Страна/регион** выберите **DEU**.

    7.  Нажмите **ОК**.

    8.  В диалоговом окне **Создание заказа на продажу** выберите **ОК**.

    9.  На экспресс-вкладке **Строки заказа** **на продажу** в поле **Код номенклатуры** выберите **0001**.

    10. На панели операций выберите **Сохранить**.

    11. На панели операций на вкладке **Накладная** в группе **Создать** выберите **Накладная**.

    12. В диалоговом окне **Разноска накладных** на экспресс-вкладке **Параметры** в разделе **Параметр** в поле **Количество** выберите **Все**. Затем выберите **ОК**, чтобы разнести накладную.

8.  Создайте отчет DEB:

    1.  Перейдите в раздел **Налог** &gt; **Декларации** &gt; **Внешняя торговля** &gt; **Интрастат**:

    2.  В области действий. выберите **Перевести**.

    3.  В диалоговом окне **Интрастат (перемещение)** в разделе **Параметры** установите для параметра **Накладная клиента** значение **Да**. Затем выберите **OK**.

    4.  Отсортируйте проводки по полю **Дата**. Верхняя проводка представляет собой результирующую проводку. Поле **NGP** устанавливается автоматически.

    5.  В области действий выберите **Вывод** &gt; **Отчет**.

    6.  В диалоговом окне **Отчет Интрастат** на экспресс-вкладке **Параметры** в разделе **Дата** выберите месяц созданного заказа на продажу.

    7.  В разделе **Параметры экспорта** установите для параметра **Создать файл** значение **Да**.

    8.  В поле **Имя файла** введите требуемое имя.

    9.  Выберите **ОК** и проверьте отчет.
