# <a name="polish-intrastat"></a>Польский Интрастат

[!include[banner](../includes/banner.md)]

Страница **Интрастат** используется для создания и подготовки отчетов о торговле между странами ЕС. Польская декларация Интрастат содержит информацию о торговле товарами для отчетности.

Следующие поля включены в польскую декларацию Интрастат. Все поля включаются в приходы и отправки, за исключением **RodzajTransportu** (режим перевозки) и **KrajPochodzenia** (страна или регион происхождения), которые не включаются в отправки, и **IdKontrahenta** (номер иностранного плательщика НДС клиента), который не включается в приемки.

| Имя поля | Описание |
|-------------------------|-------------------------|
| Deklaracja Data | Дата создания документа. |
| Miesiac | Базисный месяц в декларации. |
| Rok | Базисный год в декларации. |
| Numer | Номер декларации в отчетном периоде. |
| Wersja | Номер версии декларации. |
| NrWlasny | Идентификатор декларации. Значение генерируется автоматически. |
| Typ | Направление отчета.</br><li>Для поступлений печатается "P".</li><li>Для отправок печатается "W".</li> |
| Rodzaj | Тип декларации. Значение указывает, является ли отчет исходной декларацией или корректирующей декларацией. |
| UC | Код подразделения, которому адресована декларация Интрастат. Значение указывается в поле **Код налогового освобождения** в разделе **Налог** на вкладке **Агент** на странице **Параметры внешней торговли**. |
| Nazwa | Наименование компании. |
| Miejscowosc, UlicaNumer, KodPocztowy | Полный адрес юридического лица. |
| Nip | Польский идентификационный код налогоплательщика (код налога на добавленную стоимость [НДС]). |
| Regon | Польский статистический идентификационный номер (номер предприятия). |
| LacznaWartoscFaktur | Сумма значений накладной. |
| LacznaWartoscStatystyczna | Сумма статистических значений. |
| LacznaLiczbaPozycji | Общее число номенклатур товаров. |
| PozId | Порядковый номер заданной номенклатуры товаров. |
| OpisTowaru | Товарное имя товара. |
| KrajPochodzeniaWysylki | Код международной организацией по стандартизации (International Organization for Standardization, ISO) для страны или региона контрагента. |
| WarunkiDostawy | Код Интрастат для условий поставки. |
| RodzajTransakcji | Код, указывающий природу проводки. Компании в Польше используют коды проводок с двумя цифрами. |
| KodTowarowy | Код товара из восьми цифр в соответствии с объединенной номенклатурой. |
| RodzajTransportu | Код Интрастат для условий транспортировки. |
| KrajPochodzenia | Код ISO для страны или региона, где были произведены или изготовлены товары. |
| MasaNetto | Чистая масса в полных килограммах. |
| IloscUzupelniajacaJm | Количество в дополнительных единицах измерения, в целых числах. |
| WartoscFaktury | Сумма накладной по всем проводкам, охватываемым одной номенклатурой. |
| WartoscStatystyczna | Статистическая стоимость. |
| Wypelniajacy: NazwiskoImie, Telefon, Faks, Email | Имя и фамилия, номер телефона, номер факса и адрес электронной почты лица, который отправляет декларацию. |
| IdKontrahenta | Номер иностранного плательщика НДС клиента в стране-члене ЕС. |


## <a name="set-up-intrastat"></a>Настройка отчета Интрастат

Из глобального репозитория импортируйте последнюю версию следующих конфигураций электронной отчетности:

   - Модель Интрастат
   - Отчет Интрастат
   - Интрастат (PL)

Подробнее см. в [Загрузка конфигураций электронной отчетности (ER) из глобального репозитория Configuration Service](../../fin-ops-core/dev-itpro/analytics/er-download-configurations-global-repo.md).

## <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>Настройка идентификатора НДС и номера предприятия для компании

### <a name="create-registration-types-for-company-codes"></a>Создание типов регистраций для кодов компаний

Необходимо создать два типа регистраций для кодов компании: один для кода НДС (код NIP), а другой — для номера предприятия (код Regon).

1. Откройте **Управление организацией** > **Глобальная адресная книга** > **Типы регистрации** > **Типы регистрации**.
2. На панели операций выберите **Создать** для создания типа регистрации для идентификатора НДС.
3. В диалоговом окне **Введите сведения о типе регистрации** в поле **Имя** введите имя для нового типа регистрации. Например, введите **NIP**.
4. В поле **Страна/регион** выберите **POL**.
5. Выберите **Создать**.
6. На панели операций выберите **Создать** для создания типа регистрации для номера предприятия.
7. В диалоговом окне **Введите сведения о типе регистрации** в поле **Имя** введите имя для нового типа регистрации. Например, введите **Regon**.
8. В поле **Страна/регион** выберите **POL**.
9. Выберите **Создать**.

### <a name="match-the-registration-types-with-registration-categories"></a>Сопоставление типов регистрации с категориями регистрации

1. Откройте **Управление организацией** > **Глобальная адресная книга** > **Типы регистрации** > **Категории регистрации**.
2. В области действий выберите **Создать**, чтобы создать связь между всеми созданными типами регистрации и категорией регистрации.

    - Для типа регистрации для кода плательщика НДС (код NIP) выберите категорию регистрации **Код НДС**.
    - Для типа регистрации для номера предприятия (код Regon) выберите категорию регистрации **Код предприятия (COID)**.

### <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>Настройка идентификатора НДС и номера предприятия для компании

1. Перейдите в раздел **Управление организацией** > **Организации** > **Юридические лица**.
2. В сетке выберите вашу компанию.
3. В области операций выберите **Коды регистрации**.
4. На экспресс-вкладке **Код регистрации** выберите **Добавить**.
5. В поле **Тип регистрации** выберите один из ранее созданных типов регистрации.
6. Введите код плательщика НДС вашей компании (код NIP) или номер предприятия (код Regon), в зависимости от типа регистрации, выбранного на предыдущем шаге.
7. Повторите шаги с 4 по 6 для других типов регистрации, созданных ранее.

## <a name="set-up-a-company-address"></a>Настройка адреса компании

1. Перейдите в раздел **Управление организацией** > **Организации** > **Юридические лица**.
2. В сетке выберите вашу компанию.
3. На вкладке **Адреса** выберите **Правка**.
4. В диалоговом окне **Изменить адрес** в поле **Почтовый индекс** выберите почтовый индекс компании.
5. В поле **Улица** введите свой адрес.
6. В поле **Город** выберите свой город.

## <a name="set-up-foreign-trade-parameters"></a>Настройка параметров внешней торговли

1. Перейдите в раздел **Налог** > **Настройка** > **Параметры внешней торговли**.
2. На вкладке **Интрастат** на экспресс-вкладке **Электронная отчетность** в поле **Сопоставление форматов файлов** выберите **Интрастат (PL)**.
3. В поле **Сопоставление форматов отчетов** выберите **Отчет Интрастат**.
4. На экспресс-вкладке **Иерархия кодов товаров** в поле **Иерархия категорий** выберите **Интрастат**.
5. В поле **Код проводки** выберите код проводки для передачи собственности. Используйте этот код будет для проводок, которые приводят к фактическому или запланированному изменению собственности за компенсацию (финансовому или иному). Он также используется для корректировок. Компании в Польше используют коды проводок с двумя цифрами.
6. В поле **Кредит-нота** выберите код проводки для возврата товаров.
7. На вкладке **Свойства страны/региона** в поле **Страна/регион** перечислите все страны или регионы, с которыми ваша компания имеет деловые отношения. Для каждой страны или региона в составе ЕС выберите **ЕС** в поле **Тип страны/региона**, чтобы страна или регион отображались в отчете Интрастат. Для Польши выберите **Внутренний** в поле **Тип страны/региона**.
8. На вкладке **Агент** на экспресс-вкладке **Агент** в разделе **Налог** в поле **Код налогового освобождения** введите **420000**, чтобы указать код подразделения, которому адресована декларация Интрастат.
9. На вкладке **Контакт** введите имя, номер телефона, номер факса и адрес электронной почты лица, который отправляет декларацию.
10. На вкладке **Номерные серии** в поле **Код номерной серии** для ссылки **Номер файла XML** укажите прерывистую номерную серию, которая содержит не более девяти символов. Это поле используется для автоматического создания значения поля **Код декларации** в отчете Интрастат.

## <a name="set-up-product-parameters-for-the-intrastat-declaration"></a>Настройка параметров продукта для декларации Интрастат

1. Щелкните **Управление сведениями о продукте** > **Продукты** > **Выпущенные продукты**.
2. В сетке выберите продукт.
3. На экспресс-вкладке **Внешняя торговля** в разделе **Интрастат** в поле **Товар** выберите соответствующий товар. Название товара будет напечатано в поле **Описание товаров** в отчете Интрастат.
4. В разделе **Происхождение** в поле **Страна/регион** выберите страну или регион происхождения продукта.
5. На экспресс-вкладке **Управление запасами** в поле **Вес нетто** введите вес продукта в килограммах.

## <a name="set-up-compression-of-intrastat"></a>Настроить свертку Интрастат

-   Перейдите в раздел **Налог** > **Настройка** > **Внешняя торговля** > **Сжатие Интрастат** и выберите поля, которые должны сравниваться, когда суммируются сведения Интрастат. Для Интрастат для Польши выберите следующие поля:

    - Товар
    - Код проводки
    - Страна или регион происхождения
    - Транспорт
    - Условия поставки
    - Страна/регион отправителя
    - Страна/регион
    - Корректировка
    - Номер налогового освобождения
    - Направление
    - Накладная

## <a name="set-up-the-transport-method-and-delivery-terms"></a>Настройка метода транспортировки и условий поставки

1.  Настройка кодов транспортировки.

    1. Перейдите в раздел **Налог** > **Настройка** > **Внешняя торговля** > **Метод транспортировки**.
    2. В области действий выберите **Создать**.
    3. В поле **Транспорт** введите уникальный код. Компании в Польше используют коды транспортировки с одной цифрой.

2.  Настройте коды Интрастат для режима поставки.

    1. Перейдите в раздел **Закупки и источники** > **Настройка** > **Распределение** > **Условия поставки**.
    2. В сетке выберите набор условий поставки.
    3. На экспресс-вкладке **Общие** в поле **Код Интрастат** введите уникальный код.

## <a name="intrastat-transfer"></a>Перенос Интрастат

На странице **Интрастат** в области действий можно выбрать **Перенос** для автоматического переноса информации о торговле внутри сообщества из заказов на продажу, накладных с произвольным текстом, заказов на покупку, накладных поставщиков, поступлений продуктов поставщиков, накладных по проектам и заказов на перемещение. Будут перенесены только те документы, для которых в качестве страны или региона назначения или консигнации указана страна ЕС.

Можно также ввести проводки вручную, выбрав **Создать** в области действий.

### <a name="generate-an-intrastat-report"></a>Создание отчета Интрастат

1. Перейдите в раздел **Налог** > **Декларации** > **Внешняя торговля** > **Интрастат**.
2. В области действий выберите **Вывод** &gt; **Отчет**.
3. В диалоговом окне **Отчет Интрастат** задайте следующие поля.

    | Поле | Описание |
    |-------------------------|-------------------------|
    | С даты | Выберите начальную дату для отчета. |
    | Создать файл | Установите для этого параметра значение **Да**, чтобы создать XML-файл для отчета Интрастат. |
    | Имя файла | Введите имя файла XML. |
    | Создать отчет | Установите для этого параметра значение **Да**, чтобы создать XLSX-файл для отчета Интрастат. |
    | Имя файла отчета | Введите имя файла XLSX. |
    | Направление | Выберите **Поступления** для отчета о поступлениях внутри сообщества.</br>Выберите **Отправки** для отчета об отправках внутри сообщества. |
    | Код декларации | Код документа создается автоматически и может быть обновлен. |
    | Тип декларации | Выберите **Декларация** для исходной декларации.</br>Выберите **Коррекция декларации - замена** для корректирующей декларации, которая предназначена для полной замены существующего, ранее отправленного оригинала или корректирующей декларации. |
    | Город создания документа | Введите значение, которое должно печататься в поле **Miejscowosc** в декларации Интрастат. |
    | Дата создания документа | Введите значение, которое должно печататься в поле **Deklaracja Data** в декларации Интрастат. |
    | № документа | Введите значение, которое должно печататься в поле **Numer** в декларации Интрастат. |
    | Версия документа | Введите значение, которое должно печататься в поле **Wersja** в декларации Интрастат. |

4. Нажмите **ОК** и просмотрите созданные отчеты.

## <a name="example"></a>Пример

В этом примере показано, как разносить прибытия и отправки для Интрастат с использованием юридического лица **DEMF**.

### <a name="preliminary-setup"></a>Предварительная настройка

Импортируйте последнюю версию из перечисленных ниже конфигураций электронной отчетности:

   - Модель Интрастат
   - Отчет Интрастат
   - Интрастат (PL)

### <a name="set-up-a-company-address"></a>Настройка адреса компании

1. Перейдите в раздел **Управление организацией** > **Глобальная адресная книга** > **Адреса** > **Настройка адреса**.
2. На вкладке **Город** выберите **Создать**.
3. В поле **Страна/регион** выберите **POL**.
4. В поле **Город** введите **Варшава**.
5. На вкладке **Почтовый индекс** выберите **Создать**.
6. В поле **Страна/регион** выберите **POL**.
7. В поле **Город** выберите **Варшава**.
8. В поле **Почтовый индекс** введите **00-844**.
9. Перейдите в **Администрирование организации** > **Организация** > **Юридические лица** и выберите юридическое лицо **DEMF**.
10. На экспресс- вкладке **Адреса** выберите **Правка**.
11. В поле **Страна/регион** выберите **POL**.
12. В поле **Почтовый индекс** выберите **31-111**.
13. В поле **Улица** введите **Статистичная 22/1**.
14. В поле **Город** выберите **Варшава**.
15. Нажмите **ОК**.

## <a name="set-up-a-vat-id-and-an-enterprise-number-code-for-your-company"></a>Настройка идентификатора НДС и кода номера предприятия для компании

### <a name="create-registration-types-for-company-codes"></a>Создание типов регистраций для кодов компаний

1. Откройте **Управление организацией** > **Глобальная адресная книга** > **Типы регистрации** > **Типы регистрации**.
2. На панели операций выберите **Создать** для создания типа регистрации для идентификатора НДС (код NIP).
3. В диалоговом окне **Введите сведения о типе регистрации** в поле **Имя** введите **NIP**.
4. В поле **Страна/регион** выберите **POL**.
5. Выберите **Создать**.
6. На панели операций выберите **Создать** для создания типа регистрации для номера предприятия (код Regon).
7. В диалоговом окне **Введите сведения о типе регистрации** в поле **Имя** введите **Regon**.
8. В поле **Страна/регион** выберите **POL**.
9. Выберите **Создать**.

### <a name="match-the-registration-types-with-registration-categories"></a>Сопоставление типов регистрации с категориями регистрации

1. Откройте **Управление организацией** > **Глобальная адресная книга** > **Типы регистрации** > **Категории регистрации**.
2. В области действий выберите **Создать**, чтобы создать связь между всеми созданными типами регистрации и категорией регистрации.

    - Для типа регистрации **NIP** выберите категорию регистрации **Код НДС**.
    - Для типа регистрации **Regon** выберите категорию регистрации **Код предприятия (COID)**.

### <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>Настройка идентификатора НДС и номера предприятия для компании

1. Перейдите в раздел **Управление организацией** > **Организации** > **Юридические лица**.

### <a name="set-up-a-transport-method"></a>Настройка метода транспортировки

1. Создайте способ транспортировки.

    1. Перейдите в раздел **Налог** > **Настройка** > **Внешняя торговля** > **Метод транспортировки**.
    2. В области действий выберите **Создать**.
    3. В поле **Транспорт** введите **3**.
    4. В поле **Описание** введите **Автомобильный транспорт**.

2. Назначьте новый способ транспортировки в режим поставки. Таким образом настраиваются значения по умолчанию, используемые для метода перевозки, когда выбран соответствующий режим поставки.

    1. Перейдите к **Закупки и источники** > **Настройка** > **Распределение** > **Способы доставки**.
    2. В сетке выберите **10**.
    3. На экспресс-вкладке **Внешняя торговля** в поле **Транспортировка** выберите **3**.

3. Выберите режим поставки по умолчанию для клиента.

    1. Перейдите в раздел **Расчеты с клиентами** > **Клиенты** > **Все клиенты**.
    2. В сетке выберите **DE-016**.
    3. На экспресс-вкладке **Накладные и поставка** в поле **Режим поставки** выберите **10**.

4. Выберите режим поставки по умолчанию для поставщика.

    1. Перейдите в раздел **Расчеты с поставщиками** > **Поставщики** > **Все поставщики**.
    2. В сетке выберите **DE-001**.
    3. На экспресс-вкладке **Накладные и поставка** в поле **Режим поставки** выберите **10**.

### <a name="create-a-sales-order-with-an-eu-customer"></a>Создайте заказ на продажу с клиентом в ЕС

1. Перейдите в раздел **Расчеты с клиентами** > **Заказы** > **Все заказы на продажу**.
2. В области действий выберите **Создать**.
3. В диалоговом окне **Создать заказ на продажу** на экспресс-вкладке **Клиент** в разделе **Клиент** в поле **Счет клиента** выберите **DE-016**.
4. На экспресс-вкладке **Общие** в разделе **Аналитики хранения** в поле **Сайт** выберите **1**.
5. В поле **Склад** выберите **11**.
6. На вкладке **Адрес** проверьте, что поле **Адрес** имеет значение **Teichgasse 12, Kiel, 24103, DEU**, так как клиент находится в Германии.
7. Нажмите **ОК**.
8. На вкладке **Верхний колонтитул** на экспресс-вкладке **Поставка** убедитесь, что поле **Условия поставки** имеет значение **CIF**, а в поле **Режим поставки** установлено значение **10**.
9. На вкладке **Строки** на экспресс-вкладке **Строки заказа на продажу** в поле **Код номенклатуры** выберите **D0001**. Затем в поле **Количество** введите **8**.
10. На экспресс-вкладке **Сведения по строке** на вкладке **Внешняя торговля** убедитесь, что в поле **Код проводки** указано значение **11**, поле **Товар** имеет значение **100 200 30**, а в поле **Страна/регион происхождения** установлено значение **POL**.
11. На панели операций выберите **Сохранить**.
12. На панели операций на вкладке **Накладная** в группе **Создать** выберите **Накладная**.
13. В диалоговом окне **Разноска накладных** на экспресс-вкладке **Параметры** в разделе **Параметр** в поле **Количество** выберите **Все**.
14. На экспресс-вкладке **Настройка** в поле **Дата продажи** выберите **10/18/2021** (18 октября 2021 г.).
15. Выберите **ОК**, чтобы разнести накладную.

### <a name="transfer-the-transaction-to-the-intrastat-journal-and-review-the-result"></a>Переместите проводку в журнал Интрастат и проверьте результат

1. Перейдите в раздел **Налог** > **Декларации** > **Внешняя торговля** > **Интрастат**.
2. В области действий выберите **Перемещение**.
3. В диалоговом окне **Интрастат (перемещение)** в разделе **Параметры** установите для параметра **Накладная клиента** значение **Да**.
4. Выберите **Фильтр**.
5. В диалоговом окне **Фильтр Интрастат** на вкладке **Диапазон** выберите первую строку и убедитесь, что в поле **Поле** установлено значение **Дата**.
6. В поле **Критерий** выберите текущую дату.
7. Выберите **ОК**, чтобы закрыть диалоговое окно **Фильтр Интрастат**.
8. Выберите **ОК**, чтобы закрыть диалоговое окно **Интрастат (перемещение)** и просмотреть результат. Строка представляет заказ на продажу, созданный вами ранее.

    ![Строка, которая представляет заказ на продажу на странице Интрастат](media/intrastat_pl_1.png)

9. Выберите строку проводки, а затем перейдите на вкладку **Общие**, чтобы просмотреть дополнительные сведения.

    ![Сведения о заказе на продажу на вкладке "Общие" на странице "Интрастат"](media/intrastat_pl_2.png)

10. В области действий выберите **Вывод** > **Отчет**.
11. В диалоговом окне **Отчет Интрастат** на экспресс-вкладке **Параметры** в разделе **Дата** в поле **Начальная дата** выберите первый день текущего месяца.
12. В разделе **Параметры** **экспорта** установите для параметра **Создать файл** значение **Да**. Затем в поле **Имя файла** введите требуемое имя.
13. Для параметра **Создать отчет** выберите значение **Да**. Затем в поле **Имя файла отчета** введите требуемое имя.
14. В поле **Направление** выберите **Отправки**.
15. В разделе **Сопоставление формата файла** убедитесь, что в поле **Тип декларации** задано значение **Декларация**.
16. В поле **Город создания документа** введите **Краков**.
17. В поле **Дата создания документа** выберите **10/19/2021** (19 октября 2021 г.).
18. В поле **№ документа** введите **11**.
19. В поле **Версия документа** введите **22**.

### <a name="create-a-purchase-order"></a>Создание заказа на покупку

1. Перейдите в раздел **Расчеты с поставщиками** > **Заказы на покупку** > **Все заказы на покупку**.
2. В области действий выберите **Создать**.
3. В диалоговом окне **Создание заказа на покупку** в поле **Счет поставщика** выберите **DE-001**.
4. В поле **Сайт** выберите **1**.
5. В поле **Склад** выберите **11**.
6. Нажмите **ОК**.
7. На вкладке **Верхний колонтитул** на экспресс-вкладке **Поставка** убедитесь, что поле **Режим поставки** имеет значение **10**, а в поле **Условия поставки** установлено значение **CIF**.
8. На вкладке **Строки** на экспресс-вкладке **Строки заказа на покупку** в поле **Код номенклатуры** выберите **D0003**. Затем в поле **Количество** введите **6**.
9. На экспресс-вкладке **Сведения по строке** на вкладке **Внешняя торговля** убедитесь, что в поле **Код проводки** установлено значение **11**, поле **Транспорт** имеет значение **3**, в поле **Товар** задано значение **100 200 30**, а в поле **Страна/регион происхождения** установлено значение **DEU**.
10. В области действий на вкладке **Покупка** в группе **Действия** выберите **Подтвердить**.
11. На панели операций на вкладке **Накладная** в группе **Создать** выберите **Накладная**.
12. На панели действий выберите **По умолчанию из**, затем в поле **Количество по умолчанию для строк** выберите **Заказанное количество**. Затем выберите **OK**.
13. На экспресс-вкладке **Заголовок накладной поставщика** в разделе **Код накладной** в поле **Номер** введите **00010**.
14. В разделе **Даты накладной** в поле **Дата накладной** выберите текущую дату. Эта дата будет использоваться для переноса в Интрастат.
15. В поле **Дата получения документа** выберите **10/18/2021** (18 октября 2021 г.).
16. В области действий выберите **Разнести**, чтобы разнести накладную.

### <a name="create-an-intrastat-declaration-for-arrivals"></a>Создание декларации Интрастат для прибытий

1. Перейдите в раздел **Налог** > **Декларации** > **Внешняя торговля** > **Интрастат**.
2. В области действий выберите **Перемещение**.
3. В диалоговом окне **Интрастат (перемещение)** задайте для параметр **Накладная поставщика** значение **Да**.
4. Выберите **ОК**, чтобы переместить проводки и проверьте журнал Интрастат.

    ![Строка, которая представляет заказ на покупку на странице Интрастат](media/intrastat_pl_4.png)

5. Просмотрите сведения на вкладке **Общие** для заказа на покупку.

    ![Сведения о заказе на покупку на вкладке "Общие" на странице "Интрастат"](media/intrastat_pl_5.png)

8. В разделе **Параметры** **экспорта** установите для параметра **Создать файл** значение **Да**. Затем в поле **Имя файла** введите требуемое имя.
9. Для параметра **Создать отчет** выберите значение **Да**. Затем в поле **Имя файла отчета** введите требуемое имя.
10. В поле **Направление** выберите **Прибытия**.
11. В разделе **Сопоставление формата файла** убедитесь, что в поле **Тип декларации** задано значение **Декларация**.

13. В поле **Дата создания документа** выберите **10/19/2021** (19 октября 2021 г.).
