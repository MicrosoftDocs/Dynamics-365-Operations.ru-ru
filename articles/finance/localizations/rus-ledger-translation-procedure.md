# <a name="ledger-translation-russia"></a>Трансляция счетов (Россия)

[!include [banner](../includes/banner.md)]

Трансляция используется для переноса учетных проводок из одной компании в другую и, одновременно, с целью преобразования этих проводок, чтобы они соответствовали особенностям коммерческой деятельности или международным стандартам отчетности. Во время трансляции, счета ГК обычно консолидируются в соответствии с определенным правилом. Иногда данные должны пересчитываться из одной валюты в другую или должны быть изменены финансовые аналитики. Например, учет в одной компании может быть выполнен в рублях, в то время как бизнес-отчетность выполняется в долларах.

Трансляция может осуществляться между одной компанией и несколькими другими компаниями. Например, компания, выполняющая учет, может транслировать данные в одну компанию для подготовки международной отчетности, а в другую компанию — для подготовки коммерческой отчетности.

Трансляция может выполняться только для проводок ГК.

Доступны следующие типы трансляции ваучеров:

- **Один к одному** — если национальная валюта не отличается от валюты отчетности, выполняется трансляция проводок из исходной компании в целевую компанию, в которой используется такая же валюта.
- **Исторический** — если национальная валюта и валюта отчетности различаются, выполните трансляцию ваучеров, используя валютный курс, который действовал на дату разноски ваучера.
- **Взвешенный** — выполните трансляцию ваучеров, используя взвешенный средний валютный курс за период.
- **Текущие** — преобразование ваучеров с использованием валютного курса, который действует на дату трансляции.
- **Специальная** — трансляция выполняется с использованием валютного курса, указанному в проводке, вместо валютного курса, действовавшего на дату ваучера или на дату трансляции.
- **Указанный пользователем** — трансляция ваучеров выполняется по валютному курсу, указанному пользователем.

Компания настраивает правила трансляции на основе ее политик учета. Правила трансляции описывают, какой счет ГК в проводке с планом счетов для России должен быть перенесен, в какой счет главной книги в плане счетов международных стандартов финансовой отчетности (МСФО) проводки должны быть перенесены, какой следует использовать валютный курс, и с какими финансовыми аналитиками проводки должны быть перенесены Проводки переносятся из исходной компании, которая использует российскую бухгалтерскую отчетность, в целевую компанию, которая использует международную или управленческую отчетность.

Необходимо создать целевые компании, выполняющие трансляцию. Перед созданием целевых компаний выполните следующие обязательные условия:

1. Настройте планы счетов.
2. Создайте финансовые аналитики.
3. Настройте валюты и валютные курсы.
4. Создайте финансовые периоды.

Проводки можно транслировать вместе с сохраненной корреспонденцией исходной компании или без нее.

Процедура трансляции выполняется периодически. Она переносит проводки из исходной компании (которая является базой данных для российской отчетности) в целевую компанию (которая представляет собой компанию МСФО или бизнес-компанию).

## <a name="preliminary-setup"></a>Предварительная настройка
Параметры трансляции должны быть настроены в компании-источнике.

### <a name="set-up-a-translation-group"></a>Настройка группы трансляции
Можно создать правила для группы трансляции, чтобы указать все параметры трансляции.

1. Выберите **Главная книга** \> **Настройка перевода** \> **Группы трансляции**.
2. Создайте строку.
3. Введите в поле **Группа правил** уникальный код для группы трансляции.
4. В поле **Компания** введите код для исходной компании.
5. В поле **В компанию** выберите код для целевой компании.

> [!NOTE]  
> Значение стандартной валюты целевой компании автоматически обновляется в поле **В валюту**.

6. В поле **Описание** введите описание группы правил трансляции.
7. На экспресс-вкладке **Методы трансляции** выберите **Инициализация методов** для создания списка методов трансляции.
8. Выберите **Добавить**, затем в поле **Метод** выберите **Трансляция ГК**.

    ![Группы трансляции](media/translation-groups01.jpg)

9. На экспресс-вкладке **Параметры ГК** в поле **Группа валют по умолчанию** выберите группу валют, которая должна использоваться по умолчанию для правил трансляции счетов.
10. В поле **Группа аналитики по умолчанию** выберите группу аналитик, которая должна использоваться по умолчанию для правил трансляции счетов.
11. В поле **Приоритеты по умолчанию** введите приоритет, который должен использоваться по умолчанию для правил трансляции счетов.
12. Установите для параметра **Новые ваучеры ГК** значение **Да**, чтобы создать новые коды ваучеров ГК при выполнении трансляций. Если для параметра **Новые ваучеры ГК** установлено значение **Нет**, проводки счета ГК транслируются на счет ГК целевой компании с использованием номера ваучера исходной компании.
13. В поле **Код номерной серии** выберите код номерной серии для ваучеров.

> [!NOTE]  
> Поле **Код номерной серии** доступно только в том случае, если для параметра **Новые ваучеры ГК** задано значение **Да**.

14. В разделе **Трансляционная разница** определяется уровень, на котором должна быть проверена разница при трансляции, и счет, на который должны быть переведены суммы разницы при трансляции. Разница при трансляции — это дисбаланс, который происходит в основной и вспомогательной валютах целевой компании из-за того, что оценка соответствующих записей выполняется с разными курсами.
15. В поле **Проверка трансляционной разницы** выберите один из следующих методов проверки для трансляционной разницы:

    - **Ошибка** — если проводка разносится на счет трансляционной разницы, отображается сообщение об ошибке и сеанс трансляции отменяется.
    - **Предупреждение** — когда сеанс трансляции завершен, сообщение о суммах, разнесенных на счет трансляционной разницы, отображается. Все проводки транслируются на счет ГК целевой компании.

16.  В поле **Счет учета** выберите номер счета ГК, который должен отражать трансляционные разницы для валюты по умолчанию и вторичной валюты целевой компании.

> [!NOTE]  
> Трансляционная разница может возникать, когда дебетовые и кредитные строки проводки транслированы в целевую компанию по разным валютным курсам. Трансляционная разница может также возникать из-за ошибок округления.
>
> В разделе **Дисбаланс** определяется уровень, на котором должен проверяться дисбаланс, и счет, на который должны быть переведены суммы дисбаланса. Дисбаланс — это разница, который происходит в сумме проводки в целевой компании из-за того, что оценка соответствующих проводок выполняется с разными курсами.

17. В поле **Проверка дисбаланса** выберите метод проверки для дисбаланса:

    - **Ошибка** — если проводка разносится на счет дисбаланса, отображается сообщение об ошибке и сеанс трансляции отменяется.
    - **Предупреждение** — когда сеанс трансляции завершен, сообщение о суммах, разнесенных на счет дисбаланса, отображается. Все проводки транслируются на счет ГК целевой компании.

18. В поле **Счет учета** выберите номер счета, который должен использоваться для накопления сумм дисбаланса в валюте трансляции целевой компании.

> [!NOTE]  
> Дисбаланс может произойти во время трансляции, если дебетовая сторона проводки транслируется в целевую компанию, однако кредитная сторона проводки не транслируется.

19. Выберите **Сохранить** и закройте страницу.

## <a name="set-up-currency-groups-so-that-they-have-currency-conversion-rules"></a>Настройка групп валют, чтобы они имели правила преобразования валют
Настройка групп валют для указания правил конвертации валют, используемых в процессе трансляции.

1. Выберите **Главная книга \> Настройка перевода \> Группы валют**.
2. В левой области в поле **Группа правил** выберите группу правил, для которой вы настраиваете правило конвертации валют.

> [!NOTE]  
> Можно настроить несколько правил преобразования валют для одной группы правил.

![Определения пересчета валют](media/currency-groups02.jpg)


3. Создайте группу валют.
4. В поле **Группа валют** введите уникальный код для группы валют.
5. В поле **Описание** введите описание группы валют.
6. На экспресс-вкладке **Определения пересчета валют** выберите **Добавить**, чтобы создать строку.
7. В поле **Валюта** выберите код для валюты в компании-источнике, для которой настраивается метод конвертации.
8. В поле **Вариант курса** выберите метод конвертации валюты, который должен использоваться при трансляции проводок из исходной компании в целевую компанию:

    - **Без изменений** — в целевой компании создается транслированная проводка, в которой сумма проводки и валюта равна сумме проводки и валюте исходной компании. В транслированной проводке суммы в валютах учета и отчетности рассчитываются с использованием валютного курса целевой компании.
    - **Использовать индексы** — если выбран этот вариант, на экспресс-вкладке **Индексы и суммы** определите правила использования индексов для определения сумм проводок (в валюте проводки, основной валюте и вторичной валюте) для целевой компании:
    
        - В поле **Индекс (вал)** выберите индекс преобразования для валюты проводки.
        - В поле **Индекс (осн)** выберите индекс преобразования для валюты учета.
        - В поле **Индекс (втор)** выберите индекс преобразования для валюты отчетности.

    - **Использовать курсы** — если выбран этот вариант, на экспресс-вкладке **Индексы и суммы** определите правила использования валютных курсов для определения сумм проводок (в валюте проводки, основной валюте и вторичной валюте) для целевой компании:

        - В поле **Сумма (вал.)** выберите правило для конвертации суммы проводки в валюту проводки.
        - В поле **Сумма (осн.)** выберите правило для конвертации суммы проводки в валюту учета.
        - В поле **Сумма (втор.)** выберите правило для конвертации суммы проводки в валюту отчетности.
        - В этих трех полях **Сумма...** можно выбрать следующие правила:

            - **Из валюты проводки** — сумма рассчитывается на основе суммы исходной проводки путем конвертации по валютному курсу целевой компании.
            - **Ноль** — подставляется значение **0** (ноль).
            - **Сумма в валюте проводки** — сумма копируется из суммы в валюте проводки исходной проводки.
            - **Сумма в валюте учета** — сумма копируется из суммы в валюте учета исходной проводки.
            - **Сумма в валюте отчетности** — сумма копируется из суммы в валюте отчетности исходной проводки.
            - **1 (один)** — подставляется значение **1** (единица).

9.  Выберите **Сохранить** и закройте страницу.

## <a name="set-up-an-inflation-index"></a>Настройка индекса инфляции

1. Выберите **Главная книга \> Настройка перевода \> Индексы инфляции**.
2. В левой области в поле **Группа правил** выберите группу трансляции, для которой вы настраиваете индекс инфляции.

> [!NOTE]  
> Можно настроить несколько индексов инфляции для одной группы правил.

3. Выберите **Создать**, чтобы создать индекс инфляции.
4. В поле **Индекс инфляции** введите уникальный код для индекса инфляции.
5. В поле **Описание** введите описание индекса инфляции.
6. На экспресс-вкладке **Правила подстановки аналитик** выберите **Добавить**, чтобы создать строку.
7. В поле **Начальная дата** выберите начальную дату, с которой начинает действовать этот индекс.

>  [!NOTE]  
>  Настроенный индекс начинает работать в выбранную дату начала и продолжает работать до следующей даты начала другого настроенного индекса. Можно также настроить индекс без указания начальной даты. В этом случае индекс будет действовать для всех проводок перед датой начала другого индекса.

8. В поле **Валютный курс** введите значение индекса, которое должно использоваться в течение периода действия индекса.

![Экспресс-вкладка "Правила подстановки аналитик", поле "Валютный курс"](media/inflation-indexes03.jpg)

9. Выберите **Сохранить** и закройте страницу.

## <a name="set-up-a-dimension-group-and-conversion-rules-for-dimensions"></a>Настройка группы аналитик и правил преобразования для аналитик
Страницу **Группы аналитики** можно использовать для настройки правил аналитик трансляции из исходной компании в целевую компанию. Отдельная группа правил создается для преобразования аналитик для каждой группы трансляции, а затем для этой группы указывается набор параметров преобразования аналитик.

1. Выберите **Главная книга** \> **Настройка перевода** \> **Группы аналитики**.
2. В левой области в поле **Группа правил** выберите группу трансляции, для которой вы настраиваете правила преобразования аналитик.

> [!NOTE]  
> Можно настроить несколько правил преобразования для аналитик одной группы правил.

![Экспресс-вкладка "Правила подстановки аналитик"](media/dimension-groups04.jpg)

3. Выберите **Создать**, чтобы создать группу аналитик.
4. В поле **Группа аналитик** введите уникальный код для группы аналитик.
5. В поле **В компанию** введите код целевой компании, которая должна использоваться для проводок во время трансляции.
6. В поле **Описание** введите описание группы аналитик.
7. На экспресс-вкладке **Правила подстановки аналитик** выберите **Добавить**, чтобы создать строку.
8. В поле **Описание** выберите аналитику в исходной компании, для которой настраивается правило трансляции.
9. В полях **С кода аналитики** и **Код целевой аналитики** определите диапазон значений аналитики.
10. В поле **Правило обработки** выберите метод, используемый для преобразования аналитики транслированной проводки из исходной компании:

    - **Без изменений** — исходная аналитика переносится без изменений.
    - **Код компании** — текущая аналитика изменяется на код для исходной компании.
    - **Значение** — текущая аналитика заменяется значением, введенным в поле **Код целевой аналитики**.
    - **Клиент/поставщик** — текущая аналитика заменяется кодом клиента или поставщика.

11. В поле **В аналитику** выберите тип аналитики в целевой компании, в которую должны переноситься значения.
12. В поле **Код целевой аналитики** выберите код для аналитики в целевой компании, в которую следует переместить аналитику проводки в исходной компании в процессе трансляции.

> [!NOTE]  
> Это поле доступно только в том случае, если в поле **Правило обработки** выбрано значение **Значение**.

13.  Выберите **Сохранить** и закройте страницу.

|-----------------------------------|-------------------|--------------------------------------|-------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

| Тип аналитики                    | Клиент/поставщик   | Тип аналитики                       | Нет              | Аналитики преобразуются в указанный код клиента или поставщика (то есть, если проводка была начислена для клиента или поставщика, вместо аналитики подставляется номер счета). |

## <a name="set-up-account-translation-rules"></a>Настройка правил трансляции счета
Страница **Правила трансляции счетов ГК** используется для настройки правил трансляции для счетов ГК. Эти правила используются для сопоставления счетов исходной компании со счетами целевой компании и включают правила преобразования валюты и аналитик.

1. Выберите **Главная книга \> Настройка перевода \> Трансляция счетов ГК**.
2. В верхней области страницы в поле **Группа правил** выберите группу трансляции, для которой вы настраиваете правила преобразования счетов.

> [!NOTE]  
> Можно настроить несколько правил преобразования для одной группы правил.

3. Выберите **Создать**, чтобы создать строку.
4. На вкладке **Обзор** в поле **На счет** выберите счет ГК из плана счетов целевой компании, на который должны транслироваться проводки в исходной компании. После сохранения изменений в строке название целевой компании автоматически выбирается в поле **Имя счета**.
5. В поле **Счет ГК** выберите счета учета в исходной компании, который должен использоваться для трансляции проводок в целевую компанию.
6. В поле **Корр. счет** выберите корр. счет в исходной компании, который должен использоваться для трансляции проводок, имеющих указанную корреспонденцию. Оставьте это поле пустым, чтобы выбрать все проводки, независимо от корр. счета.
7. В поле **Кредитовый признак** выберите тип проводок для трансляции:

    - **Нет** — транслируются только проводки дебетового счета.
    - **Да** — транслируются только проводки кредитного счета.

Оставьте это поле пустым, чтобы транслировать кредитные и дебетовые проводки.

В зависимости от значений полей **Корр. счет** и **Кредитовый признак** транслируются следующие проводки:

   - Все проводки
   - Только проводки по дебету
   - Только проводки по кредиту
   - Проводки по счету, которые соответствуют определенному корр. счету
   - Проводки дебетового счета, которые соответствуют определенному кредитовому корр. счету
   - Проводки кредитового счета, которые соответствуют определенному дебетовому корр. счету

В приведенной ниже таблице показаны примеры для различных возможных значений полей **Счет ГК**, **Кредитовый признак** и **Корр. счет**.

| Счет ГК | Кредитовый признак | Корр. счет | Результат                                                                              |
|--------------|-----------|----------------|-------------------------------------------------------------------------------------|
| А            | Пустой     | Пустой          | Все проводки по счету А транслируются.                                       |
| А            | Нет        | Пустой          | Проводки по дебету по счету А транслируются.                                     |
| А            | Да       | Пустой          | Проводки по кредиту по счету А транслируются.                                    |
| А            | Пустой     | В              | Проводки по счету A, который соответствует счету В, транслируются.              |
| А            | Нет        | В              | Проводки по дебету по счету A, который соответствует кредитовому счету В, транслируются. |
| А            | Да       | В              | Проводки по кредиту по счету A, который соответствует дебетовому счету В, транслируются. |

-   Дебит A, Кредит \*

-   Дебит \*, Кредит A

-   Дебит A, Кредит \*

-   Дебит \*, Кредит A

-   Дебит A, Кредит B

-   Дебит B, Кредит A

-   Дебит A, Кредит B

-   Дебит B, Кредит A

8. В поле **Группа валют** выберите код для группы преобразования валют. Если группа валют по умолчанию настраивается для группы правил трансляции, выбранной в поле **Группа правил**, значение вводится автоматически. Однако если требуется, это значение можно изменить.
9. В поле **Группа аналитик** выберите код для группы преобразования аналитик. Если группа аналитик по умолчанию настраивается для группы правил трансляции, выбранной в поле **Группа правил**, значение вводится автоматически. Однако если требуется, это значение можно изменить.
10. В поле **Приоритет** введите приоритет правила. Если приоритет по умолчанию выбран для группы правил трансляции, выбранной в поле **Группа правил**, значение вводится автоматически. Однако если требуется, это значение можно изменить.

> [!NOTE]  
> Приоритет определяет порядок, в котором обрабатываются правила во время трансляции. Правило, которое имеет самый низкий приоритет, обрабатывается первым. Рекомендуется указывать приоритет, если имеется несколько взаимоисключающих правил.

![Строки "Правила трансляции счетов ГК"](media/translation-ledger-account-rules05.jpg)

11. На вкладке **Общие сведения** в первом поле **Тип разноски** выберите тип операции, чтобы транслировать только проводки, имеющие указанный тип ваучера. Например, если выбран вариант **Кассовые операции**, транслируются только проводки в банковском модуле.
12. Во втором поле **Тип разноски** выберите тип операции, чтобы транслировать только проводки, имеющие указанный тип ваучера для корр. счета.
13. В поле **Слой разноски** выберите слой разноски, чтобы транслировать только те проводки, которые имеют этот слой разноски.
14. В поле **Только сторно** выберите **Да**, чтобы транслировать только реверсирующие проводки, или выберите **Нет**, чтобы транслировать только обычные проводки. Оставьте это поле пустым, чтобы транслировать как обычные, так и реверсирующие проводки.
15. В поле **В тип учета** выберите слой разности, которому должны назначаться транслированные проводки в целевой компании.

![Поля "Правила трансляции счетов ГК"](media/translation-ledger-account-rules06.jpg)

16. Выберите **Инициализировать**, чтобы автоматически создать строку, которая имеет правило трансляции счета для всех счетов ГК в целевой компании. В диалоговом окне **Инициализация** можно выбрать **Фильтр** для фильтрации необходимых счетов.
17. Нажмите **ОК**. Система создает строку, имеющую правило для каждого выбранного счета ГК.

## <a name="set-up-translation-parameters"></a>Настройка параметров трансляции

1. Перейдите в раздел **Главная книга** \> **Настройка главной книги** \> **Параметры главной книги**.
2. На вкладке **Трансляция** установите для параметра **Останов на первой ошибке** значение **Да**, чтобы прервать сеанс трансляции при обнаружении первой ошибки.
3. В поле **Группа правил** выберите группу правил трансляции, которая должна использоваться по умолчанию при запуске сеанса трансляции.
4. На вкладке **Номерные серии** в поле **Код номерной серии** выберите код номерной серии для ссылки **Сессия трансляции**.
5. Закройте страницу.

## <a name="do-a-single-translation"></a>Выполнение отдельной трансляции
При создании нового сеанса трансляции открывается мастер **Трансляция**. Используйте этот мастер для выполнения следующих операций:

1. Проверьте правильность параметров преобразования.
2. Выберите даты трансляции.
3. Примените правила трансляции для всех операций в указанном периоде.
4. Отобразите результаты преобразования проводок на странице **Просмотр**.
5. Сохраните результаты трансляции в целевую компанию.

Выполните следующие действия, чтобы использовать мастер **Трансляция** для выполнения одной трансляции.

1. Перейдите в раздел **Главная книга** \> **Периодические задачи** \> **Сессии трансляции**.
2. В верхней части страницы в поле **Группа правил** выберите группу трансляций.
3. Выберите **Мастер трансляции**, чтобы открыть мастер **Трансляция**.
4. На странице **Добро пожаловать** выберите **Далее**.

![Страница приветствия](media/welcome07.jpg)


5. На странице **Проверка настроек** выберите **Проверка настроек** для проверки параметров преобразования.

![Страница "Проверка настроек трансляции данной компании"](media/consistency-check08.jpg)

6. В диалоговом окне **Проверка настроек** укажите сведения, затем выберите **ОК**.

> [!NOTE]  
> Если параметр **Останов на первой ошибке** на вкладке **Трансляция** страницы **Параметры главной книги** имеет значение **Да**, перейти на следующую страницу мастера невозможно до тех пор, пока не будет выполнена проверка параметров преобразования на наличие дополнительных ошибок.

![Страница "Проверка настроек"](media/consistency-check09.jpg)

7. Выберите **Далее**.
8. На странице **Выбор** в полях **Начальная дата** и **Конечная дата** укажите период, для которого должна быть выполнена трансляция.
9. Выберите **Запрос**, чтобы задать дополнительные критерии выбора трансляции.

![Страница выбора](media/selection10.jpg)

10. Выберите **Далее**.
11. На странице **Обработка проводок** выберите **Далее**, чтобы выполнить преобразование данных.

![Страница "Обработка проводок"](media/translation-processing11.jpg)

12. Выберите **Просмотр**, чтобы посмотреть результаты преобразования.
13. Выберите **Далее** два раза, чтобы импортировать данные трансляции в целевую компанию.
14. После завершения трансляции выберите **Готово**.

Когда сеанс трансляции для исходной компании успешно завершен, выполняются следующие действия:

   - Страница **Сессии трансляции** автоматически создает строку для статуса сессии трансляции. Поля **Статус экспорта** и **Статус импорта** автоматически обновляются на значение **Завершено**.
   - Поле **Сессия трансляции** обновляется до номера сессии трансляции.
   - Поля **Начальная дата** и **Конечная дата** обновляются до дат начала и окончания периода, для которого была выполнена трансляция.
   - Поле **Число проводок** обновляется до количества операций, которые были успешно обработаны в течение сеанса трансляции.

15. Перейдите к целевой компании и убедитесь, что проводки были успешно перемещены. Дата проводок, которые переносятся в целевую компанию, будет датой разноски проводок в исходной компании.

> [!NOTE]  
> В целевой компании, если выбрано значение **Трансляционная разница** в поле **Тип разноски** на вкладке **Общие сведения** на странице **Правила трансляции счетов ГК** (**Главная книга \> Настройка перевода \> Трансляция счетов ГК**), корреспонденция сохраняется при трансляции проводок. В противном случае проводки транслируются без корреспонденции.

## <a name="batch-translation"></a>Пакетная трансляция
Можно настроить трансляцию так, чтобы она периодически выполнялась в соответствии с определенными правилами в пакетном режиме.

1. Перейдите в раздел **Главная книга** \> **Периодические задачи** \> **Сессии трансляции**.
2. В верхней части страницы в поле **Группа правил** выберите группу для пакетной трансляции.
3. Выберите **Пакетная трансляция**, чтобы открыть диалоговое окно **Пакетная трансляция**.
4. На экспресс-вкладке **Параметры** в полях **Начальная дата** и **Конечная дата** укажите период, для которого должна быть выполнена пакетная трансляция.

![Страница "Пакетная трансляция"](media/batch-translation12.jpg)

5. На экспресс-вкладке **Выполнять в фоновом режиме** выберите **Повторение**, чтобы открыть диалоговое окно **Определение повторения**, в котором можно настроить повторения пакетной трансляции.

![Диалоговое окно "Определение повторения"](media/define-recurrence13.jpg)

6. В диалоговом окне **Пакетная трансляция** выберите **ОК**, чтобы выполнить пакетную трансляцию.

## <a name="translation-cancellation"></a>Отмена трансляции
Выполненная трансляция не может обрабатываться более одного раза. Чтобы повторно выполнить трансляцию, необходимо сначала отменить обработанную трансляцию.

1. Перейдите в раздел **Главная книга \> Периодические задачи \> Сессии трансляции**.
2. В нижней части страницы выберите трансляцию, которую требуется отменить.
3. Выберите **Отмена**. Поля **Статус экспорта** и **Статус импорта** автоматически обновляются на значение **Отменено**.
