# <a name="tax-calculation-overview"></a>Обзор расчета налогов

[!include [banner](../includes/banner.md)]

Расчет налогов представляет собой гипермасштабируемую многоклиентскую службу, которая позволяет модулю Global Tax Engine автоматизировать и упростить процесс определения и расчета налогов. Модуль налогов полностью настраиваемый. Элементы, которые могут быть настроены, включают в себя (но не ограничиваются этим) модель налогооблагаемых данных, налоговый код, матрицу налоговой применимости и формулу расчета налога. Модуль обработки налогов выполняется на платформе Microsoft Azure Core Services и предлагает современные технологии и экспоненциальную масштабируемость.

Расчет налогов интегрируется с Dynamics 365 Finance и Dynamics 365 Supply Chain Management. В конечном итоге она также интегрируется с Dynamics 365 Project Operations, Dynamics 365 Commerce и другими собственными приложениями и приложениями сторонних разработчиков.

> [!IMPORTANT]
> При включении расчета налогов можно выполнить некоторые операции с соответствующими данными в центрах обработки данных, отличном от центра обработки данных, в котором хранятся данные службы. Перед включением расчета налогов просмотрите [Условия](../../fin-ops-core/fin-ops/get-started/public-preview-terms.md). Мы заботимся о конфиденциальности ваших данных. Чтобы получить дополнительные сведения, ознакомьтесь с нашим [заявлением о конфиденциальности](https://go.microsoft.com/fwlink/?LinkId=521839).

Расчет налога — это подсистема налогов на основе микрослужб, которая обеспечивает экспоненциальную масштабируемость и может помочь в выполнении следующих задач:

- Автоматически определяет правильную налоговую группу, налоговую группу номенклатур и налоговые коды с помощью улучшенного механизма определения.
- Поддерживает несколько налоговых регистрационных номеров для одного юридического лица и может автоматически определить правильный налоговый регистрационный номер для налогооблагаемых проводок.
- Поддержка определения налога, расчета, разноски и сопоставления заказов на перемещение.
- Определение настраиваемых формул расчета налогов и условий для определенных деловых требований.
- Предоставьте общий доступ к решению определение налогового учета и расчета для юридических лиц, чтобы сэкономить усилия по обслуживанию и избежать ошибок.
- Поддержка определения кода регистрации налога для клиента и поставщика.
- Определение кода списка поддержки.
- Поддержка параметров расчета налога на уровне налоговой инспекции.

Чтобы использовать расчет налога, установите надстройку расчета налогов из проекта в Microsoft Dynamics Lifecycle Services. Затем выполните настройку в [Regulatory Configuration Service](https://marketing.configure.global.dynamics.com/) и включите расчет налогов в Finance и Supply Chain Management. Дополнительные сведения см. в разделе [Начало работы с налоговой службой](global-get-started-with-tax-calculation-service.md).

## <a name="availability"></a>Доступность

Расчет налога обычно доступен в рабочих средах для всех клиентов по состоянию на версию 10.0.21.

Новые функции будут по-прежнему предоставляться. Регулярно проверяйте самый актуальный план выпуска, чтобы узнать о зоне действия и области поддерживаемых функций.

Расчет налогов развернут в следующих географических регионах Azure. Большее число географических регионов Azure будет развернуто на основе потребностей клиентов.

- Азиатско-тихоокеанский
- Австралия
- Канада
- Европа
- Япония
- Швейцария
- Соединенное Королевство
- Соединенные штаты

> [!NOTE]
> Расчет налога не поддерживает более раннюю версию Dynamics 365, например, Dynamics AX 2012 или локальное развертывание Dynamics 365.

## <a name="data-flow"></a>Поток данных

Ниже приведена структура процесса потока данных для расчета налога. 

1. В RCS можно просмотреть и импортировать конфигурации моделей налогооблагаемых документов и конфигурации сопоставления моделей. Если необходимо расширить конфигурации для расширенного сценария, см. раздел [Добавление полей данных в налоговые конфигурации](tax-service-add-data-fields-tax-configurations.md).
2. В RCS создайте или настройте налоговые функции. Налоговые функции можно использовать для ведения ставок налогов и правил применимости налогов.
3. После завершения настройки налоговых функций опубликуйте налоговые конфигурации и налоговые функции из RCS в глобальном репозитории.
4. В Finance выберите версию настройки налоговой функции, которая будет использоваться для конкретного юридического лица.
5. В модуле Finance и Supply Chain Management проводки действуют обычным образом. Когда требуется расчет налога, клиент будет собирать сведения из проводки, такой как заказ на продажу или заказ на покупку, и упаковывать информацию в качестве полезных данных. Затем для расчета налога будет отправлен запрос.
6. Запрос расчета налога получен от клиента и расчет завершен. Затем налоговый результат возвращается клиенту.
7. Клиент Dynamics 365 получает налоговый результат и отображает результат расчета налога на странице "Налог".

## <a name="supported-transactions"></a>Поддерживаемые проводки

Расчет налога может быть включен по проводкам. 

В версии 10.0.21 поддерживаются следующие проводки: 

- Прод.

    - Предложение по продажам
    - Заказ на продажу
    - Подтверждение
    - Сборочный лист
    - Отборочная накладная
    - Накладная заказа на продажу
    - Кредит-нота
    - Заказ на возврат
    - Накладные расходы по заголовку
    - Накладные расходы по строке

- Покупка

    - Заказ на покупку
    - Подтверждение
    - Список прихода
    - Поступление продуктов
    - Накладная по покупке 
    - Накладные расходы по заголовку
    - Накладные расходы по строке
    - Кредит-нота
    - Заказ на возврат
    - Заявка на закупку
    - Накладные расходы строки заявки на покупку
    - Запрос предложения
    - Накладные расходы заголовка запроса предложения
    - Накладные расходы строки запроса предложения

- Запасы

    - Заказ на перемещение — отгрузка
    - Заказ на перемещение — получение

В версии 10.0.23 поддерживаются следующие проводки: 

- Накладная с произвольным текстом

В версии 10.0.26 поддерживаются следующие проводки:

В версии 10.0.28 поддерживаются следующие проводки: 

- Журнал платежей поставщика
