#  <a name="vat-declaration-for-egypt-eg-00002"></a>Декларация по НДС для Египта (EG-00002)

[!include[banner](../includes/banner.md)]

[!include[banner](../includes/banner.md)]

В этой теме объясняется, как настроить и создать форму возврата НДС и книги продаж и покупок для юридических лиц в Египет.

Форма возврат НДС для Египта — это официальный документ, в котором суммируются итоговая сумма исходящего НДС к уплате, итоговая сумма входящего НДС к возмещению, и соответствующие обязательства по сумме НДС. Форма используется для всех типов налогоплательщиков и должна заполняться вручную через портал налогового органа. Форма возврата НДС обычно называется подача заявления на возврат НДС.

Форма возврата НДС в Dynamics 365 Finance содержит следующие отчеты:

- Форма возврата НДС номер 10, которая обеспечивает разбивку сумм, корректировок и сумм НДС по номенклатурам строки в форме возврата НДС, как описано в законодательстве.
- Журнал проводок по продажам
- Журнал проводок по покупкам

## <a name="prerequisites"></a>Необходимые условия
Основной адрес юридического лица должен быть в Египте.
В рабочей области **Управление функциями** включите следующую функцию:

   - (Египет) Иерархия категорий для налогового отчета по продажам и покупкам.

Дополнительные сведения о включении этих функций см. в разделе [Обзор управления функциями](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).

В рабочей области **Электронная отчетность** импортируйте из репозитория следующий формат электронного отчета:

- Excel декларации по НДС (EG)

> [!NOTE]
> Вышеприведенный формат основан на **модели налоговой декларации** и использует **сопоставление модели налоговой декларации**. Дополнительные конфигурации будут импортированы автоматически.

Дополнительные сведения об импорте конфигураций электронной отчетности см. в разделе [Загрузка конфигураций электронной отчетности из Lifecycle Services](../../fin-ops-core/dev-itpro/analytics/download-electronic-reporting-configuration-lcs.md).

## <a name="download-electronic-reporting-configurations"></a>Загрузка конфигураций электронной отчетности

Реализация формы возврата НДС для Египта основана на конфигурациях электронной отчетности (ER). Дополнительные сведения о возможностях и концепциях настраиваемой отчетности см. в разделе [Электронная отчетность](../../fin-ops-core/dev-itpro/analytics/general-electronic-reporting.md).

О средах производства и приемочном тестировании пользователем (UAT) см. раздел [Загрузка конфигураций электронной отчетности из Lifecycle Services](../../fin-ops-core/dev-itpro/analytics/download-electronic-reporting-configuration-lcs.md).

Чтобы создать форму возврата НДС и соответствующие отчеты в юридическом лице в Египте, необходимо загрузить следующие конфигурации:

- Tax declaration model.version.70.xml или более поздняя версия
- Tax declaration model mapping.version.70.120.xml или более поздняя версия
- VAT Declaration Excel (EG).version.70.32 или более поздняя версия

После загрузки конфигураций электронной отчетности из Lifecycle Services (LCS) или глобального репозитория выполните следующие действия.

1. Перейдите в раздел рабочей области **Электронная отчетность** и выберите плитку **Конфигурации отчетности**.
1. На странице **Конфигурации** на панели действий выберите **Exchange** > **Загрузить из XML-файла**.
1. Загрузите файлы в перечисленной выше последовательности. После загрузки всех конфигураций дерево конфигурации должно присутствовать в Финансах.

## <a name="set-up-application-specific-parameters"></a>Настройка параметров конкретного приложения

Зависящие от приложения параметры позволяют установить критерии классификации и расчета налоговых проводок в каждой строке при создании отчета. Определение основано на конфигурации налоговой группы номенклатур, налоговой группы, налогового кода и других критериев, установленных в определении поиска.

Отчеты по продажам и покупкам для Египет включают набор столбцов, соответствующий отдельным классификациям проводок в виде типов операций, продуктов и документов, которые относятся к Египет. Вместо включения этих новых классификаций в качестве новые данные при разноске проводок классификации будут определяться на основе различных уточняющих запросов, введенных в **конфигурации** > **Настройка параметров для конкретного приложения** > **Настройка** в соответствии с требованиями отчетов по НДС для Египет. 

![Страница параметров для конкретного приложения.](media/egypt-vat-declaration-setup1.png)

Следующие конфигурации поиска используются для классификации проводок в отчетах книг входящего и исходящего НДС:

- **PurchaseItemTypeLookup** > столбец O: тип номенклатуры
- **VATRateTypeLookup** > столбец B: тип налога
- **VATRateTypeLookup** > столбец C: тип элемента таблицы
- **PurchaseOperationTypeLookup** > столбец A: тип документа
- **CustomerTypeLookup** > столбец A: тип документа
- **SalesOperationTypeLookup** > столбец N: тип операции
- **SalesItemTypeLookup** > столбец O: тип номенклатуры

Выполните следующие действия, чтобы настроить различные подстановки, используемые для создания декларации по НДС и соответствующих отчетов по книгам. 

1. В рабочей области **электронной отчетности** выберите **конфигурации** > **Настройка**, чтобы настроить правила для идентификации налоговой проводки в соответствующем поле в форме возврата НДС.
2. Выберите текущую версию и на экспресс-вкладке **Поиск** выберите имя поиска. Например, **SalesItemTypeLookup**. Этот поиск определяет список классификаций в книге исходящего НДС, которые требуются для налогового органа.
3. На экспресс-вкладке **условия** выберите **Добавить** и в новой строке столбца **результат поиска** выберите соответствующую строку, представляющую классификацию в **столбце O**.
4. В столбце **налоговая группа** выберите налоговую группу, которая используется для идентификации классификации. Например, **Накладная по продаже внутри страны**. Также можно использовать налоговую группу номенклатур, налоговый код или комбинацию атрибутов дерева, если конфигурация определена другим способом. 
5. В столбце **имя** выберите классификацию налоговой проводки.
6. Повторите шаги 3-5 для всех доступных поисков.
7. Выберите **Добавить** для включения конечной строки записи, а в столбце **Результат поиска** выберите **неприменимо**. 
8. В столбцах остаток выберите **не пусто**. 
9. В поле **Состояние** выберите **Завершено**.
10. Выберите **Сохранить**, затем закройте страницу **Параметры для конкретного приложения**.

> [!NOTE]
> При добавлении последней записи, **Неприменимо**, определяется следующее правило: когда налоговая группа, налоговая группа номенклатур, налоговый код и имя, переданные в качестве аргумента, не удовлетворяют никаким предыдущим правилам, проводки не включаются в книгу исходящего НДС. Хотя это правило не используется при создании отчета, это правило позволяет избежать ошибок в формировании отчета, когда отсутствует конфигурация правила.

## <a name="set-up-general-ledger-parameters"></a>Настройте параметры главной книги

Чтобы создать отчет по форме возврата НДС в формате Microsoft Excel, укажите формат электронной отчетности на странице **Параметры главной книги**.

1. Перейти к **Налог** > **Настройка** > **Параметры главной книги**.
2. На вкладке **налог** в разделе **Параметры налогов** в поле **Сопоставление формата отчета по НДС** выберите **Excel декларации по НДС (EG)**. Если оставить это поле пустым, стандартный налоговый отчет будет создан в формате SSRS.
3. Выберите **иерархию категорий**. Эта категория активирует товарный код в проводках на вкладке "Внешняя торговля", чтобы позволить пользователям выбирать и классифицировать товары и услуги. Описание этой классификации подробно описано в отчетах по проводкам продаж и покупки. Эта конфигурация не является обязательной.

![Форма декларации.](media/egypt-vat-declaration-setup2.png)


## <a name="generate-a-vat-return-report"></a>Создание отчета о возврате НДС
Процесс подготовки и подачи отчета о возврате НДС за период основан на проводках по оплате налога, которые были разнесены в ходе сопоставления и разноски налога. Дополнительные сведения о сопоставлении налогов и отчетности см. в [Обзор налога](../general-ledger/indirect-taxes-overview.md).

Выполните следующие шаги для создания налоговой декларации.

1. Перейти к **налог** > **декларации** > **налог** > **Отчет о налоге для периода сопоставления** или **Сопоставить и разнести налог**.
2. Выберите **Период сопоставления**.
3. Выберите начальную дату, а затем выберите версию налогового платежа.
5. Нажмите **ОК**. 
6. Введите сумму кредита предыдущего периода, если это применимо, или оставьте нулевую сумму.
7. В поле **Сведения об отчетах** выберите один из доступных параметров. 
   - **Книга входящего НДС**: создание подробного отчета о налоговых проводках по покупкам.
   - **Книга исходящего НДС**: создание подробного отчета о налоговых проводках по продажам.
   - **Декларация по НДС**: создание только формы декларации на возврат НДС.
8. Ввод имени лица, зарегистрированного для назначения формы.
9. Выберите язык. Все отчеты переводятся в **en-us** и **ar-eg**.
