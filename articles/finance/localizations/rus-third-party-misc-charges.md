# <a name="third-party-miscellaneous-charges"></a>Сторонние накладные расходы

[!include [banner](../includes/banner.md)]

В этой теме объясняется, как обрабатывать услуги, которые приобретаются, когда отдельная накладная используется в качестве накладных расходов. Затраты на эти услуги могут быть включены в себестоимость купленных товаров или списаны как затраты, например накладные расходы, которые определены в накладной для покупки товаров. Кроме того, задолженность по этим услугам может быть перевыставлена для третьих сторон, таких как клиенты или поставщики. В этом случае создается накладная, которая не будет включаться в книгу продаж или книгу покупок.

Необходимо купить номенклатуру типа **Услуга** для выполнения учета в качестве накладных расходов, которые будут распределены позже. Также необходимо указать относящиеся к номенклатуре параметры для кода накладных расходов.

Существует три режима распределения накладных расходов:

 - **Включить в себестоимость** — распределить накладные расходы по другим накладным номенклатур, чтобы распределения можно было включить в себестоимость номенклатуры. База распределения является строками накладной. Распределение приводит к корректировке затрат на номенклатуру.
 - **Перевыставить задолженность** — распределение накладных расходов для перевыставления задолженности третьим сторонам. Базой распределения является контрагент (клиент или поставщик), которому повторно назначается задолженность. Распределение заставляет систему создавать накладные для оплаты и фактуры для выбранного контрагента.
 - **Списание на затраты** — распределение накладных расходов на счет ГК. База распределения является счетом учета. Распределение приводит к списанию суммы расходов на выбранный счет.
 
## <a name="setup"></a>Настройка
### <a name="set-up-a-charges-code"></a>Настройка кода сборов

1. Перейдите в раздел **Расчеты с клиентами** > **Настройка расходов** > **Код расходов**.
2. Выберите **Создать** для создания кода расходов.
3. В поле **Код расходов** введите код для расходов.
4. В поле **Описание** введите описание расходов.
5. Необязательно: в поле **Налоговая группа номенклатур** выберите налоговую группу номенклатур.
6. В поле **Максимальная сумма** введите максимальную сумму, разрешенную для этого кода расходов. Это поле используется для проверки накладных расходов для накладных поставщиков.

    > [!NOTE]
    > Чтобы включить функции для проверки расходов, на странице **Параметры модуля расчетов с поставщиками** на вкладке **Проверка накладной** установите параметр **Включить проверку сопоставления накладных** как **Да**.

7. На экспресс- вкладке **Разноска** в разделе **Дебет** выполните следующие шаги:

      a. В поле **Тип** выберите **Счет ГК** для разноски приобретенной услуги в качестве накладных расходов.
      
    b. В поле **Разноска** выберите **Распределение** в качестве типа разноски для дебетового счета.
      
    c. В поле **Счет** выберите номер счета, на который дебетуется сумма заказа на покупку или строки журнала накладных.

8. В разделе **Кредит** в поле **Тип** выберите **Клиент/поставщик**.
9. Нажмите **Сохранить**.

[![Страница кода расходов.](./media/1-charges-codes.jpg)](./media/1-charges-codes.jpg)
 
### <a name="create-a-released-product-for-a-service-item"></a>Создание выпущенного продукта для номенклатуры "Услуга"

1. Выберите **Управление сведениями о продукте** > **Продукты** > **Выпущенные продукты**, чтобы открыть страницу **Сведения о выпущенном продукте**.
2. Создайте продукт.
3. В поле **Тип продукта** выберите **Услуга**.
4. В поле **Группа номенклатурных моделей** выберите группу номенклатурных моделей для номенклатур, которые не учитываются в запасах, и в поле **Номенклатурная группа** выберите номенклатурную группу для услуг.
5. В поле **Группа аналитик отслеживания** выберите группу аналитик.
6. В поле **Код расходов** выберите код расходов, созданный ранее.

[![Страница выпущенных продуктов.](./media/2-released-product-details.jpg)](./media/2-released-product-details.jpg)
 
### <a name="set-up-a-sales-tax-group-that-has-exempt-tax-codes"></a>Настройка налоговой группы, которая содержит коды освобождения от налога
Если необходимо переписать задолженность для услуг, предоставляемых третьей стороне, можно создать счет-фактуру для перевыписанных сумм, которые не отражаются в книге покупок или книге продаж. Настройте налоговую группу так, чтобы в ней были включены налоговые коды, и убедитесь, что установлены флажки **Освобожденный**. После завершения настройки система не будет создавать проводки ГК для налога. Однако для налоговых кодов должна быть указана соответствующая налоговая ставка, чтобы гарантировать, что в напечатанной форме повторно выданной накладной будет отражен налог.

1. Перейдите в группы **Налог** > **Налог** > **Налог**.
2. Создайте налоговую группу.
3. На экспресс- вкладке **Настройка** в поле **Налоговый код** выберите налоговые коды для всех ставок налога на добавленную стоимость (НДС).
4. Для каждого налогового кода установите флажок **Освобожденный**, чтобы показать, что для налоговый код не рассчитывается.
5. Нажмите **Сохранить**.

[![Налоговые группы с кодами освобождения от налога.](./media/3-sales-tax-groups.jpg)](./media/3-sales-tax-groups.jpg)
 
### <a name="define-a-transit-tax-group-for-a-sales-tax-group"></a>Определение транзитной налоговой группы для налоговой группы
Эта транзитная налоговая группа используется в накладной, которая создается при проводке по перевыставлению задолженности.

1. Перейдите в группы **Налог** > **Налог** > **Налоговые группы**.
2. Создайте налоговую группу.
3. На экспресс-вкладке **Общее** в поле **Транзитная налоговая группа** выберите налоговую группу, содержащую освобождаемый налоговый код, который используется для перевыставления расходов.
4. На экспресс-вкладке **Настройка** в поле **Налоговая группа** выберите налоговый код.
5. Нажмите **Сохранить**.

[![Налоговые группы с транзитным налогом.](./media/4-sales-tax-groups.jpg)](./media/4-sales-tax-groups.jpg)
  
### <a name="set-up-a-transit-account"></a>Настройка транзитного счета
Транзитный счет главной книги — это счет, который будет кредитован при перевыставлении проводок третьим сторонам. Необходимо настроить счет ГК для перевыставления расходов третьим сторонам.

1. Перейдите в раздел **Расчеты с поставщиками** > **Настройка** > **Параметры расчетов с поставщиками**.
2. На вкладке **Общее** в поле **Транзитный счет** выберите транзитный счет ГК, который будет использоваться для проводки по перевыставлению задолженности.
3. На вкладке **Номерная серия** в поле **Код номерной серии** выберите код номерной серии для **Ваучер расходов**, **Проводка распределения**, **Внешняя накладная поставщика** и **Счет на оплату**.

## <a name="purchase-services-charges-for-allocation"></a>Покупка услуг (расходы) для распределения
Можно купить услуги для распределения либо из журнала накладных, либо через заказ на покупку.

### <a name="purchase-services-charges-from-the-invoice-journal"></a>Покупка услуг (расходы) из журнала накладных

1. Перейдите в раздел **Расчеты с поставщиками** > **Накладные** > **Журнал накладных**.
2. Создайте журнал накладных.
3. Выберите **Строки** и создайте строку.
4. На вкладке **Общее** выполните следующие действия:

      a. В разделе **Налог** выберите налоговую группу продаж и налоговую группу номенклатур.

    > [!NOTE]
      > При повторном выставлении задолженности третьим лицам, возможно, потребуется выбрать налоговую группу с кодами освобождения от налога.

      b. В разделе **Расходы** в поле **Код расходов** выберите код расходов, созданный в разделе [Настройка кода расходов](#set-up-a-charges-code).
  
  [![Страница журнала накладной поставщика.](./media/5-vendor-invoice-journal.jpg)](./media/5-vendor-invoice-journal.jpg)
 
5. Укажите другие сведения, а затем выполните разноску журнала.

### <a name="purchase-services-charges-through-a-purchase-order"></a>Покупка услуг (расходы) через заказу на покупку

1. Создание заказа на покупку.
2. В строках заказа на покупку выберите услугу, созданную ранее.
3. На вкладке **Настройка** выполните следующие действия:

      a. В разделе **Налог** выберите налоговую группу номенклатур и созданную ранее налоговую группу.
  
      > [!NOTE]
      >  При повторном выставлении задолженности третьим лицам, возможно, потребуется выбрать налоговую группу с кодами освобождения от налога продажи.

      b. В разделе **Расходы** в поле **Код накладных** расходов проверьте, что выбран код расходов, созданный ранее.
    
      c. В разделе **Разноска** в поле **Счет ГК** убедитесь, что выбран дебетовый счет, указанный для кода расходов, который был создан ранее.
  
  [![Страница "Все заказы на покупку", вкладка "Настройка".](./media/6-all-purchase-orders.jpg)](./media/6-all-purchase-orders.jpg)
 
4. На вкладке **Продукт** в поле **Профиль учета** выберите профиль учета.

[![Страница "Все заказы на покупку", вкладка "Продукт".](./media/7-all-purchase-orders.jpg)](./media/7-all-purchase-orders.jpg)

5. Укажите другие сведения заказа на покупку и разнесите накладную.

## <a name="allocate-charges"></a>Распределить накладные расходы

Можно распределить накладные расходы на странице **Распределение расходов**. Эта страница состоит из трех частей:

  - В разделе **Источник распределения** отображаются строки накладной, которые будут распределены.
  - В разделе **База распределения** можно выбрать строки накладной, по которым будет распределяться накладные расходы из раздела **Источник распределения**.
  - В нижней правой части показан результат распределения сумм по исходным строкам в базовой строке.

### <a name="allocate-miscellaneous-charges-to-the-item-cost-price"></a>Распределение накладных расходов по себестоимости номенклатуры
1. Перейдите в раздел **Расчеты с поставщиками** > **Запросы и отчеты** > **Накладная** > **Журнал накладных**.
2. Выберите накладную для созданного ранее заказа на покупку, включающего эту услугу. Затем выберите **Расходы** > **Распределение**.

    [![Страница "Журнал накладных", раскрывающийся список "Расходы".](./media/8-invoice-journal.jpg)](./media/8-invoice-journal.jpg)

3. На странице **Распределение расходов** выберите **Правка**.

    [![Страница "Распределение расходов", "Правка".](./media/9-charges-allocation.jpg)](./media/9-charges-allocation.jpg)

4. В поле **Распределение расходов** выберите один из следующих методов для распределения накладных расходов:

      - **Чистая сумма** — расходы распределяются в соответствии с суммой каждой строки по отношению к общей чистой сумме накладной на услуги.
      - **Количество** — расходы распределяются в соответствии с числом единиц измерения для каждой строки по отношению к общему числу единиц измерения в накладной на услуги.
      - **Вес брутто** — расходы распределяются поровну в соответствии с весом брутто полученных номенклатур.
      - **Объем** — расходы распределяются поровну в соответствии с объемом полученных номенклатур.

5. В поле **Дата разноски** выберите дату разноски.
6. В поле **Режим распределения** выберите один из трех режимов распределения накладных расходов:

      - Включить в себестоимость
      - Перевыставить задолженность
      - Списание на затраты

    Чтобы распределить расходы по себестоимости продуктов, выберите **Включить в себестоимость**.

7. В разделе **База распределения** выберите **Выбрать**, чтобы указать критерии выбора, которые используются для переноса строк накладной в базу распределения.
8. В диалоговом окне **Выбор** выберите **Добавить**, чтобы добавить строку, и введите следующие сведения строки:

      - В поле **Таблица** выберите значение.
      - В поле **Поле** выберите поле, для которого задаются критерии. В поле **Критерии** введите критерии.

9. Нажмите **ОК**. Создаются базовые строки распределения.
10. В разделе **Источник распределения** установите флажок **Выбрать** для разнесенной накладной по накладным расходам. В поле **Доступно для распределения** можно изменить сумму, доступную для распределения.
11. В разделе **База распределения** можно установить флажок **Выбрать** для строк накладной, по которым будет распределяться накладные расходы.
12. Выберите **Распределение** для распределения накладных расходов. Суммы в выбранных базовых строках распределяются по всем выбранным базовым строкам. Результаты распределения можно просмотреть в нижней правой части страницы. Для каждой строки из базы распределения отображается сумма распределения и код накладных расходов.

    > [!NOTE]
    >
    > - Метод распределения можно изменить в поле **Метод распределения** по мере необходимости. После этого будет выведено сообщение, предлагающее пересчитать строки результата распределения. Нажмите **Да**, чтобы пересчитать результат.
    > - Можно вручную скорректировать результат распределения в поле **Сумма** в правой нижней части страницы. Если введенное значение превышает общую сумму, доступную для распределения, будет выведено информационное сообщение.

13. Выберите **Разнести**, чтобы разнести проводку распределения.
14. Выбор **Запросы** > **Ваучер** для просмотра проводок, созданных в связи с распределением.

### <a name="allocate-miscellaneous-charges-to-redraw-debt-to-third-parties"></a>Распределение накладных расходов для перевыставления задолженности третьим сторонам

1. На странице **Журнал накладных** выберите накладную для созданного ранее заказа на покупку, включающего эту услугу. Затем выберите **Расходы** > **Распределение**.

    [![Страница распределения расходов для распределения накладных расходов.](./media/10-charges-allocation.jpg)](./media/10-charges-allocation.jpg)
 
2. В поле **Распределение расходов** выберите метод для распределения накладных расходов.
3. В поле **Дата разноски** выберите дату разноски.
4. В поле **Режим распределения** выберите **Перевыставить задолженность**.
5. В разделе **База распределения** укажите, могут ли клиенты или поставщики перевыставлять задолженность автоматически или вручную:

      a. **Автоматически**: выберите **Выбрать**, укажите критерии выбора, а затем нажмите **ОК**. Базовые строки распределения создаются автоматически.

      > [!NOTE]
      > Базовые строки распределения группируются по полям **Тип контрагента**, **Код договора** и **Профиль разноски**. Если имеются накладные для одного контрагента, но значения этих параметров различаются, для этого контрагента будет использоваться несколько строк в базе распределения.
  
      b. **Вручную**: чтобы вручную создать базовые строки распределения, выберите **Создать**. В поле **Тип контрагента** выберите **Клиент** или **Поставщик**. В поле **Номер счета** выберите номер счета контрагента. Можно также выбрать значения в полях **Код соглашения** и **Профиль разноски**.
  
6. Выберите исходные и базовые строки для распределения, а затем измените поле **К распределению** по мере необходимости.
7. Выберите **Распределение** для распределения накладных расходов. Суммы в выбранных базовых строках распределяются по всем выбранным базовым строкам. Результаты распределения можно просмотреть в нижней правой части страницы. Для каждой строки из базы распределения отображается сумма распределения и код накладных расходов.

    > [!NOTE]
    >
    > - Для вручную созданных базовых строк распределения результат распределения равен 0 (нулю). В этом случае необходимо вручную ввести суммы накладных расходов в поле **Суммы** в правой нижней части страницы. Также можно изменить суммы для автоматически создаваемых базовых строк распределения. Если введенное значение превышает общую сумму, доступную для распределения, будет выведено информационное сообщение.
    > - Метод распределения можно изменить в поле **Распределение накладных расходов** по мере необходимости. Для автоматически создаваемых базовых строк распределения суммы можно пересчитать. Для вручную созданных базовых строк распределения введенные суммы будут равны 0 (нулю).

8. Выберите **Разнести**, чтобы разнести проводку распределения.
9. В диалоговом окне **Разноска** выполните следующие действия:

      a. Установите для параметра **Корректировка по кредиту** значение **Да**, чтобы разнести распределение как проводку сторно. В противном случае распределение будет создано как реверсирующая проводка.
    
      b. Установите параметр **Создать счет на оплату** как **Да**, чтобы создать счет для оплаты при разноске распределения.
    
      c. Нажмите **ОК**. Выполняется операция повторного распределения задолженности по контрагентам. Проводки будут созданы для всех выбранных строк в результате распределения.

   Из-за разноски происходит одно из следующих событий, в зависимости от того, была ли накладная, включающая услугу для распределения, создана из журнала накладных поставщика или из заказа на покупку:

  - Если накладная была создана из журнала накладных поставщика:
    
    - Если перевыставляется задолженность по поставщикам, журналы накладных создаются и разносятся автоматически. Если параметр **Создать счет на оплату** задан как **Да**, счет для оплаты будет автоматически разнесен.
        
    - При повторном выставлении задолженности для клиентов автоматически создаются накладные с произвольным текстом. Накладные для заказов на продажу и платежей разносятся автоматически.

- Если накладная была создана в заказе на покупку:
    
    - Если идет перевыпуск задолженности по поставщикам, заказы на покупку, по которым будут разноситься накладные, создаются автоматически. Если параметр **Создать счет на оплату** задан как **Да**, счет для оплаты будет автоматически разнесен.
        
    - При повторном выставлении задолженности для клиентов автоматически создаются заказы на продажу. Накладные для заказов на продажу и платежей разносятся автоматически.

10. Выбор **Запросы** > **Ваучер** для просмотра проводок, созданных в связи с распределением.

18. Для просмотра сведений о распределении задолженности на странице **Журнал накладных** выберите **Расходы > История распределения накладных расходов**. Затем в поле **Режим распределения** выберите **Перевыставить задолженность**.

### <a name="allocate-miscellaneous-charges-to-the-ledger-account"></a>Распределение накладных расходов на счет ГК

1. На странице **Журнал накладных** выберите накладную для созданного ранее заказа на покупку, включающего эту услугу. Затем выберите **Расходы** > **Распределение**.

    [![Страница распределения расходов для распределения накладных расходов на счет книги учета.](./media/16-charges-allocation.jpg)](./media/16-charges-allocation.jpg)

2. В поле **Распределение расходов** выберите метод для распределения накладных расходов.
3. В поле **Дата разноски** выберите дату разноски.
4. В поле **Режим распределения** выберите **Списание на затраты**.
5. В разделе **База распределения** укажите счета ГК, в которые будут списываться накладные расходы.
6. Выберите **Создать**, чтобы создать базовую строку распределения.
7. В поле **Счет ГК** выберите счет ГК.
8. Выберите исходные и базовые строки для распределения, а затем измените поле **К распределению** по мере необходимости.
9. Выберите **Распределение** для распределения накладных расходов. Система создает строки, в которых сумма равна 0 (нулю). Для выбранной базы распределения в поле **Суммы** в правой нижней части страницы следует ввести сумму накладных расходов.

    > [!NOTE]
    > Если введенное значение превышает общую сумму, доступную для распределения, будет выведено информационное сообщение.

10.    Выберите **Разнести**, чтобы разнести проводку распределения. Выполняется операция списания накладных расходов на счет ГК. Проводки создаются для всех выбранных строк в результате распределения.
11.    Выбор **Запросы** > **Ваучер** для просмотра проводок, созданных в связи с распределением.

## <a name="view-the-miscellaneous-charges-that-are-allocated-to-the-items-cost-price"></a>Просмотр накладных расходов, распределенных по себестоимости номенклатуры
Себестоимость номенклатуры может быть скорректирована несколько раз с помощью функций распределения накладных расходов. Можно просмотреть проводки, которые корректируют затраты номенклатуры на страницах **Сопоставления** и **Анализатор себестоимости**. Сначала необходимо выполнить операцию закрытия склада.
1. Выберите **Управление сведениями о продукте** > **Продукты** > **Выпущенные продукты**, чтобы открыть страницу **Сведения о выпущенном продукте**.
2. Выберите номенклатуру, затем, в области действий, на вкладке **Управление затратами**, в группе **Проводки затрат**, выберите **Проводки**, чтобы открыть страницу **Складские проводки**.
3. Выберите проводку для полученной номенклатуры, для которой необходимо просмотреть структуру затрат.
4. Чтобы просмотреть проводки корректировки себестоимости запасов, в области действий на вкладке **Запасы** в группе **Расчет себестоимости** выберите **Сопоставления**. Страница **Сопоставления** содержит следующие поля:

      - **Код расходов** — код накладных расходов, который был распределен для проводки прихода номенклатуры.
      - **Счет поставщика** — счет поставщика, у которого накладные расходы были приобретены.
      - **Накладная** — номер накладной по закупке накладных расходов.
  
      [![Страница сопоставлений.](./media/17-settlements.jpg)](./media/17-settlements.jpg)
  
      > [!NOTE]
      > Если поля счета поставщика и накладной не заполнены, операция корректировки затрат была выполнена стандартным методом распределения накладных расходов.
  
5. Чтобы просмотреть дополнительные сведения о списке и суммах накладных расходов, которые были включены в себестоимость разнесенной номенклатуры, на странице **Складские проводки** в области действий на вкладке **Запасы** в группе **Учет затрат** выберите **Анализатор себестоимости**.

    [![Страница анализатора себестоимости.](./media/18-cost-explorer.jpg)](./media/18-cost-explorer.jpg)
 
## <a name="view-the-allocation-history-for-miscellaneous-charges"></a>Просмотр истории распределения для накладных расходов
### <a name="view-the-allocation-history-from-the-invoice-journal"></a>Просмотр истории распределения из журнала накладных

1. На странице **Журнал накладных** выберите созданную ранее накладную по покупке, включающую эту услугу.
2. Выберите **Расходы** > **История распределения накладных расходов**, чтобы открыть страницу **Распределение накладных расходов**.
В разделе **База распределения** отображается база распределения и результат распределения, в зависимости от значения в поле **Режим распределения**. Чтобы изменить режим распределения, выберите **Правка**, а затем в поле **Режим распределения** выберите новое значение.

    [![Страница распределение расходов, раздел базы распределения.](./media/19-charges-allocation.jpg)](./media/19-charges-allocation.jpg)

3. В заголовке страницы **Распределение накладных расходов** установите для параметра **Показать отмененные операции** значение **Да**, чтобы отобразить проводки распределения накладных расходов, которые были отменены, и проводки, которые корректируют их.
4. Выберите **Запросы** > **Ваучер** для просмотра проводок, созданных в связи с распределением.

### <a name="view-the-allocation-history"></a>Просмотр истории распределения
История распределения может отображаться либо для конкретной накладной, либо для всех накладных, для которых была выполнена операция распределения накладных расходов.

1. Перейдите **Расчеты с поставщиками** > **Запросы и отчеты** > **История распределения накладных расходов**.
2. Выберите **Правка**, а затем в полях **Дата начала** и **Дата окончания** период для фильтрации накладных для покупки по накладным расходам.

    В разделе **Источник распределения** показаны исходные строки для всех накладных за данный период. В разделе **База распределения** отображается база распределения и результат распределения.

    [![Страница распределение расходов, раздел источника распределения.](./media/20-charges-allocation.jpg)](./media/20-charges-allocation.jpg)
 
## <a name="cancel-miscellaneous-charges-allocation"></a>Отмена распределения накладных расходов
1. Перейдите **Расчеты с поставщиками** > **Запросы и отчеты** > **История распределения накладных расходов**. В качестве альтернативы на странице **Журнал накладных** выберите **Расходы** > **История распределения накладных расходов**.
2. В поле **Режим распределения** выберите режим распределения, для которого нужно отменить распределение.
3. Выберите строки для отмены.

    > [!NOTE]
    > Строки, созданные одной операцией распределения, не могут быть отменены частично. При выборе одной строки из операции распределения автоматически выбираются все строки из этой операции.

4. В поле **Дата отмены** введите дату отмены.

    [![Страница распределение расходов, поле даты отмены.](./media/21-charges-allocation.jpg)](./media/21-charges-allocation.jpg)

5. Выберите **Отмена**.

    > [!NOTE]
    > 
    > - Если дата отмены предшествует дате по крайней мере одной проводки, помеченной для отмены, операция не будет выполнена, и будет выведено сообщение об ошибке.
    > - Если перевыписка задолженности отменяется, будет получено информационное сообщение, содержащее ссылки на документы, созданные для клиентов или поставщиков.

6.    Выберите **Запросы** > **Ваучер** для просмотра проводок, созданных в связи с отменой.
