# <a name="russia-advance-holders-overview"></a>Обзор подотчетных лиц России
[!include [banner](../includes/banner.md)]

## <a name="set-up-parameters-for-advance-holders"></a>Настройка параметров для подотчетных лиц

1. Выберите **Расчеты с поставщиками** \> **Настройка** \> **Параметры расчетов с поставщиками**.
2. На вкладке **Подотчетные лица** в поле **Профиль разноски** выберите профиль разноски, который должен использоваться для выполнения проводок по подотчетным лицам по умолчанию.

    > [!NOTE]
    > Дополнительные сведения см. в разделе "Настройка профилей разноски для учетных ваучеров" далее в данной теме.

3. Установите для параметра **Подотчетное лицо по умолчанию** значение **Да**, чтобы установить текущего пользователя в качестве подотчетного лица.
4. Установите для параметра **Сортировка подотчетных лиц** значение **Да**, чтобы подотчетные лица отображались вверху сетки на странице **Подотчетные лица**.
5. Установите для параметра **Выдача под отчет при открытом сальдо** значение **Да**, чтобы разрешить выдачу денежных авансов подотчетным лицам с открытым положительным сальдо.
6. Установите для параметра **Автоматическое сопоставление** значение **Да**, если требуется автоматическое сопоставление проводок.
7. Установите для параметра **Сопоставление по профилям** значение **Да**, чтобы сопоставлялись только проводки с идентичным профилем разноски.
8. На экспресс-вкладке **Закрытие сальдо** в области **Закрытие сальдо через кассу** в поле **Имя** выберите код журнала кассовых ордеров. Этот код журнала используется для создания расходных кассовых ордеров и приходных ордеров при закрытии сальдо подотчетного лица по кассе.

    > [!NOTE]
    > Все журналы имеют тип **Денежные средства**.

9. В поле **Денежные средства** выберите кассовый счет для определения ваучеров, которые используются для закрытия сальдо подотчетного лица.
10. В области **Закрытие сальдо через банк** в поле **Имя** выберите код журнала для проводок для закрытия сальдо через банк.
11. В поле **Тип счета** выберите **Банк** для закрытия сальдо подотчетного лица через банк.
12. В поле **Счет ГК** выберите код банковского счета для закрытия сальдо подотчетного лица через банк.
13. На вкладке **Номерные серии** определите коды номерных серий для следующих ссылок:

    - Авансовый отчет
    - Ваучер ГК по АО
    - Ваучер сопоставления
    - Ваучер курсовой разницы (Подотчетные лица)

14. Выберите **Сохранить** или закройте страницу.

## <a name="set-up-an-advance-holder-group"></a>Настройка группы подотчетных лиц.

Необходимо создать группу подотчетных лиц, чтобы сгруппировать подотчетные лица. Например, можно сгруппировать подотчетные лица по ваучерам, рассчитываемым в одном счете.

1. Выберите **Расчеты с поставщиками** \> **Настройка** \> **Подотчетные лица** \> **Группы подотчетных лиц**.
2. Выберите **Создать**, чтобы создать новую группу подотчетных лиц, или **Редактировать** для корректировки существующей группы.
3. В поле **Группа** введите код для новой группы подотчетных лиц.
4. В поле **Описание** введите описание группы подотчетных лиц.
5. В поле **Корр. счет** выберите корреспондентский счет ГК, используемый для группы подотчетных лиц.
6. Выберите **Сохранить** или закройте страницу.

## <a name="set-up-posting-profiles-for-accounting-vouchers"></a>Настройка профилей разноски для учетных ваучеров

Для счета операций можно установить следующие профили разноски:

- Один счет ГК для всех подотчетных лиц
- Отдельный счет ГК для каждой группы подотчетных лиц
- Отдельный счет ГК для каждого подотчетного лица

1. Выберите **Расчеты с поставщиками** \> **Настройка** \> **Подотчетные лица** \> **Профили разноски по подотч. лицам**.
2. Выберите **Создать**, чтобы создать новый профиль разноски, или **Редактировать** для корректировки существующего профиля.
3. В поле **Профиль разноски** введите идентификатор обрабатываемого профиля разноски.
4. В поле **Описание** введите описание профиля разноски.
5. На экспресс-вкладке **Табличные ограничения** в поле **Разрешить пустое значение аналитики** выберите условие, при котором сопоставления проводок, связанных с начислениями и операциями закрывающих платежей, могут обрабатываться, если не указаны значения аналитик. Имеются следующие варианты:

    - **Нет** — проводки не сопоставляются независимо от того, указаны значения аналитик или нет.
    - **Авто** — автоматические проводки сопоставляются независимо от того, указаны значения аналитик или нет.
    - **Вручную** — сделанные вручную проводки сопоставляются независимо от того, указаны значения аналитик или нет.
    - **Всегда** — все проводки сопоставляются независимо от того, указаны значения аналитик или нет.

    > [!NOTE]
    > Это поле позволяет игнорировать значения аналитик при создании отдельных проводок.

6. На экспресс-вкладке **Настройка** выберите **Добавить**, чтобы создать новую запись для данных профиля разноски.
7. В поле **Действительно для** выберите уровень группирования для настройки профиля разноски. Имеются следующие варианты:

    - **Таблица** – этот вариант устанавливается только для одного подотчетного лица.
    - **Группа** – этот вариант устанавливается для группы подотчетных лиц.
    - **Все** – этот вариант устанавливается для всех подотчетных лиц.

8. В поле **Ссылка** выберите соответствующую ссылку.

    > [!NOTE]
    > Значение в этом поле можно выбрать, только если выбран вариант **Таблица** или **Группа** в поле **Действительно для**.

9. В поле **Суммирующий счет** выберите счет для отображения проводок по подотчетным лицам.
10. В поле **Задать** выберите фокус аналитик для управления сопоставлением.

    > [!NOTE]
    > Определенный набор аналитик показывает, что контроль аналитики активирован.

11. Выберите **Сохранить** или закройте страницу.

## <a name="set-up-the-terms-of-payment"></a>Настройка условий оплаты

Необходимо установить условия оплаты при регистрации покупки товаров у поставщика через подотчетное лицо. Закупку выполняет подотчетное лицо.

1. Выберите **Расчеты с поставщиками** \> **Настройка платежей** \> **Условия оплаты**.
2. Выберите **Создать**, чтобы создать новую запись условий оплаты, или **Редактировать** для корректировки существующей записи.
3. В поле **Условия оплаты** введите код для условий оплаты.
4. В поле **Описание** введите описание для условий оплаты.
5. На экспресс-вкладке **Настройка** в поле **Способ оплаты** выберите **Наложенный платеж**.
6. Установите для параметра **Наличная оплата** значение **Да**.
7. Установите для параметра **От подотчетного лица** значение **Да**.
8. Выберите **Сохранить** или закройте страницу.

## <a name="set-up-the-per-diem-expense-rates"></a>Настройка ставок суточной оплаты расходов

Ставки расходов используются для создания авансового отчета по суточным расходам. Они также могут использоваться для автоматического расчета стандартных сумм и сумм перерасхода.

1. Выберите **Расчеты с поставщиками** \> **Настройка** \> **Подотчетные лица** \> **Ставки расходов**.
2. Выберите **Создать**, чтобы создать новый расход, или **Редактировать** для корректировки существующего расхода.
3. В поле **Расход** введите код расхода.
4. В поле **Описание** введите описание расхода.
5. В поле **Ставка** введите стандартное установленное значение для данного расхода.
6. В поле **Валюта** выберите код валюты, в котором регистрируются операции для данного расхода.
7. На экспресс-вкладке **Настройка** в области **Норма** в поле **Счет ГК** выберите бухгалтерский отчет для отражения обычных расходов.
8. В области **Сверх нормы** в поле **Счет ГК** выберите бухгалтерский отчет для отражения расходов сверх нормы.
9. В области **Налог** и **Налоги сверх нормы** в поле **Налоговая группа** выберите налоговую группу для расчета налога для продаж и покупок на суммах в пределах нормы и сверх нормы.
10. В области **Налог** и **Налоги сверх нормы** в поле **Налоговая группа номенклатур** выберите налоговую группу номенклатур для расчета налога для номенклатуры на суммах в пределах нормы и сверх нормы.
11. Если суммы включают налог, в областях **Налог** и **Налоги сверх нормы** установите для параметра **Суммы включают налог** значение **Да**.
12. В разделе **Налоги сверх нормы** в поле **Разноска сверх нормы** выберите операцию, которая должна использоваться для разноски расходов сверх нормы.
13. На экспресс-вкладке **Финансовые аналитики** в областях **Финансовые аналитики** и **Финансовые аналитики сверх нормы** выберите значения аналитик в проводке для связанных аналитик.
14. Выберите **Сохранить** или закройте страницу.

## <a name="worker-table-numbers-for-advance-holders"></a>Табельные номера работников для подотчетных лиц

Работник может являться сотрудником или подрядчиком и занимать в организации несколько должностей. Сотрудник или подрядчик также может являться подотчетным лицом, отвечающим за сумму расходов, предоставляемую организацией. Чтобы задать работника в качестве подотчетного лица, введите уникальный код работника в поле **Код работника** на странице **Работник**. Также необходимо установить для параметра **Подотчетное лицо** значение **Да** на странице **Подотчетные лица**, чтобы указать, что выбранный работник является подотчетным лицом.

Разноска проводок по таким работникам может выполняться при помощи счетов подотчетных лиц. Указанный для каждого подотчетного лица идентификатор работника можно использовать для отслеживания всех проводок подотчетного лица. Этот номер используется в качестве номера счета для проводок по подотчетному лицу на страницах **Общие журналы** и **Проводки по подотч. лицам**.
