# <a name="tax-feature-support-for-transfer-orders"></a>Поддержка функций налогов для заказов на перемещение

[!include [banner](../../includes/banner.md)]

В этой теме приводятся сведения об интеграции расчета налога и разноски в заказах на перемещение. Эта функция позволяет настраивать расчет налогов и разноску в заказах на перемещение для перемещения запасов. В положениях по налогу на добавленную стоимость (НДС) в Европейском Союзе (ЕС) перемещение запасов считается поставками внутри сообщества и приобретением внутри сообщества.

Для настройки и использования этой функции необходимо выполнить три основных шага:

1. **Настройка RCS:** в службе Regulatory Configuration Services настройте функцию налогов, налоговые коды и применимость налоговых кодов для определения налоговых кодов в заказах на перемещение.
2. **Настройка Finance:** в Microsoft Dynamics 365 Finance включите функцию **Налог в заказах на перемещение**, настройте параметры налоговой службы для запасов и настройте основные параметры налога.
3. **Настройка запасов:** настройка конфигурации запасов для проводок по заказам на перемещение.

## <a name="set-up-rcs-for-tax-and-transfer-order-transactions"></a>Настройка RCS для проводок налогов и заказов на перемещение

Выполните следующие действия, чтобы настроить налог, который участвует в заказе на перемещение. В показанном здесь примере используется заказ на перемещение из Нидерландов в Бельгию.

1. На странице **Налоговые функции** на вкладке **Версии** выберите черновую версию функции, затем выберите **Правка**.

    ![Выбор правки](../media/tax-feature-support-01.png)

2. На странице **Настройка налоговых функций** на вкладке **Налоговые коды** выберите **Добавить** для создания новых налоговых кодов. В данном примере создаются три налоговых кода: **NL-Exempt**, **BE-RC-21** и **BE-RC+21**.

    - Когда заказ на перемещение отгружается со склада в Нидерландах, применяется налоговый код, освобожденный от уплаты НДС (**NL-Exempt**).
      
        Создайте код налога **NL-Exempt**.
        1. Выберите **Добавить**, введите **NL-Exempt** в поле **Налоговый код**.
        2. Выберите **По чистой сумме** в поле **Налоговый компонент**.
        3. Нажмите **Сохранить**.
        4. Выберите **Добавить** в таблице **Ставка**.
        5. Переключите **Не облагается** в значение **Да** в разделе **Общие**.

        ![Код налога NL-Exempt](../media/tax-feature-support-02.png)

    - Когда заказ на перемещение поступает на склад в Бельгии, механизм возмещения расходов применяется с использованием налоговых кодов **BE-RC-21** и **BE-RC+21**.
        
        Создайте налоговый код **BE-RC-21**.      
        1. Выберите **Добавить**, введите **BE-RC-21** в поле **Налоговый код**.
        2. Выберите **По чистой сумме** в поле **Налоговый компонент**.
        3. Нажмите **Сохранить**.
        4. Выберите **Добавить** в таблице **Ставка**.
        5. Введите **-21** в поле **Ставка налога**.
        6. Переключите **Является возмещением расходов** в значение **Да** в разделе **Общие**.
        7. Нажмите **Сохранить**.

        ![Налоговый код BE-RC-21 для возмещения расходов](../media/tax-feature-support-03.png)
        
        Создайте налоговый код **BE-RC+21**.
        1. Выберите **Добавить**, введите **BE-RC-21** в поле **Налоговый код**.
        2. Выберите **По чистой сумме** в поле **Налоговый компонент**.
        3. Нажмите **Сохранить**.
        4. Выберите **Добавить** в таблице **Ставка**.
        5. Введите **21** в поле **Ставка налога**.
        6. Нажмите **Сохранить**.

        ![Налоговый код BE-RC+21 для возмещения расходов](../media/tax-feature-support-04.png)

3. Определите применимость налоговых кодов.

    1. Выберите **Управление столбцами**, затем выберите столбцы, которые следует использовать для построения таблицы применимости.

        > [!NOTE]
        > Обязательно добавьте столбцы **Бизнес-процесс** и **Направления налогов** в таблицу. Оба столбца являются важными для функций налогов в заказах на перемещение.

    2. Добавьте правила применимости. Не оставляйте поля **Налоговые коды**, **Налоговая группа** и **Налоговая группа номенклатуры**.
        
        Добавьте новое правило для отгрузки заказа на перемещение.
        1. Выберите **Добавить** в таблице **Правила применимости**.
        2. В поле **Бизнес-процесс** выберите **Запасы**, чтобы сделать правило применимым для заказа на перемещение.
        3. В поле **Поставка из страны/региона** введите **NLD**.
        4. В поле **Поставка в страну/регион** введите **BEL**.
        5. В поле **Направление налога** выберите **Вывод**, чтобы применить правило к отгрузке заказов на перемещение.
        6. В поле **Налоговые коды** выберите **NL-Exempt**.
        7. В полях **Налоговая группа** и **Налоговая группа номенклатур** введите соответствующую налоговую группу и налоговую группу номенклатур, которые определены в системе Finance.
        
        Добавьте другое правило для приемки заказа на перемещение.
        1. Выберите **Добавить** в таблице **Правила применимости**.
        2. В поле **Бизнес-процесс** выберите **Запасы**, чтобы сделать правило применимым для заказа на перемещение.
        3. В поле **Поставка из страны/региона** введите **NLD**.
        4. В поле **Поставка в страну/регион** введите **BEL**.
        5. В поле **Направление налога** выберите **Ввод**, чтобы применить правило к поступлению заказов на перемещение.
        6. В поле **Налоговые коды** выберите **BE-RC+21** и **BE-RC-21**.
        7. В полях **Налоговая группа** и **Налоговая группа номенклатур** введите соответствующую налоговую группу и налоговую группу номенклатур, которые определены в системе Finance.

        ![Правила применимости](../media/image5.png)

4. Выполните и опубликуйте новую версию функции налога.

    [![Изменение статуса новой версии](../media/image6.png)](../media/image6.png)

## <a name="set-up-finance-for-transfer-order-transactions"></a>Настройка Finance для проводок заказов на перемещение

Выполните следующие действия, чтобы включить и настроить налоги для заказов на перемещение.

1. В Finance перейдите в раздел **Рабочие области** \> **Управление функциями**.
2. В списке найдите и выберите функцию **Налог в заказе на перемещение**, затем выберите **Включить**, чтобы включить ее.

    > [!IMPORTANT]
    > Функция **Налог в заказах на перемещение** полностью зависит от налоговой службы. Таким образом, она может быть включена только после установки налоговой службы.

    ![Функция налога в заказах на перемещение](../media/image7.png)

3. Включите налоговую услугу и выберите бизнес-процесс **Запасы**.

    > [!IMPORTANT]
    > Необходимо выполнить этот шаг для каждого юридического лица в Finance, в котором вы хотите получить доступ к налоговой службе и функции налога в заказах на перемещение.

    1. Перейдите в раздел **Налог** \> **Настройка** \> **Конфигурация налога** \> **Настройка налоговой службы**.
    2. В поле **Бизнес-процесс** выберите **Запасы**.

    ![Задание поля бизнес-процесса](../media/image8.png)

4. Убедитесь, что механизм возмещения расходов настроен. Перейдите в раздел **Главная книга** \> **Настройка** \> **Параметры**, а затем на вкладке **Возмещение расходов** убедитесь, что для параметра **Включить возмещение расходов** установлено значение **Да**.

    ![Включение параметра возмещения расходов](../media/image9.png)

5. Убедитесь, что соответствующие налоговые коды, налоговые группы, налоговые группы номенклатур и регистрационные номера НДС были настроены в Finance в соответствии с указаниями налоговой службы.
6. Настройте промежуточный транзитный счет. Этот шаг требуется только в том случае, если налог, применяемый к заказу на перемещение, не применяется к механизму налогового освобождения или возмещения расходов.

    1. Перейдите в раздел **Налог** \> **Настройка** \> **Налог** \> **Группы разноски ГК**.
    2. В поле **Промежуточный транзитный** выберите счет ГК.

    ![Выбор промежуточного транзитного счета](../media/image10.png)

## <a name="set-up-basic-inventory-for-transfer-order-transactions"></a>Настройка базовых запасов для проводок заказов на перемещение

Выполните следующие действия, чтобы настроить основные запасы, чтобы включить проводки заказов на перемещение.

1. Создайте сайты отгрузки и получения для складов в разных странах или регионах и добавьте основной адрес для каждого сайта.

    1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Склад** \> **Сайты**.
    2. Выберите **Создать**, чтобы создать сайт, который будет назначен складу позже.
    3. Повторите шаг 2 для всех других сайтов, которые необходимо создать.

    > [!NOTE]
    > Один из созданных сайтов должен иметь имя **Транзитный**. В последующих шагах этой процедуры этот сайт будет назначен транзитному складу, так что ваучеры запасов, связанные с налогами, могут быть разнесены в проводках "отгрузка" и "получение" для заказов на перемещение. Адрес транзитного сайта не имеет отношения к расчету налога. Поэтому поле может быть пустым.

    ![Настройка сайтов](../media/image11.png)

2. Создайте склад отгрузки, транзитный склад и склад получения. Любая адресная информация, которая хранится на складе, будет переопределять адрес сайта во время расчета налога.

    1. Перейдите в раздел **Управление складом** \> **Настройка** \> **Склад** \> **Склады**.
    2. Выберите **Создать**, чтобы создать склад, и назначьте его соответствующему складу.
    3. Повторите шаг 2 для создания склада для каждого сайта, как это необходимо.

    ![Настройка складов](../media/image12.png)

    > [!NOTE]
    > Для склада отгрузки в поле **Транзитный склад** для проводок заказов на перемещение должен быть выбран транзитный склад.
    >
    > ![Выбор транзитного склада](../media/image13.png)

3. Убедитесь, что конфигурация разноски запасов настроена для проводок заказов на перемещение.

    1. Выберите **Управление запасами** \> **Настройка** \> **Разноска** \> **Разноска**.
    2. На вкладке **Запасы** убедитесь, что счет ГК настроен для разноски как **Расхода запасов**, так и **Поступления запасов**.

        ![Настройка разноски расхода запасов и поступления запасов](../media/image14.png)

    3. Убедитесь, что счет ГК настроен для разноски **Внутрихолдинговые расчеты с поставщиками**.

        ![Настройка разноски внутрихолдинговых расчетов с поставщиками](../media/image15.png)

    4. Убедитесь, что счет ГК настроен для разноски **Внутрихолдинговые расчеты с клиентами**.

        ![Настройка разноски внутрихолдинговых расчетов с клиентами](../media/image16.png)
