# <a name="deferrals-register"></a>Регистр расходов будущих периодов

[!include [banner](../includes/banner.md)]

Отдельная функция расходов будущих периодов используются для учета отложенных расходов в системе. Эта функция позволяет использовать неограниченное число моделей для учета РБП. Можно параллельно вести налоговый учет и бухгалтерский учет РБП.

РБП создаются в следующих местах:

- Вручную на странице **Расходы будущих периодов**.
- Автоматически из других модулей в Microsoft Dynamics 365 Finance. Например, РБП создаются автоматически при продаже амортизируемых активов с убытком или при учете сумм стандартных расходов выше нормы.

РБП автоматически списываются в каждом периодов. Сумма списания определяется на основе периода списания и суммы РБП.

Ниже приведены несколько примеров расходов будущих периодов:

- Убытки, понесенные налогоплательщиком в налоговый период. Эти убытки могут быть перенесены в последующие налоговые периоды в течение 10 лет. Общая сумма перенесенных убытков не должна превышать 30 процентов базы налога.
- Расходы на исследование работы или экспериментальную разработку. Эти расходы также включаются в прочие расходы за три года с первого дня месяца после месяца, когда исследование было завершено.
- Убытки от продажи амортизируемого имущества.

## <a name="deferrals-setup"></a>Настройка расходов будущих периодов

Дополнительные сведения о настройке расходов будущих периодов см. в разделе [Настройка расходов будущих периодов (Россия)](https://docs.microsoft.com/dynamics365/finance/localizations/rus-set-up-deferrals).

### <a name="set-up-deferral-creation-when-you-sell-fixed-assets"></a>Настройка создания РБП при продаже основных средств

При продаже основных средств с потерей в бухгалтерском учете и налоговом учете необходимо создать расходы будущего периода для суммы, которая была потеряна при продаже. Для автоматического создания РБП выполните следующие шаги перед продажей основного средства.

1. Перейдите в раздел **Основные средства (Россия) \> Общие \> Основные средства.** В главной записи для основных средств на панели операций выберите **Модель стоимости**.
2. Выберите модель стоимости **НАЛОГ**.
3. На экспресс-вкладке **Расходы будущего периода** выберите **Добавить**, чтобы добавить строку, затем установите следующие поля:

    - В поле **Код модели** выберите модель **НАЛОГ** для учета РБП.
    - В поле **Группа расходов будущих периодов** выберите группу расходов будущих периодов.
    - В поле **Код расхода** выберите код расходов.


Можно также настроить автоматическое создание РБП для группы амортизации основных средств.

1. На странице **Амортизационные группы** выберите модель стоимости **НАЛОГ** и группу амортизации.
2. На экспресс-вкладке **Расходы будущих периодов** добавьте строку и задайте поля, описанные в предыдущей процедуре. При создании моделей стоимости в главной записи основных средств экспресс-вкладка **Расходы будущих периодов** заполняется автоматически.

## <a name="deferrals-register"></a>Регистр расходов будущих периодов

Регистр РБП создается для суммирования информации о расходах, которые могут быть включены в расходы для целей налогообложения в будущих периодах. Эти расходы называются *расходами будущих периодов*.

Записи регистра выполняются при создании или списании РБП.

Необходимо организовать аналитический учет РБП для каждого объекта (основного средства, исследовательской работы или экспериментальной разработки) и каждого типа расходов, который приводит к созданию расходов будущих периодов. Данная организация гарантирует, что создаваемые строки регистра РБП имеют одинаковый уровень аналитического учета.

Сведения, необходимые для создания регистра, хранятся в главной записи РБП. (Эти сведения включают дату создания РБП, код расхода, сумму РБП, период списания и сумму списания для периода.) Регистр содержит сведения обо всех расходах будущих периодов, которые еще не списаны в период, для которого был создан регистр.

В регистре рассчитываются итоги по каждому коду расхода. Общие суммы регистра используются для расчета итоговых значений для кодов выручки и расходов.

Строки регистра также отображают следующую информацию:

- **Номер строки**
- **Дата проводки** — дата проводки расходов будущих периодов.
- **Тип расхода** — код расхода, назначенный в проводке.
- **Код расхода** — описание кода расходов в проводке.
- **Имя объекта** — имя расхода будущего периода.
- **Сумма РБП** — сумма расхода будущего периода.
- **Срок списания расходов будущих периодов** — период списания расхода будущего периода.
- **Ежемесячная сумма** — ежемесячная сумма списания РБП.
- **Дата начала счета** — начальная дата учета РБП.
- **Количество месяцев** — количество месяцев, когда была разнесена проводка списания, до текущего месяца.
- **Сумма списания** — сумма списания для РБП в текущем регистре.

### <a name="set-up-the-deferrals-register"></a>Настройка регистра для расходов будущих периодов

Настройте регистр РБП, следуя указаниям в журнале регистров налога на прибыль. В поле **Тип регистра** выберите **Расходы будущих периодов**.

### <a name="examples"></a>Примеры

В следующих двух примерах используется компания **RUMF**:

   - Расходы будущих периодов создаются автоматически, когда основные средства продаются с убытками
   - РБП создаются автоматически с помощью периодической задачи
   

#### <a name="example-deferrals-are-automatically-created-when-fixed-assets-are-sold-at-a-loss"></a>Пример. Расходы будущих периодов создаются автоматически, когда основные средства продаются с убытками

1. На странице **Группы РБП** (**Главная книга \> Расходы будущих периодов**) создайте следующую группу РБП.

| **Группа расходов** | **Код модели** | **Метод списания** | **Время списания** | **Дата начала списания** | **Дата выбытия** |
|---------------------|------------------|----------------------|--------------------|---------------------------------|-------------------|
| Ежегодно              | НАЛОГ              | Линейный               | 12                 | С даты создания             | Конец года          |

2. На странице **Коды расходов и доходов** (**Налог \> Настройка \> Налог на прибыль \> Коды расходов**) убедитесь, что созданы следующие коды расходов.

| **Код расхода** | **Тип кода** | **Родительский код** | **Код налога** |
|------------------|---------------|-----------------|--------------------|
| 902000000        | Расход         |                 | НП                 |
| 902030000        | Расход         | 902000000       | НП                 |

3. На странице **Группы амортизации** (**Основные средства (Россия) \> Настройка \> Группы амортизации**) создайте следующую группу амортизации, использующую модель стоимости **НАЛОГ**.

| **Амортизационная группа** | **Метод амортизации** | **Модель стоимости** | **Дата начала начисления амортизации** |
|------------------------|-------------------------|-----------------|-----------------------------|
| Убыток ОС                | Линейный                  | НАЛОГ             | С начала следующего месяца            |

4. На экспресс-вкладке **Расходы будущих периодов** создайте следующую строку.

| **Код модели** | **Группа расходов** | **Код расхода** |
|------------------|---------------------|------------------|
| НАЛОГ              | Ежегодно              | 902030000        |

5. На странице **Группы ОС** (**Основные средства (Россия) \> Настройка \> Группы ОС**) создайте следующую группу основных средств (ОС).

| **Группа ОС** | **Наименование**  |
|--------------|-----------|
| Запасы    | Запасы |

6. На странице **Выпущенные продукты** (**Управление сведениями о продукте \> Продукты \> Выпущенные продукты**) создайте продукт. В разделе **Основные средства (Россия)** в поле **Группа ОС** выберите только что созданную группу основных средств.

| **Номер продукта** | **Наименование продукта:** | **Группа ОС** |
|--------------------|------------------|--------------|
| Inventory01        | Inventory01      | Запасы    |

7. На странице **Налоговые регистры** (**Налог \> Настройка \> Налог на прибыль \> Регистры**) создайте следующие регистры:

    - ОC — информация об объекте
    - ОС - амортизация
    - ОС и НМА - реализация, расходы будущих периодов

      Для каждого регистра, на экспресс-вкладке **скрыть**, выберите поля, которые должны быть скрыты из регистра.

8. На странице **Основные средства** (Основные средства (Россия) \> Общее \> Основные средства) создайте следующее ОС.

| **Группа ОС** | **Число** | **Дата приобретения** | **Стоимость приобретения** |
|--------------|------------|----------------------|----------------------|
| Запасы    | 4051       | 01/01/2019           | 300,000.00           |

9. Добавьте модель стоимости **НАЛОГ** для основного средства. На экспресс-вкладке **Общие** настройте следующие поля.

| **Модель стоимости** | **Амортизационная группа** | **Цена ввода в эксплуатацию** |
|-----------------|------------------------|-----------------------------------|
| НАЛОГ             | Убыток ОС                | 300,000.00                        |

10. Убедитесь, что на экспресс-вкладке **Расходы будущих периодов** имеется следующая строка.

| **Код модели** | **Группа расходов** | **Код расхода** |
|------------------|---------------------|------------------|
| НАЛОГ              | Ежегодно              | 902030000        |

11. Создайте журнал основных средств для ввода в эксплуатацию и выполните предложение по вводу в эксплуатацию, используя 1 января 2019 г. (1/1/2019) в качестве даты проводки. Затем разнесите журнал.
12. Создайте журнал основных средств для амортизации и выполните предложение по амортизации, используя 01 апреля 2019 (4/1/2019) в качестве даты проводки. Убедитесь, что созданы следующие строки.

| **Учет** | **Дата** | **Тип счета** | **Счет** | **Описание**    | **По кредиту** |
|----------------|----------|------------------|-------------|--------------------|------------|
| RAP            | 01/02/2019 | Основные средства      | 4051        | Ам. на 28/02/2019 | 25,000.00  |
| RAP            | 01/03/2019 | Основные средства      | 4051        | Ам. на 31/03/2019 | 25,000.00  |
| НАЛОГ            | 01/02/2019 | Основные средства      | 4051        | Ам. на 28/02/2019 | 25,000.00  |
| НАЛОГ            | 01/03/2019 | Основные средства      | 4051        | Ам. на 31/03/2019 | 25,000.00  |

13. Разноска журнала.
14. На странице **Все заказы на продажу** (**Расчеты с клиентами \> Заказы \> Все заказы на продажу**) создайте заказ на продажу со следующей строкой:

| **Код номенклатуры** | **Количество** | **Цена за ед. изм.** | **Инвентарный номер ОС** |
|-----------------|--------------|----------------|-------------------------|
| Inventory01     | 1            | 200,000.00     | 4051                    |

15. Переключитесь в вид **заголовка**, а затем на экспресс-вкладке **финансовых аналитик** в поле **ExpenseAndIncomeCode** выберите **902030000**. В поле **Дата накладной** выберите **3/31/2019**.
16. Разноска накладной.
17. На странице **Расходы будущих периодов** (**Главная книга \> Расходы будущих периодов**) убедитесь, что была создана главная запись РБП.
18. Убедитесь, что на странице **Модели расходов будущих периодов** имеется следующая строка.

| **Код модели** | **Группа расходов** | **Дата начала списания** | **Сумма расхода** |
|------------------|---------------------|-----------------------------------|-------------------|
| НАЛОГ              | Ежегодно              | 31.03.2019                         | 50,000.00         |

19. Перейдите к представлению **Общие**. Должно отобразиться, что значение в поле **Время списания** равняется оставшемуся сроку эксплуатации основного средства. Обратите внимание, что **50 000,00** = 300 000,00 – 25 000,00 × 2 – 200 000,00.
20. Создайте журнал налоговых регистров для одного месяца 2019 календарного года, рассчитайте все регистры, затем утвердите журнал. Повторите этот шаг за два месяца 2019 календарного года.
21. Создайте журнал налоговых регистров для трех месяцев 2019 календарного года и рассчитайте все регистры. В регистре **Расходы будущих периодов** должны отобразиться следующие сведения.

| **Столбец**                   | **значение**                                                                                   |
|------------------------------|---------------------------------------------------------------------------------------------|
| Дата проводки             | 31.03.2019                                                                                   |
| Вид расхода                 | 902030000                                                                                   |
| Код расхода                 | «Прочая амортизация ОС линейным методом» (ENU: «Other FA depreciation using linear method») |
| Наименование объекта                  | Выбытие ОС № 4051 от 31.03.2019                                                            |
| Сумма расхода                | 50,000.00                                                                                   |
| Срок списания расхода | 10                                                                                          |
| Ежемесячная сумма               | \-5 000,00                                                                                  |
| Дата начала учета           | 31.03.2019                                                                                   |
| Количество месяцев             | 1.00                                                                                        |
| Сумма списания              | \-5 000,00                                                                                  |

22. Утвердить журнал.
23. Создайте журнал налоговых регистров для четырех месяцев 2019 календарного года и рассчитайте все регистры. В регистре **Расходы будущих периодов** должна отобразиться та же строка, которая была показана в регистре **Расходы будущих периодов** за три месяца. Однако в поле **Сумма месяцев** будет установлено значение **2,00** вместо **1,00**.

#### <a name="example-deferrals-are-automatically-created-by-using-a-periodic-task"></a>Пример. РБП создаются автоматически с помощью периодической задачи

1. Перейдите в раздел **Главная книга \> Настройка РБП \> Модели стоимости** и убедитесь, что созданы следующие модели стоимости и группы РБП.

| **Код модели** | **Группа расходов** | **Время списания** | **Дата начала списания** |
|------------------|---------------------|--------------------|---------------------------------|
| RAP              | 1                   | 6                  | С даты создания             |
| НАЛОГ              | Ежегодно              | 12                 | С даты создания             |

2. На странице **Нормативные расходы - последовательности** создайте следующую последовательность.

| **Последовательность** | **Описание**             | **Канал** | **Ссылка на канал** |
|--------------|-----------------------------|-------------|-----------------------|
| AutDef       | Автоматическое создание РБП | Расход буд. периодов    | 1. Ежегодно              |

3. На панели операций выберите **Счетчики**.
4. На странице **Настройка счетчика** создайте последовательность. В верхней области в поле **Код расхода** выберите **902010100**, а затем на экспресс-вкладке **Строки** создайте следующие строки.

| **Номер строки** | **Оператор** | **Tип строки** | **Типы периодов** | **Вывести**  |
|-----------------|--------------|---------------|------------------|-------------|
| 1               |              | Количество      | Нет               |             |
| 2               | \*           | Цена,         | Нет               | Вывод данных |

5. Перейдите в раздел **Расчеты с поставщиками \> Накладные \> Журнал накладных** и создайте журнал.
6. На панели операций выберите **Строки**, задайте следующие поля, затем разнесите журнал.

| **Столбец**          | **значение**                                         |
|---------------------|---------------------------------------------------|
| Дата                | 1/1/2019                                          |
| Тип счета        | Поставщик                                            |
| Счет             | RUMF-000002                                       |
| Счет-фактура             | 0007                                              |
| описание         | «Расходы перечисление» (ENU: «Expenses transfer») |
| По кредиту              | 10,000.00                                         |
| Валютное            | руб.                                               |
| Тип корр. счета | Ledger                                            |
| Корр. счет      | 97.120-902010100                                  |

7. На странице **Создание РБП** на панели операций выберите созданную последовательность расчета, а затем выберите **Рассчитать отмеченные**.
8. На экспресс-вкладке **Параметры** задайте следующие поля:

    - В поле **Дата начала** выберите **01.01.2019**.
    - В поле **Дата окончания** выберите **31.01.2019**.
    - Установите для параметра **Предварительный просмотр** значение **Да**.

9. Выберите **ОК**, затем на открывшейся странице задайте для поля **Имя** значение **«Расходы перечисление» (ENU: «Expenses transfer»)**.
10. В области действий выберите **Создать РБП**, а затем на странице **Расходы будущих периодов** убедитесь, что была создана главная запись РБП.
11. Убедитесь, что на странице **Модели расходов будущих периодов** имеются следующие строки.

| **Код модели** | **Группа расходов** | **Дата начала списания** | **Сумма расхода** |
|------------------|---------------------|---------------------------------|-------------------|
| RAP              | 1                   | 1/1/2019                        | 10,000.00         |
| НАЛОГ              | Ежегодно              | 1/1/2019                        | 10,000.00         |

12. На странице **Журнал расходов будущих периодов** создайте новый журнал, а затем в области действий выберите **Строки**.
13. На странице **Ваучер журнала** в области действий выберите **Групповые операции \> Списание**.
14. На странице **Списание расходов будущих периодов** в поле **Дата проводки** выберите **01.03.2019**, а затем выберите **ОК**.
15. Убедитесь, что две строки были созданы для модели стоимости **НАЛОГ**. Для обеих строк тип операции должен быть **Списание**, а сумма должна равняться 10 000,00 ÷ 12 = **833,33**. После завершения разнесите журнал.
16. Создайте журнал налоговых регистров для одного месяца 2019 календарного года, затем рассчитайте все регистры. В регистре **Расходы будущих периодов** должны отобразиться следующие сведения.

| **Столбец**                   | **значение**                                                                                                    |
|------------------------------|--------------------------------------------------------------------------------------------------------------|
| Дата проводки             | 1/1/2019                                                                                                     |
| Вид расхода                 | 902010100                                                                                                    |
| Код расхода                 | «Прямые расходы на производство (сырье и материалы)» (ENU: «Direct production expenses (raw and materials)») |
| Наименование объекта                  | «Расходы перечисление» (ENU: «Expenses transfer»)                                                            |
| Сумма расхода                | 10,000.00                                                                                                    |
| Срок списания расхода | 12                                                                                                           |
| Ежемесячная сумма               | \-833,33                                                                                                     |
| Дата начала учета           | 1/1/2019                                                                                                     |
| Количество месяцев             | 1.00                                                                                                         |
| Сумма списания                | \-833,33                                                                                                     |

17. Утвердите журнал, затем создайте журнал налоговых регистров для двух месяцев 2019 календарного года.
18. Рассчитайте регистры. В регистре **Расходы будущих периодов** должна отобразиться та же строка, которая была показана в регистре **Расходы будущих периодов** за один месяц. Однако в поле **Сумма месяцев** будет установлено значение **2,00** вместо **1,00**.
