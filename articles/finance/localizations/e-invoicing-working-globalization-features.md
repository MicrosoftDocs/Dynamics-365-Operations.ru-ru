# <a name="globalization-feature-components"></a>Компоненты функции глобализации

[!include [banner](../includes/banner.md)]

Функция глобализации представляет собой набор компонентов, определяющих правила для преобразования данных в конфигурациях электронной отчетности (ER). К этим компонентам относятся инструкции по обработке электронных документов, и затем их отправки или получения из внешних каналов. Они также включают в себя условия, определяющие, когда должна выполняться функция для входящих бизнес-данных.

Все компоненты зависят друг от друга. Функции глобализации облегчают создание и поддержку этой зависимости и поддержку управления жизненным циклом и управления версиями наборов компонентов.

## <a name="access-electronic-invoicing-feature-components"></a>Доступ к компонентам функций электронного выставления накладных 

1. Выполните вход в учетную запись Regulatory Configuration Services (RCS).
2. В рабочей области **Функция глобализации** в разделе **Функции** выберите плитку **Электронное выставление накладных**.

    Для функций глобализации есть следующие компоненты. Страница **Функции электронного выставления накладных** содержит отдельную вкладку для каждого компонента.

    - **Версия** — этот компонент поддерживает управление жизненным циклом функции. Его можно использовать для управления статусом для других версий функции.
    - **Конфигурации** — этот компонент позволяет управлять, просматривать и редактировать связанный формат электронной отчетности и конфигурации сопоставления форматов, которые используются в конвейере обработки.
    - **Настройки** — этот компонент позволяет пользователям служб глобализации, например службы электронных накладных, управлять настройкой связанной версии функции. Таким образом, он поддерживает гибкое создание правил обмена данными и реакции.
    - **Среды** — этот компонент позволяет пользователям служб глобализации, например службы электронных накладных, управлять средой, в которой используется настройка версии функций. Он также позволяет им предоставлять авторизацию пользователям, которые будут иметь доступ к настройке версии функции.
    - **Организации** — этот компонент позволяет пользователям использовать функцию совместно с внешними организациями.
    - **Теги** — этот компонент позволяет обозначить функции, которые могут быть использованы в схеме глобализации для данной ссылки.
