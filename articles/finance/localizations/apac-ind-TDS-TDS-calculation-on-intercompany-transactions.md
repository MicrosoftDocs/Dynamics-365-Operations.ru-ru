# <a name="tds-calculation-on-intercompany-transactions"></a>Расчет ЕTD на внутрихолдинговых проводках

[!include [banner](../includes/banner.md)]

В этой теме описывается процесс, используемый для расчета налога, вычитаемого в источнике (TDS), по внутрихолдинговым проводкам в виде фаз.

При создании внутрихолдингового заказа на покупку или заказа на продажу группа TDS по умолчанию, которая определена для клиента или поставщика, используется для расчета суммы TDS. Сумма TDS разносится на возмещаемые счета или счета кредиторской задолженности после разноски накладной.

Внутрихолдинговый заказ на продажу или заказ на покупку создается автоматически для исходного заказа на покупку или заказа на продажу. Группа TDS по умолчанию отображается во внутрихолдинговом заказе, чтобы можно было рассчитать TDS. Группу TDS можно изменить. Накладная может быть разнесена только в том случае, если чистая сумма накладной по внутрихолдинговому заказу, которая создается автоматически, совпадает с чистой суммой накладной в исходном заказе. (Чистая сумма представляет собой сумму накладной после вычета TDS.)

Например, накладная заказа на продажу создается для 50 000, и к ней привязывается TDS **10%**. Разнесенная сумма по накладной составляет 45 000, а разнесенная сумма TDS для накладной заказа на продажу составляет 5000. Заказ на покупку автоматически создается для внутрихолдингового заказа на продажу, и к нему прикрепляется **10%** TDS. Если изменить группу TDS на **15%**, невозможно будет разнести накладную, так как чистая сумма накладной для внутрихолдингового заказа на продажу, которая была автоматически создана, не совпадает с чистой суммой накладной исходного заказа на продажу.

Журнал платежей, созданный для внутрихолдинговой накладной, разносится автоматически. Этот журнал платежей может быть разнесен только в том случае, если чистая сумма платежа в ней совпадает с суммой чистого платежа по исходному журналу платежей.
