# <a name="no-matching-result-could-be-found"></a>Не удается найти соответствующие результаты

[!include [banner](../includes/banner.md)]

В этой теме объясняется порядок устранения неполадок, если вы получаете ошибку "Не удается найти соответствующие результаты" в службе расчета налогов.

## <a name="symptom"></a>Проблема

Появляется следующее сообщение об ошибке: "Заголовок/строки - 1, налоговая группа, не удается найти соответствующие результаты".

```json
======================Tax service calculation result JSON:===========================
{
    "taxDocument": {
        "Header": [
            {
                "Lines": [
                    {
                        ...
                        "Errors": [
                            {
                                "Code": "TaxSetup20000",
                                "Message": "Header/Lines - 1, Tax group applicability, no matching result could be found."
                            }
                        ],
                        "Adjustment": null
                    }
                ],
                "Measures": {
                    ...
                },
                ...
            }
        ]
    },
    ...
}
```

## <a name="cause"></a>Причина

Эта проблема возникает, когда настройка функций в Regulatory Configuration Service (RCS) неправильна

## <a name="troubleshoot"></a>Устранение неполадок

1. Загрузите файл устранения неполадок. Дополнительные сведения см. в разделе [Включение режима отладки для устранения неполадок](tcs-troubleshooting-enable-debug-mode.md).
2. Сравните входные данные расчета налоговой службы с настройкой функций, чтобы исправить ошибку настройки.

    В следующем примере показан входные данные расчетов налоговой службы.

    ```json
    ===============================Tax service calculation input JSON:=====================================
    {
        "TaxableDocument": {
            "Header": [
                {
                    "Lines": [
                        {
                            ...
                        }
                    ],
                    "Business Process": "Sales",
                    "Ship From Zip Code": "30159",
                }
            ]
        },
        "Parameter": {
            ...
        },
        "Adjustment": {
            "Lines": {}
        }
    }
    ```

    В следующей таблице перечислены условия применимости налоговой группы в RCS.

    | Заголовок.Бизнес-процесс | Строки.Бизнес-единица | Заголовок.Отгрузка из почтового индекса | Налоговая группа |
    |-------------------------|---------------------|---------------------------|-----------|
    | Журнал                 |                     |                           | Группа A   |
    | Продажи                   |                     | 30160                     | Группа B   |

    Согласно входным данным расчета налоговой службы значение **Бизнес-процесс** в заголовке равно **Продажи**, а значение **Отгрузка из почтового индекса** в заголовке равно **30159**. Эти входные данные основаны на настройке правил применимости в RCS. Так как соответствующая строка отсутствует, возникает ошибка.

    > [!NOTE]
    > Если значение в правиле применимости не заполнено, правило применяется к любому значению.

## <a name="mitigation"></a>Снижение риска

Выполните следующие действия для устранения этой ошибки.

1. В RCS перейдите в **Функции глобализации** \> **Расчета налога**.
2. Создайте новую версию функции.
3. Добавьте строку для соответствующей информации.

    | Заголовок.Бизнес-процесс | Строки.Бизнес-единица | Заголовок.Отгрузка из почтового индекса| Налоговая группа |
    |-------------------------|---------------------|--------------------------|-----------|
    | Журнал                 |                     |                          | Группа A   |
    | Продажи                   |                     | 30160                    | Группа B   |
    | Продажи                   |                     | 30159                    | Группа B   |

4. Опубликуйте версию настройки функций.
5. В Microsoft Dynamics 365 Finance перейдите к пункту **Налог** \> **Настройка** \> **Конфигурация налога** \> **Параметры расчета налогов** и выберите новую версию.
