# <a name="get-started-with-the-electronic-invoicing-add-on-for-brazil"></a>Приступая к работе с дополнительным компонентом электронных накладных для Бразилии 

[!include [banner](../includes/banner.md)]

В этой теме объясняется, как начать работать с надстройкой электронного выставления накладных для Бразилии. Процедуры данного раздела посвящены выполнению шагов настройки, зависящих от страны, в Regulatory Configuration Services (RCS), и дополнят шаги, описанные в разделе, [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

## <a name="country-specific-configuration-for-brazilian-nf-e-br-electronic-invoicing-feature"></a>Конфигурация для определенной страны для Бразилии NF-e (BR) функции электронного выставления накладных

При настройке функции выставления электронных накладных для Бразилии NF-e (BR) необходимо выполнить указанные шаги. Некоторые из параметров конфигураций публикуются со значениями по умолчанию, поэтому их необходимо проверить и обновить, чтобы они лучше соответствовали вашей бизнес-операции.

### <a name="prerequisites"></a>Необходимые условия

Прежде чем выполнить процедуру, описанную в этом разделе, создайте функцию электронного выставления накладных для Бразилии NF-e (BR) в рамках вашей организации, как описано в подразделе **Создание функции электронного выставления накладных в вашем поставщике организации** раздела [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

1. В RCS в разделе **Функции** рабочей области **Функция глобализации** выберите плитку **Надстройка электронных накладных**.
2. На странице **Функции надстройки электронных накладных** убедитесь, что выбрано созданная функция электронных накладных для **Бразилия NF-e (BR)**.
3. На вкладке **Версии** убедитесь, что выбрана версия **Черновик**.
4. На вкладке **настройки** в сетке выберите **Отправить**, а затем выберите **Изменить**.
5. На вкладке **действия** в группе поле **действия** выберите действие **(Предварительная версия) Подпишите XML-документ**.
6. В группе полей **параметры** выберите **имя сертификата**, а в поле **значение** введите имя сертификата, связанного с вашим параметром хранилища ключей.
7. На вкладке **действия** в группе поле **действия** выберите действие **(Предварительная версия) Вызов бразильской службы SEFAZ**.
8. В группе полей **параметры** выберите параметр **URL-адрес**.
9. В поле **значение**, если необходимо, проверьте и обновите URL-адрес веб-служб, опубликованный в документации SEFAZ для вашего состояния, а затем выберите **Сохранить**.
10. На вкладке **Правила применимости** в группе полей **правила применимости** проверьте и обновите критерии поля **состояние**, если это необходимо, для того же состояния, на которое ссылается URL-адрес веб-служб.
11. Выберите **Сохранить** и закройте страницу.
12. Чтобы настроить настройку приложения, см. раздел [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

## <a name="country-specific-configuration-of-application-setup-for-brazilian-nf-e-br-electronic-invoicing-feature"></a>Конфигурация настройки приложения для определенной страны для Бразилии NF-e (BR) функции электронного выставления накладных

При настройке настройки приложения для функции электронного выставления накладных для Бразилии NF-e (BR) необходимо выполнить указанные шаги. Выполните эти шаги перед развертыванием функции электронного выставления накладных в среде службы надстройки электронного выставления накладных.

### <a name="prerequisites"></a>Необходимые условия

Прежде чем выполнить процедуру, описанную в этом разделе, создайте и инициируйте настройку приложения функции электронного выставления накладных для Бразилии NF-e (BR), как описано в разделе **Настройка настройки приложения** в разделе [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

1. В RCS в разделе **Функции** рабочей области **Функция глобализации** выберите плитку **Надстройка электронных накладных**.
2. На странице **Функции надстройки электронных накладных** убедитесь, что выбрана функция электронных накладных для **Бразилия NF-e (BR)**.
3. На вкладке **Версии** убедитесь, что выбрана версия **Черновик**.
4. На вкладке **настройки** выберите **Настройка приложения** и в поле **подключенное приложение** выберите приложение, в котором необходимо выполнить развертывание.
5. В поле **Имя таблицы** убедитесь, что выбрано **Заголовок финансового документа**.
6. Выберите **типы откликов**, а затем выберите **Создать**.
7. В поле **тип отклика** введите "отклик" как фиксированное значение, а в поле **Описание** введите "Описание".
8. В поле **Статус отправки** выберите **Ожидание**.
9. В поле **Сопоставление моделей** выберите **Сопоставление модели из ответного сообщения** с **(Предварительная версия) Формат импорта ответного сообщения**, а затем выберите **Сохранить**.
10. Выберите **Создать** и в поле **тип отклика** введите "ResponseData" как фиксированное значение, а в поле **Описание** введите "Описание".
11. В поле **Статус отправки** выберите **Ожидание**.
12. В поле **Сопоставление моделей** выберите **импорт данных ответа** с **(Предварительная версия) Формат импорта данных ответа NF-e (BR)**, а затем выберите **Сохранить**.
13. Для развертывания функции электронного выставления накладных см. [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

## <a name="country-specific-configuration-for-brazilian-nfs-e-abrasf-curitiba-br-electronic-invoicing-feature"></a>Конфигурация для определенной страны для функции электронного выставления накладных Бразилии NFS-e ABRASF Куритиба (BR)

При настройке функции выставления электронного выставления накладных Бразилии NFS-e ABRASF Куритиба (BR) необходимо выполнить указанные шаги. Некоторые из параметров конфигураций публикуются со значениями по умолчанию, поэтому их необходимо проверить и обновить, чтобы они лучше соответствовали вашей бизнес-операции.

### <a name="prerequisites"></a>Необходимые условия

Прежде чем выполнить процедуру, описанную в этом разделе, создайте функцию выставления электронного выставления накладных Бразилии NFS-e ABRASF Куритиба (BR) в организации. Это описано в разделе **Настройка функции электронного выставления накладных** в теме, [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

1. В RCS в разделе **Функции** рабочей области **Функция глобализации** выберите плитку **Надстройка электронных накладных**.
2. На странице **Функции надстройки электронного выставления накладных** убедитесь, что выбрано созданная функция электронного выставления накладных **Бразилии NFS-e ABRASF Куритиба (BR)**.
3. На вкладке **версии** убедитесь, что выбрана **черновая** версия, и на вкладке **настройки** в сетке выберите **Отправить**.
4. Выберите **Изменить** и на вкладке **действия** в группе полей **действия** выберите первый случай возникновения **(Предварительная версия) Подпишите XML-документ**.
5. В группе полей **параметры** выберите параметр **Имя сертификата**.
6. В поле **значение** введите имя сертификата, связанного с вашим параметром хранилища ключей.
7. В группе полей **действия** выберите второй случай возникновения **(Предварительная версия) документ Signxml**.
8. В группе полей **параметры** выберите параметр **Имя сертификата**.
9. В поле **значение** введите имя сертификата, связанного с вашим параметром хранилища ключей.
10. На вкладке **действия** в группе поле **действия** выберите действие **(Предварительная версия) Вызов бразильской службы SEFAZ**.
11. В группе полей **параметры** выберите параметр **URL-адрес**.
12. В поле **значение**, если необходимо, проверьте и обновите URL-адрес веб-служб, опубликованный налоговый отделом для города Куритиба, а затем выберите **Сохранить**.
13. На вкладке **настройки** в сетке выберите **Запрос**, а затем выберите **Изменить**.
14. На вкладке **действия** в группе поле **действия** выберите действие **(Предварительная версия) Вызов бразильской службы SEFAZ**.
15. В группе полей **параметры** выберите параметр **URL-адрес**.
16. В поле **значение**, если необходимо, проверьте и обновите URL-адрес веб-служб, опубликованный налоговый отделом для города Куритиба.
17. Выберите **Сохранить** и закройте страницу.
18. Чтобы настроить настройку приложения, см. раздел [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

## <a name="country-specific-configuration-of-application-setup-for-brazilian-nfs-e-abrasf-curitiba-br-electronic-invoicing-feature"></a>Конфигурация настройки приложения для определенной страны для функции электронного выставления накладных Бразилии NFS-e ABRASF Куритиба (BR)

При настройке приложения для функция электронного выставления накладных для Бразилии NFS-e ABRASF Куритиба (BR) необходимо выполнить определенные шаги, прежде чем развертывать функцию электронного выставления накладных в среде службы надстройки электронного выставления накладных.

### <a name="prerequisites"></a>Необходимые условия

Прежде чем выполнить процедуру, описанную в этом разделе, создайте и инициируйте функцию электронного выставления накладных для Бразилии NFS-e ABRASF Куритиба (BR), как описано в разделе **Настройка настройки приложения** в разделе [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md).

1. В RCS в разделе **Функции** рабочей области **Функция глобализации** выберите плитку **Надстройка электронных накладных**.
2. На странице **Функции надстройки электронного выставления накладных** убедитесь, что выбрано функция электронного выставления накладных **Бразилии NFS-e ABRASF Куритиба (BR)**.
3. На вкладке **версии** убедитесь, что выбрана **черновая** версия, и на вкладке **настройки** выберите **Настройка приложения**.
4. В поле **подключенное приложение** выберите приложение, в котором необходимо выполнить развертывание.
5. В поле **Имя таблицы** убедитесь, что выбран заголовок финансового документа.
6. Выберите **типы откликов**, и выберите **Создать**.
7. В поле **тип отклика** введите "ABRASFCuritibaSubmitResponse" как фиксированное значение, а в поле **Описание** введите "Описание".
8. В поле **Статус отправки** выберите **Ожидание**.
9. В поле **Сопоставление моделей** выберите **Импорт ответного сообщения** с **(Предварительная версия) Импорт ответного сообщения NFS-e ABRASF Куритиба (BR)**, а затем выберите **Сохранить**.
10. Выберите **Создать** и в поле **тип отклика** введите "ABRASFCuritibaSubmitResponseData", а в поле **Описание** введите "Описание".
11. В поле **Статус отправки** выберите **Ожидание**.
12. В поле **Сопоставление моделей** выберите **импорт данных ответа** с **(Предварительная версия) Формат импорта данных ответа NFS-e ABRASF (BR)**, а затем выберите **Сохранить**.
13. Выберите **Создать** и в поле **тип отклика** введите "ABRASFCuritibaInquireResponse", а в поле **Описание** введите "Описание".
14. В поле **Статус отправки** выберите **Ожидание**.
15. В поле **Сопоставление моделей** выберите **Импорт ответного сообщения** с **(Предварительная версия) Импорт ответного сообщения NFS-e ABRASF Куритиба (BR)**.
16. Выберите **Сохранить**, а затем вернитесь к теме, [Начало работы с надстройкой электронного выставления накладных](e-invoicing-get-started.md), чтобы развернуть функции электронного выставления накладных.

## <a name="privacy-notice"></a>Уведомление о конфиденциальности
При включении функций **Федеральный NF-e — электронная накладная для Бразилии (BR)** и **NFS-e — бразильская служба (город) электронная накладная** могут потребоваться отправку ограниченных данных, включая код налоговой регистрации организации. Эти данные передаются сторонним агентствам, которые уполномочены налоговыми органами для этой цели отправки электронных накладных в этот налоговый орган в заранее определенном формате, требуемом для интеграции с веб-службой правительственного учреждения. Администратор может включать или отключать функции **Федеральный NF-e — электронная накладная для Бразилии (BR)** и **NFS-e — бразильская служба (город) электронная накладная**. Следующие шаги показывают, как это сделать: 

1. Перейдите в раздел **Администрирование организации** > **Настройка** > **Параметры электронных документов**. 
2. На вкладке **функции** выберите строку, которая содержит **Федеральный NF-e — электронная накладная для Бразилии (BR)** или **NFS-e — бразильская служба (город) электронная накладная**, и выберите функцию. На данные, импортируемые из этих внешних систем в данную веб-службу Dynamics 365, распространяется наше [заявление о конфиденциальности](https://go.microsoft.com/fwlink/?LinkId=512132). Дополнительные сведения см. в разделах "Уведомление о конфиденциальности" в документации по компонентам, относящимся к конкретной стране.
