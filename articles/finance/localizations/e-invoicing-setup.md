# <a name="set-up-the-electronic-invoicing-add-on"></a>Настройка дополнения электронных накладных

[!include [banner](../includes/banner.md)]


Настройка дополнительной функции электронных накладных — это процесс создания необходимой конфигурации с помощью среды Regulatory Configuration Services (RCS) и публикации этой конфигурации на сервере дополнения электронных накладных. Настройка позволяет создать настраиваемые правила, которые позволяют дополнению электронных накладных использовать защищенный протокол в Интернете для обмена данными с третьим лицом через веб-службы.

Возможность настройки конфигурации основывается на конфигурации формата электронной отчетности (ER), которая позволяет создавать содержимое, отправляемое и получаемое с помощью цифровых файлов. Она также опирается на согласование действий связи для отправки запросов и получения откликов от сторонних веб-служб, не требуя написания кода.

## <a name="overview"></a>Обзор

"Дополнительная функция электронных накладных" — это универсальное имя для ресурса, который настраивается и публикуется для использования сервера дополнения электронных накладных. Настройка функций объединяет, помимо всего прочего, использование форматов конфигурации электронной отчетности для создания настраиваемых файлов экспорта и импорта и использования действий и потоков действий для разрешения создания настраиваемых правил для отправки запросов, импорта ответов и обработки содержимого ответов.

На следующем рисунке показаны основные компоненты дополнительной функции электронных накладных.

![Обзор дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Overview-e-Invoicing-feature.png)

Из-за различия форматов накладных и потоков действий настройка функции может отличаться в зависимости от страны или региона, либо в соответствии с деловыми требованиями.

## <a name="set-up-the-electronic-invoicing-add-on-feature"></a>Настройка дополнительной функции электронных накладных

Процесс настройки должен быть завершен в вашей среде RCS. Выполните следующие действия, чтобы создать новую дополнительную функцию электронных накладных.

1. Выполните вход в свою среду RCS.
2. В рабочей области **Функции глобализации** в разделе **Функции** выберите плитку **Дополнение электронных накладных**.
3. На странице **Дополнительные функции электронных накладных** выберите **Импорт** , чтобы импортировать конфигурацию модели данных электронной отчетности из глобального репозитория.
4. Выберите **Добавить** для создания дополнительной функции электронных накладных. Можно либо создать функцию с нуля, либо наследовать ее из существующей дополнительной функции электронных накладных.

    ![Добавление дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Select-Add-e-Invoicing-feature.png)

> [!NOTE]
> При создании новой дополнительной функции электронных накладных она имеет номер версии, и ее статус по умолчанию устанавливается в значение **Черновик**.

### <a name="configurations"></a>Конфигурации

Конфигурации содержат конфигурации формата электронной отчетности, которые необходимы для преобразований и для создания файлов, которые будут обмениваться при обмене данными с веб-службами сторонних производителей. Дополнительная функция электронных накладных может иметь столько конфигураций формата файлов электронной отчетности, сколько требуется, на основе технической спецификации интеграции, предоставляемой поставщиком веб-службы.

Выполните следующие действия, чтобы добавить форматы электронной отчетности в дополнительную функцию электронных накладных.

1. На странице **Дополнительные функции электронных накладных** на вкладке **Конфигурации** выберите **Добавить** , чтобы добавить конфигурации форматов файлов электронной отчетности для дополнительной функции электронных накладных.

    ![Добавление конфигураций дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Select-Add-e-Invoicing-feature-Configurations.png)

    > [!NOTE]
    > При создании дополнительной функции электронных накладных с нуля необходимо вручную добавить все конфигурации форматов файлов электронной отчетности. При создании производной дополнительной функции электронных накладных из существующей функции конфигурации форматов файлов электронной отчетности создаются автоматически, поскольку они наследуются из исходной дополнительной функции электронных накладных.

2. Выберите **Правка** , чтобы открыть страницу **Конструктор форматов** , на которой можно изменить конфигурацию форматов файлов электронной отчетности.

    ![Изменение конфигураций дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Select-Edit-e-Invoicing-feature-Configurations.png)

    > [!NOTE]
    > При редактировании формата в качестве состояния версии конфигурации устанавливается значение **Черновик**.

3. Страница **Конструктор форматов** используется для изменения конфигурации форматов файлов. Дополнительные сведения см. в разделе [Создание конфигураций электронных документов](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/analytics/electronic-reporting-configuration).

    ![Страница конструктора форматов](media/e-Invoicing-services-feature-setup-ER-Format-designer.png)

### <a name="feature-setups"></a>Настройки функций

Настройки функций инкапсулируют правила связи и безопасности с веб-службой независимых разработчиков. Дополнительная функция электронных накладных может иметь столько настроек функций, сколько необходимо, в зависимости от бизнес-правила, которое необходимо выполнить.

Выполните следующие действия, чтобы добавить настройки функций в дополнительную функцию электронных накладных.

1. На странице **Дополнительные функции электронных накладных** на вкладке **Настройки** выберите **Добавить** , чтобы добавить настройки функций в дополнительную функцию электронных накладных.

    ![Добавление настроек дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Select-Add-e-Invoicing-feature-Setups.png)

    > [!NOTE]
    > При создании дополнительной функции электронных накладных с нуля необходимо вручную добавить все требуемые настройки функций. При создании производной дополнительной функции электронных накладных из существующей функции все настройки функций создаются автоматически, поскольку они наследуются из исходной дополнительной функции электронных накладных.

2. Выберите **Правка** , чтобы изменить настройку версии функции.

    ![Изменение настроек дополнительной функции электронных накладных](media/e-Invoicing-services-feature-setup-Select-Edit-e-Invoicing-feature-Setups.png)

3. Страница **Настройка версии функции** используется для настройки действий, правил применимости и переменных.

    ![Действия, правила применимости и переменные](media/e-Invoicing-services-feature-setup-View-Actions-Applicability-Rules-Variables.png)

### <a name="actions"></a>Действия

Действия представляют собой заранее определенный список операций, которые выполняются в последовательном порядке. Этот список представляет разбивку шагов, необходимых для полного выполнения дополнительной функции электронных накладных. Эти действия могут инкапсулировать, в одной и той же дополнительной функции электронных накладных, связь в обоих направлениях: отправка запроса для пункта назначения и получение отклика и анализ его содержимого.

Каждое действие содержит заранее определенный список параметров, которые необходимы для выполнения действия. При желании можно указать дополнительные параметры.

#### <a name="actions-fasttab"></a>Экспресс-вкладка действий

На странице **Настройка версий функций** на вкладке **Действия** на экспресс-вкладке **Действия** выполните одно или оба этих шага, чтобы управлять действиями:

- Выберите **Создать** или **Удалить** , чтобы добавить новые действия или удалить существующие действия.
- Выберите **Вверх** или **Вниз** , чтобы переместить выбранные действия вверх или вниз по сетке и, соответственно, изменить порядок, в котором они выполняются. Действия выполняются в том порядке, в котором они расположены в сетке, сверху вниз.

![Управление действиями](media/e-Invoicing-services-feature-setup-Manage-Actions.png)

| Поле        | описание |
|--------------|-------------|
| Действие       | Имеется восемь предопределенных действий:<ul><li><strong>Подписать документ</strong></li><li><strong>FileStoreActionName</strong></li><li><strong>Преобразование документа</strong></li><li><strong>Обработка ответа</strong></li><li><strong>Вызов веб-службы REST</strong></li><li><strong>Вызов мексиканской службы PAC</strong></li><li><strong>Вызов бразильской службы SEFAZ</strong></li><li><strong>Вызов итальянской службы SDI</strong></li></ul> |
| Имя действия  | Имя действия и порядок его выполнения. |
| описание  | Описание действия. |
| Включить повторную попытку | Установленный флажок указывает на то, что действие можно повторить, если предыдущая попытка завершилась неудачей. |
| Повторить действие | В случае повторной попытки действие, с которого начинается повторная попытка. Затем повторная попытка заканчивается на текущем действии (инклюзивная повторная попытка). Для действий, которые их имеют, параметры **Минимальная скорость отката** и **Максимальная скорость отката** указывают минимальное и максимальное число повторных попыток. |

### <a name="applicability-rules"></a>Правила применимости

Правила применимости позволяют создавать логические правила, определяющие контекст использования для настройки функций. Таким образом, сопоставление между контекстом, заданным деловым документом, отправляемым для обработки, и критерием правила применимости определяет, какая из дополнительных функций электронных накладных используется для обработки данной отправки.

#### <a name="set-up-applicability-rules"></a>Настройка правил применимости

1. На странице **Настройка версии функции** на вкладке **Правила применимости** выберите **Создать** , чтобы добавить правило применимости.

    ![Управление правилами применимости](media/e-Invoicing-services-feature-setup-Manage-Actions-Applicability-rules.png)

2. В сетке выберите предложения, которые необходимо сгруппировать.
3. Выберите **Сгруппировать предложения**.

    ![Группирование предложений](media/e-Invoicing-services-feature-setup-Manage-Applicability-rules-Group-clause.png)

    При группировке предложений в сетку добавляется новый столбец. В этом столбце указывается логический оператор для сгруппированных предложений.

    ![Логический оператор для сгруппированных предложений](media/e-Invoicing-services-feature-setup-Manage-Applicability-rules-Group-criterias.png)

Чтобы разгруппировать предложения, выберите сгруппированные предложения, которые необходимо разгруппировать, затем выберите вариант **Разгруппировать предложение**.

![Разгруппирование предложений](media/e-Invoicing-services-feature-setup-Manage-Applicability-rules-UnGroup-criterias.png)

> [!NOTE]
> При разгруппировании предложения всегда начинайте с самого вложенного уровня группировки.

В следующей таблице описываются поля, которые доступны на вкладке **Правила применимости**.

| Поле         | описание |
|---------------|-------------|
| И/или        | Логический оператор. |
| Поле         | Поле, используемое для создания правила. |
| Тип оператора | Тип оператора:<ul><li>Одинаковые</li><li>Не равно</li><li>Больше/Меньше</li><li>Больше или равно/Меньше или равно</li><li>Содержит</li><li>Начинается с</li></ul> |
| значение         | Критерий для правила. |

### <a name="variables"></a>Переменные

### <a name="validate-the-feature-setup"></a>Проверка настройки функции

   ![Нажатие кнопки "Проверить"](media/e-Invoicing-services-feature-setup-Select-Validate-Button.png)

При проверке выполняется проверка согласованности всей конфигурации. Например, если конкретный параметр для действия является обязательным, но его значение остается пустым, проверка обнаруживает это несоответствие и выдается предупреждение.

## <a name="environments"></a>Среды

Среда дополнения электронных накладных должна быть связана с дополнительной функцией электронного выставления накладных и включена для нее. Среды дополнений электронного выставления накладных должны быть созданы и опубликованы заранее с помощью настройки функций глобализации в учетной записи RCS организации.

Выполните эти шаги, чтобы включить среду дополнения электронных накладных для дополнительной функции электронных накладных.

1. На странице **Дополнительные функции электронных накладных** на вкладке **Среды** выберите **Включить** , чтобы добавить среду дополнительной функции электронных накладных.
2. В поле **Действует с** введите дату, когда новая среда становится действительной.

![Включение среды дополнения электронных накладных](media/e-Invoicing-services-feature-setup-Select-Enable-e-Invoicing-feature-Environment.png)

## <a name="organizations"></a>Организации

Дополнительная функция электронных накладных может использоваться одновременно в нескольких организациях.

- На странице **Дополнительные функции электронных накладных** на вкладке **Организации** выберите пункт **Общий доступ** , чтобы добавить организацию, в которой будет использоваться дополнительная функция электронных накладных.

Для прекращения совместного использования дополнительной функции электронных накладных в организации выберите вариант **Отменить общий доступ**.

## <a name="versions"></a>Версии

Версии помогают управлять жизненным циклом дополнительной функции электронных накладных, управляя ее статусом. Можно создать новую версию существующей дополнительной функции электронных накладных, или, если все настройки для дополнительной функции электронных накладных завершены, можно изменить статус функции на **Завершено** , затем на **Опубликовать**.

### <a name="create-a-new-version-of-an-existing-electronic-invoicing-add-on-feature"></a>Создание новой версии существующей дополнительной функции электронных накладных

1. На странице **Дополнительные функции электронных накладных** в левой сетке выберите дополнительную функцию электронных накладных.
2. На вкладке **Версии** выберите **Создать** , чтобы добавить новую версию дополнительной функции электронных накладных.

### <a name="change-the-status-of-the-electronic-invoicing-add-on-feature"></a>Изменение статуса дополнительной функции электронных накладных

Выполните следующие шаги для управления жизненным циклом дополнительной функции электронных накладных.

1. На странице **Дополнительные функции электронных накладных** в левой сетке выберите дополнительную функцию электронных накладных.
2. На вкладке **Версии** выберите **Изменить статус** , затем измените статус с **Черновик** на **Завершено**.
3. Вам будет предложено подтвердить, что вы хотите завершить дополнительную функцию электронных накладных и все ее компоненты. Выберите **Да** , чтобы подтвердить действие, или **Нет** , чтобы отменить его.

    > [!NOTE]
    > При выборе значения **Да** статус версий конфигурации, являющихся компонентами дополнительной функции электронных накладных, автоматически изменяется со значения **Черновик** на **Завершено**.

4. Выберите **Изменить статус** , затем измените статус с **Завершено** на **Опубликовать**.
5. Вам будет предложено подтвердить, что вы хотите опубликовать дополнительную функцию электронных накладных и все ее компоненты в глобальном репозитории. Выберите **Да** , чтобы подтвердить действие, или **Нет** , чтобы отменить его.

    > [!NOTE]
    > Когда выбрано значение **Да** , статус версий конфигурации автоматически изменяется с **Завершено** на **Общий**.
