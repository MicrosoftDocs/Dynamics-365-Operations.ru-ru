# <a name="prepayments-management"></a>Управление Предоплатами
[!include [banner](../includes/banner.md)]

По закону продавцы обязаны платить налог на добавленную стоимость (НДС) с предоплат, которые они получают от клиентов (покупателей). Затем они должны выдавать счета-фактуры на предоплату клиентам. Клиенты могут затем вычесть НДС на основе счета-фактуры на предоплату, полученной от продавца.

![Автоматически создается снимок экрана описания мобильного телефона](media/1%20Scheme%20english.jpg)


## <a name="processing-prepayments-from-customers"></a>Обработка предоплат клиентов
Когда продавцы выпускают счет-фактуру по предоплате клиенту, они также оплачивают указанную сумму НДС. Когда продавцы выпускают счет-фактуру на товары, они также оплачивают указанную сумму НДС и затем вычитают сумму НДС из счета-фактуры по предоплате.

В следующей таблице показаны транзакции главной книги (ГК) и операции пользователя для учета НДС в сценарии предоплаты.

| Транзакция ГК                         | Транзакция ГК                        | Пользовательская операция                                                                                  | Сведения об источнике                                                                                                                                      |
|----------------------------------------|---------------------------------------|--------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------|
| D 51 [Банк]                            | C 62 [Задолженность]/Авансы                 | Регистрация предоплаты от клиента                                                           | Страница **журнала платежей клиента**                                                                                                     |
| D 76 [Прочая задолженность и обязательства]/НДС | C 68 [Налоги]/НДС                      | Регистрация счета-фактуры                                                                                   | Страница **создания счета-фактуры** (**Расчеты с клиентами \> Периодические задачи \> Создание счетов-фактур на предоплату**)                                |
| D 62 [Задолженность] D 90 [Продажи]/НДС          | C 90 [Продажи]/Выручка C 68 [Налоги]/НДС | Отгрузка товаров: отгрузка товаров, выручка: исходящий НДС                                             | Регистрация заказа на продажу и счета-фактуры                                                                                                  |
| D 62 [задолженность]/Аванс D 68 [налоги]/НДС  | D 62 [Задолженность] C 76 [Прочее]/НДС         | Сопоставление предоплаты со счетом-фактурой на реверсирование НДС по отгрузке из счета-фактуры на предоплату | Сопоставление предоплаты и счета                                                                                                     |
| Нет                                   | Нет                                  | Регистрация счета-фактуры на предоплату в книге покупок для вычета НДС                                 | Страница **Книга покупок (обработка входящего НДС)** (**Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **Обработка входящего НДС**) |

Функциональные возможности обработки предоплат от клиентов позволяют выполнять следующие действия:

-   Настройка отдельных счетов ГК для разноски предоплат.
-   Расчет суммы НДС по авансовым платежам.
-   Печать счетов-фактур на предоплату.
-   Включите в книгу продаж счета-фактуры, которые были выпущены для предоплат.
-   Автоматически создает транзакцию для реверсированной суммы НДС из предоплат для сопоставленных платежей и счетов, а также включает в книгу покупок счета-фактуры на предоплату.

## <a name="setup"></a>Настройка
### <a name="set-up-a-posting-profile-for-prepayments"></a>Настройка профиля разноски для предоплат

Чтобы создать профиль разноски для предоплат, следуйте процедуре, описанной в разделе [Профили разноски по клиенту](../accounts-receivable/customer-posting-profiles.md).

> [!NOTE]
> Для удовлетворения требований российской законодательства следует использовать следующие настройки:
>
> - В поле **Итоговый счет** обычно должно быть установлено на счет учета **62 [Задолженность]**, субсчет **Полученные авансы**.
> - Поле **Счет налога по предоплате** обычно должно быть установлено на счет учета **76 [Налоги]**, субсчет **НДС**.

### <a name="set-up-a-sales-tax-payable-ledger-account"></a>Настройка счета учета уплачиваемого налога

1.  Перейдите в раздел **Налог** \> **Настройка** \> **Налог** \> **Группы разноски ГК**.
2.  Выберите группу разноски отдельной главной книги для полученных авансов.
3.  В поле **Исходящий налог** выберите счет учета для исходящего налога.

> [!NOTE]
> В соответствии с требованиями российского законодательства этот счет учета обычно должен быть счетом учета **68 [Налоги]**, субсчет **НДС**.

Дополнительные сведения о создании групп разноски ГК см. в разделе [Настройка групп разноски ГК для налога](../general-ledger/tasks/set-up-ledger-posting-groups-sales-tax.md).

### <a name="set-up-a-number-sequence-for-the-facture"></a>Настройка номерной серии для счета-фактуры
После регистрации предоплаты счет-фактура создается автоматически. Номер счета-фактуры создается на основе номерной серии, настроенной на странице Параметры главной книги.

1.  Перейдите в раздел **Главная книга** \> **Настройка главной книги** \> **Параметры главной книги**.
2.  На вкладке **Номерные серии** в поле **Код номерной серии** выберите код номерной серии для ссылки **Счет-фактура по предоплате**.

> [!NOTE]
> Выберите одну и ту же номерную серию для ссылок на **счет-фактуру по предоплате** и **счет-фактуру**. Таким образом, вы гарантируете, что соблюдаете законное требование для последовательной нумерации всех счетов-фактур.

### <a name="set-up-accounts-receivable-parameters-for-prepayments"></a>Настройка параметров модуля "Расчеты с клиентами" для предоплат

1. Перейдите в раздел **Расчеты с клиентами** \> **Настройка** \> **Параметры модуля расчетов с клиентами**.
2. На вкладке **Главная книга и налог** на экспресс-вкладке **платежа** настройте следующие поля:

    - Установите параметр **Налог по ваучеру журнала предоплат** на **да**, чтобы указать, что при разноске предоплаты необходимо рассчитывать налог. (Этот параметр является стандартным полем.)
    - В поле **Профиль разноски с ваучером журнала предоплат** укажите профиль разноски для предоплаты. (Это поле является стандартным полем.)
    - Установите для параметра **Реверс на дату накладной** значение **Да**, чтобы разнести НДС при отмене транзакций по предоплате во время сопоставления предоплаты и накладной на дату накладной.
    - В поле **Тип проводки по НДС** выберите тип транзакции, которая будет использоваться для НДС при отмене транзакции предоплаты во время сопоставления:

        - **Реверсировать** – Разноска отмен проводок НДС в виде реверсирований
        - **Реверсирование** – Разноска отмен проводки НДС как "красных" проводок сторно.

    - В поле **Налоговая группа для предоплаты** выберите налоговую группу для предоплаты.
    - В поле **Налоговая группа номенклатур** выберите налоговую группу номенклатур для предоплаты.
    - В поле **Обработка предоплаты** выберите **Простой**.
    - Установите параметр **Автоматическое создание счета-фактуры** на **Да**, чтобы автоматически создавать счета-фактуры при разноске предоплат.

> [!NOTE]
> Если установить параметр **Автоматическое создание счета-фактуры** на **Нет**, счета-фактуры по предоплате создаются в отложенном режиме. Значение этого параметра наследуется, и его можно изменить в следующих местоположениях:
>
> - На вкладке **Платежи** на странице **платежи клиента**. (Чтобы открыть эту страницу, перейдите к **Расчеты с клиентами \> Платежи \> Журнал платежей клиентов**, а затем, на странице **Журнал платежей клиентов** в области действий выберите **Строки**.)
>
> - В диалоговом окне **Конвертировать в предоплату**. (Чтобы открыть это диалоговое окно, перейдите **Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **Журнал книг продаж**, а затем, на странице **Журнал книг продаж** в области действий выберите **Функции \>Преобразовать в предоплату**. Затем на странице **Несопоставленные платежи** в области действий выберите **Обработка предоплаты**.)
>
> - Установите параметр **Наследовать аналитики накладной** на **Да**, чтобы копировать финансовые аналитики из накладной в платеж, когда используется предложение по оплате.

3. Закройте страницу.

## <a name="operations"></a>Operations

### <a name="register-a-prepayment-from-a-customer-and-print-the-facture"></a>Регистрация предоплаты от клиента и печать счета-фактуры

1. Перейдите в раздел **Расчеты с клиентами** \> **Платежи** \> **Журнал платежей клиентов**.
2. Выберите **Создать**, чтобы создать журнал платежей, а затем в поле **Имя** выберите имя платежа.
3. в области действий выберите **Строки**, чтобы создать строку журнала платежей, а затем на вкладке **список** введите требуемые сведения.
4. На вкладке **общие** в разделе **счет** в поле **Код соглашения** укажите код для соглашения, для которого была получена предоплата.
5. На вкладке **платеж** в разделе **документ** в полях **документ** и **Дата документа** укажите номер и дату платежного документа.
6. Убедитесь, что параметр **Ваучер журнала предоплат** установлен на **Да**.

> [!NOTE]
> Если параметр **Ваучер журнала предоплат** имеет значение **да**, поля **Настройка налоговой группы** и **Налоговая группа номенклатур** на вкладке **общие** и поле **Профиль разноски** на вкладке **платеж** устанавливаются автоматически в зависимости от настройки системы.

7. Убедитесь, что для параметра **Автоматическое создание счета-фактуры** установлено значение **Да**, чтобы автоматически создавать счета-фактуры при разноске предоплат.

> [!NOTE]
> Если установить параметр **Автоматическое создание счета-фактуры** на **Нет**, счета-фактуры по предоплате создаются в отложенном режиме.

8. В области действий выберите **Разнести** \> **Разнести**, чтобы разнести предоплаты.
9. Перейдите к **Расчеты с клиентами \> Запросы и отчеты \> Счет-фактура**, чтобы просмотреть созданный счет-фактуру по оплате.
10. В нижней части страницы **Журнал счетов-фактур** на вкладке **общие** в поле **Примечание** введите список товаров, для которых зарегистрирована предоплата.
11. Выберите **Заполнить детали строки**, а затем в диалоговом окне **Журнал счетов на оплату** установите флажок **метка** для всех счетов на оплату, для которых наименования товаров (работы и услуги) должны быть включены в описание счета-фактуры на предоплату. Дополнительные сведения см. в разделе [Счета на оплату](rus-invoice-payment.md)

![Диалоговое окно Журнал счетов на оплату](media/2%20Journal%20of%20invoices%20for%20payment.png)

12.  Нажмите **ОК**. Имена товаров добавляются в поле **примечания**.
13.  Выберите строку для счета-фактуры, затем в области действий выберите **Печать**.

### <a name="transform-a-customer-payment-to-a-customer-prepayment"></a>Преобразование платежа клиента в предоплату клиента

Если несопоставленные платежные документы (то есть часть суммы платежа) остаются на конец отчетного периода, платежные документы (или часть документов) должны быть преобразованы в предоплаты и включены в книгу продаж.

1. Разноска платежа клиента обычным способом.
2. Выберите **Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **Журнал книг продаж**.
3. В области действий выберите **функции** \> **преобразовать в предоплату**, чтобы открыть страницу **несопоставленные платежи**.
4. Установите флажок **Пометка** для строки платежа, которую нужно преобразовать в предоплату, а затем в области действий выберите **Обработка предоплаты**, чтобы открыть диалоговое окно **Конвертировать в предоплату**.
5. В поле **Дата** выберите дату проводки.

> [!NOTE]
>Если необходимо создать предоплату на дату проводки основного платежа, можно установить в поле **Та же дата** значение **Да**. Затем будут созданы проводки по предоплате и счета-фактуры на дату проводки.

6. Установите параметр **Автоматическое создание счета-фактуры** на **Да**, чтобы автоматически создавать счет-фактуру на предоплату.

> [!NOTE]
> Если установить параметр **Автоматическое создание счета-фактуры** на **Нет**, вы должны создать счета-фактуры по предоплате в отложенном режиме.

7. Выберите **ОК**, чтобы преобразовать платеж клиента в предоплату клиента.

> [!NOTE]
> После нажатия кнопки **ОК** проводки платежа реверсируются, и создаются проводки по предоплате. Проводки по предоплате датированы в соответствии с настройками в диалоговом окне **Преобразование в предоплату**. Счета-фактуры предоплаты создаются и включены в книгу продаж после ее обновления.

Также можно преобразовать платеж в предоплату с помощью страницы **Проводки по клиенту**. Перейдите в раздел **Расчеты с клиентами** \> **Клиенты** \> **Все клиенты**. В области действий на вкладке **Клиент** в группе **Проводки** выберите **Проводки** чтобы открыть страницу **Проводки по клиенту**. Выберите проводку платежа, а затем в области действий выберите **Обработка предоплаты** \> **Разноска** , чтобы преобразовать ее в предоплату.

Кроме того, можно преобразовать предоплату обратно в платеж. В этом случае ранее созданный счет-фактура на предоплату будет удален.

### <a name="create-factures-at-the-end-of-a-reporting-period"></a>Создание счетов-фактур в конце отчетного периода

В конце отчетного периода можно просмотреть предоплаты и создать счета-фактуры для всех предоплат, для которых на этапе разноски не создавались счета-фактуры автоматически.

1. Перейдите к **Расчеты с клиентами** \> **Периодические задачи** \> **Создание счетов-фактур на предоплату**. Страница **Создание счета-фактуры** показывает предоплаты, для которых не были автоматически созданы счета-фактуры.
2. Установите флажок **Пометка** для каждой предоплаты, для которой необходимо создать фактуру. (Можно также нажать кнопку **Пометить все** все в области действий, чтобы выбрать все предоплаты на странице.) Затем в области действий выберите **создать счет-фактуру**.
3. В диалоговом окне **Создать счет-фактуру** в поле **Дата регистрации** выберите дату регистраций счета-фактуры.

> [!NOTE]
> Выбранная дата должна быть датой в пределах периода книги продаж.

### <a name="settle-a-customer-prepayment-with-a-facture-for-shipment"></a>Сопоставление предоплаты клиента со счетом-фактурой для отгрузки

1. Перейдите в раздел **Расчеты с клиентами** \> **Клиенты** \> **Все клиенты**.
2. В области действий на вкладке **Сбор** в группе **сопоставление** выберите **Сопоставление счетов-фактур с оплатами**, чтобы открыть страницу **Сопоставление счетов-фактур с оплатами**. Эта страница позволяет фильтровать накладные по номеру счета-фактуры.
3. Установите флажок **Пометка** для каждой проводки, которую необходимо сопоставить, а затем в области действий выберите **обновить**. Сопоставляется накладная, которая является источником для счета-фактуры и платежа.

При сопоставлении проводок происходят следующие события:

-  Сумма платежа с субсчета **Полученные авансы** перемещается в субсчет **Дебиторская задолженность**.
-  Сумма НДС, которая начислена по предоплате, реверсируется.
-  Реверсированная предоплата включается на странице **Обработка входящего НДС** при обработке входящего НДС.
-  После обработки входящего НДС и обновления строк книги покупок запись реверсирования авансового платежа клиента появляется в книге покупок.

## <a name="processing-prepayments-to-suppliers"></a>Обработка предоплат поставщикам

Клиенты могут взять указанный НДС для вычета, когда они получают счет-фактуру для предоплаты от продавца. Когда покупатели получают товар от продавца, они могут взять указанный НДС для вычета. В то же время они необходимы для восстановления НДС из счета-фактуры на предоплату, которая была ранее удержана.

В следующей таблице показаны транзакции ГК и операции пользователя для учета НДС.

| Транзакция ГК                             | Транзакция ГК                                         | Пользовательская операция                                      | Сведения об источнике                                                                                                                                                                                                                                                                                                                                                                                                                                                                |

### <a name="set-up-a-posting-profile-for-prepayments"></a>Настройка профиля разноски для предоплат

Чтобы создать профиль разноски для предоплат, следуйте процедуре, описанной в разделе [Профили разноски по поставщикам](../accounts-payable/vendor-posting-profiles.md).

> [!NOTE]
> Для удовлетворения требований российской законодательства следует использовать следующие настройки:
>
>  - В поле **Итоговый счет** обычно должно быть установлено на счет учета **60 [Обязательства]**, субсчет **Выплаченные авансы**.
>  - Поле **Счет налога по предоплате** обычно должно быть установлено на счет учета **68 [Налоги]**, субсчет **НДС**.

### <a name="set-up-an-incoming-tax-payment-ledger-account"></a>Настройка счета учета для оплаты входящего налога

1. Перейдите в раздел **Налог** \> **Настройка** \> **Налог** \> **Группы разноски ГК**.
2. Выберите группу разноски отдельной главной книги для выплаченных авансов.
3. В поле **Оплата входящего налога** выберите счет учета для входящих налоговых платежей.

> [!NOTE]
> В соответствии с требованиями российского законодательства этот счет учета обычно должен быть счетом учета **76 [Прочая задолженность и обязательства]**, субсчет **НДС от предоплаты**.

### <a name="set-up-accounts-payable-parameters-for-prepayments"></a>Настройка параметров Расчеты с поставщиками для предоплат

1. Перейдите в раздел **Расчеты с поставщиками** \> **Настройка** \> **Параметры расчетов с поставщиками**.
2. На вкладке **Главная книга и налог** на экспресс-вкладке **платежа** настройте следующие поля:

    - Настройте параметр **Налог по предоплате в журнале платежей** на **Нет**, чтобы исключить разноску налогов до регистрации входящего счета-фактуры.
    - В поле **Профиль разноски для журнала платежей с предоплатой** укажите профиль разноски для платежа. 
    - Настройте параметр **Реверс на дату накладной** на **Да**, чтобы разнести реверсирующую проводку НДС на дату накладной.
    - В поле **Тип проводки по НДС** выберите тип транзакции, которая будет использоваться для НДС при отмене транзакции предоплаты во время сопоставления:

        - **Реверсировать** – Разноска отмен проводок НДС в виде реверсирований.
        - **Реверсирование** – Разноска отмен проводки НДС как "красных" проводок сторно.

    - В поле **Налоговая группа для предоплаты** выберите налоговую группу для предоплаты.
    - В поле **Налоговая группа номенклатур** выберите налоговую группу номенклатур для предоплаты.
    - В поле **Обработка предоплаты** выберите **Простой**.
    - Установите параметр **Создать фактуру на неразнесенные платежи** на **Да**, чтобы зарегистрировать счет-фактуру для неразнесенных предоплат.

3. На экспресс-вкладке **Книга покупок** в поле **Срок выписки счета-фактуры** введите количество дней, чтобы исключить счета-фактуры, выданные поставщиком. Исключение должно быть позже, чем определенный крайний срок.

> [!NOTE]
> Этот параметр действует только для счетов-фактур на предоплату. Если ввести **0** (ноль), счета-фактуры будут отражены на странице **Книга покупок (Обработка входящего НДС)**, независимо от даты счета-фактуры.

4. Закройте страницу.

### <a name="set-up-a-number-sequence-for-incoming-vat-processing"></a>Настройка номерной серии для обработки входящего НДС

1.  Перейдите в раздел **Расчеты с клиентами** \> **Настройка** \> **Параметры модуля расчетов с клиентами**.
2.  На вкладке **Номерные серии** в поле **Код номерной серии** выберите код номерной серии для ссылки **Ваучер ГК по обработке НДС**.

### <a name="register-a-suppliers-facture-for-a-prepayment"></a>Регистрация счета-фактуры поставщика для предоплаты

1. Перейдите к **Расчеты с поставщиками** \> **платежи** \> **Журнал платежей поставщиков** и создайте журнал.
2. В области действий выберите **Строки**, а затем на вкладке **Список** в строке журнала платежей задайте поля **дата**, **счет**, **Описание**, **дебет**, **Тип корр. счета**, **Корр. счет** и **валюта**.
3. На вкладке **общие** в разделе **счет** в поле **Код соглашения** укажите код соглашения, для которого была получена предоплата.
4. На вкладке **платеж** в разделе **документ** в полях **документ** и **Дата документа** укажите номер и дату платежного документа.
5. Убедитесь, что параметр **Ваучер журнала предоплат** установлен на **Да**.

> [!NOTE]
> Если **Ваучер журнала предоплат** имеет значение **да**, поля **Настройка налоговой группы** и **Налоговая группа номенклатур** на вкладке **общие** и поле **Профиль разноски** на вкладке **платеж** устанавливаются автоматически в зависимости от настройки системы.

> [!IMPORTANT]
> Чтобы предотвратить разноску налога с продаж до обработки входящих счетов-фактур на предоплату, необходимо установить для параметра **Налог по предоплате в журнале платежей** значение **Нет** на странице **Параметры модуля расчетов с поставщиками**. Кроме того, необходимо выбрать счет учета для входящих налоговых платежей в поле **Оплата входящего налога** на странице **Группы разноски ГК**.

6. Разноска журнала, который имеет предоплату поставщику.
7. Перейдите к **Расчеты с поставщиками** \> Периодические задачи \> Счет-фактура \> Создание счетов-фактур на предоплату.
8. в области действий выберите **Выбрать**, чтобы открыть диалоговое окно **Выбрать предоплаты**. Это диалоговое окно можно использовать для поиска необходимого платежа, например, кода поставщика и сведений о платежном поручении. Страница **Создать счет-фактуру на предоплату** показывает предоплаты поставщикам, для которых еще не созданы счета-фактуры. Можно зарегистрировать счет-фактуру для неразнесенной предоплаты.

9. Нажмите кнопку **ОК**, чтобы добавить строки для обработки. Затем на вкладке **Общие** в разделе **Счет-фактура** в поле **Счет-фактура** введите число полученных счетов-фактур.
10. В поле **Дата регистрации** введите дату регистрации счета-фактуры.
11. Если дата регистрации отличается от даты документа, установите поле даты **Дата счета-фактуры**.
12. Задайте поля **Налоговая группа** и **Налоговая группа номенклатур**.

> [!NOTE]
>
> - Рассчитанный налог будет записан в строку для созданного счета-фактуры. Установите флажок **отмечено** для каждой строки для счета-фактуры, а затем в области действий выберите **создать**.
>   
> - Если не заданы даты или номера счетов-фактур, выводится предупреждающее сообщение.

Созданный счет-фактура будет отражен на странице **Журнал счетов-фактур**, где в поле **Источник счетов-фактур** будет установлен тип **Ваучер журнала предоплат**.

### <a name="process-incoming-vat-on-the-received-facture-for-the-prepayment"></a>Обработка входящего НДС по полученному счету-фактуре по предоплате

После регистрации счета-фактуры на предоплату сумму НДС из счета-фактуры на предоплату можно обработать для вычета.

1. Перейти к **Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **Обработка входящего НДС**.
2. в области действий выберите **выбрать**, чтобы найти требуемый счет-фактуру.

> [!NOTE]
> В поле **операции** для счета-фактуры на предоплату имеется значение **предоплата**.

3. Пометьте счет-фактуру и выполните разноску операции. После разноски имеется налоговая проводка и проводка ГК для суммы НДС:

    - **Дебет 68 [налог]/НДС — Кредит 76 [Прочая задолженность и обязательства]/НДС от предоплаты — по сумме НДС**

> [!NOTE]
> После обработки входящего НДС удалить счет-фактуру на предоплату невозможно. Для отмены обработки входящего НДС используется страница **Книга покупок (Отмена обработанного НДС)** (**Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **Отмена обработанного НДС**).

4. Перейти к **Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **журнал обработки НДС** для просмотра обработанных счетов-фактур на предоплату.
5. Перейти к **Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **Журнал книг покупок** для создание и обновление книги покупок за отчетный период. Чтобы просмотреть обработанную счет-фактуру в книге покупок, выберите **строки** в области действий.

### <a name="recover-previously-accepted-deductible-vat-on-prepayment"></a>возмещение ранее принятого подлежащего вычету НДС по предоплате

При получении счета-фактуры для покупки товаров от продавца клиент вычитает указанный НДС по счету-фактуре на покупку. В том же налоговом периоде счет-фактура должен возмещать НДС в бюджете, вычтенный из полученной предоплаты.

После разноски входящего счета-фактуры на товары, работы и услуги необходимо выполнить дополнительные действия. Например, если продавец производит доставку, то будет выполнено обновление входящей накладной и входящего счета-фактуры для этой поставки, и накладная и ранее зарегистрированная предоплата будут сопоставлены.

1. Перейти к **Расчеты с поставщиками** \> **Периодические задачи** \> **Книга покупок** \> **Обработка входящего НДС**.
2. в области действий выберите **Выбрать**, чтобы найти необходимый платеж, а затем нажмите **OK**, чтобы добавить строки для обработки.
3. Пометьте счет-фактуру и выполните разноску операции.
4. Чтобы возместить ранее удержанный НДС из полученного счета-фактуры на предоплату, необходимо настроить параметры для возмещения НДС. Перейдите к **Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **Параметры обработки НДС** и создайте строку.
5. В поле **Код операции по НДС** введите код операции для обработки НДС.
6. В поле **тип операции** введите **возмещение НДС**.
7. В поле **тип возмещения** выберите **ваучер журнала предоплат**.
8. Установите для параметра **По умолчанию** значение **Да**, чтобы указать, что данная проводка является проводкой по умолчанию для обработки НДС.
9. Установите для параметра **Включить в книгу** значение **Да**, если счета-фактуры, обрабатываемые с использованием этого кода проводки, должны быть включены в книгу продаж.
10. Чтобы обработать исходящий НДС, перейдите к **Расчеты с клиентами** \> **периодические задачи** \> **Книга продаж** \> **Обработка исходящего НДС**.

Для сопоставленных платежей, в которых входящий НДС был обработан ранее, счета-фактуры имеют тип проводки **Сторно предоплаты**. Номер счета-фактуры соответствует номеру зарегистрированного счета-фактуры на предоплату. Сумма НДС для возмещения равна сумме обработанного входящего НДС для полученных товаров, которые попадают в сопоставленную часть накладной.

11. Пометьте счет-фактуру и выполните разноску операции. Система создает налоговую проводку и проводку ГК для обработанной суммы налога.

> [!NOTE]
> Проводка по возмещению НДС является реверсированием проводки по вычету НДС в главной книге:
>
>  **Дебет 76 [Прочая задолженность и обязательства] / НДС по предоплате — Кредит 68 [Налоги] / НДС — по сумме НДС**

12. Перейти к **Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **журнал обработки НДС** для просмотра обработанных счетов-фактур.
13. Выберите **Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **Журнал книг продаж**. Обновление книги продаж и проверка того, что счет-фактура отражается в журнале книг продаж для обработанной суммы.

По умолчанию, если для одной и той же счета-фактуры на предоплату в одном и том же периоде было несколько возмещений НДС, счет-фактура на предоплату будет отражена в нескольких строках журнала книг продаж. При печати книги продаж используйте параметр **Сгруппировать по фактурам**. Таким образом, отчет по счету-фактуре будет отражать одну строку, в которой указана общая стоимость НДС, которая была возмещена за период.

Невозможно закрыть журнал книг продаж, если имеются входящие счета-фактуры на предоплату в периоде, для которого не был возмещен НДС.

Отмена сопоставления по предоплате и накладной невозможна, если НДС был возмещен в период. Вы должны сначала отменить возмещение НДС в этот период.
Для отмены возмещения НДС используется страница **Книга продаж (Отмена обработанного НДС)** (**Расчеты с клиентами** \> **Периодические задачи** \> **Книга продаж** \> **Отмена обработанного НДС**).

Невозможно отменить обработку входящего НДС, если только не отменено возмещение НДС по предоплате, которая сопоставляется с накладной.
