# <a name="get-started-with-the-tax-calculation-add-in-preview"></a>Начало работы с надстройкой расчета налога (предварительная версия)

[!include [banner](../includes/banner.md)]

[!include [banner](../includes/preview-banner.md)]

В этой теме приводятся сведения о начале работы с надстройкой расчета налога. Сначала приводятся инструкции по выполнению шагов настройки в службах Microsoft Dynamics Lifecycle Services (LCS), Regulatory Configuration Service (RCS), Dynamics 365 Finance и Dynamics 365 Supply Chain Management. Затем рассматривается общий процесс использования надстройки расчета налога в проводках Finance и Supply Chain Management.

Настройка состоит из четырех основных шагов:

1. В LCS установите надстройку расчета налога.
2. В RCS настройте функцию расчета налога. Эта настройка не является специфичной для юридического лица. Она может использоваться для всех юридических лиц в модулях Finance и Supply Chain Management.
3. В Finance и Supply Chain Management настройте параметры надстройки расчета налога по юридическим лицам.
4. В Finance и Supply Chain Management создайте проводки, такие как заказы на продажу, и используйте надстройку расчета налога, чтобы определить и рассчитать налоги.

## <a name="prerequisites"></a>Необходимые условия

Прежде чем вы сможете выполнить процедуры из данной темы, необходимо выполнить следующие предварительные требования:

- У вас есть доступ к вашей учетной записи LCS и вы развернули проект LCS со средой уровня 2 (или выше), в котором работает Dynamics 365 версии 10.0.18 или новее.
- У вас есть доступ к вашей учетной записи RCS.
- Вы связывались с корпорацией Майкрософт, чтобы включить фокус-тестирование в развернутой среде Finance или Supply Chain Management.

## <a name="set-up-the-tax-calculation-add-in-in-lcs"></a>Настройка надстройки расчета налога в LCS

1. Вход в [LCS](https://lcs.dynamics.com)
2. Выполните настройку для интеграции Microsoft Power Platform. Дополнительные сведения см. в разделе [Обзор надстроек](../../fin-ops-core/dev-itpro/power-platform/add-ins-overview.md).
3. Выберите одну из развернутых непроизводственных сред, затем выберите **Установить новую надстройку**.
4. Выберите **Расчета налога (предварительная версия)**.
5. Прочитайте и примите условия, затем выберите **Установить**.

## <a name="set-up-the-tax-calculation-add-in-in-rcs"></a>Настройка надстройки расчета налога в RCS

Шаги данного раздела не относятся к конкретному юридическому лицу. Эту процедуру необходимо выполнить только один раз, и ее можно выполнить в любом юридическом лице в RCS.

1. Войдите в [RCS](https://marketing.configure.global.dynamics.com/).
2. В Finance в рабочей области **Электронная отчетность** добавьте нового поставщика конфигурации. В качестве имени поставщика используйте название своей компании. Дополнительные сведения см. в [Создание поставщиков конфигураций и пометка их как активных](../../fin-ops-core/dev-itpro/analytics/tasks/er-configuration-provider-mark-it-active-2016-11.md).
3. Выберите только что созданного поставщика конфигурации, затем выберите **Установить активной**.
4. Выберите поставщика конфигурации **Microsoft**, затем выберите **Репозитории**.
5. В поле **Тип** выберите **Глобальный**.
6. Выберите **Открыть**.
7. Перейдите в **Модель налоговых данных**, разверните дерево файлов, затем выберите **Налоговая конфигурация — Европа**.
8. Выберите последнюю версию, затем выберите **Импорт**.
9. Вернитесь в рабочую область **Функции глобализации (предварительная версия)**, выберите **Функции**, выберите плитку **Расчет налога**, затем выберите **Добавить**.
10. Выбор один из следующих типов функции:

    - **Создать функцию** — создание настройки функции с пустым содержимым.
    - **На основе существующей функции** — создание функции из существующей функции и копирование содержимого из существующей настройки функции.

11. Введите название и описание новой функции, затем выберите **Создать функцию**.

    После создания функции черновая версия создается автоматически.

12. Выберите черновую версию функции, затем выберите **Изменить**. Будет заполнена страница **Настройка расчета налога**.
13. Выберите **Версия конфигурации**. Должна отобразиться версия конфигурации, которая была импортирована на шаге 8.

    Майкрософт предоставляет конфигурацию налога по умолчанию для надстройки расчета налога. Эта конфигурация охватывает большинство требований к поведению при расчете налогов. Она будет обновлена в соответствии с обратной связью от рынка. Если необходимо расширить конфигурацию, чтобы она удовлетворяла определенным требованиям, см. раздел [Как построить расширение в налоговой службе](https://go.microsoft.com/fwlink/?linkid=2138483) для получения сведений о том, как создавать и выбирать собственную конфигурацию налога.

    После выбора пункта **Версия конфигурации** появятся несколько дополнительных вкладок:

    - **Налоговые коды** — эта вкладка является обязательной для службы расчета налогов. Она используется для ведения основных данных для налоговых кодов. Все налоговые коды, созданные на этой вкладке, автоматически синхронизируются с Finance при включении текущей версии настройки функции налогов в юридическом лице.
    - **Применимость налоговых кодов** — эта вкладка является обязательной для надстройки расчета налога. Она используется для определения матрицы, определяющей налоговый код, налоговую группу и налоговую группу номенклатур. Определяемый налоговый код используется для расчета суммы налога. Значения полей **Налоговый код**, **Налоговая группа** и **Налоговая группа номенклатуры** возвращаются в Finance.
    - **Применимость номера налоговой регистрации клиента** — эта вкладка является необязательной для надстройки расчета налога. При наличии нескольких налоговых регистрационных номеров для одного клиента надстройка расчета налога может автоматически определить правильный налоговый регистрационный номер. В матрице на этой вкладке определяются правила, используемые надстройкой для выполнения определения. В противном случае Finance и Supply Chain Management продолжат использовать налоговый регистрационный номер по умолчанию в налогооблагаемых документах для проводок по продажам.
    - **Применимость номера налоговой регистрации поставщика** — эта вкладка является необязательной для надстройки расчета налога. При наличии нескольких налоговых регистрационных номеров для одного поставщика надстройка расчета налога может автоматически определить правильный налоговый регистрационный номер. В матрице на этой вкладке определяются правила, используемые надстройкой для выполнения определения. В противном случае Finance и Supply Chain Management продолжат использовать налоговый регистрационный номер по умолчанию в налогооблагаемых документах для проводок по покупкам.
    - **Применимость кода списка** — эта вкладка является необязательной для надстройки расчета налога. Она может помочь автоматически определить значение поля **Код списка** с помощью более гибких и настраиваемых правил. В матрице на этой вкладке вы можете определить правила, используемые надстройкой для выполнения определения. В противном случае Finance и Supply Chain Management продолжат использовать код по умолчанию в налогооблагаемых документах.

14. На вкладке **Коды налога** выберите **Добавить** и введите налоговый код и описание.
15. Выберите **Налоговый компонент**. Налоговый компонент — это группа методов расчета налогов, которая была определена в предыдущей версии выбранной налоговой конфигурации. Доступны следующие налоговые компоненты:

    - По чистой сумме
    - По валовой сумме
    - По количеству
    - По марже
    - Налог на налог

16. Нажмите **Сохранить**. В зависимости от выбранного налогового компонента становятся доступны дополнительные поля.
17. Используйте следующие параметры для определения природы налогового кода:

    - Не облагается
    - Является налогом за пользование
    - Является возмещением расходов
    - Исключить при расчете базовой суммы

    Для сценария налога на использование настройте один налоговый код, который имеет положительную налоговую ставку, и пометьте его как **Налог за пользование**.

    Для сценария с возмещением расходов настройте два налоговых кода, один из которых имеет положительную налоговую ставку, а второй имеет отрицательную ставку налога, но одинаковое значение ставки. Отметьте отрицательный налоговый код как **Является возмещением расходов**. Дополнительные сведения о решении налога с покупателя в модуле Finance см. в разделе [Механизм возмещения расходов для схемы НДС/GST](emea-reverse-charge.md).
    
    Для некоторых типов налогов, которые должны быть исключены из расчета суммы налоговой базы для проводок с инклюзивными ценами, например, таможенный налог в некоторых странах, установите флажок **Исключить из базовой суммы**.

    Поддерживайте ставки налога и лимиты сумм налога для этого налогового кода.

18. Повторите шаги с 14 по 17 для добавления всех остальных требуемых налоговых кодов.
19. На вкладке **Применимость кодов налогов** выберите столбцы, необходимые для определения правильного налогового кода, затем выберите **Добавить**.
20. Введите или выберите значения для каждого столбца. Поля **Налоговый код**, **Налоговая группа** и **Налоговая группа номенклатуры** выводятся из этой матрицы.
21. Повторите шаги с 19 по 20, чтобы настроить применимость номеров налоговой регистрации клиентов, номеров налоговой регистрации поставщиков и кодов списка.
22. Выберите **Сохранить** и закройте страницу.
23. Выберите **Изменить статус** \> **Завершить**. После того как статус изменится на **Завершено**, версию нельзя будет изменить.
24. Выберите **Изменить статус** \> **Опубликовано**. Эта версия настройки налоговых функций будет отправлена в глобальный репозиторий и будет видна для каждого юридического лица в Finance.

## <a name="dynamics-365-setup"></a>Настройка Dynamics 365

После завершения настройки в RCS, как описано в предыдущем разделе, будет опубликована версия функции налогов. Выполните следующие действия, чтобы настроить надстройку расчета налога в Finance.

Настройка в этом разделе выполняется по юридическим лицам. Необходимо настроить ее для каждого юридического лица, для которого требуется включить надстройку расчета налогов в Finance.

1. В Finance перейдите к пункту **Налог** \> **Настройка** \> **Конфигурация налога** \> **Настройка надстройки расчета налога (предварительная версия)**.
2. На вкладке **Общие** установите следующие поля:

    - **Включить надстройку расчета налога** — установите этот флажок, чтобы включить надстройку расчета налога для юридического лица. Если надстройка расчета налога не активирована для текущего юридического лица, юридическое лицо будет продолжать использовать существующий налоговый модуль для определения и расчета налога.
    - **Настройка функции** — выберите для юридического лица опубликованную настройку и версию функции налога. Дополнительные сведения о порядке настройки и выполнения опубликованной функции налога см. в предыдущем разделе данной темы.
    - **Бизнес-процесс** — выберите бизнес-процессы, которые необходимо включить для надстройки расчета налога.
    - **Включить корректировку налогового кода** — установите для этого параметра значение **Да**, чтобы включить корректировки налогового кода на странице налога.

3. На вкладке **Расчет** определите ожидаемое правило округления для юридического лица.
4. На вкладке **Обработка ошибок** определите ожидаемый метод обработки ошибок для юридического лица. Для каждого кода результата из надстройки расчет налога доступны три варианта:

    - Нет
    - Предупреждение
    - Ошибка

5. Сохраните настройку надстройки расчета налога.
6. Повторите шаги с 1 по 5 для каждого дополнительного юридического лица.

## <a name="transaction-processing"></a>Обработка проводок

После выполнения всех процедур настройки можно использовать надстройку расчета налога, чтобы определить и рассчитать налог в Finance. Шаги для обработки проводок остаются неизменными. В Finance версии 10.0.18 поддерживаются следующие проводки:

- Процесс продаж

    - Предложение по продажам
    - Заказ на продажу
    - Подтверждение
    - Сборочный лист
    - Отборочная накладная
    - Накладная заказа на продажу
    - Кредит-нота
    - Заказ на возврат
    - Расходы заголовка
    - Накладные расходы по строке

- Процесс покупки

    - Заказ на покупку
    - Подтверждение
    - Список прихода
    - Поступление продуктов
    - Накладная по покупке 
    - Расходы заголовка
    - Накладные расходы по строке
    - Кредит-нота
    - Заказ на возврат
    - Заявка на закупку
    - Расходы строки заявки на покупку
    - Запрос предложения
    - Расходы заголовка запроса предложения
    - Расходы строки запроса предложения

- Процесс запасов

    - Заказ на перемещение — отгрузка
    - Заказ на перемещение — получение
