# <a name="inventory-reports"></a>Отчеты о запасах
[!include [banner](../includes/banner.md)]


В России доступно четыре локальных отчета о запасах:

- Оборотная ведомость по складу
- Ведомость прихода-расхода
- Корректировки
- Коррекция (вал.)

## <a name="preliminary-setup"></a>Предварительная настройка

Прежде чем можно будет создать локальные отчеты о запасах, выполните следующие шаги для настройки номерных серий для комбинации аналитик запасов в наличии.

1. Перейдите в раздел **Управление запасами** \> **Настройка** \> **Параметры управления запасами и складами**.
2.  На вкладке **Номерные серии** выберите код номерной серии для ссылки **Комбинация аналитик запасов в наличии на указанную дату**, затем назначьте номерную серию в поле **Код номерной серии**.

## <a name="inventory-balance-turnover"></a>Оборотная ведомость по складу

С помощью запроса **Оборотная ведомость по складу** можно просмотреть сальдо и обороты в контексте аналитик. Можно выбрать аналитики для включения в отчет и указать критерии выбора для аналитик при использовании фильтров.

### <a name="calculate-on-hand-inventory-on-inventory-closing-date"></a>Вычисление запасов в наличии на дату закрытия склада

Чтобы оптимизировать производительность запроса **Оборотная ведомость по складу**, используйте периодическую процедуру **Запасы в наличии на указанную дату**. Запасы в наличии рассчитываются на дату, когда запасы ранее были закрыты.

1. Перейдите в раздел **Управление запасами \> Периодические задачи \> Запасы в наличии на указанную дату**.
2. На странице **Расчет запасов в наличии на указанную дату** на вкладке **Параметры** в поле **Дата расчета запасов в наличии** укажите дату закрытия запасов, для которой должен быть выполнен расчет запасов. По умолчанию это первая дата закрытия запасов без расчета запасов.
3. Установите переключатель **Обновить вычисления** в значение **Да**, если необходимо стереть результаты предыдущего расчета запасов на указанную дату закрытия запасов и выполнить полный расчет запасов.

    ![Страница "расчет запасов в наличии на указанную дату"](media/1_Calculation_on-hand_inventory_on_date.png)

4. После завершения расчета запасов на странице **Закрытия и корректировки** для соответствующей строки закрытия запасов на вкладке **Сеанс** для параметра **Расчет запасов в наличии на указанную дату** будет установлено значение **Да**.

    ![Страница Закрытие и коррекция](media/2_Closing_and_adjustment.png)

Если попытаться снова выполнить расчет запасов для закрытия запасов, для которых параметр **Расчет запасов в наличии на указанную дату** имеет значение **Да**, а параметр **Обновить вычисления** имеет значение **Нет**, возникнет ошибка.

Результаты расчета запасов в наличии на дату закрытия запасов удаляются из системы при отмене соответствующего закрытия запасов.

### <a name="inventory-balance-turnover-inquiry"></a>Запрос оборотной ведомости по складу

1. Перейдите к пункту **Управление запасами** \> **Запросы и отчеты** \> **Отчеты о проводках (Россия)** \> **Оборотная ведомость по складу**.

    ![Страница Оборотная ведомость по складу](media/3_Inventory_balance_turnover.png)

2. В диалоговом окне **Оборотная ведомость по складу** на вкладке **Общие** в полях **Начальная дата** и **Конечная дата** укажите период отчета.
3. Задайте для параметра **Развернуть обороты** значение **Да**, чтобы разбить столбцы товарооборота на две группы в результатах запроса:

    - Внутренний оборот, который *не* изменяет итоговые сальдо компании. Сюда входят перемещения запасов в журналах перемещения запасов и заказах на перемещение.

        >[!NOTE]
        > Следующие документы не рассматриваются как внутренний оборот, так как номенклатура со склада может быть списана:
        >
        > - Карантинные заказы
        > - Отходы заказа на перемещение
        > - Заказы контроля качества

    - Внешний оборот, который *изменяет* итоговые сальдо компании. Сюда входят перемещения запасов по заказам на покупку или продажу и журналы запасов.

4. Установите для параметра **Показывать физический товарооборот** значение **Да**, чтобы отделить физический оборот от финансового оборота. Физические обороты включают в себя отборочные накладные и производственные заказы, отгрузочные накладные, журналы побочных продуктов и журналы приемки для производственных заказов. Финансовые обороты включают счета на покупку или продажу, журналы запасов и расчет производственных заказов. Физический оборот будет выделен в отдельную группу столбцов. Если для параметра **Показывать физический товарооборот** установлено значение **Нет**, физический оборот добавляется в финансовый оборот.

  > [!NOTE]
  > При использовании механизма физического сторно для складских проводок во время финансового обновления, физический оборот и сторно физический оборот для финансово обновленных складских проводок отражается в одном столбце как приход или расход, но с противоположными знаками. То есть, они в сумме дают ноль, если в период запроса включены как физические, так и финансовые даты складской проводки. Если физическая дата включена в период запроса, а финансовая — нет, запрос будет содержать только физическое перемещение, но обратные/сторнированные физические перемещения и финансовые перемещения не будет учитываться.
  >
  > Например, для следующих складских проводок отображаются покупка и кредит-нота для покупки.

  >  | Вид документа                | Тип проводки по запасам | Корректировка по кредиту (физическая) | Количество |
  >  |------------------------------|-------------------------------|------------------------------|----------|
  >  | Заказ на покупку               | Чек                       | Нет                           | 10       |
  >  | Кредит-нота для покупки | Расход                         | Да                          | \-3      |
  >
  > Если аналитика **Сторно** не выбрана, в отчете будет показана строка со следующими количествами:
  >
  >  | Количество (приход) | Количество (расход) |
  >  |--------------------|------------------|
  >  | 7                  | 0                |
  >
  > Если физическое или финансовое обновление складской проводки не было разнесено в главную книгу, значение в аналитике **Балансовый счет** для складских проводок остается пустым.

5. Установите для параметра **Показать итоги** значение **Да**, чтобы просмотреть строки с промежуточными суммами по аналитикам.
6. Установите для параметра **Показывать нулевые обороты** значение **Да**, чтобы просмотреть строки с нулевым оборотом (но с ненулевым начальным сальдо) за период. Этот параметр выбирается автоматически, если выбран параметр **Показать итоги**.
7. Установите для параметра **Показать имя номенклатуры** значение **Да**, чтобы в дополнение к коду продукта отображалось название продукта.
8. На вкладке **Аналитики** выберите аналитики, которые требуется включить в отчет.

    ![Отчет по обороту сальдо запасов, вкладка Аналитики](media/4_Inventory_balance_turnover.png)

    Доступны следующие аналитики:
    
     - **Код номенклатуры**
     - **Складские аналитики**
     - **Вид деятельности**: если у складского перемещения нет аналитики **Профиль запасов**, отображается значение **Не указано**.
     - **Сальдо по счету**
     - **Договор**
     - **Сторно** позволяет отделить проводки сторно.
       
      - Если выбрана аналитика **Сторно**, складские проводки сторно отдельно отображаются в отчете отдельно на отдельных строках. Эти строки имеют значение **Да** в столбце **Сторно**. Проводки отображаются с другим направлением и знаком суммы. Сторнирование входящих складских проводок отображается в столбце **Расход** с отрицательным знаком, а сторнирование исходящих складских проводок отображается в столбце **Приход** с отрицательным знаком.        
      - Если аналитика **Сторно** не выбрана, складские проводки сторно уменьшают значение исходной проводки в соответствующих столбцах. Сторнирование входящих складских проводок имеет уменьшенное значение в столбце **Расход**, а сторнирование исходящих складских проводок имеет уменьшенную стоимость в столбце **Приход**.
      
      Например, для следующих складских проводок показаны покупка и кредит-нота для покупки:

      | Вид документа                | Тип проводки по запасам | Корректировка по кредиту (физическая) | Количество |
      |------------------------------|-------------------------------|------------------------------|----------|
      | Заказ на покупку               | Чек                       | Нет                           | 10       |
      | Кредит-нота для покупки | Расход                         | Да                          | \-3      |

      Если аналитика **Сторно** выбрана, в отчете будут показаны строки со следующими количествами:

      | Сторно | Количество (приход) | Количество (расход) |
      |--------|--------------------|------------------|
      | Нет     | 10                 | 0                |
      | Да    | \-3                | 0                |

9. На вкладке **Включаемые записи** используйте фильтр для определения проводок, которые будут включены в отчет.
10. Щелкните **ОК**, чтобы создать отчет.

    ![Отчет по обороту сальдо запасов, Стандартное представление](media/5_Inventory_balance_turnover.png)

> [!NOTE]
> В столбце **Тип строки** можно просмотреть тип строки: **Проводки** или **Итоги**
>
> Столбцы количества (**Количество (в начале)**, **Количество (в конце)**, **Количество (приход)**, **Количество (расход)**) отображаются только в том случае, если **Код номенклатуры** выбран в качестве одной из аналитик отчета. В этих полях данные отображаются в единицах измерения запасов для данной номенклатуры.
> 
> Если дата соответствующих складских сопоставлений попадает в период запроса, поля с суммами, **Сумма (в начале)**, **Сумма (приход)**, **Сумма (расход)**, **Сумма (в конце)** содержат финансовую сумму соответствующих складских проводок в валюте учета и корректировки финансовой суммы.
>
> Количества и суммы в отчете также включают складские проводки, которые были физически обновлены. Используется физическая сумма складской проводки. Складские проводки, которые обновляются финансово, включаются в запрос с датой финансовой проводки. Складские проводки, которые обновляются только физически, включаются в запрос с датой физической проводки. Сопоставления запасов отражаются в отчете с датой сопоставления запасов.

11. В зависимости от типа складской проводки запрос определяет направление оборота запасов следующим образом:

    | Тип проводки по запасам         | Направление                                                             |
    |---------------------------------------|-----------------------------------------------------------------------|
    | Заказ на покупку                        | Чек                                                               |
    | Заказ на продажу                           | Расход                                                                 |
    | Рабочий                            | Чек                                                               |
    | Строка производства                       | Расход                                                                 |
    | Побочные продукты                           | Чек                                                               |
    | Прочие типы проводок по запасам | Приход для входящих проводок и расход для исходящих проводок |

12. Если выбрана аналитика **Код номенклатуры**, можно выбрать строку с типом строки, **Проводка**, и на панели **Область действий** выбрать **Проводка**. На открытой странице **Складские проводки** можно просмотреть список проводок, в которых сформированы обороты в выбранной строке. Если финансовая дата не попадает в период запроса, но в рамках периода запроса имеются сопоставления запасов, проводка не будет отражена на этой странице.
13. Выберите **Пересчитать**, чтобы задать новые параметры запроса и повторно создать результаты.
14. Выберите **Печать** для просмотра или сохранения отчета в виде документа Microsoft Excel. На экспресс- вкладке **Параметры** установите для параметра **Диапазон печати** значение **Да**, если требуется, чтобы в отчете отображались фильтры, указанные в параметрах запроса, а также фильтры, применяемые к результатам запроса.

### <a name="clean-up-temporary-data"></a>Очистка временных данных

При расчете данных для запроса **Оборотная ведомость по складу** система создает временные данные, но сохраняет данные в постоянных таблицах базы данных. После закрытия формы запроса данные удаляются из системы. Однако временные данные могут быть удалены не полностью (например, в случае системных сбоев). Для очистки временных данных используются периодические операции **Расчет запасов в наличии на указанную дату** и **Очистка ссылок на записи**.

1. Перейдите в раздел **Управление запасами** \> **Периодические задачи** \> **Очистка** \> **Расчет запасов в наличии на указанную дату**.

    ![Страница "Очистка расчета запасов в наличии на указанную дату"](media/6_Cleanup_on-hand_inventory_calculation_on_date.png)

2. На экспресс-вкладке **Параметры** в поле **Создано до** укажите дату. Временные данные с датой создания до указанной даты будут удалены. Укажите дату, равную текущей дате минус один (1) день, чтобы избежать удаления данных, используемых другим пользователем в данный момент.
3. Нажмите **ОК**.
4. Перейдите к пункту **Администрирование организации** \> **Периодические** \> **Очистка ссылок на записи**.

    ![Страница очистки ссылок на записи](media/7_Record_references_cleanup.png)

5. На экспресс-вкладке **Параметры** в поле **Создано до** укажите дату. Все временные данные с датой создания до указанной даты будут удалены.
6. Нажмите **ОК**.

## <a name="pay-sheet-of-receipts-and-expenditure-report"></a>Отчет "Ведомость прихода-расхода"

В этом отчете отображаются приходы и расходы по всем номенклатурам.

1. Выберите **Управление запасами** \> **Запросы и отчеты** \> **Отчеты о проводках** \> **Ведомость прихода-расхода**.

    ![Отчет Ведомость прихода-расхода](media/8_Pay-sheet_of_receipts_and_expenditure.png)

2.  В диалоговом окне **Ведомость прихода-расхода** на экспресс-вкладке **Параметры** укажите значение параметра **Исключить сторно**:

    - Если для параметра **Исключить сторно** выбрано значение **Да**, система отображает обороты по сторно в общих складских передвижениях без выделения оборотов по сторно в отдельную строку.
    - Если для параметра **Исключить сторно** выбрано значение **Нет**, система отображает строку с исходными проводками для каждой номенклатуры и строку с проводками сторно, если таковые имеются. Если исходная проводка и проводка сторно находятся в одном и том же отчетном периоде, они не добавляют обороты для номенклатуры, но если они находятся в разных периодах, проводка, попадающая в период запроса, повлияет на обороты этого периода.
    
3. На экспресс-вкладке **Включаемые записи** выберите **Фильтр**, чтобы применить фильтры для отчета.
4. Выберите **ОК**, чтобы создать отчет.

    ![Отчет по созданной ведомости прихода-расхода](media/9_Pay-sheet_of_receipts_and_expenditure.png)

## <a name="adjustments-report"></a>Отчет по корректировкам

В этом отчете отображаются все корректировки, которые были выполнены во время пересчета или закрытия склада. Для этого перейдите в раздел **Управление запасами** \> **Запросы и отчеты** \> **Отчеты о проводках** \> **Корректировки**. Укажите параметры, затем выберите **ОК**, чтобы создать отчет.

![Отчет о созданных корректировках](media/10_Adjustments.png)

## <a name="adjustments-in-currency-report"></a>Корректировка в отчете по валюте

В этом отчете отображаются все корректировки, которые были выполнены во время пересчета или закрытия склада, в валюте отчетности. Для этого перейдите в раздел **Управление запасами** \> **Запросы и отчеты** \> **Отчеты о проводках** \> **Корректировка (вал.)**. Укажите параметры и выберите **ОК**, чтобы создать отчет.
