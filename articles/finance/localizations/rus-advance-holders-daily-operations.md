# <a name="daily-operations-for-advance-holders-in-russia"></a>Ежедневные операции по подотчетным лицам в России

[!include [banner](../includes/banner.md)]

В этой теме содержится информация о ежедневных операциях по подотчетным лицам для России. К этим операциям относится обработка денежных средств и исходящих сальдо.

## <a name="create-and-post-disbursement-slips-for-advance-holders"></a>Создание и разноска расходных кассовых ордеров для подотчетных лиц

Эта процедура используется для создания и разноски расходных кассовых ордеров со сведениями о подотчетном лице. Сальдо по подотчетному лицу разносятся на балансовый счет соответствующего сотрудника.

1. Выберите **Управление банком и кассовыми операциями** \> **Проводки по кассе** \> **Журнал кассовых ордеров**.
2. Выберите **Создать**, чтобы создать журнал кассовых ордеров. 
3. На странице **Строки** в столбце **Тип корр. счета** выберите **Подотчетное лицо**. Затем выберите подотчетное лицо в столбце **Корр. счет** и введите сумму в столбце **Кредит**.
4. Утвердите и разнесите расходный кассовый ордер. 
5. Для запроса сведений о связанных проводках и сальдо по подотчетному лицу, выберите **Расчеты с поставщиками** \> **Подотчетные лица** \> **Подотчетные лица**, выберите сотрудника, а затем выберите **Проводки** или **Сальдо**.

## <a name="create-and-post-vendor-invoices-that-have-advance-holder-details"></a>Создание и разноска накладных поставщиков со сведениями о подотчетных лицах

Эта процедура используется для создания и разноски накладных поставщиков со сведениями о подотчетном лице. Сальдо подотчетных лиц разносятся на балансовый счет сотрудника, а не балансовый счет поставщика.

1. Выберите **Расчеты с поставщиками** \> **Заказы на покупку** \> **Все заказы на покупку**.
2. Выберите **Создать**, чтобы создать заказ на покупку. Дополнительные сведения см. в разделе [Создание заказа на покупку](../../supply-chain/procurement/tasks/create-purchase-order.md).
3. На странице **Заказа на покупку** перейдите в представление **Заголовок**.
4. На экспресс-вкладке **Цена и скидка** в поле **Условия оплаты** выберите условия оплаты.

    > [!NOTE]
    > Выберите условия оплаты, где для параметра **От подотчетного лица** установлено значение **Да** на странице **Условия оплаты**.

5. В поле **Подотчетное лицо** выберите подотчетное лицо для заказа на покупку.
6. Введите все необходимые сведения по строке, а затем выберите **Покупка** \> **Действия** \> **Подтвердить**, чтобы подтвердить заказ на покупку.
7. Выберите **Накладная** \> **Создать** \> **Накладная**, чтобы создать накладную поставщика для заказа на покупку.
8. На странице **Накладная поставщика** в группе полей **Идентификатор накладной** в поле **Номер** введите номер накладной.
9. Выберите **Разнести**, чтобы разнести накладную, а затем закройте страницу.
10. Выберите **Расчеты с поставщиками** \> **Запросы и отчеты** \> **Запросы и отчеты о подотчетных лицах** \> **Проводки** для просмотра связанных проводок по подотчетным лицам.
11. Выберите **Ваучер**, чтобы открыть страницу **Проводки ваучера**. Балансовый счет поставщика заменяется балансовым счетом сотрудника, а значение в поле **Тип разноски** меняется на **Сальдо сотрудника**.

## <a name="create-and-post-advance-reports"></a>Создание и разноска авансовых отчетов

Эти процедуры используются для создания, изменения, разноски и печати авансовых отчетов, а также для получения сведений о них.

### <a name="create-an-advance-report"></a>Создание авансового отчета

1. Выберите **Расчеты с поставщиками** \> **Подотчетные лица** \> **Авансовый отчет**.
2. Выберите **Создать**, чтобы создать авансовый отчет. Кроме того, можно создать авансовый отчет непосредственно со страницы **Подотчетные лица**, выбрав **Создать** \> **Авансовый отчет**.
3. В поле **Дата** введите дату авансового отчета.

    Номер авансового отчета в поле **Авансовый отчет** автоматически создается на основе номерной серии, которая определяется в параметрах.

4. В группе полей **Номер сотрудника** выберите подотчетное лицо.
5. В поле **Назначение аванса** введите назначение аванса.
6. Перейдите к представлению **Заголовок**.
7. На экспресс-вкладке **Параметры** определите даты подтверждения.
8. На экспресс-вкладке **Должностные лица** определите ответственных менеджеров.

### <a name="generate-advance-report-lines-from-expenses"></a>Создание строк авансового отчета из расходов

1. Перейдите к представлению **Строки**.
2. Выберите **Копировать из расходов**, чтобы открыть страницу **Создать строки авансового отчета**. На этой странице отображаются все типы расходов, которые были ранее определены в параметрах.
3. Выберите строку расхода.
4. Введите дату проводки в поле **Дата**.
5. В поле **Номер документа** введите номер ваучера подтверждающего расходы.

    > [!NOTE]
    > Значения в полях **Наименование документа**, **Курс** и **Валюта** автоматически берутся из справочника ставок расхода.

6. В поле **Количество дней** введите количество дней командировки.
7. В поле **В день недели** введите сумму суточных расхода.

    > [!NOTE]
    > В поле **Итоговая сумма** отображается итоговая сумма расхода.

8. Выберите **Сохранить**, чтобы перенести записи на страницу **Авансовый отчет**.

    > [!NOTE]
    > Значение в поле **Счет ГК** основано на настройке разноски ставок расхода. Если фактические расходы превышают установленные ставки, создается дополнительная строка для суммы сверх нормы, когда расходы переносятся в авансовый отчет. В этом случае флажок **Сверх нормы** устанавливается автоматически. В поле **Тип строки** авансового отчета, который создается с помощью функции **Копирование из расходов**, устанавливается значение **Расход**.

### <a name="generate-advance-report-lines-from-sources"></a>Создание строк авансового отчета из источников

1. Перейдите к представлению **Строки**.
2. Выберите **Копировать из источников**, чтобы открыть страницу **Создать строки**. На этой странице отображается список документов подотчетного лица (кредитовых проводок), где дата совпадает с датой авансового отчета.

    > [!NOTE]
    > В верхней области в поле **Тип источника** отображается тип идентификации исходного документа. Источником для типа **Счет-фактура** служит заказ на покупку, а для типа **Кредиторская задолженность** — проводка по поставщику. В поле **Сумма в валюте проводки** отображается общая сумма во вторичной валюте проводки, в поле **Валюта** отображается код валюты, а в поле **Сумма** отображается сумма в стандартной валюте компании. В нижней области в поле **Номер** отображается номер счета-фактуры поставщика. В поле **Описание** отображается наименование номенклатуры. В поле **Валюта** отображается валюта ваучера, в поле **Сумма в валюте проводки** отображается сумма по строке, а в поле **Сумма налога в валюте** отображается сумма налога.

3. В верхней области установите флажки **Пометка**, чтобы выбрать документы, которые требуется скопировать.
4. Выберите **OK**, чтобы перенести данные расходов в авансовый отчет.

    В поле **Номер документа** авансового отчета номер исходного счета-фактуры копируется, если в поле **Тип строки** авансового отчета установлено значение **Счет-фактура**. Если в поле **Тип строки** установлено значение **Поставщик**, копируется номер проводки.

    > [!NOTE]
    > Редактировать поля **Сумма**, **Валюта** и **Счет ГК** невозможно.

### <a name="manually-generate-advance-report-lines"></a>Создание строк авансового отчета вручную

Эта процедура используется для создания и разноски строк авансового отчета вручную. Можно распределить сумму расходов между различными аналитиками учета. Можно просмотреть созданные записи ГК и проверить распределение суммы расходов.

1. На экспресс-вкладке **Строки авансового отчета** выберите **Добавить строку**, чтобы создать строку.
2. В поле **Дата документа по расходу** введите дату документа, описывающего расход.
3. В поле **Номер документа** введите номер подтверждающего документа.
4. В поле **Наименование документа** введите наименование подтверждающего документа.
5. В поле **Валюта** выберите валюту, которая используется для проводки.
6. В поле **Сумма** введите потраченную сумму проводки.
7. В поле **Подтвержденная сумма авансового отчета** введите подтвержденный расход для авансового отчета.
8. В поле **Счет ГК** выберите счет ГК, к которому относится расход.
9. Выберите **Распределить суммы**, чтобы открыть страницу **Распределение по бухгалтерским счетам**.
10. В поле **Принцип распределения** выберите, следует ли распределять суммы без учета скидок и наценок или с процентом скидки. Можно создать распределения следующими способами:

    - Чтобы создать несколько распределений с одинаковым количеством, процентом или распределением суммы, выберите **Разбиение** для каждого распределения. Выберите счета ГК для каждого распределения и выберите **Распределить поровну**.
    - Для создания распределения по одному выберите **Разбиение**. Выберите счет ГК для распределения строки накладной и введите количество, процент или сумму для распределения. Например, если вы выбрали **Процент** в поле **Принцип распределения**, введите процент в поле **Процент**.

11. Выберите **Сохранить**, а затем закройте страницу **Распределение по бухгалтерским счетам**.
12. На экспресс-вкладке **Сведения по строке** на вкладке **Настройка** в поле **Налоговая группа** выберите код для налоговой группы для авансовых отчетов.
13. В поле **Налоговая группа номенклатур** выберите код налоговой группы номенклатур для авансового отчета.
14. Установите для параметра **Цены включают налог** значение **Да**, чтобы указать, что сумма в авансовом отчете включает налог.

### <a name="post-an-advance-report"></a>Разноска авансового отчета

Эта процедура используется для разноски авансового отчета после заполнения всех сведений в заголовке и строках авансового отчета.

- В области действий выберите **Разнести** \> **Разнести**, чтобы разнести авансовый отчет.

### <a name="print-an-advance-report"></a>Печать авансового отчета

Эта процедура используется для печати разнесенного финансового отчета. Авансовый отчет можно напечататься после сопоставления авансовой накладной с накладной, выданной подотчетному лицу.

> [!NOTE]
> Суммы на странице **Авансовые отчеты** пересчитываются на дату разноски журнала платежей для подотчетного лица.

1. В области действий выберите **Разнести** \> **Разнести**, чтобы открыть страницу **Авансовый отчет**.
2. Выберите **ОК**, чтобы напечатать авансовый отчет в формате Microsoft Excel.

### <a name="reject-an-advance-report"></a>Отклонение авансового отчета

Можно отклонять авансовые отчеты, которые были разнесены и имеют статус **Разнесено**. Эти отчеты не должны были быть сопоставлены. Счета-фактуры для этих отчетов не создаются.

1. Выберите авансовый отчет, а затем в области действий выберите **Разнести** \> **Отклонение**.
2. Выберите **Да**, чтобы подтвердить действие.
3. Для авансового отчета устанавливается статус **Отклонено**.

    > [!NOTE]
    > При отклонении авансового отчета сторнированные проводки создаются для подотчетного лица. Для параметра **Сторнировано** автоматически устанавливается значение **Да** на странице **Проводки по подотч. лицам**. Проводки сопоставляются и реверсируются автоматически.

4. Закройте страницу.
5. Выберите **Расчеты с поставщиками** \> **Подотчетные лица** \> **Подотчетные лица**.
6. Выберите **Проводки**, чтобы открыть страницу **Проводки по подотч. лицам**.
7. Выберите проводку, выберите **Ваучер** и убедитесь, что параметр **Коррекция** автоматически установлен в значение **Да**.

Можно проверить распределение расходов для реверсированной проводки. Реверсированные проводки или учетные проводки со сторно распределяются, как исходные. Однако распределения для сторнированных проводок или учетных проводок со сторно имеют отрицательный знак.

## <a name="close-balances-for-an-advance-holder"></a>Закрытие сальдо по подотчетным лицам

Закрыть сальдо по подотчетному лицу можно через кассу или через банк.

### <a name="close-balances-through-cash"></a>Закрытие сальдо через кассу

1. Выберите **Расчеты с поставщиками** \> **Запросы и отчеты** \> **Запросы и отчеты о подотчетных лицах** \> **Сальдо**.
2. В поле **На дату** введите дату, на которую нужно получить сальдо по подотчетным лицам.
3. Выберите **Закрытие через кассу**, чтобы открыть страницу **Закрытие через кассу**.
4. В поле **Дата платежа** введите дату платежа.
5. В поле **Сумма для переноса** введите сумму сальдо при закрытии.

    > [!NOTE]
    > По умолчанию в этом поле отображается сумма из поля **Сумма** на странице **Сальдо**.

6. Установите для параметра **Автоматически** значение **Да**, чтобы автоматически создать и разнести журнал ордеров.

    > [!NOTE]
    > Если установить параметр **Автоматически** в значение **Нет**, журнал создается автоматически, и отображается номер журнала. После этого журнал необходимо разнести вручную.

7. Выберите **OK**, чтобы создать журнал ордеров.

    > [!NOTE]
    > После обработки журнала ордеров создается расходный или приходный кассовый ордер для подотчетного лица, когда сальдо закрываются. Если сумма в поле **Сумма для переноса** отрицательная, создается расходный кассовый ордер. Если сумма в поле **Сумма для переноса** положительная, создается приходный кассовый ордер.

8. Закройте страницу.

### <a name="close-balances-through-a-bank"></a>Закрытие сальдо через банк

Процедура закрытия сальдо через банк аналогична описанной выше процедуре закрытия через кассу. Однако на шаге 3 необходимо выбрать **Закрытие через банк** вместо **Закрытие через кассу**. Коды для журнала и для банка можно настроить на вкладке **Подотчетные лица** на странице **Параметры расчетов с поставщиками**.

## <a name="complete-the-settlement-for-an-advance-holder"></a>Выполнение сопоставления для подотчетного лица

Можно выполнять ручное сопоставление или периодическое сопоставление для подотчетного лица. Если вы установили для параметра **Автоматическое сопоставление** значение **Нет** на странице **Параметры модуля расчетов с поставщиками**, разноску или закрывающие проводки необходимо сопоставить вручную или с помощью функции периодического сопоставления.

### <a name="manually-settle-advance-holder-transactions"></a>Сопоставление проводок по подотчетному лицу вручную

Эта процедура используется для сопоставления проводок по подотчетному лицу вручную.

1. Выберите **Расчеты с поставщиками** \> **Подотчетные лица** \> **Подотчетные лица**.
2. Выберите подотчетное лицо.
3. Выберите **Сопоставить** \> **Сопоставление открытых проводок**, чтобы открыть страницу **Изменение открытых проводок в нескольких валютах**. В верхней области отображаются проводки по дебету для данного подотчетного лица, а в нижней — проводки по кредиту.
4. В верхней и нижней областях установите флажки **Пометка**, чтобы выбрать проводки. В поле **Сальдо** отображается сальдо после сопоставления проводок.
5. Выберите **Обновить**, чтобы завершить сопоставление.
6. Закройте страницу.
7. На странице **Подотчетные лица** выберите **Сопоставления**, чтобы открыть страницу **Сопоставления проводок**, где можно проверить выполненные сопоставления.

    > [!NOTE]
    > Если корректировки курсовой разницы возникают при сопоставлении проводок подотчетного лица, проводки по курсовой разнице создаются из сопоставления. Корректировка курсовой разницы отображается в поле **Сумма курсовой разницы** на странице **Сопоставления проводок**.

8. Закройте страницу.
9. Выберите **Расчеты с поставщиками** \> **Подотчетные лица** \> **Авансовые отчеты**.
10. Выберите авансовый отчет, созданный для выбранного подотчетного лица.
11. На вкладке **Финансы** выберите **Просмотр распределений**, чтобы убедиться, что распределение суммы расходов верно.

### <a name="periodically-settle-advance-holder-transactions"></a>Периодическое сопоставление проводок по подотчетному лицу

Эта процедура используется для периодического сопоставления проводок по подотчетным лицам. При использовании функции периодических сопоставлений все открытые проводки сопоставляются в хронологическом порядке.

1. Выберите **Расчеты с поставщиками** \> **Периодические задачи** \> **Подотчетные лица** \> **Периодическое сопоставление**.
2. В поле **Дата проводки** выберите дату проводки по подотчетному лицу. Все проводки, разнесенные до этой даты, будут сопоставлены.
3. Установите для параметра **Сопоставление по профилям** значение **Да**, чтобы сопоставлялись только проводки с идентичным профилем разноски.
4. На экспресс-вкладке **Включаемые записи** определите дополнительную фильтрацию для сопоставления.
5. Выберите **ОК**, чтобы сопоставить проводки.

### <a name="cancel-a-periodic-settlement"></a>Отмена периодического сопоставления

Эта процедура используется для отмены периодического сопоставления проводок подотчетного лица.

1. Выберите **Расчеты с поставщиками** \> **Периодические задачи** \> **Подотчетные лица** \> **Периодический реверс**.
2. В поле **Дата проводки** выберите дату проводки по подотчетному лицу. Все проводки, сопоставленные до этой даты, будут реверсированы.
3. На экспресс-вкладке **Включаемые записи** определите дополнительную фильтрацию проводок, сопоставления которых требуется отменить.
4. Выберите **ОК** для отмены периодического сопоставления.
