# <a name="transactions-through-intermediary"></a>Проводки через посредников
[!include [banner](../includes/banner.md)]

В экономических действиях, включающих посредников, существует три участника:

- Лицо, которое продает или покупает товары (работы или услуги)
- Сторонний поставщик или покупатель
- Посредник между двумя первыми участниками

Российский законодательство определяет три юридические формы посреднических услуг:

- Комиссионный договор, в котором доверитель (называемая также принципалом) нанимает комиссионера
- Агентский договор, в котором доверитель нанимает агента
- Договор цессии, в котором доверитель нанимает адвоката

Посредник (то есть, комиссионер, агент или юрист) выполняет юридические действия и другие действия (проводки) от имени доверителя. Посредники могут выполнять эти действия одним из двух способов:

- От своего имени, но за счет доверителя
- От имени доверителя и за счет доверителя

В этой теме термин *доверитель* относится к субъекту, который нанимает агента.

## <a name="overview"></a>Обзор

Текущая реализация этой функциональности имеет следующие ограничения:

- Кредит-ноты могут регистрироваться только в модуле, в котором были зарегистрированы исходные счета-фактуры.
- Вложенная комиссия не поддерживается.
- Комиссионные премиальные не рассчитываются.
- Никакие проводки не создаются в главной книге на основе отчета для доверителя.

## <a name="preliminary-setup"></a>Предварительная настройка

### <a name="set-up-an-inventory-profile-for-an-agents-transactions"></a>Настройка профиля учета для проводок агента

1. Перейдите в раздел **Управление запасами** \> **Настройка** \> **Аналитики** \> **Профили учета**.
2. Выберите **Создать**, чтобы создать профиль учета.
3. В поле **Профиль учета** введите имя для профиля учета.
4. В поле **Имя** введите описание.
5. В поле **Вид деятельности** выберите **Комиссионер**.
6. Задайте для параметра **Не подбирать** значение **Нет**.
7. В поле **Вид запасов** укажите **Общие**.
8. Нажмите **Сохранить**.

![Страница профилей учета](media/1_Inventory_profiles.jpg)

### <a name="set-up-the-inventory-profile-and-owner-tracking-dimensions"></a>Настройка аналитик отслеживания профиля учета и владельца

1. Откройте **Управление сведениями о продукте** \> **Настройка** \> **Группы аналитик и вариантов** \> **Группы аналитик отслеживания**.
2. Выберите **Создать**, чтобы создать группу аналитик.
3. В поле **Имя** введите название группы аналитик.
4. В поле **Описание** введите описание.
5. На экспресс-вкладке **Аналитики отслеживания** настройте аналитики отслеживания "Профиль учета" и "Владелец":

    - В строке **Профиль учета** установите флажки **Активный**, **Первичная аналитика хранения**, **Физические запасы**, **Финансовые запасы**, **План покрытия по аналитикам** и **Перемещение**.
    - В строке **Владелец** установите флажки **Активный**, **Физические запасы** и **Перемещение**.

6. Нажмите **Сохранить**.

![Страница групп аналитик отслеживания](media/2_Tracking_dimension_groups.jpg)

### <a name="set-up-a-number-sequence-for-the-report-for-the-principal"></a>Настройка номерной серии для отчета для доверителя

1. Перейдите в раздел **Главная книга** \> **Настройка главной книги** \> **Параметры главной книги**.
2. На вкладке **Номерные серии** в поле **Код номерной серии** выберите код номерной серии для ссылки **Код отчета**.

Дополнительные сведения см. в разделах [Покупки по комиссии](rus-purchases-on-commission.md) и [Продажи по комиссии](rus-sales-on-commission.md).
