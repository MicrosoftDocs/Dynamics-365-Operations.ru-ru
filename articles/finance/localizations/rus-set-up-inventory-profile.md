# <a name="set-up-an-inventory-profile"></a>Настройка профиля учета 
[!include [banner](../includes/banner.md)]

## <a name="create-an-inventory-profile"></a>Создание профиля учета

1. Перейдите в раздел **Управление запасами** \> **Настройка** \> **Аналитики** \> **Профили учета**.
2. Выберите **Создать**, чтобы создать новый профиль учета.
3. В поле **Профиль учета** введите идентификатор профиля учета.
4. В поле **Имя** введите описание профиля учета.
5. На Экспресс-вкладке **настройки** в поле **вид деятельности** выберите вид деятельности для профиля учета.
6. Установите для параметра **Не подбирать** значение **Да**, если профиль учета не обязательно должен участвовать в автоматическом выборе профилей учета из запасов в наличии при создании строк заказа на продажу посредством выбора пункта **создать строки**.
7. На экспресс-вкладке **Приоритет подбора** используйте кнопки **вверх** и **вниз**, чтобы задать порядок автоматического выбора профилей учета при создании строк заказа на продажу, выбрав **Добавить строки**.

    ![Страница профилей учета, экспресс-вкладка "Приоритет подбора".](media/1_Inventory_profiles.png)

8. Нажмите **Сохранить**.

> [!NOTE]
> Удалить профиль учета можно только в том случае, если в нем нет сальдо.

## <a name="set-up-compatible-inventory-profiles"></a>Настройка совместимых профилей учета

1. Перейдите в раздел **Управление запасами** \> **Настройка** \> **Аналитики** \> **Профили учета**.
2. Выберите профиль учета, для которого необходимо настроить совместимые профили учета.

    - На экспресс-вкладке **совместимые профили учета** в списке **Совместимые** отображаются профили учета, совместимые с выбранным профилем учета. В списке **Прочие** перечислены другие профили учета.

3. Настройка списка совместимых профилей учета.

    - Выберите →, чтобы переместить выбранный профиль учета из списка **Прочие** в список **совместимые**. Профиль учета добавляется в конец списка **совместимые**.
    - Выберите →, чтобы удалить выбранный профиль учета из списка **Совместимые** и добавить его в список **Прочие**.
    - Выберите ⊞→, чтобы переместить все профили учета из списка **Прочие** в список **совместимые**. Профили последовательно добавляются в конец списка **совместимые**.
    - Выберите ←⊞, чтобы удалить все профили учета из списка **Совместимые** и добавить их в список **Прочие**.
    - Используйте **вверх** или **вниз** для задания порядка сопоставления, который используется для профилей учета, совместимых с выбранным профилем учета, когда профили учета в наличии автоматически выбираются из запасов в наличии в заказе на продажу.

4. Нажмите **Сохранить**.

![Страница "профиль учета", экспресс-вкладка "совместимые профили учета".](media/2_Inventory_profiles.png)

## <a name="set-up-an-inventory-profile-in-tracking-dimension-groups"></a>Настройка профиля учета в группах аналитика отслеживания

1. Откройте **Управление сведениями о продукте \> Настройка \> Группы аналитик и вариантов \> Группы аналитик отслеживания**.
2. В левой области выберите группу аналитик или выберите **создать**, чтобы создать новую группу аналитик.
3. В поле **Имя** введите полное имя группы аналитик.
4. В поле **Описание** введите описание группы аналитик.
5. На экспресс-вкладке **аналитики отслеживания** в строке **профиля учета** установите флажок **активно**, чтобы активировать профиль учета.

    Флажки **Первичная аналитика хранения**, **Физические запасы**, **Финансовые запасы** и **Перемещение** установлены по умолчанию.

6. Нажмите **Сохранить**.

![Страница Аналитики отслеживания.](media/3_Tracking_dimension_groups.png)

## <a name="activate-transaction-combinations-for-inventory-profiles"></a>Активация сочетаний проводок для профилей учета.

1. Выберите **Управление запасами \> Настройка \> Разноска \> Комбинации проводок**.
2. Выберите **Правка**.
3. В разделе **отношение профиля учета** настройте следующие значения:

    - Установите флажок **Активировать отношение профиля учета Все**, если необходимо, чтобы можно было настроить разноску запасов, не указывая профиль учета.
    - Установите флажок **Активировать отношение профиля учета Группы**, если необходимо, чтобы можно было настроить разноску запасов для видов деятельности.
    - Установите флажок **Активировать отношение профиля учета Таблица**, если необходимо, чтобы можно было настроить разноску запасов для профилей учета.

4. Нажмите **Сохранить**.

![Страница Комбинации проводок.](media/4_Transaction_combinations.png)

## <a name="set-up-inventory-posting-in-the-context-of-an-inventory-profile"></a>Настройка профиля учета в контексте профиля учета.

1. Выберите **Управление запасами** \> **Настройка** \> **Разноска** \> **Разноска**.
2. В левой области выберите тип разноски и создайте новую строку.
3. В поле **отношение профиля учета** выберите одно из следующих значений:

    - **Профиль** — Настройка разноски запасов для определенного профиля учета.
    - **Тип** — Настройка разноски запасов для определенного вид деятельности.
    - **Все** — Настройка разноски запасов без указания вида деятельности или профиля учета.

4. В поле **вид деятельности** выберите вид деятельности, для которого необходимо настроить разноску запасов. Это поле доступно, только если выбран вариант **Тип** в поле **отношение профиля учета**.
5. В поле **Профиль учета** выберите профиль учета, для которого необходимо настроить разноску запасов. Это поле доступно, только если выбран вариант **Профиль** в поле **отношение профиля учета**.

> [!IMPORTANT]
> Параметры профиля учета в настройке разноски запасов имеют приоритет над другими настройками. Например, имеется два набора параметров разноски:
>   
>   - Параметры для определенной номенклатуры и любого профиля учета
>   - Параметры для определенного профиля учета и всех номенклатур
>   
>   В этом случае система выберет второй набор параметров.

## <a name="set-up-a-relation-between-inventory-profiles-and-customer-and-vendor-posting-profiles"></a>Настройте связь между профилями учета и профилями разноски клиента и поставщика

1. Выберите **Управление запасами** \> **Настройка** \> **Разноска** \> **Профиль учета – профиль разноски**.
2. Выберите **Создать** для создания новой строки.
3. В поле **отношение профиля учета** выберите одно из следующих значений:

    - **Профиль** — Настройка профилей разноски по клиенту и поставщику для определенного профиля учета.
    - **Тип** — Настройка профилей разноски по клиенту и поставщику для определенного вид деятельности.
    - **Все** — Настройка профилей разноски по клиенту и поставщику без указания вида деятельности или профиля учета.

4. В поле **вид деятельности** выберите вид деятельности, для которого необходимо настроить профили разноски. Это поле доступно, только если выбран вариант **Тип** в поле **отношение профиля учета**.
5. В поле **Профиль учета** выберите профиль учета, для которого необходимо настроить профили разноски. Это поле доступно, только если выбран вариант **Профиль** в поле **отношение профиля учета**. При выборе определенного профиля учета поле **вид деятельности** автоматически устанавливается на соответствующий вид деятельности.
6. Настройка полей **Профиль разноски по поставщику** и **Профиль разноски по клиенту**.
7. Нажмите **Сохранить**.

![Страница Профиль учета-профиль разноски.](media/5_Inventory_profile_-_posting_profile.png)

## <a name="set-up-a-default-inventory-profile-for-boms"></a>Настройка профиля учета по умолчанию для спецификаций.

1. Перейдите **Управление запасами \> Записи журнала \> Номенклатуры \> Спецификации**.
2. Выберите журнал спецификации и создайте строку.
3. На экспресс-вкладке **Сведения о строке** на вкладке **Складские аналитики** в поле **Профиль учета** выберите профиль учета.

> [!NOTE]
> Невозможно развернуть или принять спецификацию или создать производственный заказ, если выполняются все следующие условия:
>
>   - Для номенклатуры в строке спецификации активен профиль учета.
>   - В строке спецификации не указана аналитика **профиля запасов**.
>   - Поле **Профиль запасов** на вкладке **спецификации** модуля **Управление запасами и складами** пусто.

## <a name="set-up-a-default-inventory-profile-for-purchase-orders"></a>Настройка профиля учета по умолчанию для заказов на покупку

На странице **Параметры модуля "Закупки и источники"** можно выбрать профиль учета по умолчанию. Этот профиль учета будет затем автоматически вводиться в заказы на покупку, если не выбраны профили учета по умолчанию и виды деятельности для записи поставщика и записи договора покупки.

1. Выберите **Закупки и источники \> Настройка \> Параметры модуля "Закупки и источники"**.
2. На вкладке **Общие** на экспресс-вкладке **значения и параметры по умолчанию** в поле **Профиль учета** выберите профиль учета, который должен использоваться автоматически в заказах на покупку.

    ![Страница Параметры модуля "Закупки и источники", вкладка Общие.](media/6_Procurement_and_sourcing_parameters.png)

3. Выберите **Обновление строк заказа**, чтобы открыть диалоговое окно **Обновление строк заказа**.

    ![Диалоговое окно Обновление строк заказа.](media/7_Update_order_lines.png)

4. В разделе **Профиль учета** в поле **Обновить профиль учета** выберите одно из следующих значений для настройки правил, используемых для обновления поля **профиль учета** в строках заказа на покупку при изменении соответствующего поля в заголовке заказа на покупку:

    - **Никогда** — профиль учета в строках заказа не должен обновляться автоматически при изменении профиля учета в заголовке заказа.
    - **Всегда** — профиль учета в строках заказа должны всегда обновляться автоматически при изменении профиля учета в заголовке заказа.
    - **Запрос** — система должна предложить вам обновить профиль учета в строках заказа при изменении профиля учета в заголовке заказа.

5. Закройте диалоговое окно **Обновление строк заказа**.
6. На странице **Параметры модуля "Закупки и источники"** на вкладке **Сводное обновление** в разделе **Разделение на основании** установите два варианта **Поступление продуктов**:

    - Установите для параметра в столбце **профиль разноски** значение **Да**, если требуется разбиение поступления продуктов заказа на покупку по профилям разноски по поставщикам, указанным в строках заказа на покупку.
    - Установите параметр в столбце **Вид деятельности** в значение **Да**, если требуется разбиение отборочных накладных по заказу на покупку на виды деятельности, которые включают профили учета, указанные в строках заказа на покупку.

![Страница Параметры модуля "Закупки и источники", вкладка Сводное обновление.](media/8_Procurement_and_sourcing_parameters.png)

> [!NOTE]
> Параметр **Накладная** в обоих столбцах **профиль разноски** и столбца **Вид деятельности** всегда имеет значение **Да** и не может быть изменено. Таким образом, накладные, связанные с заказом на покупку всегда разбиваются на профили разноски по поставщику и виды деятельности.

## <a name="set-up-a-default-inventory-profile-for-sales-orders"></a>Настройка профиля учета по умолчанию для заказов на продажу

На странице **Параметры модуля расчетов с клиентами** можно выбрать профиль учета по умолчанию. Этот профиль учета будет затем автоматически выбран в заказах на продажу, если не выбраны профили учета по умолчанию и виды деятельности для записи клиента и записи договора продажи.

1. Перейдите в раздел **Расчеты с клиентами** \> **Настройка** \> **Параметры модуля расчетов с клиентами**.
2. На вкладке **Общие** на экспресс-вкладке **Настройка продаж** в поле **Профиль учета** выберите профиль учета, который должен использоваться автоматически в заказах на продажу.
3. Установите параметр **использовать совместимые профили учета** на **Да**, если в заказах на продажу должны автоматически использоваться совместимые профили учета.
4. Установите параметр **разбивка строк заказа по профилям учета** на **Да**, чтобы автоматически разбить строки заказа на продажу по профилям учета при использовании **Создать строки** из доступного физического количества.

    ![Страница Параметры расчетов с клиентами, вкладка "Общие".](media/9_Accounts_receivable_parameters.png)

5. На вкладке **Сводное обновление** на экспресс-вкладке **Разделение на основании** установите параметр **Профиль разноски** в столбце **Отборочная накладная** на **Да** для разбиения отборочных накладных заказа на продажу по профилям разноски по клиенту, указанным в строках заказа на продажу.
6. Установите параметр **Вид деятельности** в столбце **отборочная накладная** на **Да**, если требуется разбиение отборочных накладных по заказу на покупку на виды деятельности, которые включают профили учета, указанные в строках заказа на продажу.

    ![Страница "Параметры модуля расчетов с клиентами", вкладка Сводное обновление.](media/10_Accounts_receivable_parameters.png)

    > [!NOTE]
    > В столбце **Накладная** параметры **профиль разноски** и **Вид деятельности** всегда имеют значение **Да** и не могут быть изменено. Таким образом, накладные по заказам на продажу всегда разбиваются на профили разноски по клиенту и виды деятельности.

7. На вкладке **обновления** на Экспресс-вкладке **строки заказа** выберите **Обновление строк заказа**, чтобы открыть диалоговое окно **Обновление строк заказа**.
8. В разделе **Режим доставки** в поле **Обновить профиль учета** выберите одно из следующих значений:

    - **Никогда** — профиль учета в строках заказа не должен обновляться автоматически при изменении профиля учета в заголовке заказа.
    - **Всегда** — профиль учета в строках заказа должны всегда обновляться автоматически при изменении профиля учета в заголовке заказа.
    - **Запрос** — система должна предложить вам обновить профиль учета в строках заказа при изменении профиля учета в заголовке заказа.

![Страница Обновление строк заказа.](media/11_Update_order_lines.png)

## <a name="set-up-a-default-inventory-profile-for-transfer-orders"></a>Настройка профиля учета по умолчанию для заказов на перемещение

На странице **Параметры модуля "Управление запасами и складами"** можно выбрать профиль учета по умолчанию. Этот профиль учета будет затем автоматически выбран в заказах на перемещение, если не выбраны профиль учета по умолчанию и вид деятельности для записи склада.

1. Перейдите в раздел **Управление запасами** \> **Настройка** \> **Параметры управления запасами и складами**.
2. На вкладке **Общие** в разделе **Профиль учета** в поле **Профиль учета** выберите профиль учета, который должен использоваться автоматически в заказах на перемещение.
3. Установите параметр **использовать совместимые профили учета** на **Да**, если параметр **использовать совместимые профили учета** в заказах на перемещение должен быть установлен на **Да** по умолчанию.

    ![Страница Параметры управления запасами и складами, вкладка Общие.](media/12_Inventory_and_warehouse_management_parameters.png)

4. На вкладке **спецификации** в поле **Профиль учета** выберите профиль учета, который должен использоваться по умолчанию при создании и развертывании спецификаций, и когда обрабатывается как готовое, если в строках спецификации не указан профиль учета.

!Страница "Параметры управления запасами и складами", вкладка "Спецификации"](media/13_Inventory_and_warehouse_management_parameters.png)
