# <a name="financial-dimensions-and-posting"></a>Финансовые аналитики и разноска 

[!include [banner](../includes/banner.md)]

При планировании и настройке плана счетов необходимо учитывать, как различные компоненты будут взаимодействовать при разноске документа или журнала. К этим компонентам относятся структуры счетов, расширенные правила, балансирование и фиксированные аналитики. В этой теме объясняется, что представляет собой каждый из компонентов и как эти компоненты взаимодействуют.

## <a name="chart-of-accounts-and-financial-dimension-components"></a>Компоненты плана счетов и финансовых аналитик

Основанная на правилах система используется для определения допустимых сочетаний счетов ГК и значений финансовых аналитик. В этом разделе дается краткий обзор функциональных возможностей каждого компонента и объясняется, где можно найти компонент.

### <a name="account-structures"></a>Структуры счетов

Структура счета является обязательной при настройке вашей главной книги. Необходимо определить и активировать хотя бы одну структуру счета и назначить ее главной книге. В структуру счета должен входить счет ГК. Можно определить порядок сегментов, который лучше всего подходит вашей компании. После определения счета ГК система может определить используемую структуру счета. Поместив счет ГК на первое место или в начале структуры, вы можете ограничить набор значений и также дать системе возможность применять последнее известное допустимое значение в качестве значения по умолчанию. В структуру счета может входить до 10 дополнительных финансовых аналитик. Структура счета определяет, какие значения аналитики являются допустимыми в сочетании с другими значениями. Он также определяет, требуется ли вводить значения аналитики.

### <a name="advanced-rules"></a>Дополнительные правила

Расширенные правила — дополнительный компонент, используемый при настройке плана счетов. В структуру счета можно добавить сколько угодно расширенных правил. Расширенные правила часто используются для обработки сценариев, где при выполнении определенных критериев необходимо отслеживать дополнительные финансовые аналитики. Например, при использовании счета "Командировочные расходы" может возникнуть необходимость отслеживания дополнительной информации, например мероприятия, на которое едет сотрудник. Если расширенных правил несколько, они применяются в алфавитном порядке, исходя из имен правил. Сегменты, которые добавляет правило, могут быть применены только после сегментов в структуре счета.

### <a name="balancing-dimension"></a>Аналитика балансирования

При необходимости можно определить финансовую аналитику балансирования. На странице **Книга учета** можно задать финансовую аналитику, которая должна балансироваться. Затем каждый раз, когда проводки разносятся по этой финансовой аналитике, система автоматически создает и учитывает операции, чтобы сделать финансовую аналитику сбалансированной.

### <a name="defaultfixed-financial-dimensions-on-the-main-account"></a>Аналитики по умолчанию/фиксированные аналитики для счета ГК

Аналитики по умолчанию поступают из различных мест, таких как основные записи (например, записи клиента или поставщика), заголовки документов и счет ГК. Эта тема посвящена аналитикам по умолчанию для счета ГК по юридическому лицу. Можно определить, **Фиксировано** или **Не фиксировано** значение у счета ГК для каждой финансовой аналитики, используемой во всех структур счетов для главной книги. Если значение финансовой аналитики **Не фиксировано**, используется значение по умолчанию, но это значение может быть перезаписано. Такое поведение применяется ко всем значениям по умолчанию в системе, даже значений по умолчанию, поступающих из основных записей. Если значение финансовой аналитики **Фиксировано**, это значение применяется всегда, независимо от того, поступило ли оно откуда-либо как значение по умолчанию или его ввел пользователь.

## <a name="order-in-which-default-dimensions-are-applied-during-posting"></a>Порядок применения аналитик по умолчанию во время разноски

Часто возникают вопросы о том, в каком порядке выполняются все эти компоненты. Очень важно понимать порядок, в котором применяются аналитики по умолчанию, так как это влияет на подход к настройке.

> [!NOTE]
> Эти сведения относятся только к применению аналитик по умолчанию в приложении. При импорте данных с помощью Microsoft Excel или платформа управления данными поведение будет другим.

### <a name="example-1"></a>Пример 1

**Структура счета**

| Счет ГК            | Бизнес-единица           | Подразделение              | Место возникновения затрат             |
|-------------------------|-------------------------|-------------------------|-------------------------|
| Все значения допустимы. | Все значения допустимы. | Все значения допустимы. | Все значения допустимы. |

**Счет ГК**

| Счет ГК | Наименование          | Информация о компании | Подразделение                                 |
|--------------|---------------|--------------|--------------------------------------------|
| 401100       | Продажи продукта | USMF         | Фиксировано — 022 Отдел продаж и маркетинга |

На следующем рисунке показана фиксированная аналитика по умолчанию, установленная для счета ГК 401100.

[![Финансовые аналитики по умолчанию](./media/default-dimensions.png)](./media/default-dimensions.png)

В этом очень простом примере мы введем общий журнал, где для аналитики "Отдел" задано использование значения по умолчанию **023** (Операции). Мы введем и разнесем счет книги учета. На следующем рисунке показана финансовая аналитика по умолчанию в заголовке главной книги.

[![Общие журналы](./media/general-journal.png)](./media/general-journal.png)

Из-за аналитики по умолчанию в заголовке журнала к строке счета продаж по умолчанию будет применяться отдел 23. На следующем рисунке показана строка общего журнала, где применено значение аналитики по умолчанию из заголовка, т. е. **023**.

[![Операция журнала](./media/journal-voucher.png)](./media/journal-voucher.png)

Тем не менее при разноске строки применяется фиксированная аналитика, и строка разносится на отдел 022. На следующем рисунке показан разнесенный ваучер, где для счета продажи применена фиксированная аналитика.

[![Проводки ваучера](./media/voucher-transactions.png)](./media/voucher-transactions.png)

### <a name="example-2"></a>Пример 2

В этом примере используется такая же настройка, как в первом. Однако мы добавим второй компонент и будем использовать аналитику "Отдел" в качестве аналитики балансирования. На следующем рисунке аналитика **Отдел** установлена в качестве финансовой аналитики балансирования для книги учета USMF.

[![Главная книга](./media/ledger.png)](./media/ledger.png)

При использовании той же настройки заголовка журнала и разноске той же проводки сначала применяется фиксированная аналитика. Затем применяется логика балансирования, чтобы гарантировать, что у каждого отдела будет балансирующая запись. На следующем рисунке показаны проводки ваучера, которые включают балансирующую запись после применения фиксированной аналитики.

[![Проводки ваучера](./media/voucher-transactions2.png)](./media/voucher-transactions2.png)

### <a name="example-3"></a>Пример 3

В этом примере мы добавим расширенное правило. Расширенное правило указывает, что, если используются счет продаж 401100 и отдел 022 (продажи и маркетинг), система должна отслеживать дополнительный сегмент, который называется "Клиент".

Этот пример важен из-за порядка действий. Структура счета определяется после ввода счета ГК. Если сослаться на настройку структуры счета, система сможет определить, что необходимы счет ГК, подразделение, отдел и центр затрат. На этом этапе расширенное правило еще не было инициировано, поскольку фиксированные аналитики не применяются, пока не будут применены аналитики по умолчанию для ваучера журнала во время разноски. На следующем рисунке сегмент "Клиент" отсутствует, поскольку критерии для дополнительного правила не были выполнены.

[![Счет ГК](./media/drop-down.png)](./media/drop-down.png)

Разноска не будет успешной, поскольку фиксированная аналитика была применена в конце процесса. Проверка аналитики определяет, что сегмент "Клиент" является обязательным, если счет ГК — 401100, а отдел — 022. Разноска невозможна из-за ошибки проверки. На следующем рисунке показано сообщение, которое появляется после того, как проверки аналитики определяет, что "Клиент" — обязательный сегмент.

[![Сведения о сообщении](./media/message.png)](./media/message.png)

В этом примере необходимо перезаписать значение по умолчанию таким образом, чтобы инициировалось расширенное правило и вы могли ввести сегмент "Клиент". Однако это решение не всегда возможно, и некоторые пользователи не знают о правилах разноски. Следовательно, при настройке плана счетов важно понимать порядок, в котором применяются аналитики по умолчанию.

Чтобы достичь желаемого результата в этом примере, можно изменить конфигурацию несколькими способами. Например, можно создать новую структуру счета для счетов продаж и включить в структуру сегмент "Клиент". Можно также добавить дополнительные строки в существующую структуру счета и указать счет ГК и допустимые значения отделов. Кроме того, в дополнительной структуре клиента, может быть целесообразно иметь отдельную структуру счета для счетов продаж, где присутствует сегмент "Клиент".
