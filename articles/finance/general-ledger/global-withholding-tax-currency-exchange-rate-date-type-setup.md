# <a name="enable-the-global-withholding-tax-currency-exchange-rate-type-and-date-type-setup"></a>Включить настройку типа валютного курса глобального подоходного налога и типа даты

[!include[banner](../includes/banner.md)]

[!include[banner](../includes/preview-banner.md)]

В этой статье объясняется, как включить глобальную настройку типа даты и типа валютного курса подоходного налога. Теперь можно выбрать отдельный тип валютного курса и тип даты расчета валютного курса для валюты подоходного налога. Эти выборки определяют обменный курс иностранной валюты, используемый для расчета суммы подоходного налога в валюте подоходного налога в проводках по оплате.

Эта функция доступна в Microsoft Dynamics 365 Finance версии 10.0.29 и выше.

1. Перейдите в раздел **Налог** \> **Настройка** \> **Параметры** \> **Параметры главной книги**.
2. На вкладке **Подоходный налог** установите значение **Да** для параметра **Включить валюту авансового подоходного налога**.
3. В поле **Тип валютного курса** выберите тип валютного курса для валюты подоходного налога.
4. В поле **Тип даты расчета** выберите тип даты расчета, который определяет обменный курс валюты подоходного налога:

    - **Дата оплаты** — определить валютный курс на основе даты разноски журнала платежей.
    - **Дата накладной** — определить валютный курс на основе даты накладной журнала накладных. Если дата накладной ваучера пуста, будет использоваться дата разноски накладной. 
    - **Дата документа** — определить валютный курс на основе даты документа журнала платежей. Если дата документа ваучера пуста, будет использоваться дата оплаты.

5. Нажмите **Сохранить**.

Если валюта проводки отличается от валюты подоходного налога, которая определена в коде подоходного налога, сумма в валюте подоходного налога будет рассчитана, исходя из валюты проводки на основе предыдущих настроек, и будет записана в разнесенные проводки по подоходному налогу.
