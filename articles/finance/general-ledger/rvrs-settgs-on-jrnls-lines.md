# <a name="reverse-settings-on-journals-and-lines"></a>Параметры реверсирования в журналах и строках

[!include [banner](../includes/banner.md)]

В этом разделе описывается, почему запись реверсирования, которая была введена в общем журнале, может быть не включена в разнесенную проводку.  

## <a name="symptom"></a>Симптом

Общий журнал включает запись реверсирования и дату реверсирования в журнале. При разноске журнала ни один из ваучеров не реверсируется. Почему это происходит?

## <a name="resolution"></a>Решение

При разноске журнала в процессе реверсирования рассматриваются только параметры **Запись реверсирования** и **Дата реверсирования** в разделе **Строки** ваучера. Такой подход позволяет включить в журнал некоторые ваучеры, которые помечены для реверсирования, и другие ваучеры, которые не помечены для реверсирования.

Значения из журнала используются только в качестве значений по умолчанию при добавлении *новых* строк. Изменение значений в журнале не влияет на существующие строки. В этом примере сначала были введены строки ваучеров. При вводе сведений о строке перед назначением журнала как реверсированного, назначение в качестве записи и даты реверсирования не будет применено к существующим строкам.

Изменение записи реверсирования или даты реверсирования в существующей строке распространяет это изменение на другие строки в том же ваучере. Для оптимизации производительности сетка не обновляется после распространения изменений на другие строки. Вы можете отобразить обновленные значения, обновив сетку.
