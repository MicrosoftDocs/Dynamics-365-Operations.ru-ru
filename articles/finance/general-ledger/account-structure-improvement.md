# <a name="account-structure-activation-performance-enhancement"></a>Улучшение процесса активации структуры счета

[!include [banner](../includes/banner.md)]
[!include [banner](../includes/preview-banner.md)]

Благодаря этому улучшению производительности можно ускорить активацию структур счетов за счет одновременного обновления нескольких проводок. Кроме того, сама структура помечается как активная после ее проверки, а обработка проводок может быть продолжена, пока существующие неразнесенные проводки обновляются в новой структуре. Поскольку обновление проводок может занять некоторое время, вы можете следить за статусом активации, выбрав команду **Просмотреть статус активации** над сеткой на странице **Структуры счета**. Вы также можете узнать статус активации, нажав **Просмотреть** на панели операций, а затем выбрав **Статус активации** в раскрывающемся меню.

[![Страница структур счетов.](./media/AccountStructure1.png)](./media/AccountStructure1.png)

После выбора команды **Просмотреть статус активации** откроется диалоговое окно, в котором будут показаны отдельные задачи, выполняемые в рамках процесса активации. Для каждой задачи можно посмотреть ее статус, а для завершенных задач — дату и время завершения.

[![Временная шкала активации структуры счета.](./media/AccountStructureTimeline.png)](./media/AccountStructureTimeline.png)

## <a name="account-structure-activation-tips"></a>Советы по активации структуры счета

В диалоговом окне "Активация структуры счета", которое появляется при выборе команды **Активировать** для черновика структуры счеты счета, есть вкладка **Обновление неразнесенных проводок**. На ней можно найти параметр **Принудительное обновление**. Выберите его, чтобы обновить все неразнесенные проводки до текущей структуры. Однако этот параметр следует использовать только в том случае, если произошла ошибка, или если вы получили соответствующие указания от службы поддержки Майкрософт.

Вот некоторые факторы, которые могут влиять на время выполнения процесса активации:

- большое число неразнесенных записей журнала;
- большое число открытых записей документов-источников, таких как накладные с произвольным текстом, накладные поставщиков и связанные документы, которые используют структуру документов-источников и имеют открытые распределения по бухгалтерским счетам;
- объем данных в TaxUncommitted;
- объем данных открытых бюджетов;
- импорт данных журнала в то время, когда задачи активации все еще выполняются.
