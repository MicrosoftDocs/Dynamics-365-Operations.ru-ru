# <a name="production-postings"></a>Разноски по производству

В этом разделе представлена информация о различных типах разносок в процессе производственных заказов.


## <a name="material-consumption"></a>Материальное потребление

Материалы регистрируются как потребленные во время производства, когда разносится журнал ведомостей комплектации производства. Этот процесс создает проводки расхода, которые уменьшают запасы в наличии. В **параметрах производства** можно указать, должна ли стоимость обрабатываемого сырья (незавершенное производство НЗП) быть разнесена в книге учета.

Стоимость незавершенного производства сырья (НЗП) проводится по счетам **Оценочная стоимость потребленных материалов** и **Оценочная стоимость потребленных материалов, НЗП**. Процесс листа подбора для производственного заказа является физическим обновлением складских проводок, имеющих отношение к данному производственному заказу. Когда производственный заказ регистрируется как завершенный, физические проводки реверсируются и соответствующие складские проводки обновляются финансово. При разноске конечного журнала используются типы **Стоимость потребленных материалов** и **Стоимость потребленных материалов, НЗП**.

Вкладка **Производство** на странице **Профили разноски запасов** используется для управления порядком разноски производственных заказов в главной книге. Это используется, когда в поле **Разноска ГК** установлено значение **Номенклатура и ресурс** или **Номенклатура и категория** на странице **Параметры контроля производства**. 

Чтобы разнести журнал сборочных листов в главную книгу для производственного заказа, должны быть выполнены следующие условия:

-   На странице **Параметры контроля производства** должен быть установлен флажок **Разнести сборочный лист в ГК**. Эта настройка может быть переопределена для определенного сайта на странице **Параметры управления производством по сайтам**.
-   Флажок **Разносить физические запасы** должен быть установлен на странице **Группы номенклатурных моделей** для номенклатуры, выбранной в строке заказа на покупку.
-   Счета ГК должны быть указаны на странице **Профиль разноски запасов** для следующих типов разноски:
    -   **Оценочная стоимость потребленных материалов**
    -   **Оценочная стоимость потребленных материалов, НЗП**

## <a name="time-consumption"></a>Затраты времени

Время, которое работники тратят на производственные задания, записывается в **Журнал карт маршрутов** или **Журнал карт заданий**. Когда эти журналы разносятся, обрабатывается разноска на отдельный счет для ресурсов в незавершенном производстве. Эта разноска представляет стоимость времени, которое потрачено на производственный заказ. После регистрации производственного заказа как "закончено", сопоставляются счета НЗП.

Существует три способа разноски потребления времени в зависимости от параметра, выбранного в поле **Разноска ГК** на странице **Параметры управления производством**.

## <a name="reporting-finished-goods-and-error-quantities"></a>Отчетность по готовым товарам и количеству ошибок

Когда производственный заказ **принят**, количество готовых товаров, которые были завершены, обновляется в запасах через журнал **Приемка**. Если вы используете учет незавершенного производства (НЗП), в журнале ГК уменьшается количество на счетах НЗП и увеличиваются запасы готовой продукции. Журнал использует стандартные затраты, которые определены для продукта. Если выбран параметр **Использовать оценочную цену** на странице **Параметры управления производством**, будет использоваться оценочная стоимость из производственного заказа.

Стоимость незавершенного производства сырья (НЗП) проводится по счетам **Оценочная стоимость производства** и **Оценочная стоимость производства, НЗП**. Процесс **Приемки** для производственного заказа является физическим обновлением складских проводок, имеющих отношение к данному производственному заказу. Когда производственный заказ завершен, физические проводки реверсируются и соответствующие складские проводки обновляются финансово. При разноске конечного журнала используются типы **Стоимость производства** и **Стоимость производства, НЗП**.

Вкладка **Производство** на странице **Профили разноски запасов** используется для управления порядком разноски заказов на производство в главную книгу. Это используется, когда в поле **Разноска ГК** установлено значение **Номенклатура и ресурс** или **Номенклатура и категория** на странице **Параметры контроля производства**. На вкладке **Главная книга** на странице **Производственные группы** можно также управлять разноской для производственных заказов, когда для поля установлено значение **Производственные группы**.

Чтобы разнести журнал **Приемка** в главную книгу для производственного заказа, должны быть выполнены следующие условия:
-   На странице **Параметры контроля производства** должен быть установлен флажок **Разнести приемку в ГК**. Эта настройка может быть переопределена для определенного сайта на странице **Параметры управления производством по сайтам**.
-   Флажок **Разносить физические запасы** должен быть установлен на странице **Группы номенклатурных моделей** для номенклатуры, выбранной в строке заказа на покупку.
-   Счета ГК должны быть указаны на странице **Профиль разноски запасов** для следующих типов разноски:
    -   **Оценочная стоимость производства**
    -   **Оценочная стоимость производства, НЗП**

## <a name="ending-the-production-order"></a>Завершение производственного заказа

До завершения производственного заказа для произведенного количества рассчитываются фактические затраты. Все ожидаемые затраты на материал, оплату труда и накладные расходы реверсируются и заменяются фактическими затратами. Общие затраты на произведенную номенклатуру дебетуются со счета **Стоимость производства** и кредитуется на счет **Стоимость производства, НЗП**. Материалы, потребляемые в ходе производственного заказа, кредитуются в **Стоимость потребленных материалов** и дебетуются в **Стоимость материалов, НЗП**.

Если вы установили флажок **Заключительное задание** при выполнении расчета стоимости, статус производственного заказа изменяется на **Завершено**. Этот статус предотвращает непреднамеренную разноску дополнительных затрат в завершенном производственном заказе. Можно указать, что значение количества брака, полученное во время приемки, должно распространятся на количества годных изделий, учтенных как готовые. Альтернативно, вы можете определить, чтобы значение количества ошибок должно быть разнесено на отдельный счет отходов. 

Чтобы указать конкретную учетную запись брака, выполните следующие действия:
1.  Перейдите в раздел **Управление производством** > **Настройка** > > **Параметры управления производством**.
2.  Перейдите на вкладку **Стандартное обновление**.
3.  В поле **Метод отходов** выберите **Счет для учета отходов**.
4.  В поле **Счет для учета отходов** выберите счет ГК, в который должны разноситься отходы для ошибочного количества, когда производственный заказ заканчивается. Например, выберите счет расходов, такой как 510380: Производственный брак.

Чтобы разнести конечный журнал в главную книгу для производственного заказа, должны быть выполнены следующие условия:
-   Счета ГК должны быть указаны на странице **Профиль разноски запасов** или **Производственные группы** для следующих типов разноски:
    -   **Стоимость потребленных материалов**
    -   **Стоимость потребленных материалов, НЗП**
    -   **Стоимость производства**
    -   **Стоимость производства, НЗП**
-   Счета ГК должны быть указаны на странице **категории затрат**, **Ресурсы** или **Группы ресурсов** для следующих типов:
    -   **Распределенная стоимость производства**
    -   **Стоимость потребленного производства, НЗП**

## <a name="indirect-costs-for-production-orders"></a>Косвенные затраты для производственных заказов

При обработке проводок для производственного заказа можно настроить косвенные затраты для сбора затрат на накладные расходы или дополнительных сборов в главной книге. Физические проводки для косвенных затрат определяются в запасах при разноске журнала отгрузочных накладных или журнала приемки. Финансовые проводки для косвенных затрат определяются в запасах при разноске конечного журнала.

Имеется возможность распознавать косвенные затраты по потреблению времени, связанным с журналами **карты маршрута** или журналами **карты задания**. Эти проводки реверсируются и разносятся на финансовые счета при завершении производственного заказа. Для получения дополнительных сведений перейдите в раздел [Схема калькуляции](/supply-chain/cost-management/costing-sheets.md).

## <a name="controlling-the-level-of-ledger-posting"></a>Управление уровнем разноски ГК

На странице **Параметры управления производством** вы можете использовать поле **Разноска ГК**, чтобы задать уровень разноски ГК для производственных процессов. 

Доступны следующие поля:

### <a name="item-and-resource"></a>Номенклатура и ресурс

При выборе параметра **Номенклатура и ресурс** в поле **Разноска главной книги** счета разноски берутся из ресурса или группы ресурсов в операции маршрута. Счета для данного ресурса будут первыми вариантами разноски. Если счет не указан, будут использоваться счета, указанные в группе ресурсов. Каждая строка операции в маршруте может иметь один ресурс, указанный как **Ресурс для расчета себестоимости**. Этот ресурс используется для определения счета, который будет использоваться. Этот параметр обычно используется, когда организация назначает операционные затраты для ресурсов. Затраты часто повышаются для ресурсов и используются для того, чтобы помочь им в принятии решений о покупке. Это наиболее подробная конфигурация проводки, которая позволяет наиболее точно контролировать проводки и проводить очень подробную аналитику производственного процесса.

### <a name="item-and-category"></a>Номенклатура и категория

При выборе параметра **Номенклатура и категория** в поле **Разноска главной книги** счета разноски будут поступать со страницы **Категория затрат**, имеющей отношение к каждой строке маршрута. Каждая строка операции в маршруте может иметь три категории затрат: **Время настройки**, **Время выполнения** и **Количество**. Этот вариант обычно используется, когда основное внимание вашей организации уделяется эффективности процессов и продолжительности операций. Этот вариант представляет собой более обобщенный способ проводки и по-прежнему позволяет группировать затраты по категориям.

### <a name="production-group"></a>Производственная группа

При выборе параметра **Производственные группы** в поле **Разноска главной книги** счета разноски поступают со страницы **Производственные группы**. **Производственные группы** обычно используются, когда более чем одна строка производства выполняет аналогичные продукты или аналогичное оборудование. Эту опцию можно использовать для сравнения затрат между двумя разными производственными группами.  
  
> [!NOTE]
> Если используется стандартный метод расчета затрат, это влияет и на конечные проводки. Если фактические затраты и затраты, рассчитанные стандартным методом отличаются, разница разносится на счет прибылей или убытков.

### <a name="route-groups-and-ledger-posting"></a>Маршрутные группы и разноски ГК

Каждая строка операции в маршруте имеет указанную **Маршрутную группу**. Поля **Время настройки**, **Время выполнения** и **Количество** в **маршрутной группе** в группе **Оценка и калькуляция затрат** используются для управления время будет использоваться для калькуляции затрат при размещении **Карточки заданий** или **Журнала карт маршрутов** в производственном заказе. Если параметры отключены, ваучер не будет создан в главной книге на время.

Чтобы разнести **Карту маршрутов** или **Журнал карт заданий** в главную книгу для производственного заказа, должны быть выполнены следующие условия:

-   Поле **Время настройки**, **Время выполнения** или **Количество** должно быть выбрано в группе **Оценка и расчет себестоимости** на странице **Маршрутная группа**.
-   Счета ГК должны быть указаны на странице **Категория затрат**, **Маршрут**, **Маршрутная группа** или **Производственные группы** для следующих типов разноски:
    -   Распределенная оценочная стоимость производства
    -   Оценочная стоимость потребленного производства, НЗП

На следующем рисунке показана связь маршрутной группы с расчетом общих затрат для каждой операции (строки маршрута) в данном маршруте. Каждая строка маршрута имеет одну маршрутную группу. Маршрутная группа управляет параметрами времени настройки, времени выполнения и количества. На вкладке **Категория затрат** имеются три параметра для **настройки**, **выполнения** и **количества**, которые включены в соответствии с настройкой маршрутных групп. Экспресс-вкладка **расчеты времени** содержит три поля, которые основаны на маршрутной группе.

Используемая формула зависит от того, включен ли параметр в маршрутной группе. Стоимость выбранной категории затрат умножается на количество, введенное в расчетах по времени для расчета общих затрат.
