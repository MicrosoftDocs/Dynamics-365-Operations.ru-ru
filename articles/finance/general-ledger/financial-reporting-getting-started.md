# <a name="get-started-with-financial-reporting"></a>Начало работы с финансовой отчетностью 

[!include [banner](../includes/banner.md)]

В этой статье описывается доступ к финансовой отчетности и использование возможностей финансовой отчетности. Она также включает в себя описание финансовых отчетов по умолчанию, которые предоставляются.

## <a name="enable-financial-reporting"></a>Включение финансовой отчетности
Чтобы использовать службу финансовой отчетности Financial Reporting для вашей организации, администратор Lifecycle Services (LCS) должен включить эту службу на портале LCS для вашей организации. Если служба Financial Reporting не была подготовлена для вашей среды, обратитесь к администратору LCS, чтобы включить эту службу. 

## <a name="accessing-financial-reporting"></a>Доступ к службе Financial Reporting

Меню **Финансовая отчетность** находится в следующих местах:

- **Главная книга** > **Запросы и отчеты**
- **Бюджетирование** > **Запросы и отчеты** > **Основное бюджетирование**
- **Бюджетирование** > **Запросы и отчеты** > **Бюджетное планирование**
- **Бюджетирование** > **Запросы и отчеты** > **Бюджетный контроль**
- Консолидации

Чтобы создать и сгенерировать финансовые отчеты для юридического лица, необходимо настроить следующую информацию для этого юридического лица:

- Фискальный календарь
- Ledger
- План счетов
- Валютное
- Разноска проводки по крайней мере на один счет
- MainAccount указан в столбце **Выбрано** на странице **Настройка финансовой отчетности** (**Главная книга > Настройка главной книги > Настройка Financial Reporting**)

## <a name="report-deletions-and-expirations"></a>Удаления отчетов и истечения срока действия

Пользователи, создающие отчет, могут удалять свои собственные отчеты. Пользователи с полномочиями **Сохранение безопасности финансовой отчетности** могут удалять отчеты других пользователей. 

В выпуске 10.0.8 была введена концепция дат истечения срока годности. Новая необходимая функция включается на странице **Все** в рабочей области управления функциями. Функция **Политики хранения финансовых отчетов** содержит следующие изменения:
* Вновь созданные отчеты будут автоматически помечены как имеющие дату окончания срока действия 90 дней с момента их создания.
* Всем существующим отчетам, установленным до реализации функции, будет предоставлен срок действия 90 дней. Дата может отображаться как пустая в течение короткого периода времени, пока не будет запущена служба финансовой отчетности, будет создан отчет и служба выполнит обновление существующих отчетов, используя пустую дату окончания срока действия. 
* У пользователей с разрешением **Сохранение безопасности финансовой отчетности** есть доступ к этой функциональности. Любому пользователю с полномочиями **Ведение финансовых отчетов** с привилегией **Истечение срока действия ведения финансовых отчетов**, также будет предоставлена возможность изменять срок действия. В настоящее время есть два варианта хранения: 

    * Срок действия 90 дней.
    * Возможность задать для отчета отсутствие ограничения срока действия.

Когда выбран срок действия, такой как 90 дней, он применяется к 90 дням, начиная с сегодняшнего дня. Это другое поведение, чем 90 дней с исходной даты генерации, установленные при создании отчета. 

В будущих функциях будут рассмотрены дополнительные параметры. Срок действия 90 дней будет установлен по умолчанию, а пользователи с соответствующими разрешениями могут переопределить значение по умолчанию на странице списка **Финансовые отчеты**.

## <a name="opening-financial-reports"></a>Открытие финансовых отчетов

При выборе меню **Финансовая отчетность** отображается список финансовых отчетов по умолчанию для компании. После этого можно открыть или изменить отчет. Чтобы открыть один из отчетов по умолчанию, выберите имя отчета. При открытии отчета в первый раз он автоматически создается за предыдущий месяц. Например, при открытии отчета в первый раз в августе 2019 г. отчет создается за 31 июля 2019 г. После открытия отчета можно начать его изучение, выполнив детализацию до определенных частей данных и изменив параметры отчета.

## <a name="creating-and-modifying-financial-reports"></a>Создание и изменение финансовых отчетов

В списке финансовых отчетов можно создать новый отчет или изменить существующий отчет. Если имеются соответствующие разрешения, можно создать новый финансовый отчет, выбрав **Создать** в области действий. Программа конструктора отчетов загружается на ваше устройство. Затем после запуска конструктора отчетов можно создать новый отчет. После сохранения новый отчет отобразится в списке финансовых отчетов. В списке отображаются только отчеты, которые были созданы для компании, используемой в Dynamics 365 Finance. 

## <a name="reporting-tree-definitions"></a>Определения дерева отчетности

Одним из компонентов, которые используются для создания финансовых отчетов, является определение дерева отчетности. Определение дерева отчетности помогает определить структуру и иерархию организации. Это перекрестная аналитическая иерархическая структура, основанная на отношениях аналитик в финансовых данных. Она предоставляет информацию на уровне единицы отчетности и общем уровне для всех единиц в дереве.

Вы можете создать неограниченное количество аналитических структур, чтобы отобразить данные своей организации различными способами. Каждое дерево отчетности может содержать любую комбинацию отделов и блоков сводных данных, но одновременно с определением отчета можно связать только одно дерево отчетности. 

## <a name="update-the-financial-reporting-version-through-slipstreaming"></a>Обновление версии Financial Reporting посредством интегрированной установки

Приложения для финансов и операций обновляются каждый месяц. Однако Financial Reporting не всегда обновляется с этим периодом. Более того, у клиентов есть больше вариантов выбора, когда они реализуют обновления для приложений для финансов и операций. Обновления Financial Reporting устанавливаются автоматически. Финансовая отчетность имеет назначенную версию, которая потребляется в клиентской среде при реализации обновления службы, при инициации времени простоя или при нахождении среды клиента в режиме обслуживания. Этот процесс называется *интегрированная установка* или *сверка*, так как все реализации клиентов настроены на одну и ту же версию Financial Reporting.

Изменения, которые выпускаются в каждой версии, можно найти в разделе [Что нового и что изменилось в Dynamics 365 Finance](../../finance/get-started/whats-new-home-page.md). Обновления платформы и исправления ошибок можно найти в разделе "Дополнительные ресурсы" в нижней части страницы для каждого выпуска.

Выбранная версия интегрированной установки представляет собой проверенную и утвержденную версию Financial Reporting, готовую для производства. Она совместима с любой предыдущей или будущей версией Dynamics 365 Finance. Например, Financial Reporting может находиться в новейшей сборке 10.0.19, пока клиент все еще находится в версии приложения 10.0.16.

> [!NOTE]
> Единственная ситуация, в которой клиенты могут перейти к предыдущей версии (сценарий перехода на предыдущую версию), возникают, если корпорация Майкрософт прекратила истинное развертывание из-за проблемы. После того как исправление станет доступным, оно будет применяться автоматически.

Процесс интегрированной установки полностью автоматический и не требует каких-либо действий клиента. Три топологии используют интегрированную установку, каждая несколько разным образом:

- **Локальная** — в локальных развертываниях интегрированная установка и согласование не поддерживаются.
- **Инфраструктура как услуга (IaaS)** — логика интегрированной установки применяется во время любой операции, которая предпринимает попытку обновления Financial Reporting. Она включает в себя двоичные обновления или рассылки, содержащие двоичные обновления.
- **Самообслуживание** — любая операция, требующая простоя Financial Reporting, применяет логику интегрированной установки:

    - Двоичные обновления или рассылки, включающие двоичные обновления
    - S-исправления или другое время простоя инфраструктуры
    - Развертывания пакетов AOT

## <a name="troubleshooting-issues-opening-report-designer"></a>Проблемы, связанные с устранением неполадок при открытии конструктора отчетов

Существует несколько общих проблем, которые могут вызвать проблемы при открытии конструктора отчетов. Эти проблемы и шаги по их устранению приведены ниже.

Проблема 1. При выборе пункта **Создать** или **Изменить** конструктор отчетов не запускается.

* В Internet Explorer выберите **Параметры**, затем выберите **Свойства браузера**. Перейдите на вкладку **Безопасность**. Выберите "Надежные сайты", затем выберите **Сайты**. В поле **Добавить в зону следующий узел** введите "\*\.dynamics.com" (без кавычек), затем выберите **Добавить**. 
* В Internet Explorer выберите **Параметры**, затем выберите **Свойства браузера**. Перейдите на вкладку **Безопасность**. Выберите "Надежные сайты". В области с меткой "Уровень безопасности для этой зоны" измените значение параметра на **Ниже среднего**.
* Отключите блокировку всплывающих окон в браузере.
* На рабочие станции необходимо установить Microsoft .NET Framework 4.7.2 или более поздней версии. Эту версию платформы Microsoft .NET Framework можно загрузить и установить из [центра загрузки Майкрософт](https://dotnet.microsoft.com/download/dotnet-framework/net472).
* При использовании браузера Chrome необходимо установить расширение ClickOnce, чтобы можно было загрузить клиент конструктора отчетов. При работе в Chrome в режиме инкогнито убедитесь, что расширение ClickOnce включено для работы в режиме инкогнито. Дополнительные сведения о расширении Chrome ClickOnce см. в документе [Требования к системе для облачных развертываний](../../fin-ops-core/fin-ops/get-started/system-requirements.md).
* При использовании Microsoft Edge с браузером Chrome нет необходимости устанавливать расширение ClickOnce для Microsoft Edge Chromium. Однако для загрузки клиента конструктора отчетов необходимо включить параметр ClickOnce. При работе в режиме инкогнито убедитесь, что расширение ClickOnce включено для работы в режиме инкогнито.

    1. Откройте новый браузер в Microsoft Edge.
    2. Введите **edge://flags** и выберите **Ввод**.
    3. Выполните поиск параметра **Поддержка ClickOnce** или используйте эту прямую ссылку: **edge://flags/#edge-click-once**.
    4. Задайте для параметра раскрывающегося меню значение **Включено**.
    5. Выберите **Перезапустить браузер**.

Проблема 2. Пользователю не были назначены необходимые разрешения для использования Financial Reporting. 

* Чтобы проверить, имеет ли пользователь разрешение, выберите **Да** при ошибке "Не удается подключиться к серверу Financial Reporting. Выберите Да, если хотите продолжить, и укажите другой адрес сервера." Затем выберите **Тест подключения**. Если у вас нет разрешения, появится сообщение "Сбой попытки подключения. У пользователя нет соответствующих разрешений для подключения к серверу. Обратитесь к системному администратору."
* Необходимые разрешения перечисляются выше в разделе [Предоставление безопасного доступа к Financial Reporting](#granting-security-access-to-financial-reporting). Безопасность в Financial Reporting основана на этих привилегиях. У вас нет доступа, если вам не назначены такие привилегии (или другая роль безопасности, которая включает эти привилегии). 
* Задача интеграции **Поставщик пользователей компании к компании** (которая также отвечает за и называется интеграцией пользователей) выполняется с 5-минутным интервалом. Для вступления в силу изменений разрешений в Financial Reporting может потребоваться до 10 минут. 

    Если другой пользователь может открыть конструктор отчетов, выберите **Инструменты**, затем выберите **Состояние интеграции**. Убедитесь, что карта интеграции "Поставщик пользователей компании к компании" выполнена успешно, так как вам были назначены разрешения на использование Financial Reporting. 

* Возможно, другая ошибка не позволила выполниться задаче **Интеграция пользователя Dynamics с пользователем Financial Reporting**. Возможно также, что сброс киоска данных был запущен и еще не завершен, либо произошла другая системная ошибка. Попробуйте запустить процесс еще раз позже. Если проблема остается, обратитесь к администратору системы.

Проблема 3. Можно продолжить работу после страницы входа в **ClickOnce для Report Designer**, но невозможно завершить вход в Report Designer. 

* Время, установленное на локальном компьютере при вводе учетных данных для входа, не должно отличаться от времени на сервере Financial Reporting более чем на пять минут. Если имеется разница более пяти минут, система не будет разрешать вход в систему. 
* Если время на компьютере отличается от времени сервера Financial Reporting, рекомендуется включить параметр Windows, чтобы автоматически задать время компьютера. 

## <a name="troubleshoot-report-designer-issues-with-event-viewer"></a>Устранение неполадок, связанных с конструктором отчетов, в средстве просмотра событий

Средство просмотра событий можно использовать для анализа некоторых проблем, возникающих при использовании Financial Reporting. 

### <a name="what-happens-when-you-have-connections-issues-with-financial-reporting"></a>Что происходит при наличии проблем с подключениями к Financial Reporting? 

Ниже приводится несколько шагов, которые могут быть полезными, чтобы сделать беседу со службой поддержки Майкрософт более эффективной и быстрее найти решение. 
 
Следующие шаги описывают процесс включения сообщений средства просмотра событий для Financial Reporting. Журналы, создаваемые средством просмотра событий, помогут инженерам в службе поддержке быстро определить источник проблемы подключения. При подключении к службе технической поддержки отправьте копии этих журналов вместе с обращением.


1. Скопируйте файл RegisterETW.zip на клиентскую рабочую станцию (предпочтительно на рабочий стол) и извлеките [RegisterETW.zip](https://mbs2.microsoft.com/fileexchange/?fileID=60b1106b-d5f8-4e0f-8041-039102505122).
2. Убедитесь, что средство просмотра событий Windows закрыто.
3. Откройте командную строку администратора PowerShell и перейдите в каталог, где расположен RegisterETW.ps1.
4. Выполните следующую команду: .\RegisterETW.ps1

    Успешный вывод в PowerShell будет проверен с сообщением, **Сценарий RegisterETW выполнен**.

    Снова откройте окно просмотра событий, и вы увидите следующие журналы в **Microsoft > Dynamics**:

    * MR-Client
    * MR-DVT
    * MR-Integration
    * MR-Logger
    * MR-Reporting
    * MR_SchedulerTasks
    * MR-Sql
    * MR-TraceManager

5. Воспроизведите проблему в конструкторе отчетов.
6. Экспортируйте события средства журнала MR с помощью средства просмотра событий.

## <a name="troubleshoot-issues-connecting-to-financial-reporting"></a>Устранение неполадок при подключении к Financial Reporting

Проблема: вы получаете сообщение об ошибке "Не удается подключиться к серверу Financial Reporting".

* Определите, происходит ли проблема в браузере Chrome и Microsoft Edge.
* Если проблема возникает только в одном браузере, это может быть связано с технологией ClickOnce. 
* При появлении сообщения об ошибке подключения выберите **Проверить**, чтобы проверить подключение, чтобы увидеть, какое сообщение отображается. 
* Проблема может быть следствием того, что другой пользователь не имеет доступа к Financial Reporting. Если у пользователя нет доступа, он получит сообщение с уведомлением о том, что у него нет разрешений.
* Если проблема возникает в нескольких браузерах, убедитесь, что для часов рабочей станции установлен режим "авто".
* Войдите в систему с учетной записью пользователя, обладающего правами администратора безопасности в Dynamics 365 Finance, а также административными правами доступа к домену в сети, чтобы убедиться, что можно подключиться к вашей рабочей станции. Если они могут подключиться, проблема может быть связана с сетевыми разрешениями.
* Временно отключите брандмауэр на рабочей станции. Если после этого удается подключиться к Report Designer, проблема связана с брандмауэром. Для решения проблемы обратитесь к ИТ-отделу организации.
