# <a name="enable-delayed-tax-calculation-on-journal"></a>Включение отсроченного расчета налога для журнала
[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой теме объясняется, как использовать функцию **Включить отсроченный расчет налога для журнала**, чтобы повысить производительность расчета налога, когда объем строк журнала является огромным.

Текущее поведение расчета налога в журнале запускается в режиме реального времени, когда пользователь обновляет поля, соответствующие налогам, например, налоговая группа/налоговая группа номенклатур. Любое обновление на уровне строки журнала приводит к пересчету суммы налога во всех строках журнала. Он позволяет пользователю просмотреть сумму рассчитанного налога в реальном времени, но может также привести к снижению производительности, если объем строк журнала является огромным.

Эта функция предоставляет возможность отложенного расчета налогов для решения проблем с производительностью. Если эта функция включена, сумма налога будет вычисляться только тогда, когда пользователь нажимает команду "Налог" или разносит журнал.

Пользователь может включить/отключить параметр на трех уровнях:
- По юридическому лицу
- По наименованию журнала
- По заголовку журнала

Система принимает значение параметра в заголовке журнала как окончательное. Значение параметра в заголовке журнала по умолчанию используется из наименования журнала. При создании наименования журнала значение параметра для наименования журнала по умолчанию берется из параметра главной книги.

Поля "Фактическая сумма налога" и "Рассчитанная сумма налога" в журнале будут скрыты, если этот параметр включен. Это сделано, чтобы не путать пользователя, поскольку значение этих двух полей всегда будет показывать 0, пока расчет налога не будет запущен пользователем.

## <a name="enable-delayed-tax-calculation-by-legal-entity"></a>Включение отсроченного расчета налога по юридическим лицам

1. Перейдите в раздел **Главная книга > Настройка главной книги > Параметры главной книги**.
2. Откройте вкладку **Налог**
3. На экспресс-вкладке **Общие** найдите параметр **Отсроченный расчет налога**, включите или выключите его

![](media/delayed-tax-calculation-gl.png)



## <a name="enable-delayed-tax-calculation-by-journal-name"></a>Включение отсроченного расчета налога по наименованию журнала

1. Перейдите в раздел **Главная книга > Настройка журнала > Наименования журналов**.
2. На экспресс-вкладке **Общие** найдите параметр **Отсроченный расчет налога**, включите или выключите его

![](media/delayed-tax-calculation-journal-name.png)

## <a name="enable-delayed-tax-calculation-by-journal"></a>Включение отсроченного расчета налога по журналам

1. Перейдите в раздел **Главная книга > Записи в журнале > Общие журналы**
2. Щелкните **Создать**
3. Выберите название журнала
4. Щелкните **Настройка**
5. Найдите параметр **Отсроченный расчет налога**, включите или выключите его

![](media/delayed-tax-calculation-journal-header.png)
