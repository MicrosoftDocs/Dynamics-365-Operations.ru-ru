# <a name="year-end-close"></a>Закрытие на конец года

[!include [banner](../includes/banner.md)]

В этом разделе описывается необходимая настройка и действия для запуска процесса закрытия главной книги на конец года.

В конце финансового года необходимо выполнить процесс закрытия на конец года, чтобы перенести входящие сальдо в новый год. В большинстве организаций процесс закрытия на конец года выполняется несколько раз. Первое выполнение переносит сальдо в новый финансовый год. Затем процесс можно запустить заново, любое требуемое число раз, для переноса сальдо из корректируемых записей в новый финансовый год.

Во время процесса закрытия на конец года могут создавать два типа возможных проводок. Открывающая проводка создается всегда и используется для создания начальных сальдо в новом финансовом году. Открывающая проводка показывает сальдо счета ГК балансового отчета в новом финансовом году и сальдо из сальдо счета ГК прибылей и убытков на счете учета нераспределенной чистой прибыли в новом финансовом году. Закрывающие проводки являются необязательными и могут создаваться для сведения сальдо счетов прибылей и убытков к нулю в закрываемом финансовом году.

## <a name="prepare-to-run-the-year-end-close"></a>Подготовка к выполнению закрытия на конец года

Перед выполнением процесса закрытия на конец года проверьте следующие параметры:

На странице **Счет ГК**:

- Убедитесь, что поле **Тип счета ГК** правильно настроено для каждого счета ГК. Тип счета ГК определяет, будет ли сальдо счета ГК переносится вперед как входящее сальдо или закрываться в нераспределенной чистой прибыли в открывающей проводке.
- Сальдо счета ГК может быть перенесено на новый счет ГК во время закрытия на конец года. Введите новый счет ГК в поле **Открытие счета**. Обычно это поле используется для счетов ГК балансового отчета, когда счета ГК деактивируется, и новый счет ГК используется для нового финансового года.

На странице **Параметры главной книги** в разделе **Закрытие финансового года**:

- Параметр **Удаление существующих записей на конец года при повторном закрытии года** используется для указания, следует ли удалять созданные системы открывающие проводки из предыдущего закрытия на конец года при повторном выполнении закрытия на конец года. Если этот параметр имеет значение **Да**, предыдущая открывающая и необязательная закрывающая проводки удаляется и создается новая открывающая или закрывающая проводка на основе текущего сальдо. Если этот параметр имеет значение **Нет**, предыдущая открывающая и необязательная закрывающая проводки остаются и создается дополнительная открывающая или закрывающая проводка для переноса сальдо вперед из корректирующих проводок, разнесенных после предыдущего закрытия на конец года.
- Параметр **Создать закрывающие проводки при переносе** используется для создания закрывающих проводок в закрываемом финансовом году, чтобы сделать сальдо счетов прибылей и убытков равным 0 (нулю). Если этот параметр имеет значение **Да**, создается как открывающая проводка, так и закрывающая проводка. Если этот параметр имеет значение **Нет**, открывающая проводка создается в следующем финансовом году для переноса сальдо. Сальдо счетов прибылей и убытков остаются в конце финансового года.
- Параметр **Задать для финансового года статус закрытого на постоянной основе** используется для перевода финансового года в состояние постоянно закрытого. Этот параметр следует использовать аккуратно, поскольку периоды c окончательно закрытым статусом невозможно открыть повторно. Таким образом, корректировки не могут быть разнесены в этот финансовый год. Рекомендуется установить для этого параметра значение **Нет**.
- Параметр **Код ваучера должен быть заполнен** удален. Ваучер теперь требуется при выполнении процесса закрытия на конец года. В этот момент код ваучера вводится вручную.

На странице **Фискальный календарь**:

- Следующий финансовый год должен существовать перед выполнением закрытия на конец года. В противном случае начальные сальдо не могут быть созданы в открывающем периоде.

На странице **Календарь книги учета**:

- Необязательно: каждый финансовый период закрываемого финансового года можно установить в состояние **Заблокировано**, чтобы исключить ввод новых проводок. При обнаружении записей корректировки периоды в состоянии "Заблокировано" можно повторно открыть, чтобы можно было разнести записи корректировки, даже если процесс закрытия на конец года уже был выполнен.

На странице **Настройка шаблона закрытия на конец года**:

- Когда функция **Улучшения главной книги на конец года** включена, процесс настройки шаблона закрытия на конец года отделен от процесса выполнения закрытия на конец года. Шаблон можно определить на странице **Настройка шаблона закрытия на конец года** (**Главная книга \> Настройка книги учета \> Настройка шаблона закрытия на конец года**), или к нему можно получить доступ из процесса закрытия на конец года.

## <a name="define-year-end-close-templates"></a>Определение шаблонов закрытия на конец года

После настройки системы можно запустить процесс закрытия на конец года. На странице **Настройка шаблона закрытия на конец года** можно определить шаблона для группы юридических лиц, для которых будет выполняться процесс закрытия на конец года. Шаблон можно повторно использовать при каждом закрытии на конец года, но его можно изменить при изменении вашей организации.

Во-первых, задайте поле **Имя группы** для шаблона и выберите финансовый календарь. Имя группы должно указывать группу включенных юридических лиц. При определении групп юридических лиц помните, что юридические лица могут быть включены в одну и ту же группу только в том случае, если для них выбран один и тот же финансовый календарь. Например, шаблоны могут настраиваться на основе географического положения, и отдельные группы могут быть созданы для юридических лиц в Северной Америке, юридических лиц в Европе, Среднем Востоке и Африке (EMEA) и юридических лиц в Азиатско-Тихоокеанском бассейне (APAC).

Затем эти юридические лица могут быть добавлены в шаблон. Юридические лица можно добавлять, выбирая организационную иерархию или выбирая юридические лица. Если выбрана организационная иерархия, только юридические лица в этой иерархии, использующие выбранный финансовый календарь, будут добавлены в шаблон. При использовании юридических лиц для добавления в шаблон можно добавлять только юридические лица с одинаковым финансовым календарем. Один и тот же финансовый календарь необходим, поскольку закрытие на конец года выполняется путем выбора финансового года, который может быть разным в разных календарях.

После добавления юридических лиц определите счета ГК нераспределенной чистой прибыли для каждого юридического лица.

Вкладка **Финансовая аналитика** используется для определения того, какие финансовые аналитики будут использоваться в открывающей проводке. Обратите внимание, что настройка на этой вкладке относится только к юридическому лицу, выбранному в сетке **Юридические лица**. Настройку необходимо повторить для каждого юридического лица в сетке.

Параметр **Переместить аналитики балансового отчета** используется для указания, требуется ли сохранить в открывающей проводке финансовые аналитики по проводкам, разнесенным на балансовые счета. Рекомендуется установить для этого параметра значение **Да**. Настройка аналитик балансового отчета не влияет на существующие сальдо в счетах ГК нераспределенной чистой прибыли. Эти сальдо определяются аналитиками прибылей и убытков, которые определены в следующем разделе. Например, финансовый год 2019 — это первый год, который был закрыт, а параметр **Закрыть все** использовался для выбора аналитик **Подразделение** и **Центр затрат** для закрытия. В этом случае отдельный счет нераспределенной чистой прибыли был создан для каждой комбинации подразделения и центра затрат. При закрытии года в конце финансового года 2020 нераспределенная прибыль из предыдущего года остается в точном соответствии с тем, как она была разнесена, даже если для параметра **Переместить аналитики балансового отчета** задано значение **Нет**. Сальдо, учитываемые в нераспределенной чистой прибыли от предыдущих закрытий на конец года, никогда не изменяются.

Раздел **Переместить аналитики прибыли и убытков** используется для указания того, какие финансовые аналитики для проводок, разнесенных на счета прибыли и убытков, будут перемещены на счет нераспределенной чистой прибыли в ГК. Сначала укажите финансовые аналитики, относящиеся к выбранному юридическому лицу. Эти финансовые аналитики включают любые финансовые аналитики, по которым выполнялась разноска в течение года, даже если эта финансовая аналитика не является частью активной структуры счетов. Затем определите каждую аналитику как **Закрыть один** или **Закрыть все**. Значение по умолчанию — **Закрыть все**. Этот параметр сохраняет исходные значения финансовой аналитики из разнесенных проводок и использует их для создания входящих сальдо для счета нераспределенной чистой прибыли. Отдельные начальные сальдо нераспределенной чистой прибыли будут созданы для каждой уникальной комбинации значений финансовой аналитики. Если выбрано значение **Закрыть один**, все разнесенные проводки, имеющие эту финансовую аналитику, будут суммироваться в начальном сальдо нераспределенной чистой прибыли для значения аналитики, введенного в поле, которое появляется после **Закрыть один**. Например, все проводки для финансового года были разнесены со структурой счета **Счет ГК - подразделение**. В финансовой аналитике **Подразделение** в шаблоне выбрано значение **Закрыть один** и введено значение **100** в качестве значения аналитики. Если общий доход по всем разнесенным проводкам для подразделений 200, 300 и 400 составляет 100 000 долларов, будет создано одно входящее сальдо для **Нераспределенная чистая прибыль - 100**. Если выбрать **Закрыть один**, но оставить значение финансовой аналитики пустым, все проводки будут разнесены на нераспределенную чистую прибыль, а значением аналитики будет пустым.

## <a name="run-the-year-end-close-process"></a>Выполнение процесса закрытия на конец года

После создания шаблонов закрытия на конец года можно начать процесс закрытия на конец года на странице **Закрытие на конец года** (**Главная книга \> Закрытие периода \> Закрытие на конец года**). Перед выполнением закрытия на конец года можно добавить новые шаблоны закрытия на конец года или изменить существующие шаблоны, выбрав **Настройка шаблона закрытия на конец года**, чтобы открыть страницу настройки для шаблонов.

Выберите шаблон закрытия на конец года, а затем в области действий выберите **Выполнить закрытие на конец года**. Выберите все юридические лица или подмножество юридических лиц из шаблона, для которых требуется выполнить закрытие на конец года. Если закрытие на конец года выполняется впервые в финансовом году, скорее всего вы выберите все юридические лица для создания начальных сальдо для всех из них. Если вы ранее выполняли закрытие на конец года, можно захотеть повторно выполнить его только для юридических лиц, для которых были разнесены записи корректировки.

Далее выберите финансовый год, для которого требуется запустить процесс закрытия на конец года. Если для последнего периода финансового года имеется более одного периода закрытия, поле **Название периода** становится доступным. Затем можно выбрать закрывающий период, который будет использоваться для разноски закрывающей проводки, если настройка была определена для создания закрывающей проводки.

Затем введите код ваучера. Для всех юридических лиц, выбранных для процесса закрытия на конец года, будет использоваться тот же код ваучера. Код ваучера не создается с использованием номерной серии.

По умолчанию процесс закрытия на конец года выполняется в пакетном режиме. Поэтому после его инициирования можно вернуться к другим действиям.

Поскольку структуры счетов могут меняться в течение финансового года, соответствующая структура счета не всегда может быть определена. Поэтому процесс закрытия на конец года не придерживается структур счетов. При создании открывающих проводок сальдо переносится вперед с финансовыми аналитиками, как определено в шаблоне закрытия на конец года. Записи начальных сальдо могут включать финансовые аналитики, которые больше не входят в текущую структуру счетов, и комбинации сегментов, которые больше не являются допустимыми в текущей структуре счетов. Если ваша организация хочет исключить финансовой аналитики для начального сальдо нераспределенной чистой прибыли, определите для этой финансовой аналитики значение **Закрыть один** и оставьте значение аналитики пустым.

После завершения процесса создается запись для каждой комбинации юридического лица и финансового года. Вы увидите записи только для юридических лиц, к которым у вас есть доступ. Каждая запись включает в себя системные дату и время, когда был выполнен процесс, и прямую ссылку на ваучеры, созданные для закрытия на конец года.

[![Записи, созданные на экспресс-вкладке «История закрытия на конец года» на странице закрытия на конец года](./media/run-yr-end-close.png)](./media/run-yr-end-close.png)

Если вы повторно перезапустите закрытие на конец года, вы увидите одну или несколько записей для каждой комбинации юридического лица и финансового года, в зависимости от настройки параметра **Удаление существующих записей на конец года при повторном закрытии года** на странице **Параметры главной книги**:

- Если для этого параметра задано значение **Да**, ваучер для предыдущего закрытия на конец года удаляется. Запись помечается как **Сторнированная**, с отметкой даты и времени, когда сторнирование было выполнено. Кроме того, связь с кодом ваучера удаляется. После завершения закрытия на конец года новая запись будет создана для нового создаваемого ваучера.
- Если для параметра установлено значение **Нет**, запись для предыдущего закрытия на конец года остается, вместе со ссылкой на ваучер. При каждом повторном запуске закрытия на конец года будет создана новая запись вместе со ссылкой на новые ваучеры.

## <a name="reverse-the-year-end-close-process"></a>Сторнирование процесса закрытия на конец года

На странице **Закрытие на конец года** можно сторнировать закрытие на конец года. Выберите запись для комбинации юридического лица и финансового года, которую необходимо сторнировать, затем выберите **Сторнировать закрытие на конец года**. В процессе сторнирования удаляются все открывающие и закрывающие ваучеры, которые были созданы для финансового года. Запись помечается как **Сторнированная**, с отметкой даты и времени, когда сторнирование было выполнено. Кроме того, связь с кодом ваучера удаляется. Как и процесс закрытия на конец года, сторнирование выполняется в пакетном режиме.

Флажок **Показывать сторнированные**, расположенный над сеткой, позволяет скрыть или отобразить записи для сторнированных процессов закрытия на конец года. После выполнения процесса **Сторнировать закрытие на конец года** для просмотра записи, возможно, потребуется установить флажок **Показывать сторнированные**. Можно настроить дополнительные фильтры для просмотра других сведений.

[![Использование флажка «Показать сторнированные» для просмотра записей для сторнированных процессов закрытия на конец года на странице закрытия на конец года](./media/rvrs-yr-end-close.png)](./media/rvrs-yr-end-close.png)

Дополнительные сведения см. в разделах [Закрытие ГК в конце периода](close-general-ledger-at-period-end.md) и [Закрытие финансового года](tasks/close-fiscal-year.md).
