# <a name="documents-pending-accounting"></a>Документы, ожидающие учета

[!include [banner](../includes/banner.md)]

[!include [banner](../includes/preview-banner.md)]

В этой статье описываются функции страницы **Документы, ожидающие учета**.

В Microsoft Dynamics 365 Finance 10.0.30 доступна функция **Повышенная производительность платформы учета документов-источников**. Она оптимизирует процесс разноски документов-источников, начиная с накладных с произвольным текстом.

Если эта функция включена, разноска документа субкниги выполняется отдельно от создания учета или *журнализации*, при которой создаются полные данные учета, переносимые в главную книгу. Например, в процессе разноски накладной с произвольным текстом накладная клиента в модуле **Расчеты с клиентами** записывается до создания полного учета. Эта улучшенная функция позволяет записывать накладные клиентов быстрее, откладывая создание учета.

На странице **Документы, ожидающие учета** доступны следующие кнопки.

- **Создать учет** — отправляет документ, который в настоящее время ожидает учета, на журнализацию.
- **Просмотр распределений** — показывает распределение по бухгалтерским счетам для выбранного документа в списке.
- **Просмотр журнала ошибок** — показывает сведения об ошибках для документов, состояние учета которых указывает на ошибки. Те же сведения об ошибке можно посмотреть, если выбрать ссылку **Журнал ошибок** в строке документа.

Создание учета выполняется автоматическим фоновым процессом, который называется **Процессор платформы учета**. Дополнительные сведения о настройке всех процессов автоматизации см. в разделе [Автоматизация процессов](../../fin-ops-core/dev-itpro/sysadmin/process-automation.md).
