# <a name="subledger-transfer-to-the-general-ledger"></a>Перенос субкниги в главную книгу

[!include [banner](../includes/banner.md)]

В этой теме описываются возможности, которые относятся к правилам переноса партий записей журнала субкниги.

В версии 8.1 были сделаны изменения, разрешающие перенос правил, поэтому параметр **Синхронный** устарел. Дополнительные сведения см. в разделе [Удаленные или устаревшие функции для Finance and Operations](../../fin-ops-core/dev-itpro/migration-upgrade/deprecated-features.md?toc=%2fdynamics365%2ffinance%2ftoc.json#finance-and-operations-81-with-platform-update-20).

Для переноса партий субкниги доступны следующие параметры:

- **Асинхронно** — перенос записей учета субкниги в главную книгу планируется немедленно. Ваучер ГК будет записан, как только будут доступны ресурсы, чтобы обработать этот запрос на сервере.
- **Запланированная партия** — записи учета субкниги, которые должны быть перенесены, добавляются в очередь обработки в главной книге. Записи в очереди будут обрабатываться в том порядке, в котором они получены. Каждый ваучер ГК будет обновлять счета в запланированное время, как только станут доступны ресурсы, чтобы обработать это пакетное задание на сервере.

В версии 10.0.8 были сделаны усовершенствования, повышающие производительность параметра **Асинхронно**. Эта функция включается в рамках функции **Оптимизация производительности переноса субкниги в главную книгу**.

Функции асинхронной передачи партий вспомогательной книги позволяют улучшить перенос данных из субкниги в главную книгу. Благодаря группировке наборов более мелких проводок и передаче проводок группами, функциональность обрабатывает транзакции более эффективно. При группировке проводок ресурсы сервера пакетной обработки используются более эффективно.

Для асинхронной передачи пакетов субкниги требуется, чтобы сервер пакетной обработки был настроен, подключен к сети и работал, так как задачи пакетной обработки создаются для немедленного выполнения на сервере обработки пакетных заданий. Когда функция **Оптимизация производительности переноса субкниги в главную книгу** включена, необходимо также включить системное пакетное задание **Автоматизация процессов** с именем **Системное задание по опросу автоматизации процессов**. Подробнее см. в разделе [Автоматизация процессов](../../fin-ops-core/dev-itpro/sysadmin/process-automation.md).

Изменение эффективности на уровне пакетов использует одно повторяющееся пакетное задание для всех юридических лиц в системе. Во время выполнения создается новое пакетное задание для обработки необходимых записей, которые еще не были перемещены. Дополнительные параметры можно контролировать на странице **Автоматизация процессов** администратора системы. На этой странице можно изменить фоновый процесс, изменить частоту и определить период сна.

Дополнительные сведения о процессе автоматической настройки см. в разделе [Автоматизация процессов](../../fin-ops-core/dev-itpro/sysadmin/process-automation.md).
