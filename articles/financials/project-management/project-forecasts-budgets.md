# <a name="project-forecasts-and-budgets"></a>Прогнозы и бюджеты по проекту

[!include [banner](../includes/banner.md)]

Microsoft Dynamics 365 for Finance and Operations предоставляет два способа управления проектами: прогнозы проекта и бюджеты проекта. 

Используйте прогнозирование проекта, если организация имеет рабочую перспективу и фокусируется на выручке и затратах по конкретным проводкам. Используйте бюджетирование проекта, если организация больше фокусируется на финансовых суммах. 

Прогнозы проекта и бюджеты проекта используют прогнозные модели, чтобы включить прогнозируемые суммы проводки. 

Каждый способ имеет свои преимущества. Рекомендуется рассмотреть следующие аспекты перед выбором способа для организации.

Поскольку прогнозы основываются на определенной дате, необходимо ввести период с определенной даты. | Можно распределить проводки по всему проекту или финансовому периоду. ## <a name="project-forecasts"></a>Прогнозы проекта
При использовании прогнозирования проекта можно вводить прогнозные проводки в формы прогнозов для каждого типа проводки. Все атрибуты, предусмотренные для фактической проводки, можно использовать для прогнозной проводки (например, рентабельности по строке, атрибуты строки, работников или описания). Также можно прогнозировать сроки от возникновения затрат до выставления накладных клиенту. 

Проводки прогнозирования по проекту основываются на единицах измерения и суммах. 

Каждый прогноз проекта должен быть связан с прогнозной моделью. При вводе прогнозной проводки прогнозная модель предлагается автоматически. Прогнозная модель играет роль контейнера для прогнозных проводок. 

Можно назначить модели прогноза как подмодели для модели. Это позволит выполнять прогноз по региону, периоду времени или отделу. Можно скопировать прогнозные проводки в модели, чтобы создать новую модель, а также перенести проводки в главную книгу. Поскольку между прогнозом и моделью имеется однозначное соответствие, каждая прогнозная модель представляет собой отдельный бюджет для проекта. 

В прогнозных моделях может использоваться сокращение прогноза в качестве контрольного механизма для проектов. При сокращении прогноза фактические проводки снижают остатки прогнозных проводок. Однако, поскольку снижение прогноза применяется к верхнему проекту в иерархии, видимость изменений в прогнозе ограничена. Например, если работник связан с подпроектом, фактические проводки по работнику разносятся в родительский проект. В результате может быть трудно отслеживать изменения, поскольку сложно определить, какая проводка в каком подпроекте вызвала уменьшение прогнозной суммы. Поэтому рекомендуется создать копию модели прогноза для уменьшения прогноза. Затем можно использовать отчеты для просмотра того, что было спрогнозировано первоначально. 

Прогнозы проектов можно пересматривать, копировать, удалять или перемещать в бюджет главной книги. Однако процесс не контролируется. Любой работник с разрешением на форму прогноза может вносить изменения без проверки.

-   **Пересмотр** — можно вносить изменения в прогнозную проводку в тех же формах, где изначально были сделаны записи.
-   **Копирование или удаление** — при копировании прогнозных проводок строки проводок одной прогнозной модели копируются в другую прогнозную модель. При удалении прогноза прогнозные проводки удаляются из прогнозной модели. Для ограничения количества прогнозных проводок, которые копируются или удаляются, выберите конкретные типы проводок и даты. Это позволяет скопировать или удалить только определенные части прогноза.
-   **Перемещение** — при перемещении прогноза проекта в бюджет главной книги прогнозные проводки из прогнозной модели переносятся в бюджет главной книги. Можно перезаписать любые ранее перемещенные проводки в том бюджете главной книги, куда вы перемещаете свой прогноз проекта.

## <a name="project-budgets"></a>Бюджеты проекта
Бюджетирование проекта — более простой метод, чем прогнозирование, хотя он интегрируется с прогнозными моделями. Этот метод использует одну и ту же форму ввода исходных сведений о бюджете и различных изменений. Кроме того, этот метод позволяет создавать проекции на основе только суммы, категории или мероприятия. 

В бюджетировании проекта все исходные бюджеты и изменения должны отправляться в workflow-процесс проекта на утверждение. Workflow-процессы обеспечивают более жесткий контроль над процессами и позволяют вести историю изменений. 

Бюджетирование проекта похоже на бюджетирование ГК, однако настроить его можно быстрее и проще. Многие параметры в бюджете Главной книги, такие как номерные серии или валюта, не нужно отдельно настраивать для проектов.

Бюджеты проектов автоматически связываются с 2 моделями прогнозирования: одна — для исходного бюджета, другая — для оставшегося. Следовательно, отчеты, основанные на прогнозных моделях, могут использовать бюджетные данные. При отправке бюджета проекта система создает прогнозные проводки на базе связанных моделей, которые используются для целей отчетности и контроля.

## <a name="forecast-models"></a>Модели прогнозов
Прогнозные модели должны иметь однослойную иерархию. Это значит, что один прогноз проекта должен быть связан с одной прогнозной моделью.

При использовании прогнозирования проекта можно определить модели в качестве подмоделей. Затем можно создавать прогнозы по подразделениям, периоду времени или региону. Например, можно создать прогнозную модель на год, а затем можно создать подмодели для прогнозов северо-восточного, юго-восточного, северо-западного регионов, которые отправляются главами региона. Выбирая различные параметры в доступных в отчетах, можно просмотреть сведения по общему прогнозу или по подмодели.
