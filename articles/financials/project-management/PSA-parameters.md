# <a name="project-service-automation-integration-parameters"></a>Параметры интеграции Project Service Automation

На странице **параметры интеграции Project Service Automation** можно конфигурировать параметры интеграции Project Service Automation с Finance and Operations. Для проектов, которые должны быть успешно синхронизированы из Project Service Automation в Finance and Operations, необходимо создать следующее.

> [!NOTE]
> Dynamics 365 for Finance and Operations версии 8.0. доступна интеграция задач проекта, категорий проводок расходов, оценки часов, оценки затрат и блокировка функциональности.




| **Вкладка**                      | **Поле**                          | **Описание**                    |
|------------------------------|------------------------------------|--------------------------------|
| **Общие сведения**                  | **Тип проекта по умолчанию**               | Выбор значения по умолчанию для **типа проекта**, которое является при синхронизации проектов из Project Service Automation, если не указано значение по умолчанию в шаблоне интеграции. Во время синхронизации новые проекты будут иметь **тип проекта**, заданный на это значение и могут быть обновлен при синхронизации строк контракта проекта из Project Service Automation.               |
| **Общие сведения**                  | **Категория времени**                      | Выбор значения по умолчанию для **категории времени**, являющегося при синхронизации оценки часов из Project Service Automation. Во время синхронизации часовые прогнозы нового проекта в Finance and Operations будут иметь **Категорию** установленную на это значение когда оценки часов и фактические значения времени синхронизируются из Project Service Automation.   |
| **Общие сведения**                  | **Категория сборов**                       | Выбор значения по умолчанию для **категории сбора**, являющегося при синхронизации оценки сборов из Project Service Automation. Во время синхронизации новые транзакции по сборам в Finance and Operations будут иметь **Категорию** установленную на это значение когда оценки сборов и фактические значения времени синхронизируются из Project Service Automation.          |
| **Группа проектов по умолчанию**   | **Тип проекта** | Щелкните **создать** для добавления строки, где можно выбрать тип проекта, для которого требуется задать группу проекта по умолчанию. В конфигурации только один раз можно выбрать конкретный тип проекта.              
|                              | **Группа проектов**          | Выбор группы проектов по умолчанию для указанного типа проекта. Когда новые проекты синхронизируются с Project Service Automation, **группа проектов** будет задана по умолчанию на основе типа проекта, если не указано значение по умолчанию в шаблоне интеграции.  |
| **Значения по умолчанию для типа оплаты**    | **Тип выставления счетов** | Щелкните **Создать** для добавления строки, где можно выбрать тип выставления счетов, для которого требуется задать свойство строки по умолчанию. В конфигурации только один раз можно выбрать конкретный тип выставления счетов.          |
|                              | **Свойство строки**| Выбор свойства строки по умолчанию для указанного типа выставления счетов. Когда новые оценки часов, новые оценки расходов или новые фактические данные синхронизируются из Project Service Automation, **свойство строки** будет задано по умолчанию на основе типа выставления счетов.          |
| **Блокировка функциональности**    |                   | Выберите функцию для отключения в Finance and Operations для проектов и контрактов, которые происходят из Project Service Automation. Например, можно отключить возможность редактирования контрактов и проектов, создать структурные декомпозиции работ и ввести табели учета рабочего времени в Finance and Operations. Поля, связанные с учетом, будут оставаться включенными, даже если они становятся недоступными из-за настройки параметра. По умолчанию все функциональные возможности будут включены.           |
