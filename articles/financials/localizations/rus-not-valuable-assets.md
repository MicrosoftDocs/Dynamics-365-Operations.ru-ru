# <a name="not-valuable-fixed-assets-nvfas-russia"></a>Малоценные основные средства (МОС) (Россия)

[!include [banner](../includes/banner.md)]

Малоценные номенклатуры с высоким износом, которые используются на рабочем месте, можно отслеживать и учитывать как особый тип основных средств, которые называются *малоценные основные средства* (МОС). МОС — это номенклатуры, стоимость которых меньше определенного предела. Общая стоимость МОС должна списываться для амортизации в первом месяце использования.

При приобретении основных средств регулярные основные средства и МОС делятся на основе цены покупки (затрат). Значение, заданное для поля **Макс. стоимость МОС**, когда вы [настраиваете параметры ОС для МОС](#set-up-fixed-assets-parameters-for-nvfas), определяет стоимость, которая отличает обычные основные средства и МОС.

После приобретения и регистрации МОС можно выполнять следующие задачи:

- Автоматизация процесса ввода МОС в эксплуатацию и добавления их в последующие проводки амортизации.
- Печать отчетов МБ-2, МБ-4 и МБ-8.

## <a name="set-up-fixed-assets-parameters-for-nvfas"></a>Настройка параметров ОС для МОС 

1. Перейдите в раздел **Основные средства (Россия)** \> **Настройка** \> **Параметры основных средств**.
2. В поле **Базовая модель стоимости** выберите базовую модель стоимости по умолчанию.
3. В поле **Макс. стоимость МОС** введите максимальный лимит для себестоимости МОС.
4. В поле **Профиль учета МОС** выберите профиль учета номенклатуры для МОС.
5. На вкладке **Номерные серии** в поле **Код номерной серии** выберите код номерной серии для типов ссылок **Инвентарный номер МОС** и **Номер журнала выдачи МОС**.
6. На вкладке **Документ** на экспресс-вкладке **Типы документов** в поле **Код номерной серии** выберите код номерной серии для типов документов **Акт выбытия МОС (МБ-4)** и **Акт на списание МОС (МБ-8)**.

## <a name="set-up-fixed-asset-groups-for-nvfa"></a>Настройка групп ОС для МОС

1. Откройте **Основные средства (Россия)** \> **Настройка** \> **Группы ОС**.
2. На странице **Группы ОС** создайте запись и заполните поля. В поле **Тип группы** выберите **МОС**.

## <a name="set-up-the-identification-of-fixed-asset-groups-for-nvfas"></a>Настройка идентификации групп ОС для МОС

1. Выберите **Основные средства (Россия)** \> **Настройка** \> **Спецодежда, спецоснастка и МОС** \> **Идентификация групп ОС**.
2. На странице **Условие идентификации групп ОС** создайте запись и заполните поля.
3. Выберите **Соответствие** и укажите, как номенклатура связана с группой основных средств.

    > [!NOTE]
    > Система использует этот параметр для заполнения полей **Группа ОС** и **Срок службы** на странице **Строки журнала выдачи спецодежды, спецоснастки и МОС**. Дополнительные сведения см. в разделе [Создания записей МОС, ввод в эксплуатацию и проводки амортизации для МОС](#Generate-nvfa-records,-putting-into-operation,-and-depreciation-transactions-for-nvfas).

## <a name="set-up-inventory-dimensions-for-nvfas"></a>Настройка складских аналитик для МОС

Используйте эту процедуру, чтобы настроить складские аналитики для МОС, спецодежды и элементов спецоснастки.

1. Откройте **Управление сведениями о продукте** \> **Настройка** \> **Группы аналитик и вариантов** \> **Группы аналитик отслеживания**.
2. Создайте группу аналитик.
3. Для аналитики **Номер партии** выберите флажок **Активная**, чтобы включить пакетный учет МОС. Номенклатуры, имеющие разные цены, учитываются разными партиями.
4. Также установите флажки **Первичная аналитика хранения** и **Финансовые запасы** для аналитики **Номер партии**.
5. Для аналитики **Серийный номер** установите флажок **Активно**. Данная аналитика используется, когда актив вводится в эксплуатацию.
6. Также установите флажки **Пропуск для приходов** и **Пропуск для расходов** для аналитики **Серийный номер**. Когда эти флажки установлены, можно выполнять операции склада, без необходимости задания серийного номера.
7. Для аналитики **Профиль учета** установите флажки **Активно** и **Первичная аналитика хранения**, чтобы включить учет профиля запасов для МОС.

## <a name="set-up-items-as-nvfas"></a>Настройка номенклатур как МОС

1. Щелкните **Управление сведениями о продукте** \> **Продукты** \> **Выпущенные продукты**.
2. Создайте новую номенклатуру или дважды щелкните номенклатуру, чтобы открыть запись номенклатуры.
3. Выберите **Настройка** \> **Группы аналитик** для настройки групп аналитик хранения и отслеживания для номенклатуры.
4. В поле **Группа моделей номенклатур** выберите группу моделей номенклатур.
5. В поле **Группа ОС** выберите группу основных средств, которая имеет тип группы **МОС** для номенклатуры.

## <a name="set-up-officials-for-the-nvfa-statement-of-writing-off-no-mb-8"></a>Настройка должностных лиц для отчета "Акт на списание МОС (№ МБ-8)

Эта процедура используется для настройки участников и руководителя комиссии, ответственных за акт о списании МОС (№ МБ-8).

1. Откройте **Управление организацией** \> **Настройка** \> **Контакты** \> **Должностные лица**.
2. На вкладке **Основные средства** выберите **Акт на списание МОС (МБ-8)**.
3. Выберите **Добавить**, чтобы создать запись.
4. В поле **Должность** выберите **Член комиссии** или **Председатель комиссии**, чтобы указать, является выбранный сотрудник членом комиссии или ее председателем. Можно выбрать только одного сотрудника в качестве руководителя.
5. В поле **Имя** выберите имя сотрудника.

## <a name="register-an-nvfa-by-using-a-purchase-order"></a>Регистрация МОС с помощью заказа на покупку

1. Откройте **Расчеты с поставщиками** \> **Общее** \> **Заказы на покупку** \> **Все заказы на покупку**.
2. В области **Панель операций** выберите **Создать** \> **Заказ на покупку**.
3. В поле **Счет поставщика** выберите необходимый счет поставщика, затем выберите **ОК**.
4. Создайте строку заказа на покупку.
5. В поле **Код номенклатуры** выберите код номенклатуры.
6. В поле **Количество** введите заказываемое количество номенклатуры.
7. В поле **Цена за единицу** введите цену покупки для единицы номенклатуры. Если цена покупки основного средства меньше, чем стоимость, указанное в поле **Макс. стоимость МОС** на странице **Параметры основных средств** без суммы налога, номенклатуры рассматриваются как МОС. В этом случае полю **Профиль учета** на вкладке **Продукт** экспресс-вкладки **Сведения по строке** автоматически присваивается значение, введенное на странице **Параметры основных средств**.
8. Выполните разноску журнала накладных поставщика.

> [!NOTE]
> Если система определяет номенклатуру в строке покупки как МОС, не следует указывать инвентарный номер ОС для этой строки в заказе на покупку.

## <a name="generate-nvfa-records-putting-into-operation-transactions-and-depreciation-transactions-for-nvfas"></a>Создания записей МОС, проводки ввода в эксплуатацию и проводки амортизации для МОС

Эта процедура используется для автоматического создания записей МОС на странице **Малоценные ОС** (**Основные средства (Россия)** \> **Общее** \> **Малоценные ОС**) и автоматического создания журналов ОС для ввода в эксплуатацию и амортизации МОС.

1. Выберите **Основное средство (Россия)** \> **Журналы** \> **Выдача спецодежды, спецоснастки и МОС**.
2. Выберите **Создать** для создания журнала, затем заполните поля.
3. Выберите **Строки**, затем выберите **Создать** для создания записи.
4. Заполните поля **Номенклатура** и **Количество**. Значения в полях **Ответственное лицо** и **Местоположение** заполняются из журнала. Поля **Срок службы** и **Группа ОС** заполняются автоматически со страницы **Условие идентификации групп ОС**.
5. Проверьте и заполните поля на вкладке **Аналитики продуктов**.
6. Закройте страницу **Строки журнала выдачи спецодежды, спецоснастки и МОС**.
7. Выберите **Закрыть** на странице **Журнал выдачи спецодежды, спецоснастки и МОС**. Система устанавливает флажок **Разнесено** и создает журналы ОС для ввода в эксплуатацию и амортизации МОС. 
8. Выберите **Журналы ОС** \> **Журнал ОС (ввод в эксплуатацию)** и **Журналы ОС** \> **Журнал ОС (амортизация)** для проверки и разноски журналов основных средств.

> [!NOTE]
> Система создает записи на странице **Малоценные ОС** после разноски журнала основных средств для ввода МОС в эксплуатацию.

## <a name="register-an-nvfa-by-using-the-not-valuable-fas-page"></a>Регистрация МОС используя страницу "Малоценные ОС" 

1. Откройте **Основные средства (Россия)** \> **Общее** \> **Малоценные ОС**.
2. Выберите **Основное средство**, чтобы создать МОС.
3. В поле **Группа ОС** выберите группу ОС для МОС.
4. В поле **Стоимость приобретения** введите сумму приобретения основного средства.
5. В поле **Примечание** введите любые дополнительные сведения для актива.
6. В поле **Ресурс** выберите ресурс или группу ресурсов, назначенных активу.
7. Выберите **Основное средство** \> **Комплектующие** \> **Комплектующие**.
8. Выберите **Добавить**, чтобы создать строку. В поле **Код номенклатуры** выберите номенклатуру.
9. Выберите **Склад** \> **Отображение аналитик** для включения аналитик **Номер партии** и **Серийный номер** для номенклатуры.
10. В поле **Номер партии** выберите номер партии для номенклатуры.
11. В поле **Серийный номер** выберите серийный номер для номенклатуры.
12. В поле **Первоначальное количество** введите исходное количество используемой номенклатуры.

## <a name="generate-a-disposal-transaction-and-print-the-nvfa-statement-of-disposal-no-mb-4-from-the-fixed-asset-journal"></a>Создание проводки списания и печать акта о выбытии МОС (№ МБ-4) из журнала основных средств 

Эта процедура используется для создания и печати отчета **Акт выбытия МОС (МБ-4)** из журнала ОС после ввода проводок выбытия или списания. Для проводок со статусом **Списано** или **Списано (продажа)** отчет можно также распечатать со страницы **Спецодежда**, **Спецоснастка** или **Малоценные ОС**. Этот отчет формируется каждым из подразделений, которое использует спецодежды, спецоснастку и МОС.

> [!NOTE]
> Можно создать отчет со страниц **Спецодежда**, **Спецоснастка** и **Малоценные ОС** только после создания отчета из журнала основных средств.

1. Откройте **Основные средства (Россия)** \> **Журналы** \> **Журнал ОС**.
2. Создайте журнал.
3. В поле **Имя** выберите имя журнала.
4. Выберите **Строки**, чтобы открыть страницу **Ваучер журнала**.
5. Выберите **Создать**, чтобы открыть диалоговое окно **Добавление в журнал**.
6. Выберите дату проводки в поле **Дата проводки**.
7. В поле **Тип проводки** выберите **Выбытие (продажа)**.
8. В поле **Инвентарный номер ОС** выберите инвентарный номер для основного средства.
9. В поле **Модель стоимости** выберите модель стоимости для основного средства.
10. В поле **Код причины** выберите код причины для проводки.
11. В поле **Комментарий к причине** введите комментарий или описание проводки.
12. Нажмите **ОК**. Строки списания всех моделей стоимости, зарегистрированных на счету ОС, создаются в журнале.
    –или–

Выберите **Групповые операции** \> **Выбытие (продажа)** после шага 4, чтобы создать проводки выбытия.

5. В поле **Дата выбытия** выберите дату выбытия для транзакции.
6. На вкладке **Записи для добавления** выберите **Фильтр**. Затем укажите критерии выбора, которые используются для создания проводок.
7. Выберите **ОК**, затем снова выберите **ОК**, чтобы создать проводки выбытия или списания.
8. Выберите пункт меню **Проверка** \> **Проверка**, чтобы проверить проводки списания.
9. Выберите **Разнести** \> **Разнести**, чтобы разнести транзакцию.
10. Закройте страницу.
11. На странице **Журнал ОС** выберите **печать** \> **Акт о выбытии МОС (№ МБ-4)**, чтобы открыть страницу **Акт выбытия МОС (МБ-4)**.

    > [!NOTE]
    > Кнопка **Печать** доступна только для журналов, которые были разнесены. Проводка списания, созданная на странице **Ваучер журнала**, появляется на вкладке **Обзор**. Эта проводка имеет номер документа. Номер документа создается на основе номерной серии, настроенной на странице **Параметры основных средств**.

22. В поле **Комментарий** введите причину для выбытия ОС.
23. На вкладке **Строки** можно просмотреть номер документа, номер основных средств и код модели стоимости.
24. Выберите **Печать**, чтобы создать отчет **Акт о выбытии МОС (номер МБ-4)**.

## <a name="generate-a-writing-off-transaction-and-print-the-nvfa-statement-of-writing-off-no-mb-8-from-the-fixed-asset-journal"></a>Создание проводки списания и печать акта о списании МОС (№ МБ-8) из журнала основных средств

Эта процедура используется для создания и печати отчета **Акт на списание МОС (МБ-8)** из журнала ОС после ввода проводок списания или выбытия. Для проводок со статусом **Списано** или **Списано (разборка)** отчет можно также распечатать со страницы **Спецодежда**, **Спецоснастка** или **Малоценные ОС**.

1. Откройте **Основные средства (Россия)** \> **Журналы** \> **Журнал ОС**.
2. Создайте журнал.
3. В поле **Имя** выберите имя журнала.
4. Выберите **Строки**.
5. Выберите **Создать**, чтобы открыть диалоговое окно **Добавление в журнал**.
6. Выберите дату проводки в поле **Дата проводки**.
7. В поле **Тип транзакции** выберите **Списание**.
8. В поле **Инвентарный номер ОС** выберите инвентарный номер основного средства.
9. Нажмите **ОК**. Строки модели стоимости для записи средства создаются в журнале.
    –или– Выберите **Групповые операции** \> **Списание** после выполнения шага 4, чтобы создать проводки списания.

5. В поле **Дата выбытия** введите дату транзакции списания.
6. На вкладке **Записи для добавления** выберите **Фильтр**. Затем укажите критерии выбора, которые используются для создания проводок.
7. Выберите **ОК**, затем снова выберите **ОК**, чтобы создать проводки выбытия или списания.
8. На странице **Ваучер журнала** выберите **Проверить** \> **Проверить**, чтобы проверить журнал.
9. Выберите **Разнести** \> **Разнести**, чтобы разнести журнал.
9. Выберите **Печать** \> **Акт на списание МОС (МБ-8)**.
10. На вкладке **Обзор** в поле **Номер заказа** введите номер заказа, для которого необходимо разнести проводку списания. В поле **Дата приказа** выберите дату заказа.
11. На вкладке **Строки** можно проверить номер документа, номер основных средств и код модели стоимости.
12. На вкладке **Должностные лица** можно просмотреть сведения о должностных лицах, назначенных на странице **Должностные лица**. Можно также изменить сведения о сотруднике, если требуется.
13. Выберите **Печать**, чтобы создать отчет **Акт на списание МОС (номер МБ-8)**.
