# <a name="vat-declaration-russia"></a>Декларация НДС (Россия)

[!include [banner](../includes/banner.md)]

Чтобы начать работать с декларацией по налогу на добавленную стоимость (НДС), выполните следующие действия.

1. Из [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com/V2) в общей библиотеке активов загрузите последние версии конфигураций электронной отчетности (ER) для формата декларации НДС.

    Например, для создания декларации по НДС за отчетный период 2018 года загрузите последние версии следующих конфигураций:

    - Модель декларации по НДС (RU)
    - Сопоставление модели декларации по НДС (RU)
    - Формат декларации по НДС 5.05

    Дополнительные сведения см. в разделе [Загрузка конфигураций электронной отчетности из Lifecycle Services](../../dev-itpro/analytics/download-electronic-reporting-configuration-lcs.md).

2. Необязательно: настройте финансовый отчет для декларации по НДС. Настройте правила расчета для ячеек финансового отчета.

    При использовании финансовых отчетов для создания данных для некоторых ячеек в разделе 3 декларации по НДС на основе настройки ячеек финансового отчета следует выполнить этот необязательный шаг.

    Дополнительные сведения о настройке финансовых отчетов для России см. в разделе [Финансовая отчетность (Россия)](rus-financial-reports.md).

3. Необязательно: для первого использования можно отправить параметры пакета управления данными для работы с декларацией по НДС в России. Выполните следующие действия.

    1. В общей библиотеке активов LCS выберите **Пакет данных** как тип актива. Загрузите пакет с именем **Настройка демонстрации декларации по НДС RU**. Загруженный файл называется "RU VAT Declaration.zip".
    2. В Microsoft Dynamics 365 for Finance and Operations в рабочей области **Управление данными** выберите **Импорт**.
    3. В разделе **Сведения о задании** установите следующие значения:

        - Введите имя должности.
        - В поле **Формат исходных данных** выберите **Пакет**.

    4. В поле **Отправить файл данных** выберите **Отправить**, затем выберите файл **RU VAT Declaration.zip**, загруженный ранее.
    5. После отправки информационных объектов выберите **Импорт**.
    6. Выберите пункты **Главная книга \> Настройка финансовых отчетов \> Финансовые отчеты** и проверьте финансовый отчет, который вы импортировали.

## <a name="russian-vat-declaration-version-505-in-xml-format"></a>Декларации по НДС для России версии 5.05 в формате XML

Декларация по НДС для России версии 5.05 в формате XML настраивается в модуле **Электронная отчетность**. В нем содержится следующая информация:

- **Раздел 1** — общая сумма НДС, которая должна быть выплачена в бюджет или возмещена из бюджета. В этом разделе содержатся общие суммы из разделов с 3 по 7.
- **Раздел 2** — сумма НДС, которая должна быть выплачена в бюджет налоговым агентом.
- **Раздел 3** — расчет сумм НДС, которые должны быть выплачены в бюджет или возмещены из бюджета.

    - **Приложение 1 к разделу 3** — суммы НДС, которые были ранее удержаны, а теперь должны быть восстановлены и уплачены в бюджет для основных средств, которые использовались в операциях, не облагаемых налогом.

- **Раздел 4** — расчет сумм НДС для экспортных продаж, для которых была подтверждена налоговая ставка 0 процентов за период.
- **Разделе 5** — расчет подлежащих вычету сумм НДС за отчетный период для экспортных продаж, в которых налоговая ставка 0 процентов была подтверждена либо не была подтверждена в предыдущих периодах.
- **Раздел 6** — расчет сумм НДС для экспортных продаж, для которых налоговая ставка 0 процентов должны была быть подтверждена за период, но не была подтверждена.
- **Раздел 8** — сведения из книги покупок.

    - **Раздел 8.1** — приложение к разделу 8. Сведения из дополнительных листов для книги покупок.

- **Раздел 9** — сведения из книги продаж.

    - **Раздел 9.1** — приложение к разделу 9. Сведения из дополнительных листов для книги продаж.

- **Раздел 10** — сведения о промежуточных операциях из журнала выписанных счетов-фактур.
- **Раздел 11** — сведения о промежуточных операциях из журнала полученных счетов-фактур.

Дополнительные сведения см. в разделе [Книги продаж, книги покупок и журналы счетов-фактур](rus-sales-books-purchase-books.md).

## <a name="section-2--the-vat-amount-that-must-be-paid-to-budget-by-the-tax-agent"></a>Раздел 2 — сумма НДС, которая должна быть выплачена в бюджет налоговым агентом

Раздел 2 декларации по НДС содержит суммы НДС, рассчитанные на основе проводок налогового агента.

Дополнительные сведения см. в разделе [Налог на добавленную стоимость (НДС) для налоговых агентов (Россия)](rus-tax-agent.md).

Для каждого налогового агента создается отдельный раздел. В каждом разделе содержится следующая информация:

- Название организации, для которой юридическое лицо выполняет функции налогового агента (строка 020)
- Код налоговой регистрации типа организации, для которой юридическое лицо выполняет функции налогового агента (строка 030)
- Код бюджетной классификации (строка 040)
- Код территории (строка 050)
- Сумма НДС, рассчитанная для проводок налогового агента (строка 060)
- Код операции НДС (строка 070)

## <a name="section-3--calculation-of-vat-amounts-that-must-be-paid-to-budget-or-reclaimed"></a>Раздел 3 — расчет сумм НДС, которые должны быть выплачены в бюджет или возмещены из бюджета

### <a name="customize-section-3-of-the-vat-declaration"></a>Настройка раздела 3 декларации по НДС

В конфигурации электронной отчетности **Модель декларации по НДС (RU)** раздел 3 представлен элементом **Элементы декларации**. В конфигурации электронной отчетности **Сопоставления модели декларации по НДС (RU)** этот элемент привязан к источнику данных **$RRG.$Section3.$data**. Этот источник данных настроен как **$RRG.$Section3.$data = LISTJOIN(@.'$dataStd', @.'$dataCustom')**. Он ссылается на два источника данных:

- **$RRG.$Section3.$data.$dataStd** — этот источник данных настроен для доступа к классу приложений **Вспомогательное приложение декларации по НДС (RU)** путем вызова метода **getSection3data**. Этот метод создает список, содержащий суммы нескольких полей в разделе 3.
- **$RRG.$Section3.$data.$dataCustom** — этот источник данных настроен для доступа к классу приложений **LedgerRRGCustomReportHelper\_RU** путем вызова метода **getCustomReportData**. Этот метод создает список записей, содержащих вычисленные суммы для некоторых ячеек финансовых отчетов, которые были настроены пользователем.

Чтобы настроить декларацию, выполните одно из следующих действий.

- Настройте финансовый отчет для вычисления ячеек, которые не рассчитывается автоматически с помощью класса **Вспомогательное приложение декларации по НДС (RU)**.

    В этом случае следует настроить правила расчета финансовых отчетов для ячеек отчета, которые не помечены звездочкой (\*) в таблицах в предыдущем разделе.

- Настройте финансовый отчет для полного вычисления ячеек на основе настроек пользователя вместо автоматического расчета.

    В этом случае выполните следующие действия.

    1. Настройте правила расчета финансовых отчетов для всех требуемых ячеек, включая ячейки, которые помечены звездочкой (\*) в таблицах в предыдущем разделе.
    2. Создайте настроенную конфигурацию электронной отчетности **Сопоставлений модели**. Создайте ее на основе конфигурации, предоставляемый поставщиком конфигураций Microsoft.
    3. В настроенной конфигурации электронной отчетности **Сопоставление модели**, созданной на предыдущем шаге, заново определите источник данных сопоставления модели таким образом, чтобы он был настроен как **$RRG.$Section3.$data = '$dataCustom'**, а не как **$RRG.$Section3.$data = LISTJOIN(@.'$dataStd', @.'$dataCustom')**.

- Расширьте класс **Вспомогательное приложение декларации по НДС (RU)** (используйте методы, которые начинаются с **getSection3data**, такие как **getSection3dataDomesticVAT**). Или, аналогичным образом, создайте альтернативные вспомогательные методы, которые рассчитывают значения требуемых ячеек.

    В этом случае выполните следующие действия.

    1. Создайте настроенную конфигурацию электронной отчетности **Сопоставлений модели**. Создайте ее на основе конфигурации, предоставляемый поставщиком конфигураций Microsoft.
    2. Переопределите элемент источника данных **$RRG.$Section3.$data.$dataStd** сопоставления модели.

Дополнительные сведения о порядке создания производной версии конфигураций электронной отчетности для настройки см. в разделе [Электронная отчетность — обновление формата путем принятия новой базовой версии этого формата](../../dev-itpro/analytics/tasks/er-upgrade-format.md).

Дополнительные сведения об определении сопоставлений моделей см. в разделе [Определение сопоставлений моделей электронной отчетности и выбор источников данных для них](../../dev-itpro/analytics/tasks/er-define-model-mapping-select-data-sources-2016-11.md).

## <a name="application-1-to-section-3"></a>Приложение 1 к разделу 3

Приложение 1 содержит сведения об основных средствах, которые используются в операциях, не облагаемых налогом. Оно создается в последнем отчетном периоде календарного года и содержит восстановленные суммы НДС для основного средства, рассчитанные за весь календарный год. Данные для приложения 1 создаются на основе записей журнала восстановления НДС.

Приложение 1 содержит следующую информацию:

- Название основного средства (строка 010).
- Дата, когда основное средство было введено в эксплуатацию (строка 030)
- Дата начала амортизации (строка 040)
- Исходная стоимость основного средства (строка 050)
- Входящая сумма НДС, которая связана с этим основным средством и которая была вычтена (строка 060)
- Год, когда начинается восстановление НДС, до десятого года амортизации для основного средства (строка 070, столбец 1)
- Дата, когда основное средство было впервые использовано в операциях, не облагаемых налогом (строка 070, столбец 2)
- Процент от выручки для не облагаемых налогом товаров и экспортных операций (строка 070, столбец 3)
- Сумма НДС, которая была восстановлена для этого основного средства в году (строка 070, столбец 4)

## <a name="sections-for-exports-that-are-taxed-at-a-rate-of-0-percent"></a>Разделы для экспорта, которые облагаются налогом по ставке 0 процентов

Прежде чем создать разделы декларации по НДС для экспорта, которые облагаются налогом по ставке 0 процентов, выполните следующие действия.

1. Выберите пункты **Налог \> Настройка \> Налог \> Коды операций по НДС** и создайте коды операций по НДС для экспортной торговли.
2. Выберите **Налог \> Косвенные налоги \> Налог \> Налоговые коды** и назначьте коды операций по НДС налоговым кодам.

### <a name="overview-and-example"></a>Общие сведения и пример

Разделы для экспорта создаются на основе регистрации счетов-фактур экспорта и записей журнала восстановления НДС. Дополнительные сведения о порядке обработки проводок экспорта см. в разделе "Обработка счетов-фактур для экспортной торговли" (в настоящее время недоступно).

**Пример**

- **Январь 2018:** приобретение продуктов и получение документов Накладная V1 и Счет-фактура1.

    - Накладная V1/Счет-фактура1: 10 штук на общую сумму 118 000 российских рублей (руб.), которая включает 18% НДС (18 000 руб.)

- **Январь 2018:** удержание входящего НДС для счета-фактуры1.

    - 82 600 руб., которые включают НДС 18 процентов (12 600 руб.)

    Обратите внимание, что входящая фактура включает в себя только некоторые из товаров, которые были приобретены с использованием накладной V1.

- **Апрель 2018:** продукты продаются для экспорта по валютному курсу, равному 45 руб. за 1 доллар США (USD).

    - Накладная C1: 2 шт. по 600 долларов США = 27 000 руб. (соответствующая входящая сумма НДС — 3 600 руб.).
    - Накладная C2: 8 шт. по 2 400 долларов США = 108 000 руб. (соответствующая входящая сумма НДС — 14 400 руб.).

    В этом примере сентябрь 2018 г. является крайним сроком для сбора всех документов для подтверждения экспорта по ставке 0 процентов.

- **Август 2018:** экспорт для накладной C1 подтвержден до наступления крайнего срока.
- **Сентябрь 2018:** экспорт для накладной C2 не подтвержден до наступления крайнего срока. Таким образом, НДС для данной накладной рассчитывается по ставке НДС внутри страны, равной 18 процентам (108 000 × 18 процентов = 19 440 руб.).

    Сумма соответствующего вычета входящего НДС для неподтвержденного экспорта составляет 9 000 руб.

- **Ноябрь 2018:** счет-фактура2 получена для остальных товаров, которые были приобретены с использованием накладной V1. Таким образом, вычет входящего НДС для остальной части накладной V1 разносится.

    - 35 400 руб., которые включают НДС 18 процентов (5 400 руб.) (это сумма для товаров, которые были приобретены с использованием накладной C2).

Декларация по НДС за третий квартал 2018 г. будет содержать раздел 4 и раздел 6.

Декларация по НДС за четвертый квартал 2018 г. будет содержать раздел 5.

Следующие разделы содержат дополнительные сведения для данного примера.

### <a name="section-4--calculation-of-vat-amounts-for-export-sales-where-a-tax-rate-of-0-percent-was-confirmed-during-the-period"></a>Раздел 4 — расчет сумм НДС для экспортных продаж, для которых была подтверждена налоговая ставка 0 процентов за период

Раздел 4 декларации по НДС представляет суммы для счетов-фактур экспорта, если ставка 0 процентов была подтверждена в течение отчетного периода. Данные группируются по коду операции НДС.

Экспортируются следующие данные:

- Код операции НДС (строка 010)
- Налоговая база (строка 020)
- Подлежащая вычету сумма НДС с покупки номенклатур, которые используются в экспорте, которые были подтверждены со ставкой 0 процентов (строка 030)
- Корректировка: подлежащая вычету сумма НДС, уплаченная в предыдущих периодах, когда ставка 0 процентов еще не подтверждена (строка 040)
- Корректировка: сумма восстановленного НДС, которая была удержана за предыдущие периоды для неподтвержденного экспорта (строка 050)

Если товары были возвращены, следующие данные экспортируются для исправления сумм:

- Код операции НДС (строка 060)
- Налоговая база (строка 070)

**Пример**

В предыдущем примере в разделе 4 декларации по НДС для третьего квартала 2018 г. будет присутствовать следующие данные.

| Реквизит | Линейная | Стоимость |
|---|---|---|
| Код операции по НДС | 010 | 1011410 (в соответствии с настройкой пользователя для соответствующего налогового кода) |
| Налоговая база | 020| 27 000 |
| Подлежащая вычету сумма НДС с покупки номенклатур, которые используются в экспорте, которые были подтверждены со ставкой 0 процентов | 030 | 3600 |

### <a name="section-5--calculation-of-deductible-vat-amounts-for-the-reporting-period-for-export-sales-where-a-tax-rate-of-0-percent-was-either-confirmed-or-unconfirmed-in-previous-periods"></a>Разделе 5 — расчет подлежащих вычету сумм НДС за отчетный период для экспортных продаж, в которых налоговая ставка 0 процентов была подтверждена либо не была подтверждена в предыдущих периодах

Раздел 5 декларации по НДС представляет суммы НДС, подлежащие вычету в отчетном периоде для продаж на экспорт, для которых ставка 0 процентов было подтверждена либо не подтверждена в предыдущих периодах (то есть, "последующие" вычеты сумм НДС). Данные группируются по периоду, когда соответствующий экспорт был объявлен, а также по коду операции НДС.

Экспортируются следующие данные:

- Год, когда соответствующий экспорта был объявлен (строка 010)
- Код для отчетного периода, когда соответствующей экспорт был объявлен (строка 020)
- Код операции НДС (строка 030)
- Налоговая база по покупке номенклатур, которые используются в подтвержденном экспорте (строка 040)
- Подлежащая вычету сумма НДС по покупке номенклатур, которые используются в подтвержденном экспорте (строка 050)
- Налоговая база по покупке номенклатур, которые используются в неподтвержденном экспорте (строка 060)
- Подлежащая вычету сумма НДС по покупке номенклатур, которые были использованы в неподтвержденном экспорте (строка 070)

**Пример**

В предыдущем примере в разделе 5 декларации по НДС для четвертого квартала 2018 г. будет присутствовать следующие данные.

| Реквизит | Линейная | Стоимость |
|---|---|---|
| Год, когда соответствующий экспорта был объявлен | 010 | 2018 |
| Код для отчетного периода, когда соответствующей экспорт был объявлен | 020 | 23 (код периода для третьего квартала) |
| Код операции по НДС | 030 | 1011410 |
| Налоговая база по покупке номенклатур, которые используются в неподтвержденном экспорте | 060 | 108 000 |
| Подлежащая вычету сумма НДС по покупке номенклатур, которые используются в неподтвержденном экспорте | 070 | 5 400 |

### <a name="section-6--calculation-of-vat-amounts-for-export-sales-where-a-tax-rate-of-0-percent-should-have-been-confirmed-during-the-period-but-wasnt-confirmed"></a>Раздел 6 — расчет сумм НДС для экспортных продаж, для которых налоговая ставка 0 процентов должны была быть подтверждена за период, но не была подтверждена

Раздел 6 декларации по НДС представляет суммы для счетов-фактур экспорта, если ставка 0 процентов не была подтверждена и если крайний срок истек в течение отчетного периода. Данные группируются по коду операции НДС.

Экспортируются следующие данные:

- Код операции НДС (строка 010)
- Налоговая база (строка 020)
- Сумма НДС к оплате, которая вычисляется по налоговой базе для неподтвержденного экспорта (строка 030)
- Подлежащая вычету сумма НДС по покупке номенклатур, которые используются в неподтвержденном экспорте (строка 040)

Если товары были возвращены, следующие данные экспортируются для исправления сумм:

- Код операции НДС (строка 070)
- Налоговая база возврата товаров (строка 080)
- Сумма НДС возврата товаров, на которую уменьшается подлежащая оплате сумма НДС (строка 090)
- Корректировка: сумма НДС по покупке номенклатур, которые используются в экспорте, которая была ранее удержана и должна быть восстановлена для возврата товаров (строка 100)

В предыдущем примере в разделе 6 декларации по НДС для третьего квартала 2018 г. будет присутствовать следующие данные.

| Реквизит | Линейная | Стоимость |
|---|---|---|
| Код операции по НДС | 010 | 1011410 (в соответствии с настройкой пользователя для соответствующего налогового кода) |
| Налоговая база | 020 | 108 000 |
| Сумма НДС к оплате, которая вычисляется по налоговой базе для неподтвержденного экспорта | 030 | 19 440 |
| Подлежащая вычету сумма НДС по покупке номенклатур, которые используются в неподтвержденном экспорте | 040 | 9000 |
