# <a name="fixed-assets-russia"></a>Основные средства (Россия)

[!include [banner](../includes/banner.md)]


Для управления основными средствами компании можно использовать страницу **Основные средства** для сохранения всей информации (как финансовой, так и не финансовой) об активах. В следующих примерах показаны типы финансовой информации, которая относится к основным средствам:

-   Стоимость приобретения основного средства
-   Корректировка стоимости
-   Сумма амортизации основного средства
-   Бюджет основного средства
-   Информация по налогам

В следующих примерах показаны типы нефинансовой информации, которая относится к основным средствам:

-   Техн. информация
-   Местоположение
-   Штриховые коды и инвентарные номера
-   Страхование
-   Связи между различными активами

## <a name="register-fixed-assets"></a>Регистрация основных средств

Перед созданием проводок для любого основного средства или нематериального актива необходимо сначала зарегистрировать запись актива и предоставить основную информацию о нем.

1. Для регистрации основного средства выберите **Основные средства (Россия)** \> **Основные** \> **Основные средства**. Выберите **Создать**, чтобы создать новое основное средство.

    > [!NOTE]
    > Можно создать новое основное средство, используя функцию копирования (кнопка **ОСНОВНОЕ СРЕДСТВО** \> **Копировать ОС**). Система копирует выбранные основные средства со всеми его параметрами, но с другим инвентарным номером.

![Вкладка "Общие" страницы ОС](media/RUS_FA_1%20General%20tab.JPG)

2. Заполните следующие поля на экспресс-вкладке **Общее**:

- **Группа ОС** — выберите группу основных средств, с которой должно быть связано основное средство.

- **Инвентарный номер** — если автоматическая нумерация настроена в **Параметры основных средств** или для группы основных средств, сведения заполняются автоматически.

- **Имя** — введите название основного средства.

- **Дата приобретения** — по умолчанию это поле заполняется автоматически текущей датой системы, но может быть изменено.

- **Стоимость приобретения** — введите стоимость приобретения основных средств в основной валюте компании. Если основные средства получены из модели РБП, отображается сумма накладной без налогов в основной валюте компании.

- **Примечания** — введите любые дополнительные сведения об этом основном средстве.

- **Тип** — выбор типа основного средства. В зависимости от значения в этом поле другие поля активируются на экспресс-вкладках **Технические сведения** и **Налоговая отчетность**. Ниже приведены примеры типов основного средства. 

    - **Транспортное средство** — этот флажок устанавливается для отображения группы полей **Государственная регистрация** на экспресс-вкладке **Техническая информация**.

    - **Недвижимость** — этот флажок устанавливается, чтобы поле **Налоговая база** отображалось на экспресс-вкладке **Налоговая отчетность**.

    - **Земельный участок** — этот флажок устанавливается для отображения поля **Дата начала строительства** на экспресс-вкладке **Техническая информация**.

    > [!NOTE]
    > Следующие типы не могут быть выбраны на странице **Основные средства**: МОС, Спецодежда, Спецоснастка. Для этих типов имеются специальные страницы. Дополнительные сведения см. в разделах [Учет малоценных основных средств (МОС) для России](https://docs.microsoft.com/en-us/dynamics365/unified-operations/financials/localizations/rus-not-valuable-assets) и [Учет спецодежды и спецоснастки для России](https://docs.microsoft.com/dynamics365/unified-operations/financials/localizations/rus-working-clothes-instruments-accounting).

- Группа полей **Страхование** — при необходимости заполните поля для страховки.

- **Флаг владения** — выберите соответствующее значение, по умолчанию это значение равно **Собственность**.

- **Штрих-код** — можно ввести штрих-код вручную или штрих-код может быть создан автоматически (то же, что и инвентарный номер актива). Для автоматического создания штрих-кода выберите **Штрих-код совпадает с инвентарным номером** в параметрах **Основные средства** (**Основные средства \> Основные средства** \> **Параметры**).

    > [!NOTE]
    > Поля **Местоположение** и **Код работника** заполняются автоматически при создании записи в списке страницы **Передача ОС**. (**Основные средства (Россия)** \> **Общее** \> **Основные средства**, нажмите **ОСНОВНОЕ СРЕДСТВО** \> **ИСТОРИЯ** \> **Перемещение**).

- **Выпуск/пробег** — введите размер предполагаемого производства или пробега основного средства, если профиль амортизации **Выпуск продукции/пробег** выбран для основного средства в поле **Модель учета**. В этом случае необходимо ввести записи в список страницы **Выпуск продукции/пробег**.

![Страница выпуска или пробега](media/RUS_FA_2%20Output_Mileage%20page.JPG)

3. Если основное средство включено в состав другого основного средства, выберите экспресс-вкладку **Структура**. Выберите код основного средства в поле **Главное ОС**.

4. В поле **Структура ОС** все счета ОС, имеющие ссылку на этот актив, отображаются в главном ОС.

5. Выберите экспресс-вкладку **Покупка/продажа**. Отображаются сведения о покупке и продаже основного средства.

## <a name="fixed-asset-value-models"></a>Модели стоимости ОС

Выполните следующие действия, чтобы обновить или создать модели стоимости для основного средства.

1. Выберите кнопку **Модели стоимости**.

    > [!NOTE]
    > Если модели стоимости настроены для группы основных средств, которой назначено основное средство (**Основные средства (Россия) \> Настройка \> Группы ОС**), то система автоматически создает модели стоимости, и можно обновлять значения полей.

2. Необходимо создать модель стоимости для каждого ОС, и настройки используются для регистрации проводки. Выберите **Создать** для создания новой строки. Заполните следующие поля на экспресс-вкладке **Общее**.

![Модель учета ОС](media/RUS_FA_3%20value%20models.JPG)

- **Модель стоимости** — выберите корд модели для основного средства.

- **Амортизационные группы** — выберите группу амортизации в модели стоимости, к которой относится основное средство.

- **Методы амортизации** — выберите профиль амортизации. По умолчанию отображается значение, заданное для группы амортизации, но его можно изменить.

- **Профиль разноски** — выберите профиль разноски, который будет использоваться в модели стоимости. По умолчанию значение основано на параметрах модели стоимости.

- **Стоимость приобретения** — введите сумму приобретения основного средства. Если проводка покупки актива была создана, отображается сумма проводки без налогов. Если валюта приобретения ОС отлична от валюты, настроенной для этой модели стоимости, валюты преобразуются на дату приобретения.

- **Дата начала амортизации** — выберите дату начала начисления амортизации. По умолчанию это поле заполняется в соответствии с настройкой, выбранной в поле **Группа амортизации**. Если параметр **Дата начала амортизации** настроен как **С начала следующего месяца**, то это поле заполняется периодом, который следует за периодом приобретения.

- **Валюта** — выберите валюту для модели стоимости для актива. Отображается валюта по умолчанию из параметров модели стоимости.

- **Остаточная стоимость после списания** — введите остаток суммы ОС после списания. Если значение заполнено, то остаточная стоимость, используемая в расчете амортизации, меньше значения в этом поле. После создания проводки списания сумма, равная стоимости остатков после списания, может быть разнесена на склад или на счет главной книги.

- **Не блокировать** — установите для этого параметра значение **Да**, чтобы создать проводки амортизации для актива. По умолчанию этот параметр имеет значение **Да**.

- **Дата последней амортизации, Дата выбытия** и **Стоимость выбытия** — заполняется автоматически при создании проводок амортизации и списания.

3. Выберите экспресс-вкладку **Аналитика**. Выберите коды финансовой аналитики по умолчанию, которые будут регистрировать проводки для актива.

4. Выберите экспресс-вкладку **Аренда** и заполните поле **Профиль разноски**, выберите профиль разноски, который используется для сдачи основных средств в аренду. Если для основного средства следует создать проводку аренды, установите для параметра **Арендованное** значение **Да**.

5. Выберите **Аналитика арендованного ОС**. Выберите коды аналитик, которые должны использоваться при учете для проводок с данным основным средством, когда оно сдается в аренду.

6. Выберите **Счета амортизации**. На этой странице можно настроить сумму разноски амортизации ОС для счетов ГК в необходимых пропорциях по мере необходимости. Это может потребоваться, например, чтобы вычислить амортизацию для здания, используемого для различных целей.

## <a name="inquiries-on-the-fixed-asset-page"></a>Запросы на странице основных средств

### <a name="view-asset-transactions"></a>Просмотр проводок активов

Выполните следующие действия, чтобы просмотреть проводки с основными средствами.

1. Выберите **Основные средства (Россия) \> Основные средства \> Модели стоимости \> Проводки**.

2. Все проводки с основными средствами, выполненные для модели стоимости, перечисляются на странице **Проводки ОС**, независимо от того, в каком модуле они были разнесены:

- Модуль **Основные средства (Россия)**.
- Модуль **Главная книга**.
- Модуль **Расчеты с поставщиками**.
- Модуль **Расчеты с клиентами**.

3.  На вкладках **Обзор** и **Общие** отображаются подробные сведения о проводках с основными средствами.

### <a name="view-the-balance-of-an-asset"></a>Просмотр баланса основного средства

Стоимость актива на конкретную дату отражает результаты всех проводок для актива. Выполните следующие действия, чтобы просмотреть сальдо актива.

1. Выберите **Основные средства (Россия)\> Общее \> Основные средства \> Модели стоимости**.

2. Выберите модель учета, для которой необходимо просмотреть баланс.

3. Выберите кнопку **Сальдо**.

4. В поле **Дата проводки** ведите дату для баланса. По умолчанию используется текущая дата.

5. Суммы на странице основных средств **Баланс по ОС** отображаются в валюте модели учета основных средств. Если валюта модели учета отличается от валюты компании по умолчанию, валюта по умолчанию также отображается.

![Сальдо модели стоимости ОС](media/RUS_FA_4%20model%20balance.JPG)
