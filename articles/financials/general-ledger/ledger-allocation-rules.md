# <a name="ledger-allocation-rules"></a><span data-ttu-id="1fad8-104">Правила распределения ГК</span><span class="sxs-lookup"><span data-stu-id="1fad8-104">Ledger allocation rules</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="1fad8-105">В этой статье представлена информация о правилах распределения ГК.</span><span class="sxs-lookup"><span data-stu-id="1fad8-105">This article provides information about ledger allocation rules.</span></span> <span data-ttu-id="1fad8-106">Она содержит описание различных компонентов этих правил распределения и методов распределения, которые можно использовать для них.</span><span class="sxs-lookup"><span data-stu-id="1fad8-106">It describes the various components of these allocation rules and the allocation methods that can be used for them.</span></span>

<span data-ttu-id="1fad8-107">Правила распределения книги учета используются для автоматического расчета и создания журналов распределения и записей счетов для распределения сальдо ГК или фиксированных сумм.</span><span class="sxs-lookup"><span data-stu-id="1fad8-107">Ledger allocation rules are used to automatically calculate and generate allocation journals and account entries for the allocation of ledger balances or fixed amounts.</span></span> <span data-ttu-id="1fad8-108">Методы распределения могут быть переменными или фиксированными.</span><span class="sxs-lookup"><span data-stu-id="1fad8-108">Allocation methods can be variable or fixed.</span></span> <span data-ttu-id="1fad8-109">Следующие методы распределения можно использовать для правил распределения книги учета:</span><span class="sxs-lookup"><span data-stu-id="1fad8-109">The following allocation methods can be used for ledger allocation rules:</span></span>

-   <span data-ttu-id="1fad8-110">**Базовый** — этот переменный метод используется, когда распределение зависит от фактического сальдо ГК на основе критериев фильтра.</span><span class="sxs-lookup"><span data-stu-id="1fad8-110">**Basis** – This variable method is used when the allocation depends on the actual ledger balance, based on filter criteria.</span></span> <span data-ttu-id="1fad8-111">Например, можно распределить расходы на рекламу на основе объема продаж каждого подразделения пропорционально общему объему продаж по всем подразделениям.</span><span class="sxs-lookup"><span data-stu-id="1fad8-111">For example, advertising expenses can be allocated based on each department's sales in proportion to the total departmental sales.</span></span>
-   <span data-ttu-id="1fad8-112">**Фиксированный процент** и **Фиксированный вес** — для этих методов процент распределения или вес определяются непосредственно для правила.</span><span class="sxs-lookup"><span data-stu-id="1fad8-112">**Fixed percentage** and **Fixed weight** – For these methods, the allocation percentage or weight is defined directly for the rule.</span></span> <span data-ttu-id="1fad8-113">Например, расходы на рекламу можно распределить так, что отдел A получает 70 процентов расходов на рекламу, а отдел B получает 30 процентов.</span><span class="sxs-lookup"><span data-stu-id="1fad8-113">For example, advertising expenses can be allocated so that Department A receives 70 percent of the advertising expense and Department B receives 30 percent.</span></span>
-   <span data-ttu-id="1fad8-114">**Равно** — этот метод распределяет количество поровну для каждого определенного назначения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-114">**Equally** – This method distributes the amount equally to each defined destination.</span></span> <span data-ttu-id="1fad8-115">Например, если назначения определены для отдела A и отдела B, то расходы на рекламу можно распределить так, что отделы A и B получат по 50 процентов расходов на рекламу.</span><span class="sxs-lookup"><span data-stu-id="1fad8-115">For example, if destinations are defined for Department A and Department B, advertising expenses can be allocated so that both Department A and Department B receive 50 percent of the advertising expense.</span></span>

<span data-ttu-id="1fad8-116">Если для правила распределения используется метод распределения "Базовый", необходимо также создать отдельные базовые правила распределения ГК.</span><span class="sxs-lookup"><span data-stu-id="1fad8-116">If Basis is used as the allocation method for an allocation rule, you must also define separate ledger allocation basis rules.</span></span> <span data-ttu-id="1fad8-117">Процесс "Обработать запрос на распределение" позволяет пользователям обрабатывать правило распределения книги учета и предварительно просматривать получающиеся в результате записи журнала распределения, прежде чем разнести или удалить рассчитанные распределения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-117">The "Process allocation request" process lets users process the ledger allocation rule and preview the resulting allocation journal entries before they either post or delete the calculated allocations.</span></span>

## <a name="components-of-ledger-allocation-rules"></a><span data-ttu-id="1fad8-118">Компоненты правил распределения книги учета</span><span class="sxs-lookup"><span data-stu-id="1fad8-118">Components of ledger allocation rules</span></span>
<span data-ttu-id="1fad8-119">Все правила распределения включают четыре элемента: общие, источник, назначение и поправка.</span><span class="sxs-lookup"><span data-stu-id="1fad8-119">Each allocation rule has four components: general, source, destination, and offset.</span></span> <span data-ttu-id="1fad8-120">Если используется метод распределения "Базовый", требуется дополнительный компонент: базовые правила распределения книги учета.</span><span class="sxs-lookup"><span data-stu-id="1fad8-120">An additional component, ledger allocation bases rules, is required if Basis is used as the allocation method.</span></span> <span data-ttu-id="1fad8-121">Каждый из компонентов обеспечивает важный сегмент информации, необходимой для выполнения распределений.</span><span class="sxs-lookup"><span data-stu-id="1fad8-121">Each component provides a critical piece of the information that is required in order to process allocations.</span></span>

-   <span data-ttu-id="1fad8-122">**Общий** — в этом компоненте пользователь указывает различные параметры, такие как метод распределения, параметры внутрихолдинговых правил и признак активации правила.</span><span class="sxs-lookup"><span data-stu-id="1fad8-122">**General** – This component is where the user specifies options such as the allocation method, intercompany rule settings, and whether the rule is active.</span></span>
-   <span data-ttu-id="1fad8-123">**Источник** — в этом компоненте пользователь определяет исходные данные для распределения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-123">**Source** – This component is where the user specifies the source data for the allocation.</span></span> <span data-ttu-id="1fad8-124">Распределение может основываться на сальдо ГК (**Источник данных** = **Книга учета**) или фиксированных суммах (**Источник данных** = **Фиксированное значение**).</span><span class="sxs-lookup"><span data-stu-id="1fad8-124">Allocation can be based on ledger balances (**Data source** = **Ledger**) or fixed amounts (**Data source** = **Fixed value**).</span></span> <span data-ttu-id="1fad8-125">Если для параметра **Источник данных** задано значение **Книга учета**, необходимо определить критерии фильтра по источнику для правила распределения книги учета (например, для расходов на рекламу).</span><span class="sxs-lookup"><span data-stu-id="1fad8-125">When **Data source** is set to **Ledger**, source filter criteria must be defined for the ledger allocation rule (for example, for the advertising expenses).</span></span>
-   <span data-ttu-id="1fad8-126">**Место назначения** — этот компонент определяет, как результат расчета распределения следует распределять и учитывать.</span><span class="sxs-lookup"><span data-stu-id="1fad8-126">**Destination** – This component defines how the result of the allocation calculation should be distributed and accounted for.</span></span> <span data-ttu-id="1fad8-127">Например, может быть одна строка места назначения для каждого подразделения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-127">For example, there can be one destination line for each department.</span></span>
-   <span data-ttu-id="1fad8-128">**Корр. счет** — этот компонент определяет, как счета ГК и аналитики должны быть определены для записей корреспондентских счетов, которые балансируют записи места назначения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-128">**Offset** – This component defines how main accounts and dimensions should be determined for the offset entries that balance the destination entries.</span></span> <span data-ttu-id="1fad8-129">Определенные пользователем параметры обычно используются вместо счетов и аналитик, основанных на источнике.</span><span class="sxs-lookup"><span data-stu-id="1fad8-129">User-defined options are typically used instead of accounts and dimensions that are based on the source.</span></span> <span data-ttu-id="1fad8-130">Когда для параметра **Источник данных** задано значение **Фиксированное значение**, **Источник** нельзя использовать как вариант.</span><span class="sxs-lookup"><span data-stu-id="1fad8-130">When **Data source** is set to **Fixed value**, **Source** can't be used as an option.</span></span>
-   <span data-ttu-id="1fad8-131">**Базовые правила распределения ГК** — эти правила используют свои собственные критерии фильтрации источника для того, чтобы определить, какие сальдо ГК должны использоваться для распределения (например, доход по подразделениям).</span><span class="sxs-lookup"><span data-stu-id="1fad8-131">**Ledger allocation basis rules** – These rules use their own source filter criteria to determine which ledger balances should be used for allocation (for example, the revenue per department).</span></span> <span data-ttu-id="1fad8-132">Каждое базовое правило распределения может использоваться вместе с несколькими правилами распределения.</span><span class="sxs-lookup"><span data-stu-id="1fad8-132">Each allocation basis rule can be used with multiple allocation rules.</span></span>
