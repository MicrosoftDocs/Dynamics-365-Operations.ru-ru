# <a name="cost-object-dimensions"></a>Аналитики объекта затрат

[!include[banner](../includes/banner.md)]


Когда вы анализируете затраты, вы используете аналитики элемента затрат, чтобы определить направление потока затрат. Аналитики объекта затрат можно использовать для определения того, где необходимо назначить затраты. В этом разделе представлены сведения об аналитиках объекта затрат.

Объект затрат может быть любым типом объекта, который требуется оценить, распределить на него затраты или измерить напрямую. Типичные объекты затрат включают продукты, проекты, ресурсы, подразделения, центры затрат и географические регионы. Руководство использует объекты затраты для определения количественной меры затрат, а также и для проведения анализа прибыльности.

## <a name="cost-object-dimensions-and-cost-object-dimension-members"></a>Аналитики объектов затрат и члены аналитик объектов затрат
Объекты затрат известны как *аналитики объектов затрат*. После того как вы решили, на какой объект должна ссылаться аналитика объекта затрат, необходимо указать отдельные значения аналитики или импортировать их в модуль "Учет затрат" из других исходных систем. Эти отдельные значения аналитики называются *членами аналитики объекта затрат*. Например, вы хотите использовать финансовую аналитику, которая называется "Центр затрат" как аналитику объекта затрат. Чтобы посмотреть, как затраты переносятся в отдельные центры затрат, необходимо импортировать члены аналитики объекта затрат. В этом случае члены аналитики объекта затрат являются фактическими центрами затрат, такими как продажи, производство, администрирование и географические местоположения. Следующий снимок экрана показывает пример центров затрат как аналитик объекта затрат со своими фактическими центрами затрат в качестве членов аналитики объекта затрат.

## <a name="import-cost-object-dimension-members-through-data-connectors"></a>Импорт членов аналитики объекта затрат через соединители данных
Чтобы упростить импорт членов аналитики объекта затрат используются соединители данных для извлечения значений из объектов, которые необходимо использовать в качестве аналитик объекта затрат. Можно использовать предварительно подготовленные соединители данных или пользовательские соединители данных, собранные вами.
