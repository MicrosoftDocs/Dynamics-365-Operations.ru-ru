# <a name="establish-customer-method-of-payment"></a>Установка способов оплаты для клиентов

[!include [task guide banner](../../includes/task-guide-banner.md)]

Создайте способ оплаты для платежей клиентов. В этой задаче используется демонстрационная компания USMF.

1. Перейдите в раздел "Расчеты с клиентами" > "Настройка платежей" > "Способы оплаты".
2. Щелкните "Создать".
3. В поле "Способ оплаты" введите код способа оплаты.
    * Код способа оплаты отображается в накладных и платежах, поэтому присвойте ему довольно описательное имя, чтобы пользователи понимали, какой тип платежа записывается и для какого банковского счета.  
4. В поле "Описание" введите описание.
5. Выберите, какой статус оплаты требуется для разноски платежей.
    * При создании платежа клиента платеж можно разнести, только если статус оплаты совпадает со статусом оплаты, определенным здесь.  
6. Выберите способ создания платежей клиентов для накладных.
    * Этот параметр используется только при выполнении предложения по оплате. Предложение по оплате можно использовать для платежей клиентов при прямом дебетовании и извлечении средств с банковских счетов клиентов.  
7. Выберите тип платежа.
    * Тип платежа поможет определить, будет ли выполняться проверка платежа.  
8. Выберите, куда будут разноситься платежи типа счета.
    * Обычно в качестве значения этого параметра выбирают "Банк".  
9. Выберите банковский счет, на который будет записываться этот платеж.
10. Введите тип банковской проводки, чтобы определить тип платежа, используемого банком.
    * Тип банковской проводки используется в процессе банковской выверки и упрощает выверку.  
11. Укажите, будет ли этот платеж временно разноситься на промежуточный счет.
    * Если вы хотите выделить резервное время для того, чтобы платеж прошел клиринг в банке, используйте функцию "Промежуточный счет". Платеж временно разносится на счет ГК до клиринга в банке, после чего платеж будет помещен на банковский счет, указанный здесь.  
12. Введите счет ГК, используемый для разноски на промежуточный счет.
    * Это счет ГК, на который временно разносится платеж при использовании промежуточного счета.  
13. Используйте вкладку "Формат файла", чтобы определить настройки для электронных платежей.
14. Используйте вкладку "Контроль платежей", чтобы определить обязательные поля.
    * Например, если необходимо депонировать все платежи с этим способом оплаты, можно выбрать этот параметр на этой вкладке.  
15. Используйте вкладку "Атрибуты оплаты", чтобы определить атрибуты оплаты для использования с данным способом оплаты.
16. Нажмите кнопку "Сохранить".
