# <a name="base-price-and-trade-agreements"></a>Базовая цена и коммерческие соглашения

[!include[task guide banner](../includes/task-guide-banner.md)]

Эта процедура содержит инструкции по созданию коммерческих предложений по ценам продажи для конкретных каналов. В этой процедуре используется компания с демонстрационными данными USRT.

1. В **области переходов** выберите **Модули > Розничная торговля и коммерция > Управление ценообразованием и скидками > Группы цен > Все группы цен**. Ценовые группы определяют, каким образом коммерческие соглашения назначаются конкретным каналам. Использование ценовых групп для назначения коммерческих соглашений каналам делает возможным ценообразование на уровне каналов.  
2. Нажмите кнопку **Создать**.
3. В поле **Группы цен** введите значение.
4. В поле **Имя** введите значение.
5. Нажмите кнопку **Сохранить**.
6. Закройте страницу.
7. В области **Область переходов** выберите **Модули > Розничная торговля и коммерция > Каналы > Розничные магазины > Все розничные магазины**.
8. В списке выберите "Нью-Йорк".
9. В области действий щелкните **Магазин**.
10. Щелкните **Группы цен**.
11. Нажмите кнопку **Создать**.
12. В поле **Группы цен** нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
13. В списке найдите и выберите требуемую запись.
14. Нажмите кнопку **Сохранить**.
15. Закройте страницу.
16. Закройте страницу.
17. В **области переходов** выберите **Модули > Розничная торговля и коммерция > Продукты и категории > Выпущенные продукты по категориям**.
18. В списке перейдите по ссылке в выбранной строке.
19. Выберите **Изменить**.
20. Разверните экспресс-вкладку **Продажа**.
21. В поле **Цена** введите число. Эта цена используется, если не найдены подходящие коммерческие соглашения.  
22. Нажмите кнопку **Сохранить**.
23. В области **Панель операций** щелкните **Продажа**.
24. Щелкните **Создать коммерческие соглашения**.
25. Нажмите кнопку **Создать**.
26. В поле **Имя** нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
27. В списке выберите "Розница". В демонстрационных данных имя журнала "Розница" по умолчанию имеет связь "Цена (продажи)". Это означает, что все новые созданные строки по умолчанию будут иметь цены продажи из коммерческих соглашений.  
28. В **Панели операций** щелкните **Строки**.
29. В поле **Код счета** выберите "Группа".
30. В поле **Выбор счета** нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
31. В списке найдите и выберите требуемую запись. Это завершить связь от канала к ценовой группе и коммерческому соглашению.  
32. В поле **Связь номенклатуры** введите значение.
33. В поле **Сумма в валюте** введите число.
34. На экспресс-вкладке **Сведения** установите или снимите флажок **Найти следующий**. Если поле **Найти далее** имеет значение "Да" модуль ценообразования продолжает поиск применимых коммерческих соглашений с более низкой ценой продажи. Когда поле **Найти далее** имеет значение "Нет", модуль ценообразования прекращает поиск и использует коммерческое соглашение.  
35. Щелкните **Разнести**.
36. Щелкните **OK**.
37. Закройте страницу.
38. В области **Панель операций** щелкните "Продажа".
39. Щелкните **Цена продажи**.
