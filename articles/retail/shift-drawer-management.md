## <a name="shift-operations"></a><span data-ttu-id="228e2-178">Операции со сменами</span><span class="sxs-lookup"><span data-stu-id="228e2-178">Shift operations</span></span>
<span data-ttu-id="228e2-179">Можно выполнять различные действия для изменения состояния смены или для увеличения или уменьшения суммы денег в денежном ящике.</span><span class="sxs-lookup"><span data-stu-id="228e2-179">Various actions can be taken to change the state of a shift or to increase or decrease the amount of money in the drawer.</span></span> <span data-ttu-id="228e2-180">В приведенном ниже разделе эти операции со сменами для Modern POS и Cloud POS в Dynamics 365 for Retail.</span><span class="sxs-lookup"><span data-stu-id="228e2-180">The section below describes these shift operations for Dynamics 365 for Retail Modern POS and Cloud POS.</span></span>

<span data-ttu-id="228e2-181">**Открыть смену**</span><span class="sxs-lookup"><span data-stu-id="228e2-181">**Open shift**</span></span>

<span data-ttu-id="228e2-182">POS требует, чтобы пользователь имел активную открытую смену для выполнения любых операций, которые приводят к выполнению финансовой проводки, например заказа на продажи, возврата или заказа пользователя.</span><span class="sxs-lookup"><span data-stu-id="228e2-182">POS requires that a user has an active, open shift to perform any operations that would result in a financial transaction such as a sale, return, or customer order.</span></span>  

<span data-ttu-id="228e2-183">При входе в POS система сначала проверяет, имеет ли пользователь открытую доступную смену на текущей ККМ.</span><span class="sxs-lookup"><span data-stu-id="228e2-183">When logging into the POS, the system first checks to see if the user has an active shift available on the current register.</span></span> <span data-ttu-id="228e2-184">Если нет, пользователь может выбрать открыть новую смену, возобновить существующую смену или продолжить вход в режиме "без денежного ящика", в зависимости от конфигурации системы и его разрешения.</span><span class="sxs-lookup"><span data-stu-id="228e2-184">If not, the user can then choose to open a new shift, resume an existing shift, or continue to login in “non-drawer” mode, depending on the system configuration and their permissions.</span></span>

<span data-ttu-id="228e2-185">**Объявить начальную сумму**</span><span class="sxs-lookup"><span data-stu-id="228e2-185">**Declare start amount**</span></span>

<span data-ttu-id="228e2-186">Эта операция часто является первым действием, которое выполняется новой открытой сменой.</span><span class="sxs-lookup"><span data-stu-id="228e2-186">This operation is often the first action that is taken a newly opened shift.</span></span> <span data-ttu-id="228e2-187">Пользователи указывают начальную сумму в денежном ящике для смены.</span><span class="sxs-lookup"><span data-stu-id="228e2-187">Users specify the starting cash amount in the drawer for the shift.</span></span> <span data-ttu-id="228e2-188">Это важно из-за расчета превышения/недостачи, который выполняется при закрытии счетов смены для этой суммы.</span><span class="sxs-lookup"><span data-stu-id="228e2-188">This is important because the over/short calculation that occurs when closing a shift accounts for this amount.</span></span>

<span data-ttu-id="228e2-190">Ввод остатка в кассе являются не связанными с продажей проводками, которые выполняются в активной смене, и они увеличивают сумму денег в денежном ящике.</span><span class="sxs-lookup"><span data-stu-id="228e2-190">Float entries are non-sales transactions that are performed in an active shift and they increase the amount of the cash in the drawer.</span></span> <span data-ttu-id="228e2-191">Обычным примером ввода остатка в кассе является добавление дополнительной мелочи в денежный ящик, когда ее остается мало.</span><span class="sxs-lookup"><span data-stu-id="228e2-191">A common example of a float entry would be to add additional change to the drawer when it is running low.</span></span>

<span data-ttu-id="228e2-192">**Удаление платежного средства**</span><span class="sxs-lookup"><span data-stu-id="228e2-192">**Tender removal**</span></span>

<span data-ttu-id="228e2-193">Изъятия денежных средств являются не связанными с продажей проводками, которые выполняются в активной смене, чтобы уменьшить сумму денег в денежном ящике.</span><span class="sxs-lookup"><span data-stu-id="228e2-193">Tender removals are non-sales transactions that are performed in an active shift to reduce the amount of the cash in the drawer.</span></span> <span data-ttu-id="228e2-194">Наиболее часто это используется в сочетании с вводам остатка в кассе в другую смену.</span><span class="sxs-lookup"><span data-stu-id="228e2-194">This is most commonly used in conjunction with a float entry on a different shift.</span></span> <span data-ttu-id="228e2-195">Например, в ККМ 1 осталось мало мелочи, поэтому пользователь ККМ 2 выполняет изъятие денежных средств для уменьшения суммы в кассе.</span><span class="sxs-lookup"><span data-stu-id="228e2-195">For example, Register 1 is running low on change, so the user on Register 2 performs a tender removal to reduce the drawer amount.</span></span> <span data-ttu-id="228e2-196">Пользователь ККМ 1 затем выполнит ввод остатка в кассе, чтобы увеличить сумму.</span><span class="sxs-lookup"><span data-stu-id="228e2-196">The user on Register 1 would then perform a float entry to increase their amount.</span></span>

<span data-ttu-id="228e2-198">Пользователи могут приостановить свою активную смену для освобождения текущей ККМ для другого пользователя или для перемещения своей смены на другую ККМ (это часто называется "перемещением кассы").</span><span class="sxs-lookup"><span data-stu-id="228e2-198">Users can suspend their active shift to free up the current register for another user, or to move their shift to a different register (this is often referred to as a “floating till”).</span></span> 

<span data-ttu-id="228e2-199">Приостановка смены запрещает любые новые проводки или изменения до возобновления смены.</span><span class="sxs-lookup"><span data-stu-id="228e2-199">Suspending the shift prevents any new transactions or changes to the shift until it resumed.</span></span>

<span data-ttu-id="228e2-201">Эта операция позволяет пользователю возобновить ранее приостановленную смену на ККМ, в которой еще нет активной смены.</span><span class="sxs-lookup"><span data-stu-id="228e2-201">This operation allows a user to resume a previously suspended shift on a register that does not already have an active shift.</span></span>
