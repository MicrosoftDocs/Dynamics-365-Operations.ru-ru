# <a name="set-up-fraud-alerts"></a><span data-ttu-id="e9929-104">Настройка оповещений о мошенничестве</span><span class="sxs-lookup"><span data-stu-id="e9929-104">Set up fraud alerts</span></span>

[!include[banner](includes/banner.md)]

<span data-ttu-id="e9929-105">В этом разделе объясняется, как настроить критерии и правила для блокировки потенциально мошеннических заказов на продажу для дополнительного исследования.</span><span class="sxs-lookup"><span data-stu-id="e9929-105">This topic explains how to set up criteria and rules to place potentially fraudulent sales orders on hold for further review.</span></span> <span data-ttu-id="e9929-106">Функция проверки на мошенничество используется для определения действительности сведений в заказе на продажу.</span><span class="sxs-lookup"><span data-stu-id="e9929-106">Fraud review functionality is used to determine the validity of information in a sales order.</span></span> <span data-ttu-id="e9929-107">Если сведения в заказе на продажу выглядят недостоверными на основе принятых в организации критериев и правил обнаружения мошенничества, заказ может быть заблокирован для дополнительной проверки администратором.</span><span class="sxs-lookup"><span data-stu-id="e9929-107">If the information in the sales order appears to be questionable based on an organization’s fraud criteria and rules, the order may be put on hold for further review by an administrator.</span></span>

> [!NOTE]
> <span data-ttu-id="e9929-108">Эта функция может использоваться только с обработкой заказа на продажу в канале центра обработки вызовов Retail.</span><span class="sxs-lookup"><span data-stu-id="e9929-108">This feature can only be used with Retail Call Center channel sales order processing.</span></span> 

<span data-ttu-id="e9929-109">Если канал центра обработки вызовов определен, параметр **Включить заполнение заказа** должен иметь значение **Да**.</span><span class="sxs-lookup"><span data-stu-id="e9929-109">When the Call Center channel is defined, **Enable Order Completion** must be set to **Yes**.</span></span> <span data-ttu-id="e9929-110">Если завершение заказа включено, пользователи могут просмотреть краткие сведения по заказу и щелкнуть **Отправить** для завершения заказа.</span><span class="sxs-lookup"><span data-stu-id="e9929-110">When order completion is enabled, users can view the order recap and click **Submit** to finalize the order.</span></span> <span data-ttu-id="e9929-111">Пользователям могут также быть предоставлены возможности вручную блокировать заказы на продажу для проверки на предмет мошенничества.</span><span class="sxs-lookup"><span data-stu-id="e9929-111">Users will also be provided options to manually place the sales order on fraud review hold.</span></span> <span data-ttu-id="e9929-112">Заказы на продажу, введенные пользователем центра обработки вызовов, обрабатываются в рамках правил и критериев проверки на мошенничество во время процесса отправки.</span><span class="sxs-lookup"><span data-stu-id="e9929-112">Sales orders keyed in by a Call Center user are processed through fraud check rules and criteria during the submit process.</span></span>

<span data-ttu-id="e9929-113">Существует два типа критериев мошенничества, которые используются системой для проверки, требуется ли заблокировать заказ для проверки на мошенничество:</span><span class="sxs-lookup"><span data-stu-id="e9929-113">There are two types of fraud criteria that the system will reference to check if the order should be held for fraud review:</span></span>

-   <span data-ttu-id="e9929-114">**Статические правила** используют конкретное значение, например номер телефона, который был внесен в черный список или адрес электронной почты, помеченный как использовавшийся для прошлых мошеннических операций.</span><span class="sxs-lookup"><span data-stu-id="e9929-114">**Static rules** use a specific value, such as a phone number that has been blacklisted or an email address that has been flagged as being known as used for past fraudulent transactions.</span></span> <span data-ttu-id="e9929-115">На странице **Статические данные о мошенничестве** сведения о мошенничестве могут быть добавлены вручную или с помощью импорта данных вместе с оценками, добавленными к сведениям о мошенничестве.</span><span class="sxs-lookup"><span data-stu-id="e9929-115">On the **Static Fraud Data** page, fraud information can be added manually or through data import, with scores attached to the fraudulent information.</span></span> <span data-ttu-id="e9929-116">Если включена проверка на мошенничество, каждый введенный заказ на продажу будет сравниваться со статическими данными.</span><span class="sxs-lookup"><span data-stu-id="e9929-116">If fraud checking is turned on, every sales order entered will be compared to the static data.</span></span> <span data-ttu-id="e9929-117">Если данные находятся в адресе клиента для выставления счетов или адресе поставки, или если данные находятся в адресе доставки в строке продаж, оценки будут суммироваться для всех уникальных совпадений.</span><span class="sxs-lookup"><span data-stu-id="e9929-117">If the data is found in either the customer’s billing or delivery address, or if the data is found in the delivery address on the sales line, the scores of all unique matches will be summed.</span></span>  
-   <span data-ttu-id="e9929-118">**Динамические правила** состоят из переменных и условий, определенных для этих переменных.</span><span class="sxs-lookup"><span data-stu-id="e9929-118">**Dynamic rules** are composed from variables and conditions defined for those variables.</span></span> <span data-ttu-id="e9929-119">С помощью динамических правил можно проверять другие критерии, не включенные в статические правила.</span><span class="sxs-lookup"><span data-stu-id="e9929-119">With dynamic rules, other criteria not defined in the static rules can be checked.</span></span> <span data-ttu-id="e9929-120">Более сложные операторы "И/ИЛИ" могут использоваться для учета несколько условий, чтобы определить, имеется ли положительное соответствие условиям правила в отправляемом заказе на продажу.</span><span class="sxs-lookup"><span data-stu-id="e9929-120">More complex “AND/OR” statements can be used to consider multiple conditions to determine if there is a positive match against the rule criteria and the sales order being submitted.</span></span> <span data-ttu-id="e9929-121">Например, если пользователь хочет блокировать для проверки на мошенничество все заказов клиентов, связанных к конкретным значением группы клиентов, которые заказывают определенный продукт, условия для проверки клиента и проверка продуктов будет определяться на странице **Правила** с условием типа "И".</span><span class="sxs-lookup"><span data-stu-id="e9929-121">For example, if a user wants to hold for fraud review all orders for customers that are tied to a specific customer group value and that ordered a specific product, the conditions to validate customer and validate products would be defined on the **Rules** page with an “AND” condition.</span></span> <span data-ttu-id="e9929-122">Заказ блокируется с целью проверки на мошенничество только в том случае, если оба условия выполнены и если значение оценки, назначенное правилу, делает общую оценку вероятности мошенничества для заказа больше минимальной оценки вероятности мошенничества, определенной в параметрах центра обработки вызовов.</span><span class="sxs-lookup"><span data-stu-id="e9929-122">The order would fall into fraud hold only if both conditions were true and if the score value assigned to the rule put the order’s total fraud score over the minimum fraud score as defined in Call Center parameters.</span></span>

<span data-ttu-id="e9929-123">Пользователь центра вызовов может также вручную поместить заказ в очередь блокировки для проверки на мошенничество.</span><span class="sxs-lookup"><span data-stu-id="e9929-123">A call center user may also manually place an order into the fraud hold queue.</span></span> <span data-ttu-id="e9929-124">Если пользователь, который вводит заказ, считает, что клиент, размещающий заказ, может быть подозрительным, и хочет, чтобы какой-то другой пользователь дополнительно проверил действительность заказа перед его обработкой, пользователь, который вводит заказ, может выбрать параметр **Удержание вручную по мошенничеству** в раскрывающемся меню **Удержания** на странице **Сводка по заказу на продажу** (отображается после вызова функции заказа **Завершено**).</span><span class="sxs-lookup"><span data-stu-id="e9929-124">If the user entering the order believes the customer placing the order may be suspicious and wants someone else to further review the validity of the order before it is processed, the user entering the order can choose the **Manual Fraud Hold** option from the **Holds** dropdown menu on the **Sales Order Summary** page (this appears after the **Complete** order function is called).</span></span> <span data-ttu-id="e9929-125">Пользователю будет предложено ввести заметку для дополнительного пояснения, почему он считает, что заказ может быть мошенническим, чтобы проверяющий имел дополнительный контекст.</span><span class="sxs-lookup"><span data-stu-id="e9929-125">The user will be prompted to enter a note to further explain why they feel the order may be fraudulent so that the reviewer has more context.</span></span>

<span data-ttu-id="e9929-126">Все заказы, заблокированные с помощью ручной блокировки мошеннических заказов или в результате вычисления оценки вероятности мошенничества системой, отображаются на странице **Удержания заказов**, на которой заказ может быть проверен и либо отменен, либо одобрен для обработки.</span><span class="sxs-lookup"><span data-stu-id="e9929-126">All orders held through manual fraud hold or by systematic calculation of fraud scores will appear on the **Order Holds** page, where the order can be reviewed and then either canceled or released to processing.</span></span>

> [!NOTE]
> <span data-ttu-id="e9929-127">Использование нескольких правил или слишком сложных правил приведет к низкой производительности системы при отправке заказов на продажу.</span><span class="sxs-lookup"><span data-stu-id="e9929-127">Use of multiple rules or overly complex rules will result in poor system performance when sales orders are submitted.</span></span> <span data-ttu-id="e9929-128">Функция оповещения о мошенничестве не были оптимизированы для обработки большого количества статических записей данных о мошенничестве и большого числа активных правил.</span><span class="sxs-lookup"><span data-stu-id="e9929-128">The fraud alert feature hasn't been optimized to handle a large volume of static fraud data entries and many active rules.</span></span> <span data-ttu-id="e9929-129">Важно помнить, что каждое правило оценивается при выполнении функции отправки записи заказа на продажу в центре вызова.</span><span class="sxs-lookup"><span data-stu-id="e9929-129">It’s important to remember that every rule is evaluated during the submit function of call center sales order entry.</span></span> <span data-ttu-id="e9929-130">Правила оцениваются по заголовку заказа на продажу и всем строкам заказа.</span><span class="sxs-lookup"><span data-stu-id="e9929-130">The rules are evaluated against the sales order header and all order lines.</span></span> <span data-ttu-id="e9929-131">Чем больше правил и чем сложнее инструкции правил, тем больше времени потребуется для их обработки.</span><span class="sxs-lookup"><span data-stu-id="e9929-131">The more rules there are and the more complex the rule statements are, the longer it will take to process.</span></span> <span data-ttu-id="e9929-132">Если имеется большое количество строка номенклатуры в заказе, а также большое количество активных правил и статических записей данных, этот систематический процесс просмотра и утверждения всех данных и расчета оценки мошенничества может иметь очень значительное влияние на быстродействие.</span><span class="sxs-lookup"><span data-stu-id="e9929-132">If you have a large number of line items on your order, and a large number of active rules and static data entries, this systematic process of reviewing and validating all of the data and calculating a fraud score can have a severe performance impact.</span></span>  <span data-ttu-id="e9929-133">Организации, использующие этот компонент, должны всегда тестировать и проверять, что время обработки отправки заказа является приемлемым, прежде чем применять какие-либо изменения в правилах или статических критериях выявления мошенничества в рабочей среде.</span><span class="sxs-lookup"><span data-stu-id="e9929-133">Organizations using this feature should always test and confirm that the order submission processing time is acceptable before applying any changes to rules or static fraud criteria into the production environment.</span></span>
