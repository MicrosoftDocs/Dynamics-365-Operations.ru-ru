# <a name="call-center-functionality"></a>Функциональность центра обработки вызовов

[!include[banner](includes/banner.md)]


Эта статья содержит обзор возможностей продаж в центре обработки вызовов в Microsoft Dynamics 365 for Retail.

Dynamics 365 for Retail поддерживает центры обработки вызовов как тип канала розничной торговли. В центре обработки вызовов работники принимают заказы от клиентов по телефону и создают заказы на продажу. К функциям центра обработки вызовов относится возможность более быстро принимать заказы по телефону и обслуживать клиентов в рамках процесса выполнения заказа. Например, сотрудники центра обработки вызовов могут вводить сведения о платежах непосредственно в заказ на продажу и просматривать подробную сводку расходов и платежей перед отправкой заказа. Работники также могут контролировать ценообразование и имеют доступ к различным данным о клиентах, продуктах и ценах на странице **заказ на продажу**. Кроме того, В центрах обработки вызовов также доступна расширенная функциональность для отслеживания истории клиентов и статуса заказов. Каждый центр обработки вызовов может иметь собственных пользователей, способы оплаты, ценовые группы, финансовые аналитики и режимы доставки. Эти параметры можно настроить при создании центра обработки вызовов. Кроме того, Страница **Центр обработки вызовов** используется для включения или отключения следующих групп функций, свойственных центрам обработки вызовов.

-   **Завершение заказа** — эта группа содержит функции, связанные с платежами и завершением заказа в странице **заказ на продажу**.
-   **Направленные продажи** — эта группа содержит функции, связанные с кодами источников, сценариями и запросами каталогов.

Если эти функции включить в настройках центра обработки вызовов, они становятся доступны на странице **Заказ на продажу** для пользователей, связанных с центром обработки вызовов. Для большинства этих функций требуется дополнительная настройка, прежде чем их можно будет использовать. Прежде чем пользователи могут создать заказы центра обработки вызовов, вы должны добавить тех пользователей к центру обработки вызовов как пользователей центра обработки вызовов. Этот шаг включает конфигурацию и функции для центра обработки вызовов. Здесь некоторые примеры функциональности, которая будет доступной:

-   Направленные продажи обеспечивает опции конфигурации для сценариев телепродаж и изображения продукта, которые помогают и направляют сотрудников отдела продаж, пока они принимают заказы.
-   Заказы нельзя завершить до тех пор пока сотрудник отдела продаж не запишет по крайней мере один метод оплаты.
-   Правила расширения продаж и перекрестной продажи можно установить для того, чтобы предложить сотрудникам отдела продаж продвинуть определенные продукты для клиента.
-   Сотрудники отдела продаж могут захватить исходный код для каталога, из которого клиент заказывает.
-   Сотрудники отдела продаж могут добавить купоны розничного торговца к заказу.
-   Сотрудники отдела продаж могут продать программы непрерывности.
-   Заказы можно заблокировать вручную или автоматически, для того, чтобы показать, что дополнительное расследование необходимо, прежде чем заказ можно обрабатывать.
