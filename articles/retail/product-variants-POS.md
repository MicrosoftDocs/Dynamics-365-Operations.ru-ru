# <a name="inventory-lookup-in-the-point-of-sale-pos"></a>Поиск запасов в POS

[!include [banner](includes/banner.md)]

Поиск запасов в POS помогает предприятиям розничной торговли успешно работать в режиме реального времени и получать аналитические данные путем подключения магазинов, POS и бэк-офиса. Эта функция предоставляет точное представление в режиме реального времени запасов продуктов по магазинам и центрами распределения. Она также помогает предприятиям розничной торговли обеспечивать дополнительную эффективность и снизить затраты, улучшая планирование запасов в режиме реального времени.

Точное представление в режиме реального времени запасов по всей организации помогает продавцам магазинов предоставлять отличное и своевременное обслуживания клиентов. Самый важный момент — это когда клиент готов принять решение о покупке. Важно, чтобы кассирам в магазине была легко доступна информация о запасах в режиме реального времени, чтобы они могли точно гарантировать доставку и отправку продукта.

Можно открыть страницу **Поиск запасов** из рабочей области **Retail Modern POS** или **Retail Cloud POS**.

![Плитка поиска запасов на домашней странице POS](media/POSHomepage.png)

На странице **Поиск запасов** можно использовать цифровую клавиатуру для ввода номера продукта. Затем можно просмотреть количество в наличии для нескольких магазинов и складов.

![Стандартная страница поиска запасов](media/InventoryLookUp.png)

Для каждого местонахождения также отображаются количества **Зарезервировано** и **Заказано**.

- **Зарезервировано** — это количество означает значение **Физически зарезервировано** из бэк-офиса для указанного номера продукта в местонахождении.
- **Заказано** — это количество означает значение **Всего заказано** из бэк-офиса для указанного номера продукта в местонахождении.

## <a name="locations-that-inventory-availability-information-is-shown-for"></a>Местонахождения, для которых отображается информация о доступности запасов

Список местоположений включает в себя два вида объектов:

- **Розничные магазины** — список магазинов, которые настраиваются с использованием группы указателей магазинов для текущего магазина в "Розничная сеть - Центральный офис".
- **Центры распределения** — различные типы центров распределения (такие как склады) могут быть настроены в Microsoft Dynamics 365 for Retail. Однако в списке отображаются сведения о доступности запасов только для центров распределения типа по умолчанию **Стандартный**.

    > [!NOTE]
    > Сведения о доступности запасов не отображаются для складов типов **Транзитный**, **Карантинный** и **Товаров в пути** для POS.

На странице **Поиск запасов** можно просматривать доступные для резервирования (ATP) количества для каждого магазина, помимо текущих количеств в наличии, зарезервированных количеств и заказанных количеств. Выберите магазин, для которого требуется просмотреть информацию о доступности для резервирования (ATP), затем выберите **Показать доступность магазина**.

![Количества ATP](media/ATP.png)

## <a name="opening-the-dimension-based-matrix-view-to-show-all-variants"></a>Открытие аналитики на основе матричного представления для отображения всех вариантов

На странице **Сведения о продукте** шаблона продукта или на странице **Поиск запасов** выберите **Просмотр всех вариантов** на панели приложения в нижней части страницы. Представление **Матрица на основе аналитик** для исходного запуска с этих страниц содержит сведений о доступности запасов для всех вариантов продукта для текущего магазина.

> [!NOTE]
> Кнопка **Просмотр всех вариантов** доступна только для шаблонов продуктов номенклатуры, которые имеют варианты продукта. Она недоступна для отдельных продуктов или наборов.

![Кнопка просмотра всех вариантов на странице поиска запасов](media/StandardToMatrix.png)

Выберите **Просмотр всех вариантов** на странице **Сведения о продукте** шаблона продукта или на странице **Поиск запасов** без выбора местоположения, чтобы перейти к представлению **Матрица на основе аналитик** для просмотра сведений о доступности запасов для всех вариантов продукта для текущего магазина.

![Представление матрицы на основе аналитик для страницы поиска запасов](media/Matrix.png)

> [!NOTE]
> На приведенном выше рисунке аналитики отображаются в алфавитном порядке, так как порядок отображения аналитик не был настроен для выбранного продукта.

В представлении **Матрица на основе аналитик** ячейки вариантов продукта включают значение в наличии в правом нижнем углу. В следующей таблице описывается значение различных значений.

| Стоимость запасов в наличии                            | описание |
|------------------------------------------|-------------|
| Числовое значение, которое больше 0 (нуля) | Вариант выпущен в выбранном местоположении, и вы можете выполнять дополнительные действия в ячейке. (Эти действия описаны более подробно ниже в этом разделе.) |
| **0** (ноль)                             | Вариант был выпущен в выбранном местоположении, но номенклатура недоступна в указанном местоположении. Тем не менее можно выполнять дополнительные действия в ячейке. (Эти действия описаны более подробно ниже в этом разделе.) |
| **н/д** или неактивная ячейка              | Вариант не был выпущен в выбранном местоположении, и вы не можете выполнять дополнительные действия в ячейке. |

Можно также изменить опорный элемент для аналитик, выбрав новую аналитику для использования.

![Изменение опорного элемента](media/ChangePivot.png)

![Опорный элемент изменен](media/PivotChanged.png)

> [!NOTE]
> На предыдущем рисунке порядок отображения аналитик для выбранного продукта является пользовательским (не по алфавиту). Он основан на порядке отображения аналитик, заданном в бэк-офисе.

Кроме того, в представлении **Матрица на основе аналитик** дополнительные действия могут выполняться для повышения производительности сотрудников магазина. Далее приводятся некоторые примеры.

- Изменение местоположения магазина для поиска доступности запасов всех вариантов продукта в других расположениях. Эти местоположения включают другие магазины в группе указателей магазина и центры распределения типа по умолчанию **Стандартный**.
- Продажа индивидуального варианта продукта клиенту с помощью продажи без доставки при оплате наличными, получения в магазине или отгрузки в адрес.
- Сообщите клиенту сведений о доступности для резервирования для индивидуального варианта продукта в определенном местоположении.

![Дополнительные действия на плитках вариантов](media/VariantActions.png)

> [!NOTE]
> На приведенном выше рисунке аналитики отображаются в алфавитном порядке, так как порядок отображения аналитик не был настроен для выбранного продукта.

Следующая таблица содержит дополнительные сведения о дополнительных действий, которые доступны.

| Действие               | описание |
|----------------------|-------------|
| Продать сейчас             | Добавление выбранного варианта номенклатуры в проводку и перенаправление пользователя на экран проводки. (Это действие недоступно, когда выбранное местоположение является центром распределения.) |
| Самовывоз из магазина     | Создание заказа клиента для варианта продукта, который клиент заберет из выбранного местонахождения, и перенаправление пользователя на экран проводки. (Это действие недоступно, когда выбранное местоположение является центром распределения.) |
| Отгрузить продукт         | Создание заказа клиента для варианта продукта, который будет отгружен из выбранного местонахождения, и перенаправление пользователя на экран проводки. |
| Доступность         | Отображение сведений о доступности для резервирования для выбранной комбинации вариантов для выбранного местонахождения. |
| Показать все местоположения   | Переключение на стандартное представление поиска запасов и выделение сведений о доступности запасов для варианта номенклатуры по всем магазинам в группе указателей магазинов, а также в центрах распределения типа **Стандартный/По умолчанию**. |
| Просмотр сведений о продукте | Перенаправление пользователя на страницу **Сведения о продукте** связанного шаблона продукта. |
