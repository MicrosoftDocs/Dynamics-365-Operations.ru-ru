# <a name="system-requirements-for-on-premises-deployments"></a>Требования к системе для локальных развертываний

[!include[banner](../includes/banner.md)]

В этой теме перечислены требования к системе для текущей версии Microsoft Dynamics 365 for Finance and Operations, Enterprise edition, для локальных сред. Перед установкой Finance and Operations при необходимости убедитесь, что система, с которой вы работаете, соответствует минимальным требованиям к сети, оборудованию и программному обеспечению или превышает их.

## <a name="network-requirements"></a>Требования к сети
Microsoft Dynamics 365 for Finance and Operations, Enterprise edition (локальная версия) может работать в сетях, в которых используется Internet Protocol Version 4 (IPv4) или Internet Protocol Version 6 (IPv6). Учитывайте сетевую среду при планировании системы и воспользуйтесь следующими инструкциями.

### <a name="network-response-time"></a>Время отклика сети
В следующей таблице перечислены минимальные требования к сети для связи между веб-браузером и сервером Application Object Server (AOS), а также для связи между AOS и базой данных в локальной системе.

| Стоимость     | Веб-браузер к AOS                      | AOS к базе данных |
|-----------|-----------------------------------------|------------------------------------------------------------------------------------------|
| Пропускная способность. | 50 Кбайт в секунду (Кбит/с) для каждого пользователя | 100 МБ в секунду (Мбит/с) |
| Задержка   | Менее 250 – 300 миллисекунд (мс)     | Менее 1 мс (только локальная сеть). AOS и база данных должны быть размещены совместно. |

- Приложение Finance and Operations (локальная версия) предназначено для сетей с задержкой не более 250-300 мс. Это задержка от клиента браузера до центра данных, на котором размещено приложение Finance and Operations.
- Требования к пропускной способности для Finance and Operations (локальная версия) зависят от конкретного сценария. Типичные сценарии требуют пропускную способность более 50 КБ/с между браузером и сервером Finance and Operations. Однако для сценариев с высокими требованиями к полезной нагрузке, например при использовании рабочих областей или широкой индивидуальной настройке, рекомендуется большая пропускная способность. Конкретная пропускная способность зависит от использования.

Не поддерживаются развертывания, в которых AOS и база данных Microsoft SQL Server находятся в разных центрах данных. AOS и база данных SQL Server должны быть размещены совместно. 

Как правило, приложение Finance and Operations оптимизировано для снижения количества обращений между браузером и сервером. Число обращений клиента браузера к центру данных равно нулю или единице для каждого взаимодействия, и вся полезная нагрузка сжимается.

> [!WARNING]
> Не вычисляйте требования к пропускной способности из местоположения клиента путем умножения числа пользователей на минимальные требования к пропускной способности. Параллельное использование для конкретного местоположения очень сложно рассчитать. Рекомендуется использовать реальное моделирование в непроизводственной среде Finance and Operations для оптимальной оценки производительности в каждом конкретном случае. 

### <a name="lan-environments"></a>Среды LAN
В средах на базе локальной сети (LAN) удаленный рабочий стол (Майкрософт) в Microsoft Windows Server для подключения к Finance and Operations не требуется. Тем не менее, может он может потребоваться для операций обслуживания на виртуальных машинах, составляющих развертывания серверов.

### <a name="wan-environments"></a>Среды WAN
В средах на базе глобальной сети (WAN) удаленный рабочий стол в Windows Server не требуется для подключения к Finance and Operations.

### <a name="internet-connectivity-requirements"></a>Требования подключения к Интернету
Finance and Operations (локальная версия) не требует наличия подключения к Интернету от рабочих станций конечных пользователей. Тем не менее, некоторые функции без подключения к Интернету будут недоступны.

|                    |   |
|--------------------|---|
| **Клиент браузера** | Сценарий интрасети без подключения к Интернету является расчетной точкой для варианта локального развертывания. Некоторые функции, которые требуют облачные службы, будут недоступны, например справка и библиотеки проводников по задачам в Microsoft Dynamics Lifecycle Services (LCS). |
| **Сервер**         | Уровень AOS или Microsoft Azure Service Fabric должен иметь возможность обмена данными с LCS. Локальному браузерному клиенту доступ к Интернету не требуется. |
| **Телеметрия**      | Данные телеметрии могут быть потеряны в случае длительного отсутствия подключения. Перерывы в подключении к LCS не влияют на функциональность локального приложения. |
| **LCS**            | Подключение к LCS является обязательным для развертывания, развертывание кода и операций по обслуживанию. |

## <a name="telemetry-data-transfer-to-the-cloud"></a>Перемещение данных телеметрии в облако
Большинство данных телеметрии хранится локально, и доступ к ним можно получить с помощью средства просмотра событий в Microsoft Windows. Небольшое подмножество событий телеметрии переносится в конвейер телеметрии Майкрософт в облаке для диагностики. Данные клиентов и идентифицируемые данные конечных пользователей не входят в состав данных телеметрии, отправляемых в корпорацию Майкрософт. Имена виртуальных машин отправляются в Майкрософт для использования в управлении средой и диагностике на портале LCS.

## <a name="domain-requirements"></a>Требования к домену
Учитывайте следующие требования к домену во время установки Finance and Operations (локальная версия):

- Виртуальные компьютеры, на которых размещаются компоненты Finance and Operations (локальная версия), должны принадлежать к домену Active Directory. Доменные службы Active Directory (AD DS) должны быть настроены в собственном режиме.
- Виртуальные компьютеры, на которых выполняются компоненты Finance and Operations (локальная версия), должны иметь доступ друг к другу. Этот доступ настраивается в доменных службах Active Directory. 
- Контроллером домена должен быть Microsoft  Windows Server 2012 R2 или более поздней версии с функциональным уровнем домена 2012 R2 или выше.

## <a name="hardware-requirements"></a>Требования к оборудованию
В этом разделе описывается оборудование, необходимое для работы Finance and Operations (локальная версия).

В зависимости от конфигурации системы, состава данных, а также приложений и компонентов, которые вы решите использовать, фактические требования к оборудования могут быть иными. Вот несколько факторов, которые могут повлиять на выбор подходящего оборудования для Finance and Operations (локальная версия):

- Количество проводок в час
- Количество пользователей, которые работают одновременно

## <a name="minimum-infrastructure-requirements"></a>Минимальные требования к инфраструктуре

SQL Server необходимо настроить в высокодоступной настройке HADRON, которая имеет по крайней мере два узла для производственного использования.

## <a name="storage"></a>Хранение

- **AOS** — Finance and Operations (локальная версия) будет использовать файловый ресурс общего доступа Server Message Block (SMB) версии 3.0 для хранения неструктурированных данных. Дополнительные сведения см. в разделе [Локальные дисковые пространства в Windows Server 2016](/windows-server/storage/storage-spaces/storage-spaces-direct-overview).
- **SQL** — возможны следующие варианты:

    - Высокодоступная система на базе SSD
    - Сеть хранения данных, оптимизированная для оперативной транзакционной обработки данных
    - Высокопроизводительное непосредственно подключенное хранилище (DAS) 

- **IOPS SQL Server и управления данными** — хранилище для управления данными и SQL Server должно обеспечивать хотя бы 2 000 операций ввода/вывода в секунду. Производственные IOPS зависят от множества факторов. Дополнительные сведения см. в разделе [Определение параметров оборудования для локальных сред](hardware-sizing-on-premises-environments.md).
- **IOPS виртуальных машин** — на каждой виртуальной машине число операций ввода-вывода в секунду на запись должно составлять не менее 100.

## <a name="virtual-host-requirements"></a>Требования к виртуальному узлу
При настройке виртуальных узлов для среды Finance and Operations (локальная версия) см. следующие рекомендации: [Планирование и подготовка кластера Service Fabric](/azure/service-fabric/service-fabric-cluster-standalone-deployment-preparation) и [Описание кластера Service Fabric](/azure/service-fabric/service-fabric-cluster-resource-manager-cluster-description). Каждый виртуальный узел должен иметь достаточно ядер для инфраструктуры, размер которой изменяется. Несколько расширенных конфигураций возможны там, где SQL Server находится на физическом оборудовании, но все остальные компоненты являются виртуальными. Если SQL Server вируализован, подсистема диска должна быть быстрой SAN или эквивалентной. Во всех случаях убедитесь в том, что основная настройка виртуального узла является высокодоступной и избыточной. Во всех случаях при использовании виртуализации не следует делать снимки виртуальных машин.

## <a name="software-requirements-for-database-servers"></a>Требования к программному обеспечению для серверов баз данных

- Поддерживаются только 64-разрядные версии SQL Server 2016.
- В производственной среде рекомендуется установить последнее накопительное обновление для версии SQL Server, которую вы используете.
- Finance and Operations (локальная версия) поддерживает сопоставления в формате Юникода без учета регистра, с учетом диакритических знаков, каны и ширины. Сопоставление должно соответствовать языку Windows компьютеров, на которых запущены экземпляры AOS. При настройке новой установки рекомендуется выбрать сопоставление Windows вместо сопоставления SQL Server. Дополнительные сведения о выборе сопоставления для базы данных SQL Server см. в [документации по SQL Server](/sql/sql-server/sql-server-technical-documentation).

-   Для просмотра документов, созданных с помощью функции экспорта в Excel или экспорта в Word должен быть установлен пакет Microsoft Office 2007 или более поздняя версия.
