# <a name="enter-payroll-beginning-balances"></a>Ввод начальных сальдо зарплат

[!INCLUDE [banner](../../includes/banner.md)]

В разделе описываются шаги, необходимые для ввода начальных сальдо для кодов дохода, вычетов, льгот и налогов. Эта информация полезна для партнеров, которые выполняют перенос данных для новой реализации зарплаты из другой системы. Чтобы подготовиться к вводу начальных сальдо заработной платы, мы проверяем следующее:

> * Записи сотрудников введены и доступны в системе
> * Следующие После данные настроены и назначены сотрудникам:
> 
> > * Циклы оплаты и платежные периоды
> > * Коды прибыли
> > * Налоги
> > * Льготы и вычеты
> 
> * Компании должны выбрать дату, где можно настроить начальные сальдо заработной платы.
> 
> * Сведения были собраны по всем доходам, льготами или вычетами, льготным вкладам, налогам сотрудников и налогам работодателя и их суммы с начала года из старой системы.

Перед вводом начальных сальдо решите, насколько подробные данные должны быть. Большинство компаний вводят единую, консолидированную сумму с начала года. Однако если требуется более подробная информация, сальдо можно вводить с шагом по кварталам. Для каждого сотрудника необходимо выбрать уровень детализации, который определит количество выписок об оплате, создаваемых вручную. Для единой суммы с начала года для сотрудника требуется только одна выписка по оплате. Для этого используйте суммы с начала года из окончательной выписки по оплате из предыдущей системы как сумму, введенную в новую систему учета зарплаты.

В следующем примере показано, каким образом вводятся начальные сальдо зарплаты сотрудников, включая коды дохода, льготы или вычеты и налоги. В примере реальной ситуации у вас будет элемент строки для каждого кода дохода, льготного вычета, льготного вклада, налога сотрудника и налога работодателя с введенной суммой, которая будет являться суммой с начала года. С помощью этого списка кодов и сумм выполните действия по созданию выписки до доходах и оплате вручную с отключенным учетом для начальных сальдо для целей зарплаты.  Учет отключается, поскольку не требуется выполнять разноску этой выписки по оплате для начального сальдо в главную книгу. Эта делалось в старой системе и перейдет в новую систему при настройке начальных сальдо в главной книге.

### <a name="a-how-to-set-up-earnings-codes-to-be-used-on-payroll-beginning-balances"></a>A. Порядок настройки кодов доходов для использования на начальных сальдо зарплаты
При вводе начальных сальдо заработной платы убедитесь, что используемые коды дохода настроены с включенным параметром "Разрешить изменение ставок выписки по оплате". Это позволит вручную ввести сумму из старой системы. 

### <a name="b-create-earnings-statement-for-an-employee-to-have-a-beginning-balance"></a>B. Создание выписки о доходах для сотрудника для начального сальдо
Этот шаг вручную создает выписку о доходах для каждого работника за последний период оплаты старой системы, которая создает строки выписки о доходах в новой системе расчета зарплаты. Введите одну строку на код дохода, сумму с начала года и часы. Операции из примера:

1. Откройте страницу **Выписки о всех доходах** и нажмите **Создать**.  

Введите следующее: 

| Поле      | Стоимость                 |
|------------|-----------------------|
| Рабочий     | Майкл Редмонд       |
| Цикл оплаты  | sm                    |
| Платежный период | 6.16.2017 - 6.30.2017 |

2. На вкладке **Строка выписки о доходах** введите следующее:

Строка 1: вкладка **Строка выписки о доходах**

| Поле            | Стоимость       |
|------------------|-------------|
| Код доходов    | Обычная оплата |
| Количество         | 1.00        |
| Rage             | 30 000      |
| Вкладка "Сведения по строке" |             |
| Руководство           | (отмечено)    |

Строка 2: вкладка **Строка выписки о доходах**

| Поле            | Стоимость    |
|------------------|----------|
| Код доходов    | Премия    |
| Количество         | 1.0000   |
| По норме             | 4250.00  |
| Вкладка "Сведения по строке" |          |
| Руководство           | (отмечено) |

Строка 3: вкладка **Строка выписки о доходах**

| Поле           | Стоимость      |
|-----------------|------------|
| Код доходов   | Комиссия |
| Количество        | 1.0000     |
| По норме            | !,299.00   |
| По норме            | 1,299.00   |
| Вкладка "Сведения по строке" |            |
| Руководство          | (отмечено)   |

> [!NOTE]
> Установка ползунка **Вручную** на значение **Да** на вкладке **Сведения по строке** для каждой строки выписки о доходах требуется для того, чтобы начальные сальдо зарплаты были введены для каждого работника.

3. В области **Действия** нажмите **Запустить выписку о доходах** USA-FED-ER-FICA.

4. В области **Действия** щелкните **Выписка по оплате** для открытия страницы **Создать выписки по оплате** и настройте следующее:

| Поле              | Стоимость     |
|--------------------|-----------|
| Дата оплаты       | 6.30.2017 |
| Тип выполнения платежа   | Руководство    |
| Отключить учет |   Да     |

> [!NOTE] 
> Это доступно, только если тип выполнения платежа — "вручную" и если пользователь хочет отключить учет на период оплаты.

Щелкните **ОК** и закройте **Infolog**.

#### <a name="why-the-disable-accounting-slider-needs-to-set-to-yes-when-generating-pay-statements"></a>Почему ползунок "Отключить учет" необходимо установить на значение "Да" при создании выписок по оплате?
Установка ползунка на значение **Да** предотвращает распределение строк в выписке по оплате в главную книгу. Суммы главной книги были обновлены ранее при вводе сальдо из старой системы. Ввод начальных сальдо для заработной платы позволяет создавать отчеты, которые включают сведения из предыдущего года, а также для определения ограничений для льгот и в налоговых целях.   

### <a name="c-create-pay-statements-for-employees"></a>C. Создание выписок по оплате для сотрудников
После создания выписок по оплате с начальными сальдо необходимо проверить, что выписки по оплате точно отражают данные о заработной плате. Необходимо также вручную обновить сведения о льготах и налогах, чтобы они соответствовали значения в предыдущей системе зарплаты. Убедившись, что суммы из предыдущей системы заработной платы соответствуют суммам по текущим выпискам по оплате, необходимо финализировать выписки по оплате.

1. Откройте страницу **Все выписки по оплате**.

2. Выделите для последнюю созданную выписку по оплате для Майкла Редмонда

3. Щелкните **Изменит ь**, чтобы открыть страницу **Выписка по оплате**.

4. Откройте вкладку **Льготные вычеты** и введите следующее:

|       Поле       |      Стоимость       |
|-------------------|------------------|
|      Льгота      | Сумма вычета |
|       401K        |   Участие    |
|      Dental       |      SubSp       |
| Расходы на вуз care |   Участие    |
|      Vision       |      SupSp       |

5. На вкладке **Льготные вклады** и введите следующее:

|  Поле  |        Стоимость        |
|---------|---------------------|
| Льгота | Сумма вклада |
|  401K   |     Участие     |
| Dental  |        SubSp        |
| Vision  |        SubSp        |

6. На вкладке **Налоговые вычеты** введите следующее:

| Поле           | Стоимость            |
|-----------------|------------------|
| Код налога        | Сумма вычета |
| USA-FED-ER-FICA | 1600.00          |
| USA-FED-ER-MEDI | 825.75           |

7. На вкладке **Налоговые вклады** введите следующее:

8. Щелкните **Рассчитать**.
   > [!IMPORTANT] 
   > Проверьте итоги в выписке по оплате,чтобы они соответствовали значениям с начала года в старой системе для работника. Можно не торопиться с финализацией на следующем шаге, чтобы выполнить общую проверку всех выписок по оплате в совокупности. После проверки просмотрите все выписки по оплате и финализируйте их.

При необходимости такой же процесс может выполняться с шагом по кварталу для всех предыдущих кварталов каждого года. Это необходим только в том случае, если клиенту нужно просмотреть данные по кварталам без возвращения к старой системе.

## <a name="if-you-make-a-mistake-entering-beginning-balances-for-an-employee"></a>Если была допущена ошибка при вводе начальных сальдо для сотрудника
Имеется возможность отменить ввод и повторить проводки. Чтобы отменить проводку, необходимо выполнить следующие действия на странице **Все выписки по оплате**.

1. Щелкните **Отменить**.

2. Щелкните **Да** при появлении сообщения "При сторнировании этой выписки по оплате будет создана сторнирующая выписка по оплате, корреспондирующая с данной выпиской. Ни одну из выписок по оплате нельзя будет изменить. Вы действительно хотите сторнировать эту выписку по оплате? . 

После реверсирования выписки по оплате можно создать новую выписку по оплате для работника из выписке о доходах, созданной ранее. Не забудьте исправить все неправильные строки в выписке о доходах перед созданием новой выписки по оплате, а затем создайте новую выписку по оплате с правильными суммами.
