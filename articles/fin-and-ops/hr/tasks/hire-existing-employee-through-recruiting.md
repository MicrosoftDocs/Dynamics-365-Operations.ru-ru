[!include [task guide banner](../../includes/task-guide-banner.md)]

<span data-ttu-id="e3590-104">Иногда открытые позиции могут заполняться кандидатами, которые уже являются сотрудниками организации.</span><span class="sxs-lookup"><span data-stu-id="e3590-104">Sometimes open positions can be filled by candidates who are already employees in your organization.</span></span> <span data-ttu-id="e3590-105">Эта процедура демонстрирует шаги приема на работу существующего сотрудника посредством процесса набора сотрудников.</span><span class="sxs-lookup"><span data-stu-id="e3590-105">This procedure walks through the steps of hiring an existing employee through the recruiting process.</span></span> <span data-ttu-id="e3590-106">В этой процедуре проект по набору сотрудников уже настроен, и существующий сотрудник уже подал заявление в проект по набору сотрудников.</span><span class="sxs-lookup"><span data-stu-id="e3590-106">In this procedure, a recruitment project has already been set up, and an existing employee has already submitted an application for the recruitment project.</span></span> <span data-ttu-id="e3590-107">В качестве компании с демонстрационными данными для создания этой процедуры используется USMF.</span><span class="sxs-lookup"><span data-stu-id="e3590-107">The demo data company used to create this procedure is USMF.</span></span>

1. <span data-ttu-id="e3590-108">Перейдите в раздел "Управление персоналом" > "Набор сотрудников" > "Заявления" > "Заявления".</span><span class="sxs-lookup"><span data-stu-id="e3590-108">Go to Human resources > Recruitment > Applications > Applications.</span></span>

3. <span data-ttu-id="e3590-111">Щелкните "Статус заявления".</span><span class="sxs-lookup"><span data-stu-id="e3590-111">Click Application status.</span></span>
    * <span data-ttu-id="e3590-112">Статус заявления показывает ход процесса приема на работу.</span><span class="sxs-lookup"><span data-stu-id="e3590-112">The application status indicates where an application is at in the recruitment process.</span></span>  <span data-ttu-id="e3590-113">Каждый из следующих шагов является необязательным.</span><span class="sxs-lookup"><span data-stu-id="e3590-113">Each of these steps is optional.</span></span> <span data-ttu-id="e3590-114">Как правило, статусы заявки меняются в следующем порядке: получено, подтверждено, проведено собеседование.</span><span class="sxs-lookup"><span data-stu-id="e3590-114">Typically, an application would move the statuses in the following order:  received, confirmed, and interviewed.</span></span> <span data-ttu-id="e3590-115">После собеседования принимается решение о приеме на работу.</span><span class="sxs-lookup"><span data-stu-id="e3590-115">After the interview process, a hiring decision would be made.</span></span>  
4. <span data-ttu-id="e3590-116">Щелкните "Изменить должность".</span><span class="sxs-lookup"><span data-stu-id="e3590-116">Click Change position.</span></span>
5. <span data-ttu-id="e3590-117">Выберите должность, на которую нанимается сотрудник.</span><span class="sxs-lookup"><span data-stu-id="e3590-117">Select the position that you are hiring the employee into.</span></span>
6. <span data-ttu-id="e3590-118">В поле "Назначить дату начала" введите дату, с которой сотрудник начнет работать на новой должности.</span><span class="sxs-lookup"><span data-stu-id="e3590-118">In the New Assignment Start Date field, enter the date that the employee will begin working in the new position.</span></span>  
7. <span data-ttu-id="e3590-119">В поле "Назначить дату окончания" введите дату, с которой сотрудник начнет работать на новой должности.</span><span class="sxs-lookup"><span data-stu-id="e3590-119">In the Assignment end date, enter the date that the employee will stop working in their current position.</span></span>
    * <span data-ttu-id="e3590-120">Начальная дата для новой должности и конечная дата старой должности могут перекрываться.</span><span class="sxs-lookup"><span data-stu-id="e3590-120">The starting date for the new position and the ending date of the old position may overlap.</span></span> <span data-ttu-id="e3590-121">Это может произойти в случае, когда в течение переходного периода человек выполняет обязанности обеих должностей.</span><span class="sxs-lookup"><span data-stu-id="e3590-121">This can happen when a person is performing duties for both positions during a transition period.</span></span>

9. <span data-ttu-id="e3590-124">Щелкните "Изменить должность".</span><span class="sxs-lookup"><span data-stu-id="e3590-124">Click Change position.</span></span>
    * <span data-ttu-id="e3590-125">В это время также можно изменить компенсацию.</span><span class="sxs-lookup"><span data-stu-id="e3590-125">You may also change compensation at this time.</span></span> <span data-ttu-id="e3590-126">Если не назначать компенсацию в это время, изменить ее можно, перейдя в форму работника и выбрав "Фиксированный план" на вкладке "Компенсация".</span><span class="sxs-lookup"><span data-stu-id="e3590-126">If you do not assign compensation at this time, you can change it by going to the worker form, selecting the Compensation tab, and choosing 'Fixed Plan'.</span></span> <span data-ttu-id="e3590-127">После нажатия кнопки "Изменить должность" статус заявки меняется на "Принят на работу".</span><span class="sxs-lookup"><span data-stu-id="e3590-127">After you select 'Change position', the status on the application will be updated to 'Employed'.</span></span>
