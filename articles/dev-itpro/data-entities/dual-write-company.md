## <a name="company-concept-in-common-data-service"></a><span data-ttu-id="6f872-103">Концепция компании в Common Data Service</span><span class="sxs-lookup"><span data-stu-id="6f872-103">Company concept in Common Data Service</span></span>

[!include [preview](../includes/preview-banner.md)]

<span data-ttu-id="6f872-104">В Microsoft Dynamics 365 for Finance and Operations концепция *компании* является как правовой конструкцией, так и бизнес-конструкцией.</span><span class="sxs-lookup"><span data-stu-id="6f872-104">In Microsoft Dynamics 365 for Finance and Operations, the concept of a *company* is both a legal construct and a business construct.</span></span> <span data-ttu-id="6f872-105">Она также задает границу безопасности и видимости для данных.</span><span class="sxs-lookup"><span data-stu-id="6f872-105">It's also a security and visibility boundary for data.</span></span> <span data-ttu-id="6f872-106">Пользователи всегда работают в контексте одной компании, и большая часть данных ограничивается компанией.</span><span class="sxs-lookup"><span data-stu-id="6f872-106">Users always work in the context of a single company, and most of the data is striped by company.</span></span>

<span data-ttu-id="6f872-107">В Common Data Service нет эквивалентной концепции.</span><span class="sxs-lookup"><span data-stu-id="6f872-107">Common Data Service doesn't have an equivalent concept.</span></span> <span data-ttu-id="6f872-108">Ближайшей концепцией является *бизнес-подразделение*, которое в первую очередь является границей безопасности и видимости для пользовательских данных.</span><span class="sxs-lookup"><span data-stu-id="6f872-108">The closest concept is *business unit*, which is primarily a security and visibility boundary for user data.</span></span> <span data-ttu-id="6f872-109">Эта концепция не имеет тех же юридических или деловых последствий, как концепция компании в Finance and Operations.</span><span class="sxs-lookup"><span data-stu-id="6f872-109">This concept doesn't have the same legal or business implications as the company concept in Finance and Operations.</span></span>

<span data-ttu-id="6f872-110">Поскольку бизнес-подразделение и компания не являются эквивалентными понятиями, невозможно принудительно задать взаимно-однозначное (1:1) сопоставление между ними в целях интеграции Common Data Service.</span><span class="sxs-lookup"><span data-stu-id="6f872-110">Because business unit and company aren't equivalent concepts, it isn't possible to force a one-to-one (1:1) mapping between them for the purpose of Common Data Service integration.</span></span> <span data-ttu-id="6f872-111">Однако поскольку пользователи должны, по умолчанию, иметь возможность видеть одинаковые записи в Finance and Operations и Common Data Service, корпорация Майкрософт представила новую сущность в Common Data Service, которая называется cdm\_Company.</span><span class="sxs-lookup"><span data-stu-id="6f872-111">However, because users must, by default, be able to see the same records in Finance and Operations and Common Data Service, Microsoft has introduced a new entity in Common Data Service that is named cdm\_Company.</span></span> <span data-ttu-id="6f872-112">Эта сущность эквивалентна сущности компании в Finance and Operations.</span><span class="sxs-lookup"><span data-stu-id="6f872-112">This entity is equivalent to the Company entity in Finance and Operations.</span></span> <span data-ttu-id="6f872-113">Чтобы помочь гарантировать, что видимость записей эквивалентна между Finance and Operations и Common Data Service в готовом виде, рекомендуется следующая настройка данных в Common Data Service:</span><span class="sxs-lookup"><span data-stu-id="6f872-113">To help guarantee that visibility of records is equivalent between Finance and Operations and Common Data Service out of the box, we recommend the following setup for data in Common Data Service:</span></span>

+ <span data-ttu-id="6f872-114">Для каждой записи компании в Finance and Operations, которая включена для двойной записи, создается связанная запись cdm\_Company.</span><span class="sxs-lookup"><span data-stu-id="6f872-114">For each Finance and Operations Company record that is enabled for dual-write, an associated cdm\_Company record is created.</span></span>
+ <span data-ttu-id="6f872-115">Когда запись cdm\_Company создается и включается для двойной записи, создается бизнес-подразделение по умолчанию, которое имеет то же имя.</span><span class="sxs-lookup"><span data-stu-id="6f872-115">When a cdm\_Company record is created and enabled for dual-write, a default business unit is created that has the same name.</span></span> <span data-ttu-id="6f872-116">Хотя рабочая группа по умолчанию автоматически создается для этого бизнес-подразделения, бизнес-подразделение не используется.</span><span class="sxs-lookup"><span data-stu-id="6f872-116">Although a default team is automatically created for that business unit, the business unit isn't used.</span></span>
+ <span data-ttu-id="6f872-117">Создается отдельная рабочая группа владельцев с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="6f872-117">A separate owner team is created that has the same name.</span></span> <span data-ttu-id="6f872-118">Она также связана с бизнес-подразделением.</span><span class="sxs-lookup"><span data-stu-id="6f872-118">It's also associated with the business unit.</span></span>
+ <span data-ttu-id="6f872-119">По умолчанию владелец любой записи, созданной в Finance and Operations записанной для двойной записи в Common Data Service, задается для рабочей группы "DW Owner", которая связана с соответствующим бизнес-подразделением.</span><span class="sxs-lookup"><span data-stu-id="6f872-119">By default, the owner of any record that is created in Finance and Operations and dual-written to Common Data Service is set to the "DW Owner" team that is linked to the associated business unit.</span></span>
