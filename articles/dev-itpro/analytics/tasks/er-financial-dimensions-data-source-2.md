# <a name="map-models-to-use-financial-dimensions-as-data-sources"></a>Сопоставление моделей для использования финансовых аналитик как источников данных 

[!include [task guide banner](../../includes/task-guide-banner.md)]

В следующих шагах поясняется, как пользователь, которому назначена роль системного администратора или разработчика электронной отчетности, может настроить модель электронной отчетности (ER) для использования финансовых аналитик как источника данных для отчетов электронной отчетности. Эти шаги можно выполнить в любой компании.

Для выполнения этих действий необходимо сначала выполнить шаги в процедуре "Электронная отчетность — Использование финансовых аналитик как источника данных (Часть 1. Модель проектных данных)".


## <a name="add-required-data-sources-to-model-mapping"></a>Добавление необходимых источников данных в сопоставление модели
1. Перейдите в раздел "Управление организацией" > "Электронная отчетность" > "Конфигурации".
2. В дереве выберите "Пример модели финансовых аналитик".
3. Выберите Конструктор.
4. Щелкните "Сопоставить модель с источником данных".
5. Щелкните "Создать".
6. В поле "Определение" выберите "Запись".
7. В поле "Имя" введите "Сопоставление данные аналитик".
8. В поле "Описание" введите "Сопоставление данные аналитик".
9. Нажмите кнопку "Сохранить".
10. Выберите Конструктор.
11. В дереве выберите "Dynamics 365 for Operations\Таблица".
12. Щелкните "Добавить корень".
13. В поле "Имя" введите "Компания".
14. В поле "Таблица" введите "CompanyInfo".
15. Нажмите кнопку OK.
16. В дереве выберите "Функции\Сведения о финансовых аналитиках".
17. Щелкните "Добавить корень".
    * Этот источник данных определяет, как область финансовых аналитик будет определена для любого отчета, который будет использовать эту модель в качестве источника данных.  
18. В поле "Имя" введите значение.
19. Выберите "Да" в поле "Запросить аналитики".
    * Выберите "Да", чтобы разрешить пользователю выбирать аналитики во время выполнении в форме диалогового окна пользователя. Если задано значение "Нет", по умолчанию будут использоваться все финансовые аналитики текущего экземпляра.  
20. В поле "Выбор финансовых аналитик", выберите "Юридическое лицо".
    * Выберите "Все", чтобы разрешить пользователю выбирать требуемые аналитики для текущего экземпляра в поле поиска.  Выберите "Юридическое лицо", чтобы разрешить пользователю выбирать требуемые аналитики для компании в поле поиска.  Выберите "Аналитика", чтобы разрешить пользователю выбирать аналитики с использованием одного набора аналитик.  
21. Выберите "Да" в поле "Запросить счет ГК".
    * Установите для параметра "Запросить счет ГК", чтобы разрешить пользователям выбирать счет ГК как часть списка аналитик.   Если задано значение "Нет", счет ГК не будет включен в список аналитик и будет включен параметр "Счет ГК является обязательным". Если для параметра "Счет ГК является обязательным" задано значение "Да", включите счет ГК в список аналитик независимо от значения, выбранного пользователем.  
22. Нажмите кнопку "OК".
23. В дереве выберите "Dynamics 365 for Operations\Записи таблицы".
24. Щелкните "Добавить корень".
25. В поле "Имя" введите "LedgerJournal".
26. Выберите "Да" в поле "Запросить запрос".
27. В поле "Таблица" введите "LedgerJournalTable".
28. Нажмите кнопку "OК".

## <a name="map-data-model-elements-to-added-data-sources"></a>Сопоставление элементов модели данных с добавленными источниками данных
1. В дереве разверните узел "Журнал".
2. В дереве разверните узел "Журнал\Проводки".
3. В дереве разверните узел "Журнал\Проводки\Данные аналитик".
4. В дереве разверните узел "Настройка аналитик".
5. В дереве разверните узел "LedgerJournal".
6. В дереве разверните узел "LedgerJournal\<Связи".
7. В дереве разверните узел "LedgerJournal\<Связи\LedgerJournalTrans".
8. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Ваучер".
9. В дереве выберите "Журнал\Проводка\Ваучер".
10. Щелкните "Связать".
11. В дереве выберите "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)".
    * Обратите внимание, что для любой ссылки на финансовые аналитиками, которая установлена, например, на LedgerDimension, соответствующий элемент источника данных доступен (LedgerDimension.Dimension). Этот элемент источника данных предлагает финансовые аналитики набора аналитик в виде списка записей.  
12. В дереве разверните узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)".
13. В дереве разверните узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики".
14. В дереве разверните узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики\Значение".
15. В дереве разверните узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики\Определение".
16. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики\Определение\Имя".
17. В дереве выберите узел "Журнал\Проводки\Данные аналитик\Имя".
18. Щелкните "Связать".
19. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики\Значение\Описание".
20. В дереве выберите узел "Журнал\Проводки\Данные аналитик\Описание".
21. Щелкните "Связать".
22. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики\Значение\Код".
23. В дереве выберите узел "Журнал\Проводки\Данные аналитик\Код".
24. Щелкните "Связать".
25. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Account.Dimension(LedgerDimension.Dimension)\Счет ГК и аналитики".
26. В дереве выберите узел "Журнал\Проводки\Данные аналитик".
27. Щелкните "Связать".
28. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Debit(AmountCurDebit)".
29. В дереве выберите "Журнал\Проводка\Дебет".
30. Щелкните "Связать".
31. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Date(TransDate)".
32. В дереве выберите "Журнал\Проводка\Дата".
33. Щелкните "Связать".
34. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Currency(CurrencyCode)".
35. В дереве выберите "Журнал\Проводка\Валюта".
36. Щелкните "Связать".
37. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans\Credit(AmountCurCredit)".
38. В дереве выберите "Журнал\Проводка\Кредит".
39. Щелкните "Связать".
40. В дереве выберите узел "LedgerJournal\<Связи\LedgerJournalTrans".
41. В дереве выберите "Журнал\Проводка".
42. Щелкните "Связать".
43. В дереве выберите «LedgerJournal\Номер партии журнала(JournalNum)".
44. В дереве выберите "Журнал\Партия".
45. Щелкните "Связать".
46. В дереве выберите "LedgerJournal".
47. В дереве выберите "Журнал".
48. Щелкните "Связать".
49. В дереве разверните узел "Аналитики".
50. В дереве разверните узел "Аналитики\Счет ГК и аналитики".
51. В дереве разверните узел "Аналитики\Счет ГК и аналитики\Определение".
52. В дереве выберите узел "Аналитики\Счет ГК и аналитики\Определение\Имя".
53. В дереве выберите узел "Настройка аналитик\Код".
54. Щелкните "Связать".
55. В дереве выберите узел "Аналитики\Счет ГК и аналитики\Определение\Имя столбца отчета".
56. В дереве выберите узел "Настройка аналитик\Имя".
57. Щелкните "Связать".
58. В дереве выберите узел "Аналитики\Счет ГК и аналитики".
59. В дереве выберите узел "Настройка аналитик".
60. Щелкните "Связать".
61. В дереве выберите "Компания".
62. Щелкните "Изменить".
63. В поле expressionAsStringText введите "Company.'find()'.'name()'".
    * Company.'find()'.'name()'  
64. Нажмите кнопку "Сохранить".
65. Закройте страницу.
66. Нажмите кнопку "Сохранить".
67. Закройте страницу.
