# <a name="import-configurations-to-generate-documents-with-application-data"></a>Импорт конфигураций для создания документов с обновлением данных приложения

[!INCLUDE [task guide banner](../../includes/task-guide-banner.md)]

Для выполнения действий в этой процедуре необходимо сначала выполнить процедуру "Электронная отчетность — Создание поставщика конфигурации и пометка его как активного".

В этой процедуре поясняется разработка конфигураций электронной отчетности для формирования электронного документа. В этой процедуре вам предстоит импортировать необходимые конфигурации электронной отчетности, созданные для демонстрационной компании Litware, Inc., а затем использовать их для формирования электронных документов. Эта процедура предназначена для пользователей, которым назначена роль "Системный администратор" или "Разработчик электронной отчетности". Действия можно выполнять с использованием набора данных DEMF. Прежде чем начать, загрузите и сохраните файлы, перечисленные в разделе справки "Создание электронных документов и обновление данных в приложении с помощью электронной отчетности" (generate-electronic-documents-update-application-data/). Эти файлы — Intrastat (model).xml, Intrastat (mapping).xml и Intrastat (format).xml.

1. Перейдите в раздел "Управление организацией" > "Рабочие области" > "Электронная отчетность".
    * Убедитесь, что поставщик конфигурации для демонстрационной компании Litware, Inc. доступен и помечен как активный. Если вы не видите этого поставщика конфигурации, необходимо сначала выполнить шаги в процедуре "Создание поставщика конфигурации и пометка его как активного".  
    * Шаги в этой процедуре показывают, как использовать возможности электронной отчетности для выполнения обновления данных приложения и формирования отчета Интрастат. Сведения о процессе составления отчетности архивируются в таблицах приложения. В настоящее время при активации процесса составления отчетности из формы Интрастат архивация осуществляется на основе логики, запрограммированной в существующем исходном коде. В этой процедуре вам предстоит настроить аналогичную, однако упрощенную логику данных приложения использованием исключительно платформы электронной отчетности. Изменения в исходный код вноситься не будут.   

## <a name="import-er-configurations"></a>Импорт конфигураций электронной отчетности
1. Щелкните "Конфигурации отчетности".
2. Щелкните "Обменять".
3. Щелкните "Загрузить из XML-файла".
    * Импортируйте конфигурацию модели электронной отчетности, содержащую модель данных, предназначенную для использования в качестве источника данных для формирования отчета Интрастат. Позднее вам предстоит расширить это определение модели данных, чтобы использовать его для обновления данных приложения для архивирования сведения о процессе составления отчетности Интрастат.   
    * Щелкните "Обзор" и выберите файл Intrastat (model).xml.  
4. Нажмите кнопку "OК".
5. В дереве выберите "Intrastat (model)".
6. Выберите Конструктор.
7. В дереве разверните "Для исходящего документа".
8. В дереве разверните "Для исходящего документа\Проводки".
    * Просмотрите структуру импортированной модели данных. Обратите внимание, что корневой элемент "Для исходящего документа" определен для задания потока данных, обеспечивающего получение данных из приложения и использование его в качестве источника данных для формирования отчета Интрастат. "Проводки (список записей)" используется для представления списка проводок Интрастат, которые должны быть включены в отчет. Поскольку вы будете архивировать включенные в отчет коды товаров, в этом потоке необходим уникальный идентификатор — код отдельного товара "Код записи товара (Int64)".   
9. Закройте страницу.
10. Щелкните "Обменять".
11. Щелкните "Загрузить из XML-файла".
    * Импортируйте конфигурацию сопоставлений электронной отчетности, которая задает поток данных для получения данных из приложения с последующим их использованием для формирования отчета Интрастат. Позднее вам предстоит расширить это определение сопоставления модели, чтобы получать данные из отчета Интрастат и использовать их для обновления данных приложения для архивирования сведений о процессе составления отчетности Интрастат.   
    * Щелкните "Обзор" и выберите файл Intrastat (mapping).xml.  
12. Нажмите кнопку "OК".
13. В дереве разверните узел "Intrastat (model)".
14. В дереве выберите "Intrastat (model)\Intrastat (mapping)".
15. Выберите Конструктор.
    * Обратите внимание, что текущее сопоставление модели содержит значение «В модель» в поле "Направление". Это означает, что это сопоставление модели был разработано для получения данных из приложения и сохранения их в модели данных.  
16. Выберите Конструктор.
17. В дереве разверните "Список".
18. В дереве разверните "Проводки = Список".
    * Просмотрите структуру модели сопоставления, в которой используется модель данных, фильтруемая на основании корневого элемента "Для исходящего документа". Обратите внимание, что добавленный источник данных "Список" обеспечивает доступ к необходимым данным приложения, которые представляют собой список записей из таблицы Интрастат..  
19. Закройте страницу.
20. Закройте страницу.
21. Щелкните "Обменять".
22. Щелкните "Загрузить из XML-файла".
    * Импортируйте конфигурацию формата электронной отчетности, которая задает макет отчета Интрастат и процесс заполнения отчета данными. Позднее вам предстоит расширить это определение формата, чтобы помещать данные из отчета Интрастат в модель данных, а затем использовать их для обновления данных приложения для архивирования сведений о процессе составления отчетности Интрастат.   
    * Щелкните "Обзор" и выберите файл Intrastat (format).xml.  
23. Нажмите кнопку "OК".
24. В дереве выберите "Intrastat (model)\Intrastat (format)".
25. Выберите Конструктор.
26. Щелкните "Развернуть/свернуть".
27. В дереве выберите "Файл\Декларация".
28. Перейдите на вкладку "Сопоставление".
29. В дереве выберите "Файл".
    * Просмотрите структуру формата, используемого для формирования отчета Интрастат. Обратите внимание, что он предназначен для формирования XML-файла путем заполнения его данными из модели данных, которая основана на корневом элементе "Для исходящего документа". Убедитесь, что имя для формируемого файла определяется в форме пользовательского диалогового окна (для этого используется источник данных "fn").   
30. Закройте страницу.
