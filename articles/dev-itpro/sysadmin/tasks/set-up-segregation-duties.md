# <a name="set-up-segregation-of-duties"></a>Настройка разделения обязанностей

[!include [task guide banner](../../includes/task-guide-banner.md)]

Можно настроить правила для разделения задач, которые должны выполняться разными пользователями. Это называется разделением обязанностей. Например, вряд ли имеет смысл, чтобы один и тот же человек оформлял получение товаров и обрабатывал платеж поставщику. Разделение обязанностей помогает снизить риск мошенничества, а также помогает выявлять ошибки или нарушения. Разделение обязанностей также можно использовать для реализации политик внутреннего контроля. Выполните следующую процедуру, чтобы создать правило. Для выполнения процедуры необходимо быть системным администратором. В качестве компании с демонстрационными данными для создания этой процедуры используется DAT. 

1. Перейдите в раздел "Администрирование системы" > "Контроль доступа" > "Разделение обязанностей" > "Правила разделения обязанностей".
2. Щелкните "Создать".
3. В поле "Имя" введите значение.
    * Введите имя для правила.  
4. В поле "Первая обязанность" нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
5. В списке найдите и выберите требуемую запись.
    * Выберите первое полномочие, управляемое правилом.  
6. В списке перейдите по ссылке в выбранной строке.
7. В поле "Вторая обязанность" нажмите кнопку раскрывающегося списка, чтобы открыть поиск.
8. В списке найдите и выберите требуемую запись.
    * Выберите второе полномочие, управляемое правилом.  
9. В списке перейдите по ссылке в выбранной строке.
10. В поле "Строгость" выберите один из вариантов.
    * Выберите серьезность риска, который возникает, когда один и тот же пользователь или роль имеют оба полномочия.  
11. В поле "Угроза безопасности" введите значение.
    * Введите описание риска для безопасности.  
12. В поле "Снижение риска безопасности" введите значение.
    * Введите описание действий, предпринимаемых для снижения риска безопасности. Например, для снижения риска можно проводить более подробные проверки процесса, проводить ежемесячную административную проверку или использовать ресурсы совместно с другими подразделениями.  
13. Нажмите кнопку "Сохранить".
