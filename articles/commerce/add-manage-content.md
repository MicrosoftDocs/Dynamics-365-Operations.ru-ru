# <a name="ways-to-add-content"></a>Способы добавления содержимого

[!include [banner](includes/banner.md)]

В этой теме представлен обзор и ссылки на документацию с описанием, как управлять содержимым с помощью набора инструментов веб-разработки сайтов Microsoft Dynamics 365 Commerce.

## <a name="overview"></a>Обзор

Есть много способов изменить вид, поведение и содержание сайта. В зависимости от требуемого уровня настройки многие из этих изменений могут быть реализованы не только разработчиками в построителе сайтов, в составе набора средств веб-разработки Dynamics 365 Commerce. Построитель сайтов позволяет создавать шаблоны, выбирать темы и выбирать и настраивать модули без написания кода. С другой стороны, навыки разработки необходимы для создания новой темы или модуля, так как необходимо использовать пакет средств разработки программного обеспечения (SDK) электронной коммерции и рабочий процесс развертывания Microsoft Dynamics Lifecycle Services (LCS).

Следующие разделы являются хорошими точками перехода, чтобы начать понимать способы добавления и управления содержимым сайта. В большинстве тем рассказывается об областях сайта, которым не требуется разработчик. Некоторые описывают изменения общего содержимого, в то время как другие описывают задачи администратора сайта. Каждый из этих разделов содержит задачи, для которых может потребоваться SDK. В каждом разделе предполагается, что уже был подготовлен сайт и предоставлен доступ к набору средств построителя сайтов для вашего сайта.

Выберите один из следующих разделов, чтобы начать работу.

- Чтобы ознакомиться с терминологией управления контентом, используемой в построителе сайтов и в этой документации, см. [Глоссарий по модели страниц](page-elements-overview.md).
- Чтобы понять, как работают модули в workflow-процессах управления контентом, см. раздел [Работа с модулями](work-with-modules.md).
- Чтобы изменить текст, изображения или видео на существующей странице сайта, см. раздел [Работа с модулями](work-with-modules.md).
- Чтобы увидеть, как фрагменты могут сделать управление контентом более эффективным и гибким, см. [Работа с фрагментами](work-with-fragments.md).
- Чтобы обеспечить успешное создание средств торговой марки для авторов веб-контента, см. [Обзор шаблонов и макетов](templates-layouts-overview.md) и [Работа с шаблонами](work-with-templates.md).
- Чтобы изменить расположение разделов на странице сайта, см. раздел [Работа с макетами](work-with-layouts.md).
- Чтобы изменить шрифты, цвета и общий вид страниц сайта, см. раздел [Выбор темы сайта](select-site-theme.md) или [Работа с переопределением файлов CSS](css-override-files.md).
- Сведения о том, как изменить порядок или добавить новые параметры навигации,см. в разделе [Настройка навигации по сайту](customize-site-navigation.md).
- Сведения о том, как создавать, просматривать и публиковать широкий набор одновременных изменений веб-контента см. [Работа с группами публикаций](publish-groups.md).
