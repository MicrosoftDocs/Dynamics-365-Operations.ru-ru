# <a name="return-items-across-multiple-customer-orders-and-invoices"></a>Возврат номенклатур по нескольким заказам и накладным клиента

[!include [banner](includes/banner.md)]


В этой статье описываются две функции, которые оптимизируют возвраты по заказам клиентов по нескольким накладным. 

## <a name="enable-refunds-over-multiple-captures"></a>Включить возвраты денежных средств по нескольким фиксациям

Эта функция активирует несколько связанных возвратов денежных средств для одного и того же заказа клиента. 

1. Перейдите в рабочую область **Управление функциями** и найдите функцию **Включить возвраты денежных средств по нескольким фиксациям**.
2. Выберите **Включить возвраты денежных средств по нескольким заказам**, затем нажмите **Включить**. 

## <a name="enable-proper-tax-calculation-for-returns-with-partial-quantity"></a>Включить надлежащий расчет налога для возвратов с частичным количеством

Эта функция гарантирует, что при возврате заказа с использованием нескольких накладных налоги в конечном итоге будут равняться первоначальной начисленной сумме налога. 

1. Перейдите к рабочей области **Управление функциями** и найдите функцию **Включить надлежащий расчет налога для возвратов с частичным количеством**.
2. Выберите **Включить надлежащий расчет налога для возвратов с частичным количеством**, затем нажмите **Включить**. 


## <a name="process-returns"></a>Обработка возвратов

После включения этих функций и синхронизации изменений с магазинами кассир в магазине может выбрать для клиента несколько заказов на продажу, чтобы обработать возврат.

При выборе заказов отображается список всех доступных для возврата продуктов по всем накладным для этих заказов. Кассир может затем выбрать возвращаемые продукты. Создается один заказ на возврат для всех выбранных продуктов.

Если заказ полностью возвращен, сумма налогов, возвращенная клиенту, будет равна исходной сумме начисленного налога.
