# <a name="product-collection-modules"></a>Модули семейства продуктов  

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

В этом разделе представлен обзор модулей семейства продуктов в Microsoft Dynamics 365 Commerce.

## <a name="overview"></a>Обзор

Обнаружение продукта — это основное средство, используемое розничными компаниями для работы с клиентами на веб-сайте электронной коммерции. Модули семейства продуктов помогают предприятиям розничной торговли создавать привлекательные возможности для покупок, представляя интуитивно понятный визуальный интерфейс, который может использоваться для быстрой разработки семейства продуктов.

Модули семейства продуктов представляют физические продукты и услуги на веб-сайте. Модуль семейства продуктов обычно связан со страницей сведений, где клиенты могут купить продукт или услугу, или получить дополнительные сведения о ней. 

Источниками для семейств продуктов могут быть списки трех типов:

- Редакционные списки продуктов, которые вручную определяются в Dynamics 365 Retail в качестве связанных продуктов для продукта или списков продуктов
- Алгоритмические списки, такие как списки новых, лучше всего продаваемых их трендовых продуктов
- Списки рекомендаций на основе машинного обучения

На следующем рисунке показаны различные типы семейств продуктов, используемых на сайте электронной коммерции.

![Пример различных типов коллекций продуктов на сайте электронной коммерции](./media/ProductCollectionsAcrossTheSiteUseProductPlacement.png)

> [!NOTE]
> Для отображения группы продуктов подобного типа или темы обязательно следует использовать модуль семейств продуктов

## <a name="product-collection-modules-and-types"></a>Модули и типы семейства продуктов

В следующей таблице описываются различные типы модулей семейств продуктов в Dynamics 365 Commerce.

| Модуль семейства продуктов  | Вид | Описание |
|----------------------------|------|-------------|
| Просмотр по категориям            | Редакция | В этом типе модуля семейства продуктов используется иерархия категорий навигации, которую розничный продавец создал для канала розничной торговли для отображения потока просмотра продуктов, предлагаемых в определенной категории сайта. |
| Результаты поиска             | Поисковый запрос | В этом типе модуля семейства продуктов отображается список продуктов, которые лучше всего соответствуют поисковому запросу, введенному клиентом. |
| Связанные продукты           | Редакция | В этом типе модуля семейства продуктов отображается список продуктов, которые менеджер по продажам настроил в качестве связанных продуктов в Retail, для типа отношений, выбранного автором. |
| Списки подобранных продуктов      | Редакция | В этом типе модуля семейства продуктов отображаются списки, которые менеджеры по продажам и редакторы создали в Retail. |
| Сoздать                        | Алгоритмический | В этом типе модуля семейства продуктов отображается список новейших продуктов, которые были добавлены в каналы и каталоги. |
| Лидеры продаж               | Алгоритмический | В этом типе модуля семейства продуктов отображается список продуктов, которые ранжированы по наибольшему количеству продаж. |
| Популярные                   | Алгоритмический | В этом типе модуля семейства продуктов отображается список самых эффективных продуктов для данного периода. |
| Часто покупают вместе | Искусственный интеллект или машинное обучение | Этот тип модуля семейства продуктов использует машинное обучение для анализа шаблонов покупок потребителей и рекомендует связанные номенклатуры, которые часто приобретаются вместе с данным продуктом. |
| Людям также нравится           | Искусственный интеллект или машинное обучение | Этот тип модуля семейства продуктов использует машинное обучение для анализа шаблонов покупок потребителей и рекомендует номенклатуры, связанные с определенным продуктом. |

## <a name="add-a-product-collection-module-to-a-category-page"></a>Добавление модуля семейства продуктов на страницу категории

Чтобы добавить модуль семейства продуктов на страницу категории, выполните следующие действия.

1. В Dynamics 365 Commerce перейдите на сайт и создайте страницу, использующую тот же шаблон, что и страница категории по умолчанию.
1. В структуре страницы выберите слот **Вложенный нижний колонтитул**, нажмите кнопку с многоточием (**...**), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите **Контейнер**, затем выберите **ОК**.
1. В модуле-контейнере выберите кнопку с многоточием (...), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите **Семейство продуктов**, затем выберите **ОК**.
1. Настройте параметры, выбрав соответствующий источник данных и входы для семейства продуктов.
1. В области свойства для модуля семейства продуктов выберите **Добавить список продуктов**.
1. В диалоговом окне **Выбор конфигурации списка продуктов** выберите тип списка, введите число номенклатур и выберите любые другие параметры, доступные для данного типа списка. Дополнительные сведения о типах списков см. в приведенной ниже таблице. 
1. Нажмите **ОК**.
1. Сохраните страницу и верните ее.

В следующей таблице перечислены типы списков, доступные для выбора в диалоговом окне **Выбор конфигурации списка продуктов**.

|----------------------------|-------------|------------------|-------------------------------------|-----------------------------------------------|
