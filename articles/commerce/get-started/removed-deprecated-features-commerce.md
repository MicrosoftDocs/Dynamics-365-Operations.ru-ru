# <a name="removed-or-deprecated-features-in-dynamics-365-commerce"></a>Удаленные или устаревшие функции Dynamics 365 Commerce

[!include [banner](../includes/banner.md)]

В этом разделе описываются возможности, который удалены или которые планируется удалить из Dynamics 365 Commerce.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Этот список поможет вам учитывать эти удаления и устаревания при своем собственном планировании. 

> [!NOTE]
> Подробные сведения об объектах в приложениях Finance and Operations можно найти в документе [Технический справочник по отчетам](/dynamics/s-e/). Можно сравнить различные версии этих отчетов, чтобы получить сведения об объектах, которые были изменены или были исключены в каждой версии приложений Finance and Operations.

## <a name="features-removed-or-deprecated-in-the-commerce-10021-release"></a>Функции, удаленные или устаревшие в выпуске Commerce 10.0.21

[!include [banner](../includes/preview-banner.md)]

### <a name="overlapping-discounts-handling-setting-in-commerce-parameters"></a>Настройка обработки перекрывающихся скидок в параметрах Commerce

Настройка **Обработка пересекающихся скидок** на странице **Параметры Commerce** устарели в выпуске Commerce версии 10.0.21. Далее в модуле ценообразования Commerce будет использоваться единый алгоритм для определения оптимального сочетания перекрывающихся скидок.

| &nbsp;  | &nbsp; |
|------------|--------------------|
| **Причина устаревания/удаления** | <p>Настройка **Обработка пересекающихся скидок** в параметрах Commerce Parameters определяют, как система ценообразования Commerce выполняет поиск и определяет оптимальное сочетание перекрывающихся скидок. В настоящее время предлагаются три варианта:<p><ul><li> **Оптимальная производительность** — в этом параметре используется алгоритм расширенной эвристики и метод [ранжирования по маржинальным значениям](../optimal-combination-overlapping-discounts.md) для своевременного задания приоритета, оценки и определения наиболее подходящего сочетания скидок.</li><li>**Сбалансированное вычисление** — в текущей базе кода этот параметр функционирует так же, как и параметр **Оптимальная производительность**. Таким образом, это, по сути, повторяющийся параметр.</li><li>**Исчерпывающий расчет** — этот параметр использует старый алгоритм, который перебирает все возможные комбинации скидок при расчете цены. Для заказов, имеющих большие строки и количества, этот вариант может привести к проблемам с производительностью.</li></ul><p>Чтобы упростить настройку, повысить производительность и сократить количество происшествий, вызванных старым алгоритмом, мы полностью удалим настройку **Обработка пересекающихся скидок** и обновим внутреннюю логику модуля ценообразования Commerce, чтобы в нем теперь использовался только расширенный алгоритм (то есть, алгоритм **наилучшей производительности**).</p> |
| **Заменена другой функцией?**   | Нет. Перед удалением этой функции рекомендуется, чтобы организации, использующие параметр **Сбалансированный расчет** или **Исчерпывающий расчет**, переключились на параметр **Оптимальная производительность**. |
| **Затрагиваемые области продукта**         | Цены и скидки |
| **Вариант развертывания**              | Все |
| **Состояние**                         | В выпуске 10.0.21 параметр **Обработка пересекающихся скидок** будет удален из параметров Commerce в октябре 2022 года. |

### <a name="retail-sdk-distributed-by-using-lifecycle-services"></a>Пакет Retail SDK, распространяемый с использованием Lifecycle Services

Retail SDK поставляется в Lifecycle Services (LCS). Этот режим распределения устарел в выпуске 10.0.21. Теперь справочные пакеты, библиотеки и образцы Retail SDK будут опубликованы в общественных репозиториях в GitHub.

| &nbsp;  | &nbsp; |
|------------|--------------------|
| **Причина устаревания/удаления** | Пакет Retail SDK поставляется в LCS. Для завершения процесса LCS требуется несколько часов, и процесс должен быть повторен для каждого обновления. Теперь справочные пакеты, библиотеки и образцы Retail SDK будут опубликованы в общественных репозиториях в GitHub. Примеры расширений и справочные пакеты могут быть легко использованы, а обновления завершаются через несколько минут. |
| **Заменена другой функцией?**   |  [Загрузка примеров Retail SDK и справочных пакетов из GitHub и NuGet](../dev-itpro/retail-sdk/sdk-github.md) |
| **Затрагиваемые области продукта**         | Retail SDK |
| **Вариант развертывания**              | Все |
| **Состояние**                         | Устарело: с выпуска 10.0.21 пакет SDK, поставляемый через виртуальные машины LCS, будет удален в октябре 2022 г. |

### <a name="retail-deployable-package-and-combined-pos-hardware-station-and-cloud-scale-unit-installers"></a>Распространяемый пакет Retail и объединенные программы установки POS, станции оборудования и облачной единицы масштабирования

Распространяемые пакеты Retail, созданные с помощью Retail SDK MSBuild, устарели в 10.0.21. Теперь используйте пакет облачной единицы масштабирование (CSU) для расширений облачных единиц масштабирования (Commerce Runtime, база данных канала, API консольной архитектуры коммерции, платежей и облачного POS). Используйте программы установки только с расширением для POS, станции оборудования облачных единиц масштабирования с самостоятельным размещением.

| &nbsp;  | &nbsp; |
|------------|--------------------|
| **Причина устаревания/удаления** | Развертываемый пакет Retail представляет собой объединенный пакет, содержащий полный набор пакетов расширений и установщиков. Этот комбинированный пакет делает развертывание сложным, так как расширения CSU переходят в облачную единицу масштабирования, а установщики разворачиваются в магазинах. Установщики включают расширение и базовый продукт, что затрудняет обновление. При каждом обновлении необходимо выполнить слияние кода и создание пакета. Чтобы упростить этот процесс, пакеты расширений разделяются на компоненты для простоты развертывания и управления. Благодаря новому подходу, расширения и установщики базовых продуктов разделяются и могут быть независимыми в обслуживании и обновлены без слияния кода или переупаковки.|
| **Заменена другой функцией?**   | Расширения CSU, установщики расширений POS, установщики расширений станций оборудования |
| **Затрагиваемые области продукта**         | Расширение и развертывание Dynamics 365 Commerce |
| **Вариант развертывания**              | Все |
| **Состояние**                         | Устарело: с выпуска 10.0.21 поддержка развертывания RetailDeployablePackage в LCS будет удалена в октябре 2022 г. |

### <a name="modernpossln-and-cloudpossln-in-the-retail-sdk"></a>ModernPos.Sln и CloudPOs.sln в Retail SDK

Разработка расширений POS с помощью ModernPos.sln, CloudPOs.sln, POS.Extension.csproj и папки POS устарела в выпуске 10.0.21. Теперь используется SDK независимой упаковки POS для расширений POS.

| &nbsp;  | &nbsp; |
|------------|--------------------|
| **Причина устаревания/удаления** | В предыдущих версиях пакета Retail SDK, если имеются расширения POS, для обновления до последней версии POS требуется слияние кода и упаковка. Слияние кода требовало много времени на обновление, и вам приходилось хранить полный пакет Retail SDK в репозитории. Также приходилось компилировать основной проект POS.App. При использовании независимой модели упаковки необходимо сохранять только расширение. Процесс обновления до последней версии расширений POS упрощает обновление версии пакета NuGet, используемого вашим проектом. Расширения могут быть развернуты независимо друг от друга, и службы используют установщики расширений. Базовый POS можно разворачивать и обслуживать отдельно, не требуется слияние кода или упаковка с базовым установщиком или кодом. |
| **Заменена другой функцией?**   | [SDK упаковки, независимый от POS](../dev-itpro/pos-extension/pos-extension-getting-started.md) |
| **Затрагиваемые области продукта**         | Расширение и развертывание Dynamics 365 Commerce POS |
| **Вариант развертывания**              | Все |
| **Состояние**                         | Устарело: с версии 10.0.21 поддержка комбинированных пакетов POS и модели расширений с использованием ModernPos.Sln, CloudPOs.sln и POS.Extensons.csproj в Retail SDK будет удалена в октябре 2022 г. |
