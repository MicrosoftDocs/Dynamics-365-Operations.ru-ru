# <a name="preview-of-dynamics-365-commerce-10030-november-2022"></a>Предварительная версия Dynamics 365 Commerce 10.0.30 (ноябрь 2022 г.)

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой статье перечислены новые и измененные компоненты предварительной версии Microsoft Dynamics 365 Commerce (10.0.30). Эта версия имеет номер сборки 10.0.1362 и доступна по следующему графику:

- **Предварительная версия выпуска:** сентябрь 2022
- **Общая доступность выпуска (самостоятельное обновление):** октябрь 2022 г.
- **Общая доступность выпуска (автоматическое обновление):** ноябрь 2022 г.

## <a name="features-included-in-this-release"></a>Возможности, включенные в данный выпуск

В этой таблице перечислены функции в выпуске. Мы можем обновить эту статью для включения функций, которые были добавлены в сборке после первоначальной публикации этой статьи.

| Область компонентов | Функция | Дополнительные сведения | Включено пользователем   |
|---------|------------------|----------------|--------------| 
| Служба поддержки клиентов   | Участвуйте в чате на сайте электронной коммерции с помощью бота Power Virtual Agents. | Эта функция предоставит пользователям сайта электронной коммерции выбор для использования чат-бота Power Virtual Agents для запросов поддержки. | Включено администратором/создателями для конечных пользователей |
| Аналитика  |  Используйте потоковый вывод событий оперативной аналитики POS-терминала на вашу учетную запись Application Insights. | [Журналы доступа в Application Insights](../dev-itpro/operational-insights.md#enable-diagnostic-events-in-application-insights)   |  Специалист в области ИТ/разработчик дал свое явное согласие   |
|  Платежи  | Поддержка заказов PayPal за пределами 29-дневного периода авторизации. | Максимальный период авторизации для PayPal составляет 29 дней, после которого должен быть создан новый код авторизации и заказа. В качестве альтернативы PayPal предлагает вариант, в котором на заказ PayPal можно ссылаться как на общий заказ, позволяя выполнять множественные авторизации и фиксации с одним и тем же кодом заказа вместо создания новой авторизации и кода заказа через 29 дней). | При настройке соединителя платежей PayPal в Commerce headquarters было добавлено поле **OrderIntent**, и оно может иметь два значения:<p><p>- **Авторизовать** — эта конфигурация используется по умолчанию (если поле оставлено пустым, **Авторизовать** будет действовать по умолчанию). Настройка для **OrderIntent** значения **Авторизовать** коррелирует со значением **processing_instruction** в PayPal **NO_INSTRUCTION**. Заказ будет авторизован с PayPal, и авторизация не может быть изменена, когда это значение используется.<p>- **Сохранить** — когда для параметра **OrderIntent** указано значение **Сохранить**, это соответствует значению **processing_instruction** **ORDER_SAVED_EXPLICITLY** в PayPal. Ссылки на заказ будут сохраняться в службе PayPal при использовании этого значения.  |
| Вход в POS-терминал  | [Включение функций самодиагностики для входа в POS-терминал](/dynamics365-release-plan/2022wave2/commerce/dynamics365-commerce/enable-self-serve-diagnosis-capabilities-pos-sign-in)  |  Эта функция обеспечивает функции самодиагностики в POS-терминале и Commerce headquarters, чтобы помочь пользователям и руководителям магазина быстро обнаруживать и устранять основные причины возникновения проблем с входом в POS-терминал.<p><p>- Улучшены сообщения об ошибках, отображаемые на экране входа в POS-терминал, чтобы предоставить конкретную информацию об основной причиной, которая может помочь сотрудникам магазина, использующим POS-терминал, понять, что пошло не так, чтобы они могли выполнить необходимые действия для решения проблемы.<p>- Функция **Тестовый вход** доступна на странице **Сотрудники** в Commerce headquarters, чтобы менеджеры магазинов, которые настраивают POS-терминалы, могли имитировать вход в POS-терминал. В случае сбоя входа эта функция предоставляет практическое руководство по устранению неполадок, что позволяет менеджерам проверять соответствующие конфигурации, устранять проблемы и проверять исправления.  | Включено по умолчанию |

### <a name="removed-and-deprecated-features"></a>Удаленные и устаревшие функции

В статье [Удаленные или устаревшие функции в Dynamics 365 Commerce](removed-deprecated-features-commerce.md) описываются функции, которые были удалены или устарели для Commerce.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Перед удалением каких-либо функций из продукта уведомление об устаревании будет объявлено в статье [Удаленные или устаревшие функции в Dynamics 365 Commerce](removed-deprecated-features-commerce.md) за 12 месяцев до удаления.

Для критических изменений, которые влияют только на время компиляции, но являются двоично совместимыми с песочницей и производственными средами, время устаревания будет меньше 12 месяцев. Обычно это функциональные обновления, которые должны быть выполнены для компилятора.
