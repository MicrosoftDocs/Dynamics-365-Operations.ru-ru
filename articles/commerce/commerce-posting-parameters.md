# <a name="commerce-posting-parameters"></a>Параметры разноски Commerce

[!include [banner](includes/banner.md)]
[!include [banner](includes/preview-banner.md)]

В этой теме описываются параметры, связанные с разноской финансовых и физических проводок в Microsoft Dynamics 365 Commerce. Параметры разноски Commerce находятся в модуле Commerce headquarters в разделе **Розница и коммерция \> Настройка Headquarters \> Параметры \> Параметры Commerce \> Разноска**.

## <a name="periodic-discount-parameters"></a>Параметры розничной скидки

В следующей таблице перечислены параметры розничной скидки, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Разнести розничную скидку | Этот параметр контролирует, разносятся ли периодические предложения на счета ГК. Периодические скидки включают скидки за комплект, скидки за количество и скидочные предложения. Если этот параметр включен, дополнительные поля могут быть заданы в разделе **Розничные скидки**. |
| Тип счета ГК | <p>Выберите тип счета, используемый для разноски розничных скидок:</p><ul><li>**Стандартный** — система не использует поля, соответствующие скидкам, на этой странице. Вместо этого она использует счета, определенные на странице **Разноска** в Commerce headquarters (**Управление запасами \> Настройка \> Разноска \> Формы разноски**).</li><li>**Периодический** — система использует счета скидок Commerce, которые указаны в полях, связанных со скидкой, на данной странице. Если выбрать этот параметр, следует указать счет ГК для каждого типа предложения (скидка, скидка за комплект, скидка за количество и скидка по порогу). При настройке каждой скидки можно определить счет. Когда функция разноски счета скидки используется на странице настройки скидок, выполняется дополнительная дебетовая запись и кредитовая запись для реклассификации разноски скидки со счета ГК списки Commerce на счет ГК скидки. Дополнительные сведения см. в разделе [Розничные скидки](retail-discounts-overview.md).</li></ul> |
| Разноска скидку по инфокоду | Этот параметр больше не используется в стандартном решении Commerce и будет устаревшим. |
| Разнести периодическую скидку для заказов | Этот параметр определяет, будут ли периодические скидки розничной торговли разноситься в книгу учета для заказов клиентов и заказов центра обработки вызовов. |

## <a name="inventory-update-parameters"></a>Параметры обновления запасов

В следующей таблице перечислены параметры обновления запасов, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Использовать ИД пакета по умолчанию, если не найдены номера партий | Этот параметр контролирует, используется ли ИД партии по умолчанию для номенклатур с поддержкой партий, если не найдены номера партий и допускаются отрицательные запасы. |
| ИД пакета по умолчанию | Номер партии, который используется, если номера партий для номенклатур не найдены и включен параметр **Использовать ИД пакета по умолчанию, если не найдены номера партий**. |

## <a name="aggregation-parameters"></a>Параметры агрегирования

В следующей таблице перечислены параметры агрегирования, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Сдача наличных средств в сейф | Включите этот параметр, чтобы агрегировать все проводки при разноске журнала операций и создавать одну строку в журнале для разноски вместо отдельной строки для каждой сдачи наличных. |
| Инкассация | Включите этот параметр, чтобы агрегировать все проводки при разноске журнала операций и создавать одну строку в журнале для разноски вместо отдельной строки для каждой сдачи наличных. |
| Проводки по продажам | Включите этот параметр для консолидации проводок как части процесса разноски отчета по проводкам. Рекомендуется включить этот параметр. Он ранее назывался **Проводки ваучера**. |
| Не агрегировать возвраты | Если выбран этот параметр, каждая проводка возврата будет разнесена как отдельный заказ на продажу при разноске журнала операций розничной торговли. |

## <a name="batch-processing-parameters"></a>Параметры пакетной обработки

В следующей таблице перечислены параметры пакетной обработки, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Максимальное количество параллельных операций | В этом поле определяется число пакетных задач, которые используются для разноски нескольких отчетов. |
| Максимум потоков для обработки заказов на журнал операций | Это поле представляет максимальное число потоков, используемых пакетным заданием разноски журналов для создания и выставления накладных для заказов на продажу для одного журнала операций. Общее число потоков, которые используются процессом разноски журнала операций, вычисляется путем умножения значения этого параметра на значение в параметре **Максимальное количество параллельных операций разноски журналов операций**. Если значение этого параметра слишком велико, это может негативно повлиять на производительность процесса разноски журнала операций. |
| Максимальное число строк при агрегировании проводок | Это поле определяет количество строк проводки, которые включаются в одну объединенную проводку перед созданием новой проводки. Агрегированные проводки создаются на основе других критериев агрегирования, таких как клиент, рабочая дата или финансовая аналитика. Обратите внимание, что строки из одной проводки не будут разбиты по разным агрегированным проводкам. Поэтому число строк в агрегированной проводке может быть несколько больше или меньше, в зависимости от факторов, таких как число различных продуктов. |
| Максимальное число потоков для проверки проводок магазина | Это поле определяет максимальное число потоков, которые используются для проверки проводок. Проверка проводок является обязательным шагом, который должен быть выполнен до того, как проводки могут быть вставлены в журналы операций. Также необходимо определить продукт подарочного сертификата на экспресс-вкладке **Подарочный сертификат** на вкладке **Разноска** на странице **Параметры Commerce**, даже если в организации на используются подарочные сертификаты. |

В следующей таблице перечислены рекомендуемые значения для параметров в предыдущей таблице. Эти значения следует проверять и адаптировать под конфигурацию развертывания и доступную инфраструктуру. Значения, которые превышают рекомендуемые значения, могут отрицательно повлиять на обработку других пакетов и должны быть проверены.

| Параметр | Рекомендуемое значение | Подробно |
|-----------|-------------------|---------|
| Максимальное количество параллельных операций разноски журналов операций | <p>Установите для этого параметра количество пакетных задач, доступных для группы пакетов, в которой запущено задание **Отчет**.</p><p>**Общее правило:** умножьте число виртуальных серверов Application Object Server (AOS) на число пакетных задач, доступных для виртуального сервера AOS.</p> | Этот параметр неприменим, если включена функция **Журналы операций розничной торговли — поточная разноска**. |
| Максимум потоков для обработки заказов на журнал операций | Начало проверки значений с **4**. Обычно значение не должно превышать **8**. | Этот параметр определяет число потоков, используемых для создания и разноски заказов на продажу. Представляет собой число потоков, доступных для разноски по журналам. |
| Максимальное число строк при агрегировании проводок | Начало проверки значений с **1000**. В зависимости от конфигурации Commerce headquarters, меньшие заказы могут быть более выгодными для производительности. | Этот параметр определяет число строк, которые включаются в каждый заказ на продажу при разноске журнала операций. После достижения этого номера строки будут разделены в новый заказ. Число строк продаж не будет точно равно указанному вами количеству, так как разбиение выполняется на уровне заказа на продажу. Тем не менее, число будет близко к заданному количеству. Этот параметр используется для создания заказов на продажу для проводок по розничной торговле, не имеющих именованных клиентов. |
| Максимальное число потоков для проверки проводок магазина | Рекомендуется установить для этого параметра значение **4**, и оно увеличится только в том случае, если приемлемое быстродействие не достигнуто. Число потоков, используемых этим процессом, не может превышать число процессоров, доступных серверу обработки пакетных заданий. Если количество потоков слишком большое, это может повлиять на другую пакетную обработку. | Этот параметр определяет количество проводок, которые могут быть проверены одновременно для данного магазина. |

> [!NOTE]
> Все настройки и параметры, которые относятся к разноске журналов операций и которые определяются в магазинах и на странице **Параметры Commerce**, применимы к улучшенной функции разноски журналов операций.

## <a name="invoice-parameters"></a>Параметры накладной

В следующей таблице перечислены параметры накладных, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Имя журнала | Имя журнала платежей, который используется при создании журналов платежей клиентов для платежей по заказам на продажу. Этот параметр применяется ко всем платежам по заказу, которые разносятся для заказов на POS-терминалах, в центре обработки звонков и каналах электронной коммерции. |
| Наименование счета | Имя счета платежей. |
| Приоритизация аналитик по способу оплаты | Если этот флаг включен, журналы платежей задают приоритет аналитик из метода оплаты, а не аналитик из магазина. |
| Поведение расчета налога | Этот параметр определяет поведение при выставлении накладных для заказов на продажу, созданных при разноске журнала операций:<ul><li>**Пересчитать** — повторный расчет налогов.</li><li> **Не пересчитывать** — используются суммы налога, рассчитанные в POS.</li></ul> |
| Имя журнала | Имя журнала, которое используется при создании журнала платежей клиента для возврата денежных средств. Этот параметр применяется ко всем платежам по заказу, которые разносятся для заказов на POS-терминалах, в центре обработки звонков и каналах электронной коммерции. |

## <a name="statement-parameters"></a>Параметры журнала операций

В следующей таблице перечислены параметры журнала операций, связанные с разноской финансовых и физических проводок.

| Параметр | Описание |
|-----------|-------------|
| Резервировать запасы во время расчета | Когда этот параметр включен, при расчете журнала операций складские запасы временно резервируются до разноски журнала операций. По умолчанию этот параметр отключен, что позволяет улучшить производительность при расчете журнала операций. Обновленные сведения о запасах можно рассчитать, используя пакетное задание **Разнести запасы**. Обратите внимание, что пакетное задание **Разнести запасы** больше не используется, когда включено создание заказов на основе [поточной разноски](trickle-feed.md) для проводок розничного магазина. |
| Отключить требование пересчета | Этот флаг отключает пересчет при разноске в Commerce headquarters. |
| Пересчет финансовых аналитик в случае ошибки | Когда этот параметр включен, финансовые аналитики могут быть пересчитаны в последующих разносках журналов операций в случае ошибки разности. |
| Использовать финансовые аналитики из хранилища возвратов | Когда этот параметр включен, могут быть созданы связанные заказы на продажу с использованием финансовых аналитик магазина вместо финансовых аналитик из исходной проводки. |
| Удалить связь для возвратов | Когда этот параметр включен, выписка может создавать возвраты по неразнесенным продажам как возвраты "вслепую". По умолчанию этот параметр отключен, и рекомендуется оставить его отключенным. |
| Отключение разноски разницы округления | Этот параметр отключает разноску разницы округления между платежом по проводке и валовой суммой при обработке платежей. |
| Автоматическое сопоставление депозитов клиента | Когда этот параметр включен, депозиты клиента, разнесенные во время разноски журнала розничных операций, сопоставляются с открытыми проводками клиента. |
| Включите и используйте выверку управления кассой из POS | Если этот параметр включен, выверка денежных средств выполняется в POS-терминале, а значения передаются в разноску финансового отчета розничной торговли для создания строк журнала операций. |
| Уровень детализации ваучера ГК | Этот параметр определяет уровень детализации, который включается в ваучер ГК для выбранных проводок, созданных из POS. Типы проводок включают доходы, расходы и скидки. Для скидок этот параметр учитывается только в том случае, если номер счета для периодической скидки и номер счета для обычной скидки не совпадают. Если подробности не требуются, для этих разносок рекомендуется использовать значение **Сводка**. При определении разноски на уровне сводки значение **TransactionDiscountTrans.RecID** не будет заполнено. В выпуске Commerce версии 10.0.27 этот флаг был переименован и перемещен. Ранее он назывался **Уровень детализации** и находился в разделе **Обновление запасов**. |
