# <a name="configure-a-product-to-be-purchased-for-free"></a>Настройте продукт для бесплатной покупки

[!include [banner](includes/banner.md)]


В этом разделе описывается, как настроить продукт таким образом, чтобы его можно было купить бесплатно в Microsoft Dynamics 365 Commerce.

## <a name="configure-the-product"></a>Настройка продукта

Чтобы продать продукт бесплатно в Dynamics 365 Commerce, необходимо установить для его цены значение 0 (ноль). Кроме того, для продукта необходимо настроить параметр **Нулевая цена допустима**.

Чтобы настроить параметр **Нулевая цена допустима** в центральном офисе Commerce, выполните следующие действия.

1. Перейдите в раздел **Retail и Commerce \> Продукты и категории \> Выпущенные продукты по категориям**.
1. Перейдите к продукту, который необходимо продать бесплатно. 
1. В разделе **Commerce** страницы продукта установите для параметра **Нулевая цена допустима** значение **Да**.

На следующем рисунке показан пример продукта, в котором для параметра **Нулевая цена допустима** задано значение **Да**.

![Пример продукта, в котором параметр нулевая цена допустима задан как "Да".](./media/Zero-price.png)

## <a name="configure-the-online-stores-functionality-profile"></a>Настройте функциональный профиль интернет-магазина

Прежде чем свободные проводки будут обработаны, следует настроить параметр **Разрешить оформление заказа без платежей** профиля функциональности для вашего интернет-магазина, чтобы проводки, не имеющие платежей, были разрешены. Сведения о создании профилей функциональности см. в разделе [Создание профиля функциональности для онлайн-торговли](online-functionality-profile.md).

Чтобы настроить параметр **Разрешить оформление заказа без платежей** в центральном офисе Commerce, выполните следующие действия.

1. Перейдите в раздел **Retail и Commerce \> Настройка канала \> Настройка интернет-магазина**.
1. На странице для профиля функциональности магазина в **Оформить заказ** установите для параметра **Разрешить оформление заказа без платежей** значение **Да**.

На следующем рисунке показан пример профиля интернет-магазина, в котором параметр **Разрешить оформление заказа без платежей** имеет значение **Да**.

![Пример профиля интернет-магазина, в котором параметр Разрешить оформление заказа без платежей имеет значение Да.](./media/Zero-price-profile.png)
