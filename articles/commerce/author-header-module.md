# <a name="header-module"></a>Модуль верхнего колонтитула

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

В этом разделе описываются модули заголовка, а также описывается, как создавать их в Microsoft Dynamics 365 Commerce.

## <a name="overview"></a>Обзор

Модуль заголовка — это специальный контейнер, который используется для размещения всех модулей, которые будут отображаться в заголовке страницы. Например, он может содержать эмблему сайта, ссылки на иерархию навигации, ссылки на другие страницы сайта и панель поиска.

Модуль заголовка автоматически оптимизируется для устройства, на котором просматривается сайт (то есть, для настольного устройства или мобильного устройства). Например, на мобильном устройстве панель навигации свернута в кнопку **Меню** (которая иногда называется *меню гамбургер*)

## <a name="properties-of-a-header"></a>Свойства заголовка

Как и в случае универсальных контейнеров, модуль заголовка поддерживает свойства **заголовок** и **ширина**.

В модуле заголовка имеется несколько слотов. Например, существуют гнезда для информационного сообщения, меню навигации, эмблемы, панели поиска, значка корзины, значка списка желаний и информации об учетной записи. Каждый слот поддерживает конкретный набор модулей.

## <a name="modules-that-are-available-in-a-header-module"></a>Модули, доступные в модуле заголовка

Следующие модули, которые могут быть использованы в модуле заголовка:

- **Меню навигации** — меню навигации представляет собой иерархию навигации по каналам и другие статические навигационные ссылки. Иерархия навигации по каналу может быть настроена в Dynamics 365 Retail. Сконфигурированные элементы затем отображаются в виде навигации заголовка. Кроме того, можно настроить статические ссылки навигации, а также указать относительные ссылки на другие страницы сайта электронной коммерции. Сам заголовок может быть выровнен по левому краю, по правому или по центру.
- **Значок корзины** — значок корзины представляет собой специальный значок, представляющий корзину. Он отображается в заголовке и указывает количество номенклатур в корзине. Ссылка на страницу корзины должна сопровождаться значком корзины, чтобы клиенты могли быть перенаправлены на страницу корзины при взаимодействии со значком.
- **Значок списка желаний** — значок списка желаемых товаров отображается в заголовке и указывает число номенклатур, добавленных в список желаний клиента. Ссылка на страницу списка желаний должна сопровождаться этим значком, чтобы клиенты могли быть перенаправлены на страницу списка желаний при взаимодействии со значком.
- **Модуль входа** — в заголовке отображается модуль входа в систему. Клиенты могут либо войти в учетную запись, либо зарегистрировать учетную запись. Если клиент уже выполнил вход, модуль можно настроить так, чтобы он отображал ссылки на страницу учетной записи, страницу журнала заказов или другую страницу.
- **Модуль эмблемы** — в этом модуле отображается эмблема, представляющая розничного продавца и торговую марку. Это изображение, имеющее ссылку. Ссылка обычно настраивается таким образом, чтобы она перенаправлялась на домашнюю страницу. Поэтому клиенты могут быстро вернуться на домашнюю страницу с любой страницы сайта.
- **Оповещение** — оповещение отображается в заголовке и используется для отображения встроенного сообщения, которое применяется ко всем страницам сайта. Например, оповещение может содержать сообщение "Ежегодная распродажа заканчивается через 2 дня".
- **Панель поиска** — панель поиска позволяет пользователям вводить искомые термины, чтобы они могли выполнять поиск продуктов. Модуль должен быть настроен с URL-адресом страницы результатов поиска. Можно настроить параметр строки запроса (значение по умолчанию — **"q"**). Панель поиска содержит ячейку автоматического предложения, в которую следует добавить модуль автозаполнения.
- **Автозаполнение** — автоматически предлагаемые результаты отображаются в модуле автозаполнения. Эти результаты могут быть ключевыми словами, продуктами или категориями, в которых искомый термин будет найден.

## <a name="create-a-header-module"></a>Создание модуля заголовка

Чтобы создать модуль заголовка, выполните следующие действия.

1. Создайте фрагмент страницы, содержащий модуль заголовка.
1. Добавьте модули в гнезда в модуле заголовка.
1. Обновите настройки для каждого модуля.
1. Сохраните фрагмент страницы. 
1. Верните страницу и опубликуйте его.

Чтобы гарантировать, что заголовок появляется на каждой странице, выполните следующие действия в каждом шаблоне страниц, созданном для сайта.

1. На странице по умолчанию добавьте фрагмент страницы, содержащий модуль заголовка, к заголовку в главном слоте.
1. Сохраните шаблон. 
1. Верните шаблон и опубликуйте его.
