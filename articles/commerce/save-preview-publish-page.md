# <a name="save-preview-and-publish-a-page"></a>Сохранение, предварительный просмотр и публикация страницы

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

В этом разделе описывается, как сохранить, предварительно просмотреть и опубликовать страницу в Microsoft Dynamics 365 Commerce.

## <a name="save-a-page"></a>Сохранение страницы

Чтобы сохранить страницу, необходимо, чтобы она была извлечена для вас и открыта в редакторе страниц. Страницу следует сохранить непосредственно после ее изменения, чтобы гарантировать сохранение изменений.

При сохранении страницы эти изменения видны только вам. Операция сохранения предназначена в основном для сохранения изменений, пока страница еще не готова для возврата. После завершения изменения страницы рекомендуется вернуть ее в, чтобы изменения стали видимыми для других пользователей. На этом этапе страница может также быть извлечена другими пользователями, которые должны ее изменить.

## <a name="preview-a-page"></a>Предварительный просмотр страницы

Средство разработки предлагает два вида предварительного просмотра: область просмотра "что вы видите, то вы и получаете» (WYSIWYG) в редакторе страниц и отдельное окно предварительного просмотра.

При использовании редактора страниц в центральной области появится предварительный просмотр "что вы видите, то и получите" (WYSIWYG). Этот предварительный просмотр автоматически обновляется, когда пользователь сохраняет страницу. Можно также обновить его вручную, выбрав **Обновить** на панели команд. В режиме предварительного просмотра WYSIWYG страница отображается точно в том виде, в котором она отображается пользователями сайта, но помощники по разработке отображаются поверх нее.

После завершения изменения страницы, возможно, потребуется просмотреть ее, чтобы увидеть, что увидят клиенты. Чтобы просмотреть страницу, выберите **Предварительный просмотр** на панели команд. Предварительный просмотр появится в отдельном окне браузера. Страница в окне предварительного просмотра отображается точно так же, как отображается пользователем сайта. Можно изменить размеры окна, чтобы убедиться, что отзывчивые модули правильно обрабатываются во всех портах просмотра.

> [!NOTE]
> Для предварительного просмотра неопубликованного содержимого требуются проверка подлинности и правильные разрешения. Поэтому при открытии кому-то общего доступа к URL-адресу предварительного просмотра этот человек должен иметь правильные разрешения на доступ к содержимому.

## <a name="publish-a-page"></a>Публикация страницы

После того как страница готова, следующим шагом является ее публикация, чтобы внешние пользователи могли просмотреть содержимое. Прежде чем можно будет опубликовать страницу, ее необходимо вернуть.

Можно публиковать и отменять публикацию страниц либо в инспекторе страниц, либо в редакторе страниц. Инспектор страниц отображает список страниц и позволяет выполнять массовые операции. Редактор страниц можно использовать для публикации или отмены публикации только одной страницы, которая открыта в нем.

Чтобы опубликовать одну или несколько страниц из инспектора страниц, выберите страницы, убедитесь, что они возвращены, затем выберите команду **Опубликовать** на панели команд. Страницы публикуются, и вы получаете уведомление об операции в инструменте разработки.

Процедура публикации одной страницы с помощью редактора страниц аналогична. Пока страница открыта в редакторе страниц, убедитесь, что она возвращена, затем выберите команду **Опубликовать** на панели команд. Страница публикуется, и вы получаете уведомление об операции.

При публикации страницы публикуется только ее содержимое. Вы и другие пользователи могут перейти к странице и просмотреть ее только после того, как с ней будет связан URL-адрес. URL-адрес должен быть опубликован отдельно.

> [!IMPORTANT]
> Прежде чем можно будет опубликовать страницу, все изображения или фрагменты, на которые ссылается страница, уже должны быть опубликованы.

## <a name="save-preview-and-publish-a-home-page"></a>Сохранение, предварительный просмотр и публикация домашней страницы

Чтобы сохранить опубликовать домашнюю страницу, выполните следующие действия.

1. В области **Сайты** выберите **Fabrikam** (или имя вашего сайта).
1. В области переходов слева выберите **Страницы**.
1. Найдите и выберите домашнюю страницу для ее открытия в редакторе страниц.
1. Выберите **Извлечь**.
1. Измените страницу нужным образом.
1. Выберите **Сохранить**, затем выберите **Вернуть**.
1. В поле **Комментарии** введите примечание о внесенных изменениях, затем выберите **ОК**.
1. Выберите **Предварительный просмотр** для предварительного просмотра страницы. По завершении закройте вкладку предварительного просмотра, чтобы вернуться к инструменту разработки.
1. Выберите **Опубликовать**.

## <a name="publish-a-url"></a>Публикация URL-адреса

Чтобы опубликовать URL-адрес, выполните следующие действия.

1. В области **Сайты** выберите **Fabrikam** (или имя вашего сайта).
1. В области переходов слева выберите **URL-адреса**.
1. Найдите и выберите URL-адрес, который требуется опубликовать.
1. На панели команд выберите **Опубликовать**.
