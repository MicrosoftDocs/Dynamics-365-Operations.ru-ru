# <a name="page-model-glossary"></a>Глоссарий по модели страниц


[!include [banner](includes/banner.md)]

В этой статье описываются различные элементы, которые используются на страницах сайта Microsoft Dynamics 365 Commerce.

## <a name="page-element-definitions"></a>Определения элементов страницы

В следующей таблице представлена сводка терминов, которые необходимо знать при изменении внешнего вида, поведения и содержимого сайта. Для получения более подробных пояснений и процедур воспользуйтесь ссылками.

| Срок | Описание и примечания |
|------|-----------------------|
| [Модуль](work-with-modules.md) | <p>**Определение:** модули являются строительными блоками, которые можно разработать и составить из них каркас веб-страницы. Примерами являются модули заголовка, главного имиджевого баннера или карусели.</p><p>**Где выбирается:** развернутые модули могут быть выбраны и настроены на различных стадиях процесса разработки сайта, таких как этапы шаблона, макета, страницы и разработки фрагментов.</p><p>**Где редактируется:** настраиваемые модули создаются в коде с помощью пакета средств разработки программного обеспечения (SDK). Затем они отправляются на сайт, где они становятся доступными для выбора.</p> |
| Свойство модуля | <p>**Определение:** свойства модуля — это определенные параметры, определяемые модулем. Их можно редактировать в инструментах разработки электронной коммерции. Например, свойства модуля используются для задания заголовка и фонового изображения модуля баннера.</p><p>**Где настраивается:** свойства модуля выбираются и настраиваются в области свойств, которая отображается в средах разработки (редакторах) для параметров шаблонов, макетов, страниц, фрагментов и приложений.</p> |
| [Шаблон](templates-layouts-overview.md) | <p>**Определение:** шаблоны определяют сочетания модулей и параметры, которые должны использоваться для категории страниц (например, страницы маркетинга, страницы категорий и страницы продукта).</p><p>**Где выбирается:** шаблоны могут выбираться в рабочих процессах создания страниц или макетов.</p><p>**Где редактируется:** шаблоны создаются в редакторе шаблонов. Написание кода для их создания или изменения не требуется.</p> |
| [Макет](templates-layouts-overview.md) | <p>**Определение:** макеты определяют окончательный выбор и расположение модулей из набора параметров родительского шаблона. Макет можно настроить для одной страницы (*настраиваемой макет*), либо он может быть общим для нескольких страниц (*готовый макет*).</p><p>**Где выбирается:** макеты могут выбираться во время создания новой страницы или когда для существующей страницы требуется другой макет.</p><p>**Где редактируется:** макеты создаются в редакторе макетов. Написание кода для их создания или изменения не требуется.</p> |
| [Экземпляр страницы](modify-existing-page.md) | <p>**Определение:** экземпляры страниц определяют конечное локализованное содержимое, специфическое для страницы, для одной страницы. Это содержимое извлекается из значений свойств модуля.</p><p>**Где выбирается:** страницы выбираются при назначении URL-адресов.</p><p>**Где редактируется:** страницы редактируются в редакторе страниц. Написание кода для их создания или изменения не требуется.</p> |
| [Тема](select-site-theme.md) | <p>**Определение:** темы определяют каскадные таблицы стилей (CSS) и определяют внешний вид и поведение модулей, отображаемых на странице.</p><p>**Где выбирается** : после отправки темы на сайт с помощью служб Microsoft Dynamics Lifecycle Services (LCS) она может быть выбрана в качестве свойства модуля-контейнера страницы.</p><p>**Где редактируется:** темы в настоящее время создаются и редактируются с помощью SDK. Затем они отправляются на ваш сайт с использованием служб LCS.</p> |
| [Фрагмент](work-with-fragments.md) | <p>**Определение:** фрагменты являются полностью настроенными модулями с локализованным содержанием, которые могут быть повторно использованы и централизовано обновлены на нескольких страницах. Например, фрагмент, созданный из модуля заголовка, может использоваться во всех шаблонах и на всех страницах сайта, а также централизованно обновляться в одном месте.</p><p>**Где выбирается:** фрагменты могут выбираться везде, где могут быть выбраны модули. Их можно подставить вместо модуля, чтобы повысить эффективность благодаря повторному использованию и централизованной разработке.</p><p>**Где редактируется:** фрагменты редактируются в редакторе фрагментов. Написание кода для их создания или изменения не требуется.</p> |
| [URL](create-page-URL.md) | <p>**Определение:** универсальные указатели ресурсов (URL-адреса) — это адреса, которые указывают на веб-страницы или другие URL-адреса.</p><p>**Где выбирается:** URL-адреса выбираются во всех местах, в которых требуются ссылки между страницами.</p><p>**Где редактируется:** URL-адреса редактируются в редакторе URL-адресов. Написание кода для их создания или изменения не требуется.</p> |
| ОС | <p>**Определение:** активы представляют собой двоичные файлы с расширениями JPG, DOCX, PDF или MPG.</p><p>**Где выбираются:** активы выбираются как свойства модуля для модулей, которым они необходимы.</p><p>**Где редактируются:** активы отправляются, а связанные метаданные редактируются в диспетчере активов.</p> |
