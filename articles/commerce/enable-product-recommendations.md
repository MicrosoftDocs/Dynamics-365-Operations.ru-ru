# <a name="enable-product-recommendations"></a>Включить рекомендации по продуктам

[!include [banner](includes/banner.md)]

В этой теме объясняется, как создать рекомендации продуктов, которые основаны на искусственном интеллекте и машинном обучении (AI-ML), доступном клиентам Microsoft Dynamics 365 Commerce. Дополнительные сведения о списках рекомендаций продуктов см. в разделе [Обзор рекомендаций продуктов](product-recommendations.md).

## <a name="recommendations-pre-check"></a>Предварительная проверка рекомендаций

1. Убедитесь, что у вас есть действующая лицензия Dynamics 365 Commerce Recommendations.
1. Убедитесь, что хранилище объектов связано с учетной записью Azure Data Lake Storage 2 поколения, принадлежащей заказчику. Дополнительные сведения см. в разделе [Убедитесь, что Azure Data Lake Storage было приобретено и успешно проверено в среде](enable-ADLS-environment.md).
1. Убедитесь, что конфигурация удостоверения Azure AD содержит запись для рекомендаций. Дополнительные сведения о том, как выполнить это действие, см. ниже.
1. Убедитесь, что ежедневное обновление хранилища объектов Azure Data Lake Storage 2 поколения было запланировано. Дополнительные сведения см. в разделе [Убедитесь, что обновление хранилища данных было автоматизировано](../fin-ops-core/dev-itpro/data-entities/entity-store-data-lake.md).
1. Включите RetailSale измерения для хранилища объектов. Дополнительные сведения о настройке этого процесса см. в разделе [Работа с мерами](/dynamics365/ai/customer-insights/pm-measures).

После выполнения указанных выше действий вы сможете включить рекомендации.

## <a name="azure-ad-identity-configuration"></a>Конфигурация удостоверения Azure AD

Этот шаг необходим только для клиентов, использующих конфигурацию инфраструктуры как услуги (IaaS). Конфигурация удостоверения Azure AD выполняется автоматически для клиентов на Azure Service Fabric, но рекомендуется проверить правильность настройки параметров.

### <a name="setup"></a>Настройка

1. В Commerce headquarters найдите страницу **приложений Azure Active Directory**.
1. Проверьте, существует ли запись для **RecommendationSystemApplication-1**. Если запись не существует, создайте ее, используя следующую информацию:

    - **Код клиента**: d37b07e8-dd1c-4514-835d-8b918e6f9727
    - **Имя**: RecommendationSystemApplication-1
    - **Код пользователя**: RetailServiceAccount

1. Сохраните изменения и закройте страницу. 

## <a name="turn-on-recommendations"></a>Включение рекомендаций

Чтобы включить рекомендации продуктов, сделайте следующее.

1. В Commerce Headquarters найдите **Управления функциями**.
1. Для просмотра списка доступных функций выберите **Все**. 
1. В поле поиска введите **Рекомендации**.
1. Выберите функцию **Рекомендации по продукту**.
1. В области свойств **Рекомендации продуктов** выберите **Включить**.

![Включение рекомендаций.](./media/FeatureManagement_Recommendations.PNG)

> [!NOTE]
> - Процедура выше запускает процесс создания списков рекомендаций продуктов. До того, как списки можно будет просматривать на POS-терминале или в Dynamics 365 Commerce, может потребоваться несколько часов.
> - Эта конфигурация не включает все возможности рекомендаций. Расширенные функции, такие как персонализированные рекомендации, "Выбрать похожие по виду" и "Выбрать похожие по описанию" управляются с помощью выделенных записей управления функциями. Сведения о включении этих функций в Commerce headquarters см. в разделе [Включение персонализированных рекомендаций](personalized-recommendations.md), [Включение рекомендаций "Выбрать похожие по виду"](shop-similar-looks.md) и [Включение рекомендаций "Выбрать похожие по описанию"](shop-similar-description.md).

## <a name="configure-recommendation-list-parameters"></a>Настройка параметров списка рекомендаций

По умолчанию список рекомендаций продуктов на основе AI-ML предоставляет предлагаемые значения. Предлагаемые по умолчанию значения можно изменить в соответствии с потоком бизнеса. Дополнительные сведения об изменении параметров по умолчанию см. в разделе [Управление результатами рекомендаций продуктов на основе AI-ML](modify-product-recommendation-results.md).

## <a name="include-recommendations-in-e-commerce-experiences"></a>Включить рекомендации в области возможностей электронной коммерции

После включения рекомендаций в Commerce headquarters модули Commerce, используемые для просмотра результатов рекомендаций по работе с электронной коммерцией, готовы к настройке. Дополнительные сведения см. в разделе [Модули семейства продуктов](product-collection-module-overview.md).

## <a name="show-recommendations-on-pos-devices"></a>Отображение рекомендаций на POS-устройствах

После включения рекомендаций в Commerce headquarters необходимо добавить панель рекомендаций на экран управления POS через инструмент макета. Для получения сведений об этом процессе см. [Добавление элемента управления рекомендациями на экране проводки на устройствах POS](add-recommendations-control-pos-screen.md). 

## <a name="enable-personalized-recommendations"></a>Включение персонализированных рекомендаций

В Dynamics 365 Commerce розничные продавцы могут сделать доступными персонализированные рекомендации по продуктам (также называемые персонализацией). Таким образом, персонализированные рекомендации могут быть включены в интернет-магазине и в POS-терминале. Когда функция персонализации включена, система может связать сведения о покупке пользователя и сведения о продукте, чтобы создать индивидуальные рекомендаций по продуктам.

Дополнительные сведения о персонализированных рекомендациях см. в разделе [Включение персонализированных рекомендаций](personalized-recommendations.md).
