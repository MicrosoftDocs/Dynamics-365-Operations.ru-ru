# <a name="audit-synchronization-of-customer-operations-in-headquarters"></a>Аудит синхронизации операций клиентов в Headquarters

[!include [banner](includes/banner.md)]
[!include [banner](includes/preview-banner.md)]

В этой статье объясняется, как проводить аудит синхронизации клиентских операций в Microsoft Dynamics 365 Commerce Headquarters, чтобы помочь в устранении неполадок пользователям сайта.

По мере того как организации начинают внедрять возможность создания и изменения клиентов асинхронно, администраторам сайтов требуется способ просмотра и устранения неполадок операций, основанный на запросах пользователей сайтов или сбоях процессов. В этих сценариях администраторы сайта должны иметь возможность выполнять аудит операций создания и редактирования клиентов и устранять любые сбои с помощью модели самообслуживания.

## <a name="customer-synchronization-status"></a>Статус синхронизации клиентов

Когда вы даете согласие на асинхронный режим управления клиентами и его соответствующие функции, администраторам Commerce headquarters необходимо создать и запланировать повторяющееся пакетное задание для **P-задания**, задание **Синхронизация клиентов и деловых партнеров из асинхронного режима** и задание **1010**, чтобы преобразовать всех асинхронных клиентов в синхронных клиентов в Commerce Headquarters. Синхронизация операций управления клиентами происходит каждый раз, когда выполняются эти задания. Дополнительные сведения см. в разделе [Асинхронный режим создания клиентов](async-customer-mode.md).

В качестве владельца компании вы можете выполнять следующие операции:

- Просмотр всех операций создания или изменения пользователей сайта. Подробные сведения включают статус и отметку времени.
- Операции фильтрации с помощью любого из полей таблицы клиентов и значений для сужения журнала аудита.
- Просматривайте краткие описания изменений вместе со сведениями о состоянии для понимания операций на высоком уровне.
- Для сбоев измените и исправьте проблемные поля, затем сохраните и синхронизируйте конкретные операции клиентов.

### <a name="elements-on-the-customer-synchronization-status-page"></a>Элементы на странице статуса синхронизации клиентов

Чтобы просмотреть список всех операций синхронизации, в Commerce headquarters перейдите в **Commerce и Retail \> Клиенты \> Статус синхронизации клиента**. На следующем рисунке показан пример страницы **Статус синхронизации клиента**.

![Страница состояния синхронизации клиентов в Commerce headquarters.](media/D365-Commerce-Customer-Mgmt-Audi-Async-Operations.png)

По умолчанию список операций синхронизации клиентов с левой стороны страницы **Статус синхронизации клиента** содержит следующие столбцы:

- Название канала
- Счет клиента
- Счет асинхронного клиента
- Метка времени операций
- Статус синхронизации клиента

В верхнем правом углу страницы отображаются сведения о счете клиента, такие как **Счет клиента**, **Операция Retail Async**, **Статус синхронизации клиента** и **Последняя известная ошибка**.

Раздел **Изменение описания** содержит описание типа операции управления клиентами, которая была выполнена (например, создание клиента, обновление счета или сбой синхронизации с подробными сведениями).

Раздел **Атрибуты заказчика** содержит сетку, в которой показаны все атрибуты клиентов вместе со старыми и новыми значениями. Можно изменить этот раздел, если вы хотите изменить значения атрибута клиента для исправления ошибок, а затем выполнить синхронизацию снова.
