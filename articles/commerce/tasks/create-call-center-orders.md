# <a name="create-call-center-orders"></a>Создание заказов центра обработки вызовов

[!include [banner](../includes/banner.md)]

В этой статье содержится пошаговое описание демонстрационной процедуры, в ходе которой пользователь центра обработки вызовов находит клиента, создает новый заказ, находит продукт и получает платеж от заказчика в Microsoft Dynamics 365 Commerce. В этой процедуре используются демонстрационные данные компании USRT, и она предназначена для сотрудников, занимающихся заказами на продажу. 

## <a name="prerequisites"></a>Необходимые условия

Пользователь, выполняющий эту процедуру, должен быть настроен в качестве пользователя центра обработки вызовов. Дополнительно можно опубликовать полугодовой каталог компании Fabrikam по крайней мере с одним кодом источника.

### <a name="add-yourself-as-a-call-center-user"></a>Добавление себя в качестве пользователя центра обработки вызовов

Чтобы добавить себя в качестве пользователя центра обработки вызовов, выполните следующие действия.

1. В Commerce headquarters перейдите в раздел **Розничная торговля и коммерция \> Каналы \> Центры обработки вызовов \> Все центры обработки вызовов**.
1. В поле **Пользователи** выберите пункт **Пользователи канала**.
1. В области действий выберите **Создать**.
1. В поле **Идентификатор пользователя** введите свой идентификатор пользователя.
1. В поле **Имя** введите свое имя пользователя. Имя пользователя может совпадать с идентификатором пользователя.
1. На панели операций выберите **Сохранить**.
1. Вернитесь в раздел **Розничная торговля и коммерция \> Каналы \> Центры обработки вызовов \> Все центры обработки вызовов**.
1. Выбор идентификатора розничного канала центра обработки вызовов.
1. Убедитесь, что для параметра **Включить заполнение заказа** установлено значение **Да**. Если этот параметр не отображается, этот шаг можно пропустить.

## <a name="complete-the-example-call-center-procedure"></a>Выполнение демонстрационной процедуры центра обработки вызовов

Чтобы выполнить демонстрационную процедуру центра обработки вызовов, выполните следующие действия.

1. Перейдите в раздел **Retail и Commerce \> Клиенты \> Обслуживание клиентов**.
1. На вкладке **Поиск клиента** введите условия поиска для поиска клиента. Для этой демонстрационной процедуры введите значение **Karen**.
1. Выберите **Поиск**. Появится диалоговое окно **Поиск клиента**, в котором выведены результаты поиска.
1. Выберите запись клиента **Karen Berg**, имеющую счет клиента с номером **2001**, а затем нажмите кнопку **Выбрать**.
1. В области действий выберите команду **Новый заказ на продажу**.
1. В правой части выберите вкладку **Заголовок**.
1. На экспресс-вкладке **Поставка** в поле **Способ поставки** выберите **99 Стандартная**.

    ![Выбор способа поставки.](../media/Select_Mode_of_Delivery.png)

1. В правой части выберите вкладку **Строки**.
1. В разделе **Строки заказов на продажу** в новой строке для строки новых продаж введите код искомой номенклатуры в поле **Код номенклатуры**. Для данной демонстрационной процедуры введите **81327**, а затем выберите продукт в раскрывающемся списке, чтобы добавить его в заказ на продажу.
1. В поле **Количество** введите количество для продажи.
1. В поле **Код источника** выберите код источника для каталога. Если активные коды источника отсутствуют, можно пропустить этот шаг.
1. В области действий выберите пункт **Завершить**, чтобы зафиксировать платеж клиента. Это действие открывает диалоговое окно **Сводка по заказу на продажу**, в котором отображается общая сумма к оплате. Это действие также запускает расчет всех накладных расходов, таких как отгрузка и обработка, и показывает их в диалоговом окне **Сводка по заказу на продажу**.

    ![Кнопка "Завершить".](../media/Complete_button.png)

1. В диалоговом окне **Сводка по заказу на продажу** на экспресс-вкладке **Платежи** выберите команду **Добавить** для получения платежей.

    ![Диалоговое окно "Сводка по заказу на продажу".](../media/order_summary.png)

1. В диалоговом окне **Ввод сведений о клиентских платежах** в поле **Способ оплаты** выберите способ оплаты. Для этой демонстрационной процедуры выберите значение **Наличные деньги**.
1. В поле **Сумма платежа** введите сумму платежа. В данном примере введите **120,00**, что соответствует сальдо заказа, который отображается в диалоговом окне **Сводка по заказу на продажу**. Ввод этого значения позволит завершить заказ как полностью оплаченный.
1. Нажмите **ОК**.
1. Выберите **Отправить**.
