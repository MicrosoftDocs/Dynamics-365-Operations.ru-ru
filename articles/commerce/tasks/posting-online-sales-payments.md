# <a name="posting-of-online-sales-and-payments"></a>Разноска интерактивных продаж и платежей

[!include[task guide banner](../includes/task-guide-banner.md)]

Эта процедура содержите инструкции по настройке и запуску регулярного пакетного задания для создания заказов на продажу и платежей для проводок интернета-магазина.

Разноска интернет-продаж и платежей выполняется в два этапа.

- Извлечение данных транзакций Commerce интернет-торговли в головной офис.
- Синхронизация заказов для создания заказов на продажу в головном отделении.

Извлечение данных транзакций интернет-торговли может выполнять либо вручную путем запуска P-задания, либо путем создания повторяющегося пакетного задания.

### <a name="manually-running-the-p-job"></a>Запуск P-задания вручную

1. Перейдите в раздел Все рабочие области > ИТ Retail и Commerce.
2. Щелкните "График распределения".
3. Выберите P-0001.
4. Если необходимо, скорректируйте группы баз данных каналов.
5. Щелкните Запустить сейчас.
6. Щелкните Да.

### <a name="scheduling-a-recurring-p-job"></a>Планирование повторяющегося P-задания

1. Перейдите в раздел Все рабочие области > ИТ Retail и Commerce.
2. Щелкните "График распределения".
3. Выберите P-0001.
4. Щелкните "Создать пакетное задание".
5. Щелкните "Выполнять в фоновом режиме".
5. Включите пакетную обработку.
6. Щелкните "Повторение".
7. Выберите параметр "Дата завершения не указана".
8. В поле "Счетчик" введите интервал между запусками в минутах. Обычно это будет 5–10.
9. Нажмите кнопку "OК".
10. Нажмите кнопку "OК".

Заказы могут быть синхронизированы либо путем запуска задания "Синхронизировать заказы" вручную, либо путем создания повторяющегося пакетного задания.

### <a name="manually-running-order-synchronization"></a>Запуск синхронизации заказов вручную 

Выполните следующие действия, чтобы один раз запустить задание "Синхронизировать заказы" вручную.

1. Перейдите "Все рабочие области > Финансовая информация магазина".
2. Щелкните "Синхронизация заказов".
3. В поле "Организационная иерархия" выберите "Магазины по регионам".
    * Выберите конкретный интернет-магазин или узел, если вы хотите создать пакетное задание для группы магазинов.  
    * Нажмите на стрелку для добавления выбора.  
4. Щелкните вкладку "Выполнять в фоновом режиме".
5. Отключение пакетной обработки
6. Щелкните "Повторение".
7. Выбор параметра "Окончание после"
8. В поле "Окончание после" введите 1.
9. Нажмите кнопку "OК".
10. Нажмите кнопку "OК".

### <a name="scheduling-recurring-order-synchronization"></a>Планирование повторяющейся синхронизации заказов

Эта процедура содержите инструкции по настройке и запуску регулярного пакетного задания для создания заказов на продажу и платежей для проводок интернета-магазина. В этой процедуре используется компания с демонстрационными данными USRT.

1. Перейдите "Все рабочие области > Финансовая информация магазина".
2. Щелкните "Синхронизация заказов".
3. В поле "Организационная иерархия" выберите "Магазины по регионам".
    * Выберите конкретный интернет-магазин или узел, если вы хотите создать пакетное задание для группы магазинов.  
    * Нажмите на стрелку для добавления выбора.  
4. Щелкните вкладку "Выполнять в фоновом режиме".
5. Включите пакетную обработку
6. Щелкните "Повторение".
7. Выберите параметр "Дата завершения не указана".
8. В поле "Счетчик" введите интервал между запусками в минутах. Обычно это будет 2–20
9. Нажмите кнопку "OК".
10. Нажмите кнопку "OК".
