# <a name="document-states-and-lifecycle"></a>Состояния и жизненный цикл документа

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

В этом разделе рассматриваются различные состояния документов элементов страницы в Microsoft Dynamics 365 Commerce.

## <a name="document-state-descriptions"></a>Описания состояния документа

В разделе [Элементы страницы](page-elements-overview.md) перечислены различные типы документов в системе управления контентом (CMS). Эти типы документов могут иметь несколько состояний в инструменте разработки. Состояния документов помогают предотвратить конфликты данных и обеспечивать управление версиями. Они определяют, кто может изменять документы, когда документы могут быть изменены и когда изменения могут просматриваться другими пользователями.

В следующей таблице показаны возможные состояния документов элементов страницы в Commerce.

| Состояние документа | Описание |
|---|---|
| Извлеченные | Когда элемент CMS извлечен для вас, он не может быть изменен никаким другим пользователем системы, прошедшим проверку подлинности. Любые изменения, внесенные в элемент, будут видны только вам. |
| Возвращенные | Когда элемент CMS возвращен, все изменения видны другим пользователям системы, прошедших проверку подлинности, и эти пользователи могут затем извлечь элемент и редактировать его. Каждый возврат создает запись версии документа в истории элемента. |
| Опубликованные | Когда элемент CMS опубликован, он перемещается на активный сайт и становится видимым в Интернете для внешних пользователей, не прошедших проверку подлинности. Элементы могут быть опубликованы только в том случае, если они возвращены. |
| Сохранено | Изменения, внесенные в извлеченный элемент CMS, могут быть сохранены в CMS до того, как элемент будет возвращен или опубликован. Эти сохраненные изменения не видны другим пользователям системы, прошедшим проверку подлинности, до тех пор, пока элемент не будет возвращен. Они не видны внешним пользователям до тех пор, пока элемент не будет опубликован. |
| Отменено извлечение | Когда извлеченный элемент CMS отменяется, все сохраненные изменения удаляются, и элемент восстанавливается до последней возвращенной версии. |
