# <a name="document-states-and-lifecycle"></a>Состояния и жизненный цикл документа

[!include [banner](includes/banner.md)]

В этом разделе рассматриваются различные состояния документов элементов страницы в Microsoft Dynamics 365 Commerce.

## <a name="document-state-descriptions"></a>Описания состояния документа

В разделе [Элементы страницы](page-elements-overview.md) перечислены различные типы документов в системе управления контентом (CMS). Эти типы документов могут иметь несколько состояний в инструменте разработки. Состояния документов помогают предотвратить конфликты данных и обеспечивать управление версиями. Они определяют, кто может изменять документы, когда документы могут быть изменены и когда изменения могут просматриваться другими пользователями.

В следующей таблице показаны возможные состояния документов элементов страницы в Commerce.

| Состояние документа      | Действие конструктора сайтов        | описание                                                  |
| ------------------- | -------------------------- | ------------------------------------------------------------ |
| Оформленные         | Выберите **Правка**.           | Соответствующий документ извлечен вами. Если документ находится в этом состоянии, он не может быть изменен другими пользователями системы, прошедшими проверку подлинности, и любые изменения, внесенные в документ, будут видны только вам. |
| Сохранено               | Нажмите **Сохранить**.           | Изменения, внесенные в извлеченный документ, сохраняются в базе данных, но документ еще не был зарегистрирован или опубликован. Сохраненные изменения не видны другим пользователям системы, прошедшим проверку подлинности, до тех пор, пока автор не выберет **Завершить правку**. Они не видны внешним пользователям до тех пор, пока элемент не будет опубликован. |
| Отменено извлечение | Выберите **Отменить изменения**.  | Все изменения в извлеченном документе отбрасываются, и элемент восстанавливается до последней возвращенной версии. |
| Возвращенные          | Выберите **Завершить правку**. | Измененный документ возвращен. Все изменения видны другим пользователям системы, прошедшим проверку подлинности, и эти пользователи могут затем редактировать документ. Каждый возврат создает запись версии документа в истории элемента. |
| Опубликованные           | Выберите **Опубликовать**.        | Документ публикуется, и изменения отправляются на ваш активный сайт и становятся видимыми внешним пользователям. Элементы могут быть опубликованы только в том случае, если они сначала были возвращены путем выбора команды **Завершить правку**. |
