# <a name="provision-a-dynamics-365-commerce-sandbox-environment"></a>Подготовка среды песочницы Dynamics 365 Commerce

[!include [banner](includes/banner.md)]

В этой статье описывается подготовка среды песочницы Microsoft Dynamics 365 Commerce для демонстрационного использования с помощью встроенных демонстрационных данных. Процесс настройки рабочей среды аналогичен, но более сложен, поскольку в демонстрационных данных уже выполнены многие необходимые условия для среды песочницы.

Прежде чем начать, рекомендуется выполнить быструю проверку данной статьи, чтобы понять, что именно требуется для этого процесса.

Для успешной подготовки среды песочницы Commerce необходимо указать некоторые параметры для среды и Commerce Scale Unit (CSU), которые будут использоваться при подготовке Commerce позже. Инструкции в этой статье описывают все необходимые шаги, которые вы должны выполнить, и параметры, которые вы должны использовать.

После успешного предоставления среды Commerce необходимо выполнить несколько шагов после подготовки, чтобы подготовить ее к использованию. Некоторые шаги являются необязательными, в зависимости от того, какие аспекты системы вы хотите использовать. Вы всегда можете выполнить дополнительные шаги позже.

Для получения информации о том, как настроить среду Commerce после подготовки, см. в [Настройка среды песочницы Commerce](cpe-post-provisioning.md). Для сведений о том, как настроить дополнительные функции для среды Commerce, см. [Настройка дополнительных функций для среды песочницы Commerce](cpe-optional-features.md).

## <a name="prerequisites"></a>Необходимые условия

Для того, чтобы подготовить среду песочницы Commerce, необходимо предусмотреть следующие предварительные условия:

- У вас есть доступ к порталу Lifecycle Services Microsoft Dynamics (LCS)
- Вы являетесь существующим партнером или клиентом Microsoft Dynamics 365 и создали проект реализации, который уже создан и доступен для использования в LCS.  
- Вы создали группу безопасности Azure Active Directory (Azure AD), которую можно использовать в качестве группы администраторов системы Commerce, и у вас есть ее идентификатор.
- Вы создали группу безопасности Azure AD, которую можно использовать в качестве группы модератора оценок и рецензий, и у вас есть ее идентификатор. (Эта группа безопасности может быть такой же, как группа администрирования системы Commerce.)
- Вы развернули экземпляр Headquarters в LCS. Дополнительные сведения см. в разделе [Развертывание новой среды](/dynamics365/fin-ops-core/dev-itpro/deployment/deployenvironment-newinfrastructure).

Обратите внимание, что этот список не является исчерпывающим. При возникновении каких-либо проблем обратитесь за помощью к партнеру корпорации Майкрософт.

## <a name="provision-your-commerce-environment"></a>Подготовка среды Commerce

Следующие процедуры объясняют, как подготовить среду Commerce. После успешного завершения этих шагов среда Commerce будет готова к настройке. Все шаги, описанные ниже, выполняются на портале LCS.

### <a name="initialize-the-commerce-scale-unit-cloud"></a>Инициализация Commerce Scale Unit (облако)

Чтобы инициализировать CSU, выполните следующие действия.

1. В LCS выберите среду из списка.
1. В представлении **СРЕДЫ** справа выберите **Полные сведения**. Появится представление сведений о среде.
1. В разделе **Управление средой** в пункте **ФУНКЦИИ СРЕДЫ** выберите **Управление**.
1. На вкладке **Commerce Scale Units** выберите **Инициализация**. Появится представление **Добавить единицу масштабирования**.
1. В поле **РЕГИОН** выберите область, совпадающую с областью, в которую вы развернули среду, или близкий к ней.
1. В раскрывающемся списке **Версия** выберите доступную последнюю версию.
1. Выберите **Инициализация**.
1. В диалоговом окне предупреждение, в котором предлагается подтвердить инициализацию Commerce Scale Unit, выберите **Да**. CSU теперь поставлен в очередь для подготовки.
1. Перед продолжением убедитесь, что состояние CSU — **УСПЕХ**. Инициализация занимает примерно два-пять часов.

Если вы не можете найти ссылку **Управление** в представлении сведений о среде, обратитесь за помощью к своему контакту в Майкрософт.

### <a name="initialize-e-commerce"></a>Инициализация электронной коммерции

Чтобы инициализировать Commerce, выполните следующие действия.

1. На вкладке **Электронная коммерция** выберите **Настройка**.
1. В поле **Имя среды электронной коммерции** введите имя. Обратите внимание, что это имя появится в некоторых URL-адресах, указывающих на экземпляр электронной коммерции.
1. В поле **Имя Commerce Scale Unit** выберите CSU в списке. (В списке должен быть только один вариант.)

    Поле **География электронной коммерции** настраивается автоматически.

1. Выберите **Далее** для продолжения.
1. В поле **Поддерживаемые имена узлов** введите любой допустимый домен, например `www.fabrikam.com`.
1. В поле **Группа безопасности AAD для системного администратора** введите первые несколько букв имени группы безопасности, которую необходимо использовать, затем выберите символ лупы для просмотра результатов поиска. Выберите правильную группу безопасности в списке.
1.  В поле **Группа безопасности AAD для модератора оценок и отзывов** введите первые несколько букв имени группы безопасности, которую необходимо использовать, затем выберите символ лупы для просмотра результатов поиска. Выберите правильную группу безопасности в списке.
1. Оставьте для параметра **Включить службу оценок и отзывов** значение **Да**.
1. Выберите **Инициализация**. Отобразится представление **Управление Commerce**, в котором будет выбрана вкладка **Электронная коммерция**. Инициализация электронной коммерции начата.
1. Перед продолжением необходимо подождать, пока состояние инициализации Commerce не станет **УСПЕШНАЯ ИНИЦИАЛИЗАЦИЯ**.
1. В **Ссылки** в нижнем правом углу запишите URL-адреса для следующих ссылок:

    * **Сайт электронной коммерции** — ссылка на корневой узел вашего сайта электронной коммерции.
    * **Конфигуратор сайта электронной коммерции** — ссылка на средство управления сайтом.
