# <a name="restrict-access-to-a-storefront-during-testing-or-development"></a>Ограничение доступа к витрине в ходе тестирования или разработки

[!include [banner](../../includes/banner.md)]

В этой теме объясняется, как ограничить доступ к витрине в Microsoft Dynamics 365 Commerce во время внутреннего испытания или разработки.

## <a name="description"></a>описание

Во время внутреннего тестирования или активной разработки может возникнуть необходимость ограничить доступ к вашему сайту, чтобы запретить внешним пользователям доступ к опубликованным страницам витрин.

## <a name="resolution"></a>Приказ

### <a name="set-up-azure-ad-b2c-by-using-standard-user-flows"></a>Настройте Azure AD B2C с помощью стандартных потоков пользователей

Сведения о настройке Azure Active Directory B2C (Azure AD B2C) для вашего сайта электронной коммерции см. [Настройка клиента в коммерческих организациях B2C](../set-up-b2c-tenant.md).

### <a name="restrict-user-access-to-storefront-pages-and-block-the-creation-of-new-users"></a>Ограничьте доступ пользователя к страницам витрин и запретите создание новых пользователей

Чтобы ограничить доступ пользователей к страницам витрин в Commerce Site Builder, выполните следующие действия.

1. Перейдите на ваш сайт.
1. Выберите **страницы**, а затем выберите страницу, для которой требуется ограничить доступ пользователя.
1. Выберите модуль или ячейку фрагмента, а затем выберите **Изменить**.
1. В области свойств выберите **требуется вход в систему?**, а затем нажмите кнопку **Завершить правку**.
1. Выберите **Опубликовать**.

Чтобы заблокировать создание новых пользователей в Azure AD, выполните следующие действия.

1. Перейдите в раздел [порталу Azure](https://portal.azure.com/).
1. Выберите приложение Azure AD B2C, созданное для доступа к сайту.
1. В левой области переходов выберите **Потоки пользователей**.
1. В верхней части страницы **Потоки пользователей** выберите **Создать поток пользователей**.
1. На странице **Выбор типа потока пользователя** выберите **Предварительный просмотр**.
1. В середине страницы выберите **Выполните вход v2**. Затем выполните шаги, описанные в разделе [Настройка клиента B2C в Commerce](../set-up-b2c-tenant.md), чтобы настроить поток в качестве стандартного потока пользователей узла для Azure AD B2C во время тестирования или разработки.
