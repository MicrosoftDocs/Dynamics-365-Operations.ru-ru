# <a name="issues-with-the-data-reconciliation-of-a-z-report"></a>Проблемы с выверкой данных Z-отчета

[!include [banner](../../includes/banner.md)]

В этой статье содержатся инструкции по устранению неполадок, которые могут помочь при возникновении проблем с выверкой данных Z-отчета в Microsoft Dynamics 365 Commerce headquarters. В нем описываются проблемы, с которыми пользователи могут столкнуться при выверке данных, возможные основные причины и решения, которые могут помочь предотвратить будущие повторения.

## <a name="description"></a>Наименование

- Имеется несоответствие между суммами, отображаемыми в Z-отчете, и итогами, которые отображаются в расчетной выписке.
- Проводка в headquarters имеет неверное количество номенклатур строки, либо имеется несоответствие итога по строке и итоговой суммы для проводки.
- Число проводок, которые отображаются в смене в headquarters, меньше числа проводок в Z-отчете.
- Разноска журнала операций не выполнена по одной из предыдущих причин.

### <a name="root-cause"></a>Основная причина

Наиболее распространенной основной причиной ранее описанных симптомов является создание дубликатов идентификаторов проводок в базе данных канала. Дубликаты кодов проводок могут создаваться по следующим причинам:

- Повреждение хранилища локальной базы данных Modern POS (MPOS).
- MPOS имеет некоторые проводки в автономном режиме и повторно активируются с помощью учетной записи, которая не имеет доступа к автономной базе данных.
- Возникла проблема с настройкой, которая связана с созданием кодов проводок.

## <a name="resolution"></a>Решение

Обычно для создания последовательных кодов проводок в Commerce используется номерная серия. Если система не может определить, что номерная серия использовалась по какой-либо причине, создается дубликат кода проводки. 

Чтобы устранить проблему дублирования кода проводки, создайте запрос в службу поддержки для проверки возможности исправления данных проводок. В некоторых случаях, например, если в headquarters отсутствуют потери данных, исправление данных может быть невозможно или не требоваться.

Чтобы предотвратить возникновение этой проблемы в будущем, необходимо включить функцию **Включить новый код проводки, чтобы избежать дублирования кодов проводок** в headquarters. Эта функция была введена в версии Commerce 10.0.19. Это помогает предотвращать создание последовательных кодов проводок, гарантируя, что для каждой проводки создается уникальный код проводки. Дополнительные сведения об этой функции см. в разделе [Предотвращение дублирования кодов проводки](../channel-setup-retail.md#ensure-unique-transaction-ids).
