# <a name="refundable-charges-are-miscalculated-based-on-the-quantity-returned"></a>Возмещаемые расходы неправильно рассчитываются на основе возвращенного количества

[!include [banner](../../includes/banner.md)]

В этой теме содержатся указания по устранению неполадок, которые могут помочь, когда кассиры видят неправильные возмещаемые расходы в POS-терминале для количества возвращенных номенклатур.

## <a name="description"></a>Описание

Клиент возвращает частичное количество номенклатур, которые были приобретены для заказа на продажу, который имеет возмещаемые накладные расходы на уровне строки. Однако вместо отображения частичного возмещения POS-терминал отображает все расходы как возмещаемые.

Например, клиент покупает пять номенклатур по 5 долларов США за штуку, общая сумма накладных расходов 25 долларов США. Затем клиент возвращает три из пяти номенклатур. Однако на POS-терминале кассиру отображаются возмещаемые расходы 25 долларов США вместо ожидаемых возмещаемых расходов 15 долларов США за три возвращенные номенклатуры.

## <a name="resolution"></a>Решение

### <a name="turn-on-the-enable-refunding-charges-based-on-the-refunded-quantity-feature"></a>Включение функции "Включить возмещение расходов на основе возвращаемого количества"

В Microsoft Dynamics 365 Commerce версии 10.0.26 функция **Включить возмещение расходов на основе возвращаемого количества** позволяет рассчитать возмещаемые расходы на уровне строки на основе количества возвращенных номенклатур.

Чтобы включить функцию **Включить возмещение расходов на основе возвращаемого количества** в Commerce headquarters, выполните следующие действия.

1. Откройте рабочую область **Управление функциями** (**Системный администратор \> Рабочие области \> Управление функциями**).
1. В списке доступных функций выполните поиск и выберите функцию **Включить возмещение расходов на основе возвращаемого количества**.
1. В правой области выберите **Включить сейчас**.
