# <a name="choose-between-store-commerce-and-cloud-pos"></a>Выбор между Store Commerce и Cloud POS

[!include [banner](includes/banner.md)]

В этой теме объясняются основные различия между Store Commerce и Cloud POS, а также описываются различные факторы, которые должны учитывать розничные продавцы, внедряющие Dynamics 365 Commerce, чтобы помочь им в оптимальном выборе для своих требований. В ней также приводятся дополнительные общие рекомендации для специалистов по внедрению, а также сведения по факторам, которые они должны учитывать при развертывании Dynamics 365 Commerce. Изучив и соблюдая эти рекомендации как часть процесса развертывания, специалисты по внедрению смогут избежать проблем, которые могут повлиять на удовлетворенность пользователей или производительность.

## <a name="insights"></a>Аналитика

Commerce обеспечивает широкий диапазон параметров развертывания и топологии. Таким образом предприятия розничной торговли могут выбрать компоненты и конфигурацию, наилучшим образом соответствующие требованиям их бизнеса и технологии. Одним из аспектов внедрения, который требует тщательного анализа, являются выбор платформы и форм-фактор для компонента POS.

### <a name="pos-platform-and-form-factor-considerations"></a>Анализ платформы и форм-фактора POS

Commerce поддерживает следующие параметры POS:

- Store Commerce для Microsoft Windows
- Store Commerce для iOS и Android
- Cloud POS (CPOS), который поддерживает браузеры Microsoft Edge и Google Chrome
- Modern POS (MPOS) для Microsoft Windows (MPOS станет устаревшим в октябре 2023 года.) 

Во всех случаях POS (Store Commerce и CPOS) совместно используют один и тот же код ядра приложения. Этот факт важен по следующим причинам:

- Пользовательского интерфейса (UI) единообразен независимо от платформы или форм-фактора.
- Большая часть функциональных возможностей одинакова, независимо от платформы или форм-фактора. Однако существуют некоторые важные различия. Эти отличия отражены в этом разделе.
- В каждом магазине варианты POS могут комбинироваться и работать параллельно. Например, для основных ККМ предприятие розничной торговли может использовать Store Commerce на компьютерах под управлением Windows. Однако с дополнение к этим ККМ предприятие розничной торговли может использовать терминалы на основе веб-браузера или мобильных устройств.
- Настройки и расширения можно легко использовать для разных платформ и форм-факторов. Поскольку основной код приложения является общим, большая часть настроек может быть выполнена один раз, а не несколько раз.

### <a name="store-commerce-vs-cpos"></a>Сравнение Store Commerce и CPOS

Несмотря на то, что Store Commerce и CPOS во многом совпадают, существуют важные различия, которые необходимо понимать.

#### <a name="store-commerce"></a>Store Commerce

Store Commerce — это классическое приложение, которое устанавливается и обслуживается на устройстве.

- **Windows** — приложение Store Commerce для Windows содержит весь код приложения, среду Commerce Runtime (CRT) и Hardware Station (HWS).
- **iOS/Android** — на этих платформах приложение выполняет роль узла для кода приложения CPOS. Другими словами, код приложения поступает с сервера CPOS, который размещен в Commerce Scale Unit. Дополнительные сведения см. в разделе [Общие сведения о Commerce Scale Unit](dev-itpro/retail-store-system-begin.md).

#### <a name="cpos"></a>CPOS

Так как CPOS выполняется в браузере, приложение не установлено на устройстве. Вместо этого браузер обращается к коду приложения на сервере CPOS. Таким образом, CPOS не может напрямую получать доступ к оборудования POS-терминала или работать в автономном режиме.

### <a name="store-deployment-considerations"></a>Вопросы, связанные с развертыванием в магазине

В дополнение к платформе и форм-фактору, предприятиям розничной торговли также необходимо выбрать вариант развертывания в магазине. В следующей таблице представлены конфигурации, доступные на каждого варианта POS.

| Приложение POS            | Commerce Scale Unit | Доступно в автономном режиме | Поддержка локальных HWS |
|----------------------------|---------------------|-------------------|-------------------|
| Store Commerce для Windows | Облако или RSSU       | Да               | Да               |
| Store Commerce для Android | Облако или RSSU       | Нет                | Да               |
| Store Commerce для iOS     | Облако или RSSU       | Нет                | Нет                |
| Cloud POS                  | Облако или RSSU       | Нет                | Нет                |

#### <a name="commerce-scale-unit"></a>Commerce Scale Unit

Commerce Scale Unit — это компонент, в котором размещается CRT. Среда CRT содержит всю бизнес-логику, которая используется POS, и она предоставляет доступ к базе данных канала. Пока они подключены к сети, все клиенты POS в магазине используют Commerce Scale Unit. Commerce Scale Unit можно развернуть либо в облаке, либо в магазине.

#### <a name="offline-mode"></a>Автономный режим

Store Commerce для Windows поддерживает автономный режим. В автономном режиме POS-терминал может продолжать обрабатывать операции продажи, даже если он отключен от Commerce Scale Unit. Затем он может быть синхронизирован с базой данных канала при возобновлении подключения к сети. Store Commerce использует свой собственный внедренный экземпляр среды CRT и временно использует свой собственный локальный источник данных (автономную базу данных SQL Server). Дополнительные сведения о функциональной возможности работы в автономном режиме см. в разделе [Функциональность POS в автономном режиме](pos-offline-functionality.md).

### <a name="pos-peripheralhardware-considerations"></a>Вопросы, связанные с периферийными устройствами и оборудованием POS-терминала

Предприятия розничной торговли также должны учитывать, как POS-терминал будет осуществлять доступ к устройствам и периферийному оборудованию, таким как принтеры, денежные ящики и платежные терминалы. Станции оборудования могут быть закреплены за конкретной ККМ POS или использоваться совместно контрольно-кассовыми машинами в магазине.

| Приложение POS            | Локальное HWS OPOS | Сетевые периферийные устройства | Поддержка общих HWS |
|----------------------------|----------------|---------------------|--------------------|
| Store Commerce для Windows | Да            | Да                 | Да                |
| Store Commerce для Android | Нет             | Да                 | Да                |
| Store Commerce для iOS     | Нет             | Нет                  | Да                |
| Cloud POS                  | Нет             | Нет                  | Да                |

Дополнительные сведения о том, как установить станцию оборудования, см. в разделе [Конфигурация и установка станции оборудования Retail](retail-hardware-station-configuration-installation.md).

## <a name="implementation-considerations"></a>Вопросы реализации

При планировании реализации POS в магазинах учитывайте следующую информацию:

- **Функциональные требования** — ключевые бизнес-процессы и возможности одинаковы, независимо от платформы, форм-фактора или топологии развертывания. Таким образом, большинству предприятий розничной торговли нет требуется учитывать функциональные требования, когда они планируют реализацию.
- **Подключение** — доступность сети (глобальной сети \[WAN\] и локальной сети \[LAN\]) является основным фактором, который требует тщательного анализа. Все преимущества не требующих места для установки облачных решений с точки зрения простоты и затрат будут потеряны, если система будет недоступна для критически важных бизнес-процессов.

    Кроме случаев, когда подключение для данного устройства очень надежное и устойчивое или когда некоторое время простоя допустимо для розничной торговли, рекомендуется один из следующих вариантов:

    - Используйте Store Commerce в Windows и включите автономный режим.
    - Развертывание локальной версии Commerce Scale Unit.

    Эти два варианта не являются взаимно исключающими. Для наиболее надежной топологии предприятия розничной торговли могут развернуть локальный RSSU, чтобы уменьшить зависимость от подключения к Интернету или доступности Azure; также они могут развернуть ККМ POS, в который включен автономный режим, если имеется проблема с локальным сервером или сетью.

- **Оборудование/периферийные устройства** — один важный аспект системы Retail POS заключается в возможности использования периферийного оборудования POS, такого как принтеры, денежные ящики и платежные терминалы. Несмотря на то, что все доступные варианты POS могут использовать периферийные устройства, только Store Commerce для Windows поддерживает их непосредственно. Для других приложений необходима одна или несколько станций оборудования. Хотя такой подход добавляет гибкости, дополнительные компоненты необходимо развертывать, настраивать и обслуживать.
- **Требования к системе** — требования к системе для POS-приложения различаются. Обязательно проверьте последние сведения, прежде чем делать выбор. Например, так как приложение CPOS выполняется в браузере, оно поддерживает широкий диапазон операционных систем. Дополнительные сведения о требованиях к системе см. в документе [Требования к системе для облачных развертываний](../fin-ops-core/fin-ops/get-started/system-requirements.md).
- **Развертывание и обслуживание** — требования к сложности развертывания и обслуживания могут различаться в зависимости от выбора приложения и развертывания. Например, для развертывания CPOS с размещением в облаке не требуется устанавливать и обновлять каждое устройство. Таким образом, этот подход значительно уменьшает сложность и стоимость. Однако если выполнить развертывание Store Commerce на каждой ККМ и включить автономный режим, а также развернуть общую станцию оборудования, значительно увеличивается количество конечных точек, которыми требуется управлять.
