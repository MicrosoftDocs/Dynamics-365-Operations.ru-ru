# <a name="set-up-a-b2b-e-commerce-site"></a>Настройка сайта электронной коммерции B2B

[!include [banner](../../includes/banner.md)]

Сайты электронной коммерции бизнес-бизнес (B2B) предоставляют некоторые ключевые возможности, которые оптимизируют рабочий процесс для пользователей B2B. В этом разделе описывается, как настроить сайт электронной коммерции B2B в Microsoft Dynamics 365 Commerce. Он проходит через модули и настройки сайта, которые должны быть настроены для включения сценариев, связанных с B2B.

## <a name="prerequisites"></a>Необходимые условия

- Для настройки сайта электронной коммерции B2B необходимо включить и настроить отдельные функции в Commerce headquarters, как описано в этом разделе.
- Основные возможности, такие как обнаружение продукта, страницы сведений о продукте, корзина и оформление заказа, обеспечиваются теми же модулями, которые используются для сайтов электронной коммерции для сценариев "бизнес-потребитель" (B2C). Авторы сайтов должны ознакомиться со всеми поддерживаемыми модулями Dynamics 365 Commerce. Дополнительные сведения см. в разделе [Обзор библиотеки модулей](../starter-kit-overview.md).
- В этом разделе предполагается, что авторы сайта имеют представление об основах конфигуратора сайта Commerce, шаблонов, фрагментов и страниц, чтобы они могли включить возможности B2B для сайтов электронной коммерции.

## <a name="site-level-settings"></a>Настройки уровня сайта

Доступ к параметрам уровня сайта в конструкторе сайтов можно получить в разделе **Параметры сайта \> Расширения**. Следующие два параметра уровня сайта применимы к сценариям B2B:

- **Включить платежи счетов клиентов** — это свойство позволяет пользователям оплачивать заказы, используя счета клиентов. Доступные значения: **Включены для клиентов B2B**, **Включены для клиентов B2C**, **Включены для всех клиентов** и **Отключены для всех клиентов**. Если сайт B2B поддерживает счета клиентов, следует выбрать **Включены для клиентов B2C**.
- **Включить пределы количества заказов** — это свойство позволяет установить ограничения на число номенклатур, которые могут быть заказаны для каждого продукта или категории. Доступные значения: **Включены для клиентов B2B**, **Включены для клиентов B2C**, **Включены для всех клиентов** и **Отключены для всех клиентов**.

> [!NOTE]
> При обновлении библиотеки модулей до последней версии необходимо выполнить дополнительные шаги, чтобы убедиться, что ранее описанные настройки сайта доступны в вашей среде. Дополнительные сведения см. в разделе [Обновление файла app.settings.json](../e-commerce-extensibility/sdk-updates.md#update-the-appsettingsjson-file).

## <a name="create-business-partner-sign-up-pages"></a>Создание страниц регистрации для деловых партнеров

Чтобы стать деловым партнером, пользователи должны сначала представить запрос по деловому партнеру. Ссылка на страницу запроса делового партнера будет доступна на домашней странице B2B, чтобы пользователи могли начать процесс. После того как пользователи отправили запрос делового партнерства, они получат подтверждение, что запрос был отправлен. 

### <a name="create-a-business-partner-request-page"></a>Создание страницы запроса делового партнерства

Модуль **Регистрация партнера** на странице запроса делового партнерства используется для создания запросов пользователей, которые хотят стать деловыми партнерами. Этот модуль позволяет собирать сведения о пользователе, необходимые для процесса регистрации. Кроме того, модуль **Бизнес-адрес организации** используется для записи адреса пользователя.

Чтобы установить и настроить страницу запроса делового партнерства в конфигураторе сайта выполните следующие действия.

1. Создайте шаблон с именем **Регистрация**. Этот шаблон должен включать следующие модули:

    - Регистрация для партнеров
    - Навигация
    - Верхний колонтитул
    - Нижний колонтитул
    - Блок содержимого
    - Текстовый блок
    - Коллекция продуктов

1. Используйте шаблон **Регистрация** для создания страницы с именем **Запрос делового партнера**.
1. В гнезде **Заголовок** добавьте фрагмент заголовка, предварительно настроенный с заголовком сайта.
1. В гнезде **Нижний колонтитул** добавьте фрагмент нижнего колонтитула, предварительно настроенный с нижним колонтитулом сайта.
1. В гнезде **Главное** добавьте модуль **Контейнер**. В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**.
1. В слоте **Контейнер** добавьте модуль **Иерархическая навигация**. В области свойств модуля в области **Ссылки** настройте ссылку на главную страницу и введите **Главная** в качестве текста ссылки.
1. В гнезде **Контейнер** добавьте модуль **Регистрация партнера** под модулем **Иерархическая навигация**. В области свойств модуля в области **Заголовок** введите **Станьте деловым партнером**.
1. В слоте **Регистрация партнера** добавьте модуль **Бизнес-адрес организации**.
1. В гнезде **Контейнер** добавьте модуль **Текстовый блок** под модулем **Регистрация партнера**. Здесь можно определить некоторые условия для процесса регистрации.
1. Выберите **Сохранить**, затем выберите **Предварительный просмотр**, чтобы просмотреть страницу.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.
1. Опубликуйте URL-адрес для страницы.

### <a name="create-a-business-partner-request-confirmation-page"></a>Создание страницы подтверждения запроса делового партнерства

После отправки запроса по деловому партнеру для подтверждения отправки пользователю должно быть показана страница подтверждения. 

Чтобы установить и настроить страницу подтверждения запроса в конфигураторе сайта выполните следующие действия.

1. Используйте шаблон **Регистрация**, созданный ранее, для создания страницы с именем **Подтверждение запрос партнера**.
1. В гнезде **Заголовок** добавьте фрагмент заголовка, предварительно настроенный с заголовком сайта.
1. В гнезде **Нижний колонтитул** добавьте фрагмент нижнего колонтитула, предварительно настроенный с нижним колонтитулом сайта.
1. В гнезде **Главное** добавьте модуль **Контейнер**. В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**.
1. В гнезде **Контейнер** добавьте модуль **Блок содержимого**. В области свойств модуля в области **Заголовок** введите **Запрос отправлен**. В поле **Форматированный текст** введите **Ваш запрос отправлен**. В области **Ссылки** настройте ссылку на главную страницу и введите **Вернуться к покупкам** в качестве текста ссылки.
1. Добавьте другой модуль **Контейнер** и добавьте в него модуль **Семейство продуктов**.
1. Настройте модуль **Семейство продуктов** с рекомендацией или списком категорий, которые необходимо показать на странице.
1. Выберите **Сохранить**, затем выберите **Предварительный просмотр**, чтобы просмотреть страницу.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.
1. Опубликуйте URL-адрес для страницы.

Чтобы добавить ссылку на страницу подтверждения запроса, в конфигураторе сайта выполните следующие действия.

1. Перейдите на страницу **Запроса делового партнера**, которая была создана ранее, и выберите **Правка**. 
1. Выберите гнездо модуля **Регистрация партнера**. В области свойств в области **Ссылка на страницу подтверждения регистрации** настройте ссылку на страницу запроса делового партнера, созданную ранее. 
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.

### <a name="add-a-business-partner-request-link-to-the-home-page"></a>Добавление запроса делового партнера на главную страницу

После создания страниц регистрации и подтверждения делового партнера необходимо сделать страницу регистрации доступной с помощью ссылки на главной странице. Эту задачу можно выполнить, добавив ссылку в любой модуль **Блок содержимого** на главной странице.

Чтобы добавить ссылку запроса делового партнера на главную страницу, в конфигураторе сайта выполните следующие действия.

1. Перейдите на главную страницу для вашего сайта и выберите **Изменить**.
1. Выберите гнездо модуля **Блок содержимого**. В области свойств модуля в области **Ссылки** настройте ссылку на страницу запроса делового партнера, которая была создана ранее, и введите **Зарегистрируйтесь, чтобы стать деловым партнером** или аналогичный текст. Добавьте соответствующее изображение.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.

## <a name="account-management-landing-page"></a>Целевая страница управления учетной записью

Целевая страница управления учетными записями включает все сведения для управления учетными записями, необходимые для сайтов электронной коммерции B2B и B2C. Эту страницу могут просматривать только пользователи, которые вошли в систему.

Чтобы создать и настроить главную страницу управления учетными записями B2B, в конфигураторе сайта выполните следующие действия.

1. Создайте шаблон с именем **Управление учетными записями**. Этот шаблон должен включать следующие модули:

    - Верхний колонтитул
    - Нижний колонтитул
    - Навигация
    - Плитка приветствия для учетной записи
    - Общая плитка учетной записи
    - Плитка адреса учетной записи
    - Плитка списка пожеланий учетной записи
    - Плитка адреса учетной записи
    - Плитка лояльности учетной записи
    - Плитка сальдо клиента учетной записи
    - Плитка шаблонов заказов учетной записи
    - Пользователи организации
    - Список бизнес-организаций
    - Сальдо счета клиента
    - Строки шаблона заказа
    - Список шаблонов заказов
    - Плитка накладных учетных записей
    - Список накладных
    - Сведения о накладной

1. Используйте шаблон **Управление учетными записями** для создания страницы с именем **Моя учетная запись**.
1. В гнезде **Заголовок** добавьте фрагмент заголовка, предварительно настроенный с заголовком сайта.
1. В гнезде **Нижний колонтитул** добавьте фрагмент нижнего колонтитула, предварительно настроенный с нижним колонтитулом сайта.
1. В гнезде **Главное** добавьте модуль **Контейнер**. В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. 
1. В слоте **Контейнер** добавьте модуль **Иерархическая навигация**. В области свойств модуля в области **Ссылки** настройте ссылку на главную страницу и введите **Главная** в качестве текста ссылки.
1. В гнезде **Контейнер** добавьте модуль **Плитка приветствия**. В области свойств модуля в области **Заголовок** введите **Добро пожаловать**.
1. В гнезде **Главный** добавьте еще один модуль **Контейнер** (**Контейнер 2**). В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. Установите для параметра **Отображаются дочерние** значение **Два**. 
1. В гнезде **Контейнер 2** добавьте модуль **Универсальная плитка учетной записи**. В области свойств модуля в области **Заголовок** введите **Мой профиль**. В области **Ссылки** настройте ссылку на страницу **Мой профиль**. 
1. В гнезде **Контейнер 2** добавьте еще один модуль **Универсальная плитка учетной записи**. В области свойств модуля в области **Заголовок** введите **История заказов**. В области **Ссылки** настройте ссылку на страницу истории заказов.
1. В гнезде **Главный** добавьте еще один модуль **Контейнер** (**Контейнер 3**). В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. Установите для параметра **Отображаются дочерние** значение **Два**. 
1. В гнезде **Контейнер 3** добавьте модуль **Плитка адреса учетной записи**. В области свойств модуля в области **Заголовок** введите **Мой адрес**. В области **Ссылки** настройте ссылку на страницу **Мой адрес**. 
1. В гнезде **Контейнер 3** добавьте модуль **Плитка списка желаний учетной записи**. В области свойств модуля в области **Заголовок** введите **Мой список желаний**. В области **Ссылки** настройте ссылку на страницу **Мой список желаний**.
1. В гнезде **Главный** добавьте еще один модуль **Контейнер** (**Контейнер 4**). В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. Установите для параметра **Отображаются дочерние** значение **Два**. 
1. В гнезде **Контейнер 4** добавьте модуль **Пользователи организации**. В области свойств модуля в области **Заголовок** введите **Пользователи организации**. 
1. В гнезде **Контейнер 4** добавьте модуль **Плитка сальдо клиента учетной записи**. В области свойств модуля в области **Заголовок** введите **Кредит учетной записи**. 
1. В гнезде **Главный** добавьте еще один модуль **Контейнер** (**Контейнер 5**). В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. Установите для параметра **Отображаются дочерние** значение **Два**. 
1. В модуле **Контейнер 5** добавьте модуль **Плитка шаблонов заказов учетной записи**. В области свойств модуля в области **Заголовок** введите **Шаблоны заказов**. 
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.

> [!NOTE] 
> Некоторые из разделов, добавленные на шагах с 13 по 18, не будут отображаться на холсте "что вы видите, то и получаете" (WYSIWIG) конфигуратора сайтов, так как для них требуется учетная запись B2B с выполненным входом.

## <a name="create-and-configure-customer-balance-pages-and-modules"></a>Создание и настройка страниц и модулей сальдо клиента 

Учетные записи клиентов могут использоваться для оплаты по заказам. Доступное сальдо на счету клиента можно просмотреть на странице управления счетами пользователей.

### <a name="create-a-customer-balance-page"></a>Создание страницы сальдо клиента 

Чтобы выполнившие вход пользователи B2B могли просмотреть свое сальдо клиента, необходимо создать страницу сальдо клиента. 

Чтобы создать страницу сальдо клиента, в конструкторе сайтов выполните следующие действия.

1. Используйте созданный ранее шаблон **Управление счетами**, чтобы создать страницу с именем **Сальдо клиента**.
1. В гнезде **Заголовок** добавьте фрагмент заголовка, предварительно настроенный с заголовком сайта.
1. В гнезде **Нижний колонтитул** добавьте фрагмент нижнего колонтитула, предварительно настроенный с нижним колонтитулом сайта.
1. В гнезде **Главный** добавьте еще один модуль **Контейнер** (**Контейнер 3**). В области свойств модуля установите значение **Ширина** как **Вписать в контейнер**. Установите для параметра **Отображаются дочерние** значение **Два**.
1. В слоте **Контейнер** добавьте модуль **Иерархическая навигация**. В области свойств модуля в области **Ссылки** настройте ссылку на целевую страницу управления счетами и введите **Мой счет** в качестве текста ссылки.
1. В гнезде **Контейнер** добавьте модуль **Сальдо счета клиента**. В области свойств модуля в области **Заголовок** введите **Сальдо счета**.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.
1. Опубликуйте URL-адрес для страницы.
1. Перейдите на целевую страницу управления счетами (**Мой счет**), которая была создана ранее.
1. В области свойств модуля **Плитка сальдо клиента учетной записи** добавьте ссылку на страницу сальдо клиента. 
1. Сохраните и опубликуйте страницу.

Теперь страница сальдо клиента создана, и пользователи могут получить доступ к ней с целевой страницы управления счетами.

### <a name="configure-a-checkout-page-so-that-the-customer-balance-can-be-used-as-a-form-of-payment"></a>Настройка страницы оформления заказа, чтобы сальдо клиента можно было использовать как форму платежа

Модуль **Платеж со счета клиента** требуется, чтобы разрешить использование сальдо клиента в качестве формы платежа. Этот модуль следует добавить на страницу оформления заказа как форму платежа. Сведения о настройке страницы оформления заказа и о модулях, которые необходимы для оформления заказа, включая все сведения об оплате, см. в разделе [Модуль оформления заказа](../add-checkout-module.md).

После настройки страницы оформления заказа необходимо добавить модуль **Платеж со счета клиента** в раздел платежей, а затем сохранить и опубликовать страницу. Пользователи B2B смогут войти на сайт электронной коммерции и применить доступное сальдо клиента к заказам во время оформления заказа.

Кроме того, в разделе **Конструктор сайтов \> Расширения** необходимо убедиться, что для свойства **Включить платежи со счета клиента** задано значение **Разрешено для клиентов B2B**.

## <a name="create-order-template-pages"></a>Создание страниц шаблона заказа

Для сайта электронной коммерции B2B можно настроить две страницы шаблонов заказов: страницу списка шаблонов заказов и страницу строк шаблонов заказов.

Страница списка шаблонов заказов содержит список всех доступных шаблонов заказов. Она настраивается с помощью модуля **Список шаблонов заказов**. Страница списка шаблонов заказов позволяет создавать или удалять шаблоны и добавлять номенклатуры в шаблоне в корзину.

На странице строк шаблона заказа отображаются сведения о шаблоне заказа, выбранном на странице списка шаблонов заказов. Она настраивается с помощью модуля **Строки шаблона заказа**. Когда пользователь выбирает имя шаблона на странице списка шаблонов заказов, отображается страница строк шаблона заказа, в которой отображаются подробные сведения о шаблоне. Затем пользователь может просматривать и редактировать номенклатуры в шаблоне.

1. Опубликуйте URL-адрес для страницы.
1. Перейдите на целевую страницу управления счетами (**Мой счет**), которая была создана ранее.
1. В области свойства модуля **Плитка шаблонов заказов учетной записи** в разделе **Ссылки** настройте ссылку на страницу списка шаблонов заказов, которая была только что создана.
1. Сохраните и опубликуйте страницу.

Теперь страница списка шаблонов заказов создана, и пользователи могут получить доступ к ней с целевой страницы управления счетами.

## <a name="add-a-quick-add-module-to-the-cart-page"></a>Добавление модуля быстрого добавления на страницу корзины

Модуль быстрого добавления позволяет быстро добавлять в корзину несколько номенклатур, используя коды номенклатур (также называемые кодами единиц измерения складского учета \[SKU\]). Модуль быстрого добавления добавляется на страницу корзины сайта.

Чтобы добавить модуль быстрого добавления на страницу корзины в Commerce, выполните следующие действия.

1. Перейдите **Шаблоны** и выберите шаблон страницы корзины сайта.
1. Выберите **Правка**.
1. В слоте **Главный** модуля **Страница по умолчанию** выберите многоточие (**...**), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите модуль **Контейнер**, затем выберите **ОК**.
1. В ячейке **Контейнер** выберите многоточие (**...**), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите модуль **Быстрое добавление**, затем выберите **ОК**.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата шаблона, затем нажмите кнопку **Опубликовать**, чтобы опубликовать его.
1. Перейдите **Страницы** и выберите страницу корзины сайта.
1. В слоте **Главный** модуля **Страница по умолчанию** выберите многоточие (**...**), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите модуль **Контейнер**, затем выберите **ОК**.
1. В области свойств модуля **Контейнер** задайте для свойства **Ширина** значение **Вписать в контейнер**.
1. В ячейке **Контейнер** выберите многоточие (**...**), затем выберите **Добавить модуль**.
1. В диалоговом окне **Добавить модуль** выберите модуль **Быстрое добавление**, затем выберите **ОК**.
1. Выберите **Сохранить**, выберите **Завершить редактирование** для возврата страницы, затем нажмите кнопку **Опубликовать**, чтобы опубликовать ее.

> [!NOTE] 
> Модуль быстрого добавления доступен в выпуске Commerce версии 10.0.17. Если выполняется обновление с более ранней версии Commerce, необходимо вручную обновить файл appsettings.json. Дополнительные инструкции см. в разделе [Обновления пакета SDK и библиотеки модулей](../e-commerce-extensibility/sdk-updates.md#update-the-appsettingsjson-file).
