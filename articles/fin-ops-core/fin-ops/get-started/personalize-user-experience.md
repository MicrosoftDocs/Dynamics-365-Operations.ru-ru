# <a name="personalize-the-user-experience"></a>Персонализация пользовательского интерфейса

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

В этой теме описывается порядок персональной настройки приложения.

Существует три основных класса личных настроек.

- Персонализации, сделанные на странице настройки. Примеры включают в себя цвет темы и часовой пояс.
- Персонализации, которые относятся к использованию страниц. Такие персонализации называются *неявными* персонализациями. Например, система отслеживает ширину столбцов таблиц, если вы ее отрегулировали, и состояние экспресс-вкладок (свернуто или развернуто).
- Персонализация, которую пользователь вносит в внешний вид страницы, изменяя способ отображения или действия элемента на этой странице, часто в интерактивном режиме персонализации. Такие персонализации называются *явными* персонализациями. Например, пользователь может добавить, скрыть или изменить порядок элементов на странице.

Любая персонализация, которую вносит пользователь, предназначена только для этого пользователя, вне зависимости от типа персонализации или компании, с которой пользователь в текущее время взаимодействует. Изменения, вносимые одним пользователем в отношении страницы, никак не сказываются на других пользователей системы.

## <a name="system-wide-options-for-the-current-user"></a>Параметры для текущего пользователя, действующие в масштабах всей системы

Страница **Параметры пользователя** содержит несколько общесистемных настроек для текущего пользователя. Чтобы открыть страницу **Параметры пользователя**, выберите кнопку **Параметры** (символ шестеренка) на панели навигации и выберите **Параметры пользователя**. Страница **Параметры пользователя** содержит четыре вкладки, которые содержат различные пользовательские настройки:

- **Внешний вид** — Выберите цвет темы и размер по умолчанию для элементов на ваших страницах.
- **Предпочтительные параметры** — Выбор значений по умолчанию, которые используются каждый раз при открытии системы. Эти значения включают компанию, начальную страницу и режим просмотра/редактирования по умолчанию. (Режим просмотра/редактирования определяет, является ли страница заблокированной для просмотра или открывается для редактирования при каждом открытии.) Эта вкладка также содержит параметры для языка, часового пояса, даты, времени и форматов номера. Наконец, эта вкладка содержит несколько предпочтительных параметров, которые изменяются от запуска в производство к запуску в производство.
- **Счет** — скорректируйте ваше имя пользователя и другие параметры, связанных с ученой записью.
- **Workflow-процесс** – выберите параметры, связанные с workflow-процессом.

В дополнение к изменению пользовательских настроек можно также использовать страницу **Параметры пользователя**, чтобы просмотреть и удалить данные об использовании и персонализации. Просто выберите **данные об использовании** в области действий.

В процессе использования приложения многие выбранные значения сохраняются, чтобы вам было проще пользоваться системой в будущем. На вкладке **Персонализация**, можно просматривать и управлять личными изменениями, сделанными на страницах в системе. На этой вкладке можно также сбросить выноски с описанием функций (то есть всплывающие окна, которые представляют новые системные функции). Затем снова появится оповещение об обнаруженных ранее функциях.

> [!NOTE]
> Если включена функция [Сохраненные представления](saved-views.md), можно просматривать личные настройки и управлять ими, выбирая **Персонализация** в области действий на странице **Параметры пользователя**.

## <a name="implicit-personalizations"></a>Неявная персонализация

Неявная персонализация — это та персонализация, которую вы вносите просто путем взаимодействия с определенными элементами управления, которые сохраняют текущее состояние отображения.

- **Столбцы сетки** — вы можете отрегулировать ширину столбца в сетке, выбрав маркер изменения размера слева или справа от заголовка столбца и сдвинув его влево или вправо до желаемой ширины столбца. Приложение хранит значения ширины, которые были заданы для столбца. Затем при следующем открытии страницы, содержащей эту сетку, размер столбца будет изменен до этой ширины.
- **Экспресс-вкладки** — на некоторых страницах есть разворачиваемые разделы, которые называются *экспресс-вкладками*. Приложение сохраняет сведения об экспресс-вкладках, которые были развернуты и свернуты. Затем в следующий раз, когда вы откроете страницу, те же экспресс-вкладки будут либо развернуты, либо свернуты, на основании вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание экспресс-вкладки может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для экспресс-вкладки до тех пор, пока вы они не будут развернуты. Как это описано далее в этой теме, можно также изменить порядок экспресс-вкладок на странице.
- **Информационные поля** — на некоторых страницах имеется панель **Связанные сведения**, в которой отображаются сведения только для чтения, которые относятся к текущей теме страницы. Каждый раздел в панели **Связанные сведения** называется *информационным полем*. Вы можете развернуть или свернуть панель **Связанные сведения**, а также развернуть или свернуть отдельные информационные поля. Приложение сохраняет эти предпочтения. Затем в следующий раз, когда вы откроете страницу, панель **Связанные сведения** и отдельные информационные поля будут развернуты или свернуты, в зависимости от вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание информационного поля может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для информационных полей до тех пор, пока вы они не будут развернуты.
- **Панели операций** – *панель операций* появляется в верхней части большинства страниц. Панель операций содержит кнопки для многих действий, которые можно выполнить на текущей странице. Часто эти кнопки расположены на вкладках. Можно закрепить всю панель операций, или можно держать ее свернутой по умолчанию. Затем в следующий раз, когда вы откроете страницу, панель операций будет либо открыта, либо свернута, на основании вашего последнего взаимодействия со страницей. Если область действий закреплена открытой, будет показана последняя использованная вкладка.
- **QuickFilters** — *QuickFilter* появляется над многими сетками. QuickFilter позволяет фильтровать сетку, на основе выбранного столбца. Приложение хранит столбец, который был отфильтрован. Затем, в следующий раз, когда вы откроете страницу, содержащую эту сетку, сетка будет отфильтрована по тому же столбцу. Однако вы можете выбрать другой столбец для фильтрации сетки.
- **Фильтры заголовков столбцов** — используется при фильтрации сетки с помощью *фильтров заголовков столбцов*, вы можете изменить оператор фильтра, чтобы найти нужные данные. Например, можно изменить оператор с **начинается с** на **совпадает с**. Каждый раз, когда вы используете фильтр заголовка столбца и изменяете оператор фильтра, приложение сохраняет это изменение. Затем, в следующий раз, когда вы фильтруете этот столбец, будет восстановлен оператор фильтра.
- **Область перехода** — можно открыть *Область перехода*, нажав кнопку **Развернуть область переходов** в левом верхнем углу любой страницы. (Эта кнопка иногда называется _кнопка **Меню**_, *гамбургер*, *меню гамбургер* или *кнопка гамбургер*). Можно закрепить область перехода в открытом состоянии или можно оставить ее свернутой по умолчанию. После того, как вы закрепите область перехода в открытом состоянии, приложение будет держать ее открытой, пока вы ее не свернете.

## <a name="explicit-personalizations"></a>Явная персонализация

У разных людей и компаний есть различные перспективы в отношении наиболее важных для них данных, и данных, которые им не нужны ввиду того, как они управляют своим бизнесом. Можно настроить способ заказа и работы с вашими сообщениями. Можно также указать, что некоторые сообщения должны быть скрыты. Эти возможности являются ключом к персональному и эффективному опыту и являются примерами явной персонализации. Явная персонализация — это персонализация, которую вы выполняете явным образом, потому что вы хотите изменить внешний вид или поведение элемента или страницы.

### <a name="shortcut-menu-options"></a>Параметры контекстного меню

Контекстные меню предоставляют несколько способов явно изменить страницу, чтобы она лучше соответствовала вашим требованиям или требованиям вашей компании. (Контекстное меню также называются *меню правой кнопки мыши* или *контекстное меню*.)

Некоторые из наиболее типичных и важных изменений, выполненные на страницу доступны непосредственно, как параметры в контекстном меню. Например, начиная с Platform update 17, чтобы добавить или скрыть столбцы в сетке, просто щелкните правой кнопкой мыши заголовок столбца, а затем выберите **Добавить столбцы** или **Скрыть этот столбец**.

Кроме того, основные типы явных настроек доступны по щелчку элемента правой кнопкой мыши с последующим выбором **Персонализация**. (Обратите внимание, что не все элементы на странице можно персонализировать.) При использовании этого метода персонализации появится окно свойств элемента.

![Настройка свойств элемента](./media/personalization-element-properties.png)

С помощью окна свойств можно осуществлять персонализацию элемента следующим образом:

- Изменение метки элемента.
- Скройте этот элемент, чтобы он не отображается на странице. Данные в поле не удаляются и не изменяются. Данные просто не отображается на странице больше.
- Включите сообщение в раздел сводки экспресс-вкладки (если элемент находится на экспресс-вкладке).
- Пропустите поле, чтобы оно никогда не получало фокус при переходе по странице.
- Предотвратить редактирование данных в поле (для любой записи).

Окно свойств может содержать других функций персонализации, в зависимости от элемента. Например, окно свойств для плитки может позволить переместить ее на панель мониторинга, а окно свойств для панели мониторинга может предусматривать создание новой рабочей области на этой панели мониторинга.

### <a name="the-personalization-toolbar"></a>Панель инструментов персонализации

Если требуется внести несколько изменений на странице или внести изменения, которые недоступны посредством другого механизма (например, если требуется изменить порядок элементов), можно использовать панель инструментов **Персонализация**. Чтобы открыть панель инструментов **Персонализация**, выполните одно из следующих действий:

- Выберите **персонализировать эту форму** в окне свойств элемента.
- Выберите **Персонализировать эту страницу** в группе **Персонализация** на вкладке **Параметры** панели действий любой страницы.
- Нажмите кнопку **Параметры** (символ шестеренки) на панели переходов, а затем выберите **Персонализация**.

[![Панель инструментов персонализации](./media/restyledPersonalizationToolbar.png)](./media/restyledPersonalizationToolbar.png)

#### <a name="navigating-the-page"></a>Навигация по странице

Когда открыта панель инструментов **Персонализация**, соответствующая страница доступна только для чтения (иными словами, редактирование данных невозможно), но по-прежнему является интерактивной. В частности, можно развернуть или свернуть панель **Связанные сведения**, переключить вкладки и разворачивать или сворачивать разделы точно так же, как обычно они выполняются на странице. Чтобы применить персонализацию к свертываемому разделу или вкладке (например, скрыть экспресс-вкладку), просто нужно выбрать кнопку, которая отображается рядом с разделом или вкладкой, когда они получают фокус клавиатуры или когда на них наводится курсор.

#### <a name="personalization-tools"></a>Инструменты персонализации

Доступны следующие средства на панели инструментов **Персонализация**:

- Используйте инструмент **Выбрать**, чтобы выбрать и изменить свойства элемента. Для использования этого средства выберите кнопку **Выбрать** на панели инструментов, а затем выберите нужный элемент. Открывается окно свойств элемента, в котором вы можете изменить любое из свойств для этого элемента. Вы можете повторить этот процесс для других персонализируемых элементов на странице. Обратите внимание, что некоторые свойства персонализации могут оказаться недоступными в некоторых сценариях. Например, нельзя заблокировать обязательное поле.
- Используйте инструмент **Скрыть**, чтобы скрыть элемент на странице. Для использования этого средства выберите кнопку **Скрыть** на панели инструментов, а затем выберите элемент для скрытия. При использовании инструмента **Скрыть** все элементы, которые в настоящий момент скрыты становятся видимыми, но отображаются в контейнере серого цвета. Затем можно сделать элемент видимым, выбрав его. Чтобы увидеть, как будет выглядеть страница, когда элементы скрыты, переключитесь на другой инструмент персонализации.

    Можно скрыть обязательные поля и разделы, содержащие обязательные поля. Таким образом, вы можете создать упрощенный интерфейс, в котором обязательные поля не отображаются, если они имеют значения по умолчанию, заполненные бизнес-логикой. Скрытые обязательные поля временно становятся видимыми, если они не заполнены, когда пользователь пытается сохранить страницу.

- Используйте инструмент **Добавить поле**, чтобы добавить на страницу поле. При использовании этого инструмента можно добавлять только те поля, которые являются частью определения страницы. Сведения о том, как создавать новые поля, которые не являются частью определения текущей страницы см. в разделе [Пользовательские поля](user-defined-fields.md). После выбора кнопки **Добавить поле** на панели инструментов необходимо сначала выбрать группу или область, где требуется добавить поле. Появится диалоговое окно со списком полей, связанных с выбранной группой или областью. В диалоговом окне выберите одно или более поле для добавления и выберите **Вставить**. Чтобы удалить поле, добавленное ранее, повторите описанные выше действия, но снимите флажок поля в диалоговом окне.
- Используйте инструмент **Переместить**, если вы хотите выбрать элемент и переместить его в другое место в текущей группе элементов. Обратите внимание, что переместить элемент за пределы его родительской группы нельзя. Для использования этого средства выберите кнопку **Переместить** на панели инструментов, а затем выберите элемент для перемещения. При выборе элемента, приложение определяет места, куда элемент может быть перемещен. Эти местоположения называются *зонами посадки*. При перетаскивании элемента в текущей группе каждая зона посадки отображается как выделенная цветом строка жирным шрифтом рядом с областью, где элемент может быть размещен.
- Используйте инструмент **Пропустить**, чтобы удалить элемент из последовательности перехода по клавише TAB . При выборе кнопки **Пропустить** на панели инструментов все элементы, которые в настоящий момент пропущены, отображаются в контейнере серого цвета. Можно в интерактивном режиме удалять или добавлять поля в последовательность перехода.
- Используйте инструмент **Показать в заголовке**, когда требуется отображение поля в разделе сводки экспресс-вкладки. При выборе кнопки **Показать в заголовке** на панели инструментов все поля, которые были выбраны как поля сводки отображаются в контейнере серого цвета. В интерактивном режиме можно добавить поля в сводку экспресс-вкладки и удалить поля из нее путем выбора полей.
- Используйте инструмент **Блокировать**, когда вы хотите пометить элемент как редактируемый или нередактируемый. При выборе кнопки **Блокировать** на панели инструментов все элементы, которые в настоящий момент нередактируемые, отображаются в контейнере серого цвета. После этого можно сделать их редактируемыми снова. Обратите внимание, что некоторые поля являются обязательными, и сделать нередактируемыми их нельзя. Символ замка появляется рядом с этими полями.
- Используйте кнопку **Добавить PowerApp** для внедрения на страницу приложения, созданного с помощью Microsoft PowerApps. Детальная информация о том, как внедрить на странице приложения PowerApps, см. раздел [Внедрение PowerApps](embed-power-apps.md).
- Используйте инструмент **Очистить**, чтобы сбросить страницу в ее предусмотренное по умолчанию (как после установки) состояние. Вся персонализация на текущей странице будет сброшена. Нет отмененного действия. Таким образом, используйте этот инструмент только в том случае, если Вы действительно хотите выполнить сброс страницы.
- Используйте инструмент **Импорт**, чтобы загрузить персонализацию из файла персонализации, которую вы или кто-то другой ранее создали. При импорте персонализаций для страницы можно указать, следует ли их добавлять в список или заменить все существующие персонализации для страницы. Нет отмененного действия. Поэтому после импорта персонализаций необходимо вручную удалить или отменить все ненужные изменения.
- Используйте инструмент **Экспорт** для сохранения персонализаций для страницы в файл. Можно обмениваться персонализациями с другими пользователями. Этим пользователям следует только осуществить импорта файла, содержащего ваши персонализации для страницы.
- Нажмите кнопку **Закрыть**, чтобы закрыть панель инструментов **Персонализация** и вернуть страницу к предыдущему интерактивному состоянию.

Обычно при использовании панели инструментов **Персонализация** персонализации вступают в силу после их создания. Однако, если функция [сохраненные представления](saved-views.md) включена, необходимо явно сохранить персонализации в выбранном вами представлении.

В некоторых случаях при выборе того или иного инструмента рядом с элементом появляется значок замка. Этот символ указывает, что невозможно изменить свойства элемента, которые относятся к выбранному инструменту, поскольку изменения этих свойств будет препятствовать правильной работе страницы.

### <a name="adding-a-tile-list-or-link-to-a-workspace"></a>Добавление плитки или ссылки в рабочую область

Для некоторых страниц, содержащих списки, функция персонализации **Добавить в рабочую область** доступна в группе **Персонализация** на вкладке **Параметры** панели операций. Эта функция позволяет переместить соответствующие сведения из текущего списка в конкретную рабочую область. Сведения, отображаемые в рабочей области, могут быть основаны либо на целом списке, либо на основе фильтрованных и отсортированных версий списка. Также можно указать, показывать сообщения в рабочей области в виде списка, плитки сводки, где может быть указано число элементов в списке, или в виде ссылки.

> [!NOTE]
> Если включена функция [сохраненных представлений](saved-views.md), то содержимое, которое вы перемещаете к рабочей области, непосредственно связано с представлением. Запрос представления используется для выборки данных в рабочей области, а соответствующая плитка или ссылка в рабочей области открывает страницу в этом представлении, таким образом к ней применяются запросы и личные настройки представления.

[![Добавить в рабочую область](./media/personalization-addtoworkspace.png)](./media/personalization-addtoworkspace.png)

- Чтобы добавить список в рабочую область, сначала отсортируйте или отфильтруйте список на странице, чтобы он отображал сообщение в рабочей области, как вы хотите. (Если функция Сохраненные представления включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Список**. После выбора **Конфигурировать** появится диалоговое окно, где можно выбрать столбцы для отображения в списке в рабочей области. Можно также указать метку, используемую для списка в рабочей области.
- Чтобы добавить в рабочую область плитку, сначала отфильтруйте список на странице, чтобы в нем были представлены данные, сводку по которым требуется видеть или к которым нужен быстрый доступ. (Если функция Сохраненные представления включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Плитка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для плитки в рабочей области. Можно также указать, должна ли плитка отображать подсчет. После добавления плитки в рабочую область можно выбрать ее, чтобы открыть текущую страницу из рабочей области. Затем можно просмотреть отфильтрованный список, связанный с плиткой.
- Чтобы добавить ссылку в рабочую область, сначала отфильтруйте список на странице, чтобы отобразить данные, в которых вы заинтересованы. (Если функция Сохраненные представления включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление**, выберите **Ссылка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для ссылки. Можно также указать метку для нового раздела, который содержит эту ссылку.

После того, как список, плитка или ссылка добавлены в рабочую область, можно открыть эту рабочую область и изменить порядок элементов по вашему усмотрению.

### <a name="adding-a-summary-from-a-workspace-to-a-dashboard"></a>Добавление сводки из рабочей области на панель мониторинга

Некоторые рабочие области содержат плитки подсчета (то есть плитки с номерами), и, возможно, вы захотите разместить такие плитки на панели мониторинга. В рабочей области щелкните правой кнопкой мыши плитку счетчика, выберите **Персонализация**, а затем в окне свойств плитки выберите **Закрепить на панели мониторинга**. Затем, когда вы в следующий раз откроете и обновите выбранную панель мониторинга, вы увидите результат подсчета рядом с навигационной плиткой этой рабочей области. Можно сделать, чтобы подсчет осуществлял переход непосредственно к данным, который он представляет.

### <a name="personalizing-your-dashboard"></a>Персонализация вашей панели мониторинга

Во многих случаях панель мониторинга — это первая страница, которую вы видите, открывая приложение. Панели мониторинга можно настроить так, чтобы он содержал только плитки рабочей области, которые нужно отобразить. Можно также изменить порядок плиток, чтобы они были появлялись в том порядке, в котором вам удобно, переименовывать плитки навигации вашей области или добавить новую плитку в рабочую область.

Чтобы персонализировать панель мониторинга, щелкните правой кнопкой мыши любую плитку, а затем выберите **Персонализация** для открытия окна свойств.

- Если вы хотите скрыть или переименовать выбранную плитку, это изменение можно сделать непосредственно в окне свойств.
- Чтобы изменить порядок плиток в окне свойств, выберите **Персонализировать эту форму**, чтобы открыть панель инструментов **Персонализация**. После этого вы сможете с помощью инструмента **Переместить** перегруппировать плитки в требуемом порядке.
- Чтобы добавить новую плитку в рабочей области в окне свойств, выберите **Добавление рабочей области**. В нижней части панели мониторинга будет создана новая плитка рабочей области. Можно переименовать эту новую плитку рабочей области по желанию. Также можно добавить списки, плитки и ссылки в рабочую область как описано в разделе [добавление списков, плиток или ссылок в рабочие области,](#adding-a-tile-list-or-link-to-a-workspace) см.

## <a name="administration-of-personalizations"></a>Администрирование персонализаций

После его персонализации страницы такими настройками можно поделиться с другими пользователями, экспортировав персонализированную страницу. Затем можно попросить других пользователей открыть персонализированную страницу и импортировать созданный вами файл персонализации. Кроме того, можно передать ваши персонализации пользователю, который имеет привилегии администратора. Пользователь может применить файл персонализации ко многим пользователям одновременно.

Если у пользователей есть привилегии администратора, они также могут управлять персонализацией для других пользователей на странице **Персонализация**.

Для клиентов, которые не включили функцию [Сохраненные представления](saved-views.md), страница имеет четыре вкладки:

- **Применить** — вы можете импортировать или выбрать персонализацию для одного или нескольких пользователей. Чтобы применить персонализацию для одного или нескольких пользователей, выберите роль и того, кто получит эту роль. Затем либо выберите существующую персонализацию для применения к выбранным пользователям, либо импортируйте файл персонализации. Персонализация будет проверена и применена для всех выбранных пользователей, когда они в следующий раз откроют выбранную страницу.
- **Очистить** – вы можете очистить персонализации для страницы или рабочей области для одного или нескольких пользователей. Сначала выберите страницу или рабочую область, чтобы просмотреть список пользователей, у которых эта страница персонализирована. Затем выберите пользователей, для которых необходимо очистить персонализацию для этой страницы или рабочей области, и выберите **Очистить**. Все персонализации, которые выбранные пользователей применили к выбранной странице или рабочей области, будут удалены. Это действие невозможно отменить. Однако если для страницы или рабочей области есть сохраненная персонализация, ее можно повторно импортировать.
- **Пользователи** — выберите пользователя, чтобы просмотреть список страниц, которые этот пользователь персонализировал. Затем можно включить или отключить возможность этого пользователя использовать персонализации для определенных страниц или для всей системы. Можно также импортировать, экспортировать или очистить персонализации для выбранного пользователя. Кроме того, можно сбросить выноски с описанием функций для пользователя. В этом случае, если пользователь ранее закрыл все всплывающие окна с новыми функциями, они снова будут отображаться при следующем обнаружении этих функций пользователем.
- **Система** — здесь можно временно отключить все персонализацию для всех пользователей в системе. В этом случае все персонализации удаляются для всех пользователей, и все страницы сбрасываются в состояние по умолчанию. Если вы впоследствии снова включаете персонализацию, все персонализации заново применены. Можно также безвозвратно удалить все персонализации для всех пользователей в системе. Нет способа восстановить персонализации, которые были удалены. Поэтому, прежде чем выполнять этот шаг, убедитесь, что вы экспортировали все персонализации, которые вам понадобятся впоследствии.

Для клиентов, которые включили функцию [Сохраненные представления](saved-views.md), страница **Персонализация** имеет пять вкладок:

- **Опубликованные представления** — эти представления опубликованы в Организации. Чтобы изменить пользователей, которые предназначены для этих представлений, можно изменить роли безопасности или юридические лица, которые связаны с каждым представлением. Имеется также возможность экспортировать или удалить одно или несколько опубликованных представлений.
- **Неопубликованные представления** — эти представления являются представлениями шаблонов, которые были импортированы в систему, но еще не были опубликованы. Эти представления можно опубликовать, экспортировать или удалить.
- **Персональные представления** – эти представления созданы пользователями системы. Можно опубликовать персональное представление в организации или скопировать одно или несколько из этих представлений другим пользователям. Эти представления можно также экспортировать или удалить, при необходимости.
- **Пользователи** — выберите пользователя, чтобы просмотреть список страниц, которые этот пользователь персонализировал. Затем можно включить или отключить возможность этого пользователя использовать персонализации для определенных страниц или для всей системы. Можно также импортировать, экспортировать или очистить персонализации для выбранного пользователя. Кроме того, можно сбросить выноски с описанием функций для пользователя. В этом случае, если пользователь ранее закрыл все всплывающие окна с новыми функциями, они снова будут отображаться при следующем обнаружении этих функций пользователем.
- **Система** — здесь можно временно отключить все персонализацию для всех пользователей в системе. В этом случае все персонализации удаляются для всех пользователей, и все страницы сбрасываются в состояние по умолчанию. Если вы впоследствии снова включаете персонализацию, все персонализации заново применены. Можно также безвозвратно удалить все персонализации для всех пользователей в системе. Нет способа восстановить персонализации, которые были удалены. Поэтому, прежде чем выполнять этот шаг, убедитесь, что вы экспортировали все персонализации, которые вам понадобятся впоследствии.

Пользователи, которые имеют доступ к странице **Персонализация**, могут также импортировать персональные или шаблонные представления, используя кнопку **импорт представлений** в области действий.

## <a name="personalizing-inventory-dimensions"></a>Персонализация складских аналитик

При персонализации настройки складских аналитик на странице примите во внимание параметры, созданные с помощью команды **Отобразить аналитики**. Например, персонализации позволяют скрыть столбец для складской аналитики "Номер партии", но столбец отобразится, когда страница будет открыта в следующий раз. Это происходит, поскольку параметры **Отображение аналитики** управляют столбцами складской аналитики, которые отображаются. Параметры **Отображение аналитики** применяются ко всем страницам, и эти параметры будут переопределять собой всю персонализированную настройку полей складских аналитик на отдельных страницах.

Таким образом, в предыдущем примере, если вы не хотите, чтобы столбец для складской аналитики "Номер партии" отображался на странице, следует удалить эту аналитику в рамках параметра **Отображение аналитик** для этой страницы.
