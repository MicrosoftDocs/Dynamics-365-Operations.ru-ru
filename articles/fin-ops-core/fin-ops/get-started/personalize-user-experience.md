# <a name="personalize-the-user-experience"></a>Персонализация пользовательского интерфейса

[!include [banner](../includes/banner.md)]

В этой теме объясняется, как можно персонализировать приложение и рассматриваются следующие темы: 

- **Параметры на уровне системы** — эти параметры персонализации сделаны на странице настройки и доступны всем пользователям. Примеры включают в себя цвет темы и часовой пояс. 
- **Ограниченный доступ к личным данным** — на этом уровне доступа, действия пользователя, связанные с типичным использованием страниц, автоматически сохраняются приложением и восстанавливаются при следующем посещении страницы. Например, приложение хранит ширину столбцов таблиц, если вы ее отрегулировали, и состояние экспресс-вкладок (свернуто или развернуто). 
- **Полный доступ к персонализации** — на этом уровне доступа у пользователей есть доступ ко всем возможностям персонализации в приложении. В частности, у них есть доступ к панели инструментов **Персонализация**. 
- **Общий доступ к персонализациям** — пользователи с полным доступом к персонализации могут экспортировать свои личные настройки страниц и поделиться ими с другими пользователями.
- **Администрирование персонализаций** — привилегированные пользователи могут получить доступ к странице администрирования **Персонализация**, чтобы управлять всеми персонализациями на уровне организации. 

## <a name="system-wide-options-for-the-current-user"></a>Параметры для текущего пользователя, действующие в масштабах всей системы

Страница **Параметры пользователя** содержит несколько общесистемных настроек для текущего пользователя. Эти параметры доступны всем пользователям, даже пользователям, которым не предоставлен доступ к персонализации. Чтобы открыть страницу **Параметры пользователя**, выберите кнопку **Параметры** на панели навигации и выберите **Параметры пользователя**. Страница **Параметры пользователя** содержит четыре вкладки, которые содержат различные пользовательские настройки:

- **Внешний вид** — Выберите цвет темы и размер по умолчанию для элементов на ваших страницах.
- **Предпочтительные параметры** — Выбор значений по умолчанию, которые используются каждый раз при открытии системы. Эти значения включают компанию по умолчанию, начальную страницу и режим просмотра/редактирования по умолчанию. (Режим просмотра/редактирования определяет, является ли страница заблокированной для просмотра или открывается для редактирования при каждом открытии.) Эта вкладка также содержит параметры для языка, часового пояса, даты, времени и форматов номера. Наконец, эта вкладка содержит несколько предпочтительных параметров, которые изменяются от запуска в производство к запуску в производство.
- **Счет** — просмотр или корректировка имени пользователя и других параметров, связанных с учетной записью.
- **Workflow-процесс** – выберите параметры, связанные с workflow-процессом.

В дополнение к изменению пользовательских настроек можно также использовать страницу **Параметры пользователя**, чтобы просмотреть и удалить данные об использовании и персонализации. Для просмотра данных об использовании выберите **Использование данных** на панели операций. На вкладке **Персонализация**, можно просматривать и управлять личными изменениями, сделанными на страницах в системе. На этой вкладке можно также сбросить выноски с описанием функций (то есть всплывающие окна, которые представляют новые системные функции). Затем снова появится оповещение об обнаруженных ранее функциях.

> [!NOTE]
> Если включена функция [Сохраненные представления](saved-views.md), можно просматривать личные настройки и управлять ими, выбирая **Персонализация** в области действий на странице **Параметры пользователя**.

## <a name="restricted-personalization-access-formerly-implicit-personalizations"></a>Ограниченный доступ к персонализации (ранее "неявные персонализации")

На уровне **Ограниченный доступ к персонализации** действия пользователя, связанные с типичным использованием страниц, автоматически сохраняются приложением и восстанавливаются при следующем посещении страницы. Явная операция сохранения не является обязательной. 

Ниже приводится список действий, характерных для обычной работы со страницей и рассматриваемых как ограниченный доступ к персонализации: 

- **Ширина столбцов сетки** — вы можете отрегулировать ширину столбца в сетке, выбрав маркер изменения размера слева или справа от заголовка столбца и сдвинув его влево или вправо до желаемой ширины столбца. Приложение хранит значения ширины, которые были заданы для столбца. Затем при следующем открытии страницы размер столбца будет изменен до этой ширины.
- **Нижний колонтитул сетки и итоги по столбцам** — *(доступно только в том случае, когда включен новый элемент управления сетки)* можно указать, следует ли показывать итоговое значение в нижней части любого числового столбца в сетке, а также доступен ли нижний колонтитул сетки. Приложение сохраняет эти данные и применяет их при следующем открытии страницы. Дополнительные сведения см. в разделе [Возможности сетки](grid-capabilities.md). 
- **Экспресс-вкладки** — на некоторых страницах есть разворачиваемые разделы, которые называются *экспресс-вкладками*. Приложение сохраняет сведения об экспресс-вкладках, которые были развернуты или свернуты. В следующий раз, когда вы откроете страницу, те же экспресс-вкладки будут либо развернуты, либо свернуты, на основании вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание экспресс-вкладки может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для экспресс-вкладки до тех пор, пока вы они не будут развернуты. Как это описано далее в этой теме, можно также изменить порядок экспресс-вкладок на странице.
- **Информационные поля** — на некоторых страницах имеется панель **Связанные сведения**, в которой отображаются сведения только для чтения, которые относятся к текущей теме страницы. Каждый раздел в панели **Связанные сведения** называется *информационным полем*. Вы можете развернуть или свернуть панель **Связанные сведения**, а также развернуть или свернуть отдельные информационные поля. Приложение сохраняет эти предпочтения. В следующий раз, когда вы откроете страницу, панель **Связанные сведения** и отдельные информационные поля будут развернуты или свернуты, в зависимости от вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание области **Связанные сведения** или информационного поля может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для информационных полей до тех пор, пока вы они не будут развернуты.
- **Панели операций** – *панель операций* появляется в верхней части большинства страниц. Панель операций содержит кнопки для многих действий, которые можно выполнить на текущей странице. Часто эти кнопки расположены на вкладках. Можно *закрепить* всю панель операций, или можно держать ее свернутой по умолчанию. В следующий раз, когда вы откроете страницу, панель операций будет либо открыта, либо свернута, на основании вашего последнего взаимодействия со страницей. Если область действий закреплена открытой, будет показана последняя использованная вкладка.
- **QuickFilters** — *QuickFilter* появляется над многими сетками. QuickFilter позволяет фильтровать сетку, на основе одного выбранного столбца. Приложение хранит столбец, который был отфильтрован. Затем, в следующий раз, когда вы откроете страницу, сетка будет использовать тот же столбец для фильтрации по умолчанию. Однако вы можете выбрать другой столбец для фильтрации сетки.
- **Фильтры заголовков столбцов** — используется при фильтрации сетки с помощью *фильтров заголовков столбцов*, вы можете изменить оператор фильтра, чтобы найти нужные данные. Например, можно изменить оператор с **начинается с** на **совпадает с**. Каждый раз, когда вы используете фильтр заголовка столбца и изменяете оператор фильтра, приложение сохраняет это изменение. Затем, в следующий раз, когда вы фильтруете этот столбец, будет восстановлен оператор фильтра.
- **Область перехода** — можно открыть *Область перехода*, нажав кнопку **Развернуть область переходов** в левом верхнем углу любой страницы. (Эта кнопка иногда называется _кнопка **Меню**_, *гамбургер*, *меню гамбургер* или *кнопка гамбургер*). Можно закрепить область перехода в открытом состоянии или можно оставить ее свернутой по умолчанию. После того, как вы закрепите область перехода в открытом состоянии, приложение будет держать ее открытой, пока вы ее не свернете.

## <a name="full-personalization-access-formerly-explicit-personalizations"></a>Полный доступ к персонализации (ранее "явные персонализации")

На уровне **Полный доступ к персонализации** у пользователей есть доступ ко всем возможностям персонализации в приложении. Поскольку различные сотрудники и компании имеют различные потребности при взаимодействии с приложением, особенно в плане используемых полей, персонализация предоставляет средства, позволяющие пользователям и организациям настроить способ упорядочивания и работы с данными в приложении. Эти возможности являются ключевыми для обеспечения упрощенной, оптимизированной работы с приложением, адаптированном для вас и вашей организации. 

Если включена функция [Сохраненные представления](saved-views.md), необходимо явное сохранение, чтобы сохранить эти изменения во взаимодействии пользователей для конкретного представления. Когда функция **Сохраненные представления** отключена, эти изменения сохраняются автоматически.

В следующих разделах рассматриваются возможности персонализации, доступные пользователям с **полным уровнем доступа к персонализации**. Вот некоторые из этих возможностей:

- Параметры контекстного меню
- Панель инструментов **Персонализация**
- Добавление плиток, списков и ссылок в рабочие области
- Добавление сводки из рабочей области на панель мониторинга
- Персонализация панели мониторинга

### <a name="shortcut-menu-options"></a>Параметры контекстного меню

Контекстные меню предоставляют один способ изменить интерфейс страницы, чтобы она лучше соответствовала вашим требованиям или требованиям вашей организации. (Контекстное меню также называются *меню правой кнопки мыши* или *контекстное меню*.)

Некоторые из наиболее типичных и важных изменений, выполненные на страницу доступны непосредственно, как параметры в контекстном меню. Например, чтобы добавить или скрыть столбцы в сетке, просто щелкните правой кнопкой мыши заголовок столбца, а затем выберите **Вставить столбцы** или **Скрыть этот столбец**.

Кроме того, основные типы персонализаций доступны по щелчку элемента правой кнопкой мыши с последующим выбором **Персонализация**. (Обратите внимание, что не все элементы на странице можно персонализировать.) При использовании этого метода персонализации появится *окно свойств элемента*.

![Настройка свойств элемента.](./media/cli-element-property-window.png)

С помощью окна свойств можно осуществлять персонализацию элемента следующим образом:

- Изменение метки элемента.
- Скройте этот элемент, чтобы он не отображается на странице. Данные в поле не удаляются и не изменяются. Данные просто не отображается на странице больше.
- Включите сообщение в раздел сводки экспресс-вкладки (если элемент находится на экспресс-вкладке).
- Пропустите поле, чтобы оно никогда не получало фокус при переходе по странице.
- Предотвратить редактирование данных в поле (для любой записи).
- Назначьте поле, необходимое для ввода данных. Если в это поле не было введено никакого значения, оно будет отображаться с красной границей и звездочкой, чтобы обозначить это состояние. Этот параметр доступен только начиная с версии 10.0.11, когда включены функции [Сохраненные представления](saved-views.md) и **Назначение полей как обязательных с помощью персонализации**.

Окно свойств может содержать других функций персонализации, в зависимости от элемента. Например, окно свойств для плитки может позволить переместить ее на панель мониторинга, а окно свойств для элементов на панели мониторинга по умолчанию может предусматривать создание новой настраиваемой рабочей области.

### <a name="personalization-toolbar"></a>Панель инструментов персонализации

Если требуется внести несколько изменений на странице или внести изменения, которые недоступны посредством других механизмов (например, если требуется изменить порядок элементов), можно использовать панель инструментов **Персонализация**. Чтобы открыть панель инструментов **Персонализация**, выполните одно из следующих действий:

- Нажмите сочетание клавиш **CTRL+SHIFT+P** для любого элемента на странице.
- Выберите **Персонализировать эту страницу** в окне свойств элемента.
- Выберите **Персонализировать эту страницу** в группе **Персонализация** на вкладке **Параметры** панели действий любой страницы.
- Нажмите кнопку **Параметры** (символ шестеренки) на панели переходов, а затем выберите **Персонализация**.

[![Панель инструментов персонализации.](./media/restyledPersonalizationToolbar.png)](./media/restyledPersonalizationToolbar.png)

#### <a name="navigating-the-page"></a>Навигация по странице

Когда открыта панель инструментов **Персонализация**, соответствующая страница доступна только для чтения (иными словами, редактирование данных невозможно), но по-прежнему является интерактивной. В частности, можно развернуть или свернуть панель **Связанные сведения**, переключить вкладки и разворачивать или сворачивать разделы точно так же, как обычно они выполняются на странице. Чтобы применить персонализацию к свертываемому разделу или вкладке (например, скрыть экспресс-вкладку), просто нужно выбрать кнопку, которая отображается рядом с разделом или вкладкой, когда они получают фокус клавиатуры или когда на них наводится курсор.

#### <a name="personalization-tools"></a>Инструменты персонализации

Доступны следующие средства на панели инструментов **Персонализация**:

- Используйте инструмент **Выбрать**, чтобы выбрать и изменить свойства элемента. Для использования этого средства выберите кнопку **Выбрать** на панели инструментов, а затем выберите нужный элемент. Открывается окно свойств элемента, в котором вы можете изменить любое из свойств для этого элемента. Вы можете повторить этот процесс для других персонализируемых элементов на странице. Обратите внимание, что некоторые свойства персонализации могут оказаться недоступными в некоторых сценариях. Например, нельзя заблокировать обязательное поле.
- Используйте инструмент **Скрыть**, чтобы скрыть элемент на странице. Для использования этого средства выберите кнопку **Скрыть** на панели инструментов, а затем выберите элемент для скрытия. При использовании инструмента **Скрыть** все элементы, которые в настоящий момент скрыты становятся видимыми, но отображаются в контейнере серого цвета. Затем можно сделать элемент видимым, выбрав его. Чтобы посмотреть, как страница будет выглядеть, когда элементы скрыты, переключитесь на другое средство персонализации или закройте панель инструментов персонализации.
- Используйте инструмент **Добавить поля**, чтобы добавить на страницу поля. При использовании этого инструмента можно добавлять только те поля, которые являются частью определения страницы. Сведения о том, как создавать новые поля, которые не являются частью определения текущей страницы см. в разделе [Создание настраиваемых полей и работа с ними](user-defined-fields.md). После выбора кнопки **Добавить поле** на панели инструментов необходимо сначала выбрать сетку или раздел, где требуется добавить поле. Появится диалоговое окно со списком полей, связанных с выбранной сеткой или разделом. В диалоговом окне выберите одно или более поле для добавления и выберите **Обновить**. Чтобы удалить поле, добавленное ранее, повторите описанные выше действия, но снимите флажок поля в диалоговом окне.
- Используйте инструмент **Переместить**, если вы хотите выбрать элемент и переместить его в другое место в текущей группе элементов. Обратите внимание, что переместить элемент за пределы его родительской группы нельзя. Для использования этого средства выберите кнопку **Переместить** на панели инструментов, а затем выберите элемент для перемещения. При выборе элемента, приложение определяет места, куда элемент может быть перемещен. Эти местоположения называются *зонами посадки*. При перетаскивании элемента в текущей группе каждая зона посадки отображается как выделенная цветом строка жирным шрифтом рядом с областью, где элемент может быть размещен.
- Используйте инструмент **Пропустить**, чтобы удалить элемент из последовательности перехода по клавише TAB . При выборе кнопки **Пропустить** на панели инструментов все элементы, которые в настоящий момент пропущены, отображаются в контейнере серого цвета. Можно в интерактивном режиме удалять или добавлять поля в последовательность перехода.
- Используйте инструмент **Показать в заголовке**, когда требуется отображение поля в разделе сводки экспресс-вкладки. При выборе кнопки **Показать в заголовке** на панели инструментов все поля, которые были выбраны как поля сводки отображаются в контейнере серого цвета. В интерактивном режиме можно добавить поля в сводку экспресс-вкладки и удалить поля из сводки путем выбора полей.
- Используйте инструмент **Обязательный**, чтобы назначить элемент, обязательный для ввода данных. При выборе кнопки **Обязательный** на панели инструментов все элементы, которые были настроены как обязательные, отображаются в контейнере с заливкой. После этого можно сделать их снова необязательными. Этот параметр доступен только начиная с версии 10.0.12 и более поздней, когда включена функция **Назначение полей как обязательных с помощью персонализации**.
- Используйте инструмент **Блокировать**, когда вы хотите пометить элемент как редактируемый или нередактируемый. При выборе кнопки **Блокировать** на панели инструментов все элементы, которые в настоящий момент нередактируемые, отображаются в контейнере серого цвета. После этого можно сделать их редактируемыми снова. Обратите внимание, что некоторые поля являются обязательными, и сделать нередактируемыми их нельзя. Символ замка появляется рядом с этими полями.
- Используйте средство **Добавить приложение из Power Apps** для внедрения на страницу приложения, созданного с помощью Microsoft Power Apps. Детальная информация о том, как внедрить на странице приложения из Power Apps, см. раздел [Внедрение приложений из Power Apps](embed-power-apps.md). Этот параметр доступен только в том случае, если функция [Сохраненные представления](saved-views.md) отключена.
- Используйте кнопку **Добавить приложение**, чтобы внедрить приложение, созданное из Microsoft Power Apps или стороннего решения, на страницу. Этот параметр доступен только в том случае, если функция [Сохраненные представления](saved-views.md) включена. 
- Используйте инструмент **Очистить**, чтобы сбросить страницу в ее предусмотренное по умолчанию (как после установки) состояние. Вся персонализация на текущей странице будет сброшена. Это действие невозможно отменить. Таким образом, используйте этот инструмент только в том случае, если Вы действительно хотите выполнить сброс страницы. Когда функция **Сохраненные представления** включена, этот инструмент удаляет персонализации для текущего представления.
- Используйте инструмент **Импорт**, чтобы загрузить персонализацию из файла персонализации, которую вы или кто-то другой ранее создали. 

    - Когда функция **Сохраненные представления** отключена, можно указать, следует ли добавить или заменить существующие персонализации на персонализации, импортируемые для страницы. Это действие невозможно отменить. Поэтому после импорта персонализаций необходимо вручную удалить или отменить все ненужные изменения.
    - Когда функция **Сохраненные представления** включена, импортируемые персонализации станут представлением на странице. Если представление уже существует, можно пропустить импорт, заменить текущее представление с тем же именем или переименовать импортированное представление.

- Используйте инструмент **Экспорт** для сохранения персонализаций для страницы в файл. Можно обмениваться персонализациями с другими пользователями. Этим пользователям следует только осуществить импорта файла, содержащего ваши персонализации для страницы. Когда функция **Сохраненные представления** включена, этот инструмент сохраняете текущую персонализацию для файла для общего доступа.
- Нажмите кнопку **Закрыть**, чтобы закрыть панель инструментов **Персонализация** и вернуть страницу к предыдущему интерактивному состоянию.

Обычно при использовании панели инструментов **Персонализация** персонализации вступают в силу после их создания. Однако, если функция [сохраненные представления](saved-views.md) включена, необходимо явно сохранить персонализации в выбранном вами представлении.

В некоторых случаях при выборе того или иного инструмента рядом с элементом появляется значок замка. Этот символ указывает, что невозможно изменить свойства элемента, которые относятся к выбранному инструменту, поскольку изменения этих свойств будет препятствовать правильной работе страницы.

### <a name="adding-tiles-lists-and-links-to-a-workspace"></a>Добавление плиток, списков и ссылок в рабочую область

Для некоторых страниц, содержащих списки, функция персонализации **Добавить в рабочую область** доступна в группе **Персонализация** на вкладке **Параметры** панели операций. Эта функция позволяет переместить соответствующие сведения из текущего списка в конкретную рабочую область. Сведения, отображаемые в рабочей области, могут быть основаны либо на целом списке, либо на основе фильтрованных и отсортированных версий списка. Также можно указать, показывать сообщения в рабочей области в виде списка, плитки сводки, где может быть указано число элементов в списке, или в виде ссылки.

> [!NOTE]
> Если включена функция [сохраненных представлений](saved-views.md), то содержимое, которое вы перемещаете к рабочей области, непосредственно связано с представлением. Запрос представления используется для получения данных в рабочей области, а соответствующая плитка или ссылка в рабочей области открывает страницу в этом представлении, таким образом к ней применяются запросы и личные настройки представления. Если представление обновляется, соответствующие элементы рабочей области будут скорректированы в соответствии с новым определением представления.

[![Добавить в рабочую область.](./media/personalization-addtoworkspace.png)](./media/personalization-addtoworkspace.png)

- Чтобы добавить список в рабочую область, сначала отсортируйте или отфильтруйте список на странице, чтобы он отображал сообщение в рабочей области, как вы хотите. (Если функция **Сохраненные представления** включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Список**. После выбора **Конфигурировать** появится диалоговое окно, где можно выбрать столбцы для отображения в списке в рабочей области. Можно также указать метку, используемую для списка в рабочей области.
- Чтобы добавить в рабочую область плитку, сначала отфильтруйте список на странице, чтобы в нем были представлены данные, сводку по которым требуется видеть или к которым нужен быстрый доступ. (Если функция **Сохраненные представления** включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Плитка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для плитки в рабочей области. Можно также указать, должна ли плитка отображать подсчет. После добавления плитки в рабочую область можно выбрать ее, чтобы открыть текущую страницу из рабочей области. Затем можно просмотреть отфильтрованный список, связанный с плиткой.
- Чтобы добавить ссылку в рабочую область, сначала отфильтруйте список на странице, чтобы отобразить данные, в которых вы заинтересованы. (Если функция **Сохраненные представления** включена, продолжение работы невозможно до тех пор, пока не будет сохранено представление с данными условиями). Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление**, выберите **Ссылка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для ссылки. Можно также указать метку для нового раздела, который содержит эту ссылку.

После того, как список, плитка или ссылка добавлены в рабочую область, можно открыть эту рабочую область и изменить порядок элементов по вашему усмотрению.

### <a name="adding-a-summary-from-a-workspace-to-a-dashboard"></a>Добавление сводки из рабочей области на панель мониторинга

Некоторые рабочие области содержат плитки подсчета (то есть плитки с номерами), и, возможно, вы захотите разместить такие плитки на панели мониторинга. В рабочей области щелкните правой кнопкой мыши плитку счетчика, выберите **Персонализация**, а затем в окне свойств плитки выберите **Закрепить на панели мониторинга**. Когда вы в следующий раз откроете и обновите панель мониторинга, вы увидите результат подсчета рядом с навигационной плиткой этой рабочей области. Можно сделать, чтобы подсчет осуществлял переход непосредственно к данным, который он представляет.

### <a name="personalizing-your-dashboard"></a>Персонализация вашей панели мониторинга

Во многих случаях панель мониторинга — это первая страница, которую вы видите, открывая приложение. Ее можно персонализировать как любую другую страницу в системе, используя те же механизмы, которые описаны ранее в этом разделе. 

> [!WARNING]
> В настоящее время при скрытии содержимого на панели мониторинга важно непосредственно ориентироваться на плитку, а не на пространство вокруг нее. Если скрыть группу вокруг плитки, могут возникнуть неожиданные результаты, если позже добавляется большее количество плиток или если система переключается на другой язык.

Одна уникальная функция персонализации, доступная на панели мониторинга, позволяет добавлять плитки. 

- Если функция **Приложения полной страницы** отключена, можно добавить новую плитку, щелкнув правой кнопкой мыши элемент на панели мониторинга, а затем выбрав **Добавить рабочую область**. В нижней части панели мониторинга будет создана новая плитка рабочей области. Можно переименовать эту новую плитку рабочей области по желанию. Также можно добавить списки, плитки и ссылки в рабочую область, как описано в разделе [Добавление плиток, списков и ссылок в рабочую область](personalize-user-experience.md#adding-tiles-lists-and-links-to-a-workspace).
- Если функция **Приложения полной страницы** отключена, можно добавить новую плитку, щелкнув правой кнопкой мыши элемент на панели мониторинга, а затем выбрав **Добавить приложение**. В диалоговом окне выберите, следует ли добавить плитку для новой рабочей области или плитки с содержимым Power Apps или сайтом. Затем выполните шаги для настройки выбранного параметра. В нижней части панели мониторинга будет создана новая плитка. 

## <a name="sharing-personalizations"></a>Общий доступ к персонализациям

После персонализации страницы есть несколько способов, которые можно использовать, чтобы поделиться своими персонализациями с другими пользователями. В следующем списке методы упорядочены по возрастанию от наиболее рекомендованных к минимально рекомендованным.

1. Публикация представлений для пользователей.
2. Копирование представлений или персонализаций для пользователей.
3. Экспорт и импорт представлений или персонализаций.

### <a name="publish-views-to-users"></a>Публикация представлений для пользователей

Если включена функция [Сохраненные представления](saved-views.md) и если страница поддерживает представления, лучшим способом совместного использования персонализаций с другими пользователями является публикация представления для пользователей с одной или несколькими ролями безопасности. Дополнительные сведения см. в разделе [Публикация представлений](saved-views.md#publishing-views).

### <a name="copy-views-or-personalizations-to-users"></a>Копирование представлений или персонализаций для пользователей

Если функция [Сохраненные представления](saved-views.md) выключена или страница не поддерживает представления, рекомендуемым способом совместного использования персонализаций является их копирование между пользователями. Этот метод доступен только пользователям с особыми привилегиями (например, системным администраторам). Однако администраторы могут найти конкретную персонализацию пользователя в системе (включая личное представление пользователя, если включены сохраненные представления) и копировать конфигурацию для других пользователей.

Если включены сохраненные представления, выполните следующие действия, чтобы скопировать персонализации.

1. Перейдите в раздел **Администрирование системы \> Настройка \> Персонализация**.
2. Чтобы скопировать личные представления, необходимо выполнить следующие действия:

    1. Выберите **Личные представления**.
    2. В списке выберите требуемое представление.
    3. Выберите **Копировать для пользователей**.
    4. Выберите пользователей, которым необходимо распределить представления.

    Выполните следующие действия, чтобы скопировать персонализации для страниц, не поддерживающих представления:

    1. Выберите **Параметры пользователя**.
    2. Выберите пользователя, для которого необходимо распределить персонализацию.
    3. Выберите **Управление всеми персонализациями**.
    4. В списке выберите требуемую персонализацию.
    5. Выберите **Копировать для пользователей**.
    6. Выберите пользователей, которым необходимо распределить персонализации.

Если не включены сохраненные представления, выполните следующие действия, чтобы скопировать персонализацию.

1. Перейдите в раздел **Администрирование системы \> Настройка \> Персонализация**.
2. Выберите **Применить**.
3. Выберите пользователей, которым необходимо распределить персонализацию.
4. Выберите **Выбрать существующую персонализацию**.
5. Найдите и выберите (одну) персонализацию, которая вас заинтересовала.
6. Нажмите **ОК**.

### <a name="export-and-import-views-or-personalizations"></a>Экспорт и импорт представлений или персонализаций

Другим способом совместного использования персонализаций является экспорт и импорт. Отдельные пользователи или администратор, действующий от своего имени, могут использовать этот метод для экспорта персонализаций или представлений, а затем передать экспортированный файл другим пользователям для импорта. Или же пользователи могут предоставить экспортированные персонализации пользователю с правами администратора, и этот пользователь может затем использовать страницу администрирования **Персонализация** для одновременного применения файла персонализации для нескольких пользователей.

#### <a name="export"></a>Экспорт

В общем случае можно экспортировать одно из собственных представлений или персонализаций, открыв соответствующую страницу, открыв панель инструментов **Персонализация**, а затем выбрав **Экспорт**. Дополнительные сведения о панели инструментов см. в разделе [Панель инструментов персонализации](#personalization-toolbar) ранее в этой теме. Кроме того, если [сохраненные представления](saved-views.md) включены, можно перейти **Параметры \> Параметры пользователя \> Персонализация**, чтобы просмотреть список всех ваших персонализаций в системе. В нем можно выбрать представления или персонализации для экспорта, а затем выбрать **Экспорт**.

Кроме того, администраторы могут экспортировать персонализации других пользователей, выполнив следующие шаги:

1. Перейдите в раздел **Администрирование системы \> Настройка \> Персонализация**.
2. На вкладке **Пользователи** выберите требуемого пользователя.
3. Найдите и выберите представление или персонализацию, которая вас заинтересовала.
4. Выберите **Экспорт**.

#### <a name="import"></a>Импорт

Чтобы импортировать представление или персонализацию, можно просто открыть панель инструментов **Персонализация** и выбрать **Импорт**. Кроме того, администраторы могут импортировать файл и немедленно передать его одному или нескольким пользователям.

Если включены сохраненные представления, выполните следующие действия.

1. Перейдите в раздел **Администрирование системы \> Настройка \> Персонализация**.
2. На панели операций выберите **Импорт представлений \> Представления пользователей**.
3. Выберите режим импорта:

    - **Выбрать конкретных пользователей** — предоставьте представление или персонализацию выбранным пользователям.
    - **Импортировать как есть** — импорт представления или персонализации для того же пользователя, который ее экспортировал.

4. Выберите **Обзор**, найдите и выберите импортируемую персонализацию.
5. Выберите **Далее**.
6. Если на шаге 3 выбрано **Выбрать конкретных пользователей**, выберите пользователей, для которых требуется импортировать персонализацию.
7. Выберите **Импорт**.
8. Разрешите конфликты, если это необходимо.

Если не включены сохраненные представления, выполните следующие действия.

1. Перейдите в раздел **Администрирование системы \> Настройка \> Персонализация**.
2. Выберите **Применить**.
3. Выберите пользователей, которым необходимо распределить персонализацию.
4. Выберите **Импорт персонализаций из файла**.
5. Выберите **Обзор**, найдите и выберите импортируемую персонализацию.
6. Нажмите **ОК**.

## <a name="administration-of-personalizations"></a>Администрирование персонализаций

Страница **Персонализация** является центральной системой для управления персонализациями на уровне организации. Содержимое и возможности на этой странице зависят от того, была ли включена функция **Сохраненные представления**.

Для клиентов, которые включили функцию **Сохраненные представления**, см. раздел "Глобальное управление представлениями" в теме [Сохраненные представления](saved-views.md).

Для клиентов, которые не включили функцию [Сохраненные представления](saved-views.md), страница имеет четыре вкладки:

- **Применить** — вы можете импортировать или выбрать персонализацию для одного или нескольких пользователей. Чтобы применить персонализацию для одного или нескольких пользователей, выберите роль и того, кто получит эту роль. Затем либо выберите существующую персонализацию для применения к выбранным пользователям, либо импортируйте файл персонализации. Персонализация будет проверена и применена для всех выбранных пользователей, когда они в следующий раз откроют выбранную страницу.

- **Очистить** – вы можете очистить персонализации для страницы или рабочей области для одного или нескольких пользователей. Сначала выберите страницу или рабочую область, чтобы просмотреть список пользователей, у которых эта страница персонализирована. Затем выберите пользователей, для которых необходимо очистить персонализацию для этой страницы или рабочей области, и выберите **Очистить**. Все персонализации, которые выбранные пользователей применили к выбранной странице или рабочей области, будут удалены. Это действие невозможно отменить. Однако если для страницы или рабочей области есть сохраненная персонализация, ее можно повторно импортировать.

- **Пользователи** — выберите пользователя, чтобы просмотреть список страниц, которые этот пользователь персонализировал. Затем можно включить или отключить возможность этого пользователя использовать персонализации для определенных страниц или для всей системы. Можно также импортировать, экспортировать или очистить персонализации для выбранного пользователя. Кроме того, можно сбросить выноски с описанием функций для пользователя. В этом случае, если пользователь ранее закрыл все всплывающие окна с новыми функциями, они снова будут отображаться при следующем обнаружении этих функций пользователем.

- **Система** — здесь можно временно отключить все персонализацию для всех пользователей в системе. В этом случае все персонализации удаляются для всех пользователей, и все страницы сбрасываются в состояние по умолчанию. Если вы впоследствии снова включаете персонализацию, все персонализации заново применены. Можно также безвозвратно удалить все персонализации для всех пользователей в системе. Нет способа восстановить персонализации, которые были удалены. Поэтому, прежде чем выполнять этот шаг, убедитесь, что вы экспортировали все персонализации, которые вам понадобятся впоследствии.

## <a name="personalizing-inventory-dimensions"></a>Персонализация складских аналитик

При персонализации настройки складских аналитик на странице примите во внимание параметры, созданные с помощью команды **Отобразить аналитики**. Например, персонализации позволяют скрыть столбец для складской аналитики "Номер партии", но столбец отобразится, когда страница будет открыта в следующий раз. Это происходит, поскольку параметры **Отображение аналитики** управляют столбцами складской аналитики, которые отображаются. Параметры **Отображение аналитики** применяются ко всем страницам, и эти параметры будут переопределять собой всю персонализированную настройку полей складских аналитик на отдельных страницах.

Таким образом, в предыдущем примере, если вы не хотите, чтобы столбец для складской аналитики "Номер партии" отображался на странице, следует удалить эту аналитику в рамках параметра **Отображение аналитик** для этой страницы.
