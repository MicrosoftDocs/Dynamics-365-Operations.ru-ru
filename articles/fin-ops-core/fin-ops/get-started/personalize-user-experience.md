# <a name="personalize-the-user-experience"></a>Персонализация пользовательского интерфейса

[!include [banner](../includes/banner.md)]

В этой теме описывается порядок персональной настройки приложения.

Существует три основных класса личных настроек.

- Персонализация на странице настройки. Примеры включают в себя цвет темы и часовой пояс.
- Персонализация, которая связана с использованием страницы, называемая *неявная* персонализация. Например, система отслеживает ширину столбцов таблиц, если вы ее отрегулировали, и состояние экспресс-вкладок (свернуто или развернуто).
- Персонализация пользователя позволяет изменить внешний вид страницы, изменив способ отображения или действия элемента на этой странице, часто в интерактивном режиме персонализации. Такая персонализация называется *явной* персонализацией. Например, пользователь может добавить, скрыть или изменить порядок элементов на странице.

Любая персонализация, которую вносит пользователь, предназначена только для этого пользователя, вне зависимости от типа персонализации или компании, с которой пользователь в текущее время взаимодействует. Изменения, вносимые одним пользователем в отношении страницы, никак не сказываются на других пользователей системы.

## <a name="system-wide-options-for-the-current-user"></a>Параметры для текущего пользователя, действующие в масштабах всей системы

Страница **Параметры пользователя** содержит несколько общесистемных настроек для текущего пользователя. Чтобы открыть страницу **Параметры пользователя**, выберите меню **Параметры** (символ шестеренка) на панели навигации и выберите **Параметры пользователя**. Страница **Параметры пользователя** содержит четыре вкладки, которые содержат различные пользовательские настройки:

- **Внешний вид** — Выберите цвет темы и размер по умолчанию для элементов на ваших страницах.
- **Предпочтительные параметры** — Выбор значений по умолчанию, которые используются каждый раз при открытии системы. Эти значения включают компанию, начальную страницу и режим просмотра/редактирования по умолчанию. (Режим просмотра/редактирования определяет, является ли страница заблокированной для просмотра или открывается для редактирования при каждом открытии.) Эта вкладка также содержит параметры для языка, часового пояса, даты, времени и цифрового формата. Наконец, эта вкладка содержит несколько предпочтительных параметров, которые изменяются от запуска в производство к запуску в производство.
- **Счет** — скорректируйте ваше имя пользователя и другие параметры, связанных с ученой записью.
- **Workflow-процесс** – выберите параметры, связанные с workflow-процессом.

В дополнение к изменению пользовательских настроек можно также просмотреть и удалить данные об использовании и личные настройки, нажав кнопку **Данные использования**. В процессе работы с приложением многие выбранные значения сохраняются, чтобы вам было проще пользоваться системой в будущем. Вкладка **Персонализация**, в частности, позволяет просматривать и управлять личными изменениями, сделанными на страницах в системе. Выноски с описанием функций, всплывающие окна, в которых описываются новые функции в системе (доступно в обновлении платформы Platform update 26), можно также сбросить на этой вкладке, чтобы вы снова получали оповещения о ранее встреченных функциях.  

## <a name="implicit-personalizations"></a>Неявная персонализация

Неявная персонализация — это та персонализация, которую вы вносите просто путем взаимодействия с определенными элементами управления, которые запоминают свое текущее состояние отображения.

- **Столбцы сетки** — вы можете отрегулировать ширину столбца в сетке, выбрав маркер изменения размера слева или справа от заголовка столбца и сдвинув его влево или вправо до желаемой ширины столбца. Приложение хранит значения ширины, которые были заданы для столбца. Затем изменяется размер столбца на эту ширину каждый раз, когда вы открываете страницу, содержащую эту сетку.
- **Экспресс-вкладки** — на некоторых страницах есть разворачиваемые разделы, которые называются *экспресс-вкладками*. Приложение сохраняет сведения об экспресс-вкладках, которые были развернуты и свернуты. Затем каждый раз, когда вы возвращаетесь на страницу, те же экспресс-вкладки будут либо развернуты, либо свернуты, на основании вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание экспресс-вкладки может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для этой экспресс-вкладки до тех пор, пока вы ее не развернете. Как описано далее в этой теме, можно также изменить порядок экспресс-вкладок на странице.
- **Информационные поля** — на некоторых страницах есть раздел, который называется *информационным полем*. Это поле содержит доступную только для чтения информацию, связанную с предметом текущей страницы. Каждый раздел в информационной области называется *информационным полем*. Вы можете скрыть или показать всю панель информационных полей, а также расширить или свернуть отдельные информационные поля. Приложение сохраняет ваши предпочтения. Затем каждый раз, когда вы возвращаетесь на страницу, состояние панели информационных полей и отдельные информационные поля будут восстановлены на основе вашего последнего взаимодействия со страницей. В некоторых случаях сворачивание информационного поля может улучшить производительность системы, потому что приложению не нужно будет извлекать информацию для этого информационного поля до тех пор, пока вы ее не развернете.
- **Панели операций** – *панель операций* появляется в верхней части большинства страниц. Панель операций содержит кнопки для многих действий, которые можно выполнить на текущей странице. Часто эти кнопки расположены на вкладках. Можно открыть и закрепить всю панель операций, или можете держать ее свернутой по умолчанию. Затем при следующем открытии страницы приложение восстановит закрепленное состояние панели операций. Если панель операций закреплена в открытом состоянии, приложение покажет вкладку операций, которую вы использовали последней.
- **QuickFilters** — *QuickFilter* появляется над многими сетками. QuickFilter позволяет фильтровать сетку, на основе выбранного столбца. Приложение хранит столбец, который был отфильтрован. Затем, в следующий раз, когда вы откроете страницу, содержащую эту сетку, сетка будет отфильтрована по тому же столбцу. Однако вы можете фильтровать сетку по другому столбцу.
- **Фильтры заголовков столбцов** — используется при фильтрации сетки с помощью *фильтров заголовков столбцов*, вы можете изменить оператор фильтра, чтобы найти нужные данные. Например, можно изменить оператор с **начинается с** на **совпадает с**. Каждый раз, когда вы используете фильтр заголовка столбца и изменяете оператор фильтра, приложение сохраняет это изменение. В следующий раз, когда вы фильтруете этот столбец, будет восстановлен оператор фильтра.
- **Область перехода** — можно открыть *Область переходов*, нажав кнопку **Меню** в левой части любой страницы. (Кнопка **Меню** иногда называется *"гамбургер"*, *меню "гамбургер"* или *кнопка "гамбургер"*.) Можно закрепить область перехода в открытом состоянии или можно оставить ее свернутой по умолчанию. После того, как вы закрепите область перехода в открытом состоянии, приложение будет держать ее открытой, пока вы ее не свернете.

## <a name="explicit-personalizations"></a>Явная персонализация

У разных людей и компаний есть различные перспективы в отношении наиболее важных для них сообщений, или сообщений, которые им не нужны ввиду того, как они управляют своим бизнесом. Можно настроить способ заказа и работы с вашими сообщениями. Можно также указать, что некоторые сообщения должны быть скрыты. Эти возможности являются ключом к персональному и эффективному опыту и являются примерами явной персонализации. Явная персонализация — это персонализация, которую вы выполняете явным образом, чтобы изменить внешний вид или поведение элемента или страницы, путем выбора меню персонализации.

### <a name="shortcut-menu-options"></a>Параметры контекстного меню

Контекстные меню предоставляют несколько способов явно изменить страницу, чтобы она больше подходила под ваши требования или требования вашей компании. (Контекстное меню также называются *меню правой кнопки мыши* или *контекстное меню*.)

Некоторые из наиболее типичных и важных изменений, выполненные на страницу доступны непосредственно, как параметры в контекстном меню. Например, начиная с Platform update 17, чтобы добавить или скрыть столбцы в сетке, просто щелкните правой кнопкой мыши заголовок столбца сетки, а затем выберите **Добавить столбцы** или **Скрыть этот столбец**.

Кроме того, основные типы явных настроек доступны по щелчку элемента правой кнопкой мыши с последующим выбором **Персонализация**. (Обратите внимание, что не все элементы на странице можно персонализировать.) При использовании этого метода персонализации появится окно свойств элемента.

![Настройка свойств элемента](./media/personalization-element-properties.png)

С помощью окна свойств можно осуществлять персонализацию элемента следующим образом:

- Изменение метки элемента.
- Скройте этот элемент, чтобы он не отображается на странице. Данные в поле не удаляются и не изменяются. Данные просто не отображается на странице больше.
- Включите сообщение в раздел сводки экспресс-вкладки (если элемент находится на экспресс-вкладке).
- Пропустите поле при нажатии клавиши Tab для перемещения между полями на странице.
- Данные в поле (любая запись) не редактируется.

Окно свойств может содержать других функций персонализации, в зависимости от элемента. Например, окно свойств для плитки может позволить переместить ее на панель мониторинга, а окно свойств для панели мониторинга может предусматривать создание новой рабочей области на этой панели мониторинга.

### <a name="the-personalization-toolbar"></a>Панель инструментов персонализации

Если требуется внести несколько изменений на странице или внести изменения, которые недоступны через другие механизмы (например, изменить порядок элементов), можно использовать панель инструментов **Персонализация**. Чтобы открыть панель инструментов **Персонализации**, выберите **Персонализировать эту форму** в окне свойств элемента. Можно также выбрать **Персонализировать эту форму** в группе **Персонализация** на вкладке **Параметры** панели операций для каждой страницы.

[![Панель инструментов персонализации](./media/restyledPersonalizationToolbar.png)](./media/restyledPersonalizationToolbar.png)

#### <a name="navigating-the-page"></a>Навигация по странице

Возможность навигации по странице при открытой **Панели инструментов персонализации**, зависит от используемой версии платформы.

- До Platform update 19, когда панель инструментов **Персонализация** открыта, страница доступна только для чтения (ничего вводить нельзя) и неинтерактивна (можно только вносить изменения в видимые элементы на странице). Если требуется внести изменения в элементы в свернутом разделе или на другой вкладке, необходимо закрыть панель инструментов **Персонализация**, развернуть раздел или перейи на нужную вкладку и снова открыть панель инструментов **Персонализация**.

- Начиная с Platform update 19, если панель инструментов **Персонализация** открыто, страница по-прежнему доступна только для чтения, но гораздо более интерактивна. В частности, можно разворачивать или сворачивать область информационной панели, переходить с вкладки на вкладку и разворачивать или сворачивать разделы при открытой панели инструментов **Персонализации** точно так же, как обычно на странице. Чтобы применить изменение персонализации к свертываемому разделу или вкладке (например, скрыть экспресс-вкладку), необходимо нажать кнопку, которая отображается рядом со свертываемым разделом или вкладкой, когда они получают фокус клавиатуры или когда на них наводится курсор.

#### <a name="personalization-tools"></a>Инструменты персонализации

Доступны следующие средства на панели инструментов **Персонализация**:

- Используйте инструмент **Выбрать**, чтобы выбрать и изменить свойства элемента. Выберите инструмент **Выбрать**, а затем выберите элемент, чтобы изменить его свойства. Когда вы выбираете элемент, открывается окно свойств, в котором вы можете изменить любое из свойств для этого элемента. Вы можете повторить этот процесс для других персонализируемых элементов на этой странице. Тем не менее из-за того, что некоторые элементы используются, приложение не позволит изменить некоторые их свойства. Таким образом, при выборе элемента вы можете увидеть, что некоторые из его свойств не могут быть изменены. Например, нельзя скрыть обязательное поле.
- Используйте инструмент **Переместить**, если вы хотите выбрать элемент и переместить его в другое место в пределах текущей группы элементов. (Переместить элемент за пределы его родительской группы нельзя.) Выберите инструмент **Переместить**, а затем выберите элемент для перемещения. При выборе элемента, приложение просматривает страницу, чтобы определить, куда можно переместить элемент. Затем функция создает серию «зон посадки». При перетаскивании элемента в пределах текущей группы каждая «зона посадки» отображается как выделенная цветом строка жирным шрифтом рядом с областью, где элемент может быть размещен.
- Используйте инструмент **Скрыть**, чтобы скрыть элемент на странице. Выберите инструмент **Скрыть**, а затем выберите элемент для скрытия. При выборе инструмента **Скрыть** все элементы, которые в настоящий момент скрыты становятся видимыми и отображаются в контейнере серого цвета. Затем можно отобразить их. Выбрав инструмент **Выбрать**, вы увидите, как будет выглядеть страница, когда выбранные элементы будут скрыты.

    - Начиная с Platform update 18 можно скрыть обязательные поля и разделы, содержащие обязательные поля. Это позволяет создавать упрощенный интерфейс, когда обязательные поля, которые по умолчанию заполняются бизнес-логикой, не отображаются. Скрытые обязательные поля также временно становятся видимыми, если на момент попытки сохранения они пустые.

- Используйте инструмент **Сводка**, когда требуется отображение элемента в разделе сводки экспресс-вкладки. Инструмент "Сводка" применим только к полям, которые содержатся в разделе экспресс-вкладки. При выборе инструмента **Сводка**, все поля, которые были выбраны как поля сводки отображаются в контейнере серого цвета. В интерактивном режиме можно добавить поля в сводку экспресс-вкладки и удалить поля из сводки экспресс-вкладки путем выбора полей.
- Используйте инструмент **Пропустить**, чтобы удалить элемент из последовательности перехода по клавише TAB . При выборе инструмента **Пропустить** все элементы, которые в настоящий момент пропущены, отображаются в контейнере серого цвета. После этого можно сделать их частью последовательности перехода снова.
- Используйте инструмент **Правка**, когда вы хотите пометить элемент как редактируемый или нередактируемый. При выборе инструмента **Правка** все элементы, которые в настоящий момент являются нередактируемыми, отображаются в контейнере серого цвета. После этого можно сделать их редактируемыми снова. Обратите внимание, что некоторые поля являются обязательными, и сделать нередактируемыми их нельзя. Символ замка появляется рядом с этими полями.
- Используйте кнопку **Вставить**, чтобы просмотреть список элементов, которые могут быть вставлены на странице.

    - Выберите инструмент **Поле** под **Вставить**, чтобы добавить поле на страницу. При использовании инструмента **Поле** можно добавлять только поля, являющиеся частью определения страницы, но не в данный момент отображаемые на странице. Сведения о том, как создавать новые поля, которые не являются частью определения текущей страницы см. в разделе [Пользовательские поля](user-defined-fields.md). После выбора инструмента **Поле** необходимо сначала выбрать группу или область, где требуется добавить поле. Появится диалоговое окно со списком полей, связанных с выбранной группой или областью. В диалоговом окне выберите одно или более поле для добавления и выберите **Вставить**. Чтобы удалить поле, добавленное ранее, повторите описанные выше действия, но снимите флажок поля в диалоговом окне.
    - Выберите инструмент **PowerApp** под **Вставить** для внедрения на страницу приложения, созданного с помощью Microsoft PowerApps. Детальная информация о том, как внедрить на странице приложения PowerApps, см. раздел [Внедрение PowerApps](embed-power-apps.md).

- Выберите кнопку **Управление**, чтобы увидеть список параметров управления, связанных со всей персонализацией для текущей страницы.

    - Выберите **Очистить**, чтобы сбросить страницу в ее предусмотренное по умолчанию (как после установки) состояние. Вся персонализация на текущей странице будет сброшена. Нет отмененного действия. Таким образом, используйте этот параметр только в том случае, если Вы действительно хотите выполнить сброс страницы.
    - Выберите **Импорт**, чтобы загрузить персонализацию из файла персонализации, которую вы или кто-то другой ранее создали для данной страницы. Все вашей текущие персонализации для страницы заменяются персонализациями из выбранного файла.
    - Выберите **Экспорт** для сохранения персонализации для страницы в файл. Можно обмениваться персонализациями с другими пользователями. Этим пользователям следует только осуществить импорта файла, содержащего ваши персонализации для страницы.

- Нажмите кнопку **Закрыть**, чтобы закрыть панель инструментов **Персонализация** и вернуть страницу к предыдущему интерактивному состоянию.

Когда используется панель инструментов **Персонализация** , операции сохранения являются неявными. Ваша персонализация вступает в силу по мере осуществления, и нет необходимости нажимать кнопку **Сохранить**. В некоторых случаях при выборе того или иного инструмента рядом с элементом появляется значок замка. Этот символ указывает, что невозможно изменить свойства элемента, которые относятся к выбранному инструменту, поскольку изменения этих свойств будет препятствовать правильной работе страницы.

### <a name="adding-a-tile-list-or-link-to-a-workspace"></a>Добавление плитки или ссылки в рабочую область

Для некоторых страниц, которые содержат списки, доступна дополнительная персонализация. Кнопка **Добавить в рабочую область** в группе **Персонализация** на вкладке **Параметры** панели операций позволяет отобразить сообщение из текущего списка в конкретной рабочей области. В рабочей области можно отобразить отфильтрованное и отсортированное сообщение или можно отобразить представление по умолчанию. Также можно указать, показывать сообщения в рабочей области в виде списка, плитки сводки, где может быть указано число элементов в списке, или в виде ссылки.

[![Добавить в рабочую область](./media/personalization-addtoworkspace.png)](./media/personalization-addtoworkspace.png)

- Чтобы добавить список в рабочую область, сначала отсортируйте или отфильтруйте список на странице, чтобы он отображал сообщение в рабочей области, как вы хотите. Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Список**. После выбора **Конфигурировать** появится диалоговое окно, где можно выбрать столбцы для отображения в списке в рабочей области. Можно также указать метку, используемую для списка в рабочей области.
- Чтобы добавить в рабочую область плитку, сначала отфильтруйте список на странице, чтобы в нем были представлены данные, сводку по которым вы хотите видеть или к которым вы хотите иметь быстрый доступ. Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление** выберите **Плитка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для плитки в рабочей области. Можно также указать, должна ли плитка отображать подсчет. После добавления плитки в рабочую область, можно выбрать возможность открытия для текущей страницы из рабочей области и просмотреть отфильтрованный список, который связан с плиткой.
- Чтобы добавить ссылку в рабочую область, сначала отфильтруйте список на странице, чтобы отобразить данные, в которых вы заинтересованы. Затем выберите **Добавить в рабочую область**. Выберите рабочую область, а затем в поле **Представление**, выберите **Ссылка**. После выбора **Конфигурировать** появляется диалоговое окно, где можно указать метку, используемую для ссылки в рабочей области. Можно также указать метку для нового раздела, который будет содержать эту ссылку.

После того, как ваш список, плитка или ссылка добавлены в рабочую область, можно открыть эту рабочую область и изменить порядок элементов по вашему желанию.

### <a name="adding-a-summary-from-a-workspace-to-a-dashboard"></a>Добавление сводки из рабочей области на панель мониторинга

Некоторые рабочие области содержат плитки подсчета (то есть плитки с номерами), и, возможно, вы захотите разместить такие плитки на панели мониторинга. В рабочей области щелкните плитку подсчета правой кнопкой мыши и выберите **Персонализировать**. Затем выберите в окне свойств плитки **Закрепить на панели мониторинга**. Когда вы в следующий раз откроете (обновите) выбранную панель мониторинга, вы увидите результат подсчета рядом с навигационной плиткой этой рабочей области. Можно сделать, чтобы подсчет осуществлял переход непосредственно к данным, который он представляет.

### <a name="personalizing-your-dashboard"></a>Персонализация вашей панели мониторинга

Во многих случаях панель мониторинга — это первая страница, которую вы видите, открывая приложение. Панели мониторинга можно настроить так, чтобы он содержал только плитки рабочей области, которые нужно отобразить. Можно также изменить порядок плиток, чтобы они были расположены в том порядке, в котором вам удобно, переименовывать плитки навигации вашей области или добавить совершенно новую плитку в рабочую область.

Чтобы персонализировать панель мониторинга, щелкните правой кнопкой мыши любую плитку, а затем выберите **Персонализация** для открытия окна свойств.

- Если вы хотите скрыть или переименовать выбранную плитку, это изменение можно сделать непосредственно в окне свойств.
- Чтобы изменить порядок плиток в окне свойств, выберите **Персонализировать эту форму**, чтобы открыть панель инструментов **Персонализация**. После этого вы сможете с помощью инструмента **Переместить** расположить плитки в требуемом порядке.
- Чтобы создать новую плитку в рабочей области в окне свойств, выберите **Добавление рабочей области**. В нижней части панели мониторинга будет создана новая плитка рабочей области. Можно переименовать эту новую плитку рабочей области по желанию. Также можно добавить списки, плитки и ссылки в рабочую область как описано в разделе [добавление списков, плиток или ссылок в рабочие области,](personalize-user-experience.md#adding-a-tile-list-or-link-to-a-workspace) см.

## <a name="administration-of-personalization"></a>Администрирование персонализации

После его персонализации страницы такими настройками можно поделиться с другими пользователями, экспортировав персонализированную страницу. Затем можно попросить других пользователей открыть персонализированную страницу и импортировать созданный вами файл персонализации. Кроме того, можно передать вашу персонализацию пользователю, который имеет права администратора. Пользователь может применить файл персонализации ко многим пользователям одновременно.

Если у пользователей есть привилегии администратора, они также могут управлять персонализацией для других пользователей на странице **Персонализация**. На этой странице четыре вкладки:

- **Применить** — вы можете импортировать или выбрать персонализацию для одного или нескольких пользователей. Чтобы применить персонализацию для одного или нескольких пользователей, выберите роль и того, кто получит эту роль. Затем либо выберите существующую персонализацию для применения к выбранным пользователям, либо импортируйте файл персонализации. Персонализация будет проверена и применена для всех выбранных пользователей, когда они в следующий раз откроют выбранную страницу.
- **Очистить** – вы можете очистить персонализации для страницы или рабочей области для одного или нескольких пользователей. Сначала выберите страницу или рабочую область, чтобы просмотреть список пользователей, у которых эта страница персонализирована. Затем выберите пользователей, для которых необходимо очистить персонализацию для этой страницы или рабочей области, и выберите **Очистить**. Все персонализации, которые выбранные пользователей применили к выбранной странице или рабочей области, будут удалены. Это действие невозможно отменить. Однако если для страницы или рабочей области есть сохраненная персонализация, ее можно повторно импортировать.
- **Управление по отдельным пользователям** — выберите пользователя, чтобы просмотреть список страниц, которые этот пользователь персонализировал. Затем можно включить или отключить для выбранных пользователь возможность использования персонализации для конкретных страниц или для всей системы. Можно также импортировать, экспортировать или очистить персонализацию для выбранного пользователя. Кроме того, можно сбросить выноски с описанием функций для выбранного пользователя, в результате чего все ранее закрытые всплывающие окна с описанием новых функций будут снова отображаться, когда пользователь встретит эти функции.   
- **Система** — здесь можно временно отключить все персонализации для всех пользователей в системе. В этом случае персонализации будут удалены. Все страницы только сбрасываются в состояние по умолчанию для всех пользователей. Если вы впоследствии снова включаете персонализацию, все эти настройки будут заново применены. Можно также безвозвратно удалить все персонализации для всех пользователей в системе. Нет способа восстановить персонализации, которые были удалены. Поэтому, прежде чем выполнять этот шаг, убедитесь, что вы экспортировали все персонализации, которые вам понадобятся впоследствии.

## <a name="personalization-of-inventory-dimensions"></a>Персонализация складских аналитик

При персонализации настройки складских аналитик на странице примите во внимание параметры, созданные с помощью команды **Отобразить аналитики**. Например, персонализации позволяют скрыть столбец для складской аналитики "Номер партии", но столбец отобразится, когда страница будет открыта в следующий раз. Это происходит, поскольку параметры **Отображение аналитики** управляют столбцами складской аналитики, которые отображаются.

Параметры **Отображение аналитики** применяются ко всем страницам, и эти параметры будут переопределять собой всю персонализированную настройку полей складских аналитик на каждой отдельной странице.

В результате, в предыдущем примере, если вы не хотите, чтобы столбец для складской аналитики "Номер партии" отображался на странице, следует удалить эту аналитику в рамках параметра **Отображение аналитик** для этой страницы.
