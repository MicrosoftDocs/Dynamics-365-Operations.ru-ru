# <a name="view-workflow-history"></a>Просмотр истории бизнес-правила

[!include [banner](../../includes/banner.md)]

В этом разделе описываются шаги для просмотра статуса документа, отправленного в систему workflow-процессов для обработки и утверждения. В качестве компании с демонстрационными данными для создания этой процедуры используется USMF.

1. Перейдите к **Область перехода > Модули > Общее > Запросы > Workflow-процесс > История workflow-процесса**.
    - Данная форма предназначена для просмотра статуса документа, отправленного в систему workflow-процессов для обработки и утверждения.  
    - **Код экземпляра** — это идентификационный код экземпляра workflow-процесса, в рамках которого обрабатывается или обработан данный документ.  
    - **Статус** — это статус workflow-процесса документа.  
    - **Код workflow-процесса** — это идентификационный код workflow-процесса, в рамках которого обрабатывается или обработан данный документ.  
    - **Документ** — это идентификационный код документа.  
    - **Тип документа** — это тип документа, который был отправлен на обработку.  
    - **Workflow-процесс** — это имя workflow-процесса, в рамках которого обрабатывается или обработан данный документ.  
    - **Версия** — это номер версии workflow-процесса, в рамках которого обрабатывается или обработан данный документ.  
    - **Дата и время создания** — это дата и время отправки документа.  
    - **Затраченное время** — это время, прошедшее с момента отправки документа.  
    - Кнопка **Возобновить** позволяет возобновить workflow-процесс для выбранного документа.  
    - Кнопка **Отзыв** позволяет отозвать выбранный документ, чтобы он не обрабатывался.   
2. В списке выберите ссылку в требуемой строке.
    - Убедитесь, что раздел **Рабочие элементы** развернут. В этом разделе можно просмотреть рабочие элементы, которые связаны с выбранным документом. Например, необходимо завершить задачу, либо документ должен быть утвержден.  
    - Кнопка **Назначить повторно** откроет диалоговое окно, в котором имеется возможность повторно назначить рабочий элемент другому пользователю.  
    - Убедитесь, что раздел **Отслеживание сведений** развернут. В этом разделе можно просмотреть историю workflow-процесса для выбранного документа.
