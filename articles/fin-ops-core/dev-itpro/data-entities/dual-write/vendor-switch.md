# <a name="switch-between-vendor-designs"></a>Переключение между вариантами дизайна поставщиков

[!include [banner](../../includes/banner.md)]



## <a name="vendor-data-flow"></a>Поток данных поставщика 

Если выбрано использование объекта **Организация** для хранения поставщиков типа **Организация** и объекта **Контакт** для хранения поставщиков типа **Физическое лицо** , настройте следующие рабочие процессы. В противном случае такая конфигурация не требуется.

## <a name="use-the-extended-vendor-design-for-vendors-of-the-organization-type"></a>Использование расширенного дизайна поставщиков для поставщиков типа "Организация"

Пакет решения **Dynamics365FinanceExtended** содержит следующие шаблоны рабочих процессов. Будет создан рабочий процесс для каждого шаблона.

+ Создание поставщиков в объекте "Организации"
+ Создание поставщиков в объекте "Поставщики"
+ Обновление поставщиков в объекте "Организации"
+ Обновление поставщиков в объекте "Поставщики"

Чтобы создать новые рабочие процессы с помощью шаблонов рабочих процессов, выполните следующие действия.

1. Создайте рабочий процесс для объекта **Поставщик** и выберите шаблон рабочих процессов **Создание поставщиков в объекте "Организации"**. Затем выберите **OK**. Этот workflow-процесс обрабатывает сценарий создания поставщиков для объекта **Организация**.

    ![Рабочий процесс создания поставщиков в объекте "Организации"](media/create_process.png)

2. Создайте рабочий процесс для объекта **Поставщик** и выберите шаблон рабочих процессов **Обновление поставщиков в объекте "Организации"**. Затем выберите **OK**. Этот workflow-процесс обрабатывает сценарий обновления поставщиков для объекта **Организация**.
3. Создайте рабочий процесс для объекта **Организация** и выберите шаблон рабочих процессов **Создание поставщиков в объекте "Поставщики"**.
4. Создайте рабочий процесс для объекта **Организация** и выберите шаблон рабочих процессов **Обновление поставщиков в объекте "Поставщики"**.
5. Рабочие процессы можно настроить как рабочие процессы в реальном времени или в фоновом режиме, в зависимости от требований пользователя. Чтобы настроить рабочий процесс в качестве фонового рабочего процесса, выберите **Преобразовать в фоновый рабочий процесс**.

    ![Кнопка преобразования в фоновый рабочий процесс](media/background_workflow.png)

6. Активируйте рабочие процессы, созданные для объектов **Организация** и **Поставщик** , чтобы начать использовать объект **Организация** для хранения информации о поставщиках типа **Организация**.

## <a name="use-the-extended-vendor-design-for-vendors-of-the-person-type"></a>Использование расширенного дизайна поставщиков для поставщиков типа "Физическое лицо"

Пакет решения **Dynamics365FinanceExtended** содержит следующие шаблоны рабочих процессов. Будет создан рабочий процесс для каждого шаблона.

+ Создание поставщиков типа "Физическое лицо" в объекте "Поставщики"
+ Создание поставщиков типа "Физическое лицо" в объекте "Контакты"
+ Обновление поставщиков типа "Физическое лицо" в объекте "Контакты"
+ Обновление поставщиков типа "Физическое лицо" в объекте "Поставщики"

Чтобы создать новые рабочие процессы с помощью шаблонов рабочих процессов, выполните следующие действия.

1. Создайте рабочий процесс для объекта **Поставщик** и выберите шаблон рабочих процессов **Создание поставщиков типа "Физическое лицо" в объекте "Контакты"**. Затем выберите **OK**. Этот workflow-процесс обрабатывает сценарий создания поставщиков для объекта **Контакты**.
2. Создайте рабочий процесс для объекта **Поставщик** и выберите шаблон рабочих процессов **Обновление поставщиков типа "Физическое лицо" в объекте "Контакты"**. Затем выберите **OK**. Этот workflow-процесс обрабатывает сценарий обновления поставщиков для объекта **Контакт**.
3. Создайте рабочий процесс для объекта **Контакт** и выберите шаблон рабочих процессов **Создание поставщиков типа "Физическое лицо" в объекте "Поставщики"**.
4. Создайте рабочий процесс для объекта **Контакт** и выберите шаблон рабочих процессов **Обновление поставщиков типа "Физическое лицо" в объекте "Поставщики"**.
5. Рабочие процессы можно настроить как рабочие процессы в реальном времени или в фоновом режиме, в зависимости от требований пользователя. Чтобы настроить рабочий процесс в качестве фонового рабочего процесса, выберите **Преобразовать в фоновый рабочий процесс**.
6. Активируйте рабочие процессы, созданные для объектов **Контакт** и **Поставщик** , чтобы начать использовать объект **Контакт** для хранения информации о поставщиках типа **Физическое лицо**.
