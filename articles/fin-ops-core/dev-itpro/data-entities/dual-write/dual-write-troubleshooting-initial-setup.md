# <a name="troubleshoot-issues-during-initial-setup"></a>Устранение неполадок при начальной настройке

[!include [banner](../../includes/banner.md)]

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]



Эта тема предоставляет информацию по устранению неполадок для интеграции двойной записи между приложениями Finance and Operations и Dataverse. Конкретно, в этом разделе содержатся сведения об устранении неполадок, которые могут возникнуть в процессе начальной настройки интеграции двойной записи.

> [!IMPORTANT]
> Для некоторых вопросов, связанных с этим разделом, может потребоваться роль системного администратора или учетные данные администратора клиента Microsoft Azure Active Directory (Azure AD). В разделе для каждого выпуска объясняется, требуются ли конкретная роль или учетные данные.

## <a name="you-cant-link-a-finance-and-operations-app-to-dataverse"></a>Невозможно связать приложение Finance and Operations с Dataverse

**Необходимая роль для настройки двойной записи:** системный администратор в приложениях Finance and Operations и в Dataverse.

Ошибки на странице **Настройка ссылки на Dataverse** обычно вызываются проблемами с неполной настройкой или разрешениями. Убедитесь, что вся проверка работоспособности пройдена на странице **Настройка ссылки на Dataverse**, как показано на следующем рисунке. Связывание двойной записи возможно только после полной проверки работоспособности.

![Успешная проверка работоспособности](media/health_check.png)

Необходимы учетные данные администратора клиента Azure AD, чтобы можно было связать среды Finance and Operations и Dataverse. После связывания сред пользователи могут войти в систему, используя учетные данные своих учетных записей, и обновить существующее сопоставление таблиц.

## <a name="error-when-you-open-the-link-to-dataverse-page"></a>Ошибка при открытии ссылки на страницу Dataverse

**Необходимые учетные данные для устранения проблемы:** администратор клиента Azure AD

При открытии страницы **Связать с Dataverse** в приложении Finance and Operations может появиться следующее сообщение об ошибке:

*Код статуса отклика не указывает на успешное выполнение: 404 (Не найден).*

Эта ошибка возникает, когда шаг разрешения не был завершен. Для проверки того, был ли выполнен шаг разрешения, войдите на portal.Azure.com с помощью учетной записи администратора клиента Azure AD и проверьте, появляется ли приложение независимого разработчика с идентификатором **33976c19-1db5-4c02-810e-c243db79efde** в списке Azure AD **Корпоративные приложения**. Если нет, необходимо предоставить согласие приложению.

Чтобы предоставить согласие приложению, выполните следующие действия.

1. Откройте следующий URL-адрес, используя учетные данные администратора. Вам должно быть предложено предоставить разрешение.

    <https://login.microsoftonline.com/common/oauth2/authorize?client_id=33976c19-1db5-4c02-810e-c243db79efde&response_type=code&prompt=admin_consent>

2. Выберите **Принять**, чтобы указать, что вы предоставляете согласие на установку приложения с идентификатором **33976c19-1db5-4c02-810e-c243db79efde** в вашем клиенте.

    > [!TIP]
    > Это приложение необходимо для связывания Dataverse и приложений Finance and Operations. При возникновении проблем с этим шагом откройте браузер в режиме "Инкогнито" (в Google Chrome) или в режиме InPrivate (в Microsoft Edge).

## <a name="verify-that-company-data-and-dual-write-teams-are-set-up-correctly-during-linking"></a>Проверка правильности настройки данных компании и групп двойной записи во время связывания

Чтобы обеспечить правильную работу двойной записи, в среде Dataverse создаются компании, выбранные во время настройки. По умолчанию эти компании доступны только для чтения, и свойство **IsDualWriteEnable** имеет значение **True**. Кроме того, создаются ответственный подразделения и группы по умолчанию, которые включают название компании. Перед включением карт убедитесь, что указан владелец группы по умолчанию. Чтобы найти объект **Компании (CDM\_Company)**, выполните следующие действия.

1. В приложении на основе модели в Dynamics 365 выберите фильтр в верхнем правом углу.
2. В раскрывающемся списке выберите **Компания**.
3. Выберите **Выполнить**, чтобы увидеть результаты.
4. Выбор компанию, которая была связана при настройке двойной записи.
5. Убедитесь, что поле **Ответственная рабочая группа по умолчанию** имеет значение. На следующем рисунке в поле **Ответственная рабочая группа по умолчанию** установлено значение **Двойная запись USMF**.

    ![Проверка ответственной рабочей группы по умолчанию](media/default_owning_team.png)

## <a name="find-the-limit-on-the-number-of-legal-tables-or-companies-that-can-be-linked-for-dual-write"></a>Определение предельного числа юридических лиц или компаний, которые могут быть связаны с двойной записью

При попытке включения сопоставлений может появиться следующее сообщение об ошибке:

*Ошибка двойной записи - сбой регистрации подключаемого модуля: \[(Не удается получить карту разделов для проекта DWM-1ae35e60-4bc2-4905-88ea-69efd3b29260-7f12cb89-1550-42e2-858e-4761fc1443ea. Ошибка Превышено максимальное число разделов, допустимое для сопоставления DWM-1ae35e60-4bc2-4905-88ea-69efd3b29260-7f12cb89-1550-42e2-858e-4761fc1443ea)\], Возникла одна или несколько ошибок.*

Текущий предел при связывании сред составляет примерно 40 юридических лиц. Эта ошибка возникает, если при попытке включить сопоставления между этими средами связано более 40 юридических лиц.
