# <a name="removed-or-deprecated-platform-features"></a>Удаленные или устаревшие функции Platform

[!include [banner](../includes/banner.md)]

В этом разделе описываются возможности, который удалены или которые планируется удалить в обновлениях платформы приложений Finance and Operations.

- *Удаленная* функция больше недоступна в продукте.
- *Устаревшая* функция не находится в активной разработке и может быть удалена в следующем обновлении.

Этот список поможет вам учитывать эти удаления и устаревания при своем собственном планировании. 

> [!NOTE]
> Подробные сведения об объектах в приложениях Finance and Operations можно найти в документе [Технический справочник по отчетам](https://mbs.microsoft.com/customersource/northamerica/AX/downloads/reports/axtechrefrep). Можно сравнить различные версии этих отчетов, чтобы получить сведения об объектах, которые были изменены или были исключены в каждой версии приложений Finance and Operations.

## <a name="platform-updates-for-version-10011-of-finance-and-operations-apps"></a>Обновления платформы для версии 10.0.11 приложений Finance and Operations

### <a name="field-groups-containing-invalid-field-references"></a>Группы полей, содержащие недопустимые ссылки на поля

|   |  |
|------------|--------------------|
| **Причина устаревания/удаления** | Группы полей в таблице определения метаданных могут содержать недопустимые ссылки на поля. Если эти группы полей развернуты, они могут привести к ошибкам во время выполнения в Financial Reporting и службах Reporting Services Microsoft SQL Server (SSRS). В обновлении платформы Platform update 23 введено *предупреждение* компилятора, которое позволило устранить эту проблему с метаданными. Обновления платформы для версии 10.0.11 приложений Finance and Operations классифицируют эту проблему как ошибку *компилятора*.<p>Чтобы устранить эту проблему, выполните следующие действия.</p><ol><li>Удалите недопустимые ссылки полей из определения группы полей таблицы.</li><li>Заново выполните компиляцию.</li><li>Убедитесь в том, что все ошибки устранены.</li></ol> |
| **Заменена другой функцией?**   | Эта ошибка компилятора полностью заменяет предупреждение компилятора.  |
| **Затрагиваемые области продукта**         | Средства разработки Visual Studio |
| **Вариант развертывания**              | Все |
| **Состояние**                         | **Устарело:** предупреждение компилятора является ошибкой компилятора в обновлениях платформы для версии 10.0.11 приложений Finance and Operations. |

### <a name="isv-licenses-created-by-using-the-sha1-hashing-algorithm"></a>Лицензии ISV, созданные с использованием алгоритма хеширования SHA1

|   |  |
|------------|--------------------|
| **Причина устаревания/удаления** | Изменен процесс создания лицензий независимых поставщиков программного обеспечения (ISV). Дополнительные сведения см. в разделе [Лицензирование независимых поставщиков программного обеспечения (ISV)](../dev-tools/isv-licensing.md#appendix-create-self-signed-certificates-for-test-purposes). |
| **Заменена другой функцией?**   | Да. Создайте лицензии с помощью Windows PowerShell. |
| **Затрагиваемые области продукта**         | Средства разработки Visual Studio |
| **Вариант развертывания**              | Все |
| **Состояние**                         | <strong>Устарело:</strong> лицензии ISV, которые были созданы с использованием алгоритма хеширования SHA1. Этот алгоритм зависит от сертификатов, созданных с помощью служебной программы MakeCert, и эта утилита устарела.<p><strong>Устарело:</strong> использование алгоритма SHA1 в целях безопасности или хэширования. Алгоритм SHA1 перестанет работать в начале 2021 года. Поэтому он больше не должен использоваться.<p><strong>Удалено:</strong> поддержка входящих или исходящих запросов протоколов TLS 1.0 и TLS 1.1. |

## <a name="platform-update-32"></a>Обновление платформы update 32

### <a name="workflow-request-change-dialog-box-no-longer-includes-user-selection-drop-down-list"></a>Диалоговое окно "запрос на изменение бизнес-правила" больше не включает раскрывающийся список выбора пользователя
|   |  |
|------------|--------------------|
| **Причина устаревания/удаления** | Это связано с безопасностью, поскольку запрос на изменение может быть отправлен непредусмотренному пользователю. Это также проблема для удобства использования, поскольку требует от пользователя определить, кто был инициатор workflow-процесса, и вручную выбрать их.  |
| **Заменена другой функцией?**   | Нет |
| **Затрагиваемые области продукта**         | Рабочий процесс |
| **Вариант развертывания**              | Все |
| **Состояние**                         | Раскрывающийся список выбора пользователя был удален из диалогового окна запроса на изменение в обновлении платформы 32. Запросы на изменение запроса будут автоматически отправлены инициатору по назначению. Дополнительные сведения об этих функциях см. в разделе [Действия в процессах утверждения workflow-процесса](https://docs.microsoft.com/dynamics365/fin-ops-core/fin-ops/organization-administration/workflow-actions?toc=%2Fdynamics365%2Fcommerce%2Ftoc.json#request-change). |

### <a name="embedded-drill-through-links-are-no-longer-supported-in-paginated-documents-rendered-by-the-cloud-hosted-service"></a>Встроенные глубокие ссылки больше не поддерживаются в разбитых на страницы документах, которые визуализируются размещенной в облаке службой 
|   |  |
|------------|--------------------|
| **Причина устаревания/удаления** | URL-адреса переходов, встроенные в документы, отображаемые службой, могут содержать конфиденциальные деловые данные. Мы удаляем поддержку внедренных глубоких ссылок в документах в качестве меры предосторожности для дополнительной защиты данных клиента. В результате этого изменения пользователи также получать преимущества от повышенной производительности при интерактивной подготовке документов.  |
| **Заменена другой функцией?**   | Нет |
| **Затрагиваемые области продукта**         | Отчетность |
| **Вариант развертывания**              | Все |
| **Состояние**                         | Эта функция активно удаляется из службы.<br><br>Современный клиент предлагает множество параметров для создания представлений, включающих автоматически созданные ссылки, которые помогут в навигации по приложению. Разбиваемые на страницы документы, отображаемые службой, рекомендуются для внешних сообщений, которые отправляются по электронной почте,архивируются и печатаются для получателей. Мы улучшили взаимодействие предварительного просмотра документов непосредственно в браузере, который обеспечивает прямой доступ к локальным принтерам. Дополнительные сведения см. в разделе [Предварительный просмотр PDF-документов со встроенным средством просмотра](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/analytics/preview-pdf-documents). |
