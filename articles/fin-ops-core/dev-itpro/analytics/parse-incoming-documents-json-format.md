# <a name="parse-incoming-documents-in-json-format"></a>Разбор входящих документов в формате JSON

[!include[banner](../includes/banner.md)]

Форматы электронной отчетности можно разрабатывать для разбора входящих электронных документов, представляющих данные в текстовом формате, основанном на JavaScript (иными словами, файлы в формате JavaScript Object Notation \[JSON\]).

Чтобы получить дополнительные сведения об этой функции, загрузите файлы в приведенной ниже таблице, а затем запустите руководство по задаче "Электронная отчетность — Создание конфигурации формата для импорта данных из внешнего файла JSON". В этом руководстве по задаче показано, как можно использовать формат ER для разбора входящего файла JSON с целью обновления данных приложения.

| Название должности                                  | Имя файла |
|----------------------------------------|-----------|
| Конфигурация формата электронной отчетности                | [Формат для импорта trans_JSON.xml поставщиков](https://go.microsoft.com/fwlink/?linkid=874111) |
| Пример входящего файла в формате .CSV | [1099entries_JSON.txt](https://go.microsoft.com/fwlink/?linkid=874111) |

## <a name="requirements"></a>Требования

Перед завершением руководства по задаче "Электронная отчетность — Создание конфигурации формата для импорта данных из внешнего файла JSON" необходимо выполнить следующие требования:

- Родительскими элементами в JSON-файле могут быть только элементы объектов.
- Вложенные элементы для объекта должны быть элементами свойств, чтобы была создана допустимая структура JSON.
- Массивы JSON могут быть только вложенными элементами элементов свойств объекта.
- Массивы JSON могут содержать только объекты JSON. Они не могут содержать прямые строковые/числовые значения и вложенные массивы. Элементы в этих массивах будут анализироваться в порядке, в котором они указаны в формате. Будут учитываться настройки кратности для каждого объекта JSON.

Кроме того, необходимо выполнить руководство по задаче [ER Создание необходимых конфигураций для импорта данных из внешнего файла](tasks/er-required-configurations-import-data.md), если оно еще не было выполнено. Загрузите следующий файл для завершения проводника по задаче.

| Название должности                  | Имя файла |
|------------------------|-----------|
| Конфигурация модели ER | [1099model.xml](https://go.microsoft.com/fwlink/?linkid=874111) |
