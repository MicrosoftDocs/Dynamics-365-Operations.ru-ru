# <a name="generate-financial-reports"></a>Создание финансовых отчетов

[!include [banner](../includes/banner.md)]

В этой статье содержится информация о создании финансового отчета.

Для того, чтобы произвести отчет, откройте определение отчета и на панели инструментов выберите **Создать**. Страница **Статус очереди отчета** откроется и покажет расположение вашего отчета в очереди.

По мере выполнения создания отчета на странице **Состояние очереди отчетов** могут быть видны следующие индикаторы состояния очереди отчетов.

| Состояние          | Статус | Описание|
|-----------------|--------|--------------------|
| Поставлено в очередь        | Промежуточный |Определение отчета проверяется до постановки отчета в очередь создания.                    |
| Поставлено в очередь          | Промежуточный | Отчет поступает в очередь создания отчетов и ожидает обработки.                      |
| Обработка      | Промежуточный | Этот статус обычно следует за статусом **В очереди** и обычно переводится в состояние **Конечное** после завершения обработки.       |
| PostProcessing | Промежуточный | Этот статус соответствует статусу **Обработка** и указывает на то, что все данные отчета собраны, но выполняются такие производные действия, как вычисление и сведение.            |
| Отмена      | Промежуточный | Отчетность отменяется по запросу пользователя. Это состояние является результатом запрошенной пользователем отмены для отчета в состоянии **В очереди** или **Обработка**. Система пытается поместить отчет в состояние **Отменено**, если только система не продвинулась слишком далеко и не должна завершить его в другом состоянии. |
| Отменено        | Заключительный | Обработка отчета завершена, но он не был завершен из-за остановки, запрошенной пользователем.            |
| Завершено       | Заключительный | Отчет готов к использованию.                      |
| Сбой          | Заключительный | Обработка отчета завершилась неудачно, но он не должен использоваться. |

По умолчанию созданный отчет будет открыт в веб-средстве просмотра. Для формирования отчетов предусмотрены следующие варианты:

- Настройте план, чтобы автоматически произвести отчет или группу отчетов
- Проверьте на предмет пропущенных счетов или данных в отчете и проверьте точность отчета

Когда вы создаете отчет, используются параметры, которые вы определили на вкладках Определение отчета.

## <a name="generate-a-financial-report"></a>Создание финансового отчета

Чтобы создать финансовый отчет, откройте **Главная книга** \> **Запросы и отчеты** \> **Финансовые отчеты**.

- Выберите отчет, который требуется создать, и выберите **Создать**.
- Заполните поле **Дата отчета** и выберите **ОК**.

После создания отчета он будет доступен для просмотра в разделе **Отчеты**.

Для отчета можно выбрать операцию **Показать** или **Удалить**.

Для создания отчета с помощью средства **Конструктор отчетов** откройте определение отчета, затем выберите кнопку **Создать** на панели инструментов. Страница **Статус очереди отчета** откроется и покажет расположение вашего отчета в очереди. По умолчанию созданный отчет будет открыт в веб-средстве просмотра.

## <a name="report-groups"></a>Группы отчетов

Группы отчетов являются эффективным способом создания нескольких отчетов одновременно. Пусть, к примеру, вы знаете, что в течение месяца пользователи создают восемь отчетов каждый месяц. Создайте группу отчетов, и вместо выбора **создать** для каждого из восьми отчетов в группе, можно выбрать **создать** для группы отчетов, и восемь отчетов будут созданы за один шаг. Когда отчеты в выбранной группе отчетов были созданы, можно перейти к **финансовым отчетам** (**Главная книга > запросы и отчеты > Финансовые отчеты**), чтобы просмотреть отдельные отчеты. Чтобы настроить группу отчетов, выполните следующие действия.

1. В **Report Designer** выберите **Группы отчетов**. 
2. Выбор существующих определений отчета для включения в группу отчетов. 
3. Выберите параметр переопределение настроек компании, сведений и даты из каждого отчета, который будет включен в группу.
   Мы рекомендуем настроить для каждого отчета **компания**, **период**, **год** и **уровень детализации**. 
4. Сохраните группу отчетов.

## <a name="schedule-report-generation"></a>Формирование отчета по расписанию
Многие компании имеют основной набор отчетов, выполняемых в запланированные интервалы, соответствующие бизнес-процессам компании. Вы можете запланировать отчет для производства регулярно, например ежедневно, еженедельно, ежемесячно, или ежегодно. Это может быть отдельного отчет или группа отчетов по нескольким компаниям. Необходимо ввести учетные данные для каждой из заданных компании, например тех, что указаны в определении аналитической структуры. Если учетные данные недействительны, то отчет покажет только информацию, к которой в вас есть разрешение на доступ, например компания, к которой вы подключены сейчас. Выходная информация прочитана сперва из группы отчета, и после этого от индивидуальных отчетов.

По мере того как создаются и сохраняются планы отчетов, они показываются в панели навигации в разделе **Планы отчетов**. Для упорядочения отчетов используются папки. Если один отчет расписания не запускается, выполнение остальных будет продолжено.

> [!IMPORTANT]
> Только пользователи, которым назначена роль конструктора или администратора, могут создавать, изменять, удалять и генерировать группы отчетов. Когда отчет выполняется, учетные данные пользователя, который создал план, используются для того, чтобы произвести отчет.

### <a name="create-a-report-schedule"></a>Создание плана отчетов

1. В **Report Designer** в меню **Файл** выберите **Создать**, а затем выберите **План отчетов**. Открывается диалоговое окно **Новый план отчетов**.
2. В разделе **Параметры** выберите отдельный отчет или группу отчетов, которые запланировать. Доступны только отчеты, группы отчетов для компании или выборка шаблонов, в которые вы выполнили вход.
3. Установите флажок **Активный**, чтобы включить план отчетов. Только создатель отчета или администратор может включать и выключать расписание отчета.
4. Выберите кнопку **Разрешения**, чтобы вписать учетные данные компании. По умолчанию ваши учетные данные используются для компании, к которой вы подключились. Если включены другие компании, например в определениях дерева отчетности, выберите **Использовать отдельные учетные данные** и введите учетные данные для любой другой компании, которая включена в план отчета. Вы можете выбрать **Проверку подлинности Windows** или ввести имя пользователя и пароль для каждой компании. Выберите флажок **Сохранить учетные данные**, чтобы сохранить учетные данные для этих компаний, и после этого выберите **ОК**, чтобы закрыть диалоговое окно.
5. В разделе **Частота** в поле **Начало повторения** выберите дату, когда план начать. По умолчанию выбирается текущая системная дата клиентского компьютера.
6. В поле **Выполнить отчет в** выберите время когда отчет должен быть выполнен. Если указать время более ранее, чем текущее системное время, отчет запустится в следующую запланированную дату.
7. В области **Шаблон повторения** определите, как часто отчет выполняется. По умолчанию выбрано **Ежедневно** со значением **Интервал (дни)**, равным **1**. Другие варианты включают **Еженедельно**, **Ежемесячно** и **Ежегодно**.
8. В области **Диапазон повторения** выберите, когда следует перестать создавать отчет.

    - **Без даты окончания** — план отчета выполняется бесконечно.
    - **Установленное число повторений** — план отчета выполняется определенное количество раз, и после этого деактивируется.
    - **Завершить к** — план отчета завершается на определенную дату.

9. Нажмите **Сохранить**. В диалоговом окне **Сохранить как** впишите уникальное имя и описание для плана отчета.

Чтобы скопировать план отчетов, вы должны иметь роль Конструктора или администратора. Даже если администратор изменяет план отчета, отчет поддерживает учетные данные пользователя, который создал отчет.

### <a name="copy-a-report-schedule"></a>Создание плана отчетов

1. В Report Designer выберите **Планы отчетов** в панели навигации и раскройте план отчета, который необходимо скопировать.
2. В меню **Файл** выберите **Сохранить как** и введите новое имя и описание графика в диалоговое окно **Сохранить как**. Выберите **ОК**, и новый план показан в Области перехода.
3. В новом плане измените поля и информацию, как требуется, и выберите **Сохранить** на панели инструментов или выберите **Сохранить** в меню **Файл**.

Чтобы удалить план отчета, вы должны быть владельцем плана отчета или иметь роль администратора.

### <a name="delete-a-report-schedule"></a>Удаление плана отчетов

1. В Report Designer, в панели навигации, выберите **Планы отчетов**.
2. Выберите план отчетов, которую необходимо удалить, и выберите кнопку **Удалить** или нажмите клавишу **Удалить**.
3. В диалоговом окне подтверждения удаления, выберите **Да**, чтобы навсегда удалить план отчета. Если вы не имеете разрешений удалить план, то отображается сообщение и отчет не удаляется.

### <a name="credentials-and-report-schedules"></a>Учетные данные и расписания отчетов

Если не ввести учетные данные, которые необходимы для всех компаний, включаемых в отчеты, вы получите следующее сообщение при сохранении расписания отчетов: "Необходимо ввести учетные данные для компаний, содержащихся в этом расписании отчетов. Нажмите кнопку **Разрешения**, чтобы ввести свои учетные данные".

Например, пользователь выполняет вход в компанию А с помощью своего имени входа и пароля. Пользователь создает план для отчета, который использует определение дерева отчетности для того, чтобы собрать данные от множественных компаний. При сохранении данного расписания у пользователя запрашивается ввод учетных данных для других компаний, указанных в определении аналитической структуры, Когда ваши учетные данные устаревают, соответствующие отчеты в план отчетов не создаются до тех пор, пока учетные данные не будут обновлены. Сообщение отображается в очереди отчета, чтобы показать, что разрешения необходимо обновить. План отчета терпит неудачу, если происходят любые следующие сценарии (потому что они требуют учетные данные):

- Новая компания была добавлена к дереву отчета для индивидуального отчета.
- Был изменен отчет в группе отчетов.
- В группу отчетов был добавлен новый отчет для дополнительной компании.

Чтобы продолжить, выберите кнопку **Разрешения** в диалоговом окне **Планирование отчетов** и после этого впишите соответствующие учетные данные.

## <a name="missing-account-analysis-feature"></a>Функция анализа отсутствующих счетов
Вы можете искать финансовые счета и аналитики, которых может не хватать во всех определениях строк, определениях дерева отчетности, и определениях отчета в группе строительных блоков. Это полезно, когда вы создаете или обновляете несколько счетов или строительных блоков за короткий период времени, и вы хотите проверить, что вся новая информация включена в ваших отчетах.

Недостающие счета определяются с помощью самых низких и самых высоких значений из определения строки или определения дерева отчетности, после чего отображается список счетов, которых нет в определении строки или определении дерева отчетности, но которые есть в финансовых данных. Если отсутствующий счет больше или меньше значений в определении строки, этот счет не включается в список отсутствующих счетов.

> [!TIP]
> В целях проверки этот процесс следует запускать перед созданием ежемесячных отчетов, а также при создании новых шаблонов.

Отчеты, которые имеют диапазоны значений, с меньше вероятностью имеют отсутствующие счета. По возможности используйте диапазоны в шаблоне для включения новых счетов при их создании. Если любое из определений отчетов настроено на компанию @ANY, то можно выполнить вход в отдельную компанию и запустить анализ отсутствующих счетов для этой компании.

> [!NOTE]
> Если была добавлена новая компания, вы должны добавить новую компанию к деревьям отчетности во всех существующих отчетах, или компания не будет включена в анализ отсутствующих счетов.

### <a name="run-missing-account-analysis"></a>Выполнение анализа отсутствующих счетов

1. В Report Designer выберите **Инструменты**, и после этого выберите **Анализ отсутствующих счетов**.
2. В поле **Фильтр компании** выберите компанию, по которой фильтровать результаты, или выбрать **Все (без фильтра)** для показа результатов от всех доступных компаний.
3. В поле **Фильтр аналитик**, выберите аналитику, по которой фильтровать результаты, или выберите **Все (без фильтра)**, чтобы посмотреть все данные аналитик для всех доступных аналитик.
4. В поле **Группировать по** выберите вариант для сортировки результатов. Результаты можно сортировать в соответствии с затрагиваемыми шаблонами или же по измерениям и наборам значений.
5. Просмотрите отображаемые результаты. При выборе элемента в верхней панели на нижней панели отображаются дополнительные сведения об исключенных данных. К ним относятся связанные измерения, значения и отчеты.
6. Чтобы открыть соответствующий элемент, выберите соответствующий значок, который показан в панели списка, или выполните правый клик элемента и выберите **Открыть**. Чтобы выбрать несколько элементов, удерживайте клавишу **Ctrl**, пока вы выбираете элементы в нижней области.
7. Если возвращены какие-либо значения, шаблоны, или отчеты, которые не должны быть включены в анализ, выполните правый клик на элементе и выберите **Исключить**, или выберите флажок **Исключить** рядом с элементом, чтобы удалить элемент из списка. Исключенные элементы не включаются при обновлении списка. Чтобы выбрать несколько элементов, удерживайте клавишу **Ctrl**, пока вы выбираете элементы в нижней области. Чтобы просмотреть все элементы, включая и ранее исключенные из анализа результаты, установите флажок **Показать исключенные шаблоны и значения** и выберите кнопку **Обновить**.
8. Выберите **Обновить**, чтобы обновить исключения, которые вы адресовали. Выберите **Да** для того, чтобы выполнить полное обновление всех результатов, или выберите **Нет** для того, чтобы выполнить частичное обновление адресованных элементов.

    > [!NOTE]
    > Если страница не открывалась в течение последних 15 минут, то при открытии форма автоматически обновляется.

9. Когда проблемы устранены, выберите **ОК**, чтобы закрыть диалоговое окно.

## <a name="keyboard-shortcuts-for-missing-account-analysis"></a>Сочетания клавиш анализа отсутствующих счетов
При выполнении анализа отсутствующего счета доступны следующие сочетания клавиш.

| Чтобы это выполнить                           | Нажмите  |
|--------------------------------------|----------------------------|
| Фильтр по компании                    | Alt+C                      |
| Фильтрация по измерению                  | Alt+D                      |
| Выбор поля Группировать по            | Alt+G                      |
| Показывать исключенные шаблоны и значения      | Alt+S                      |
| Обновить результаты                      | Alt+R                      |
| Исключение выбранного шаблона  | Alt+X                      |
| Исключение выбранного определения строки  | Ctrl+B                     |
| Исключение выбранного значения измерения | Ctrl+D                     |
| Открытие выбранного определения отчета  | Ctrl+R                     |
| Открытие выбранного определения строки     | Ctrl+O                     |
