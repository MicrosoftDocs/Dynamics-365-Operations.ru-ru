# <a name="generate-financial-reports"></a>Создание финансовых отчетов

[!include [banner](../includes/banner.md)]

В данном разделе содержится информация о создании финансового отчета.

Для того, чтобы произвести отчет, раскройте определение отчета и после этого щелкните кнопку Создать в панели инструментов. Окно Состояние очереди отчета откроется и покажет расположение вашего отчета в очереди. По умолчанию созданный отчет будет открыт в веб-средстве просмотра.

Для формирования отчетов предусмотрены следующие варианты:

- Настройте план, чтобы автоматически произвести отчет или группу отчетов
- Проверьте на предмет пропущенных счетов или данных в отчете и проверьте точность отчета

Когда вы создаете отчет, используются параметры, которые вы определили на вкладках Определение отчета.

## <a name="generate-a-financial-report"></a>Создание финансового отчета

Чтобы создать финансовый отчет, откройте **Главная книга** \> **Запросы и отчеты** \> **Финансовые отчеты**.

- Выберите отчет, который требуется создать, и щелкните **Создать**.
- Заполните поле **Дата отчета** и щелкните **ОК**.

После создания отчета он будет доступен для просмотра в разделе **Отчеты**.

Для отчета можно выбрать операцию **Показать** или **Удалить**.

Для создания отчета с помощью средства **Конструктор отчетов** откройте определение отчета, затем нажмите кнопку "Создать" на панели инструментов. Окно Состояние очереди отчета откроется и покажет расположение вашего отчета в очереди. По умолчанию созданный отчет будет открыт в веб-средстве просмотра.

## <a name="schedule-report-generation"></a>Формирование отчета по расписанию
Многие компании имеют основной набор отчетов, выполняемых в запланированные интервалы, соответствующие бизнес-процессам компании. Вы можете запланировать отчет для производства регулярно, например ежедневно, еженедельно, ежемесячно, или ежегодно. Это может быть отдельного отчет или группа отчетов по нескольким компаниям. Необходимо ввести учетные данные для каждой из заданных компании, например тех, что указаны в определении аналитической структуры. Если учетные данные недействительны, то отчет покажет только информацию, к которой в вас есть разрешение на доступ, например компания, к которой вы подключены сейчас. Выходная информация прочитана сперва из группы отчета, и после этого от индивидуальных отчетов.

По мере того как создаются и сохраняются планы отчетов, они показываются в Области перехода в разделе Планы отчетов. Для упорядочения отчетов используются папки. Если один отчет расписания не запускается, выполнение остальных будет продолжено.

> [!IMPORTANT]
> Только пользователи, которым назначена роль конструктора или администратора, могут создавать, изменять, удалять и генерировать группы отчетов. Когда отчет выполняется, учетные данные пользователя, который создал план, используются для того, чтобы произвести отчет.

### <a name="create-a-report-schedule"></a>Создание плана отчетов

1. В конструкторе отчетов в меню Файл щелкните Создать, а затем выберите План отчетов. Открывается диалоговое окно Новый план отчетов.
2. В разделе Установки выберите отдельный отчет или группу отчетов, которые запланировать. Доступны только отчеты, группы отчетов для компании или выборка шаблонов, в которые вы выполнили вход.
3. Установите флажок Активно, чтобы включить расписание отчета. Только создатель отчета или администратор может включать и выключать расписание отчета.
4. Нажмите кнопку Разрешения, чтобы ввести учетные данные для компании. По умолчанию ваши учетные данные используются для компании, к которой вы подключились. Если включены другие компании, например, в определениях аналитических структур, установите флажок Использовать разные учетные данные и введите данные для остальных компаний, входящих в расписание отчета, Можно выбрать параметр Использовать проверку подлинности Windows или ввести имя пользователя и пароль для каждой компании. Выберите флажок Сохранить учетные данные, чтобы сохранить учетные данные для этих компаний, и после этого щелкните ОК, чтобы закрыть диалоговое окно.
5. В разделе Частота в поле Начало повторения выберите дату, когда план начать. По умолчанию выбирается текущая системная дата клиентского компьютера.
6. В поле Запускать отчет в укажите время, когда отчет должен быть запущен. Если указать время более ранее, чем текущее системное время, отчет запустится в следующую запланированную дату.
7. В области Шаблон повторения укажите как часто запускать отчет. По умолчанию выбрано "Ежедневно" со значением "Интервал (дни)", равным 1. Другие варианты включают "Еженедельно", "Ежемесячно" и "Ежегодно".
8. В области Диапазон повторение выберите дату, когда следует прекратить формирование отчета.

    - Без конечной даты – отчет должен выполняться бесконечно.
    - Заданное кол-во повторение – отчет выполняется указанное количество раз, затем деактивируется.
    - Завершить – расписание отчета завершается на определенной дате.

9. Нажмите кнопку Сохранить в панели инструментов. В диалоговом окне Сохранить как, впишите уникальное имя и описание для плана отчета.

Чтобы скопировать план отчетов, вы должны иметь роль Конструктора или администратора. Даже если администратор изменяет план отчета, отчет поддерживает учетные данные пользователя, который создал отчет.

### <a name="copy-a-report-schedule"></a>Создание плана отчетов

1. В конструкторе отчетов, щелкните Планы отчетов в Области перехода, и раскройте план отчета, который скопировать.
2. В меню Файл выберите команду Сохранить как, затем введите новое название и описание расписания в окне Сохранить как. Нажмите OK и новое расписание появится в панели навигации.
3. В новом плане измените поля и информацию, как требуется, и нажмите Сохранить на панели инструментов или щелкните Сохранить в меню Файл.

Чтобы удалить план отчета, вы должны быть владельцем плана отчета или иметь роль администратора.

### <a name="delete-a-report-schedule"></a>Удаление плана отчетов

1. В конструкторе отчетов, в Области перехода, щелкните Планы отчетов.
2. Выберите удаляемое расписание и выберите команду Удалить или нажмите клавишу Delete.
3. В диалоговом окне подтверждения удаления нажмите Да. Если вы не имеете разрешений удалить план, то отображается сообщение и отчет не удаляется.

### <a name="credentials-and-report-schedules"></a>Учетные данные и расписания отчетов

Если не ввести учетные данные, которые необходимы для всех компаний, включаемых в отчеты, вы получите следующее сообщение при сохранении расписания отчетов: "Необходимо ввести учетные данные для компаний, содержащихся в этом расписании отчетов. Нажмите кнопку "Разрешения", чтобы ввести свои учетные данные".

Например, пользователь выполняет вход в компанию А с помощью своего имени входа и пароля. Пользователь создает план для отчета, который использует определение дерева отчетности для того, чтобы собрать данные от множественных компаний. При сохранении данного расписания у пользователя запрашивается ввод учетных данных для других компаний, указанных в определении аналитической структуры, Когда ваши учетные данные устаревают, соответствующие отчеты в план отчетов не создаются до тех пор, пока учетные данные не будут обновлены. Сообщение отображается в очереди отчета, чтобы показать, что разрешения необходимо обновить. План отчета терпит неудачу, если происходят любые следующие сценарии (потому что они требуют учетные данные):

- Новая компания была добавлена к дереву отчета для индивидуального отчета.
- Был изменен отчет в группе отчетов.
- В группу отчетов был добавлен новый отчет для дополнительной компании.

Чтобы продолжить, щелкните кнопку Разрешения в диалоговом окне Планирование отчетов, и после этого впишите соответствующие учетные данные.

## <a name="missing-account-analysis-feature"></a>Функция анализа отсутствующих счетов
Вы можете искать финансовые счета и аналитики, которых может не хватать во всех определениях строк, определениях дерева отчетности, и определениях отчета в группе строительных блоков. Это полезно, когда вы создаете или обновляете несколько счетов или строительных блоков за короткий период времени, и вы хотите проверить, что вся новая информация включена в ваших отчетах.

Недостающие счета определяются с помощью самых низких и самых высоких значений из определения строки или определения дерева отчетности, после чего отображается список счетов, которых нет в определении строки или определении дерева отчетности, но которые есть в финансовых данных. Если отсутствующий счет больше или меньше значений в определении строки, этот счет не включается в список отсутствующих счетов.

> [!TIP]
> В целях проверки этот процесс следует запускать перед созданием ежемесячных отчетов, а также при создании новых шаблонов.

Отчеты, которые имеют диапазоны значений, с меньше вероятностью имеют отсутствующие счета. По возможности используйте диапазоны в шаблоне для включения новых счетов при их создании. Если любое из определений отчетов настроено на компанию @ANY, то можно выполнить вход в отдельную компанию и запустить анализ отсутствующих счетов для этой компании.

> [!NOTE]
> Если была добавлена новая компания, вы должны добавить новую компанию к деревьям отчетности во всех существующих отчетах, или компания не будет включена в анализ отсутствующих счетов.

### <a name="run-missing-account-analysis"></a>Выполнение анализа отсутствующих счетов

1. В конструкторе отчетов, щелкните Инструменты, и после этого щелкните Анализ отсутствующих счетов.
2. В поле Фильтр компании выберите компанию, по которой необходимо выполнить фильтрацию результатов, или выберите Все (нет фильтра), чтобы отобразить результаты по всем доступным компаниям.
3. В поле Фильтр измерений выберите измерение, по которому необходимо выполнить фильтрацию результатов, или выберите Все (нет фильтра), чтобы отобразить сведения измерений для всех доступных измерений.
4. В поле Группировать по выберите вариант сортировки результатов. Результаты можно сортировать в соответствии с затрагиваемыми шаблонами или же по измерениям и наборам значений.
5. Просмотрите отображаемые результаты. При выборе элемента в верхней панели на нижней панели отображаются дополнительные сведения об исключенных данных. К ним относятся связанные измерения, значения и отчеты.
6. Чтобы открыть соответствующий элемент, щелкните соответствующий значок, который показан в панели списка, или выполните правый клик элемента и выберите Открыть. Чтобы выбрать несколько элементов, удерживайте клавишу Ctrl, пока вы выбираете элементы в нижней области.
7. Если возвращены какие-либо значения, строительные блоки, или отчеты, которые не должны быть включены в анализе, выполните правый клик элемента и выберите Исключить, или выберите флажок Исключить рядом с элементом, чтобы удалить элемент из списка. Исключенные элементы не включаются при обновлении списка. Чтобы выбрать несколько элементов, нажмите и удерживайте нажатой клавишу CTRL при выборе элементов на нижней панели. Чтобы просмотреть все элементы, включая и ранее исключенные из анализа результаты, установите флажок Показать исключенные шаблоны и значения и нажмите кнопку Обновить.
8. Нажмите кнопку Обновить, чтобы обновить ранее рассмотренные исключения. Нажмите кнопку Да, чтобы выполнить полное обновление результатов, или нажмите кнопку Нет, чтобы выполнить частичное обновление рассматриваемых элементов.

    > [!NOTE]
    > Если форма не открывалась в течение последних 15 минут, то при открытии она автоматически обновляется.

9. Когда проблемы устранены, щелкните ОК, чтобы закрыть диалоговое окно.

## <a name="keyboard-shortcuts-for-missing-account-analysis"></a>Сочетания клавиш анализа отсутствующих счетов
При выполнении анализа отсутствующего счета доступны следующие сочетания клавиш.

| Чтобы это выполнить                           | Используйте это сочетание клавиш |
|--------------------------------------|----------------------------|
| Фильтр по компании                    | Alt+C                      |
| Фильтрация по измерению                  | Alt+D                      |
| Выбор поля Группировать по            | Alt+G                      |
| Показывать исключенные шаблоны и значения      | Alt+S                      |
| Обновить результаты                      | Alt+R                      |
| Исключение выбранного шаблона  | Alt+X                      |
| Исключение выбранного определения строки  | Ctrl+B                     |
| Исключение выбранного значения измерения | Ctrl+D                     |
| Открытие выбранного определения отчета  | Ctrl+R                     |
| Открытие выбранного определения строки     | Ctrl+O                     |
