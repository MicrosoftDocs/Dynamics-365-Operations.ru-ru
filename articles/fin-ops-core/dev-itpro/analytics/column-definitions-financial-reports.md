# <a name="column-definitions-in-financial-reports"></a>Определения столбцов в финансовых отчетах

[!include [banner](../includes/banner.md)]

Эта статья содержит сведения об определениях столбцов. Определение столбца — это компонент отчета или строительный блок, который определяет содержимое столбцов в отчете. Как и определения строк, базовые определения столбцов можно использовать в нескольких отчетах.

## <a name="create-and-modify-a-column-definition"></a>Создайте и измените определение столбца

Определение столбца может содержать от 2 до 255 столбцов.

### <a name="create-a-column-definition"></a>Создание определения столбцов

1. В конструкторе отчетов, в Области перехода, щелкните **Определения столбцов**.
2. В меню **Файл** выберите команду **Создать**, а затем — **Определение столбца**.
3. Добавьте содержимое определения столбцов.

### <a name="open-a-column-definition"></a>Открытие определения столбцов

1. В конструкторе отчетов, в Области перехода, щелкните **Определения столбцов**.
2. Дважды щелкните определение чтобы открыть его.

### <a name="add-a-column-to-a-column-definition"></a>Добавление столбца в определение столбцов

1. В конструкторе отчетов щелкните **Определения столбцов**, и после этого раскройте определение столбца, которое надо изменить.
2. Выберите место вставки нового столбца.
3. На меню **Правка** щелкните **Вставить столбец**. Новый столбец появится слева от выбранного столбца.

### <a name="delete-a-column-from-a-column-definition"></a>Удаление столбца из определения столбцов

1. В конструкторе отчетов нажмите кнопку **Определения столбцов** и откройте изменяемое определение.
2. Выберите удаляемый столбец.
3. В меню **Правка** выберите команду **Удалить столбец**.

## <a name="contents-of-a-column-definition"></a>Содержимое определения столбцов
Определение столбцов содержит следующие данные:

- Столбец описаний определения строки.
- Столбцы сумм, в которых отображаются данные из финансовых источников или вычислений по другим данным в определении столбцов.
- Столбцы форматирования.
- Столбцы атрибутов.

Эти сведения содержатся в следующих двух областях определения столбцов:

- Область заголовков в определении столбцов содержит тексты заголовков и форматирование, с применением которого они будут отображаться в отчете. Заголовок можно применить к отдельному столбцу данных, распространить его на несколько столбцов либо применить к столбцам в зависимости от условий. Определение столбцов может содержать любое число строк заголовков столбцов.

    > [!NOTE]
    > Заголовки столбцов применяются к каждому столбцу данных в отчете. Заголовки отчета применяются ко всему отчету. Вы определяете заголовки отчета на вкладке **Заголовки и нижние колонтитулы** определения отчета.

- Строки сведений столбца находятся в определении столбцов под строками заголовка. В строках сведений столбца определяются сведения, которые будут включены в отчет. В следующей ниже таблице перечислены строки сведений столбца, а также приведены их описания.

    | Имя строки сведений столбца                                                | Описание                                                                                            |
    |-----------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
    | Тип столбца                                                           | (Обязательно) Задание типа данных в столбце.                                                     |
    | Код книги/категория атрибута                                          | Укажите данные финансовых данных для столбцов типов **FD** и **ATTR**.                       |
    | Охватываемые периоды финансового года                                    | Укажите данные финансовых данных для столбцов типа **FD**.                                     |
    | Формула                                                               | Определите формулу вычисления для столбцов типа **CALC**.                                        |
    | Столбец — дополнительное расстояние перед столбцом — переопределение формата — управление печатью | Указываются особые параметры форматирования.                                                                        |
    | Ограничения столбца                                                   | Ограничение данных.                                                                                         |
    | Элемент аналитической структуры                                                        | Ограничение столбца для отображения только данных для указанного элемента аналитической структуры.                      |
    | Отображение валюты — фильтр валюты                                      | Формат валюты.                                                                                       |
    | Фильтр измерений                                                      | Указание фильтра, в результате применения которого будут отображаться только финансовые данные конкретных элементов аналитической структуры.                           |
    | Фильтр атрибутов                                                      | Указание фильтра для ограничения финансовых данных.                                                       |
    | Дата начала —дата окончания                                                   | Ограничение финансовых данных определенным диапазоном.                                                         |
    | Выравнивание                                                         | Выравнивание по левой стороне, по центру или по правой стороне текста описания, указанного в определении строки. |

## <a name="column-restrictions-in-a-column-definition"></a>Ограничения столбцов в определении столбца
Ограничения столбцов можно использовать, чтобы указать, как определение столбца использует данные или рассчитывает информацию. Также можно ограничить столбец отчета определенным блоком или определенными датами.

> [!NOTE]
> Код **Ограничение столбца** переопределяет любые конфликтующие параметры, которые назначаются в определении строк.

### <a name="column-restrictions-cell"></a>Ячейка "Ограничения столбцов"

Ячейка **Ограничения столбца** может включить коды, которые ограничивают или подавляют информацию, такую как форматирование, детали и суммы, для того столбца.

#### <a name="add-a-column-restriction-in-a-column-definition"></a>Добавить Ограничения столбца в определении столбца

1. В конструкторе отчетов, раскройте определение столбца, которое изменить.
2. Дважды щелкните ячейку **Ограничения столбца** для столбца, который ограничить.
3. В диалоговом окне **Ограничения столбца** выберите одни или несколько кодов в списке, и после этого щелкните **ОК**.

### <a name="column-restriction-codes"></a>Коды ограничений столбцов

В следующей ниже таблице рассматриваются коды ограничений столбца.

| Код ограничения столбца | Описание |
|-------------------------|-------------|
| SU                      | Подавить подчеркивание для столбца, где введена для определения строки команда подчеркивания (**---**) или команда двойного подчеркивания (**===**). Например, вы можете не хотеть подчеркивать суммы, которое получены вычислением процента. |
| ST                      | Подавление итогов, вывод только исходных данных по столбцу (например, в столбцах статистики). |
| SD                      | Подавить детали, так, что только строки **TOT** и **CAL** (от определения строки) будут показаны в столбце. |
| DR                      | Ограничьте суммы в столбце **FD** суммами дебета. |
| CR                      | Ограничьте суммы в столбце **FD** суммами кредита. |
| ADJ                     | Ограничение сумм в столбце всеми суммами за исключением сумм корректировки за период (при наличии). |
| XAD                     | Ограничение сумм в столбце всеми суммами за исключением сумм корректировки за период. |
| PT                      | Ограничение сумм в столбце всеми суммами за исключением разнесенных проводок, если эти проводки доступны. |
| UPT                     | Ограничьте суммы в столбце, так, что только неразнесенные проводки будут включены, если эти проводки доступны.<p><strong>Примечание:</strong> Не все поставщики данных поддерживают неразнесенные проводки. </p> |

### <a name="restrict-a-column-to-a-reporting-unit"></a>Ограничение столбца элементом аналитической структуры.

1. В конструкторе отчетов, раскройте определение столбца, которое изменить.
2. Дважды щелкните ячейку **Единица отчетности** для столбца, который ограничить.
3. В диалоговом окне **Выбор единицы отчетности**, в списке **Дерево отчетности**, выберите дерево.
4. Разверните или сверните список единиц, выберите единицу отчетности, и после этого щелкните **ОК**.

## <a name="format-column-headers"></a>Форматирование заголовков столбцов
Можно добавлять, изменять и удалять заголовки, отображаемые вверху столбцов в отчете. Вы можете также настроить условные в диапазоне заголовки столбца, основанные на поле **Период** от определений столбца и поле **Базовый период** от определений отчета. Базовый период — это функция, которая экономит время при создании отчетов скользящих прогнозов.

### <a name="create-and-manage-column-headers"></a>Создание заголовков столбцов и управление ими

Вы можете использовать диалоговое окно **Заголовок столбца**, чтобы добавить, изменить или удалить заголовки, которые появляются вверху столбцов на отчете. Следующая таблица описывает поля в диалоговом окне **Заголовок столбца**.

| Поле                 | Описание |
|-----------------------|-------------|
| Текст заголовка столбца    | Это текст, отображаемый в заголовке столбца. Вы можете напечатать текст прямо в этом поле, или щелкнуть **Вставить AutoText** для того, чтобы выбрать вариант который обновляет заголовок столбца каждый раз, когда отчет произведен. Для того, чтобы включить несколько кодов autotext, щелкайте **Вставить AutoText** снова, и после этого щелкайте другой код в списке. |
| Параметры формата        | Применение форматирования к заголовку столбца, например рамки или подчеркивания. |
| Развернуть от/Развернуть до | Определение столбца или столбцов, к которым будет применяться текст заголовка. |
| Выравнивание         | Определите, как текст заголовка столбца должен быть выровнен для столбца или диапазона столбцов, который указан в полях **Распределение от** и **Распределение до**. |

### <a name="create-a-column-header"></a>Создание заголовка столбца

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку заголовка.
3. В диалоговом окне **Заголовок столбца**, впишите текст заголовка столбца. Альтернативно, щелкните **Вставить AutoText**, и выберите вариант.
4. В поле **Параметры формата** выберите формат для заголовка.
5. В Поле **Распределение от** впишите букву столбца, на которой заголовок столбца должен начаться. В Поле **Распределение до** впишите букву столбца, на которой заголовок столбца должен кончиться.
6. В разделе **Выравнивание** выберите, должен ли текст заголовка столбца быть выровнен по леву, центру или праву.
7. Нажмите кнопку **OК**.

### <a name="add-a-column-header-row"></a>Добавление строки заголовков столбцов

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Выберите ячейку в строке заголовка.
3. На меню **Правка** щелкните **Вставить строку**. Новая строка вставляется перед строкой, выбранной на шаге 2.

> [!NOTE]
> Если в отчете четыре или более заголовков отчета, при экспорте отчета на лист Excel заголовки будут накладываться друг на друга. Чтобы просмотреть все заголовки в отчете, увеличьте верхнее поле в определении отчета.

### <a name="delete-a-column-header-row"></a>Удаление строки заголовков столбцов

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. В строке заголовка выберите ячейку для удаления.
3. На меню **Правка** щелкните **Удалить строку**.

### <a name="create-an-automatically-generated-header"></a>Создайте автоматически созданный заголовок

Конструктор отчетов может автоматически создавать заголовки столбцов, основанные на кодах autotext. Коды автотекста представляют собой переменные, которые обновляются каждый раз при создании отчета. Эти коды можно включать в любой заголовок столбца, чтобы задавать сведения отчета, различающиеся для разных отчетов, например даты или номера периодов. Таким образом, одно определение столбцов можно использовать для нескольких определений отчетов, периодов времени и аналитических структур. Потому что коды autotext основываются на данных по данных календаря из строк подробностей определения столбца, они поддерживаются только для столбцов **CALC** и **FD**. Способ отображения кода автотекста в ячейке заголовка столбца влияет на вывод этих сведений в отчете. В диалоговом окне **Заголовок столбца** коды autotext появляются в смешанном регистре. Поэтому текст отображается с разным регистром в отчете. Например, в стандартном календарном году параметр **\@CalMonthLong** разрешает месяц **7** в **июле**. Если имя месяца должно отображаться прописными буквами (например, **ИЮЛЬ**), введите код автотекста прописными буквами в поле **Текст заголовка столбца**. Например, введите **\@CALMONTHLONG**. Коды можно использовать вместе с обычным текстом. Например, вы вписываете следующий текст заголовка: **Период \@FiscalPeriod-\@FiscalYear от \@StartDate к \@EndDate**. Заголовок отчета, который создан, напоминает следующий текст: **Период 1-02 от 01/01/02 до 01/31/02**.

> [!NOTE]
> Формат некоторого текста, такого как длинная дата, зависит на ваших региональных параметров на сервере. Чтобы изменить эти установки, щелкните кнопку **Пуск**, щелкните **Панель управления**, и после этого щелкните **Язык и региональные стандарты**. В следующей ниже таблице перечислены доступные для заголовка столбца параметры автотекста.


| Параметр автотекста и код                | Описание |
|-----------------------------------------|-------------|
| Название месяца (@CalMonthLong)              | При выводе на печать в заголовке столбца выводится название текущего месяца. Если выбрано округление сумм в отчете до тысяч, миллионов или миллиардов, либо ширина столбца в отчете позволяет вывести менее девяти знаков, название месяца будет сокращено до первых трех знаков. |
| Сокращенное название месяца (@CalMonthShort) | Печать сокращенного названия месяца для выбранного финансового периода. |
| Номер периода (@FiscalPeriod)           | Печать финансового периода, который определен для этого столбца, в виде числа. Если столбец охватывает несколько периодов, выполняется печать последнего периода в диапазоне. |
| Описание периода (@FiscalPeriodName)  | Печать описания финансового периода, определенного в финансовых данных. |
| Финансовый год (@FiscalYear)               | Печать финансового года для текущего столбца в виде числа. |
| Календарный год (@CalYear)                | Печать календарного года для текущего столбца в виде числа. |
| Дата начала (@StartDate)                 | Печать даты начала столбца. |
| Дата окончания (@EndDate)                     | Печать даты окончания столбца. |
| Имя блока дерева (@UnitName)         | Печать имени блока в заголовке столбца, если столбец ограничен определенным блоком аналитической структуры. |
| Описание блока (@UnitDesc)            | Печать описание блока в заголовке столбца, если столбец ограничен определенным блоком аналитической структуры. |
| Код книги (@BookCode)                   | Печать кода книги, указанного в столбце. |
| Пустая строка (@Blank)                     | Вставка пустой строки в заголовок столбца. |

### <a name="create-a-conditional-spanning-header"></a>Создание заголовка условного интервала

Заголовки условных интервалов могут распространяться на несколько столбцов, основанных на данных определенных периодов. Например, если имеется бюджетный отчет по финансовому году и необходимо отобразить фактические бюджеты прошедших месяцев вместе с прогнозируемыми бюджетами будущих месяцев, можно воспользоваться заголовком условного интервала для автоматического обновления заголовка отчета. При создании заголовка условного интервала необходимо учитывать следующие моменты.

- Любое условие прекращения (поле **Распределение до**), которое сопоставляется до условия старта (поле **Распределение от**) игнорируется. Например, для столбца B имеется условие распределения, определенное как от BASE+1 до BASE, BASE находится в столбце C и BASE+1 находится в столбце D. В этом случае условие остановки в столбце C игнорируется, а печать заголовка начинается со столбца D.
- При указании перекрывающихся заголовков столбцов они выводятся на печать в отчете с перекрытием. Отчет формируется, но в поле **Состояние очереди отчетов** появляется следующее предупреждение: "Заголовки столбцов, использующие пересечение BASE с другими заголовками столбцов, могут привести к перекрыванию текста". Например, определение заголовка в столбце B — от B до BASE+1, и определение заголовка на столбце D — от BASE+1 до F. В этом случае заголовки печатаются друг на друге и не читаются. Когда BASE использовано в определении **Распределение от/Распределение до**, обязательно посмотрите созданный отчет, чтобы увидеть, перекрываются ли заголовки.
- Если указать BASE в определении распределения в столбце "Не печатать" (**NP**), оно игнорируется, независимо от того, что определено в определении столбца. По сути, этот сценарий — то же,, как не создавать определение заголовка столбца.
- Для столбцов условной печати (**P&lt;B**, **P&gt;=B**) условные в диапазоне заголовки поступают так же, как любое обычное определение заголовка столбца. Например, если условие окажется ложным, то любой последующий столбец, соответствующий условию развертывания, вызовет начало печати заголовка.

#### <a name="create-a-conditional-spanning-header"></a>Создание заголовка условного интервала

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку заголовка.
3. В диалоговом окне **Заголовок столбца**, впишите текст заголовка столбца. Альтернативно, щелкните **Вставить AutoText**, и выберите вариант.
4. В поле **Параметры формата** выберите стиль форматирования для заголовка.
5. Укажите период относительно базового периода, который задается при создании отчета. В полях **Распределение от** и **Распределение до**, введите одно из следующих значений: **BASE**, **BASE-X** или **BASE+X**, где X — число периодов от базового периода. Например, если вы вводите **BASE** в поле **Распределение от**, то текст условного в диапазоне заголовка столбца начинается в заголовке столбца, где значение **Базовый период** определения отчета равняется значению **Период** определения столбца. Он кончается в столбце, который показан в поле **Распределение до**. Поэтому, если распространение — от BASE до M, и значение **Базовый период** определения отчета — **4**, то заголовок начинается в столбце, где период установлен как **4** и кончается в столбце M. Заголовки оканчиваются и начинаются только на печатаемых столбцах.
6. В разделе **Выравнивание** выберите, должен ли текст заголовка столбца быть выровнен по леву, центру или праву.
7. Щелкните **OK**.

## <a name="apply-column-justification"></a>Применение выравнивания столбцов
Ячейка **Выравнивание** используется для того, чтобы применить форматирование выравнивания к столбцу описания в отчете. Этот параметр влияет только на описания столбцов, а не на фактические значения.

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку **Выравнивание**.
3. Выберите одно из следующих значений в списке:

    - **Нет** — никакое выравнивание не применяется.
    - **По левому краю** — Выравнивание описаний столбца По левому краю.
    - **По центру** — выравнивание описаний столбца По центру.
    - **По правому краю** — Выравнивание описаний столбца По правому краю.

## <a name="add-special-formatting-options"></a>Добавление специальных параметров форматирования
В определении столбца строки сведений о столбце форматирования применяют к выбранным столбцам особое форматирование. Хотя некоторые из параметров **Управление печатью** и **Ограничения столбца** относятся конкретно к столбцам **FD**, большинство параметров относится ко всем типам столбцов. Форматирование, которое указано в определении столбца, переопределяет форматирование, которое указано в определении отчета. Однако форматирование, которое указано в определении строки, переопределяет форматирование, которое указано в определении столбца. Следующие строки считаются строками форматирования:

- Ширина столбца
- Дополнительное расстояние перед столбцом
- Переопределение формата/валюты
- Управление печатью

### <a name="changing-the-column-width"></a>Изменение ширины столбца

Ячейка **Ширина столбца** определяет число символов для использования для ширины этого столбца в напечатанном отчете. Ширина столбца важна для столбцов, которые содержат суммы (столбцы типа **CALC**, **WKS**, или **FD**), описания (столбцы типа **DESC**), или заполнение (столбцы типа **FILL**). По умолчанию выбран вариант **AutoFit** выбран, так что ширина каждого столбца автоматически регулируется так, чтобы помещалось содержимое.

#### <a name="specify-the-width-of-a-column-on-a-report"></a>Задание ширины столбца в отчете

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. В ячейке **Ширина столбца**, введите число пробелов для ширины столбца. Максимальная ширина любого столбца — 255 символов (этот номер включает центы, запятые и скобки). Либо, чтобы позволить дизайнеру отчетов выбрать соответствующую ширину столбца, основанную на содержимом ячейки, дважды щелкните ячейку **Ширина столбца** и после этого щелкните **Автоподгонка**.

### <a name="add-space-between-columns"></a>Добавление пробела между столбцами

Ячейка **Дополнительные пробелы перед столбцом** задает ширину разделителя между одним столбцом и смежными столбцами в определении столбца. Настройка **Дополнительные пробелы перед столбцом** влияет на все строки сведений столбца для столбца, но не строки заголовка столбца. Используйте этот вариант для того, чтобы отделить группы столбцов или добавить несколько пробелов перед описанием, что позволяет сдвинуть столбец описаний от выровненных по левому краю заголовков, включенных в отчет. По умолчанию число пробелов между столбцами составляет два. Вы можете изменить эту установку на вкладке **Настройки** в определении отчета.

#### <a name="specify-the-space-between-columns"></a>Указание интервалов между столбцами

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. В ячейке **Дополнительный пробел перед столбцом** впишите число пробелов, которое ввести между столбцами.

### <a name="specify-a-format-currency-override"></a>Укажите перекрытие формата/валюты

Ячейка **Перекрытие формата/валюты** определяет форматирование десятичной дроби, валюты, и сумм процента в столбце. Это форматирование переопределяет любое форматирование, указанное в определении отчета или системных параметрах по умолчанию.

#### <a name="assign-a-format-currency-override-to-a-report-column"></a>Назначение переопределения формата и валюты для столбца отчета

1. В конструкторе отчетов выберите изменяемое определение столбца.
2. Дважды щелкните ячейку **Переопределение формата/валюты** в столбце суммы.
3. В диалоговом окне **Перекрытие формата** выберите варианты форматирования.

### <a name="add-a-print-control-code"></a>Добавление кода управления печатью

Ячейка **Управление печатью** может содержать коды, которые регулируют отображение или печать символов столбца. Существует два типа кодов управления печатью: обычные коды управления печатью и условные коды управления печатью.

## <a name="column-types"></a>Типы столбцов
Тип информации, который включает каждый столбец в отчете, определен значением в строке **Типе столбца** в определении столбца. Каждое определение столбца должно содержать по меньшей мере один столбец описания (**DESC**) и один столбец суммы (**FD**, **WKS** или **CALC**).

### <a name="financial-dimensions-column"></a>Столбец "Финансовая аналитика"

Следующие определения строки **Оределение столбца** применяются к столбцам которые имеют тип столбца **FD** (суммы из финансовых аналитик).

#### <a name="book-codeattribute-category-cell"></a>Ячейка "Код книги/категория атрибута"

Ячейка **Код книги/Атрибут категории** определяет код книги для данных в столбце **FD**. Определение столбца может включать несколько столбцов с фактическими, бюджетными и статистическими данными. Определение столбца также может отображать различные периоды, например текущий или с начала года, и различные суммы. Список кодов книг отражает фактические, бюджетные, и статистические (не-финансовые) параметры, которые были установлены в ваших финансовых данных.

#### <a name="fiscal-year-cell"></a>Ячейка "Финансовый год"

Ячейка **Финансовый год** определяет финансовый год, который столбец должен включить. Год можно указать относительно базового года, который определяется при создании отчета. Доступны следующие параметры.

| Параметр  | Описание                                                                                                                  |
|---------|------------------------------------------------------------------------------------------------------------------------------|
| BASE    | Используйте базовый год, который определен во время отчета.                                                                          |
| BASE+\# | Используйте год, следующий через \# лет после базового года. Например, чтобы использовать третий год после базового года, введите **BASE+3**. |
| BASE-\# | Используйте год за \# лет до базового года. Например, чтобы использовать предыдущий год, введите **BASE-1**.                 |
| \#      | Введите фактический финансовый год.                                                                                                |

#### <a name="period-cell"></a>Ячейка периода

Ячейка **Период** определяет финансовые периоды, которые столбец должен включить. Период можно указать относительно базового периода, который определяется при создании отчета. Имеются следующие варианты.

| Параметр          | описание |
|-----------------|-------------|
| BASE            | Используйте базовый период. |
| BASE+\#         | Используйте период, который следует через \# периодов после базового периода. Например, чтобы использовать третий период после базового периода, введите **BASE+3**. |
| BASE-\#         | Используйте период, который был за \# периодов до базового периода. Например, чтобы использовать предыдущий период, введите **BASE-1**. |
| BASE-\#:BASE    | Используйте множественные периоды, от нескольких периодов перед базовым периодом до базового периода. Например, для использования трех предыдущих периодов и базового периода, введите **BASE-3:BASE**. |
| BASE:BASE+\#    | Используйте множественные периоды, от базового периода до нескольких периодов после базового периода. Например, для использования базового периода и следующих двух периодов, введите **BASE:BASE+2**. |
| BASE-\#:BASE+\# | Используйте множественные периоды, от нескольких периодов перед базовым периодом до нескольких периодов после базового периода. Например, для использования трех предыдущих периодов, базового периода и следующих двух периодов, введите **BASE-3:BASE+2**. |
| 1:BASE          | Используйте множественные периоды, от первого периода до нескольких периодов после базового периода. |
| \#              | Всегда используйте специфический номер периода. Мы не рекомендуем использовать этот вариант, потому что оно уменьшает гибкость определения столбца. |
| \#:\#           | Всегда используйте специфический диапазон периодов. Мы не рекомендуем использовать этот вариант, потому что оно уменьшает гибкость определения столбца. |

При указании периода можно переходить на другой финансовый год, диапазон периодов может охватывать несколько финансовых лет. Например, вы задаете периоды как **BASE-5** (представляет прошедшие шесть периодов) и выполняете отчет, который имеет базовый период 2. В этом случае отчет показывает данные для первых двух периодов указанного финансового года и последних четырех периодов предыдущего финансового периода.

### <a name="specify-the-periods-for-an-fd-column"></a>Задание периодов для столбца FD

1. В конструкторе отчетов выберите изменяемое определение столбца.
2. В столбце **FD**, выполните двойной клик ячейки в строке **Период**, и после этого выберите вариант в списке.
3. В строке формул над Областью перехода, или в ячейке **Период**, завершите формулу. Замените знак любого числа (\#) соответствующим значением.

#### <a name="periods-covered-cell"></a>Ячейка Покрытые периоды

Ячейка **Покрытые периоды** определяет сумму, которую столбец должен отобразить. Эта сумма — по отношению к значению в ячейках **Финансовый год** и **Период** для столбца. Доступны следующие параметры.

| Параметр      | Описание                                                                 |
|-------------|-----------------------------------------------------------------------------|
| PERIODIC    | Отображение суммы мероприятия для текущего периода или диапазона периодов. |
| PERIODIC/BB | Отображение начального сальдо для текущего периода или диапазона периодов.   |
| YTD         | Отображение суммы действия с начала года.                               |
| YTD/BB      | Отображение начального сальдо для данного года.                                 |

### <a name="specify-the-periods-that-are-covered-for-an-fd-column"></a>Задание периодов, захватываемых для столбца FD

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. В столбце **FD**, выполните двойной клик ячейки в строке **Покрытые периоды**, и после этого выберите вариант в списке.

### <a name="attribute-filter-in-a-column-definition"></a>Фильтр атрибутов в определении столбцов

Атрибуты представляют собой финансовые данные, дополнительно определяющие счет или транзакцию. Атрибуты счета включают **Актив**, **Задолженность**, **Доход** и **Расход**. Атрибуты проводки включают **Описание проводки** и **Дата применения проводки**. В разных системах Microsoft Dynamics ERP осуществляется разная поддержка атрибутов. Ячейка **Фильтр атрибута** ограничивает данные в столбце **FD** определенными значениями или диапазонами для категорий атрибутов. Хотя эту функцию можно использовать вместе со столбцом **ATTR**, столбец **ATTR** не необходим. В столбце **FD** есть предел на счета или проводки, который отчет включит от фильтра атрибута.

> [!NOTE]
> В руководстве по интеграции вашей системы есть список поддерживаемых атрибутов.

#### <a name="apply-an-attribute-filter-for-an-fd-column-on-a-report"></a>Применение фильтра атрибутов для столбца FD в отчете

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку **Фильтр атрибутов** столбца **FD**.
3. В диалоговом окне **Фильтр атрибутов** дважды щелкните ячейку **Атрибут** и выберите тип фильтра.
4. Чтобы дополнительно ограничивать результаты, введите диапазон в столбцы **От** и **До**. Ячейка **От** должна содержать значение.
5. Нажмите кнопку **OК**.

### <a name="dimension-filter-in-a-column-definition"></a>Фильтр аналитики в определении столбца

Фильтр аналитики используется, чтобы ограничить столбец **FD** специфическими значениями аналитики. Фильтр может содержать отдельное измерение, диапазон измерений или группу измерений. Фильтр также может включать наборы значений измерений. Так как значения аналитик могут меняться, система на основе ..\\финансовые_аналитики\\аналитика не обязательно должна соответствовать точной длине. Фильтр применен, независимо от того, включает ли отчет дерево отчетности. Вы можете использовать подстановочный знак (\* или ?) в любом положении. Когда вы определяете множество счетов, поставьте запятую между счетами, как в следующем примере: +Счет=\[1200\], +Счет=\[1100\], Подразделение=\[01?\] Для того, чтобы получить все отделы для специфического счета, вы можете исключить аналитику "Подразделение" из фильтра аналитик. Например, оба из следующих фильтров аналитик обрабатываются одним и тем же образом:

- +Счет=\[1100\],Подразделение
- +Счет=\[1100\]

Вы можете также использовать любое сочетание цифробуквенные символов для точного сопоставления, и вы можете определить частичные размеры. Например, **Местонахождение = \[10\*\]** включает все значения аналитики положения, которые начинаются с 10.

#### <a name="apply-a-dimension-filter-for-a-column-on-a-report"></a>Применение фильтра измерений для столбца в отчете

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку **Фильтр аналитик** для столбца **FD**.
3. В диалоговом окне **Аналитики**, введите применяемые.
4. Нажмите кнопку **OК**.

### <a name="format-a-multiple-currency-report-in-a-column-definition"></a>Форматирование отчета в нескольких валютах в определении столбца

В отчетах с несколькими валютами суммы могут отображаться в валюте учета ГК, валюте отчетности ГК, исходной валюте проводки или переведенной валюте отчетности. Валюта учета компании определена в настройке главной книги. Не путайте эту настройку с настройкой региональных параметров операционной системы, в которых можно задавать символ валюты по умолчанию, используемый в отчетах. В определении столбцов содержатся следующие связанные с валютой ячейки:

- **Отображение валюты** — укажите тип валюты (учета, отчетности, проводки или переведенной отчетности), в которой отображаются проводки. Функция перевода в валюту отчетности иногда называется преобразованием валюты. Она представляет возможность формировать отчеты по суммам главной книги в валюте, которая может отличаться от функциональной валюты или валюты отчетности компании и от валюты, в которой была введена проводка.
- **Фильтр валюты** — определите фильтр валюты. Только проводки которые введены в выбранной валюте, показаны на отчете.

> 
Чтобы определить валюту учета компании, выполните следующие шаги.

1. В конструкторе отчетов в меню **Компания** выберите **Компании**.
2. В диалоговом окне **Компании** выберите компанию, а затем нажмите **Просмотр**.
3. В диалоговом окне **Просмотр компании**, под **Региональные параметры**, вы можете посмотреть валюту, которая определена для выбранной компании.

#### <a name="specify-the-currency-on-a-multiple-currency-report"></a>Задание валюты для отчета в нескольких валютах

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. Дважды щелкните ячейку **Отображение валюты** в соответствующем столбце **FD**, затем выберите вариант для отображения информации валюты: **Валюта учета главной книги**, **Валюта отчетности ГК**, валюта проводки или выберите перевод в другую валюту отчетности.
3. Дважды щелкните ячейку **Фильтр валюты** в соответствующем столбце **FD**, а затем выберите соответствующий код валюты в списке. Только проводки которые введены в этой валюте, показаны на отчете.


### <a name="example-for-currency-display-and-currency-filter-cells"></a>Пример ячеек "Отображение валюты" и "Фильтр валюты"

- **Фильтр валюты:** Йена
- **Отображение валюты:** валюта учета из книги учета (доллары США)

### <a name="calculation-column-in-a-column-definition"></a>Столбец вычисления в определении столбца

Тип столбца **CALC** в определении столбца поддерживает сложные вычисления в ячейке **Формула** и может включать операторы **+**, **-**, **\**_ и _*/**, а также утверждения **IF/THEN/ELSE**. Столбец вычисления может также сослаться на любой другой столбец, даже последующие столбцы. Дополнительно столбец вычислений может также включать финансовый год и период для поддержки заголовков столбца. Длина формулы вычисления не может превышать 1024 знаков. Чтобы выразить результат выражения в процентах, нужно воспользоваться специальным переопределением формата.

> [!NOTE]
> В результаты расчетных формул не включаются значения столбцов, которые не выводятся на печать. Например, **A:D** печатает **0** (ноль), тогда как **A+B+C** для значений не для печати высчитывает значение.

#### <a name="operators-in-calculation-columns"></a>Операторы в столбцах вычислений

Чтобы добавить, отнять, умножить и разделить столбцы введите литеры столбцов в порядке вычисления и затем используйте соответствующий знак оператора для разделения каждой литеры. Следующая таблица объясняет операторы, которые вы можете использовать в столбце вычисления.

| Оператор | Пример расчета | описание |
|----------|---------------------|-------------|
| +        | A+C                 | Сложить количество в столбце A к количеству в столбце C. |
| :        | A:C A:C-D           | Добавление последовательности столбцов. Например, формула **A:C** складывает суммы столбцов от A до С и формула **A:C-D** складывает суммы столбцов от A до C, а затем вычитает столбец D. |
| -        | A-C                 | Отнимание значения в столбце А от столбца С.<p><strong>Примечание.</strong> Можно также использовать знак "минус" (-), чтобы реверсировать знак в столбце. Например, используйте <strong>-A+B</strong> для того, чтобы добавить обратную сумму в столбце A к сумме в столбце B.</p> |
| \*       | A\*C                | Умножить количество в столбце A на количество в столбце C. |
| /        | A/C                 | Разделить количество в столбце A на количество в столбце C. |

#### <a name="use-a-calculation-formula-in-a-column-definition"></a>Использование формулы вычисления в определении столбца

1. В конструкторе отчетов выберите редактируемое определение столбца.
2. В соответствующем столбце **CALC**, впишите формулу в ячейку **Формула**.

#### <a name="complex-calculations"></a>Сложные вычисления

Сложное вычисление может содержать ссылки на ячейки, операторы, значения и скобки нескольких уровней в любом сочетании. Например, для того, чтобы вычислить среднее столбцов A и B, используйте формулу вычисления **((A+B)/2)**.

#### <a name="specify-report-cells-in-a-column-calculation"></a>Задание ячеек отчета в вычислении столбца

Можно также указать определенную ячейку отчета, введя букву столбца и код строки. Например, **B.100** ссылается на коду строки 100 в столбце B. Вы можете разделить весь столбец на специфическое сумму ячейки отчета, которая в том же столбце. Например, вычисление **B/B.100** значит, что сумма в столбце B должно быть разделено на значение в коде строки 100 в столбце B. Если вычисление ссылается на столбец, который зависит от другого столбца, зависимый столбец разрешается первым. Если столбец ссылается на другой столбец, который, в свою очередь, ссылается на первый столбец, возникает ошибка циклической ссылки.

> [!NOTE]
> Вычисление может дать неверный результат, если изменить приоритеты вычислений для отчета. Вы можете установить приоритет вычисления на вкладке **Установки** определения отчета.

#### <a name="multiply-or-divide-a-column-by-a-base-row"></a>Умножение и деление столбца на базовую строку

Можно создать столбец, в котором будут отображаться значения определенного столбца в виде процента от базового числа. Таким образом, можно отобразить связи между строками, например процент от строки продаж или процент строки общих затрат. Для того, чтобы умножить или разделить каждую строку в определенном столбце на базовую строку, введите столбец, который использовать в вычислении, и после этого введите **\*BASEROW** или **/BASEROW**. Например, впишите **C\*BASEROW** или **C/BASEROW**.

> [!NOTE]
> Если в определении столбцов используется вычисление по базовой строке, убедитесь, что каждое из определений строк, используемых с этим определением столбцов, содержит по меньшей мере одну базовую строку для вычислений.

#### <a name="divide-the-amount-in-a-column-by-the-number-of-periods"></a>Деление суммы в столбце на число периодов

Сумму в столбце можно разделить на указанное число периодов. Например, формула **B/Periods** разделяет значение в столбце B на число периодов в столбце B. Если вычисление занимает множество столбцов, укажите число периодов для использования в вычислении. Например, формула **(B+C)/Periods** складывает количество в столбце B и столбце C, и после этого разделяет результат на значение периода.
