# <a name="whats-new-or-changed-in-dynamics-365-human-resources-september-20-2021"></a>Что нового и что изменилось в Dynamics 365 Human Resources 20 сентября 2021 г.

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

В этой статье описываются новые, измененные или ожидаемые в ближайшее время функции в Microsoft Dynamics 365 Human Resources.

Дополнительные сведения о нашем процессе обновления и графике см. в разделе [Процесс обновления](hr-admin-setup-update-process.md).

Дополнительные сведения о новых функциях и ожидаемых датах общей доступности см. в разделе [Обзор волны 2 выпуска Dynamics 365 Human Resources 2021](/dynamics365-release-plan/2021wave2/human-resources/dynamics365-human-resources/).

## <a name="in-this-release"></a>В данном выпуске

Этот выпуск включает следующие новые функции и исправления ошибок. Изменения применяются для номера сборки 8.1.4464.

### <a name="new-features"></a>Новые возможности

В этой версии следующие функции стали общедоступными.

| Функция | План выпуска | Документация |
|---|---|---|
| Включить упрощенную интеграцию зарплаты (API интеграции зарплаты) | [Включение упрощенной интеграции с поставщиками зарплаты](/dynamics365-release-plan/2021wave1/human-resources/dynamics365-human-resources/enable-simplified-integration-payroll-providers) | [API интеграции зарплаты](hr-admin-integration-payroll-api-introduction.md) |
| Разрешение маркировки сотрудников как готовых к оплате | [Разрешение маркировки сотрудников как готовых к оплате](/dynamics365-release-plan/2021wave1/human-resources/dynamics365-human-resources/enable-employees-be-marked-as-ready-pay) | [Готовы платить](/dynamics365/human-resources/hr-compensation-payroll) |

### <a name="bug-fixes"></a>Исправления ошибок

Этот выпуск содержит следующие исправления ошибок.

> [!NOTE]
> Наша цель — предоставить эту информацию вам как можно скорее. Мы можем обновить эту статью для включения в нее исправлений ошибок, которые были сделаны в сборке после первоначальной публикации этой статьи.

| Номер проблемы | Проблема | Описание |
|---|---|---|
| 619774 | Изменение описания адреса не синхронизируется с Dataverse в режиме реального времени. | При изменении описания для адреса работника обновленное описание не синхронизируется в реальном времени с Dataverse. Подписка в таблице **Местонахождение логистики** обновлена для отправки обновления. |
| 614603| Ошибка на странице **Работник**, если не выбран параметр **Действия персонала работника**. | При найме нового сотрудника или переходе на страницу **Работник** появляется следующее сообщение об ошибке: "Поле **Тип действия сотрудника** должно быть заполнено", даже если **Действия персонала** отключены. |
| 615367 | Вкладка **Утвержденное отсутствие** отображает предупреждение и отображается неправильно. | Если для единицы отпуска установлено значение **Дни**, после чего включается функция **Настройка единиц измерения отпуска для каждого типа отпуска**, на вкладке **Утвержденное отсутствие** отображается недопустимое предупреждение и неправильные столбцы. |

### <a name="benefits-statement-known-issues"></a>Известные проблемы выписки по льготам

| Проблема | Описание |
|---|---|
| Неправильная страница **Параметры отчета** в разделе **Самообслуживание сотрудников** для выписки по льготам. | При переходе к отчету **Выписка по льготам** в модуле **Самообслуживание сотрудников** на странице **Параметры отчета** отображаются экспресс-вкладки **Записи для включения** и **Выполнять в фоновом режиме**.  Их необходимо удалить. |
| Закрытые и будущие периоды отображаются на странице **Параметры отчетов** для выписки по льготам. | При переходе к странице назначения **Отчет выписки по льготам** пользователь может выбрать периоды планов льгот, которые были закрыты или которые помечены датами в будущем, что приведет к открытию пустой страницы. В списке должен отображаться только текущий период плана льгот. |
|Ошибка при отправке отчета по электронной почте с использованием пункта назначения отчета GER. | Для выписки по льготам можно задать использования параметров электронной почты на странице **Пункты назначения отчетов GER**. При завершении настройки и печати отчета пользователь получит сообщение об ошибке форматирования, и выписка по льготам не будет отправлена.|
