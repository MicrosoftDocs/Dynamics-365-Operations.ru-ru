# <a name="create-a-benefits-plan"></a>Создание плана льгот

В этой статье показано, как настроить планы льгот в Dynamics 365 Human Resources.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Создать**.

3. На вкладке **Общие** укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **План** | Уникальный идентификатор для плана. |
   | **Описание** | Описание плана. |
   | **Тип плана** | При создании нового плана необходимо указать тип плана. Тип плана является высокоуровневой группировкой различных типов льгот. Каждый тип плана указывает, может ли сотрудник регистрироваться в нескольких планах этого типа, указывает, являются ли контакты бенефициарами или иждивенцами, и определяет параметры покрытия. Можно создавать новые типы пользовательских планов, соответствующие потребностям ваших льгот. Основные типы планов льгот: <ul><li>401K</li><li>ADD</li><li>Пособие на зубоврачебное обслуживание</li><li>Фитнес</li><li>FSA</li><li>Страхование жизни</li><li>LTD</li><li>Медицинское обслуживание</li><li>PTO</li><li>STD</li><li>Зрение</li></ul> |
   | **Код типа плана** | Код типа плана для типа плана. |
   | **Программа** | Задает программу для необязательного назначения плана. |
   | **Набор** | Задает набор для необязательного назначения плана. |
   | **Сводная** | Указывает, является ли план сводным планом в пакете, которому он назначен. |
   | **Состояние** | Означает текущий статус плана льгот. Значение по умолчанию — "активно". При изменении статуса на "неактивно", план не будет доступен в качестве выбора во время регистрации. |
   | **Действительно с даты и времени** | Дата и время начала плана. Значение по умолчанию — текущая системная дата. |
   | **Действительно до даты и времени** | Дата и время завершения плана (статус задается "неактивный"). Значение по умолчанию — 12/31/2154, что означает "никогда". |

4. На вкладке **Конфигурация** укажите значения для следующих полей в зависимости от типа создаваемого плана:

   | Тип плана | Поле | Описание |
   | --- | --- | --- |
   | Медицина (медицина, стоматология, зрение, HMO) | COBRA | Указывает, распространяется ли на план COBRA (Consolidated Omnibus Budget Reconciliation Act). |
   | Медицина (медицина, стоматология, зрение, HMO) | HIPAA | Указывает, распространяется ли на план HIPAA (Health Insurance Portability and Accountability Act). |
   | <ul><li>Медицина (медицина, стоматология, зрение, HMO)</li><li>Другое (PTO, фитнес)</li><li>Прочее</li><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li><li>Экономия (например, 401 (k))</li><li>FSA</li></ul> | База до учета налогов, допустимая | Указывает, могут ли в план быть сделаны вклады перед применением налогов. |
   | <ul><li>Медицина (медицина, стоматология, зрение, HMO)</li><li>Другое (PTO, фитнес)</li><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li><li>Экономия (например, 401 (k))</li><li>FSA</li></ul> | Разнести налог, допустимый | Указывает, могут ли в план быть сделаны вклады после применения налогов. |
   | <ul><li>Медицина (медицина, стоматология, зрение, HMO)</li><li>Другое (PTO, фитнес)</li><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li><li>Экономия (например, 401 (k))</li><li>FSA</li></ul> | Участник | Указывает, кто участвовал в плане — сотрудник, работодатель или оба варианта. |
   | <ul><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li></ul> | Минимальное покрытие | Минимальная сумма страхового покрытия, требуемая для плана. |
   | <ul><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li></ul> | Максимальное покрытие | Максимальная сумма страхового покрытия, требуемая для плана. |
   | <ul><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li></ul> | Использовать приращения покрытия | Указывает, следует ли проверять соответствие суммы покрытия с допустимой добавочной суммой. |
   | <ul><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li></ul> | Сумма приращения | Добавочная сумма страхового покрытия для плана. Например, если добавочная сумма — 1000, сотрудник не может иметь страховку 200 500 долл. США, ему придется округлить их до 201 000 или до 200 000 долл. США. |
   | <ul><li>Долговременная нетрудоспособность</li><li>ADD (базовое страхование жизни, добровольное страхование жизни)</li></ul> | Инкрементное направление | Определяет направление округления — либо вверх, либо вниз — если сумма покрытия не соответствует значению приращения. |
   | ADD (базовое страхование жизни, добровольное страхование жизни) | Свидетельство о возможности страхования | Указывает, должен ли сотрудник предоставлять свидетельство страхового обеспечения. |
   | ADD (базовое страхование жизни, добровольное страхование жизни) | Сумма, руб. | Сумма в валюте учета. Это поле будет активно только тогда, когда установлен флажок свидетельства страхового обеспечения. |
   | <ul><li>Экономия (например, 401 (k))</li><li>FSA</li></ul> | Минимальный годовой взнос | Минимальная сумма вклада, требуемая для плана. |
   | <ul><li>Экономия (например, 401 (k))</li><li>FSA</li></ul> | Максимальный годовой взнос | Максимальная сумма вклада, требуемая для плана. |
   | Экономия (например, 401 (k)) | Максимальная годовая сумма работодателя | Максимальная сумма, которую может предоставить работодатель для участия в плане экономии сотрудника в течение льготного периода. Чтобы использовать это поле, необходимо установить флажок соответствия работодателя. |
   | Экономия (например, 401 (k)) | Соответствие работодателя | Указывает, участвовал ли работодатель в плане экономии сотрудника. |
   | Экономия (например, 401 (k)) | Процент соответствия работодателя | Процент от вклада сотрудника, которому будет соответствовать работодатель. |
   | Экономия (например, 401 (k)) | Ограничение соответствия работодателя | Максимальный процент, которому будет соответствовать работодатель. Например, если работодатель соответствует 100% вкладов сотрудника до 6% оплаты сотруднику, соответствие работодателя составляет 6%. |

5. На вкладке **Настройка** укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Разрешить/продолжить регистрацию** | Указывает, могут ли сотрудники регистрироваться в плане, если они удовлетворяют требованиям к доступности.</br></br>Если для этого поля установлено значение "Нет", план не будет доступен сотрудникам при обработке доступности. |
   | **Автоматическая регистрация за предыдущий год** | Указывает, следует ли автоматически регистрировать допустимых сотрудников в плане, если они были зарегистрированы в течение предыдущего года. |
   | **Автоматическая регистрация по умолчанию** | Указывает, нужно ли выбирать план регистрации по умолчанию. План не является обязательным, поэтому сотрудник может изменить выбор по умолчанию. |
   | **Закрыто для новых регистраций** | Указывает, следует ли ограничить план только подходящими сотрудниками, зарегистрированными в плане за предыдущий год. |
   | **Обязательный план** | Указывает, следует ли автоматически регистрировать сотрудников в плане. Сотрудники не могут изменить выбор регистрации. |
   | **Дата начала действия** | Дата создания плана в компании. |
   | **Счет поставщика** (поставщик льгот) | Поставщик, которому компания выплачивает вознаграждение для этого плана. |
   | **Имя** (поставщик льгот) | Название поставщика. |
   | **Ссылка поставщика** (поставщик льгот) | Ссылка поставщика для плана. Например, номер группового плана компании. |
   | **Альтернативная ссылка** (поставщик льгот) | Альтернативная ссылка поставщика для плана. Например, номер счета компании. |
   | **Валюта** (поставщик льготы) | Валюта, используемая для выплаты вознаграждения поставщику. |
   | **Счет расходов** (поставщик льгот) | Счет ГК, который используется как счет расходов для премий плана. |
   | **Счет поставщика** (администратор льгот) | Поставщик, которому компания оплачивает администрирование для плана. Если план самостоятельно администрируется, оставьте это поле пустым. |
   | **Имя** (администратор льгот) | Имя поставщика администратора льгот. |
   | **Ссылка поставщика** (администратор льгот) | Ссылка поставщика администратора для плана. |
   | **Альтернативная ссылка** (администратор льгот) | Альтернативная ссылка поставщика администратора для плана. |
   | **Валюта** (администратор льгот) | Валюта, используемая для выплаты администратору льгот. |
   | **Счет расходов** (администратор льгот) | Счет ГК, который используется как счет расходов для затрат, связанных с администрированием плана. |

6. На вкладке **Фильтры** выберите нужный фильтр. Можно выполнять фильтрацию по следующим полям:

   - **Бизнес-единица**
   - **Отдел**
   - **Информация о компании**
   - **Местоположение**
   - **Занимаемая должность**

7. На вкладке **Правила допустимости** укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Номер строки** | Номер строки правила допустимости. |
   | **Правило допустимости** | Правило допустимости, применяемое к плану льгот. Это правило допустимости будет применяться к соответствующему типу действия и связано с заданным периодом ожидания покрытия и вычетами. |
   | **Тип действия** | Действие, применяемое к правилу допустимости для: регистрация льгот или окончание срока действия льгот. |
   | **Период ожидания покрытия** | Значение из формы периодов ожидания. Период ожидания покрытия определяет количество дней или месяцев, в течение которых сотрудник ждет покрытия льгот или истечения срока льготы на основе критериев в правиле допустимости и типе действия. |
   | **Период ожидания вычета** | Значение из формы периодов ожидания. Период ожидания вычета определяет количество дней или месяцев, в течение которых сотрудник ждет вычета льгот из оплаты на основе критериев в правиле допустимости и типе действия. |

8. Нажмите **Сохранить**.

## <a name="view-enrolled-workers"></a>Просмотр зарегистрированных работников

Можно просмотреть работников, которые зарегистрированы в выбранном плане льгот.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Зарегистрированные работники**.

## <a name="attach-coverage-options"></a>Присоединить параметры покрытия

Можно добавить параметры покрытия в выбранный план льгот. Прикрепление параметров покрытия приводит к совместной настройке ставки и вычета для параметра покрытия.  Пример: для медицинского плана пользователь должен выбрать параметр покрытия "семья".  Затем они должны выбрать ставку для семьи для связанного плана (в настройке ставки) и вычет для связанного плана (в настройке ставки). Это позволяет получить затраты для работодателя и сотрудника для выбранного покрытия. Затем выполняется повтор процесса для покрытия "сотрудник+1" или покрытие сотрудника.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Прикрепить параметры покрытия**.

## <a name="override-eligibility-rules"></a>Переопределения правил допустимости

Можно добавлять работников в план в качестве исключений в правилах допустимости. Каждый добавляемый работник будет иметь право на план льгот.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Переопределение правил допустимости**.

## <a name="view-attached-periods"></a>Просмотр присоединенных периодов

Можно просмотреть список доступных периодов льгот.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Периоды**.

## <a name="view-plan-information"></a>Просмотр информации о плане

Можно создать описание плана, чтобы помочь сотрудникам в выборе льгот. Введенные здесь сведения о плане отображаются в самообслуживании сотрудников при наведении указателя на план в списке параметров покрытия.

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Сведения о плане**.

## <a name="view-flex-credit-programs"></a>Просмотр программ Flex Credit

1. В рабочей области **Управления льготами** в **Планы** выберите **Планы льгот**.

2. Выберите **Гибкие программы кредитования**.
