# <a name="manage-features"></a>Управление функциями

В рамках нашего непрерывного процесса развертывания новых возможностей для Microsoft Dynamics 365 Human Resources мы хотим предоставить клиентам возможность испытать новые функции. Мы предоставляем предварительную версию, которая почти готова и прошла множество тестов. Мы хотели бы получить окончательные отзывы клиентов и выполнить проверку до выпуска общедоступной версии.

Дополнительные сведения о новых возможностях в Human Resources см. в разделах [Что нового в Human Resources](hr-admin-whats-new.md) и [План выпуска Dynamics 365 и Power Platform](https://docs.microsoft.com/dynamics365/release-plans/#pivot=products&panel=products1).

Рабочая область **Управление функциями** содержит список функций, предоставляемых в каждом выпуске. По умолчанию новые функции отключены. Можно использовать рабочую область для их включения и просмотра документации по ним. Дополнительные сведения об управлении функциями см. в разделе [Обзор управления функциями](https://docs.microsoft.com/dynamics365/fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview).

Все новые возможности остаются в виде предварительного просмотра по крайней мере на 30 дней, а обычно 30-60 дней. Основные функции обычно доступны в октябре и апрель каждого года, следующего за периодом предварительного просмотра. После отображения новых возможностей в рабочей области **Управление функциями** их можно включить. Некоторые функции могут быть включены по умолчанию.

Когда функция обычно доступна, она может быть включена или выключена в производственных средах. Рабочая область **Управление функциями** показывает, что функция предварительного просмотра станет обязательной. Эта дата обычно приходится на 1 октября или 1 апреля, чтобы соответствовать полугодовым плановым выпускам. Вы не можете отключить обязательные функции. До тех пор, пока она не станет обязательной, можно включать и выключать функцию во всех средах.

## <a name="enable-or-disable-preview-features"></a>Включение и отключение функций предварительного просмотра

Чтобы получить доступ к предварительным версиям функций, необходимо сначала включить их в своей среде. Включение или отключение функций предварительного просмотра зависят от среды.

> [!IMPORTANT]
> Функции предварительной версии доступны только в средах **Песочница**. При включении функции предварительного просмотра вы включаете ее для всех пользователей в организации, имеющихся в данной среде. При отключении функции предварительного просмотра вы отключаете ее и делаете ее недоступной для пользователей. Функции предварительного просмотра имеют ограниченную поддержку в Human Resources. К ним могут применяться меньше требований к соблюдению конфиденциальности и безопасности, и они не включаются в соглашение об уровне обслуживания (SLA) Human Resources. Не следует использовать функции предварительного просмотра для обработки личных данных (то есть любой информации, которая может идентифицировать пользователя) или для обработки других данных, на которые действуют юридические или нормативные требования.

1. В модуле Human Resources выберите **Администрирование системы**.

2. Выберите плитку **Управление функциями**.

3. Чтобы включить функцию предварительного просмотра, выберите ее из списка, а затем выберите **Включить**. Чтобы отключить функцию предварительного просмотра, выберите ее из списка, а затем выберите **Отключить**.

## <a name="preview-feature-benefits-management"></a>Предварительная версия функции: управление льготами

Управление льготами предоставляет гибкое решение, которое поддерживает широкий спектр вариантов льгот, в том же, как и простое в использовании впечатление сотрудников, что позволяет продемонстрировать ваши предложения. Дополнительные сведения о настройке и использовании управления льготами см. в разделе [Обзор управления льготами](hr-benefits-management-overview.md).

Управление льготами заменяет функциональные возможности в рабочей области **Льготы**. После включения функции предварительного просмотра управления льготами вы не сможете получить доступ к следующим формам в рабочей области **Льготы**:

- **Льготы**
- **Элементы льготы**
- **Ставки расчета вкладов**
- **Результаты регистрации льготы**
- **Результаты сокращения или продления срока действия льготы**
- **Типы правил политики права на льготу**
- **Политики прав на льготы**
- **События допустимости**

Сведения в этих формах можно просматривать в режиме только для чтения. Если необходимо изменить данные, необходимо сначала отключить функцию предварительного просмотра управления льготами.

### <a name="benefits-management-known-issues"></a>Известные проблемы управления льготами

#### <a name="life-events"></a>Жизненные события

При обработке событий срока службы пользователь получит сообщение об ошибке:

Дата начала покрытия должна находиться в диапазоне *от начала периода плана* до *окончания периода плана*. 

Срок жизни события будет по-прежнему обрабатываться должным образом.

#### <a name="eligibility-processing"></a>Обработка допустимости

При выполнении проверки допустимости льгот, использующих сумму покрытия 1-5X зарплаты, % от зарплаты и фиксированной суммы, дата сведений о льготе должна быть установлена на дату начала сотрудника в **журнале занятости** с отработанными часами, частотой оплаты и суммой зарплаты годовых льгот. Если для работника существует постоянная компенсация, введите количество часов, отработанных с частотой выплаты, и будет вычисляться сумма годовой заработной платы. Если сотруднику начисляется заработная плата, отработанное время не нужно. При создании новых работников рекомендуется сначала ввести фиксированную компенсацию. Чтобы обновить запись сведений о льготе: перейдите **Работник > Журнал работника> Сведения о занятости**. Скорректируйте дату на дату начала работника.

#### <a name="employee-self-service"></a>Самообслуживание сотрудника

Сотрудники могут выбрать план, для которого они не квалифицированы, и оформить выдачу. Например, у работника нет иждивенцев, но ему разрешено выбирать медицинской план с параметром семейного покрытия.

Сумма сотрудника не рассчитывается при обновлении суммы покрытия для страхования жизни. Например, когда сотруднику предлагается план страхования жизни, они могут выбрать 50 000 долл. США в покрытии при стоимости 0,36 долл. США на 1000 долл. США покрытия.  Когда сотрудник обновляет сумму покрытия, связанные затраты сотрудника остаются в нулевом виде.

Если план льготы разрешает только один выбор этого типа плана, пользователь получит сообщение об ошибке, если попытается отказаться от плана после выбора плана. Например, пользователь выбирает медицинский план и помещает его в корзину. После этого пользователь выбирает **Отказ** для другого медицинского плана. При этом пользователь получит сообщение об ошибке.

## <a name="preview-features-in-leave-and-absence"></a>Предварительные функции в разделах отпуска и отсутствия

Предварительные функции в разделах отпуска и отсутствия:

- **Календарь отпуска и отсутствий** — параметры отпуска и отсутствия будут перенесены из **параметров Human Resources** на новый экран с названием **Параметры отпуска и отсутствия**. Новый экран включает новую вкладку **Календарь**. Этот предварительная версия включает только подмножество параметров. Доступ к новому экрану осуществляется со вкладки **Ссылки** рабочей области **Отпуск и отсутствие**. Календари включают:
  - **Календарь компании** — показывает все запросы на отгулы сотрудников. Пользователи с ролью **Управление персоналом** могут получить доступ к этому календарю со вкладки **Ссылки** рабочей области **Отпуск и отсутствие**.
  - **Календарь команды менеджера** — отображаются все непосредственные запросы отгулов. Менеджеры могут получить доступ к календарю со вкладки **Моя группа** в модуле самообслуживания сотрудников в **Отпуск и отсутствие**. 

- **Календари праздников с отпуском и отсутствием** — типы отпусков включают новый параметр **Праздник**, используемый в сочетании с календарем рабочего времени. Дни, определяемые праздниками и нерабочими днями, в данный момент обозначаются как **Праздник** при создании рабочих дней. При обработке начислений выполняется корректировка сотрудников, назначенных для календаря, для учета праздников, выпадающих на рабочий день.

- **Аудита начислений по отпуску** — новый экран позволяет просмотреть, когда начисления были обработаны и удалены как всеми сотрудниками, так и отдельными сотрудниками. Доступ к этому экрану осуществляется со вкладки **Ссылки** рабочей области **Отпуск и отсутствие**.

- **Удаление начисления по отпуску** — теперь можно удалить записи начисления для определенного плана отпусков. Доступ к этому параметру осуществляется со вкладки **Ссылки** рабочей области **Отпуск и отсутствие**. Для отдельных сотрудников этот параметр отображается в группе **Отпуск и отсутствие** в профиле сотрудника. 

- **Округление начисления по отпуску** — новые параметры для **Типа отпуска** определяют, какой тип округления следует использовать, и точность округления в десятичных знаках в процессе начисления. При обработке начислений для записей начисления применяются округления и точность. 

- **Настройка нескольких типов отпуска для одного плана отпуска** — новый столбец в плане начислений для типов отпусков позволяет определять несколько типов отпусков в отпуске и плане отсутствия с различными планами начисления. Предыдущее поле **Тип отпуска** удаляется. При регистрации сотрудника сальдо для типов отпусков сейчас отображается в таблице, а не в верхней части экрана.

  > [!IMPORTANT]
  > После включения эту функцию отключить невозможно.

- **Использовать полную занятость сотрудника (FTE) для начисления** — новый столбец в плане начисления по отпуску позволяет использовать FTE для начисления. Когда начисления обработаны, приложение использует основную должность сотрудника и FTE, определенное для определения пропорциональной суммы начисления.

  > [!NOTE]
  > Эта функция доступна только в том случае, если включено **Настройка нескольких типов отпусков по плану отпуска**.
