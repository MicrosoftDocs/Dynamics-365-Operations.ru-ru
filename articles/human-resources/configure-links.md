# <a name="create-links-from-human-resources-to-another-finance-environment"></a>Создание ссылок из Human Resources на другую среду Finance

Клиент может иметь две среды Dynamics 365, в которых он работает. В качестве примера у клиента может быть среда Dynamics 365 Human Resources в инфраструктуре Finance, и необходимо подключиться к другой среде Dynamics 365 Finance. 

Эта функция разрешит ссылки со страницы Human Resources на конкретную страницу в другой среде Finance. Когда ссылки настроены, можно указать, где будет доступна ссылка в модуле Human Resources, а также целевую страницу, которая будет открыта в другой среде.

> [!Note] 
> Для получения этой функции нужно включить **Улучшения пользовательского интерфейса Human Resources** в **Управлении функциями**.

## <a name="configure-target-systems"></a>Настройка целевых систем

В модуле Human Resources системные администраторы могут определять ссылки, которые отображаются на **Human Resources**. Частями конфигурация являются среды Finance, в которые вы хотите переходить в качестве "цели" ссылки. 

Настроить целевую систему
1. На странице **Настройка ссылок** нажмите **Настроить целевую систему**.  
2. Введите имя целевой системы и укажите URL-адрес среды Finance. После настройки ваших целевых систем можно определить ссылки.

## <a name="configure-links"></a>Настроить ссылки

В каждой созданной ссылке будут определены следующие сведения.
 - **Ссылка**: наименование ссылки. Использовано только в целях идентификации.
 - **Включить эту ссылку** — задайте значение **Да**, если требуется отобразить эту ссылку для пользователей Human Resources.
 - **Отображаемое имя** — определите имя, которое будет отображаться как ссылка на вторичную среду. 
 - **Поверхностная ссылка на форму** — выберите, на какой странице должна отображаться эта ссылка. Ссылки могут быть представлены только в рабочей области **Самообслуживания сотрудников** и на страницах **Задание**, **Позиция**, **Рабочий** и **Оптимизированные рабочие роли**.
 - **Группа**: можно организовать ссылки, используя группы. Выберите существующую группу или создайте новый столбец **Группа**.
 - **Целевая система** — выберите целевую систему, которая была создана с помощью параметра **Настроить целевую систему**. Это будет вторичная среда, которая будет использоваться при переходе с помощью ссылки.
 - **Использовать текущую компанию пользователя** — выберите **Да**, если вы хотите использовать контекст текущей компании пользователя при переходе к Finance. Выберите **Нет**, чтобы выбрать компанию, которая будет использоваться.
 - **Целевой** пункт меню — введите пункт меню из среды Finance, который ссылка должна использовать при переходе. Доступны пункты меню, к которым можно перейти напрямую. 

   Чтобы найти необходимый пункт меню:
   1. Откройте среду Finance и откройте страницу, которая является целевой для навигации. 
   2. Скопируйте пункт меню из URL-адрес. Например, если вы хотите, чтобы ссылка вела на список сотрудников в Finance and Operations, введите значение, которое появляется после "&mi" в URL-адресе. 
   3. Пункт меню для перехода к странице списка сотрудников в этом примере имеет вид: HcmWorkerListPage_Employees.

 - **Ссылка на источник данных** — выберите источник данных, на который ссылается ссылка. Доступны наиболее распространенные источники, такие как **Рабочий** и **Должность**.

### <a name="access-to-links"></a>Доступ к ссылкам

Системные администраторы будут видеть вновь созданные ссылки на определенные страницы, даже если для параметра **Включить эту ссылку** задано значение **Нет**. Это можно использовать для тестирования ссылок перед их отображением другим сотрудникам. Все другие роли будут видеть настроенные ссылки только после того, как для параметра **Включить эту ссылку** было задано значение **Да**. Сотрудники, которые имеют доступ к страницам, на которых отображаются ссылки, будут иметь доступ к ссылкам.

Кроме того, пользователи должны иметь права доступа в рамках вторичной среды, определяемой доступом к страницам в этой среде. Если они отсутствуют, при использовании ссылки будет отображаться диалоговое окно безопасности.
