# <a name="configure-eligibility-rules-and-options"></a>Настройка правил и параметров приемлемости

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

После настройки необходимых параметров для управления льготами в Microsoft Dynamics 365 Human Resources можно создать правила допустимости, наборы, периоды и программы, которые будут связаны с планами льгот.

## <a name="create-an-eligibility-rule"></a>Создание правила допустимости

Правила допустимости определяют, какие сотрудники могут регистрироваться в каждом плане льгот. После определения правил допустимости их можно назначить планам льгот. Затем можно обработать допустимость регистрации, чтобы увидеть, какие сотрудники могут получить право для каждого плана. 

Во время открытой регистрации сотрудники могут выбирать планы льгот. Если они станут недоступны для плана льгот на основе правил допустимости после того, как они уже были зарегистрированы, их регистрация не будет автоматически отменена. Обычно, когда происходит жизненное событие, которое влияет на допустимость для плана, период регистрации инициируется для сотрудника, который выбирает допустимые для себя планы. 

1. В рабочей области **Управление льготами** в **Настройка** выберите **Плавила и параметры допустимости**.

2. На вкладке **Правила допустимости** выберите **Создать**, чтобы создать правило допустимости. Для просмотра планов, связанных с правилом допустимости, выберите **Присоединенные планы**.

3. Укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Правило допустимости** | Уникальный идентификатор для правила допустимости. |
   | **Описание** | Описание правила допустимости. |
   | **Действительно с даты и времени** | Дата начала правила допустимости. | 
   | **Действительно до даты и времени** | Дата окончания правила допустимости. |
   | **Тип сотрудника пользователя** | Указывает, следует ли использовать тип сотрудника в правиле допустимости льготы. |
   | **Тип работника** | Тип работника, если переключатель **Использовать тип сотрудника** задан как **Да**. |
   | **Использовать статус сотрудника** | Указывает, следует ли использовать статус занятости в правиле допустимости льготы. |
   | **Состояние** | Статус работника, если переключатель **Использовать статус сотрудника** задан как **Да**. Если переключатель **Использовать статус сотрудника** задан как **Нет**, поле не используется. |
   | **Использовать категорию трудоустройства** | Указывает, следует ли использовать значение сотрудника **Категория занятости** в правиле допустимости льготы. | 
   | **Категория трудоустройства** | Категория занятости сотрудника, если для переключателя **Использовать категорию занятости** установлено значение **Да**. |
   | **Использовать новое правило приема на работу** | Указывает, следует ли использовать значение нового периода найма на работу в рамках правила допустимости льгот. |
   | **Период регистрации** | Период времени, в течение которого разрешается новая регистрация для приема на работу. Если также задать это в параметрах, настройка параметра имеет приоритет над данным значением параметра. |
   | **Использовать бывший статус сотрудника** | Указывает, следует ли использовать предыдущий статус трудоустройства сотрудника в правиле допустимости льгот. Например, можно указать правило применимости, которое отменяет период ожидания покрытия для всех сотрудников, которые перешли со статуса **Снят с работы** в статус **Трудоустроены** в течение 90 дней со своего предыдущего трудоустройства. |

4. В **Дополнительные критерии** выберите следующие параметры и добавьте необходимые сведения:

   | Параметр | Описание |
   | --- | --- |
   | **Допустимый срок** | Указывает возрастной диапазон или диапазоны, необходимые для удовлетворения правилу допустимости. |
   | **Допустимое подразделение** | Указывает подразделение или подразделения, в которых должен быть сотрудник, для соответствия правилу допустимости. |
   | **Допустимый тип трудоустройства** | Указывает тип занятости или типы категорий, в которых должен быть сотрудник, для соответствия правилу допустимости. Например, полное или частичное время. |
   | **Допустимое задание** | Указывает задание или задания, которые удовлетворяют правилу допустимости. Задания связаны с должностями, и должности заполняются сотрудниками. |
   | **Допустимая функция должности** | Указывает функциональные обязанности или функции, которые удовлетворяют правилу допустимости. Например, работники по продажам или техники. |
   | **Допустимый тип должности** | Указывает тип задания или типы, которые удовлетворяют правилу допустимости. Например, клерки или руководители. |
   | **Допустимые юридические лица** | Указывает юридическое лицо или юридические лица, удовлетворяющие правилу допустимости. Например, Contoso Entertainment System USA. |
   | **Приемлемый регион компенсации** | Указывает местоположение сотрудника, удовлетворяющее правилу допустимости. Например, центральная часть США. |
   | **Допустимая должность** | Указывает должность или должности, которые удовлетворяют правилу допустимости. Например, помощник по кадрам или менеджер по кадрам. |
   | **Допустимый тип должности** | Указывает тип должности или типы, которые удовлетворяют правилу допустимости. Например, полное время. |
   | **Допустимое состояние** | Указывает регионы, которые удовлетворяют правилу допустимости. Например, Северная Дакота США или Британская Колумбия, Канада. |
   | **Допустимые условия трудоустройства** | Указывает условия занятости, удовлетворяющее правилу допустимости. Например, без заключения коллективного договора или с его заключением. |
   | **Допустимый профсоюз** | Указывает участие в профсоюзе, которые удовлетворяют правилу допустимости. Например, операторы вилочных погрузчиков Америки. </br></br>При использовании правила допустимости на основе профсоюза в записи профсоюза работника должна быть указана дата окончания. Поле не может быть пустым. |
   | **Допустимый почтовый индекс** | Указывает почтовые индексы, которые удовлетворяют правилу допустимости. Например, 58104. |

5. В разделе **Дополнительные сведения** можно просмотреть следующие дополнительные сведения:

   | Поле | Описание |
   | --- | --- |
   | **Поле допустимого пользователя** | Задает дополнительные правил допустимости на основе полей, определяемых пользователем. |
   | **Тип допустимости** | Указывает категорию критериев, выбранную в **Дополнительные критерии**. |
   | **Ссылка на допустимость** | Указывает значения, выбранные в **Дополнительные критерии**. |
   | **Описание** | Описание, выбранное в **Дополнительные критерии**. |

6. Нажмите **Сохранить**.

## <a name="configure-bundles"></a>Настройка наборов

Наборы представляют собой набор связанных планов льгот. Наборы льгот можно использовать для группировки планов льгот, которые сотрудник должен выбрать для регистрации в определенных планах льгот, которые могут зависеть от других регистраций планов льгот. Примеры использования наборов:

- Набор плана здравоохранения, который включает в себя медицинскую страховку с большими вычетами с соответствующей HSA.

- План страхования жизни с обязательным планом страхования жизни сотрудника, объединенного с зависимым планом жизни. Это обеспечит сотруднику возможности выбора зависимого покрытия жизни без регистрации для покрытия сотрудника.

1. В рабочей области **Управление льготами** в **Настройка** выберите **Плавила и параметры допустимости**.

2. На вкладке **Наборы** выберите **Создать**, чтобы создать набор. Для просмотра планов, связанных с набором, выберите **Присоединенные планы**.

3. Укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Набор** | Уникальный идентификатор для набора. |
   | **Описание** | Описание набора. |
   | **Сводная** | Указывает, должен ли один из планов в наборе быть помечен как сводный план. Сводный план должен быть выбран во время открытой регистрации в составе набора до того, как администратор льгот может подтвердить выбор льгот сотрудником. |
   | **Действительно с даты и времени** | Дата и время, когда набор становится активным. |
   | **Действительно до** | Дата истечения срока действия набора. Значение по умолчанию — 12/31/2154, что означает "никогда". |

4. Нажмите **Сохранить**.

## <a name="configure-periods"></a>Настройка периодов

Периоды определяют, когда действуют льготы и разрешена ли регистрация для сотрудников. Можно создать столько периодов, сколько необходимо, чтобы обеспечить поддержку открытой регистрации льгот и периодов покрытия льгот. Годы планов льгот могут соответствовать или не соответствовать календарному году. 

1. В рабочей области **Управление льготами** в **Настройка** выберите **Плавила и параметры допустимости**.

2. На вкладке **Периоды** выберите **Создать**, чтобы создать период. Чтобы выполнить процесс, который присоединит все допустимые активные планы льгот к периоду льгот, выберите **Прикрепить планы**. Для просмотра планов, связанных с набором, выберите **Присоединенные планы**. 

3. Укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Период** | Уникальный идентификатор для периода. |
   | **Действительно с даты и времени** | Дата и время начала, когда период льгот активен. |
   | **Действительно до даты и времени** | Дата и время, когда период льгот неактивен. |
   | **Начать регистрацию** | Дата начала открытой регистрации. Открытая регистрация используется для того, чтобы сотрудник мог выбрать льготы при самообслуживании. |
   | **Завершение регистрации** | Дата завершения открытой регистрации. |
   | **Открытие** | Указывает, открыт ли период на основе системной даты, а также допустимых дат и времени начала и окончания. | 
   | **Предыдущий период** | Указывает период льгот, предшествующий выбранному периоду льгот. Эти сведения используются во время регистрации допустимости льгот, чтобы назначить планы, параметры покрытия и назначаемых лиц из предыдущего года. |
 
4. Нажмите **Сохранить**.

## <a name="use-a-flex-credit-program"></a>Использование гибкой кредитной программы

Можно использовать программы гибкого кредитования для регистрации сотрудников в льготах в соответствии с предварительно определенным количеством кредитов по гибкому графику. Сотрудники могут выбрать способ распределения гибких кредитов. Например, если на сотрудника распространяется страховой план здравоохранения супруги, он может использовать кредиты, которые в противном случае использовались бы на покрытие здравоохранения, для других льгот.

1. В рабочей области **Управление льготами** в **Настройка** выберите **Плавила и параметры допустимости**.

2. На вкладке **Периоды** выберите **Гибкие программы кредитования**.

3. Выбор применимой гибкой программы кредитования. Поле содержит следующие сведения:

   | Поле | Описание |
   | --- | --- |
   | Код кредита льготы | Уникальный код гибкой кредитной программы. |
   | Описание | Описание программы гибкого кредитования. | 
   | С даты | Дата, в которую программа гибкого кредитования становится активной. |
   | До даты | Дата, в которую программа гибкого кредитования закрывается. Можно оставить значение по умолчанию (12/31/2154), чтобы указать на то, что программа гибкого кредитования не имеет запланированного окончания срока действия. |
   | Итоговое значение по кредиту | Число кредитов, которое потребуется использовать сотруднику для получения льгот. |
   | Правило пропорции | Правило использования для пропорции гибких кредитов, когда сотрудник принимается на работу в середине гибкого кредитного периода. </br></br><ul><li>**Нет** — сотрудник не получает кредитов по гибкому графику работы, если он принят на работу после начала периода действия кредитной программы по гибкому графику работы.</li><li>**Полный кредит** — сотрудник получает полную сумму кредитов по гибкому графику, независимо от того, когда он принят на работу.</li><li>**Пропорция** — сотрудник получает пропорциональную сумму кредитов по гибкому графику на основе даты их начала.</li></ul> |
   | Формула расчета пропорции гибкого кредита | Правило использования для пропорции гибких кредитов, когда сотрудник принимается на работу в середине периода льгот для программы гибкого кредитного периода. Пропорция базируется на дате начала трудоустройства. Это поле используется только если выбрано значение **Пропорция** в поле **Правило пропорции**. </br></br><ul><li>**Ежедневно** — пропорция количества кредитов по гибкому графику, который сотрудник получает на уровне дня. Общее число гибких кредитов делится на число дней в периоде. Например, если период льготы составляет 400 дней, система разделяет общее число кредитов по гибкому графику на 400 для расчета количества кредитов, полученных сотрудниками по гибкому графику работы в день.</li><li>**Текущий месяц** — пропорциональное количество кредитов по гибкому графику, которые сотрудник получает на уровне месяца, округляется до текущего месяца. Общее число гибких кредитов делится на число месяцев в периоде. Например, если период льготы составляет 15 месяцев, система разделяет общее число кредитов по гибкому графику на 15 для расчета количества кредитов, полученных сотрудниками по гибкому графику работы в месяц.</li><li>**Следующий месяц** — пропорциональное количество кредитов по гибкому графику, которые сотрудник получает на уровне месяца, округляется до следующего месяца. Общее число гибких кредитов делится на число месяцев в периоде. Например, если период льготы составляет 15 месяцев, система разделяет общее число кредитов по гибкому графику на 15 для расчета количества кредитов, полученных сотрудниками по гибкому графику работы в месяц.</li></ul> |
   
   Убедитесь, что каждый план льгот зарегистрирован только в одной гибкой кредитной программе за льготный период. В противном случае система не сможет определить, какая гибкая кредитная программа будет использоваться для предоставления гибких кредитов и при этом будут обнаружены проблемы. 

## <a name="configure-programs"></a>Настройка программ

Программы представляют собой набор планов льгот, использующих общий набор правил допустимости. Можно определить правила допустимости для всей программы, а не для каждого отдельного плана. Например, программа Contoso Canada FTE или программа на уровне руководителей Contoso Europe. 

1. В рабочей области **Управление льготами** в **Настройка** выберите **Плавила и параметры допустимости**.

2. На вкладке **Программы** выберите **Создать**, чтобы создать программу. Чтобы сделать исключения для сотрудников, не отвечающих требованиям к правилу допустимости, выберите **Переопределение правила допустимости**. Для просмотра планов, связанных с программой, выберите **Присоединенные планы**.

3. Укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Программа** | Уникальный идентификатор для программы. |
   | **Описание** | Описание программы. | 
   | **Действительно с даты и времени** | Дата и время, когда программа становится активной. |
   | **Действительно до даты и времени** | Дата и время, когда истекает срок действия программы. Значение по умолчанию — 12/31/2154, что означает "никогда". |
   | **Период ожидания покрытия** | Период, в течение которого сотрудник должен ожидать перед началом покрытия для программы льгот. |
   | **Период ожидания вычета** | Период, в течение которого сотрудник ожидаем перед началом вычета для программы льгот. |
   | **Правила допустимости** | Выберите правила допустимости, которые будут применяться к программе льгот. Правила допустимости можно определить на вкладке **Правила допустимости** на этой странице. |
   
4. Нажмите **Сохранить**.
