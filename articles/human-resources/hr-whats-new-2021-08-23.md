# <a name="whats-new-or-changed-in-dynamics-365-human-resources-august-23-2021"></a>Что нового и что изменилось в Dynamics 365 Human Resources (23 августа 2021 г.)

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

В этой теме описываются новые, измененные и ожидающиеся компоненты в Microsoft Dynamics 365 Human Resources.

Дополнительные сведения о нашем процессе обновления и графике см. в разделе [Процесс обновления](hr-admin-setup-update-process.md).

Дополнительные сведения о новых функциях и ожидаемых датах общей доступности см. в разделе [Обзор волны 2 выпуска Dynamics 365 Human Resources 2021](/dynamics365-release-plan/2021wave2/human-resources/dynamics365-human-resources/).

## <a name="in-this-release"></a>В данном выпуске

Этот выпуск включает следующие новые функции и исправления ошибок. Изменения применяются для номера сборки 8.1.4419.

### <a name="bug-fixes"></a>Исправления ошибок

Этот выпуск содержит следующие исправления ошибок.

> [!NOTE]
> Наша цель — предоставить эту информацию вам как можно скорее. Можно обновить этот раздел для включения исправлений ошибок, которые были сделаны в сборке после первоначальной публикации этого раздела.

| Номер проблемы | Проблема | описание |
| --- | --- | --- |
| 594066 | Невозможно удалить контактную информацию | При выборе удаления записи контактной информации для сотрудника вместо этого удаляется запись контактной информации, отличная от выбранной записи. |
| 611339 | Добавление личной настройки приводит к тому, что банковский счет игнорирует фильтр и выбирает первую запись | Добавление личной настройки приводит к тому, что список банковских счетов запускает запрос персонализации после выполнения запроса datasource, в результате чего запрос получает верхнюю запись независимо от сотрудника, для которого просматриваются эти сведения. |
| 591806 | Неправильно рассчитываются задачи даты выполнения подключения | Даты выполнения рассчитываются неправильно, когда выполняются следующие условия: созданные контрольные списки используют календарь, в котором используется расширенный диапазон времени (например, с 2005 по 2050), а в параметрах пользователя используется часовой пояс, отличный от центрального поясного времени. |   
| 592749 | Сальдо отпуска в **службе самообслуживания сотрудников** неверное | Если сотрудник трудоустроен в нескольких юридических лицах и включен режим просмотра по всем компаниям, сальдо отпуска может быть неверным. |
| 603133 | Непредвиденное предупреждение при запросе отгула | Когда запрашивается отгул, заполните поле **Половина дня**, прежде чем поле **сумма** выдаст непредвиденное предупреждение. |
| 606546 | Выберите поле, которое не отображается в утвержденные отгулы | Флажок для выбора нескольких утвержденных запросов на отпуск не виден. |
| 597059 | Сотрудник не отображается в **календаре компании отпуск и отсутствия** | Работник не отображается в **календаре отпусков и отсутствия компании**, если примененный диапазон дат включает любой день, для которого они отклонили запрос на отпуск. |
