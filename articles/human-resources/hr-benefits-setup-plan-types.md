# <a name="create-plan-types"></a><span data-ttu-id="45cf7-102">Создание типов планов</span><span class="sxs-lookup"><span data-stu-id="45cf7-102">Create plan types</span></span>

[!include [banner](includes/preview-feature.md)]

<span data-ttu-id="45cf7-103">Тип плана в Microsoft Dynamics 365 Human Resources является высокоуровневой группировкой различных типов льгот.</span><span class="sxs-lookup"><span data-stu-id="45cf7-103">A plan type in Microsoft Dynamics 365 Human Resources is a high-level grouping of specific types of benefits.</span></span> <span data-ttu-id="45cf7-104">Каждый тип плана имеет код типа плана, который определяет правила для данного типа плана.</span><span class="sxs-lookup"><span data-stu-id="45cf7-104">Each plan type has a plan type code that determines rules for the plan type.</span></span> <span data-ttu-id="45cf7-105">Например, тип плана "основное страхование жизни" будет иметь код типа плана "жизненный", так как это вид плана страхования жизни, и должен соответствовать правилам, установленным для кода типа плана "жизнь".</span><span class="sxs-lookup"><span data-stu-id="45cf7-105">For example, the plan type Basic life would have the plan type code Life because it’s a kind of life insurance plan and must conform to rules established for the Life plan type code.</span></span> <span data-ttu-id="45cf7-106">Другой тип плана может быть "дополнительное страхование жизни", также кодом типа плана "жизнь".</span><span class="sxs-lookup"><span data-stu-id="45cf7-106">Another plan type might be Supplemental life, also with plan type code Life.</span></span>

<span data-ttu-id="45cf7-107">Каждый тип плана указывает, может ли сотрудник регистрироваться в одном или нескольких планах его типа.</span><span class="sxs-lookup"><span data-stu-id="45cf7-107">Each plan type indicates whether an employee can enroll in one plan of its type or multiple.</span></span> <span data-ttu-id="45cf7-108">Например, сотрудник, скорее всего, сможет регистрироваться для полисов "базового страхования жизни" и "дополнительного страхования жизни" типа плана "жизнь".</span><span class="sxs-lookup"><span data-stu-id="45cf7-108">For example, an employee would likely be able to enroll in both the Basic life and the Supplemental life policies of plan type Life.</span></span> <span data-ttu-id="45cf7-109">Вероятно, сотруднику разрешается регистрировать только в одном полисе типа "медицина".</span><span class="sxs-lookup"><span data-stu-id="45cf7-109">An employee would likely be allowed to enroll in only one policy of type Medical.</span></span>

<span data-ttu-id="45cf7-110">Если тип плана включает контакты, тип плана указывает, являются ли контакты бенефициарами или иждивенцами.</span><span class="sxs-lookup"><span data-stu-id="45cf7-110">If a plan type involves contacts, the plan type indicates whether contacts are beneficiaries or dependents.</span></span> <span data-ttu-id="45cf7-111">Например, у типа плана "основная жизнь" могут быть бенефициары, в то время как у типа плана "основной медицинский" будут иждивенцы.</span><span class="sxs-lookup"><span data-stu-id="45cf7-111">For example, a Basic life plan type would have beneficiaries, while a Basic medical plan type would have dependents.</span></span> <span data-ttu-id="45cf7-112">В некоторых случаях в плане могут отсутствовать личные контакты.</span><span class="sxs-lookup"><span data-stu-id="45cf7-112">In some cases, a plan may not have any personal contacts.</span></span> <span data-ttu-id="45cf7-113">Например, план сбережений на случай непредвиденных расходов или премии за парковку.</span><span class="sxs-lookup"><span data-stu-id="45cf7-113">For example, a Flexible Spending Account or Parking allowance.</span></span>

<span data-ttu-id="45cf7-114">Для типа плана можно определить параметры покрытия.</span><span class="sxs-lookup"><span data-stu-id="45cf7-114">A plan type may define coverage options.</span></span> <span data-ttu-id="45cf7-115">Параметры покрытия определяются в форме параметров покрытия.</span><span class="sxs-lookup"><span data-stu-id="45cf7-115">The coverage options are defined in the Coverage option form.</span></span> <span data-ttu-id="45cf7-116">В параметре покрытия можно указать сумму льготы или контакты, которые могут быть допустимыми для типа плана.</span><span class="sxs-lookup"><span data-stu-id="45cf7-116">A coverage option can specify the amount of the benefit or the contacts who are eligible for the plan type.</span></span> <span data-ttu-id="45cf7-117">Например, если тип контакта — "бенефициар", параметр покрытия должен определять условия того, что бенефициар имеет право получить, когда используется льгота.</span><span class="sxs-lookup"><span data-stu-id="45cf7-117">For example, if the contact type is Beneficiary, the coverage option should define the terms of what the beneficiary is eligible to receive when the benefit is utilized.</span></span> <span data-ttu-id="45cf7-118">Если тип контакта — "Иждивенец", параметр покрытия должен определять связь между иждивенцем и сотрудником.</span><span class="sxs-lookup"><span data-stu-id="45cf7-118">If the contact type is Dependent, the coverage option should define the relationship between the dependent and employee.</span></span>
