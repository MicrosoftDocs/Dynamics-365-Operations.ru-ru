# <a name="configure-life-event-types"></a>Настройка типов событий жизненного цикла

Microsoft Dynamics 365 Human Resources использует типы жизненных событий для определения событий, когда допускается обновление регистрации льгот для сотрудников. Например, свадьба или рождение ребенка. Каждый код типа жизненного события может быть связан только с одним типом жизненного события. Например, если создается код жизненного события "Изменение адреса", который связан с типом жизненного события "Изменение адреса сотрудника", невозможно создать другой код с меткой "Изменение адреса сотрудника" и связать его с типом жизненного события "Изменение адреса сотрудника". 

После создания типов жизненных событий их необходимо связать с типами планов. Дополнительные сведения см. в разделе [Создание типов планов](hr-benefits-setup-plan-types.md).

   > [!NOTE]
   > После создания жизненного события его необходимо связать с типом плана. Дополнительные сведения см. в разделе [Создание типов планов](hr-benefits-setup-life-event-types.md).

## <a name="create-a-life-event-type"></a>Создание типа жизненного события

1. В рабочей области **Управление льготами** в разделе **Настройка** выберите **Типы жизненных событий**.

2. Выберите **Создать**.

3. Укажите значения для следующих полей:

   | Поле | Описание |
   | --- | --- |
   | **Идентификатор типа жизненного события** | Уникальный код типа жизненного события. |
   | **Описание** | Описание типа жизненного события. |
   | **Тип жизненного события** | Катализатор для обновления регистрации льгот сотрудника. Список жизненных событий см. в разделе [Жизненные события](hr-benefits-setup-payment-frequencies.md?life-events). |

4. Нажмите **Сохранить**.

## <a name="view-attached-plans"></a>Просмотр присоединенных планов

Можно просмотреть список планов, присоединенных к выбранному типу жизненного события. Жизненные события присоединяются к типу плана, а типы планов связаны с планом. 

1. В рабочей области **Управление льготами** в разделе **Настройка** выберите **Типы жизненных событий**.

2. Выберите **Действия**.

3. Выберите **Присоединенные планы**.

## <a name="life-events"></a>Жизненные события

При создании типа жизненного события можно выбрать следующие жизненные события:

| Жизненное событие | Местоположение | Триггер |
| --- | --- | --- |
| **Изменение семейного положения** | Работник > Профиль > Личные данные > Семейное положение| Изменение семейного положения |
| **Изменение статуса сотрудника** | <ul><li>Работник > Занятость</li><li>Страница истории занятости</li></ul> | Изменение статуса занятости |
| **Изменение адреса сотрудника** | <ul><li>Работник > Профиль > Адреса </li><li>Работник > Личная информация > Личные контакты > Адрес</li></ul> Добавление, изменение или удаление адреса |
| **Изменение для иждивенца** | <ul><li>Работник > Профиль > Личные сведения > Личные контакты > Добавление или удаление иждивенца</li><li>Самообслуживание сотрудника</li></ul> | Добавленный или удаленный иждивенец. Связь личных контактов должна быть ребенок, супруг, гражданский супруг или бывший супруг. Обновление **Действительно с** запускает жизненное событие. Если эта дата не была обновлена, жизненное событие не будет запущено. |
| **Рождение или усыновление, удочерение (иждивенец)** | <ul><li>Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца</li><li>Самообслуживание сотрудника</li></ul> | Заполнено поле **Дата усыновления, удочерения**. Дата рождения ребенка является обязательной. |
| **Потеря покрытия (супруг / супруга / сожитель)** | Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца > Потеря покрытия | **Потеря покрытия**, выбранная для личного контакта, и **Действует с** |
| Изменение в трудоустройстве для сожителя | Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца > Принятый на работу. | <ul><li>Созданная запись сведений об иждивенцах и поле **Личный контакт трудоустроен** = Да</li><li>Поле **Личный контакт трудоустроен** изменено (да или нет)</li></ul> |
| **Отпуск (супруг / супруга / сожитель)** | Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца > Отпуск | <ul><li>Запись об иждивенцах создана и заполнена **EhrLOAEffectiveDate**</li><li>**personPrivateDetails.EhrIsLOA** изменено (да или нет)</li><li>**personPrivateDetails.EhrLOAEffectiveDate** изменено</li></ul> |
| **Изменение в покрытии (должность)** | <ul><li>Работник > Назначение на должность > Назначения работника на должность</li><li>Должности > Должности</li></ul> | <ul><li>Изменение на должность в записях назначения работника на должность</li><li>Изменение назначения работника на должность</li></ul> |
| **Программа Medicare (сотрудник / иждивенец)** | Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца > Medicare действует с | Не запускается автоматически, когда личный контакт вводит дату вступления в силу. |
| **Обеспечение по решению суда** | Работник > Профиль > Личные сведения > Личные контакты > Иждивенец > Обеспечение по решению суда (QMSCO/QDRO и даты вступления в силу) | Не запускает автоматические обновления. Оно не влияет на допустимость; записывает жизненное событие. |
| **Покойный** | Работник > Профиль > Личные данные > Дата смерти | Вводится дата смерти |
| **Свидетельство страхования** | <ul><li>Работник > > Работник > Версии > История занятости > Диспетчер дат > Сведения о льготах</li><li> Работник > Занятость > Сведения о льготах > Дата проверки</li></ul> | <ul><li>Работник вводит дату проверки</li><li>Работник задает для поля EvidenceOfInsurability значение **Да**</li></ul> |
| **Бенефициар** | Работник > Профиль > Личная информация > Личные контакты | Добавляется личный контакт и поля **Бенефициар** и **Действует с** заполняются. Личный контакт должен иметь тип **Child**, **Spouse**, **DomesticPartner**, **Sibling**, **FamilyContact**, **OtherContact**, **Parent**, **BeneficiaryEstate**, **BeneficiaryOrg** или **BeneficiaryTrust**. |
| **Программа Medicare сотрудника** | Работник > > Работник > Версии > История занятости > Диспетчер дат > Сведения о льготах | <ul><li>**EhrMedicareEligibilityDate** изменяется</li><li>Для **MedicareEligibile** задано значение **Да**</li></ul> |
| **День рождения** | Работник > Профиль > Личные сведения > Личные контакты > Сведения иждивенца > Дата рождения | Дата рождения добавляется или обновляется (не после обработки изменения жизненного события). Пример: Если **Параметры допустимости личного контакта** для ребенка задано как возраст: 26 в Настройка > Льготы > Параметры допустимости личного контакта, и если сотрудник отдела кадров выполняет обработку изменений жизненных событий в любой день после того, как иждивенцу исполнится 26 лет, будет получено сообщение об ошибке, появится соответствующее уведомление о том, что иждивенец больше не соответствует покрытию. |
| **Изменение допустимости работника (не касается только США)** | <ul><li>Работник > Занятость</li><li>Работник > Работник > Версии > Журнал занятости</li></ul> | <ul><li>Изменяется тип сотрудника, категория занятости или пять полей допустимости, определяемых пользователем</li><li>**HcmEmploymentDetail.EhrEmploymentType** изменяется (обрабатывается только для *измененных* записей сведений о сотруднике, не обрабатывается для *новых* записей о занятости, например, для повторного найма и увольнения)</li></ul> |
| **Новое переопределение допустимости (не только для США)** | Расширенные кадровые ресурсы > Льготы > Планы > Льготы > Переопределение правил допустимости | Использование обработки жизненных событий | EhrBenefitEligibilityRuleOverride.ValidFrom |
| **Изменение переопределения правила допустимости (не только для США)** | Расширенные кадровые ресурсы > Льготы > Планы > Льготы > Переопределение правил допустимости | Использование обработки жизненного события (только учитывает изменения в полях **ValidFrom** и **ValidTo** в переопределении правила допустимости) |
| **Окончание срока действия переопределения правила допустимости (не только для США)** | Расширенные кадровые ресурсы > Льготы > Планы > Льготы > Переопределение правил допустимости | Использование обработки изменения жизненных событий. Например, если изменить дату окончания срока действия переопределения правила допустимости для плана на сегодня на 17:00, на любое время после 17:00 или на следующие дни, а затем запустить обработку изменения жизненного события, появится сообщение о том, что срок действия переопределения правила допустимости истек. |
| **Новый план льгот (не только для США)** | Расширенные кадровые ресурсы > Льготы > Планы > Создать | <ul><li>К текущему плану будут добавлены параметры допустимости</li><li>Добавляется новый план с прикрепленными параметрами допустимости</li></ul></br></br>Сотрудники отдела кадров должны выполнить обработку допустимости жизненных событий в данном экземпляре. |
| **Изменение правила допустимости (не только для США)** | Расширенные кадровые ресурсы > Льготы > Правила/параметры > Правила допустимости | Использование обработки допустимости жизненных событий. Записывается в журнал, когда у записей **EhrBenefitEligibilityRule** изменятся следующие значения: **UseEmplCategory**, **UseEmplStatus** или **UseEmplType**. Обновляет только проводки жизненных событий, которые уже существуют для измененного правила или критериев допустимости. |
