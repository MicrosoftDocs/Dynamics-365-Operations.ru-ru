# <a name="registration-ids"></a>Коды регистрации

В этом разделе приводятся сведения о настройке и использовании регистрации идентификаторов.

Многие страны и регионы имеют разные правила и требования для записи регистрационные номера или идентификаторы. В этом разделе представлен обзор необходимые параметры и обработка типов поддерживаемых регистрации для субъектов в разных странах/регионах. Все страны/регионы имеют свои требования для поддержки различных локальная функции, относящиеся к регистрационных номеров, предоставляемые офисов другое состояние. Регистрационный номер примеры, европейская НДС код ЕС НДС, идентификационный номер налогоплательщика (TIN) и страховой номер (SSN). Эта возможность предоставляет единой среды для всех стран во всех областях, учитывая требования к учетной записи Локальная некоторых европейских стран. В следующих разделах описаны сведения, которые используются для настройки и обработки регистрации идентификаторов общий поток.

## <a name="registration-type-creation"></a>Создание типа регистрации
Прежде чем можно будет ввести код регистрации, необходимо настроить типы регистрации для разных видов регистрационные номера, которые каждый субъект подлежит. Последовательно выберите пункты **управления организацией**&gt;**глобальной адресной книги**&gt;**типов регистрации**&gt;**типов регистрации** страницы для создания и управления типов регистрации для клиентов, поставщиков, работников и юридические лица в разных странах/регионах.

|Поле                 |описание      |
|------------------------------|----------------------------|                                                                           
| Наименование                | Имя типа регистрации. |                                                                           
| описание         | Описание типа регистрации. |
| Страна/регион      | Уникальный код страны/региона.|
| Налоговый орган       | Налоговый орган, который связан с типом регистрации.|
| Ограничен       | Тип ограничения, используемый для типа налоговой регистрации: нет, лица, организации.|
| Форматировать              | Формат проверки типа регистрации.|
| Возможно обновление      | Определяет, если регистрационный номер для типа налоговой регистрации, которые могут быть обновлены после ввода.|
| Уникальный              | Определяет, если регистрационный номер для типа налоговой регистрации является уникальным. |
| Основной для страны | Если сторона связана с одной или нескольких адресов в частности страны и код регистрации является действительным для всех адресов, необходимо назначить один адрес как основной для страны. Только один код можно зарегистрироваться в качестве основного. Определяет, если только для основного адреса страны можно ввести регистрационный номер. |

## <a name="assign-a-registration-type-to-a-registration-category"></a>Назначить тип регистрации категории регистрации
Категории регистрации — идентификатор страны/региона регистрации, утвержденные для использования в частности страны/региона для налога, таможенной декларации и других целей. Эта категория определяет правила проверки регистрации конкретного кода (в том числе проверка цифр и т.д.) и код регистрации для включения в различных отчетах. Страница **управления организацией**&gt;**глобальной адресной книги**&gt;**типов регистрации**&gt;**категории регистрации** назначения тип регистрации для конкретной страны регистрации поддерживаемые категории.

| Поле            | описание|
|-----------------------|----------------|
| Тип регистрации     | В частности, регистрация введите страны/региона.|
| Ограничен         | Тип ограничения, используемой для типа налоговой регистрации: нет, лица, организации.|
| Категория регистрации | Идентификатор уникального регистрации утверждены для использования в стране. Полный список поддерживаемых в AX7.1 является категорий ниже. |

## <a name="enter-registration-ids-for-global-address-book-records"></a>Введите регистрацию идентификаторы записей глобальной адресной книги
Глобальная адресная книга в Microsoft Dynamics 365 для операций содержит консолидированной адреса для клиентов, поставщиков, контактов, деловых отношений и юридические лица. Для получения дополнительных сведений см. в [Обзор глобальной адресной книги](/dynamics365/operations/organization-administration/overview-global-address-book). Записи субъекта, которые хранятся в глобальной адресной книге может содержать одну или несколько записей адресов. Эти адреса используются в разных целях, например для выставления счетов или доставки. Можно настроить регистрацию идентификаторы для сведений об адресе для клиентов, поставщиков, работников и юридические лица. Найти запись субъекта (юридического лица, поставщика, клиента, работник), для которого требуется ввести идентификатор регистрации, а затем щелкните **идентификаторы регистрации** в формах, связанных с стороны, юридического лица, поставщика, клиента, работника, чтобы открыть **управления адресами** страницы. На **налоговой регистрации**, щелкните **добавить**и введите следующую информацию о идентификатор регистрации

|Поле                |описание                                                |
|---------------------|-----------------------------------------------------------|
| Тип регистрации   | Тип регистрации выбранной страной/регионом.     |
| Регистрационный номер | Идентификатор стороны регистрации                                |
| описание         | Описание регистрационный номер.               |
| Секция             | Дополнительные сведения о регистрационный номер. |
| Выпускающее агентство      | Полномочия, выпустившей этот номер регистрации.        |
| Дата выдачи         | Дата выдачи для регистрационный номер.              |
| Действует           | Дата вступления в силу для регистрационный номер.           |
| Истечение срока          | Дата окончания срока действия для регистрационный номер.          |

> [!NOTE]
> Налог, освобождения юридического лица, поставщика, клиента, которые могут быть выбраны из регистрации идентификаторов относящихся к КОДУ НДС и введенных субъекта.

## <a name="search-for-records-by-registration-id"></a>Поиск записей по КОДУ регистрации
Поиск записи субъекта на основе кода регистрации доступен в формах, связанных с стороны юридического лица, поставщика, клиента и работника. Щелкните **поиск кода регистрации** чтобы открыть **критерий поиска код регистрации** страницы. Укажите критерии поиска и нажмите кнопку **найти**. Система будет отображать выбранной записи из глобальной адресной книги, а также связанные типы запись субъекта.
