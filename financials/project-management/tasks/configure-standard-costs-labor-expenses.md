# <a name="configure-standard-costs-for-labor-and-expenses"></a>Настройка стандартной стоимости труда и расходов

[!include[task guide banner](../../includes/task-guide-banner.md)]

В этой процедуре показано, как настроить стандартные затраты на труд и расходы по проекту. В этой задаче используется набор данных компании USSI.

1. Перейдите в раздел "Управление и учет по проектам" > "Настройка" > "Цены" > "Себестоимость (час)".
2. Щелкните "Создать".
3. В поле "Действует с" введите дату.
4. В поле "Себестоимость" введите число.
    * Можно настроить стандартную себестоимость категории проекта или настроить себестоимость по коду работника, номеру проекта, категории, дате или любой их комбинации. Применяемая себестоимость будет себестоимостью при самом высоком уровне детализации.  
5. Нажмите кнопку "Сохранить".
6. Закройте страницу.
7. Перейдите в раздел "Управление и учет по проектам" > "Настройка" > "Цены" > "Цена продажи (время)".
8. Щелкните "Создать".
9. В поле "Действует с" введите дату.
10. В поле "Допустимо для" выберите один из вариантов.
11. В поле "Ценообразование" введите число.
    * Можно настроить стандартную цену продажи для почасовых проводок или категории проектов. Можно также настроить цены продажи по коду работника, номеру проекта, категории, дате проводки или любой их комбинации. Фактическая цена продажи, которая применяется, когда работник вводит проводку в журнале регистрации времени, является ценой продажи с самым высоким уровнем детализации. Например, если существует общая цена продажи и цена продажи для работника, то будет использоваться цена продажи для работника.  
12. Нажмите кнопку "Сохранить".
13. Закройте страницу.
14. Перейдите в раздел "Управление и учет по проектам" > "Настройка" > "Цены" > "Себестоимость (расход)".
15. Щелкните "Создать".
16. В поле "Действует с" введите дату.
17. В поле "Себестоимость" введите число.
    * Можно заполнить несколько полей, но это минимум, необходимый для сохранения записи.  
18. Нажмите кнопку "Сохранить".
19. Закройте страницу.
20. Перейдите в раздел "Управление и учет по проектам" > "Настройка" > "Цены" > "Цена продажи (расход)".
21. Щелкните "Создать".
22. В поле "Действует с" введите дату.
23. В поле "Допустимо для" выберите один из вариантов.
24. В поле "Ценообразование" введите число.
    * Фактической ценой продажи, которая используется, когда работник заносит проводки в журнал затрат, является цена продажи с самым высоким уровнем детализации. Например, если существует общая цена продажи и цена продажи для работника, то будет использоваться цена продажи для работника.  
25. Нажмите кнопку "Сохранить".
