# <a name="project-resourcing"></a>Выделение ресурсов проекту

В этом разделе сведения о выделению ресурсов проекта.

Одна из проблем для руководителей проектов и руководители ресурсов во время этапа планирования проекта является выделение ресурсов, где они определяют и зарезервировать необходимого ресурса для работы по проекту. В Microsoft Dynamics 365 для операций распределения ресурсов возможности для проектов позволяют определить роли, которые рассматриваются как временные ресурсы, которые можно зарезервировать для определенных обязательств или часть договорам. Этот тип выделения ресурсов позволяет менеджерам проекта и менеджерам по ресурсам выполнять указанные ниже задачи.

-   Определить роль, имеющую необходимые компетенции, чтобы упростить сопоставление ресурсов.
-   Роли можно используйте для определения расписания первоначальной обязательств на основе зарезервированных ресурсов.
-   Оценить затраты и определить исходный бюджет в зависимости от назначенных ролей и ресурсов для проекта.
-   Роли можно используйте для оценки количества резервирования ресурсов, которые необходимы для каждого обязательств.
-   Оценить количества ресурсов, необходимых для всего жизненного цикла проекта.
-   Составить черновик структурной декомпозиции работ (СДР), используя исходные назначения ресурсов.

[![Жизненный цикл проекта](./media/projectresourcing02-1024x812.jpg)](./media/projectresourcing02.jpg) 

Как выручка планирования проекта спланированного ресурсы могут быть заменены с и обсуживаются ресурсы. Руководитель проекта можно вернуться назад и обновление resourcing резервирования во время любой из этапов проекта.

## <a name="set-up-project-resources"></a>Настройка ресурсов проектов
Необходимо настроить календарь и связать его с сотрудником или работником. Календарь используется для планирования проекта и рабочего времени ресурсов, зарезервированных для проекта. При настройке календаря менеджеры проектов могут выполнять выравнивание ресурсов как часть оптимизации ресурсов. На основе расписания календаря на ресурсы можно накладывать ограничения. Можно настроить календарь на **календари** страницы. 

При настройке работника в качестве ресурса проекта, можно выбрать один из сотрудников, работающих в компании, для которой выполняется настройка ресурсов или работников можно выбрать один из других компаний в вашей организации. Это внутрихолдинговых ресурсов. Следующие процедуры показывают, как настроить работника в качестве ресурса проекта в рамках вашей компании и настройке ресурса внутрихолдинговых проекта.

### <a name="set-up-a-worker-as-a-project-resource"></a>Настройка работника в качестве ресурса проекта

1.  На **работников** странице в **работников** выберите работника, который добавляется в качестве ресурса проекта, а откройте запись работника.
2.  В области действий щелкните **проекта**&gt;**установки**&gt;**настройки проекта**.
3.  Выберите календарь, а затем закройте страницу.

Можно также указать проекты по умолчанию для ресурса в качестве типа предварительного назначения. Предварительные назначения можно использовать, когда менеджер ресурсов или менеджер проекта заранее знают, над какими проектами будет работать ресурс. Предварительные назначения могут также основываться на запросе спонсора проекта или клиента. Чтобы задать предварительные назначения для проекта, на странице **Назначение проектов** на вкладке **Проекты** в списке **Остальные проекты** выберите соответствующий проект.

### <a name="set-up-an-intercompany-resource"></a>Настройка внутрихолдинговых ресурсов

При настройке работника в качестве внутрихолдинговых ресурса, необходимо выполнить настройку в аренде привлекающее дополнительных работников с компанией. 

**В аренде компании:**

1.  В Dynamics 365 для операций убедитесь, что выбран сдаче в аренду компании и выполните описанную выше процедуру, «Настройка работника в качестве ресурса проекта».
2.  Последовательно выберите пункты ** главной книги **&gt; ** настройку разноски **&gt;**для внутрихолдингового учета**. Click **New**.
3.  В ** код юридического лица ** выберите сдаче в аренду компании. Заполните оставшиеся поля и нажмите кнопку **Сохранить**.
4.  Переход ** управление и учет по проектам **&gt; ** настройки **&gt;**цены ** &gt;**перемещение цены**.** **
5.  На ** цена перемещения ** щелкните **Нью**и в ** получающее юридическое лицо ** выберите соответствующую компанию.
6.  Если требуется только временно привлекающее дополнительных работников компании ресурса, созданной в начале этого раздела в **ресурсов** укажите имя ресурса, который был создан. Чтобы освободить все ресурсы в компании на компанию, привлекающее дополнительных работников, оставьте ** ресурсом ** пустым.
7.  Последовательно выберите пункты ** Управление и учет по проектам **&gt; ** настройки **&gt;**управления и параметры учета проекта**и на ** внутрихолдинговых ** вкладке ** Включение внутрихолдингового планирования ресурсов и табели учета рабочего времени ** на **Да**.

**В компании, привлекающее дополнительных работников:**

1.  Последовательно выберите пункты **управление и учет по проектам**&gt;**ресурсы по проекту**&gt;**список ресурсов**.
2.  В фильтрах поиска введите имя ресурса, созданного в предыдущей процедуре для выдачи компании, чтобы убедиться, что имя включается в список ресурсов для компании, привлекающее дополнительных работников.

## <a name="manage-resource-competencies"></a>Управление компетенциями ресурса
Компетенции ресурса являются важной частью управление ресурсами. Компетенции можно использовать как базис для определения ресурсов, которые имеют правильный баланс навыков, образования, наличия сертификатов и проектного опыта. Необходимо настроить эту информацию для каждого из ресурсов и регулярно обновлять ее. Таким образом можно максимально увеличить возможности, когда компетенции определенных трудовых ресурсов совпадают во время назначения ресурсов проекта. [![Примеры навыков, сертификация, образование и опыт работы по проекту](./media/projectresourcing06-1024x383.jpg)](./media/projectresourcing06.jpg) 

Следующие процедуры показывают, как настроить некоторые компетенции для ресурса. 

Чтобы настроить компетенции для работника, можно использовать или страницу списка **Работники** в модуле "Управление персоналом" или страницу списка **Ресурсы** в модуле "Управление и учет по проектам". Следующие процедуры **работников** используется страница списка в модуле управления персоналом.

### <a name="set-up-competencies-certificates"></a>Настройка компетенций: сертификаты

1.  На странице списка **Работники** выберите строку работника, для которой добавляются сведения о сертификатах.
2.  В области действий на вкладке **Работник** в группе **Компетенции** щелкните **Сертификаты**.
3.  Нажмите **Создать**.
4.  В поле **Тип сертификата** выберите **PMP**.
5.  В поле **Дата начала** выберите **01.10.2015**.
6.  Щелкните **Сохранить** и закройте страницу.

### <a name="set-up-competencies-skills"></a>Настройка компетенций: навыки

1.  На странице списка **Работники** убедитесь, что все еще выбран работник, который использовался в предыдущей процедуре. Затем в области действий на вкладке **Работник** в группе **Компетенции** щелкните **Навыки**.
2.  Нажмите **Создать**.
3.  В поле **Навык** выберите **Управление проектами**.
4.  В поле **Уровень** выберите **5 Эксперт**.
5.  В поле **Дата уровня** выберите **14.1-.2014**.
6.  В поле **Опыт в годах** введите **10**.
7.  Щелкните **Сохранить** и закройте страницу.

## <a name="create-a-new-project"></a>Создать новый проект
1.  Щелкните **управление и учет по проектам**&gt;**рабочих областей**&gt;**управление проектами**.
2.  Щелкните **Создать проект** и введите следующие значения:
    -   **Тип проекта** - время и расходы
    -   **Имя проекта** -2 этапа обновления XYZ
    -   **Группа проектов** -TM\_НЗП
    -   **Код контракта по проекту** -00000002
3.  Щелкните **Создать проект**.

### <a name="assign-a-resource-to-a-project"></a>Назначение ресурса проекту

1.  Щелкните **персоналом**&gt;**работников**&gt;**работников**.
2.  В списке **Работники** выберите запись для работника, для которого ранее настраивали компетенции, и откройте эту запись работника.
3.  В области действий на вкладке **Проект** в группе **Настройка** выберите **Назначение проектов**.
4.  На странице **Назначения проектов проверок ресурсов** выберите вкладку **Проекты**.
5.  В **добавить проект в выбранные проекты**, фильтр по проекту XYZ обновить этап 2
6.  В области **Остальные проекты** области выберите проект, затем нажмите на стрелку, чтобы добавить его в область **Выбранные проекты**.
7.  Закройте страницу.

Если нужно, можно также назначить категории для ресурса. Тип категории либо "Затраты", либо "Выручка". Это определяется вашей организацией. Если не назначенных категории для ресурса, Dynamics 365 для операций будет искать категории по умолчанию по листам почасовых затрат и выручки совпадали.

### <a name="set-up-project-resource-and-role-characteristics"></a>Настройка ресурс проекта и характеристик роли

Менеджер проекта может использовать функцию выделения ресурсов проекта для создания ролей, необходимых для проекта. Роли можно использовать при подтвержденной ресурсы по-прежнему неизвестны при резервировании ресурсы. Роли можно временно зарезервированы как запланированных ресурсы, что этапы планирования проекта можно продолжить. 

[![Пример роли](./media/projectresourcing05.jpg)](./media/projectresourcing05.jpg) 

**Сценарий:** Contoso нанята для выполнения проекта типа "Время и расходы", у которого имеется утвержденный устав проекта. Младший менеджер проекта все еще определяет объем работ по проекту. Диспетчер ресурсов — в настоящее время определить конкретные ресурсы, которые будут зарезервированы для работы над новым проектом. Одна из ролей, запрошенные спонсором проекта, из-за критически важную природу проекта, является старший руководитель. Диспетчер ресурсов необходимо приобрести новый ресурс и определите роль в системе, в том случае, если руководитель проекта младшие требует сведения о ресурсах на этапе планирования проекта. 

Ниже показано, как настроить роль диспетчера главный проект и связать с ним характеристик ресурсов диспетчера ресурсов. Позднее эту роль можно использовать для поиска доступных ресурсов, которые соответствуют обязательным компетенциям ресурса.

1.  Щелкните **управление и учет по проектам**&gt;**установки**&gt;**ресурсов**&gt;**настройки ролей**.
2.  Щелкните **Создать** и введите следующие значения:
    -   **Роль с Идентификатором** -старший менеджер проекта
    -   **Описание** -старший менеджер проекта
3.  Щелкните **Создать**.
4.  Выберите роль **Старший менеджер проекта**, затем нажмите **Настроить характеристики**.
5.  В поле **Тип характеристик** выберите **Навык**.
6.  В поле **Доступные характеристики** введите навык, который вы ищете.
7.  В поле **Тип характеристики** выберите **Сертификат**.
8.  В поле **Доступные характеристики** введите тип сертификата, который вы ищете.
9.  Щелкните **ОК** и закройте страницу.

### <a name="assign-a-project-resource-to-a-project"></a>Назначение ресурса проекта проекту

1.  Щелкните **управление и учет по проектам**&gt;**обычные**&gt;**проекты**&gt;**все проекты**и откройте **2 этапа обновления XYZ** проекта.
2.  На вкладке **Проектная группа и планирование** нажмите **Добавить**.
3.  В поле **Роль** выберите **Член группы**.
4.  Щелкните **Зарезервировать из календаря**.
5.  На странице **Доступность ресурсов** щелкните **Просмотр параметров**.
6.  На странице **Настройка параметров представления** введите следующие значения:
    -   **Формата для представления даты диапазона** - день
    -   **Отображение описания доступности** - Да
    -   **Отображение оставшегося резерва** - Да
7.  В списке ресурсов выберите ресурс.
8.  Щелкните **окончательно**&gt;**полной мощности**.
9.  Закройте страницу.

### <a name="assign-a-resource-to-a-default-role"></a>Назначение ресурса роли по умолчанию

Чтобы помочь менеджерам проекта или ресурса, можно выполнить дальнейшую детализацию еще больше ресурсов, может быть зарезервировано для проекта. Можно связать роль по умолчанию с существующим ресурсом или вновь приобретенным ресурсом. Например когда Дэниэла был принят на работу, у него есть опыт и навыки для заполнения аналитик бизнес-роли. Диспетчер ресурсов назначены этой роли как Дэниэла роль по умолчанию. Таким образом диспетчер ресурсов добавлены Дэниэла пул бизнес-аналитики, которые доступны для работы над проектами. 

Во время резервирования ресурсов руководители проектов можно отфильтровать роль ресурсов, доступных для работы в рамках проектов. Они могут использовать эту информацию как один из критериев, когда они выполняют анализ решений с несколькими критериями во время заполнения ресурса. Они могут также добавить другие характеристики ресурса в фильтр для поиска ресурсов с конкретными навыками, образованием и опытом для данного проекта. 

**Сценарий:** проекте утвержденных запущен, а роль диспетчера старших проекта было зарезервировано как запланированных ресурс во время этап планирования проекта. Менеджер по ресурсам теперь получил ресурс для заполнения роли старшего менеджера проекта.

1.  Щелкните **управление и учет по проектам**&gt;**ресурсы по проекту**&gt;**список ресурсов**.
2.  В списке **Ресурс** выберите **Даниэль Голдшмидт**.
3.  Щелкните **ресурсов по проекту**&gt;**Сохранить**&gt;**роль ресурса**.
4.  Щелкните **Создать** и введите следующие значения:
    -   **Действия** - (текущая дата)
    -   **Дата истечения срока действия** - никогда не
    -   **Роль** -старший менеджер проекта
5.  Щелкните **Сохранить** и закройте страницу.
6.  На вкладке **Компетенции** добавьте навык **ProjectMgmt** и сертификат **PMP**.

## <a name="set-up-role-based-pricing"></a>Настройка ценообразования на основе роли
Все затраты, продажи и трансферные цены можно настроить для ролей.

1.  Щелкните **управление и учет по проектам**&gt;**установки**&gt;**цены**&gt;**цены продажи (часы)**.
2.  Click **New**.
3.  Ввести дату вступления в силу.
4.  В столбце **Роль** выберите роль.
5.  В столбце **Ценообразование** введите цену для выбранной роли ресурса.

## <a name="form-a-project-team"></a>Форма группы проекта
Для использования ролей, которые ранее были настроены в проекте руководителем проекта необходимо связать с проектом ролей. Несколько ролей, которые могут быть назначены для проекта и Dynamics 365 для операций автоматически означает, что подпись этих ролей во время резервирования, чтобы избежать путаницы. Например если менеджер проекта требует трех инженеров по программному обеспечению трех ролей разработчик программного обеспечения, имеющие разработчик программного обеспечения 1, разработчик программного обеспечения 2 и разработчик программного обеспечения 3 как их подписи автоматически создаются. Если характеристики роли уже были настроены для роли, они применяются как фильтр во время поиска ресурса. При необходимости могут быть добавлены дополнительные характеристики для дальнейшего уточнения поиска. 

Параметры просмотра можно также настраивать, чтобы обеспечить лучшее представление доступности ресурсов. Предусмотрены параметры для отображения доступности по часам, по дням, по неделям, по месяцам, по кварталам и по годам. Также предусмотрен параметр для отображения доступности и оставшихся возможностей ресурсов. Этот параметр полезен для контроля времени при оценке доступного времени для действий или доступности ресурса. 

Руководитель проекта можно выбрать роль на странице и в случае освобождения ресурса, соответствующему данной потребности, выберите для резервирования ресурсов для выполнения роли. Обратите внимание, что ресурсы не были зарезервированы в этой точке на этапе планирования. При создании СДР ролей можно заменить службы ресурсы для проекта. Если роли заменяются службы ресурсов СДР, Настройка ресурсов автоматически обновляет группе проекта со списком и планирования. 

[![Группы проекта со списком, который включает в себя роли и фактические ресурсы](./media/projectresourcing03-1024x368.jpg)](./media/projectresourcing03.jpg) 

У менеджера проекта есть несколько вариантов резервирования ресурса для проекта, такие как **Остаток мощности**, **Полная мощность**, **Процент мощности** и **Укажите часы**. Эти параметры резервирования можно отменить в любое время, если назначения ресурсов изменяются. Поддерживаются два типа резервирования:

-   **Окончательно** – резервирование ресурсов был утвержден и подтверждено для работы по проектной на указанное время.
-   **Предварительно зарезервировать** – резервирование ресурсов под вопросом было задано для работы по проектной на указанное время.

В следующей процедуре показано, как создать проектную группу.

### <a name="create-a-project-team"></a>Создание проектной группы

1.  На странице списка **Все проекты** выберите проект, затем нажмите **Правка**.
2.  На **группы и планирования проекта** вкладке **Дата окончания расписания** введите Дата начала расписания плюс один месяц. Например, если дата начала расписания — 24 июня 2017 г. (24/06/2017 г.), введите **24/07/2017 г.**.
3.  Click **Add**.
4.  В области **Добавление ролей к проекту** в поле **Роль** выберите **Старший менеджер проекта**.
5.  Щелкните **Необходимые компетенции**.
6.  На странице **Выбрать характеристики** по умолчанию выбраны характеристики, которые ранее были заданы для роли старшего менеджера проекта. Нажмите кнопку **OК**.
7.  На странице **Добавление ролей к проекту** в поле **Число ресурсов** введите **1**.
8.  В поле **Ресурс** поиск показывает все ресурсы, которые имеют необходимые компетенции. Выберите **Даниэль Голдшмидт**, затем нажмите **Создать**.
9.  На странице **Проект** нажмите **Добавить**.
10. В области **Добавление ролей к проекту** в поле **Роль** выберите **Член группы**. В поле **Число ресурсов** введите **5**.
11. Щелкните **Создать**.
12. На странице **Проекты** нажмите **Обеспечить ресурс**.

## <a name="resource-capacity-synchronization"></a>Синхронизация мощностей ресурсов
Процессы для синхронизации ресурсов помогают гарантировать, что информация из календаря и базового календаря передавалась в планирование ресурсов проекта. Если календарь изменен, процессы выполняют необходимые обновления в планировании ресурсов проекта. Процессы также помогают улучшить производительность, поскольку сведения о ресурсе в календаре заранее синхронизируются, чтобы обновления информации планирования ресурсов происходили быстрее. Рекомендуется планировать процессы пакетами, а не по одному. В противном случае имеется риск, что кто-то забудет включающие даты, когда информация синхронизировалась в последний раз. Если включающие даты не используются, несоответствия могут иметь место во время синхронизации даты.

### <a name="calendar-synchronizationmediaprojectresourcing04-1024x471jpg"></a>![Синхронизация календаря](./media/projectresourcing04-1024x471.jpg)

**Synchronize resource capacity roll-ups**

Процесс синхронизации предназначен для синхронизации всех данных календаря ресурса. Эти сведения включают данные базового календаря обо всех изменениях в таблице мощности календаря ресурсов проекта. Если в проект добавляются новые ресурсы, синхронизации помогает гарантировать доступность данных обновленных календаря. Эта синхронизация может быть выполнена в любое время. 

Рекомендуется использовать партию. Параметры доступны в синхронизации резервирований мощности.

-   Щелкните **управление и учет по проектам**&gt;**Периодические операции**&gt;**мощности синхронизации**&gt;**синхронизация сверток мощности ресурсов**.

| Параметр | описание                                                                                                                                                                                          |
|--------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Да    | Синхронизация всех данных ресурса с данными календаря и базового календаря и замена всех данных в календаре мощности ресурсов проекта.                                                  |
| Нет     | Синхронизация данных ресурса на основании кода интервала дат и указанных дат начала и окончания. Этот параметр не удаляет существующие данные и обновляет сведения только для вновь добавленных ресурсов. |

[![Процесс синхронизации](./media/projectresourcing09.jpg)](./media/projectresourcing09.jpg)

## <a name="resource-fulfillment-for-planned-resources"></a>Обеспечение ресурсов для запланированных ресурсов
Менеджер проекта может запланировать необходимые роли ресурсов для проекта. Менеджер по ресурсам видит эти запланированные ресурсы в виде запросов на странице **Обеспечение ресурса** и может назначить фактические ресурсы.

1.  Щелкните **управление и учет по проектам**&gt;**проекты**&gt;**все проекты**.
2.  В списке выберите проект **Этап 2 обновления XYZ**.
3.  Щелкните **Проект**.
4.  Выберите **Изменить**.
5.  На **группы и планирования проекта** вкладке ** ** щелкните **добавить**.
6.  В диалоговом окне **Добавить роли** выберите роль **Разработчик программного обеспечения**.
7.  Щелкните **Создать**.
8.  Закройте страницу проекта.
9.  Щелкните **управление и учет по проектам**&gt;**ресурсы по проекту**&gt;**ресурсов на исполнение**.
10. Выберите **Разработчик программного обеспечения 1** для проекта **Этап 2 проекта обновления XYZ**.
11. Выберите работника, затем нажмите **Назначить**.
12. Убедитесь, что строка для **Разработчик программного обеспечения 1** удалена из проекта **Этап 2 проекта обновления XYZ**.

## <a name="requests-for-project-resources"></a>Запросы ресурсов проекта
Функциональные возможности планирования ресурсов проекта поддерживает только руководители ресурсов, чтобы распределить ресурсы службы по обязательствам или проектов. Чтобы включить эту функцию, необходимо выполнить следующие задачи, или убедитесь, что они были внесены.

-   Настройка номерных серий.
-   Настройте управление проектами и учет бизнес-правил.
-   Включите workflow-процесс запроса ресурса.

После проверены или выше задачи выполнены, можно выполнить следующие задачи, при необходимости.

-   Создайте запрос на ресурс из ресурса службы предварительные.
-   Мониторинг запросов ресурсов.
-   Выполнения запросов ресурсов.
-   Запросите ресурс службы СДР.
-   Книга ресурсов в проект без запроса для ресурса службы.
