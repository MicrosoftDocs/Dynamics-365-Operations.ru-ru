# <a name="statistical-dimension-members-and-statistical-measure-provider-templates"></a>Элементы статистической аналитики и шаблоны поставщика статистической меры

[!include[banner](../includes/banner.md)]

Статистическая аналитика и ее элементы используются для регистрации и контроля немонетарных записей в модуле "Учет затрат". Элементы статистической аналитики можно использовать для двух целей:

- В качестве базы распределения в таких политиках, как распределение затрат и распределение стоимости
- Для отчетности по потреблению немонетарных ресурсов

## <a name="statistical-dimension"></a>Статистическая аналитика

Статистическая аналитика имеет уникальное имя и набор уникальных элементов аналитики. Статистическая аналитика назначается коду книги учета затрат. Это отношение связывает все соответствующей элементы статистической аналитики с книгой учета затрат. Таким образом, все статистические записи будут создаваться в контексте главной книги модуля "Учет затрат".

> [!NOTE]
> Статистическая аналитика может быть назначена нескольким книгам учета затрат.

Ниже приведен пример статистической аналитики.

| Наименование                        | Соединитель данных для элементов аналитик |
|-----------------------------|--------------------------------------|
| Общие статистические элементы | Импортированные элементы аналитики           |

Ниже приведен пример статистической аналитики, назначенной книге учета затрат.

| Наименование                  | Валюта учета | Тип валютного курса | Фискальный календарь | Аналитика элемента затрат | Статистическая аналитика       |
|-----------------------|---------------------|--------------------|-----------------|------------------------|-----------------------------|
| Управленческий учет | американский доллар                 | Постоянная валюта  | Учетный период   | Общие элементы затрат   | Общие статистические элементы |

## <a name="statistical-dimension-members"></a>Элементы статистической аналитики

Элемент статистической аналитики представляет объект, для которого вы хотите регистрировать немонетарные меры. Эти меры можно использовать либо как базу распределения, либо просто для включения в отчетность немонетарных значений.

Элементы статистической аналитики можно создавать вручную. Кроме того, их можно импортировать из файла с помощью средства импорта и экспорта данных в модуле "Управление данными".

Элемент статистической аналитики автоматически становится предопределенной базой распределения. Ее можно использовать в качестве базы распределения в политиках или в качестве входных данных для других типов баз распределения.

Вот несколько примеров типичных элементов статистической аналитики.

| Имя статистической аналитики  | Статистические элементы | описание             | Ед. изм. |
|-----------------------------|----------------------|-------------------------|------|
| Общие статистические элементы | FTE                  | Сотрудники с полной занятостью     | шт.  |
| Общие статистические элементы | Электричество          | Потребление энергии | кВт-ч  |
| Общие статистические элементы | Pack CC              | Центр затрат "Упаковка"   | ч |

## <a name="statistical-measure-provider-template"></a>Шаблон поставщика статистической меры

Статистические меры могут поступать из различных источников. Microsoft Dynamics 365 for Finance and Operations, Enterprise edition — прекрасный источник для извлечения статистических мер. Для конфигурирования статистических мер, которые вы хотите извлекать, удобно использовать шаблон поставщика статистической меры.

Определение шаблона поставщика статистической меры является универсальным, и его можно использовать в нескольких элементах статической аналитики.

> [!NOTE]
> Все таблицы, которые содержат финансовые аналитики, можно использовать в качестве источников для статистических мер.

**Пример. Число сотрудников по центрам затрат**

Число сотрудников по центрам затрат — статистическая мера, которую можно использовать для различных целей управленческого анализа:

- В качестве меры статистической отчетности по центру затрат
- В качестве базы распределения для различных типов расходов
- Внутренние ставки затрат по центру затрат:

    - Затраты по сотруднику
    - Выручка по сотруднику

Таблица HcmEmployment содержит список всех сотрудников в экземпляре. Эта таблица является глобальной таблицей. Следовательно, эти записи не связаны с конкретным кодом области данных.

Ниже приведен пример сотрудников в таблице HcmEmployment.

| Наименование       | Место возникновения затрат | описание   | Тип работника |
|------------|-------------|----|-------------|
| Сотрудник 1 | CC001       | Отдел кадров | Код сотрудника    |
| Сотрудник 2 | CC002       | FI | Код сотрудника    |
| Сотрудник 3 | CC002       | FI | Код сотрудника    |
| Сотрудник 4 | CC003       | СП | Код сотрудника    |
| Сотрудник 5 | CC003       | СП | Код сотрудника    |
| Сотрудник 6 | CC002       | FI | Подрядчик  |

При создании записи **Шаблон поставщика статистической меры** необходимо решить, какую функцию использовать:

- **Счетчик** — передается количество записей по объекту стоимости.
- **Сумма** — передается сумма записей по объекту стоимости. (Поля **Сумма** и **Дата** являются обязательными.)

## <a name="using-the-sum-function"></a>Использование функции "Сумма"

Центр производственных затрат CC010 (упаковка) отвечает за упаковку продуктов, прежде чем они отгружаются клиентам. Прямые трудовые затраты добавляются к продукции через спецификацию и маршрут. Косвенные затраты на обеспечение функционирования центра также должны распределяться по произведенным продуктам. Зачастую наилучшей статистической мерой для подобного распределения является количество зарегистрированных производственных часов, приходящееся на продукт в течение заданного периода.

## <a name="imported-statistical-measures"></a>Импортированные статистические меры

Статистические меры можно импортировать в модуль "Учет затрат" с помощью средства импорта и экспорта данных в модуле "Управление данными".

Информационный объект, используемый для импорта, называется "Импортированные статистические меры".

> [!NOTE]
> Этот информационный объект допускает не более пяти уникальных значений аналитики на запись.

Потребление энергии регистрируется в Microsoft Excel с использованием предопределенного формата информационного объекта. Рассмотрим пример:

| Финансовая дата | Имя элемента аналитики 1 | Имя элемента аналитики 2 | Имя элемента аналитики 5 | Величина  | Идентификатор источника |
|-----------------|------------------------|------------------------|------------------------|------------|-------------------|
| 31.01.2017      | CC001                  |                        |                        | 2,450.00   | Электричество       |
| 31.01.2017      | CC002                  |                        |                        | 4,100.00   | Электричество       |
| 31.01.2017      | CC003                  |                        |                        | 15,000.00  | Электричество       |

При импорте данных через модуль "Управление данными", данные будут сохранены в промежуточной таблице модуля "Учет затрат". Так импортированные данные могут использоваться в нескольких книгах учета затрат. Перезагрузка данных не требуется.

Чтобы импортировать данные, выберите **Импортированные данные** \> **Информационный объект** \> **Импортированные статистические меры**.

| Идентификатор источника | Финансовая дата | Величина  | Имя элемента аналитики 1 | Имя элемента аналитики 2 | Имя элемента аналитики 5 |
|-------------------|-----------------|------------|------------------------|------------------------|------------------------|
| Электричество       | 31.01.2017      | 2,450.00   | CC001                  |                        |                        |

Прежде чем статистические меры можно будет получить в модуле "Учет затрат", необходимо установить связь между идентификатором источника и элементом статистической аналитики. Эта связь создается для книги учета затрат и версии. Связь состоит из соединителя данных и поставщика данных. У вас может быть несколько соединителей данных и поставщиков данных на каждый элемент статистической аналитики.

> [!NOTE]
> В этом примере мы создадим связь только для версии **Фактическая версия**.

Выберите **Книга учета затрат** \> **Фактическая версия** \> **Управление** \> **Статистические меры**, чтобы установить связь. Для этого сценария выберите соединитель данных **Импортированные статистические меры**, потому что данные были импортированы из сторонней системы в модуль "Учет затрат" через Excel.
