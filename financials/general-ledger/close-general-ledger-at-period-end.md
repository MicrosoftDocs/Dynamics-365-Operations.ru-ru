# <a name="close-the-general-ledger-at-period-end"></a>Закрытие ГК в конце периода

В этом разделе описаны задачи, которые обычно выполняются при выполнении закрытия периода для главной книги. 

В главной книге можно выполнять процедуры закрытия для периода или года. Процессы закрытия подготавливают систему к новому периоду. Для подготовки системы для нового года, необходимо выполнить процесс закрытия года окончания. Каждая организация имеет разные процессы и действия, выполняемые на конец периода. Ниже приведены некоторые дополнительные шаги для окончания периода.

-   Завершите все задачи для всех других модулей, например Расчеты с поставщиками, Расчеты с клиентами и Запасы.
-   Проверьте, что все журналы разнесены.
-   Выполните переоценку в иностранной валюте для создания любых сумм внереализационной прибыли или убытков.
-   Сопоставьте проводки для каждого счета ГК.
-   Обработайте все требуемые распределения.
-   Вручную разнесите корректировки на конец периода.
-   Учтите в субкниге проводки и просмотрите отчет **Журнал ГК**.
-   Выполните консолидацию, используя консолидированную компанию или финансовую отчетность.
-   Создайте финансовые отчеты на конец периода с помощью финансовой отчетности.
-   Задайте для периодов ГК статус **Заблокировано**, чтобы было невозможно выполнить разноску. Также можно ограничить период определенной группой пользователей, пока выполняются действия на конец периода, для более эффективного управления. Не рекомендуется настроить периоды, которые должны **без возможности восстановления закрытых**, так как нельзя повторно открыть период, который был закрыт.

Финансовый периода закрытия рабочей области используются для организации и отслеживания задач, необходимых для различных процессов на конец периода. См. в разделе [финансового периода закрытия рабочей области](financial-period-close-workspace.md) и [закрытие в конце года](Year-end-close.md) Дополнительные сведения.
