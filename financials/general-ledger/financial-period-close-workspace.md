# <a name="financial-period-close-workspace"></a>Рабочая область закрытия финансового периода

Эта статья предоставляет обзор Рабочей области "Закрытие финансового периода" и соответствующего конфигурации.

Рабочая область закрытия финансового периода

**Закрытие финансового периода** рабочей области позволяет отслеживать свои процессы финансового закрытия всех компаний, областей и людей. В зависимости от вида **Закрытие финансового периода** рабочей области, вы увидите, как все задачи и статусы для закрытия расписание или задач, которые назначены вам. 

Необходимо сначала выбрать расписание закрытия в верхней части рабочей области. Все данные, которые отображаются в рабочей области выберите фильтруется по выбранной заключительной графика.

### <a name="summary-tiles"></a>Плитки сводки

Плитки **Сводки** дают обзор процесса, и индикаторы помогают поддерживать процесс закрытия. Можно просмотреть задачи, которые являются преодолеть срок выполнения, оставшихся задач для текущей задачи, которые заканчиваются сегодня, но блокированы из-за зависимостей и всех остальных задач для процесса. Эти сведения предназначены для всех компаний, включенных в расписание закрытие.

### <a name="tasks-and-status-section"></a>Раздел задач и статуса

В **задач и статус** разделе статус общего расписания закрытия разделяется различными способами: статус компанией, статус по областям, а статус, ответственный. Можно просматривать статус для всех задач в Закрытие планирование только задачи, срок которых наступает в настоящее время или задачи, которые просроченных путем изменения фильтра в верхней части списка карт. Можно также выбрать фильтр компании для просмотра статуса для конкретной компании. На каждой вкладке Статус дает разбивку процента завершения и число задач, которые остались. Выберите карточку, или **Просмотр сведений о** действия для фильтрации списка подробных задач по выбранной карты. 

Для списка подробных задач является последней. Этот список отображается список задач полное и могут быть отфильтрованы, чтобы отобразить только задач, которые заинтересованы в его. Можно отфильтровать список задач несколькими способами. Например можно отфильтровать по задаче, дате оплаты, связанная организация и связанные области. Можно также выбрать отображение или скрытие завершенных задач в списке задач. 

Два индикатора использованы для задач:

-   Значок с восклицательным знаком указывает задачу просроченных. Для задач, просроченные срок также выделяется красным цветом.
-   Значок замка указывает, что задача зависит от других задач, которые еще не выполнены. Заблокированного зависимостей задачи не может быть помечена как завершенная. Можно настроить зависимостей для задачи с помощью **зависимости** действия.

Название задачи будет гиперссылки на Microsoft Dynamics 365 для страницы "операции" или другие веб-страницы, которой должно перейти пользователь для завершения работы. Эту гиперссылку можно настроить с помощью **связи задач** поле при создании задачи или изменить. 

Можно прикрепить файлы, примечания, изображения и URL-адресов задачи с помощью **вложения** действия. Например можно указать номеров журналов, которые используются как часть задачи, добавление комментариев о выполнении конкретной задачи или присоединить файл отчета, который был распечатан для выполнения задачи. Значок появляется в **вложение** столбца для задачи, если имеется вложение. 

**Завершить задачу** параметр необходимо выбрать вручную после завершения задачи. Когда задача помечена как завершенная, **Дата завершения** автоматически обновляется на текущую дату и время. Индикаторы зависимости также обновляются соответствующим образом.

## <a name="all-financial-period-close-tasks-list-page"></a>Страница списка всех задач закрытия финансового периода
Можно просмотреть все текущего и предыдущего периода закрытия задачи из **все финансового периода закрытия задачи** страницы списка. На этой странице списка рекомендуется использовать для исторического анализа процесса закрытия, так как оно содержит сведения о запланированной дате оплаты, дата фактического выполнения и лица, заполнившего задачи. Сведения на этой странице списка можно легко экспортировать в Microsoft Excel для составления отчетности и целей аудита.

## <a name="financial-period-close-configuration-page"></a>Страница Конфигурация закрытия финансового периода
Прежде чем можно будет использовать **Закрытие финансового периода** рабочей области, необходимо настроить процесс в Microsoft Dynamics 365 для операций с помощью **финансового периода закрытия конфигурации** страницы. (Щелкните **главной книги**&gt;**Закрытие периода**&gt;**финансового периода закрытия конфигурации**.)

### <a name="resources"></a>Ресурсы

На **ресурсов** вкладке определить лиц, которые участвуют в процессе закрытия. Любого сотрудника, который будет нести ответственность за закрытия задачи необходимо сначала назначить здесь. Необходимо также указать сотрудника вид рабочей области. Имеются следующие варианты:

-   **Только заданные задачи** — пользователь увидит только задачи, которые назначены ему или ей.
-   **Все задачи и статус** — Пользователь увидит все задачи закрытия и состояние общего процесса.

Потребители, которые имеют разрешения осмотреть только их заданные задачи, не смогут добавить задачи к списку задач, редактировать задачи, или удалить задачи из списка задач.

### <a name="task-areas"></a>Области задач

Вы используете зоны задач для того, чтобы группировать закрывающие задачи в логически зоны владения в пределах вашей организации. Например, расчеты с поставщиками, Расчеты с клиентами, или главную книгу могли быть использованы как зоны задачи.

### <a name="calendars"></a>Календари

Создание и редактирование закрытия финансовых календарей с помощью вкладки "календари".  Это, где будет определять рабочие дни для закрытия процессов, а также будет использоваться для планирования задач закрытия.  Создание нового календаря, а также указать рабочие дни, которые будут использоваться для планирования задач.  Лучше всего создать календарь для длительного периода времени, например за год или несколько лет, так как оно может быть изменено после создания.  После создания календаря, нажмите на кнопку Правка, чтобы обновить календарь для конкретных дней, такие как праздничные дни.  Закрытие задачи будут планироваться в те дни, когда значение элемента управления установлено на "открыто".  Если закрытие задачи не должно быть расписание на определенный день, тот же день должна иметь значение закрыто значение элемента управления.

### <a name="templates"></a>Шаблоны

Шаблон финансового закрытия используются для определения всех задач, которые являются частью процесса закрытия. Задача закрытия является повторяющееся трудозатраты, назначенного одному пользователю для выполнения в рамках каждого процесса закрытия. В шаблоне относительного срока выполнения для каждой задачи закрытия должна быть определена дата. Срок исполнения относительные даты — количество дней до или после определенного периода конечной даты, задачи должны быть выполнены за каждый период. Время завершения также назначены для каждой задачи. Требуемое время завершения устанавливается с использованием контекста часовой пояс и будут преобразованы в часовом поясе для каждого пользователя. 

Можно назначить задачу в шаблоне для одной или нескольких компаниях, где применяется этой задачи. Если другое лицо назначен для завершения этого рабочих операций в каждой компании, может оказаться полезным для создания нескольких задач для затрат рабочего времени. Создайте одну задачу для каждой компании. 

**Связи задач** пункт меню связан с рабочих операций задач и может использоваться для прямого перехода к соответствующей странице из связи задач в рабочей области. Например, закрытия задание Переоценка валюты для модуля Расчеты с поставщиками могут быть связаны с соответствующими **переоценки в иностранной валюте** страницы в Microsoft Dynamics 365 для операций. Вы можете также сделать ссылку на внешний URL-адрес. 

> [! Подсказка] Если требуется связать определенный отчет Management Reporter финансового периода закрытия задачи, можно использовать URL-адрес отчета. Для доступа к URL-адрес отчета, откройте отчет в конструкторе отчетов и нажмите кнопку **файл**&gt;**просмотреть отчет о**, чтобы открыть отчет в веб-обозревателе. Вы можете после этого скопировать URL-адрес в строке адреса браузера и вставить его в поле **URL** **ссылки на задачу**. 

Можно определить зависимости задач в шаблоне. Если задача было настроено зависят от одной или нескольких задач, эта задача не может быть помечен как завершенные, пока не будут завершены все зависимости. 

Можно создать несколько шаблонов финансового закрытия. Затем можно использовать различные шаблоны для отслеживания процедуры закрытия для различных типов периодов, например на конец месяца или в конце года, или для отслеживания компаний, использующих различные закрытия процессов. После создания одного шаблона можно скопировать в новый шаблон и внесите необходимые изменения. Можно назначить только один шаблон для каждого графика закрытия.

### <a name="closing-schedules"></a>Планы закрытия

График закрытия используется для назначения шаблона финансового закрытия конкретного финансового периода, должны быть закрыты. Задач из шаблона автоматически создаются для указанного периода, а новое расписание закрытия добавляется в рабочую область. При создании нового графика закрытия, **Дата окончания периода** поле используется для определения срока выполнения фактической даты задач закрытия, относительный срок оплаты на основании даты, назначенные в шаблоне финансового закрытия. 

Назначьте календарь, подходящий для закрытия расписание, чтобы указать рабочие дни для использования при планировании задачи. Если не определить конкретный календарь, задачи из-за даты будет использовать все дни недели. 

Необходимо также определить компании, которые будут связаны с закрытия графика. Если шаблон задачи назначены к нескольким компаниям, отдельные задачи создается для каждой компании, в расписании закрытия и назначена задача шаблона. 

После завершения закрытия расписание, выберите **закрыто** для него вариант. Журнал задач по-прежнему будут доступны из **все финансового периода закрытия задачи** страницы списка, но закрытия графика будут удалены из рабочей области. После закрытия расписание был помечен как **закрыто**, нельзя будет добавлять к, изменение задач или удалять задачи из него.
