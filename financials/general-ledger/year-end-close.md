# <a name="year-end-close"></a>Закрытие на конец года

В этом разделе описываются настройки и действия для запуска процесса закрытия главной книги на конец года. 

В конце финансового года необходимо выполнить процесс закрытия на конец года для переноса начальных сальдо для нового года. В большинстве организаций будет установлен несколько раз процесс закрытия конца года. В первый раз можно получить в новом финансовом году перенести сальдо. Закрытие года можно запустить заново, затем как часто требуются для перемещения сальдо из корректируемых операций в новый финансовый год. 

Процесс закрытия года, существует два типа возможных созданных проводок. Открывающая операция создается всегда и используется для создания начальных сальдо в новый финансовый год. Открывающая операция показывает сальдо счета ГК балансового отчета в новый финансовый год и сальдо из сальдо счета ГК прибылей и убытков бухгалтерского счета нераспределенной чистой прибыли в новом финансовом году. Закрывающие проводки при необходимости создается для объединения сальдо счетов прибылей и убытков до нуля закрываемого финансового года.

## <a name="prepare-to-run-the-year-end-close"></a>Подготовка к закрытию на конец года
Перед выполнением процесса закрытия на конец года, проверьте следующие параметры: 

На странице **Счет ГК**:

-   Проверка **основной тип счета** должным образом определяется для каждого счета ГК. Основной тип счета используется для определения того, является ли сальдо счета ГК встраивания Переслать как начальное сальдо и закрытых в нераспределенной чистой прибыли в открывающие проводки.
-   **Открывающий счет** поле может использоваться для переноса сальдо счета ГК в новый счет ГК во время закрытия года. Новый счет ГК вводится в **открывающий счет** поле. Обычно это будет использоваться для балансовых счетов ГК при деактивируется счета ГК, а новый счет ГК для нового финансового года.

На **параметры главной книги** странице под **закрытия финансового года**:

-   **Удаления закрыть год проводок** параметр используется для указания, следует ли удалять открывающие проводки созданное системой из закрытия предыдущего года при повторном выполнении закрытия года. Если этот параметр имеет значение **Да**, предыдущей открывающие проводки удаляется и создается новая проводка открывающие созданного на основе текущего сальдо. Если этот параметр имеет значение **нет**, остается предыдущей открывающие проводки и дополнительную проводку открывающие создается для перемещения сальдо по сравнению с корректировкой проводки, разнесенные после закрытия предыдущего года.
-   **Создание закрывающих проводок при переносе** параметр используется для создания закрывающих проводок в финансовый год закрыт для переноса сальдо счетов прибылей и убытков, равным нулю. Если этот параметр имеет значение **Да**, создается проводка открывающие и закрывающие проводки. Если этот параметр имеет значение **нет**, открывающая операция создается в следующего финансового года для переноса сальдо. Сальдо счетов прибылей и убытков, остаются в конце финансового года.
-   **Установить статус финансового года без возможности восстановления закрытых** параметр используется для присвоения статуса без возможности восстановления закрытого финансового года. Этот параметр следует используйте с осторожностью, так как все периоды со статусом без возможности восстановления закрытых повторно открыть нельзя, предотвращающих корректировок разноску в финансовый год. Рекомендуется установить **нет**.
-   **Необходимо указать код операции** параметр используется для определения, необходим ли код операции при запуске процесса закрытия на конец года. Это рекомендуемый способ требуют код операции для легко идентифицировать открывающие проводки.

На **финансового календаря** страницы:

-   Следующего финансового года должен существовать до выполнения закрытия года. Нужно создать начальные сальдо в открытом периоде следующего финансового года.

На **календаря книги** страницы:

-   Необязательно: Каждого финансового периода для закрытия финансового года можно задать **заблокированной** для предотвращения новых проводок для введения. При записи корректировки идентифицируются, периода сохранения Далее может быть повторно открыт для учета корректировки операций, даже если уже запущен процесс закрытия конца года.

## <a name="define-year-end-close-templates"></a>Определение шаблонов на конец года закрытия
После настройки системы можно запустить процесс закрытия конца года. На **закрытие в конце год** странице шаблона можно определить для группы компаний, для которых процесс закрытия года будет выполняться. Шаблон повторно в конце каждого года закрытия, но могут быть изменены при изменении вашей организации. 

Во-первых, определите **имя группы** для шаблона, выберите финансовый календарь. Имя группы следует указать группу по юридическим лицам, включенным.  Например шаблоны могут настраиваться на основе по географическому положению, с отдельных групп, которые созданы для североамериканских компаниях, юридические лица EMEA и юридические лица СТРАНЫ. 

После этого юридические лица могут быть добавлены в шаблон. Юридические лица могут быть добавлены, выбрав организационной иерархии или выбрав юр. Если установлен организационной иерархии, только допустимых объектов в иерархии, использующие выбранного финансового календаря будет добавлен в шаблон. При использовании юридические лица для добавления в шаблон можно добавлять только юридические лица с один и тот же финансовый календарь. Один и тот же финансовый календарь необходим из-за закрытия года выполняется путем выбора финансового года, могут различаться в зависимости от календаря календаря. 

После добавления юридические лица, определяются счета ГК, нераспределенной чистой прибыли для каждого юридического лица. **Закрыть Дата завершения последнего года** поле обновляется каждый раз, выполнении закрытия конца года для юридического лица. 

**Финансовой аналитики** вкладка используется для определения того, какие финансовые аналитики, которые будут использоваться в открывающие проводки. Обратите внимание, что параметры, вы определяете важны для юридического лица, выбранного в **компаниями** сетки. Настройка будет повторяться для каждого юридического лица в сетке. 

**Переноса аналитики балансового отчета** используется для определения, следует ли хранить на открывающие проводки финансовых аналитик для проводок, разнесенных на балансовые счета. Рекомендуется установить этот параметр на **Да**. **Переноса аналитики прибылей и убытков** используется для определения того, какие финансовые аналитики для проводок, разнесенных по прибыли и убытков перемещается на счета ГК нераспределенной чистой прибыли. Во-первых следует определите финансовые аналитики, относящихся к выбранному юридическому лицу. Это будет включать все финансовые аналитики, разноска в течение года, даже если финансовая аналитика не является частью активной структуре счета. Затем определите каждое измерение как **закрыть один** или **закрыть все**.  Значение по умолчанию — **закрыть все**, содержит исходный финансовой аналитики значения из разнесенных проводок и использует их для создания входящее сальдо для счета нераспределенной прибыли. Для каждой уникальной комбинации значений финансовой аналитики будут создаваться отдельный нераспределенной чистой прибыли начальных сальдо. Если **закрыть одним** — флажок установлен, все проводки, разнесенные с этой финансовой аналитики будут суммироваться в нераспределенной прибыли, начальное сальдо для значения аналитики, введенного в поле после **закрыть одним**. Например предположим, что все для финансового года были разнесены со структурой счета счет ГК - подразделение. На финансовые аналитики подразделения в шаблоне **закрыть один** установлен и введенное значение 100. Если общий доход по всем проводкам, разнесенным в подразделения, 200, 300 до 400 $100,000, для расчета зарплаты сохраненное - 100 создается одно входящее сальдо. При выборе **закрыть один** и оставьте пустым значение связанной финансовой аналитики, нераспределенной чистой прибыли разноски всех транзакций с соответствующими значениями измерения не определен. 

Процесс закрытия конца года не соответствует структур счетов. Это потому, что структуры счетов может меняться в течение финансового года, и не всегда можно определить структуру соответствующего счета из-за этих изменений.  При создании открывающих проводок, сальдо встраивания вперед с финансовыми аналитиками, определенное в шаблоне закрыть год окончания. Начало записи сальдо может включать финансовые аналитики больше не в комбинациях структуры и сегмент текущего счета, которые больше не являются допустимыми в структуре текущего счета. Если ваша организация хочет исключить финансовой аналитики для нераспределенной доход начальное сальдо, набора финансовых аналитик должен быть **закрыть одним** и оставить пустым значение аналитики.

## <a name="run-the-year-end-close-process"></a>Запустите процесс закрытия года
Создав шаблоны закрытия на конец года, инициируется процесс закрытия конца года, выбрав **выполнение финансового года** в области действий. Выбор всех или подмножества юридические лица из шаблона, для которого требуется выполнить закрытия года. После запуска на конец года закроется в первый раз в финансовом году, скорее всего будет выбрана юридические лица для создания начальные сальдо для всех компаний. Если вы работаете на конец года закрытие еще раз, можно выполнить только для юридических лиц, для которых были разнесены записи корректировки. 

Выберите финансовый год, который вы хотите запустить конец процесс закрытия года от. Если существует более одного закрытия периода для последнего периода финансового года **название периода** поля будут доступны, если указано установки Чтобы создать закрывающие проводки, можно выбрать для разноски проводки закрытия, какой период закрытия. 

Введите операцию номер, который или операция не является обязательной, в зависимости от настройки в главной книге параметры. Для всех юридических лиц, выбранных для процесс закрытия года окончания будет использоваться тот же код операции. Код операции не генерируется с использованием номерной серии. Рекомендуется ввести номер операции, даже если это не является обязательным. Ввод номера операции облегчает поиск открывающих проводок в новый финансовый год. Если не ввести код операции, операции будет пустым для открывающих проводок. 

Если вы хотите реверсировать закрытия предыдущего года окончания для выбранного финансового года, установите **отменить закрытие предыдущих** для **Да**. Закрытие года будут отменены, но процесс можно перезапустить в любое время. Если необходимо реверсировать закрытия на конец года **закрыть Дата завершения последнего года** будут не доступны. 

В конце года процесса закрытия по умолчанию запускать в пакетном режиме. Рекомендуется запустить процесс в пакетном режиме, дает пользователю будут возвращаться другие действия. По завершении процесса закрытия конец года, **закрыть Дата завершения последнего года** будет обновлено с даты сеанса.

Дополнительные сведения см. в разделе [Закрытие ГК в конце периода](close-general-ledger-at-period-end.md).
