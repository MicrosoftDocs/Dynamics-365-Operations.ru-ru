# <a name="intercompany-accounting-setup"></a>Настройки внутрихолдингового учета

В этом разделе объясняется, как настроить внутрихолдингового учета, чтобы межфирменные журналы можно использовать для распределения ГК и финансовых журналов, например ежедневных журналов, журналы накладных поставщика и журналы платежей.

Межфирменные журналы могут создаваться в различных сценариях, как, например для ежедневных журналов, журналы накладных поставщика, распределениях ГК и централизованных платежей. Чтобы эти сценарии работали, необходимо настроить внутрихолдинговый учет.

## <a name="define-main-accounts"></a>Определите счета ГК
Во-первых, необходимо создать внутрихолдинговые счета ГК для использования в записях учета "Задолженность компании" и "Задолженность от". Рекомендуется использовать уникальные счета ГК для каждой компании, чтобы упростить выверку и закрытие записей внутрихолдингового учета. При использовании торговым партнером или эквивалентным аналитики для идентификации субъекта внутрихолдинговых этого измерения можно определить как фиксированная аналитика на счет ГК, определенный во внутрихолдинговом учете. При настройке счета ГК следует установить **основной тип счета** на **балансового отчета** на **счетах ГК** страницы.

## <a name="define-journal-names"></a>Определять имена журналов
Затем следует определить имя журнала. Настройка **тип журнала** на **ежедневные** на **имена журналов** страницы. Рекомендуется использовать определенное имя журнала для внутрихолдингового учета.

## <a name="define-intercompany-accounting-setup"></a>Определите настройки внутрихолдингового учета
**Для внутрихолдингового учета** страница используется для создания пар юридическим лицам, которые могут transact друг с другом. Внутрихолдинговом учете настройки является общим, таким образом, Настройка доступной для просмотра в всех компаний. При создании новой пары юридического лица, убедитесь, что находитесь известно, какое юридическое лицо определяется как исходную компанию и компании-получателя. При вводе внутрихолдинговые проводки, проводки определяет, какое юридическое лицо инициализацией или рассчитанные проводки. Например внутрихолдинговом учете настраивается для USMF (рассчитанные) и USSI (назначение). Если пользователь является активным в USSI и вводит внутрихолдинговой проводки с USMF, проводки не будет разноситься из-за внутрихолдинговых расчетов определяется только для USMF выполняется инициатор. Если транзакция может быть создано любой компании, необходимо создать вторую пару юридического лица для установки соответствующих друг другу. 

Выберите **дебетовый счет (долг)** и **Кредитовый счет (выполнения)** для юридического лица источника и назначения. Определить, какие **имя журнала** будет использоваться при создании проводки в компании-получателя. Поскольку выбранный пользователем при создании внутрихолдинговой проводки журнала для исходной компании уже известен. 

Наконец выберите, какое юридическое лицо будет получать учета для поддержки сумм, таких как скидка при оплате наличными или реализованных прибылей/убытков для централизованных платежей. 

Второй стороны отношения можно легко настроить на **для внутрихолдингового учета** страницу с использованием **создать второй стороны отношения** кнопку после создания первой паре юридического лица. При создании второй стороны паре сведения для компании-получателя копируются в исходную компанию и наоборот. Журнал, определенные для компании-получателя, останется. В большинстве организаций используется одному соглашению об именах для их имена журналов, чтобы имя журнала совпадает с. Если указано другое имя журнала, предупреждение появится в поле, чтобы то, что можно выбрать другой журнал и журнала не существует.
