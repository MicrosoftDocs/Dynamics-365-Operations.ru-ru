# <a name="sepa-credit-transfer-overview"></a>Обзор кредитных переводов SEPA

Приводятся общие сведения о перемещениях ISO 20022 кредит, включая переносы кредита одной области платежей евро (SEPA) и других электронных платежей для поставщиков. Перенос кредита SEPA является особым типом платежа в евро из одной компании или отдельных для другой компании, а также отдельных. Раздел также объясняется, как настроить и отправить файл кредит перемещение платежа.

## <a name="what-is-a-credit-transfer-message"></a>Что такое перемещение сообщение кредит
Кредит перемещения сообщения является запрос, который отправляет инициирующей стороны (вашей компании), чтобы переместить фондов из своей собственной учетной записью кредитору. Существуют многие реализации и страной/регионом для конкретных банковских кредит передачи сообщений. Некоторые из них используются в пределах одной страны/региона, и некоторые становятся все стандарты. Один хорошо известной общий стандарт является ISO 20022 и его запуска сообщения, такие как перенос кредита. На следующем рисунке показана связей и покрытия для выбранной кредит передачи сообщений. 
![Передача кредит-](./media/credit-transfer.jpg) кредит перемещение сообщений\[/заголовок\] 

## <a name="what-are-iso-20022-and-sepa-payments"></a>Что такое ISO 20022 и платежи SEPA
Единая зона платежей в евро (SEPA) настраивается европейской комиссией и определяет, что все электронные платежи считается внутренними, независимо от страны или региона, где находятся лицо, бизнес или организация. Нет никакой разницы между национальных платежей и зарубежных платежей. SEPA включает 28 государств-членов Европейского Союза (ЕС) и также Исландии, Лихтенштейн, Норвегию, Швейцарии, Монако и Сан-Марино. SEPA помогает формировать единый рынок для платежных проводок в пределах Европейского экономического союза (EEA). Наконец, как ожидается, SEPA уменьшит число форматов платежей, с которыми должны работать банки организации и лица. Европейская комиссия установила правовую основу для платежей SEPA с помощью Директивы о платежных услугах (PSD). Европейский совет по платежным системам (EPC) поддерживает SEPA следующим образом:

-   Он устанавливает стандарты для электронных платежей SEPA с помощью формата XML ISO 20022 "Универсальная схема сообщений финансовой индустрии".
-   Он устанавливает правила и нормы обработки платежей в евро.

EPC, состоящий из европейских банков, разрабатывает коммерческие и технические платформы для платежных инструментов SEPA. Используется три типа платежей SEPA:

-   Кредитные переводы
-   Прямое дебетование
-   Карты

## <a name="what-is-a-sepa-credit-transfer"></a>Что такое кредитный перевод SEPA?
Кредитный перевод SEPA — это платеж одной компании или физического лица другой компании или физическому лицу. Платежи должны осуществляться в евро и должны включить номер IBAN и код БИК обеих сторон. (BIC называется также организация по всему миру Межбанковский Financial Telecommunication \[SWIFT\] кода.) Кроме того операционные издержки должны быть распределены между сторонами. Кредитные переводы, которые происходят между сторонами, должны использовать XML-файлы, соответствующие стандартам обработки платежей ISO 20022 и формату XML, как указано EPC.

## <a name="how-is-a-credit-transfer-implemented"></a>Способ реализации переносить по кредиту
Кредит перемещение формата платежа для стран Европы реализуется с помощью электронной отчетности (ER) и методы оплаты функций в Dynamics 365 для операций. Несколько форматов кредит перемещения, которые используются в других регионах по-прежнему использовать структуру устаревшего платежа. Среди множества других форматов существует 12 ISO 20022 кредит перемещение форматы файлов, доступные. Эти форматы экспорта соответствует стандарту SEPA ISO 20022 XML. Они используются для создания платежей не в единицах евро для стран/регионов, где они используются и выплат евро, перечисленным в версии 8.2 SEPA кредит перенос схемы Rulebook, освобождает EPC. Прежде чем реализовать переносы кредита, необходимо обратиться в свой банк для получения программного обеспечения, который необходим для отправки файлов электронных банковских операций. Это программное обеспечение будет использоваться для передачи XML-файлы, содержащие платежных поручений в ваш банк.

## <a name="what-credit-transfer-formats-are-currently-supported-in-dynamics-365-for-operations"></a>Какие форматы передачи кредит в настоящее время поддерживаются в Dynamics 365 для операций?
Перейдите в библиотеку общих средств на основе Microsoft Dynamics Lifecycle services (LCS) и должны просмотреть последнюю версию списка доступных файлов, которые имеют тип средств **конфигурации Немецкий**. Следующий раздел, «Что нужно настроить?», ссылка на раздел, объясняется, как создать LCS репозиторий для просмотра допустимых конфигураций и импортировать выбранных конфигурациях.

## <a name="what-do-i-have-to-set-up"></a>Что следует настроить?
-   Перед созданием кредит передачи файлов, по крайней мере один активный конфигурированию переноса должна быть импортирована в вашей конфигурации аварийного восстановления. Инструкции см. в [загрузить электронной отчетности конфигурации из Lifecycle Services](https://docs.microsoft.com/en-us/dynamics365/operations/dev-itpro/analytics/download-electronic-reporting-configuration-lcs).
-   При настройке расчетов с поставщиками способы оплаты, выберите **универсального электронной отчетности** флажок, а затем выберите формат перемещение соответствующие кредит (например, **переносить по кредиту ISO 20022 (AT)**) как конфигурацию формата экспорта.
-   Также необходимо настроить юридического лица и банковский счет информацию в Dynamics 365 для операций.
-   Другие идентификаторы требуются для создания платежей перемещение допустимый кредит или номеров банковских счетов, IBANs и иногда SWIFT коды (BICs). Таким образом необходимо настроить их для банковского счета поставщика и банковского счета организации, запрашиваемой перемещения.
-   Дополнительные сведения могут потребоваться, такие как номера налога на добавленную стоимость (НДС) для субъектов, которые имеются ссылки в сообщении перемещение кредит. Эти сведения должны настраиваться для поставщиков и для юридического лица при ее запросе.
-   Некоторые Расчеты с поставщиками методов оплаты, главным образом на базе ISO 20022 методов оплаты, может потребоваться дополнительная настройка для **наборов кодов формата платежей**, таких как **тип службы** = **SLEV**. Эти коды используются как дополнительные теги для проводок по платежам, во время обработки платежей. Значения кодов платежей по умолчанию, как **назначение категории**, **расходы предъявителя**, **локальной инструмент** и **уровень обслуживания** может задаваться в двух местах. В первом месте не **заголовок журнала подлежащих оплате счетов**, второй — **Расчеты с поставщиками методы для платежей**. При создании строк журнала оплаты значения кода оплаты на заголовка журнала платежей будут перенесены в строку журнала, если не установлено, используются значения из методов оплаты.

## <a name="what-are-ibans-and-bics"></a>Что такое номера IBAN и коды БИК?
Номер международного банковского счета (IBAN) и идентификатор кода BIC (Bank) используются для идентификации любой учетной записи в многих странах/регионах по всему миру. К ним относятся 34 SEPA стран/регионов. Ввод BIC в **SWIFT-код** поле и IBAN в **IBAN** поле. По кредиторам банковский счет и банковский счет клиента, эти поля находятся на **дополнительной идентифицирующей** экспресс-на **банковского счета** вкладке **банковские счета** страницы. Использование для платежей SEPA BIC больше не является обязательным.

## <a name="how-do-i-transmit-a-payment-file-to-the-bank"></a>Как отправить файл платежа в банк?
При создании платежей, будет создан файл платежа, и выводится запрос на сохранение ее в любое место, доступное в веб-обозревателе. Далее необходимо для отправки в ваш банк XML-файл. Этот процесс зависит от банка. Следуйте инструкциям банка, чтобы отправить файлы в банк на обработку.
