# <a name="project-cost-accrual-on-purchase-receipts"></a>Начисление затрат проекта на Прих.

В этой теме описывается, как начисленные затраты по проекту можно отслеживать приходы в Microsoft Dynamics 365 для операций покупок. 

Накладные по проекту часто поступают позже, чем товары и доставляются службы, который может оказать значительное влияние на проект ключевые индикаторы производительности (KPI). Необходимо иметь возможность отслеживания этих проводок в обоих финансовых и проекта он сообщает.

Это показано в следующем примере сценарии. 

Компания Contoso обратиться начала нового проекта развертывания в облаке. Заказ на покупку создается приобрести компьютер для проекта. Компьютер будет $1500 и службы установки будет затратами $150. Поставщик распространяться и устанавливаться как компьютер, но накладная еще не достиг Contoso обратиться. Руководитель проекта хотим начисление затрат проекта $1650 перед доставкой накладной. Эта стоимость также должно быть отражено в компании месяц окончания финансовые отчеты. 

Начисленные расходы должен записываться на уровне проекта и финансовые уровня для целей отчетности. В Dynamics 365 для операций можно отслеживать финансовые обновления отборочной накладной для номенклатуры, категориях закупок. 

Для номенклатур на **параметры расчетов с поставщиками** странице **Разноска поступления продуктов в книге** параметр.
[![accruals1](./media/accruals1-1024x409.png)](./media/accruals1.png) 

Для категорий закупаемой продукции на **правило политики для категории** странице **торгового** политики, а затем выберите **начислений покупки расходы при поступлении** для каждой категории закупаемой продукции.
[![accruals2](./media/accruals2-1024x569.png)](./media/accruals2.png) 

**Без выставления накладной расходы по закупкам** и **начисления покупок** счетов в **Настройка разноски** будет использоваться при учете операций, относящихся к поступления продуктов.
[![accruals3](./media/accruals3-1024x429.png)](./media/accruals3.png) 

Используя этот же сценарий, посмотрим, как разноска поступления продуктов влияют на главную книгу и сведений о проекте. 

**Шаг 1:** создать и подтвердить новый заказ на покупку для проекта, чтобы записать покупки компьютера для $1500 и установке служб для $150.
[![accruals4](./media/accruals4-1024x497.png)](./media/accruals4.png) 

При подтверждении заказа на покупку создаются операции для подтвержденных затрат для проекта. 
[![accruals5](./media/accruals5-1024x219.png)](./media/accruals5.png) 

> [!NOTE]
> Проводки для подтвержденных затрат будут иметь **источник проводки** задано значение **заказа на покупку**. Создание и подтверждения заказа на покупку не создает проводок по проекту. 

**Шаг 2:** полученных товаров и услуг, а также поступление продуктов будет зарегистрирована. 

Разноска поступления продуктов создаст и разнесет операции в главную книгу. Операция расхода покупки, корр. счет по дебету и кредитовать счет начислений покупки. 
[![accruals6](./media/accruals6-1024x214.png)](./media/accruals6.png)

> [!NOTE]
> Разноска поступления продуктов будет использовать настройку разноски для категории закупаемой продукции и продуктов и не настройку разноски для категорий проекта. Эта настройка для правильно показать результат выполнения накопления средств, должен быть выровнены. 

Имеется возможность сопоставить категории закупаемой продукции для категории проекта на **категория закупаемой продукции** страницы.
[![accruals7](./media/accruals7-1024x390.png)](./media/accruals7.png)

**Шаг 3:** создать накладную поставщика платежного поручения. 

В Dynamics 365 для операций Разноска поступления продуктов не влияет на сведения о проекте. Чтобы избежать этого можно либо создать накладную поставщика платежного поручения непосредственно после учета приходной накладной покупки. Последовательно выберите пункты **заказа на покупку** страница &gt;**вкладке Счет**&gt;**формирование**&gt;**накладной**. При этом создается документ ожидающую обработки накладную, обновляет сведения о проекте. 

Создание накладной поставщика платежного поручения будет создавать ожидающих проводок по проекту. 
[![accruals8](./media/accruals8-1024x225.png)](./media/accruals8.png) 

В **подтвержденные затраты** страницы, будет закрыт записей, созданных на шаге 1, и новые записи будут создаваться с учетом затрат обязательства, поступающих от поставщиков, ожидающие обработки накладной. **Источник проводки** поле подтвержденных затрат будет иметь значение **накладной поставщика**.
[![accruals9](./media/accruals9-1024x200.png)](./media/accruals9.png)

Накладной поставщика останется в состоянии ожидания, пока не приходит накладная поставщика Фактическая.
