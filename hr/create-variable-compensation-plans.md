# <a name="create-variable-compensation-plans"></a>Создание планов переменных компенсаций

Переменная компенсация составляет нерегулярную заработную плату сотрудника, такую как как премии или вознаграждений акциями. Эта статья описывает компоненты, которые необходимо настроить перед использованием переменной компенсации и регистрации сотрудника на план с переменной компенсацией.

Расчет сумм переменной компенсации для сотрудников может основываться на нескольких факторах, таких как производительность сотрудника, уровень компенсации сотрудника и производительность подразделения.

## <a name="variable-compensation-components"></a>Компоненты переменной компенсации
### <a name="create-compensation-types"></a>Создание типов компенсации

**Типы переменной компенсации **— обязательный компонент. Типы переменной компенсации позволяют описываться виды переменной компенсации, которые предлагает организация. Они также позволяют указать, будет ли компенсация выдаваться наличными или в неденежной форме, например запасе.

### <a name="describe-vesting-rules"></a>Описание правил по компенсации по положению о передаче прав

Дополнительно компании могут настроить **правила по компенсации по положению о передаче прав**. Правила вестирования описания распределенных переменного вознаграждения со временем. Например правило вестирования может быть указано, что сотрудник получит 25 процентов от общего вознаграждения свой каждый год для следующих четырех лет. Правила вестирования нужны только являются информационными.

## <a name="variable-compensation-plans"></a>Планы переменной компенсации
**План переменной компенсации** содержит правила, методы расчета и значения по умолчанию для расчета переменной компенсации для зарегистрированных сотрудников. При создании плана переменной компенсации, необходимо задать тип переменной компенсации. Тип переменной компенсации, определяет ли система рассчитывает валютной суммы или количества единиц в качестве вознаграждения. Также следует задать метод расчета:

-   **Момент времени** — для расчетов переменного вознаграждения по фиксированной компенсации, где был сотрудника в конкретный день. Что указана в событии процесса при обработке сумм по новым компенсации.
-   **Составной** — сумма вознаграждения рассчитывается для каждой уникальной ставки оплаты фиксированной компенсации, которую имел сотрудник между датой начала цикла и датой окончания цикла в событии процесса. Затем добавляются курсы друг с другом, для определения окончательного размера вознаграждения. Например во время цикла, сотрудник перенесены в другое место, где был ставку оплаты. В этом случае переменное вознаграждение корректируется в соответствии с периодом времени, в течение которого сотрудник имел каждую ставку оплаты.

Сумма переменного вознаграждения может основываться на проценте от обычного базового дохода сотрудника или от заданного числа единиц.

-   Выберите параметр **Процент базы**, чтобы ввести процент по умолчанию, и укажите, следует ли в качестве базы использовать фиксированную ставку оплаты сотрудника или контрольную точку уровня компенсации сотрудника. Уровень компенсации устанавливается на задании сотрудника. Одной из опорных точек в структуре компенсации может устанавливаться как контрольную точку для плана фиксированной компенсации. Система будет использовать уровень компенсации сотрудника заданию и перекрестных ссылок его с контрольной точкой, перечисленного в плане фиксированной компенсации для сотрудника для нахождения суммы точки управления для уровня компенсации для сотрудника. Сумма точки управления будет использовать вместо фиксированной ставки зарплаты сотрудника как основу для вознаграждения.
-   Выберите параметр** Число единиц**, чтобы ввести число единиц по умолчанию, стоимость каждой единицы и валюту стоимости единицы, если план компенсации предназначен для неденежного вознаграждения (например, 200 единиц запаса по цене 40 долларов США), или только число единиц, если план компенсации предназначен для денежного вознаграждения. Для денежных вознаграждений сотрудник получит указанное число единиц валюты, используемый для плана фиксированной компенсации свой (например, 500 единицы 1 USD). Однозначное управления используется для указания, является прямой однозначное сопоставление между числом единиц измерения и значения. При создании плана переменной компенсации для плана, основанный на наличных с использованием числа единиц, этот параметр автоматически блокируется для **Да**, а значение единицы измерения **1,0000**.

**Правила найма** установка позволяет указать, должно ли получать же увеличение, независимо от даты их найма всех сотрудников (**правила найма** = **нет**), или ли сотрудники должны получить процент премии на основании продолжительности занятости во время цикла (**правила найма** = **%**). 

**Поощрение** позволяет корректировать вознаграждения для сотрудника, в зависимости от производительности сотрудника отдела. Метрики производительности могут быть настроены для каждого отдела в **отделы** странице в области **связанные формы**&gt;**компенсации**&gt;**производительности**. Вознаграждение, которое сотрудников этого отдела зависит от значения **Процент достигнутого целевого уровня** поля, указывающего отдел производительности:

-   Если производительность подразделения составляет 100 процентов, вознаграждение сотрудников в этом подразделении факторизуется по проценту, который задан в поле** Выплата при 100%**.
-   Если производительность подразделения более 100 процентов, система добавляет процент, который задан в поле **На 1% больше задания**, к проценту, который задан в поле **Выплата при 100%**, пока не будет достигнуто значение, заданное в поле **Наибольшая возможная выплата**.
-   Если производительность подразделения менее 100 процентов, система вычитает процент, который задан в поле **На 1% меньше задания**, из процента, который задан в поле **Выплата при 100%**, пока не будет достигнуто значение, заданное в поле **Наименьшая возможная выплата**.

Можно задать** уровни допуска** в процентах порога, чтобы отображалось предупреждение, если зависимость приводит к тому, что процент выходит за пределы процента порога. 

По умолчанию система выполняет поиск, устанавливаемая на должность сотрудника в подразделение. Тем не менее вознаграждения для некоторые сотрудники могут зависеть от производительности нескольких отделов. В этом случае в регистрации переменной компенсации сотрудника можно настроить разные подразделения и процент вознаграждения, которые распределяются на производительность каждого отдела. Дополнительные сведения см. в разделе «регистрации дополнительного вознаграждения», следующая за инструкцией. 

Зависимость используется, только если выбран параметр **Оплата за производительность** при выполнении процесса компенсации. 

**Уровни переопределения** вкладке можно переопределить по умолчанию процент вознаграждения или число единиц, основанное на уровне компенсации сотрудника. Если **разрешить переопределение для уровней** установлено значение **Да** для сотрудников, которые участвуют в плане переменной компенсации, система имеет уровень от задания для сотрудника, а затем ищет его на уровнях переопределяет таблицы, чтобы определить процент или число единиц для этого уровня. Если уровень не найден в уровень переопределяет таблицы, по умолчанию процент или количество единиц из **Общие** вкладка используется. Процент и количество единиц можно также переопределить в регистрации сотрудника в плане переменной компенсации.

## <a name="variable-compensation-enrollment"></a>Регистрация переменной компенсации
### <a name="determine-who-is-eligible-for-the-plan"></a>Определение лиц для регистрации в плане

Первым шагом регистрации сотрудников в плане переменной компенсации является определение того, кто имеет право получать компенсацию, которая определена в плане. Пока не будет определено право на компенсацию, будет невозможно назначить план сотрудникам. Чтобы настроить право на компенсацию, откройте страницу **Правила приемлемости**, чтобы создать новое правило приемлемости для плана, и определите критерии, которым должны отвечать сотрудники, чтобы иметь право на этот план компенсации. Право на план можно ограничить на основании подразделения, профсоюза, региона компенсации (расположения), должности, функциональных обязанностей или уровня компенсации. Сотрудники могут зарегистрироваться в плане компенсации, только если они отвечают **всем** критериям, заданным в правиле приемлемости. 

**Примечание.** Правила приемлемости определяют приемлемости как в планах фиксированной, так и переменной компенсации. Правила приемлемости учитывают значения следующих полей в записях заданий, должности и сотрудника для определения того, имеет ли сотрудник право на план компенсации.

-   На странице **Задание**:
    -   Поле **Задание**
    -   На вкладке **Классификация задний** поля **Функция** и **Тип задания**
    -   На вкладке **Компенсация** поле **Уровень**
-   На странице **Должности** поля **Подразделение** и **Регион компенсации**
-   На **сотрудников** страница: сведения о профсоюзов, связанной с сотрудником в разделе **личные сведения**&gt;**профсоюзов** на *** работника *** вкладку

### <a name="enable-enrollment-for-the-variable-compensation-plan"></a>Включение регистрации для плана переменной компенсации

На странице **Планы переменных компенсаций** задайте для параметра **Включить регистрацию** значение **Да**, чтобы разрешить соответствующим сотрудникам регистрироваться в плане.

### <a name="enroll-the-employee"></a>Регистрация сотрудника

Теперь можно зарегистрировать сотрудников в плане переменной компенсации. Чтобы зарегистрировать сотрудника, перейдите на страницу **Сотрудники** и выберите сотрудника. В области действий нажмите кнопку **компенсации**&gt;**переменный план**. 

**Примечание.** Для параметра **Регистрация** необходимо задать значение **Да** в плане переменной компенсации. **План** поле отображается только планы, которые сотрудник имеет право для на основе правил проверки, которые были настроены для этих планов. Если значение указано правила приемлемости для плана, нет сотрудников будет производиться для этого плана. 

Убедитесь, что **даты вступления в силу** поля установлено неправильно. Если в плане переменной компенсации используются **составного** метод расчета даты вступления в силу регистрации может рассматриваться как при расчете суммы вознаграждения для сотрудника. 

Можно использовать **переопределяет** вкладке для переопределения значений, определенных для сотрудника. Например если **правила найма** установлено значение **%** в плане и дату приема на работу должен использоваться при расчете процента найма сотрудника, даты приема на работу, задаваемых в **Дата приема на работу** поле. Можно также переопределить либо **процент вознаграждения** значение или **число единиц** значение для конкретного сотрудника, в зависимости от настройки плана. Эти значения будут по-прежнему разложить правила найма, коэффициенты производительности и других параметров в плане. 

**Переопределение организации** используются для вознаграждения для сотрудника на производительность одного или нескольких отделов. Процент, который распределяется между подразделениями должны составлять 100 процентов. Также рассматривается производительности отдельного сотрудника. Эти параметры используются только в том случае, если **оплата за производительность** выбирается при запуске процесса компенсации.
