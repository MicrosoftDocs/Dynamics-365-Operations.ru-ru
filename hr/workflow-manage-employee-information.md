# <a name="use-workflows-to-manage-employee-information"></a>Использовать workflow-процессы для управления сведений о сотруднике

В этом разделе объясняется, как возможности рабочего процесса для управления персоналом используется для управления сведений о сотруднике. Например можно связать с должностью workflow-процесс и настроить процесс утверждения, которое запускается при их История изменений сотрудников.

Возможности рабочего процесса для управления персоналом предоставляет множество бизнес-правил для управления мероприятия управления персоналом. Кроме того многочисленные доступны параметры, чтобы могли изменение конкретных workflow-процессов и их связь с иерархии отчетности. Для управления изменениями в несколько стандартных типов сведений о сотруднике доступны бизнес-правил. Workflow-процесс может быть связан с должностью. Затем сотрудников изменения их записи сотрудника, запуска процесса, требующего утверждения перед сохранением новую информацию. Заранее определены бизнес-правил для следующих типов сведений, чтобы помочь эффективно управлять изменениями и сохранения правильных данных сотрудников:

-   Идентификационные номера
-   Курсы
-   Образование
-   Изображение
-   Одолженные номенклатуры
-   Опыт работы
-   Проектный опыт
-   Навыки
-   Выборные должности
-   Действия управления персоналом
-   Регистрация курса

Когда сотрудников на работу, перемещены или прекращено, бизнес-правила можно включить процесс рассмотрения. Таким образом документ можно исправить или сроки действия можно определить как часть рабочего процесса. По завершении процесса просмотра документа или действия, заполняется и перемещает шага окончательного утверждения бизнес-правила.

## <a name="associate-a-workflow-with-a-position-hierarchy"></a>Связать workflow-процесс с Иерархия должностей
Workflow-процесс может быть связан с любой иерархии, которые были настроены. Например если должность, связанная с аналитической иерархии матрицы, можно настроить workflow-процесс, направляет руководителя проекта вместо Менеджер сотрудника, связанного с этой позиции расходов для определенного проекта. Для создания нового бизнес-правила или изменения существующего рабочего процесса в **workflow-процесса управления персоналом** и щелкните **Нью**. Выберите бизнес-правило в списке, чтобы запустить конструктор рабочих процессов. Конструктор можно использовать для создания нового бизнес-правила или этапы существующего рабочего процесса. При изменении существующего рабочего процесса, изменения сохраняются в новой версии. Таким образом вы всегда можно вернуться к предыдущей версии, если необходимо.

## <a name="configure-a-human-resources-workflow"></a>Настроить workflow-процесс управления персоналом
Чтобы настроить основные рабочего процесса, который запускается при сотрудников запрос на изменение на свой личный код, выполните следующие действия.

1.  На **workflow-процессов управления персоналом** и щелкните **создать**.
2.  В списке доступных бизнес-процессов, выберите **идентификационных номеров**.
3.  Щелкните **выполнения** для запуска конструктора рабочих процессов и затем введите имя пользователя и пароль, когда появится запрос.
4.  Перетащите **утвердить идентификационный номер** элемент из списка элементов workflow-процесса на основе конструктора.
5.  Элемент утверждения для подключения **начать** и **завершения**.
6.  Дважды щелкните **утвердить элемент**, а затем щелкните правой кнопкой мыши, а затем выберите **свойства**.
7.  Выполните следующие действия, чтобы добавить номенклатуры рабочие инструкции.
    1.  Выберите **назначения**, а затем выберите **иерархии** в столбце назначения тип.
    2.  В разделе **иерархии** выбора, выберите **настраиваемой иерархии**.
    3.  Добавьте условие остановки и закрыть диалоговое окно.

8.  Выполните любые дополнительные инструкции (должны существовать никакие дополнительные предупреждения).
9.  Нажмите кнопку **Сохранить и закрыть**. Активация нового бизнес-правила, когда будет открыто диалоговое окно и выберите **сделать активным**.
10. Последовательно выберите пункты **персоналом**&gt;**позиции**&gt;**поместите типы иерархии**.
11. Выберите **матрицы**.
12. Добавление **идентификационный номер работника** рабочего процесса в список.
