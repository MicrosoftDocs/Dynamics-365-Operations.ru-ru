# <a name="setting-up-the-components-of-a-job"></a>Настройка компонентов задания

В этом разделе описываются общие элементы, что задание может включать и приводятся примеры того, как можно использовать эти элементы в вашей организации. 

Перед созданием должностей необходимо настроить определенную справочную информацию. Можно создать задание, которое содержит только имя. Тем не менее а также дополнительные сведения, например, названия должности, предоставляет значения по умолчанию для должности, назначенные задания. Кроме того некоторые данные, которые вводятся используется для фильтрации планов вознаграждения для определенных должностей. Если необходимо настроить права, можно использовать для фильтрации планов вознаграждения для конкретного задания, следует настроить типы заданий и функции, перед настройкой должностей. Когда эти значения по умолчанию, сэкономить время при добавлении позиций к заданию. 

Некоторые сведения о задании, например, должность, тип и функции, текущей датой. Если создать задание сегодня, но не добавлять эти сведения до более поздней версии, а затем просмотрите задания по состоянию на дату создания, эти данные не отображается. Таким образом необходимо создать некоторые данные ссылки прежде, чем этого требуют. Таким образом, вы можете добавить сведения о новых заданий при их создании.

## <a name="job-titles"></a>Название должности
Перед созданием должностей необходимо настроить названия для этих должностей. Позиции наследуют названия из должностей, с которыми связаны эти позиции. 

Ведение должностей с использованием **заголовки** страницы, который можно открыть с помощью функции поиска. На ** заголовков ** введите заголовки, которые планируется использовать для заданий.

## <a name="job-types"></a>Типы должностей
Используйте типы заданий для группировки похожих заданий в категории. Типы должностей не являются необходимыми. Однако если планируется использовать типы должностей при настройке правил приемлемости для управления компенсациями, необходимо настроить типы должностей перед настройкой должностей. Примеры типов заданий: полный рабочий день, неполный рабочий день, или зарплаты и почасовой оплаты. Ведение типов должностей с помощью **типы заданий** страницы. На **типы заданий** введите имя и краткое описание типа должности. В **исключить статус** выберите один из следующих вариантов для указания статуса освобождений справедливых трудовых стандартах (FLSA) заданий, имеющих данный тип задания:

-   **Освобождение** — должности освобождаются от сверхурочного времени согласно закону FLSA.
-   **Освобождаемый тег** – задания не освобождаются от сверхурочного времени согласно закону FLSA.
-   **Не применяется** — покрытие FLSA не применимо.

## <a name="job-functions"></a>Должностные функции
Задание соединения описывают высокого уровня функциональные категории и соотносятся обязанностей высокого уровня. Функциональные обязанности не являются необходимыми. Можно использовать функциональные обязанности, вместе с типами должностей для фильтрации планов вознаграждения для определенных должностей. Связать типы заданий и функции с планами компенсации, настроив правила проверки на **правила проверки** страницы. Можно прикреплять множество уровней в плане компенсации, относящиеся к конкретным сочетанием тип задания и должностных обязанностей, который определен с помощью правила проверки. (Эти функции применяются для планов фиксированной компенсации и планов переменных компенсаций.) Тем не менее если вы планируете использовать должностных функций при настройке правил приемлемости для управления компенсациями, настраиваются должностных обязанностей до настройки должностей. Ниже приведены несколько примеров должностных обязанностей.

| Задание           | Должностные функции      |
|---------------|-------------------|
| Менеджер по продажам | Диспетчер среднего уровня |
| Бухгалтер    | Профессионалов     |

Ведение должностных функций с помощью **должностных обязанностей** страницы. На **должностных обязанностей** введите идентификационный код и краткое описание функциональной обязанности.

## <a name="areas-of-responsibility"></a>Области ответственности
Используйте области ответственности, чтобы показать рабочие роли, процессы и продукты, которые несет ответственность за работника, находится в положении, для задания. Например для задания с именем «Бухгалтер», одной области ответственности, может быть «Финансовая отчетность по продукту A.» Ведение областей ответственности, используя **области ответственности** страницы, который можно найти с помощью функции поиска. На **области ответственности** введите имя и описание области ответственности. В **Примечание** при необходимости можно ввести дополнительные сведения. Примечания могут быть обновлены для конкретного задания без изменения заметки, введенные здесь.
