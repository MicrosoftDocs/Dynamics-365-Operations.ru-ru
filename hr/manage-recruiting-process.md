# <a name="manage-a-recruiting-process"></a>Управление процессом набора сотрудников

В этой теме описывается понятие, которое агентов по найму кадров можно использовать для отслеживания действия с процессом набора сотрудников, включая свои усилия на рекламу открытых вакансий и найма кандидатов, отслеживание информации о кандидате и приложения, собеседование с кандидатами и выбора одного или нескольких кандидатов для заполнения открытых должностей в вашей организации.

<a name="overview"></a>Обзор
--------

Проекты по набору сотрудников могут помочь вам организовать этапы, выполняемые при заполнении открытых вакансий в юридическом лице. Соискатель — это лицо, применяется для приема на работу для данной организации.  Приложение — выражение кандидата интерес, применяемые в компании и могут быть связаны с проект по набору сотрудников для срочной интерес к конкретной открытия.  Одного кандидата может иметь несколько приложений в рамках той же компании или в нескольких компаниях в вашей организации.

<a name="recruitment-projects"></a>Проекты по набору сотрудников
--------------------

Проекты по набору сотрудников позволяют отслеживать ход выполнения и заполнение одной или нескольких открытых вакансий работодателям.  Проект по набору сотрудников идентифицирует отделение и должность, для которой открыты один или несколько позиций. В проектах по набору сотрудников также отслеживается следующая информация для открытых вакансий:
-   Специфическое число открытых вакансий
-   Менеджера по найму и альтернативное контактное лицо для вакансии
-   Дата утверждения заявки
-   Крайний срок подачи заявления
-   Предполагаемая дата начала

Проект по набору сотрудников содержит **Объявление о приеме на работу**, используемое на **портале самообслуживания сотрудников** для того, чтобы разрекламировать вакансию. Чтобы показать вакансию работникам, проект по набору сотрудников должен иметь **Объявление о приеме на работу**, в поле **Отобразить на портале самообслуживания сотрудников** должно быть задано значение "Да", в поле **Крайний срок** должна быть задана дата в будущем и проект по набору сотрудников должен иметь **Статус проекта** "Начато". В следующей таблице перечислены статусы проектов можно по набору сотрудников и их описание.

| **Status**    | **Indicates that…**                                                                  |
|-----------|------------------------------------------------------------------------------------------|
| Запланировано | Деятельность по найму подготавливаются.  Набор сотрудников еще не началась для данного проекта. |
| Начато   | Заявления теперь принимаются для вакансий в этом проекте.                    |
| Завершено  | Все вакансии для этого проекта были заполнены.                                          |
| Отменено  | Прием на работу был отменен для этого проекта.                                           |

Специалисты по набору персонала могут также записать **СМИ**, используемые для рекламы вакансии через внешние средства, а также отслеживать **Разработки** для проекта или заявлений.

<a name="applicants"></a>Кандидаты
----------

Соискатель — это лицо, применяется для задания на предприятии.  Кандидаты, совместно используются все юридические лица в вашей организации, предоставляя большого пула способностей поиск по. Можно хранить компетенции, рекомендации, запросы на размещение и личные данные для кандидатов. При создании записи кандидата в глобальной адресной создается запись о лице для этого кандидата. Страницу **Кандидат** можно использовать для обновления следующих данных глобальной адресной книги для кандидатов:
-   Адресная информация
-   Контактная информация
-   Идентификационная информация
-   Имя подробно
-   Личные данные

## <a name="applications"></a>Заявления
Можно зарегистрировать информацию из полученных заявлений о приеме на работу на странице **Заявление**. Приложение — выражение соискателя интерес к работе, открыв в вашей организации.  Чтобы создать приложение, кандидата должна существовать как кандидата или лица в системе.
Заявления о приеме на работу, которые отправляются кандидатами через Интернет, являются запрошенными заявлениями, введенными в ответ на объявление, или незапрошенными. Запрошенные заявления автоматически связываются с проектом по набору сотрудников, для которого это объявление было создано. Заявления, поданные по собственной инициативе, связываются с проектом по набору сотрудников, указанным в области **Набор сотрудников** на странице **Параметры управления персоналом**.
### <a name="application-status"></a>Статус заявления

Статус заявления показывает ход процесса приема на работу. В следующей таблице перечислены возможные статусы заявления и их описание.

| Состояние    | Указывает, что...                                                                           |
|-----------|-------------------------------------------------------------------------------------------|
| Получено  | Заявление получено.                                                             |
| Подтвержден | Кандидату отправлено уведомление, подтверждающее получение заявления.            |
| Собеседование | Кандидату может быть отправлено приглашение на собеседование.                                     |
| Отклонение | Кандидату может быть отправлено письмо с отказом.                                          |
| Отменено  | Кандидату может быть отправлено письмо с подтверждением отмены. Этот статус присваивается вручную. |
| Принят  | Кандидату может быть направлено предложение о приеме на работу.                                         |

### <a name="correspondence-actions"></a>Соответствующие действия

Действие по обмену корреспонденцией по **заявлению** определяет шаблон документа или сообщения электронной почты, используемый для связи с кандидатом, отправившим заявление. Можно связать **закладки приложения** с соответствующие действия, чтобы можно было использовать значения из приложения, страницам проекта кандидата, интервью и по набору сотрудников, в связи с кандидатами.  **Шаблоны электронной почты приложения** могут быть созданы для соответствующие действия, чтобы быстро отправить сообщения электронной почты кандидатам, приложение вместе с определенными статус и переписки действия. Например, может отправить электронное письмо с подтверждением для всех приложений с **статуса** получено и **действия корреспонденции** получено.  После отправки сообщения электронной почты, имеется возможность автоматического обновления состояния приложения.

## <a name="application-routing"></a>Маршрутизация заявлений

Если заявление должно быть просмотрено несколькими работниками, можно использовать страницу **Маршрутизация заявлений** для создания списка обращения работников и управления процессом.

## <a name="interviews"></a>Собеседования

**Интервью с кандидатом** может планироваться от **приложения** страницы.  Используйте **отправить данные о встрече**, чтобы отправить файл календаря со сведениями расписания интервью с кандидатом и работодатель.

## <a name="skill-mapping"></a>Подбор персонала

**Подбор персонала** и **Профили подбора персонала** можно использовать для определения кандидатов, которые могут быть хорошей кандидатурой на вакансию.

## <a name="hiring-applicants"></a>Прием кандидатов на работу

Используйте страницу **Заявления**, чтобы нанять заявителя. При приеме кандидата на работу статус заявления изменится на **Принят**, а запись кандидата в глобальной адресной книге будет связана с новой записью работника. Изменения данных глобальной адресной для новой записи работника также отображаются в записи этого кандидата. Это помогает сократить объем вводимых данных, если новый работник претендует на другую должность в вашем предприятии.  Для найма существующего работника в новую должность, щелкните **изменить положение** в **статус заявления** раскрывающейся инициирование процесса переноса.
