# <a name="cost-entries"></a>Записи затрат

В этой статье представлена информация о записях затрат и когда они созданы. Запись затрат — это запись, регистрирующая количество и стоимость данного события.

Записи затрат — это агрегирования проводок по запасам, которые записываются в активных финансовых складских аналитиках.

## <a name="examples"></a>Примеры
### <a name="example-1-no-cost-entries-are-created"></a>Пример 1: записи затрат не создаются

Регистрируется событие журнала перемещений. Событие перемещает одну часть номенклатуры А из местонахождения А в местонахождение B. Складская аналитика местонахождения не считается частью объекта затрат. Поэтому событие создает две проводки по запасам и не создает записей затрат.

### <a name="example-2-cost-entries-are-created"></a>Пример 2: записи затрат создаются

Регистрируется событие журнала перемещений. Событие передает часть товара А с узла 1 узла 2. Складской аналитикой узла считается частью объект стоимости. Поэтому событие создает две проводки по запасам и две записи затрат.

### <a name="example-3-one-cost-entry-is-created"></a>Пример 3: создается одна запись затрат

Событие поступления продуктов регистрируется для заказа на покупку. Событие регистрирует 100 частей номенклатуры А по цене за единицу 10,00 долларов США (USD). Поскольку номенклатура А использует серийный номер для отслеживания цели управления запасами, уникальный серийный номер создается для каждой полученной номенклатуры. Поэтому событие создает 100 проводок по запасам и одну запись затрат.

## <a name="cost-entries-page"></a>Страница "Записи затрат"
Новая страница **Записи затрат** позволяет просматривать регистрации количеств и затрат и управлять ими. Эта страница дополняет страницы **Проводка по запасам** и **Сопоставление запасов**. Записи регистрируются в хронологическом порядке для события. Таким образом, можно быстро найти накопленные затраты определенного события или всех событий, связанных с документом, и управлять ими. Рассмотрим пример:

-   Событие поступления продуктов регистрируется для номенклатуры A. Что частей получено по цене за единицу 10,00 долларов США.
-   Несколько дней после регистрации события накладной цена увеличивается до 11,00 долларов США. Следовательно общая сумма равна 1100 долларов США. Второй ваучер создается для учета разницы в 100 долларов США.
-   Несколько дней спустя регистрируются накладные расходы на сумму 15,00 долларов США для покрытия транспортных расходов по заказу на покупку.

| Операция | Дата       | Рекомендация      | Число | Номер лота  | Ссылка на лот | Номер возвращенного лота | Количество | Cумма  |
|---------|------------|----------------|--------|---------|---------------|---------------|----------|---------|
| 00001   | 01-01-2015 | Заказ на покупку | 100001 | 0000101 |               |               | 100,00   | 1000,00 |
| 00002   | 20-01-2015 | Заказ на покупку | 100001 | 0000101 |               |               |          | 100,00  |
| 00003   | 31-01-2015 | Корректировка     | 100001 | 0000101 |               |               |          | 15,00   |

На странице **Записи затрат** можно выполнить фильтрацию по коду и дате документа. **Примечание:** записи затрат будут доступны только для [затрат объекты](cost-object.md) или выпущенные продукты.
