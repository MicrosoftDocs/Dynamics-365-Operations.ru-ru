# <a name="product-number-nomenclature"></a>Номенклатура номеров продуктов

[!include[banner](../includes/banner.md)]


В этом разделе описывается, как можно настроить номенклатуру номеров продуктов для замены фиксированного формата [Номер шаблона продукта - Конфигурация - Размер - Цвет - Стиль] на требуемый формат, который включает номер шаблона продукта, активные аналитики продукта и текстовые разделители по вашему выбору. Также можно создать номенклатуру для идентификации конфигураций, созданных с помощью конфигуратора продукта на основе ограничений. Эти номенклатуры могут содержать атрибуты по вашему выбору.

Номенклатура номеров вариантов продукта позволяет включать сегменты в ваши идентификаторы вариантов продукта. Эти сегменты могут включать номер шаблона продукта, аналитики продукта, номерные серии, текстовые константы и атрибуты. Эта функция позволяет быстро найти определенный вариант продукта при создании заказа на продажу или заказа на покупку.

## <a name="nomenclature-of-predefined-product-variants"></a>Номенклатура предопределенных вариантов продуктов
Варианты продукта создаются для шаблонов продуктов в соответствии с одной из трех технологий конфигурации:

-   Предопределенные варианты
-   На основе ограничений
-   На основе аналитик

Каждый вариант продукта имеет номер, и номенклатура идентификации варианта продукта позволяет выбрать сегменты, которые будут включены в каждый номер варианта продукта. Можно выбрать следующие сегменты на странице **Номенклатура продуктов**.

-   Номер шаблона продукта
-   Значение номерной серии
-   Текстовая константа
-   Аналитики продуктов
    -   Конфигурация
    -   Цвет
    -   Размер
    -   Стиль

После определения номенклатуры идентификации вариантов продуктов ее можно связать с группой аналитик продуктов. Следовательно, всем шаблонам продуктов, ссылающимся на эту группу аналитик продуктов, будут назначены номера вариантов продуктов в соответствии с номенклатурой. Также можно назначать номенклатуру идентификации вариантов продуктов непосредственно в шаблоне продукта; в этом случае вариантам продукта, принадлежащим этому мастеру, назначаются номера вариантов продуктов в соответствии с номенклатурой.

### <a name="example"></a>Пример

Футболка (TS1234) производится в 3 разных размерах (S, M, L), 4 разных цветах (Красный, Зеленый, Голубой, Желтый) и 2 стилях (Поло, V), что дает 24 возможных вариантов продукта. Номенклатура идентификации вариантов продукта создается со следующими сегментами:

1.  Номер шаблона продукта
2.  Текстовая константа: "-"
3.  Цвет
4.  Текстовая константа: "-"
5.  Размер
6.  Текстовая константа: "-"
7.  Стиль

Номер варианта продукта для красной футболки поло маленького размера будет: TS1234-Red-Small-Polo.

## <a name="nomenclature-of-constraintbased-configurations"></a>Номенклатура конфигураций на основе ограничений
Для конфигураций, основанных на ограничениях, можно создать специальную номенклатуру для аналитики продукта конфигурации. Можно выбрать следующие сегменты на странице **Номенклатура продуктов**.

-   Значение номерной серии
-   Текстовая константа
-   Значение атрибута 

Каждый из компонентов в модели конфигурации продукта может иметь собственную номенклатуру конфигурации. Только атрибуты, принадлежащие компоненту, могут быть использованы. Атрибуты из субкомпонентов или из требований пользователя недоступны.

### <a name="example"></a>Пример

Модель конфигурации продукта имеет корневой компонент с двумя атрибутами.

-   Материал (пластик, древесина, сталь)
-   Длина (10...100)

Номенклатура конфигурации определена, используя следующие сегменты:

1.  Значение атрибута: Материал
2.  Текстовая константа: "AAA"
3.  Значение атрибута: Длина

Код конфигурации для деревянного материала с длиной 78 получит следующий код конфигурации: WoodAAA78.

## <a name="nomenclature-of-dimensionbased-configurations"></a>Номенклатура конфигураций на основе аналитик
Для конфигураций, основанных на аналитиках, можно создать специальную номенклатуру для аналитики продукта конфигурации. Можно выбрать следующие сегменты на странице **Номенклатура продуктов**.

-   Значение номерной серии
-   Текстовая константа
-   Номенклатура конфигурационной группы

Номенклатура конфигурации могут быть определена для спецификации (BOM).

### <a name="example"></a>Пример

Спецификация имеет 4 строки спецификации, разделенные на 2 конфигурационной группы.

-   Строка спецификации: M0007, стандартный шкаф
    -   Конфигурационная группа: Шкаф
-   Строка спецификации: M0008, Шкаф верхнего сегмента
    -   Конфигурационная группа: Шкаф
-   Строка спецификации: M0021, Передняя решетка ткань
    -   Конфигурационная группа: Передняя решетка
-   Строка спецификации: M0022, Передняя решетка металл
    -   Конфигурационная группа: Передняя решетка

Номенклатура конфигурации определена, используя следующие сегменты:

1.  Конфигурационная группа: Шкаф
2.  Текстовая константа: "&"
3.  Конфигурационная группа: Передняя решетка

Код конфигурации для стандартного корпуса с тканевой передней решеткой будет: M0007&M0021.

## <a name="nomenclature-of-a-combination-of-product-variants-and-configurations"></a>Номенклатура комбинации вариантов продукта и конфигураций
При использовании технологии конфигурации на основе ограничений или на основе аналитик для конфигурации вариантов продукта для шаблона продукта варианты продукта могут получить номера вариантов продукта, которые включают номенклатуру из аналитики конфигурации. Чтобы настроить варианты, выполните следующие действия.

1.  Определите номенклатуру номеров вариантов продукта, которые включают аналитику конфигурации, на странице **Номенклатура продукта**.
2.  Назначьте эту номенклатуру группе аналитики продукта с аналитикой конфигурации.
3.  Определите номенклатуру конфигурации для компонентов или спецификаций, которые будут использоваться для настройки вариантов продукта.

### <a name="example-for-constraint-based-configurations"></a>Пример конфигураций на основе ограничений

В этом примере можно использовать номенклатуру номеров вариантов продукта, которая состоит из следующих сегментов:

1.  Номер шаблона продукта
2.  Текстовая константа "\_"
3.  Конфигурация

Номенклатура конфигурации может состоять из следующих сегментов:

1.  Значение атрибута: Материал
2.  Текстовая константа: "AAA"
3.  Значение атрибута: Длина

Можно ввести следующие значения для сегментов:

-   Номер шаблона продукта = M0099
-   Материал = Пластик
-   Длина = 12

Номер варианта продукта станет: M0099\_PlasticAAA12.

### <a name="example-for-dimension-based-configurations"></a>Пример конфигураций на основе аналитик

В этом примере можно использовать номенклатуру номеров вариантов продукта, которая состоит из следующих сегментов:

1.  Номер шаблона продукта
2.  Текстовая константа: "//"
3.  Конфигурация

Номенклатура конфигурации может состоять из следующих сегментов:

1.  Конфигурационная группа: Шкаф
2.  Текстовая константа: "&"
3.  Конфигурационная группа: Передняя решетка

Можно ввести следующие значения для сегментов:

-   Номер шаблона продукта = D0123
-   Корпус = M0008
-   Передняя решетка = M0022

Номер варианта продукта станет: D0123//M0008&M0022.

## <a name="numbering-conflicts"></a>Конфликты нумерации
Может получиться, что настроенная номенклатура номеров вариантов продуктов не обеспечивает уникальность номеров вариантов продукта. Например, это может произойти, если одна из активный аналитик продукта не включена в номенклатуру для шаблона продукта, который использует технологию конфигурации предопределенных вариантов. Конфликты обрабатываются по-разному для различных технологий конфигурации.

### <a name="predefined-variants"></a>Предопределенные варианты

Произойдет ошибка, если попытаться вручную или автоматически создать варианты продукта, когда один или несколько вариантов приводят к одинаковому номеру варианта продукта. Во избежание этого необходимо использовать все активные аналитики продукта в группе аналитик продукта либо включить номерную серию для обеспечения, что номера вариантов продукта будут уникальными.

### <a name="constraint-based-configurations"></a>Конфигурации на основе ограничений

В зависимости от номенклатуры система может попытаться назначить неуникальный номер варианта продукта для конфигурации. В этом случае система будет использовать номерную серию для аналитики конфигурации как номер варианта продукта. В этом случае появится предупреждение. Во избежание это необходимо включить достаточно атрибутов в номенклатуру для обеспечения уникальности, а также убедиться, что параметр **Повторное использование** включен для компонента.

### <a name="dimension-based-configurations"></a>Конфигурации на основе аналитик

Процесс настройки включает шаг, на котором система предложит значение конфигурация в соответствии с номенклатурой. На этом этапе можно вручную изменить значение конфигурации. При сохранении конфигурации система проверит, является ли значение конфигурации уникальным. Если это не так, отображается ошибка. Необходимо ввести уникальное значение конфигурации для сохранения конфигурации.
