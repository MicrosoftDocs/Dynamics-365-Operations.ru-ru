# <a name="warehouse-configuration"></a>Конфигурация склада

В этой статье описывается, как настроить склад. Она включает информацию о том, как включить макет склада и процессы склада.

**Примечание.** Эта статья применяется к функциям модуля** Управление складом** (расширенное складирование). Она не применяется к функциям склада в модуле** Управление запасами**.

## <a name="warehouse-layout"></a>Макет склада
Система управления складом в Microsoft Dynamics 365 for Operations предоставляет гибкие способы определения макета склада для соответствия изменяющимся потребностям, чтобы гарантировать оптимальную эффективность склада.

-   Можно определить области хранения с высоким и низким приоритетом для оптимального размещения товаров.
-   Можно разделить склад на зоны в соответствии с различными потребностями хранения, например требованиями к температуре, или различной скоростью оборота номенклатур.
-   Можно указать местонахождения склада на любом уровне (например, сайт, склад, проход, стеллаж, полка и ячейка).
-   Можно сгруппировать местонахождения с помощью параметров ограничения физической емкости.
-   Можно управлять тем, как будут храниться и комплектоваться номенклатуры, на основании правил, определенных запросом.

Для использования управления складом в Microsoft Dynamics 365 for Operations необходимо создать склад и включить его для более сложных или специализированных мероприятий управления складом. На странице **Склады** установите флажок **Использовать процессы управления складом**.

### <a name="zone-groups-zones-location-types-and-locations"></a>Группы зон, зоны, типы местонахождений и местонахождения

В рамках процесса включения макета склада необходимо определить группы зон склада, зоны, профили местонахождений, типы местонахождений и местонахождения.

-   **Группы зон** — логическая или физическая группировка зон на складе.
-   **Зоны** — логическая или физическая группировка местонахождений на складе.
-   **Профили местонахождений** — логическая или физическая группировка местонахождений, которые имеют одинаковые политики процессов местонахождений склада (например, комбинирование различных номенклатур, которые можно хранить в определенном местонахождении, и одинаковые применимые ограничения по физической емкости).
-   **Типы местонахождений** — логическая или физическая группировка местонахождений склада. Например, можно создать тип местонахождения для всех промежуточных местонахождений. Обязательные параметры на странице **Параметры управления складом** управляют процессом определения типов местонахождений и типа итогового местонахождения отгрузки.
-   **Местонахождения** — самый низкий уровень сведений о местонахождении. Местонахождения используются для отслеживания места хранения и комплектации запасов в наличии на складе.

Объекты, создаваемые для определения макета склада, используются в запросах, настроенных в шаблонах работ для обработки заказов на выполнение работ на складе. Поэтому, когда вы определяете зоны, типы местонахождений и т. д., учитывайте то, как используются различные области на складе для различных процессов. Кроме того, учитывайте факторы, такие как физические характеристики определенной области. Например может быть областей, где можно использовать только с определенным типом автопогрузчике. Или, если ваша компания имеет производства и готовых товаров в том же производственном объекте, можно создать один склад в Dynamics 365 для операций, но затем разделить две операции путем создания двух групп зоны. Предоставление объектов описательные имена, чтобы легко идентифицировать их при использовании в запросах шаблона.

### <a name="location-stocking-limits-location-profiles-and-fixed-picking-locations"></a>Лимиты хранения местонахождения, профили местонахождений и фиксированные ячейки комплектации

Необходимо учитывать физический макет склада для определения объема складского помещения (лимиты хранения местонахождения и профили местонахождений) и достижения оптимальных процессов склада. 

Местоположение хранения пределы помогают обеспечить, что работы не могут быть созданы для запроса запасов помещаться в расположении, которое не имеет физических возможностей, который должен выполняться на складе. Например если некоторые ячейки на складе, может содержать только одну палету в каждой ячейке, можно включить хранение границы ячейки. ** Количество ** может быть присвоено значение **1**и ** единицы ** может быть присвоено значение **PL** в определенном месте группе профилей. 

Если необходимы более сложные расчеты для управления вместимостью местонахождения, можно использовать параметры профилей местонахождений. В этом случае при расчете вместимости учитываются вес и объем. 

Чтобы достигнуть оптимальных исходящих процессов, следует решить, стоит ли использовать фиксированные ячейки комплектации и/или местонахождения упаковки. Часто минимальное/максимальное пополнение используется для процессов пополнения из массовой зоны в фиксированные ячейки комплектации, и можно включить несколько фиксированных ячеек комплектации на одном складе и для вариантов продукта. Подумайте о том, какой гибкости можно достигнуть, включив выделенные местонахождения пополнения спроса, которые используются только для обработки волны/пополнения загрузок.

### <a name="location-setup-wizard"></a>Мастер настройки местонахождения

Чтобы быстро создать ячейки на складе, можно использовать ** настройку склада ** мастера. В рамках этого процесса можно легко вести формат имен местонахождений.

## <a name="warehouse-processes"></a>Процессы склада
При конфигурации склада важно включить процессы склада согласно бизнес-требованиям. Самые важные компоненты, которые необходимо настроить, — это шаблоны волн, шаблоны работ, пулы работ и директивы местонахождений.

### <a name="wave-templates"></a>Шаблоны волны

Шаблоны волн помогают осуществлять исходящий процесс "Запуск на склад". После запуска строк заказа в производство (непосредственно из документов-источников, через процессы пакетных заданий или через уже созданные загрузки) используются функциональные возможности шаблона волны. 

Можно создать три типа шаблонов волна: **Отгрузка**, **производственный заказ**, и **канбана**. Параметры используются для определения того, насколько система автоматически добавляется в обработку исходящих рабочих. Шаблон волны выбирается на основании последовательности шаблонов волн и критериев, указанных в шаблоне. Если шаблон указан вверху последовательности, то в первую очередь проверяются критерии в этом шаблоне. Если критерии можно выполнить, шаблон волны обрабатывается. В противном случае проверяются критерии в следующем шаблоне и так далее. Поэтому рекомендуется поместить шаблон с самыми точными критериями наверх списка последовательности шаблонов волн, чтобы он обрабатывался в первую очередь. Например, требуется обработать всю работу для определенного перевозчика сегодня и временно отложить обработку работы для других перевозчиков. В этом случае шаблон волны, который выбирает работу для этого перевозчика, следует поместить выше, чем другие шаблоны в последовательности. В противном случае работа для других перевозчиков может обрабатываться раньше завышения работы для этого перевозчика. 

Необходимо указать методы обработки волн в каждом шаблоне волны. Доступные методы зависят от типа шаблона волны.

### <a name="work-templates"></a>Шаблоны работ

Определения шаблонов волн играют важную роль в определении процессов работ управления складом. Они определяют, какая работа выполняется и какая работа завершена. Шаблоны могут также содержать код директивы, который связан с директивой местонахождения для определения места выполнения работы. Шаблоны работ включают запрос, который определяет критерии для работы. Каждый шаблон должен включать по крайней мере одну операцию комплектации и одну операцию размещения для выполнения базовой операции работы по перемещению запасов в наличии из одного местонахождения в другое. 

Если несколько работников должны иметь возможность обрабатывать работу по некоторым операциям склада, может потребоваться использовать принцип *промежуточной обработки* для запасов и разделить выполнение работы на различные классы работы.

### <a name="work-pools"></a>Пулы работ

Пулы работ используются для организации работы в группы. Например, можно создать пул работ для классификации работы, которая выполняется в определенном местонахождении склада. Для всех типов работ кроме инвентаризации можно назначить пул работ шаблону работы. В случае подсчета циклов можно назначить пул работ на следующих страницах:

-   Планы подсчета циклов
-   Пороги подсчета циклов
-   Работа подсчета циклов по месту хранения
-   Работа подсчета циклов по номенклатуре

При использовании шаблонов работ для создания работы пул работ автоматически присваивается работе. 

Коды пулов работ можно использовать для ограничения работы, которая направлена определенному работнику склада, при условии, что эта функциональность настроена в соответствующем пункте меню мобильного устройства.

### <a name="location-directives"></a>Директивы местонахождений

Как предполагает имя, директивы местонахождений используются для направления проводок работы в соответствующие местонахождения на складе. Другими словами, они определяют место комплектации и размещения. 

Чтобы упростить и ускорить определение действий, которые связаны с каждой строкой директивы местонахождений, используйте одну из предопределенных стратегий. Например, можно использовать стратегию **Пустое местонахождение без входящих работ** для поиска свободных местонахождений на складе или можно использовать стратегию **Резервирование партии по FEFO** для исходящей комплектации для продаж.
