# <a name="configure-app-field-names-in-warehousing-app"></a>Настройка имен полей приложения в приложении "складом"

В этом разделе описывается, как определить и настроить имена полей приложения склада и приоритеты в Dynamics 365 для операций. 

**Примечание:** этот раздел относится к функциям управления складом. Он не применяется к функции в разделе Управление запасами. Dynamics 365 для операций - складские операции является приложение, которое можно использовать для выполнения задач на склад. Вы можно определить и настроить имена полей, которые используются в приложении, а также настроить приоритет, для которых назначены имена полей. В этом разделе объясняется, как определить и настроить эти имена полей приложения склада и приоритетов, а порядок их использования в Dynamics 365 для операций - складские операции. Подробные сведения о порядке настройки подключения Dynamics 365 for Operations - складских операций, см. руководство [установить и настроить Dynamics 365 for Operations - складские операции](install-configure-warehousing-app.md).

<a name="configure-warehouse-app-field-names"></a>Настройка имен полей приложения склада
===================================

При использовании Dynamics 365 для операций - складские операции на мобильном устройстве, можно настроить способ отображения метаданных на устройстве на **склад имена полей приложения** страницы. В новой компании в Dynamics 365 для операций, выберите **создать настройку по умолчанию** для создания все имена полей, которые будут использоваться в бизнес-правила, склад мобильном устройстве, а затем назначьте предпочтительного режима ввода и тип входных данных для них. После создания всех имен полей, можно выбрать следующие параметры ввода.

<a name="configure-warehouse-app-field-priority"></a>Настройка склада приложению поле приоритет
======================================

На **склад приложению поле приоритет** страницы, имена полей можно поместить на другой приоритет группы. Это дает возможность определить, какие сведения должны отображаться на странице основная задача Если складские работники выполняют задачи с помощью приложения. При нажатии кнопки **создать настройку по умолчанию**, создается набор групп приоритет по умолчанию. Можно создавать столько групп приоритет, при необходимости, но будут отображаться только трех групп приоритетов на странице задач. Dynamics 365 for Operations приложение отправляет метаданных, каждое поле назначит относительный приоритет в зависимости от его приоритет группы и приложение будет отображать первой трех приоритетов групп, содержащихся в метаданных на странице задач. Например, когда работник склада выполняет задачу на мобильном устройстве, если метаданные, которые будут отображаться в приложении состоит из следующих полей:

-   Раздел
-   Количество
-   Единица измерения
-   Описание номенклатуры
-   Размер и местоположение

На основании склада приложению поле приоритет в приведенной выше таблице, следующие 3 строки данных будут отображаться на странице задач:

-   Строка 1: Единица измерения номенклатуры, количество,
-   Строки 2: Описание номенклатуры
-   Строка 3: размер

Оставшиеся метаданные, например, местоположение, не будет отображаться на странице задач, но будет отображаться на веб-страницу.
