# <a name="replenishment"></a>Пополнение

[!include[banner](../includes/banner.md)]


В этой статье описываются стратегии пополнения, доступные для складов, использующих функциональные возможности, доступные в модуле "Управление складом".

В этой статье описываются стратегии пополнения, доступные для складов, использующих функциональные возможности, доступные в модуле "Управление складом". Эта информация не относится к решению складирования, доступному в модуле "Управление запасами". Доступны три стратегии пополнения:

-   **Пополнение, основанное на спросе волны** — эта стратегия создает работу пополнения для исходящих заказов или загрузок, если запасы недоступны при создании работы в волне. Например, работу пополнения можно создать, если количество, необходимое для заказа на продажу, недоступно при обработке волны.
-   **Пополнение, основанное на минимальном и максимальном количествах** — эта стратегия использует минимальные и максимальные лимиты хранения для определения времени пополнения местонахождений. Критерии номенклатур и местонахождений определяют запасы, которые оцениваются для пополнения. Шаблоны пополнения, основанного на минимальном и максимальном количествах, являются основным механизмом для поддержания оптимального уровня в ячейках комплектации. Чтобы гарантировать доступность достаточного количества запасов для комплектации для удовлетворения спроса, можно использовать пополнение спроса как дополнение между циклами пополнения, основанного на минимальном и максимальном количествах.
-   **Пополнение, основанное на спросе загрузки** — эта стратегия суммирует спрос для нескольких загрузок и создает работу пополнения, необходимую для пополнения запасов в соответствующих ячейках комплектации. Эта стратегия гарантирует, что создаваемые загрузки можно скомплектовать на складе после их выпуска.

Все три стратегии создают работу пополнения, основанную на шаблоне пополнения.

## <a name="wave-demand-replenishment"></a>Пополнение, основанное на спросе волны
Пополнение, основанное на спросе волны, создает работу пополнения на основании спроса, если количество, необходимое для исходящих заказов или загрузок не доступно при создании работы в волне. Шаблон пополнения содержит сведения о критериях номенклатуры, единице измерения, увеличении спроса и местонахождении. 

Директивы местонахождений используются для определения того, какие местонахождения следует пополнять. Эти директивы местонахождений можно связать с шаблоном пополнения с помощью поля **Код директивы**. Если в поле **Код директивы** не задано значение, используются запросы для определения того, какие директивы местонахождений должны использоваться. Обратите внимание, что если код директивы не указан в шаблоне пополнения и директива местонахождения имеет код директивы, директива местонахождения будет игнорироваться, даже если запрос в директиве местонахождения правильный. Директивы местонахождений комплектации используются для определения того, откуда можно получить запасы для пополнения. 

Помимо создания шаблона необходимо указать некоторые параметры пополнения в шаблоне волны. Шаблон волны должен содержать шаг волны для пополнения, который выполняется, только если распределение номенклатуры не выполнено успешно. Этот шаг волны пополнения использует код шага волны для определения шаблона пополнения, который следует использовать. Помимо шага волны для пополнения необходимо убедиться в том, **пополнение** выбрано в разделе **Методы** шаблона волны. 

Страница **Шаблон пополнения** включает флажок **Разрешить волне спроса использовать незарезервированные количества**. Этот флажок следует установить, если нужно, чтобы при пополнении спроса вычитались незарезервированные количества из работы, созданной на основе выбранного шаблона пополнения. Чтобы использовать эту логику в шаблонах пополнения спроса, флажок должен быть установлен для всех существующих шаблонов пополнения. Когда пополнение спроса запускается на складе, спрос вычитается из существующей работы пополнения с незарезервированными количествами, если эта работа создана на основе шаблонов пополнения с установленным флажком **Разрешить волне спроса использовать незарезервированные количества**.

## <a name="minmax-replenishment"></a>Пополнение, основанное на минимальном и максимальном количествах
В пополнении, основанном на минимальном и максимальном количествах, запасы пополняются таким образом, чтобы они находились в заданных пределах между минимальным и максимальным количествами. Обычно этот процесс происходит один раз в день, чтобы гарантировать, что все ячейки комплектации заполняются до максимального уровня перед началом комплектации. 

Минимальная и максимальная суммы задаются в шаблоне пополнения. Многие другие параметры в шаблоне напоминают параметры в шаблонах, используемых в пополнении, основанном на спросе волны. Шаблон должен включать одну строку для каждой номенклатуры и местонахождения. При выполнении пополнения с помощью пакетного задания Microsoft Dynamics 365 for Operations будет оценивать, требуется ли пополнение, в последовательности, в которой организованы строки. 

Обратите внимание, что при использовании стратегии пополнения, основанной на минимальном и максимальном количествах, невозможно пополнить пустое местонахождение, если оно не задано как фиксированное местонахождение для номенклатуры. Если местонахождение, которое необходимо пополнить, не является фиксированным местонахождением, Dynamics 365 for Operations не может определить, какую номенклатуру пополнять. Таким образом, необходимо по крайней мере какое-то количество в запасе до выполнения пополнения.

## <a name="load-demand-replenishment"></a>Пополнение спроса загрузки
Пополнение, основанное на спросе загрузки, суммирует спрос для нескольких загрузок и создает работу пополнения, необходимую для пополнения запасов в соответствующих ячейках комплектации. Пополнение, основанное на спросе загрузки, похоже на пополнение, основанное на спросе волны, во многих отношениях. Основное различие состоит в том, как и когда выполняется пополнение в обоих случаях. Как и при пополнении, основанном на минимальном и максимальном количестве, пополнение, во основанное на спросе загрузки, выполняется с помощью пакетного задания. Чтобы настроить пакетное задание, на странице **Пополнение, основанное на спросе загрузки** выберите шаблон пополнения для использования и задайте запрос фильтра, чтобы указать, какие загрузки используются для определения спроса. Запрос местонахождения определяет местонахождения, из которых будет вычитаться все доступное количество для удовлетворения агрегированного спроса загрузок.

## <a name="replenishment-prerequisites"></a>Необходимые условия для пополнения
| Необходимые условия            | Описание                                                                                                                                                                                                                                        |
|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Товар                    | Для процессов управления складом должна быть включена номенклатура.                                                                                                                                                                                       |
| Склад               | Для процессов управления складом должен быть включен склад. Чтобы включить склад для процессов управления складом, на странице **Склады** выберите склад и установите флажок **Использовать процессы управления складом**. |
| Шаблоны пополнения | Необходимо настроить хотя бы один шаблон пополнения для пополнения, основанного на минимальном и максимальном количествах, пополнения, основанного на спросе загрузки, или пополнения, основанного на спросе загрузки.                                                                                                             |
| Местоположения               | Необходимо создать местонахождения и подключить их к профилю местонахождения.                                                                                                                                                                                     |
| Профили местонахождения       | Для создания местонахождений требуются профили местонахождений.                                                                                                                                                                                       |
| Директивы для мест хранения     | Директивы местонахождений являются обязательными для направления работы в местонахождения, в которых требуется пополнение, и местонахождения, из которых берутся запасы.                                                                                     |
| Шаблоны работ          | Шаблоны работы типа **Пополнение** являются обязательными для создания работы пополнения, чтобы запасы можно было переместить в нужные местонахождения.                                                                                           |
