# <a name="replenishment"></a><span data-ttu-id="e78c2-103">Пополнение</span><span class="sxs-lookup"><span data-stu-id="e78c2-103">Replenishment</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="e78c2-104">В этой теме описываются стратегии пополнения, доступные для складов, использующих функциональные возможности, доступные в модуле "Управление складом".</span><span class="sxs-lookup"><span data-stu-id="e78c2-104">This topic describes the replenishment strategies that are available for warehouses which use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="e78c2-105">Эта информация не относится к решению складирования, доступному в модуле "Управление запасами".</span><span class="sxs-lookup"><span data-stu-id="e78c2-105">The information doesn't apply to the warehousing solution that is available in Inventory management.</span></span> <span data-ttu-id="e78c2-106">Доступны следующие стратегии пополнения:</span><span class="sxs-lookup"><span data-stu-id="e78c2-106">The following replenishment strategies are available:</span></span>

-   <span data-ttu-id="e78c2-107">**Пополнение, основанное на спросе волны** — эта стратегия создает работу пополнения для исходящих заказов или загрузок, если запасы недоступны при создании работы в волне.</span><span class="sxs-lookup"><span data-stu-id="e78c2-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="e78c2-108">Например, работу пополнения можно создать, если количество, необходимое для заказа на продажу, недоступно при обработке волны.</span><span class="sxs-lookup"><span data-stu-id="e78c2-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
-   <span data-ttu-id="e78c2-109">**Пополнение, основанное на минимальном и максимальном количествах** — эта стратегия использует минимальные и максимальные лимиты хранения для определения времени пополнения местонахождений.</span><span class="sxs-lookup"><span data-stu-id="e78c2-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="e78c2-110">Критерии номенклатур и местонахождений определяют запасы, которые оцениваются для пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="e78c2-111">Шаблоны пополнения, основанного на минимальном и максимальном количествах, являются основным механизмом для поддержания оптимального уровня в ячейках комплектации.</span><span class="sxs-lookup"><span data-stu-id="e78c2-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="e78c2-112">Чтобы гарантировать доступность достаточного количества запасов для комплектации для удовлетворения спроса, можно использовать пополнение спроса как дополнение между циклами пополнения, основанного на минимальном и максимальном количествах.</span><span class="sxs-lookup"><span data-stu-id="e78c2-112">To ensure that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
-   <span data-ttu-id="e78c2-113">**Пополнение, основанное на спросе загрузки** — эта стратегия суммирует спрос для нескольких загрузок и создает работу пополнения, необходимую для пополнения запасов в соответствующих ячейках комплектации.</span><span class="sxs-lookup"><span data-stu-id="e78c2-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="e78c2-114">Эта стратегия гарантирует, что создаваемые загрузки можно скомплектовать на складе после их выпуска.</span><span class="sxs-lookup"><span data-stu-id="e78c2-114">This strategy helps to ensure that the loads that are created can be picked in the warehouse after they are released.</span></span>

<span data-ttu-id="e78c2-115">Все три стратегии создают работу пополнения, основанную на шаблоне пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-115">All three strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="e78c2-116">Пополнение, основанное на спросе волны</span><span class="sxs-lookup"><span data-stu-id="e78c2-116">Wave demand replenishment</span></span>

<span data-ttu-id="e78c2-117">Пополнение, основанное на спросе волны, создает работу пополнения на основании спроса, если количество, необходимое для производственных заказов, канбанов, исходящих заказов или загрузок не доступно при создании работы в волне.</span><span class="sxs-lookup"><span data-stu-id="e78c2-117">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="e78c2-118">Шаблон пополнения содержит сведения о критериях номенклатуры, единице измерения, увеличении спроса и местонахождении.</span><span class="sxs-lookup"><span data-stu-id="e78c2-118">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="e78c2-119">Директивы местонахождений используются для определения того, какие местонахождения следует пополнять.</span><span class="sxs-lookup"><span data-stu-id="e78c2-119">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="e78c2-120">Эти директивы местонахождений можно связать с шаблоном пополнения с помощью поля **Код директивы**.</span><span class="sxs-lookup"><span data-stu-id="e78c2-120">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="e78c2-121">Если в поле **Код директивы** не задано значение, используются запросы для определения того, какие директивы местонахождений должны использоваться.</span><span class="sxs-lookup"><span data-stu-id="e78c2-121">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="e78c2-122">Обратите внимание, что если код директивы не указан в шаблоне пополнения и директива местонахождения имеет код директивы, директива местонахождения будет игнорироваться, даже если запрос в директиве местонахождения правильный.</span><span class="sxs-lookup"><span data-stu-id="e78c2-122">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="e78c2-123">Директивы местонахождений комплектации используются для определения того, откуда можно получить запасы для пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-123">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="e78c2-124">Помимо создания шаблона необходимо указать некоторые параметры пополнения в шаблоне волны.</span><span class="sxs-lookup"><span data-stu-id="e78c2-124">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="e78c2-125">Шаблон волны должен содержать шаг волны для пополнения, который выполняется, только если распределение номенклатуры не выполнено успешно.</span><span class="sxs-lookup"><span data-stu-id="e78c2-125">The wave template should contain a wave step for replenishment that is run only if allocation of an item isn't successful.</span></span> <span data-ttu-id="e78c2-126">Этот шаг волны пополнения использует код шага волны для определения шаблона пополнения, который следует использовать.</span><span class="sxs-lookup"><span data-stu-id="e78c2-126">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="e78c2-127">Помимо шага волны для пополнения необходимо убедиться в том, что в разделе **Методы** шаблона волны выбрано **Пополнение**.</span><span class="sxs-lookup"><span data-stu-id="e78c2-127">In addition to having a wave step for replenishment, you need to make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span> 

<span data-ttu-id="e78c2-128">Страница **Шаблон пополнения** включает флажок **Разрешить волне спроса использовать незарезервированные количества**.</span><span class="sxs-lookup"><span data-stu-id="e78c2-128">The **Replenishment template** page includes an **Allow wave demand to use unreserved quantities** check box.</span></span> <span data-ttu-id="e78c2-129">Этот флажок следует установить, если нужно, чтобы при пополнении спроса вычитались незарезервированные количества из работы, созданной на основе выбранного шаблона пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-129">You should select this check box if you want to allow demand replenishment to deduct unreserved quantities from work that is generated from the selected replenishment template.</span></span> <span data-ttu-id="e78c2-130">Чтобы использовать эту логику в шаблонах пополнения на основании спроса, установите этот флажок для всех существующих шаблонов пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-130">To enable demand replenishment templates to use this logic, select this check box for every existing replenishment template.</span></span> <span data-ttu-id="e78c2-131">Когда пополнение спроса запускается на складе, спрос вычитается из существующей работы пополнения с незарезервированными количествами, если эта работа создана на основе шаблонов пополнения с установленным флажком **Разрешить волне спроса использовать незарезервированные количества**.</span><span class="sxs-lookup"><span data-stu-id="e78c2-131">When demand replenishment is triggered in the warehouse, it will deduct the demand from existing replenishment work that has unreserved quantities, if the work originates from replenishment templates where the **Allow wave demand to use unreserved quantities** check box is selected.</span></span>


<span data-ttu-id="e78c2-132">Пополнение на основании спроса поддерживается для заказов на продажу, заказов на перемещение, производственных заказов и канбанов.</span><span class="sxs-lookup"><span data-stu-id="e78c2-132">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="e78c2-133">Пополнение, основанное на минимальном и максимальном количествах</span><span class="sxs-lookup"><span data-stu-id="e78c2-133">Min/Max replenishment</span></span>
<span data-ttu-id="e78c2-134">В пополнении, основанном на минимальном и максимальном количествах, запасы пополняются таким образом, чтобы они находились в заданных пределах между минимальным и максимальным количествами.</span><span class="sxs-lookup"><span data-stu-id="e78c2-134">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="e78c2-135">Обычно этот процесс происходит один раз в день, чтобы гарантировать, что все ячейки комплектации заполняются до максимального уровня перед началом комплектации.</span><span class="sxs-lookup"><span data-stu-id="e78c2-135">Typically, this process occurs one time every day to ensure that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="e78c2-136">Минимальная и максимальная суммы задаются в шаблоне пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-136">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="e78c2-137">Многие другие параметры в шаблоне напоминают параметры в шаблонах, используемых в пополнении, основанном на спросе волны.</span><span class="sxs-lookup"><span data-stu-id="e78c2-137">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="e78c2-138">Шаблон должен включать одну строку для каждой номенклатуры и местонахождения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-138">The template should contain one line for each item and location.</span></span> <span data-ttu-id="e78c2-139">При выполнении пополнения с помощью пакетного задания Finance and Operations оценивает, требуется ли пополнение, в последовательности, в которой организованы строки.</span><span class="sxs-lookup"><span data-stu-id="e78c2-139">When you run replenishment by using the batch job, Finance and Operations evaluates whether replenishment is required, in the sequence that the lines are organized in.</span></span> 

<span data-ttu-id="e78c2-140">Обратите внимание, что при использовании стратегии пополнения, основанной на минимальном и максимальном количествах, невозможно пополнить пустое местонахождение, если оно не задано как фиксированное местонахождение для номенклатуры.</span><span class="sxs-lookup"><span data-stu-id="e78c2-140">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="e78c2-141">Если местонахождение, которое необходимо пополнить, не является фиксированным местонахождением, невозможно определить, какую номенклатуру пополнять.</span><span class="sxs-lookup"><span data-stu-id="e78c2-141">If the location that must be replenished isn't a fixed location, it's not possible to determine which item to replenish.</span></span> <span data-ttu-id="e78c2-142">Таким образом, необходимо по крайней мере какое-то количество в запасе до выполнения пополнения.</span><span class="sxs-lookup"><span data-stu-id="e78c2-142">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
