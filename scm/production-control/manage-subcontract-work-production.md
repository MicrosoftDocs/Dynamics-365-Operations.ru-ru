# <a name="manage-subcontracting-work-in-production"></a>Управление Субподряд незавершенного производства

В этом разделе объясняется, как субподрядных операций осуществляется в Microsoft Dynamics 365 для операций. Другими словами в ней объясняется, как производственных операций, которые распределяются по ресурсу осуществляется поставщиком.

В [производственных процессов](production-process-overview.md), может выполняться рабочих ресурсов, которые принадлежат или для администрирования поставщиков. Как правило, поставщик ресурсы используются для уровня периодических лишние требование, превосходит доступной мощности компании принадлежат ресурсы. Также можно попытаться предложить конкретного поставщика [ресурсов возможности](resource-capabilities.md)или ресурсов по более низкой цене.  

В зависимости от поставщика ресурсов, используемых в процессе производства [маршрут](routes-operations.md) часто имеются дополнительные требования, логистические, поскольку материалов и незаконченных продуктов необходимо сначала необходимо переместить узел поставщика. Затем необходимо передавать результат субподрядной операции, либо для склада, в который распределяется на следующую операцию или со складом готовой продукции.  

При использовании Субподряд операции или мероприятия влияет на все этапы операций из определения операций, которые требуются для производства, цены, прогнозирование, планирование и планирование, в управлении логистики материалов, незаконченных продуктов и готовых товаров. Наконец эти ресурсы требуются собственные процессы для учета и управление затратами.  

Для внутренних ресурсов обычно выделяется ставки постоянных затрат для периода. Напротив затраты ресурсов субподрядных основана на связанных сервисных цена покупки. Служба представляет собой другой продукции и используется для диска закупаемой продукции и процессы для субподрядных данной операции на покупку.  

В настоящее время отсутствует явные понятие незаконченных продуктов в Microsoft Dynamics 365 for Operations. Для производственного заказа, более чем одной операции для преобразования в готовый продукт сырья готовой продукции разносится обратно на склад только в последней операции. Незаконченных продукты, которые ранее операции приводят к учитываются в незавершенное производство (НЗП), но они не разнесен или не отслеживаются на складе. Несмотря на то, что можно разделить на несколько небольших блоков маршруты и спецификации (BOM), этот подход увеличивает количество продуктов, спецификаций и маршрутов, которыми необходимо управлять.  

Существует два метода для моделирования Субподряд работы для производственных операций. Эти методы различаются способами, что процесс Субподряд могут быть смоделированы, способ полуфабрикаты, представлены в процессе и управляет способом управления затратами.

-   Субподряд операций маршрута в производственных заказов или заказов партий
    -   Обслуживание продукта необходимо учитываемых в запасах продуктов, и он должен быть частью Спецификации.
    -   Этот метод поддерживает первым в, сначала ФИФО (FIFO) или стандартной себестоимости.
    -   Полуфабрикаты представлены в процессе обслуживания продукта.
    -   Управление затратами распределяет затраты, связанные с субподрядной работы, которая затраты на материалы.
-   Субподряд потока мероприятий производственного потока бережливого производства
    -   Служба является продуктом службы не хранятся на складе, и он не является частью Спецификации.
    -   Этот метод использует договоров покупки как соглашения на обслуживание.
    -   Этот метод использует backflush-расчета себестоимости.
    -   Этот метод позволяет статистическим и асинхронных закупаемой продукции. (Потоков материалов не зависит от процесса закупок.)
    -   Управление затратами выделяет субподрядной работы в блок разбиение затрат.

## <a name="subcontracting-of-route-operations"></a>Субподряд операций маршрута
Чтобы использовать Субподряд операций маршрута для производства или заказов партий, обслуживание продукта, используемый для закупок службы должен быть определен как произведение **службы** типа. Кроме того, необходимо иметь группу номенклатурных моделей с **Учитываемый в запасах продукт** под флажком **запасами политики** значение **Да**. Этот параметр определяет, учитывается ли продукта как запасов в поступлении продуктов (**Учитываемый в запасах продукт** = **Да**), или ли продукт списывается на счет прибылей и убытков (**Учитываемый в запасах продукт** = **нет**). Несмотря на то, что это поведение выглядит противоречащих друг другу, именно на основе того факта, что только продукты, для которых установлена данная политика создаст складских проводок, которые могут использоваться в управления затратами для расчета спланированных затрат и определения фактических затрат по завершении производственного заказа.  

Считается планирования и расчет себестоимости, необходимо добавить в спецификацию обслуживания. В строке Спецификации должен иметь **поставщика** типа и его должна быть выделена, служба резервируется для операции маршрута. Эта операция маршрута должны иметь ресурс для расчета себестоимости и потребности в ресурсах, указывая ресурс из **поставщика** тип, который подключается к счет поставщика соответствующие операции и связанной службы.  

При использовании этой конфигурации для связанных сервисных продуктов, на основе оценки производственного заказа создается заказ на покупку. Заказ на покупку службы используется в качестве базы для субподрядных операций. Можно управлять посредством субподрядных рабочих **Субподряд рабочих** страницы списка в управлении производством. Субподрядной работы используется для поставки сырья и, в конечном итоге незаконченных продукта поставщика при подготовке к операции. Он пригоден получения результирующий произведение субподрядной операции в прибытия номенклатуры, где продукт обслуживания используется для идентификации прибытия незаконченных продукта. При получении строки заказа на покупку, производственной операции обновляется как завершенные.  

Производственный заказ может иметь много операций и каждой операции, которые могут быть распределены по другому поставщику. Таким образом начала до конца производственного заказа могут запускать несколько заказов на покупку.

## <a name="subcontracting-of-production-flow-activities"></a>Субподряд мероприятий производственного потока
[Бережливого производства](lean-manufacturing-overview.md)решений модели Субподряд работы для службы, связанной с действием по [производственного потока](http://ax.help.dynamics.com/en/wiki/create-a-production-flow-version/) (в разделе Руководство по задаче). Таким образом, этот тип Субподряд также называется [продукцию субподрядчиков на основе операций.](activity-based-subcontracting.md) Специальный тип группы, затрат **прямой аутсорсинг**, были введены, и Субподряд служб, не являющихся частью Спецификации готовой продукции. При использовании бережливого производства, все действия определяются канбанов, которые могут быть связаны с одним или несколькими мероприятий производственного потока. На данный момент объяснение выглядит так же, как и объяснение производственных заказов. Тем не менее в то время как производственные заказы всегда должен заканчиваться готового продукта, можно создать канбаны поставку незаконченных продукта. Не нужно представить новый продукт и уровень Спецификации.  

Поскольку правила канбана могут быть очень динамический, можно модель различные варианты поставок на один программный продукт, в производственного потока. При использовании тонкого строго разделяются Субподряд, потоков материалов и финансовых потоках. Весь поток материалов представлен канбана мероприятий. Заказы на покупку для обновления продуктов и разноски прихода этих служб можно автоматизировать, на основе статуса заданий канбана в производственном потоке. Задания канбана может быть запускается и выполняется даже до создания заказов на покупку. Документы субподряда (заказ на покупку и приходная накладная покупки службы) могут быть агрегированы по периодам и обслуживания. Таким образом число документов на покупку и строк могут храниться небольшой, даже в очень повторяющихся операций, где поставщики предоставляют услуги субподряда в потоке один фрагмент.

### <a name="modeling-subcontracting-in-a-production-flow"></a>Моделирование продукцию субподрядчиков в производственном потоке

В [производственного потока бережливого](lean-manufacturing-modeling-lean-organization.md), мероприятие процесса может быть определен, так как Субподряд при распределенного производственной ячейки (группа ресурсов), для которого ресурс одному поставщику. При является субподрядным производственной ячейки, процесса, связанные действия должны быть привязаны строку соглашения активного покупки, содержащий сервисного товара и цены за услуги. Действия соглашения на обслуживание определяет также расчет отношение между количеством продуктов задания канбана и результирующее количество услуг. Можно выбрать ли служба количество рассчитывается на основе количества заданий, количество выпуск качественного продукта, включенный в отчет по заданиям, или общее количество продукта (это общее количество включает переведенных продуктов).  

Перемещение действий может быть также определена как Субподряд. Это определение происходит неявно при выборе Ответственный субъект для отгрузки в мероприятие перемещения. При выборе **грузоотправителем** или **получателя**, если соответствующий источник или цель склада хранилища управляемого поставщика, действие считается Субподряд. При выборе **перевозчика**, действие всегда является субподрядным. Процесс субподрядных мероприятий, таких как мероприятие перемещения субподрядных должен быть подключен к соглашению на обслуживание, прежде чем можно активировать производственный поток.

### <a name="backflush-costing"></a>Backflush-расчет себестоимости

Учет затрат субподрядной работы, полностью интегрирован в калькуляции затрат для бережливого производства решения (backflush-расчета себестоимости). При разноске получения заказа покупки службы или при выставлении накладных, стоимость обслуживания выделяется производственного потока. Backflush-расчета себестоимости, расхождение служб субподрядных расчета путем смещения Субподряд блок стандартной себестоимости полученных продуктов с количествами фактической службы полученные и с выставленным счетом.

## <a name="material-supply-for-subcontracted-operations"></a>Поставки материалов для операции субподряда
Незаконченных продуктов и других связанных с ней материалов необходимо перенести туда, где физически выполняется работа. При использовании субподрядных операций и мероприятий, передача часто связана дополнительная передача на сайт поставщиков-выполнении операций. Распределение материала в Спецификации для субподрядной операции, объявить, что материалы должны предварительно сохраняться в место хранения на складе группы ресурсов для распределенного ресурса. Сводное планирование и затем экономичным пополнением Подготовка материалов в это местоположение.  

Для моделирования запасов, расположенный в точке поставщика, рекомендуется в сфере определить склад управляемого поставщика. Склад управляемого поставщика легко можно определить, создав новый склад и присвоив счет поставщика. Чтобы зафиксировать этот материал необходимо перенести поставщика перед выполнением операции, следует выделить управляемого поставщика склада на склад получения группы ресурсов, в котором содержится ресурс.  

Для пополнения материалов в данном складе, можно использовать несколько стратегий:

-   Заказы на перемещение
-   Журналы переносов
-   Канбаны изъятия
-   Прямой покупки в ячейку поставщика

Незаконченных продуктами являются исключением из этого правила. Чтобы перенести полуфабрикаты, ограничен этих параметров:

-   Для производственных заказов и заказов партии, частично готовой продукции может быть передано только логически, используя журнал отгрузочных накладных с **Субподряд рабочих** страницы списка. Этот журнал будет создан документ транспортной накладной, который может использоваться для переноса незаконченных и сырья поставщику.
-   Для субподрядных операций в производственных потоков перемещение незаконченных продуктов описана путем получения канбаны изъятия или производства в местонахождении поставщика. Для моделирования действия явные перемещение, можно завершить канбан производства с мероприятием, дополнительную перенос.

**Примечание:** не может выходить за несколько узлов маршрута производства для одного производственного заказа. Это правило также применяется к субподрядной работы. Таким образом склады, представляющие управляемого поставщика материала, который ячеек должны определяться в том же сайте как внутренние ресурсы, которые используются в маршруте. Несмотря на то, что производственные потоки могут пересекать узлов, их нельзя передачи незаконченных продукты с одного узла на другой, из-за этого операция предполагает изменение контекста затрат.  

Как правило склад выпуска и расположение группы ресурсов субподрядных распределяются на склад и место следующего этапа в потоке маршрута или производственной операции. Эта настройка позволяет сократить объем задания, отчетность, который истек или число дополнительных операций, которые необходимо моделировать.
