# <a name="operations-resources"></a>Ресурсы Operations

[!include[banner](../includes/banner.md)]


Операционные ресурсы выполняют действия проекта или производственного процесса. Они могут быть разных видов и могут иметь различные возможности. 

<a name="operations-resources"></a>Ресурсы Operations
--------------------

Операционные ресурсы — это машины, инструменты, работники, средства, физические области или поставщики, которые выполняют работы проекта или производственного процесса. Они могут быть разных видов и могут иметь различные возможности.

-   **Поставщик** — внешний ресурс, который выполняет мероприятия проекта или производственные операции. Пример — субподрядчик. Связывая ресурсы поставщика со счетом поставщика, вы можете создавать покупки для субподрядчиков, основанные на строках спецификаций или строках производства.
-   **Человеческие ресурсы** — работник проекта или производства, который выполняет работу, самостоятельно или как оператор инструмента или машины. Если вы используете функциональность "Управление персоналом", вы можете связать человеческие ресурсы с работником. После этого подсистема планирования может выделять ресурсы на основе компетенций, которые определены для соответствующего работника.
-   **Машина** — машина или другое производственное оборудование, которое необходимо для производства.
-   **Инструмент** — инструмент или прибор, который обычно используются вместе с другим ресурсом для выполнения деятельности в проекте или на производстве.
-   **Местонахождение** — фактическое месторасположение специфического размера, которое необходимо для выполнения деятельности. Пример — зона сборки.
-   **Средство** — здание или фиксированная структура, которая необходима для выполнения деятельности.

## <a name="calendars"></a>Календари
Календарь может быть назначен операционному ресурсу и описывает емкость (в часах) этого ресурса. Значения рабочего времени операционного ресурса определяются календарем, который назначен группе ресурсов, в которую входит операционный ресурс. Вы можете задать дату начала действия и дату окончания срока действия для назначенного календаря. Затем можно назначить несколько календарей для операционного ресурса. Однако даты назначенных календарей не могут перекрываться, и операционному ресурсу можно назначить только один календарь одновременно. **Примечание.** Если для группы ресурсов отсутствуют действующие рабочие календари (например если истек срок действия последнего или единственного назначенного календаря), доступ к операционным ресурсам, назначенным группе ресурсов для планирования производства и операционной деятельности, закрыт. Вы можете также назначить календарь группе ресурсов, чтобы определить значения рабочего времени как для группы ресурсов, так и для всех операционных ресурсов, которые заданы к группе ресурсов. Емкость группы ресурсов рассчитывается как сумма емкостей каждого операционного ресурса, который задан в этой группе ресурсов. Календарь, назначенный операционной группе, можно изменять с течением времени.

## <a name="resource-capabilities"></a>Возможности ресурса
Возможность — это способность операционного ресурса участвовать в определенном мероприятии. Можно назначить несколько возможностей для операционного ресурса. Механизм планирования затем выделяет ресурсы, сопоставляя требования каждого мероприятия к ресурсам с возможностями доступных операционных ресурсов. Возможности можно задавать для всех типов операционных ресурсов (**Инструмент**, **Поставщик**, **Машина**, **Человеческие ресурсы**, **Местонахождение** или **Средство**). Чтобы назначить возможности операционным ресурсам на временной основе, задайте дату начала и дату окончания срока действия назначения возможности. Дополнительные сведения см. в разделе [Возможности ресурса](resource-capabilities.md).

## <a name="resource-groups"></a>Группы ресурсов
Группа ресурсов представляет собой набор операционных ресурсов, который представляет степень детализации, с которой вы хотите планировать ресурсы при использовании метода планирования операций. Таким образом, группы ресурсов обычно соответствуют физической организации производственных ячеек, обозначенных желтыми линиями в цехе производства. Группа ресурсов определяет сайт, производственное подразделение и контекст склада для операционных ресурсов, назначенных группе. При назначении операционного ресурса группе ресурсов этот ресурс можно запланировать на сайте, на котором находится группа ресурсов. Назначать группу ресурсов создаваемым операционным ресурсам необязательно. Однако операционному ресурсу должна быть назначена группа ресурсов, чтобы его можно было запланировать для выполнения работы. Операционный ресурс может быть назначен группе ресурсов в течение ограниченного времени. Вы можете также назначить операционному ресурсу несколько групп ресурсов, чтобы можно было совместно использовать этот ресурс на нескольких сайтах. Однако даты вступления в силу и даты истечения срока действия не могут перекрываться. Другими словами, невозможно назначить операционный ресурс для двух различных групп ресурсов одновременно. Изменения в назначениях групп ресурсов действуют только для новых распределений ресурсов. Ужа запланированные резервирования мощности для прогнозируемых затрат времени для заданий, операций и проектов не затрагиваются. **Примечание.** Когда вы задаете операционные ресурсы типа **Поставщик** группе ресурсов, все операционные ресурсы, назначенные этой группе ресурсов, должны быть типа **Поставщик** и должны быть связаны с одним и тем же счетом поставщика.

## <a name="production-units"></a>Производственные подразделения
Производственное подразделение - это административное подразделение, которое является коллекцией групп ресурсов. Производственное подразделение может содержать несколько групп ресурсов, но группа ресурсов может быть назначена только одному производственному подразделению. Производственное подразделение отражает физическое распределение производственных ресурсов и не влияет на проводки или способ их обработки. Производственное подразделение необходимо сопоставить с площадкой. Для производственного подразделения можно назначить склад комплектации и склад хранения. Можно использовать производственное подразделение для консолидации и фильтрации данных по производству. Например, менеджер цеха может просматривать наглядный обзор неиспользуемой рабочей нагрузки и имеющихся мощностей для конкретного производственного подразделения. Можно изменить производственное подразделение, назначенное группе ресурсов. Можно также удалить производственное подразделение. Однако эти изменения производственного подразделения вступают в силу только для новых заказов, созданных после проведения сводного планирования. Если вы хотите применить изменение производственного подразделения к существующим заказам, следует сделать это изменение вручную.

## <a name="resource-scheduling"></a>Планирование ресурсов
Операционные ресурсы назначаются действиям при планировании проекта или производства. Предусмотрены два метода планирования: планирование операций и планирование заданий. Когда вы используете планирование операций, каждая операция или действие проекта планируется для группы ресурсов, которая содержит операционные ресурсы, соответствующие потребностям в ресурсах, которые определены для операции. Если определенный операционный ресурс необходим для операции, планирование резервирует модности только определенного операционного ресурса в группе. Планирование заданий является более подробным способом планирования по сравнению с планированием операций. Каждая операция разбивается на отдельные задачи или задания. Затем эти задания назначаются операционным ресурсам, которые будут их выполнять. При планировании резервируется мощность в группах ресурсов на основе времени операций, заданных в маршруте производства или действиях проекта. Если вы работаете с конечной мощностью, график зависит от доступности операционных ресурсов, необходимых для выполнения действия. Для планирования операций мощность группы ресурсов равна сумме доступных мощностей операционных ресурсов, входящих в эту группу. Резервирования мощностей, которые уже существуют для операционных ресурсов, считаются недоступной мощностью. Если доступные мощности недостаточны для производства, производственные заказы могут быть задержаны или даже остановлены. На странице **Ресурсы** вы можете определить несколько свойств, которые контролируют порядок резервирования мощности:

-   **Мощность** — определите мощность операционного ресурса в час в единицах измерения мощности.
-   **Объем пакета серии** — определите максимальное число единиц, которые может обработать этот операционный ресурс за один прогон.
-   **Процент эффективности** —укажите процент эффективности, ожидаемый от операционного ресурса. Процент эффективности корректирует пропускную способность операционного ресурса и влияет на время, зарезервированное для ресурса. Время упреждения операций, использующих операционный ресурс, также изменяется соответствующим образом. Следующая формула используется для вычисления: Время планирования = Время × 100 ÷ Процент эффективности В этой формуле *Время* включает в себя время выполнения и время настройки.
-   **Процент планирования операций** — укажите максимальный процент мощности операционного ресурса, который требуется использовать при планировании операций. Для обеспечения гибкости при планировании заданий следует задать значение менее 100 процентов.
-   **Ограничение по мощности** — задайте для этого параметра значение **Да**, если операционный ресурс должен планироваться на основании фактической доступной мощности и если должны учитываться существующие резервирования мощностей. Если для этого параметра задать значение **Нет**, будет предполагаться, что операционный ресурс имеет неограниченную мощность и поэтому может быть зарезервирован с превышением.
-   **Ограничение по свойствам** — установите для этого параметра значение **Да**, если вы хотите, чтобы операционный ресурс планировался на действительной мощности, которая доступна с точки зрения свойств планирования необходимого рабочего времени.
-   **Исключающее** — установите для этого параметра значение **Да**, если операционный ресурс не должен быть доступен для другого задания или операции до тех пор, пока не будет завершено текущее производство. В таком случае операционный ресурс нельзя использовать, даже если существуют перерывы во времени работы ресурса.
-   **Минимальный ресурс** — определите операционный ресурс как минимальный ресурс. Минимальный ресурс планируется с использованием ограничения по мощности, когда выбраны поля **Ограничение по мощности** и **Планирование узких мест** на странице **Сводные планы**.

Чтобы запланировать несколько операционных ресурсов одновременно, для выполнения, например операции при производстве, вы должны определить требования для различных ресурсов путем использование первичных и вторичных операций. Вы можете после этого также зарезервировать несколько операционных ресурсов, имеющих различную мощность. Операционный ресурс, который запланирован для основной операции, определяет продолжительность действия.

## <a name="lean-work-cells"></a>Экономичные производственные ячейки
Вы можете определить, что группа ресурсов является экономичной производственной ячейкой. Группа ресурсов может после этого быть частью производственного потока. Указав группу ресурсов как экономичную производственную ячейку, вы также предотвращаете выделение группу ресурсов и назначенных операционных ресурсов операциям производственного заказа или прогноза часов проекта. Для каждой группы ресурсов, которая действует как экономичная производственная ячейка, вы должны определить следующую информацию:

-   **Календарь** — рабочий календарь производственной ячейки, которая должна иметь свойство стандартного рабочего дня.
-   **Склад и место получения** — местонахождение по умолчанию, которое используется для комплектации материала для действия.
-   **Склад и место выпуска** — местонахождение по умолчанию, куда помещаются все результаты работы производственной ячейки.
-   **Категория затрат во время работы** — эту категорию необходимо определить для производственной ячейки, если прямые трудовые затраты отслеживаются в учете затрат.

Когда группа ресурсов используется как экономичная производственная ячейка, мощность производственной ячейки указывается непосредственно в группе ресурсов. Поэтому не требуется назначать операционные ресурсы группе ресурсов. Только когда производственная ячейка управляется субподрядчиком, операционный ресурс типа **Поставщик** должен быть назначен производственной ячейке. Если вы назначаете операционный ресурс группе ресурсов, который помечен как производственная ячейка, мощность производственной ячейки не затрагивается.

## <a name="costing-resources"></a>Ресурсы для расчета себестоимости
Когда вы определяете действие, такое как операция маршрута или часовой прогноз проекта, можно указать требование для определенного операционного ресурса или группы ресурсов. Однако вы можете также указать требование для операционного ресурса определенного типа или операционных ресурсов, которые имеют определенную мощность или компетенцию. По этой причине фактическое назначение ресурса не выполняется до тех пор, пока действие не будет запланирована и мощность не будет зарезервирована. Поэтому в операции маршрута можно указать оценку, а расчет спецификации должен быть основан на конкретном операционном ресурсе. Этот операционный ресурс называется ресурсом для расчета себестоимости. Вы можете также переносить категории цены и значения времени операции из ресурса для расчета себестоимости в действие. При планировании операции оценка и расчет спецификации выполняются с использованием фактически запланированного операционного ресурса.
