# <a name="demand-forecasting-overview"></a>Прогнозирование обзора спроса

Прогнозирование спроса используется для прогнозирования независимого спроса по заказам на продажу и зависимого спроса в любой точке разъединения для заказов клиента. Расширенные возможности спроса прогноза сокращения правила обеспечивают идеальное решение для массового настройки.

Чтобы создать базовый прогноз, сводка проводок за прошлые периоды передается сервису Microsoft Azure Machine Learning, размещенному в Azure. Поскольку этот сервис не находится в общем доступе пользователей, его легко настроить в соответствии с отраслевыми требованиями. Dynamics 365 для операций можно использовать для визуализации прогноза, настроить прогноз и просмотреть ключевые индикаторы производительности (KPI) о точности прогноза.

## <a name="key-features-of-demand-forecasting"></a>Ключевые функции прогнозирования спроса
Здесь представлены некоторые из основных функций прогнозирования спроса.

-   Создание статистического базового прогноза на основе данных за прошлые периоды.
-   Использование динамического набора аналитик прогноза.
-   Визуализация тенденций спроса, интервалов доверия и корректировок прогноза.
-   Авторизация скорректированного прогноза для использования в процессах планирования.
-   Удаление выбросов.
-   Создание измерений точности прогноза.

## <a name="major-themes-in-demand-forecasting"></a>Основные аспекты прогнозирования спроса
В прогнозировании спроса выделяют три основных аспекта.

-   **Модальность** — прогнозирование спроса отличается модальностью, его легко настроить. Можно включить или отключить функциональные возможности, изменив ключ конфигурации по **коммерческих**&gt;**Прогноз запасов**&gt;**прогнозирование спроса**.
-   **Повторное использование стек Microsoft** — Корпорация Майкрософт начала реализацию платформы машинного обучения в феврале 2015. Машинного обучения, который является частью пакета Microsoft Cortana аналитики, позволяет быстро и легко создавать упреждающего анализа экспериментов, например экспериментов оценка спроса, с помощью алгоритмов R или языков программирования Python и простой интерфейс и перетаскивания.
    -   Можно загрузить Dynamics 365 для операций спроса прогнозирование экспериментов, изменять их для соответствия требованиям бизнеса и опубликовать их в качестве веб-службы в Azure позволяют создавать прогнозы спроса. Если вы приобрели Dynamics 365 для операций подписки для планировщик производства в качестве пользователя на уровне предприятия, эксперименты доступны для загрузки.
    -   Можно загрузить любой из доступных в настоящее время экспериментов прогнозирования спроса в [Галерее аналитики Cortana](https://gallery.cortanaanalytics.com/). В то время как Dynamics 365 для прогнозирования экспериментов спроса операций автоматически интегрированы с Dynamics 365 для операций, клиентам и партнерам должна поддерживать интеграции экспериментов, которые они загрузить с [Cortana аналитики коллекции](https://gallery.cortanaanalytics.com/). Таким образом, использующей из [Cortana аналитики коллекции](https://gallery.cortanaanalytics.com/) не так просто для использования в качестве Dynamics 365 для операций спроса прогнозирование экспериментов. Необходимо изменить код эксперименты, чтобы они использовали Dynamics 365 для операций интерфейс программирования (API).
    -   В Microsoft Azure Machine Learning Studio можно создавать собственные эксперименты, публиковать их как сервисы в Azure и использовать для создания прогнозов спроса.
    -   Если высокая производительность не требуется или не требуется обработка крупных массивов данных, можно воспользоваться уровнем бесплатного пользования Machine Learning. Рекомендуется всегда начинать с этого уровня, особенно на этапах внедрения и тестирования. Если требуется высокая производительность и дополнительные ресурсы хранения, можно воспользоваться стандартным уровнем Machine Learning. Этот уровень требует подписки Azure и сопряжен с дополнительными затратами. Подробные сведения о ценах Machine Learning см. в разделе <http://aka.ms/machine-learning-price-info>.
-   **Прогноз сокращения в любом случае, отделение** — прогнозирования Dynamics 365 для построения операций по эта функция позволяет зависимый и независимый спрос в любом случае, отделение прогноза спроса.

## <a name="basic-flow-in-demand-forecasting"></a>Базовая процедура прогнозирования спроса
На следующей схеме показана базовая процедура прогнозирования спроса. 

[![Введение схеме спроса для прогнозирования](./media/demand-forecasting-introduction.png)](./media/demand-forecasting-introduction.png)

Начинается с создания в Dynamics 365 для операций прогноза спроса. Статистические данные о транзакциях из Dynamics 365 для операции транзакций базы данных собираются и заполняет промежуточную таблицу. В этой таблице промежуточной подается позже для обслуживания машинного обучения. Выполняя минимальной настройки, можно подключить различных источников данных в промежуточную таблицу. Источники данных могут включать файлы Microsoft Excel, файлы с разделителями-запятыми (CSV) и данные из Microsoft Dynamics AX 2009 и Microsoft Dynamics AX 2012. Таким образом можно создавать прогнозы спроса, которые следует учитывать статистические данные, которые распределяются между несколькими системами. Основные данные, такие как названия номенклатур и единицы измерения, должны совпадать в разных источниках данных.

При использовании Dynamics 365 для прогнозирования экспериментов машинного обучения спроса операций, они выполняют поиск наилучшего один из пяти прогнозирования методы время серии для расчета прогноза базового плана. Параметры для этих методов прогнозирования осуществляется в Dynamics 365 для операций. 

Любые изменения, которые были сделаны в прогнозы спроса в предыдущих итераций, прогнозы и статистические данные затем доступны в Dynamics 365 для операций. 

Dynamics 365 для операций можно использовать для визуализации и изменения базовых прогнозов. Ручные регулировки необходимо авторизовать, прежде чем прогнозы можно будет использовать для планирования.

## <a name="limitations"></a>Ограничения
Прогнозирование спроса в Dynamics 365 для операций является средством, которое помогает клиентам в промышленных Создание прогнозирования процессов. Он обеспечивает основные функциональные возможности спроса, прогнозированию решения и разработан таким образом, ее можно легко расширить. Требовать прогнозированию может не быть подобрать ширину для клиентов в отраслях, таких как розничной торговли, оптом, складские операции, Транспортировка и других профессиональных услуг.
