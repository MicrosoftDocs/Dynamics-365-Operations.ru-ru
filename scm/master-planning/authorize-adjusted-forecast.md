# <a name="authorize-an-adjusted-forecast"></a>Авторизовать уже скорректированных прогноза

Не все данные прогноза необходимо авторизовать немедленно. Эта статья объясняет, как задать период, для которого авторизован прогноз. Она также объясняет, как авторизовать прогноз для определенных компаний и моделей прогноза.

Не все данные прогноза необходимо авторизовать немедленно. Можно задать начальные и конечные даты периода, для которого авторизован прогноз. Эта функциональность позволяет блокировать конкретные периоды. 

Даты начала и окончания, задаваемых должно соответствовать даты начала и окончания сегмента, создаваемого в прогноз. Система применяет это ограничение и изменяется автоматически даты, если требуется корректировки. 

На вкладке **Сведения** страницы **Авторизация** можно просмотреть сведения о недавно созданном прогнозе. 

Можно выбрать компании и модели прогноза, чтобы авторизовать прогноз, который требуется использовать. По умолчанию сетка включает все компании, для которых был создан прогнозируемый спрос. Для каждой компании модель прогноза, которая соответствует текущему плану прогноза, который настроен в параметрах сводного планирования, заполняется предварительно. Однако можно изменить эту модель прогноза на любую модель прогноза, которая относится к этой компании. Если никакие данные прогнозируемого спроса не созданы для выбранной компании, во время импорта отображается предупреждение. 

Важно понимать принцип работы флажка **Сохранить ручные корректировки базового прогноза спроса**. Если сделанные настройки вручную статистических базового прогноза скорректированные значения авторизован для использования, даже если этот флажок снят. Однако изменения после авторизации сбрасываются. Следовательно, в следующий раз при создании прогноза этот прогноз является только статистическим и не имеет никаких переопределений вручную, даже если выбрано значение **Перенести ручные корректировки в прогноз спроса**. Поэтому можно рассматривать флажок **Сохранить ручные корректировки базового прогноза спроса** как механизм, который позволяет сохранить все внесенные вручную изменения или отказаться от них.
