# <a name="manage-the-electronic-reporting-configuration-lifecycle"></a>Управление жизненным циклом конфигураций электронной отчетности

В этом разделе описывается управление жизненного цикла электронной отчетности конфигурации (ER) для Microsoft Dynamics 365 для решения операций.

<a name="overview"></a>Обзор
--------

Электронная отчетность (ER) — это предусмотренный в Microsoft Dynamics 365 for Operations механизм для поддержки обязательных электронных документов, существующих в различных странах. Обычно ER подразумевает возможность выполнять следующие задачи для одного электронного документа. Дополнительные сведения см. в разделе [электронной отчетности Обзор](general-electronic-reporting.md).

-   Создание шаблона для электронного документа:
    -   Определение необходимых источников данных, которые могут быть представлены в документе:
        -   Нижележащие Dynamics 365 для данных, операций, таких как таблицы данных, объекты данных и классы.
        -   Свойства, зависящие от процесса, например дату выполнения и время и часовой пояс.
        -   Входные параметры пользователя, задаваемого конечным пользователем во время выполнения.
    -   Определение необходимых элементов документа и их топологии для задания формата окончательного документа.
    -   Конфигурирование желаемого потока данных из выбранных источников данных в заданные элементы документа (посредством привязки источников данных к компонентам формата документа) и задание логики управления процессом.
-   Обеспечение доступности шаблона, чтобы его можно было использовать в других экземплярах Dynamics 365 for Operations:
    -   Преобразование шаблона документа, созданного в Dynamics 365 for Operations, в конфигурацию ER и экспорт конфигурации из текущего экземпляра Dynamics 365 for Operations в виде XML-пакета, который можно сохранить локально или в LCS.
    -   Преобразование конфигурации ER в шаблон документа Dynamics 365 for Operations.
    -   Импорт XML-пакета, который хранится локально или в LCS, в текущий экземпляр Dynamics 365 for Operations.
-   Настройка шаблона электронного документа:
    -   импорт шаблона из LCS в текущий экземпляр Dynamics 365 for Operations в качестве конфигурации ER;
    -   конструирование пользовательской версии конфигурации ER с сохранением ссылки на ее базовую версию.
-   Интеграция шаблона с определенным бизнес-процессом, чтобы он был доступен в Dynamics 365 for Operations:
    -   Настройка параметров так, чтобы Dynamics 365 for Operations начал использовать конфигурацию ER, путем ссылки на эту конфигурацию в параметре, связанном с процессом. Например обратитесь к конфигурации ER в определенный метод подлежащих оплате счетов позволяет создать сообщение электронного платежа для обработки накладных.
-   Использование шаблона в конкретном бизнес-процессе:
    -   ER конфигурации запуска конкретного бизнес-процесса. Например позволяет создать сообщение электронного платежа для обработки накладных, когда метод оплаты, ссылается на конфигурацию ER установлен.

## <a name="concepts"></a>Концепции
Следующие роли и связанные действия связаны с жизненный цикл настройки аварийного восстановления.

| Роль                                       | Мероприятия                                                      | описание                                                                                                                                                                                                                  |
|--------------------------------------------|-----------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Консультант по функциональным возможностям электронной отчетности | Создание компонентов ER (модели и форматы) и управление ими.           | Business пользователь разрабатывает модели данных, заданной для определенного домена ER, макеты требуемые шаблоны для электронных документов и связывает их соответствующим образом.                                                                           |
| Разработчик электронной отчетности             | Создание сопоставлений моделей данных и управление ими.                          | Dynamics 365 для специалиста по операциям, выбирает требуется Dynamics 365 для источников данных, операций и привязывает их к модели данных, заданной для определенного домена аварийного восстановления.                                                                 |
| Супервизор по учету                      | Задание настроек, связанных с процессом, которые ссылаются на артефакты ER. | Например **учет супервизор** роль, позволяющая параметры конфигурации ER для использования в конкретный метод подлежащих оплате счетов позволяет создать сообщение электронного платежа для обработки накладных. |
| Сотрудник, обрабатывающий платежи по расчетам с поставщиками            | Использование артефактов ER в конкретном бизнес-процессе.                | Например **сотрудник отдела расчетов с поставщиками платежей** роль, позволяющая электронных платежей сообщений для обработки накладных, в зависимости от формата ER, настроенного на специальный метод оплаты.           |

## <a name="er-configuration-development-lifecycle"></a>Жизненный цикл разработки конфигурации ER
По следующим связанным с ER причинам рекомендуется разрабатывать конфигурации ER в среде разработки — отдельном экземпляре Dynamics 365 for Operations:

-   Пользователи, которые выполняют роли **Разработчик электронной отчетности** или **Консультант по функциональным возможностям электронной отчетности**, могут редактировать конфигурации и запускать их в целях тестирования. Этот сценарий может привести к вызовам методов классов и таблиц, которые могут повредить бизнес-данные и негативно сказаться на работе экземпляра Dynamics 365 for Operations.
-   Вызовы методов классов и таблиц как источников данных ER конфигураций ER не ограничиваются точками входа Dynamics 365 for Operations и содержанием, принадлежащим вошедшей в систему компании. Это значит, что пользователи, которые играют роли **Разработчик электронной отчетности** или **Консультант по функциональным возможностям электронной отчетности**, могут получить доступ к конфиденциальным бизнес-данным.

Тестовая среда для оценки конфигурации (интеграции с нужными процессами, правильность результатов и максимальной производительности) и качества, например, правильность права доступа на основе ролей и разделения обязанностей можно загрузить ER конфигураций, разработанных в среде разработки. Функции обеспечения обмена настройки аварийного восстановления можно использовать для этой цели. Наконец имеющийся ER конфигурации могут быть отправлены в LCS, где они могут использоваться совместно с подписчиками службы, или в производственной среде для внутреннего использования, как показано на следующем рисунке. ![Жизненный цикл настройки аварийного восстановления](./media/ger-configuration-lifecycle.png)
