# <a name="cost-accounting-analysis-power-bi-content"></a>Аналитики учета затрат содержимое питания бизнес-Аналитики

В этой теме описывается содержание анализа содержимого питания бизнес-Аналитики учета затрат. Объясняется, как получить доступ к отчетам бизнес-Аналитики питания и содержит сведения о модели данных и объекты, которые использовались для создания содержимого.

<a name="overview"></a>Обзор
--------

**Аналитики учета затрат** Microsoft Power BI содержимое предназначено для контроллеров затрат или любому, кто несет ответственность за выполнение управления затратами организации. В нем содержатся ключевые параметры, такие как затраты, величина и ставки затрат, фактические затраты, бюджетные затраты и затрат в гибкий бюджет. Она использует данные проводки учета затрат в Microsoft Dynamics 365 для операций и предоставляет статистического представления затрат в рамках всей организации в одной валюте отчетности. Менеджеры данные можно фильтровать объектами затрат для проведения контроля затрат их организационных подразделений, даже если организация может иметь несколько юр. Так как **аналитики учета затрат** содержимое Power BI выделяет расхождения между фактическими затратами и бюджетных затрат, можно получать уведомления о тенденции положительные и отрицательные менеджеры их работоспособными блоков. Менеджеры может перейти к иерархии элемент затрат или затрат отдельные элементы можно получить подробные ответы как возникли отклонений по стоимости, а затем будет эффективные действия. **Аналитики учета затрат** Power BI содержимого проанализируем бухгалтерами затрат как затраты проходит через объекты затрат по всей организации. Дополнительные сведения об учете затрат см. в разделе [учет затрат Домашняя страница](/dynamics365/operations/financials/cost-accounting/cost-accounting-home-page.md). Определив безопасности уровень доступа в учете затрат и его объединения с безопасности уровня строк в питания бизнес-Аналитики, можно предоставить доступ к всех владельцев объектов затрат **аналитики учета затрат** Power BI содержимого. Все данные в визуализации затем фильтруются на основе на уровень доступа, который управляется в учете затрат. Для получения дополнительных сведений о уровень доступа и безопасность на уровне строк, см. в [настройке безопасности для содержимого учета затрат для бизнес-Аналитики питания](setup-security-cost-accounting-content-pack.md).

## <a name="understanding-the-data-model-and-entities"></a>Понимание модели данных и объектов
Dynamics 365 для данных операций используется для заполнения страниц отчета **аналитики учета затрат** Power BI содержимого. Эти данные представлены как общий измерений, которые помещаются на временное хранение в хранилище объекта — базе данных Microsoft SQL, оптимизированная для аналитики. Дополнительные сведения см. в разделе [Обзор Power BI интеграции с магазином объекта](power-bi-integration-entity-store.md). Следующие статистические измерениям ключа используются в качестве основы содержимого.

| Объект                  | Ключевые совокупного измерения | Источник данных для Dynamics 365 для операций | Поле     | описание                                   |
|-------------------------|---------------------------|---------------------------------------------|-----------|-----------------------------------------------|
| Операции учета затрат | SUM(Amount)               | CAMDATAAggregatedCostEntry                  | Сумма, руб.    | Сумма в валюте ГК учета затрат |
| Статистические записи     | SUM(Magnitude)            | CAMDATAAggregatedStatisctialEntry           | Величина |                                               |

Следующая таблица показывает, как ключевых статистических измерений используются для создания нескольких вычисляемых мер в наборе данных содержимого.

| Мера                                       | Вычисление меры                                                                                          |
|-----------------------------------------------|------------------------------------------------------------------------------------------------------------------------|
| Фактические затраты                                   | РАССЧИТАТЬ ("записи учета затрат"\[измерения\], "Проводки версии"\[ISSOURCEVERSIONBUDGET\_значение\] = 0)            |
| Бюджетные затраты                                   | РАССЧИТАТЬ ("записи учета затрат"\[измерения\], "Проводки версии"\[ISSOURCEVERSIONBUDGET\_значение\] = 1)            |
| Отклонение бюджета затрат                          | \[Бюджетирование затрат\] - \[фактическая стоимость\]                                                                                      |
| Процент расхождения бюджета                    | IF (\[бюджетные затраты\] = 0, blank(), \[бюджетное отклонение\] / \[бюджетные затраты\])                                                |
| Фактическая величина                              | РАССЧИТАТЬ ("Статистических записи"\[FullMagnitude\], "Проводки версии"\[ISSOURCEVERSIONBUDGET\_значение\] = 0)          |
| Величина бюджета                              | РАССЧИТАТЬ (\[FullMagnitude\], «Проводки версии»\[ISSOURCEVERSIONBUDGET\_значение\] = 1)                               |
| Отклонение статистических бюджета                   | \[Бюджет величины\] - \[факт величины\]                                                                            |
| Процент расхождения статистических бюджета        | IF (\[бюджет величины\] = 0, blank(), \[статистических бюджетное отклонение\] / \[бюджет величины\])                          |
| Фактическая ставка затрат                              | IF (\[фактическая величина\] = 0, BLANK(), \[фактическая стоимость\] / \[фактическая величина\])                                          |
| Бюджетная ставка затрат                              | IF (\[бюджет величины\] = 0, BLANK(), \[бюджетные затраты\] / \[бюджет величины\])                                          |
| Расхождение курса бюджетных затрат                     | \[Ставка бюджетные затраты\] - \[факт ставки затрат\]                                                                            |
| Процент отклонения ставка бюджетные затраты          | IF (\[бюджетные затраты\] = 0, blank(), \[бюджетные затраты Расхождение курса\] / \[ставка бюджетные затраты\])                                 |
| Фиксированный бюджет затрат                             | РАССЧИТАТЬ (\[бюджетные затраты\], «Операции учета затрат»\[COSTBEHAVIOR\] = 1)                                              |
| Переменные бюджетные затраты                          | РАССЧИТАТЬ (\[бюджетные затраты\], «Операции учета затрат»\[COSTBEHAVIOR\] = 2)                                              |
| Основные гибкий бюджет затрат                    | \[Фиксированный бюджет затрат\]                                                                                                  |
| Переменные гибкий бюджет затрат                 | IF (\[бюджет величины\] = 0, BLANK(), (\[переменных бюджетные затраты\] / \[бюджет величины\]) \*\[фактическая величина\])       |
| Гибкий бюджет затрат                          | \[Фиксированные гибкий бюджет\] + \[переменных гибкий бюджет затрат\]                                                     |
| Отклонение гибкого бюджета                      | \[Гибкий бюджет затрат\] - \[фактическая стоимость\]                                                                             |
| Процент расхождения гибкого бюджета           | IF (\[затрат в гибкий бюджет\] = 0, BLANK(), \[отклонение гибкий бюджет\] / \[затрат в гибкий бюджет\])                     |
| Ставка затрат гибкого бюджета                     | IF (\[фактическая величина\] = 0, BLANK(), \[затрат в гибкий бюджет\] / \[фактическая величина\])                                 |
| Расхождение курса гибкий бюджет затрат            | \[Ставка затрат в гибкий бюджет\] - \[факт ставки затрат\]                                                                   |
| Процент отклонения ставки затрат гибкого бюджета | IF (\[ставки затрат в гибкий бюджет\] = 0, BLANK(), \[курс отклонение по стоимости гибкий бюджет\] / \[ставки затрат в гибкий бюджет\]) |
