# <a name="cost-management-power-bi-content"></a>Стоимость управления контентом Power BI

В этой теме описывается содержание управление стоимостью Power BI содержимого. Объясняется, как получить доступ к отчетам бизнес-Аналитики питания и содержит сведения о модели данных и объекты, которые используются для построения содержимого.

# <a name="overview"></a>Обзор

**Стоимость управления** бизнес-Аналитики Майкрософт для Power содержимое предназначено для запасов бухгалтерами или лицам в организации, ответственных за запасов. **Стоимость управления** содержимое Power BI предоставляет менеджера представление о складских и запасов работы НЗП (НЗП), и как затрат проходит через их по категориям со временем. Информация может использоваться также в подробных дополнение к финансовой отчетности.

## <a name="key-measures"></a>Ключевых показателя

+ Начальное сальдо
+ Конечное сальдо
+ Чистое изменение
+ Оборот в %
+ Распределение по срокам

## <a name="key-performance-indicators"></a>Ключевые показатели эффективности
+ Оборот запасов
+ Точность запасов

Первичный источник данных для CostAggregatedCostStatementEntryEntity является таблицей CostStatementCache. В этой таблице можно управлять структурой набора данных кэш-памяти. По умолчанию таблица обновляется каждые 24 часа, но имеется возможность ручного обновления в конфигурации кэша данных. Затем можно выполнить обновление вручную **стоимость управления** или **анализ затрат** рабочей области. После запуска обновление CostStatementCache, необходимо обновить подключение OData на питание BI.com увидеть обновленные данные на узел. Расхождение (Покупка, производство) мер в данный материал Power BI относятся только элементы, которые являются оценки методом стандартной себестоимости запасов. Отклонение цены производства от рассчитывается как разница между активных затрат и реализованные затраты. Расхождение производственного будет рассчитана, если производственный заказ имеет статус **завершено**. Дополнительные сведения о типы производственных отклонений и рассчитывается как каждого типа см. [об анализе расхождений в выполненном производственном заказе](https://technet.microsoft.com/en-us/library/gg242850.aspx).

## <a name="accessing-the-power-bi-content"></a>Доступ к содержимому питания бизнес-Аналитики
**Стоимость управления** Power BI содержимое доступно из PowerBI.com. Дополнительные сведения о том, как для подключения и загрузки данных Microsoft Dynamics 365 для данных операций см. в разделе [содержимого Power BI доступ из PowerBI.com](power-bi-home-page.md).

## <a name="understanding-the-data-model-and-entities"></a>Понимание модели данных и объектов
Dynamics 365 для данных операций используется для заполнения страниц отчета в **стоимость управления** Power BI содержимого. Эти данные представлены как общий измерений, которые помещаются на временное хранение в хранилище объекта — базе данных Microsoft SQL, оптимизированная для аналитики. Дополнительные сведения см. в разделе [Обзор Power BI интеграции с магазином объекта](power-bi-integration-entity-store.md). Следующие статистические измерениям ключа используются в качестве основы содержимого.
