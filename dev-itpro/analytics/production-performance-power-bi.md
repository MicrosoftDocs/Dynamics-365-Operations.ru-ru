# <a name="production-performance-power-bi-content"></a>Содержимое Power BI "Эффективность производства"

[!include[banner](../includes/banner.md)]

В этой теме описывается, что входит в содержимое Microsoft Power BI **Эффективность производства**. В нем описывается порядок доступа к отчетам Power BI и предоставляется информация о модели данных и объектах, которые использовались для построения пакета содержимого.

## <a name="overview"></a>Обзор

Содержимое Power BI **Эффективность производства** предназначено для руководителей производства или других специалистов в организации, ответственных за контроль производства.

Входящие в него отчеты позволяют использовать Power BI для мониторинга эффективности производственных операций в отношении своевременности выполнения, качества и затрат. В отчетах используются данные проводок из производственных заказов и заказов партий для формирования как агрегированного представления производственных показателей по всей компании, так и разбивки показателей по продукту и ресурсу.

Содержимое Power BI позволяет понять, способна ли организация осуществлять производство вовремя и в полном объеме. Будущая эффективность прогнозируется на основе производственных планов. Подробные отчеты позволяют получить глубокое представление о браке продуктов, вызванном производством, а также о проценте брака по ресурсам и по операциям.

Это содержимое Power BI также дает возможность анализировать отклонения производственных затрат. Отклонение цены производства от себестоимости рассчитывается как разница между оценочной стоимостью и реализованной стоимостью. Отклонения цены производства от себестоимости рассчитываются при достижении производственными заказами или партионными заказами состояния **Завершено**.

Содержимое Power BI **Эффективность производства** включает данные, которые поступают из производственных заказов и заказов партий. Данные, связанные с производством по системе "канбан", в отчеты не входят.

## <a name="accessing-the-power-bi-content"></a>Доступ к содержимому Power BI
При использовании Microsoft Dynamics 365 for Finance and Operations, Enterprise edition, обновление от июля 2017 г. содержимое Power BI **Эффективность производства** отображается на странице **Эффективность производства** (**Управление производством** > **Запросы и отчеты** > **Анализ эффективности производства** > **Эффективность производства**).

## <a name="extending-the-power-bi-content"></a>Расширение содержимого Power BI
С помощью пакетов содержимого, доступных в Microsoft Dynamics Lifecycle Services (LCS), можно повысить эффективность аналитики для тех, кто не использует Microsoft Dynamics 365. Эти пакеты содержимого можно изменить, чтобы они включали других отчеты или визуальные элементы, и затем опубликовать пакеты содержимого для владельца Power BI.com для анализа.

Содержимое Power BI **Эффективность производства** можно найти в библиотеке общих ресурсов в LCS. Дополнительные сведения о том, как загрузить содержимое и реализовать его в вашей организации, см. в разделе [Содержимое Power BI в LCS от Майкрософт и ваших партнеров](power-bi-content-microsoft-partners.md). Чтобы просмотреть демонстрацию, которая показывает, как реализовать содержимое Power BI, см. [Содержимое Power BI от Майкрософт и ваших партнеров в службах Dynamics Lifecycle Services](https://mix.office.com/watch/9puyb1b2xs1w) (Office mix).

Следите за тем, чтобы загружаемое содержимое Power BI **Эффективность производства** соответствовало версии Dynamics 365, которую вы используете.

> [!NOTE]
> При использовании Microsoft Dynamics 365 for Finance and Operations версии 1611 необходимым условием для использования этого содержимого Power BI является реализация KB 4011327. После входа в LCS этот KB можно найти здесь: https://fix.lcs.dynamics.com/issue/results/?q=kb4011327.

## <a name="understanding-the-data-model-and-entities"></a>Понимание модели данных и объектов

Следующие данные используются для заполнения страниц отчета в содержимом Power BI **Эффективность производства**. Эти данные представлены как общие измерения, которые помещаются на временное хранение в хранилище объектов. Хранилище объектов является базой данных Microsoft SQL Server, оптимизированной для аналитики. Дополнительные сведения о хранилище объектов см. в разделе [Интеграция Power BI с хранилищем объектов](power-bi-integration-entity-store.md).

В следующей таблице приведены ключевые агрегированные измерения, используемые в качестве основы для содержимого Power BI.

|--------------------------|-----------------------------|----------------------------------------|--------------------|
| CostCalculation          | CostAmount                  | ProdCalcTransExpanded                  | CostAmount         |
| CostCalculation          | CostMarkup                  | ProdCalcTransExpanded                  | CostMarkup         |
| CostCalculation          | ActualCostAmount            | ProdCalcTransExpanded                  | RealCostAmount     |
| CostCalculation          | RealCostAdjustment          | ProdCalcTransExpanded                  | RealCostAdjustment |
| RouteTransactions        | ErrorQuantity               | ProdRouteTransExpanded                 | QtyError           |
| RouteTransactions        | GoodQuantity                | ProdRouteTransExpanded                 | QtyGood            |
| ProductionOrder          | DaysDelayed                 | ProdTableExpanded                      | DaysDelayed        |
| ProductionOrder          | LeadTime                    | ProdTableExpanded                      | LeadTime           |
| ProductionOrder          | PlannedLeadTime             | ProdTableExpanded                      | PlannedLeadTime    |

В следующей таблице показано, как ключевые агрегированные измерения используются для создания нескольких вычисляемых мер в наборе данных содержимого.

| Мера                  | Как вычисляется мера |
|--------------------------|-------------------------------|
| Отклонение цены производства от себестоимости, %   | SUM('Отклонение цены производства от себестоимости'[Отклонение цены производства от себестоимости]) / SUM('Отклонение цены производства от себестоимости'[Оценочная стоимость]) |
| Все спланированные заказы       | COUNTROWS('Спланированный производственный заказ') |
| С опережением                    | COUNTROWS(FILTER('Спланированный производственный заказ', 'Спланированный производственный заказ'[Запланированная дата окончания] \< 'Спланированный производственный заказ'[Дата потребности])) |
| С опозданием                     | COUNTROWS(FILTER('Спланированный производственный заказ', 'Спланированный производственный заказ'[Запланированная дата окончания] \> 'Спланированный производственный заказ'[Дата потребности])) |
| Вовремя                  | COUNTROWS(FILTER('Спланированный производственный заказ', 'Спланированный производственный заказ'[Запланированная дата окончания] = 'Спланированный производственный заказ'[Дата потребности])) |
| % вовремя                | IF ( 'Спланированный производственный заказ'[Вовремя] \<\> 0, 'Спланированный производственный заказ'[Вовремя], IF ('Спланированный производственный заказ'[Все спланированные заказы] \<\> 0, 0, BLANK()) ) / 'Спланированный производственный заказ'[Все спланированные заказы] |
| Зaвершенo                | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Принят] = TRUE)) |
| Процент брака (миллионных долей)     | IF( 'Производственный заказ'[Общее количество] = 0, BLANK(), (SUM('Производственный заказ'[Количество дефектных]) / 'Производственный заказ'[Общее количество]) \* 1000000) |
| Процент продукции, произведенной с опозданием  | 'Производственный заказ'[\# произведенных с опозданием] / 'Производственный заказ'[Завершено] |
| С опережением и в полном объеме          | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен в полном объеме] = TRUE && Производственный заказ'[Выполнен с опережением] = TRUE)) |
| \# с опережением                 | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен с опережением] = TRUE)) |
| % с опережением                  | IFERROR( IF('Производственный заказ'[\# с опережением] \<\> 0, 'Производственный заказ'[\# с опережением], IF('Производственный заказ'[Всего заказов] = 0, BLANK(), 0)) / 'Производственный заказ'[Всего заказов], BLANK()) |
| Не завершено               | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен в полном объеме] = FALSE && 'Производственный заказ'[Принят] = TRUE)) |
| % не в полном объеме             | IFERROR( IF('Производственный заказ'[Не завершено] \<\> 0, 'Производственный заказ'[Не завершено], IF('Производственный заказ'[Всего заказов] = 0, BLANK(), 0)) / 'Производственный заказ'[Всего заказов], BLANK()) |
| Выполнен с опозданием               | 'Производственный заказ'[Принят] = TRUE && 'Производственный заказ'[Значение задержки] = 1 |
| Выполнен с опережением                 | 'Производственный заказ'[Принят] = TRUE && 'Производственный заказ'[Дней задержки] \< 0 |
| Выполнен в полном объеме               | 'Производственный заказ'[Годное количество] \>= 'Производственный заказ'[Запланированное количество] |
| Принят                | 'Производственный заказ'[Значение производственного статуса] = 5 \|\| 'Производственный заказ'[Значение производственного статуса] = 7 |
| С опозданием и в полном объеме           | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен в полном объеме] = TRUE && Производственный заказ'[Выполнен с опозданием] = TRUE)) |
| \# с опозданием                  | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен с опозданием] = TRUE)) |
| % с опозданием                   | IFERROR( IF('Производственный заказ'[\# с опозданием] \<\> 0, 'Производственный заказ'[\# с опозданием], IF('Производственный заказ'[Всего заказов] = 0, BLANK(), 0)) / 'Производственный заказ'[Всего заказов], BLANK()) |
| Вовремя и в полном объеме        | COUNTROWS(FILTER('Производственный заказ', 'Производственный заказ'[Выполнен в полном объеме] = TRUE && 'Производственный заказ'[Выполнен с опозданием] = FALSE && 'Производственный заказ'[Выполнен с опережением] = FALSE)) |
| % вовремя и в полном объеме      | IFERROR( IF('Производственный заказ'[Вовремя и в полном объеме] \<\> 0, 'Производственный заказ'[Вовремя и в полном объеме], IF('Производственный заказ'[Завершено] = 0, BLANK(), 0)) / 'Производственный заказ'[Завершено], BLANK()) |
| Всего заказов             | COUNTROWS('Производственный заказ') |
| Общее количество           | SUM('Производственный заказ'[Годное количество]) + SUM('Производственный заказ'[Количество дефектных]) |
| Процент брака (миллионных долей)        | IF( 'Маршрутные проводки'[Обработанное количество] = 0, BLANK(), (SUM('Маршрутные проводки'[Количество дефектных]) / 'Маршрутные проводки'[Обработанное количество]) \* 1000000) |
| Процент брака смешанный (миллионных долей) | IF( 'Маршрутные проводки'[Общее смешанное количество] = 0, BLANK(), (SUM('Маршрутные проводки'[Количество дефектных]) / 'Маршрутные проводки'[Общее смешанное количество]) \* 1000000) |
| Обработанное количество       | SUM('Маршрутные проводки'[Годное количество]) + SUM('Маршрутные проводки'[Количество дефектных]) |
| Общее смешанное количество     | SUM('Производственный заказ'[Годное количество]) + SUM('Маршрутные проводки'[Количество дефектных]) |
