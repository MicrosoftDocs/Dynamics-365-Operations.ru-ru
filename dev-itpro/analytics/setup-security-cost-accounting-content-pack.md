# <a name="set-up-security-for-the-cost-accounting-analysis-power-bi-content"></a>Настройка безопасности для анализа содержимого питания бизнес-Аналитики учета затрат

В этом разделе объясняется, как можно распространить уровень доступа безопасности для защиты на уровне строк в Microsoft Power BI при учете затрат. Данная функция позволяет гарантировать, что пользователи видят только данные бизнес-Аналитики Опытные пользователи получают доступ к.

<a name="overview"></a>Обзор
--------

**Аналитики учета затрат** содержимое Microsoft Power BI использует безопасность на уровне строк питания бизнес-Аналитики для ограничения прав доступа пользователя. Безопасность основана на организационной иерархией уровень доступа, настроенный в параметрах учета затрат. Дополнительные сведения о **аналитики учета затрат** содержимого, Power BI см. [аналитики учета затрат Power BI содержимого](cost-accounting-analysis-content-pack.md).

## <a name="setup"></a>Настройка
Чтобы распространить уровень доступа безопасности питания бизнес-аналитики, владельца содержимого Power BI, выполните следующие действия. **Примечание:** пользователя, который публикует **аналитики учета затрат** содержимое Power BI, автоматически становится владельцем. Только владелец может настройки безопасности в питания бизнес-Аналитики. Кроме того, пока владелец добавляет другим пользователям на PowerBI.com, никто, кроме владельца можно просматривать данные в **аналитики учета затрат** Power BI содержимого.

1.  Публикация файла определения питания бизнес-аналитики.
2.  Выполните вход в PowerBI.com.
3.  Найти набор данных для **аналитики учета затрат** Power BI содержимого.
4.  Откройте страницу безопасности. 

    [![Открыв страницу «безопасность»](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-1.png)](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-1.png)

5.  **Стоимость объекта контроллера** роль уже создана. Добавление членов, являющихся частью организационной иерархии уровень доступа Учет затрат. 

    [![Добавление элементов](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-2.png)](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-2.png)

Пользователи, которые будут добавлены к **стоимость объекта контроллера** роль будет отображаться только те данные, они могут увидеть, в соответствии с определениями в организационной иерархии уровень доступа учета затрат. **Примечание:** безопасность на уровне строк применяется к фрагментов и отчеты в Microsoft Dynamics 365 для операций, которые встроены из питания бизнес-Аналитики.

## <a name="updating-security"></a>Обновление безопасности
Если вносить обновления безопасности уровень доступа в учете затрат, и вам требуется питания бизнес-Аналитики, чтобы отразить эти обновления, необходимо обновить хранилище объекта для **аналитики учета затрат** Power BI содержимого. После завершения обновления хранилища объекта из Dynamics 365 для операций, необходимо обновить артефакты на PowerBI.com. Дополнительные сведения о том, как выполняется обновление хранилища объекта см. в разделе [обновление объекта хранения](power-bi-integration-entity-store.md#update-entity-store). Владелец **аналитики учета затрат** содержимое питания бизнес-Аналитики также необходимо выполнить обновление хранилища объекта, если новых пользователей, имеющих доступ к организационной иерархии. Кроме того, владелец необходимо добавить новых пользователей, чтобы **стоимость объекта контроллера** роли на PowerBI.com, поэтому для них применяется, безопасность на уровне строк.

## <a name="disabling-security"></a>Отключение безопасности
Предположим, что ваша организация может ограничить доступ к данным. Если для какой-либо причине, параметры безопасности будут отключены при выполнении учета затрат, владельца необходимо добавить пользователей к **бухгалтер, учитывающий затраты** роль в Power BI, вместо этого. При изменении безопасности из включенном состоянии отключена, рекомендуется удалять пользователей из **стоимость объекта контроллера** роли. И наоборот, при повторном включении безопасности. Пользователи могут принадлежать к обе роли. Совместный доступ является объединение обе роли. В частности **аналитики учета затрат** Power BI содержимого пользователям, имеющим совместный доступ не ограничен доступ к данным. Если для применения ограниченный доступ, пользователям должны быть назначены только к **стоимость объекта контроллера** роли. Эти обновления безопасности уровня строк вступают в силу немедленно. Соответствующих пользователей следует обновить обозреватель.
