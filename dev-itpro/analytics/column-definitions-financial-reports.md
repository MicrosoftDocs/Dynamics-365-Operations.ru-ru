# <a name="column-definitions-in-financial-reports"></a>Определения столбцов в финансовых отчетах

[!include[banner](../includes/banner.md)]


Эта статья содержит сведения об определениях столбцов. Определение столбца — это компонент отчета или строительный блок, который определяет содержимое столбцов в отчете. Как и определения строк, базовые определения столбцов можно использовать в нескольких отчетах.

<a name="create-and-modify-a-column-definition"></a>Создайте и измените определение столбца
-------------------------------------

Определение столбца может содержать от 2 до 255 столбцов.

### <a name="create-a-column-definition"></a>Создание определения столбца

1.  В конструкторе отчетов, в Области перехода, щелкните **Определения столбцов**.
2.  В меню **Файл** выберите команду **Создать**, а затем — **Определение столбца**.
3.  Добавьте содержимое определения столбца.

### <a name="open-a-column-definition"></a>Откройте определение столбца

1.  В конструкторе отчетов, в Области перехода, щелкните **Определения столбцов**.
2.  Дважды щелкните определение столбца, чтобы открыть его.

### <a name="add-a-column-to-a-column-definition"></a>Добавьте столбец к определению столбца

1.  В конструкторе отчетов щелкните **Определения столбцов**, и после этого раскройте определение столбца, которое надо изменить.
2.  Выберите столбец, где новый столбец должен быть вставлен.
3.  На меню **Правка** щелкните **Вставить столбец**. Новый столбец кажется слева от столбца, что вы выбрали.

### <a name="delete-a-column-from-a-column-definition"></a>Удалите столбец из определения столбца

1.  В конструкторе отчетов щелкните **Определения столбцов**, и после этого раскройте определение столбца, которое надо изменить.
2.  Выберите столбец, который требуется удалить.
3.  В меню **Правка** щелкните **Удалить столбец**.

## <a name="contents-of-a-column-definition"></a>Содержимое определения столбца
Определение столбца содержит следующую информацию:

-   Столбец описаний для определения строки
-   Столбцы количества, которые показывают данные из финансовых данных, лист Microsoft Excel, или вычисления, которые основаны на других данных в определении столбца
-   Столбцы форматирования
-   Столбцы атрибута

Эта информация появляется в следующих зонах в определении столбца:

-   Зона заголовков определения столбца содержит текст заголовка и форматирование, которое появляется в отчете. Заголовок может применяться к одному столбцу данных, может распространяться на множественные столбцы, или может применяться к столбцам на основании условий. Определение столбца может включать столько строк заголовка столбца, сколько требуется. **Примечание:** Заголовки столбца применяются к каждому столбцу данных в отчете. Заголовки отчета применяются ко всему отчету. Вы определяете заголовки отчета на вкладке **Заголовки и нижние колонтитулы** определения отчета.
-   Строки сведений столбцов — строки под строками заголовка в определении столбца. Строки сведений столбцов определяют информацию, которая включена в отчете. Следующая таблица перечисляет и описывает строки сведений столбцов.

    | Имя строки сведений столбца                                                | Описание                                                                                            |
    |-----------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
    | Тип столбца                                                           | (Требуется) Определите тип данных в столбце.                                                     |
    | Код книги/Атрибут категории                                          | Укажите данные финансовых данных для столбцов типов **FD** и **ATTR**.                       |
    | Покрытые периоды периода бюджетного года                                    | Укажите данные финансовых данных для столбцов типа **FD**.                                     |
    | Формула                                                               | Определите формулу вычисления для столбцов типа **CALC**.                                        |
    | Дополнительные пробелы ширины столбца перед форматом столбца перекрывают управление печатью | Укажите специальные варианты формата.                                                                        |
    | Ограничения столбца                                                   | Ограничение данных.                                                                                         |
    | Единица отчетности                                                        | Ограничьте столбец, так, что он покажет данные только для определенного единицы отчетности.                      |
    | Фильтр валюты отображения валюты                                      | Форматирование валюты.                                                                                       |
    | Фильтр аналитик                                                      | Определите фильтр для того, чтобы ограничить данные определенными единицами отчетности финансовых данных.                           |
    | Фильтр атрибута                                                      | Определите фильтр для того, чтобы ограничить финансовые данные.                                                       |
    | Дата начала Дата окончания                                                   | Ограничить финансовые данные определенными датами.                                                         |
    | Обоснование                                                         | Выравнивание текста описания, который определен в определении строки, по левому краю, центру или правому краю. |

## <a name="column-restrictions-in-a-column-definition"></a>Ограничения столбца в определении столбца
Вы можете использовать ограничения столбца для того, чтобы определить, как определение столбца использует данные или высчитывает информацию. Вы можете также ограничить столбец отчета определенной единицей или датами. **Примечание:** Код **Ограничения столбца** перекрывает любые конфликтующие параметры, назначенные в определении строки.

### <a name="column-restrictions-cell"></a>Ячейка ограничений столбца

Ячейка **Ограничения столбца** может включить коды, которые ограничивают или подавляют информацию, такую как форматирование, детали и суммы, для того столбца.

#### <a name="add-a-column-restriction-in-a-column-definition"></a>Добавить Ограничения столбца в определении столбца

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Ограничения столбца** для столбца, который ограничить.
3.  В диалоговом окне **Ограничения столбца** выберите одни или несколько кодов в списке, и после этого щелкните **ОК**.

Например, вы можете не хотеть подчеркивать суммы, которое получены вычислением процента.                                                                        |

| SD                      | Подавить детали, так, что только строки **TOT** и **CAL** (от определения строки) будут показаны в столбце.                                                                                                                                                                                                                              |
| УС                      | Ограничьте суммы в столбце **FD** суммами дебета.                                                                                                                                                                                                                                                                              |

| UPT                     | Ограничьте суммы в столбце, так, что только неразнесенные проводки будут включены, если эти проводки доступны. **Примечание:** Не все поставщики данных поддерживают неразнесенные проводки. Дополнительные сведения см. в [руководстве по интеграции данных](http://go.microsoft.com/fwlink/?LinkID=162565) для вашей системы Microsoft Dynamics ERP. |

### <a name="restrict-a-column-to-a-reporting-unit"></a>Ограничьте столбец единицей отчетности

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Единица отчетности** для столбца, который ограничить.
3.  В диалоговом окне **Выбор единицы отчетности**, в списке **Дерево отчетности**, выберите дерево.
4.  Разверните или сверните список единиц, выберите единицу отчетности, и после этого щелкните **ОК**.

## <a name="format-column-headers"></a>Форматирование Заголовков столбцов
Вы можете добавить, изменить или удалить заголовки, которые появляются вверху столбцов на отчете. Вы можете также настроить условные в диапазоне заголовки столбца, основанные на поле **Период** от определений столбца и поле **Базовый период** от определений отчета. Функция Базовый период помогает сохранить время, когда вы создаете отчеты о текущих прогнозах.

### <a name="create-and-manage-column-headers"></a>Создание и управление заголовками столбцов

Вы можете использовать диалоговое окно **Заголовок столбца**, чтобы добавить, изменить или удалить заголовки, которые появляются вверху столбцов на отчете. Следующая таблица описывает поля в диалоговом окне **Заголовок столбца**.

| Поле                 | Описание                                                                                                                                                                                                                                                                                                              |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Текст заголовка столбца    | Этот текст появляется в заголовке столбца. Вы можете напечатать текст прямо в этом поле, или щелкнуть **Вставить AutoText** для того, чтобы выбрать вариант который обновляет заголовок столбца каждый раз, когда отчет произведен. Для того, чтобы включить несколько кодов autotext, щелкайте **Вставить AutoText** снова, и после этого щелкайте другой код в списке. |
| Параметры формата        | Применяйте форматирование к заголовку столбца, например рамка или подчеркивание.                                                                                                                                                                                                                                                           |
| Распределение от Распределение до | Определите столбец или столбцы, к которым применяется текст заголовка.                                                                                                                                                                                                                                                            |
| Обоснование         | Определите, как текст заголовка столбца должен быть выровнен для столбца или диапазона столбцов, который указан в полях **Распределение от** и **Распределение до**.                                                                                                                                                               |

### <a name="create-a-column-header"></a>Создать заголовок столбца

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Выполнить двойной клик ячейки заголовка.
3.  В диалоговом окне **Заголовок столбца**, впишите текст заголовка столбца. Альтернативно, щелкните **Вставить AutoText**, и выберите вариант.
4.  В поле **Параметры формата** выберите формат для заголовка.
5.  В Поле **Распределение от** впишите букву столбца, на которой заголовок столбца должен начаться. В Поле **Распределение до** впишите букву столбца, на которой заголовок столбца должен кончиться.
6.  В разделе **Выравнивание** выберите, должен ли текст заголовка столбца быть выровнен по леву, центру или праву.
7.  Нажмите кнопку **OК**.

### <a name="add-a-column-header-row"></a>Добавьте строку заголовка столбца

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Выберите ячейку в строке заголовка.
3.  На меню **Правка** щелкните **Вставить строку**. Новая строка вставляется над строкой, выбранной на шаге 2. **Примечание.** Если имеется четыре или более заголовков отчета в отчете, заголовки будут перекрываться при экспорте отчета в лист Excel. Для того, чтобы осмотреть все заголовки на отчете, увеличьте верхнее поле в определении отчета.

### <a name="delete-a-column-header-row"></a>Удаление строку заголовка столбца

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В строке заголовка, выберите ячейку для удаления.
3.  На меню **Правка** щелкните **Удалить строку**.

### <a name="create-an-automatically-generated-header"></a>Создайте автоматически созданный заголовок

Конструктор отчетов может автоматически создавать заголовки столбцов, основанные на кодах autotext. Коды Autotext — переменные которые обновляются каждый раз, когда отчет создан. Любой заголовок столбца может включать эти коды для того, чтобы определить данные отчета, которые могут меняться, такие как даты или номера периода. Поэтому, вы можете использовать одно определение столбца для множественных определений отчетов, периодов времени и деревьев отчетности. Потому что коды autotext основываются на данных по данных календаря из строк подробностей определения столбца, они поддерживаются только для столбцов **CALC**, **FD** и **WKS**. Способ, которым код autotext появляется в ячейке заголовка столбца, влияет на то, как информация появляется в отчете. В диалоговом окне **Заголовок столбца** коды autotext появляются в смешанном регистре. Поэтому, текст появляется в смешанном регистре в отчете. Например, в стандартном календарном годе **@CalMonthLong** разрешает месяц **7** до **июля**. Если имя месяца должно быть в верхнем регистре (например **ИЮЛЬ**), то впишите код autotext символами в верхнем регистре в поле **Текст заголовка столбца**. Например, введите **@CALMONTHLONG**. Вы можете смешать коды и текст. Например, можно ввести следующий текст заголовка: **Период @FiscalPeriod-@FiscalYear от @StartDate до @EndDate**. Заголовок отчета, который создан, напоминает следующий текст: **Период 1-02 от 01/01/02 до 01/31/02**. **Примечание.** Формат некоторого текста, такого как длинная дата, зависит на ваших региональных параметров на сервере Dynamics 365 for Operations. Чтобы изменить эти установки, щелкните кнопку **Пуск**, щелкните **Панель управления**, и после этого щелкните **Язык и региональные стандарты**. Следующая таблица перечисляет доступные варианты autotext для заголовков столбца.

| Вариант и код Autotext                | описание                                                                                                                                                                                                                                                                                      |
|-----------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Название месяца (@CalMonthLong)              | Напечатайте имя текущего месяца в заголовке столбца. Если вы решаете округлить суммы в отчете до тысяч, миллионов или миллиардов, или если вы устанавливаете ширину столбца в отчете до меньше чем 9 символов, то имя месяца сокращается до первых 3 символов. |
| Сокращенное имя месяца (@CalMonthShort) | Печатать сокращенное имя месяца для выбранного фискальный период.                                                                                                                                                                                                                          |
| Номер периода (@FiscalPeriod)           | Печатать численную форму фискального периода, который определен для этого столбца. Если столбец включает множественные периоды, то последний период в диапазоне печатается.                                                                                                                                   |
| Описание периода (@FiscalPeriodName)  | Печатать описание фискального периода, которое определено в финансовых данных.                                                                                                                                                                                                                    |
| Финансовый год (@FiscalYear)               | Печатать Финансовый год для столбца в численной форме.                                                                                                                                                                                                                                            |
| Календарный год (@CalYear)                | Печатать календарный год для столбца в численной форме.                                                                                                                                                                                                                                          |
| Дата начала (@StartDate)                 | Печатать начальную дату столбца.                                                                                                                                                                                                                                                             |
| Дата окончания (@EndDate)                     | Печатать Дату окончания столбца.                                                                                                                                                                                                                                                               |
| Имя единицы из дерева (@UnitName)         | Если вы ограничиваете столбец определенной единицей дерева отчетности, напечатайте имя единицы в заголовке столбца.                                                                                                                                                                                     |
| Описание единицы (@UnitDesc)            | Если вы ограничиваете столбец определенной единицей дерева отчетности, напечатать описание единицы в заголовке столбца.                                                                                                                                                                              |
| Код книги (@BookCode)                   | Напечатать Код книги, указанный в столбце.                                                                                                                                                                                                                                             |
| Пустая строка (@Blank)                     | Вставьте пустую строку в заголовок столбца.                                                                                                                                                                                                                                                       |

### <a name="create-a-conditional-spanning-header"></a>Создайте условный в диапазоне заголовок

Условный в диапазоне заголовок может занимать множественные столбцы, которые основаны на специфических данных периода. Например, если вы имеете отчет по бюджету на финансовый год и хотите показать фактические бюджеты прошлых месяцев вместе с запроектированными бюджетами будущих месяцев, то вы можете использовать условный в диапазоне заголовок для автоматического обновления заголовка отчета. Будьте в курсе следующих ситуаций, когда вы создаете условный в диапазоне заголовок:

-   Любое условие прекращения (поле **Распределение до**), которое сопоставляется до условия старта (поле **Распределение от**) игнорируется. Например, для столбца B имеется условие распределения, определенное как от BASE+1 до BASE, BASE — столбец C и BASE+1 — в столбце D. В этом случае, условие прекращение в столбце C, и печатание заголовка начинается на столбце D.
-   Если вы определяете заголовки столбца, которые перекрываются, они перекрываются, когда они печатаются в отчете. Отчет создан, но следующее предупреждение появляется в поле **Состояние очереди отчетов**: "Заголовки столбца с использованием Base пересекаются с другими заголовками столбца и могут причинить перекрывающийся текст". Например, определение заголовка в столбце B — от B до BASE+1, и определение заголовка на столбце D — от BASE+1 до F. В этом случае заголовки печатаются друг на друге и не читаются. Когда BASE использовано в определении **Распределение от/Распределение до**, обязательно посмотрите созданный отчет, чтобы увидеть, перекрываются ли заголовки.
-   Если указать BASE в определении распределения в столбце "Не печатать" (**NP**), оно игнорируется, независимо от того, что определено в определении столбца. По сути, этот сценарий — то же,, как не создавать определение заголовка столбца.
-   Для столбцов условной печати (**P&lt;B**, **P&gt;=B**) условные в диапазоне заголовки поступают так же, как любое обычное определение заголовка столбца. Например, если условие ложно, любое последующее сопоставление столбца для условия распределения начинает печатание заголовка.

#### <a name="create-a-conditional-spanning-header"></a>Создайте условный в диапазоне заголовок

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Выполнить двойной клик ячейки заголовка.
3.  В диалоговом окне **Заголовок столбца**, впишите текст заголовка столбца. Альтернативно, щелкните **Вставить AutoText**, и выберите вариант.
4.   В поле **Параметры формата** выберите стиль форматирования для заголовка.
5.  Определите период по отношению к базовому периоду, который определен, когда отчет произведен. В полях **Распределение от** и **Распределение до**, введите одно из следующих значений: **BASE**, **BASE-X** или **BASE+X**, где X — число периодов от базового периода. Например, если вы вводите **BASE** в поле **Распределение от**, то текст условного в диапазоне заголовка столбца начинается в заголовке столбца, где значение **Базовый период** определения отчета равняется значению **Период** определения столбца. Он кончается в столбце, который показан в поле **Распределение до**. Поэтому, если распространение — от BASE до M, и значение **Базовый период** определения отчета — **4**, то заголовок начинается в столбце, где период установлен как **4** и кончается в столбце M. Заголовки оканчиваются и начинаются только на печатаемых столбцах.
6.  В разделе **Выравнивание** выберите, должен ли текст заголовка столбца быть выровнен по леву, центру или праву.
7.  Нажмите кнопку **OК**.

## <a name="apply-column-justification"></a>Применение выравнивания столбца
Ячейка **Выравнивание** используется для того, чтобы применить форматирование выравнивания к столбцу описания в отчете. Этот вариант влияет только на описания столбца, а не натуральные величины.

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Выравнивание**.
3.  Выберите одно из следующих значений в списке:
    -   **Нет** — никакое выравнивание не применяется.
    -   **По левому краю** — Выравнивание описаний столбца По левому краю.
    -   **По центру** — выравнивание описаний столбца По центру.
    -   **По правому краю** — Выравнивание описаний столбца По правому краю.

## <a name="add-special-formatting-options"></a>Добавление специальных вариантов форматирования
В определении столбца строки сведений форматирующего столбца применяют специальное форматирование к выбранным столбцам. Хотя некоторые из параметров **Управление печатью** и **Ограничения столбца** относятся конкретно к столбцам **FD**, большинство параметров относится ко всем типам столбцов. Форматирование. которое определено в определении столбца, перекрывает форматирование которое определено в определении отчета. Однако Форматирование. которое определено в определении строки, перекрывает форматирование которое определено в определении столбца. Следующие строки считаются форматирующими строками:

-   Ширина столбца
-   Дополнительный пробел перед столбцом
-   Перекрытие формата/валюты
-   Управление печатью

### <a name="changing-the-column-width"></a>Изменение ширины столбца

Ячейка **Ширина столбца** определяет число символов для использования для ширины этого столбца в напечатанном отчете. Ширина столбца важна для столбцов, которые содержат суммы (столбцы типа **CALC**, **WKS**, или **FD**), описания (столбцы типа **DESC**), или заполнение (столбцы типа **FILL**). По умолчанию выбран вариант **AutoFit** выбран, так что ширина каждого столбца автоматически регулируется так, чтобы помещалось содержимое.

#### <a name="specify-the-width-of-a-column-on-a-report"></a>Определите ширину столбца в отчете

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В ячейке **Ширина столбца**, введите число пробелов для ширины столбца. Максимальная ширина любого столбца — 255 символов (этот номер включает центы, запятые и скобки). Либо, чтобы позволить дизайнеру отчетов выбрать соответствующую ширину столбца, основанную на содержимом ячейки, дважды щелкните ячейку **Ширина столбца** и после этого щелкните **Автоподгонка**.

### <a name="add-space-between-columns"></a>Добавление пробела между столбцами

Ячейка **Дополнительные пробелы перед столбцом** задает ширину разделителя между одним столбцом и смежными столбцами в определении столбца. Настройка **Дополнительные пробелы перед столбцом** влияет на все строки сведений столбца для столбца, но не строки заголовка столбца. Используйте этот вариант для того, чтобы отделить группы столбцов или добавить немного пробелов перед описанием, так, что столбец описания будет С отступом от выровненных влево названий на отчете. Количество пробелов по умолчанию между каждыми столбцами — два. Вы можете изменить эту установку на вкладке **Настройки** в определении отчета.

#### <a name="specify-the-space-between-columns"></a>Укажите пробел между столбцами

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В ячейке **Дополнительный пробел перед столбцом** впишите число пробелов, которое ввести между столбцами.

### <a name="specify-a-currency"></a>Определите валюту

Ячейка **Перекрытие формата/валюты** определяет форматирование десятичной дроби, валюты, и сумм процента в столбце. Это форматирование перекрывает любое форматирование, которое определено в определении отчета по умолчанию или значения системы по умолчанию.

#### <a name="assign-a-format-currency-override-to-a-report-column"></a>Задайте перекрытие валюты формата для столбца отчета

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Выполните двойной клик ячейки **Перекрытие формата/валюты** в столбце суммы.
3.  В диалоговом окне **Перекрытие формата** выберите варианты форматирования.

### <a name="add-a-print-control-code"></a>Добавьте код управления печатью

Ячейка **Управление печатью** может содержать коды, которые регулируют отображение или печать символов столбца. Есть два типа кодов управления печатью: регулярные коды управления печатью и условные коды управления печатью.

## <a name="column-types"></a>Типы столбцов
Тип информации, который включает каждый столбец в отчете, определен значением в строке **Типе столбца** в определении столбца. Каждое определение столбца должно содержать по крайней мере один столбец описания (**DESC**) и один столбец количества (**FD**, **WKS**, или **CALC**). **Примечание:** Коды типа столбца не применяются ко всем системам учета. Если вы выбираете тип, который не действителен для вашей системы учета, этот столбец будет пуст на отчете.

### <a name="financial-dimensions-column"></a>Строка Финансовые аналитики

Следующие определения строки **Оределение столбца** применяются к столбцам которые имеют тип столбца **FD** (суммы из финансовых аналитик).

#### <a name="book-codeattribute-category-cell"></a>Ячейка Код книги/Атрибут категории

Ячейка **Код книги/Атрибут категории** определяет код книги для данных в столбце **FD**. Определение столбца может включить много столбцов фактических, бюджета, и статистических. Определение столбца может также отображать различные периоды, такие как текущий или с начала года, и различное суммы. Список кодов книг отражает фактические, бюджетные, и статистические (не-финансовые) параметры, которые были установлены в ваших финансовых данных.

#### <a name="fiscal-year-cell"></a>Ячейка Финансовый год

Ячейка **Финансовый год** определяет финансовый год, который столбец должен включить. Год может быть относительно к базовому году, который определен, когда отчет произведен. Имеются следующие варианты.

| Параметр  | Описание                                                                                                                  |
|---------|------------------------------------------------------------------------------------------------------------------------------|
| BASE    | Используйте базовый год, который определен во время отчета.                                                                          |
| BASE+\# | Используйте год, следующий через \# лет после базового года. Например, чтобы использовать третий год после базового года, введите **BASE+3**. |
| BASE-\# | Используйте год за \# лет до базового года. Например, чтобы использовать предыдущий год, введите **BASE-1**.                 |
| \#      | Введите фактический финансовый год.                                                                                                |

#### <a name="period-cell"></a>Ячейка периода

Ячейка **Период** определяет финансовые периоды, которые столбец должен включить. Период может быть по отношению к базовому периоду, который определен, когда отчет произведен. Имеются следующие варианты.

| Параметр          | описание                                                                                                                                                                                                                          |
|-----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BASE            | Используйте базовый период.                                                                                                                                                                                                                 |
| BASE+\#         | Используйте период, который следует через \# периодов после базового периода. Например, чтобы использовать третий период после базового периода, введите **BASE+3**.                                                                                               |
| BASE-\#         | Используйте период, который был за \# периодов до базового периода. Например, чтобы использовать предыдущий период, введите **BASE-1**.                                                                                                                 |
| BASE-\#:BASE    | Используйте множественные периоды, от нескольких периодов перед базовым периодом до базового периода. Например, для использования трех предыдущих периодов и базового периода, введите **BASE-3:BASE**.                                                |
| BASE:BASE+\#    | Используйте множественные периоды, от базового периода до нескольких периодов после базового периода. Например, для использования базового периода и следующих двух периодов, введите **BASE:BASE+2**.                                                  |
| BASE-\#:BASE+\# | Используйте множественные периоды, от нескольких периодов перед базовым периодом до нескольких периодов после базового периода. Например, для использования трех предыдущих периодов, базового периода и следующих двух периодов, введите **BASE-3:BASE+2**. |
| 1:BASE          | Используйте множественные периоды, от первого периода до нескольких периодов после базового периода.                                                                                                                                                                 |
| \#              | Всегда используйте специфический номер периода. Мы не рекомендуем использовать этот вариант, потому что оно уменьшает гибкость определения столбца.                                                                                       |
| \#:\#           | Всегда используйте специфический диапазон периодов. Мы не рекомендуем использовать этот вариант, потому что оно уменьшает гибкость определения столбца.                                                                                    |

Вы можете выйти за границы финансового года в любых спецификациях периода, и вы можете смешать годы в диапазоне периодов. Например, вы задаете периоды как **BASE-5** (представляет прошедшие шесть периодов) и выполняете отчет, который имеет базовый период 2. В этом случае отчет показывает данные для первых двух периодов указанного финансового года и последних четырех периодов предыдущего финансового периода.

### <a name="specify-the-periods-for-an-fd-column"></a>Определите периоды для столбца FD

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В столбце **FD**, выполните двойной клик ячейки в строке **Период**, и после этого выберите вариант в списке.
3.  В строке формул над Областью перехода, или в ячейке **Период**, завершите формулу. Замените знак любого числа (\#) соответствующим значением.

#### <a name="periods-covered-cell"></a>Ячейка Покрытые периоды

Ячейка **Покрытые периоды** определяет сумму, которую столбец должен отобразить. Эта сумма — по отношению к значению в ячейках **Финансовый год** и **Период** для столбца. Имеются следующие варианты.

| Параметр      | Описание                                                                 |
|-------------|-----------------------------------------------------------------------------|
| PERIODIC    | Отображение суммы мероприятия для текущего периода или диапазона периодов. |
| PERIODIC/BB | Отображение начального сальдо для текущего периода или диапазона периодов.   |
| YTD         | Отображение суммы мероприятия с начала года.                               |
| YTD/BB      | Покажите начальное сальдо на год.                                 |

### <a name="specify-the-periods-that-are-covered-for-an-fd-column"></a>Определите периоды, покрытые для столбца FD

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В столбце **FD**, выполните двойной клик ячейки в строке **Покрытые периоды**, и после этого выберите вариант в списке.

### <a name="attribute-filter-in-a-column-definition"></a>Фильтр атрибута в определении столбца

Атрибуты — значения финансовых данных, которые добавочно определяют счет или проводку. Атрибуты счета включают **Актив**, **Задолженность**, **Доход** и **Расход**. Атрибуты проводки включают **Описание проводки** и **Дата применения проводки**. Поддержка атрибута может отличаться между системами Microsoft Dynamics ERP. Ячейка **Фильтр атрибута** ограничивает данные в столбце **FD** определенными значениями или диапазонами для категорий атрибутов. Хотя эту функцию можно использовать вместе со столбцом **ATTR**, столбец **ATTR** не необходим. В столбце **FD** есть предел на счета или проводки, который отчет включит от фильтра атрибута. **Примечание:** Чтобы Увидеть, которые атрибуты ваша система ERP поддерживает, посмотрите руководство по интеграции для вашей системы.

#### <a name="apply-an-attribute-filter-for-an-fd-column-on-a-report"></a>Примените фильтр атрибута для столбца FD в отчете

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Фильтр атрибута** для столбца **FD**.
3.  В диалоговом окне **Фильтр атрибутов**, дважды щелкните ячейку в столбце **Атрибут**, и после этого выберите тип фильтра.
4.  Чтобы дополнительно ограничивать результаты, введите диапазон в столбцы **От** и **До**. Ячейка **От** должна содержать значение.
5.  Нажмите кнопку **OК**.

### <a name="dimension-filter-in-a-column-definition"></a>Фильтр аналитики в определении столбца

Фильтр аналитики используется, чтобы ограничить столбец **FD** специфическими значениями аналитики. Фильтр может включать одну аналитику, диапазон аналитик или группу аналитик. Фильтр может также включать наборы значений аналитик. Так как значения аналитик могут меняться, ..\финансовые-аналитики\основанная на аналитиках система не должна соответствовать точной длине. Фильтр применен, независимо от того, включает ли отчет дерево отчетности. Вы можете использовать подстановочный знак (\* или ?) в любом положении. Когда вы определяете множество счетов, поставьте запятую между счетами, как в следующий примере: +Счет=\[1200\], +Счет=\[1100\], Подразделение=\[01?\] Для того, чтобы получить все отделы для специфического счета, вы можете исключить аналитику Подразделение из фильтра аналитик. Например, оба из следующих фильтров аналитик обрабатываются одним и тем же образом:

-   +Счет=\[1100\],Подразделение
-   +Счет=\[1100\]

Вы можете также использовать любое сочетание цифробуквенные символов для точного сопоставления, и вы можете определить частичные размеры. Например, **Местонахождение = \[10\*\]** включает все значения аналитики положения, которые начинаются с 10.

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Фильтр аналитик** для столбца **FD**.
3.  В диалоговом окне **Аналитики**, введите применяемые.
4.  Нажмите кнопку **OК**.

### <a name="format-a-multiple-currency-report-in-a-column-definition"></a>Форматируйте отчет с несколькими валютами в определении столбца

Отчет о нескольких валютах может показать суммы в естественной (местной) валюте, функциональной (по умолчанию) валюте, или валюте отчетности. Функциональная валюта компании определена в системе Microsoft Dynamics ERP. Не путайте эту установку ERP с настройками региональных параметров операционной системы, где вы можете установить символы валюты по умолчанию, которые используются в отчетах. Следующие связанные с валютой ячейки доступны в определении столбца:

-   **Отображение валюты** – укажите тип валюты (основная, функциональная или учетная), в которой отображаются проводки. Эта функция иногда называется преобразованием валюты. Перевод валюты — это способность сообщать суммы в главной книге в валюте, которая может быть не функциональной валютой компании или не валютой, в которой введена проводка.
-   **Фильтр валюты** — определите фильтр валюты. Только проводки которые введены в выбранной валюте, показаны на отчете.

> [!NOTE]
> Для того, чтобы создать отчеты, которые используют множественные валюты, вы должны выбрать флажок **Включить все валюты отчетности** на вкладке **Отчет** определения отчета. Для того, чтобы определить функциональную валюту компании, выполните эти шаги.

1.  В конструкторе отчетов, в меню **Компания**, щелкните **Компании**.
2.  В диалоговом окне **Компании** выберите компанию, а затем нажмите **Просмотр**.
3.  В диалоговом окне **Просмотр компании**, под **Региональные параметры**, вы можете посмотреть валюту, которая определена для выбранной компании.

#### <a name="specify-the-currency-on-a-multiple-currency-report"></a>Определите валюту в отчете с несколькими валютами

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  Дважды щелкните ячейку **Отображение валюты** в соответствующем столбце **FD**, и после этого выберите вариант для отображения информации валюты: **Естественная/исходная валюта**, **Функциональная валюта из данных о компании**, или валюта отчетности.
3.  Дважды щелкните ячейку **Фильтр валюты** в соответствующем столбце **FD**, а затем выберите соответствующий код валюты в списке. Только проводки которые введены в этой валюте, показаны на отчете.

> [!NOTE]
> Варианты, которые описаны здесь, могут отличаться, в зависимости от системы ERP. Дополнительные сведения см. в [документации системы Microsoft ERP](https://www.microsoft.com/en-us/download/details.aspx?id=5916).

### <a name="example-for-currency-display-and-currency-filter-cells"></a>Пример для ячеек Отображение валюты и Фильтр валюты

Филлис сделала следующие выборы валюты в своем определении столбца:

-   **Фильтр валюты:** Йена
-   **Отображение валюты:** Функциональное (доллары США)

Из-за фильтра валюты, который выбрала Филлис, отчет включает только проводки, которые были введены в японских йенах (JPY). Из-за отображения валюты, которое она выбрала, отчет показывает эти проводки в функциональной валюте, долларах США (USD).

### <a name="calculation-column-in-a-column-definition"></a>Столбец вычисления в определении столбца

Тип столбца **CALC** в определении столбца поддерживает сложные вычисления в ячейке **Формула** и может включать операторы **+**, **-**, **\*** и **/**, а также утверждения **IF/THEN/ELSE**. Столбец вычисления может также сослаться на любой другой столбец, даже последующие столбцы. Дополнительно, столбец вычисления может также включить финансовый год и период для того, чтобы поддержать заголовки для столбца. Формула вычисления может включать в себя до 1024 символов. Чтобы выразить результат вычисления как процент, используйте специальное перекрытие формата. **Примечание.** Результаты формул вычисления не включают значения диапазонах столбцов не для печати. Например, **A:D** печатает **0** (ноль), тогда как **A+B+C** для значений не для печати высчитывает значение.

#### <a name="operators-in-calculation-columns"></a>Операторы в столбцах вычисления

Для того, чтобы добавить, вычесть, умножить, или разделить столбцы, впишите буквы столбцов в заказ вычисления, и после этого используйте соответствующего оператора для того, чтобы отделить каждую букву столбца. Следующая таблица объясняет операторы, которые вы можете использовать в столбце вычисления.

| Оператор | Пример расчета | описание                                                                                                                                                                                                                                    |
|----------|---------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| +        | A+C                 | Сложить количество в столбце A к количеству в столбце C.                                                                                                                                                                                          |
| :        | A:C A:C-D           | Сложить диапазон последовательных столбцов. Например, формула **A:C** складывает суммы столбцов от A до С и формула **A:C-D** складывает суммы столбцов от A до C, а затем вычитает столбец D.                          |
| -        | A-C                 | Вычтите количество в столбце A от количества в столбце C. **Примечание:** Вы можете также использовать знак минус (-), чтобы обратить знаки в столбце. Например, используйте **-A+B** для того, чтобы добавить обратную сумму в столбце A к сумме в столбце B. |
| \*       | A\*C                | Умножить количество в столбце A на количество в столбце C.                                                                                                                                                                                     |
| /        | A/C                 | Разделить количество в столбце A на количество в столбце C.                                                                                                                                                                                       |

#### <a name="use-a-calculation-formula-in-a-column-definition"></a>Использование формулы вычисления в определении столбца

1.  В конструкторе отчетов, раскройте определение столбца, которое изменить.
2.  В соответствующем столбце **CALC**, впишите формулу в ячейку **Формула**.

#### <a name="complex-calculations"></a>Сложные вычисления

Сложное вычисление может содержать любые сочетания ссылок на ячейки, операторы, значения, и уровни вложенных скобок. Например, для того, чтобы вычислить среднее столбцов A и B, используйте формулу вычисления **((A+B)/2)**.

#### <a name="specify-report-cells-in-a-column-calculation"></a>Определите ячейки отчета в вычислении столбца

Вы можете сослаться к определенную ячейку отчета путем ввода буквы столбца и кода строки. Например, **B.100** ссылается на коду строки 100 в столбце B. Вы можете разделить весь столбец на специфическое сумму ячейки отчета, которая в том же столбце. Например, вычисление **B/B.100** значит, что сумма в столбце B должно быть разделено на значение в коде строки 100 в столбце B. Если вычисление ссылается на столбец, который зависит от другого столбца, зависимый столбец разрешается первым. Если вы ссылаетесь в столбце на другой столбец, который ссылается назад к первому столбцу, вы вызовете ошибку циклической ссылки. **Примечание:** Вычисление могло быть неправильно, если вы изменяете приоритет вычисления для отчета. Вы можете установить приоритет вычисления на вкладке **Установки** определения отчета.

#### <a name="multiply-or-divide-a-column-by-a-base-row"></a>Умножение или разделение столбца на базовую строку

Вы можете создать столбец, который показывает все значения в определенном столбце как процент от базового числа. Поэтому, вы можете показать отношения между строками, такие как процент строки продаж или процент строки общих расходов. Для того, чтобы умножить или разделить каждую строку в определенном столбце на базовую строку, введите столбец, который использовать в вычислении, и после этого введите **\*BASEROW** или **/BASEROW**. Например, впишите **C\*BASEROW** или **C/BASEROW**. **Примечание.** Когда вы используете вычисление по базовой строке в определении столбца, убеждайтесь что каждое определение строки, которое использовано с этим определением столбца, содержит по крайней мере одну базовую строку для вычислений.

#### <a name="divide-the-amount-in-a-column-by-the-number-of-periods"></a>Разделите количество в столбце на число периодов

Можно разделить количество в столбце на указанное число периодов. Например, формула **B/Periods** разделяет значение в столбце B на число периодов в столбце B. Если вычисление занимает множество столбцов, укажите число периодов для использования в вычислении. Например, формула **(B+C)/Periods** складывает количество в столбце B и столбце C, и после этого разделяет результат на значение периода.
