# <a name="purchase-spend-analysis-power-bi-content"></a>Покупки Анализ содержимого Power BI расходов

В этой теме описывается, что входит в покупке тратить содержимого пакета анализа для бизнес-Аналитики Майкрософт для Power. Объясняется, как получить доступ к отчетам, которые включены в пакет содержимого и содержит сведения о модели данных и объекты, которые используются для построения содержимого пакета.

<a name="overview"></a>Обзор
--------

Покупки анализ, содержимого, что был создан пакет для бизнес-Аналитики Майкрософт для Power для закупок менеджеры и руководители, ответственные за бюджетов расходов. Она предназначена для их следите за расходы на покупку. Используется для операций покупки данные о транзакциях из Microsoft Dynamics 365 и предоставляет как статистического представления цифр покупки в масштабе всей компании, так и детализация расходов по поставщикам и продуктам заказа на покупку. Отчеты выделить изменений в покупке расходы со временем. Таким образом они могут использоваться для менеджеров оповещения о положительных и отрицательных тенденций расходы для конкретных поставщиков и продуктов. Диаграммы показывают покупку расходов для закупок различных категорий и групп поставщика. Категория и региональных менеджеров бывает полезно, когда нужно использовать эти диаграммы для выявления изменений в поведение на расходы. Пакет содержимого закупку руководители и менеджеры, ответственные за бюджетов проанализируем покупку расходы следующими способами:

-   Покупки с начала года (по группе поставщиков и отдельных поставщиков категории закупаемой продукции и отдельных продуктов и местонахождение поставщика)
-   Изменение покупки по годам (по поставщику группы и категорию закупаемой продукции)

## <a name="accessing-the-content-pack"></a>Доступ к содержимого пакета
Покупки анализ, содержимого пакета публикуется как реализация основных средств в Microsoft Dynamics Lifecycle Services (LCS) и можно получить доступ из Microsoft Dynamics 365 для операций расходов. Дополнительные сведения о способах доступа и открытые отчеты Power BI см. в разделе [содержимое Power BI в LCS корпорации Майкрософт и ее партнеры](power-bi-content-microsoft-partners.md).

## <a name="data-model-and-entities"></a>Модель данных и объектов
Dynamics 365 для операций, данные используются для отчета в покупке тратить содержимого пакета анализа. Эти данные представлены как общий измерений, которые помещаются на временное хранение в хранилище объекта — базе данных Microsoft SQL, оптимизированная для аналитики. Дополнительные сведения о хранилище сущности см. [Power BI интеграции с магазином объекта в Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/) в блоге. Общий измерения в этот пакет содержимого являются подмножество совокупного измерений, которые были доступны в кубе покупки в Microsoft Dynamics AX 2012 и Microsoft Dynamics 365 для операций 2012 R3. Для размещения совокупного измерений куба в хранилище объекта, необходимо сделать их развертывания. Для получения дополнительных сведений см. процедуру промежуточного хранения статистических измерений в хранилище объекта в [Power BI интеграции с магазином объекта в Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/) в блоге. Следующие ключевые совокупного измерениям доступны непосредственно из объекта строки накладной и используются в качестве основы для содержимого пакета.

| Объект        | Ключевые совокупного измерений | Источник данных для Dynamics 365 для операций | Поле              | описание                           |
|---------------|----------------------------|---------------------------------------------|--------------------|---------------------------------------|
| Строки накладной | Покупка                   | VendInvoiceTrans                            | SUM(LineAmountMST) | Сумма в валюте учета |

В следующей таблице показаны ключевые измерениями, которые рассчитываются в пакет содержимого из объекта строки накладной.

| Мера               | Расчет                                                                                         |
|-----------------------|-----------------------------------------------------------------------------------------------------|
| Текущий год покупки | Покупки текущий год = сумм ("Строки накладной"\[покупку\])                                            |
| Покупки прошлого года    | Покупки прошлый год = РАССЧИТАТЬ (сумма ("Строки накладной"\[покупку\]), SAMEPERIODLASTYEAR (даты\[Дата\])) |
| Рост YOY покупки   | Рост на покупку YOY = \[текущий год на покупку\] — \[на покупку прошлого года\]                            |

Следующие основные размеры в пакет содержимого используются как фильтры для разделить совокупного измерений, таким образом, можно получить более подробную детализацию и более глубокое представление аналитической.

| Объект                 | Примеры атрибутов                                |
|------------------------|-------------------------------------------------------|
| Поставщики                | Групп поставщиков, поставщиков страны или регионы, имя поставщика |
| Товары               | Номер продукта, название продукта, имя группы номенклатуры        |
| Категории закупаемой продукции | Категория закупаемой продукции, имена категорий закупаемой продукции      |
| Юридические лица         | Имя информации о компании                                     |
| Даты                  | Даты, смещение года                                    |

По умолчанию пакет содержимого отображает данные для текущего календарного года. Тем не менее можно изменить фильтр даты в разделе фильтры отчета. Можно также изменить фильтр компании.
